# 等保 2.0 三级测评实施全流程指南

## 1. 等保 2.0 体系总览

### 1.1 核心标准

| 标准 | 编号 | 作用 |
|------|------|------|
| 《信息安全技术 网络安全等级保护基本要求》 | GB/T 22239-2019 | 核心要求（安全技术+安全管理） |
| 《信息安全技术 网络安全等级保护测评要求》 | GB/T 28448-2019 | 测评机构测评项 |
| 《信息安全技术 网络安全等级保护定级指南》 | GB/T 22240-2020 | 定级 |
| 《信息安全技术 网络安全等级保护安全设计技术要求》 | GB/T 25070-2019 | 设计与建设 |

### 1.2 五级体系（实际常用 二 / 三级）

| 级别 | 适用对象 |
|------|---------|
| 第一级 | 一般信息系统；内网小系统 |
| 第二级 | 小型企业、门户网站、一般办公系统 |
| 第三级 | 重点单位、关键业务、面向公众服务的较大型系统 |
| 第四级 | 金融、电力、轨道交通等关键信息基础设施 |
| 第五级 | 国家核心秘密（极少） |

### 1.3 实施全流程

```
定级 → 备案 → 差距分析 → 建设整改 → 测评 → 定期复测
  ↑                                                   │
  └────────────────── 每年一次测评 / 重大变更重新定级 ──┘
```

## 2. 定级

- **明确定级对象**：按业务系统或子系统为单位划分
- **确定受侵害的客体**：公民/法人/其他组织；社会秩序/公共利益；国家安全
- **确定对客体的侵害程度**：一般损害 / 严重损害 / 特别严重损害
- **查表得出等级**（GB/T 22240-2020 附录表）
- **编写《定级报告》** + 专家评审 + 上级主管部门审批
- 在公安部门（等保 110）办理《备案证明》，获得备案编号

## 3. 基本要求（GB/T 22239）总览

### 3.1 安全技术要求（S1-S10）

| 大类 | 典型控制点 |
|------|-----------|
| S1 安全物理环境 | 物理位置选择、物理访问控制、防盗窃、防火、防水、防静电、温湿度、电力供应、电磁防护 |
| S2 安全通信网络 | 网络架构、通信传输、可信验证、数据保密性/完整性 |
| S3 安全区域边界 | 边界防护、访问控制、入侵防范、恶意代码防范、安全审计、可信验证 |
| S4 安全计算环境 | 身份鉴别、访问控制、安全审计、入侵防范、恶意代码防范、数据完整性/保密性/备份恢复、个人信息保护 |
| S5 安全管理中心 | 系统管理、审计管理、安全管理、集中管控 |

### 3.2 安全管理要求（M1-M5）

| 大类 | 典型控制点 |
|------|-----------|
| M1 安全管理制度 | 安全策略、各种制度、发布、评审 |
| M2 安全管理机构 | 岗位设置、人员配备、授权审批、沟通合作、审核检查 |
| M3 安全管理人员 | 人员录用、离岗、考核、安全教育培训 |
| M4 安全建设管理 | 方案设计、采购、软件开发、工程实施、测试验收、交付、服务商管理 |
| M5 安全运维管理 | 环境/资产/设备/介质/ vulnerability 运维、外包、变更、监控、审计、应急 |

三级要求覆盖以上大多数控制点；四级要求更为严格。

## 4. 差距分析（Gap Analysis）

方法：对照《基本要求》逐条判断：

| 控制点 | 当前情况 | 差距 | 整改建议 | 责任人 |
|--------|---------|------|---------|--------|
| S1.1 物理位置选择 | 机房在写字楼 3 层 | 存在漏水风险（靠近水管） | 加装水浸传感器 + 改道 | 运维部 |
| S3.1 边界防护 | 只有防火墙，无 IPS | 无法主动阻断入侵 | 部署 IPS 或下一代防火墙 | 信息中心 |
| S4.1 身份鉴别 | 部分系统只有用户名密码 | 无双因子 | 关键系统启用 MFA | 应用部 |
| S4.5 安全审计 | 日志分散，无集中审计 | 无法关联分析 | 部署 SIEM / 日志平台 | 信息中心 |
| M4.3 软件开发 | 无代码审计流程 | 上线前无安全测试 | 引入 SAST / SCA，制定 SDL | 开发部 |

## 5. 整改建设建议清单（三级）

1. **网络 / 边界**：防火墙、下一代防火墙（NGFW）、IPS/IDS、WAF、上网行为管理、APT 检测、堡垒机、运维审计、负载均衡、VPN、日志集中收集
2. **主机 / 终端**：EDR / 主机防病毒、补丁管理、基线合规（CIS Benchmark）、主机审计
3. **应用 / 数据**：安全开发流程（SDL）、SAST/DAST/IAST、Web 应用防火墙、数据加密、备份与灾备、数据库审计、DLP
4. **管理**：制度体系、应急预案、应急演练、定期安全培训、安全事件上报与处置流程
5. **审计与日志**：日志保存 ≥ 6 个月，有审计管理员角色分离

## 6. 测评配合

- 选择具有等保测评资质的机构
- 提供：《定级报告》《备案证明》《差距分析报告》《安全方案》
- 配合测评人员进行：**文档审查 + 访谈 + 配置核查 + 工具扫描 + 渗透性测试**
- 对不符合项在规定时限内整改并复测

## 7. 典型高风险点 Top 10

1. 关键系统无双因素认证（MFA）
2. 日志未集中收集、保留不足 6 个月
3. 弱口令 / 共享账号 / 默认口令
4. 数据库未启用审计、未限制访问源
5. 公网暴露敏感服务（Redis/DB/管理后台）
6. 无入侵检测 / 防护设备
7. 无正式安全管理制度与应急预案
8. 无定期漏洞扫描与渗透测试
9. 无数据备份与恢复演练
10. 外包 / 第三方接口无安全协议与权限最小化

## 8. 交付物清单

- 《定级报告》
- 《备案证明》
- 《安全方案 / 差距分析报告》
- 《安全管理制度体系》（8 大类文档）
- 《上线安全测试报告》
- 《等级测评报告》（由测评机构出具）
- 《整改复测报告》
- 《应急预案》+ 演练记录

---

> 等保不是一次性项目，而是持续合规与安全运营的过程。年度测评 + 重大变更重新定级 + 常态化漏洞管理是关键。
