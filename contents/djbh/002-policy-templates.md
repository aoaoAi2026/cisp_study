# 安全管理制度体系建设模板

> 等保项目需建立 8 大类制度文档，形成完整的信息安全管理制度体系。

## 1. 制度体系结构

```
信息安全方针（最高层纲领）
 ├── 安全策略类（策略/规划/总体方针）
 ├── 组织与人员类（岗位职责/录用/离岗/培训/考核）
 ├── 物理与环境类（机房/门禁/设备管理）
 ├── 网络与系统类（网络/主机/数据库/中间件安全/补丁）
 ├── 应用与数据类（开发安全/上线测试/数据分类分级/加密/备份）
 ├── 运维与操作类（变更/配置/账号权限/审计/日志/外包）
 ├── 应急与事件类（应急响应/容灾备份/演练/事件上报）
 └── 监督与评审类（合规/审计/评审/持续改进）
```

## 2. 核心制度清单与模板要点

### 2.1 《信息安全管理方针》（1 级文件）

- 目标：定义信息安全目标、范围、管理层承诺
- 关键段落：
  - 信息安全总体目标（CIA 三性：保密性/完整性/可用性）
  - 适用范围（组织、系统、人员）
  - 管理层声明与资源保障
  - 发布、评审、修订周期（至少每年一次）

### 2.2 《信息安全组织机构与岗位职责》

- 成立信息安全领导小组（组长：CEO/CIO）
- 设立信息安全管理部门 / 信息安全员
- 明确岗位职责：
  - **系统管理员**：系统配置、账号、备份
  - **安全管理员**：权限管理、策略、漏洞、监控
  - **审计管理员**：日志、审计、报告（与前两者角色分离）
- 明确授权审批流程（重要系统变更需要双人签字）

### 2.3 《人员安全管理制度》

- 人员录用：背景调查、保密协议、安全教育、权限最小化
- 人员离岗：权限回收、设备回收、签署保密承诺、账号禁用
- 安全培训：新员工入职培训 + 年度培训 + 专项培训（至少每年一次）
- 考核与奖惩：信息安全纳入 KPI

### 2.4 《物理环境安全管理制度》

- 机房门禁、监控、访客登记（留存 ≥ 3 个月）
- 出入审批、钥匙/门禁卡管理
- 设备进出登记、资产标签
- 防火/防水/防静电/温湿度监控/电力供应 UPS 检查
- 电磁防护、介质存储与销毁（粉碎、消磁）

### 2.5 《网络与系统安全管理制度》

- 网络架构、VLAN 划分、边界防火墙策略审批
- 服务器基线配置（CIS Benchmark 对标）
- 账号管理（命名规范、生命周期、锁定策略）
- 密码策略（复杂度 ≥ 8、更换周期 90 天、禁用历史 5 次）
- 补丁管理（月度扫描、分类补丁窗口、紧急补丁 72 小时）
- 中间件、数据库、操作系统定期加固
- 远程运维管理（VPN + 堡垒机 + MFA）

### 2.6 《应用与数据安全管理制度》

- 软件开发安全：需求安全评审、代码审计、上线安全测试
- 上线前安全检查清单（SQL 注入、XSS、越权、弱口令、文件上传）
- 数据分类分级：公开 / 内部 / 秘密 / 机密
- 数据传输加密（TLS）、存储加密（数据库字段加密、整库 TDE）
- 个人信息保护符合《个人信息保护法》与 GB/T 35273
- 数据备份：本地 + 异地，定期恢复演练（至少每年一次）
- 数据销毁流程

### 2.7 《运维与操作安全管理制度》

- 变更管理（申请 → 评审 → 测试 → 审批 → 实施 → 验证）
- 配置管理（CMDB、配置基线、版本控制）
- 账号与权限：最小权限原则、定期评审、申请审批流程
- 第三方 / 外包运维：安全协议、权限管控、访问审计、退出
- 介质管理：U盘、移动硬盘、笔记本加密、登记、定期盘点
- 日志管理：集中收集（≥ 6 个月）、访问审计、日志备份

### 2.8 《应急预案与事件处置管理制度》

- 事件分级（Ⅰ/Ⅱ/Ⅲ/Ⅳ）
- 应急组织与联络表
- 典型事件处置 SOP（勒索软件、DDoS、数据外泄、账号被盗、钓鱼）
- 上报流程（内部、上级主管部门、公安网安）
- 应急演练（至少每半年一次）
- 事后复盘、整改、制度更新

## 3. 评审与持续改进

- **月度**：安全例会、漏洞回顾
- **季度**：合规检查、制度执行情况检查
- **年度**：管理评审、全面风险评估、制度修订、管理评审

## 4. 记录与留痕

以下记录至少保留 6 个月至 3 年：

- 人员录用与离岗记录
- 培训记录（签到 + 课件）
- 机房出入登记
- 变更申请与审批单
- 权限申请与审批单
- 补丁安装记录
- 漏洞扫描与修复记录
- 事件处置记录与复盘报告
- 应急演练记录
- 安全审计日志

---

> 建议：制度发布后，应针对核心制度进行全员培训与考试；制度应每年或在重大业务/组织变化时重新评审。
