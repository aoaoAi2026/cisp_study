# V2X 通信安全与 PKI 证书体系

---

## 一、V2X 通信标准

### 1.1 两种技术路线

```
V2X (Vehicle-to-Everything) 通信方式：

DSRC/802.11p (美国/日本主推):
  基于IEEE 802.11p + IEEE 1609.x (WAVE)
  频率：5.85-5.925 GHz
  特点：成熟、低延迟(≤10ms)
  劣势：覆盖范围有限(≤300m)，需要路侧单元(RSU)密集部署

C-V2X (中国主推):
  基于4G LTE-V2X (PC5接口) → 5G NR-V2X (演进)
  频率：5.905-5.925 GHz
  特点：
    - PC5直连通信(无需基站)：V2V/V2I
    - Uu接口(通过基站)：V2N(云端)
    双模同时工作
  优势：覆盖范围大，可利用现有4G/5G基站
  国产芯片：华为Balong 765、大唐/高通

V2X 通信类型：
  V2V (Vehicle-to-Vehicle): 相邻车辆碰撞预警
  V2I (Vehicle-to-Infrastructure): 信号灯/限速牌
  V2P (Vehicle-to-Pedestrian): 行人/自行车检测
  V2N (Vehicle-to-Network): 云端服务
```

### 1.2 V2X 消息格式

```
BSM (Basic Safety Message) — SAE J2735:
  每秒10次广播
  内容：位置/GPS/速度/方向/加速度/刹车状态
  大小：约300-400字节

攻击后果：
  伪造BSM → "前方急刹车"虚假预警 → 引发幽灵堵车
  伪造BSM → "我在这里" — Ghost Vehicle攻击
  重放BSM → 欺骗碰撞预警系统
```

## 二、V2X 安全标准

### 2.1 三大安全标准对比

```
IEEE 1609.2 (美国):
  WAVE安全服务标准
  证书格式：显式证书(ECDSA签名)
  签名算法：ECDSA P-256 + SHA-256

ETSI ITS (欧洲):
  证书格式：显式 + 隐式证书
  支持ECQV(隐式证书) → 更小带宽开销
  支持多种签名方案

CCSA (中国):
  基于国密算法：
    签名：SM2
    加密：SM4
    哈希：SM3
  证书格式：国密X.509证书
  协议：基于IEEE 1609.2修改的国密版
```

## 三、V2X PKI 证书体系

### 3.1 SCMS (美国安全凭证管理系统)

```
SCMS = Security Credential Management System

证书三层结构：

1. 注册证书 (Enrollment Certificate)
   有效期：数月至数年
   作用：车辆的身份凭证(类似"出生证明")
   车辆出厂时预置(EVITA HSM存储私钥)

2. 假名证书 (Pseudonym Certificate) ★核心★
   有效期：数分钟至数天
   作用：签名BSM消息(不暴露车辆真实身份)
   隐私保护：同一车辆定期更换假名证书
   换证机制：每5分钟更换一次 → 防止追踪

3. 应用证书 (Application Certificate)
   用于非BSM消息(如特殊车辆信号灯优先)

SCMS核心组件：
  RA (Registration Authority): 证书注册
  PCA (Pseudonym CA): 假名证书签发
  MA (Misbehavior Authority): 不当行为检测+吊销
  LA (Linkage Authority): 链接值用于吊销(需两组协作)
```

### 3.2 中国 C-V2X 安全体系

```
中国 V2X 安全体系 (CCSA标准):

证书类型（类似SCMS）：
  注册证书 + 假名证书 + 应用证书
  全部使用SM2算法

特点：
  - 集中式PKI (不同于美国联邦式SCMS)
  - 国家SM2根CA → 行业CA → 车辆证书
  - 车联网安全信任根管理平台(工信部车联网身份认证系统)

关键性能要求：
  签名生成：<10ms (毫秒级，影响BSM发送频率)
  签名验证(接收端)：每秒验证≥2000条BSM
  证书大小：<200字节(减少无线带宽消耗)
  假名证书更换：<100ms(切换过程不掉消息)
```

## 四、V2X 隐私保护

```
隐私悖论：
  需要身份认证(防止伪造消息)
  但不需要知道"是谁"(保护隐私)

假名证书方案：
  T=0min → 使用假名证书A
  T=5min → 切换到假名证书B (广播空消息过渡)
  T=10min → 切换到假名证书C
  ...
  T=60min → 切换到假名证书A (循环)

  如果一直用同一个证书 → 路径追踪 → 隐私泄露
  如果频繁切换 → 追踪难度指数增加

不当行为检测与吊销：
  某车辆发送虚假BSM → MA检测到不当行为
  → 通过Linkage Authority将假名证书关联到注册证书
  → 吊销注册证书 → 该车辆所有假名证书失效
  → 隐私保护：正常车辆无法被追踪
```

#    录制BSM → 延迟重放 → 检测RSU/车辆是否拒绝

#    使用无效证书签名 → 发送BSM →

#    发送矛盾消息(位置1秒内跳变1km) →

# 4. Sybil攻击测试
#    单个物理车辆伪造多个假名同时发送 →
#    制造"幽灵车群"→ 定位一致性检测
