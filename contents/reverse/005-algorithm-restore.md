# 算法还原：加密/校验/混淆函数识别与解密

逆向工程中，最常见也最有成就感的任务之一就是"算法还原"——从编译后的二进制里识别加密、哈希或自定义混淆算法，并恢复出原始算法逻辑、密钥与输入输出关系。本文从识别、分析、还原三个阶段介绍通用方法。

## 一、为什么要做算法还原

现实场景中算法还原的需求非常广泛：

- **漏洞研究**：还原固件/协议中的加密算法，才能进一步构造 PoC；
- **CTF**：Reverse 题普遍要求还原自定义混淆、加密算法；
- **恶意代码分析**：识别勒索软件的加密变体与密钥生成逻辑；
- **协议逆向**：恢复私有协议（如游戏、IoT 设备）中的校验、加密字段；
- **安全评估**：判定自研加密是否"自研但不安全"。

## 二、快速识别：靠常量与特征

很多标准算法会在代码中留下明显的"指纹"，逆向时优先关注以下特征：

### 2.1 标准加密算法的常量指纹

| 算法 | 识别线索 |
| --- | --- |
| AES | `0x63, 0x7c, 0x77, 0x7b…`（S-box）、轮常数 `0x01, 0x02, 0x04…` |
| DES / 3DES | IP 表、E 表、S-box 表、`0x01010101, 0xFEFEFEFE` 等奇偶表 |
| RC4 | 初始化 S-box `0..255` 循环；典型的 `i, j, S[i], S[j]` 交换 |
| TEA/XTEA | 魔法常数 `0x9E3779B9`；`sum += delta` 循环 |
| ChaCha20 | 常量 `"expand 32-byte k"` 或 `0x61707865, 0x3320646E, 0x79622D32, 0x6B206574` |
| SM4 | 固定 S-box、轮常数 `FK/Ck` 等中国商密标准参数 |
| RSA | 大数运算（big integer）、模幂 `ModExp` 结构、`p*q = n`、私钥 `d` |

### 2.2 哈希/校验的常量指纹

| 算法 | 识别线索 |
| --- | --- |
| MD5 | 初始化常量 `0x67452301, 0xEFCDAB89, 0x98BADCFE, 0x10325476` |
| SHA-1 | `0x67452301, 0xEFCDAB89, 0x98BADCFE, 0x10325476, 0xC3D2E1F0` |
| SHA-256 | `0x6a09e667, 0xbb67ae85, 0x3c6ef372, 0xa54ff53a…` 开头 8 个素数平方根 |
| CRC32 | 多项式 `0xEDB88320` 与 256 项查找表；`0xFFFFFFFF` 初始值/最终异或 |
| Adler-32 | `A = 1, B = 0`、`A = (A + data) % 65521, B = (B + A) % 65521` |

### 2.3 其他通用特征

- 大量 **移位/异或/加法轮换** 指令；
- 函数内部 **循环固定次数**（AES 10/12/14 轮、SHA-1 80 轮、MD5 64 轮）；
- 使用 **立即数表**（LUT）替代实时计算；
- 频繁使用 `rol`/`ror`（循环移位）的 x86/x64 指令。

## 三、还原通用步骤

### 步骤 1：定位关键函数

- 从外部接口出发：
  - Windows：`CreateFile`、`ReadFile`、`InternetReadFile`、`CryptDecrypt`、`BCryptDecrypt` 等 API 往往离算法最近。
  - Linux：`read`、`recv`、`SSL_read`、`EVP_CipherUpdate` 等。
- 对网络协议样本，在 `send`/`recv` 上下断点，观察入参和出参；
- 对固件样本，关注加密前后内存区域的数据差异（"已知明文攻击"思路）。

### 步骤 2：识别算法类型

- 通过前面的常量指纹快速在 IDA/Ghidra 中搜索（`Search → sequence of bytes`，或使用 FindCrypt/FindCrypt2 插件）。
- 若无标准常量，判断是"标准算法 + 自定义常量/表"还是"全新混淆算法"。
- 对自定义混淆，用 **输入-输出对（IO pairs）** 做黑盒建模：固定部分输入变化，观察输出变化规律。

### 步骤 3：还原算法结构

- 将目标函数 F5 反编译，清理变量名；
- 判断是否是常见算法的变体（AES 改 S-box、SHA 改初始常量、TEA 改 delta）；
- 若是自定义算法，根据"可逆/不可逆、是否有密钥、输入输出是否定长"等特征分类；
- 对加密算法，进一步区分"对称还是非对称、ECB/CBC/CFB/OFB/CTR/GCM 等模式"。

### 步骤 4：提取密钥/参数

- 密钥可能：
  - 硬编码在二进制常量中（最常见）；
  - 通过配置文件/注册表读取；
  - 通过 `PBKDF2/Scrypt/Argon2` 从口令派生；
  - 使用 RNG 动态生成但可被复现（例如错误地使用 `rand()` 作为 CSPRNG）。
- 对对称加密，若能拿到 1~2 对 明文/密文，即可暴力穷举较短密钥；
- 对非对称加密，往往只能还原公钥，私钥仍需在目标环境中获取或通过数学攻击（如低加密指数、共模攻击等）。

### 步骤 5：验证与复现

- 用 C/Python/Go 重写算法实现；
- 对二进制捕获的输入输出做回归测试；
- 若是协议逆向，用 Scapy/Socket 构造真实报文验证。

## 四、实战例子：识别并还原一个变种 TEA

1. IDA 中发现某函数包含常量 `0x9E3779B9`，且循环 `sum += 0x9E3779B9`；
2. F5 后看到典型的 `v0 += (((v1 << 4) ^ (v1 >> 5)) + v1) ^ (sum + key[sum & 3])` 结构；
3. 判断为 **XTEA**，但注意到 `sum & 3` 变为 `(sum>>11) & 3`，说明是自定义变体；
4. 从二进制中取出 4 个 32 位密钥；
5. 用 Python 复现 XTEA 解密，对捕获的密文解密验证。

## 五、常见难点与对策

| 难点 | 应对思路 |
| --- | --- |
| 算法被混淆/虚拟化（VMProtect） | 先在关键 API 上 Hook，捕获明文；或基于 trace 做黑盒还原 |
| 大量 inline 代码、无明确函数边界 | 通过字符串引用、常量反推位置；trace 中观察循环次数 |
| 使用自定义 S-box 与轮常数 | 识别算法模板（如 AES 结构）+ 提取自定义常量 |
| 密钥由用户输入、运行时获取 | 通过 Hook `recv`/`fgets`/密码框控件 API 捕获输入 |
| 算法是多次调用的组合（AES + HMAC + Base64） | 分段识别、每段独立还原 |

## 七、学习建议

- **先熟悉标准算法**：手写一遍 AES、SHA-1、TEA、RSA 的玩具实现，再去读 F5 反编译代码时会有"似曾相识"的感觉。
- **多做 CTF reverse 题**：复现题目时重点关注"我是怎么识别出算法的"，并把特征整理到自己的 cheat sheet。
- **结合 Frida 做运行时辅助**：对静态分析不清晰的算法，直接 Hook 关键函数，打印输入输出做黑盒分析。
- **总结"算法模板"**：把 AES/SHA/TEA/RSA 等结构写成模板，遇到变体时只需要替换常量和小改动。

算法还原的核心，其实是"从汇编/反编译代码里看出结构"。先识别，再建模，最后复现——这套流程反复操练，就能从"看不懂"逐步进阶到"还原得比原作者还清晰"。
