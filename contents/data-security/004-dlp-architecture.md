## 二、四层防线详解

### 2.1 终端DLP (Endpoint DLP)

```
管控点：
├── USB存储设备（U盘/移动硬盘）
├── 打印机（本地/网络打印）
├── 剪切板（复制/粘贴）
├── 截屏/录屏
├── 蓝牙/红外/WiFi Direct
├── 应用程序：即时通讯(微信/企业微信/钉钉)、
│   网盘客户端、FTP客户端、邮件客户端
├── 文件操作：重命名/压缩/加密外带
└── 进程注入检测（绕过DLP的恶意行为）

代理部署方式：
- Agent模式（最全面）：轻量级Agent常驻终端
- Agentless模式：通过RDP/VDI代理实现（局限较多）
```

### 2.2 网络DLP (Network DLP)

```
部署位置：网络出口（旁路镜像/串行代理/ICAP集成）

协议解析：
├── HTTP/HTTPS（SSL解密/证书替换代理）
├── SMTP/IMAP/POP3（邮件外发）
├── FTP/SFTP（文件传输）
├── 即时通讯协议（微信/企业微信/钉钉/飞书）
├── 网盘上传（百度网盘/阿里云盘/OneDrive等）
├── 云服务API（S3上传/GitHub推送）
└── 自定义TCP/UDP协议

检测技术：
├── 关键字/正则匹配
├── 文件指纹（文档DNA）
├── 数据标识符（身份证/银行卡/手机号正则）
├── 机器学习分类（文档内容分类）
├── 文件类型检测（防后缀名伪装）
└── 加密流量检测（异常加密流量行为分析）
```

### 2.3 邮件DLP

```
集成方式：
- MTA代理（Postfix/Sendmail Milter）
- Exchange Transport Agent
- O365/Gmail API集成
- SMTP网关前置过滤

检测策略：
├── 邮件主题/正文敏感词扫描
├── 附件内容扫描（Office/PDF/图片OCR）
├── 附件文件类型检查
├── 收件人域外检查（非企业域名收件人）
├── 大批量外发检测
├── 定时外发规律分析
└── 加密附件中转解密检测
```

### 2.4 云DLP/存储DLP

```
云DLP场景：
├── CASB集成（Microsoft Defender for Cloud Apps / Netskope / MVISION Cloud）
├── SaaS应用DLP（O365/Google Workspace/Salesforce/Box）
├── IaaS存储扫描（AWS S3/阿里云OSS/腾讯云COS）
├── 数据库DLP（SQL查询审计+结果集控制）
└── 大数据平台DLP（HDFS/Hive/Kafka）

存储扫描：
├── 定期扫描（全量+增量）
├── 敏感数据发现（关键字/正则/指纹）
├── 异常存储检测（公开Bucket/无加密存储）
└── 权限审计（过度授权/共享链接）
```

## 三、核心技术：内容识别

### 3.1 基于关键词/正则

```python
# DLP规则配置示例
DLP_RULES = {
    "身份证泄露": {
        "pattern": r'[1-9]\d{5}(19|20)\d{2}(0[1-9]|1[0-2])(0[1-9]|[12]\d|3[01])\d{3}[\dXx]',
        "threshold": 5,          # 检测到≥5条身份证号触发
        "action": "block",
        "severity": "high"
    },
    "银行卡泄露": {
        "pattern": r'\b\d{16,19}\b',
        "threshold": 3,
        "action": "block",
        "severity": "high"
    },
    "源代码泄露": {
        "keywords": ["password", "secret_key", "private_key", "token", "api_key", "access_key"],
        "file_types": [".py", ".java", ".go", ".js", ".ts", ".env", ".config", ".yml", ".yaml"],
        "action": "block",
        "severity": "critical"
    },
    "机密文档泄露": {
        "keywords": ["机密", "绝密", "内部资料", "敏感", "商业秘密"],
        "threshold": 3,          # 同时匹配≥3个关键词
        "action": "alert",
        "severity": "medium"
    }
}
```

### 3.2 基于文档指纹（Exact Data Matching）

```
文档指纹技术：
1. 选取需要保护的敏感文档
2. 提取文档内容，生成"指纹"（哈希分片/SimHash）
3. 配置指纹规则到DLP引擎
4. 检测外发内容，计算与指纹库的相似度
5. 相似度超过阈值则触发告警/阻断

优势：精确匹配，误报率低
局限：文档修改后可能漏检
```

### 3.3 基于机器学习

```
分类模型训练：
1. 标注训练集（正常文档 / 含敏感信息文档）
2. 特征提取（TF-IDF / Word2Vec / BERT embedding）
3. 训练分类模型（SVM / XGBoost / Transformer）
4. 模型部署到DLP检测引擎
5. 持续反馈优化（误报/漏报标注回馈）

优势：上下文理解，减少误报
局限：需要标注数据，模型可解释性弱
```

## 四、部署架构

### 4.1 典型企业部署

```
                    Internet
                       │
                 ┌─────▼─────┐
                 │ 网络DLP    │ ← 旁路镜像/串行
                 │ (出口网关) │
                 └─────┬─────┘
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
    ┌──────────┐ ┌──────────┐ ┌──────────┐
    │终端DLP   │ │邮件DLP   │ │CASB      │
    │(Agent)   │ │(MTA集成) │ │(API集成) │
    └────┬─────┘ └────┬─────┘ └────┬─────┘
         │            │            │
         └────────────┼────────────┘
                      ▼
               ┌─────────────┐
               │DLP管理平台   │
               │├ 策略管理    │
               │├ 事件中心    │
               │├ 报表审计    │
               │└ 响应流程    │
               └──────┬──────┘
                      ▼
               ┌─────────────┐
               │SIEM/SOC     │
               └─────────────┘
```

### 4.2 分阶段实施路线

```
Phase 1: 监控模式（1-2个月）
- 部署DLP，策略设为 "仅告警不阻断"
- 收集基线数据，了解数据流转现状
- 调整误报率，优化检测规则

Phase 2: 关键场景阻断（2-4个月）
- 核心敏感数据（代码/客户数据）外发阻断
- USB管控 + 邮件外发管控
- 建立应急审批通道（紧急外发需求）

Phase 3: 全面防护（4-6个月）
- 覆盖全部终端+网络+邮件+云
- 自动响应+SOAR编排（自动隔离/通知/回收）
- 数据安全态势感知大屏

Phase 4: 智能运营（持续）
- AI辅助误报识别
- 行为基线异常检测（UEBA集成）
- 数据安全风险评估自动化
```

## 五、运营与指标

### 5.1 事件处理SOP

```
DLP告警触发 → 事件分类分级 → 分析研判 → 处置 → 闭环
                              ├─ 误报：标记+优化规则
                              ├─ 正常业务：审批放行
                              └─ 真实泄露：阻断+溯源+追责
```

### 5.2 KPI指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| DLP Agent覆盖率 | ≥98% | 终端/服务器 |
| 策略命中准确率 | ≥85% | 告警中真实泄露占比 |
| 误报率 | ≤15% | 误报/总告警 |
| 事件响应时间 | ≤30min | 高优事件 |
| 数据泄露事件数 | 同比下降≥50% | 年度同比 |
| 敏感数据外发发现率 | ≥95% | 已知场景覆盖 |
| 扫描覆盖率 | 100% | 存储DLP全量扫描 |
