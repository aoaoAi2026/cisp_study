# HW 行动（护网）蓝队防御实施方案（模板）

## 1. 方案概述

本方案适用于**省级/市级/央企护网行动**期间的蓝队防御实施。目标是在不影响业务可用性的前提下，最大化降低风险暴露面、提升威胁发现与响应速度，做到：

- **事前**：资产与风险梳理、基线加固、防御策略预部署
- **事中**：7×24 值守、监测、研判、处置、溯源
- **事后**：复盘、整改、长效机制建设

## 2. 组织与分工

### 2.1 组织架构

```
总指挥（甲方 CISO / 信息中心主任）
  ├── 蓝队防御组（监测 + 研判 + 处置）
  │     ├── 监测组：SIEM / SOC / 告警值班
  │     ├── 研判组：攻击链分析、误报排除、关联分析
  │     └── 处置组：封禁、隔离、策略调整、取证
  ├── 业务保障组：业务方 / 运维 / 应用负责人
  ├── 外联组：与公安 / 等保测评机构 / 主管单位沟通
  └── 红队组：内部对抗演习（可选）
```

### 2.2 角色职责

| 角色 | 职责 |
|------|------|
| 总指挥 | 重大事件决策、资源协调、对外沟通 |
| 监测岗 | 7×24 监控安全设备告警、发现异常 |
| 研判岗 | 分析告警、确认是否为真实攻击、判断风险 |
| 处置岗 | 执行封禁、隔离、策略修改、应急操作 |
| 业务岗 | 配合业务影响评估、协助回滚/下线 |
| 值守经理 | 班次管理、交接班、汇总报告 |

## 3. 事前准备阶段（开战前 7~14 天）

### 3.1 资产梳理（参考配套资产 Checklist）

- 业务系统清单、对外暴露 IP/域名、开放端口清单
- 云资产 / IDC / 第三方代维资产全覆盖
- 关键业务资产标注优先级（T0/T1/T2）

### 3.2 风险自查

- 弱口令扫描：SSH/RDP/数据库/中间件管理后台
- 端口最小化：关闭非必要对外端口
- 高危服务：RMI / JNDI / Redis 未授权访问
- 历史漏洞：CVE-2021-44228(Log4j)、Spring RCE、ThinkPHP、Exchange SSRF 等

### 3.3 基线加固

- 操作系统基线（参照 CIS Benchmark / GB/T 22239）
- 中间件基线：Nginx/Apache/Tomcat/IIS 默认版本升级
- 数据库基线：删除测试库、限制访问源、最小权限账号
- 边界设备：防火墙策略收敛、NAT 策略梳理、ACL 最小权限

### 3.4 防御策略预部署

- WAF：默认规则 + 虚拟补丁（如近期 1day）
- IPS/IDS：启用常见 Web 攻击、暴力破解、扫描规则
- HIDS / ED R：Agent 全覆盖，关键服务器启用进程/文件/网络监控
- DNS 流量监控：识别 DGA、TUNNEL 行为
- 邮件网关：SPF / DKIM / DMARC 检查，钓鱼邮件识别规则

## 4. 事中值守与响应阶段（7×24）

### 4.1 值守排班（参考配套 7×24 排班模板）

- 采用 4 班 3 运转或 3 班运转，每班 8 小时
- 每班至少 3 人：1 监测 + 1 研判 + 1 处置
- 交接班必须面对面 + 纸质/电子记录双签字

### 4.2 事件分级

| 级别 | 场景示例 | 响应时限 |
|------|---------|---------|
| Ⅰ级（重大） | 核心系统被攻陷、勒索成功、数据外泄 | 立即上报总指挥 |
| Ⅱ级（高危） | 高危漏洞被利用、核心系统告警异常 | 30 分钟内处置 |
| Ⅲ级（中危） | 持续扫描、低危利用尝试、钓鱼攻击 | 2 小时内处置 |
| Ⅳ级（低危） | 外部扫描、误报 | 当班内完成 |

### 4.3 事件响应 SOP

1. **发现**：告警触发 / 情报推送 / 业务报障
2. **研判**：判断是否真实攻击、攻击类型、影响范围
3. **遏制**：临时封禁 IP、下线攻击资产、阻断端口
4. **根除**：清理 webshell、修复漏洞、清除后门
5. **恢复**：业务恢复、观察、回写策略
6. **取证与上报**：保存日志、截图、抓包；填写工单；向上级单位上报

### 4.4 常见攻击场景处置要点

#### 场景 A：Web 应用被植入 webshell

1. 立即隔离被攻陷主机（断网或 ACL 阻断）
2. 保留 webshell 样本（用于溯源/取证）与访问日志
3. 搜索同类文件（时间/路径/关键字）
4. 修复漏洞（SQL 注入、文件上传、RCE）
5. 重置受影响账号密码
6. WAF 增加对应攻击特征规则

#### 场景 B：内部账号密码被爆破

1. 立即锁定被爆破账号 / 修改密码
2. 查看登录来源 IP、时间、失败次数
3. 判断是否为真实弱口令 / 内网横向
4. 启用登录失败锁定、加 MFA、加异地登录告警
5. 回溯登录成功记录，核查是否存在后续操作

#### 场景 C：钓鱼邮件成功投递

1. 邮件网关紧急投递样本到同类收件人
2. 已点击附件/链接的用户终端排查
3. 恶意域名封禁（DNS / 代理 / 防火墙）
4. 全员安全意识邮件提醒

## 5. 事件上报与通报

### 5.1 上报渠道

- 内部工单系统（每起事件一单）
- 护网行动指挥系统 / 上报平台（按主管单位要求）
- 重大事件电话 + 邮件 + 微信同步

### 5.2 日报模板（当班每日 09:00 前）

```
【XX日护网日报】
- 当班累计攻击事件：X 起（Ⅰ级 Y，Ⅱ级 Z ...）
- 封禁 IP 数量：X 个
- 新增/调整 WAF 规则：X 条
- 业务影响情况：无 / 影响 XXX 业务 XX 分钟
- 重点事件摘要：...
- 需上级协调事项：...
```

## 6. 复盘与长效整改（护网结束后 7 天内）

- 攻击链复盘：每起真实事件从源头 → 漏洞利用 → 横向 → 影响全链路分析
- 扣分点梳理：按资产/系统/人员/流程四类归因
- 整改优先级矩阵（概率 × 影响）
- 30/60/90 天整改计划与责任人排期

详见配套 [护网复盘报告模板](./hw-006)。

## 7. 法律与合规注意事项

- 所有攻防操作必须在授权范围内进行
- 溯源操作不得超越授权（避免对第三方系统发起主动扫描）
- 保留完整的日志、截图、操作记录（至少 6 个月）
- 涉及个人信息的数据需按《个人信息保护法》处理
