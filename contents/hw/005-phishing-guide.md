# 护网钓鱼邮件识别与处置实战指南

## 1. 钓鱼邮件常见套路

### 1.1 社会工程学手段

- **伪装领导**：CEO / 上级微信 / 邮件紧急指令「请立即转账 / 请登录以下系统」
- **伪装 HR / 行政**：「工资条 / 考勤调整 / 防疫通知 / 体检报告」
- **伪装业务伙伴**：「订单合同 / 发票 / 邀请函」
- **伪装 IT 管理员**：「密码即将过期 / 邮箱配额已满请登录修改」
- **伪装国家部委 / 公检法**：「请核查贵单位信息 / 接受调查」

### 1.2 常见载体

| 类型 | 示例 |
|------|------|
| 恶意附件 | `.docm` / `.doc`（含宏）、`.xlsb` / `.xlsm`、`.lnk`、`.iso`、`.zip`（内藏 `.hta` / `.js` / `.wsf`） |
| 恶意链接 | 仿冒登录页（域名极似）、`bit.ly` 等短链接、Unicode 字符同形 |
| HTML smuggling | HTML 中利用 JS 动态生成下载 |
| 图片鱼叉式 | 二维码 / 图片诱导扫码至恶意站点 |

## 2. 识别特征速查

### 2.1 邮件头

- 发件域名与官方不一致（`@huawei.co` vs `@huawei.com`）
- `Reply-To` 与 `From` 不一致
- `Received` 中显示中继服务器可疑
- SPF 检查失败（`Received-SPF: fail` / `softfail`）
- DKIM / DMARC 失败

### 2.2 内容特征

- 紧迫语气：「限时 2 小时」「今天必须完成」
- 涉及金钱、账号密码、个人信息索取
- 链接文本与实际 href 不一致（鼠标悬停查看）
- 附件后缀可疑：`.docm`、`.js`、`.lnk`、`.iso`、`.bat`
- 语法不自然、来自翻译器痕迹
- 要求使用「屏幕共享 / 远程协作」工具

### 2.3 链接/附件识别命令

```bash
# 检查文件类型（即便改名也要看真正魔数）
file suspicious.doc
file suspicious.zip

# 解压缩并检查内容
7z l suspicious.zip
unzip -l suspicious.zip

# 检查宏（需 olevba / 工具）
olevba -a suspicious.docm

# 查询域名/IP 威胁情报
curl -s https://api.abuseipdb.com/api/v2/check?ipAddress=X.X.X.X
```

## 3. 处置 SOP

1. **隔离**：对已收到的邮件，在网关侧删除同类邮件
2. **通报**：向全员快速发提醒（短信/邮件/IM），提供样本截图
3. **分析**：提取附件哈希 / 恶意 URL，提交沙箱并加入黑名单
4. **终端排查**：对已点击/下载员工的终端进行 ED R / 进程检查
5. **阻断**：
   - 邮件网关：域名、附件哈希、发件 IP
   - DNS / 代理：恶意域名
   - 防火墙：恶意 IP / C2
6. **溯源**：分析投递链、受害员工、是否进一步发生数据外泄或横向移动
7. **留存**：保留完整邮件（eml）、附件样本、日志截图

## 4. 应急处置脚本（模板，仅供参考）

```bash
# 1) 提取邮件附件哈希并加黑名单
sha256sum malware.docm   # 输出哈希 -> 填入 EDR/邮件网关

# 2) 封禁恶意 IP（Linux iptables 示例）
iptables -A INPUT -s 192.0.2.0/24 -j DROP
iptables -A OUTPUT -d 192.0.2.0/24 -j DROP
iptables-save > /etc/iptables/rules.v4

# 3) 封禁域名（本地 DNS / hosts 文件示例）
echo "0.0.0.0 evil.example.com" >> /etc/hosts

# 4) 搜索终端上的可疑文件路径
find / -name "*.docm" -o -name "*.lnk" -newer /tmp/flag 2>/dev/null
```

## 5. 长效建设

- SPF + DKIM + DMARC（拒绝策略 `p=reject`）
- 邮件网关沙箱 + 宏默认禁用
- Office 宏禁用（GPO 策略「禁用所有无数字签名的宏」）
- 员工安全意识培训与钓鱼演练（至少每季度一次）
- 建立举报邮箱与快速响应流程

## 6. 真实案例场景分析

**场景：HR 部门收到「2025 年度调薪通知.docm」**

1. 邮件网关检测：发现包含宏 VBA，触发预警
2. 人工复核：「发件人并非 HR，附件后缀为 .docm，且来自外部邮箱」
3. 沙箱分析：运行后外联至 `192.0.2.66:443`，尝试下载 payload
4. 处置：
   - 邮件网关紧急过滤同类邮件
   - 恶意域名与 IP 全网络黑名单
   - 对已点击员工进行终端隔离与查杀
   - 发全员提醒 + HR 部门重点沟通
5. 复盘：更新邮件规则，增加「外部邮件附件 .docm 默认隔离」策略
