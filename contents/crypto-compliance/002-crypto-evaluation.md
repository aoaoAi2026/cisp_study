# 商用密码应用安全性评估(密评)实战指南

---

## 一、密评标准体系

### 1.1 核心标准

```
密评核心标准演进：
  GM/T 0054-2018 → GB/T 39786-2021 (国家标准)

GB/T 39786-2021 《信息安全技术 信息系统密码应用基本要求》

两大组成部分：
  1. 密码应用技术要求
  2. 密码应用管理要求

测评依据：GM/T 0115-2021 + GM/T 0116-2021
```

### 1.2 密评范围

```
需要密评的信息系统范围：
  - 关键信息基础设施 (CII)
  - 网络安全等级保护三级及以上系统
  - 国家政务信息系统

密评的两种形式：
  1. 密码应用方案评估（建设前）
     → 审查密码应用方案是否满足GB/T 39786要求
  2. 密码应用安全性评估（建设后/运行中）
     → 评估实际部署情况是否符合方案和标准
     → 评估周期：关基≥1次/年，等保三级≥1次/年
```

## 二、密评五大维度

### 2.1 物理和环境安全

```
测评项：
  1. 身份鉴别 — 进入机房/机柜的双因素认证
     → 采用密码技术的门禁卡+指纹/PIN
     → 国密实现：SM2签名证书或SM4加密的RFID卡

  2. 电子门禁记录完整性 — 进出记录防篡改
     → 门禁日志附加HMAC-SM3完整性校验码

  3. 视频监控记录完整性 — 监控录像防篡改
     → 视频文件的SM3哈希值链
```

### 2.2 网络和通信安全

```
测评项 ★重点★：
  1. 通信双方身份鉴别
     → 国密TLCP协议(SM2证书双向认证)
     → 或IPSec VPN使用国密算法

  2. 通信数据机密性
     → SM4加密（对称，替代AES）
     → 密钥长度≥128位

  3. 通信数据完整性
     → SM3-HMAC完整性校验

  4. 证书体系
     → 使用国家认可的CA签发的国密证书
     → 或经国家密码管理局认可的第三方CA
```

### 2.3 设备和计算安全

```
测评项：
  1. 身份鉴别 — 运维人员登录使用密码技术双因素
     → 国密USBKey (SM2) + PIN码
     → 堡垒机集成国密证书认证

  2. 远程管理通道安全
     → SSH使用国密算法(SM2密钥交换+SM4加密+SM3 MAC)
     → 或通过堡垒机国密TLCP代理

  3. 系统资源访问控制信息完整性
     → 配置文件/ACL清单的SM2签名/SM3-HMAC

  4. 日志记录完整性
     → 系统日志附加SM3-HMAC标签
     → 安全审计日志SM2签名保护

  5. 重要可执行程序完整性
     → 可执行文件SM3哈希值校验
```

### 2.4 应用和数据安全

```
测评项：
  1. 身份鉴别 — 应用系统登录使用密码技术
     → 登录密码SM3存储（不存储明文/简单哈希）
     → 或SM2数字证书登录

  2. 访问控制信息完整性
     → 权限配置表SM3-HMAC保护

  3. 重要数据传输机密性
     → SM4加密传输（前端应用→后端服务）

  4. 重要数据存储机密性
     → 数据库敏感字段SM4加密
     → 数据库透明加密(使用SM4国密算法)

  5. 重要数据存储完整性
     → SM3-HMAC或SM2数字签名

  6. 不可否认性
     → 关键操作(审批/确认/签名)使用SM2数字签名
     → 带时间戳的签名(Timestamp)
```

### 2.5 密钥管理

```
测评项：
  1. 密钥全生命周期管理
     ├── 密钥生成：使用国密局批准的随机数生成器
     ├── 密钥存储：加密存储（不用明文）
     ├── 密钥分发：加密传输
     ├── 密钥使用：仅授权实体可用
     ├── 密钥更新：定期更换
     ├── 密钥备份：加密备份（门限方案）
     ├── 密钥恢复：多因素授权恢复
     ├── 密钥归档：安全归档+审计
     └── 密钥销毁：安全擦除(不可恢复)

  2. 密码服务统一管理
     → 密码服务平台/密码资源池
     → 实现密码运算服务化（加解密/签名/验签/密钥管理API化）
```

## 三、密评评分

### 3.1 评分规则

```
GM/T 0115-2021 测评结果计算：

每个测评单元：符合/部分符合/不符合/不适用

量化评分（简化）：
  - 全部符合 + 无高风险项 → 符合（通过）
  - 部分不符合 但无高风险 → 部分符合（有条件通过，限期整改）
  - 存在高风险项 → 不符合（不通过，必须整改后复测）

高风险项（一票否决）：
  ✗ 使用已被攻破/不安全的密码算法（如DES/MD5/SHA-1）
  ✗ 密码产品无国家密码管理局认证证书
  ✗ 核心系统明文传输（无任何加密措施）
  ✗ 密钥管理严重缺陷（硬编码密钥/密钥明文存储）
```

## 四、高分过评策略

### 4.1 常见扣分项与应对

```
Top 10 扣分项：
  1. 未使用国密算法(使用AES/RSA/SHA-256)
     → 替换为国密算法或双算法支持

  2. 未使用经认证的密码产品
     → 选购带有国密局型号证书的产品

  3. 密钥明文存储(配置文件/hardcode)
     → 迁移到KMS/HSM

  4. 证书管理不规范
     → 建立证书生命周期管理(签发/更新/吊销)

  5. 日志无完整性保护
     → 日志系统增加SM3-HMAC链条

  6. 远程管理通道不安全
     → 启用国密TLCP堡垒机

  7. 数据存储未加密
     → 数据库国密透明加密

  8. 密码应用管理制度缺失
     → 编写密码管理制度文件

  9. 无密码应用方案或方案未经评审
     → 委托密评机构进行方案评审

  10. 人员无密码相关培训记录
      → 组织国密培训+考试留档
```

## 五、密评流程

```
密评全流程：

Step 1: 确定密评范围
  → 确定测评系统边界和等级(关基/三级/政务)

Step 2: 签订密评合同
  → 选择有资质的密评机构(国家密码管理局公布的机构名录)

Step 3: 方案/系统评估
  → 方案评估：120人天(约6周)
  → 系统评估：150-200人天(约8-10周)

Step 4: 现场测评
  → 文档审查+配置检查+渗透测试+访谈

Step 5: 出具报告
  → 测评报告经国家密码管理局审核备案

Step 6: 整改后复测(如需)
  → 按不符合项逐项整改 → 复测通过

Step 7: 年度监督
  → 关基+等保三级系统 ≥1次/年
```
