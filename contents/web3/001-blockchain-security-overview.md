# 区块链安全全景：共识/网络/应用层攻击面

---

## 一、区块链分层安全模型

```
区块链分层架构 × 安全风险：

L4 应用层 (DApp/DeFi/NFT/钱包)
  ├── 智能合约漏洞(重入/整数溢出/逻辑漏洞)
  ├── 前端攻击(XSS/供应链攻击/DNS劫持)
  └── 钓鱼攻击(假DApp/签名骗取/Approve授权)

L3 合约层 (EVM/WASM/Move VM)
  ├── 虚拟机实现漏洞(RCE/DoS)
  ├── Gas消耗攻击(超额Gas→交易失败)
  └── 预编译合约漏洞(EVM预编译合约)

L2 共识层 (PoW/PoS/BFT/DPoS)
  ├── 51%攻击(算力/质押超过半数→重写历史)
  ├── 长程攻击(获得历史私钥→重新分叉)
  ├── Nothing at Stake(PoS早期问题)
  ├── 自私挖矿(Strategic Mining→获利超过份额)
  └── 时间劫持(操纵节点时间→影响区块时间)

L1 网络层 (P2P/Gossip)
  ├── 日蚀攻击(Eclipse Attack): 隔离节点→控制其视角
  ├── 路由劫持(BGP Hijack): 重定向挖矿流量
  ├── 女巫攻击(Sybil): 大量虚假节点→污染网络
  └── DoS攻击(洪泛交易/区块→网络拥塞)

L0 数据层 (区块链数据结构)
  ├── 区块篡改(需>51%算力重计算所有后续区块)
  ├── 默克尔树攻击
  └── 交易可塑性(TxID可被修改→双花风险)
```

## 二、51%攻击 深度分析

### 2.1 攻击原理

```
PoW 51%攻击：
  攻击者获得全网>50%算力 → 
  秘密挖一条比主链更长的分叉 → 
  在分叉中包含双重支付交易(主链中已花费的币) → 
  当分叉长度>主链时广播 → 全网切换到攻击者分叉 → 
  主链上的交易被回滚 → 攻击者成功双花

攻击成本估算 (BTC为例 2024):
  全网算力: ~600 EH/s
  攻击1小时所需: ~3600 EH
  租用算力成本(NiceHash): $1.5M-$3M/小时
  → BTC攻击成本极高(防御优势)

  对于小算力PoW链:
  全网算力<1 TH/s → 租用成本<$100/小时 → 可攻击!

PoS 51%攻击(称为>33%或>67%攻击)：
  PoS用质押代替算力
  34% 质押: 可做小规模审查(阻止交易被打包)
  51% 质押: 可做最终确定攻击(修改已经Finalized的区块)
  67% 质押: 完全控制

防止51%攻击的机制：
  - 经济罚没(Slashing): PoS中作恶=没收质押金
  - Social Consensus(社会共识): 社区可硬分叉移除攻击者的链
  - 检查点(Checkpoint): 深度足够的历史区块不可回滚
  - 被动防御: 联合挖矿(Merged Mining,借用大链算力)
```

## 三、日蚀攻击

```
Eclipse Attack (日蚀攻击)：

原理：
  攻击者垄断目标节点的所有P2P连接
  目标节点看到的区块链世界完全由攻击者控制

执行：
  P2P网络选择出站连接的机制被利用
  → 攻击者创建大量节点(拥有特定IP/端口组合)
  → 不断尝试连接目标节点
  → 填满目标节点的出站连接槽位(通常8-125个)
  → 目标节点所有出站连接都是攻击者控制的节点

攻击效果：
  - 双花：目标节点看到的是攻击者的"假链"
  - 交易审查：目标节点的交易永不被广播
  - 挖矿劫持：攻击者窃取目标节点的挖矿奖励

防御(Bitcoin Core 0.15+):
  - 出站连接限制：每个/16网段最多1-2个连接
  - 连接多样性：优先连接不同AS/不同国家节点
  - Anchored Connections(锚定连接): 少数长期信任连接
```

## 四、各主流链安全模型

| 链 | 共识 | 安全特性 | 关键风险 |
|-----|------|---------|---------|
| **Bitcoin** | PoW (SHA256) | 最大算力、最长历史 | 51%攻击(理论可能/经济不可行) |
| **Ethereum** | PoS (Gasper: Casper + GHOST) | 经济最终确定性 | 客户端多样性不足(>80% Geth) |
| **Solana** | PoH+PoS (Tower BFT) | 高吞吐、低延时 | 中心化(验证人数量少)、网络中断历史 |
| **Cosmos** | Tendermint BFT | 即时最终确定性 | 验证人合谋风险 |
| **Sui/Aptos** | 共识各不相同 | Move语言安全性 | 新兴链、生态不成熟 |
