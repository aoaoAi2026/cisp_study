## 二、分类说明

### 2.1 要求独立 Windows 操作系统（Kali Linux 无法满足）

**靶场清单**：
- Day091 IThema-Labs：必须 Windows Server 2008 R2 Enterprise SP1 x64
- Day094 WebBug：必须 Windows + PHPStudy + PHP 5.x
- Day105 红日靶场（域环境）：必须多台 Windows Server（域森林/子域/父域/域控）
- Day106 企业级靶场：含防火墙/WAF/IDS 等，多 OS 多网段

**核心原因**：
Kali Linux 是 Debian 系 Linux 发行版，无法运行 Windows 原生驱动、IIS、MSSQL、AD 域控等 Windows 专属组件。要搭建这些靶场必须额外安装独立 Windows 虚拟机（VMware / VirtualBox / Hyper-V），超出当前 Kali 单 OS 环境范围。

---

### 2.2 要求多 VM 内网 / 域环境（单机不适用）

**靶场清单**：
- Day095 VulStack：Win7 + Win2008 + Win2016 + Kali 多机内网
- Day104 WebSphere/CVE 2020 系列：多 OS 容器 + VM 混合组网
- Day105 红日域环境靶场：多 Windows Server 组域
- Day106 企业级 APT 靶场：多网段、多硬件、多安全设备

**核心原因**：
这些靶场用于模拟真实企业内网渗透场景，需要网络分段、域信任关系、跳板代理等多机协作环境。单台 Kali VM 即使资源足够，也无法在逻辑上满足多机拓扑要求。

---

### 2.3 官方 ISO 镜像类（需要新 VM 安装）

**靶场清单**：
- Day099 WebForPentester I/II & NodeBug

**核心原因**：
官方以 ISO 镜像形式分发，相当于一个独立的 OS 发行版，需要作为单独虚拟机完整安装启动，无法作为 Kali 内部的一个服务或目录直接部署。

---

### 2.4 依赖 Vulhub Git 仓库（仓库 1.3GB 暂未完整落盘）

**靶场清单**：
- Day100 Fastjson 反序列化
- Day101 Log4j2 CVE-2021-44228
- Day102 Shiro 反序列化 CVE-2016-4437
- Day103 Nginx 解析漏洞

**核心原因**：
以上 4 个靶场的 Docker Compose 配置文件均位于 Vulhub 仓库对应目录中。因 Vulhub 仓库 1.3GB 体积过大，国内网络 clone 超过 300s 阈值暂未完成落盘。

**缓解方案（可用但暂未执行）**：
无需完整仓库，可以直接 `docker pull` 对应官方镜像 + 手动 docker-compose.yml 启动。详见《搭建Fastjson+Log4j2+Shiro靶场过程问题及后续方案.md》。

---

### 2.5 状态待验证类（可能已存在，需 curl 确认）

**靶场清单**：
- Day086 SQLi-Labs（靶场2）
- Day088 Pikachu（靶场4）

**核心原因**：
之前的部署记录中目录 `/var/www/html/sqli-labs/` 和 `/var/www/html/pikachu/` 可能已存在，但未经过 HTTP 级别的完整验证。

**验证方式**：
详见《day086-靶场2-SQLi-Labs和day088-靶场4-Pikachu验证过程.md》，文档中包含：
- curl 验证是否存在
- 若不存在的 Docker 一键启动方式
- 文件存在但 HTTP 404 的处理方式
- 备用 Dockerfile

## 三、后续搭建建议（优先级排序）

| 优先级 | 内容 | 工作量 | 说明 |
|--------|------|--------|------|
| P0 | 验证 SQLi-Labs + Pikachu 当前状态 | 30分钟 | curl 即可确认，缺什么补什么（Docker 5分钟内拉起） |
| P0 | 补起 Fastjson + Log4j2 + Shiro + Nginx 四个 Docker 单容器 | 1小时 | 无需 Vulhub 仓库，直接 docker pull 对应镜像 |
| P1 | 夜间低峰期浅克隆 Vulhub 仓库（`--depth=1`） | 2~4小时 | 只取最新 commit，体积从 1.3GB 缩小很多 |
| P2 | 额外申请一台 Windows Server 2008/2016 VM | 资源申请 | 用于 IThema-Labs、WebBug 等 Windows 靶场 |
| P3 | 规划 4~6 台 VM 内网组域 | 高 | VulStack、红日、企业级靶场 |
