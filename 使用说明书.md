## 1. 平台概述

### 1.1 项目简介

**CISP 网络安全学习平台** 是一个基于 **React 18 + TypeScript + Vite 6** 的前端单页应用（SPA），专注于 CISP（注册信息安全专业人员）认证考试备考、网络安全知识学习、代码实训练习、在线工具集等功能。

本平台的核心理念：

- 📚 **系统化学习**：90 天完整学习路径 + 专项学习计划
- 💻 **安全实操**：XSS、SQL 注入、CTF、密码破解、RSA 等多种安全实验
- 🧪 **在线工具箱**：200+ 纯前端工具，覆盖编码解码/哈希/生成器/换算
- 🎮 **游戏化激励**：XP/等级/徽章/连续打卡/Confetti 动画
- 🛡️ **稳定性优先**：ErrorBoundary 全局错误边界 + 本地持久化

### 1.4 平台特色

- 📚 系统化学习：90 天完整 CISP 学习路径 + 专项计划
- 🔧 在线工具箱：200+ 实用工具，纯前端实现，无需后端，无数据上传
- 🎮 游戏化激励：XP 经验值、等级系统、连续打卡、徽章成就

- 📝 模拟考试：历年真题库 + 模拟考试 + 错题本 + 章节测验

- 🛡️ 稳定性：全局 ErrorBoundary，单个页面异常不影响整个应用

## 2. 快速开始

### 2.1 环境要求

- **Node.js** ≥ 18
- **npm** ≥ 9
- Windows / macOS / Linux 均支持

### 2.2 安装依赖

```bash
cd cisp
npm install
```

### 2.3 启动开发服务器

```bash
npm run dev
```

默认端口：http://localhost:6776

如果 6776 被占用，Vite 会自动切换到下一个可用端口（如 6777、5173 等）。

### 2.4 生产构建

```bash
# 1. TypeScript 类型检查 + Vite 构建（推荐）
npm run build

# 2. 预览构建产物（本地测试）
npm run preview
# 访问 http://localhost:4173
```

构建产物输出目录：`dist/`

### 2.5 类型检查（仅检查，不构建）

```bash
npm run check
```

等价于 `tsc --noEmit`，可在 CI/CD 中使用。

### 2.6 Lint 检查

```bash
npm run lint
```

## 4. 核心页面功能详解

平台共 **24 个页面**，按功能划分为 7 大板块。

### 4.1 登录/注册 (AuthPage)

**路由**: `/auth`（公开页面）

#### 功能

- 登录表单（用户名 + 密码）
- 注册表单（用户名 + 邮箱 + 密码）
- 一键游客登录（自动创建临时账号）
- 错误提示（用户名不存在、密码错误、网络异常）
- 表单验证（用户名 ≥3 字符，密码 ≥6 字符）

#### 数据持久化

- 纯前端：数据写入 `localStorage`
- 完整模式：数据写入后端 SQLite

### 4.2 首页仪表盘 (Dashboard)

**路由**: `/`（需要登录）

#### 布局区域

1. **欢迎区**：用户昵称、连续学习天数、距 CISP 考试倒计时
2. **统计卡片**：累计学习天数、连续打卡、已获徽章、总 XP
3. **每日挑战**：每天随机生成 8 种挑战
4. **本周打卡日历**：周一至周日已完成/未完成状态
5. **学习小贴士**：随机刷新一条学习建议
6. **学习进度环**：环形进度条（Recharts 风格）
7. **今日任务列表**：勾选式学习任务
8. **12 周学习路径概览**：快速跳转到任意周
9. **学习数据统计**：番茄钟、笔记、徽章总览

### 4.7 代码运行中心 (CodeRunner)

**路由**: `/lab/code-runner`

#### 功能

- **多标签页编辑器**：同时打开多份代码
- **语言切换**：Python / JavaScript / Java
- **执行方式**：
  - Pyodide（浏览器内 Python，无需后端）

- **stdin 输入区**：支持命令行参数输入

- **性能基准测试**：多次运行耗时对比（柱状图）
- **执行历史**：保存/加载历史运行
- **代码 Diff**：两个版本代码差异对比
- **片段库**：11 个内置安全代码片段
- **代码格式化** / **全屏模式** / **下载文件**

## 10. 错误边界与稳定性

### 10.1 全局 ErrorBoundary

**文件**: `src/components/ErrorBoundary.tsx`

这是 v2.0 最重要的稳定性增强。其作用：

- ✅ **防止单点故障导致整体崩溃**：单个页面/组件出错不会导致整个应用白屏
- ✅ **用户友好提示**：显示清晰的错误信息，而非 React 开发错误页面
- ✅ **一键重试**：提供"刷新页面"按钮
- ✅ **开发者友好**：开发模式下显示详细错误栈

**使用位置**：`src/App.tsx` → 包裹整个 `<Layout />`

### 10.2 其他容错机制

| 机制 | 位置 | 说明 |
|------|------|------|
| 路由降级 | `App.tsx` | `*` → `/auth` |
| localStorage 异常保护 | `LocalStorageDB.ts` | 读写异常时降级为临时内存存储 |
| API 超时处理 | `api/client.ts` | >5s 自动降级为纯前端模式 |
| 单个工具计算容错 | `OnlineTools.tsx` | try-catch 包裹工具计算函数，异常时显示错误信息而非整体报错 |
| React StrictMode | `main.tsx` | 开发模式启用双重渲染以检测不安全的副作用 |

## 15. 常见问题 FAQ

### Q: 必须启动后端才能使用吗？

**A**: 不需要。大部分功能（课程浏览、Python 代码运行、测验、实验、在线工具箱等）纯前端即可使用。后端仅提供用户注册/登录、社区发帖、后端执行多语言代码等增强功能。

### Q: 数据存在哪里？会丢失吗？

**A**: 浏览器中使用 `localStorage`（封装在 `src/data/LocalStorageDB.ts`），APK 中使用 `@capacitor/preferences`（封装在 `NativeStorage.ts`）。只要不清空浏览器存储/卸载 App，数据不会丢失。建议定期在"个人中心 → 导出数据"备份。

### Q: 在线工具箱的数据会上传到服务器吗？

**A**: 绝对不会。所有 200+ 工具的计算都在浏览器端完成，纯前端实现，无任何网络请求。适合处理敏感数据（密码、密钥、Token 等）。

### Q: 在线工具箱有多少工具？如何分类？

**A**: v2.0 提供 200+ 工具，分为 12 大类：编码解码、哈希计算、进制转换、安全工具、开发工具、生成器、文本处理、颜色/图像、时间/日期、网络工具、计算工具、其他实用。详见 **第 5 章**。

### Q: 代码运行支持哪些语言？

**A**:

- **Pyodide（浏览器端，无需后端）**：Python
- **后端 API（需启动后端）**：Python / JavaScript / Java

### Q: 模拟考试可以中途退出/保存吗？

**A**: 模拟考试有暂停功能（隐藏题目），但不支持保存进度。交卷或超时后自动评分。

### Q: 题库/课程内容可以修改吗？

**A**: 可以。所有题库、课程内容存储在 `src/data/` 目录下的 31 个 TypeScript 文件中。编辑对应 `.ts` 文件后重新 `npm run build` 即可生效。

### Q: 如何重置学习进度？

**A**: 个人中心 → 学习数据管理 → 重置数据（⚠️ 会清空所有进度，请先导出数据备份）。

### Q: 为什么使用 Zustand 而不是 Redux？

**A**: Zustand 是一个极简的状态管理库（API 仅 2-3 个函数），相比 Redux：
- ✅ 无 Provider 嵌套，代码简洁
- ✅ TypeScript 类型推导友好
- ✅ 对中等规模项目足够用
- ✅ 支持 middleware（persist 等）
- ✅ 学习成本极低

### Q: 为什么要加 ErrorBoundary？它解决了什么问题？

**A**: 任何 React 组件内部抛出未捕获异常时（例如数据文件格式错误、空指针、API 变化），React 默认会把整个应用渲染树卸载变成白屏。ErrorBoundary 相当于"try-catch for UI"，捕获异常并显示友好错误页，保证其他功能仍能正常使用。

### Q: 构建失败怎么办？

**A**: 请按以下步骤排查：

1. `npm install` 重新安装依赖
2. `npm run check` 查看具体类型错误
3. 查看控制台报错中的文件路径，定位问题
4. 如修改了 `src/data/*.ts`，确认导出对象格式与其他文件一致

### Q: 为什么会有路由重定向？

**A**: v2.0 对路由结构进行了规范化整理，但保留旧路径映射以兼容历史链接。如果你有分享过旧的链接（例如 `/learning/20`），访问时会自动跳到新路径。

### Q: 如何添加新的工具到在线工具箱？

**A**: 在 `src/pages/OnlineTools.tsx` 中：
1. 在工具数组中添加新工具项（`id`/`name`/`icon`/`desc`/`category`）
2. 在 `specialProcess()` 函数中添加该工具 `case` 分支，实现输入→输出的逻辑
3. 如果工具需要 encode/decode 切换模式，在 `encode()` / `decode()` 中添加对应 `case`
4. 在 `isSpecial` 列表中加入对应 `id`（如果它不走 encode/decode 模式）

### Q: 如何添加新的专项学习计划？

**A**: 1) 在 `src/data/` 创建一个数据文件；2) 在 `CyberLearningMain.tsx` 或 `InterviewLearningMain.tsx` 中添加入口卡片；3) 添加对应路由到 `App.tsx`。

### Q: 为什么 React 18 不是 React 19？

**A**: React 18 是目前最稳定、生态最完善的版本。所有依赖（Monaco、Milkdown、Recharts、Zustand v5）都已验证兼容。

### Q: 为什么 Vite 6 不是 Vite 7？

**A**: Vite 6 是 LTS 版本，已足够成熟稳定。升级不会带来用户可感知的功能提升，会增加兼容性风险。

## 16. 故障排除

| 现象 | 可能原因 | 解决方案 |
|------|---------|---------|
| 打开页面显示"加载中..."且不消失 | 登录态恢复失败 | 刷新页面；若无效 → 清空 localStorage |
| 页面白屏但有 React 错误 | 某个组件渲染异常 | v2.0 已加 ErrorBoundary，会显示错误页。刷新即可重试 |
| 端口 6776 不可用 | 被其他程序占用 | Vite 会自动选择下一个端口；查看终端输出确认实际端口 |
| `npm run build` 失败 | TS 类型错误或未定义变量 | 先 `npm run check` 查看具体错误行 |
| Monaco Editor 无法加载 | 静态资源路径问题 | 检查 `dist/assets` 是否包含 Monaco worker 文件 |
| APK 安装后无法打开 | Android 版本/架构不匹配 | 使用兼容的模拟器/真机，或重新构建 |
| 学习进度消失（刷新后） | 浏览器隐私模式 / 已清空存储 | 非隐私模式；从"个人中心 → 导出数据"定期备份 |
| 社区发帖失败 | communityStore 异常 | 刷新页面重新初始化；检查 localStorage 配额 |
| 在线工具箱某个工具不工作 | 特定工具的输入格式不兼容 | 尝试其他输入；查看浏览器 Console |
| 移动端登录后数据未同步 | Capacitor Preferences 与 localStorage 隔离 | Android/iOS 有独立存储容器，属正常现象 |
