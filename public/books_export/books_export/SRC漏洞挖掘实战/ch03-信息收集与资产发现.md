# 第三章 信息收集与资产发现

## 3.1 信息收集概述

信息收集是漏洞挖掘的第一步，也是最关键的环节。俗话说"磨刀不误砍柴工"，充分的信息收集能让你事半功倍。一个优秀的信息收集可以帮你发现：
- 更多的攻击面
- 隐藏的资产
- 潜在的漏洞入口
- 系统架构信息
- 员工信息和组织架构
- 技术栈和开发框架
- 历史漏洞和安全事件

在SRC漏洞挖掘中，信息收集的质量直接决定了你能发现多少漏洞。很多新手一上来就拿着工具乱扫，结果往往一无所获。而经验丰富的白帽子，往往在信息收集阶段就能找到很多高价值的目标。

### 3.1.1 信息收集的分类

**被动信息收集**
- 不直接与目标交互
- 通过第三方获取信息
- 不易被目标发现
- 信息可能不够准确
- 不会留下攻击痕迹
- 可以大规模进行

被动信息收集是信息收集的第一步，也是最安全的一步。通过搜索引擎、公开数据、第三方服务等渠道，我们可以在不被目标察觉的情况下获取大量信息。

**主动信息收集**
- 直接与目标交互
- 主动探测目标资产
- 可能被目标发现
- 信息相对准确
- 可能触发安全告警
- 需要注意测试范围

主动信息收集是在被动收集的基础上，对发现的资产进行进一步探测。这时候需要注意遵守SRC平台的测试规则，避免超出测试范围。

### 3.1.2 信息收集的目标

**域名信息**
- 主域名
- 子域名
- 历史域名
- 关联域名
- 兄弟域名
- 备案域名

**IP信息**
- 服务器IP
- CDN节点IP
- 云服务IP
- 关联IP
- C段资产
- ASN信息

**端口与服务**
- 开放端口
- 运行服务
- 服务版本
- 中间件信息
- 数据库服务
- 远程管理服务

**Web信息**
- Web框架
- CMS系统
- 编程语言
- 数据库类型
- 前端框架
- 服务器软件

**敏感信息**
- 源代码泄露
- 配置文件泄露
- 员工信息
- 技术文档
- 账号密码
- API密钥

### 3.1.3 信息收集的完整流程

一个完整的信息收集流程应该遵循"由远及近、由表及里"的原则，逐步深入：

**第一阶段：外围信息收集（被动）**
1. 公司/组织背景调查
2. 域名Whois信息查询
3. 备案信息查询
4. 子域名被动收集
5. 历史DNS记录查询
6. 证书透明度日志查询
7. 搜索引擎信息收集
8. GitHub/GitLab代码泄露搜索
9. 员工信息和邮箱收集
10. 网盘和文档搜索

**第二阶段：资产识别（半主动）**
1. 子域名主动探测
2. 子域名爆破
3. CDN识别与真实IP查找
4. IP反查域名
5. C段资产探测
6. 端口扫描
7. 服务识别
8. 存活URL探测

**第三阶段：深入收集（主动）**
1. Web指纹识别
2. CMS识别
3. 技术栈识别
4. WAF识别
5. 目录扫描
6. 敏感文件探测
7. 参数发现
8. 接口分析

**第四阶段：资产整理与优先级排序**
1. 资产去重和分类
2. 资产价值评估
3. 测试优先级排序
4. 制定测试计划

## 3.2 域名信息收集

### 3.2.1 Whois查询

Whois可以查询域名的注册信息，包括注册人、注册时间、DNS服务器等。

**在线工具**
- https://whois.chinaz.com
- https://whois.aliyun.com
- https://www.whois.com
- https://whois.icann.org
- https://www.whoisxmlapi.com

**命令行查询**
```bash
whois example.com
```

**关键信息**
- 注册人姓名/公司
- 注册邮箱
- 注册电话
- DNS服务器
- 注册时间/过期时间
- 域名状态
- 注册商信息

**反查技巧**
通过注册人信息反查其他域名：
```bash
# 通过邮箱反查
whois -h whois.example.com "email@example.com"

# 通过注册人反查
whois -h whois.example.com "Company Name"
```

**高级Whois查询**
```bash
# 查询详细信息
whois -H example.com

# 指定Whois服务器
whois -h whois.verisign-grs.com example.com

# 批量查询
cat domains.txt | while read domain; do
    whois $domain >> whois_results.txt
done
```

**国内备案信息查询**
- 工信部备案查询：https://beian.miit.gov.cn
- 天眼查：https://www.tianyancha.com
- 企查查：https://www.qcc.com
- 爱站网备案查询：https://beian.aizhan.com

备案信息可以帮助我们找到目标公司的所有备案域名，这是发现关联域名的重要途径。

### 3.2.2 子域名收集（10+种方法详解）

子域名收集是信息收集的核心环节，可以发现大量隐藏资产。下面介绍15种常用的子域名收集方法，包括被动和主动方式。

#### 方法一：搜索引擎发现（被动）

利用搜索引擎的索引来发现子域名，这是最基础也是最有效的被动收集方法。

```
# Google搜索
site:example.com
site:*.example.com

# 百度搜索
site:example.com

# Bing搜索
site:example.com

# 多个搜索引擎组合使用效果更好
```

**高级搜索技巧：**
```
# 排除特定子域名
site:example.com -www

# 查找特定页面的子域名
site:example.com intitle:"index of"
site:example.com inurl:admin
site:example.com inurl:login
```

#### 方法二：证书透明度日志（被动）

证书透明度（Certificate Transparency，CT）日志记录了所有SSL证书的颁发情况，可以从中提取域名信息。

**在线查询**
- https://crt.sh
- https://censys.io
- https://www.certificate-transparency.org
- https://sslmate.com/certspotter/
- https://developers.facebook.com/tools/ct/

**crt.sh查询**
```bash
# 查询域名证书
curl -s "https://crt.sh/?q=%25.example.com&output=json" | jq -r '.[].name_value' | sort -u

# 查询通配符证书
curl -s "https://crt.sh/?q=%.example.com&output=json" | jq -r '.[].name_value' | sort -u
```

**Censys查询**
```bash
# 安装censys
pip install censys

# 配置API
censys config

# 搜索证书中的子域名
censys search "parsed.names: example.com" --fields parsed.names
```

#### 方法三：DNS区域传输（被动/主动）

DNS区域传输是指从主DNS服务器复制整个区域文件到从DNS服务器。如果配置不当，攻击者可以获取整个区域的所有域名记录。

```bash
# 区域传输尝试
dig axfr @ns1.example.com example.com
dig axfr @ns2.example.com example.com

# 使用nslookup
nslookup -type=any example.com
server ns1.example.com
ls -d example.com

# 使用host命令
host -l example.com ns1.example.com
```

**自动化工具：**
```bash
# dnsrecon
dnsrecon -d example.com -t axfr

# fierce
fierce -dns example.com
```

#### 方法四：DNS记录枚举（被动）

查询各种类型的DNS记录，从中发现子域名。

```bash
# 查询所有记录类型
dig example.com ANY
nslookup -type=any example.com

# 查询特定记录类型
dig example.com A
dig example.com AAAA
dig example.com MX
dig example.com NS
dig example.com TXT
dig example.com SRV
dig example.com CNAME
dig example.com SOA

# 批量查询
for type in A AAAA MX NS TXT SRV CNAME; do
    echo "=== $type ==="
    dig example.com $type +short
done
```

#### 方法五：子域名爆破（主动）

通过字典暴力猜解子域名，这是最常用的主动收集方法。

**Sublist3r**

**OneForAll（推荐）**
```bash

cd OneForAll

# 批量扫描

```

**Subfinder**

#### 方法六：JS文件分析（被动/主动）

#### 方法十：子域名关联发现（被动）


### 3.2.3 子域名收集的最佳实践

**多工具组合使用**
没有任何一个工具能覆盖所有子域名，组合使用多个工具效果最好：

```bash
#!/bin/bash
DOMAIN="example.com"
OUTPUT="subdomains.txt"

# 清空输出文件
> $OUTPUT

# 1. Subfinder
echo "[+] 运行 Subfinder..."
subfinder -d $DOMAIN -silent >> $OUTPUT

# 2. Amass 被动模式
echo "[+] 运行 Amass 被动模式..."
amass enum -passive -d $DOMAIN -silent >> $OUTPUT

# 3. crt.sh
echo "[+] 查询 crt.sh..."
curl -s "https://crt.sh/?q=%25.$DOMAIN&output=json" | jq -r '.[].name_value' 2>/dev/null | grep -oP '[a-zA-Z0-9.-]+\.'"$DOMAIN" >> $OUTPUT

# 4. 去重
echo "[+] 去重..."
sort -u $OUTPUT -o $OUTPUT

echo "[*] 收集完成，共 $(wc -l < $OUTPUT) 个子域名"
```

**存活验证**
收集到的子域名不一定都存活，需要进行存活验证：

```bash
# httpx
cat subdomains.txt | httpx -silent -status-code -title -tech-detect -o alive.txt

# httprobe
cat subdomains.txt | httprobe -p http:80 -p https:443 > alive.txt
```

## 3.3 IP信息收集

### 3.3.1 IP地址查询

**获取域名IP**
```bash
# 使用ping
ping example.com

# 使用dig
dig example.com +short

# 使用nslookup
nslookup example.com

# 使用host
host example.com

# 使用curl
curl -s https://api.ipify.org?format=json
```

**批量获取子域名IP**
```bash
# 从子域名列表获取IP
cat subdomains.txt | while read sub; do
    ip=$(dig +short $sub | head -1)
    if [ -n "$ip" ]; then
        echo "$sub: $ip"
    fi
done

# 使用dig批量
dig -f subdomains.txt +short

# 使用httpx获取IP
httpx -l subdomains.txt -ip -silent
```

### 3.3.3 C段扫描

C段扫描可以发现同一网段的其他服务器。

**工具推荐**

**Nmap**
```bash
# 扫描C段存活主机
nmap -sn 192.168.1.0/24

# 扫描C段开放端口
nmap -sS -p 80,443,8080 192.168.1.0/24

# 快速扫描常用端口
nmap -T4 -F 192.168.1.0/24

# 扫描C段Web服务
nmap -p 80,443,8080,8443 --open 192.168.1.0/24
```

**Masscan**
```bash
# 快速扫描
masscan -p 80,443,8080 192.168.1.0/24 --rate=1000

# 全端口扫描
masscan -p1-65535 192.168.1.0/24 --rate=10000

# 输出结果
masscan -p 80,443 192.168.1.0/24 -oL result.txt
```

**Naabu（快速端口扫描）**
```bash
# 安装
go install -v github.com/projectdiscovery/naabu/v2/cmd/naabu@latest

# 基本使用
naabu -host 192.168.1.1

# 批量扫描
naabu -l ips.txt -p 80,443,8080

# 快速模式
naabu -l ips.txt -rate 1000
```

**注意事项**
- C段扫描可能触发IDS/IPS告警
- 部分SRC平台禁止C段扫描
- 扫描前确认是否在测试范围内
- 控制扫描速率，避免影响目标业务
- 优先扫描目标明确的资产

### 3.3.4 CDN识别与绕过

**CDN识别**

```bash
# 多地ping判断
ping example.com

# 使用在线工具
# https://www.cdnplanet.com/tools/cdnfinder/
# https://whatismyip.com.cn/cdn/

# 查看响应头
curl -I https://example.com
# 查找 X-Cache、Via、X-CDN 等头

# 查看DNS解析
nslookup example.com
# 如果返回多个IP且分布在不同地区，很可能使用了CDN

# 使用wafw00f同时识别WAF和CDN
wafw00f https://example.com
```

**常见CDN厂商特征：**
- 阿里云CDN：X-Cache: HIT/MISS
- 腾讯云CDN：X-Cache-Lookup
- Cloudflare：CF-Cache-Status、server: cloudflare
- 网宿科技：X-Cache、X-Via
- 七牛云：X-Log
- 又拍云：X-Request-Id

**CDN绕过技巧**

1. **子域名查询**
   - CDN通常只保护主域名
   - 子域名可能未使用CDN
   - 特别是内部系统、测试系统
   - 常见的未加CDN的子域名：api、test、dev、staging、admin、mail、ftp等

2. **历史DNS记录**
   - 查询历史DNS解析记录
   - 可能找到真实IP
   - 目标迁移CDN之前的IP可能仍然有效
   - 查询平台：SecurityTrails、DNSDB、ViewDNS

3. **SSL证书查询**
   - 查询证书关联的IP
   - https://censys.io
   - https://crt.sh
   - 证书透明度日志中可能包含真实IP信息

4. **邮件头分析**
   - 注册/登录获取邮件
   - 查看邮件头中的服务器IP
   - 邮件服务器通常不经过CDN
   - 查看Received头信息

5. **子域名爆破**
   - 爆破子域名
   - 寻找未使用CDN的子域名
   - 使用大字典进行爆破
   - 关注冷门子域名

6. **网站漏洞反查**
   - SSRF漏洞可以探测内网和真实IP
   - 命令执行漏洞可以查看网络配置
   - XSS漏洞可以获取客户端信息

7. **多地解析对比**
   - 使用不同地区的DNS服务器解析
   - 如果结果相同，可能没有CDN
   - 如果结果不同，使用了CDN

8. **favicon哈希搜索**
   - 计算网站favicon的hash值
   - 在Shodan/Fofa中搜索
   - 找到使用相同favicon的真实IP

```python
# 计算favicon hash
import mmh3
import requests
import codecs

response = requests.get('https://example.com/favicon.ico')
favicon = codecs.encode(response.content, 'base64')
hash = mmh3.hash(favicon)
print(f"favicon hash: {hash}")

# 然后在Shodan中搜索：http.favicon.hash:{hash}
```

9. **域名解析历史**
   - 查询目标域名的IP变更历史
   - 找到最早使用的IP
   - 很多小网站最初没有CDN

10. **第三方服务**
    - 网站使用的第三方服务可能暴露真实IP
    - 例如：统计代码、客服系统、CDN回源地址等

### 3.5.4 目录扫描的技巧和注意事项

**1. 字典选择**
- 根据网站语言选择字典（PHP、Java、Python等）
- 根据CMS类型选择字典（WordPress、Drupal、Joomla等）
- 自定义行业专用字典
- 动态生成字典（从网站内容提取）

**2. 扫描策略**
- 先用小字典快速扫描，再用大字典深度扫描
- 先扫描常见文件，再扫描目录
- 递归扫描要控制深度
- 关注403状态码的目录

**3. 绕过WAF**
- 降低扫描速率
- 使用代理池
- 随机User-Agent
- 添加随机延迟
- 使用HTTP/2

**4. 结果分析**
- 200：文件存在，重点关注
- 301/302：重定向，可能有价值
- 403：禁止访问，目录存在
- 404：不存在
- 500：服务器错误，可能有漏洞
- 大小异常的响应值得关注

```python
        timestamp = time.strftime("%H:%M:%S", time.localtime())
        print(f"{timestamp} {level} {message}")
    
    def save_to_file(self, filename, data):
        filepath = os.path.join(self.output_dir, filename)
        with open(filepath, 'w', encoding='utf-8') as f:
            if isinstance(data, list):
                for item in data:
                    f.write(str(item) + '\n')
            elif isinstance(data, set):
                for item in sorted(data):
                    f.write(str(item) + '\n')
            else:
                f.write(str(data))
        self.log(f"结果已保存到 {filepath}")
    
    def subdomain_passive_crt(self):
        """通过crt.sh被动收集子域名"""
        self.log("通过 crt.sh 收集子域名...")
        try:
            url = f"https://crt.sh/?q=%25.{self.domain}&output=json"
            response = requests.get(url, timeout=30, verify=False)
            if response.status_code == 200:
                data = response.json()
                for entry in data:
                    name = entry.get('name_value', '')
                    if '\n' in name:
                        names = name.split('\n')
                        for n in names:
                            if n.endswith(self.domain) and '*' not in n:
                                self.subdomains.add(n.strip().lower())
                    else:
                        if name.endswith(self.domain) and '*' not in name:
                            self.subdomains.add(name.strip().lower())
                self.log(f"crt.sh 找到 {len(self.subdomains)} 个子域名")
        except Exception as e:
            self.log(f"crt.sh 查询失败: {e}", "[-]")
    
    def subdomain_passive_virustotal(self):
        """通过VirusTotal被动收集子域名"""
        self.log("通过 VirusTotal 收集子域名...")
        try:
            url = f"https://www.virustotal.com/ui/domains/{self.domain}/subdomains?limit=40"
            headers = {
                'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'
            }
            response = requests.get(url, headers=headers, timeout=30, verify=False)
            if response.status_code == 200:
                data = response.json()
                for item in data.get('data', []):
                    subdomain = item.get('id', '')
                    if subdomain.endswith(self.domain):
                        self.subdomains.add(subdomain.lower())
        except Exception as e:
            self.log(f"VirusTotal 查询失败: {e}", "[-]")
    
    def subdomain_bruteforce(self, wordlist=None):
        """子域名爆破"""
        self.log("开始子域名爆破...")
        
        if not wordlist:
            wordlist = [
                'www', 'mail', 'ftp', 'localhost', 'test', 'blog', 'dev', 'api',
                'admin', 'web', 'cdn', 'img', 'static', 'upload', 'bbs', 'shop',
                'user', 'demo', 'portal', 'mobile', 'm', 'app', 'vpn', 'mail',
                'smtp', 'pop', 'imap', 'pop3', 'ns1', 'ns2', 'dns', 'dns1',
                'backup', 'test', 'dev', 'staging', 'prod', 'stage', 'new',
                'old', 'beta', 'alpha', 'master', 'slave', 'mirror', 'proxy',
                'gateway', 'firewall', 'router', 'switch', 'server', 'monitor',
                'status', 'system', 'manage', 'manager', 'oa', 'crm', 'erp',
                'hr', 'finance', 'docs', 'doc', 'wiki', 'confluence', 'jira',
                'gitlab', 'github', 'jenkins', 'svn', 'bugzilla', 'redmine'
            ]
        else:
            with open(wordlist, 'r', encoding='utf-8', errors='ignore') as f:
                wordlist = [line.strip() for line in f if line.strip()]
        
        def check_subdomain(sub):
            try:
                subdomain = f"{sub}.{self.domain}"
                import socket
                socket.gethostbyname(subdomain)
                return subdomain
            except:
                return None
        
        count = 0
        with ThreadPoolExecutor(max_workers=self.threads) as executor:
            futures = {executor.submit(check_subdomain, sub): sub for sub in wordlist}
            for future in as_completed(futures):
                result = future.result()
                if result:
                    self.subdomains.add(result.lower())
                    count += 1
        
        self.log(f"子域名爆破完成，新增 {count} 个")
    
    def check_alive(self):
        """验证子域名存活状态"""
        self.log("验证子域名存活状态...")
        
        urls = []
        for subdomain in self.subdomains:
            urls.append(f"http://{subdomain}")
            urls.append(f"https://{subdomain}")
        
        def check_url(url):
            try:
                response = requests.get(url, timeout=5, verify=False, allow_redirects=True,
                    headers={'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'})
                title = ''
                match = re.search(r'<title>(.*?)</title>', response.text, re.IGNORECASE | re.DOTALL)
                if match:
                    title = match.group(1).strip()[:100]
                return {
                    'url': response.url,
                    'status_code': response.status_code,
                    'title': title,
                    'length': len(response.text)
                }
            except:
                return None
        
        results = []
        with ThreadPoolExecutor(max_workers=self.threads) as executor:
            futures = {executor.submit(check_url, url): url for url in urls}
            for future in as_completed(futures):
                result = future.result()
                if result:
                    results.append(result)
                    self.alive_urls.append(result['url'])
                    self.log(f"{result['status_code']} {result['url']} {result['title']}", "[+]")
        
        self.log(f"存活URL数量: {len(results)}")
        self.save_to_file("alive_urls.txt", [f"{r['status_code']}\t{r['url']}\t{r['title']}" for r in results])
        return results
    
    def scan_ports(self, target_ips=None):
        """端口扫描"""
        self.log("开始端口扫描...")
        if not target_ips:
            target_ips = list(self.subdomains)[:10]  # 只扫前10个，避免太慢
        
        common_ports = [21, 22, 23, 25, 53, 80, 110, 143, 443, 445, 
                        1433, 1521, 3306, 3389, 5432, 6379, 7001, 
                        8000, 8080, 8443, 9090, 27017]
        
        def scan_port(target, port):
            import socket
            try:
                sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
                sock.settimeout(1)
                result = sock.connect_ex((target, port))
                sock.close()
                if result == 0:
                    return (target, port)
            except:
                pass
            return None
        
        open_ports = []
        with ThreadPoolExecutor(max_workers=self.threads) as executor:
            futures = []
            for target in target_ips:
                try:
                    import socket
                    ip = socket.gethostbyname(target)
                    for port in common_ports:
                        futures.append(executor.submit(scan_port, ip, port))
                except:
                    pass
            
            for future in as_completed(futures):
                result = future.result()
                if result:
                    open_ports.append(result)
                    self.log(f"{result[0]}:{result[1]} 开放", "[+]")
        
        self.save_to_file("open_ports.txt", [f"{r[0]}:{r[1]}" for r in open_ports])
        return open_ports
    
    def dir_scan(self, url):
        """目录扫描"""
        self.log(f"开始目录扫描: {url}")
        
        common_dirs = [
            'admin', 'administrator', 'backup', 'config', 'data', 'db',
            'debug', 'docs', 'files', 'images', 'include', 'install',
            'logs', 'manager', 'old', 'private', 'public', 'secret',
            'test', 'tmp', 'upload', 'user', 'api', 'app', 'auth',
            'cache', 'cgi-bin', 'console', 'cms', 'dashboard', 'download',
            'forum', 'login', 'member', 'portal', 'register', 'static',
            'sys', 'system', 'template', 'uploadfile', 'web', 'wiki'
        ]
        
        common_files = [
            'index.php', 'index.html', 'index.asp', 'index.aspx', 'index.jsp',
            'admin.php', 'login.php', 'config.php', 'phpinfo.php', 'info.php',
            'robots.txt', 'sitemap.xml', '.git/HEAD', '.env', 'web.config',
            'package.json', 'composer.json'
        ]
        
        paths = common_dirs + common_files
        
        def check_path(path):
            try:
                full_url = f"{url.rstrip('/')}/{path}"
                response = requests.get(full_url, timeout=3, verify=False,
                    headers={'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'})
                if response.status_code not in [404]:
                    return (full_url, response.status_code, len(response.text))
            except:
                pass
            return None
        
        results = []
        with ThreadPoolExecutor(max_workers=self.threads) as executor:
            futures = {executor.submit(check_path, path): path for path in paths}
            for future in as_completed(futures):
                result = future.result()
                if result:
                    results.append(result)
                    self.log(f"{result[1]} {result[0]} (len:{result[2]})", "[+]")
        
        output_file = f"dirscan_{url.replace('://', '_').replace('/', '_')}.txt"
        self.save_to_file(output_file, [f"{r[1]}\t{r[0]}\t{r[2]}" for r in results])
        return results
    
    def check_sensitive_files(self, url):
        """检测敏感文件"""
        self.log(f"检测敏感文件: {url}")
        
        sensitive_files = [
            '.git/HEAD', '.git/config', '.svn/entries', '.svn/wc.db',
            '.DS_Store', '.env', '.env.local', '.env.production',
            'web.config', 'config.php', 'config.inc.php',
            'wp-config.php', 'application.yml', 'application.properties',
            'database.yml', 'settings.py', 'package.json',

            'backup.zip', 'www.zip', 'web.zip', 'backup.sql',
            'robots.txt', 'sitemap.xml', 'phpinfo.php', 'info.php'
        ]
        
        def check_file(path):
            try:
                full_url = f"{url.rstrip('/')}/{path}"
                response = requests.get(full_url, timeout=5, verify=False,
                    headers={'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'})
                if response.status_code == 200:
                    return (full_url, response.status_code)
            except:
                pass
            return None
        
        results = []
        with ThreadPoolExecutor(max_workers=self.threads) as executor:
            futures = {executor.submit(check_file, path): path for path in sensitive_files}
            for future in as_completed(futures):
                result = future.result()
                if result:
                    results.append(result)
                    self.log(f"发现敏感文件: {result[0]}", "[!]")
        
        output_file = f"sensitive_{url.replace('://', '_').replace('/', '_')}.txt"
        self.save_to_file(output_file, [f"{r[0]}" for r in results])
        return results
    
    def run_full(self):
        """运行完整信息收集流程"""
        self.log(f"开始对 {self.domain} 进行完整信息收集")
        self.log("=" * 50)
        
        # 1. 被动子域名收集
        self.log("第一阶段：被动子域名收集")
        self.subdomain_passive_crt()
        self.subdomain_passive_virustotal()
        self.log(f"被动收集到 {len(self.subdomains)} 个子域名")
        self.log("")
        
        # 2. 主动子域名爆破
        self.log("第二阶段：主动子域名爆破")
        self.subdomain_bruteforce()
        self.log(f"共收集到 {len(self.subdomains)} 个子域名")
        self.save_to_file("subdomains.txt", self.subdomains)
        self.log("")
        
        # 3. 存活验证
        self.log("第三阶段：存活验证")
        alive_results = self.check_alive()
        self.log("")
        
        # 4. 敏感文件检测
        self.log("第四阶段：敏感文件检测")
        for url in self.alive_urls[:10]:  # 只扫前10个
            self.check_sensitive_files(url)
        self.log("")
        
        # 5. 目录扫描（对主要站点）
        self.log("第五阶段：目录扫描")
        for url in self.alive_urls[:3]:  # 只扫前3个
            self.dir_scan(url)
        self.log("")
        
        self.log("=" * 50)
        self.log("信息收集完成!")
        self.log(f"子域名总数: {len(self.subdomains)}")
        self.log(f"存活URL数: {len(self.alive_urls)}")
        self.log(f"结果保存在: {self.output_dir}")

def main():
    parser = argparse.ArgumentParser(description='信息收集自动化工具')
```

## 3.9 资产整理与优先级判断

### 3.9.1 资产整理方法

信息收集会产生大量数据，如果不加以整理，就会陷入数据海洋，无法有效利用。

**资产分类方法：**

1. **按资产类型分类**
   - Web应用
   - 移动应用
   - API接口
   - 中间件
   - 数据库
   - 网络设备
   - 云服务

2. **按重要性分类**
   - 核心系统（主站、交易系统、用户中心）
   - 重要系统（管理后台、内部系统）
   - 一般系统（宣传页、博客、论坛）
   - 边缘系统（测试环境、旧系统）

3. **按技术栈分类**
   - Java系统
   - PHP系统
   - Python系统
   - .NET系统
   - 前端静态页面

**资产整理工具：**

1. **表格整理法**
   | 资产 | 类型 | 技术栈 | 重要性 | 状态 | 备注 |
   |------|------|--------|--------|------|------|
   | www.example.com | Web | PHP+MySQL | 高 | 正常 | 主站 |
   | admin.example.com | Web | Java+Oracle | 高 | 正常 | 管理后台 |
   | api.example.com | API | Node.js | 中 | 正常 | API接口 |
   | test.example.com | Web | PHP+MySQL | 低 | 正常 | 测试环境 |

2. **思维导图法**
   - 使用XMind、MindMaster等工具
   - 以主域名为中心，展开子域名
   - 每个子域名记录技术栈、端口、漏洞等

3. **资产数据库**
   - 使用ARL、水泽等工具的资产管理功能
   - 自建资产管理系统
   - 使用Confluence、Notion等文档工具

**资产去重：**
- 相同IP的资产去重
- 相同站点（不同域名指向同一系统）去重
- CDN节点去重
- 重复页面去重

### 3.9.2 优先级判断

面对大量资产，我们需要判断哪些资产最有价值，优先进行测试。

**资产价值评估维度：**

1. **业务重要性**
   - 核心业务系统 > 一般业务系统 > 边缘系统
   - 涉及资金交易的系统优先级最高
   - 涉及用户数据的系统优先级高
   - 管理后台优先级高

2. **技术栈风险**
   - 使用已知漏洞的框架（如Struts2、WebLogic、ThinkPHP等）
   - 旧版本软件
   - 使用不常见的技术（可能测试不充分）
   - 开源CMS（插件漏洞多）

3. **暴露程度**
   - 暴露在公网的系统 > 内网系统
   - 功能多的系统 > 简单页面
   - 有用户交互的系统 > 静态页面
   - 有输入参数的系统 > 无参数页面

4. **安全防护**
   - 没有WAF的系统优先级高
   - 弱口令风险的系统（FTP、SSH、数据库）
   - 未授权访问的服务（Redis、MongoDB、Elasticsearch等）
   - 有明显错误信息的系统

**优先级排序方法：**

**高优先级资产（优先测试）：**
1. 管理后台、OA、CRM等内部系统
2. API接口服务
3. 测试/开发/预发布环境
4. 使用已知漏洞框架的系统
5. 有敏感文件泄露的系统
6. 数据库直接暴露在公网
7. Redis、MongoDB等未授权访问
8. 旧系统和遗留系统
9. 子域名爆破发现的冷门系统
10. 有错误信息泄露的系统

**中优先级资产（其次测试）：**
1. 主站和主要业务系统
2. 用户中心、个人中心
3. 论坛、社区等互动系统
4. 电商、支付相关系统
5. 移动端API

**低优先级资产（有时间再测）：**
1. 纯静态宣传页面
2. 只有展示功能的页面
3. 第三方系统（非目标自有）
4. 明确说明不接收测试的系统
5. 防护非常严格的系统

**快速判断技巧：**
1. 先看403页面，说明目录存在
2. 先看有参数的页面，可能有注入
3. 先看登录注册页面，可能有逻辑漏洞
4. 先看上传、下载功能，可能有文件漏洞
5. 先看搜索、查询功能，可能有注入和XSS

### 3.10.1 案例一：电商网站信息收集

**目标：** 某中型电商网站 example-shop.com

**第一步：外围信息收集**

1. **公司背景调查**

   - DNS服务器：dns1.example-shop.com, dns2.example-shop.com
   - 注册时间：2018年

3. **备案信息查询**
   - 备案号：京ICP备XXXXXXXX号

## 3.11 信息收集的注意事项

### 3.11.1 避免触发WAF和安全告警

**1. 控制扫描速率**
- 不要使用过高的线程数
- 添加随机延迟
- 分时段进行扫描
- 先快后慢，先广后深

**2. 伪装请求**
- 使用真实的User-Agent
- 随机User-Agent
- 添加Referer头
- 模拟正常用户访问路径

**3. 使用代理**
- 使用HTTP代理池
- 使用Tor网络
- 使用VPN轮换IP
- 使用云函数/Serverless代理

**4. 绕过WAF的技巧**
- 变换请求方式（GET/POST/HEAD/OPTIONS）
- URL编码/Unicode编码
- 大小写混合
- 添加垃圾参数
- 分块传输编码
- HTTP走私

### 3.11.2 隐藏自己的踪迹

**1. 网络层隐藏**
- 使用代理/VPN/Tor
- 不使用真实IP
- 轮换出口IP

**2. 应用层隐藏**
- 随机User-Agent
- 不要使用明显的扫描器特征
- 修改请求头特征
- 模拟正常用户行为

**3. 账号层隐藏**
- 不要使用真实账号注册
- 使用临时邮箱
- 使用虚拟手机号
- 不要在目标网站留下真实信息

### 3.11.3 遵守SRC规则

**1. 确认测试范围**
- 仔细阅读SRC的测试规则
- 确认测试范围
- 不在范围外的系统上测试
- 有疑问及时联系厂商

**2. 禁止的操作**
- 禁止DDoS攻击
- 禁止删除/修改数据
- 禁止获取大量用户数据
- 禁止横向渗透
- 禁止提权操作
- 禁止内网探测（除非授权）

**3. 漏洞报告要求**
- 提供详细的复现步骤
- 提供漏洞证明截图
- 说明漏洞危害
- 给出修复建议
- 不泄露漏洞细节

### 3.11.4 信息收集的误区

**误区一：信息收集越多越好**
- 质量比数量重要
- 重点关注高价值资产
- 不要陷入数据海洋

**误区二：只收集不整理**
- 收集的数据要及时整理
- 建立资产管理体系
- 定期更新资产信息

**误区三：被动收集就够了**
- 被动收集只能发现已知资产
- 主动探测能发现更多隐藏资产
- 两者结合效果最好

**误区四：信息收集只做一次**
- 资产是动态变化的
- 定期重新收集
- 关注目标的新功能和新系统

**误区五：信息收集阶段不会触发告警**
- 现在的WAF和IDS很智能
- 扫描行为很容易被识别
- 即使被动收集也可能留下痕迹
