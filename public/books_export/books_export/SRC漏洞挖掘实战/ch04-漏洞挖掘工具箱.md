# 第四章 漏洞挖掘工具箱

## 4.1 Burp Suite详解

Burp Suite是Web安全测试必备工具，功能强大且灵活。本章详细介绍Burp Suite的各项功能、高级技巧和实战应用。

### 4.1.1 Burp Suite安装与配置

**版本选择**
- Community（免费版）：基础功能，适合学习
- Professional（专业版）：完整功能，推荐使用
- Enterprise（企业版）：团队协作，自动化扫描

**安装步骤**
1. 下载Burp Suite安装包
2. 安装Java运行环境（JRE 8+）
3. 运行安装程序
4. 激活许可证（专业版）

**浏览器代理配置**

Firefox配置：
1. 打开设置 → 网络设置
2. 手动代理配置：127.0.0.1:8080
3. 勾选"也将此代理用于HTTPS"

Chrome配置：
1. 安装SwitchyOmega插件
2. 新建情景模式：Burp
3. 代理服务器：127.0.0.1:8080
4. 代理协议：HTTP

**证书安装**
1. 访问 http://burp
2. 下载CA证书
3. 浏览器导入证书
4. 信任证书颁发机构

**初始配置优化**
```
1. 修改默认端口：Proxy → Options → Proxy Listeners
2. 设置拦截规则：Proxy → Options → Intercept Client Requests
3. 配置SSL：Project options → SSL
4. 设置线程数：Project options → Connections
5. 配置自动保存：Project options → Misc
```

### 4.1.6 Intruder暴力破解

**攻击类型**
- Sniper：单个位置，逐个测试
- Battering ram：多个位置，相同payload
- Pitchfork：多个位置，一一对应
- Cluster bomb：多个位置，排列组合

**使用步骤**
1. 拦截请求 → 发送到Intruder
2. 标记攻击位置（§符号）
3. 选择攻击类型
4. 设置Payload
5. 开始攻击
6. 分析结果

**Payload类型**
- Simple list：简单列表
- Runtime file：从文件读取
- Numbers：数字序列
- Dates：日期序列
- Brute forcer：暴力破解
- Custom iterator：自定义迭代器
- Character substitution：字符替换
- Case modification：大小写变换
- Recursive grep：递归搜索
- Copy other payload：复制其他payload

**实战案例：密码爆破**
```
# 请求示例
POST /login HTTP/1.1
Host: example.com
Content-Type: application/x-www-form-urlencoded

username=admin&password=§password§

# Payload设置
- 类型：Simple list
- 内容：常用密码字典

# 结果分析
- 长度不同的响应可能是成功
- 状态码不同的响应需要关注
- 响应时间异常的需要关注
- 重定向到不同页面的可能是成功
```

**实战案例：目录扫描**
```
# 请求示例
GET /§path§ HTTP/1.1
Host: example.com

# Payload设置
- 类型：Simple list
- 内容：目录字典

# 结果分析
- 状态码200：目录存在
- 状态码403：目录存在但无权限
- 状态码404：目录不存在
- 状态码301/302：重定向
```

**实战案例：验证码绕过**
```
# 测试验证码是否可以重复使用
# 测试验证码是否可以为空
# 测试验证码是否可以预测
```

## 4.2 SQLMap详解

SQLMap是自动化SQL注入工具，支持多种数据库和注入类型。

### 4.2.1 SQLMap安装

```bash
# Kali Linux自带
sqlmap -h

# 其他系统安装
git clone --depth 1 https://github.com/sqlmapproject/sqlmap.git sqlmap-dev
cd sqlmap-dev
python sqlmap.py -h

# 或使用pip
pip install sqlmap
```

### 4.2.2 基本使用

**测试注入点**
```bash
# GET请求
sqlmap -u "http://example.com/page?id=1"

# POST请求
sqlmap -u "http://example.com/login" --data="username=admin&password=123"

# Cookie注入
sqlmap -u "http://example.com/page" --cookie="id=1"

# HTTP头注入
sqlmap -u "http://example.com/page" --headers="X-Forwarded-For: 1*"
sqlmap -u "http://example.com/page" -H "User-Agent: *"

# Referer注入
sqlmap -u "http://example.com/page" --referer="*"
```

**指定参数**
```bash
# 指定注入参数
sqlmap -u "http://example.com/page?id=1&name=test" -p id

# 排除参数
sqlmap -u "http://example.com/page?id=1&name=test" --skip=name

# 使用*标记注入点
sqlmap -u "http://example.com/page/1*/news.html"
```

**请求文件**
```bash
# 从Burp导出的请求文件
sqlmap -r request.txt

# 指定POST数据
sqlmap -r request.txt --data="param=value"
```

### 4.2.3 注入类型检测

```bash

sqlmap -u "http://example.com/page?id=1"

# B: Boolean-based blind
# E: Error-based
# U: Union query-based
# S: Stacked queries
# T: Time-based blind
# Q: Inline queries

# 只测试布尔盲注
sqlmap -u "http://example.com/page?id=1" --technique=B

# 跳过时间盲注（比较慢）

```


**方法9：使用--hex**

**方法10：使用--prefix和--suffix**

## 4.4 Nmap详解

Nmap是网络扫描和主机发现的利器。

### 4.4.1 基本扫描

```bash
# 扫描单个主机
nmap 192.168.1.1

# 扫描多个主机
nmap 192.168.1.1 192.168.1.2

# 扫描网段
nmap 192.168.1.0/24

# 扫描范围
nmap 192.168.1.1-100

# 从文件读取目标
nmap -iL targets.txt
```

### 4.4.2 端口扫描

```bash
# 指定端口
nmap -p 80,443,8080 192.168.1.1

# 端口范围
nmap -p 1-1000 192.168.1.1

# 所有端口
nmap -p- 192.168.1.1

# 常用端口
nmap -p 21,22,23,25,80,110,443,445,1433,3306,3389,5432,6379,8080 192.168.1.1

# 快速扫描（top 100）
nmap -F 192.168.1.1

# top N端口
nmap --top-ports 1000 192.168.1.1
```

### 4.4.3 扫描技术

```bash
# TCP连接扫描
nmap -sT 192.168.1.1

# SYN扫描（半开放扫描，需要root）
nmap -sS 192.168.1.1

# UDP扫描
nmap -sU 192.168.1.1

# FIN扫描
nmap -sF 192.168.1.1

# NULL扫描
nmap -sN 192.168.1.1

# Xmas扫描
nmap -sX 192.168.1.1

# ACK扫描
nmap -sA 192.168.1.1

# 窗口扫描
nmap -sW 192.168.1.1

# 空闲扫描
nmap -sI zombie_host 192.168.1.1
```

### 4.4.4 服务识别

```bash
# 服务版本探测
nmap -sV 192.168.1.1

# 详细版本探测
nmap -sV --version-intensity 5 192.168.1.1

# 操作系统探测
nmap -O 192.168.1.1

# 综合扫描
nmap -A 192.168.1.1

# 系统和服务版本
nmap -sV -O 192.168.1.1
```

### 4.4.5 脚本扫描

```bash
# 默认脚本
nmap -sC 192.168.1.1

# 指定脚本

# 漏洞扫描
nmap --script=vuln 192.168.1.1

nmap --script=http-enum 192.168.1.1

# 认证测试
nmap --script=auth 192.168.1.1

# 发现脚本类别
nmap --script=discovery 192.168.1.1

# 恶意软件检测
nmap --script=malware 192.168.1.1

# 脚本参数
nmap --script=http-enum --script-args http-enum.url-path=/admin 192.168.1.1
```

### 4.4.6 输出格式


### 4.4.7 时间与性能

```bash

nmap -T3 192.168.1.1  # 正常（默认）

```

### 4.4.8 防火墙绕过


### 4.6.1 用Python编写简单的Web漏洞扫描器

下面是一个用Python编写的简单Web漏洞扫描器示例，可以检测常见的Web漏洞：

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
简单Web漏洞扫描器
功能：SQL注入检测、XSS检测、敏感文件发现、目录扫描
"""

import sys
import requests
import argparse
import re
from urllib.parse import urljoin, urlparse
from concurrent.futures import ThreadPoolExecutor, as_completed
from bs4 import BeautifulSoup

requests.packages.urllib3.disable_warnings()

class WebScanner:
    def __init__(self, target, threads=10):
        self.target = target
        self.threads = threads
        self.vulnerabilities = []
        self.session = requests.Session()
        self.session.headers.update({
            'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'
        })
    
    def log(self, message, level="[*]"):
        print(f"{level} {message}")
    
    def get_params(self, url):
        """获取URL参数"""
        parsed = urlparse(url)
        params = {}
        if parsed.query:
            for param in parsed.query.split('&'):
                if '=' in param:
                    key, value = param.split('=', 1)
                    params[key] = value
        return params
    
    def test_sql_injection(self, url, param, value):
        """测试SQL注入"""
        payloads = [
            ("'", "SQL syntax error", "MySQL error"),
            ("1' AND '1'='1", "", "Boolean true"),
            ("1' AND '1'='2", "", "Boolean false"),
            ("1 AND SLEEP(3)", "time_delay", "Time-based blind"),
        ]
        
        base_url = url.split('?')[0]
        params = self.get_params(url)
        
        for payload, indicator, desc in payloads:
            test_params = params.copy()
            test_params[param] = value + payload
            
            try:
                import time
                start = time.time()
                response = self.session.get(base_url, params=test_params, timeout=10, verify=False)
                elapsed = time.time() - start
                
                if indicator == "time_delay":
                    if elapsed > 3:
                        vuln = {
                            'type': 'SQL Injection (Time-based)',
                            'url': url,
                            'parameter': param,
                            'payload': payload,
                            'evidence': f'Response time: {elapsed:.2f}s'
                        }
                        self.vulnerabilities.append(vuln)
                        self.log(f"发现SQL注入: {url} 参数: {param}", "[!]")
                        return True
                elif indicator and indicator.lower() in response.text.lower():
                    vuln = {
                        'type': 'SQL Injection (Error-based)',
                        'url': url,
                        'parameter': param,
                        'payload': payload,
                        'evidence': indicator
                    }
                    self.vulnerabilities.append(vuln)
                    self.log(f"发现SQL注入: {url} 参数: {param}", "[!]")
                    return True
            except:
                pass
        
        return False
    
    def test_xss(self, url, param, value):
        """测试XSS"""
        payloads = [
            '<script>alert(1)</script>',
            '<img src=x onerror=alert(1)>',
            '"><script>alert(1)</script>',
            '\'><script>alert(1)</script>',
            'javascript:alert(1)',
            '<svg/onload=alert(1)>',
        ]
        
        base_url = url.split('?')[0]
        params = self.get_params(url)
        
        for payload in payloads:
            test_params = params.copy()
            test_params[param] = payload
            
            try:
                response = self.session.get(base_url, params=test_params, timeout=10, verify=False)
                if payload in response.text:
                    vuln = {
                        'type': 'XSS (Reflected)',
                        'url': url,
                        'parameter': param,
                        'payload': payload,
                        'evidence': 'Payload reflected in response'
                    }
                    self.vulnerabilities.append(vuln)
                    self.log(f"发现XSS: {url} 参数: {param}", "[!]")
                    return True
            except:
                pass
        
        return False
    
    def scan_sensitive_files(self):
        """扫描敏感文件"""
        self.log("扫描敏感文件...")
        
        sensitive_files = [
            '.git/HEAD',
            '.git/config',
            '.env',
            '.env.local',
            'robots.txt',
            'sitemap.xml',
            'phpinfo.php',
            'info.php',
            'admin.php',
            'login.php',
            'config.php',
            'backup.zip',
            'www.zip',
            'web.config',
            '.DS_Store',
            '.svn/entries',
            'crossdomain.xml',
            'clientaccesspolicy.xml',
        ]
        
        def check_file(path):
            url = urljoin(self.target, path)
            try:
                response = self.session.get(url, timeout=5, verify=False)
                if response.status_code == 200:
                    # 判断不是404页面
                    if len(response.text) > 100 and '404' not in response.text[:200]:
                        vuln = {
                            'type': 'Sensitive File',
                            'url': url,
                            'parameter': '-',
                            'payload': path,
                            'evidence': f'Status: {response.status_code}, Size: {len(response.text)}'
                        }
                        self.vulnerabilities.append(vuln)
                        self.log(f"发现敏感文件: {url}", "[!]")
                        return True
            except:
                pass
            return False
        
        with ThreadPoolExecutor(max_workers=self.threads) as executor:
            futures = [executor.submit(check_file, f) for f in sensitive_files]
            for future in as_completed(futures):
                future.result()
    
    def scan_url_parameters(self, url):
        """扫描URL参数"""
        self.log(f"扫描URL参数: {url}")
        
        params = self.get_params(url)
        if not params:
            self.log("  没有发现URL参数")
            return
        
        for param, value in params.items():
            self.log(f"  测试参数: {param}")
            
            # 测试SQL注入
            self.test_sql_injection(url, param, value)
            
            # 测试XSS
            self.test_xss(url, param, value)
    
    def crawl_links(self, url):
        """爬取页面链接"""
        links = set()
        try:
            response = self.session.get(url, timeout=10, verify=False)
            soup = BeautifulSoup(response.text, 'html.parser')
            
            for link in soup.find_all('a'):
                href = link.get('href')
                if href:
                    full_url = urljoin(url, href)
                    if self.target in full_url:
                        links.add(full_url.split('#')[0])
            
            # 提取表单
            for form in soup.find_all('form'):
                action = form.get('action', '')
                method = form.get('method', 'get').lower()
                form_url = urljoin(url, action)
                
                inputs = []
                for inp in form.find_all(['input', 'textarea', 'select']):
                    name = inp.get('name')
                    if name:
                        inputs.append(name)
                
                if inputs:
                    param_str = '&'.join([f'{inp}=test' for inp in inputs])
                    full_url = f"{form_url}?{param_str}" if method == 'get' else form_url
                    if self.target in full_url:
                        links.add(full_url)
            
        except Exception as e:
            self.log(f"爬取失败: {e}", "[-]")
        
        return links
    
    def run(self):
        """运行完整扫描"""
        self.log(f"开始扫描目标: {self.target}")
        self.log("=" * 50)
        
        # 1. 敏感文件扫描
        self.log("[1/3] 敏感文件扫描")
        self.scan_sensitive_files()
        self.log("")
        
        # 2. 爬取链接
        self.log("[2/3] 爬取页面链接")
        links = self.crawl_links(self.target)
        self.log(f"发现 {len(links)} 个链接")
        
        # 3. 扫描每个链接的参数
        self.log("[3/3] 扫描URL参数漏洞")
        for link in list(links)[:20]:  # 限制数量，避免太慢
            self.scan_url_parameters(link)
        
        self.log("")
        self.log("=" * 50)
        self.log(f"扫描完成，共发现 {len(self.vulnerabilities)} 个漏洞")
        
        # 输出报告
        if self.vulnerabilities:
            self.log("\n漏洞详情:")
            for i, vuln in enumerate(self.vulnerabilities, 1):
                self.log(f"\n{i}. {vuln['type']}")
                self.log(f"   URL: {vuln['url']}")
                self.log(f"   参数: {vuln['parameter']}")
                self.log(f"   Payload: {vuln['payload']}")
                self.log(f"   证据: {vuln['evidence']}")
        
        return self.vulnerabilities

def main():
    parser = argparse.ArgumentParser(description='简单Web漏洞扫描器')
    parser.add_argument('-u', '--url', required=True, help='目标URL')
    parser.add_argument('-t', '--threads', type=int, default=10, help='线程数')
    parser.add_argument('--sql', action='store_true', help='只测试SQL注入')
    parser.add_argument('--xss', action='store_true', help='只测试XSS')
    parser.add_argument('--sensitive', action='store_true', help='只扫描敏感文件')
    
    args = parser.parse_args()
    
    scanner = WebScanner(args.url, args.threads)
    
    if args.sql or args.xss or args.sensitive:
        if args.sensitive:
            scanner.scan_sensitive_files()
        if args.sql or args.xss:
            scanner.scan_url_parameters(args.url)
    else:
        scanner.run()

if __name__ == '__main__':
    main()
```

**使用方法：**
```bash

# 只扫描敏感文件

# 只测试SQL注入
```

### 4.6.2 编写子域名爆破工具

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
简单子域名爆破工具
"""

import sys
import socket
import argparse
from concurrent.futures import ThreadPoolExecutor, as_completed

class SubdomainBrute:
    def __init__(self, domain, wordlist, threads=20):
        self.domain = domain
        self.wordlist = wordlist
        self.threads = threads
        self.results = []
    
    def load_wordlist(self):
        """加载字典"""
        with open(self.wordlist, 'r', encoding='utf-8', errors='ignore') as f:
            return [line.strip() for line in f if line.strip()]
    
    def check_subdomain(self, sub):
        """检查子域名是否存在"""
        try:
            subdomain = f"{sub}.{self.domain}"
            ip = socket.gethostbyname(subdomain)
            return (subdomain, ip)
        except:
            return None
    
    def run(self):
        """运行爆破"""
        print(f"[*] 开始爆破子域名: {self.domain}")
        print(f"[*] 字典文件: {self.wordlist}")
        print(f"[*] 线程数: {self.threads}")
        print()
        
        wordlist = self.load_wordlist()
        print(f"[*] 字典大小: {len(wordlist)}")
        print()
        
        count = 0
        with ThreadPoolExecutor(max_workers=self.threads) as executor:
            futures = {executor.submit(self.check_subdomain, sub): sub for sub in wordlist}
            for future in as_completed(futures):
                result = future.result()
                if result:
                    count += 1
                    self.results.append(result)
                    print(f"[+] {result[0]} -> {result[1]}")
        
        print()
        print(f"[*] 爆破完成，共发现 {count} 个子域名")
        return self.results

def main():
    parser = argparse.ArgumentParser(description='子域名爆破工具')
    parser.add_argument('-d', '--domain', required=True, help='目标域名')
    parser.add_argument('-w', '--wordlist', required=True, help='字典文件')
    parser.add_argument('-t', '--threads', type=int, default=20, help='线程数')
    parser.add_argument('-o', '--output', help='输出文件')
    
    args = parser.parse_args()
    
    brute = SubdomainBrute(args.domain, args.wordlist, args.threads)
    results = brute.run()
    
    if args.output and results:
        with open(args.output, 'w') as f:
            for sub, ip in results:
                f.write(f"{sub}\t{ip}\n")
        print(f"[*] 结果已保存到: {args.output}")

if __name__ == '__main__':
    main()
```

## 4.11 工具最佳配置方案

### 4.11.1 Burp Suite最佳配置

**内存配置：**
```bash
# 增加JVM内存（启动参数）
java -Xms512m -Xmx4g -jar burpsuite_pro.jar

# 4G内存推荐配置
java -Xmx4g -jar burpsuite_pro.jar
```

**代理配置：**
- 监听器：127.0.0.1:8080
- 拦截：默认关闭，需要时开启
- 响应拦截：默认关闭
- 不可见代理：按需开启

**扫描配置：**
- 主动扫描：中等速度
- 被动扫描：开启实时分析
- 线程数：10-20
- 插入点：URL参数、表单、Cookie、Header

**插件配置：**
- 必备插件：Autorize、Logger++、JS Link Finder、Hackvertor
- 按需安装：SQLiPy、Param Miner、Retire.js
- 不常用的插件禁用，提升性能

### 4.11.2 SQLMap最佳配置

**常用别名（写入.bashrc）：**
```bash
alias sqlmap='sqlmap --batch --random-agent --tamper=space2comment'
```

**配置文件 sqlmap.conf：**
```
batch = True
randomAgent = True
threads = 10
tamper = space2comment
delay = 0
timeout = 30
retries = 3
```

**性能优化：**
- 根据目标调整线程数
- 指定数据库类型（已知的情况下）
- 指定注入技术
- 使用--smart智能检测
- 合理使用--level和--risk

### 4.11.3 Nmap最佳配置

**常用扫描组合：**
```bash
# 快速扫描
nmap -T4 -F <target>

# 标准扫描
nmap -T4 -sV -O <target>

# 全面扫描
nmap -T4 -sV -sC -O -p- <target>

# 隐蔽扫描
nmap -T2 -f -D RND:5 <target>
```

**性能优化：**
- 根据网络调整时间模板
- 只扫描必要的端口
- 使用--open只显示开放端口
- 合理使用--min-rate/--max-rate
