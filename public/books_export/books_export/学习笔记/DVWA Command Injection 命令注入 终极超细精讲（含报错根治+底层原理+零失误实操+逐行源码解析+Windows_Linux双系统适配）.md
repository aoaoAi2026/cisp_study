# DVWA Command Injection 命令注入 终极超细精讲（含报错根治\+底层原理\+零失误实操\+逐行源码解析\+Windows/Linux双系统适配）

## 0\. 零基础彻底吃透：命令注入（Command Injection）核心本质（人话\+底层原理）

### 0\.1 官方名词通俗拆解

**Command Injection 全称：操作系统命令注入漏洞**

逐词通俗解读，摒弃晦涩术语，小白一看就懂：

Command（命令）：服务器系统自带的原生指令，比如Windows查IP、Linux查目录，是服务器底层执行的操作命令

Injection（注入）：利用网站输入框，**拼接、插入恶意系统命令**，欺骗服务器执行非授权操作

**终极一句话定义（面试/实操通用）：**命令注入是由于网站后端未对用户输入做任何过滤、转义和拦截，导致攻击者可以在正常输入内容后拼接系统恶意命令，让服务器直接解析执行，从而非法获取服务器信息、篡改文件、控制系统的高危漏洞，属于Web超级高危漏洞。

### 0\.2 黄金生活化类比（小白秒懂攻击逻辑）

场景对标：**网站输入框 = 传话机器人，服务器系统 = 执行工人**

正常业务：用户输入正常指令（查IP），机器人原封不动传话，工人正常执行查询IP操作，合规安全。

注入攻击：机器人**不会筛选、不会过滤、不会拦截话术**，攻击者在正常指令后面偷偷加一堆恶意指令，机器人全部传给工人，工人无脑全部执行。

核心特点：网站把「用户输入的所有内容」全部当成合法系统命令执行，不区分对错、不区分用途，直接交给服务器底层运行，危害极大。

### 0\.3 命令注入核心分类（考试/实操必懂）

**1、直接命令注入（无过滤）**：输入内容直接拼接系统命令，无任何拦截，Low、Medium级别核心攻击方式

**2、过滤绕过命令注入**：网站屏蔽部分符号（如\&\&、\|），可通过替代符号、大小写、空格绕过，Medium级别专属考点

**3、完全过滤命令注入**：严格正则匹配，屏蔽所有拼接符号、恶意关键词，仅能通过特殊漏洞绕过，High级别防护

### 0\.4 命令注入成立三大必要条件（缺一不可）

1、**用户可控输入点**：存在输入框、URL参数等可自由输入、可控的传参位置

2、**后端直接调用系统命令**：代码将用户输入拼接进系统执行函数，不做隔离

3、**无过滤无转义**：未过滤命令拼接符号、未屏蔽恶意命令、未做参数净化

### 0\.5 新手必懂：命令注入为什么是高危漏洞？

不同于CSRF、暴力破解仅能操作网站账号，**命令注入可以直接控制服务器底层系统**。一旦攻击成功，攻击者可以查服务器IP、看文件目录、读写文件、植入后门、拿下服务器最高权限，是渗透测试中危害性顶尖的漏洞。

### 0\.6 【重点补强】命令注入六大核心拼接符号 零基础超细精讲（原理\+用法\+适用关卡\+踩坑全覆盖）

命令注入的核心本质：**利用特殊符号截断/拼接原有系统命令，让服务器执行多条指令**。DVWA三关所有出题逻辑、过滤规则、绕过思路，全部围绕这6个符号展开，零基础必须吃透每一个符号的执行逻辑、区别、适配场景。

#### 0\.6\.1 分号 ; （基础拼接符）

**核心作用**：**不管前一条命令成功还是失败，强制顺序执行后一条命令**，无逻辑判断，纯拼接换行执行。

**通俗理解**：相当于换行，第一条命令跑完，无条件接着跑第二条。

**标准用法**：127\.0\.0\.1; ipconfig

**执行结果**：先执行ping 127\.0\.0\.1，再执行ipconfig，两条命令全部执行。

**关卡适配**：仅**Low可用**，Medium、High直接被黑名单过滤拦截。

**高频踩坑**：Medium关卡用;直接失效，新手不要在中高级使用分号拼接。

#### 0\.6\.2 单与符 \& （并行执行符）

**核心作用**：**前后两条命令同时执行，互不影响**，前置命令报错不影响后置恶意命令执行。

**通俗理解**：两条命令打包一起跑，不管前面通不通，后面恶意命令必跑。

**标准用法**：127\.0\.0\.1 \& ipconfig

**执行结果**：ping命令、ipconfig命令全部执行，稳定出结果。

**关卡适配**：Low、Medium可用，**High被过滤完全失效**。

**绕过价值**：Medium过滤了\&\&、\|\|、;，但漏过了单\&，是中级核心绕过Payload之一。

#### 0\.6\.3 双与符 \&\& （条件执行符）

**核心作用**：**前置命令执行成功，才会执行后置恶意命令**；前置报错，后置直接不执行。

**通俗理解**：前面正常通，后面恶意命令才放行，属于“顺利才执行”。

**标准用法**：127\.0\.0\.1 \&\& ipconfig

**执行结果**：ping通 → 执行ipconfig；ping不通 → 无任何恶意结果。

**关卡适配**：仅Low可用，**Medium、High直接拦截**。

**新手缺点**：不稳定，前置网络波动会导致攻击失效，不推荐作为主力Payload。

#### 0\.6\.4 单管道符 \| （覆盖执行符｜最强万能符）

**核心作用**：**直接舍弃、屏蔽前置命令，只执行后置恶意命令**，优先级所有符号最高！

**通俗理解**：前面命令直接作废，强制只运行你写的恶意命令，百分百出结果。

**标准用法**：127\.0\.0\.1 \| ipconfig

**执行结果**：忽略ping检测，页面只返回ipconfig系统信息，成功率全网最高。

**关卡适配**：Low、Medium通杀，**High被过滤失效**。

**核心地位**：DVWA低中级**首选Payload**，稳定、不挑环境、不报错。

#### 0\.6\.5 双或符 \|\| （报错执行符）

**核心作用**：**前置命令执行失败，才执行后置恶意命令**；前置成功，后置直接不执行。

**通俗理解**：前面报错、不通，后面恶意命令才会触发。

**标准用法**：127\.0\.0\.2 \|\| ipconfig（故意填错误IP）

**执行结果**：ping错误 → 触发ipconfig；ping正确 → 无结果。

**关卡适配**：仅Low可用，中高级直接拦截。

**新手缺点**：需要前置命令报错才能触发，使用麻烦、极不稳定，不推荐使用。

#### 0\.6\.6 %0a （URL换行编码｜高级唯一绕过符）

**核心作用**：URL编码换行符，**强制截断原有命令、新开一行独立执行恶意命令**。

**底层原理（新手必懂）**：

DVWA高级黑名单只过滤「可见特殊符号」（\| \& ; \|\| \&\&），但是**不会解码过滤URL编码字符**，服务器接收参数后会自动解码%0a为换行符，直接截断命令实现注入。

**标准用法**：127\.0\.0\.1%0aipconfig

**执行逻辑拆解**：

原后端命令：ping 【用户输入】

传入Payload后：ping 127\.0\.0\.1（换行）ipconfig

最终效果：两条独立命令，恶意命令强制执行。

**关卡适配**：**High级别唯一可用Payload**，通杀高级所有过滤规则。

**硬性规范**：必须小写%0a，大写%0A部分环境无法解析，禁止加空格。

#### 0\.6\.7 六大符号终极选用口诀（新手永久记忆）

Low随便拼，单竖最稳定；

Medium禁双留单，竖线稳通关；

High符号全拉黑，编码换行破防御。

## 1\. DVWA\-Command Injection 模块通用底层逻辑（三关完全共用）

### 1\.1 模块核心功能

整个模块只有一个核心业务：**Ping网络连通性检测**。用户输入一个IP地址，网站调用服务器系统命令，执行ping检测，返回ping结果，用来判断目标IP是否通畅。

### 1\.2 正常手动操作完整流程

用户访问命令注入页面 → 输入合法IP地址（如127\.0\.0\.1） → 点击提交 → 网站后端拼接ping系统命令 → 服务器执行ping检测 → 返回执行结果。

### 1\.3 漏洞核心根源（三关通用底层问题）

DVWA命令注入模块的核心致命缺陷：**直接将用户输入的内容拼接到系统命令中执行，没有参数隔离**。

正常开发应该固定ping命令，仅接收用户输入的IP参数；而DVWA直接把用户输入当成完整命令的一部分，导致用户可以随意拼接恶意指令，篡改系统执行逻辑。

### 1\.4 新手高频报错\&失效通用根因（提前避坑）

1、符号输入错误：使用中文符号、全角符号，服务器无法识别，执行无结果

2、命令不兼容：Windows/Linux系统命令混用，导致命令执行报错

3、空格多余/缺失：拼接格式错误，导致命令拆分失效

4、安全等级未切换：关卡配置未保存，防护机制不匹配

5、被过滤符号命中：中级高级使用被屏蔽符号，直接拦截攻击

6、符号场景用错：高级使用\|、\&等常规符号，百分百拦截失效

7、%0a格式错误：大写编码、前后带空格，高级无法截断命令

### 1\.5 关键前置：Windows \& Linux 命令注入核心区别（必考，解决90%执行报错）

命令注入**最大新手误区**：通用拼接符号通用，但**系统指令不通用**，跨系统混用直接报错、无回显、执行失败。DVWA会自动识别服务器系统拼接对应ping命令，攻击者必须适配系统指令。

#### 1\.5\.1 Windows系统 DVWA环境特征

自动执行无次数限制ping、命令不区分大小写、常用查询指令：ipconfig、dir、whoami、systeminfo

#### 1\.5\.2 Linux系统 DVWA环境特征

自动执行 ping \-c 4 限制4次数据包、命令严格区分大小写、常用查询指令：ifconfig、ls、pwd、id、uname \-a

#### 1\.5\.3 双系统通用拼接规则（符号完全通用）

;  \&  \&\&  \|  \|\|  %0a **六大拼接符号在Windows、Linux完全通用，仅后置恶意命令需要区分系统**。

# 第一关：Command Injection Low 低级（无防护裸奔｜超细原理\+零报错实操\+逐行源码解析\+双系统Payload）

## 1\.1 Low级别源码防护原理（完全无防护）

Low级别源码**无任何过滤、无任何拦截、无字符检测**，直接将用户输入的所有内容拼接进system系统执行函数。

通俗理解：服务器完全“无脑执行”，用户输入什么，服务器就运行什么，不管是正常ping命令还是恶意系统指令，全部直接执行，属于完全裸奔的超高危漏洞。

## 1\.2 Low级别完整源码 \+ 逐行超细解析（必考）

```php
<?php
if( isset( $_POST[ 'Submit' ] ) ) {
    // Get input
    $target = $_POST[ 'ip' ];

    // Determine OS and execute the ping command.
    if (stristr(PHP_OS, 'WINNT')) {
        // Windows
        $cmd = shell_exec( 'ping  ' . $target );
    } else {
        // Linux
        $cmd = shell_exec( 'ping  -c 4 ' . $target );
    }

    // Feedback for the end user
    echo "<pre>{$cmd}</pre>";
}
?>
```

### 逐行白话拆解：

**第2行**：判断用户是否点击了提交按钮，触发后续命令执行逻辑。

**第4行【致命漏洞行】**：直接通过POST获取用户输入的ip参数，**无过滤、无转义、无校验**，用户输入完全可控。

**第7\-10行**：自动判断服务器系统，Windows直接拼接ping命令，Linux拼接带次数限制的ping命令。

**核心漏洞点**：代码固定ping命令前缀，**后半段完全拼接用户可控输入**，用户可以用; \| \&等符号截断原有ping命令，拼接任意恶意指令。

**第13行**：调用shell\_exec高危函数，将拼接后的完整命令提交给服务器系统执行，并返回执行结果。

**源码总结**：全程无任何安全防护，信任所有用户输入，是最标准、最原始的命令注入漏洞源码模型。

## 1\.3 Low漏洞深层原理

后端代码未对输入字符做任何限制，支持所有命令拼接符号（\|、\|\|、\&、\&\&、;），用户可以随意拼接多条系统命令，实现多指令连续执行。

输入格式：**正常IP \+ 拼接符号 \+ 恶意命令**，即可绕过原有ping功能，强制执行自定义系统命令。

## 1\.4 Low级核心攻击思路（双系统专属Payload全覆盖）

利用单管道符\|优先级最高的特性，屏蔽前面的ping指令，强制执行后置恶意系统命令，快速获取服务器信息。

### Windows系统 Low 全部可用Payload

首选稳定：127\.0\.0\.1 \| ipconfig

拓展通用：127\.0\.0\.1;ipconfig  /  127\.0\.0\.1 \& ipconfig  /  127\.0\.0\.1 \&\& ipconfig

拓展探测：127\.0\.0\.1 \| whoami（查看当前系统权限）、127\.0\.0\.1 \| dir（查看当前目录文件）

### Linux系统 Low 全部可用Payload

首选稳定：127\.0\.0\.1 \| ifconfig

拓展通用：127\.0\.0\.1;ifconfig  /  127\.0\.0\.1 \& ifconfig  /  127\.0\.0\.1 \&\& ifconfig

拓展探测：127\.0\.0\.1 \| ls（查看网站目录）、127\.0\.0\.1 \| pwd（查看当前路径）、127\.0\.0\.1 \| uname \-a（查看系统内核）

## 1\.5 零失误通关实操步骤（新手逐行照做）

步骤1：进入DVWA Security安全设置，等级选择Low，点击Submit保存；

步骤2：进入Command Injection命令注入模块，清空输入框原有内容；

步骤3：根据服务器系统输入对应纯净Payload（全部英文半角符号，无空格错误）；

步骤4：点击Submit提交，服务器直接执行后置系统命令；

步骤5：页面成功返回系统信息，说明命令注入执行成功，Low关卡通关。

## 1\.6 Low级所有失败场景\+根治方案（全覆盖）

1、无执行结果：使用中文全角符号、多余空格，替换为英文半角符号即可；

2、仅执行ping命令：拼接符号错误、符号前后空格过多，严格按照标准Payload输入；

3、命令报错：系统指令混用，Windows必须用ipconfig/dir，Linux必须用ifconfig/ls；

4、页面无响应：输入非法特殊字符，清空输入框重新输入纯净Payload。

# 第二关：Command Injection Medium 中级（关键字过滤｜缺陷绕过\+超细实操\+逐行源码解析\+双系统Payload）

## 2\.1 Medium新增防护底层原理（通俗白话讲解）

开发者修复Low裸奔漏洞，新增**基础字符过滤机制**，后台黑名单屏蔽了高频拼接符号：**\&\&、\|\|、;**

**超通俗解释**：服务器学会了基础拦截，禁止用户使用双与、双或、分号拼接命令，但**过滤不彻底、不全面**，存在大量漏网符号。

**Medium后台校验规则**：匹配到\&\&、\|\|、;直接清空替换、拦截执行；未匹配到的符号正常放行。

## 2\.2 Medium完整源码 \+ 逐行超细解析（必考绕过原理）

```php
<?php
if( isset( $_POST[ 'Submit' ] ) ) {
    // Get input
    $target = $_POST[ 'ip' ];

    // Filter the user input
    $target = str_replace( '&&', '', $target );
    $target = str_replace( '||', '', $target );
    $target = str_replace( ';', '', $target );

    // Determine OS and execute the ping command.
    if (stristr(PHP_OS, 'WINNT')) {
        // Windows
        $cmd = shell_exec( 'ping  ' . $target );
    } else {
        // Linux
        $cmd = shell_exec( 'ping  -c 4 ' . $target );
    }

    // Feedback for the end user
    echo "<pre>{$cmd}</pre>";
}
?>
```

### 逐行白话拆解\&漏洞分析：

**第6\-8行（核心防护行）**：使用str\_replace字符串替换函数，将匹配到的\&\&、\|\|、;直接替换为空，实现黑名单过滤。

**致命缺陷（面试高频考点）**：该过滤属于**局部黑名单过滤**，只屏蔽了三种拼接符号，**完全遗漏 \| 、\& 两个核心注入符号**，攻击者可直接绕过。

**漏洞本质**：开发者误以为封禁高频符号即可防护，忽略了同等效果的替代拼接符号，过滤逻辑不完整。

**其余代码**：和Low级别完全一致，依旧直接拼接用户输入、无参数隔离、无其他校验，底层漏洞依旧存在。

## 2\.3 中级漏洞核心缺陷（绕过核心原理｜必考）

该防护属于**局部弱智过滤，治标不治本**，只屏蔽了部分拼接符号，遗漏了最核心、最好用的**单管道符 \|** 和 **单与符 **\&。

通俗说：服务器封了简单符号，漏掉了杀伤力最强的符号，攻击者可以直接使用未被屏蔽的符号继续注入攻击。

**终极绕过本质**：放弃被屏蔽的\|\|、\&\&、;，使用未拦截的单\|、单\&符号拼接命令，完美绕过过滤机制。

## 2\.4 LowPayload在中级失效的原因

Low级常用的\|\|、\&\&、;拼接方式，在中级会被后台黑名单直接匹配、清空替换，导致命令拼接失效，仅执行默认ping操作，无注入效果。

## 2\.5 中级零报错绕过双系统Payload（仅可用，无失效）

### Windows系统 Medium 专属可用Payload

最优稳定：127\.0\.0\.1 \| ipconfig

备用绕过：127\.0\.0\.1 \& ipconfig

拓展探测：127\.0\.0\.1 \| whoami 、 127\.0\.0\.1 \| systeminfo

**绝对禁止**：; 、\|\| 、\&\& 全部拦截失效

### Linux系统 Medium 专属可用Payload

最优稳定：127\.0\.0\.1 \| ifconfig

备用绕过：127\.0\.0\.1 \& ifconfig

拓展探测：127\.0\.0\.1 \| ls 、 127\.0\.0\.1 \| id

**绝对禁止**：所有双符号、分号拼接全部失效

## 2\.6 中级零报错实操步骤（超细适配过滤规则）

步骤1：DVWA Security切换等级为Medium，Submit保存配置；

步骤2：进入命令注入页面，输入未被过滤的对应系统纯净Payload；

## 命令注入漏洞危害（实操/面试必背，通俗易懂）

1、**服务器信息泄露**：恶意查询服务器IP、目录、系统配置、用户信息，泄露核心资产；

2、**任意系统命令执行**：可创建用户、修改系统配置、读写服务器本地文件；

3、**网站权限沦陷**：查看网站源码、配置文件，获取数据库账号密码、后台密钥；

4、**服务器完全被控**：可植入后门、木马、提权，直接拿下服务器最高管理员权限，危害远超普通Web漏洞。

## 命令注入终极防御方案（根治漏洞的5种核心手段，通俗解读）

1、**白名单参数校验（最优方案）**：不采用黑名单过滤，仅允许合法IP字符输入，拒绝所有特殊符号、编码字符，从根源杜绝注入；

2、**固定系统命令，参数隔离**：后端固定ping命令，用户输入仅作为独立参数，不拼接进命令主体，无法篡改执行逻辑；

3、**特殊字符转义过滤**：对\|、\&、%0a、;等所有命令符号、编码字符统一转义拦截；

4、**禁用高危系统函数**：代码层面限制system、exec、shell\_exec等危险命令执行函数，禁止随意调用系统指令；

5、**最小权限运行服务**：网站运行账号配置最低权限，即使被注入，也无法执行高危系统操作。
