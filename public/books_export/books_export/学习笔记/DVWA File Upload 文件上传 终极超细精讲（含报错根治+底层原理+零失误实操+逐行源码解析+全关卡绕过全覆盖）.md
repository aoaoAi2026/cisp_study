# DVWA File Upload 文件上传 终极超细精讲（含报错根治\+底层原理\+零失误实操\+逐行源码解析\+全关卡绕过全覆盖）

## 0\. 零基础彻底吃透：文件上传（File Upload）核心本质（人话\+底层原理）

### 0\.1 官方名词通俗拆解

**File Upload 全称：任意文件上传漏洞**

逐词通俗解读，摒弃晦涩术语，小白一看就懂：

File（文件）：用户前端上传的所有文件，包含图片、压缩包、文本、脚本、后门木马等格式文件

Upload（上传）：网站业务提供文件上传功能，允许用户将本地文件传输至服务器保存、解析、展示

**终极一句话定义（面试/实操通用）：**文件上传漏洞是由于网站后端未对上传文件的**后缀、MIME类型、文件内容、文件路径**做严格校验，导致攻击者可上传恶意脚本文件至服务器，直接解析执行代码、植入持久后门，最终造成服务器完全沦陷的超高危漏洞。

### 0\.2 黄金生活化类比（小白秒懂攻击逻辑）

场景对标：**文件上传 = 小区快递代收点，服务器 = 小区仓库，校验规则 = 快递安检**

正常业务：小区代收点只允许存放生活用品（图片、静态文件），安检严格排查违禁品，仓库只存放合规物品，安全可控。

上传攻击：安检机制失效、检查不全面，攻击者将违禁危险品（PHP木马、恶意脚本）混入上传，仓库直接收纳并允许运行，最终导致整个小区被控制。

核心特点：文件上传漏洞是**直接落地型漏洞**，无需二次触发，上传成功即代码执行，危害优先级高于文件包含、命令注入。

### 0\.3 文件上传核心分类（考试/实操必懂）

**1、无校验裸奔上传（Low级）**：无任何过滤，任意后缀、任意文件均可上传，直接上传木马一键拿权

**2、前端JS校验绕过（Medium级）**：仅前端限制文件类型，后端校验缺失，可抓包篡改绕过

**3、后端严格后缀校验（High级）**：后端黑名单拦截脚本后缀，需文件伪装、截断、二次绕过

**4、超严防护绕过（Impossible级）**：多维度校验\+随机文件名，彻底封堵上传漏洞

### 0\.4 文件上传成立三大必要条件（缺一不可）

1、**存在上传入口**：网站具备文件上传功能，支持用户提交本地文件

2、**校验机制缺失/不完善**：仅前端校验、仅校验后缀、仅校验MIME类型，无综合校验

3、**文件可解析访问**：上传目录可执行脚本、文件路径可访问、服务器支持对应脚本解析

### 0\.5 新手必懂：文件上传为什么是Top级高危漏洞？

文件上传属于Web漏洞中**危害天花板级别**，区别于间接触发漏洞：文件包含需要二次访问、命令注入需要参数触发，而文件上传**上传成功即落地、访问即执行**。可直接植入一句话木马、大马，获取网站权限、服务器权限，是渗透测试中最直接、最高效的拿权方式。

### 0\.6 【重点补强】上传校验四大维度\&核心绕过思路（零基础必背）

所有DVWA关卡、实战上传漏洞，全部围绕这四大校验维度攻防对抗，吃透即可通杀所有上传题型。

#### 0\.6\.1 前端JS校验（最弱防护）

仅在网页端限制文件后缀，抓包可直接篡改、禁用JS可绕过，无任何后端防护效果。

#### 0\.6\.2 后缀名校验（黑白名单）

黑名单：禁止php、asp、jsp等脚本后缀（可通过大小写、特殊后缀绕过）

白名单：仅允许jpg、png、gif等图片后缀（相对安全，需配合其他绕过）

#### 0\.6\.3 MIME类型校验（文件头类型）

校验文件Content\-Type类型，检测是否为图片/静态文件，可通过抓包修改数据包绕过。

#### 0\.6\.4 文件内容校验（最强防护）

检测文件头部内容、文件数据流，验证是否为真实图片，仅修改后缀无法绕过。

## 1\. DVWA\-File Upload 模块通用底层逻辑（四关完全共用）

### 1\.1 模块核心功能

整个模块核心业务：**用户上传图片文件，服务器接收并保存至指定目录**。正常业务仅允许上传jpg、png、gif格式图片，用于头像、素材上传等场景。

### 1\.2 正常手动操作完整流程

用户进入上传模块 → 选择本地图片文件提交 → 服务器校验文件合法性 → 保存至服务器目录 → 页面返回上传路径，正常展示图片。

### 1\.3 漏洞核心根源（四关通用底层问题）

DVWA文件上传模块迭代核心问题：**防护维度单一，仅做片面校验，未实现「后缀\+类型\+内容\+路径\+文件名」全维度校验**。低中高级均存在校验缺失，导致各类绕过方式生效，仅最高级实现完整安全机制。

### 1\.4 新手高频报错\&失效通用根因（提前避坑）

1、路径访问错误：不清楚上传文件保存目录，导致木马上传成功无法访问

2、文件被拦截：未对应关卡绕过规则，使用错误Payload被过滤

3、脚本无法执行：服务器目录禁止脚本执行、PHP环境异常

4、缓存/权限问题：旧文件缓存未清除、目录写入权限不足

5、格式绕过错误：大小写、后缀拼接、MIME类型修改不规范

### 1\.5 关键前置：DVWA默认上传路径（全关卡通用必考）

**DVWA统一上传保存目录**：`./hackable/uploads/`

文件上传成功后，访问格式：`http://DVWA地址/hackable/uploads/上传文件名`

双系统路径逻辑一致，无需适配系统差异，全环境通用。

# 第二关：File Upload Medium 中级（前端\+MIME类型校验｜双重绕过｜逐行源码解析）

## 2\.1 Medium新增防护底层原理（通俗白话讲解）

开发者修复Low裸奔漏洞，新增**双重防护机制**：前端JS后缀校验 \+ 后端MIME类型校验。仅允许image/jpeg、image/png类型文件上传，拦截脚本文件，但防护存在致命缺陷：**只校验文件类型，不校验文件内容、不校验后缀**，抓包可直接篡改绕过。

**通俗理解**：安检只看文件标签（MIME类型），不看文件真实内容，攻击者可伪造标签骗过安检，上传恶意脚本。

## 2\.2 Medium完整源码 \+ 逐行超细解析（必考绕过原理）

```php

    // Check if file type is image
```

**致命缺陷（面试高频考点）**：MIME类型由前端数据包可控，攻击者抓包即可随意篡改，后端无文件内容校验、无后缀校验，伪造类型即可绕过。

**漏洞本质**：基于客户端可控数据做服务端校验，校验数据可伪造，防护完全失效。

## 2\.3 中级漏洞核心缺陷\&绕过原理（必考）

1、**前端JS校验可直接跳过**：禁用浏览器JS、直接抓包上传，无需修改文件；

2、**MIME类型可伪造**：抓包修改数据包中Content\-Type为图片类型，欺骗后端校验；

3、**无文件内容校验**：即使文件是PHP木马，只要类型伪造成功，即可正常上传落地。

## 2\.4 LowPayload在中级失效的原因

直接上传原生PHP文件，数据包Content\-Type为application/x\-php，被后端类型校验直接拦截，上传失败。

### 2\.5\.2 标准伪造数据包参数

修改前：`Content-Type: application/x-php`

修改后：`Content-Type: image/jpeg`

## 2\.6 中级零报错实操步骤（超细适配过滤规则）

步骤1：DVWA Security切换等级为Medium，Submit保存配置；

步骤2：开启BurpSuite抓包，选择PHP木马文件点击上传；

步骤5：访问上传路径，脚本正常解析执行，中级关卡通关。

## 2\.7 中级高频报错\&失效根源（精准避坑）

1、未抓包修改类型：原生PHP文件直接上传被拦截；

```php
            if( !move_uploaded_file( $uploaded_tmp, $target_path ) ) {
```

## 10\. 面试高频增量考点（上传漏洞专项拔高）

1、**为什么MIME类型校验不安全？** MIME类型由客户端数据包自定义可控，服务端未核验真实文件内容，可直接抓包伪造，属于前端可控数据服务端校验，防护完全失效。

2、**图片马为什么不能直接执行？** 服务器仅对php后缀文件解析PHP代码，jpg/png后缀会被当作静态图片渲染，必须通过文件包含、本地解析漏洞触发代码执行。

3、**黑名单和白名单哪个更安全？** 白名单绝对安全，仅放行合规后缀；黑名单存在遗漏风险，小众后缀、变形后缀可轻松绕过。

4、**根治上传漏洞的核心关键是什么？** 禁止信任客户端所有数据，服务端强制内容校验\+后缀白名单\+文件名随机化\+目录权限隔离，四重闭环防护。

补充企业站点高频上传绕过方式，适配真实渗透场景：

**1、大小写后缀绕过**：黑名单拦截php，可上传Shell\.PHP、Php后缀绕过解析；

**2、特殊后缀绕过**：利用phtml、php7、phar等小众脚本后缀绕过常规黑名单；

**3、00截断上传绕过**：文件名shell\.php%00\.jpg，截断后缀解析为PHP脚本；

**4、二次渲染绕过**：针对图片压缩渲染防护，制作不死马图片马，规避内容篡改。
