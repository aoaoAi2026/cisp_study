# DVWA Command Injection 命令注入 全三关保姆级通关教程（零基础超通俗版）

## 前置统一说明（全部关卡通用）

本模块功能：网站提供一个「Ping 测试」输入框，让用户输入 IP，后台帮你测试网络通不通。

漏洞核心根源（三关通用）：**网站把用户输入的内容，直接拼接到服务器系统命令中执行，没有做安全隔离**。

你可以利用电脑特殊符号，把「一条命令」强行变成「多条命令」，实现操控服务器。

---

## 一、命令注入 终极通俗原理（三关通用）

### 1\. 生活大白话类比

网站就像一个**只会执行 Ping 功能的机器人**：

机器人规则：你输入什么，我就原封不动交给服务器电脑执行。

正常输入：127\.0\.0\.1 → 机器人只执行 ping，安全。

恶意输入：127\.0\.0\.1 \| whoami → 机器人看不懂特殊符号，服务器看懂了，直接执行两条命令：

① ping 测试网络

② whoami 查询服务器账号

**本来只能 ping，现在能随便操控服务器，这就是命令注入。**

### 2\. 漏洞诞生三个必要条件（缺一不可）

|序号|必要条件|通俗解释|
|---|---|---|
|1|调用系统原生命令|网站自己不会 ping，只能调用服务器系统自带 ping 工具|
|2|用户输入直接拼接|你写的所有文字，不检查、不隔离，直接粘进系统命令|
|3|未严格过滤分隔符|; \| \& \|\| \&\& 等可以拆分多条命令的符号没有被封完|

### 3\. 所有命令分隔符 通俗大全表（通关核心）

|分隔符号|超通俗人话原理（核心本质）|执行逻辑 \& 触发条件|DVWA 关卡可用性|小白专属备注|
|---|---|---|---|---|

|**\&\& 双与号**|条件联动符，两条命令绑定，必须前置命令正常完成才能执行后续命令|前置成功才执行：仅ping通、无报错，才运行恶意命令|✅Low可用 ❌Medium/High被过滤|必须填正确IP，错IP直接失效|
|**\|\| 双或号**|条件兜底符，和\&\&完全相反，前置命令失败才会执行后续命令|前置失败才执行：ping超时、IP错误、报错崩溃，才运行恶意命令|✅Low/Medium可用 ❌High被过滤|必须填错误IP，正确IP反而失效|

|**\| 管道符**|数据传递\+强制执行符，将前一条命令的执行结果，交给后一条命令处理|强制后置执行：彻底忽略前置命令成败，后置命令必定运行|✅Low/Medium可用 ❌High被过滤|全场最稳，中级关卡通关核心|
|**%0a 换行符**|浏览器编码换行符，页面输入无法直接打出，仅URL传参生效|强制换行截断：将一行命令拆分为两行独立命令，绕过符号黑名单|✅High专属 ✅Low/Medium通用|高级关卡唯一可绕过符号|

## 二、Command Injection Low 低级通关（第一关）

### 1\. 源码防护状态

**完全无过滤、无黑名单、无检查**。

用户输入什么，服务器就执行什么，裸奔状态。

### 2\. 漏洞原理

后台直接拼接命令：ping \+ 用户输入

没有删除、没有拦截、没有转义，所有分隔符全部能用。

### 3\. Low 全部可用 Payload【逐条超级详细拆解｜每条单独讲懂】

Low 级别无任何过滤，所有命令分隔符全部生效。下面 5 条 payload**每一条原理、符号作用、执行逻辑、适用场景**我全部拆开讲，零基础彻底看懂。

**前置统一格式：**所有 payload 结构 = 正常 ping 地址 \+ 特殊分隔符 \+ 想要执行的恶意命令（whoami 查询服务器账号）

---

#### Payload 1：127\.0\.0\.1;whoami

**分隔符：; 分号**

**人话原理：**分号是「彻底切断两条命令，互不影响」。

意思是：不管前面 ping 成功还是失败，**后面的命令 100% 强制执行**。

**服务器真实执行两条命令：**

1、ping \-c 4 127\.0\.0\.1（正常测试网络）

2、whoami（查询服务器当前登录用户）

**优点：**最稳定、百分百成功、不受任何情况影响。

**缺点：**Medium 中级会被过滤，中级不能用。

**总结：Low 首选万能通关指令**

---

#### Payload 2：127\.0\.0\.1\&\&whoami

**分隔符：\&\& 双与号**

**人话原理：前面成功，后面才执行**

必须满足条件：**前面 ping 通、没有报错**，后面 whoami 才会运行。

**执行逻辑：**

1、ping 127\.0\.0\.1 能通 → 成功

2、触发 \&\& 执行后面 whoami

**如果 IP 填错 ping 不通 → 整条注入失效**

**特点：**条件执行、不是万能、只能用正确 IP

**注意：**Medium 中级直接屏蔽，中级完全用不了

---

#### Payload 3：127\.0\.0\.1 \| whoami

**分隔符：\| 管道符（最强符号）**

**人话原理：忽略前面所有结果，强制执行后面命令**

管道符的真正作用：**把前一条命令的结果，交给后一条命令处理**

安全最大漏洞：**不管前面命令对错、报错、崩溃，后面命令一定执行**

**优势无敌：**

就算你填错误 IP：999\.999\.999\.999 \| whoami 照样注入成功

**适用关卡：Low、Medium 都能用（中级通关核心符号）**

---

#### Payload 4：127\.0\.0\.1 \& whoami

**分隔符：\& 单与号**

**人话原理：两条命令并行、同时运行、互不干扰**

相当于一次性告诉服务器：这两条命令全部给我执行，不分先后。

**执行效果：**

ping 执行的同时，whoami 也执行

**特点：**成功率高，但是页面输出容易乱码、排版错乱

**适用：Low、Medium 均可绕过**

---

#### Payload 5：1\.1\.1\.1 \|\| whoami

**分隔符：\|\| 双或号**

**人话原理：前面失败，后面才执行**

和 \&\& 完全相反！

**必须满足条件：前面 ping 不通、报错**

所以这条 payload 故意用 **1\.1\.1\.1（延迟高/本机ping不通）**

**执行逻辑：**

1、ping 1\.1\.1\.1 失败报错

2、触发 \|\| 执行 whoami

**特点：必须错误IP才能生效，正确IP反而失效**

---

### 4\. 五条 Payload 一句话终极区别（小白必背）

1、**;** ：不管成败，后面必执行（Low 最简单）

2、**\&**\& ：前面成功，后面才执行（正确IP才生效）

3、**\|** ：强制后面执行，万能最强（Low/Medium通用）

4、\& ：两条一起跑，并行执行

5、**\|\|** ：前面失败，后面才执行（错误IP专用）

### 6\. 逐行源码拆解（看懂彻底无过滤原理）

Low级别核心源码，全程无任何过滤、无黑名单、无字符删除：

// 直接获取用户输入的所有内容，不做任何处理

$target = $\_REQUEST\['ip'\];

// 直接拼接系统ping命令，用户输入是什么，就执行什么

$cmd = shell\_exec\('ping \-c 4 ' \. $target\);

**核心漏洞：**代码完全信任用户输入，不拦截、不删除特殊符号，所有分隔符（; \| \& \&\& \|\|）都能正常解析，直接触发多条命令执行。

### 7\. 超详细分步通关实操（零基础照做必过）

**步骤1：修改DVWA安全等级**

1、打开DVWA顶部导航栏，点击【DVWA Security】安全设置；

2、下拉选择【Low】，点击底部Submit按钮提交保存；

3、页面自动刷新，代表等级切换成功。

**步骤2：进入漏洞模块**

1、左侧菜单栏点击【Command Injection】命令注入模块；

2、进入IP Ping测试输入页面，确认功能正常。

**步骤3：输入通关Payload并提交**

输入框填写新手万能指令：**127\.0\.0\.1;whoami**，点击Submit提交。

**步骤4：验证通关结果**

1、页面上方输出ping 127\.0\.0\.1的网络连通数据；

2、页面下方输出服务器系统用户（www\-data）；

3、出现额外命令执行结果，证明注入成功，Low关卡通关。

1、DVWA 安全等级切 Low

2、进入 Command Injection

3、输入上面 payload 点提交

4、页面出现 whoami 结果，通关

## 三、Command Injection Medium 中级通关（第二关｜你主学的关卡）

### 1\. 中级完整源码逐行拆解（看懂过滤绕过核心）

Medium级别专属防护代码，漏洞根源一目了然：

// 1\. 定义黑名单数组，仅封禁两个符号

$substitutions = array\( '\&\&' =\> '', ';' =\> '' \);

// 2\. 只删除;分号、\&\&双与号，其余符号全部保留

$target = str\_replace\(array\_keys\($substitutions\), $substitutions, $target\);

// 3\. 依旧直接拼接用户输入，无其他安全校验

$cmd = shell\_exec\('ping \-c 4 ' \. $target\);

**源码漏洞总结：**开发者偷懒只屏蔽两个常用符号，漏掉了**\| 管道符、\&单与号、\|\|双或号**，导致可以直接绕过防护。

开发者做了**简陋黑名单**：

只删除两个符号：**;（分号）和 \&\&（双与号）**

其余所有核心分隔符：**\|（管道符）、\&（单与号）、\|\|（双或号）** 完全没有过滤，可直接绕过

### 2\. 失效Payload超细报错原理（为什么Low的指令不能用）

**失效案例1：127\.0\.0\.1;whoami**

后台过滤流程：检测到**;**分号，直接删除清空；

过滤后残留内容：**127\.0\.0\.1whoami**；

服务器识别：这是一个非法无效IP，无法执行ping命令，更无法拆分指令，注入彻底失败。

**失效案例2：127\.0\.0\.1\&\&whoami**

后台过滤流程：检测到**\&**\&双与号，直接删除清空；

过滤后残留内容：**127\.0\.0\.1whoami**；

同样变成无效IP，命令拼接失效，注入失败。

输入 127\.0\.0\.1;whoami → ; 被删掉 → 变成乱码，执行失败

输入 127\.0\.0\.1\&\&whoami → \&\& 被删掉 → 执行失败

### 3\. 精准绕过思路（零基础必懂）

不跟黑名单硬碰，**彻底放弃被过滤的;和\&**\&，专门使用黑名单中不存在的符号。

中级安全核心漏洞：只防常用符号，不防冷门通用符号，**\| 管道符**全程无过滤、无删除，是中级通关万能钥匙。

**弃用被过滤符号，使用未过滤符号**

中级最强万能符号：**\| 管道符**

### 4\. 中级所有可用Payload 逐条超细拆解

#### ✅ 首选万能Payload（100%稳定）：127\.0\.0\.1 \| whoami

**符号：\| 管道符（未被过滤）**

**完整执行过程：**

1、用户输入完整内容，后台检测黑名单：无;、无\&\&，内容完整保留；

2、后台拼接完整系统命令：ping \-c 4 127\.0\.0\.1 \| whoami；

3、Linux系统识别管道符，强制执行两条独立命令；

4、优先执行ping测试网络，无论ping成功或失败，强制执行whoami查询服务器用户；

5、页面输出双重结果，注入成功。

**优势：**不受IP对错影响、不受执行结果影响、零报错、百分百通关。

#### ✅ 备选Payload1：127\.0\.0\.1 \& ls

**符号：\& 单与号（未被过滤）**

原理：两条命令并行后台运行，ping和ls查询文件命令同时执行，无先后顺序；

缺点：页面输出容易乱序，展示效果不清晰，仅作为备用方案。

#### ✅ 备选Payload2：1\.1\.1\.1 \|\| id

**符号：\|\| 双或号（未被过滤）**

原理：故意填写外网IP 1\.1\.1\.1，本机ping测试大概率超时失败；

前置ping命令报错后，触发\|\|符号，执行id查询服务器权限命令；

缺点：部分网络环境可ping通1\.1\.1\.1，会导致注入失效，稳定性差。

**首选稳定：127\.0\.0\.1 \| whoami**

备选1：127\.0\.0\.1 \& ls

备选2：1\.1\.1\.1 \|\| id

### 5\. 中级标准通关实操步骤（一步不缺）

**步骤1：切换安全等级**

DVWA Security → 选择Medium → Submit保存。

**步骤2：进入命令注入模块**

左侧点击Command Injection，进入Ping测试页面。

**步骤3：输入最优Payload**

输入框填写：**127\.0\.0\.1 \| whoami**

**步骤4：提交验证结果**

点击Submit，页面同时显示ping数据和服务器用户信息，中级通关。

1、用户输入：127\.0\.0\.1 \| whoami

2、后台检测黑名单：没有 \| 符号，不做任何删除

3、拼接命令：ping \-c 4 127\.0\.0\.1 \| whoami

4、系统识别管道符，强制执行两条命令

5、页面输出服务器账号，注入成功

### 6\. 中级拓展高权限指令

读取账号文件：127\.0\.0\.1 \| cat /etc/passwd

读取网站密码：127\.0\.0\.1 \| cat /var/www/html/DVWA/config/config\.inc\.php

## 四、Command Injection High 高级通关（第三关｜最难关卡超细拆解）

### 1\. 高级顶级防护规则（彻底看懂为什么普通Payload全失效）

High级别重写黑名单规则，**封禁所有常规命令分隔符**：

拦截符号：**; 分号、\&\&双与号、\|\|双或号、\&单与号、\|管道符**

**通俗解释：**Low/Medium所有能用的符号，高级全部拉黑，常规拼接命令的方法100%失效，没有任何例外。

高级黑名单非常狠，直接拦截：**; \| \& \&\& \|\|**

所有基础分隔符全部失效，普通写法全部过不去。

### 2\. 高级核心漏洞（唯一突破口）

黑名单只拦截**可视化命令分隔符号**，完全忽略**URL编码隐形符号**。

重点漏防符号：**%0a （URL换行符）**

**小白通俗理解：**

%0a是浏览器专属的隐形换行代码，页面输入框肉眼打不出来，后台黑名单规则没有收录这个符号；

服务器系统可以识别%0a为合法换行，直接把一行命令拆分成两行独立命令，完美绕过所有防护。

黑名单依旧**没有封死所有符号**，漏掉了**换行符 %0a**

换行符可以强行截断前命令、开启新命令，实现注入。

### 3\. 高级专属Payload 超细逐字拆解

**唯一稳定通关指令：127\.0\.0\.1%0awhoami**

**整句拆分讲解：**

1、127\.0\.0\.1：正常ping测试IP，无任何风险，不会被拦截；

2、%0a：隐形换行截断符（核心绕过点，无过滤、无删除）；

3、whoami：后置恶意查询命令。

**服务器真实执行逻辑：**

原本拼接的一行命令：ping \-c 4 127\.0\.0\.1%0awhoami

服务器识别%0a换行，自动拆分为两行独立命令：

第一行：ping \-c 4 127\.0\.0\.1（正常网络测试）

第二行：whoami（独立执行恶意查询命令）

两行命令互不干扰，成功实现命令注入，绕过顶级黑名单防护。

**127\.0\.0\.1%0awhoami**

你遇到的情况很典型，`%0a` 在 DVWA 的 Command Injection 高难度下无效，通常是因为后端过滤机制与你的预期不同。我们来分析原因并给出确切的绕过方法。

## 1. 为什么 `127.0.0.1%0awhoami` 没有反应？

DVWA High 难度的源码使用了**黑名单**过滤，而不是转义函数。典型代码如下：

```php
$substitutions = array(
    '&'  => '',
    ';'  => '',
    '| ' => '',   // 注意：只过滤了"管道符+空格"
    '-'  => '',
    '$'  => '',
    '('  => '',
    ')'  => '',
    '`'  => '',
    '||' => '',
);
$target = str_replace(array_keys($substitutions), $substitutions, $target);
```

它并**没有过滤换行符** `\n`（即 `%0a`），按说 `%0a` 应该可以生效。你没有看到反应，最常见的原因是：

- **输出被浏览器的 `<pre>` 标签渲染方式干扰** —— `whoami` 的结果可能输出了，但和 ping 结果混在一起，或在页面某个不显眼的位置，你没有注意到。
- **目标系统是 Windows**（DVWA 可部署在 Windows + XAMPP 上）。Windows 的命令行（`cmd.exe`）中，换行符不能分隔两条命令，必须用 `&` 或 `&&`。你的 `%0a` 会被当做 ping 参数的一部分，从而注入失败。
- **某些定制版 DVWA 或 WAF 层面额外过滤了换行符**（极少见）。

但不论如何，我们只需要换一个**更通用、肯定能绕过当前黑名单**的 payload 即可。

## 2. 稳定可用的 High 难度绕过 Payload

由于黑名单只过滤了 `| `（管道符 + 空格）和 `||`，而没有过滤**单独的管道符** `|`，也没有过滤**没有空格紧跟命令**的情形。Linux shell 允许管道符前后不加空格，因此可以这样写：

```
127.0.0.1|whoami
```

最终在服务器上执行的命令会变成：
```bash
ping -c 4 127.0.0.1|whoami
```
管道将 ping 的输出传给 whoami（无意义但会执行），**whoami 的结果会追加显示在输出中**。

如果你想执行带参数的命令（例如 `cat /etc/passwd`），空格没有被过滤，可以直接用：
```
127.0.0.1|cat /etc/passwd
```

如果你想更保险一些，防止某些极端环境空格被过滤（虽然当前黑名单没过滤空格），可以使用 `${IFS}` 替换空格：
```
127.0.0.1|cat${IFS}/etc/passwd
```

## 3. 若仍无回显，尝试输出重定向或时间延迟验证

如果页面仍然只显示 ping 结果，可能你的命令执行了但输出被某种方式抑制。此时可以这样做：

- **写文件验证**：  
  `127.0.0.1|whoami>/var/www/html/out.txt`  
  然后直接访问 `http://你的DVWA地址/out.txt` 查看内容。
- **用 ping 延迟检测**：  
  `127.0.0.1|ping${IFS}-c${IFS}5${IFS}127.0.0.1`  
  如果页面加载明显变慢 5 秒，说明命令注入成功（`-c` 参数会因黑名单过滤 `-` 被吃掉，但这里 ping 后的参数里 `-c 5` 中的 `-` 会被删除，变成 `c 5`，可能报错。更准确的延迟检测可以用 `sleep`）：  
  `127.0.0.1|sleep${IFS}5`  
  页面延迟 5 秒响应即为注入成功。

## 七、命令注入 终极根治防御方案（逐点通俗拆解）

### 1\. 绝对禁止使用黑名单防护（最大误区）

黑名单永远防不住，原因：命令分隔符、编码绕过符号无数，开发者不可能全部收录，永远存在遗漏漏洞，DVWA三关全程印证了这个问题。

### 2\. 最优方案：白名单严格校验（根治漏洞）

针对Ping测试功能，**只放行合法IP字符**：仅允许数字0\-9、小数点\.；

只要输入框出现\| ; \& %等特殊符号，直接拦截、报错、拒绝提交，从根源杜绝注入。

### 3\. 禁止直接拼接系统命令（代码层修复）

高危错误写法（DVWA漏洞根源）：shell\_exec\("ping "\.$user\_input\)

安全规范写法：分离命令参数，用户输入仅作为独立参数，不参与命令拼接，系统不会解析特殊符号。

### 4\. 全局转义所有高危特殊字符

对所有系统高危符号做转义处理，让服务器只把特殊符号当成普通文字，无法识别为命令分隔符，丧失注入条件。

### 5\. 服务器降权运行（兜底防护）

网站程序不使用root最高权限运行，仅配置低权限账号；即便被注入成功，攻击者也无法读取核心系统文件、篡改服务器配置，最大限度降低危害。

1、**不用黑名单（永远防不住）**

2、优先**白名单**：ping 输入框只允许数字和小数点

3、禁止直接拼接系统命令

4、特殊符号全部转义，让系统无法识别命令分隔符
