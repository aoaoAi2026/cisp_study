# 第26章 移动安全测试环境

## 26.1 移动安全测试概述

### 26.1.1 什么是移动安全测试

移动安全测试是指对移动应用（Android、iOS等）进行安全评估，发现其中的安全漏洞和隐私问题的过程。随着移动设备的普及，移动应用安全越来越重要。

移动安全测试主要包括：
- **静态分析（Static Analysis）**：不运行应用，分析APK/IPA文件结构、代码、资源等
- **动态分析（Dynamic Analysis）**：运行应用，观察其行为、网络通信、数据存储等
- **渗透测试（Penetration Testing）**：模拟攻击者，尝试利用漏洞获取敏感信息或控制设备
- **逆向工程（Reverse Engineering）**：反编译应用，分析其实现逻辑
- **漏洞挖掘（Vulnerability Research）**：发现应用中的0day漏洞

### 26.1.2 移动安全测试工具分类

| 工具类型 | 代表工具 | 平台 | 主要用途 |
|---------|---------|------|---------|
| IDE/SDK | Android Studio、Xcode | Win/Mac | 应用开发和调试 |
| 反编译工具 | Apktool、JADX、ApkStudio | Win/Linux/Mac | APK反编译和回编译 |
| 动态调试 | Frida、Xposed、Substrate | Android/iOS | 运行时Hook和修改 |
| 综合扫描 | MobSF、QARK、Androbugs | Win/Linux/Mac | 自动化安全扫描 |
| 抓包工具 | Charles、Burp Suite、Fiddler | Win/Linux/Mac | 网络流量分析 |
| 渗透框架 | Drozer、Metasploit | Win/Linux/Mac | 移动应用渗透测试 |
| 逆向工具 | IDA Pro、Ghidra | Win/Linux/Mac | Native代码分析 |
| 模拟器 | AVD、Genymotion | Win/Linux/Mac | 应用运行和测试 |

### 26.1.3 环境要求

#### 硬件要求
- CPU：4核8线程以上
- 内存：16GB以上（推荐32GB）
- 硬盘：200GB以上可用空间
- 测试设备：Android手机（推荐Root）、iOS手机（推荐越狱）

#### 软件环境
- 操作系统：Windows 10/11、macOS或Linux
- Java环境：JDK 11+
- Python环境：Python 3.7+
- Android SDK
- 虚拟化软件（可选）：VMware/VirtualBox

### 26.2.6 Root环境配置（Magisk）

**【通用】Magisk Root配置**

**操作位置：Android手机 + 电脑**

**Magisk简介：**
Magisk是目前最流行的Android Root方案，采用Systemless Root方式，不修改系统分区，更容易通过SafetyNet检测。

**准备工作：**
- 解锁Bootloader（不同品牌方法不同）
- 下载对应版本的Magisk APK
- 提取手机的boot.img

**安装步骤：**

```bash
# 方法1：通过Recovery安装（推荐）

# 1. 下载Magisk APK
# 访问：https://github.com/topjohnwu/Magisk/releases
# 下载 Magisk-v26.4.apk

# 2. 重命名为zip
# 将 Magisk-v26.4.apk 重命名为 Magisk-v26.4.zip

# 3. 进入Recovery模式
adb reboot recovery

# 4. 刷入Magisk
# 在TWRP中选择Install
# 选择Magisk-v26.4.zip
# 滑动确认刷入
# 重启系统

# 5. 验证Root
# 安装Magisk APK
# 打开Magisk Manager
# 显示Magisk已安装即为成功
```

```bash
# 方法2：修补boot.img

# 1. 提取手机的boot.img
# 从固件中提取
# 或使用 dd 命令备份

# 2. 将boot.img传到手机
adb push boot.img /sdcard/Download/

# 3. 使用Magisk Manager修补
# 打开Magisk Manager
# 点击"安装" → "选择并修补一个文件"
# 选择boot.img
# 等待修补完成

# 4. 将修补后的文件传回电脑
adb pull /sdcard/Download/magisk_patched-xxx.img

# 5. 刷入修补后的boot.img
adb reboot bootloader
fastboot flash boot magisk_patched-xxx.img
fastboot reboot

# 6. 验证Root
adb shell
su
# 手机弹出授权提示，点击允许
whoami  # 应该显示root
```

**Magisk常用模块：**
- **MagiskHide**：隐藏Root，绕过检测（已集成到Magisk中）
- **LSPosed**：Xposed框架的Magisk版本
- **BusyBox**：Linux工具集
- **Systemless Hosts**：修改hosts文件
- **Riru**：注入框架（LSPosed依赖）

### 26.2.7 常见问题

**Q1: ADB找不到设备？**

A: 排查步骤：
1. 确认USB线是数据线（不是充电线）
2. 确认已开启USB调试
3. 更换USB接口或USB线
4. 重启ADB服务：`adb kill-server && adb start-server`
5. 安装手机驱动（Windows需要）
6. 撤销USB调试授权后重新连接
7. 尝试切换USB配置（MTP/PTP/仅充电）

**Q2: 模拟器启动失败？**

A: 常见原因：
1. CPU没有开启VT-x/AMD-V → BIOS中开启虚拟化
2. 内存不足 → 减少模拟器内存
3. HAXM未安装 → 安装Intel HAXM
4. Hyper-V冲突 → 禁用Hyper-V（Windows）
5. 显卡驱动问题 → 更新显卡驱动
6. 使用软件渲染：`emulator -avd <name> -gpu swiftshader_indirect`

### 26.3.1 Apktool反编译

**【Windows环境】【Linux环境】Apktool**

**Apktool简介：**
Apktool是最常用的APK反编译工具，可以将APK反编译为smali代码和资源文件，也可以将修改后的内容回编译为APK。

**安装步骤：**

```bash

# 或安装最新版
cd ~/Downloads

chmod +x apktool
sudo mv apktool /usr/local/bin/
sudo mv apktool_2.9.3.jar /usr/local/bin/apktool.jar

# 【Windows环境】
# 下载：https://ibotpeaches.github.io/Apktool/install/
# 下载apktool.bat和apktool.jar
```

```bash
# 基本反编译

apktool d -r app.apk

# 保持调试信息
apktool d -d app.apk

# 强制覆盖输出目录
apktool d -f app.apk
```

**APK签名：**

```bash
# 生成签名密钥（首次使用）
keytool -genkeypair -v \
  -keystore my-release-key.keystore \
  -alias my-key-alias \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000

# 使用apksigner签名（推荐）

  --ks-key-alias my-key-alias \

# 使用jarsigner签名（旧方法）

  -keystore my-release-key.keystore \
  modified.apk my-key-alias
```

### 26.3.2 jadx反编译

**【Windows环境】【Linux环境】JADX**

**操作位置：命令行终端 + GUI**

**JADX简介：**
JADX是一款优秀的Java/Android反编译工具，可以将DEX/APK文件反编译为Java源代码，支持命令行和GUI两种模式。

**安装步骤：**

```bash
sudo ln -s ~/tools/jadx/bin/jadx /usr/local/bin/jadx
sudo ln -s ~/tools/jadx/bin/jadx-gui /usr/local/bin/jadx-gui

# 【Windows环境】
# 下载jadx-1.4.7.zip
# 解压到 C:\Tools\jadx\
# 添加bin目录到PATH
```

**命令行使用：**

```bash
# 反编译APK
jadx -d output_dir app.apk

# 反编译DEX
jadx -d output_dir classes.dex

# 反编译JAR
jadx -d output_dir app.jar

# 只反编译指定类
jadx --class "com.example.MainActivity" app.apk

# 反编译为Gradle项目结构
jadx --export-gradle -d project_dir app.apk

# 显示反编译进度
jadx -v app.apk

# 不生成资源
jadx --no-res app.apk

# 不反编译源码（只分析）
jadx --no-src app.apk
```

**GUI使用：**

```bash
# 启动JADX GUI
jadx-gui

# 或直接打开文件
```

```javascript
    var MainActivity = Java.use("com.example.app.MainActivity");
    
    // Hook onCreate方法
    MainActivity.onCreate.implementation = function(savedInstanceState) {
        console.log("[*] MainActivity.onCreate called");
        
        // 打印调用栈
        console.log(Java.use("android.util.Log").getStackTraceString(
            Java.use("java.lang.Exception").$new()
        ));
        
        // 调用原方法
        this.onCreate(savedInstanceState);
    };
    
    // Hook isAdmin方法，修改返回值
    MainActivity.isAdmin.implementation = function() {
        console.log("[*] isAdmin called, returning true");
        return true;
    };
    
    // Hook加密函数，打印参数和返回值
    var CryptoUtil = Java.use("com.example.app.CryptoUtil");
    CryptoUtil.encrypt.implementation = function(data) {
        console.log("[*] encrypt called with: " + data);
        var result = this.encrypt(data);
        console.log("[*] encrypt result: " + result);
        return result;
    };
});

// Hook Native函数
Interceptor.attach(Module.findExportByName("libnative-lib.so", "native_encrypt"), {
    onEnter: function(args) {
        console.log("[*] native_encrypt called");
        console.log("arg0: " + args[0].readCString());
    },
    onLeave: function(retval) {
        console.log("[*] native_encrypt return: " + retval.readCString());
    }
});
```

## 26.4 iOS安全测试环境

### 26.4.1 iOS越狱环境介绍

**iOS越狱简介：**
iOS越狱是指绕过苹果的限制，获取iOS系统最高权限（root权限）的过程。越狱后可以安装第三方应用、修改系统、进行安全测试等。

**越狱类型：**

| 类型 | 说明 | 特点 |
|-----|------|------|
| 完美越狱 | 重启后越狱状态保留 | 最理想，但越来越少 |
| 不完美越狱 | 重启后需要重新越狱 | 常见，需要电脑引导 |
| 半完美越狱 | 重启后部分功能受限 | 介于两者之间 |
| 半越狱（Taurine等） | 用户空间越狱，不修改系统 | 较新的越狱方式 |

**常见越狱工具：**
- **checkra1n**：基于BootROM漏洞，支持A5-A11芯片，iOS 12-14
- **unc0ver**：支持iOS 11-14，使用内核漏洞
- **Taurine**：支持iOS 14，基于libkrw
- **palera1n**：基于checkra1n，支持iOS 15-16
- **Dopamine**：支持iOS 15.2-16.5.1，无根越狱
- **TrollStore**：利用CoreTrust漏洞，永久签名

**注意事项：**
1. 越狱有风险，可能导致设备变砖
2. 越狱会失去保修
3. 越狱后设备安全性降低
4. 建议使用测试设备，不要用主力机
5. 重要数据提前备份

### 26.6.4 HTTPS抓包绕过SSL Pinning

**SSL Pinning简介：**
SSL Pinning（证书钉扎）是指应用在代码中内置了服务器的证书或公钥，只信任指定的证书，即使系统安装了其他CA证书也无法拦截HTTPS流量。

**绕过方法：**

**方法1：使用Frida Hook（推荐）**

```javascript
// ssl_pinning_bypass.js
Java.perform(function() {
    console.log("[*] SSL Pinning Bypass loaded");
    
    // 绕过OkHttp证书验证
    try {
        var CertificatePinner = Java.use("okhttp3.CertificatePinner");
        CertificatePinner.check.overload('java.lang.String', 'java.util.List').implementation = function(host, peerCertificates) {
            console.log("[*] Bypassing OkHttp CertificatePinner.check for: " + host);
            return;
        };
        console.log("[+] OkHttp CertificatePinner hooked");
    } catch(e) {
        console.log("[-] OkHttp CertificatePinner not found: " + e.message);
    }
    
    // 绕过WebView证书验证
    try {
        var WebViewClient = Java.use("android.webkit.WebViewClient");
        WebViewClient.onReceivedSslError.implementation = function(view, handler, error) {
            console.log("[*] Bypassing WebView SSL error");
            handler.proceed();
        };
        console.log("[+] WebView SSL error hooked");
    } catch(e) {
        console.log("[-] WebViewClient not found: " + e.message);
    }
    
    // 绕过X509TrustManager
    try {
        var TrustManagerFactory = Java.use("javax.net.ssl.TrustManagerFactory");
        var X509TrustManager = Java.use("javax.net.ssl.X509TrustManager");
        var SSLContext = Java.use("javax.net.ssl.SSLContext");
        
        // 创建信任所有证书的TrustManager
        var TrustAllManager = Java.registerClass({
            name: "com.example.TrustAllManager",
            implements: [X509TrustManager],
            methods: {
                checkClientTrusted: function(chain, authType) {},
                checkServerTrusted: function(chain, authType) {},
                getAcceptedIssuers: function() { return []; }
            }
        });
        
        console.log("[+] TrustAllManager created");
    } catch(e) {
        console.log("[-] SSL bypass error: " + e.message);
    }
});
```

```javascript
Interceptor.attach(Module.findExportByName("libnative-lib.so", "Java_com_example_app_MainActivity_nativeEncrypt"), {
```

### 26.7.6 加固脱壳环境

**Android加固简介：**
为了防止应用被反编译和破解，很多应用会使用加固服务（加壳），将dex文件加密隐藏，运行时动态解密加载。

**常见加固服务商：**
- 360加固保
- 腾讯乐固
- 百度加固
- 阿里聚安全
- 梆梆安全
- 爱加密
- 网易易盾

**脱壳方法：**

**1. 内存dump法**
```bash
# 原理：
# 加固的应用在运行时会解密dex并加载到内存
# 从内存中dump出dex文件

# 工具：
# - Frida + dump脚本
# - FART（基于Frida的脱壳工具）
# - Youpk
# - dumpDex
# - IDA Pro动态调试dump

# Frida脱壳示例：
# 搜索内存中的dex特征（dex.035/036/037/038）
# 找到后dump到文件
```

**2. Hook脱壳**
```bash
# 原理：
# Hook dex加载相关函数
# 在dex加载完成后dump

# 关键函数：
# - DexFile.loadDex
# - DexFile.openDexFile
# - ClassLoader.loadClass
# - 等等
```

**3. 定制系统脱壳**
```bash
# 原理：
# 修改Android系统源码
# 在dex加载时自动dump

# 代表：
# - FART（基于Android源码修改）
# - Youpk（定制Recovery）
```

**脱壳工具推荐：**
- **FART**：功能强大的脱壳工具，基于Frida
- **Youpk**：基于定制Recovery的脱壳方案
- **BlackDex**：Android 8+脱壳工具
- **Refine**：开源脱壳框架
- **Frida-unpack**：Frida脱壳脚本集合

**注意事项：**
1. 脱壳有难度，不同加固方法不同
2. 很多加固有反调试、反Frida检测
3. 可能需要组合使用多种方法
4. 脱壳后的dex可能需要修复
5. 仅供学习研究使用，请遵守法律法规

### 26.7.7 常见问题

**Q1: 推荐先学哪个漏洞环境？**

A: 学习路径建议：
1. 先从DIVA开始，涵盖各种基础漏洞
2. 然后是Sieve，配合Drozer练习组件测试
3. 再试InsecureBankv2，更贴近真实应用
4. 最后可以尝试CTF中的移动安全题目
5. 边学边练，逐步深入

**Q2: 逆向so层太难了怎么办？**

A: 学习建议：
1. 先学好ARM汇编基础
2. 从简单的Native程序开始分析
3. 多用Frida Hook，观察参数和返回值
4. 结合静态分析和动态调试
5. 多做练习，熟能生巧
6. 可以先学x86汇编，再学ARM
7. 利用好搜索和社区资源

**Q3: 加固的应用怎么分析？**

A: 加固应用分析步骤：
1. 先识别用了什么加固（查壳工具）
2. 尝试静态分析（看manifest、资源、so库）
3. 尝试动态分析（Frida Hook、抓包）
4. 尝试脱壳（根据加固类型选择方法）
5. 脱壳后再做完整分析
6. 脱壳有难度，不要放弃，多尝试不同方法
7. 有些加固可以通过Hook直接分析，不一定非要脱壳
