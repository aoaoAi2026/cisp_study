### 27.1.4 网卡模式切换（monitor模式）

#### 【Linux环境】使用airmon-ng切换监听模式

```bash
# 操作位置：Kali Linux终端
# 步骤1：查看网络接口
sudo airmon-ng

# 预期输出：
# PHY	Interface	Driver		Chipset
# phy0	wlan0		rt2800usb	Ralink Technology, Corp. RT2870/RT3070

# 步骤2：停止可能干扰的进程
sudo airmon-ng check kill

# 预期输出：
# Killing these processes:
#   PID Name
#   1234 NetworkManager
#   5678 wpa_supplicant

# 步骤3：启动监听模式
sudo airmon-ng start wlan0

# 预期输出：
# PHY	Interface	Driver		Chipset
# phy0	wlan0mon	rt2800usb	Ralink Technology, Corp. RT2870/RT3070
# 
# 	(mac80211 monitor mode vif enabled for [phy0]wlan0 on [phy0]wlan0mon)
# 	(mac80211 station mode vif disabled for [phy0]wlan0)

# 步骤4：验证监听模式
iwconfig wlan0mon

# 预期输出：
# wlan0mon  IEEE 802.11  Mode:Monitor  Frequency:2.457 GHz  Tx-Power=20 dBm
#           Retry short limit:7   RTS thr:off   Fragment thr:off
#           Power Management:off
```

#### 【Linux环境】使用iw命令切换监听模式

```bash
# 操作位置：Kali Linux终端
# 步骤1：关闭网卡
sudo ip link set wlan0 down

# 步骤2：切换到监听模式
sudo iw wlan0 set type monitor

# 步骤3：开启网卡
sudo ip link set wlan0 up

# 步骤4：验证模式
iw wlan0 info

# 预期输出：
# Interface wlan0
# 	ifindex 3
# 	wdev 0x1
# 	addr aa:bb:cc:dd:ee:ff
# 	type monitor
# 	channel 6 (2437 MHz), width: 20 MHz, center1: 2437 MHz
# 	txpower 20.00 dBm
```

#### 【Linux环境】切回托管模式

```bash
# 操作位置：Kali Linux终端
# 使用airmon-ng
sudo airmon-ng stop wlan0mon

# 或使用iw命令
sudo ip link set wlan0 down
sudo iw wlan0 set type managed
sudo ip link set wlan0 up

# 重启网络服务
sudo systemctl start NetworkManager
```

### 27.1.5 常见问题

**Q: 无线网卡插入后不显示怎么办？**

A: 按以下步骤排查：
```bash
# 操作位置：Kali Linux终端
# 查看USB设备
lsusb

# 查看PCI设备
lspci | grep -i wireless

# 查看内核日志
dmesg | grep -i wireless

# 查看网络接口
ip link show
```

**Q: airmon-ng start 失败怎么办？**

A: 常见原因和解决方法：
1. 网卡不支持监听模式 - 更换支持的网卡
2. 被其他进程占用 - 使用 `airmon-ng check kill` 终止进程
3. 驱动问题 - 更新驱动或更换内核版本

**Q: 注入测试失败怎么办？**

A: 检查以下几点：
1. 确认网卡芯片支持注入
2. 距离AP不要太远，信号强度足够
3. 尝试更换频道
4. 检查网卡功率设置 `iwconfig wlan0mon txpower 30`

#### 【Linux环境】airmon-ng（监听模式）

airmon-ng用于管理无线网卡的监听模式。

**常用命令：**

```bash
# 操作位置：Kali Linux终端
# 查看所有无线网卡
sudo airmon-ng

# 检查并终止干扰进程
sudo airmon-ng check kill

# 启动指定网卡的监听模式
sudo airmon-ng start wlan0

# 启动监听模式并指定频道
sudo airmon-ng start wlan0 6

sudo airmon-ng stop wlan0mon

airmon-ng --help
```

### 27.3.7 常见问题

**Q: WEP破解需要多少IVs？**

A: 一般情况下：
- 64位WEP：约10,000-50,000个IVs
- 128位WEP：约50,000-200,000个IVs
- 使用PTW攻击方法可以大大减少所需IVs数量

**Q: 为什么获取不到WPA握手包？**

A: 常见原因：
1. 没有客户端连接到AP - 需要有在线客户端
2. Deauth包没有到达客户端 - 调整位置或增加发送数量
3. 客户端自动重连太快 - 增加Deauth包数量
4. 信号太弱 - 靠近AP或使用高增益天线

**Q: hashcat速度太慢怎么办？**

A: 提升破解速度的方法：
1. 使用GPU加速（比CPU快10-100倍）
2. 使用更有针对性的字典
3. 使用规则（rules）扩展字典
4. 使用分布式破解

### 27.4.1 Evil Twin（邪恶双胞胎）

Evil Twin是创建一个与目标AP同名的伪造AP，诱使用户连接。

#### 【Linux环境】使用airbase-ng创建Evil Twin

```bash

# 创建DHCP配置
cat > /etc/dhcp/dhcpd.conf << EOF
default-lease-time 600;
max-lease-time 7200;
subnet 192.168.1.0 netmask 255.255.255.0 {
```

### 27.4.7 常见问题

**Q: Evil Twin AP创建后用户连不上怎么办？**

A: 检查以下几点：
1. 确认DHCP服务器正常运行
2. 检查网卡是否支持AP模式
3. 确认没有IP冲突
4. 查看hostapd日志

**Q: DNS欺骗不生效怎么办？**

A: 可能的原因：
1. 客户端使用DNS over HTTPS (DoH)
2. 客户端缓存了DNS记录
3. ARP欺骗未成功
4. 防火墙拦截了DNS请求

**Q: SSL Strip对很多网站无效怎么办？**

A: 现代网站普遍使用HSTS，SSL Strip效果有限：
1. HSTS保护的网站无法降级
2. 只能针对未启用HSTS的网站
3. 可以配合DNS欺骗使用

### 27.7.6 常见问题

**Q: Proxmark3无法连接怎么办？**

A: 排查步骤：
```bash
# 检查USB连接
lsusb

# 检查串口设备
ls /dev/ttyACM*

# 检查权限
sudo chmod 666 /dev/ttyACM0

# 添加用户到dialout组
sudo usermod -a -G dialout $USER
```

**Q: ACR122U在Linux下无法使用怎么办？**

A: 可能的解决方法：
1. 卸载内核模块 `sudo modprobe -r pn533_usb pn533 nfc`
2. 黑名单相关模块
3. 检查pcscd服务状态
4. 确认驱动正确安装

**Q: 复制的门禁卡不能用怎么办？**

A: 可能的原因：
1. 不是普通的Mifare卡，使用了加密方案
2. 卡的UID被门禁系统绑定
3. 使用了非标准的扇区数据
4. 门禁系统使用了双向认证
