# 第17章 工控安全实验环境搭建

## 17.1 工控安全实验环境概述

### 17.1.1 什么是工控安全

工业控制系统（Industrial Control System，ICS）是国家关键基础设施的核心组成部分，广泛应用于电力、水务、石油化工、交通运输、制造业等领域。工控安全是指保护工业控制系统免受未经授权的访问、攻击、破坏或干扰，确保工业生产过程的安全、稳定和可靠运行。

工控系统与传统IT系统存在显著差异：
- **实时性要求高**：工业控制过程对响应时间有严格要求，延迟可能导致生产事故
- **可用性优先**：工控系统强调7×24小时不间断运行，可用性优先级高于保密性
- **生命周期长**：工控设备使用寿命长达10-20年，系统升级困难
- **协议特殊性**：使用Modbus、S7、DNP3等专用工业协议，缺乏安全设计
- **网络隔离性**：传统工控系统与互联网物理隔离，但随着工业互联网发展，边界逐渐模糊

## 17.2 OpenPLC工控仿真环境

### 17.2.1 OpenPLC简介

OpenPLC是一个开源的可编程逻辑控制器（PLC）项目，完全兼容IEC 61131-3标准，支持多种编程语言（梯形图、功能块图、结构化文本、指令表、顺序功能图）。OpenPLC可以运行在多种硬件平台上，也可以纯软件仿真运行，非常适合工控安全学习和研究。

**主要特点：**
- 完全开源免费
- 支持IEC 61131-3标准的5种编程语言
- 支持Modbus RTU/TCP协议
- 可在Windows、Linux、macOS上运行
- 支持多种硬件平台（Raspberry Pi、Arduino等）
- 提供Web管理界面

### 17.2.4 常见问题与排错

#### 【通用】常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| Web界面无法访问 | 服务未启动或端口被占用 | 检查服务状态：`./openplc status`；检查端口占用：`netstat -tlnp \| grep 8080` |
| Modbus连接失败 | 端口502需要root权限 | 使用sudo启动；或修改端口号；或设置capabilities |
| 程序编译失败 | 程序语法错误或依赖缺失 | 查看编译日志；检查程序语法；确保依赖完整 |
| 运行后无输出 | 程序逻辑错误或IO未配置 | 在线监控程序执行状态；检查IO映射配置 |
| Docker容器无法启动 | 端口被占用或权限不足 | 检查宿主机端口占用；使用 `--privileged` 参数 |

#### 【Linux环境】特殊问题

```bash
# 问题：编译时报错 "boost/version.hpp not found"
# 解决方法：安装Boost开发库
sudo apt install -y libboost-all-dev

# 问题：Modbus库找不到
# 解决方法：手动编译安装libmodbus
cd /tmp
wget https://github.com/stephane/libmodbus/releases/download/v3.1.10/libmodbus-3.1.10.tar.gz
tar -xzf libmodbus-3.1.10.tar.gz
cd libmodbus-3.1.10
./configure --prefix=/usr/local
make && sudo make install
sudo ldconfig
```

#### 【Docker环境】特殊问题

```bash
# 问题：容器内无法访问外部网络
# 解决方法：检查Docker网络配置
docker network ls
docker network inspect bridge

# 问题：端口已占用
# 解决方法：修改宿主机端口映射
docker run -d --name openplc2 \
  -p 8081:8080 \
  -p 503:502 \
  openplc/openplc:latest

# 问题：数据持久化
# 解决方法：挂载数据卷
docker run -d --name openplc \
  -p 8080:8080 \
  -p 502:502 \
  -v /path/to/data:/opt/openplc/data \
  --privileged \
  openplc/openplc:latest
```

## 17.3 SCADA仿真环境搭建（Ignition）

### 17.3.1 Ignition简介

Ignition是由Inductive Automation开发的一款工业应用平台，提供完整的SCADA（数据采集与监视控制系统）、HMI（人机界面）、MES（制造执行系统）等功能。Ignition具有跨平台、模块化、可扩展等特点，是学习SCADA系统安全的理想工具。

**主要特点：**
- 完整的SCADA/HMI功能
- 支持多种工业协议（Modbus、Siemens S7、OPC UA等）
- 基于Java的跨平台架构
- 提供免费的Maker版本（非商业用途）
- 支持Web端和移动端访问
- 内置数据库和报警系统

### 17.4.1 Modbus协议模拟

#### Modbus简介

Modbus是由Modicon（现为施耐德电气）在1979年发明的串行通信协议，是工业领域最广泛使用的协议之一。Modbus有多种变体：
- **Modbus RTU**：基于RS-232/RS-485串行链路，使用二进制编码
- **Modbus ASCII**：基于串行链路，使用ASCII字符编码
- **Modbus TCP**：基于以太网TCP/IP，端口号502

```bash
cat > /tmp/modbus_server.py << 'EOF'
#!/usr/bin/env python3
"""
Modbus TCP服务器示例
用于工控安全实验环境，模拟Modbus PLC设备
"""
from pymodbus.server import StartTcpServer
from pymodbus.datastore import ModbusSequentialDataBlock
from pymodbus.datastore import ModbusSlaveContext, ModbusServerContext
import threading
import time
import random

def update_datastore(context):
    """定期更新数据存储，模拟真实传感器数据"""
    slave_id = 1
    while True:
        try:
            # 模拟温度数据（20-30度波动）
            temp = int((20 + random.random() * 10) * 100)  # 扩大100倍存储
            context[slave_id].setValues(3, 0, [temp])       # 保持寄存器地址0
            
            # 模拟压力数据
            pressure = int((1.0 + random.random() * 0.5) * 1000)
            context[slave_id].setValues(3, 1, [pressure])
            
            # 模拟流量数据
            flow = int((50 + random.random() * 30) * 10)
            context[slave_id].setValues(3, 2, [flow])
            
            # 模拟DI输入状态（8个开关量）
            di_values = [random.randint(0, 1) for _ in range(8)]
            context[slave_id].setValues(2, 0, di_values)
            
            # 模拟DO输出状态
            do_values = [1, 0, 1, 1, 0, 1, 0, 0]
            context[slave_id].setValues(1, 0, do_values)
            
            time.sleep(2)
        except Exception as e:
            print(f"数据更新错误: {e}")

def run_server():
    # 创建数据块
    store = ModbusSlaveContext(
        di=ModbusSequentialDataBlock(0, [0]*100),   # 离散输入
        co=ModbusSequentialDataBlock(0, [0]*100),   # 线圈
        hr=ModbusSequentialDataBlock(0, [0]*100),   # 保持寄存器
        ir=ModbusSequentialDataBlock(0, [0]*100),   # 输入寄存器
    )
    
    context = ModbusServerContext(slaves=store, single=True)
    
    # 启动数据更新线程
    thread = threading.Thread(target=update_datastore, args=(context,))
    thread.daemon = True
    thread.start()
    
    # 启动服务器
    print("=" * 50)
    print("Modbus TCP Server starting...")
    print("监听地址: 0.0.0.0:502")
    print("按 Ctrl+C 停止服务器")
    print("=" * 50)
    StartTcpServer(context=context, address=("0.0.0.0", 502))

if __name__ == "__main__":
    run_server()
EOF
```

```bash
#!/usr/bin/env python3
"""
Modbus TCP客户端示例
用于测试连接Modbus服务器
"""
from pymodbus.client import ModbusTcpClient
from pymodbus.exceptions import ModbusException

# 配置连接参数
HOST = '127.0.0.1'  # 服务器地址
PORT = 502           # Modbus TCP端口

def main():
    # 连接到Modbus服务器
    print(f"正在连接到 {HOST}:{PORT}...")
    client = ModbusTcpClient(HOST, port=PORT)
    
    if not client.connect():

        return
    
    print("连接成功！")

    try:

        # 地址0，读取10个寄存器

        else:
            print(f"读取错误: {result}")
        
        print()
        
        # 读取输入寄存器（功能码04）
        print("读取输入寄存器 (功能码04)...")
        result = client.read_input_registers(address=0, count=5, slave=1)
        
        if not result.isError():
            print(f"输入寄存器[0-4]: {result.registers}")
        else:
            print(f"读取错误: {result}")
        
        print()
        
        # 读取线圈（功能码01）
        print("读取线圈状态 (功能码01)...")
        result = client.read_coils(address=0, count=8, slave=1)
        
        if not result.isError():
            print(f"线圈[0-7]: {result.bits[:8]}")
        else:
            print(f"读取错误: {result}")
        
        print()
        
        # 读取离散输入（功能码02）
        print("读取离散输入 (功能码02)...")
        result = client.read_discrete_inputs(address=0, count=8, slave=1)
        
        if not result.isError():
            print(f"离散输入[0-7]: {result.bits[:8]}")
        else:
            print(f"读取错误: {result}")
        
        print()
        
        # 写入单个寄存器（功能码06）
        print("写入保持寄存器 (功能码06)...")
        result = client.write_register(address=10, value=1234, slave=1)
        
        if not result.isError():
            print("写入成功！")
            # 验证写入
            result = client.read_holding_registers(address=10, count=1, slave=1)
            if not result.isError():
                print(f"验证写入: 寄存器[10] = {result.registers[0]}")
        else:
            print(f"写入错误: {result}")
        
        print("-" * 50)
        
    except ModbusException as e:
        print(f"Modbus异常: {e}")
    finally:
        # 关闭连接
        client.close()
        print("连接已关闭")

if __name__ == "__main__":
    main()
EOF
```

#### 【通用】S7 PLC模拟器

**操作位置：安装Python Snap7的任意系统**

**步骤1：创建S7模拟器脚本**

```bash
cat > /tmp/s7_simulator.py << 'EOF'
#!/usr/bin/env python3
"""
S7 PLC模拟器示例
用于工控安全实验环境，模拟西门子S7 PLC
"""
import snap7
from snap7.server import Server
import time
import random
import threading
import struct

def update_data(server):
    """定期更新数据块内容"""
    while True:
        try:
            # 更新DB1中的数据
            db_data = bytearray(256)
            
            # 模拟温度（REAL类型，字节偏移0）
            temp = 25.5 + random.random() * 5
            temp_bytes = struct.pack('>f', temp)
            db_data[0:4] = temp_bytes
            
            # 模拟压力（REAL类型，字节偏移4）
            pressure = 1.2 + random.random() * 0.3
            pressure_bytes = struct.pack('>f', pressure)
            db_data[4:8] = pressure_bytes
            
            # 模拟流量（REAL类型，字节偏移8）
            flow = 100 + random.random() * 50
            flow_bytes = struct.pack('>f', flow)
            db_data[8:12] = flow_bytes
            
            # 模拟液位（INT类型，字节偏移12）
            level = int(50 + random.random() * 30)
            level_bytes = struct.pack('>h', level)
            db_data[12:14] = level_bytes
            
            # 模拟开关状态（字节偏移14）
            # 模拟8个开关量
            switch_byte = 0
            for i in range(8):
                if random.random() > 0.5:
                    switch_byte |= (1 << i)
            db_data[14] = switch_byte
            
            # 写入DB1
            server.db_write(1, 0, db_data)
            
            time.sleep(1)
        except Exception as e:
            print(f"数据更新错误: {e}")

def main():
    # 创建S7服务器
    server = Server()
    server.start()
    
    print("=" * 50)
    print("S7 PLC Simulator starting...")
    print("监听地址: 0.0.0.0:102")
    print("Rack: 0, Slot: 1")
    print("按 Ctrl+C 停止服务器")
    print("=" * 50)
    
    # 注册数据块DB1，大小256字节
    db1 = bytearray(256)
    server.register_area(snap7.types.srvAreaDB, 1, db1)
    
    # 注册输入区（过程映像输入）
    input_data = bytearray(128)
    server.register_area(snap7.types.srvAreaPA, 0, input_data)
    
    # 注册输出区（过程映像输出）
    output_data = bytearray(128)
    server.register_area(snap7.types.srvAreaMK, 0, output_data)
    
    # 注册标记区（M寄存器）
    marker_data = bytearray(512)
    server.register_area(snap7.types.srvAreaTM, 0, marker_data)
    
    # 启动数据更新线程
    thread = threading.Thread(target=update_data, args=(server,))
    thread.daemon = True
    thread.start()
    
    try:
        while True:
            time.sleep(1)
    except KeyboardInterrupt:
        print("\n停止服务器...")
        server.stop()
        server.destroy()
        print("服务器已停止")

if __name__ == "__main__":
    main()
EOF
```

#### 【通用】S7客户端测试

**步骤1：创建S7客户端测试脚本**

```bash
cat > /tmp/s7_client.py << 'EOF'
#!/usr/bin/env python3
"""
S7 PLC客户端示例
用于测试连接S7 PLC
"""
import snap7
import struct

# 配置连接参数
HOST = '127.0.0.1'  # PLC地址
RACK = 0             # 机架号
SLOT = 1             # 槽号

def main():
    # 创建客户端
    print(f"正在连接到 {HOST} (Rack: {RACK}, Slot: {SLOT})...")
    client = snap7.client.Client()
    
    try:
        # 连接PLC
        client.connect(HOST, RACK, SLOT)
        
        # 检查连接状态
        if client.get_connected():
            print("连接成功！")
            print("-" * 50)
        else:
            print("连接失败！")
            return
        
        # 读取数据块DB1
        print("读取数据块 DB1...")
        db_data = client.db_read(1, 0, 256)
        
        # 解析REAL类型数据
        temp = struct.unpack('>f', db_data[0:4])[0]
        pressure = struct.unpack('>f', db_data[4:8])[0]
        flow = struct.unpack('>f', db_data[8:12])[0]
        
        # 解析INT类型数据
        level = struct.unpack('>h', db_data[12:14])[0]
        
        # 解析开关状态
        switch_byte = db_data[14]
        switches = [(switch_byte >> i) & 1 for i in range(8)]
        
        print(f"温度: {temp:.2f} °C")
        print(f"压力: {pressure:.3f} MPa")
        print(f"流量: {flow:.2f} m³/h")
        print(f"液位: {level} %")
        print(f"开关状态: {switches}")
        
        print("-" * 50)
        
        # 读取CPU状态
        cpu_state = client.get_cpu_state()
        print(f"CPU状态: {cpu_state}")
        
        # 读取库信息
        library_info = client.get_library_info()
        print(f"库版本: {library_info}")
        
        print("-" * 50)
        
        # 写入数据测试
        print("写入数据测试...")
        new_temp = struct.pack('>f', 30.0)
        client.db_write(1, 0, new_temp)
        print("温度已更新为30.0°C")
        
        # 重新读取验证
        db_data = client.db_read(1, 0, 4)
        new_temp_value = struct.unpack('>f', db_data[0:4])[0]
        print(f"验证: 温度 = {new_temp_value:.2f}°C")
        
    except Exception as e:
        print(f"错误: {e}")
    finally:
        # 断开连接
        client.disconnect()
        print("连接已关闭")

if __name__ == "__main__":
    main()
EOF
```

### 17.4.3 常见问题与排错

#### 【通用】Modbus常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 连接超时 | 防火墙阻止或服务器未启动 | 检查防火墙设置；确认Modbus服务器正在运行 |
| 数据读取异常 | 寄存器地址错误或功能码不匹配 | 检查地址偏移；注意Modbus地址从0开始 |
| 字节序问题 | 大端/小端字节序不一致 | Modbus使用大端序；解析时注意转换 |
| 连接被拒绝 | 端口未开放或权限不足 | 使用sudo运行；检查端口占用 |

#### 【通用】S7协议常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 连接被拒绝 | Rack/Slot配置错误 | 确认Rack=0, Slot=1；检查端口102是否开放 |
| 权限不足 | 需要管理员权限 | 使用sudo运行；或设置cap_net_bind_service |
| 读取数据失败 | DB块未注册或地址错误 | 确保DB块已注册；检查偏移地址 |
| 连接不稳定 | 网络问题或超时设置 | 增加超时时间；检查网络连接 |

#### 【Linux环境】Snap7特殊问题

```bash

# 问题：Permission denied

```
