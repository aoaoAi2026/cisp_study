# 第19章 电子取证分析环境搭建

## 19.1 电子取证概述

### 19.1.1 什么是电子取证

电子取证（Digital Forensics）是指利用计算机科学和技术手段，对电子设备中存储、传输或处理的数据进行获取、保存、分析和呈现，以提供法庭可采纳的证据的过程。电子取证是网络安全领域的重要分支，广泛应用于刑事案件调查、民事纠纷取证、企业内部调查、网络攻击溯源等场景。

**电子取证的基本原则：**
- **合法性原则**：取证过程必须符合法律法规要求
- **客观性原则**：如实反映客观事实，不主观臆断
- **关联性原则**：证据需与案件事实相关
- **完整性原则**：确保证据完整，不被篡改或遗漏
- **可重复性原则**：相同条件下可重复获得相同结果

**电子取证的主要特点：**
- **无形性**：电子证据以二进制形式存在，不可直接感知
- **易变性**：电子数据容易被修改、删除、伪造
- **多样性**：存储介质和文件格式多种多样
- **海量性**：数据量巨大，需要高效的分析工具
- **技术性**：需要专业的技术工具和知识

### 19.1.2 电子取证分类

根据取证对象和目的的不同，电子取证可分为以下几类：

| 取证类型 | 取证对象 | 主要内容 |
|---------|---------|---------|
| 计算机取证 | 台式机、笔记本、服务器 | 硬盘取证、文件分析、系统日志分析 |
| 移动设备取证 | 手机、平板、智能设备 | 通话记录、短信、APP数据、位置信息 |
| 网络取证 | 网络流量、日志 | 流量分析、入侵检测、攻击溯源 |
| 内存取证 | 物理内存、虚拟内存 | 进程分析、密码提取、恶意代码分析 |
| 数据库取证 | 数据库系统 | 数据恢复、操作日志、审计记录 |
| 物联网取证 | 智能设备、工控设备 | 设备日志、传感器数据、固件分析 |

### 19.1.3 取证流程

标准的电子取证流程包括以下阶段：

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 证据获取  │───▶│ 证据保存  │───▶│ 证据分析  │───▶│ 证据呈现  │
│ (Acquire)│    │ (Preserve)│    │ (Analyze) │    │ (Present) │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

1. **证据获取（Acquisition）**
   - 保护现场，防止数据被破坏
   - 记录设备状态和环境信息
   - 使用专业工具制作取证镜像
   - 计算哈希值验证完整性

2. **证据保存（Preservation）**
   - 对原始介质进行写保护
   - 制作工作副本，分析使用副本
   - 建立证据链，记录所有操作
   - 安全存储原始证据

3. **证据分析（Analysis）**
   - 文件系统分析
   - 应用程序数据分析
   - 网络活动分析
   - 时间线重建
   - 恶意代码分析

4. **证据呈现（Presentation）**
   - 整理分析结果
   - 生成取证报告
   - 以可视化方式呈现
   - 准备法庭出庭

### 19.1.4 实验环境要求

#### 硬件要求
- **CPU**：6核12线程以上，推荐8核以上
- **内存**：32GB以上，推荐64GB（内存取证需要大内存）
- **硬盘**：
  - 系统盘：256GB SSD以上
  - 数据盘：2TB以上（存储取证镜像）
  - 推荐使用独立硬盘存放证据
- **外设**：
  - 写保护设备（硬盘写保护锁）
  - 各类接口转换器（USB、SATA、M.2等）
  - 大容量移动硬盘

#### 软件环境
- **主系统**：Windows 10/11 或 Linux
- **取证专用系统**：Kali Linux、Parrot OS、FTK、CAINE
- **虚拟机**：VMware Workstation、VirtualBox
- **必备工具集**：见后续章节

## 19.2 Kali Linux取证工具集

### 19.2.1 Kali Linux简介

Kali Linux是基于Debian的Linux发行版，专为数字取证和渗透测试设计。Kali Linux预装了数百款安全工具，涵盖渗透测试、漏洞评估、逆向工程、数字取证等多个领域，是网络安全从业者的必备工具。

**取证工具分类：**
- **取证工具集（Forensics）**：Autopsy、Ghidra、Volatility等
- **取证启动镜像**：支持Live模式，不修改原始磁盘
- **哈希校验工具**：md5sum、sha256sum、hashdeep等
- **数据恢复工具**：testdisk、photorec、foremost等
- **网络取证工具**：wireshark、tcpdump、networkminer等

### 19.2.4 常用取证命令行工具

#### 【通用】哈希校验工具

**操作位置：终端**

**步骤1：计算文件哈希**

```bash
# 计算MD5哈希
md5sum evidence.img

# 计算SHA256哈希
sha256sum evidence.img

# 计算多种哈希
md5sum evidence.img > hash.txt
sha256sum evidence.img >> hash.txt
```

**预期输出：**
```

e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855  evidence.img
```

**步骤2：批量计算目录哈希**

```bash
# 使用hashdeep计算目录哈希
hashdeep -c md5,sha256 -l -r /path/to/evidence/ > hash_list.txt

# 验证哈希
hashdeep -a -k hash_list.txt /path/to/evidence/
```

**预期输出：**
```
hashdeep: audit passed
         file: evidence/file1.txt
         file: evidence/file2.txt
         ...
```

### 19.2.5 常见问题与排错

#### 【通用】常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 无法识别磁盘 | 权限不足或磁盘被占用 | 使用sudo；确保磁盘未挂载 |
| 镜像制作失败 | 磁盘有坏道或接口问题 | 使用conv=noerror,sync；检查数据线 |
| 工具找不到 | 未安装或PATH问题 | apt install安装；检查环境变量 |
| 中文显示乱码 | 编码设置问题 | 安装中文字体；设置locale |

#### 【Linux环境】特殊问题

```bash
# 问题：dd制作镜像速度太慢
# 解决方法：增加块大小
sudo dd if=/dev/sda of=evidence.dd bs=64M conv=noerror,sync

# 问题：磁盘只读无法读取
# 解决方法：检查硬件写保护开关；使用分区设备而非整盘
sudo hdparm -r0 /dev/sdb  # 尝试解除只读

# 问题：md5sum计算结果不一致
# 解决方法：确保使用相同的块大小；检查是否有坏道
```

### 19.3.4 常见问题与排错

#### 【Windows环境】常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 无法识别磁盘 | 权限不足或被系统占用 | 以管理员身份运行；关闭磁盘管理 |
| 镜像制作速度慢 | 磁盘性能问题或接口限制 | 使用USB 3.0以上接口；检查磁盘健康状态 |
| 镜像损坏 | 写入中断或目标磁盘问题 | 使用E01格式可部分恢复；检查目标磁盘空间 |
| 无法验证哈希 | 镜像文件损坏 | 检查文件是否完整传输 |

#### 镜像格式选择建议

```bash
# Raw (dd)
# 优点：兼容性好，读取速度快
# 缺点：文件大，不支持压缩
# 适用：快速获取，跨平台使用

# E01 (EnCase)
# 优点：支持压缩，可分卷，嵌入元数据
# 缺点：专有格式，其他工具兼容性差
# 适用：正式取证，证据存档

# AFF (Advanced Forensic Format)
```

## 19.5 Volatility内存取证

### 19.5.1 内存取证简介

内存取证（Memory Forensics）是指从计算机的物理内存（RAM）中提取和分析有价值信息的过程。内存中包含着系统运行时的大量敏感信息，如进程列表、网络连接、打开的文件、加密密钥、用户密码等，这些信息在硬盘上可能无法获取。

**内存取证的价值：**
- 获取运行中的进程信息
- 发现隐蔽的恶意代码（无文件攻击）
- 提取网络连接和套接字信息
- 恢复内存中的文件和数据
- 获取明文密码和加密密钥
- 分析内核级Rootkit
- 重建用户操作行为

### 19.6.4 网络取证分析流程

#### 【通用】常用分析命令

```bash
# 1. 统计协议分布
tshark -r capture.pcap -q -z io,phs

# 2. 提取HTTP请求
tshark -r capture.pcap -Y "http.request" -T fields \
    -e ip.src -e http.request.method -e http.host -e http.request.uri

# 3. 提取DNS查询
tshark -r capture.pcap -Y "dns.flags.response == 0" \
    -T fields -e ip.src -e dns.qry.name

# 4. 提取FTP登录凭据
tshark -r capture.pcap -Y "ftp.request.command == USER || ftp.request.command == PASS" \
    -T fields -e ip.src -e ftp.request.command -e ftp.request.arg

# 5. 提取所有HTTP对象
# Wireshark中：File → Export Objects → HTTP
```

**预期输出（DNS查询）：**
```

192.168.1.100    api.google.com
192.168.1.100    mail.qq.com
```
