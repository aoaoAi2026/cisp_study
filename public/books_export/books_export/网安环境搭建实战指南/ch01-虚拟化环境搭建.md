# 第一章 虚拟化环境搭建

## 1.1 环境概述

网络安全学习与实战离不开安全的实验环境。通过虚拟化技术，我们可以在一台物理机上搭建多台虚拟机，模拟真实的网络环境，进行各类安全实验而不影响生产系统。

本章将介绍主流虚拟化软件的安装配置，以及基础环境的准备工作。

---

## 1.2 VMware Workstation Pro 安装

> **适用操作系统：Windows / Linux**
>
> **说明：** 本节以 Windows 环境为例进行详细演示，Linux 环境安装步骤类似。

### 1.2.1 软件介绍

VMware Workstation Pro 是业界最流行的桌面虚拟化软件，支持在 Windows/Linux 主机上运行多个操作系统，具有强大的网络模拟能力和快照功能，是安全研究人员的首选工具。

### 1.2.2 系统要求

| 组件 | 最低要求 | 推荐配置 |
|------|----------|----------|
| 操作系统 | Windows 10 64位 / Linux 64位 | Windows 11 / Ubuntu 22.04 |
| CPU | 双核处理器，支持VT-x/AMD-V | 四核及以上，支持VT-x/AMD-V |
| 内存 | 4GB | 16GB及以上 |
| 硬盘 | 50GB可用空间 | 500GB SSD |
| 显卡 | 支持OpenGL 3.3 | 独立显卡 |

### 1.2.3 安装前准备：启用CPU虚拟化技术

**【Windows环境】**

#### 检查虚拟化是否已启用

**步骤1：打开任务管理器**

- 在 Windows 桌面上，右键点击任务栏空白处
- 在弹出菜单中选择 **"任务管理器"**

**步骤2：查看虚拟化状态**

- 在任务管理器界面，点击 **"性能"** 选项卡
- 左侧点击 **"CPU"**
- 观察右下角 **"虚拟化"** 项显示为 **"已启用"** 或 **"已禁用"**

**截图说明：** 任务管理器 -> 性能 -> CPU -> 查看右侧"虚拟化"状态

#### 如未启用，进入BIOS启用

**步骤1：重启电脑**

- 保存所有打开的工作
- 点击 **"开始"** 菜单 -> **"电源"** 按钮
- 按住 **Shift** 键同时点击 **"重启"**

**步骤2：进入BIOS设置**

- 计算机重启时，反复按下 **Del** 键 或 **F2** 键 或 **F10** 键（不同品牌主板按键不同）
- 截图说明：开机画面底部通常会显示 "Press DEL to enter setup" 等提示

**步骤3：在BIOS中找到虚拟化选项**

- 进入 BIOS 后，使用方向键切换到 **"Advanced"** 或 **"CPU Configuration"** 或 **"Security"** 选项卡
- 查找以下类似选项：
  - **Intel Virtualization Technology**（Intel CPU）
  - **AMD-V** 或 **SVM Mode**（AMD CPU）
- 截图说明：BIOS界面 -> Advanced选项卡 -> CPU Configuration -> Virtualization Technology

**步骤4：启用虚拟化**

- 使用方向键选中该选项
- 按 **Enter** 键，在弹出菜单中选择 **"Enabled"**
- 截图说明：选择 Enabled 选项

**步骤5：保存并退出**

- 按 **F10** 键保存设置
- 在确认对话框中选择 **"Yes"** 或 **"OK"**
- 计算机会自动重启

#### 验证虚拟化已启用

**打开 PowerShell（管理员）**

- 右键点击 **"开始"** 菜单
- 选择 **"终端管理员(Admin)"** 或 **"PowerShell (管理员)"**

**执行验证命令**

```powershell
systeminfo | findstr /i "虚拟化"
```

**预期输出**

```
Hyper-V 要求: 已安装虚拟化功能是
```

或在任务管理器的性能-CPU页面查看"虚拟化"显示"已启用"

**常见问题**

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 找不到虚拟化选项 | 主板品牌不同，位置不同 | 查阅主板说明书，或尝试在 Security -> Virtualization 中查找 |
| 按键无法进入BIOS | 时机不对或按键错误 | 关机后立即反复按Del/F2/F10，开机时屏幕底部有提示 |
| 虚拟化已启用但仍报错 | 可能需要关闭Hyper-V | 控制面板 -> 程序 -> 启用或关闭Windows功能 -> 取消勾选Hyper-V |

### 1.2.5 安装 VMware Workstation Pro

### 1.2.8 虚拟网络编辑器配置

**【Windows环境】**

#### 步骤1：打开虚拟网络编辑器

- 在 VMware 主界面，点击 **"编辑(E)"**
- 选择 **"虚拟网络编辑器(N)"**
- 截图说明：编辑菜单 -> 虚拟网络编辑器

#### 步骤2：获取管理员权限

- 点击 **"更改设置(C)"** 按钮（需要管理员权限）
- 截图说明：点击"更改设置"

#### 步骤3：配置VMnet0（桥接模式）

- 在网络列表中选择 **"VMnet0"**
- 勾选 ☑ **"桥接模式(B)"**
- 桥接到：**自动**（或选择特定网卡，如 Realtek PCIe ...）
- 点击 **"应用(A)"**
- 截图说明：VMnet0 -> 桥接模式 -> 桥接到自动 -> 应用

#### 步骤4：配置VMnet8（NAT模式）

- 在网络列表中选择 **"VMnet8"**
- 确认已勾选 ☑ **"NAT模式(N)"**
- 点击 **"NAT设置(S)..."** 查看网关IP
- 记录显示的 **"网关IP"**，如 `192.168.180.2`
- 点击 **"确定"**
- 截图说明：VMnet8 -> NAT设置 -> 查看网关IP -> 确定

#### 步骤5：配置DHCP

- 在网络列表中选择 **"VMnet8"**
- 点击 **"DHCP设置(D)..."**
- 查看并记录：
  - 起始IP地址：如 `192.168.180.128`
  - 结束IP地址：如 `192.168.180.254`
- 点击 **"确定"**
- 截图说明：VMnet8 -> DHCP设置 -> 查看IP范围 -> 确定

#### 步骤6：确认并退出

- 点击 **"确定"** 保存所有设置
- 截图说明：虚拟网络编辑器 -> 确定

**常见问题**

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| VMnet0桥接时找不到网卡 | 网卡未识别或驱动问题 | 更新网卡驱动，或手动选择网卡 |
| NAT模式下无法上网 | VMware服务未启动 | 开始菜单 -> 运行 -> services.msc -> 重启 VMware DHCP/NAT 服务 |
| VMnet8显示"已禁用" | VMnet8被关闭 | 在虚拟网络编辑器中点击"恢复默认" |

### 1.2.9 VMware Workstation 常见错误与解决

**【Windows环境】**

| 错误信息 | 原因 | 解决方法 |
|----------|------|----------|
| "VMware Workstation is not installed" | 安装损坏或不完整 | 控制面板 -> 程序和功能 -> 卸载VMware -> 重新安装 |
| "虚拟化技术未启用" | BIOS中未启用VT-x/AMD-V | 重启电脑，进入BIOS，启用虚拟化技术 |
| "网络连接不可用" | VMware桥接协议服务未运行 | 开始 -> 运行 -> services.msc -> 重启 "VMware Bridge Protocol" 服务 |
| "无法获取许可证" | 未输入有效密钥 | 选择"我有许可证密钥"输入，或使用免费试用版 |
| "此主机不支持64位客户机操作系统" | CPU不支持或未启用VT-x | 确认CPU支持64位虚拟化，并在BIOS中启用虚拟化 |
| 虚拟机运行缓慢 | 内存/CPU分配不足 | 编辑虚拟机设置，增加内存和CPU核心数 |
| 虚拟机黑屏 | ISO未正确挂载或损坏 | 检查虚拟机设置 -> CD/DVD -> 确认ISO路径正确 |

### 1.2.10 快照管理

**【Windows/Linux环境通用】**

#### 创建快照

- 关闭虚拟机或保持运行状态均可
- 在 VMware 主界面，选择目标虚拟机
- 点击 **"虚拟机"** 菜单 -> **"快照"** -> **"拍摄快照"**
- 截图说明：虚拟机菜单 -> 快照 -> 拍摄快照

- 在弹出窗口中填写：
  - 名称：如 `Win10-初始安装完成`
  - 描述：（可选）记录快照状态
- 点击 **"拍摄快照(S)"**

### 1.3.7 VirtualBox 虚拟机网络模式详解

| 模式 | 说明 | 虚拟机可访问 | 主机可访问 | 虚拟机间互通 |
|------|------|-------------|-----------|-------------|
| NAT | 共享主机IP上网 | 外网+主机 | 否 | 否 |
| NAT网络 | 多个虚拟机通过NAT上网 | 外网+主机 | 端口转发后 | 是 |
| 桥接网卡 | 获得独立IP直连网络 | 全部 | 是 | 是 |
| 内部网络 | 仅虚拟机间通信 | 仅内部虚拟机 | 否 | 是 |
| Host-Only | 仅与主机和同网络VM通信 | 主机+同网络VM | 是 | 是 |
| 通用驱动 | 使用第三方驱动 | - | - | - |

```bash
# 在Windows中访问Linux文件
```

```powershell
# 在PowerShell或文件管理器中
cd \\wsl$\Ubuntu-22.04\home\student
```

**从Linux访问Windows文件**

```bash
# 在WSL终端中，Windows C盘挂载在 /mnt/c
cd /mnt/c/Users/YourUsername
```

> **适用操作系统：Windows 10 64位专业版/企业版/教育版 / Windows 11**
>
> **说明：** Windows Home版需要通过WSL2运行Docker

### 1.5.1 软件介绍

Docker 是现代安全环境搭建的重要工具。很多靶场、漏洞环境和安全工具都提供Docker镜像，可以快速部署和管理。

### 1.5.2 系统要求

| 组件 | 最低要求 | 推荐配置 |
|------|----------|----------|
| 操作系统 | Windows 10 64位专业版/教育版/企业版（Home版需WSL2） | Windows 11 |
| CPU | 支持虚拟化（VT-x/AMD-V） | 多核处理器 |
| 内存 | 4GB RAM | 8GB RAM及以上 |
| 硬盘 | 16GB可用空间 | 50GB SSD |
| BIOS | 启用SLAT (Second Level Address Translation) | - |

### 1.5.3 Windows Home版准备（安装WSL2）

**【Windows环境】**

如果使用的是 Windows Home 版，需要先安装 WSL2：

```powershell
# 以管理员身份打开PowerShell
wsl --install
```

**预期输出**

```
正在下载: WSL2 内核更新包
正在安装: WSL2 内核更新包
安装成功!
下载: Ubuntu-22.04 LTS
正在安装: Ubuntu-22.04 LTS
安装成功!
```

安装完成后重启电脑，然后设置Ubuntu用户名和密码。

**常见问题**

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| "WSL2 installation is incomplete" | WSL2内核未更新 | 以管理员运行PowerShell，执行 `wsl --update` |
| "Hyper-V not installed" | Hyper-V未启用（专业版） | 控制面板 -> 程序 -> 启用或关闭Windows功能 -> 勾选Hyper-V |
| "Docker Desktop not starting" | 服务未启动 | 管理员PowerShell执行 `net stop com.docker.service && net start com.docker.service` |
| "Hardware assisted virtualization" | 虚拟化未启用 | 进入BIOS启用VT-x/AMD-V |
| 安装后报错"WSL 2 installation is incomplete" | WSL2内核问题 | 下载并安装 https://wslstorestorage.blob.core.windows.net/wslblob/wsl_update_x64.msi |

### 1.5.6 Docker Compose 安装

**【Windows/Linux环境通用】**

Docker Desktop for Windows 已自带 Docker Compose。如需单独安装：

```bash
# 下载Linux版Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 添加执行权限
sudo chmod +x /usr/local/bin/docker-compose

# 验证
docker-compose --version
```

**预期输出**

```
Docker Compose version v2.20.0
```

## 1.8 实验网络拓扑搭建

> **适用操作系统：VMware/VirtualBox通用**

### 1.8.1 简单实验网络

适合初学者的简单拓扑：

```
┌─────────────┐     ┌─────────────┐
│   Kali      │     │  Windows 10 │
│ (攻击机)    │─────│  (靶机)     │
│ 192.168.1.10│     │ 192.168.1.20│
└─────────────┘     └─────────────┘
       │                   │
       └───────────────────┘
          VMnet1 (仅主机模式)
```

#### 配置步骤

**步骤1：创建Kali虚拟机**

- 按照 1.7.1 节创建Kali虚拟机
- 网络适配器设置为 **"仅主机模式"**
- 选择 **"VMnet1"**
- 截图说明：虚拟机设置 -> 网络适配器 -> 仅主机模式 -> VMnet1

**步骤2：创建Windows 10虚拟机**

- 按照 1.6.1 节创建Windows 10虚拟机
- 网络适配器设置为 **"仅主机模式"**
- 选择 **"VMnet1"**
- 截图说明：虚拟机设置 -> 网络适配器 -> 仅主机模式 -> VMnet1

**步骤3：配置静态IP**

**Kali配置IP：**

```bash
sudo vim /etc/network/interfaces
```

添加内容：

```
auto eth0
iface eth0 inet static
address 192.168.1.10
netmask 255.255.255.0
```

```bash
sudo systemctl restart networking
```

**Windows 10配置IP：**

1. 打开 **"控制面板"** -> **"网络和共享中心"**
2. 点击 **"更改适配器设置"**
3. 右键点击 **"以太网"** -> **"属性"**
4. 双击 **"Internet协议版本4(TCP/IPv4)"**
5. 选择 **"使用下面的IP地址"**
   - IP地址：`192.168.1.20`
   - 子网掩码：`255.255.255.0`
   - 默认网关：留空（仅主机模式）
6. 点击 **"确定"**
7. 截图说明：Windows IP设置

**步骤4：验证连接**

**在Kali中：**

```bash
ping 192.168.1.20
```

**在Windows 10中：**

```powershell
ping 192.168.1.10
```

**预期输出：** 显示来自对方IP的回复

### 1.8.2 多层网络拓扑

模拟企业内网环境：

```
                    互联网/NAT
                        │
                   ┌────┴────┐
                   │  防火墙  │
                   │ pfSense │
                   └────┬────┘
                        │ 192.168.1.1
            ┌───────────┴───────────┐
            │ 192.168.1.0/24 (DMZ)  │
       ┌────┴────┐           ┌─────┴─────┐
       │ Web服务器│           │ DNS服务器 │
       │192.168.1.10│        │192.168.1.11│
       └─────────┘           └───────────┘
                        │
                   ┌────┴────┐
                   │  路由器  │
                   └────┬────┘
                        │ 192.168.2.1
            ┌───────────┴────────────┐
            │ 192.168.2.0/24 (内网)  │
       ┌────┴────┐            ┌─────┴─────┐
       │  AD域控  │            │  客户端PC  │
       │192.168.2.10│          │192.168.2.20│
       └─────────┘            └───────────┘
```

#### 实现方式

**步骤1：在VMware中创建多个虚拟网络**

1. 打开 **编辑** -> **虚拟网络编辑器** -> **更改设置**
2. 添加 **VMnet2**，设置为 **NAT模式**，子网 `192.168.1.0/24`
3. 添加 **VMnet3**，设置为 **仅主机模式**，子网 `192.168.2.0/24`
4. 点击 **应用** -> **确定**
5. 截图说明：虚拟网络编辑器 -> 添加VMnet2/3

**步骤2：安装pfSense防火墙**

1. 下载pfSense ISO：
```
https://www.pfsense.org/download/
```

2. 创建虚拟机：
   - 选择 **"FreeBSD"** 或 **"Other"** -> **"FreeBSD 12.x 64位"**
   - CPU：2核，内存：1GB
   - 硬盘：8GB

3. 添加第二块网卡：
   - 虚拟机设置 -> **"添加..."** -> **"网络适配器"**
   - 第一块：NAT（连接外网）
   - 第二块：自定义 **VMnet2**（连接DMZ）

4. 安装pfSense（按照向导安装）

5. 配置pfSense LAN IP：
   - WAN（em0）：DHCP自动获取
   - LAN（em1）：手动设置 `192.168.1.1/24`

**步骤3：配置多网卡虚拟机**

**例如Web服务器：**

1. 创建虚拟机，安装Ubuntu Server
2. 添加第二块网卡：
   - 第一块：自定义 **VMnet2**（DMZ）
   - 第二块：自定义 **VMnet3**（内网）
3. 配置IP地址：
   - eth0（DMZ）：`192.168.1.10/24`，网关 `192.168.1.1`
   - eth1（内网）：`192.168.2.10/24`

**步骤4：配置路由和防火墙规则**

在pfSense Web界面（https://192.168.1.1）：

1. **设置LAN规则：**
   - 允许内网访问DMZ的特定端口（如80, 443）
   - 禁止内网直接访问外网（可选）

2. **设置DMZ规则：**
   - 允许DMZ访问外网DNS（53）
   - 禁止DMZ访问内网
