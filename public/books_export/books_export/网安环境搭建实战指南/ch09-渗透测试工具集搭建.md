# 第9章 渗透测试工具集搭建

## 9.1 渗透测试工具概述

渗透测试（Penetration Testing）是通过模拟恶意黑客的攻击方法，来评估计算机系统安全性的一种方法。工欲善其事，必先利其器，掌握常用的渗透测试工具是每个安全从业者的必备技能。

### 9.1.1 渗透测试工具分类

根据功能不同，渗透测试工具大致可以分为以下几类：

1. **信息收集工具**：Nmap、Amass、theHarvester、Whois等
2. **漏洞扫描工具**：Nessus、OpenVAS、Nuclei等
3. **Web应用测试工具**：Burp Suite、sqlmap、Nikto、Wfuzz等
4. **密码攻击工具**：Hydra、John the Ripper、Hashcat、Medusa等
5. **漏洞利用框架**：Metasploit、Cobalt Strike等
6. **目录/文件扫描工具**：dirsearch、Gobuster、DirBuster等
7. **CMS专用工具**：WPScan、JoomScan、Droopescan等
8. **后渗透工具**：Mimikatz、Empire、Covenant等
9. **逆向工程工具**：IDA Pro、Ghidra、x64dbg等
10. **社会工程学工具**：Social Engineering Toolkit等

### 9.1.2 推荐系统环境

- **Kali Linux**：最流行的渗透测试发行版，内置大量安全工具
- **Parrot Security OS**：另一个优秀的安全发行版
- **BlackArch**：基于Arch Linux的渗透测试发行版
- **自定义环境**：在普通Linux发行版上手动安装所需工具

## 9.2 Nmap 端口扫描工具

### 9.2.1 Nmap简介

Nmap（Network Mapper）是一个免费开源的网络发现和安全审计工具，被称为"扫描器之王"。它可以用于：

- 主机发现
- 端口扫描
- 服务/版本检测
- 操作系统检测
- 漏洞扫描（配合NSE脚本）

**官方网站**：https://nmap.org/

Kali Linux默认已安装Nmap，如果没有可以使用以下命令安装：

```bash
# 验证安装

# 预期输出：Nmap version 7.94
```


### 9.2.3 Nmap基本使用

#### 1. 基础端口扫描

```bash
# 操作位置：终端
# 扫描单个主机
nmap 192.168.1.1
# 预期输出：显示主机端口状态

# 扫描多个主机
nmap 192.168.1.1 192.168.1.2

# 扫描网段
nmap 192.168.1.0/24

# 扫描IP范围
nmap 192.168.1.1-100
```

#### 2. 端口指定扫描

```bash
# 操作位置：终端
# 扫描指定端口
nmap -p 22,80,443 192.168.1.1

# 扫描端口范围
nmap -p 1-1000 192.168.1.1

# 扫描所有端口（65535个）
nmap -p- 192.168.1.1

# 扫描常用100个端口
nmap -F 192.168.1.1

# 扫描前1000个常用端口
nmap --top-ports 1000 192.168.1.1
```

#### 3. 扫描技术

```bash
# 操作位置：终端
# TCP SYN扫描（半开放扫描，需要root权限）
sudo nmap -sS 192.168.1.1
# 预期输出：显示开放的端口

# TCP Connect扫描（全连接扫描）
nmap -sT 192.168.1.1

# UDP扫描（较慢）
sudo nmap -sU 192.168.1.1

# Ping扫描（主机发现）
nmap -sn 192.168.1.0/24

# 无Ping扫描
nmap -Pn 192.168.1.1

# 综合扫描
sudo nmap -sS -sU -p T:1-1000,U:53,161 192.168.1.1
```

#### 4. 服务和版本检测

```bash
# 操作位置：终端
# 服务版本检测
nmap -sV 192.168.1.1
# 预期输出：显示服务名称和版本

# 详细的版本检测（强度0-9）
nmap -sV --version-intensity 5 192.168.1.1
```

#### 5. 操作系统检测

```bash
# 操作位置：终端
# 操作系统检测（需要root权限）
sudo nmap -O 192.168.1.1
# 预期输出：显示操作系统类型

# 全面扫描（OS+版本+脚本）
sudo nmap -A 192.168.1.1
```

#### 6. 主机发现

```bash
# 操作位置：终端
# 仅主机发现，不扫描端口
nmap -sn 192.168.1.0/24

# 不进行Ping，直接扫描
nmap -Pn 192.168.1.1
```

#### 7. NSE脚本使用

```bash
# 操作位置：终端
# 使用默认脚本扫描
nmap -sC 192.168.1.1

# 使用指定脚本
nmap --script=http-title 192.168.1.1

# 漏洞扫描
nmap --script=vuln -p 80,443 192.168.1.1
```

#### 8. 输出格式

```bash
# 操作位置：终端
# 标准输出到文件
nmap -oN scan.txt 192.168.1.1

# XML格式输出
nmap -oX scan.xml 192.168.1.1

# Grepable格式
nmap -oG scan.gnmap 192.168.1.1

# 输出所有格式
nmap -oA scan 192.168.1.1
```

### 9.2.4 Nmap常用脚本

```bash
# 操作位置：终端
# HTTP相关
nmap --script=http-enum 192.168.1.1        # 目录枚举
nmap --script=http-title 192.168.1.1        # 获取网页标题
nmap --script=http-robots.txt 192.168.1.1   # 检查robots.txt

# SMB相关
nmap --script=smb-enum-shares 192.168.1.1   # 枚举共享
nmap --script=smb-os-discovery 192.168.1.1  # SMB信息
nmap --script=smb-vuln-ms17-010 192.168.1.1 # 永恒之蓝检测

# SSH相关
nmap --script=ssh-auth-methods 192.168.1.1  # SSH认证方式
nmap --script=ssh-brute 192.168.1.1         # SSH暴力破解

# 数据库相关
nmap --script=mysql-info 192.168.1.1        # MySQL信息
nmap --script=ms-sql-info 192.168.1.1       # MSSQL信息

# 漏洞检测
nmap --script=vuln 192.168.1.1              # 漏洞扫描
nmap --script=vulscan/vulscan.nse 192.168.1.1  # 使用vulscan脚本
```

### 9.2.5 Nmap高级选项

```bash
# 操作位置：终端
# 分片数据包（躲避防火墙）
nmap -f 192.168.1.1

# 指定MTU
nmap --mtu 24 192.168.1.1

# 诱饵扫描
nmap -D RND:10 192.168.1.1

# MAC地址欺骗
nmap --spoof-mac Cisco 192.168.1.1

# 调整时间模板（T0-T5，T5最快）
nmap -T4 192.168.1.1

# 调整并行数
nmap --min-parallelism 100 192.168.1.1
```

### 9.2.6 常见问题

**问题1：Operation not permitted**

```
原因：SYN扫描、操作系统检测等需要root权限
解决方法：使用sudo运行nmap
```

**问题2：扫描速度太慢**

```
解决方法：
1. 调整时间模板：nmap -T4 192.168.1.1
2. 调整并行数：nmap --min-parallelism 100 192.168.1.1
3. 减少扫描端口范围
```

**问题3：Nmap输出乱码**

```
解决方法：设置编码 export LANG=C
```

## 9.3 Burp Suite Web安全测试工具

### 9.3.1 Burp Suite简介

Burp Suite是Web应用安全测试的行业标准工具，由PortSwigger公司开发。它包含了一系列功能强大的工具，能够执行各种Web安全测试任务。

**主要功能**：
- **Proxy**：HTTP/HTTPS代理，用于拦截和修改请求响应
- **Repeater**：重放请求，用于手动测试
- **Intruder**：自动化攻击，用于爆破、fuzzing等
- **Scanner**：漏洞扫描器（专业版）
- **Spider**：爬虫（专业版）
- **Decoder**：编解码工具
- **Comparer**：对比工具
- **Extender**：插件扩展

**官方网站**：https://portswigger.net/burp

### 9.3.2 Burp Suite版本

- **Community Edition（社区版）**：免费，功能有限
- **Professional Edition（专业版）**：付费，功能完整
- **Enterprise Edition（企业版）**：企业级自动化扫描

```
1. 确认代理设置为127.0.0.1:8080
2. 确认Burp Suite的Proxy Listeners已启用
3. 检查浏览器代理插件（如FoxyProxy）设置
```

## 9.4 sqlmap SQL注入工具

### 9.4.1 sqlmap简介

sqlmap是一个开源的自动化SQL注入工具，功能强大，支持多种数据库和注入技术。

**主要功能**：
- 自动检测SQL注入漏洞
- 支持多种数据库（MySQL、Oracle、PostgreSQL、SQL Server等）
- 支持多种注入技术（布尔盲注、时间盲注、报错注入、联合查询、堆叠查询）
- 数据提取、文件读写、命令执行
- 数据库枚举、数据表导出

**官方网站**：https://sqlmap.org/
**GitHub**：https://github.com/sqlmapproject/sqlmap

### 9.4.3 sqlmap基本使用

#### 1. 检测注入点

```bash
# 操作位置：终端
# 检测GET参数
sqlmap -u "http://example.com/page?id=1"

# 检测POST参数
sqlmap -u "http://example.com/login" --data="username=admin&password=123"

# 检测Cookie参数
sqlmap -u "http://example.com/page" --cookie="id=1"

# 检测User-Agent
sqlmap -u "http://example.com/page" --user-agent="*"

# 检测所有参数
sqlmap -u "http://example.com/page?id=1" --level 3 --risk 2
```

#### 2. 指定数据库类型

```bash
# 操作位置：终端
sqlmap -u "http://example.com/page?id=1" --dbms=mysql
sqlmap -u "http://example.com/page?id=1" --dbms=postgresql
sqlmap -u "http://example.com/page?id=1" --dbms=mssql
sqlmap -u "http://example.com/page?id=1" --dbms=oracle
```

#### 3. 获取信息

```bash
# 操作位置：终端
# 数据库版本
sqlmap -u "http://example.com/page?id=1" -b

# 当前数据库用户
sqlmap -u "http://example.com/page?id=1" --current-user

# 当前数据库名
sqlmap -u "http://example.com/page?id=1" --current-db

# 列出所有数据库
sqlmap -u "http://example.com/page?id=1" --dbs

# 列出指定数据库的表
sqlmap -u "http://example.com/page?id=1" -D database_name --tables

# 列出表的列
sqlmap -u "http://example.com/page?id=1" -D database_name -T table_name --columns

# 导出数据
sqlmap -u "http://example.com/page?id=1" -D database_name -T table_name -C "col1,col2" --dump
```

#### 4. 提权与文件操作

```bash
# 操作位置：终端
# 读取文件
sqlmap -u "http://example.com/page?id=1" --file-read="/etc/passwd"

# 写入文件
sqlmap -u "http://example.com/page?id=1" --file-write="/path/to/shell.php" --file-dest="/var/www/html/shell.php"

# 执行系统命令（MySQL）
sqlmap -u "http://example.com/page?id=1" --os-shell

# 执行SQL查询
sqlmap -u "http://example.com/page?id=1" --sql-shell
```

#### 5. 高级用法

```bash
# 操作位置：终端
# 使用代理
sqlmap -u "http://example.com/page?id=1" --proxy="http://127.0.0.1:8080"

# 指定线程数
sqlmap -u "http://example.com/page?id=1" --threads 10

# 延时设置
sqlmap -u "http://example.com/page?id=1" --delay 1

# 从Burp日志文件加载请求
sqlmap -l burp_log.txt

# 从请求文件加载
sqlmap -r request.txt

# 批量扫描
sqlmap -m urls.txt

# 自动跳过询问
sqlmap -u "http://example.com/page?id=1" --batch
```

#### 6. 注入技术指定

```bash
# 操作位置：终端
# 只使用布尔盲注
sqlmap -u "http://example.com/page?id=1" --technique=B

# 只使用时间盲注
sqlmap -u "http://example.com/page?id=1" --technique=T

# 使用联合查询+报错注入
sqlmap -u "http://example.com/page?id=1" --technique=UE

# 技术参数说明：
# - B: Boolean-based blind（布尔盲注）
# - E: Error-based（报错注入）
# - U: Union query（联合查询）
# - S: Stacked queries（堆叠查询）
# - T: Time-based blind（时间盲注）
# - Q: Inline queries（内联查询）
```

### 9.4.4 常用Tamper脚本

```bash
# 操作位置：终端
# 绕过空格过滤
sqlmap -u "URL" --tamper=space2comment

# 编码绕过
sqlmap -u "URL" --tamper=base64encode
sqlmap -u "URL" --tamper=charunicodeencode

# 绕过关键字过滤
sqlmap -u "URL" --tamper=equaltolike
sqlmap -u "URL" --tamper=apostrophemask

# 组合使用多个tamper
sqlmap -u "URL" --tamper="space2comment,equaltolike"

# 其他常用tamper：
# - between：替换 > 为 BETWEEN
# - charencode：URL编码
# - equaltolike：LIKE 替换 =
# - space2hash：空格替换为 ###
# - randomcase：随机大小写
# - charunicodeencode：Unicode编码
```

### 9.4.5 常见问题

**问题1：检测不到注入**

```
解决方法：
1. 提高level和risk等级：--level 3 --risk 2
2. 确认参数确实是动态的

4. 尝试使用--random-agent
```

## 9.13 工具使用注意事项

### 9.13.1 法律合规

1. **获得授权**：只对授权的目标进行测试

3. **验证结果**：自动化工具可能有误报，需要手动验证
4. **控制速度**：避免对目标造成拒绝服务

2. **手动测试**：不要过度依赖自动化工具
