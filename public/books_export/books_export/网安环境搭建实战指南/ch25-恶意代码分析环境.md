# 第25章 恶意代码分析环境

## 25.1 恶意代码分析环境概述

### 25.1.1 什么是恶意代码分析

恶意代码分析（Malware Analysis）是指对恶意软件（病毒、木马、蠕虫、勒索软件等）进行解剖和研究，以了解其功能、传播方式、攻击目标和清除方法的过程。

恶意代码分析主要分为以下几类：
- **静态分析（Static Analysis）**：不运行恶意代码，通过分析文件结构、字符串、反汇编代码等了解其功能
- **动态分析（Dynamic Analysis）**：在隔离环境中运行恶意代码，观察其行为（文件系统、注册表、网络、进程等）
- **代码分析（Code Analysis）**：反汇编/反编译恶意代码，深入理解其实现逻辑
- **内存分析（Memory Analysis）**：分析内存镜像，发现隐藏的恶意代码和注入行为

### 25.1.2 分析环境分类

| 环境类型 | 用途 | 工具 |
|---------|------|------|
| 沙箱环境 | 自动化动态分析 | Cuckoo Sandbox、ANY.RUN、Joe Sandbox |
| 动态分析 | 行为监控分析 | ProcMon、Process Explorer、Wireshark、RegShot |
| 静态分析 | 代码结构分析 | PEiD、IDA Pro、Ghidra、x64dbg |
| 在线沙箱 | 快速初步分析 | VirusTotal、Hybrid Analysis、微步沙箱 |
| 脱壳环境 | 加壳程序分析 | UPX、Exeinfo PE、x64dbg |
| 内存取证 | 内存镜像分析 | Volatility、Redline |

### 25.1.3 安全注意事项

⚠️ **重要：分析恶意代码存在风险，请务必遵守以下安全规则：**

1. **隔离环境**：务必在虚拟机或沙箱中分析，切勿在物理机上运行
2. **网络隔离**：分析时断开网络或使用仅主机模式
3. **快照备份**：分析前创建虚拟机快照，便于恢复
4. **专用工具**：使用专门的分析工具，不要使用日常办公环境
5. **法律合规**：仅分析授权样本，遵守相关法律法规
6. **防护软件**：在分析环境外部署防护，防止意外感染

### 25.1.4 环境要求

#### 硬件要求
- CPU：4核8线程以上（推荐8核16线程）
- 内存：16GB以上（推荐32GB）
- 硬盘：500GB以上可用空间（SSD更佳）
- 网络：支持虚拟机网络配置

#### 软件环境
- 虚拟化软件：VMware Workstation / VirtualBox
- 分析主机：Windows 10/11（主系统）
- 目标虚拟机：Windows XP/7/10、Linux
- 分析工具：各类静态/动态分析工具

### 25.2.6 常见问题

**Q1: Cuckoo虚拟机无法连接？**

A: 检查以下几点：
1. 确认虚拟机已启动并可ping通
2. 确认Agent已运行（检查启动项）
3. 检查防火墙是否阻止了连接
4. 确认虚拟机IP配置正确
5. 查看Cuckoo日志：`~/.cuckoo/log/cuckoo.log`

**Q2: 分析任务一直挂起？**

A: 可能原因：
1. 虚拟机资源不足，增加内存和CPU
2. 分析时间设置太短，增加timeout
3. 恶意代码在等待用户交互
4. 查看任务日志定位问题

### 25.3.1 Process Monitor（ProcMon）配置

**【Windows环境】Process Monitor配置**

**操作位置：Windows 虚拟机（分析机）**

**ProcMon简介：**
Process Monitor是微软官方提供的高级监视工具，可以实时监控文件系统、注册表、进程/线程活动和网络活动。

**安装步骤：**

```bash

# 访问微软官方下载页：
# https://docs.microsoft.com/en-us/sysinternals/downloads/procmon

# 或直接下载：
# https://download.sysinternals.com/files/ProcessMonitor.zip

# 解压到任意目录
# 例如：C:\Tools\ProcMon\
```

**预期输出：**
```
解压后文件：
ProcMon/
├── Procmon.exe        # 32位版本
├── Procmon64.exe      # 64位版本
├── Eula.txt
└── Procmon.chm        # 帮助文档
```

**基本使用：**

```bash
# 1. 启动ProcMon
# 双击 Procmon64.exe
# 接受许可协议

# 2. 开始/停止监控
# 工具栏上的放大镜图标：开始/停止捕获
# 或按 Ctrl+E

# 3. 清除显示
# 工具栏上的清除图标
# 或按 Ctrl+X

# 4. 保存日志
# File → Save...
# 选择保存格式：PML（原生格式）、CSV、XML
```

**过滤配置：**

```bash
# 添加过滤规则（Ctrl+L）
# 常用过滤条件：
# - Process Name：只监控特定进程
# - Path：只监控特定路径的文件/注册表
# - Operation：只监控特定操作类型
# - Result：只监控成功/失败的操作

# 示例：只监控notepad.exe的文件操作
# 1. Filter → Filter... (Ctrl+L)
# 2. 添加规则：
#    Process Name | is | notepad.exe | Include
#    Operation | is | WriteFile | Include
# 3. 点击"Add" → "Apply" → "OK"
```

**常用操作类型：**

| 操作类型 | 说明 |
|---------|------|
| Process Create | 进程创建 |
| Process Exit | 进程退出 |
| Thread Create | 线程创建 |
| Load Image | 加载DLL/驱动 |
| CreateFile | 创建/打开文件 |
| WriteFile | 写入文件 |
| ReadFile | 读取文件 |
| SetDispositionInformationFile | 删除文件 |
| RegCreateKey | 创建注册表项 |
| RegSetValue | 设置注册表值 |
| RegQueryValue | 查询注册表值 |
| TCP Send | TCP发送数据 |
| TCP Receive | TCP接收数据 |

**使用示例（分析恶意代码）：**

```bash

# 1. 启动ProcMon
```

### 25.3.2 Process Explorer使用

**【Windows环境】Process Explorer**

**操作位置：Windows 虚拟机（分析机）**

**Process Explorer简介：**
Process Explorer是Sysinternals提供的进程管理工具，可以看作是任务管理器的增强版，能够显示进程的详细信息、DLL加载、句柄等。

**安装步骤：**

```bash
# 下载Process Explorer
# 访问：https://docs.microsoft.com/en-us/sysinternals/downloads/process-explorer
# 下载解压到 C:\Tools\ProcExp\
```

**主要功能：**

```bash
# 1. 进程树视图
#    显示进程之间的父子关系
#    蓝色：当前用户进程
#    粉色：服务进程
#    黄色：新创建的进程
#    红色：已终止的进程

# 2. 查看进程DLL
#    选中进程 → View → Lower Pane View → DLLs (Ctrl+D)
#    查看进程加载的所有DLL文件

# 3. 查看进程句柄
#    选中进程 → View → Lower Pane View → Handles (Ctrl+H)
#    查看进程打开的文件、注册表、互斥量等

# 4. 进程详情
#    双击进程打开属性窗口
#    - Image：镜像路径、命令行参数
#    - Performance：CPU、内存、I/O统计
#    - Threads：线程列表
#    - TCP/IP：网络连接
#    - Strings：进程内存中的字符串

# 5. 替换任务管理器
#    Options → Replace Task Manager
#    以后按Ctrl+Shift+Esc就启动Process Explorer
```

**恶意代码分析技巧：**

```bash
# 1. 查找恶意进程
#    - 查看没有公司名称的进程
#    - 查看路径可疑的进程（临时目录、AppData）
#    - 查看伪装成系统进程的程序（路径不对）

# 2. 检查可疑DLL
#    - 查看进程加载的DLL
#    - 查找没有签名的DLL
#    - 查找路径可疑的DLL

# 3. 检测注入
#    - 查看进程加载的DLL是否合理
#    - 比如notepad.exe加载了网络相关的DLL就很可疑

# 4. 验证数字签名
#    - 选项中启用"Verify Signature"
#    - 查看进程和DLL是否有合法签名
```

**Wireshark简介：**
Wireshark是最流行的网络协议分析工具，可以捕获和分析网络流量，用于恶意代码的网络行为分析。

**安装步骤：**


**基本使用：**

```bash
# 1. 启动Wireshark
# 2. 选择捕获接口（网卡）
# 3. 点击开始捕获（鲨鱼鳍图标）

# 4. 常用过滤表达式：
#    - ip.addr == 192.168.1.100  特定IP
#    - tcp.port == 80            TCP端口
#    - http                      HTTP协议
#    - dns                       DNS协议
#    - ip.src == 192.168.1.100   源IP
#    - ip.dst == 192.168.1.200   目的IP
#    - http.request              HTTP请求
#    - http.response             HTTP响应
```

**恶意代码网络分析：**

```bash
# 分析步骤：
# 1. 启动Wireshark抓包
# 2. 运行恶意代码
# 3. 等待2-3分钟
# 4. 停止抓包
# 5. 分析流量

# 重点关注：
# - DNS查询：查找C2域名
# - HTTP/HTTPS请求：与C2通信
# - 异常端口：非标准端口的连接
# - 外发数据：数据窃取行为
# - 入站连接：后门/木马连接

# 常用过滤组合：
# 查看HTTP POST请求：
# http.request.method == "POST"

# 查看DNS查询：
# dns.flags.response == 0

# 查看所有HTTP流量：
# http

# 导出HTTP对象：
# File → Export Objects → HTTP...
```

**命令行工具Tshark：**

```bash
# 实时抓包
tshark -i eth0

# 捕获HTTP请求
tshark -i eth0 -Y 'http.request'

# 读取pcap文件
tshark -r capture.pcap

# 统计HTTP请求
tshark -r capture.pcap -Y http -T fields -e http.host -e http.request.uri
```

RegShot是一款注册表对比工具，可以快速比较两次快照之间注册表的变化，用于分析恶意代码对注册表的修改。

### 25.3.7 常见问题

**Q1: ProcMon日志太多找不到重点？**

A: 使用过滤技巧：
1. 先排除系统进程（svchost、explorer、services.exe等）
2. 根据进程名过滤（只看恶意进程及其子进程）
3. 根据操作类型过滤（只看WriteFile、RegSetValue等写操作）
4. 使用"Exclude Events Before/After"聚焦时间段
5. 使用Tools菜单下的Summary功能汇总

**Q2: Wireshark抓包看不到HTTPS内容？**

A: HTTPS流量默认是加密的，需要解密：
1. SSL/TLS解密：需要获取服务器私钥（不现实）
2. 使用代理工具（Fiddler/Burp）：客户端信任代理证书
3. 检查恶意代码是否忽略证书验证（常见于恶意代码）
4. 分析TLS握手信息（SNI、证书等）也能获得很多信息

4. 使用64位版本
5. 使用SSD硬盘

### 25.7.3 YARA规则编写

**YARA简介：**

**YARA规则基础：**

```
rule RuleName : Tag1 Tag2
{
    meta:
        description = "规则描述"
        author = "作者"
        date = "2024-06-25"
        version = "1.0"
        hash = "sha256hash值"
        category = "malware"

    strings:
        $string1 = "字符串1"
        $string2 = { 68 74 74 70 }  // 十六进制
        $string3 = /regex\d+/       // 正则表达式

    condition:
        2 of them  // 满足2个条件
        // 或 $string1 and $string2
        // 或 any of them
}
```

**示例规则：**

```yara
malware_emotet_strings : emotet banking
{
    meta:
        description = "Detects Emotet malware strings"
        author = "Analyst"
        date = "2024-06-25"
        severity = "high"

    strings:
        $c2_url = "http://*.com/connect" nocase wide
        $mutex = "Global\\EmotetMutex"
        $reg_key = "Software\\Microsoft\\Windows\\CurrentVersion\\Run\\Emotet"

        // 十六进制特征码
        $code = { 68 74 74 70 3A 2F 2F }  // "http://"

    condition:
        uint16(0) == 0x5A4D and  // PE文件
        (
            2 of ($c2_url, $mutex, $reg_key)
            or
            $code
        )
}
```

### 25.8.6 常见问题

**Q1: Volatility提示找不到符号/配置文件？**

A: 确保使用正确的配置文件和符号：
1. Volatility 3会自动检测系统版本
2. 确保内存镜像完整、格式正确
3. Windows需要对应的PDB符号
4. 可以使用--info查看支持的插件
5. 检查镜像格式（raw、elf、dmp等）

**Q2: 如何判断内存中是否有恶意代码？**

A: 检查以下几个方面：
1. 可疑进程：路径、名称、父进程异常
2. 可疑网络连接：连接恶意IP、异常端口
3. 可疑DLL：路径异常、无签名、注入到正常进程
4. 注册表启动项：新增的自启动项
5. 恶意注入：Malfind发现的可疑内存
6. Rootkit：隐藏的进程、驱动
7. 内存中的字符串：搜索IP、域名、密钥等
