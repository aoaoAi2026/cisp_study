## 16.1 环境概述

随着云计算的快速发展，云安全已成为网络安全领域的重要方向。搭建云安全实验环境，可以帮助学习者了解云平台架构、掌握云安全配置、学习云原生安全工具和技术。

本章将介绍云安全实验环境的搭建方法，包括主流云平台使用、容器安全、Kubernetes安全、云原生安全工具等内容。


**步骤2：安装依赖包**

> 操作位置：Linux终端

```bash
# 安装依赖
sudo apt install -y apt-transport-https ca-certificates curl gnupg lsb-release
```

---

**步骤3：添加Docker GPG密钥**

> 操作位置：Linux终端

```bash
# 添加Docker官方GPG密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
```

---

**步骤4：添加Docker仓库**

> 操作位置：Linux终端

```bash
# 添加Docker APT仓库
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

---

**步骤5：安装Docker**

> 操作位置：Linux终端

```bash
# 更新软件包列表
sudo apt update

# 安装Docker
sudo apt install -y docker-ce docker-ce-cli containerd.io

# 验证安装
docker --version
```

**预期输出：**
```
Docker version 24.x.x, build xxxxxxx
```

**步骤6：验证Docker运行**

> 操作位置：Linux终端

```bash
sudo systemctl enable docker

# 运行测试镜像
```

### 16.4.3 Docker安全配置

> 操作位置：Linux终端

```bash
# 创建配置目录
sudo mkdir -p /etc/docker

# 创建daemon.json
sudo nano /etc/docker/daemon.json
```

添加以下配置：

```json
{
    "icc": false,
    "live-restore": true,
    "no-new-privileges": true,
    "userland-proxy": false,
    "storage-driver": "overlay2",
    "log-driver": "json-file",
    "log-opts": {
        "max-size": "10m",
        "max-file": "3"
    }
}
```

---

**步骤3：逃逸方法1 - 写宿主机cron**

> 操作位置：容器内

### 16.4.6 Docker安全最佳实践

| 实践 | 说明 |
|------|------|
| 使用官方镜像 | 使用受信任的基础镜像 |
| 定期扫描漏洞 | 使用Trivy等工具扫描镜像 |
| 非root用户 | 以非root用户运行容器 |
| 最小化镜像 | 使用Alpine等最小化镜像 |
| 限制资源 | 使用--memory限制内存 |
| 禁用特权 | 避免使用--privileged |
| 最小化权限 | 移除不必要的权限 |
| 启用内容信任 | 设置DOCKER_CONTENT_TRUST=1 |
| 定期更新 | 更新Docker版本和镜像 |

```yaml

  name: restricted

  privileged: false
  allowPrivilegeEscalation: false
  requiredDropCapabilities:
    - ALL
```

## 16.11 常见问题汇总

### 【通用】云平台问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 账户被锁定 | 多次登录失败 | 通过邮箱或手机验证解锁 |
| 费用异常 | 免费额度用完或配置错误 | 检查账单，调整配置 |
| 无法创建资源 | 配额不足 | 申请提高配额 |
| 服务无法访问 | 区域不支持或配置错误 | 切换区域，检查配置 |

---

### 【Docker环境】容器问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 容器启动失败 | 镜像问题或配置错误 | 检查镜像和配置 |
| 网络不通 | 网络模式或防火墙问题 | 检查网络配置 |
| 磁盘空间不足 | 日志或数据占用过多 | 清理日志和数据 |
| 性能问题 | 资源限制过严 | 调整资源限制 |

---

### 【Kubernetes环境】K8s问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| Pod无法启动 | 镜像拉取失败或资源不足 | 检查镜像和资源配置 |
| Service无法访问 | selector不匹配 | 检查Service配置 |
| Ingress不工作 | 域名解析或证书问题 | 检查DNS和证书 |
| 存储挂载失败 | PV/PVC配置错误 | 检查存储配置 |
