# 第20章 综合攻防靶场搭建

## 20.1 攻防靶场概述

### 20.1.1 什么是攻防靶场

攻防靶场（Pentesting/Red Team Training Range）是专门用于模拟真实网络攻击和防御场景的训练环境。通过在靶场中进行各种安全演练，安全人员可以学习攻击技术、测试防御手段、提升应急响应能力，而无需担心对真实系统造成影响。

**靶场的主要用途：**
- **渗透测试学习**：学习各种渗透测试技术和工具
- **红蓝对抗演练**：模拟真实攻防对抗，提升团队协作能力
- **漏洞验证复现**：验证和复现已知漏洞，理解攻击原理
- **防御能力测试**：测试企业安全防护体系的有效性
- **应急响应演练**：模拟安全事件，训练应急响应流程
- **技能认证考核**：用于安全技能认证和人员考核

### 20.1.2 靶场架构分类

根据应用场景和复杂度，靶场可分为以下几类：

| 靶场类型 | 规模 | 复杂度 | 适用场景 |
|---------|------|--------|---------|
| 单机靶场 | 单台虚拟机 | 低 | 漏洞复现、工具学习 |
| 小型靶场 | 3-5台虚拟机 | 中 | Web渗透测试入门 |
| 中型靶场 | 10-20台虚拟机 | 高 | 综合渗透演练 |
| 大型靶场 | 20+虚拟机+网络设备 | 极高 | 红蓝对抗、攻防竞赛 |
| 云靶场 | 分布式集群 | 高 | 大规模演练、竞赛 |

### 20.1.4 实验环境要求

#### 硬件要求
- **CPU**：8核16线程以上，支持虚拟化（VT-x/AMD-V）
- **内存**：32GB以上（推荐64GB），多虚拟机环境需要更大内存
- **硬盘**：500GB以上可用空间（SSD更佳）
- **网络**：千兆网卡，推荐多网卡支持

#### 软件环境
- **宿主机系统**：Windows 10/11 Pro 或 Linux
- **虚拟化软件**：VMware Workstation Pro 16+ 或 VirtualBox
- **靶场镜像**：见各节详述
- **网络配置**：熟悉VMware网络虚拟化

### 20.2.1 Vulnstack靶场简介

Vulnstack（红日安全团队靶场）是一套基于真实企业网络环境搭建的综合渗透测试学习平台。Vulnstack靶场模拟了真实的企业内网环境，包含多层网络结构、多种漏洞场景，是国内最受欢迎的渗透测试学习靶场之一。

**靶场特点：**
- 真实企业网络环境模拟

## 20.3 Metasploit靶机环境

### 20.3.1 Metasploit Framework简介

Metasploit Framework（ MSF）是世界上最流行的开源渗透测试框架，提供了大量的漏洞利用模块、Payload生成器、辅助模块等。Metasploit支持Windows、Linux、macOS平台，是每个渗透测试工程师必须掌握的工具。

**主要功能：**
- 漏洞利用（Exploit）
- 漏洞扫描和验证
- Payload生成和传输
- 后渗透模块（Post-Exploitation）
- 辅助模块（Auxiliary）
- 社会工程学工具包（SET）

## 20.4 Vulhub Docker漏洞环境

### 20.4.1 Vulhub简介

Vulhub是一个基于Docker和Docker-compose的漏洞环境集合，包含了大量热门漏洞的真实复现环境。Vulhub环境开箱即用，学习者可以快速搭建各种漏洞环境进行学习和测试。

**特点：**
- 基于Docker，隔离安全
- 一键启动，部署简单
- 包含多种流行漏洞
- 持续更新
- 文档完善

### 20.4.2 Docker安装

#### 【Linux环境】Ubuntu/Debian安装Docker

**操作位置：终端**

**步骤1：更新软件包**

```bash
sudo apt update
sudo apt upgrade -y
```

**步骤2：安装依赖包**

```bash
sudo apt install -y apt-transport-https ca-certificates \
    curl gnupg lsb-release software-properties-common
```

**步骤3：添加Docker官方GPG密钥**

```bash
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
    sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
```

**预期输出：**
```
OK
```

**步骤4：设置Docker仓库**

```bash
echo "deb [arch=$(dpkg --print-architecture) \
    signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] \
    https://download.docker.com/linux/ubuntu \
    $(lsb_release -cs) stable" | \
    sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

**步骤5：安装Docker**

```bash
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io \
    docker-compose-plugin
```

**预期输出：**

## 20.5 ELK监控部署

### 20.5.1 ELK Stack简介

ELK Stack是由Elasticsearch、Logstash、Kibana三个开源工具组成的日志分析和监控解决方案，广泛应用于安全监控、运维分析、日志审计等领域。

**组件介绍：**
- **Elasticsearch**：分布式搜索引擎，用于存储和搜索日志数据
- **Logstash**：日志收集处理管道，支持收集、过滤、转发日志
- **Kibana**：可视化分析平台，用于图表展示和交互分析
- **Beats**：轻量级数据采集器（可选）

### 20.5.2 ELK安装部署

#### 【Docker】Docker Compose方式部署（推荐）

**操作位置：终端**

**步骤1：创建工作目录**

```bash
mkdir -p /opt/elk
cd /opt/elk
```

**步骤2：创建Docker Compose配置文件**

```bash
cat > /opt/elk/docker-compose.yml << 'EOF'
version: '3'
services:
  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.11.0
    container_name: elasticsearch
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
      - "ES_JAVA_OPTS=-Xms512m -Xmx512m"
    volumes:
      - esdata:/usr/share/elasticsearch/data
    ports:
      - "9200:9200"
      - "9300:9300"
    networks:
      - elk
    restart: unless-stopped

  logstash:
    image: docker.elastic.co/logstash/logstash:8.11.0
    container_name: logstash
    volumes:
      - ./logstash.conf:/usr/share/logstash/pipeline/logstash.conf:ro
    ports:
      - "5044:5044"
    networks:
      - elk
    depends_on:
      - elasticsearch
    restart: unless-stopped

  kibana:
    image: docker.elastic.co/kibana/kibana:8.11.0
    container_name: kibana
    environment:
      - ELASTICSEARCH_HOSTS=http://elasticsearch:9200
    ports:
      - "5601:5601"
    networks:
      - elk
    depends_on:
      - elasticsearch
    restart: unless-stopped

volumes:
  esdata:
    driver: local

networks:
  elk:
    driver: bridge
EOF
```

**步骤3：创建Logstash配置文件**

```bash
cat > /opt/elk/logstash.conf << 'EOF'
input {
  beats {
    port => 5044
  }
  tcp {
    port => 5000
  }
}

filter {
  if [message] =~ /^\{/ {
    json {
      source => "message"
      target => "parsed"
    }
  }
  date {
    match => [ "[@timestamp]", "ISO8601" ]
  }
}

output {
  elasticsearch {
    hosts => ["elasticsearch:9200"]
    index => "logs-%{+YYYY.MM.dd}"
  }
  stdout { codec => rubydebug }
}
EOF
```
