## 22.1 AD域环境详细搭建

### 22.1.1 【Windows环境】Windows Server 2019域控制器安装

#### 环境说明
- 操作系统：Windows Server 2019 Standard/Datacenter
- 主机名：DC01
- 域名：hacklab.local
- IP地址：192.168.1.10/24
- 网关：192.168.1.1

#### 操作位置：Windows Server 2019 服务器（本地或远程桌面）

#### 步骤1：设置静态IP地址

**操作位置：设置 → 网络和Internet → 以太网**

1. 打开"设置" → "网络和Internet" → "以太网"
2. 点击当前网络连接
3. 点击"IP设置"下的"编辑"按钮
4. 选择"手动"，启用IPv4
5. 配置如下：
   - IP地址：192.168.1.10
   - 子网前缀长度：24
   - 网关：192.168.1.1
   - 首选DNS：127.0.0.1

**或使用PowerShell配置：**

```powershell
# 以管理员身份运行PowerShell
New-NetIPAddress -InterfaceIndex (Get-NetAdapter).ifIndex -IPAddress 192.168.1.10 -PrefixLength 24 -DefaultGateway 192.168.1.1
Set-DnsClientServerAddress -InterfaceIndex (Get-NetAdapter).ifIndex -ServerAddresses "127.0.0.1"
```

**预期输出：**
```

InterfaceIndex    : 3
InterfaceAlias    : Ethernet
AddressFamily      : IPv4
Type             : Unicast
PrefixLength     : 24
PrefixOrigin     : Manual
SuffixOrigin     : Manual
AddressState     : Preferred
```

#### 步骤2：修改计算机名

**操作位置：系统属性**

1. 右键"此电脑" → "属性" → "更改设置" → "更改"
2. 计算机名输入：DC01
3. 点击"确定"，重启计算机

**或使用PowerShell：**

```powershell
Rename-Computer -NewName "DC01" -Restart
```

#### 步骤3：安装Active Directory域服务（图形化安装

**操作位置：服务器管理器**

1. 打开"服务器管理器"（Server Manager）
2. 点击"管理" → "添加角色和功能"
3. 点击"下一步"到"服务器角色"页面
4. 勾选"Active Directory域服务"
5. 在弹出的对话框中点击"添加功能"
6. 继续点击"下一步"直到"确认"页面

**预期输出：**

- zhaoliu（人事部）
- sqlservice（服务账户，技术部）

**PowerShell批量创建用户：**

```powershell
# 定义用户列表
$users = @(
    @{Name="zhangsan"; DisplayName="张三"; OU="OU=技术部,OU=公司总部,DC=hacklab,DC=local"; Password="P@ssw0rd123"},
    @{Name="lisi"; DisplayName="李四"; OU="OU=技术部,OU=公司总部,DC=hacklab,DC=local"; Password="P@ssw0rd123"},
    @{Name="wangwu"; DisplayName="王五"; OU="OU=财务部,OU=公司总部,DC=hacklab,DC=local"; Password="P@ssw0rd123"},
    @{Name="zhaoliu"; DisplayName="赵六"; OU="OU=人事部,OU=公司总部,DC=hacklab,DC=local"; Password="P@ssw0rd123"},
    @{Name="sqlservice"; DisplayName="SQL服务账户"; OU="OU=技术部,OU=公司总部,DC=hacklab,DC=local"; Password="P@ssw0rd123"}
)

# 批量创建用户
foreach ($user in $users) {
    $securePassword = ConvertTo-SecureString $user.Password -AsPlainText -Force
    New-ADUser -Name $user.Name `
        -DisplayName $user.DisplayName `
        -GivenName $user.DisplayName.Substring(0,1) `
        -Surname $user.DisplayName.Substring(1) `
        -SamAccountName $user.Name `
        -UserPrincipalName "$($user.Name)@hacklab.local" `
        -Path $user.OU `
        -AccountPassword $securePassword `
        -Enabled $true `
        -PasswordNeverExpires $true `
        -ChangePasswordAtLogon $false
    Write-Host "用户 $($user.Name) 创建成功"
}
```

**预期输出：**
```
用户 zhangsan 创建成功
用户 lisi 创建成功
用户 wangwu 创建成功
用户 zhaoliu 创建成功
用户 sqlservice 创建成功
```

### 22.1.6 【Windows环境】域内常见漏洞配置

#### AS-REP Roasting 漏洞配置

**漏洞原理：** 如果用户账户设置了"不需要Kerberos预身份验证"，攻击者可以请求该用户的AS-REP响应并离线破解密码。

**操作位置：ADUC（启用高级功能）**

1. 在ADUC中，点击"查看" → "高级功能"
2. 找到用户（如 zhangsan
3. 右键 → "属性" → "账户"选项卡
4. 在"账户选项"中勾选"不要求Kerberos预身份验证"
5. 点击"确定"

**PowerShell配置：**

```powershell
# 启用AS-REP Roasting配置
Set-ADAccountControl -Identity zhangsan -DoesNotRequirePreAuth $true

# 验证配置
Get-ADUser zhangsan -Properties DoesNotRequirePreAuth | Select-Object Name, DoesNotRequirePreAuth
```

**预期输出：**
```
Name     DoesNotRequirePreAuth
----     ---------------------
zhangsan                 True
```

#### Kerberoasting 漏洞配置

**漏洞原理：** 服务主体名称（SPN）的服务账户可以被请求服务票据（TGS），然后可以离线破解密码。

**配置SPN服务账户：**

```powershell
# 为sqlservice用户注册SPN
setspn -S MSSQLSvc/sql01.hacklab.local:1433 hacklab\sqlservice

# 验证SPN
setspn -L sqlservice
```

**预期输出：**
```
Registered ServicePrincipalNames for CN=SQL服务账户,OU=技术部,OU=公司总部,DC=hacklab,DC=local:
        MSSQLSvc/sql01.hacklab.local:1433
```

#### 委派攻击配置

**非约束委派配置：**

```powershell
# 配置用户或计算机账户启用非约束委派
Set-ADAccountControl -Identity sqlservice -TrustedForDelegation $true

# 验证
Get-ADUser sqlservice -Properties TrustedForDelegation | Select-Object Name, TrustedForDelegation
```

**约束委派配置：**

```powershell
# 配置约束委派（需要先设置SPN
Set-ADComputer -Identity "WEB01$" -PrincipalsAllowedToDelegateToAccount @("CIFS/fileserver.hacklab.local")

# 或使用msDS-AllowedToDelegateTo属性
Set-ADObject -Identity "CN=WEB01,OU=服务器组,OU=公司总部,DC=hacklab,DC=local" -Add @{"msDS-AllowedToDelegateTo"="CIFS/fileserver.hacklab.local"}
```

#### Silver Ticket 环境准备

**Silver Ticket利用需要：**
- 域SID（获取方法：
```powershell
# 获取域SID
Get-ADDomain hacklab.local | Select-Object DomainSID
```

**预期输出：**
```
DomainSID
---------
S-1-5-21-xxxxxxxxxx-xxxxxxxxxx-xxxxxxxxxx
```

- KRBTGT账户密码哈希（Golden Ticket需要）
- 服务账户密码哈希（Silver Ticket需要）

#### DCSync 漏洞配置

**漏洞原理：** 如果账户拥有复制目录更改权限，可以模拟域控制器复制权限，可以同步域内所有用户哈希。

**配置DCSync权限：**

```powershell
# 给用户添加DCSync权限（DC同步权限）
$user = "zhangsan"
$domain = "hacklab.local"
$distinguishedName = (Get-ADDomain).DistinguishedName

# 添加DS-Replication-Get-Changes
# 添加DS-Replication-Get-Changes-All
# 添加DS-Replication-Get-Changes-In-Filtered-Set

$account = New-Object System.Security.Principal.SecurityIdentifier (Get-ADUser $user).SID
$acl = Get-Acl "AD:\$distinguishedName"

$rights = [System.DirectoryServices.ActiveDirectoryRights]::GenericAll
$type = [System.Security.AccessControl.AccessControlType]::Allow
$inheritance = [System.DirectoryServices.ActiveDirectorySecurityInheritance]::None

$ace = New-Object System.DirectoryServices.ActiveDirectoryAccessRule($account, $rights, $type, $inheritance)
$acl.AddAccessRule($ace)
Set-Acl -Path "AD:\$distinguishedName" -AclObject $acl

Write-Host "已为用户 $user 添加DCSync权限"
```

### 22.1.7 常见问题

**Q1: 提升为域控时提示DNS错误？**
A: 确保服务器已设置静态IP且DNS指向自己（127.0.0.1）。如果是新林，安装过程会自动安装DNS。

**Q2: 无法打开Active Directory用户和计算机找不到？**
A: 需要先安装AD域服务角色和管理工具：
```powershell
Install-WindowsFeature RSAT-AD-PowerShell,RSAT-AD-AdminCenter
```

**Q3: 域用户无法登录？**
A: 1. 检查客户端DNS是否指向域控IP
2. 检查计算机是否已加入域
3. 检查用户名密码是否正确
4. 检查账户是否被禁用
