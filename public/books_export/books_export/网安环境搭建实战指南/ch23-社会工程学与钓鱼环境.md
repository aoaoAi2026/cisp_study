## 23.1 Gophish钓鱼平台

### 23.1.1 【Docker】Gophish安装

#### 环境说明
Gophish是一个开源的钓鱼平台，提供完整的钓鱼活动管理功能。

#### 操作位置：本地终端（Linux/macOS/Windows PowerShell）

**docker-compose.yml配置：**

```yaml
version: '3.8'

services:
  gophish:
    image: gophish/gophish:latest
    container_name: gophish
    ports:
      - "3333:3333"  # 管理界面
      - "8080:8080"  # 钓鱼页面
    volumes:
      - gophish_data:/opt/gophish
    restart: always

volumes:
  gophish_data:
```

**启动命令：**
```bash
docker-compose up -d
```

**预期输出：**
```
Creating network "gophish_default" with the default driver
Creating volume "gophish_gophish_data" with default driver
Creating gophish ... done
```

#### 手动安装（Linux）

**操作位置：Linux服务器终端**

```bash
# 下载Gophish
cd /opt
wget https://github.com/gophish/gophish/releases/download/v0.12.1/gophish-v0.12.1-linux-64bit.zip
unzip gophish-v0.12.1-linux-64bit.zip
cd gophish

# 修改配置文件
# 编辑 config.json，设置监听地址
cat config.json
```

**config.json配置示例：**
```json
{
	"admin_server": {
		"listen_url": "0.0.0.0:3333",
		"use_tls": true,
		"cert_path": "gophish_admin.crt",
		"key_path": "gophish_admin.key"
	},
	"phish_server": {
		"listen_url": "0.0.0.0:8080",
		"use_tls": false,
		"cert_path": "example.crt",
		"key_path": "example.key"
	},
	"db_name": "sqlite3",
	"db_path": "gophish.db",
	"migrations_prefix": "db/db_",
	"contact_address": "admin@example.com",
	"logging": {
		"filename": "",
		"level": ""
	}
}
```

**启动Gophish：**

```bash
chmod +x gophish
./gophish
```

**预期输出：**
```
time="2024-01-01T12:00:00Z" level=info msg="Starting gophish" version=0.12.1
time="2024-01-01T12:00:00Z" level=info msg="Creating new admin user: admin"
time="2024-01-01T12:00:00Z" level=info msg="Please login with the username admin and the password 随机密码"
```

**默认登录信息：**
- 管理地址：https://localhost:3333
- 用户名：admin
- 密码：首次启动时随机生成（查看终端输出）

### 23.1.6 常见问题

**Q1: Gophish管理后台无法访问？**
A: 检查防火墙是否开放3333端口，确保gophish服务正在运行。

**Q2: 邮件发送失败？**
A: 检查SMTP配置：
1. SMTP服务器地址和端口是否正确
2. 用户名密码是否正确
3. 是否需要SSL/TLS
4. 发件人地址是否与SMTP账号一致

**Q3: 钓鱼页面无法捕获密码？**
A: 确保：
1. 钓鱼页面中包含表单
2. 勾选了"Capture Submitted Data"和"Capture Passwords"
3. 表单字段名称正确

### 23.2.6 常见问题

**Q1: Evilginx2启动后无法访问？**
A: 检查：
1. 服务器防火墙是否开放80和443端口
2. 域名解析是否正确
3. SSL证书是否获取成功

**Q2: SSL证书获取失败？**
A: 确保：
1. 域名已正确解析到服务器IP
2. 80端口可访问（Let's Encrypt验证需要）
3. 没有其他服务占用80/443端口

**Q3: 无法捕获会话Cookie？**
A: 检查phishlet配置是否正确，确保目标网站支持中间人攻击。某些有额外安全措施的网站可能无法完全捕获。
