#### 【Linux环境】Postfix + Dovecot + Roundcube手动搭建

**操作位置：Linux 服务器终端**

**步骤1：安装基础软件包**

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install postfix dovecot-core dovecot-imapd dovecot-pop3d dovecot-mysql \
    roundcube roundcube-core roundcube-mysql roundcube-plugins \
    mysql-server mysql-client -y

# CentOS/RHEL
sudo yum install postfix dovecot dovecot-mysql roundcubemail mysql-server -y
```

**步骤2：配置Postfix**

```bash
# 编辑主配置文件
sudo nano /etc/postfix/main.cf
```

**配置内容：**
```
myhostname = mail.example.com
mydomain = example.com
myorigin = $mydomain
inet_interfaces = all
inet_protocols = ipv4
mydestination = $myhostname, localhost.$mydomain, localhost, $mydomain
home_mailbox = Maildir/

# SMTP认证
smtpd_sasl_type = dovecot
smtpd_sasl_path = private/auth
smtpd_sasl_auth_enable = yes
smtpd_sasl_security_options = noanonymous
smtpd_sasl_local_domain = $myhostname
smtpd_recipient_restrictions = permit_sasl_authenticated, permit_mynetworks, reject_unauth_destination

# TLS配置
smtpd_tls_cert_file = /etc/ssl/certs/ssl-cert-snakeoil.pem
smtpd_tls_key_file = /etc/ssl/private/ssl-cert-snakeoil.key
smtpd_use_tls = yes
smtpd_tls_auth_only = yes
```

**步骤3：配置Dovecot**

```bash
# 编辑Dovecot配置
sudo nano /etc/dovecot/dovecot.conf

# 启用协议
protocols = imap pop3 lmtp

# 配置认证
sudo nano /etc/dovecot/conf.d/10-auth.conf
disable_plaintext_auth = yes
auth_mechanisms = plain login

# 配置邮件存储
sudo nano /etc/dovecot/conf.d/10-mail.conf
mail_location = maildir:~/Maildir

# 配置SSL
sudo nano /etc/dovecot/conf.d/10-ssl.conf
ssl = yes
ssl_cert = </etc/ssl/certs/ssl-cert-snakeoil.pem
ssl_key = </etc/ssl/private/ssl-cert-snakeoil.key
```

**步骤4：配置Roundcube**

```bash
# 编辑Roundcube配置

$config['default_host'] = 'localhost';
$config['default_port'] = 143;
$config['smtp_server'] = 'localhost';
$config['smtp_port'] = 587;
$config['smtp_user'] = '%u';
$config['smtp_pass'] = '%p';
$config['support_url'] = '';
$config['product_name'] = 'Webmail';
```

**步骤5：创建测试用户**


## 30.2 邮件安全协议

### 30.2.1 SPF记录配置

#### 【通用】SPF记录原理

**SPF (Sender Policy Framework)** 是一种DNS TXT记录，用于验证发件人IP地址是否被授权发送该域名的邮件。

**SPF工作原理：**
```
收件服务器 → 查询域名SPF记录 → 对比发件人IP → 验证通过/失败
```

#### 【通用】SPF记录配置

**操作位置：DNS管理面板**

**SPF记录语法：**

```
v=spf1 [机制] [修饰符] ~all/-all/?all
```

**常用机制：**

| 机制 | 说明 | 示例 |
|-----|------|------|
| ip4 | IPv4地址或网段 | ip4:192.168.1.0/24 |
| ip6 | IPv6地址或网段 | ip6:2001:db8::/32 |
| a | 域名A记录 | a:mail.example.com |
| mx | 域名MX记录 | mx:example.com |
| include | 包含其他域名SPF | include:_spf.google.com |
| exists | 检查域名是否存在 | exists:%{i}.spf.example.com |

**结果修饰符：**

| 修饰符 | 说明 |
|-------|------|
| +all | 全部通过（不推荐） |
| ~all | 软失败（推荐测试用） |
| -all | 硬失败（推荐生产用） |
| ?all | 中性（不推荐） |

**SPF记录示例：**

```bash
# 基础SPF记录
v=spf1 mx a ip4:192.168.1.100 ~all

# 包含第三方邮件服务
v=spf1 include:_spf.google.com include:mailgun.org ~all

# 完整生产环境SPF
v=spf1 mx a ip4:1.2.3.4 ip4:1.2.3.5 include:_spf.example.com -all
```

**SPF检查：**

```bash
# 使用dig查询SPF记录
dig example.com TXT

# 使用nslookup查询
nslookup -type=TXT example.com

# 在线工具
# https://mxtoolbox.com/spf.aspx
# https://www.kitterman.com/spf/validate.html
```

**预期输出：**
```
; <<>> DiG 9.16.1 <<>> example.com TXT
;; ANSWER SECTION:
example.com.    300    IN    TXT    "v=spf1 mx a ip4:1.2.3.4 ~all"
```

### 30.2.2 DKIM签名配置

#### 【通用】DKIM原理

**DKIM (DomainKeys Identified Mail)** 通过在邮件头添加数字签名，让收件方验证邮件是否确实由声明的域名发送，且内容在传输过程中未被篡改。

**DKIM工作原理：**
```
发件服务器 → 用私钥签名邮件头和正文 → 收件服务器 → 查询DNS公钥 → 验证签名
```

#### 【Linux环境】OpenDKIM配置

**操作位置：Linux 服务器终端**

**安装OpenDKIM：**

```bash
# Ubuntu/Debian
sudo apt install opendkim opendkim-tools -y

# CentOS/RHEL
sudo yum install opendkim opendkim-tools -y
```

**配置步骤：**

```bash
# 生成DKIM密钥
sudo opendkim-genkey -s mail -d example.com

# 将密钥移动到配置目录
sudo mv mail.private /etc/opendkim/keys/example.com/mail.private
sudo chown opendkim:opendkim /etc/opendkim/keys/example.com/mail.private

# 查看公钥（需要添加到DNS）
cat /etc/opendkim/keys/example.com/mail.txt
```

**DNS记录内容：**
```
mail._domainkey.example.com. IN TXT "v=DKIM1; k=rsa; p=MIGfMA0GCSqGSIb3DQEBAQUAA4GNADCBiQKBgQC..."
```

**配置OpenDKIM主配置文件：**

```bash
sudo nano /etc/opendkim.conf
```

**配置内容：**
```
Domain                  example.com
KeyFile                 /etc/opendkim/keys/example.com/mail.private
Selector                mail
Socket                  inet:8891@localhost
PidFile                 /var/run/opendkim/opendkim.pid
Mode                    sv
SubDomains              no
```

**配置Postfix使用DKIM：**

```bash
sudo nano /etc/postfix/main.cf

# 添加以下内容
milter_default_action = accept
milter_protocol = 2
smtpd_milters = inet:localhost:8891
non_smtpd_milters = inet:localhost:8891
```

**重启服务：**

```bash
sudo systemctl restart opendkim
sudo systemctl restart postfix

# 设置开机自启
sudo systemctl enable opendkim
```

**DKIM验证：**

```bash
# 发送测试邮件到Gmail，查看邮件头
# 查找 "DKIM-Signature" 和 "Authentication-Results"

# 使用命令行验证
opendkim-testkey -d example.com -s mail -vvv
```

**预期输出：**
```

opendkim-testkey: key not secure

```

### 30.2.3 DMARC策略配置

#### 【通用】DMARC原理

**DMARC (Domain-based Message Authentication, Reporting & Conformance)** 建立在SPF和DKIM之上，定义了当邮件验证失败时收件方应采取的策略，并提供报告反馈机制。

**DMARC策略：**

| 策略 | 说明 |
|-----|------|
| none | 监控模式，只发送报告 |
| quarantine | 将失败邮件放入垃圾箱 |
| reject | 拒绝失败邮件 |

**DMARC工作流程：**
```
邮件到达 → SPF验证 → DKIM验证 → DMARC检查 → 执行策略 → 发送报告
```

#### 【通用】DMARC记录配置

**操作位置：DNS管理面板**

**DMARC记录语法：**

```
v=DMARC1; p=策略; pct=百分比; rua=mailto:报告地址; ruf=mailto:取证报告地址;
```

**DMARC记录示例：**

```bash
# 监控模式（推荐先使用）
v=DMARC1; p=none; rua=mailto:dmarc-reports@example.com;

# 隔离模式
v=DMARC1; p=quarantine; pct=50; rua=mailto:dmarc@example.com;

# 拒绝模式（生产环境）
v=DMARC1; p=reject; pct=100; rua=mailto:dmarc@example.com; ruf=mailto:dmarc-forensics@example.com; fo=1;
```

**常用标签：**

| 标签 | 说明 |
|-----|------|
| p | 主域名策略（none/quarantine/reject） |
| sp | 子域名策略 |
| pct | 应用策略的邮件百分比 |
| rua | 汇总报告接收地址 |
| ruf | 取证报告接收地址 |
| fo | 失败报告选项 |
| adkim | DKIM对齐模式（r/s） |
| aspf | SPF对齐模式（r/s） |

**DMARC检查：**

```bash
# 查询DMARC记录
dig _dmarc.example.com TXT

# 在线检查工具
# https://dmarcian.com/dmarc-inspector/
# https://mxtoolbox.com/DMARC.aspx
```

**渐进式部署DMARC建议：**
1. 第一阶段：p=none，收集数据
2. 第二阶段：p=quarantine，pct=10
3. 第三阶段：p=quarantine，pct=50
4. 第四阶段：p=quarantine，pct=100
5. 第五阶段：p=reject，pct=100

```bash
import dkim
import sys

def verify_dkim(email_file):
    with open(email_file, 'rb') as f:
        message = f.read()
    
    try:
        result = dkim.verify(message)
        if result:
            print("[+] DKIM验证通过")
```

```bash
cat > phish_check.py << 'EOF'
import re
import sys
from email import policy
from email.parser import BytesParser

def analyze_email(eml_file):
    with open(eml_file, 'rb') as f:
        msg = BytesParser(policy=policy.default).parse(f)
    
    print("=" * 60)
    print("钓鱼邮件分析报告")
    print("=" * 60)
    
    # 发件人信息
    print(f"\n[发件人信息]")
    print(f"From: {msg.get('From', 'N/A')}")
    print(f"Reply-To: {msg.get('Reply-To', 'N/A')}")
    print(f"Return-Path: {msg.get('Return-Path', 'N/A')}")
    
    # 认证结果
    print(f"\n[认证结果]")
    auth_results = msg.get('Authentication-Results', 'N/A')
    print(f"{auth_results}")
    
    # 检查可疑特征
    print(f"\n[可疑特征检测]")
    body = msg.get_body(preferencelist=('plain', 'html'))
    body_text = body.get_content() if body else ""
    
    suspicious_keywords = [
        'urgent', 'immediately', 'password', 'verify', 'confirm',
        'account', 'suspended', 'expire', 'click', 'link',
        '紧急', '立即', '密码', '验证', '确认', '账号', '停用', '过期'
    ]
    
    found_keywords = []
    for keyword in suspicious_keywords:
        if keyword.lower() in body_text.lower():
            found_keywords.append(keyword)
    
    if found_keywords:
        print(f"[!] 发现可疑关键词: {', '.join(found_keywords)}")
    else:
        print("[+] 未发现明显可疑关键词")
    
    # 检查URL
    urls = re.findall(r'https?://[^\s<>"]+', body_text)
    if urls:
        print(f"\n[URL分析]")
        for url in urls[:10]:
            print(f"  - {url}")
            # 检查是否使用IP地址
            if re.search(r'https?://\d+\.\d+\.\d+\.\d+', url):
                print(f"    [!] 可疑：使用IP地址")
    
    # 检查附件
    attachments = []
    for part in msg.iter_attachments():
        filename = part.get_filename()
        if filename:
            attachments.append(filename)
    
    if attachments:
        print(f"\n[附件]")
        for att in attachments:
            print(f"  - {att}")
            if att.endswith(('.exe', '.bat', '.vbs', '.ps1', '.docm', '.xlsm')):
                print(f"    [!] 可疑：可执行文件类型")

if __name__ == "__main__":
    if len(sys.argv) != 2:
        print(f"Usage: {sys.argv[0]} <email.eml>")
        sys.exit(1)
    analyze_email(sys.argv[1])
EOF

python3 phish_check.py suspicious_email.eml
```

```bash
#!/bin/bash
IP=$1
REV_IP=$(echo $IP | awk -F. '{print $4"."$3"."$2"."$1}')

RBL_LIST=(
    "sbl.spamhaus.org"
    "xbl.spamhaus.org"
    "pbl.spamhaus.org"
    "dnsbl.sorbs.net"
    "cbl.abuseat.org"
)

for rbl in "${RBL_LIST[@]}"; do
    result=$(dig +short $REV_IP.$rbl A)
    if [ -n "$result" ]; then
        echo "[LISTED] $IP is listed in $rbl ($result)"
    else
        echo "[OK] $IP not in $rbl"

done
EOF
```

## 常见问题

### Q1: 搭建邮件服务器需要哪些DNS记录？

**A:** 至少需要以下DNS记录：
1. **A记录**：mail.example.com → 服务器IP
2. **MX记录**：example.com → mail.example.com (优先级10)
3. **PTR记录**（反向DNS）：服务器IP → mail.example.com
4. **SPF记录**：TXT记录，v=spf1 mx a ~all
5. **DKIM记录**：mail._domainkey.example.com TXT
6. **DMARC记录**：_dmarc.example.com TXT

### Q2: 邮件被标记为垃圾邮件怎么办？

**A:** 排查和解决方法：
1. 检查SPF/DKIM/DMARC配置是否正确
2. 确保IP有反向DNS解析
3. 检查IP是否在RBL黑名单中
4. 优化邮件内容，避免垃圾邮件关键词
5. 提高邮件域名信誉
6. 申请加入白名单
7. 使用邮件送达率测试工具检测

### Q3: 如何防止SMTP用户枚举？

1. 禁用VRFY命令
2. 禁用EXPN命令
3. 限制RCPT TO错误次数
4. 启用速率限制
5. 使用失败延迟响应
6. 部署WAF/IDS
7. 监控异常访问日志

### Q4: 钓鱼邮件防护有哪些最佳实践？

**A:** 多层防御策略：
1. **技术层**：SPF/DKIM/DMARC、邮件网关、URL过滤、附件沙箱
2. **用户层**：安全培训、钓鱼演练、多因素认证
3. **流程层**：报告机制、应急响应、定期评估
4. **管理层**：安全策略、资源投入、持续改进

### Q5: Exchange漏洞环境搭建需要注意什么？

3. 不要暴露到公网
4. 定期重置环境
5. 遵守法律法规
6. 记录所有操作
7. 使用vulhub等合法测试环境

### Q6: 如何检测邮件头注入漏洞？

**A:** 检测方法：
1. 在所有用户输入处注入换行符
2. 检查邮件头是否被修改
3. 测试各种编码方式（URL编码、Base64等）
4. 尝试注入额外的头字段
5. 测试MIME注入
6. 使用自动化扫描工具

### Q7: SpamAssassin评分如何调整？

**A:** 调整建议：

2. 根据实际情况调整各规则的分数
3. 设置合理的阈值（通常5.0）
4. 使用白名单减少误报
5. 训练贝叶斯过滤器
6. 定期 review 垃圾邮件和正常邮件
