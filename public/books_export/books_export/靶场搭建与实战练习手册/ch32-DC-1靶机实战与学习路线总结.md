### 32.2.2 导入VMware

下载解压完了，接下来就是导入VMware了。

导入方法跟我们之前导入Jangow、Kioptrix是一样的：

1. 打开VMware Workstation
2. 点击左上角的"文件" → "打开"
3. 选择你下载的DC-1的 `.ova` 文件
4. 给虚拟机起个名字（比如"DC-1"），选个存放位置
5. 点击"导入"，等待导入完成

> ⚠️ **注意：**
>
> 导入的时候可能会弹出一个"导入失败"或者"OVF规范问题"的提示，别慌，点"重试"或者"忽略"就行，一般都能正常导入。
>
> 这就像你下载了一个压缩包，解压的时候提示"CRC错误"，但其实大部分文件都是好的，不影响用。

### 32.2.3 网络设置（重要！）

导入完成后，**先别急着开机！** 先检查一下网络设置。

跟之前的靶机一样，我们推荐使用**NAT模式**，这样Kali和靶机能在同一个局域网里，方便我们攻击。

检查方法：
1. 在VMware里选中DC-1虚拟机
2. 点击"编辑虚拟机设置"
3. 找到"网络适配器"
4. 确认网络连接是"NAT模式"
5. 如果不是，改成NAT模式
6. 点击"确定"保存

> 🔔 **为什么用NAT模式？**
>
> 就好比你家有个路由器，你的电脑、手机、平板都连在这个路由器上，互相之间可以访问，但外面的人访问不进来。
>
> NAT模式就是这样，Kali和靶机都在VMware的"虚拟路由器"下面，互相能访问，但不会影响外面的真实网络，既安全又方便。🏠

### 32.2.4 开机，找靶机IP

好了，现在可以开机了！

点击"开启此虚拟机"，等待靶机启动。启动完成后，你会看到一个登录界面，但是我们不需要登录，我们要做的是——**找到靶机的IP地址**。

怎么找呢？用我们的老朋友 `arp-scan` 或者 `netdiscover`。

打开你的Kali Linux，输入命令：

```bash
arp-scan -l
```

或者：

```bash
netdiscover
```

这两个命令就像"网络雷达"，会扫一遍你的局域网，把所有在线的设备都列出来。📡

扫完之后，你会看到一个列表，里面有：
- 你的Kali的IP
- 你的物理机的IP（可能有）
- 还有一个就是DC-1靶机的IP

怎么分辨哪个是DC-1呢？看MAC地址的厂商信息，一般是"VMware, Inc."之类的。如果你不确定，可以多记几个可疑的，后面用nmap扫一下就知道了。

> 🤔 **新手常见问题：扫不到靶机怎么办？**
>
> 别慌，按这个 checklist 检查：
> 1. 靶机开了吗？没开肯定扫不到啊
> 2. 都是NAT模式吗？Kali和靶机要在同一个网络模式下
> 3. 等一会儿，靶机刚启动可能网络还没准备好
> 4. 重启一下靶机试试
> 5. 把靶机的网络适配器移除了再重新加一个
>
> 网络问题是新手最常遇到的问题，别灰心，多试几次就好了。

### 32.2.5 验证连通性

找到靶机IP之后，我们先ping一下，看看能不能通：

```bash
ping 靶机IP
```

如果能收到回复，说明网络是通的，我们可以开始了！🎉

如果ping不通，那就要回去检查网络设置了。

那我们开始吧！

### 32.3.1 Nmap端口扫描

信息收集的第一步，永远是**端口扫描**。

用什么扫？当然是Nmap啦！这可是渗透测试界的"敲门神器"。🚪

输入命令：

```bash
nmap -sV -sC -p- 靶机IP
```

> 🔍 **参数解释：**
> - `-sV`：探测服务版本号（Version）
> - `-sC`：使用默认脚本扫描（Script）
> - `-p-`：扫描所有65535个端口（全端口扫描）
>
> 为什么要全端口扫描？因为有时候管理员会把服务开在不常见的端口上，只扫默认端口可能会漏掉。
>
> 就像你找东西，不能只找抽屉，柜子里、床底下、沙发缝里都得找找，对吧？🛋️

全端口扫描可能会慢一点，耐心等一下。喝口水，伸个懒腰，马上就好。☕

那DC-1上都开了哪些端口呢？大概是这样的（你的结果可能略有不同，但大差不差）：

| 端口 | 服务 | 版本 | 说明 |
|------|------|------|------|
| 22 | SSH | OpenSSH 6.0p1 | 远程登录 |
| 80 | HTTP | Apache 2.2.22 | 网站服务 |
| 111 | rpcbind | 2-4 | 远程过程调用 |
| 43235 | 其他 | - | 一些其他服务 |

嗯……开的端口不算多。重点关注哪几个呢？

- **22端口（SSH）**：可以试试爆破，但一般来说SSH爆破成功率不高，先放一放
- **80端口（HTTP）**：网站服务，这是重点！一般来说Web层面漏洞最多
- **111端口**：rpcbind，这个服务一般漏洞不多，暂时先不管

> 💡 **思路整理：**
>
> 端口不多，说明攻击面不大。但没关系，我们从最有可能的80端口入手，先看看网站上有什么。
>
> 这就像你去别人家偷东西（又打比方了哈哈），门窗都看了一遍，发现门锁得很严实，窗户好像没关紧，那你肯定先从窗户入手嘛。🪟

### 32.3.2 Web信息收集：看看网站长啥样

好，我们来看看80端口的网站。

打开浏览器，输入靶机IP，访问一下：

```
http://靶机IP
```

哇！你会看到一个看起来还挺不错的网站，对吧？有图片、有文字、有导航栏……看起来像是一个企业官网或者个人博客。

但是！细心的你有没有发现什么？——这个网站用的是一个**CMS**！

什么是CMS？CMS就是"内容管理系统"（Content Management System），简单说就是别人已经帮你写好的网站程序，你直接拿来用就行，不用自己从头写代码。

我们之前学过的WordPress、Discuz!，都是CMS。那这个网站用的是什么CMS呢？

仔细看看网页的底部，或者看看网页的源代码，你可能会看到一些线索。比如：
- 网页底部写着"Powered by Drupal"
- 网页源代码里有 `sites/default/` 这种路径
- 登录页面的URL是 `/user/login`

如果你看到了这些线索，那恭喜你——这个网站用的是 **Drupal** CMS！🎉

> 🤔 **Drupal是什么？**
>
> Drupal是国外一个非常有名的开源CMS，跟WordPress、Joomla并称"世界三大CMS"。
>
> 打个比方：
> - WordPress就像QQ空间，简单易用，新手也能快速上手
> - Drupal就像高级定制西装，功能强大，灵活度高，但相对复杂一点
> - Joomla介于两者之间
>
> Drupal虽然没有WordPress那么流行，但也是一个非常重要的CMS，很多政府、企业的网站都用它。🏛️

### 32.3.3 确定Drupal版本

知道了用的是Drupal，接下来我们要搞清楚——**具体是什么版本？**

为什么版本号这么重要？因为**漏洞是跟版本挂钩的**！

就像你知道了一个人的年龄，就能大概猜到他可能得什么病。知道了CMS的版本，你就能去查这个版本有没有什么已知漏洞。

那怎么看Drupal的版本呢？有好几种方法：

**方法一：看网页源代码**
> 在网页上右键 → 查看网页源代码，然后搜索 "Drupal" 或者 "version"，说不定能找到版本号。
>
> 这就像看一个人的身份证，上面写着出生日期，你一算就知道年龄了。🪪

**方法二：访问CHANGELOG.txt**
> Drupal有个习惯，会在根目录放一个 `CHANGELOG.txt` 文件，记录每个版本更新了什么。
>
> 我们直接访问：
> ```
> http://靶机IP/CHANGELOG.txt
> ```
> 如果能看到，那版本号就一目了然了！

**方法三：用专门的扫描工具**
> 比如 `droopescan`，这是一个专门扫Drupal的工具，能扫出版本、插件、主题什么的。
>
> 就像医院里的CT机，一照就知道你身体里有什么问题。🏥

我们来试试方法二，直接访问 `CHANGELOG.txt`。

打开浏览器，输入：

```
http://靶机IP/CHANGELOG.txt
```

哎？能看到吗？如果能看到的话，你会发现DC-1用的是 **Drupal 7.x** 版本（具体是7.22还是7.24我有点记不清了，反正都是7.x系列）。

好的，现在我们知道了：
- 靶机开了80端口
- 网站用的是Drupal CMS
- 版本是7.x

那接下来，我们该干什么呢？——当然是去查一下 **Drupal 7.x 有没有什么已知漏洞** 啊！😏

## 32.4 漏洞发现与Getshell

### 32.4.1 Drupal 7.x有什么漏洞？

Drupal 7.x，这可是个"赫赫有名"的版本啊！为什么这么说？因为它有一个超级有名的漏洞——**Drupalgeddon**！

什么是Drupalgeddon？我给大家讲讲：

Drupalgeddon是Drupal 7.x版本的一个**远程代码执行漏洞**（RCE），CVE编号是 **CVE-2018-7600**，Drupal官方编号是 **SA-CORE-2018-002**。

这个漏洞有多严重呢？这么说吧——
> 只要网站用了有漏洞的Drupal 7.x版本，黑客不需要任何账号、不需要任何权限，**直接就能在服务器上执行任意命令**，想干什么就干什么！

就相当于你家大门敞开着，谁都能进来，而且进来之后想拿什么就拿什么——你说可怕不可怕？😱

这个漏洞在2018年披露的时候，可是轰动一时的，全世界有几十万的网站受影响。因为Drupal用的人太多了，而且漏洞危害又大，所以被称为"Drupalgeddon"（Drupal + Armageddon，意思是"Drupal世界末日"）。

巧了不是？我们的DC-1用的正好就是Drupal 7.x，而且是有漏洞的版本！

那这就好办了——我们直接利用这个漏洞，不就能拿到Shell了吗？😎

### 32.4.2 怎么利用这个漏洞？

知道了有漏洞，那怎么利用呢？有两种方法：

**方法一：用Metasploit（MSF）**
> 简单粗暴，一键搞定，新手强烈推荐！
>
> 就像你买了个现成的外卖，打开就能吃，不用自己做饭。🍔

**方法二：手动打POC**
> 自己构造exp去打，稍微麻烦一点，但是能学到更多东西。
>
> 就像自己买菜做饭，虽然麻烦，但能学到厨艺。👨‍🍳

作为入门教程，我们今天重点讲**第一种方法**——用Metasploit来利用这个漏洞。

等你以后水平高了，再去研究第二种方法也不迟。

### 32.4.3 实战：用MSF打Drupalgeddon

好了，理论讲得差不多了，我们开始实战！大家跟着我一步一步来。

#### 第一步：打开Metasploit

首先，打开你的Kali Linux，在终端里输入：

```bash
msfconsole

#### 再试试SUID提权

好了，现在我们可能是flag4用户，也可能还是www-data用户（如果密码复用没成功的话）。

不管怎样，我们的目标是**root**！

接下来，我们试试另一种提权方法——**SUID提权**。

什么是SUID？我再用大白话讲一遍：
> 有些程序，不管是谁运行它，都能以程序所有者的权限运行。
>
> 比如，如果一个程序的所有者是root，而且设置了SUID位，那么普通用户运行这个程序的时候，就会临时获得root权限。
>
> 这就像——你是个普通人，但是你拿到了老板的印章，你盖了章的文件就有老板的效力。📎

那怎么找SUID文件呢？用这个命令：

```bash
find / -perm -u=s -type f 2>/dev/null
```

> 🔍 **命令解释：**
> - `find /`：从根目录开始找
> - `-perm -u=s`：找设置了SUID位的文件
> - `-type f`：只找文件，不找目录
> - `2>/dev/null`：把错误信息丢掉，不然会有一大堆"Permission denied"的报错，看着烦

执行完，会列出所有设置了SUID位的文件。

我们来看看，这些文件里，有没有什么可疑的？

常见的SUID文件比如 `passwd`、`su`、`sudo` 这些都是正常的。但是如果有什么奇怪的、不常见的，那就要重点关注了。

哎？你有没有看到 `find` 命令也设置了SUID位？

**什么？find命令有SUID？那这可是个大宝贝啊！** 😱

### 32.6.5 用find命令提权

find命令有SUID？那可太好了！因为find命令可以用来提权！

怎么提呢？我给大家讲讲：

find命令有个参数叫 `-exec`，意思是"找到文件之后，执行某个命令"。

比如：
```bash
find . -name "test.txt" -exec ls -la {} \;
```
意思就是：找当前目录下叫test.txt的文件，找到之后，执行 `ls -la` 命令。

那如果find命令有SUID位（所有者是root），那我们用 `-exec` 执行的命令，是不是也是以root权限执行的？

**没错！就是这样！** 🎯

那我们能不能用find来执行一个shell呢？这样我们不就拿到root权限了吗？

当然可以！方法如下：

```bash
find . -exec /bin/sh \;
```

就这么简单！

> 🔍 **解释一下：**
> - `find .`：在当前目录下找文件（随便找什么都行，只要能找到）
> - `-exec /bin/sh \;`：找到之后，执行 `/bin/sh`，也就是开一个shell
>
> 因为find命令有SUID位，所以这个shell是以root权限运行的！
>
> 就这么一行命令，我们就能变成root！是不是很神奇？

好，我们来试试！

在Shell里输入：

```bash
find . -exec /bin/sh \;
```

然后回车……

等等，怎么没反应？哦，不对，find需要找到至少一个文件才会执行 `-exec` 后面的命令。当前目录下肯定有文件的，所以应该没问题。

或者你可以换个写法，确保能找到文件：

```bash
find /etc/passwd -exec /bin/sh \;
```

（找/etc/passwd这个文件，这个文件肯定存在）

执行完之后，你会看到一个新的Shell提示符。我们来看看现在是谁：

```bash
whoami
id
```

如果返回的是 `root`，那就是成功了！🎉🎉🎉

哇！我们成功拿到root权限了！太不容易了！

### 32.6.6 找剩下的flag

好了，现在我们已经是root了，系统里我们想去哪儿就能去哪儿。

提示一下：flag3一般在网站的某个地方，可能是一个内容页面，也可能是一个文件。既然我们现在是root了，我们可以全局搜一下：

```

### 32.7.1 完整攻击链梳理

整个攻击过程，我们可以用一张图来表示：

```

    ↓

```

是不是很清晰？这就是一个**非常典型的渗透测试流程**。

### 32.7.2 每一步的关键点

我们来回顾一下每一步的关键点：

**1. 信息收集是基础**
> 还是那句话，信息收集永远是第一位的。如果我们不扫端口，就不知道有80端口；如果不看网站，就不知道用的是Drupal；不知道Drupal，就找不到Drupalgeddon漏洞。
>
> **七分收集，三分攻击**，这句话永远不会错。

**2. CMS识别很重要**
> 现在很多网站都是用CMS搭的，WordPress、Drupal、Joomla、Discuz!……每个CMS都有自己的常见漏洞。
>
> 所以，**能识别出目标用的是什么CMS**，是一项非常重要的技能。识别出来了，就可以去查对应的已知漏洞，事半功倍。

**3. 已知漏洞利用是拿Shell的捷径**
> 我们这次用的Drupalgeddon漏洞，就是一个典型的"已知漏洞利用"。不用自己去找0day，不用费劲去挖漏洞，直接用已经公开的漏洞，一键就能拿Shell。
>
> 这就像你开门，不用自己撬锁，直接用配好的钥匙开门——又快又方便。🔑
>
> 当然，能自己挖漏洞当然更厉害，但对于新手来说，先学会用已知漏洞，把流程跑通，更重要。

**4. 提权的方法有很多**
> 这次我们用的是SUID提权（find命令），但其实DC-1的提权方法不止这一种。比如：
> - 内核漏洞提权（系统版本这么老，肯定有内核漏洞）
> - 密码复用（用数据库密码登录flag4用户）
> - 定时任务提权
> - 等等……
>
> 打靶机的时候，不要满足于一种方法，多试试其他方法，你的技术才能全面提高。

**5. flag只是奖励，过程才是最重要的**
> 很多人打靶机，光顾着找flag，找完就完事了。但其实，flag只是一个"通关证明"，真正重要的是**过程**。
>
> 你在这个过程中学到了什么？遇到了什么问题？怎么解决的？这些才是最宝贵的财富。
>
> 就像玩游戏，重要的不是通关那一刻，而是闯关过程中的乐趣和成长。🎮

## 32.9 打靶场的正确姿势（再强调一遍）

### 32.9.1 先自己想，再看答案

第一点，也是最重要的一点：**遇到问题，先自己思考，实在想不出来再看答案。**

为什么这么说？因为只有自己思考过、挣扎过，你才能真正理解、真正记住。

打个比方：
- 别人直接告诉你答案，就像别人嚼过的饭喂给你——没味道，也没营养
- 自己想出来的答案，就像自己做的饭——虽然可能做得不好吃，但有成就感，也更有营养

我给大家的建议是：
> 遇到问题，至少自己思考30分钟，翻翻笔记、搜搜相关知识点。
> 实在想不出来，再去看Writeup。
> 看完Writeup之后，一定要自己再动手做一遍。

### 32.9.2 一定要动手，别光看

第二点：**不要光看教程、看视频，一定要自己动手做！**

很多人喜欢看教程，觉得"我看懂了"，但真到自己动手的时候，就各种出错。这就是典型的"一看就会，一做就废"。😂

为什么会这样？因为看和做是两回事：
- 看的时候，你跟着别人的思路走，觉得很顺畅
- 自己做的时候，会遇到各种各样意想不到的问题

只有自己动手做，遇到问题，解决问题，你才能真正掌握。

记住：**实践出真知！动手才是硬道理！** 💪

### 32.9.3 做好笔记

第三点：**一定要做好笔记！**

### 32.9.4 理解原理，不要死记硬背

第四点：**要理解原理，不要死记硬背命令。**

很多新手容易犯一个错误：就是死记硬背命令和步骤。比如打DC-1，就把步骤背下来：第一步nmap，第二步MSF，第三步set RHOSTS……

但是这样没用的。换一个靶机，换一个漏洞，你就不会了。

我们应该理解**为什么要这么做**，而不是只记住**怎么做**。

比如：
- 为什么要信息收集？→ 因为要了解目标，找到攻击面
- 为什么要扫端口？→ 因为端口对应着服务，服务可能有漏洞
- 为什么要找SUID文件？→ 因为SUID文件可能可以用来提权

理解了原理，不管遇到什么靶机，你都能有思路。

记住：**授人以鱼不如授人以渔。思路比命令重要一万倍！** 🧠

>
> 推荐资源：
> - 《代码审计：企业级Web代码安全架构》
> - 各种CMS的漏洞分析文章

#### 📖 2. 逻辑漏洞
> 什么是逻辑漏洞？简单说就是业务逻辑上的漏洞，比如：
> - 越权（水平越权、垂直越权）
> - 支付漏洞（0元购、修改价格）
> - 密码找回漏洞
> - 验证码绕过
> - 等等……
>
> 为什么重要？因为现在很多网站的技术漏洞（比如SQL注入、XSS）已经很少了，但是逻辑漏洞依然大量存在。
>
> 而且逻辑漏洞不需要什么高大上的工具，就是靠细心和思路，特别能体现一个人的安全思维。
>
> 怎么学？
> - 多打CTF题里的Web题
> - 多看看真实的漏洞案例
> - 平时上网的时候多留意，多思考"这里有没有漏洞？"

#### 📖 3. 更深的WAF绕过
> 什么是WAF？WAF就是Web应用防火墙，专门用来挡黑客的。
>
> 真实环境中，很多网站都有WAF，你直接注入的话，会被WAF拦下来。这时候就需要WAF绕过技术了。
>
> 我们之前学过一些简单的绕过（比如大小写绕过、注释绕过），但还有更多更高级的绕过技巧。
>
> 怎么学？
> - 多做绕过相关的CTF题
> - 看看WAF绕过的总结文章
> - 自己搭个WAF环境练练手

#### 📖 4. 内网渗透
> 什么是内网渗透？就是你拿到了WebShell之后，怎么在内网里继续深入，拿下更多的机器。
>
> 为什么要学？因为真实的渗透测试项目，很少有只拿一个WebShell就完事的。一般都要继续深入，拿到域控、拿下整个内网。
>
> 怎么学？
> - 先学好Windows和Linux系统知识
> - 学习常见的内网协议（SMB、RDP、SSH、WinRM等等）
> - 学习横向移动的各种方法
> - 打一些内网靶场，比如VulnStack（红日靶场）

#### 📖 5. 提权技术
> 提权我们学过一些，但那只是最基础的。提权的水很深，有很多很多技巧。
>
> 比如：
> - Windows提权和Linux提权思路不一样
> - 内核漏洞提权
> - 第三方服务提权
> - 计划任务提权
> - 数据库提权
> - 等等……
>
> 怎么学？
> - 多打靶机，每个靶机试试不同的提权方法
> - 看看提权相关的总结文章
> - 收藏一些提权cheat sheet（速查表）

### 32.10.3 如果你想往二进制方向发展

如果你对二进制安全、PWN、逆向感兴趣，那你需要补很多基础知识：

#### 📖 1. C语言
> C语言是二进制安全的基础，必须要学好。
>
> 不光要会写，还要能看懂别人的代码，理解指针、内存、结构体这些概念。

#### 📖 2. 汇编语言
> 逆向工程、PWN都需要懂汇编。
>
> 不用会写，但至少能看懂，知道每条指令是干什么的。
>
> 先学x86汇编，再学x64汇编。

#### 📖 3. 操作系统原理
> 要理解内存管理、进程、线程、系统调用这些概念。
>
> 推荐书籍：《深入理解计算机系统》（CSAPP），这本书虽然厚，但非常经典，值得反复读。

#### 📖 4. 逆向工程
> 什么是逆向工程？就是把编译好的程序反过来，还原出它的逻辑和代码。
>
> 工具：IDA Pro、Ghidra、x64dbg……
>
> 怎么学？可以从简单的CrackMe开始练手。

#### 📖 5. PWN
> PWN就是利用二进制漏洞来控制程序，拿Shell。
>
> 常见的漏洞类型：栈溢出、堆溢出、格式化字符串、UAF……
>
> 怎么学？可以去BUUCTF、CTFHub上做PWN题，从简单的开始。

二进制方向比较难，入门周期长，但是一旦入门了，发展前景也很好。如果你真的感兴趣，那就加油吧！💪

### 32.10.4 可以考的证书

很多小伙伴会问：学安全要不要考证？有什么证书可以考？

我给大家列几个常见的证书：

**初级证书：**
- **CEH（Certified Ethical Hacker）**：国际上比较有名的入门证书，偏理论，难度不大。
- **CISP**：国内的信息安全认证，分很多方向，体制内比较认可。

**进阶证书：**
- **OSCP（Offensive Security Certified Professional）**：含金量非常高的渗透测试证书，纯实操，难度大，但是考下来之后找工作非常有优势。
- **CISP-PTE**：国内的渗透测试工程师认证，也是实操型的。

**大牛证书：**
- **OSCE**：比OSCP更难更高级的证书。
- **CISSP**：信息安全领域的"黄金证书"，偏管理和架构，需要工作经验。

我的建议是：
> 新手不用急着考证，先把技术学好。
>
> 等你有一定基础了，可以考虑考个OSCP或者CISP-PTE，这两个证书的含金量比较高，对找工作帮助大。
>
> 证书只是敲门砖，真正重要的还是你的技术实力。📜

## 32.11 给新手的几句心里话

在这本书的最后，我想跟大家说几句心里话。这些话，可能比技术知识更重要。

### 32.11.1 安全这条路很长，没有终点

首先我想告诉你：**网络安全这条路很长，没有终点，永远有学不完的东西。**

真的，安全这个领域太广了：Web、二进制、移动、内网、云、工控、IoT、密码学……每个方向都够你学好几年。

而且技术更新换代很快，新的漏洞、新的技术、新的工具层出不穷。你刚学会这个，那个又出来了。

所以，永远不要觉得"我学会了"、"我够厉害了"。要保持一颗谦卑的心，永远在学习的路上。

> 学无止境，这句话在安全领域体现得淋漓尽致。📚

### 32.11.2 保持好奇心，保持学习的热情

第二点：**保持好奇心，保持学习的热情。**

好奇心是最好的老师。

- 为什么这个漏洞会存在？
- 为什么这个命令能提权？
- 背后的原理是什么？
- 有没有更好的方法？

多问几个"为什么"，多去探索，你才能学得更深、走得更远。

如果只是死记硬背，为了找工作而学，那你会学得很痛苦，也走不远。只有真正感兴趣、有好奇心，你才能在这条路上走得长久。

> 兴趣是最好的老师，好奇心是最强的动力。🔍

### 32.11.3 不要怕难，大家都是从新手过来的

第三点：**不要怕难，遇到挫折很正常，大家都是从新手过来的。**

我知道，学习安全的过程中，你肯定会遇到很多困难：
- 这个漏洞怎么都理解不了
- 那个靶机怎么都打不下来
- 学了这么久，感觉自己什么都不会
- 看着大佬们的操作，觉得自己好菜

别灰心，别着急。每个人都是这么过来的。那些你眼中的大佬，曾经也跟你一样，是个连Linux命令都不会的小白。

遇到困难的时候，告诉自己：
> 这很正常，我只是暂时不会而已。
> 慢慢来，我能学会的。
> 今天不会，明天再学；明天不会，后天再学。总有一天会学会的。

只要你不放弃，一直坚持下去，就一定能越来越好。

> 慢慢来，比较快。🐢

### 32.11.4 多动手，多实践

第四点：**多动手，多实践。不要光看不练。**

这句话我已经说过很多遍了，但我还是想再说一遍，因为它太重要了。

安全是一门实践性很强的学科。光看书、光看视频是学不会的，必须自己动手做。

- 看了一百篇教程，不如自己动手打一个靶机
- 背了一千条命令，不如自己在终端里敲一遍
- 想了一万个思路，不如实际操作验证一下

只有动手做了，遇到问题，解决问题，你才能真正掌握。

> 纸上得来终觉浅，绝知此事要躬行。🛠️

### 32.11.5 遵纪守法，技术要用在正途上

第五点，也是最重要的一点：**遵纪守法，技术要用在正途上。**

这是底线，也是红线。

我知道，很多人学安全是因为觉得"黑客很酷"、"能黑进别人的电脑很厉害"。但是我要告诉你：

**现实不是电影。**

在现实世界里：
- 未经授权入侵别人的网站/系统，是违法的
- 盗号、诈骗、搞破坏，都是要坐牢的
- 黑产、灰产，看着赚钱，其实都是在钢丝上走路，早晚有一天会进去

我们学安全，是为了**保护网络安全**，而不是破坏它。
- 你可以去做渗透测试工程师，帮企业找漏洞、修漏洞
- 你可以去做安全研究员，研究漏洞、发布预警
- 你可以去做安全开发，开发更安全的系统

技术本身没有对错，关键在于使用技术的人。

希望你能记住：**能力越大，责任越大。**

用你的技术去做好事，去保护别人，去让这个世界变得更安全——这才是一个安全从业者应该做的事。🛡️
