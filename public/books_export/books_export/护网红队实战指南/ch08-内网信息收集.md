## 8.1 为什么内网信息收集这么重要？

### 8.1.1 内网就是信息战

很多人觉得内网渗透就是
"打进去一台就完事儿"，
其实不是。

内网渗透，
很大程度上是**信息战**。
你掌握的信息越多，
你就越强。

为什么这么说？

因为内网和外网不一样，
外网你可能只有一个入口，
内网里机器多了去了，
你知道的信息越多，
能找到的路就越多，
能打的目标就越多。

比如：
- 你知道了域控是哪台，
  你就可以盯着域控打。
- 你知道了哪个用户是域管，
  你就可以盯着这个用户的机器打。
- 你知道了有哪些网段，
  你就知道该往哪个方向打。

信息收集决定了你能走多远。

### 8.1.2 内网信息收集的原则

内网信息收集，
要遵守几个原则：

**第一，动静小。**
内网里有IDS、有审计、有各种监控，
扫得太猛容易被发现。
所以要尽量安静，
能少发包就少发包，
能用被动的就不用主动的。

**第二，全面。**
能收集的都收集，
别嫌多，
信息越多越好。
现在看着没用的信息，
说不定哪天就用上了。

**第三，持续。**
信息收集不是一次就完事的，
每拿到一台新机器，
都要重新收集一遍。
每到一个新网段，
都要重新摸一遍。
信息收集是贯穿整个内网渗透始终的。

> 💡 **一句话：**
> 内网渗透就是一个不断收集信息、
> 不断扩大战果的过程。
> 信息越多，
> 滚雪球就越快。

## 8.2 本机信息收集

先从哪里开始？
先从你打下的第一台机器开始。
这台机器就是你的"根据地"，
先把它摸得清清楚楚的。

### 8.2.1 基本信息

先看看最基本的信息：

**Windows下：
```powershell
# 系统信息
systeminfo

# 主机名
hostname

# 当前用户
whoami

# 用户ID
whoami /all

# 网络配置
ipconfig /all

# 路由表
route print

# ARP表
arp -a

# 网络连接
netstat -ano
```

**Linux下：
```bash
# 系统信息
uname -a
cat /etc/issue

# 主机名
hostname

# 当前用户
whoami
id

# 网络配置
ip a
ifconfig

# 路由表
route -n
ip route

# ARP表
arp -a
ip neigh

# 网络连接
netstat -tulnp
ss -tulnp
```

这些都是最基本的，
先把这些看一遍，
心里就有数了。

### 8.2.2 用户和凭据信息

用户信息很重要，
特别是凭据，
有了凭据就能横向移动。

**Windows下：**
```powershell
# 本地用户列表
net user

# 本地管理员组
net localgroup administrators

# 当前域用户（如果在域里）
net user /domain

# 域管理员
net group "domain admins" /domain

# 企业管理员
net group "enterprise admins" /domain

# 域控制器列表
net group "domain controllers" /domain
```

**Linux下：**
```bash
# 用户列表
cat /etc/passwd

# 密码哈希（需要root）
cat /etc/shadow

# 用户组
cat /etc/group

# 当前登录的用户
w
who
users
```

这些信息很重要，
特别是管理员用户、
域管理员，
这些都是重点目标。

### 8.2.3 进程和服务

看看机器上跑了哪些进程、
哪些服务，
说不定有什么有用的东西。

**Windows下：
```powershell
# 进程列表
tasklist
tasklist /v

# 服务列表
net start
sc query

# 启动项
wmic startup get command,caption

# 计划任务
schtasks /query /fo LIST /v
```

**Linux下：
```bash
# 进程列表
ps aux
ps -ef

# 服务列表
service --status-all
systemctl list-units --type=service

# 开机启动项
ls /etc/init.d/
systemctl list-unit-files --type=service | grep enabled

# 计划任务
crontab -l
ls /etc/cron*
```

看看有没有什么敏感进程，
比如数据库、
邮件服务、
运维系统什么的，
这些都是重点。

### 8.2.4 软件和补丁

看看装了哪些软件，
系统补丁打得怎么样，
有没有什么老版本的软件有漏洞的。

**Windows下：
```powershell
# 已安装的软件
wmic product get name,version

# 已安装补丁
wmic qfe get Caption,Description,HotFixID,InstalledOn
```

**Linux下：
```bash
# 已安装软件（Debian/Ubuntu）
dpkg -l

# 已安装软件（CentOS/RHEL）
rpm -qa
```

补丁信息也很重要，
补丁打得少的话，
提权漏洞就多，
横向也容易。

### 8.2.5 历史记录和日志

看看历史记录，
能发现很多东西。

**Windows下：
```powershell
# 最近打开的文件
# （看用户最近访问过什么）

# 浏览器历史
# （各种浏览器的历史记录）

# PowerShell历史
Get-History

# 事件日志
wevtutil ...
```

**Linux下：
```bash
# bash历史
cat ~/.bash_history
history

# 系统日志
/var/log/
```

历史记录里经常能发现宝贝，
比如管理员登录过什么机器，
输过什么密码，
访问过什么网站。

## 8.3 网络信息收集

本机信息收集完了，
接下来看看网络情况。
这台机器能访问哪些网段？
内网里有哪些机器是活的？

### 8.3.1 内网网段发现

首先得知道内网有哪些网段。
怎么找？

**方法一：看网卡配置
ipconfig / ip a 就能看到网卡，
如果有多块网卡，
就能知道它在哪些网段。

**方法二：看路由表
route print 或 route -n，
看路由表，
能知道有哪些路由，
访问哪些网段走哪个网关。

**方法三：看ARP表
arp -a，
ARP表里有什么IP，
说明这些IP是跟这台机器通的。

**方法四：看配置文件
各种配置文件里，
比如数据库连接地址、
其他系统的地址，
也能发现其他网段。

**方法五：看主机名
有些机器的主机名
有时候也能猜出来，
比如WEB-01、DB-01、
WEB01.xxx.com，
也能找到规律。

### 8.3.2 存活主机探测

知道了网段，
接下来看看这个网段里
哪些机器是活的。

怎么探？

**方法有很多：**

**1. ICMP探测（ping扫描）
最简单的，ping一下。
但很多机器禁ping，
ping不通不代表不活。

**2. TCP/UDP探测
扫常见端口，
比如135、139、445、3389、22、80...
只要有端口开着，
就说明机器是活的。

**3. ARP探测
ARP扫描，
内网里特别好用，
因为ARP一般不会被拦截。

**4. DNS探测
如果有域的话，
可以通过DNS解析
找域内的机器。

**常用工具：
- Windows下：
  • nmap
  • masscan
  • 各种S扫描器
  • PowerShell脚本
- Linux下：
  • nmap
  • masscan
  • fscan
  • ......

> ⚠️ 注意：
> 内网扫描动静挺大的，
> 很容易被检测到。
> 红队在内网里，
> 不要上来就全端口、
> 全网段扫，
> 太吵了，
> 容易被发现。
>
> 先慢慢摸，
> 一个网段一个网段来，
> 先扫常用端口，
> 慢慢扩大。

### 8.3.3 端口和服务探测

找到存活主机了，
接下来看看开了哪些端口、
跑了什么服务。

常见的端口和对应的服务：
```
21    FTP
22    SSH
23    Telnet
25    SMTP
53    DNS
80    HTTP
110   POP3
135   RPC
139   NetBIOS
1433  MSSQL
443   HTTPS
445   SMB
3306  MySQL
3389  RDP远程桌面
5985  WinRM
6379  Redis
8080  HTTP代理/各种Web
......
```

看到什么服务，
就可以对应什么思路：
- 看到445，就可以试试SMB相关的漏洞、哈希传递；
- 看到3389，就可以试试弱口令、远程桌面；
- 看到3306，就可以试试MySQL弱口令；
- 看到6379，就可以试试Redis未授权；
- ......

看到什么服务，
就有对应的攻击方法。

## 8.4 域信息收集

如果内网有域的话，
那恭喜你，
域信息收集
就是重中之重了。
域环境是内网渗透的重点，
也是最有意思的部分。

### 8.4.1 先判断有没有域

怎么判断机器在不在域里？

Windows下：
```powershell
# 看系统信息里有没有域
systeminfo | findstr /B /C:"域"

# 看工作站信息
net config workstation

# 看当前用户域
echo %userdomain%

# 看环境变量
set u
```

如果在域里的话，
就可以开始收集域信息了。

### 8.4.2 基本域信息

先收集最基本的域信息：

```powershell
# 查看域信息
net view /domain

# 查看域控制器
net group "domain controllers" /domain
nltest /dclist:域名

# 查看域用户
net user /domain

# 查看域组
net group /domain

# 查看域管理员
net group "domain admins" /domain

# 查看企业管理员
net group "enterprise admins" /domain

# 查看架构管理员
net group "schema admins" /domain

# 查看域内机器
net group "domain computers" /domain

# 查看域策略
net accounts /domain
```

这些命令都是Windows自带的，
不用传工具，
直接就能用，
比较安静。

### 8.4.3 用PowerShell收集域信息

net命令能查到的有限，
PowerShell能查到的更多。

比如用PowerView，
或者用AD模块，
能查到非常详细的域信息。

常用的PowerShell脚本：
- PowerView（PowerView.ps1）
- AD模块（RSAT工具里的）
- ......

能查什么？
- 所有域用户详细信息
- 所有域机器详细信息
- 所有组、组织单位
- 域管理员、企业管理员
- 域管理员登录过的机器
- 哪些用户有SPN
- 哪些机器有哪些用户登录过
- 委派关系
- ......

域信息收集的东西，
我们域渗透那章再详细讲，
这里先有个概念。

### 8.4.4 域信息收集的重点

域信息这么多，
重点关注什么？

**1. 域管理员列表
谁是域管理员？
这些人是重点目标，
只要拿到其中一个，
域控基本就到手了。**

**2. 域控是哪台？
最终目标就是域控，
得先知道域控在哪。**

**3. 哪些机器有域管登录过？
域管登录过的机器，
如果能拿到那台机器，
就能抓到域管的哈希或者明文，
然后就可以哈希传递打域控。**

**4. SPN（服务主体名称）
有SPN的服务账号，
可以做Kerberoasting攻击，
直接能拿到服务账号的哈希，
拿下来可能权限很高。**

**5. 委派关系
有委派的机器或用户，
可以利用委派来提权、横向。**

**6. GPO（组策略）
组策略里可能有密码、
有脚本、
有各种配置，
都是宝贝。**

这些都是域渗透的关键信息，
收集到了，
域渗透就成功了一半。

## 8.5 凭据收集—— 重中之重

说了这么多信息收集，
最重要的是什么？
**凭据收集。**

为什么凭据这么重要？
因为有了凭据，
你就能登录其他机器，
就能横向移动，
就能打更多机器，
拿更多凭据，
滚雪球一样越滚越大。

### 8.5.1 Windows下的凭据收集

Windows下能收集的凭据可多了：

**1. SAM哈希
本地用户的密码哈希，
存在注册表里，
System权限就能读。
拿到了本地哈希，
可以拿去破解，
也可以哈希传递登录其他机器。**

**2. LSA 内存里的密码
用mimikatz从内存里抓明文密码、
抓哈希、抓票据，
这是最经典的。**

**3. 浏览器保存的密码
Chrome、Firefox、Edge...
各种浏览器保存的密码，
很多人都保存网站密码，
说不定就有其他系统的。**

**4. 远程桌面保存的密码
RDP保存的密码，
mstsc保存的凭据，
也能导出来。**

**5. 各种软件保存的密码
FTP客户端、
数据库客户端、
邮箱客户端、
VPN客户端...
各种软件保存的密码，
都能想办法弄出来。**

**6. WiFi密码
连过的WiFi密码，
也能看到。**

**常用工具：
- mimikatz（经典中的经典）
- LaZagne（各种密码都能抓）
- mimikittenz
- PowerShell版本的mimikatz
- 各种抓密码工具多了去了。

> ⚠️ 注意：
> mimikatz太有名了，
> 查杀率极高，
> 一上传就被杀，
> 要用的话得免杀做好，
> 或者用别的方式。

### 8.5.2 Linux下的凭据收集

Linux下也有不少：

**1. shadow文件
/etc/shadow里的密码哈希，
root权限就能读，
读出来可以爆破。**

**2. bash历史
~/.bash_history
历史命令里，
经常能看到明文密码，
比如mysql -u root -p密码，
这种。**

**3. ssh密钥
~/.ssh/里的ssh私钥，
拿到了就能登其他机器。**

**4. 各种配置文件里的密码
数据库配置、
网站配置、
各种应用的配置文件，
经常有明文密码。**

**5. 内存里的密码
也有工具可以从Linux内存里抓密码，
比如mimipenguin之类的。**

## 8.6 信息收集的思路和技巧

讲了这么多收集的内容，
再聊聊信息收集的几个思路和技巧。

### 8.6.1 由近及远，由点及面

什么意思？
就是从你打下的这台机器开始，
先把这台机器摸透，
然后从这台机器所在的网段，
然后慢慢往外扩，
一个网段一个网段地摸，
逐步扩大范围。

不要上来就全网段扫，
太吵了，
容易被发现。
稳一点，
慢慢摸，
越打越多。

### 8.6.2 被动收集优先

能用被动的方式收集信息，
就别用主动的。
什么是被动的？
比如看配置文件、
看历史记录、
看缓存、
看日志，
这些都是不用发什么包，
比较安静。

主动扫描是要发很多包，
容易被检测到。
能少扫就少扫，
能不扫就不扫。

### 8.6.3 每拿到一台新机器，
先收集信息。
每拿到一台新机器，
第一件事就是收集信息，
本机信息、
网络信息、
凭据信息，
都收集一遍。
新的信息新的信息，
新的凭据，
新的网段，
新的目标，
滚雪球就快了。

### 8.6.4 信息要整理，别乱堆。

信息收集了一大堆，
别就扔那了，
要整理。

怎么整理？
建个表，
或者整个知识库，
把收集到的信息都记下来：
- 哪些IP，对应什么机器
- 哪些用户，什么权限
- 哪些凭据，能登哪些机器
- 哪些网段，什么情况
- 哪些重要系统在哪

整理清楚了，
用的时候才方便，
不然信息乱哄哄的，
找都找不到。
