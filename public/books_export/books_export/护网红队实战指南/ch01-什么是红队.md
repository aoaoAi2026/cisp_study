# 第一章 什么是红队？—— 红队不是黑客

> 第1章 | 预计学习时间：3小时 | 难度：⭐

---

## 📖 写在前面

很多人第一次听到"红队"这俩字，脑子里都会冒出几个问题：

"红队？是不是就是黑客啊？"
"红队和渗透测试有啥不一样？"
"护网红队又是个啥？听着挺厉害的"
"学这个会不会进去啊？"

别着急，这一章我们就慢慢聊，用大白话把这些问题都讲明白。

看完你就知道红队到底是干啥的，和黑客、渗透测试有啥区别，以及——为什么说红队是网络安全里最有意思的方向之一。

坐稳了，我们开唠。

---

## 1.1 红队到底是个啥东西？

### 1.1.1 先从一个真实的故事讲起

先给你讲个真实发生的故事。

有这么一家大公司，老板姓王，我们就叫他王总吧。

王总很重视安全，公司防火墙买的是最好的，什么下一代防火墙、入侵检测、态势感知、EDR（端点检测与响应）一个不少，安全团队有二十多个人，7×24小时值班，监控大屏24小时亮着。

每年还花几十万请人做渗透测试，每次报告都写得漂漂亮亮的：
- "系统整体安全状况良好"
- "未发现高危漏洞"
- "风险可控，建议持续监控"

王总心里美滋滋的：钱没白花，我们公司固若金汤啊！

直到有一天——

他朋友老李来公司喝茶，老李是个资深安全顾问。老李参观完王总的安全监控中心，眉头皱了皱，说："老王啊，你们这都是'静态'的安全防护啊。"

王总不解："啥意思？"

老李说："这么说吧，你买了全世界最好的防盗门、防盗窗、保险柜，但是——你从来没请过一个小偷来试试，能不能撬开。你怎么知道这些东西真的能防住小偷呢？"

王总一听，觉得有道理。

老李继续说："要不这样，我找几个真正懂黑客技术的人，让他们模拟真实的网络攻击来打你。就当是真有黑客来进攻，看看你的安全团队到底能不能防住，最后能被打成什么样。"

王总想了想，觉得有道理。光自己说安全不算，得有人来"考考试"。

于是他就找了个安全公司，签了合同，说："你们来打我们吧，就当成真的黑客来打，能多隐蔽就多隐蔽，看看我们的安全团队多久能发现，以及最后能打到什么程度。"

安全公司就派了一队人，假装是黑客，偷偷摸摸地开始攻击。

结果呢？

**三天**，就打穿了整个内网，拿到了域控权限，还"偷走"了一批"机密文件"。

而王总的安全团队呢？直到**第七天**才发现不对劲。还是因为攻击者不小心留了个小马脚——有个木马的进程名起得太明显了，叫"test123.exe"。

王总看完攻击报告，吓出一身冷汗。他一直以为自己公司固若金汤，结果人家三天就打穿了，自己的人过了一周才发现。等发现的时候，人家早就把该拿的都拿了。

后来王总根据这次攻击暴露出来的问题，做了全面整改，又做了好几次类似的演练，才真正把安全能力提上来了。

---

这群"假装黑客"的人，就是我们要说的——**红队**。

### 1.1.2 给红队下个定义

说了半天故事，那到底啥是红队？

用一句话说：

> **红队，就是站在攻击者的角度，模拟真实的黑客攻击，来检验一个组织的真实安全防御能力的团队。**

我再翻译翻译，说得更直白点：

红队就是"职业假黑客"。他们有黑客的技术，用黑客的思路，学黑客的手段，但他们不是坏人，是好人花钱请来"打自己的"。

就像汽车的碰撞测试一样——

汽车厂花大价钱研发一辆车，然后专门找人用各种方式撞毁它：正面撞、侧面撞、翻滚、追尾......看看撞了之后车里的人安不安全、气囊能不能弹出来、安全带能不能保护人。

红队干的也是这个事儿：客户花钱请他们来"攻击"自己，看看自己的防线到底牢不牢，安全团队到底行不行，真正被打的时候能不能扛得住。

### 1.1.3 再用一个生活中的例子帮你理解

我再给你举个生活中的例子，帮你更直观地理解红队是干啥的。

比如你家的门锁：

- 你买了一把高级门锁，商家说"防盗性能一流，B级锁芯，密码+指纹+钥匙三合一"。

- 你觉得很安全，放心了。

- 但有一天，你请了个开锁师傅来"测试"你的锁。开锁师傅看了看，说："这把锁啊，我用特殊工具，5分钟就能打开。"

- 你不信。开锁师傅当场演示，果然3分钟就打开了。

- 你吓一跳，赶紧换了把更高级的锁。

这个开锁师傅，就相当于"红队"。他是好人（你花钱请来的），干的事看起来像坏人（开锁），但目的不是偷你家东西，而是告诉你"你的锁不安全，赶紧换"。

红队就是这样：他们看起来像黑客（用黑客的技术），但目的是帮你找出安全问题，让你变得更安全。

### 1.1.4 红蓝紫三方是怎么回事

经常听人说"红蓝对抗"、"红蓝紫三方"，那红蓝紫到底是什么关系？

我给你打个比方，你就懂了。就像拍电影打仗：

```
红队 = 进攻的军队（攻击方）
  他们的任务是：想尽一切办法打进去
  目标可能是：拿下目标系统、拿到核心数据、获取域控权限
  越隐蔽越好，越快越好，不被发现最好

蓝队 = 防守的军队（防御方）
  他们的任务是：发现攻击、拦住攻击
  最好还能溯源——找到攻击者是谁、从哪来的、用了什么手法
  如果发现不了，那就输了

紫队 = 导演部/裁判（组织方）
  他们既不攻也不守
  负责制定比赛规则、判定谁得分谁失分
  最后组织大家复盘总结：哪里做得好、哪里需要改进
```

再换个比喻，像踢足球：
- 红队 = 前锋，负责进球（打进去、拿分）
- 蓝队 = 后卫+守门员，负责守门（拦住、发现）
- 紫队 = 裁判，负责吹哨、计分（组织、评判）

这三个角色缺了哪个都不行：
- 没有红队，蓝队不知道自己防不防得住
- 没有蓝队，红队就是自己跟自己玩
- 没有紫队，乱成一锅粥，谁赢谁输都说不清

## 1.2 红队、黑客、渗透测试，到底啥区别？

很多人搞不清这三者的关系，觉得"不都是找漏洞打进去吗？"

还真不一样。我给你掰开揉碎了讲。

### 1.2.1 先说说黑客（恶意攻击者）

这个最好理解，就是我们常说的"坏家伙"。

黑客攻击的目的是什么？
- 为了钱（勒索软件把文件加密，不交钱不给解密；偷银行账号直接转账；卖数据给黑市）
- 为了名（炫技，在黑客论坛发帖"某某网站被我黑了"）
- 为了情报（商业间谍偷竞争对手的商业机密；国家级攻击偷军事机密）
- 为了报复（黑掉前任公司的网站；搞垮竞争对手的服务器）
- 为了政治/意识形态（hacktivist，Hack+Activist，黑客活动家）

他们的特点是：

**第一，没有授权。** 没经过你同意就打你，这叫"非法入侵"，是违法的。

**第二，不择手段。** 只要能达到目的，什么阴招损招都能用：黑网站、挂马、钓鱼、DDoS、供应链攻击......不管会不会搞坏你的系统，会不会泄露你的数据。

**第三，想打多久打多久。** 今天打不下来明天接着打，今年打不下来明年再来，缠上你就没完没了。不像你有周末、有假期，黑客是7×24小时全年无休。

**第四，不留痕迹。** 偷偷摸摸干完就跑，用代理、用VPN、用跳板机，最好你永远都发现不了。等你发现了，人家早跑了，或者早就把你东西偷走了。

**第五，目的就是搞破坏或者偷东西。** 黑客不是来帮你找问题的，是来搞你的。

这些人是我们的敌人，是警察叔叔要抓的人。我们要学他们，但绝不能成为他们。

### 1.2.2 再说说渗透测试

渗透测试，简单说就是：

> 请专业的安全人员，在授权范围内，对你的系统做一次安全测试，看看有哪些漏洞，最后给你一份报告和修复建议。

渗透测试的特点：

**第一，有明确的范围。** 测试哪些系统、哪些IP、哪些域名，合同里都写得清清楚楚。超范围的一律不能碰，比如合同写只测Web系统，你就不能去搞他们的交换机。

**第二，目的是找漏洞。** 渗透测试主要是"找漏洞"，把系统里的安全问题都挖出来告诉你：哪里有问题、问题有多严重、应该怎么修。

**第三，时间比较短。** 一般几天到几周，测完就走，交报告收钱。像去医院体检，一两天就做完了。

**第四，一般不强调隐蔽。** 我光明正大地测，你知道哪天来、谁来测，测完给你报告就行。不需要藏，因为你不是真的在打我，你是在"检查"我。

**第五，合法合规。** 签了合同、有授权书，一切都在阳光下进行。

**第六，方法比较"规矩"。** 主要测试已知漏洞、常见配置问题，不会用太多"歪门邪道"。

渗透测试就像是医生给病人做体检：从头到脚检查一遍，告诉你哪里有病、该怎么治。

### 1.2.3 最后说说红队

那红队又有啥不一样呢？

如果说渗透测试是"体检"，那红队就是"实战演习"。

体检是医生主动给你查，你躺着就行，查完告诉你哪里有问题。

实战演习呢？敌人真刀真枪打过来了，你得自己去发现、去应对，看看你的防御体系到底顶不顶得住。

再打个比方：

**渗透测试** = 你请了个装修师傅，帮你检查一下房子哪里有漏雨的风险

**红队** = 你请了个装修师傅，但这次他假装是小偷，试试看小偷能不能进你家、进了能偷多少东西

红队的特点：

**第一，更接近真实攻击。** 渗透测试可能只是测测已知漏洞、用用标准工具。红队不一样，他们会用真实黑客的思路和方法，想尽一切办法打进来。钓鱼？用。社工？用。物理攻击？只要授权了也用。什么好用用什么，什么有效用什么。

**第二，强调隐蔽性。** 渗透测试你知道哪天来、谁来测。红队不一样——很多时候蓝队都不知道什么时候开始的，得靠自己发现谁来了。就像真的敌人潜入一样，你能发现说明你厉害，发现不了说明防线有漏洞。

**第三，范围可能更灵活。** 渗透测试范围写得很死。红队有时候会给你更大的自由度，只要最终目标达成，中间过程你自己想办法。比如目标是"拿到域控权限"，你怎么拿是你自己的事，没人管你用啥方法。（当然也是在授权范围内的）

**第四，不只要看系统，还要看人。** 渗透测试主要测技术漏洞。红队还要测管理、测流程、测人。比如：人会不会点钓鱼邮件？前台会不会放陌生人进去？员工会不会把密码贴在显示器上？这些都是红队测试的内容。

**第五，时间更长，过程更曲折。** 渗透测试可能一周就完事了。红队项目经常持续几周甚至几个月，慢慢打、慢慢摸，追求的是"真实感"。真实黑客不也是想打多久打多久吗？

**第六，红队评估的是"防御能力"，不只是"漏洞数量"。** 渗透测试告诉你"你有10个漏洞"。红队告诉你"你有10个漏洞，而且我用了3种方法打进来，你的人3天后才发现"。后者更有价值，因为这才说明真实的防护水平。

### 1.2.4 一张图帮你记住区别

我给你画个简单的对比表：

```
┌─────────────┬────────────────┬────────────────┬────────────────┐
│             │  黑客（坏人）   │  渗透测试      │   红队          │
├─────────────┼────────────────┼────────────────┼────────────────┤
│ 目的        │ 搞破坏/偷东西   │ 找漏洞给建议    │ 检验防御能力    │
├─────────────┼────────────────┼────────────────┼────────────────┤
│ 授权        │ 没有（违法）     │ 有，范围明确    │ 有，范围可能更灵活│
├─────────────┼────────────────┼────────────────┼────────────────┤
│ 手段        │ 不择手段        │ 主要技术漏洞    │ 技术+社工+物理  │
├─────────────┼────────────────┼────────────────┼────────────────┤
│ 隐蔽性      │ 必须隐蔽        │ 不强调隐蔽      │ 必须隐蔽        │
├─────────────┼────────────────┼────────────────┼────────────────┤
│ 时间        │ 想多久多久      │ 几天到几周      │ 几周到几个月    │
├─────────────┼────────────────┼────────────────┼────────────────┤
│ 结果        │ 偷数据/搞破坏   │ 给漏洞报告      │ 评估整体防御能力 │
├─────────────┼────────────────┼────────────────┼────────────────┤
│ 合法性      │ 违法            │ 合法            │ 合法            │
└─────────────┴────────────────┴────────────────┴────────────────┘
```

简单记：
- 黑客 = 没授权的坏人 = 警察抓
- 渗透测试 = 有授权的体检医生 = 帮你找问题
- 红队 = 有授权的"假敌人" = 帮你测试防线牢不牢

## 1.3 护网红队又是啥？

前面讲了红队是什么，那经常听到的"护网"、"护网红队"又是个啥？

### 1.3.1 护网到底是啥

"护网"是简称，全称一般是**网络安全实战攻防演习**。

有的地方叫"护网行动"，有的叫"网安演练"，有的叫"攻防对抗赛"，名字不一样，本质上都是一回事。

简单说就是：

> 官方组织一大批单位，一些当攻方（红队），一些当守方（蓝队），在规定的时间里，红队想尽办法打进去，蓝队想尽办法防住，最后看谁得分高、谁排名靠前。

就像是网络安全版的"军事演习"，只不过打的是网络战，不是真枪真炮。

### 1.3.2 护网的由来

护网这东西，是从美国开始的，叫"网络风暴"（Cyber Storm）。美国国土安全部从2006年开始搞，搞了好几年，发现效果不错，后来就推广开了。

中国大概是2016年左右开始搞大规模的护网行动，一开始只在部分行业、部分单位试点，后来推广到全国，现在每年都有，成了网络安全领域最重要的事件之一。

护网的目的是：
1. 检验各单位真实的网络安全防护能力
2. 发现防护体系中的问题
3. 提高各级单位对网络安全的重视程度
4. 锻炼网络安全人才队伍
5. 检验应急响应和处置能力

### 1.3.3 护网一般怎么玩

我给你讲讲护网的大致玩法，让你有个概念（具体规则每次都不一样，大概是这个意思）。

**参加的人：**

- **红队**：可能有几十支甚至上百支队伍，每队几个人，来自不同的安全公司、高校、科研单位、民间战队等等。红队是"攻方"，到处去打。

- **蓝队**：就是防守单位，一般是政府部门、国企、金融机构、能源、交通、医疗、教育这些重要单位。蓝队是"守方"，在自己的地盘上防守。

- **组委会/裁判**：负责制定规则、判定得分、处理申诉。有些还有"工业和信息化部"、"公安部"这样的官方背书。

**比赛流程：**

一般持续1-4周不等，常见的是2周（14天）。

这十几天里，红队就对着蓝队的目标使劲打，能打进去多少算多少，能拿多少分拿多少分。

蓝队就防守，发现攻击就拦截、溯源，溯源成功还能反扣红队的分。

最后按总分排名，红队有红队的排名，蓝队有蓝队的排名。

**怎么得分？**

不同的目标有不同的分值，大概是这个意思：

```
边界突破（从外网打进去）：
  - 拿到一个WebShell → 10分
  - 拿到服务器权限 → 15分

内网渗透：
  - 控制一台普通服务器 → 15分
  - 控制一台重要服务器 → 25分

域渗透：
  - 拿到域普通用户权限 → 20分
  - 拿到域控权限 → 50分

数据窃取：
  - 拿到普通业务数据 → 30分
  - 拿到核心机密数据 → 50分

持久性（不被发现）：
  - 在目标系统潜伏N天 → 每天额外加分
  - 成功建立后门 → 加分

其他：
  - 物理渗透成功 → 加分
  - 社会工程学成功 → 加分
```

目标越重要，分值越高。拿下域控、偷走核心数据，是最值钱的。

**蓝队怎么得分？**

蓝队也不是光挨打，他们也能得分：
- 发现攻击并成功拦截 → 加分
- 成功溯源到红队（搞清楚是谁打的、怎么打的）→ 给红队扣分
- 业务正常运转没受影响 → 基础分
- 应急响应及时、处置得当 → 加分

最后综合算分，红队有红队的排名，蓝队有蓝队的排名。有些比赛还评"最佳攻击手"、"最佳防护团队"之类的单项奖。

### 1.3.4 护网红队和普通红队项目有啥不一样

都是红队，护网红队和做企业红队项目还不太一样：

```
普通红队项目：
  • 只打一个客户
  • 目标比较明确
  • 时间相对充裕（一周到一个月）
  • 追求打深打透，越隐蔽越好
  • 最后给客户一份详细报告，告诉你漏洞在哪、怎么修

护网红队：
  • 打好几个甚至几十个目标（随机分配或自己选）
  • 目标多，得挑软柿子捏
  • 时间紧（就那么几天），节奏快
  • 追求得分效率，怎么快怎么打，怎么得分怎么来
  • 打得快、得分多就是赢
```

打个比方：
- **普通红队项目**像是**外科手术**，精准、深入、细致，把一个目标研究透
- **护网红队**像是**闪电战**，快速、高效、多点开花，同时打很多目标

护网更考验红队的"节奏感"和"效率意识"——14天要打几十个目标，你得知道先打哪个、后打哪个，哪个能快速得分，哪个要花时间慢慢磨。

就像打牌一样，你手里的牌有限（时间和人手），怎么打出最高的总分，是门学问。

## 1.4 护网的亲身经历分享

说了这么多理论，我给你讲个真实的护网故事，让你感受一下护网是啥样的。

### 1.4.1 小明的第一次护网

小明是个刚入行一年的安全工程师，在一家安全公司工作。

今年夏天，他第一次被派去参加护网，跟着团队去打一个政府单位。

出发前，队长给他们开了个会：
"这次护网，我们组负责打3个目标。目标都是政府网站和政务系统，一共14天。记住几个原则：第一，安全第一，别把人家系统搞崩了；第二，能打进去就打，打不进去换目标，别死磕；第三，注意隐蔽，被发现了就低调点；第四，每天晚上复盘，总结经验教训。"

小明听了，心里又紧张又兴奋。

**护网第1天**

早上8点，护网正式开始。

小明和队友开始信息收集：扫域名、扫端口、看指纹、找子站......

忙活了一上午，扫出来十几个域名、几十个开放端口。

下午开始逐个排查，看看哪个有漏洞。

结果发现，大多数系统都打了补丁，漏洞扫出来一堆，全是误报，手动测试一个能用的都没有。

晚上复盘，队长说："第一天没进展很正常，政府系统一般防护都比较严。继续加油。"

**护网第2天**

继续扫，继续测。

下午，小明突然发现有个政务系统用的是老版本的CMS，这个CMS有个已知文件上传漏洞！

小明赶紧试试，果然能上传！

上传了一个WebShell，访问，居然成功了！

小明激动地喊："队长队长！我打进去了！"

队长过来看了看，说："干得不错！但是先别急，先稳住。"

队长让小明先不上线，继续维持在这个入口，先把入口稳住。

**护网第3天**

小明继续在已拿下的系统里收集信息。

提权？成功了！这台服务器是Administrator权限。

抓密码？运气不错，抓到了一个域用户的明文密码。

用这个账号登录别的服务器？居然也成功了！

顺着这条线，又拿下了两台服务器。

**护网第5天**

通过收集到的信息，发现了一个域管理员的账号。

利用这个账号，成功登录到了域控！

在域控上导出所有用户的哈希，拍照截图，提交flag。

队长拍了拍小明的肩膀："干得漂亮！域控到手，这波稳了。"

**护网第7天**

就在小明以为一切顺利的时候，蓝队发现了他们的痕迹。

一个蓝队工程师在查日志的时候，发现了异常的登录记录，开始追踪。

小明这边，队长的电脑响了一下，C2上线了一台机器，但很快又掉了——估计是被发现了。

队长说："没事，我们已经拿到域控了，该拿的分都拿了。降级处理，低调点。"

**护网第14天**

护网结束。

最终结果：小明他们队拿下了2个目标的域控，还打穿了多个内网系统，得分排在所有红队的第8名，还不错。

小明第一次护网，圆满结束。

---

这就是护网的样子：有紧张、有兴奋、有挫败感、有成就感。十几天里，你可能天天熬夜，也可能几天没进展急得团团转，最后拿到目标的那一刻，什么都值了。

### 1.4.2 护网中红队的酸甜苦辣

我干了这么多年护网，总结一下红队人的真实感受：

**苦：**
- 熬夜是常态，尤其是最后几天冲刺
- 有时候对着一个目标搞了3天，一点进展都没有，急死人
- 被蓝队发现了，人家把你的马全杀了，前功尽弃
- 精神压力大，14天精神紧绷，吃不好睡不好
- 封闭集训，14天见不到家人朋友

**甜：**
- 拿下目标那一刻，成就感爆棚
- 和队友一起奋斗的感情，很珍贵
- 学到很多东西，实战是最好的老师
- 护网期间大家一起吃住，感情都特别好
- 拿到好名次，发奖金的时候真香

**酸：**
- 眼睁睁看着别的队得分领先，自己就是打不进去
- 明明差一点就能拿下了，结果触发了告警，全完了
- 护网结束了，发现自己队排名中游，不上不下

**辣：**
- 蓝队溯源做得太好了，把你扒得干干净净挂出来了
- 人家系统防护太变态，试了100种方法都不行

总的来说，护网虽然辛苦，但干过的都说有意思，很锻炼人。

## 1.5 红队需要什么技能？

很多新手问我："我想做红队，需要学什么？"

我给你详细列一下，你可以对照着看自己还差哪些。

### 1.5.1 技术技能

首先是**Web安全**，这个是基础中的基础。

SQL注入、XSS、文件上传、命令注入、逻辑漏洞、反序列化......这些得熟练。毕竟大多数时候，红队都是从Web打进去的。Web安全不会，别想干红队。

然后是**内网渗透**，这个是红队的核心技能。

提权（Windows、Linux、各种提权漏洞）、哈希传递、票据传递、横向移动、端口转发、域渗透......进去之后怎么打，这个才是红队和普通渗透测试最大的区别。渗透测试可能打个Web漏洞就交报告了，红队得把整个内网都打穿。

再有就是**免杀**。

现在EDR、HIDS（主机入侵检测系统）这么多，你传个木马上去就被杀了，那还玩啥？免杀是红队的基本功。什么壳、什么混淆、什么加载器，你得会。

还有**编程/脚本能力**。

不用你写多厉害的程序，但至少Python、Shell脚本得会写。能写个小工具批量处理、能写个脚本自动化操作，这都是一个合格红队的基本要求。

还有各种**工具的使用**：

Cobalt Strike、Metasploit、Nmap、Burp Suite、Sqlmap、蚁剑、冰蝎、哥斯拉......常用工具得熟练。当然，工具只是辅助，别当脚本小子，只会点工具。

最后是**操作系统**。

Windows和Linux都得熟悉，命令行、PowerShell、bash、各种系统命令......你得把操作系统玩得跟玩自己电脑一样溜。

### 1.5.2 非技术技能

别小看非技术技能，很多时候这些比技术还重要。

**第一个是信息收集能力**，或者叫"情报分析能力"。

红队很多时候不是输在技术上，是输在信息不够。你对目标了解得越多，打起来就越容易。谁在管这个系统？用的什么技术栈？有什么历史漏洞？员工邮箱什么格式？......这些信息都可能成为突破口。

**第二个是耐心。**

红队不是CTF，不是半小时就能出flag的。可能你对着一个目标搞一周都没进展，也可能你试了一百种方法都不行，第一百零一种才成功。没有耐心是干不了红队的。

**第三个是思维灵活。**

这条路走不通就换一条，别死磕一个点。技术不行就试试社工，正面打不进去就试试侧面，Web不行就试试旁站。攻击的本质是找漏洞，没有漏洞就创造漏洞，总能找到办法的。

**第四个是社会工程学能力。**

钓鱼、伪装、话术......说真的，社工的成功率往往比技术攻击高。人永远是最薄弱的环节。一封精心设计的钓鱼邮件，可能比花一周找的0day漏洞还好使。

**第五个是写作和表达能力。**

打完了得写报告啊！你打得再精彩，报告写得乱七八糟，客户看不懂，那也是白搭。能把复杂的技术问题讲清楚，让非技术的人也能听懂，这是一项很重要的能力。

**第六个是团队协作能力。**

红队不是单打独斗，是一个团队。有人负责打点，有人负责内网，有人负责社工，有人负责写报告......你得知道怎么和队友配合，怎么分工，怎么沟通。

最后，也是最重要的——**人品和底线**。

红队掌握的技术都很敏感，用好了是保护别人，用歪了就是犯罪。人品不行，技术越好越危险。

### 1.5.3 不同阶段需要掌握的内容

我给你列个学习路线图，不同阶段该学什么：

**入门阶段（0-1年）：**
- Web安全基础（SQL注入、XSS、文件上传等）
- 了解常见的漏洞和利用方法
- 会用常见工具（Burp、Nmap、Sqlmap等）
- 了解HTTP协议、TCP/IP协议
- 搭建自己的实验环境练习

**进阶阶段（1-3年）：**
- 内网渗透技术（提权、横向移动、域渗透）
- 免杀技术基础
- 应急响应基础
- 能独立完成渗透测试项目
- 参加CTF比赛SRC漏洞挖掘

**高级阶段（3年+）：**
- 深入研究某一领域（Web安全、二进制、安全开发等）
- 独立完成红队项目
- 工具开发能力
- 漏洞挖掘能力
- 参加护网并取得好成绩

## 1.6 法律红线—— 这个必须放在最后讲

我把这一节放在最后，因为它最重要。

我知道很多人学安全，都是从"觉得黑客很酷"开始的。看完各种黑客电影、黑客纪录片，心里痒痒的，想自己也当一回"黑客"。

但我必须跟你说清楚：

**技术没有善恶，但用技术的人有。**

### 1.6.1 这些事绝对不能干

我给你列几条红线，碰了就是违法犯罪，千万别拿自己的前途开玩笑。

**第一条：没授权的测试绝对不能做**

不管你是好奇、练手、还是"我只是测一下"，只要没有对方的书面授权，就绝对不能碰别人的系统。

别心存侥幸，觉得：
- "我就扫一下端口，没事"
- "我就试试弱口令，不会搞破坏"
- "这个网站漏洞太多了，忍不住想试试"

法律可不这么认为。《刑法》第二百八十五条、二百八十六条了解一下：
- 非法侵入计算机信息系统 → 三年以下有期徒刑
- 非法获取计算机信息系统数据 → 三年以下有期徒刑
- 破坏计算机信息系统 → 五年以下有期徒刑

真抓进去了，哭都来不及。

**第二条：数据绝对不能碰**

就算你有授权测试，看到了敏感数据也只能看，不能复制、不能下载、不能外传。这是职业操守，也是法律底线。

我见过有人做项目的时候，觉得"这个数据挺有意思的"，就偷偷拷了一份。后来被发现了，工作没了不说，还差点吃官司。

不值得。

**第三条：不能搞破坏**

测试要克制，能看的不要改，能改的不要删。能不影响业务的就尽量不要影响。

打之前问清楚：哪些能测，哪些不能测；哪些系统能动，哪些不能动；哪些数据能看到，哪些不能看。

别为了得分把客户系统搞挂了，那可不是赔钱的事。你把人家系统搞崩了，人家报警，你吃官司。

**第四条：不能留后门不给客户说**

测试完了要清理干净，留下的木马、后门、账号，该删的删，该改的改。偷偷留后门是违法的。

你想想，你给客户留了个后门，以后随时能进去，这跟黑客有什么区别？被发现了，不光丢工作，还可能坐牢。

**第五条：不能用学到的技术搞黑产**

这个就不用多说了，什么盗号、诈骗、挖矿、勒索、卖数据、卖webshell......想都别想。

你以为你很聪明，干点黑产没人知道？太天真了。现在网络追踪能力这么强，你的一举一动都在监控之下。赚那点钱不够坐牢的。

**第六条：不能传播攻击工具/教程给坏人**

你写的工具、你挖的漏洞，不能随便发到网上。你不知道看到的人会拿来干什么。发到GitHub上，看起来是"开源精神"，实际上可能助纣为虐。

### 1.6.2 正确的学习姿势

那想学红队，应该怎么学才对？

**第一，在自己的环境里练。**

自己搭虚拟机、装靶机，VulnHub、HackTheBox上那么多靶机，你想怎么打就怎么打，打到天昏地暗都没人管你。

搭个靶场：Linux服务器、Windows域控、AD活动目录......全在自己家里，随便搞。

**第二，打CTF。**

CTF比赛是最合法的练习方式，各种类型的题目都有：Web、Pwn、Reverse、Crypto、Misc......还能认识很多大佬，技术提升也快。

国内常见CTF平台：CTFHub、补天、漏洞盒子、XCTF......

**第三，打SRC。**

各大公司都有漏洞赏金计划：
- 阿里云安全应急响应中心
- 腾讯安全应急响应中心
- 字节跳动安全中心
- 百度安全应急响应中心
- 补天平台
- 漏洞盒子平台

找到漏洞提交了还有钱拿，合法赚钱，名利双收。一个高危漏洞，可能给你几千甚至几万块。

**第四，找正规公司实习/工作。**

安全公司、甲方安全团队，跟着做项目成长最快。正规公司都有完整的流程：授权书、合同、保密协议，什么都齐全，你放心干活就行。

**第五，参加护网比赛。**

每年各地都有各种护网演练，加入正规的红队去打比赛，合法合规练技术，还能认识很多同行。护网期间，还有专业人员带你，进步飞快。

**第六，读安全书籍和文章。**

看书是系统学习的好方法。《Web安全深度剖析》、《内网安全攻防》、《域渗透攻防》......这些书都不错。

关注一些安全博客、公众号，看看最新的漏洞和技术动态，保持学习。

### 1.6.3 三句话送给你

最后，送给所有想入行的新手三句话：

> **第一句：技术是中立的，但人要有善恶。**
> 同样的技术，可以用来做好事，也可以用来做坏事。你选择做哪种人？

> **第二句：搞安全的，先学做人，再学技术。**
> 技术学得再厉害，人品不行，早晚出事。

> **第三句：别走歪路，走了就回不了头了。**
> 网络安全这行，名声很重要。一旦有了黑产记录，你的前途就毁了。

记住这三句话，比学会十个漏洞都重要。

## 1.7 本章总结

这一章我们聊了很多，主要是让大家对红队有个整体的认识。

我们讲了：
- 红队是什么（模拟攻击的"好人"，帮别人测安全）
- 红蓝紫三方是什么关系（攻、守、裁判）
- 红队、黑客、渗透测试的区别（目的、授权、方法都不一样）
- 护网红队是怎么回事（官方组织的攻防演习）
- 护网的真实体验（14天里发生的故事）
- 红队需要什么技能（技术+非技术+人品）
- 最重要的——法律红线（没授权不能干，干了要坐牢）

你看，红队不是黑客，不是搞破坏的。恰恰相反，红队是帮助别人变得更安全的。

就像医生研究病毒，不是为了传播疾病，而是为了治病救人。

红队研究攻击技术，不是为了搞破坏，而是为了帮客户找出弱点，让他们在真正被黑客攻击的时候，能防得住。

这就是红队的意义。
