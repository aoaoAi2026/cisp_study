# 第三章 前期侦察—— 不打无准备之仗

> 第3章 | 预计学习时间：4小时 | 难度：⭐⭐

---

## 📖 写在前面

上一章我们讲了红队的作战地图，知道了红队攻击分几步。

这一章我们来讲第一步——前期侦察。

前面我们反复强调过一句话：**信息收集决定攻击的上限。**

为什么这么说？

因为红队不是瞎打的。你对目标了解得越多，能找到的突破口就越多，打进去的概率就越大。

我给你讲个真实的故事：

有一次护网，有个队伍去打一个政府部门。他们上来就是一顿扫描，扫了半天没扫出什么东西，换了几个漏洞打也打不进去。

结果另一个队伍，也是打同一个目标，他们先做了详细的信息收集。

他们发现：
- 这个部门的网站是用某CMS做的，版本比较老
- 他们有个子公司，子公司有个外包开发的系统
- 那个外包系统的GitHub仓库里有员工的公司邮箱
- 有个员工的GitHub里居然放着内部系统的测试密码

就靠这个密码，那个队伍直接登进去了，拿下了整个目标。

而前面那个队伍呢？因为没做信息收集，就这么错过了。

**你说，信息收集重要不重要？**

## 3.1 侦察到底是个啥？

### 3.1.1 用大白话解释侦察

侦察，说白了就是"摸底"。

在动手之前，先把目标的底细摸得清清楚楚：
- 他们有哪些网站？域名是什么？
- 服务器在哪？IP多少？
- 用了什么技术？什么框架？
- 有哪些员工？叫什么名字？
- 员工邮箱是什么？
- 公司在哪、有几栋楼？
- 有没有什么泄露的密码？
- 有没有什么旧系统、测试系统？
- 有没有外包、供应商？
- ......

就像小偷作案之前要踩点一样：

你见过小偷怎么踩点吗？他们会在目标房子附近转悠，看看：
- 家里几口人？
- 什么时候出门、什么时候回家？
- 窗户朝哪？门朝哪？
- 有没有保安？保安几点巡逻？
- 有没有摄像头？摄像头朝哪？
- 小区保安严不严？

摸清楚了再动手，成功率才高。

红队侦察也是一样的道理，只不过踩的是"网络上的点"。

### 3.1.2 侦察的分类

侦察一般分成两种：**被动侦察**和**主动侦察**。

啥区别呢？

**被动侦察：**

就是你不用直接去碰目标的系统，通过公开的信息就能收集到的。比如：
- 查域名注册信息（Whois）
- 搜索引擎找信息（Google、Baidu）
- 社交网站看员工信息（LinkedIn、脉脉）
- GitHub上找泄露的代码
- 查SSL证书透明度日志
- 查DNS历史记录
- 查历史漏洞记录
- ......

被动侦察的好处是：**目标发现不了你。**

因为你根本没碰他的系统，都是公开信息，他怎么知道你在收集他的信息？

**主动侦察：**

就是你得主动去碰目标的系统，才能收集到的信息。比如：
- 端口扫描（用Nmap扫目标开了哪些端口）
- 目录扫描（扫网站有哪些目录）
- 指纹识别（识别网站用的什么技术）
- 漏洞扫描（扫网站有没有漏洞）
- 尝试弱口令（爆破试试）
- ......

主动侦察的风险是：**可能被目标发现。**

你扫他端口，他的防火墙、IDS可能会报警。态势感知可能直接通知安全人员："有人正在扫描我们！"

> 💡 **红队思路：**
>
> **优先用被动侦察。**
>
> 能不碰目标就不碰目标。
>
> 被动侦察能收集到的信息，绝对不用主动侦察去拿。
>
> 毕竟，隐蔽第一嘛。
>
> 被动侦察做完了，再考虑主动侦察。而且主动侦察也要悠着点，别太猛。

### 3.1.3 侦察都收集些啥？

很多新手以为侦察就是扫扫端口、找找子域名，那你可太天真了。

红队的侦察，那叫一个全面，只要是能用上的信息，什么都收集。

我给你列个详细的清单你感受一下：

**技术信息：**
```
域名相关：
  • 主域名是什么
  • 有哪些子域名（子公司域名、业务域名）
  • 有哪些旁站（同IP域名）
  • DNS记录是什么
  • 有没有CDN？怎么绕过？
  • 有没有邮件服务器？SPF、DKIM记录？

IP相关：
  • IP地址/IP段
  • IP定位（在哪、哪个机房）
  • IP归属（公司自有机房？还是云服务器？）
  • 开放的所有端口
  • 每个端口对应的服务
  • 服务的版本号

Web应用：
  • 有哪些Web站点
  • 用的是什么CMS（WordPress？Dedecms？PHPCMS？）
  • 用的是什么框架（Spring Boot？Django？ThinkPHP？）
  • 用的是什么中间件（Apache？Nginx？Tomcat？）
  • 用的是什么编程语言（PHP？Java？Python？）
  • 有哪些功能模块
  • 有没有API接口
  • 有没有Swagger文档
  • 有没有测试页面

App和小程序：
  • 有没有App（iOS、Android）
  • 有没有微信小程序
  • App的包名是什么
  • App的框架是什么（React Native？Flutter？）
  • App里有没有绑定的域名

漏洞情报：
  • 目标用的产品/组件版本
  • 这个版本有什么已知漏洞（CVE）
  • 历史上有过什么漏洞
  • 有没有人之前挖过漏洞（漏洞库）

敏感信息：
  • 有没有泄露的源代码（GitHub、Gitee）
  • 有没有泄露的配置文件
  • 有没有备份文件
  • 有没有.phpinfo()页面
  • 有没有.git、.svn目录泄露
  • 有没有网盘泄露（百度网盘、GitHub）
```

**人员信息：**
```
基本信息：
  • 员工姓名
  • 员工职位（CTO、开发、运维、HR、销售？）
  • 员工邮箱（工作邮箱、私人邮箱）
  • 员工手机号
  • 员工社交账号（微信、微博、知乎、抖音）
  • 员工照片（做钓鱼头像用）

社工信息：
  • 员工喜欢用什么密码（社工库查询）
  • 员工有什么习惯（用什么社交媒体、论坛）
  • 员工的生日、宠物名、家乡（常被用作密码）
  • 员工的家人信息
  • 员工的工作经历

钓鱼信息：
  • 公司邮箱格式是什么（姓.名@域名？还是名.姓@域名？）
  • 有哪些邮件网关
  • 邮件安全做得好不好（有没有SPF、DKIM、DMARC）
  • 谁是IT管理员（钓鱼要钓大的）
```

**组织信息：**
```
公司背景：
  • 公司的组织架构是什么
  • 有哪些部门
  • 有哪些子公司/关联公司
  • 有哪些合作伙伴
  • 公司去年营收多少
  • 公司有多少人

业务信息：
  • 公司做什么业务
  • 用什么技术栈
  • 有哪些供应商
  • 有没有外包/代维
  • 用什么办公系统（OA、CRM、ERP）
  • 用什么云服务（阿里云？腾讯云？AWS？）

安全信息：
  • 公司有没有安全团队
  • 用什么安全设备（防火墙、IDS、EDR？）
  • 有没有安全运营中心（SOC）？
  • 之前有没有被黑过（数据泄露、漏洞曝光）
```

**物理信息（如果有条件）：**
```
公司位置：
  • 公司地址在哪
  • 有几栋楼
  • 有几个出入口

物理防护：
  • 有没有门禁
  • 有没有摄像头
  • 有没有保安巡逻
  • 保安几点换班

WiFi信息：
  • 能不能搜到公司WiFi
  • WiFi密码是什么（有人说、有人贴出来）
  • 有没有访客WiFi
```

你看，是不是比你想象的多得多？

**别嫌麻烦，这些信息看着没用，真到打的时候，指不定哪条就用上了。**

## 3.2 被动侦察—— 悄无声息摸家底

先来说说被动侦察，这是红队最喜欢的方式，因为安全，不会被发现。

### 3.2.1 域名信息收集

域名是最基础的信息了，一般客户会给你一个主域名，比如 `example.com`。

但你以为只有这一个域名吗？太天真了。

一个大公司，可能有几十个甚至上百个域名：
- 主站一个域名
- 子公司各有一个域名
- 不同业务用不同域名
- 收购的公司也有自己的域名
- 各种活动、专题页还有临时域名

**那怎么找呢？**

**方法一：查备案信息**

国内的网站基本都要备案，通过备案号或者公司名，可以反查到这个公司备案了哪些域名。

推荐网站：
- 站长工具备案查询（beian.cn）
- 爱站网备案查询
- 天眼查、企查查
- ICP备案查询网站

**方法二：查Whois信息反查**

Whois里有注册人、注册邮箱、注册电话，用这些信息反查，可以查到同一个人/同一个邮箱注册了哪些域名。

推荐网站：
- Whois.chinaz.com
- Whois.domaintools.com
- 爱站网Whois查询

**方法三：SSL证书透明度**

现在很多网站都用HTTPS，SSL证书里会写域名。有些网站可以查证书透明度日志，能找到很多子域名和关联域名。

推荐网站：
- crt.sh（最常用的）
- censys.io
- crt.shidai.jysx.org（中文界面）
- certificatedetails.com

**方法四：搜索引擎**

用Google或者百度搜，比如：
```
site:example.com
site:*.example.com
```

能找到很多收录的子域名。

还有各种高级搜索语法：
```
inurl:example.com
intitle:example.com
related:example.com
```

**方法五：DNS历史记录**

域名解析记录会变化，有些已经删除的子域名可能还能查到。

推荐网站：
- securitytrails.com
- dnsdumpster.com
- viewdns.info

### 3.2.2 子域名收集

找到主域名之后，下一步就是找子域名。

**子域名是什么？**

比如 `www.example.com` 是 `example.com` 的子域名，`mail.example.com`、`oa.example.com`、`vpn.example.com`、`test.example.com` 都是。

**为什么要找子域名？**

因为主站往往防护做得最好，WAF、防火墙、代码审计，什么都有。

反而是一些子域名，可能是测试站、边缘业务、子公司，防护比较弱，更容易打进去。

**子域名收集的方法：**

**1. 被动查询（不碰目标，推荐优先使用）**

用各种在线工具查：
- VirusTotal（virustotal.com）
- DNSDumpster（dnsdumpster.com）
- Censys（censys.io）
- Shodan（shodan.io）
- Fofa（fofa.so）- 国内最好用
- crt.sh
- Certificate Transparency网站

**2. 暴力枚举（主动发包，会被记录）**

准备一个子域名字典，一个一个去试，看能不能解析到IP。

常用工具：
- subfinder（快，用起来简单）
- amass（功能强大）
- OneForAll（国产神器，中文界面）
- sublist3r（老牌子）
- dnsx（Go写的，速度快）

**3. 证书透明度日志（非常好用！）**

前面说过的crt.sh之类的，这个方法特别好用，经常能找到一些意想不到的子域名。

特别是那些测试环境、子公司的域名，经常只有证书里能找到。

**4. 从JS文件里找**

很多网站的JS代码里会写API地址、其他子域名的链接。爬一下网站的JS文件，从中提取域名信息。

常用工具：
- LinkFinder
- jsfinder
- JSFinder

**5. 从其他来源找**

- GitHub代码里可能有子域名
- 公开的文档里可能有
- 漏洞报告里可能有
- 各种传感器、网络空间测绘平台

**一个小技巧：**

子域名收集不是一次就完事的，建议用多种方法、多个工具一起用，然后合并结果、去重。

不同的工具能找到的子域名不一样，用的方法越多，收集得就越全。

**推荐一个实战流程：**

1. 先用证书透明度查（crt.sh）- 最快最全
2. 再用综合工具扫（OneForAll、subfinder）
3. 然后爬JS文件
4. 最后合并去重

### 3.2.3 IP和端口信息

找到域名之后，下一步就是找对应的IP地址，然后看看IP上开了哪些端口、跑了哪些服务。

**怎么找IP？**

最简单的方法就是ping一下：
```bash
ping example.com
```

或者用nslookup、dig解析：
```bash
nslookup example.com
dig example.com
```

**但要注意几点：**

**1. CDN的问题**

有的网站用了CDN（内容分发网络），你ping到的是CDN的IP，不是真实IP。

怎么判断有没有CDN？
- 多个地点ping同一个域名，看返回的IP是否相同
- 如果不同，大概率用了CDN

怎么找真实IP？
- 查历史DNS记录（securitytrails.com）
- 查邮件服务器的MX记录
- 查子域名（有些子域名可能没接入CDN）
- 查网站的SSL证书（证书里可能有真实IP）
- 让目标给你发邮件，邮件头里有真实IP

**2. 多IP段的问题**

有的公司有很多IP段，可以通过ASN（自治系统号）反查等方式找。

怎么查ASN？
- 用BGP Lookup工具
- 用Shodan搜公司名
- 用Fofa搜公司名

**怎么找端口和服务？**

最经典的工具就是**Nmap**，这个不用说了，安全从业人员必备。

**但红队扫端口要注意：别扫太猛了，容易被发现。**

所以一般会：
- 减慢扫描速度（用-T2甚至-T1参数）
- 分片扫描（今天扫一批，明天扫一批）
- 只扫常用端口（先扫最常用的几十个）
- 用跳板扫描（用代理、VPS扫）
- 尽量用被动方式（Shodan、Fofa能查到的就别自己扫）

**被动查端口的方法：**

Shodan、Fofa、Censys这些搜索引擎，可以直接查某个IP开了哪些端口，不用自己扫，也就不会被发现。

比如：
- 在Shodan搜 `ip:1.2.3.4`
- 在Fofa搜 `ip="1.2.3.4"`

> 💡 **关于Shodan和Fofa：**
>
> 这俩是好东西，网络空间搜索引擎，相当于互联网的"百度"。
>
> 你可以搜"某个城市的摄像头"、"某个版本的Apache"、"开了3389端口的机器"......
>
> 红队做信息收集必备。
>
> 国内的话Fofa更好用（数据多，中文界面），
> 国外的话Shodan更全。

### 3.2.4 Web应用信息收集

找到了Web站点，接下来就要收集Web应用的信息了。

**收集什么呢？**

**1. 指纹识别**

就是搞清楚这个网站用的是什么技术：
- 用的什么CMS？（WordPress？Drupal？Dedecms？帝国CMS？）
- 用的什么中间件？（Apache？Nginx？IIS？Tomcat？）
- 用的什么编程语言？（PHP？Java？Python？C#？）
- 用的什么框架？（Spring Boot？Django？ThinkPHP？Laravel？）
- 用的什么数据库？（MySQL？PostgreSQL？MongoDB？）
- 用的什么WAF？（阿里云盾？腾讯云？安全狗？创宇盾？）

**怎么识别指纹？**

- 看HTTP响应头（Server、X-Powered-By等）
- 看页面特征（favicon、特定路径、特定注释）
- 看返回内容的特征
- 看Cookie名称
- 看HTML源码里的特征
- 用工具：WhatWeb、Wappalyzer、御剑指纹识别

**2. 目录和文件扫描**

看看网站有哪些目录、有没有后台、有没有备份文件、有没有phpinfo、有没有测试页面、有没有泄露的源码。

常用工具：
- dirsearch（Python写的，简单好用）
- gobuster（Go写的，速度快）
- ffuf（模糊测试工具，也可以扫目录）
- 御剑（国产经典）

**还是那句话：红队做目录扫描要悠着点，别扫太猛，容易被WAF封。**

建议：
- 先扫常见的目录（admin、backup、test、phpinfo等）
- 用个中等速度就行，别太猛
- 遇到WAF就换个思路

**3. 参数和接口收集**

网站有哪些参数、有哪些API接口、有没有Swagger文档、有没有测试接口。

可以用爬虫爬，也可以用Burp Suite代理着访问一遍，Burp会自动记录所有请求。

**4. 历史URL收集**

很多URL已经不存在了，但历史记录里还有。可以用Wayback Machine、URLScan、CommonCrawl来找历史URL。

工具：
- waybackurls
- gau（GetAllUrls）
- wayback_machine_downloader

### 3.2.5 人员信息收集

前面说的都是技术信息，现在我们来说说人员信息。

**别小看人员信息，很多时候社工比技术攻击管用多了。**

我给你讲个真实案例：

有一次红队打一个目标，技术扫描没找到什么漏洞。但是人员信息收集的时候，他们发现：

- 这个公司有个CTO，经常在社交媒体上发技术博客
- 他的GitHub用的是公司邮箱注册
- 他的GitHub代码仓库里有个commit，包含了公司VPN的密码

红队用这个密码直接连上了VPN，进入内网，然后一路打穿了整个网络。

**你说，人员信息重要不重要？**

**人员信息都收集啥？**
- 员工姓名
- 职位（CTO、开发、运维、HR、销售？）
- 邮箱（工作邮箱、私人邮箱）
- 手机号
- 社交账号
- 照片（做钓鱼头像用）
- 工作经历
- 公开的密码（社工库里可能有）

**从哪收集？**

**1. 领英（LinkedIn）**

- 搜公司名，能找到一大堆员工
- 能看到职位、工作经历
- 很多人会把邮箱写在上面

**2. 脉脉、BOSS直聘**

- 国内的职场社交平台
- 也能找到很多员工信息

**3. 微博、知乎、抖音**

- 很多员工会在上面发东西
- 有时候能挖到有用的信息

**4. 公司官网**

- 很多公司官网有"关于我们"、"团队介绍"页面
- 上面有高管的照片和介绍

**5. 新闻报道、会议演讲**

- 公司的技术人员出去演讲、接受采访
- 也是信息来源

**6. GitHub、Gitee**

- 很多员工会把工作代码传到GitHub
- 运气好的话能找到：
  - 内部系统的源代码
  - 配置文件（可能含密码）
  - 数据库连接字符串
  - API密钥
  - 内网IP地址

**7. 密码泄露库（社工库）**

- 这个就不多说了，懂的都懂
- 但要提醒一句：社工库这个东西法律风险比较大，正规项目一般不用，自己知道就行

### 3.2.6 GitHub信息收集

**专门把GitHub拎出来讲，因为它太重要了。**

很多开发人员安全意识不强，把公司的代码、配置文件、甚至密码都传到GitHub上了。红队最喜欢这种"送分题"。

**怎么在GitHub上找信息？**

**搜索关键词：**
- 公司名
- 公司域名（如 `example.com`）
- 公司的项目名
- 员工的名字/ID
- 敏感词：password、pass、key、secret、token、api_key、db_password
- 数据库连接字符串
- 内网IP地址
- JWT Secret、Session Secret

**搜索技巧：**
- 按仓库搜、按代码搜、按issue搜
- 限制语言（`language:java`）
- 限制时间（`pushed:>2024-01-01`）
- 组合搜索

**GitHub高级搜索示例：**
```
example.com language:java password
example.com filename:config.inc
example.com filename:database
example.com api_key
```

**推荐工具：**
- GitRob（自动扫描GitHub敏感信息）
- GitHacker（下载整个仓库，找敏感信息）
- trufflehog（找密钥、密码之类的）
- Hawkeye（GitHub敏感信息扫描）
- x-safe（国产工具）

**其实最好用的还是GitHub自己的搜索**（github.com/search）

> 💡 **真实案例：**
>
> 我有个朋友做红队，打一个目标打了一周没打进去，最后在GitHub上找到了一个员工的私人仓库，里面放着内部OA系统的完整源代码，还有数据库密码。
>
> 直接就打穿了。
>
> 你说，这找谁说理去？

### 3.2.7 其他被动侦察的方法

再补充几个方法，都是实战中很有用的。

**1. 漏洞情报收集**

- 看看目标用的产品/组件最近有没有爆什么漏洞
- 关注CNVD、CNNVD、CVE这些漏洞库
- 如果有0day或者刚爆出来的漏洞，直接拿EXP去打，省老事了

**2. 历史漏洞信息**

- 看看这个网站/公司以前有没有被曝过漏洞
- 漏洞平台（补天、漏洞盒子、CNVD）上搜一搜
- 说不定能找到之前的漏洞详情，虽然可能已经修了，但能提供思路

**3. 网盘搜索**

- 有人会把内部资料传到百度网盘、阿里云盘
- 可以用网盘搜索工具找：`凌风云`、`小白盘`、`盘搜搜`

**4. 乌云镜像/历史漏洞库**

- 乌云曾经是国内最大的漏洞平台，虽然关了，但镜像还在
- 可以查查目标有没有历史漏洞

**5. 传感器和网络测绘平台**

- ZoomEye（知道创宇）
- FOFA（白帽汇）
- Shodan
- Censys
- Quake（360）

这些平台能帮你：
- 找某个地区的所有摄像头
- 找某个版本的Web服务器
- 找开了某个端口的所有IP
- 找某个关键词的所有资产

**6. DNS记录查询**

- 查各种DNS记录：MX、 TXT、SPF、DKIM、DMARC
- 有时候能从TXT记录里找到内部域名、内网IP

## 3.3 主动侦察—— 小心翼翼试探

被动侦察做得差不多了，接下来就是主动侦察了。

**主动侦察就是你主动去碰目标的系统，比如扫端口、扫目录、测漏洞之类的。**

### 3.3.1 端口扫描

端口扫描是最基础的主动侦察，用Nmap就行。

**但红队扫端口有讲究，不能像做渗透测试那样噼里啪啦全端口扫，那样太容易被发现了。**

**红队扫端口的原则：**

**1. 慢一点**

别用默认速度，用 `-T2` 甚至 `-T1`，慢点扫，不容易触发告警。

```bash
# 慢速扫描常用端口
nmap -T2 -sV -p 21,22,23,25,80,443,445,3389,8080,8443 1.2.3.4

# 慢速全端口扫描
nmap -T2 -p- -sV 1.2.3.4
```

**2. 分片扫**

不要一次扫完，分成几批，今天扫一批，明天扫一批。

**3. 先扫常用端口**

先扫最常用的几十个端口（Web服务、SSH、RDP、SMB、数据库等），有收获再说，不要上来就全端口。

**4. 用跳板扫**

不要用自己的IP扫，用代理、用VPS、用肉鸡，就算被封了也不怕。

```bash
# 通过代理扫描
proxychains nmap -T2 -sV -p 80,443 1.2.3.4
```

**5. 能被动查的就不主动扫**

Shodan、Fofa上能查到的端口和服务，就别自己扫了。

### 3.3.2 Web应用探测

找到了Web站点，接下来就要深入探测一下。

**探测什么？**

**1. 指纹识别**

前面说过了，用WhatWeb、Wappalyzer、御剑指纹识别之类的工具。

**2. WAF探测**

看看有没有WAF，是什么WAF。

怎么测？
- 传个带特殊字符的参数，看看会不会被拦截
- 看响应头有没有WAF的特征
- 或者用工具：wafw00f

```bash
wafw00f -a http://example.com
```

**3. 目录扫描**

还是那句话，慢点扫，别太猛。

先扫常见的目录：
```
admin/
backup/
test/
phpmyadmin/
phpinfo.php
robots.txt
.svn/entries
.git/config
```

工具：dirsearch、gobuster、御剑

```bash
# dirsearch示例
python3 dirsearch.py -u http://example.com -e php,html,js -t 10

# gobuster示例
gobuster dir -u http://example.com -w /usr/share/wordlists/dirb/common.txt
```

**4. 漏洞探测**

看情况，有明显漏洞的可以先试试。

但注意：**不要上来就用扫描器扫，太吵了。**

先手动测几个常见的：
- URL参数有没有SQL注入
- 搜索框有没有XSS
- 文件上传点有没有过滤
- 登录框能不能爆破

有发现再用工具深入测。

### 3.3.3 漏洞扫描

**要不要用漏洞扫描器？比如Nessus、AWVS、Xray这些。**

**我的建议是：慎用。**

为啥？

因为漏洞扫描器太吵了，发的包太多、太明显，很容易被蓝队发现。

你这边一开扫，那边态势感知马上就报警了。

**那什么时候用？**

- 目标防护比较弱的时候（政府网站、学校网站之类）
- 已经拿到内网权限了，在内网里扫（内网一般监控没那么多）
- 有足够的代理和跳板，不怕被溯源
- 项目快结束了还没打进去，搏一搏

**如果一定要扫，也尽量：**

- 调低扫描速度
- 只扫关键漏洞，别全扫
- 用代理
- 分时段扫
- 优先扫边缘资产（子公司、测试站）

### 3.3.4 弱口令爆破

弱口令爆破也算主动侦察，而且有时候特别管用。

**常见弱口令类型：**

```
系统弱口令：
  • admin/admin
  • admin/password
  • admin/123456
  • root/root
  • root/toor

Web后台弱口令：
  • admin/admin
  • admin/admin888
  • admin/123456
  • admin/manager
  • guest/guest

数据库弱口令：
  • root/root
  • root/123456
  • sa/sa
  • postgres/postgres

常用弱口令字典：
  • top100
  • top1000
  • 信息化发展年度报告里的常用密码
```

**爆破工具：**

- Burp Suite（Web登录爆破）
- Hydra（支持很多协议：SSH、RDP、SMB、FTP、MySQL等）
- Medusa
- CrackMapExec（内网横向用的）

**红队爆破注意事项：**

1. **先收集信息再爆破** - 知道用户名再爆，成功率更高
2. **注意账号锁定** - 有些系统登录失败几次就锁了，别一下爆太多
3. **用代理** - 防止IP被封
4. **悠着点** - 别太猛，容易被发现

## 3.4 信息整理与分析

**信息收集了一大堆，别就那么扔着，要整理、要分析。**

很多新手收集了一堆信息，但不会分析，那信息就是垃圾，没用。

### 3.4.1 信息整理

先把收集到的信息整理一下，分类放好：

```
资产清单：
  • 域名列表
  • 子域名列表
  • IP/IP段列表
  • Web站点列表（带指纹识别结果）
  • 端口服务列表
  • App/小程序列表

人员清单：
  • 员工姓名列表
  • 邮箱列表
  • 职位信息
  • 部门信息
  • IT管理员信息

敏感信息：
  • 泄露的密码
  • 泄露的源代码
  • 泄露的配置文件
  • 内部域名/IP

漏洞情报：
  • 已知的漏洞
  • 可能存在的弱点
  • 历史上的漏洞

突破口候选：
  • 可能的Web漏洞点
  • 可能的弱口令
  • 可能的社工对象
  • 可能的第三方切入点
```

**整理得清清楚楚的，后面用的时候才方便。**

建议用Excel或者数据库整理，关键字段要包含：
- 资产名称
- 类型（域名/IP/子域名/Web站点）
- 备注（发现了什么、有什么问题）
- 发现时间
- 优先级

### 3.4.2 信息分析

整理完了就要分析：哪些东西是有用的？哪些可能是突破口？先打哪个？后打哪个？

**分析思路：**

**1. 找最薄弱的点**

哪个系统看起来最菜？
- 哪个是测试站？（一般防护最弱）
- 哪个用的技术最老？（老版本漏洞多）
- 哪个是边缘业务？（不受重视）
- 哪个防护最少？（没WAF、没防火墙）

**先从软柿子捏。**

**2. 找性价比最高的点**

哪个点打进去之后收益最大？
- 能不能直接拿到内网权限？
- 能不能拿到很多凭据？
- 能不能接触到核心系统？
- 是不是域内机器？

**优先打那些打进去就能扩大战果的目标。**

**3. 找风险最低的点**

哪个点打进去最不容易被发现？
- 是不是边缘资产？
- 是不是测试站？
- 打完会不会打草惊蛇？

**如果不确定，先打风险低的试试手。**

**4. 制定攻击路径**

根据分析结果，画几条可能的攻击路径：
- 路径A：子域名A → Web漏洞 → WebShell → 内网
- 路径B：钓鱼邮件 → 办公电脑 → 内网
- 路径C：子公司漏洞 → 横向到总部

**先试最有希望的那条，不行再换。**

> 💡 **红队思维：**
>
> 不要把鸡蛋放在同一个篮子里。
>
> 多准备几条攻击路径，这条路走不通，马上换另一条。
>
> 别死磕一个点，浪费时间还容易被发现。

## 3.6 侦察的几个实用技巧

最后再分享几个实战中有用的小技巧。

### 3.6.1 从已知推未知

找到一个子域名，看看它的命名规律。

比如：
- 找到了 `oa.example.com`（OA系统）
- 那有没有 `mail.example.com`？（邮件系统）
- 有没有 `vpn.example.com`？（VPN）
- 有没有 `test.example.com`？（测试站）
- 有没有 `dev.example.com`？（开发环境）
- 有没有 `staging.example.com`？（预发布环境）

找到一个员工的邮箱，看看命名规律。

比如：
- 张三的邮箱是 `zhangsan@example.com`
- 是不是"拼音@域名"的格式？
- 那李四是不是 `lisi@example.com`？
- 王小明是不是 `wangxiaoming@example.com`？
- 领导是不是 `lingdao@example.com`？

**找到规律之后，就可以批量生成了。**

### 3.6.2 关注"边缘资产"

什么是边缘资产？就是那些不是核心业务、不怎么受重视的资产。

比如：
- 测试站（test.xxx.com、dev.xxx.com）
- 旧系统（old.xxx.com、2018.xxx.com）
- 子公司的网站
- 供应商的网站
- 年会、活动的专题页
- 内部工具的外网入口（jumpserver、gitlab、jenkins）
- 微信小程序
- App

**这些资产往往防护比较弱，容易被忽略，但价值可能很大。**

红队特别喜欢从这些地方入手。

### 3.6.3 善用"历史信息"

很多东西现在删了，但历史上存在过。

比如：

**1. Wayback Machine（互联网档案馆）**

能重现网站以前的样子。可能以前有漏洞，现在修了，但能提供思路。

网站：web.archive.org

**2. GitHub的历史提交**

可能密码之前提交过，后来删了，但历史记录里还有。

```bash
# 用trufflehog查历史
trufflehog --repo https://github.com/user/repo
```

**3. 漏洞平台的历史漏洞**

之前报过的漏洞，可能修得不干净，还能利用。

### 3.6.4 信息收集要贯穿始终

**别以为信息收集只是"前期"的事，打进去之后还要继续收集。**

每拿到一台机器，都要重新收集信息：
- 这台机器上有什么？
- 有哪些凭据？
- 能访问哪些网段？
- 有哪些新的目标？
- 域内有什么新信息？

**信息收集是贯穿整个攻击过程的，不是干完第一步就完事了。**
