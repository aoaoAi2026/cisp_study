## 4.1 C2到底是个啥？

### 4.1.1 用大白话解释C2

C2，全称是Command and Control，中文叫"命令与控制"。

啥意思呢？就是攻击者远程控制被攻陷的机器的那一套东西。

想象一下：

你打进去了一台机器，总不能每次都跑到跟前去操作吧？你得有个办法远程控制它，让它干啥就干啥，还得能传文件、拿数据、执行命令。

这就是C2干的事。

简单说：

> **C2就是红队的"遥控系统"，一边连着攻击者，一边连着被打进去的机器，攻击者通过C2给被控机器发命令，被控机器把结果传回来。**

C2就像遥控器，你在家里（攻击者这边）拿着遥控器按按钮，电视（被控机器）就会执行对应的操作。

只不过这个"电视"是别人的电脑，而这个"遥控器"需要你自己搭。

### 4.1.2 C2是怎么工作的？

C2的工作原理其实挺简单的：

```
攻击者（你）
   │
   │ 发命令
   ▼
C2服务器（VPS）
   │
   │ 通信
   ▼
被控机器（被打进去的机器）
```

步骤是这样的：

1. **被控机器上运行着"木马"（也叫Beacon、Payload、Agent）**
   这个木马是攻击者提前种上去的，它会偷偷运行在后台。

2. **木马主动连接C2服务器**
   木马会每隔一段时间（比如每几秒）去问C2服务器："有没有给我的命令？"

3. **攻击者在C2客户端上下命令**
   攻击者通过C2客户端（比如Cobalt Strike的图形界面）下达命令，比如：
   - "执行这个命令"
   - "上传这个文件"
   - "下载这个文件"
   - "截图"
   - "抓密码"

4. **C2服务器把命令转发给木马**
   C2服务器像个中转站，把攻击者的命令传给被控机器上的木马。

5. **木马执行命令，把结果传回**
   木马执行完命令，把结果（命令输出、截图、文件内容等）再传回给C2服务器。

6. **攻击者查看结果**
   攻击者在C2客户端上就能看到命令执行的结果了。

就这么个流程，一问一答，循环往复。

**那为啥要中间放个C2服务器，不能直接连被控机器吗？**

主要有两个原因：

**第一，被控机器在内网，没有公网IP。**

被控机器可能在你家、公司、学校的局域网里，它没有公网IP，你根本直接连不上它。所以得让它主动出来连你。

**第二，隐蔽性。**

有个中间层，就算被控机器被发现了，蓝队也只能追到C2服务器，追不到你本人。就像间谍不会直接和总部联系，中间有好几个联络点，就算一个联络点暴露了，也不会影响全局。

### 4.1.3 C2的核心组件

一个完整的C2系统，一般包含这几个部分：

**1. C2服务器（Team Server）**
   就是一台VPS或者服务器，攻击者把C2工具安装在这上面。被控机器会连到这里。

**2. C2客户端（Client/UI）**
   攻击者操作的界面，可以是图形界面也可以是命令行。攻击者通过客户端下达命令、查看结果。

**3. 负载/后门（Payload/Beacon/Agent）**
   运行在被控机器上的东西，负责接收命令、执行命令、返回结果。

**4. 通信协议**
   怎么传输数据？用什么协议？HTTP、HTTPS、DNS、TCP...都有。不同的协议有不同的特点。

**5. 配置文件（Profile）**
   控制C2的行为、流量特征等。比如Malleable C2 Profile可以让流量看起来像正常网站的流量。

### 4.1.4 常用的C2工具有哪些？

现在市面上C2工具可多了，我给你详细讲讲几个最有名的：

**1. Cobalt Strike（简称CS）**

这是红圈里最火的C2，没有之一。
- 功能强大，界面友好
- 支持团队协作（多人可以同时操作）
- 有很多插件和脚本
- 隐蔽性比较好，支持域前置等高级功能
- 缺点：收费，而且不便宜（正版要几万块一年）
- 因为太火了，所以查杀也比较多，但对付一般场景够用

**2. Metasploit Framework（简称MSF）**

这是安全圈最著名的开源工具。
- 开源免费
- 漏洞利用库超级全（几万个Exploit）
- 也能当C2用（Meterpreter）
- 缺点：太出名了，免杀难做
- 适合新手学习，原理都是通的

**3. Metasploit的Meterpreter**

Meterpreter是MSF默认的Payload。
- 运行在内存里，不写文件（相对隐蔽）
- 支持很多功能：提权、抓密码、端口转发...
- 缺点：特征太明显，EDR一抓一个准

**4. Empire**

- 开源的，PowerShell为主
- 适合Windows环境
- 现在好像维护得少了

**5. Covenant**

- .NET平台的C2
- 开源免费
- 界面还不错

**6. Sliver**

- 开源的，跨平台（Windows、Linux、Mac都能用）
- 最近挺火的
- 被称为"穷人的CS"
- 功能也比较全

**7. Brute Ratel（简称BRc4）**

- 商业工具，比较贵
- 号称"下一代C2"
- 功能强大，隐蔽性好

**8. Havoc**

- 开源的
- 界面挺好看的
- 一直在更新

**9. Mythic**

- 开源的，跨平台
- 用Go写的
- 模块化设计

**10. 甚至自己写一个也不是不行**

有些高级红队会自己写C2工具，不让别人知道，但开发成本太高，一般人搞不来。

> 💡 **新手建议：**
>
> 先从MSF开始学，原理都是通的，学会了MSF再去用CS之类的，上手很快。
>
> 别一开始就追求什么"高大上"的工具，原理搞懂了最重要。
>
> 工具只是手段，思路才是关键。

## 4.2 为啥要搭"基础设施"？

很多新手觉得C2不就是"买个VPS，装个CS，直接用"吗？还需要什么基础设施？

如果只是打个靶机、做个小测试，那确实够了。但如果是做红队项目、打护网，那就不够了。

### 4.2.1 直接用VPS有啥问题？

直接拿一个VPS当C2，问题可多了：

**问题一：容易被溯源**

被控机器直接连你的VPS，蓝队一看流量就知道是C2，顺着IP就找过来了。一查是哪个公司的服务器，再查谁买的（你注册的信息），你就暴露了。

**问题二：容易被封**

蓝队发现了之后，直接把你的C2 IP封了，你所有的被控机器都掉线了，一下就全没了。辛苦打进去的，一夜回到解放前。

**问题三：没有冗余**

就一个C2服务器，挂了就全挂了，连个备份都没有。

**问题四：流量特征明显**

C2的流量有特征，直接用的话，入侵检测系统一抓一个准。

### 4.2.2 基础设施是啥？

那基础设施是啥？简单说，就是在你和被控机器之间，加了好多层"掩护"，让蓝队找不到你，就算找到了一层，还有下一层。

就像间谍的联络方式——

```
普通人：间谍 ↔ 总部（直接联系，一暴露就全暴露）
间谍：间谍 → 联络点A → 联络点B → 联络点C → 总部（多级跳板，追踪困难）
```

C2基础设施也是这个思路：
```
简单：C2服务器 → 你
复杂：被控机器 → CDN → 跳转机A → 跳转机B → C2服务器 → 你
```

### 4.2.3 基础设施的几个原则

搭基础设施，要遵守几个原则：

```
1. 隐藏自己
   不能让蓝队轻易找到真实的C2服务器，
   更不能找到你本人。

2. 有冗余
   不能一个点挂了就全完了，
   要有备份，有备用线路。

3. 流量正常
   C2的流量要看起来像正常流量，
   不能一眼就看出来是C2。

4. 方便操作
   不能为了隐蔽搞太复杂，
   自己用起来都费劲，
   那也不行。

5. 安全第一
   基础设施本身也要安全，
   不能被人一打就穿。
```

## 4.3 经典的C2架构

我们来看看几种常见的C2架构，从简单到复杂，你可以根据项目需要选。

### 4.3.1 最简单的：直连型

```
被控机器 → 直接连 → C2服务器 → 你
```

最简单的，被控机器直接连C2服务器，你直接连C2服务器操作。

**优点：**
- 简单，搭起来快
- 延迟低，操作流畅

**缺点：**
- 完全没有隐蔽性
- 一抓一个准
- 被封了就全没了

**适用场景：**
- 打靶机（HackTheBox、VulnHub之类）
- 内部测试
- 短期、无所谓暴露的项目

### 4.3.2 入门级：单跳转型

```
被控机器 → 跳转机 → C2服务器 → 你
```

在被控机器和C2服务器之间，加一个"跳转机"（也叫"前置机"、"肉鸡"）。

被控机器连跳转机，转发机再转发到真实的C2服务器。这样就算跳转机被发现了，蓝队也只能追到跳转机，追不到真实的C2。

**跳转机可以是啥？**
- 被攻陷的第三方服务器
- 云函数（阿里云函数、腾讯云函数、AWS Lambda）
- CDN
- 代理服务器
- GitHub Pages、Netlify（当静态资源转发）
- ......

**优点：**
- 比直连安全一点
- 搭建难度不高

**缺点：**
- 还是只有一条路
- 跳转机挂了就全完了

### 4.3.3 进阶级：域前置 + CDN

```
被控机器 → CDN/云函数 → C2服务器 → 你
```

这个就高级一点了，用CDN或者云函数当"中间层"。

被控机器连接的是CDN的地址，CDN再把流量转发到真实的C2服务器。

为什么叫"域前置"？因为你可以用一个合法的域名（比如阿里云的域名）当"门面"，流量看起来像是在访问这个合法域名，但实际上被转到了你的C2。

**优点：**
- 隐蔽性好，流量看起来正常
- 不容易被封IP（CDN IP太多了）
- 成本不高

**缺点：**
- 配置稍微麻烦一点
- 延迟会高一点

**适用场景：**
- 大多数红队项目
- 护网比赛

### 4.3.4 高级：多层跳转 + 多C2

```
被控机器 → 多个跳转层 → 多个C2服务器 → 你
```

更高级的玩法，多层跳转，多个C2，互相备份。

就算其中一个C2被发现了，还有其他的；就算一条路断了，还有其他路。

**优点：**
- 非常隐蔽，很难溯源
- 冗余度高，不怕单点故障

**缺点：**
- 搭建复杂，维护麻烦
- 成本高
- 延迟高

**适用场景：**
- 大型、长期的红队项目
- 对隐蔽性要求特别高的场景

> 💡 **新手建议：**
>
> 不用一上来就搞最复杂的，先从简单的开始。
>
> 能把直连和单跳转玩明白，应付大多数场景就够了。等熟练了再往复杂了搞。
>
> 循序渐进，不要一口吃成胖子。

## 4.4 手把手搭一个C2

说了这么多理论，我们来实际操作一下，搭一个最简单的C2环境。

就用最经典的组合：**Linux VPS + Cobalt Strike**

（没有CS也没关系，用MSF也是一样的思路，我们主要讲思路）

### 4.4.1 准备工作

你需要准备：
- 一台Linux VPS（公网IP，系统推荐Ubuntu或CentOS）
- Cobalt Strike安装包（或者MSF也行）
- 一个域名（可选，但建议有）
- 一个SSH客户端（FinalShell、Xshell、PuTTY都可以）

### 4.4.2 第一步：买VPS

怎么选VPS？

**配置要求：**
- 配置不用太高，1核2G就够了
- 带宽不用太大，C2流量不大
- 系统选Linux，推荐Ubuntu 20.04或CentOS 7

**选哪里的？**
- 国外的话，搬瓦工（BandwagonHost）、Vultr、DigitalOcean都可以
- 国内的话，阿里云、腾讯云也行，但要注意国内服务器要备案，而且监控比较严

**注意事项：**
- 不要用自己的真实信息买！（学习测试没关系）
- 不要用自己常用的邮箱、手机号
- 最好用虚拟信息，或者找靠谱的渠道

> ⚠️ **重要提醒：**
>
> 用国内的服务器做C2要特别小心，很容易被查到。
>
> 学习测试的话没关系，真做项目的话还是用国外的吧。
>
> 而且最好用假信息、虚拟卡，别留下自己的痕迹。

### 4.4.3 第二步：连接VPS

买好VPS之后，你会收到：
- VPS的IP地址（比如 1.2.3.4）
- SSH端口（默认22，或者随机端口）
- root密码

用SSH客户端连接上去：

```bash
ssh root@1.2.3.4 -p 22
```

输入密码，就登进去了。

> 💡 **小技巧：**
>
> 可以先在本地生成SSH密钥对，然后把公钥传到VPS上，以后就不用输入密码了。
>
> ```bash
> # 本地生成密钥
> ssh-keygen -t rsa
> # 传公钥到VPS
> ssh-copy-id root@1.2.3.4
> ```

### 4.4.4 第三步：安装Java

CS是Java写的，得装Java环境。

```bash
# Ubuntu
apt update
apt install openjdk-11-jdk

# CentOS
yum install java-11-openjdk
```

装完验证一下：

```bash
java -version
```

应该能看到Java版本信息。

### 4.4.5 第四步：上传并配置CS

**1. 把安装包传到VPS上**

在你本地电脑上，把CS的压缩包（比如 cobaltstrike.tar.gz）传到VPS上。

可以用scp命令：

```bash
# 在本地的终端里执行
scp cobaltstrike.tar.gz root@1.2.3.4:/root/
```

**2. 解压**

```bash
# 在VPS上执行
cd /root/
tar zxvf cobaltstrike.tar.gz
cd cobaltstrike
```

**3. 查看目录结构**

```bash
ls -la
```

应该能看到这些文件：
- teamserver（C2服务器启动脚本）
- cobaltstrike（C2客户端启动脚本）
- update（更新脚本）
- 各种资源文件

### 4.4.6 第五步：启动Teamserver

**启动C2服务器（Teamserver）：**

```bash
./teamserver 你的VPS的IP 连接密码
```

比如：

```bash
./teamserver 1.2.3.4 mypassword123
```

> ⚠️ 注意：
>
> 这里要填VPS的公网IP，不能填127.0.0.1或内网IP。
>
> 连接密码是你自己设的，后面客户端连接要用。

**启动之后应该看到类似这样的输出：**

```
[*] Starting Cobalt Strike Strike Server 4.x
[+] Team server is up on 50050
[*] SHA256 hash of the team server: xxxxxxxx
```

就说明C2服务器起来了。

**让Teamserver在后台运行：**

上面的命令会占用终端，如果你关掉终端，C2就掉了。所以最好用screen或nohup让它在后台跑：

```bash
# 用screen
apt install screen
screen -S cs
./teamserver 1.2.3.4 mypassword123
# 然后按 Ctrl+A，D 退出screen
```

### 4.4.7 第六步：启动CS客户端

现在你可以在你自己的电脑上启动C2客户端了。

Cobalt Strike是客户端-服务器架构的：
- Teamserver跑在VPS上，是服务端
- 你本地电脑上的cobaltstrike程序是客户端

在你本地电脑上，双击cobaltstrike图标，启动客户端。

会弹出一个对话框：
- Host：填VPS的IP（1.2.3.4）
- Port：默认50050
- User：随便起个名字，比如attacker
- Password：刚才设的连接密码

点Connect，就连接上了。

### 4.4.8 第七步：创建Listener（监听器）

连上之后，先创建一个Listener（监听器）。

Listener就是等待被控机器连接的"耳朵"。

**在CS里操作：**

1. 点击左上角的菜单 → Listeners
2. 点击Add按钮，新增一个Listener
3. 填写配置：
   - Name：随便起个名字，比如http-listen
   - Payload：选Beacon（这是最常用的）
   - Bindto：默认就行
   - HTTP Host（Callback）：填你的VPS IP，或者域名
   - HTTP Port（Callback）：填端口，比如80或443（常用端口）
4. 点击Save保存

> 💡 **注意：**
>
> 如果填的是80或443端口，要确保VPS上这些端口是开放的。
>
> 有的VPS默认只开放22端口，其他端口要用iptables或安全组放行。

Listener创建成功之后，状态应该是绿色的"active"。

### 4.4.9 第八步：生成一个马（Beacon）

Listener创建好了，现在要生成一个"木马"（也叫Payload、Beacon），放到被控机器上运行。

**在CS里操作：**

1. 点击菜单 → Attacks → Packages → Windows Executable
2. 在弹出的框里：
   - Listener：选刚才创建的Listener
   - Output：选Windows EXE
3. 点击Generate，保存exe文件

这样就生成一个exe文件了，比如叫 beacon.exe。

把这个文件想办法传到目标机器上（比如U盘、邮件、下载链接等），让目标机器运行它。

### 4.4.10 第九步：等待上线

目标机器运行beacon.exe之后，CS里就会看到一个新的Beacon上线了。

一般会显示：
- 机器名
- IP地址
- 用户名
- 权限（是普通用户还是管理员）
- 系统类型（Windows 10还是Windows Server等）

右键点击Beacon，可以：
- Interact（进入交互模式，下命令）
- Sleep（设置心跳间隔）
- Shell（执行命令）
- ...

**进入交互模式试试：**

点Interact，进入Beacon交互界面，在底部的输入框里输入命令，比如：

```bash
# 查看当前目录
pwd

# 查看用户
whoami

# 查看系统信息
hostname
sysinfo
```

能执行命令，就说明C2搭好了！

> ⚠️ **再次提醒：**
>
> 这个生成的马是没有免杀的，一上去就会被杀软干掉。
>
> 免杀我们后面章节再讲，这里先了解流程。

## 4.5 怎么隐藏C2？

前面说了，直接把C2暴露在外面不行，得想办法藏起来。

怎么藏呢？我们来讲几种常用的方法。

### 4.5.1 方法一：域名 + CDN（最常用）

这是最常用也最好用的方法之一。

**原理：**

```
被控机器 → CDN节点 → 真实C2服务器
```

被控机器不直接连你的VPS，而是连一个域名，这个域名解析到CDN上，CDN再把流量转发到你的真实C2。

这样蓝队看到的是CDN的IP，不不是你真实的C2 IP。而且CDN的IP成千上万，封都封不过来。

**怎么弄？**

大概步骤：

**1. 买一个域名**

不用太好的域名，便宜的就行。比如 Godaddy、Namecheap、阿里云、腾讯云都能买。

买完之后把域名的DNS服务器改成Cloudflare的（如果你用Cloudflare的话）。

**2. 注册Cloudflare账号，开启免费CDN**

Cloudflare是全球最大的CDN服务商之一，免费版就挺好用。

注册完之后，把你的域名添加进去，Cloudflare会自动扫描DNS记录。

**3. 配置CDN**

在Cloudflare的DNS设置里：
- 添加一条A记录，指到你的VPS IP（这是CDN回源的地址）
- 开启代理（橙色云图标）

这样流量就经过Cloudflare了。

**4. 开启Cloudflare的CDN IP隐藏功能**

Cloudflare默认会暴露真实IP，这不行。

需要在Cloudflare里开启一些设置，比如：
- 开启"Proxy IP"隐藏
- 关闭一些可能泄露IP的功能

> 💡 **注意：**
>
> Cloudflare虽然能隐藏源站IP，但Cloudflare本身的一些服务（比如Cloudflare Workers、Sites）如果配置不当，可能会泄露真实IP。
>
> 要仔细配置。

**5. 配置C2的Profile**

让C2的流量看起来像正常的网站流量。

CS里有个东西叫Malleable C2 Profile，就是用来自定义流量特征的。你可以写一个Profile，让CS的流量看起来像WordPress博客、像京东商城、像任何你想模仿的网站。

**6. 生成马的时候用域名**

Payload里填域名，不要填IP。

比如原来是 `http://1.2.3.4:80`，改成 `http://yourdomain.com:80`。

这样被控机器连的是你的域名，流量经过CDN转发，蓝队很难直接找到你真实的C2 IP。

> 💡 **小知识：这就是"域前置"的基础版。**
>
> 更高级的域前置可以用大公司的域名当"门面"，比如用微软的、亚马逊的域名，流量看起来就像是在访问大公司的网站，隐蔽性更强。
>
> 不过那种配置起来比较麻烦，而且很多CDN已经封了这种用法。

### 4.5.2 方法二：云函数/Serverless（最近很火）

这个是最近几年特别火的玩法。

**啥是云函数？**

就是云计算厂商提供的"函数即服务"，你写一段代码传上去，它给你一个URL，访问这个URL就会执行你的代码。用多少算多少，很便宜。

**常见的云函数服务：**
- AWS Lambda
- 阿里云函数计算
- 腾讯云函数
- 华为云函数工作流

**怎么用来做C2跳转？**

原理很简单：

```
被控机器 → 云函数URL → 真实C2服务器
```

被控机器访问云函数的URL，云函数把请求转发给真实的C2，再把C2的响应返回给被控机器。

**优点：**
- 云函数的域名是官方的，比如腾讯云函数就是 `*.ap-beijing.tencentcloudapi.com` 这种域名一般不会被封
- 流量看起来很正常
- 溯源困难，总不能去找腾讯吧
- 成本很低，用不了多少钱

**缺点：**
- 配置稍微麻烦一点
- 延迟会高一点
- 云函数有超时限制，长时间不响应的请求会超时

**怎么弄？**

大概思路：

1. 注册一个云平台的账号（腾讯云、阿里云、AWS都可以）

2. 创建一个云函数

3. 写一个简单的转发脚本（把请求转发到你的C2）

4. 生成马的时候填云函数的URL

具体配置方法网上教程很多，这里就不细讲了，主要是让大家知道有这么个东西。

### 4.5.3 方法三：域名轮转 + 多C2

这个更狠一点，不只用一个域名一个C2，用好多个。

**怎么玩？**

- 准备好多个域名
- 准备好几台C2服务器
- 被控机器上配置多个C2地址（一个连不上就试下一个）
- 或者每隔一段时间换一个域名/IP

这样就算其中一个域名被封了，还有其他的能用，不会一下子全掉线。

而且蓝队很难封完，你封一个我换一个，打游击一样。

**DNS轮询也是一种方式：**

- 一个域名配多个C2 IP
- DNS查询返回不同的IP
- 被控机器连不同的IP，分散风险

### 4.5.4 方法四：流量伪装

光藏IP还不够，流量本身也得伪装。

C2的流量是有特征的，比如特定的URL路径、特定的请求头、特定的数据包规律。有经验的蓝队或者IDS一眼就能看出来。

怎么办？**把C2流量伪装成正常的流量。**

比如：
- 伪装成正常的HTTP/HTTPS流量（访问网站、加载图片、提交表单）
- 伪装成DNS流量
- 伪装成微信/QQ聊天流量
- 伪装成各种软件的更新流量
- 伪装成Google、YouTube的流量

**怎么伪装？**

CS里有个东西叫Malleable C2 Profile，就是用来自定义流量特征的。

你可以写一个Profile，比如：

```
让CS的请求头看起来像Chrome浏览器
让URL路径看起来像WordPress博客的路径
让响应内容看起来像正常的HTML页面
```

这样流量看起来就像是在正常上网，蓝队很难发现。

网上也有很多现成的Profile可以直接用，比如模仿京东商城、模仿WordPress、模仿Google...挑一个喜欢的就行。

### 4.5.5 方法五：双层跳板

更高级的玩法，不只用一层跳转，用多层。

```
被控机器 → 跳转机A → 跳转机B → 真实C2
```

或者：

```
被控机器 → CDN → 跳转机 → 云函数 → C2
```

多层跳转，每一层都隔离，就算追到了一层，也追不到下一层。

当然，层数越多越复杂，延迟也越高，要权衡。

## 4.6 基础设施的其他组成部分

C2只是基础设施的一部分，还有一些其他的东西也要准备。

### 4.6.1 钓鱼基础设施

如果你要做钓鱼，那还得准备：

**钓鱼域名**

要和目标域名很像的那种，比如目标是 `example.com`，你注册个：
- `examp1e.com`（数字1代替字母l）
- `example-login.com`
- `example-support.com`
- `example123.com`

不注意看就看错了。

> 💡 **注册技巧：**
>
> 用不同的后缀，比如目标用.com，你可以用：
> - .net
> - .org
> - .info
> - .xyz
> - .top
>
> 或者用相近的域名：
> - examle.com（少一个p）
> - exampl.com（换了位置）
> - exampie.com（e结尾）

**钓鱼邮件服务器**

用来发钓鱼邮件的，要做好SPF、DKIM、DMARC，不然邮件直接进垃圾箱。

或者用第三方的邮件服务：
- SendGrid
- Mailgun
- 国内的SendCloud

**钓鱼网站服务器**

用来搭钓鱼页面的，要做得和真实网站一模一样。

可以用Social Engineering Toolkit（SET）快速克隆网站。

### 4.6.2 代理和跳板

做红队，你自己 的真实IP绝对不能暴露。

所以不管是做信息收集、还是扫端口、还是连C2，都得走代理。

**常用的：**
- VPN（多买几个，换着用）
- 代理池（Proxy Pool）
- 肉鸡/跳板机（被攻陷的第三方服务器）
- Tor（慎用，速度慢而且有些环境用不了）

**原则：**
```
你自己的电脑
  ↓
代理/VPN
  ↓
VPS跳板机
  ↓
云函数/CDN（可选）
  ↓
真实C2
  ↓
被控机器

绝对不能：
你的电脑 → 直接 → 目标
你的电脑 → 直接 → C2
```

### 4.6.3 数据存储和日志

红队项目会产生很多数据，要找个地方存。而且这些数据都是敏感数据，得加密存。

比如：
- 收集到的目标信息
- 拿到的凭据
- 截图、记录
- 报告素材
- ......

**存哪里？**
- 加密的VPS
- 加密的移动硬盘
- VeraCrypt加密的磁盘镜像
- 加密的云存储（慎用，怕泄露）

反正一个原则：**数据不能丢，也不能泄露。**

### 4.6.4 团队协作工具

如果是团队作战，还得有协作工具：

- 飞书/企业微信/钉钉（沟通用，但注意不要聊敏感内容）
- 内部wiki（存资料、记笔记）
- 共享屏幕/远程协作（飞书、腾讯会议都行）
- 版本管理工具（Git，自建GitLab）

## 4.7 搭基础设施的注意事项

最后讲几个注意事项，都是血的教训。

### 4.7.1 OPSEC（操作安全）最重要

OPSEC是Operations Security的缩写，翻译过来叫"操作安全"。

简单说就是：**别犯低级错误把自己暴露了。**

比如：
- 不要用真实信息买服务器、注册域名
- 不要用常用邮箱、手机号
- 不要在个人电脑上直接操作（最好用专用的工作机）
- 不要在社交平台上瞎发项目相关的东西
- 不要在一个设备上登个人号和工作号
- 不要用微信/QQ聊项目（太容易被溯源了）
- 不要把工具放桌面/下载文件夹
- 不要用"密码123"这种弱密码
- ......

这些都是基础中的基础，但很多人就是不注意，最后栽在这上面。

> 💡 **一个真实的案例：**
>
> 有个红队队员去打项目，用的是公司的VPN（真实IP）。
>
> 结果蓝队一查日志，就知道是哪家公司派来的了。
>
> 整个队伍都被溯源了，项目成绩作废，还被客户投诉。
>
> 所以OPSEC真的很重要。

### 4.7.2 不要把鸡蛋放在一个篮子里

前面说了，不要只用一个C2、一个域名、一个IP。多准备几个，有备无患。

而且不同的项目最好用不同的基础设施，不要一个基础设施打天下。

不然一个项目暴露了，其他项目也跟着遭殃。

**建议：**
- 每个项目用独立的C2
- 每个C2用独立的域名、独立的IP
- 准备多个备用C2

### 4.7.3 定期更换

基础设施不是一劳永逸的，要定期换。

比如：
- C2服务器定期换IP
- 域名定期换
- 木马定期更新、重新免杀
- 代理定期换

用得越久，暴露的风险就越高。

特别是木马，用久了特征就会被杀软收录，就得换新的。

### 4.7.4 测试！测试！测试！

正式开打之前，一定要自己先测试一遍。

比如：
- 马能不能上线？
- 稳不稳定？
- 延不延迟？
- 掉线了能不能重连？
- 流量看起来正常不正常？
- 免杀效果怎么样？（能不能过杀软？）
- 在不同网络环境下能不能用？
- ......

别等到了目标机器上才发现有问题，那就尴尬了。

**测试环境：**
- 本地虚拟机（VirtualBox、VMware）
- 云上的VPS（测试境外目标）
- 不同操作系统（Windows 7/10/11、Server 2016/2019等）

- 会搭CS
- 会生成马

- 加CDN域前置
- 加云函数跳转
- 加多层跳板
- 加流量伪装

原理都是通的，会了基础的，高级的也就不难了。
