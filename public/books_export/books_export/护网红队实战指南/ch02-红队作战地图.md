# 第二章 红队作战地图—— 打仗先画地图

> 第2章 | 预计学习时间：3小时 | 难度：⭐⭐

---

## 📖 写在前面

上一章我们讲了红队是什么，这一章我们来讲讲红队怎么打仗。

很多新手刚接触红队的时候，经常会问一个问题：

"红队攻击的步骤是什么？先干啥，后干啥？有没有什么标准流程？"

还真有。

红队发展到今天，已经有好几套成熟的攻击框架了，就像是打仗用的作战地图，告诉你敌人在哪、怎么打、分几步打。

这一章我们就来聊聊这些"作战地图"，以及红队怎么制定作战计划，怎么安排攻击节奏。

坐稳了，我们开讲。

---

## 2.1 为啥需要"作战地图"？

### 2.1.1 新手常犯的错

我见过很多新手打靶机或者做项目，上来就是一顿乱扫，扫到漏洞就打，打进去了就提权，提完权就不知道干啥了。

就像一个人进了陌生的城市，没有地图，没有导航，瞎转悠，走到哪算哪。

运气好的话，稀里糊涂就打穿了；运气不好的话，绕来绕去就是找不到突破口，或者打进去了不知道下一步该干啥，卡在那里浪费时间。

我给你讲个真实的故事：

有个新手去参加护网，分配到一个政府网站目标。他上来就是一顿扫描，扫了半天发现有个Tomcat页面，直接弱口令进了后台。

进了后台发现有文件上传功能，他传了个WebShell，上去了！

上去之后他很兴奋，但是接下来就懵了——"我下一步干啥？"

他对着服务器看了一个小时，不知道该干啥。看看目录结构？看了。看看用户？看了。然后呢？然后就不知道了。

最后还是队长过来，问他："你抓密码了吗？你收集内网信息了吗？你看看能不能提权啊？"

他才反应过来，原来还有这么多事要做。

这就是没有"作战地图"的下场——打进去了不知道下一步干嘛，浪费时间。

### 2.1.2 有了地图就不一样了

如果有一张"作战地图"呢？

那你就知道：
- 我现在在哪一步
- 下一步该干啥
- 还有哪些路可以走
- 最终目标怎么去

就像玩游戏有任务指引，或者开车有导航，虽然路还是得你自己走，但至少你知道方向在哪。

### 2.1.3 两个最有名的"作战地图"

红队圈子里，最有名的两个攻击框架是：

```
1. Cyber Kill Chain（网络杀伤链）
   洛克希德·马丁公司搞的，
   把攻击分成7个步骤，
   比较宏观，适合理解攻击的整体过程。

2. MITRE ATT&CK
   MITRE公司搞的，
   更详细、更具体，
   把攻击的每一招每一式都列出来了，
   像是一本"攻击招式大全"。
```

这俩是红队的"必修课"，不管你是做红队还是做蓝队，都得了解。

下面我们就一个一个讲，讲透、讲明白。

---

## 2.2 Cyber Kill Chain —— 七步杀敌链

### 2.2.1 它是啥来头？

Cyber Kill Chain，中文一般叫"网络杀伤链"，也有人叫"网络攻击链"。

它是洛克希德·马丁公司（就是那个造战斗机、造导弹的军火公司）在2011年提出来的。

为啥一个军火公司会搞这个？

因为他们把网络攻击和军事打击做了类比，觉得打仗有作战流程，网络攻击也应该有攻击流程。

比如军事上有：
- 情报侦察（敌人基地在哪？有什么武器？）
- 制定计划（用什么武器？打哪个点？）
- 发射导弹（把武器送过去）
- 击中目标（武器生效了）
- 评估效果（打中没有？损失如何？）

网络攻击也有类似的流程，于是就有了这个Cyber Kill Chain模型。

### 2.2.2 七步走，一步都不能少

Cyber Kill Chain把一次完整的攻击，分成了7个步骤：

```
第1步：侦察（Reconnaissance）
  → 收集目标信息，了解敌情

第2步：武器化（Weaponization）
  → 根据情报，准备"武器"

第3步：投递（Delivery）
  → 把武器送到目标那里

第4步：利用（Exploitation）
  → 武器触发，漏洞生效

第5步：安装（Installation）
  → 装后门，留个"入口"

第6步：命令与控制（Command & Control，C2）
  → 建立远程控制渠道

第7步：行动（Actions on Objectives）
  → 开始干"正事"
```

我们一个一个来讲，用大白话给你翻译翻译，保证让你听懂。

### 2.2.3 第一步：侦察（Reconnaissance）

侦察，就是收集信息。

在动手之前，攻击者会想尽办法收集目标的各种信息。

具体收集什么呢？我给你列个清单：

**技术信息：**
- 目标有哪些网站？域名是什么？
- 网站用的是什么技术？（PHP？Java？Python？）
- 服务器用的是什么？（Apache？Nginx？iis？）
- 有哪些开放的端口？什么服务？
- 有没有用CMS？（WordPress？Dedecms？PHPCMS？）
- 有没有已知漏洞？历史漏洞？

**人员信息：**
- 公司有哪些员工？
- 员工的名字、职位、邮箱
- 员工喜欢用什么密码？
- 员工的社交媒体账号？
- 哪些人有较高的权限？（IT管理员？开发？财务？）

**组织信息：**
- 公司的组织架构是什么？
- 有哪些部门？部门之间什么关系？
- 公司用的是什么安全设备？
- 公司有没有安全团队？几个人？
- 公司有哪些供应商？哪些外包？

**物理信息：**
- 公司地址在哪？
- 有几栋楼？几个出入口？
- 有没有WiFi？密码是什么？
- 有没有门禁？门禁卡好复制吗？

这一步就像小偷踩点——

你见过小偷怎么踩点吗？他们会在目标房子附近转悠，看看：
- 家里几口人？
- 什么时候出门、什么时候回家？
- 窗户朝哪？门朝哪？
- 有没有保安？保安几点巡逻？
- 有没有摄像头？摄像头朝哪？

信息收集得越多，后面攻击的成功率就越高。

**一个真实的案例：**

有个红队去打一家公司，技术层面防护得还不错，没找到什么漏洞。但是红队做了详细的人员信息收集，发现：

- 公司有个CTO，经常在社交媒体上发技术博客
- 这个CTO的GitHub账号用的是公司邮箱注册
- 他的GitHub代码里有一个commit，包含了公司服务器的密码

红队用这个密码直接SSH登录了服务器，拿下了整台机器。

这就是信息收集的力量——技术漏洞没找到，但是人员漏洞找到了。

> 💡 **划重点：**
>
> 红队里有句话叫"信息收集决定上限"。
>
> 你对目标了解得越多，能打的牌就越多，成功率就越高。
>
> 很多时候打不进去，不是你技术不行，是你信息不够。
>
> 扫了几个端口、看了几个网站就说"没漏洞"的新手，是永远不会成为合格的红队的。

### 2.2.4 第二步：武器化（Weaponization）

侦察完了，知道目标大概是什么情况了，接下来就要准备"武器"了。

什么是武器？就是能帮你打进去的东西。

根据不同的攻击方式，要准备不同的武器：

**如果是钓鱼攻击，你要准备：**
- 钓鱼邮件的模板（看起来像真的一样）
- 钓鱼网站（克隆真实的网站）
- 恶意附件（带木马的Word、Excel、PDF）
- 恶意链接

**如果是Web攻击，你要准备：**
- 针对特定漏洞的Exploit（利用代码）
- WebShell（管理已拿下的Web服务器）
- 各种扫描工具

**如果是近源攻击，你要准备：**
- BadUSB（伪装成U盘的键盘）
- 带木马的U盘
- WiFi钓鱼设备

**如果是内网攻击，你要准备：**
- 横向移动工具（Mimikatz、PsExec等）
- 抓密码工具
- 端口转发工具
- C2木马

这一步就像小偷准备作案工具——根据踩点的结果，准备对应的工具：

- 门锁太高级？那我准备开锁工具
- 窗户有铁栅栏？那我准备撬棍
- 保安很严？那我准备迷药、假证件

### 2.2.5 第三步：投递（Delivery）

武器准备好了，接下来就要把武器"送"到目标那里。

怎么送？方法很多：

**1. 钓鱼邮件（最常见）**
- 给目标发一封邮件
- 邮件里有恶意链接
- 或者有恶意附件
- 诱骗目标点击

举个例子：
```
主题：您的工资条已生成，请查收

您好，

您的2024年1月份工资条已生成，请点击下方链接查看：
http://xxx.com/salary/202401.pdf

如有问题请联系HR。

人事部
2024年1月15日
```

这个链接看起来像PDF，实际上是个钓鱼网站，或者会下载木马。

**2. 水坑攻击**
- 找到目标经常访问的网站
- 在那个网站上挂马
- 目标访问网站就中招

比如目标公司的人经常访问某个行业新闻网站，红队就把那个网站黑了，挂上恶意代码，目标员工访问时就中招了。

**3. 丢U盘**
- 把带木马的U盘丢到目标公司
- 有人捡到插电脑上就中招
- 这种方法在内网渗透时特别有效

**4. 社工直接送**
- 伪装成维修人员、送外卖的、快递员
- 直接进入目标公司
- 找机会插U盘或者连接内网

**5. 利用已有入口**
- 如果已经有WebShell
- 就可以直接上传更多工具
- 不用再"投递"了

这一步就像小偷把"作案工具"送到目标手里——可能是假装快递员送个包裹，可能是放在门口让你捡，总而言之就是想方设法让你"接招"。

### 2.2.6 第四步：利用（Exploitation）

投递成功之后，目标"中招"了，武器开始发挥作用。

比如：
- 用户点开了带木马的Word文档，宏代码执行了
- 用户输入了钓鱼网站的密码，密码被记录了
- 攻击者利用SQL注入拿到了数据库权限
- U盘插上去自动运行了病毒

这一步就是"漏洞触发"的瞬间，攻击者从"外面"进到了"里面"，开始执行自己的代码。

这是一个转折点——之前都是在外面折腾（侦察、准备、送武器），从这一步开始，才真正进入目标内部。

### 2.2.7 第五步：安装（Installation）

好不容易打进来了，总不能打一下就走吧？

得想办法"留下来"。不然万一这次没拿到想要的，下次还得重新打，太麻烦了。

这一步就是安装后门、木马、或者别的什么东西，保证以后还能进来，而且不容易被发现。

常见的安装物：
- 一个隐藏的WebShell
- 一个计划任务后门
- 一个服务后门
- 一个隐藏账号
- 一个rookit

就像小偷进了房子，先配一把钥匙，或者偷偷留一扇窗不关，这样下次来就方便了，不用每次都撬锁。

### 2.2.8 第六步：命令与控制（C2）

后门装好了，接下来就要建立"通信渠道"了。

攻击者需要能远程控制这台机器，让它干啥就干啥。这就是C2（Command & Control，命令与控制）。

一般来说，被控的机器会主动连接攻击者的服务器（这样能穿透防火墙），然后等着攻击者发号施令。

就像间谍和总部的无线电台，悄悄联系，总部说啥间谍就干啥。

C2通信的方式：
- 直接连IP（最简单，也最容易被发现）
- 连接域名（可以用域名做流量隐藏）
- 通过合法的网站中转（更难被发现）
- 通过加密协议（更难被检测）

### 2.2.9 第七步：行动（Actions on Objectives）

最后一步，就是达到最终目的了。

攻击者想干啥？
- 偷数据？（财务报表、客户资料、源代码）
- 搞破坏？（删库、勒索）
- 横向移动？（打更多机器）
- 拿下域控？（控制整个网络）

到这一步，攻击就进入"收尾阶段"了，攻击者开始干他真正想干的事。

### 2.2.10 一张图帮你理解

我把这七步画成一张图，你一看就明白了：

```
        外围阶段                    突破阶段                    内部阶段

┌─────────┐   ┌─────────┐   ┌─────────┐
│ 1.侦察   │ → │ 2.武器化 │ → │ 3. 投递 │
│ 收集情报  │   │ 准备工具  │   │ 送武器   │
└─────────┘   └─────────┘   └─────────┘
                                  │
                                  ↓
                           ┌─────────┐
                           │ 4.利用  │
                           │ 触发漏洞│
                           └─────────┘
                                  │
                                  ↓
              ┌─────────┐   ┌─────────┐   ┌─────────┐
              │ 7.行动  │ ← │  6. C2  │ ← │ 5.安装  │
              │达成目标  │   │ 远程控制│   │ 留后门  │
              └─────────┘   └─────────┘   └─────────┘
```

简单理解就是：
- 第一到第三步：都是在外围搞事情，还没进去
- 第四步：突破防线，进去了
- 第五到第六步：站稳脚跟，建好通道
- 第七步：开始干正事

### 2.2.11 这个模型有啥用？

你可能会问："知道这七步有啥用？"

用处大了！

**对红队来说：**
- 知道攻击分几步，就知道自己现在在哪一步，下一步该干啥
- 每一步都有很多方法，这条路走不通可以换一条
- 可以帮助梳理攻击思路，不会漏掉步骤

**对蓝队来说：**
- 攻击链的每一步，都可以对应防守措施
- 只要在任意一步把攻击拦住，整个攻击就失败了
- 可以针对每一步部署防御和检测

比如：
- 侦察阶段 → 减少信息泄露（别让人那么容易收集到信息）
- 投递阶段 → 邮件过滤、防火墙（不让恶意东西进来）
- 利用阶段 → 打补丁、WAF（不让漏洞被触发）
- C2阶段 → 流量检测、威胁情报（发现异常通信）

> 💡 **一个经典的说法：**
>
> "攻击方需要每一步都成功才能打进来，防守方只要在任意一步成功就能防住。"
>
> 听起来防守方占优势对吧？但现实是——
>
> 防守方要防无数种攻击，而攻击方只要找到一个突破口就行。
>
> 这就是网络安全的"不对称性"。

## 2.3 MITRE ATT&CK —— 攻击招式大全

讲完了Cyber Kill Chain，我们再来讲讲另一个更有名的框架——MITRE ATT&CK。

### 2.3.1 它又是啥来头？

MITRE是美国的一个非营利组织，经常和政府合作搞安全研究。ATT&CK就是他们搞出来的一个知识库。

ATT&CK是啥意思？全称是：
**Adversarial Tactics, Techniques, and Common Knowledge**

翻译过来就是：
**敌手战术、技术与公共知识库**

名字很拗口，你可以简单理解为：**一本黑客攻击的"招式大全"**。

就像武侠小说里的武功秘籍，告诉你：
- 这招叫"黑虎掏心"，怎么用、什么效果
- 这招叫"猴子偷桃"，怎么用、什么效果
- 有什么破绽、怎么破解

ATT&CK就是网络安全版的"武功秘籍"，告诉你黑客有哪些"招式"，每个招式怎么用、怎么防、怎么检测。

### 2.3.2 ATT&CK和Kill Chain有啥不一样？

可能你会觉得："这不也是攻击框架吗？和Kill Chain有啥区别？"

区别大了。

打个比方：

- **Kill Chain**像是兵法，告诉你打仗分几步：侦察、冲锋、突破、纵深...比较宏观，讲的是战略。

- **ATT&CK**像是武功秘籍，告诉你具体的每一招每一式：这招叫啥、怎么用、谁用过、有什么破绽、怎么检测...非常具体，讲的是战术。

简单说：
Kill Chain是战略层面的，ATT&CK是战术层面的。

### 2.3.3 ATT&CK的结构

ATT&CK把攻击分成了很多个"战术"（Tactics），每个战术下面又有很多"技术"（Techniques），每个技术下面还有具体的"子技术"（Sub-techniques）。

就像：
```
战术 = 要干啥（比如：横向移动）
  技术 = 怎么干（比如：哈希传递）
    子技术 = 具体怎么干（比如：用哈希传递登录SMB）
```

目前ATT&CK企业矩阵里，一共有14个战术（最新版）：

```
1. 侦察（Reconnaissance）
   → 收集目标信息

2. 资源开发（Resource Development）
   → 准备攻击用的资源

3. 初始访问（Initial Access）
   → 打进去，获得第一个立足点

4. 执行（Execution）
   → 在目标机器上运行代码

5. 持久化（Persistence）
   → 留后门，让自己一直能进来

6. 权限提升（Privilege Escalation）
   → 从低权限变高权限

7. 防御绕过（Defense Evasion）
   → 躲避检测，不被发现

8. 凭据访问（Credential Access）
   → 偷密码、偷哈希、偷票据

9. 发现（Discovery）
   → 了解环境，了解网络

10. 横向移动（Lateral Movement）
    → 从一台机器到另一台机器

11. 收集（Collection）
    → 收集信息、收集数据

12. 命令与控制（Command and Control）
    → 远程控制

13. 数据渗出（Exfiltration）
    → 把偷到的数据传出来

14. 影响（Impact）
    → 搞破坏、勒索、删数据
```

你看，是不是和Kill Chain有点像？但分的更细了，而且每个战术下面都有一大堆具体的技术。

### 2.3.4 举个例子你就懂了

光说概念你可能有点懵，我举个例子你就明白了。

就拿"初始访问"（Initial Access）这个战术来说，它下面有这些常见技术：

```
初始访问（Initial Access）：
  ├── 钓鱼（Phishing）
  │   ├── 钓鱼邮件带附件（Spearphishing Attachment）
  │   ├── 钓鱼邮件带链接（Spearphishing Link）
  │   └── 通过链接钓鱼（Spearphishing via Service）
  ├── 面向公众的应用漏洞
  │   └── 利用面向Internet的应用程序的漏洞
  ├── 外部远程服务
  │   └── 如VPN、邮件网关等远程服务
  ├── 硬件连接
  │   └── 插入U盘等物理访问
  ├── 通过中间人攻击
  ├── 利用有效账户
  ├── 利用外部远程服务
  ├── 供应链攻击
  ├── 可移动介质
  └── 受信任的关系
```

再拿"横向移动"（Lateral Movement）来说：

```
横向移动（Lateral Movement）：
  ├── 远程服务（Remote Services）
  │   ├── SMB/Windows Admin Shares
  │   ├── SSH
  │   ├── RDP
  │   ├── VNC
  │   └── WinRM
  ├── 哈希传递（Pass the Hash）
  ├── 票据传递（Pass the Ticket）
  ├── 使用备用身份验证材料
  ├── 横向工具转移
  ├── 远程桌面协议
  └── 软件部署工具
```

每个技术点进去，都有详细的说明：
- 这个技术是干啥的
- 攻击怎么实现（步骤是什么）
- 有哪些黑客组织用过（APT28、APT29都用过哪些）
- 怎么检测（有什么特征）
- 怎么防御
- 用什么工具可以做到
- 有哪些真实案例

可以说非常详细了，相当于一本"黑客攻击百科全书"。

### 2.3.5 ATT&CK有啥用？

ATT&CK这么详细，它有啥用呢？

**对红队来说：**
- 可以查"还有哪些攻击方法我不会"，补充自己的技术栈
- 可以用来做攻击思路的参考，这个目标还有哪些路可以走
- 可以用来模拟特定的黑客组织（红队演练时可以模仿APT攻击）
- 可以用来规划攻击路径，哪条路最近、哪条路最难

**对蓝队来说：**
- 可以对照着检测自己覆盖了哪些攻击手法
- 可以用来评估自己的检测能力，哪些地方有盲区
- 可以用来做威胁狩猎的参考（主动去搜这些特征）
- 可以用来做红蓝对抗，蓝队知道攻击链，就知道在哪布防

**对防守方来说，**
**ATT&CK就像是一份"考点清单"——**
黑客可能用的招都在这了，你一个个去防、去检测，防住了就安全一点。

> 💡 **小知识：**
>
> ATT&CK不是一成不变的，MITRE会持续更新，新的攻击技术会被加进去，旧的会被更新。
>
> 目前最新的版本已经有好几百个技术点了，还在不断增加。
>
> 所以做安全的人要持续学习，跟上最新动态。

## 2.4 红队实战一般怎么打？

讲完了两个理论框架，我们再回到"实战"，说说红队做项目的时候，一般是怎么个流程。

理论是理论，实战是实战，两者有关系但不完全一样。实战中会更灵活、更实际。

### 2.4.1 阶段一：前期准备（战前准备）

正式开打之前，有很多准备工作要做。这些工作做得好，后面打起来就顺；做得不好，后面就会很麻烦。

**第一件事：签合同、拿授权**

这个最重要，没有授权说啥都白搭。

合同里要写清楚：
- 攻击范围（哪些IP段、哪些域名能打，哪些不能打）
- 攻击时间（什么时候开始、什么时候结束、每天几点到几点可以打）
- 允许的攻击手段（Web渗透可以，社工行不行？物理攻击呢？）
- 责任划分（把人家系统搞崩了谁负责？）
- 保密要求（不能往外说，打完要清理干净）

这些都得白纸黑字写清楚，不光是法律要求，也是保护自己。不然出了问题，说不清楚。

**第二件事：明确目标和范围**

搞清楚：
- 最终目标是什么？（拿到域控？偷到某份数据？拿下某个系统？）
- 攻击范围有多大？（哪些IP段、哪些域名、哪些系统在范围内）
- 哪些东西绝对不能碰？（比如生产数据库不能动，某些核心系统不能搞崩）
- 有没有蜜罐？（有些公司会故意放几个假目标钓鱼）

**第三件事：制定作战计划**

根据目标和范围，制定一个大概的计划：
- 先打哪，后打哪（挑软柿子先捏？）
- 有哪些可能的突破口（Web漏洞？钓鱼？弱口令？）
- 人手怎么分配（谁负责外网、谁负责内网、谁负责社工）
- 时间怎么安排（什么阶段干什么）
- 备用方案是什么（这条路不通怎么办）

计划不用太细，毕竟计划赶不上变化，但大概的方向得有。不然一群人瞎打，效率很低。

**第四件事：准备工具和基础设施**

打仗得有武器对吧？这些都得提前准备好：
- C2服务器搭好了吗？（隐蔽性够不够？）
- 免杀做好了吗？（能不能过EDR？）
- 钓鱼网站搭好了吗？（模板准备好了吗？）
- 钓鱼邮箱准备好了吗？（邮箱域名买了吗？）
- 常用工具备齐了吗？（扫描、渗透、提权、横向工具）
- 代理、VPN准备好了吗？（暴露真实IP是大忌）

这些都得提前准备好，别等到用的时候才发现没有，那就尴尬了。

**一个真实的小故事：**

有个新手红队去做项目，前期准备很充分，扫描工具、渗透工具都准备好了。

但是他忘了一件事——买钓鱼用的邮箱域名。

等开始打的时候，发现需要发钓鱼邮件，但是没有邮箱域名啊！

临时去买吧，等域名生效要好几天；用自己邮箱发吧，分分钟被封。

最后只能用别的组的资源，耽误了时间。

这个故事告诉我们：**细节决定成败，准备要全面。**

### 2.4.2 阶段二：外围侦察（情报收集）

准备工作做好了，就开始正式的侦察了。

这一步和Kill Chain的第一步"侦察"差不多，就是收集各种信息。

但红队的侦察，比普通渗透测试要细得多、全面得多。

**1. 技术信息收集**

```
域名/IP信息：
  - 主域名是什么？
  - 有哪些子域名？（子公司域名、业务域名等）
  - 对应哪些IP？（IP段是多少？）
  - DNS记录是什么？
  - 有没有CDN？怎么绕过去？

端口/服务信息：
  - 哪些端口是开放的？
  - 什么服务运行在端口上？
  - 服务的版本是什么？
  - 有没有默认端口、默认口令？

Web应用信息：
  - 有哪些Web应用？
  - 用的是什么CMS？
  - 用的是什么框架？
  - 有哪些功能模块？
  - 有没有Swagger/API文档？

漏洞信息：
  - 这个版本有什么已知漏洞？（CVE）
  - 历史上有过什么漏洞？（历史漏洞）
  - 有没有人之前挖过漏洞？（漏洞库）
```

**2. 人员信息收集**

```
员工信息：
  - 公司有哪些员工？（招聘网站、LinkedIn）
  - 员工的名字、职位、邮箱格式是什么？
  - 哪些人是管理员/开发/运维？（权限高的人）

社工信息：
  - 员工喜欢用什么密码？（社工库查询）
  - 员工有什么习惯？（用什么社交媒体、论坛）
  - 员工的生日、家乡、宠物名？（常被用作密码）

钓鱼信息：
  - 公司邮箱格式是什么？
  - 有哪些邮件网关？
  - 邮件安全做得好不好？（有没有SPF、DKIM、DMARC）
```

**3. 组织信息收集**

```
公司背景：
  - 公司的组织架构是什么？
  - 有哪些部门？
  - 有哪些子公司/关联公司？
  - 有哪些合作伙伴？

业务信息：
  - 公司做什么业务？
  - 用什么技术栈？
  - 有哪些供应商？
  - 有没有外包/代维？

安全信息：
  - 公司有没有安全团队？
  - 用什么安全设备？（防火墙、IDS、EDR？）
  - 有没有安全运营中心（SOC）？
  - 之前有没有被黑过？（数据泄露）
```

**4. 物理信息收集（如果有条件）**

```
公司位置：
  - 公司地址在哪？
  - 有几栋楼？

物理防护：
  - 有没有门禁？
  - 有没有摄像头？
  - 有没有保安巡逻？

WiFi信息：
  - 能不能搜到公司WiFi？
  - WiFi密码是什么？（有人说、有人贴出来）
```

总之一句话：**能收集的都收集，信息越多越好。**

这一步可能要花好几天，甚至一两周。别嫌慢，信息收集做得好，后面能省很多事。

**真实案例：**

有一次红队打一个目标，技术扫描没找到什么漏洞。但是人员信息收集的时候发现：

- 公司的GitHub组织下面有很多开源项目
- 这些项目里有员工的工作邮箱
- 有个员工在代码里写了测试密码：Test123456
- 这个密码居然和生产环境的某个后台一样

红队用这个密码直接登录了后台，后台上有个文件上传功能，上传了WebShell，拿下了服务器。

这就是信息收集的力量。

### 2.4.3 阶段三：边界突破（初始访问）

信息收集得差不多了，就开始想办法打进去了。

这一步在ATT&CK里叫"初始访问"（Initial Access），是整个攻击链中最关键的一步。

怎么打进去？方法很多，大概可以分成几类：

**1. 技术突破（从系统层面打进去）**

最传统的方法，找漏洞、利用漏洞：

- Web漏洞利用（SQL注入、XSS、文件上传、命令注入、逻辑漏洞...）
- 公开漏洞利用（永恒之蓝、Log4j、Spring4Shell...）
- 弱口令爆破（SSH、RDP、FTP、后台管理...）
- 未授权访问（Redis、MongoDB、Jenkins...）
- 敏感文件泄露（.git、.svn、备份文件、配置文件...）

**2. 社工突破（从人的层面打进去）**

有时候技术打不进去，就从人下手：

- 钓鱼邮件（发恶意附件或链接）
- 钓鱼网站（克隆登录页面骗密码）
- 电话社工（假装IT打电话要密码）
- 微信/QQ社工（假装熟人骗信息）

社工的成功率往往比技术攻击高，因为人是最薄弱的环节。

**3. 物理突破（从物理层面打进去）**

- 丢U盘（带木马的U盘扔到公司）
- BadUSB（伪装成键盘，一插就执行命令）
- 伪装进入（假装修空调的、送外卖的混进去）
- WiFi攻击（钓鱼WiFi、破解WiFi密码）
- 门禁卡复制

**4. 供应链/第三方突破**

- 打合作伙伴（你公司安全，合作伙伴不一定安全）
- 打外包/运维人员（他们往往有你公司的权限）
- 打云服务商（如果你的数据在云上）
- 打第三方软件（你用的软件可能有漏洞）

**红队的思路是：条条大路通罗马，哪条好走走哪条。**

这个目标Web防护做得好，那我就试试钓鱼；钓鱼不行就试试物理；物理也不行就试试第三方。

总有一款适合你。

只要能拿到第一台机器的权限，就算"突破"了。这第一台机器，业内一般叫：
- 跳板机
- 初始立足点
- First Blood（第一滴血）

### 2.4.4 阶段四：权限提升

打进去了，但可能权限很低，比如：
- 只是个WebShell，只能执行简单命令
- 只是普通用户权限，很多目录访问不了
- 只是某个服务的账户，不是管理员

那接下来就要想办法提权，拿到更高的权限，最好是：
- Windows的话，拿到System/Administrator权限
- Linux的话，拿到Root权限

怎么提权？方法很多：

**Windows提权常见方法：**
- 系统漏洞提权（MS08-067、MS17-010...）
- 错误配置提权（AlwaysInstallElevated、服务配置错误...）
- 第三方软件提权（VNC、输入法、驱动...）
- 令牌窃取
- Bypass UAC

**Linux提权常见方法：**
- 内核漏洞提权
- SUID提权
- sudo配置错误
- 计划任务提权
- 密码复用（同一密码用于多台机器）

提权是个技术活，我们后面会有专门的章节讲。

### 2.4.5 阶段五：内网信息收集

拿到一台机器的权限了，接下来该干啥？

**别急着横向移动，先收集信息。**

很多新手容易犯的错就是——提完权就想着去打下一台机器，结果发现不知道打谁、从哪打起、怎么打。

正确做法是：**先把这台机器的信息收集完整，再去打下一台。**

收集什么信息？

**本机信息：**
```
系统信息：
  - 主机名是什么？
  - IP地址是多少？
  - 是Windows还是Linux？
  - 是什么版本？打了哪些补丁？

用户信息：
  - 有哪些本地用户？
  - 有没有域用户登录过？
  - 有哪些管理员？
  - 用户密码策略是什么？

网络信息：
  - 网卡配置是什么？
  - 内网IP段是多少？
  - 有哪些网段可以访问？
  - 有没有网关、防火墙规则？

软件信息：
  - 安装了哪些软件？
  - 有没有杀毒软件？（如果有，要小心）
  - 有没有安全软件？（EDR、HIDS？）
  - 有没有远程管理软件？（VNC、AnyDesk、向日葵？）

文件信息：
  - 有没有共享文件夹？
  - 有没有敏感文件？（密码本、配置文件、备份文件）
  - 有没有浏览器历史？（可能保存了密码）
```

**凭据信息（最重要！）：**
```
密码相关：
  - 本地密码哈希（SAM数据库）
  - 内存中的明文密码（LSASS进程）
  - 管理员是否 reused password？
  - 有没有保存的RDP密码？

其他凭据：
  - 浏览器保存的密码
  - FTP客户端保存的密码
  - 数据库客户端保存的密码
  - Git凭据
  - SSH密钥
```

**内网信息：**
```
网络发现：
  - 有哪些其他机器？（ARP缓存、NetBIOS）
  - 域环境是什么样的？（域名、域控IP）
  - 有哪些域用户？
  - 有哪些域管？
  - 有哪些服务账号？

域信息：
  - 域的SID是什么？
  - 域控是谁？
  - 域的信任关系？
  - 有哪些组策略？
```

> 💡 **红队思路：**
>
> 收集信息不是一次就完事的，每拿到一台新机器，都要重新收集一遍信息。
>
> 新机器上可能有你之前没有的凭据、有你没有的内网网段、有你不知道的域信息。
>
> 信息越多，下一步的选择就越多。

### 2.4.6 阶段六：横向移动

信息收集得差不多了，就开始横向移动了——从这台机器打到另一台机器，一步步扩大战果。

怎么横向？方法很多：

**1. 远程服务利用**
- PsExec（Mimikatz套件里的，经典工具）
- WMI（Windows管理工具，可以远程执行命令）
- SMB/WinRM（通过445/5985/5986端口）
- RDP（远程桌面）
- SSH（Linux之间）

**2. 凭据攻击**
- Pass the Hash（哈希传递，用密码哈希直接登录）
- Pass the Ticket（票据传递，用Kerberos票据登录）
- 爆破密码（如果知道用户名，试试常见密码）
- 中继攻击（把获取到的哈希Relay到其他机器）

**3. 黄金白银票据**
- 域环境下特有的攻击方式
- 伪造 Kerberos 票据
- 可以获取域管权限

横向移动的同时，还要继续收集凭据、收集信息，然后用新的凭据去打新的机器，滚雪球一样，越打越多。

### 2.4.7 阶段七：达成目标 + 权限维持

打着打着，就打到最终目标了。

最终目标是什么？要看任务要求：
- 拿下域控？（控制整个域）
- 拿到核心业务系统？
- 偷到某份重要数据？
- 在某个指定位置放个flag？

达到目标了，攻击就算成功了。

但别急着结束，正规的红队项目，打到目标之后，还要做一件事——**权限维持**。

就是留几个后门，证明"我可以随时回来"。

为什么要留后门？
- 万一被发现清理了，你还能进来
- 项目还没结束，你可能还要继续打其他目标
- 证明给客户看：你是真的可以随时进来的

怎么留后门？
- 计划任务后门
- 服务后门
- 账号后门
- WebShell
- 黄金票据/白银票据（域环境）
- WMI事件订阅

（后门怎么留，我们后面专门讲）

### 2.4.8 阶段八：清理痕迹 + 写报告

目标达成了，别急着收工。

**第一件事：清理痕迹**

把你留下的东西都清掉：
- 木马、后门删了（按清单一个个清）
- 日志处理了（清掉或者恢复原样）
- 新建的账号删了
- 修改过的配置改回去
- 上传的文件删了
- 命令历史删了

总之，尽量恢复到你来之前的样子，别给客户留一堆烂摊子。

> ⚠️ 重要提醒：
>
> 项目结束了，后门必须清干净！
>
> 偷偷留后门是违法的，也是没有职业道德的行为。
>
> 客户请你来是帮你测安全的，不是让你以后能随时进来的。

**第二件事：写报告**

这个很重要。打得再精彩，报告写不好，客户也不认。

报告里要写清楚：
- 你是怎么打进去的（详细的攻击路径）
- 走了哪些步骤（每个步骤都写）
- 用了哪些漏洞（漏洞名称、描述、证明截图）
- 拿到了哪些权限（截图）
- 造成了什么影响（风险评估）
- 怎么修复（给建议）

越详细越好，最好让客户照着报告就能复现。

报告是交付给客户的最终产品，要认真对待。

## 2.5 红队作战的几个关键思路

讲完了流程，再跟你聊聊红队作战的几个关键思路。这些都是经验之谈，比具体的技术还重要。

### 2.5.1 思路比工具重要

很多新手沉迷于收集工具，硬盘里存了几十个G的工具，但是真到打的时候还是不会。

为啥？因为思路不对。

工具再多，不会用也是白搭。

红队拼到最后，拼的不是谁工具多，是谁思路广、谁脑子活。

面对一个问题：
- A只会用Sqlmap扫SQL注入，扫不出来就说"没漏洞"
- B想了想，可以用XSS打到后台，拿到Cookie后登录后台找上传点

工具一样，思路不一样，结果就不一样。

> 💡 **一句话：**
>
> 一流的红队玩思路
> 二流的玩技术
> 三流的玩工具

### 2.5.2 条条大路通罗马

红队不是考试，没有标准答案。

不是说"这个目标必须用SQL注入打进去"，也不是说"提权必须用某个漏洞"。

方法多了去了，这条不行就换一条。

- 正门进不去就走侧门
- 侧门进不去就走窗户
- 窗户进不去就走烟囱
- 烟囱也爬不进去？那从地下挖地道？

**别死磕一个点。**

在一个地方卡太久了（超过2小时没进展），就退出来，换个思路，换个方向。

继续死磕只会浪费时间，不如换个方向试试。

### 2.5.3 信息收集决定上限

前面说过，这里再强调一遍：**信息收集决定了你的攻击上限。**

你对目标了解得越多，能打的牌就越多，成功率就越高。

很多新手打不进去，不是技术不行，是信息收集做得太差了：
- 扫了几个端口、看了几个页面，就说"没漏洞，打不进去"
- 试试弱口令"admin/admin"，不行就说"没弱口令"
- 看看首页没有后台地址，就不知道去哪找后台了

那哪行啊？信息收集是要花时间的，是要用心的。

多花点时间在信息收集上，绝对不亏。

**一个建议：** 做项目的时候，信息收集阶段至少占总时间的30%。别急着上手打，先把情报收集充分了。

### 2.5.4 隐蔽第一，速度第二

红队和渗透测试不一样。

渗透测试你可以光明正大地扫，大不了被封IP，反正客户知道你在测。

红队不行。被蓝队发现了，轻则这个目标废了，重则整个队伍都被溯源，得不偿失。

所以红队的第一原则是：**隐蔽。**

怎么隐蔽？
- 能慢一点就慢一点（扫描别太猛）
- 能少发包就少发包
- 能安静就安静
- 别一上来就噼里啪啦全端口扫描
- 工具尽量用正常的系统工具，少用"黑客工具"
- 流量尽量走加密协议，看起来像正常HTTPS

**别给蓝队送分。**

### 2.5.5 人是最薄弱的环节

记住这句话：**人永远是最薄弱的环节。**

技术防护做得再好，人点了钓鱼邮件，一样白搭。

防火墙再厚，IDS再灵敏，也防不住"人傻"。

所以红队特别喜欢社工，因为：
- 成功率高（人比机器好骗）
- 不容易被发现（没有触发任何技术检测）
- 成本低（不用挖0day，不用找漏洞）

当然，我们做防守的时候也要记住：**技术是基础，人才是关键。**

光靠技术是不够的，还要做好员工的安全意识培训。

## 2.6 本章总结

这一章我们讲了红队的"作战地图"。

我们讲了：
- 为什么需要作战地图（避免瞎打、有方向感）
- Cyber Kill Chain 七步模型（侦察→武器化→投递→利用→安装→C2→行动）
- MITRE ATT&CK 攻击知识库（详细的"招式大全"）
- 红队实战的八个阶段（准备→侦察→边界突破→提权→信息收集→横向→目标→清理）
- 红队作战的五个关键思路（思路比工具重要、条条大路通罗马、信息收集决定上限、隐蔽第一、人最薄弱）

这些都是"道"层面的东西，听起来可能有点虚，但等你真正打多了就会发现，这些"虚"的东西，其实是最重要的。

技术是学不完的，漏洞每天都有新的，工具也是换了一茬又一茬。但思路、方法、原则，这些东西是不变的。

把这些东西搞明白了，你就不是一个只会用工具的"脚本小子"，而是一个有思路、有方法的红队队员。
