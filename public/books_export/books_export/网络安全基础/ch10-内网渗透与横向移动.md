**内网渗透与横向移动。**

很多人对黑客攻击的印象，
可能就是攻破一个网站，
拿到webshell，
然后就完事了。
但真实的攻击，
尤其是有组织的攻击，

拿下一台Web服务器，
往往只是第一步。
攻击者的目标可能是：
- 核心数据库里的数据
- 域控制器
- 公司的核心业务系统
- 甚至整个内网

所以，
拿下一台外网服务器之后，
攻击者会以它为跳板，
继续往内网打，
一台一台地打过去，
最终到达目标。
这个过程，
就叫**内网渗透**和**横向移动**。

横向移动是内网渗透的核心，
也是高级攻击的标志性动作。
会不会打内网，
是区分脚本小子和真正攻击者的重要标志。

我们就来讲讲内网渗透和横向移动的那些事：

信息收集怎么做、
常见的内网攻击手法、
横向移动的方式、
域环境的渗透...
让你对内网渗透有个基本的了解。

## 10.1 先讲一个故事

### 10.1.1 "从一台Web服务器到域控"

我给你讲个典型的内网渗透故事。

某公司官网被黑客攻破了，
黑客通过一个SQL注入，
拿到了Web服务器的webshell。
但这台Web服务器在DMZ区，
权限不大，
数据也不多。
黑客的目标，
是公司内网的核心数据库和域控制器。

接下来黑客干嘛了？

**第一步：信息收集**
先在这台Web服务器上收集内网信息——
IP段、网段、
域信息、当前用户权限、
网络拓扑、其他服务器...
先搞清楚内网长什么样。

**第二步：提权**
当前Web服务器的权限是普通用户，
不够用。
先想办法提权，
拿到系统权限（system/root）。
通过一个系统漏洞，
成功提权到了system。

**第三步：抓取密码和哈希**
拿到system权限之后，
用mimikatz之类的工具，
从内存里抓取这台机器上登录过的用户的密码和哈希。
还真抓到了一个域用户的凭据。

**第四步：横向移动**
有了域用户凭据，
就可以去访问内网里的其他机器了。
先扫了一下内网，
发现了几十台机器。
用抓到的凭据去试，
发现这个用户在好几台机器上都有权限。
于是用PsExec之类的工具，
登录到了另一台内网服务器上。

**第五步：继续收集、继续横向**
在第二台机器上，
又抓了更多的凭据，
又横向到了更多的机器。
像滚雪球一样，
越打越多。

**第六步：打域控**
最后，
通过一个域管理员的凭据，
成功登录到了域控制器（DC）。
拿到域控权限，
就等于拿到了整个域的控制权，
整个内网的所有机器，
想干嘛干嘛。

整个过程用了不到一周的时间。
从一台小小的Web服务器，
一步步打到了核心域控。

---

你看，
这就是典型的内网渗透——
**以点破面，
逐步渗透，
横向移动，
直到拿下整个内网。**

内网的防护往往比外网弱，
很多企业外网防火墙堆得满满的，
内网却几乎不设防，
觉得"外网打不进来，内网就是安全的"。
这是非常错误的观念。
攻击者只要在外网打开一个口子，
就能在内网里横着走。

这一章，
我们就来看看内网渗透
到底是怎么回事。

## 10.2 内网渗透概述

### 10.2.1 什么是内网渗透？

什么是内网渗透？
简单说就是：
**攻击者已经进入了内网
（或者控制了内网中的一台机器），
然后在内网中进行进一步的渗透，
获取更多机器的权限、
更多有价值的数据的过程。**

外网渗透是从外面往里面打，
内网渗透就是已经进来了，
在里面继续打。

内网渗透的目标一般是：
- 获取更多的服务器权限
- 获取核心数据（数据库、文件服务器...）
- 拿下域控制器，控制整个域
- 获取最高权限，长期潜伏
- 横向扩散，感染更多机器

### 10.2.2 内网的特点

内网和外网有什么不一样？
为什么内网渗透单独拿出来讲？
因为内网环境有自己的特点：

```
  1. 网络环境复杂
    内网可能有很多网段，
    不同网段之间有防火墙、有访问控制
    有DMZ区、办公区、
      核心业务区、管理区...
    比外网复杂多了

  2. 防护相对薄弱
    很多企业重外轻内
    外网防护很严，内网几乎不设防
    很多机器漏洞没打补丁，
    很多弱口令
    内网机器之间互相访问比较宽松

  3. 域环境（Active Directory）
    企业内网很多都是域环境
    由域控制器统一管理
    域渗透是内网渗透的重头戏
    拿下域控就拿下了一切

  4. 有价值的目标多
    核心数据库、文件服务器、
    邮件服务器、业务系统、OA...
    都在内网里
    数据都在这里

  5. 监控和检测相对薄弱
    很多企业内网没有流量监控，
    没有EDR，
    横向移动很难被发现
```

正因为这些特点，
内网往往比外网更好打，
价值也更大。
所以高级攻击者，
都会把目标放在内网。

### 10.2.3 内网渗透的一般流程

内网渗透大概是个什么流程？

```
  1. 拿到立足点（Webshell / 跳板）
     先拿下内网中的一台机器
     可以是通过外网打进来的，
       也可以是通过社工、
       钓鱼邮件搞进来的
     这台机器就是你的跳板

  2. 内网信息收集
     搞清楚内网的环境：
       • 我在哪？IP、网段、机器名
       • 我是谁？当前用户、权限
       • 这是个什么网络？域还是工作组？
       • 有哪些网段、哪些机器？
       • 域控在哪？有哪些域用户？
       • 网络拓扑大概什么样？
     信息收集是第一位的

  3. 权限提升（提权）
     如果当前权限不够，
     想办法提到更高的权限
     本地system/root权限，
     或者域管理员权限

  4. 凭据获取
     抓取这台机器上的各种凭据：
       • 本地用户密码哈希
       • 域用户密码哈希
       • 明文密码（如果有的话）
       • 服务账号、数据库密码...
     有了凭据才能横向移动

  5. 横向移动
     用拿到的凭据，
     去访问和控制内网中的其他机器
     一台变两台，两台变四台...
     像滚雪球一样

  6. 权限维持 / 持久化
     在拿下的机器上留后门，
     方便以后再进来
     就算这台机器重启、补丁了，
     也还能进来

  7. 痕迹清理
     擦除攻击痕迹，
     删除日志、清理文件...
     让管理员发现不了

  8. 达成目标
     拿到想要的数据、
     拿下域控、
     或者完成其他目标
```

整个过程，
信息收集贯穿始终，
横向移动是核心。

## 10.3 内网信息收集

和外网渗透一样，
内网渗透的第一步也是**信息收集**。
先搞清楚环境，
才能有的放矢。

内网信息收集都收集什么？
怎么收集？

### 10.3.1 本机信息收集

先从当前这台机器开始，
收集本机的信息。

Windows下常用的命令：
```
  # 查看系统信息
  systeminfo
  whoami /all
  hostname

  # 查看网络信息
  ipconfig /all
  route print
  netstat -ano

  # 查看用户
  net user
  net localgroup administrators
  net group "domain admins" /domain

  # 查看域信息
  net config workstation
  net view /domain
  nltest /dclist:域名

  # 查看进程、服务
  tasklist /svc
  net start
  wmic service list brief

  # 查看共享
  net share
  net view \\机器名

  # 查看补丁
  wmic qfe get Caption,HotFixID,InstalledOn
```

Linux下常用的命令：
```
  # 系统信息
  uname -a
  cat /etc/issue
  whoami
  id

  # 网络信息
  ifconfig / ip a
  route -n
  netstat -tulnp
  ss -tulnp

  # 用户信息
  cat /etc/passwd
  cat /etc/shadow
  sudo -l

  # 进程
  ps aux
  top

  # 历史命令
  history
  cat ~/.bash_history

  # 计划任务
  crontab -l
  cat /etc/crontab
```

先把本机摸清楚，
知道自己是谁、在哪、有什么权限。

### 10.3.2 内网存活探测

接下来要知道内网里还有哪些机器，
哪些是活的。

怎么探测？
```
  1. ICMP扫描
     ping扫描整个网段
     但很多机器禁ping

  2. ARP扫描
     内网里扫ARP，
     比ping准
     工具：arp-scan、netdiscover...

  3. 端口扫描
     扫常见端口，
     比如135、139、445、3389、22...
     有回应就是活的
     工具：nmap、masscan、
       还有很多内网专用的扫描脚本

  4. 其他方式
     • 看本机的ARP缓存、DNS缓存
     • 看hosts文件
     • 看应用的配置文件（数据库地址、
       接口地址...）
     • 嗅探内网流量
     • 域环境下可以查询域内机器列表
```

### 10.3.3 端口与服务探测

知道了哪些机器活的，
接下来就是扫端口、看服务。
和外网差不多，
但内网的端口更丰富，
也更容易找到突破口。

内网重点关注的端口：
```
  • 21    FTP
  • 22    SSH
  • 80/443  HTTP/HTTPS  Web应用
  • 135   RPC
  • 139   NetBIOS
  • 445   SMB       重点！横向移动常用
  • 3306  MySQL
  • 3389  RDP       远程桌面
  • 5432  PostgreSQL
  • 6379  Redis
  • 11211 Memcached
  • 27017 MongoDB
  • 5985/5986  WinRM  Windows远程管理
  • 389   LDAP      域相关
  • 88    Kerberos  域相关
  ...还有很多
```

内网很多服务防护都比较弱，
比如Redis未授权、
MongoDB未授权、
MySQL弱口令...
这些都是常见的突破口。

### 10.3.4 域信息收集

如果是域环境，
那域信息收集就非常重要了。

域环境下要收集什么？
```
  • 域名是什么？
  • 域控（DC）是哪台机器？IP多少？
  • 有哪些域用户？
  • 有哪些域管理员？
  • 有哪些域机器？
  • 有哪些组？
  • 域的策略是什么？
  • 等等...
```

收集的方法也很多：
- Windows自带的net命令
- PowerView脚本
- BloodHound（域渗透神器，
  能画出域内的关系图，
  找到攻击路径）
- SharpHound
- ADSI查询
- 等等...

域环境下的信息收集，
是域渗透的基础。
信息收集得好，
后面打起来事半功倍。

## 10.4 权限提升（提权）

信息收集完了，
如果当前权限不够，
就要想办法**提权**。

### 10.4.1 什么是提权？

什么是提权？
简单说就是：
**从低权限提升到高权限。**

比如：
- 普通用户 → 管理员/root
- 网站权限 → 系统权限
- 域用户 → 域管理员

提权分两类：
```
  1. 本地提权
    在当前机器上，
    从普通用户提到系统最高权限
    （Windows的system，Linux的root）

  2. 域内提权
    从普通域用户提到域管理员
    这个属于域渗透的范畴了
```

### 10.4.2 常见提权方式

怎么提权？
方法很多：

```
  系统漏洞提权：
    操作系统有漏洞，
    利用漏洞提升权限
    比如Windows的：
      • MS17-010（永恒之蓝，也能提权）
      • MS16-032
      • MS14-068（域内提权）
      • CVE-2021-40449...
    每个月补丁星期二都会出一堆漏洞
    没打补丁就能提
    工具：
      • Windows：
        MSF的提权模块、
        Empire、
        PowerSploit、
        各种提权EXP
      • Linux：
        脏牛（CVE-2016-5195）、
        各种内核漏洞、
        提权辅助脚本（LinEnum、linux-exploit-suggester...）

  错误配置提权：
    系统配置有问题，
    也能提权
    比如：
      • 服务路径有空格、未加引号
      • 服务权限配置错误
      • 计划任务权限配置错误
      • 注册表权限问题
      • 始终安装（AlwaysInstallElevated）
      • Linux下的SUID、SGID文件
      • sudo配置错误
      • 有漏洞的第三方程序
      等等...
    这种不需要打系统补丁，
    配置错了就能提权

  数据库提权：
    如果数据库以高权限运行，
    利用数据库的功能提权
    比如MySQL的UDF提权、MOF提权
    SQL Server的xp_cmdshell
    Oracle的各种提权方式

  其他提权方式：
    第三方软件漏洞提权、
    凭据窃取提权...
    等等
```

提权是内网渗透中很重要的一步，
有了高权限，
才能干更多的事情，
比如抓密码。

## 10.5 凭据窃取

拿到系统权限之后，
下一步很重要——
**窃取凭据**。
什么凭据？
就是密码、哈希、票据这些东西。
有了凭据，
才能横向移动到其他机器。

### 10.5.1 为什么要窃取凭据？

因为内网里很多机器，
管理员可能用同一个账号密码登录，
或者同一个域用户在很多机器上有权限。
抓到了一个凭据，
可能就能登录很多台机器。
这就是横向移动的基础。

那能窃取到什么凭据？
```
  • 本地用户的密码哈希（LM/NTLM哈希）
  • 域用户的密码哈希
  • 明文密码（如果在内存里的话）
  • Kerberos票据（TGT、ST）
  • 浏览器里保存的密码
  • 各种应用里保存的密码（数据库、FTP...）
  • WiFi密码
  • RDP连接密码
  等等...
```

### 10.5.2 经典神器 mimikatz

说到凭据窃取，
就不得不提 **mimikatz**。
这是最著名的凭据窃取工具，
法国一个牛人写的。
功能极其强大。

mimikatz能干嘛？
```
  • 从内存里抓取明文密码和哈希
    （sekurlsa模块）
  • Pass-the-Hash（哈希传递）
  • Pass-the-Ticket（票据传递）
  • Golden Ticket（黄金票据）
  • Silver Ticket（白银票据）
  • Kerberos相关攻击
  • 等等...
```

mimikatz是内网渗透的神器，
几乎是标配。
当然，
现在杀毒软件和EDR都能查杀mimikatz了，
所以有很多免杀的变种和替代工具，
比如：
- Mimikatz的各种修改版
- Mimipenguin（Linux版的）
- LaZagne（抓取各种密码的工具）
- SharpDump、SafetyKatz...
- 或者直接用PowerShell的脚本

原理都是一样的，
就是从内存里把凭据挖出来。

### 10.5.3 哈希传递（Pass-the-Hash）

窃取到NTLM哈希之后，
不一定非要破解出明文密码。
有个技术叫**哈希传递（PtH）**——
**直接用哈希就能登录，不用明文密码。**

什么意思？
Windows认证的时候，
有些情况下传的就是NTLM哈希，
而不是明文密码。
所以你就算不知道明文，
只要有哈希，
就能通过认证，登录系统。
这就是哈希传递。

这是内网横向移动最常用的手法之一。
抓到一个管理员的哈希，
就能用这个哈希登录所有他有权限的机器。
非常方便。

工具：
- mimikatz
- pth-toolkit
- impacket套件里的psexec、wmiexec...
- Metasploit
- Cobalt Strike

哈希传递是个老技术了，
但依然很好用。
微软也出了一些缓解措施，
但并没有彻底解决。

## 10.6 横向移动

有了凭据，
接下来就是**横向移动**了。

### 10.6.1 什么是横向移动？

什么是横向移动（Lateral Movement）？
简单说就是：
**从已经控制的机器，
移动到内网中的其他机器，
扩大控制范围。**

就像下象棋，
一个子过河了，
然后在对方阵地里横着走，
控制更多的区域。

横向移动是内网渗透的核心，
也是区分攻击水平的重要标志。

### 10.6.2 常见横向移动方式

横向移动的方式有很多，
我给你介绍几种常见的。

**1. IPC$ + 计划任务 / 服务**

最经典的方式之一。
```
  先用IPC$连接目标：
  net use \\目标IP\ipc$ "密码" /user:"用户名"

  然后可以：
    • 复制文件过去（copy）
    • 创建计划任务执行命令（at、schtasks）
    • 创建服务执行命令（sc）
    • 等等
```
老派黑客最喜欢用这个，
系统自带，不容易被查杀。

**2. PsExec**

PsExec是Sysinternals工具集里的一个工具，
本来是给管理员用的，
用来远程执行命令。
但也经常被攻击者用来横向移动。
```
  psexec \\目标IP -u 用户名 -p 密码 cmd.exe
  直接弹一个system权限的cmd回来
```
非常方便，
所以也很常用。
当然，
现在很多杀毒软件都会把PsExec标记为黑客工具。
所以也有很多替代品，
比如Metasploit的psexec模块，
impacket的psexec.py，
还有各种自己实现的版本。

**3. WMI / WinRM**

Windows的WMI（Windows管理规范）
也可以用来远程执行命令。
```
  wmic /node:目标IP /user:用户名 /password:密码 process call create "命令"
```
WMI执行命令不会留下明显的日志，
比较隐蔽，
所以也很受欢迎。
相关工具：
- wmic（系统自带）
- impacket的wmiexec.py
- CrackMapExec
- 等等...

WinRM（Windows远程管理）
也是类似的，
通过PowerShell远程执行。
现在越来越常用了。

**4. RDP 远程桌面**

有了账号密码或者哈希，
直接远程桌面登录上去。
图形界面，操作方便。
RDP哈希传递也有工具，
比如mimikatz的sekurlsa::pth
加上Restricted Admin模式。

**5. SMB / PSExec 类**
就是通过445端口的SMB协议来执行命令，
PsExec就是其中一种。

**6. SSH**

Linux机器之间，
有了密码或者密钥，
直接SSH登录，
或者scp传文件，
ssh执行命令。
这是最基础的。

**7. 数据库 / 中间件**

如果目标机器开了数据库，
有数据库账号密码，
也可以通过数据库来执行命令，
或者提权。
比如MySQL的UDF提权、
SQL Server的xp_cmdshell...

**8. 其他方式**

还有很多其他横向移动方式：
- DCOM
- WinRM
- 组策略（GPO）
- 脚本/批量执行工具
- 等等...

横向移动的方式非常多，
Windows下尤其多。
只要能远程执行命令、
远程传文件，
都能用来横向移动。

### 10.6.3 横向移动的思路

横向移动怎么个思路？
一般是这样：
```
  1. 用当前机器上抓到的凭据，
     去试内网里的其他机器
     看看能不能登录

  2. 登录上新的机器之后，
     再抓这台机器上的凭据
     凭据变多了

  3. 用新的凭据再去试更多的机器
     像滚雪球一样，
     越滚越大

  4. 直到拿到域管理员权限，
     或者拿到目标机器的权限
```

为什么能滚雪球？
因为很多企业里，
管理员图省事，
很多机器用的是同一个本地管理员密码，
或者同一个域管理员账号在很多机器上登录过。
抓到一个，
就能登录一堆。

这也是为什么内网渗透
往往能从一台机器
打到整个内网。

## 10.7 域渗透基础

讲到横向移动，
就不得不提**域渗透**。
企业内网很多都是域环境，
域渗透是内网渗透的重头戏。

### 10.7.1 什么是域？

什么是域（Domain）？
简单说就是：
**由域控制器（DC，Domain Controller）
统一管理的一组计算机和用户的集合。**

用人话说就是：
公司机器多了，
用户多了，
一台一台管太麻烦。
那就搞个域控制器，
所有机器都加入域，
所有用户都在域里统一管理。
你用域账号，
可以在任何一台域机器上登录，
权限由域控统一管理。

域的好处是**方便管理**，
坏处是——
**一旦域控被拿下，
整个域的所有机器就都沦陷了。**

域里的几个概念：
```
  • 域控制器（DC）：
    域的核心，管理整个域
    存着所有域用户、域机器的信息
    由Active Directory（活动目录，AD）服务
    拿下DC就等于拿下整个域

  • 域用户：
    在域里创建的用户账号
    可以在域内的机器上登录

  • 域管理员（Domain Admin）：
    域里权限最高的用户组
    对域内所有机器都有管理员权限
    域渗透的目标之一就是拿到域管权限

  • 组织单元（OU）：
    域里的组织单位，
    用来分类管理用户和机器

  • 组策略（GPO）：
    组策略对象，
    可以批量配置域内机器的策略
    比如开机脚本、密码策略...
    也可以用来横向移动

  • Kerberos：
    域内默认的认证协议
    域渗透里很多攻击都是针对Kerberos的
```

### 10.7.2 Kerberos 协议

说到域渗透，
就绕不开 **Kerberos 协议**。
这是域里默认的认证协议。

Kerberos是什么？
一种网络认证协议，
用票据（Ticket）来做认证，
不用每次都传密码。
名字来自希腊神话里的三头犬克尔柏洛斯，
因为它有三个参与方：
- 客户端
- 服务端
- KDC（密钥分发中心，也就是域控）

Kerberos认证的大致过程比较复杂，
这里就不展开讲了。
你只要知道，
针对Kerberos有几种著名的攻击手法：

```
  • PTT（Pass-the-Ticket，票据传递）
    和哈希传递类似，
    不过传的是Kerberos票据，不是哈希
    有了票据就能访问服务

  • Golden Ticket（黄金票据）
    伪造TGT（票据授权票据）
    有了黄金票据，
    就相当于有了域管理员权限
    想访问什么就访问什么
    前提是要拿到krbtgt用户的哈希

  • Silver Ticket（白银票据）
    伪造某个具体服务的ST（服务票据）
    只能访问指定的服务
    权限比黄金票据小，
    但更隐蔽

  • Kerberoasting
    请求服务票据，
    然后离线破解服务账号的密码
    因为服务账号密码一般设置了就不改，
      而且往往权限很高
    是域渗透中常用的手法

  • AS-REP Roasting
    针对不需要预认证的用户，
    获取AS-REP包，然后离线破解密码
    和Kerberoasting有点像
```

Kerberos相关的攻击非常多，
是域渗透的核心内容。
想深入学习的话，
可以专门去研究Kerberos协议。

### 10.7.3 域渗透的一般思路

域渗透一般怎么打？
```
  1. 先拿下一台域内机器
     不管什么方法，先搞到一个域用户权限
     或者一台域机器的权限

  2. 域信息收集
     收集域的各种信息：
       用户、机器、组、OU、ACL...
     用BloodHound画图，找攻击路径
     这个非常重要！

  3. 找域管理员的路径
    看看怎么能拿到域管理员权限：
      • 找登录了域管的机器，
        打上去抓凭据
      • Kerberoast服务账号，
        服务账号可能是域管
      • 利用ACL漏洞
      • 利用各种域漏洞
        （比如MS14-068、CVE-2020-1472等）
      • 等等...

  4. 拿到域管理员权限

  5. 拿下域控（DC）
     有了域管权限，
     登录域控，
     整个域就是你的了

  6. 权限维持
    在域里留后门，
    比如黄金票据、
    万能钥匙、
    藏个后门账号...
    长期潜伏
```

域渗透水很深，
内容非常多。
这里只是给你做个入门介绍，
让你知道有这么回事。
感兴趣的话可以深入学习。

## 10.8 权限维持与痕迹清理

拿下目标之后，
还有两件事要做：
**权限维持**和**痕迹清理**。

### 10.8.1 权限维持（持久化）

什么是权限维持？
就是**留后门**，
保证以后还能进来。
不能这次进来了，
下次人家补丁一打、密码一改，
就进不来了。

常见的权限维持方式：
```
  本地机器的维持：
    • 后门账号（隐藏用户、克隆管理员）
    • 注册表启动项
    • 计划任务
    • 服务
    • 驱动 / Rootkit
    • WebShell
    • DLL劫持
    • 映像劫持
    • COM劫持
    • 等等...
    花样非常多

  域内的维持：
    • 黄金票据
    • 白银票据
    • 万能钥匙（mimikatz的skeleton key）
    • DSRM后门
    • ACL后门
    • 留个域管理员账号
    • 等等...
```

高级的APT攻击，
权限维持的手段非常隐蔽，
有的甚至在固件里留后门，
重装系统都没用。

### 10.8.2 痕迹清理

打完了还要擦屁股——
**清理痕迹**。
不然管理员一看日志就发现了。

清理什么？
```
  • 系统日志（安全日志、系统日志、应用日志）
  • Web日志
  • 登录日志
  • 命令历史记录
  • 上传的文件、工具
  • 各种缓存
  • 等等...
```

当然，
现在很多企业有日志收集，
日志都传到远程服务器了，
本地删了没用。
所以攻击者也会想办法绕开日志，
或者打日志服务器。
攻防就是这样互相博弈。

## 10.9 本章总结

这一章我们讲了内网渗透与横向移动，
内容比较多，也比较难。
给你总结一下：

**内网渗透概述：**
- 拿下一台机器只是开始，内网才是主战场
- 内网特点：环境复杂、防护薄弱、域环境、
  价值高、监控弱
- 一般流程：立足点 → 信息收集 → 提权 →
  凭据窃取 → 横向移动 → 权限维持 → 痕迹清理

**内网信息收集：**
- 本机信息收集（系统、网络、用户、进程...）
- 内网存活探测（ARP扫描、端口扫描...）
- 端口与服务探测
- 域信息收集（用户、机器、组、域控...）

**权限提升（提权）：**
- 本地提权：系统漏洞、错误配置、数据库提权...
- 域内提权：各种域漏洞、Kerberos攻击...

**凭据窃取：**
- mimikatz —— 经典神器
- 抓明文密码、哈希、票据
- 哈希传递（PtH）—— 有哈希就能登录

**横向移动：**
- IPC$ + 计划任务/服务
- PsExec
- WMI / WinRM
- RDP
- SMB、SSH、数据库...
- 滚雪球模式：抓凭据 → 登录新机器 → 抓更多凭据

**域渗透基础：**
- 域、域控（DC）、活动目录（AD）、域管理员
- Kerberos协议：黄金票据、白银票据、
  Kerberoasting、PTT...
- 域渗透思路：一台立足点 → 信息收集 →
  找攻击路径 → 拿域管 → 拿域控

**权限维持与痕迹清理：**
- 权限维持：各种后门，保证长期访问
- 痕迹清理：删日志、清文件，擦屁股

最后送你一句话：

> **外网是门面，
> 内网是家底。
> 门面再光鲜，
> 家底被偷光了也没用。**
