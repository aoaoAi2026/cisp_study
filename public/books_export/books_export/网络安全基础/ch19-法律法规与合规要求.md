## 19.1 先讲一个故事

### 19.1.1 "删库跑路的程序员"

我给你讲个真实的案例。

某公司的一个运维工程师，
因为跟公司闹矛盾，
被开除了。
他心里不爽，
离职前偷偷留了后门。
离职之后，
他通过后门远程登录公司的服务器，
把数据库里的数据全删了，
还格式化了好几台服务器。
公司业务直接停了，
损失惨重。

然后呢？
公司报警了。
最后这个工程师，
因为**破坏计算机信息系统罪**，
被判了好几年。
不但工作没了，
还留下了案底，
人生都毁了。

还有一个案例，
某公司的程序员，
为了测试一下网站的安全性，
没经过公司同意，
就用工具扫了公司的网站，
还试了几个SQL注入。
结果被公司安全部门发现了，
报警了。
最后他因为**非法侵入计算机信息系统罪**，
被判了缓刑，还罚了钱。

很多做技术的，
法律意识很淡薄，
觉得"我就是技术研究一下"、

"公司是我家，我想怎么弄怎么弄"。
这些想法都是非常危险的。

**网络空间不是法外之地。**
不管你是好心还是恶意，
只要违反了法律，
就要承担法律责任。
很多时候，
你以为是技术问题，
其实是法律问题。

所以，
做网络安全的，
一定要懂法。
知道边界在哪里，
什么能做，什么不能做。
对自己负责，也对公司负责。

这一章，
我们就来讲讲网络安全相关的
法律法规和合规要求。

知道什么是违法的，

## 19.2 我国网络安全法律体系概述

先给你梳理一下，
我们国家的网络安全法律体系，
大概是什么样的。

```
  法律层面（全国人大及其常委会制定）：
    • 《中华人民共和国网络安全法》
    • 《中华人民共和国数据安全法》
    • 《中华人民共和国个人信息保护法》
    • 《中华人民共和国密码法》
    • 《中华人民共和国国家安全法》
    • 《中华人民共和国刑法》（相关条款）
    • 等等

  行政法规层面（国务院制定）：
    • 《关键信息基础设施安全保护条例》
    • 《网络安全等级保护条例》（制定中）
    • 《计算机信息系统安全保护条例》
    • 等等

  部门规章和规范性文件
  （各部委制定，比如公安部、网信办等）：
    • 《网络安全等级保护基本要求》等系列标准
    • 《网络安全审查办法》
    • 《互联网信息服务算法推荐管理规定》
    • 《生成式人工智能服务管理暂行办法》
    • 等等

  国家标准和行业标准：
    • GB/T 22239 等保基本要求
    • GB/T 22080/ISO 27001 信息安全管理体系
    • 等等
```

可以看到，
我们国家的网络安全法律体系
已经比较完善了，
从法律到行政法规到规章到标准，
各个层面都有。
而且还在不断地完善和加强。

下面我们挑几个最重要的来讲。

## 19.3 三大核心法律

首先是最核心的三部法律，
被称为网络安全领域的"三驾马车"：
**《网络安全法》、
《数据安全法》、
《个人信息保护法》。**

### 19.3.1 《网络安全法》

第一部，
**《中华人民共和国网络安全法》**，
简称《网安法》。
2017年6月1日起施行。

这是我国网络安全领域的
**第一部基础性、综合性法律**。
相当于网络安全领域的"母法"，
确立了网络安全的基本制度和框架。

主要内容包括哪些？
```
  1. 网络安全等级保护制度
    国家实行网络安全等级保护制度
    网络运营者要按等级保护要求履行安全义务
    这就是我们常说的"等保"的法律依据

  2. 关键信息基础设施保护
    对关键信息基础设施，实行重点保护
    专门的保护制度和要求

  3. 网络运营者的安全义务
    比如：
      • 制定内部安全管理制度和操作规程
      • 采取防范计算机病毒和网络攻击的措施
      • 采取监测、记录网络运行状态的措施
      • 按照规定留存网络日志不少于六个月
      • 采取数据分类、备份、加密等措施
      • 制定应急预案
      • 等等...
    这些都是法定的义务，
      不做就是违法

  4. 个人信息保护
    确立了个人信息保护的基本规则
    （现在有专门的《个人信息保护法》了，更详细）

  5. 网络安全监测预警与应急处置
    国家建立网络安全监测预警和
      信息通报制度
    网络运营者要制定应急预案，
      及时处置安全事件

  6. 法律责任
    违反了会怎么样？
    罚款、警告、没收违法所得、
      责令改正、暂停相关业务、
      停业整顿、关闭网站、
      吊销相关业务许可证...
    构成犯罪的，依法追究刑事责任
```

《网络安全法》是网络安全的根本大法，
做网络安全的一定要了解。

### 19.3.2 《数据安全法》

第二部，
**《中华人民共和国数据安全法》**，
简称《数安法》。
2021年9月1日起施行。

顾名思义，
这部法律是管**数据安全**的。
现在数据越来越重要，
成了生产要素，
所以专门立法来管。

主要内容：
```
  1. 数据分类分级保护制度
    国家建立数据分类分级保护制度
    根据数据的重要程度，分等级保护
    分为：一般数据、重要数据、核心数据
    重要数据和核心数据重点保护

  2. 数据安全制度
    • 数据安全风险评估
    • 数据安全应急处置
    • 数据安全审查
    • 数据出口管制
    • 数据交易管理
    等等

  3. 数据安全保护义务
    开展数据活动的组织和个人，
      要履行数据安全保护义务
    建立健全全流程数据安全管理制度
    组织开展数据安全教育培训
    采取相应的技术措施和其他必要措施
    保障数据安全

  4. 政务数据安全与开放
    对政务数据的安全和开放做了规定

  5. 法律责任
    违反规定的，
      罚款、责令改正、
      责令暂停相关业务、停业整顿、
      吊销相关业务许可证...
    构成犯罪的，追究刑事责任
    最高可以罚一千万！
```

做数据相关业务的，
一定要好好研究这部法律。

### 19.3.3 《个人信息保护法》

第三部，
**《中华人民共和国个人信息保护法》**，
简称《个保法》。
2021年11月1日起施行。

这部法律是管**个人信息保护**的，
相当于中国版的GDPR。
跟每个人都息息相关。

主要内容：
```
  1. 个人信息的定义
    什么是个人信息？什么是敏感个人信息？
    前面第13章讲过，这里就不重复了

  2. 处理个人信息的基本原则
    • 合法、正当、必要、诚信
    • 目的明确、最小必要
    • 公开透明
    • 质量保证
    • 安全保障
    • 责任原则

  3. 个人信息处理的规则
    • 取得个人同意（一般情况）
    • 告知义务：告诉个人处理什么、怎么用
    • 敏感个人信息要单独同意
    • 处理未成年人信息要监护人同意
    • 等等...

  4. 个人的权利
    知情权、决定权、查阅复制权、
    更正补充权、删除权（被遗忘权）、
    可携带权、解释说明权...

  5. 个人信息处理者的义务
    • 制定内部管理制度和操作规程
    • 对个人信息实行分类管理
    • 采取相应的加密、去标识化等安全技术措施
    • 合理确定操作权限，定期安全教育和培训
    • 制定并组织实施应急预案
    • 定期进行合规审计
    • 等等...

  6. 个人信息跨境提供
    个人信息出境的规则和要求
    不是想传出去就能传出去的

  7. 法律责任
    违反本法的，
    责令改正，给予警告，没收违法所得
    拒不改正的，并处一百万元以下罚款
    对直接负责的主管人员和其他直接责任人员
      处一万元以上十万元以下罚款

    情节严重的：
    • 五千万元以下 或者 上一年度营业额百分之五以下罚款
    • 可以责令暂停相关业务或者停业整顿
    • 通报有关主管部门吊销相关业务许可
      或者吊销营业执照
    • 直接负责的主管人员和其他直接责任人员
      处十万元以上一百万元以下罚款
    而且还可以决定禁止其在一定期限内
      担任董事、监事、高级管理人员和
      个人信息保护负责人

    这个处罚力度是相当大的！
    跟GDPR有一拼。
```

《个保法》的处罚力度非常大，
最高可处上一年度营业额5%的罚款。
对大公司来说，
就是几十亿上百亿的罚款，
非常有威慑力。
现在做互联网、做用户业务的公司，
都非常重视这部法律。

## 19.4 其他重要的法律法规

除了三大核心法律，
还有几个比较重要的，
简单给你介绍一下。

### 19.4.1 关键信息基础设施安全保护条例

**《关键信息基础设施安全保护条例》**，
2021年9月1日起施行。

什么是关键信息基础设施（CII）？
简单说就是：
**一旦遭到破坏、丧失功能或者数据泄露，
可能危害国家安全、国计民生、
公共利益的网络设施和信息系统。**

比如：
- 能源、电力、交通、水利、金融...
- 公共通信、信息服务...
- 政务、国防...
- 等等...

这些都是关键信息基础设施，
是国家的命根子，
所以要重点保护。

这个条例规定了：
- 关键信息基础设施的认定
- 运营者的安全保护义务
  （比等保要求更高）
- 保障和促进措施
- 法律责任

如果你所在的行业属于关键信息基础设施，
那安全要求会比普通企业高很多。

### 19.4.2 网络安全审查办法

**《网络安全审查办法》**，
最新版2022年2月15日起施行。

什么是网络安全审查？
就是：
**对影响或者可能影响国家安全的
网络平台运营者开展数据处理活动，
进行网络安全审查。**

简单说就是：
你这个平台如果涉及国家安全，
那就要接受审查，
看看有没有安全风险。

哪些情况要申报网络安全审查？
```
  • 关键信息基础设施运营者
    采购网络产品和服务，
    影响或者可能影响国家安全的

  • 掌握超过100万用户个人信息的
    网络平台运营者赴国外上市
    必须申报网络安全审查

  • 其他影响或者可能影响国家安全的
    数据处理活动
```

网络安全审查的重点是什么？
重点审查产品和服务、
数据处理活动
使用后可能带来的国家安全风险。

简单说，
就是关系到国家安全的，
都要审查。
准备赴美上市的互联网公司，
这个一定要注意。

### 19.4.3 《刑法》相关条款

最后讲一个最严重的——
**刑法**。
前面说的都是行政违法，
罚钱、停业什么的。
如果情节严重，
构成犯罪了，
那就要负刑事责任，
也就是要坐牢的。

刑法里跟网络安全相关的罪名，
常见的有这几个：

```
  1. 非法侵入计算机信息系统罪
    违反国家规定，侵入国家事务、
      国防建设、尖端科学技术领域的
      计算机信息系统的
    处三年以下有期徒刑或者拘役

  2. 非法获取计算机信息系统数据、
     非法控制计算机信息系统罪
    违反国家规定，侵入其他计算机信息系统
      或者采用其他技术手段，
      获取该系统中存储、处理或者传输的数据，
      或者对该系统实施非法控制
    情节严重的，
      处三年以下有期徒刑或者拘役，
      并处或者单处罚金
    情节特别严重的，
      处三年以上七年以下有期徒刑，
      并处罚金

  3. 提供侵入、非法控制计算机信息系统程序、工具罪
    提供专门用于侵入、非法控制计算机
      信息系统的程序、工具，
    或者明知他人实施该行为，
      还提供程序工具的
    情节严重的，跟上面一样

  4. 破坏计算机信息系统罪
    对计算机信息系统功能进行删除、修改、
      增加、干扰，
    造成计算机信息系统不能正常运行
    或者对系统中存储、处理或者传输的数据
      和应用程序进行删除、修改、增加操作
    后果严重的，处五年以下有期徒刑或者拘役
    后果特别严重的，处五年以上有期徒刑
    （前面讲的删库跑路那个，
      就是这个罪名）

  5. 拒不履行信息网络安全管理义务罪
    网络服务提供者不履行法律、行政法规
      规定的信息网络安全管理义务，
    经监管部门责令采取改正措施
      而拒不改正
    有法定情形的，
      处三年以下有期徒刑、拘役或者管制，
      并处或者单处罚金

  6. 帮助信息网络犯罪活动罪（帮信罪）
    明知他人利用信息网络实施犯罪，
    为其犯罪提供互联网接入、服务器托管、
      网络存储、通讯传输等技术支持，
    或者提供广告推广、支付结算等帮助
    情节严重的，
      处三年以下有期徒刑或者拘役，
      并处或者单处罚金
    这个是现在比较火的罪名，
      很多人稀里糊涂就犯了

  7. 侵犯公民个人信息罪
    违反国家有关规定，
      向他人出售或者提供公民个人信息，
    情节严重的，
      处三年以下有期徒刑或者拘役，
      并处或者单处罚金
    情节特别严重的，
      处三年以上七年以下有期徒刑，
      并处罚金

  还有很多其他相关罪名...
```

这些罪名都不是闹着玩的，
真的会坐牢的。
所以做技术的，
一定要有法律意识，
什么能做什么不能做，
心里要有数。
不要觉得"我就是玩玩"、
"我就是研究一下"，
真追究起来，
后悔都来不及。

## 19.5 企业的安全合规义务

讲了这么多法律，
那企业具体有哪些合规义务？
给你梳理一下主要的。

```
  制度与人员：
    • 建立内部安全管理制度和操作规程
    • 确定网络安全负责人
    • 组织安全培训和教育
    • 设立专门的安全管理机构和人员
      （关键信息基础设施运营者）

  技术防护：
    • 采取防范计算机病毒和网络攻击、
      网络侵入等危害网络安全行为的技术措施
    • 采取监测、记录网络运行状态、
      网络安全事件的技术措施
    • 采取数据分类、重要数据备份和加密等措施
    • 防范非法获取用户个人信息的行为
    • 等等

  日志与审计：
    • 按照规定留存相关的网络日志不少于六个月
    • （等保要求是六个月，
      很多行业要求更长）
    • 对系统操作、数据操作进行审计
    • 保留审计记录

  漏洞与补丁：
    • 及时修补系统漏洞
    • 发现安全漏洞及时处置
    • 按规定向有关部门报告

  等级保护：
    • 履行网络安全等级保护义务
    • 定级、备案、建设整改、等级测评、
      监督检查
    • 这是法定义务，不是可选的

  应急响应：
    • 制定网络安全事件应急预案
    • 及时处置系统漏洞、计算机病毒、
      网络攻击、网络侵入等安全风险
    • 发生安全事件时，
      立即启动应急预案，
      采取相应的补救措施
    • 按照规定向有关主管部门报告

  个人信息保护：
    • 合法、正当、必要原则
    • 告知并取得同意
    • 采取安全措施保护个人信息
    • 个人信息泄露了要及时通知用户和监管
    • 保障个人的各项权利

  数据安全：
    • 建立数据安全管理制度
    • 数据分类分级保护
    • 重要数据重点保护
    • 数据安全风险评估
    • 数据出境安全管理

  关键信息基础设施：
    （如果是CII运营者，还有额外义务）
    • 设置专门安全管理机构
    • 负责人安全背景审查
    • 定期开展安全检测评估
    • 采购网络产品和服务要安全审查
    • 重要数据要在境内存储，
      出境要进行安全评估
    • 每年至少进行一次网络安全检测和风险评估
    • 等等...
```

这些都是企业法定的安全义务。
不是可做可不做的，
是必须做的，
不做就是违法，
会被处罚的。

## 19.6 常见的合规体系

除了法律要求，
企业常见的合规体系还有哪些？
我们前面提到过一些，
这里再总结一下。

### 19.6.1 等级保护2.0

这个第5章讲过，
这里再简单提一下。
等级保护是我国网络安全的基本制度，
是法律要求的，
不是可选的。

五个等级：
- 第一级：自主保护级
- 第二级：指导保护级
- 第三级：监督保护级
- 第四级：强制保护级
- 第五级：专控保护级

一般企业的业务系统，
二级或者三级比较常见。

等保工作流程：
定级 → 备案 → 建设整改 → 等级测评 → 监督检查

三级系统每年测一次，
二级每两年测一次。

### 19.6.2 ISO 27001 / ISO 27002

ISO 27001是国际标准，
信息安全管理体系的标准。
第5章也讲过。

ISO 27001是国际上最通用的
信息安全管理体系标准。
很多企业做认证，
一方面是提升自身安全水平，
另一方面也是客户要求、
招投标需要。

ISO 27001和等保的区别：
- 等保是国内的、强制的（达到一定级别要求做）
- ISO 27001是国际的、自愿的（企业自愿做）
- 等保偏技术和合规
- ISO 27001偏管理体系
- 两者可以互相借鉴，一起做

### 19.6.3 行业合规要求

不同行业还有自己的合规要求，
比如：

```
  金融行业：
    • 银保监会的监管要求
    • 《网络安全法》《数据安全法》等
    • 金融行业等保要求更高
    • 金融数据安全规范
    • 个人金融信息保护
    • 等等...

  医疗行业：
    • 医疗健康数据安全
    • 患者隐私保护
    • 医院等级评审的安全要求
    • 等保要求
    • 等等...

  教育行业：
    • 教育系统网络安全规定
    • 学生个人信息保护
    • 等保
    • 等等...

  政府/政务：
    • 政务信息系统安全要求
    • 等保要求更高
    • 密码应用要求（密评）
    • 等等...

  还有很多其他行业...
```

每个行业都有自己的监管要求，
做哪个行业的安全，
就要了解哪个行业的规定。

## 19.7 给技术人员的法律建议

最后，
给做技术的朋友们
几条法律建议：

```
  1. 要有法律意识
    网络空间不是法外之地
    做什么事情之前，
      先想想合不合法
    别稀里糊涂犯了法

  2. 未经授权不要搞
    不管是测试、研究还是什么，
      不是自己的系统，
      没有明确授权，
      不要去扫、不要去测、不要去碰
    哪怕你是好意，
      哪怕你只是看看
    真追究起来都是违法的
    （白帽子也要注意，
      一定要在授权范围内测试）

  3. 不要帮人干可能违法的事
    朋友让你帮忙写个爬虫、
      做个外挂、搭个翻墙...
    看起来是小事，
    但可能已经违法了
    严重的可能构成犯罪
    （帮信罪了解一下）
    帮忙之前先想想合不合法

  4. 保护好用户数据和个人信息
    做开发、做运维的，
      能接触到很多用户数据
    不要随便看、不要随便导、
      更不能拿去卖、拿去用
    侵犯公民个人信息罪是要坐牢的

  5. 不要搞破坏
    哪怕受了委屈、跟公司闹矛盾，
      也不要做删库、删数据、
      搞破坏的事
    一时冲动，
      代价可能是你的一生
    得不偿失

  6. 学习基本的法律知识
    不用成为法律专家，
      但基本的法律常识要有
    知道什么能做、什么不能做
    知道有什么风险
    保护好自己

  7. 遇到问题找专业人士
    拿不准的事，
      找公司法务或者专业律师问问
    别自己瞎猜、瞎决定
```

一句话总结：
**守好底线，不碰红线，
对自己负责，对他人负责。**

## 19.8 本章总结

这一章我们讲了法律法规与合规要求。
总结一下：

**三大核心法律：**
- **《网络安全法》**：基础性、综合性法律，确立等保、关键信息基础设施保护等制度
- **《数据安全法》**：数据安全的专门法律，数据分类分级、数据安全制度
- **《个人信息保护法》**：个人信息保护，处罚力度大，最高上一年度营业额5%罚款

**其他重要法规：**
- 《关键信息基础设施安全保护条例》：重点保护CII
- 《网络安全审查办法》：影响国家安全的要审查
- 《刑法》相关条款：非法侵入、破坏计算机信息系统、帮信罪、侵犯公民个人信息... 严重的要坐牢

**企业的安全合规义务：**
- 制度人员、技术防护、日志审计、漏洞补丁
- 等级保护、应急响应
- 个人信息保护、数据安全
- 关键信息基础设施还有额外义务

**常见合规体系：**
- 等级保护2.0（国内强制）
- ISO 27001（国际标准，自愿）
- 各行业的合规要求

**给技术人员的建议：**
- 有法律意识
- 未经授权不要搞
- 不要帮人干违法的事
- 保护好用户数据
- 不要搞破坏
- 学法懂法，保护好自己

最后送你一句话：

> **技术是工具，
> 法律是底线。
> 技术越厉害，
> 越要知道边界在哪里。**
