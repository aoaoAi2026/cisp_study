## 8.1 先讲一个故事

### 8.1.1 "一封邮件引发的灾难"

我给你讲个真实的故事。

2017年，
有一款勒索软件席卷全球，
造成了巨大的损失，
它的名字叫——
**WannaCry（想哭）。**

WannaCry是怎么传播的？
它利用了Windows的一个永恒之蓝漏洞（EternalBlue），
这个漏洞是NSA（美国国家安全局）
开发的黑客工具，
后来被泄露出来了。
WannaCry用这个漏洞，
扫描互联网上开了445端口的Windows机器，
只要机器没打补丁，
一扫描到就自动感染，
感染了之后又继续扫描其他机器，
像蠕虫一样自我复制、快速传播。

短短几天时间，
就有超过150个国家、
30多万台机器被感染。
医院、学校、政府、企业...
都中招了。
文件被加密，
要付比特币赎金才能解密。
造成的损失估计有几十亿美元。

你可能会问：
这个漏洞微软不是已经发补丁了吗？
怎么还有这么多机器中招？
因为很多人、很多企业，
根本不打补丁。
觉得"我这机器好好的，
打什么补丁，麻烦。"
结果呢？
一爆发就全中招了。

---

WannaCry只是勒索软件的一个代表。
现在勒索软件越来越猖獗，
攻击目标从个人到企业、
从医院到政府、从学校到工厂，
无一幸免。
而且勒索软件已经形成了产业链，
有RaaS（勒索软件即服务），
不会写代码的人也能搞勒索攻击，
门槛越来越低，
攻击越来越多。

恶意代码不是过去式，
而是现在进行时，
而且愈演愈烈。
这一章，
我们就来好好认识一下它们。

## 8.2 恶意代码概述

### 8.2.1 什么是恶意代码？

什么是恶意代码（Malware，Malicious Software）？
简单说就是：
**故意制造的、
用来对计算机系统、网络或用户
造成损害或进行未授权操作的软件或代码。**

用人话说就是：
**不干好事的程序和代码，都叫恶意代码。**

恶意代码是一个总称，
它包含很多种类，
比如病毒、木马、蠕虫、
勒索软件、间谍软件、广告软件、
后门、Rootkit、僵尸网络、
挖矿木马、恶意脚本...
都是恶意代码。

### 8.2.2 恶意代码的历史

恶意代码的历史几乎和计算机一样长。
简单给你捋一捋：

```
  1970年代：
    最早的计算机病毒概念出现
    1971年的Creeper（爬行者），
    被认为是第一个计算机病毒
    不过它更像个实验，没什么危害

  1980年代：
    真正的病毒开始出现
    1986年的Brain病毒，
    第一个IBM PC病毒
    通过软盘传播
    那时候主要是软盘传播

  1990年代：
    病毒越来越多
    宏病毒、邮件病毒出现
    比如著名的"我爱你"病毒（ILOVEYOU），
    2000年的，通过邮件传播，
    造成了百亿美元的损失

  2000年代：
    蠕虫开始流行
    比如冲击波、震荡波、熊猫烧香...
    利用系统漏洞主动传播
    木马、后门也越来越多
    灰色产业链开始形成

  2010年代：
    勒索软件开始爆发
    比如CryptoLocker、WannaCry...
    挖矿木马流行
    APT攻击越来越多
    恶意代码越来越专业化、
    产业化、武器化

  2020年代：
    勒索软件攻击愈演愈烈
    RaaS（勒索软件即服务）盛行
    供应链攻击兴起
    比如SolarWinds攻击
    恶意代码更加隐蔽、更加智能
```

恶意代码一直在进化，
从简单到复杂，
从个人恶作剧到有组织的犯罪，
甚至国家级的网络武器。
道高一尺，魔高一丈。

### 8.2.3 恶意代码的传播方式

恶意代码是怎么传播的？
方式很多：
```
  1. 漏洞利用
    利用系统或软件的漏洞
    不管是远程漏洞还是本地漏洞
    比如WannaCry用永恒之蓝漏洞
    这是蠕虫常用的传播方式

  2. 钓鱼邮件 / 社会工程学
    最常见的传播方式之一
    发钓鱼邮件，
    附件是恶意文件，
    或者邮件里有恶意链接
    你一点开就中招
    很多勒索软件都是这么传播的

  3. 恶意网站 / 水坑攻击
    访问了带毒的网站，
    利用浏览器漏洞或者插件漏洞
    偷偷下载运行恶意代码
    （挂马）
    或者诱导用户下载

  4. 移动存储介质
    U盘、移动硬盘...
    插了带毒的U盘就中招
    比如最早的软盘，后来的U盘

  5. 软件捆绑 / 下载站
    从非正规渠道下载软件，
    软件里捆绑了恶意程序
    你装软件的时候一起装上了
    很多弹窗广告、流氓软件都是这么来的

  6. 供应链攻击
    攻击软件供应商，
    在软件更新里植入恶意代码
    然后所有用这个软件的用户都中招
    比如SolarWinds事件
    这种攻击危害大，隐蔽性强

  7. 内网传播
    一台机器中招了之后，
    在内网里横向移动，
    感染更多机器
    比如弱口令、漏洞、共享文件夹...
    企业内网里传播很快

  还有很多其他方式...
```

传播方式多种多样，
防不胜防。
所以安全防护要多层防御，
不能只靠某一层。

## 8.3 常见恶意代码分类

接下来，
我们一个一个讲常见的恶意代码种类。

### 8.3.1 计算机病毒（Virus）

第一个，
**计算机病毒**。
这是最古老、最广为人知的恶意代码种类。

什么是病毒？
病毒有两个特点：
```
  1. 自我复制
    把自己的代码复制到其他程序或文件里
    感染其他文件

  2. 需要宿主
    病毒不能独立运行，
    必须寄生在其他程序里
    你运行了被感染的程序，
    病毒才会运行
```

就像生物病毒一样，
必须寄生在活细胞里才能繁殖。
计算机病毒也是，
必须寄生在宿主程序里。

病毒的危害：
- 破坏文件、破坏系统
- 占用系统资源，让电脑变慢
- 窃取信息
- 等等

病毒现在相对少一些了，
因为现在的系统和杀毒软件都比较成熟，
传统病毒没那么容易存活了。
但不是没有了。

### 8.3.2 蠕虫（Worm）

第二个，
**蠕虫**。
蠕虫和病毒有点像，
都能自我复制，
但有个关键区别：

```
  病毒：需要宿主，不能独立运行
  蠕虫：不需要宿主，可以独立运行
         而且能主动通过网络传播
```

蠕虫最大的特点就是：
**自我复制 + 主动通过网络传播。**
它不需要你运行什么程序，
只要你的机器有漏洞，
它就能感染你，
然后你又去感染别人。
一传十，十传百，
传播速度非常快。

比如著名的：
- 冲击波蠕虫（Blaster）
- 震荡波蠕虫（Sasser）
- 熊猫烧香（其实是病毒蠕虫混合体）
- WannaCry（勒索软件 + 蠕虫）

蠕虫的危害很大，
因为传播速度快，
感染范围广，
往往一爆发就是大规模的。

### 8.3.3 特洛伊木马（Trojan）

第三个，
**特洛伊木马**，
简称木马（Trojan）。
名字来自特洛伊木马的故事。

什么是木马？
```
  伪装成正常的软件，
  暗地里干坏事。
```

你下载了一个"破解软件"、
"游戏外挂"、"免费电影播放器"，
看起来是正常的软件，
你运行了之后，
表面上好像没什么事，
但背地里它偷偷干坏事——
比如偷你的密码、
给你的机器开后门、
监控你的屏幕、
记录你的键盘输入...

这就是木马。
木马的特点是**伪装 + 隐蔽**，
表面正常，背地里搞事情。

木马的种类很多：
```
  • 远程控制木马（远控木马）
    攻击者可以远程控制你的电脑
    想干嘛干嘛
    比如灰鸽子、冰河...都是经典的远控木马

  • 盗号木马
    专门偷各种账号密码
    游戏账号、银行账号、社交账号...

  • 键盘记录木马
    记录你敲的所有键盘输入
    偷密码、偷聊天记录

  • 间谍软件
    收集你的各种信息，
    上网记录、文件、摄像头...
    然后发出去

  • 勒索木马
    就是勒索软件，后面专门讲

  • 挖矿木马
    偷偷用你的电脑挖矿，
    后面专门讲

  • ......还有很多
```

木马是现在最常见的恶意代码类型之一，
因为经济利益驱动，
做木马的人很多。

### 8.3.4 后门（Backdoor）

第四个，
**后门**。
什么是后门？
简单说就是：
**绕过正常的安全验证，
偷偷进入系统的通道。**

就像你家大门锁得好好的，
但有人偷偷在墙上挖了个洞，
他可以从洞里进来，
不用走大门，
你也发现不了。
这个洞就是后门。

后门怎么来的？
```
  1. 木马开的后门
    木马感染之后，
    在系统里留个后门，
    攻击者随时可以进来

  2. 开发者留的后门
    有些软件开发者，
    故意在软件里留后门，
    方便自己进去
    （当然这是违法的）

  3. 硬件后门
    硬件里的后门，更隐蔽

  4. Web后门（Webshell）
    Web应用里的后门，
    比如上传了一个webshell，
    随时可以通过网页执行命令
```

后门的危害很大，
因为它隐蔽，
你不知道它存在，
攻击者可以随时进来，
为所欲为。

### 8.3.5 Rootkit

第五个，
**Rootkit**。
Rootkit是什么？
Root + Kit，
就是获取root（最高）权限的工具包。

Rootkit的特点是**隐蔽**。
它能隐藏自己的存在——
- 隐藏进程，任务管理器里看不到
- 隐藏文件，资源管理器里看不到
- 隐藏网络连接，netstat看不到
- 隐藏注册表项...

它用各种手段，
让你发现不了它。
甚至杀毒软件也很难发现。

Rootkit有什么用？
一般是配合其他恶意代码用的。
比如木马装上之后，
再装个Rootkit，
把自己隐藏起来，
这样用户和杀毒软件都发现不了，
它就能长期潜伏。

Rootkit分几种：
```
  • 用户态Rootkit
    在用户层，隐藏自己
    相对容易检测

  • 内核态Rootkit
    在内核层，驱动级别
    权限更高，更隐蔽，也更难检测

  • 固件Rootkit
    在BIOS、固件里
    重装系统都没用
    最隐蔽，也最危险

  • 硬件Rootkit
    硬件层面的，更不用说了
```

Rootkit是比较高级的恶意代码，
一般APT攻击里比较常见。

## 8.4 勒索软件（Ransomware）

接下来单独讲一下**勒索软件**，
因为它现在太火了，
危害也太大了。
专门拿一节来讲。

### 8.4.1 什么是勒索软件？

什么是勒索软件？
简单说就是：
**加密你的文件或者锁住你的系统，
然后向你索要赎金，
说你付了钱就给你解密。**

就像现实中的绑架勒索一样，
只不过绑架的是你的数据。

勒索软件一般加密什么？
- 用户的文档、照片、视频...
- 企业的业务数据、数据库、备份...
- 只要是重要的文件，都给你加密
- 后缀名改成乱七八糟的
- 然后留个勒索信，
  告诉你怎么付赎金，
  付多少，限时多久...

为什么用比特币？
因为比特币匿名，
不好追踪，
所以勒索一般都要比特币或者其他加密货币。

### 8.4.2 勒索软件的历史

勒索软件不是什么新鲜东西，
很早就有了。
但之前一直不温不火，
直到比特币流行起来之后，
有了匿名的支付方式，
勒索软件才真正爆发。

几个重要的节点：
```
  2013年：CryptoLocker
    第一个大规模的勒索软件
    用RSA+AES加密，非常难破解
    赚了好几千万美元
    让大家看到了勒索软件的"钱景"

  2017年：WannaCry
    最著名的勒索软件事件
    利用永恒之蓝漏洞，蠕虫式传播
    全球大爆发，30多万台机器中招
    影响巨大

  2019年之后：RaaS兴起
    RaaS = Ransomware as a Service
    勒索软件即服务
    做勒索软件的团伙把勒索软件"出租"出去
    不会写代码的人也能搞勒索攻击
    只要付加盟费或者分成
    门槛大大降低，攻击数量暴增

  现在：
    勒索软件已经形成了完整的产业链
    有开发的、有分销的、有谈判的、
    有洗钱的...分工明确
    攻击目标越来越大，
    从个人到中小企业到大企业
    甚至政府机构、关键基础设施
    还搞双重勒索——
      不仅加密你数据，还偷你数据
      不给钱就把数据公开
    越来越恶劣
```

勒索软件现在已经是网络安全的头号威胁之一了。
每年造成的损失高达数百亿美元。

### 8.4.3 勒索软件怎么传播？

勒索软件的传播方式很多：
```
  1. 钓鱼邮件
    最常见的
    发钓鱼邮件，附件是恶意文档
    （宏病毒、恶意Office文件）
    或者链接到恶意网站
    你一打开就中招
    很多企业中招都是从一封邮件开始的

  2. 漏洞利用
    利用系统漏洞、服务漏洞
    比如RDP弱口令爆破、
    VPN漏洞、
    各种远程代码执行漏洞
    打进来之后再横向移动，
    全内网加密

  3. 远程桌面暴力破解
    RDP、SSH这些远程服务，
    弱口令，被爆破进去
    然后手动装勒索软件
    这种针对性很强

  4. 供应链攻击
    通过软件供应链感染
    比如被攻击的软件更新包

  5. 其他
    木马下载器、漏洞利用工具包...
```

### 8.4.4 怎么防御勒索软件？

勒索软件这么凶，
怎么防？
勒索软件的防御是个系统工程，
要多层防护。

```
  1. 补丁管理
    及时打补丁！
    很多勒索软件都是利用已知漏洞
    补丁打了就没事了
    尤其是高危漏洞，一定要及时打
    （WannaCry就是教训）

  2. 备份！备份！备份！
    重要的事情说三遍
    定期备份重要数据
    而且备份要离线、异地、
    不能和生产网络连在一起
    不然勒索软件把备份也加密了
    有备份跟没备份一样
    只要有干净的备份，
    大不了恢复备份，
    不用给勒索软件交钱
    这是对付勒索软件最有效的手段之一

  3. 杀毒软件 / EDR
    安装杀毒软件或者EDR
    能检测和拦截大部分已知的勒索软件
    但要注意更新病毒库

  4. 邮件安全
    很多勒索软件是通过邮件进来的
    所以邮件安全很重要
    垃圾邮件过滤、
    附件安全扫描、
    钓鱼邮件检测...
    还有员工的安全意识培训

  5. 最小权限原则
    用户不要用管理员权限日常使用
    就算中招了，
    普通用户权限危害也小一些

  6. 网络分段
    内网不要扁平化
    要分段隔离
    就算一段中招了，
    也不会传染到全内网
    限制横向移动

  7. 禁用宏
    Office宏是勒索软件常用的载体
    非必要的话禁用宏
    或者限制宏的执行

  8. 应用白名单
    只允许运行经过批准的程序
    其他程序都不让运行
    勒索软件自然也跑不起来
    防护效果很好
    但管理成本比较高

  9. 安全意识培训
    提高员工的安全意识
    不要随便点陌生邮件的附件和链接
    很多攻击都是从人突破的
    人是最大的安全弱点

  10. 应急响应预案
    提前做好预案
    真中招了怎么办？
    怎么隔离？怎么恢复？
    事先有预案，
    出事了才不会慌
```

对付勒索软件，
**备份是底线，
多层防护是关键，
意识是基础。**
任何单一的防护都不够，
必须多层防御，纵深防御。

## 8.5 挖矿木马

再讲一个现在也很常见的——
**挖矿木马（Cryptominer）。**

### 8.5.1 什么是挖矿木马？

什么是挖矿木马？
简单说就是：
**偷偷在你电脑上运行，
用你的CPU、GPU算力来挖加密货币，
给攻击者赚钱。**

为什么挖矿木马这么流行？
因为加密货币值钱啊，
挖了币就能卖钱，
一本万利。
而且挖矿木马相对"温和"，
它不毁你数据，
也不偷你密码，
就是偷偷用你点电、占你点CPU，
很多用户发现不了，
就算发现了也觉得"不就是卡一点吗，没什么大不了"。
所以挖矿木马泛滥成灾。

### 8.5.2 挖矿木马的传播

挖矿木马怎么传播的？
```
  1. 网页挖矿
    网站里植入挖矿脚本（JS）
    你一访问这个网站，
    浏览器就开始挖矿
    关了网页就停了
    比如以前的CoinHive
    现在被禁了，但还有其他的

  2. 软件捆绑
    下载的软件里捆绑了挖矿程序
    你装软件就一起装上了
    后台偷偷挖

  3. 漏洞利用
    利用漏洞入侵服务器，
    然后种挖矿木马
    服务器性能好，
    挖起来更赚
    很多挖矿团伙专门扫
    Redis未授权、Docker未授权...
    打进去就挖矿

  4. 木马下载器
    先下个下载器，
    下载器再下载挖矿木马
```

### 8.5.3 挖矿木马的危害

挖矿木马危害大吗？
说大不大，说小不小。

```
  对个人用户：
    • 电脑变卡、变慢
    • 耗电增加，电费变多
    • 硬件损耗加速
    一般没有直接的经济损失

  对企业服务器：
    • 服务器性能被占用
      影响正常业务
    • 电力和散热成本增加
    • 说明服务器被入侵了，
      既然能种挖矿木马，
      说不定还装了别的
      这是更大的安全隐患

  对云服务器：
    • 云服务器被入侵挖矿，
      产生高额的算力费用
    • 还有可能被拿去干别的坏事
```

挖矿木马虽然不像勒索软件那么破坏性强，
但它反映了系统已经被入侵了，
这个问题更严重。
发现挖矿木马，
不能只把木马删了就完事了，
还要查清楚是怎么进来的，
有没有其他后门，
不然还会再来。

## 8.6 僵尸网络与DDoS

再讲两个相关的——
**僵尸网络（Botnet）** 和 **DDoS攻击**。

### 8.6.1 什么是僵尸网络？

什么是僵尸网络？
简单说就是：
**攻击者控制了一大批被感染的机器（僵尸主机），
组成一个网络，
可以统一指挥，
让它们同时干一件事。**

这些被控制的机器就叫"僵尸机"或者"肉鸡"，
它们的主人往往根本不知道自己的电脑被控制了。
攻击者（叫"牧主"或者"控制者"）
通过命令与控制信道（C&C）
来控制这些僵尸机。

僵尸网络能干嘛？
```
  1. DDoS攻击
    让所有僵尸机同时向一个目标发请求
    流量巨大，直接把目标打瘫
    这是僵尸网络最常见的用途

  2. 发送垃圾邮件
    用僵尸机发垃圾邮件、钓鱼邮件
    数量巨大，还不容易被封

  3. 挖矿
    用僵尸机挖矿，赚比特币

  4. 点击欺诈
    刷广告点击，骗广告费

  5. 暴力破解
    用僵尸机爆破密码、扫漏洞

  6. 传播更多恶意代码
    继续感染更多机器，
    壮大僵尸网络
```

僵尸网络规模有多大？
大的僵尸网络能有几十万台、
甚至上百万台僵尸机。
非常可怕。

### 8.6.2 什么是DDoS？

DDoS是什么？
Distributed Denial of Service，
分布式拒绝服务攻击。

什么意思？
就是用大量的机器（分布式的），
同时向目标发送大量的请求或数据包，
把目标的带宽、连接、计算资源耗尽，
让正常用户无法访问。
也就是"拒绝服务"。

DDoS就像——
正常情况下一家店门口人不多，
大家都能进去买东西。
结果来了一万个人同时堵在门口，
真正想买东西的人根本进不去。
这就是拒绝服务。

DDoS的种类很多：
```
  • 流量型攻击
    靠大流量把带宽塞满
    比如UDP洪水、ICMP洪水、
    放大攻击（DNS放大、NTP放大...）
    攻击流量能达到几十G、上百G甚至T级别

  • 连接型攻击
    把服务器的连接数占满
    比如SYN洪水，
    利用TCP三次握手的缺陷
    发大量SYN包，不回应
    把服务器的半开连接占满

  • 应用层攻击
    针对Web应用的
    比如HTTP洪水，
    大量请求页面、接口
    把服务器、数据库打挂
    CC攻击就是一种
    这种攻击流量不一定大，
    但专门打应用的弱点
    防御起来更麻烦
```

DDoS攻击很难彻底防住，
只能缓解。
一般用高防CDN、
流量清洗、
冗余带宽...
来扛。

## 8.7 恶意代码的检测与防御

讲了这么多种恶意代码，
最后我们讲讲怎么检测和防御。

### 8.7.1 恶意代码检测技术

恶意代码怎么检测？
常见的检测技术有这些：

```
  1. 特征码检测
    最经典的检测方法
    提取恶意代码的特征（一段特定的代码）
    做成病毒库
    扫描文件的时候，
    看文件里有没有这些特征
    有就是病毒
    优点：准确，误报低
    缺点：只能检测已知病毒，
      对新病毒、变种病毒没用
      病毒库要一直更

  2. 启发式检测
    不看具体的特征码，
    而是看程序的行为和结构
    有没有可疑的地方
    比如会不会修改系统文件、
    会不会自我复制、
    结构是不是像病毒...
    优点：能检测未知病毒和变种
    缺点：误报率相对高一些

  3. 行为检测
    看程序运行时的行为
    有没有干坏事
    比如会不会修改注册表启动项、
    会不会注入其他进程、
    会不会加密大量文件、
    会不会连接可疑的C&C服务器...
    优点：能检测未知病毒，
      尤其是高级恶意代码
    缺点：要运行起来才能检测，
      可能已经造成损害了

  4. 云查杀
    把可疑文件的哈希传到云端，
    云端有更大的病毒库和分析系统
    快速判断是不是病毒
    现在的杀毒软件基本都有云查杀

  5. 沙箱 / 沙盒（Sandbox）
    把可疑文件放到一个隔离的环境里运行
    观察它的行为
    看它干不干坏事
    用来分析未知恶意代码

  6. 机器学习 / AI检测
    用机器学习算法，
    大量样本训练模型，
    来识别恶意代码
    现在越来越流行，
    对未知样本的检测效果不错

  7. EDR（端点检测与响应）
    不光是检测，
    还能监控终端的各种行为，
    发现异常及时响应
    比传统杀毒软件高级，
    适合企业用
```

检测技术一直在发展，
从特征码到启发式到行为检测到AI，
越来越高级。
但恶意代码也在进化，
免杀技术也越来越强，
攻防一直在博弈。

### 8.7.2 恶意代码防御体系

对付恶意代码，
不能只靠杀毒软件，
要**多层防御，纵深防御**。

```
  网络层：
    • 防火墙：限制不必要的端口和连接
    • 入侵检测/防御系统（IDS/IPS）
    • Web应用防火墙（WAF）
    • 邮件网关：过滤钓鱼邮件和恶意附件
    • 流量监控：发现异常流量

  终端层：
    • 杀毒软件 / EDR：基础防护
    • 补丁管理：及时打系统和软件补丁
    • 应用白名单：只允许运行可信程序
    • 最小权限：用户不用管理员权限
    • 禁用不必要的服务和功能

  数据层：
    • 数据备份：定期备份，离线备份
    • 数据加密：重要数据加密存储
    • 数据防泄漏（DLP）

  管理层：
    • 安全策略和制度
    • 安全意识培训：提高员工警惕性
    • 应急响应预案：出事了怎么办
    • 定期安全评估和渗透测试
    • 供应链安全：软件来源可信
```

多层防护，
每一层都挡住一部分威胁，
就算一层被突破了，
还有下一层。
没有哪一层是百分之百安全的，
但多层加起来，
安全性就大大提高了。

### 8.7.3 个人用户怎么防？

最后给个人用户几条简单实用的建议：

```
  1. 系统和软件及时更新打补丁
    别嫌麻烦，很多攻击都是打已知漏洞

  2. 装个靠谱的杀毒软件
    Windows自带的Defender就够用了
    不用装乱七八糟的"安全卫士"
    反而可能更不安全

  3. 软件从正规渠道下载
    别从乱七八糟的下载站下载
    别用什么"破解版"、"绿色版"
    十有八九带捆绑

  4. 不要随便点陌生邮件的附件和链接
    尤其是exe、docm、js、vbs这些
    不确定的别点开
    钓鱼邮件是最常见的传播方式

  5. 不要随便插陌生U盘
    U盘也可能带毒
    尤其是来历不明的U盘

  6. 重要数据定期备份
    备份到移动硬盘或者云盘
    不怕一万就怕万一
    真中了勒索软件还有备份

  7. 密码要复杂，不同网站用不同密码
    用密码管理器管理密码
    开启双因素认证

  8. 提高安全意识
    天上不会掉馅饼
    中奖、美女、理财...
    多半是骗局
```

做到这几点，
中招的概率就能降低很多。

## 8.8 本章总结

这一章我们讲了恶意代码与勒索软件，
内容不少。
总结一下：

**恶意代码概述：**
- 总称，包括病毒、木马、蠕虫、勒索软件等等
- 传播方式：漏洞利用、钓鱼邮件、恶意网站、
  移动存储、软件捆绑、供应链攻击...
- 一直在进化，从个人恶作剧到有组织犯罪

**常见恶意代码种类：**
- **病毒**：自我复制，需要宿主
- **蠕虫**：自我复制，不需要宿主，主动网络传播
- **木马**：伪装成正常软件，暗地里干坏事
- **后门**：绕过验证的秘密通道
- **Rootkit**：高度隐蔽，隐藏自己的存在

**勒索软件：**
- 加密文件索要赎金
- 从CryptoLocker到WannaCry到RaaS
- 愈演愈烈，产业链成熟
- 防御：打补丁、备份（最重要）、杀毒软件、
  邮件安全、网络分段、应用白名单、
  安全意识、应急预案

**挖矿木马：**
- 偷偷用你的算力挖币赚钱
- 传播：网页挖矿、软件捆绑、漏洞利用...
- 危害：性能下降、耗电、说明系统已被入侵

**僵尸网络与DDoS：**
- 僵尸网络：大量被控制的僵尸机
- DDoS：分布式拒绝服务攻击，把目标打瘫
- 类型：流量型、连接型、应用层

**检测与防御：**
- 检测技术：特征码、启发式、行为检测、
  云查杀、沙箱、AI、EDR
- 防御：多层防御、纵深防御
- 个人防护：打补丁、杀毒软件、正规渠道、
  不点陌生邮件、备份、安全意识

最后送你一句话：

> **恶意代码的战争永不停歇，
> 最好的防御，
> 是时刻保持警惕。**
