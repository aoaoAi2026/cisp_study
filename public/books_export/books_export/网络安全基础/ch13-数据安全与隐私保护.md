## 13.1 先讲一个故事

### 13.1.1 "一条数据五毛钱"

我给你讲个真实的故事。

某电商公司，
用户量很大，
有几千万用户的个人信息。
公司对数据安全不太重视，
觉得"我们又不是金融公司，

谁会来偷我们的。"

开发、测试、运营...
好多人都能直接查生产数据库，

一个开发离职了，
走之前把整个用户数据库

后来事情败露了，
公司被监管部门罚款，
还被用户集体起诉，
赔了一大笔钱，
公司声誉一落千丈，
最后差点倒闭。
那个卖数据的前员工，
也因为侵犯公民个人信息罪，
被判了刑。

## 13.2 数据安全概述

### 13.2.1 什么是数据安全？

什么是数据安全？
简单说就是：
**通过各种技术和管理手段，
保护数据的保密性、完整性、可用性，
防止数据被泄露、篡改、丢失、
滥用和非授权访问。**

还是我们第一章讲的CIA三要素：
- **保密性（Confidentiality）**：数据不能被不该看的人看到
- **完整性（Integrity）**：数据不能被随便改，改了要能发现
- **可用性（Availability）**：需要用的时候得能用

数据安全就是围绕这三个核心目标，
在数据的整个生命周期里，
全方位地保护数据。

### 13.2.2 为什么数据安全越来越重要？

为什么数据安全现在这么火？
几个原因：

```
  1. 数据越来越值钱
    数字经济时代，
    数据是重要的生产要素，
    是企业的核心资产
    数据越值钱，
    想偷的人就越多

  2. 数据泄露的危害越来越大
    数据泄露了，
    不光是经济损失，
    还可能危害国家安全、
    公共利益、个人权益
    影响越来越大

  3. 法律法规越来越严
    《网络安全法》、
    《数据安全法》、
    《个人信息保护法》...
    一部接一部
    数据安全做不好是违法的
    罚款最高可达上年营业额的5%
    还可能追究刑事责任
    （前一章那个卖数据的就是例子）

  4. 数据攻击越来越多
    现在黑客攻击的目标
    越来越偏向数据
    偷数据、勒索数据、
    倒卖数据...
    数据成了主要的攻击目标
```

所以说，
数据安全是未来安全的重要方向，
也是企业安全的重中之重。

### 13.2.3 数据安全 vs 网络安全

数据安全和网络安全是什么关系？
```
  网络安全：
    更侧重网络、系统、应用的安全
    是从技术层面、从攻防的角度
    保护整个信息系统

  数据安全：
    更侧重数据本身的安全
    围绕数据的全生命周期
    从采集、存储、传输、使用
      到共享、销毁
    全方位保护数据
```

数据安全是网络安全的一部分，
也是网络安全的目标之一。
网络安全是手段，
数据安全是目的之一。
保护网络和系统的安全，
最终也是为了保护数据和业务的安全。

## 13.3 数据分类分级

做数据安全，
第一步是什么？
**数据分类分级。**

### 13.3.1 为什么要分类分级？

为什么要给数据分类分级？
因为数据和数据不一样，
重要程度不一样，
敏感度不一样，
当然保护的程度也应该不一样。

比如：
- 公开的产品介绍，谁都能看，
  当然不用什么保护
- 公司内部的规章制度，
  只有内部员工能看，保护级别一般
- 用户的个人信息，
  泄露了会影响用户权益，
  保护级别要高
- 核心商业机密、
  核心算法、财务数据，
  泄露了公司就完蛋了，
  保护级别最高

如果所有数据都按最高级别保护，
成本太高，也影响业务效率；
都按最低级别保护，
重要数据又不安全。
所以要分类分级，
**不同级别的数据，
用不同级别的保护措施。**
恰到好处，又不浪费。

这就是数据分类分级的意义。

### 13.3.2 数据分类

什么是数据分类？
**按照数据的属性、
业务领域、用途等，
把数据分成不同的类别。**

比如按业务分：
- 用户数据
- 财务数据
- 业务数据
- 研发数据
- 人力资源数据
- 运营数据
- 行政数据
- 等等...

按数据形态分：
- 结构化数据（数据库里的表）
- 非结构化数据（文档、图片、视频...）
- 半结构化数据（JSON、XML...）

分类的方式没有标准答案，
根据企业自己的业务情况来定，
怎么方便管理怎么来。

### 13.3.3 数据分级

什么是数据分级？
**按照数据的敏感程度、
重要程度、
泄露后的危害程度，
把数据分成不同的等级。**

分几级没有统一标准，
一般分3-5级比较常见。
比如：

```
  第1级：公开数据
    对所有人公开，谁都能看
    比如官网公开的产品介绍、
      公司新闻、招聘信息...
    泄露了没什么危害

  第2级：内部数据
    公司内部员工可以看
    不对外部公开
    比如内部规章制度、
      普通的内部文档...
    泄露了有一定影响，
      但不大

  第3级：敏感数据
    只有特定部门/岗位的人能看
    泄露了会造成一定损害
    比如用户非敏感个人信息、
      普通业务数据、
      一般的商业秘密...

  第4级：重要数据 / 高敏感数据
    只有极少数人能访问
    泄露了会造成严重损害
    比如核心商业秘密、
      核心技术数据、
      用户敏感个人信息、
      重要的财务数据...

  第5级：核心数据 / 绝密数据
    最高级别，严格管控
    泄露了会造成特别严重损害
    比如国家秘密、
      公司的核心命门数据...
```

《数据安全法》里也提到了数据分级，
分为一般数据、重要数据、核心数据。
重要数据和核心数据是重点保护对象。

分级的目的，
就是为了**差异化保护**——
级别越高的数据，
保护措施越严；
级别低的数据，
保护措施可以适当放松。
既保证了安全，
又不至于过度防护影响业务。

## 13.4 数据全生命周期安全

数据安全不能只看某一个环节，
要覆盖数据的整个生命周期。
什么是数据的生命周期？
就是数据从产生到销毁的整个过程。

一般分为这几个阶段：
```
  1. 数据采集
  2. 数据传输
  3. 数据存储
  4. 数据处理 / 使用
  5. 数据共享 / 交换
  6. 数据销毁
```

每个阶段都有对应的安全风险，
也都要有对应的安全措施。
我们一个一个讲。

### 13.4.1 数据采集

第一个阶段，**数据采集**。
就是收集数据的阶段。

这个阶段的风险：
- 过度采集：采集了很多不必要的数据
- 非法采集：没经过用户同意就采集
- 采集渠道不安全：采集的时候被中间人截获

安全措施：
```
  • 最小必要原则
    只采集业务必须的数据，
    能不采集的就不采集
    数据越少，风险越小

  • 合法合规
    采集个人信息要经过用户同意，
    告知用户采集了什么、用来干嘛
    符合《个人信息保护法》的要求

  • 安全采集
    采集通道加密（HTTPS）
    防止传输过程中被截获
```

### 13.4.2 数据传输

第二个阶段，**数据传输**。
数据在网络上传输的过程。

风险：
- 被窃听：传输过程中被人截获
- 被篡改：中间人改了数据
- 被冒充：冒充发送方发假数据

安全措施：
```
  • 加密传输
    用TLS/SSL加密
    HTTP → HTTPS
    FTP → SFTP/FTP over SSL
    普通传输 → VPN
    总之，重要数据传输必须加密

  • 完整性校验
    传输完了校验一下哈希，
    看看数据有没有被改

  • 身份认证
    确认对方身份，防止冒充
```

简单说就是：
**重要数据，
传输过程中一定要加密。**

### 13.4.3 数据存储

第三个阶段，**数据存储**。
数据存在数据库、文件服务器、
云存储里的阶段。
这个阶段时间最长，
也是最容易出问题的阶段。

风险：
- 数据库被攻破，数据被拖库
- 备份泄露
- 存储介质丢失（硬盘、U盘丢了）
- 内部人员越权访问、倒卖数据

安全措施：
```
  • 加密存储
    敏感数据加密之后再存
    就算数据库被拖了，
      拿到的也是密文，没用
    加密方式：
      • 数据库透明加密（TDE）
      • 字段级加密（重要字段单独加密）
      • 文件加密
    密钥要好好管理，
      不能和数据存在一起

  • 访问控制
    严格控制谁能访问数据
    最小权限原则
    不是谁都能查数据库
    数据库账号权限最小化

  • 备份安全
    备份数据也要加密
    备份也要严格管控
    很多数据泄露都是备份出问题

  • 存储介质安全
    硬盘、磁带这些，
      物理上要安全
    报废的硬盘要消磁销毁，
      不能随便扔

  • 数据库安全
    数据库审计：谁查了什么都记下来
    数据库防火墙：防SQL注入、异常查询
    数据库脱敏：返回的时候脱敏
    漏洞防护：及时打补丁
```

加密存储是数据安全的核心手段之一，
非常重要。

### 13.4.4 数据处理与使用

第四个阶段，**数据处理和使用**。
数据被查询、分析、计算、展示的过程。
这个阶段人和数据打交道最多，
风险也很多。

风险：
- 越权访问：看了不该看的数据
- 数据滥用：拿数据干不该干的事
- 内部人员泄露：内鬼倒卖数据
- 开发测试用生产数据：容易泄露
- 数据导出泛滥：到处都是数据副本

安全措施：
```
  • 访问控制
    严格的权限控制
    谁能看什么数据，清清楚楚
    最小权限原则

  • 数据脱敏
    敏感数据在使用的时候脱敏
    比如身份证号显示中间打星号
    手机号中间四位打星号
    开发测试环境用脱敏数据，
      不用真实的生产数据

  • 数据水印
    给数据加上水印（可见或不可见）
    如果泄露了，
      可以溯源，知道是谁泄露的
    震慑内部人员

  • 操作审计
    所有数据操作都记录日志
    谁、什么时候、查了什么、导出了什么
    都记下来，出问题能查

  • 防泄漏（DLP）
    数据防泄漏系统，
    防止数据被随便下载、
      拷贝、传出去
    这个后面专门讲

  • 审批流程
    重要数据的访问、导出，
      需要审批
    不能随便看、随便导
```

数据使用阶段是最复杂的，
因为涉及到人。
人是最大的变量，
也是最难管的。

### 13.4.5 数据共享与交换

第五个阶段，**数据共享与交换**。
数据在不同部门、不同公司、
不同机构之间共享的阶段。

风险：
- 共享出去的数据失控，被滥用
- 第三方的安全水平不够，数据泄露
- 共享范围失控，越传越广

安全措施：
```
  • 最小化原则
    共享的数据越少越好，
    够用就行
    能脱敏就脱敏

  • 评估与审批
    共享之前做安全评估，
    走审批流程
    不是想共享就共享

  • 签协议
    和接收方签数据安全协议，
    明确责任和义务
    出了问题能追责

  • 脱敏 / 去标识化
    能脱敏就脱敏，
    能匿名化就匿名化
    降低泄露的危害

  • 管控共享渠道
    数据怎么传、传给谁、
      传了多少，都要有记录
    可审计、可追溯
```

数据共享是现在的大趋势，
数据要流动才能产生价值。
但流动的同时，
安全也要跟上。

### 13.4.6 数据销毁

最后一个阶段，**数据销毁**。
数据不用了，
要删除、要销毁。

风险：
- 没删干净，还能恢复
- 存储介质直接扔了，被人捡去恢复
- 备份没一起销毁

安全措施：
```
  • 彻底删除
    删除不是简单地删文件、格式化
    要多次覆写，
      或者用专业的销毁工具
    防止数据被恢复

  • 介质销毁
    不用的硬盘、磁带，
      物理销毁（消磁、粉碎、焚烧）
    绝对不能随便扔

  • 备份一起销毁
    要删就全删，
      原始数据删了备份也要删
    不然等于没删

  • 销毁记录
    销毁了什么、什么时候销毁的、
      谁负责的，都记下来
    可审计
```

很多人容易忽略销毁阶段，
觉得不用了就没事了。
但很多数据泄露事件，
就是因为旧硬盘、旧服务器
处理不当导致的。

## 13.5 数据加密

数据加密是数据安全的核心技术，
前面密码学那一章讲过加密，
这里再专门讲讲数据加密在数据安全中的应用。

### 13.5.1 加密的三个状态

数据有三种状态：
```
  1. 静态数据（Data at Rest）
    存着的数据，比如数据库里的、文件里的
    对应存储阶段

  2. 传输中的数据（Data in Transit / Data in Motion）
    网络上传输的数据
    对应传输阶段

  3. 使用中的数据（Data in Use）
    内存里正在处理的数据
    对应处理使用阶段
```

理想情况下，
三种状态的数据都要加密保护。
但使用中的数据加密最难，
因为数据要用就得解密，
内存里肯定是明文的。
现在也有一些技术，
比如同态加密、可信执行环境（TEE）...
但还不成熟，成本也高。

一般企业能做到
静态数据加密 + 传输中数据加密，
就已经很不错了。

### 13.5.2 密钥管理

加密很重要，
但比加密更重要的是**密钥管理**。
加密了之后，
密钥管不好，
等于没加密。

密钥管理包括什么？
- 密钥的生成：用安全的方式生成
- 密钥的存储：密钥存在哪里？
  不能和加密数据存在一起
- 密钥的分发：怎么安全地把密钥给需要的人
- 密钥的使用：谁能用密钥？怎么用？
- 密钥的轮换：定期更换密钥
- 密钥的销毁：不用的密钥销毁掉
- 密钥的备份：密钥丢了数据也找不回来了

密钥管理是个技术活，
也是个管理活。
大企业一般会用专门的密钥管理系统（KMS）
或者硬件加密机（HSM）
来管密钥。
小公司至少也要把密钥
和数据分开存，
不能写死在代码里。

## 13.6 数据防泄漏（DLP）

接下来讲一个专门的技术——
**DLP（Data Loss Prevention）**，
数据防泄漏，也叫数据泄露防护。

### 13.6.1 什么是DLP？

什么是DLP？
简单说就是：
**通过技术手段，
防止敏感数据被未经授权地
带出、泄露出去的系统。**

DLP能干嘛？
- 检测敏感数据在哪里
- 监控敏感数据的流动
- 防止敏感数据被非法外传
- 违规操作及时阻断和告警

### 13.6.2 DLP的分类

DLP一般分几类：

```
  1. 终端DLP
    装在员工电脑上的
    监控终端上的数据操作
    比如：
      • 能不能拷文件到U盘？
      • 能不能通过微信/QQ发文件？
      • 能不能打印敏感文件？
      • 能不能上传到网盘？
    发现违规就告警或者阻断

  2. 网络DLP
    部署在网络出口的
    监控网络流量里的敏感数据
    比如：
      • 有没有人通过邮件发敏感数据？
      • 有没有人往网盘上传敏感文件？
      • 有没有人通过HTTP外发敏感数据？
    发现异常就告警或者阻断

  3. 存储DLP
    部署在文件服务器、数据库上的
    扫描存储里有没有敏感数据
    存的位置对不对、权限对不对
    发现不合规的就告警
```

不同类型的DLP管不同的方面，
有的企业三种都上，
形成立体防护。

### 13.6.3 DLP的局限性

DLP也不是万能的：
```
  • 误报率比较高
    经常把正常操作当成违规
    用起来比较折腾
    需要花很多时间调规则

  • 可以被绕过
    比如拍屏、拍照...
      DLP管不了
    再比如改改文件名、
      压缩加密一下，
      DLP可能就识别不出来了

  • 影响用户体验
    管得太严，员工干活不方便
    管得太松，又没效果
    平衡不好掌握

  • 实施成本高
    买设备贵，
      实施和运维也贵
    中小企业可能用不起
```

DLP是个好东西，
但要用好不容易。
一般对数据安全要求比较高的行业，
比如金融、医疗、政府...
用得比较多。

## 13.7 个人信息保护与隐私保护

最后讲一个大家都很关心的话题——
**个人信息保护与隐私保护**。

### 13.7.1 什么是个人信息？

什么是个人信息？
按照《个人信息保护法》的定义：
**个人信息是以电子或者其他方式记录的
与已识别或者可识别的自然人有关的各种信息，
不包括匿名化处理后的信息。**

用人话说就是：
**能识别出你是谁的信息，
都是个人信息。**

比如：
- 姓名、身份证号、手机号、住址
- 邮箱、账号、密码
- 人脸、指纹、声纹（生物识别信息）
- 位置信息、行踪轨迹
- 医疗健康信息
- 金融账户、交易记录
- 通信记录和内容
- 等等...

个人信息里，
还有一类叫**敏感个人信息**，
保护级别更高：
- 生物识别信息
- 宗教信仰
- 特定身份
- 医疗健康
- 金融账户
- 行踪轨迹
- 不满十四周岁未成年人的个人信息

这些信息一旦泄露，
危害更大，
所以保护更严格。

### 13.7.2 个人信息保护的原则

《个人信息保护法》规定了
处理个人信息的几个基本原则：

```
  1. 合法、正当、必要、诚信原则
    不能非法收集、不能欺诈、
      不能误导

  2. 目的明确、最小必要原则
    有明确的目的，
    只收集够用的，
    不能什么都收集

  3. 公开透明原则
    告诉用户你收集了什么、
      用来干嘛、怎么用
    不能偷偷摸摸收集

  4. 质量原则
    保证个人信息准确、及时更新
    不能是错的、过时的

  5. 安全保障原则
    采取安全措施保护个人信息
    防止泄露、篡改、丢失

  6. 责任原则
    谁处理谁负责
    出了问题要承担责任
```

这些原则不只是法律要求，
也是做个人信息保护的基本思路。

### 13.7.3 个人信息主体的权利

个人对自己的信息有哪些权利？
按照《个保法》：
```
  • 知情权：知道你收集了什么、怎么用
  • 决定权：决定同不同意你处理
  • 查阅复制权：可以看你有我什么信息
  • 更正补充权：信息错了可以改
  • 删除权：可以要求你删除我的信息
    （也叫"被遗忘权"）
  • 可携带权：可以把我的信息
    转移到其他平台
  • 解释说明权：可以让你解释
    你怎么处理我的信息
```

这些都是用户的法定权利，
做产品、做系统的时候要考虑到。

### 13.7.4 隐私保护技术

有一些专门的隐私保护技术，
用来在数据使用的同时
保护个人隐私：

```
  • 数据脱敏（Data Masking）
    把敏感的部分替换掉
    比如手机号138****1234
    身份证号110***********1234
    最常用、最简单的技术

  • 去标识化 / 匿名化
    去标识化：去掉能直接识别个人的信息，
      但结合其他信息还可能识别出来
    匿名化：彻底没法识别出个人，
      匿名化后的数据不属于个人信息了
    注意：去标识化 ≠ 匿名化

  • 差分隐私（Differential Privacy）
    在数据里加一点"噪音"，
      让人分不清具体是哪个人的数据
    但整体统计结果还是准的
    适合大数据分析场景
    比较高级的技术

  • 联邦学习（Federated Learning）
    数据不用拿出来，
      就在各自本地训练
    大家一起训练一个模型，
      但数据不交换
    做到"数据可用不可见"
    现在很火的隐私计算技术

  • 同态加密
    加密之后的数据也能做计算
    算完了解密结果还是对的
    非常牛逼，但现在性能还不行
    还在发展中

  • 可信执行环境（TEE）
    在CPU里划出一块安全区域，
      数据在里面解密计算，
      外面看不到
    硬件级的安全
    也是隐私计算的一种
```

隐私保护技术是现在的热点，
也是未来的趋势。
数据要流通、要产生价值，
但又不能泄露隐私，
就得靠这些隐私保护技术。

## 13.8 本章总结

这一章我们讲了数据安全与隐私保护，
内容很多，也很重要。
总结一下：

**数据安全概述：**
- 保护数据的保密性、完整性、可用性
- 数据越来越重要，法律法规越来越严
- 数据安全是未来安全的重要方向

**数据分类分级：**
- 分类：按业务、属性等分类别
- 分级：按敏感程度、重要程度分级别
- 目的：差异化保护，级别越高保护越严
- 《数据安全法》：一般数据、重要数据、核心数据

**数据全生命周期安全：**
- 数据采集：最小必要、合法合规、安全采集
- 数据传输：加密传输、完整性校验
- 数据存储：加密存储、访问控制、备份安全、数据库安全
- 数据处理与使用：权限控制、脱敏、水印、审计、DLP
- 数据共享与交换：最小化、审批、协议、脱敏、可追溯
- 数据销毁：彻底删除、介质销毁、可审计

**数据加密：**
- 三种状态：静态数据、传输中数据、使用中数据
- 加密 + 密钥管理，缺一不可
- 密钥管理是关键，KMS/HSM

**数据防泄漏（DLP）：**
- 终端DLP、网络DLP、存储DLP
- 防止敏感数据被非法外传
- 有局限性，误报、可绕过、影响体验

**个人信息保护与隐私保护：**
- 个人信息、敏感个人信息的定义
- 基本原则：合法正当必要、公开透明、最小必要...
- 个人的权利：知情、决定、查阅、更正、删除...
- 隐私保护技术：脱敏、匿名化、差分隐私、
  联邦学习、同态加密、TEE...

最后送你一句话：

> **数据是资产，
> 也是责任。
> 保护好数据，
> 就是保护好企业的未来。**
