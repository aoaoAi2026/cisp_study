## 2.1 先讲一个故事

### 2.1.1 "网络是怎么把数据送到你手上的？"

我问你一个问题：
你现在用手机或者电脑看这本书，
这些文字、图片，
是怎么从服务器传到你屏幕上的？

你可能会说："通过网络啊。"
那网络具体是怎么工作的呢？
数据是怎么从一个地方，
传到另一个地方的？
中间经过了多少设备？
为什么不会传错？
为什么不会丢？

这些问题，
就是网络基础要回答的。

我再给你讲个生活中的例子，
你一下子就懂了。

想象一下，
你要给远方的朋友寄一封信。
这封信是怎么从你手里，
到朋友手里的？

```
  1. 你把信写好，装到信封里
     信封上写上收件人的地址和姓名
     这相当于"应用层"

  2. 你把信送到小区的快递柜/邮局
     快递员来收走
     这相当于"传输层"和"网络层"

  3. 快递到了城市分拣中心
     分拣员看地址，分到不同的路线
     这相当于"路由器"

  4. 信通过公路、铁路、飞机...
     运到对方的城市
     这相当于"物理层"和"数据链路层"

  5. 对方城市的分拣中心再分拣
     派到具体的区、具体的街道
     快递员送货上门
     朋友收到信，拆开看内容

  整个过程，
  不同的环节负责不同的事情，
  一层一层传递，
  最终信送到了目的地。
```

计算机网络的工作原理，
跟寄快递几乎一模一样。
数据要从一台电脑发到另一台电脑，
也要经过很多层、很多设备，
一层一层往下传，
到了对方再一层一层往上传，
最后对方的应用程序拿到数据。

这就是网络的分层思想。

## 2.2 OSI七层模型

说到网络分层，
就不得不提**OSI七层模型**。
这是网络领域最经典的模型，
也是每个学网络的人必须知道的。

### 2.2.1 什么是OSI七层模型？

OSI（Open Systems Interconnection，
开放系统互连）模型，
是国际标准化组织（ISO）制定的
一个网络通信的标准框架。
它把网络通信分成了七层，
每一层负责不同的功能，
各司其职，互相配合。

哪七层？
从下到上分别是：

```
  第7层 应用层（Application Layer）
  第6层 表示层（Presentation Layer）
  第5层 会话层（Session Layer）
  第4层 传输层（Transport Layer）
  第3层 网络层（Network Layer）
  第2层 数据链路层（Data Link Layer）
  第1层 物理层（Physical Layer）
```

口诀记一下：
**"应表会传网数物"**
（从7层到1层）
或者反过来：
**"物数网传会表应"**
（从1层到7层）

多念几遍就记住了。

### 2.2.2 为什么要分层？

为什么要搞这么多层？
直接发不行吗？
还真不行。
分层有很多好处：

```
  1. 各司其职，每一层只干自己的事
     不用管别的层怎么实现
     只要接口对就行

  2. 独立发展，互不影响
     某一层的技术升级了，
     不影响其他层
     比如物理层从网线换成光纤，
     上层完全不用改

  3. 容易学习和理解
     把复杂的问题拆成小问题
     一层一层地学
     不会一团乱麻

  4. 便于排错
     出问题了，
     可以一层一层排查
     看是哪层出了问题
```

就像寄快递一样，
你只管写信，
不用管快递怎么运、走哪条路。
快递员只管送货，
不用管信里写的是什么。
大家各干各的，
效率高，也不容易乱。

### 2.2.3 数据封装与解封装

那七层之间是怎么配合的呢？
这里有两个重要的概念：
**封装**和**解封装**。

简单说就是：
```
发送方：从上到下，一层一层加包头（封装）
接收方：从下到上，一层一层剥包头（解封装）
```

比如你发一个网页请求：
```
  应用层：生成HTTP请求数据
    ↓
  传输层：加上TCP头（端口号等）
    ↓
  网络层：加上IP头（源IP、目的IP等）
    ↓
数据链路层：加上以太网头（源MAC、目的MAC等）
    ↓
  物理层：变成电信号/光信号发出去
```

到了对方那里，
再倒过来，
一层一层把头剥掉，
最后应用程序拿到原始数据。

就像寄快递：
你写好信（应用层），
装信封（传输层），
装快递袋（网络层），
装车（数据链路层/物理层），
运过去。
对方收到了，
卸车、拆袋、拆信封、看信。
一个道理。

每一层的数据有不同的名字：
- 应用层：数据 / 消息（Data / Message）
- 传输层：段（Segment）
- 网络层：包 / 分组（Packet）
- 数据链路层：帧（Frame）
- 物理层：比特流（Bits）

这些术语不用死记硬背，
有个印象就行。

## 2.3 物理层

好，
我们从最底层开始，
一层一层往上讲。

先讲**第一层：物理层**。

### 2.3.1 物理层是干嘛的？

物理层，
就是最底层的那一层，
负责把数据变成信号，
在物理介质上传输。

用人话说就是：
**怎么把0和1，变成电、光、无线电波，
传出去，再收回来。**

你电脑上的网线口、
WiFi天线、
光纤接口，
这些都是物理层的东西。

### 2.3.2 传输介质

物理层的传输介质有哪些？
分两类：
**有线的**和**无线的**。

```
  有线介质：
    • 双绞线（网线） —— 最常见，比如家里的网线
    • 同轴电缆 —— 以前有线电视用的
    • 光纤 —— 速度快、距离远、抗干扰
      现在主干网基本都是光纤

  无线介质：
    • 无线电波 —— WiFi、蓝牙、移动通信
    • 微波 —— 长距离通信
    • 红外线 —— 遥控器那种，短距离
```

不同的介质，
速度不一样、距离不一样、成本不一样。
根据场景选合适的。

### 2.3.3 物理层和安全有啥关系？

有人可能会问：
物理层就是传信号的，
跟安全有啥关系？

关系大了。
```
  • 物理安全：网线插哪了？
    会不会有人偷偷接一根线进来偷听？
  • 无线信号：WiFi信号会穿墙，
    外面的人也能收到，就能破解、监听
  • 电磁泄漏：屏幕、键盘、线缆
    都会有电磁辐射，
    厉害的能通过电磁信号还原内容
  • 硬件攻击：物理接触了，
    什么安全都白搭
    （所以机房不是谁都能进的）
```

物理安全是最底层的安全，
也是最根本的。
物理上都不安全，
上面再多层防护都没用。

## 2.4 数据链路层

往上一层，
**第二层：数据链路层**。

### 2.4.1 数据链路层是干嘛的？

物理层只管传信号，
不管传的是什么、
传给谁、对不对。
数据链路层就是干这个的：
**在两个相邻的节点之间，
可靠地传输数据帧。**

用人话说就是：
**同一个局域网里，
数据怎么从一台机器传到另一台机器。**

### 2.4.2 以太网与MAC地址

说到数据链路层，
就不得不提**以太网（Ethernet）**。
现在局域网基本都是以太网。

以太网里，
每台设备都有一个唯一的地址，
叫 **MAC地址**
（Media Access Control，
介质访问控制地址）。

MAC地址长什么样？
一般是6组两位的十六进制数，
用冒号或者横杠隔开，比如：
`00:1A:2B:3C:4D:5E`

- 前3组是厂商标识（OUI）
- 后3组是设备编号

MAC地址是烧录在网卡里的，
理论上全球唯一。
就像你的身份证号一样，
是网卡的"身份证"。

### 2.4.3 交换机

数据链路层的核心设备是**交换机（Switch）**。
交换机是干什么的？
简单说就是：
**根据MAC地址，
把数据帧转发到正确的端口。**

比如一个交换机连了5台电脑，
A要发给B，
交换机看一下目的MAC地址，
知道B在2号口，
就直接把帧从2号口发出去，
其他口收不到。

这样既提高了效率，
也比集线器（Hub）那种广播式的安全一点。

（集线器是啥？
就是个共享总线的设备，
收到数据就所有口都发一遍，
谁都能收到。
现在基本淘汰了。）

### 2.4.4 ARP协议

讲数据链路层，
必须讲**ARP协议**。
这个协议太重要了，
而且跟安全关系很大。

ARP是什么？
Address Resolution Protocol，
地址解析协议。
干嘛用的？
**已知IP地址，求MAC地址。**

为什么需要ARP？
因为数据链路层通信要用MAC地址，
但我们只知道对方的IP地址，
不知道MAC地址。
这时候就需要ARP来问一下：
"谁是192.168.1.100？
你的MAC地址是多少？"

怎么问的？
**广播**。
就是对着整个局域网喊：
"喂！192.168.1.100在吗？
把你的MAC地址告诉我！"
所有机器都能听到，
但只有IP是192.168.1.100的那台会回答：
"我是192.168.1.100，
我的MAC地址是xx:xx:xx:xx:xx:xx。"

问完了之后，
双方都会把对方的IP和MAC的对应关系，
存在自己的**ARP缓存表**里，
下次直接用，不用再问了。

### 2.4.5 ARP欺骗 —— 经典的二层攻击

讲ARP，
就是为了讲**ARP欺骗（ARP Spoofing / ARP Poisoning）**。
这是一个非常经典的局域网攻击，
也是很多其他攻击的基础。

什么是ARP欺骗？
简单说就是：
**攻击者伪造ARP应答，
告诉受害者"我是网关"或者"我是某某"，
让受害者把流量发给攻击者，
从而实现中间人攻击。**

用人话说就是：
**我骗你，说我是网关，
你就把上网的流量都发给我，
我就能看到你所有的上网数据，
还能篡改。**

为什么ARP这么好骗？
因为ARP协议有个特点：
**它不验证。**
不管你有没有发过ARP请求，
只要收到ARP应答，
它就信，
而且会更新自己的缓存表。
这就是ARP协议的设计缺陷。

ARP欺骗能干嘛？
```
  • 中间人攻击：监听、篡改流量
  • 断网攻击：把流量导到一个不存在的地方，你就上不了网了
  • 会话劫持：劫持你的登录会话
  • 偷密码、偷数据
  • ......
```

非常危险。
而且局域网里，
ARP防起来还挺麻烦的。
怎么防？
```
  • 静态ARP绑定：把IP和MAC手动绑定，不相信ARP应答
  • ARP防火墙：检测异常的ARP包
  • 交换机端口安全：限制每个端口的MAC地址数量
  • 动态ARP检测（DAI）：交换机校验ARP的合法性
```

ARP欺骗是一个非常经典的二层攻击，
也是网络安全入门必学的。
后面讲攻击的时候还会详细讲。

### 2.4.6 VLAN —— 虚拟局域网

再讲一个数据链路层的重要概念：
**VLAN（Virtual Local Area Network，
虚拟局域网）。**

什么是VLAN？
简单说就是：
**把一个物理的交换机，
逻辑上分成好几个虚拟的交换机，
不同VLAN之间默认不通。**

为什么要做VLAN？
```
  1. 隔离广播域
     广播太多了会影响性能
     VLAN把广播域划小

  2. 安全隔离
     不同部门、不同安全级别的设备
     放到不同VLAN
     就算某一个VLAN出事了，
     也影响不到其他的

  3. 灵活管理
     不用改物理连线，
     改配置就能调整网络结构
```

比如一个公司，
财务部一个VLAN，
技术部一个VLAN，
人事部一个VLAN。
不同部门之间默认不通，
要通过路由器/三层交换机，
还能做访问控制。
安全很多。

VLAN也是企业网络里最基本的安全手段之一。

## 2.5 网络层

再往上一层，
**第三层：网络层**。

这一层非常重要，
内容也比较多。
这一章的后半部分，
我们就来讲网络层。

### 2.5.1 网络层是干嘛的？

数据链路层只管同一个局域网里的通信，
那不同局域网之间怎么通信？
跨网络、跨城市、跨国家，
数据怎么路由过去？
这就是网络层要干的事。

网络层的核心作用就是：
**路径选择和路由转发。**
也就是：
**帮数据包找路，
从源IP送到目的IP。**

你可以这么理解：
- 数据链路层：小区里送快递，
  按门牌号（MAC）送
- 网络层：跨省跨市送快递，
  按城市地址（IP）选路线

网络层最核心的协议就是——
**IP协议（Internet Protocol）**。
可以说，
整个互联网就是建立在IP协议之上的。

### 2.5.2 IP地址

说到IP协议，
第一个要讲的就是**IP地址**。

什么是IP地址？
简单说就是：
**互联网上每台设备的网络地址。**
就像你家的门牌号一样，
有了IP地址，
数据包才能找到你。

我们现在常用的是 **IPv4**，
也就是IP协议第4版。
IPv4地址长什么样？
32位二进制数，
分成4段，每段8位，
用点分隔，
每段的范围是0-255。
比如：
`192.168.1.1`
`10.0.0.1`
`8.8.8.8`

理论上IPv4有多少个地址？
2的32次方，
大概42亿个。
听起来挺多的，
但现在早就不够用了。
所以才有了IPv6。

IPv6是128位的，
地址多到用不完，
号称"地球上每一粒沙子都能分配一个IP"。
IPv6也在慢慢普及，
但现在主流还是IPv4。
我们先讲IPv4。

### 2.5.3 公网IP和私网IP

IP地址分两种：
**公网IP**和**私网IP**。

什么意思？
```
  公网IP：
    • 直接在互联网上能用的IP
    • 全球唯一
    • 需要向运营商申请
    • 你家宽带上公网的那个IP就是公网IP

  私网IP（内网IP）：
    • 局域网内部用的IP
    • 不能直接在公网上路由
    • 不同局域网可以重复用
    • 比如你家路由器下面的192.168.1.x就是私网IP
```

私网IP有专门的保留地址段，
这些地址公网上是不会出现的：
```
  A类私网：10.0.0.0 ~ 10.255.255.255
  B类私网：172.16.0.0 ~ 172.31.255.255
  C类私网：192.168.0.0 ~ 192.168.255.255
```

你家里的、公司内网的，
基本都是这几个网段的。
私网IP要上网怎么办？
通过**NAT（网络地址转换）**，
把私网IP转换成公网IP，
就能上网了。
路由器干的就是这事。

为什么要有私网IP？
一来是解决IPv4地址不够的问题，
二来内网不直接暴露在公网，
也更安全。

### 2.5.4 子网划分

IP地址还涉及到一个重要概念——
**子网划分**。

什么是子网划分？
简单说就是：
**把一个大的网络，
分成几个小的子网。**

为什么要划分子网？
- 提高IP地址利用率
- 隔离广播域
- 安全隔离
- 便于管理

怎么划分？
用**子网掩码**。

什么是子网掩码？
32位，
用来区分IP地址里
哪部分是网络号，
哪部分是主机号。
- 子网掩码是1的位，对应网络号
- 子网掩码是0的位，对应主机号

比如最常见的 `255.255.255.0`，
就是说前24位是网络号，
后8位是主机号。
这个网段里能有2的8次方减2（254）台主机。
（减2是因为网络地址和广播地址不能用。）

还有一种更简洁的表示方法，
叫CIDR表示法，
就是在IP后面加个斜杠，
写上网络位的位数。
比如：
`192.168.1.0/24`
就表示前24位是网络号，
跟255.255.255.0是一个意思。

子网划分是网络的基本功，
学安全也得会。
不然你连网段都看不懂，
怎么搞渗透、怎么打内网？

新手刚开始学子网划分可能觉得有点绕，
多算几个就熟练了。
不用怕。

### 2.5.5 路由器

网络层的核心设备是**路由器（Router）**。
路由器是干嘛的？
**连接不同的网络，
根据IP地址选路，
转发数据包。**

简单说就是：
**数据包的中转站。**
一个数据包要去另一个网络，
要经过很多路由器，
路由器一个一个往前传，
最终传到目的地。

路由器怎么知道往哪传？
靠**路由表**。
路由表里记录了：
"去哪个网段，
应该从哪个口发出去，
下一跳是谁。"

路由器收到一个数据包，
看一下目的IP，
查路由表，
找到匹配的条目，
就从对应的端口发出去。

这个过程就叫**路由**。
互联网上那么多路由器，
就是靠路由协议（比如OSPF、BGP）
交换路由信息，
一起把数据包送到目的地。

路由器在网络安全里也非常重要，
很多访问控制、防火墙、NAT，
都是在路由器上做的。
后面还会讲。

### 2.5.6 ICMP协议与ping

网络层还有一个重要的协议——
**ICMP协议**（Internet Control Message Protocol，
互联网控制报文协议）。

ICMP是干嘛的？
**在IP网络里传递控制消息、错误消息。**
比如网络通不通、
主机可达不可达、
数据包超时了...
这些消息都是ICMP来传的。

我们最常用的 `ping` 命令，
用的就是ICMP协议。
ping发一个ICMP Echo Request包，
对方如果收到了，
就回一个ICMP Echo Reply，
那就说明网络通的。
如果没回，
可能就不通，或者被防火墙拦了。

ping是最常用的网络测试命令，
也是渗透测试里信息收集的常用工具——
扫网段看哪些IP活着。

不过现在很多服务器、防火墙
都禁了ICMP，
ping不通不代表机器不在线，
这个要注意。

### 2.5.7 IP协议的安全问题

最后聊聊IP协议的安全问题。
IP协议设计的时候，
根本没考虑安全，
所以有很多安全缺陷。

```
常见的IP层安全问题：

  1. IP欺骗（IP Spoofing）
     攻击者可以伪造源IP地址
     把数据包的源IP改成别人的
     用来隐藏身份、发动DDoS、
     绕过基于IP的访问控制...

  2. 分片攻击
     IP数据包太大的时候会分片
     攻击者可以构造特殊的分片包
     绕过防火墙的检测
     （比如经典的ping of death、teardrop）

  3. 路由安全问题
     路由协议如果没认证的话
     可以被攻击者注入虚假路由
     把流量拐到错误的地方

  还有很多...
```

这些攻击现在虽然没那么流行了，
但原理还是要了解。

## 2.6 本章总结

这一章我们讲了OSI七层模型的下三层：
物理层、数据链路层、网络层。
内容不少，总结一下：

**OSI七层模型：**
- 7层：应用层、表示层、会话层
- 4层：传输层
- 3层：网络层
- 2层：数据链路层
- 1层：物理层
- 口诀：应表会传网数物

**数据封装与解封装：**
- 发送方：从上到下，层层加包头（封装）
- 接收方：从下到上，层层剥包头（解封装）

**物理层（第1层）：**
- 传比特流，电/光/无线信号
- 传输介质：双绞线、光纤、无线电...
- 物理安全是最根本的安全

**数据链路层（第2层）：**
- 同一局域网内的通信
- MAC地址：网卡的硬件地址，全球唯一
- 交换机：根据MAC地址转发帧
- ARP协议：IP→MAC，靠广播
- ARP欺骗：经典的二层攻击，中间人攻击基础
- VLAN：虚拟局域网，隔离广播域，安全隔离

**网络层（第3层）：**
- 跨网络通信，路由选路
- IP协议：核心协议
- IP地址：IPv4（32位）、IPv6（128位）
- 公网IP vs 私网IP
  私网段：10/8、172.16/12、192.168/16
- 子网划分与子网掩码 / CIDR
- 路由器：选路转发，路由表
- ICMP协议：ping用的就是这个
- IP层的安全问题：IP欺骗、分片攻击...

最后送你一句话：

> **网络是安全的基础，
> 地基不牢，地动山摇。**
