## 11.2 防火墙概述

### 11.2.1 什么是防火墙？

什么是防火墙（Firewall）？
简单说就是：
**部署在网络边界，
按照预先设定的安全规则，
对进出的网络流量进行检查和控制，
允许合法的通过，
阻止非法的通过的安全设备。**

就像大楼门口的保安，
检查进出的人和车辆，
好人放行，坏人拦住。
防火墙就是网络世界的保安。

防火墙的核心作用：
- **访问控制**：哪些IP能进，哪些不能进
- **隔离**：不同安全区域之间的隔离
- **记录日志**：记录访问日志，方便审计
- **NAT**：网络地址转换，隐藏内网地址
- VPN支持：远程安全接入
- 还有其他各种功能...

### 11.2.2 防火墙的发展历程

防火墙不是一成不变的，
它也经历了好几代的发展：

```
  第一代：包过滤防火墙
    最早的防火墙
    只检查IP头、TCP/UDP头这些
    源IP、目的IP、源端口、目的端口、协议
    符合规则就放行，不符合就拦
    简单、速度快，但很粗糙
    不看内容，不知道你在干嘛
    就像保安只看你身份证上的地址，
      不看你带了什么东西

  第二代：状态检测防火墙
    比包过滤高级一点
    会记录连接的状态
    比如TCP三次握手的状态
    知道这个包是哪个连接的
    动态地放行返回的数据包
    比包过滤更智能、更安全
    现在很多基础防火墙还是这一类

  第三代：应用层防火墙 / 代理防火墙
    工作在应用层
    能看懂应用层的内容
    比如HTTP、FTP协议的内容
    可以做更细粒度的控制
    比如允许访问网页，
      但不允许上传文件
    但速度慢，性能差
    现在不多见了

  第四代：下一代防火墙（NGFW）
    Next-Generation Firewall
    现在的主流
    把防火墙、IPS、应用识别、
    内容过滤、VPN、AV...
    一大堆功能都集成在一起
    既能做传统的网络层防火墙，
    又能做应用层的防护
    功能非常全面
    是现在企业的首选

  还有UTM（统一威胁管理），
    也是把很多功能集成在一起，
    跟NGFW有点像，
    但架构和侧重点不太一样，
    现在UTM的概念慢慢被NGFW取代了
```

现在企业用的基本都是**下一代防火墙（NGFW）**，
功能强大，性能也好。

### 11.2.3 防火墙的工作层次

前面网络基础讲过OSI七层模型，
防火墙工作在哪一层？
不同类型的防火墙不一样：

```
  包过滤防火墙：
    网络层、传输层
    只看IP和端口

  状态检测防火墙：
    网络层、传输层
    加上连接状态

  应用层防火墙 / 代理防火墙：
    应用层
    能看懂应用内容

  下一代防火墙（NGFW）：
    从网络层到应用层都管
    全栈防护
```

工作层次越高，
能检查的内容越多，
控制越精细，
但性能消耗也越大。

## 11.3 防火墙的核心功能

防火墙都有哪些功能？
给你介绍几个核心的。

### 11.3.1 访问控制（ACL）

最基础、最核心的功能——
**访问控制**。
也就是根据规则，
允许或者拒绝某些流量。

规则（ACL，访问控制列表）
一般是五元组：
```
  源IP地址
  目的IP地址
  源端口
  目的端口
  协议类型（TCP/UDP/ICMP...）
```

当然，
NGFW的规则可以更复杂，
可以基于应用、基于用户、
基于内容、基于时间...

防火墙的规则是从上往下匹配的，
匹配到第一条就执行相应动作，
后面的就不看了。
所以规则的顺序很重要。

### 11.3.2 NAT 网络地址转换

第二个重要功能，
**NAT（Network Address Translation）**，
网络地址转换。

什么是NAT？
简单说就是：
**把IP包头里的IP地址，
转换成另一个IP地址。**

为什么要做NAT？
```
  1. 解决公网IP不够的问题
    内网一堆机器用私网IP，
    通过NAT共享一个公网IP上网
    这是最常见的用法
    叫PAT（端口地址转换）或者NAPT

  2. 隐藏内网结构
    内网的IP和拓扑，
    外面看不到
    提高安全性

  3. 服务器映射
    内网的服务器，
    通过目的NAT（端口映射），
    把公网IP的某个端口
    映射到内网服务器的端口上
    外面的人访问公网IP:端口，
    就能访问到内网服务器
```

NAT分几种：
```
  • 源NAT（SNAT）：
    转换源IP地址
    内网用户上网用这个

  • 目的NAT（DNAT）：
    转换目的IP地址
    外网访问内网服务器用这个
    也叫端口映射

  • 双向NAT：
    源和目的都转
    特殊场景用
```

NAT是防火墙的基本功能，
几乎所有防火墙都支持。

### 11.3.3 VPN 支持

第三个常见功能，
**VPN（虚拟专用网络）**。

什么是VPN？
简单说就是：
**在公共网络上，
建立一条加密的、安全的隧道，
就像在公网里挖了一条专用的加密通道，
两端可以安全地通信，
别人看不到也改不了。**

VPN的用途：
- 远程办公：员工在家通过VPN连公司内网
- 分支机构互联：分公司和总部通过VPN连起来
- 数据传输加密：重要数据通过VPN加密传输

常见的VPN类型：
```
  • IPsec VPN
    网络层的VPN
    站点到站点（Site-to-Site）常用
    安全性高

  • SSL VPN
    应用层的VPN，基于HTTPS
    远程接入常用，
    不用装客户端，浏览器就能用
    方便，但性能差一点

  • L2TP、PPTP
    老的VPN协议，
    现在用得少了
```

企业防火墙一般都支持IPsec VPN和SSL VPN。

### 11.3.4 应用识别与控制

这是下一代防火墙（NGFW）的特色功能——
**应用识别**。

传统防火墙只能基于IP和端口控制，
但现在很多应用都跑在80、443端口上，
你光看端口根本不知道是什么应用。
比如微信、QQ、抖音、
淘宝、迅雷...
都用80/443端口。
传统防火墙分不清。

NGFW可以识别具体的应用，
基于应用来做控制。
比如：
- 允许访问网页，但禁止看视频
- 允许用微信，但禁止用文件传输
- 禁止上班时间玩游戏、刷抖音
- 禁止使用迅雷等P2P下载工具
- 等等...

这就是应用识别的好处，
管控更精细。

### 11.3.5 内容过滤

再高级一点的，
**内容过滤**。
不光知道你用什么应用，
还能看你传的内容，
过滤掉不好的东西。

比如：
- 网页过滤：禁止访问某些网站
  （色情、赌博、钓鱼...）
- 关键字过滤：过滤敏感内容
- 文件过滤：禁止传输某些类型的文件
  （exe、脚本、敏感文档...）
- 病毒过滤：查杀流量里的病毒
  （集成杀毒引擎）

内容过滤功能越做越强，
现在的NGFW基本都有。

### 11.3.6 其他功能

除了上面这些，
防火墙还有很多其他功能：
- **入侵防御（IPS）**：后面专门讲
- **防DDoS**：抗DDoS攻击
- **用户认证**：基于用户做控制
- **高可用（HA）**：双机热备，防止单点故障
- **日志与报表**：详细的日志和统计报表
- **流量监控与管理**：看谁占了多少带宽
- 等等...

现在的防火墙越做功能越多，
几乎成了全能选手。

## 11.4 防火墙的部署模式

防火墙怎么部署？
常见的部署模式有几种：

### 11.4.1 路由模式（网关模式）

最常见的部署模式——
**路由模式**。
防火墙作为三层设备，
有自己的IP地址，
作为内网的网关。
内网的流量都经过防火墙，
由防火墙做NAT和转发。

```
  互联网 → 防火墙 → 内网
            （网关）
```

优点：
- 功能最全，NAT、VPN、各种安全功能都能用
- 部署简单
缺点：
- 要改变网络拓扑，要改网关

大多数企业都是这种部署模式。

### 11.4.2 透明模式（桥模式）

第二种，
**透明模式**，也叫桥模式。
防火墙工作在二层，
像个网桥一样串在网络里。
没有IP地址，
用户感觉不到它的存在。

```
  互联网 → 防火墙（透明）→ 路由器/网关 → 内网
```

优点：
- 不用改现有网络拓扑，不用改网关
- 部署方便，透明接入
缺点：
- 有些功能用不了（比如NAT）

适合那些不想改现有网络，
又想加个防护的场景。

### 11.4.3 混合模式

第三种，
**混合模式**。
一部分接口是路由模式，
一部分是透明模式，
混在一起用。
复杂网络环境下可能会用到。

### 11.4.4 旁路部署

还有一种，
**旁路部署**。
不串在线路上，
通过镜像把流量复制一份给防火墙，
防火墙只做检测，不做阻断。
IDS一般是旁路部署，
防火墙很少这么部署。

## 11.5 防火墙的局限性

防火墙很重要，
但防火墙不是万能的。
它有很多局限性。

```
  1. 防不住应用层攻击
    传统防火墙只能防网络层、传输层
    对于SQL注入、XSS这些Web攻击，
      传统防火墙根本拦不住
    （NGFW有一定的应用层防护能力，
      但专门的事还是得专门的设备干，
      比如WAF）

  2. 防不住内部攻击
    防火墙一般部署在网络边界
    对内网之间的攻击管不了
    比如内网一台机器被攻破了，
      横向移动打其他机器，
      防火墙管不着

  3. 防不住绕过防火墙的攻击
    比如员工通过钓鱼邮件点了恶意链接，
      从内部主动连出去
    防火墙一般对出站流量限制没那么严
    还有比如通过VPN进来的，
      防火墙也管不了

  4. 0day漏洞防不住
    防火墙的规则都是已知的
    对于未知的0day漏洞攻击，
      防火墙一般拦不住

  5. 配置错误等于没装
    防火墙规则配得乱七八糟，
      什么都放行
    那跟没装一样
    甚至还不如不装，
      至少你知道不安全
```

所以说，
防火墙是基础，
但光有防火墙远远不够。
安全需要多层防护，纵深防御。

## 11.6 入侵检测系统（IDS）

讲完防火墙，
接下来讲 **IDS（Intrusion Detection System）**，
入侵检测系统。

### 11.6.1 什么是IDS？

什么是IDS？
简单说就是：
**对网络流量或者系统行为进行监控，
发现其中的入侵行为和可疑行为，
并发出告警的系统。**

用人话说就是：
**专门找坏人、找攻击的监控系统。**

防火墙是"拦"，
IDS是"看"。
防火墙在门口拦着，
IDS在里面巡逻，
看看有没有混进来的坏人，
有没有异常的行为。

### 11.6.2 IDS的分类

IDS怎么分类？
按监控的对象分：

```
  1. NIDS（网络入侵检测系统）
    监控网络流量
    串接或者旁路部署在网络上
    分析网络数据包，
      找攻击行为
    比如Snort、Suricata
    还有很多硬件NIDS设备

  2. HIDS（主机入侵检测系统）
    监控单台主机
    装在主机上的代理
    监控主机的系统调用、
      文件变化、进程、日志...
    看主机有没有被入侵
    比如OSSEC、Tripwire
    现在的EDR其实也属于HIDS的升级版
```

按检测原理分：
```
  1. 误用检测（特征检测）
    有一个攻击特征库
    把流量或者行为和特征库比对
    匹配上了就是攻击
    就像杀毒软件的特征码检测

    优点：准确，误报低
    缺点：只能检测已知攻击，
      对未知攻击、0day没用
      特征库要一直更

  2. 异常检测
    先建立一个"正常"的基线
    然后监控当前行为，
    和基线比，
    偏离太多就是异常，可能是攻击

    优点：能检测未知攻击、0day
    缺点：误报率高，
      什么算"正常"不好定义

  现在的IDS一般是两种结合着用
```

### 11.6.3 IDS的部署

NIDS一般怎么部署？
**旁路部署**。
就是不串在线路上，
通过端口镜像（SPAN口）
把流量复制一份给IDS，
IDS只负责检测和告警，
不阻断流量。

为什么不串接？
因为IDS是检测用的，
如果串接，
万一IDS挂了，
整个网络就断了，
影响业务。
旁路部署的话，
IDS挂了不影响正常业务。

```
  互联网 → 防火墙 → 交换机 → 内网
                     ↓（镜像流量）
                    IDS
```

IDS的作用是**发现攻击、发出告警**，
然后管理员去处理。
它本身不拦，
只报警。

### 11.6.4 IDS的优缺点

IDS的优点：
- 能发现防火墙漏掉的攻击
- 旁路部署，不影响业务
- 有详细的攻击日志，方便分析和取证

IDS的缺点：
- 只能报警，不能主动拦截
  （所以叫检测系统，不是防御系统）
- 有漏报和误报
  需要有人去分析告警
  误报多了容易让人麻木
- 对于加密流量（HTTPS）检测困难
  当然现在也有解密的方案

## 11.7 入侵防御系统（IPS）

既然IDS只能报警不能拦，
那能不能让它既能检测又能拦截？
可以，这就是 **IPS**。

### 11.7.1 什么是IPS？

IPS（Intrusion Prevention System），
入侵防御系统。
简单说就是：
**IDS + 阻断功能。**
不光能检测攻击，
还能主动把攻击流量拦下来。

IPS和IDS的核心区别：
```
  IDS：旁路部署，只检测，告警，不阻断
  IPS：串接部署，检测 + 主动阻断
```

IPS串在线路上，
所有流量都要经过它，
它检测到攻击，
直接就把攻击包丢掉，
不用等管理员来处理。

### 11.7.2 IPS的工作原理

IPS的工作原理和IDS差不多，
也是基于特征检测 + 异常检测。
但因为要串接、要阻断，
对性能和准确性要求更高。

- 性能要高：不能成为网络瓶颈
- 误报要低：不能把正常流量拦了，影响业务

所以IPS的技术门槛比IDS更高。

现在的**下一代防火墙（NGFW）**，
基本都集成了IPS功能。
所以很多企业就不用单独买IPS了，
直接用NGFW里的IPS就行。
当然专业的IPS设备性能和功能还是更强一些。

### 11.7.3 IPS的部署

IPS一般是**串接部署**，
串在防火墙和核心交换机之间，
或者其他关键路径上。

```
  互联网 → 防火墙 → IPS → 核心交换机 → 内网
```

当然也可以旁路部署，
但旁路就只能检测不能阻断了，
那跟IDS差不多了。

IPS部署要注意高可用，
因为它串在线路上，
它挂了网络就断了。
所以一般要做HA（高可用），
双机热备，
一台挂了另一台顶上。

## 11.8 WAF —— Web应用防火墙

讲了这么多，
还有一个很重要的防火墙——
**WAF（Web Application Firewall）**，
Web应用防火墙。
这个很多人容易和普通防火墙搞混，
我们单独拿一节来讲。

### 11.8.1 什么是WAF？

什么是WAF？
简单说就是：
**专门防护Web应用的防火墙。**

普通防火墙（包括NGFW）
主要工作在网络层和传输层，
对Web应用层的攻击（SQL注入、XSS、
文件上传、CSRF...）防护能力有限。
WAF就是专门干这个的，
工作在应用层，
专门防护Web攻击。

```
  普通防火墙：管网络层、传输层，防网络攻击
  WAF：管应用层，防Web攻击
```

### 11.8.2 WAF能防什么？

WAF主要防护Web应用的常见攻击：
```
  • SQL注入
  • XSS跨站脚本
  • CSRF跨站请求伪造
  • 文件上传漏洞
  • 命令执行、代码执行
  • 文件包含、目录遍历
  • 扫描器、爬虫
  • CC攻击、应用层DDoS
  • 0day攻击（一定程度上）
  • ......
```

基本OWASP Top 10的攻击，
WAF都能防。

### 11.8.3 WAF的工作原理

WAF怎么工作的？
```
  1. 特征匹配
    有一个Web攻击规则库，
    把请求和规则比对，
    匹配上了就拦截
    这是最基本的

  2. 白名单/黑名单
    白名单里的IP/URL放行
    黑名单里的直接拦

  3. 行为分析
    分析访问行为，
    比如一个IP短时间内大量请求
    不同页面、大量尝试注入
    一看就是扫描器或者攻击
    直接封IP

  4. 机器学习 / AI
    现在很多WAF用机器学习，
    学习正常业务的模型，
    异常的请求就拦截
    对0day和变种攻击效果更好

  5. 虚拟补丁
    刚爆出的漏洞，
    业务代码还没来得及修，
    可以先在WAF上加规则，
    临时挡住，
    给开发争取修复的时间
    这个很实用
```

### 11.8.4 WAF的部署方式

WAF怎么部署？
常见的有几种：

```
  1. 反向代理模式（透明代理模式）
    WAF作为反向代理，
    对外的域名指向WAF，
    WAF再把请求转发给后端Web服务器
    所有流量都经过WAF
    防护效果最好
    也是最常见的部署方式

  2. 透明桥模式
    串接在Web服务器前面，
    像透明网桥一样
    不用改DNS
    但功能可能受限

  3. 旁路镜像模式
    旁路部署，只检测不阻断
    相当于Web入侵检测
    用得少

  4. 云WAF / CDN WAF
    不用自己买设备，
    用云服务商的WAF服务
    域名CNAME过去就行
    部署简单，弹性好
    现在越来越流行
```

现在越来越多的企业用云WAF，
尤其是网站类的业务，
部署方便，效果也不错。

### 11.8.5 WAF的局限性

WAF也不是万能的，
它也有局限性：
```
  1. 加密流量的问题
    HTTPS流量WAF要看内容，
    得先解密
    需要把证书配在WAF上
    当然这是能解决的

  2. 误报和漏报
    规则太松漏报多，
    规则太紧误报多
    平衡不好掌握
    需要根据业务调优规则

  3. 可以被绕过
    WAF不是百分之百能防住
    很多攻击可以通过编码、
    混淆、拆分等方式绕过WAF
    WAF是辅助，根本还是要代码写得安全

  4. 业务逻辑漏洞防不住
    越权、支付漏洞、验证码绕过...
    这些逻辑层面的漏洞WAF防不了
    因为它不知道业务逻辑是什么样的
```

WAF是Web安全的重要一层，
但不是全部。
代码安全才是根本，
WAF是锦上添花，
不是救命稻草。

## 11.9 其他常见安全设备

除了防火墙、IDS/IPS、WAF，
还有很多其他的安全设备，
简单给你介绍几个：

```
  • UTM（统一威胁管理）
    把防火墙、VPN、IPS、AV、
    反垃圾邮件...一大堆功能
    都集成在一个盒子里
    一体化设备
    适合中小企业，省钱省事
    但性能和专业性不如独立设备
    现在慢慢被NGFW取代了

  • 抗DDoS设备
    专门抗DDoS攻击的
    流量清洗、异常流量过滤
    对付大流量DDoS攻击
    一般机房或者大公司会有

  • 上网行为管理
    管员工上网的
    能看员工上什么网站、
      用什么应用、发什么内容
    还能管控，禁止上班时间玩游戏
    很多企业都有装
    安全和管理都能用

  • 堡垒机
    也叫运维审计系统
    运维人员登录服务器都要经过堡垒机
    统一认证、授权、审计
    所有操作都录下来，可审计
    防止内部人员搞事情
    等保三级一般要求有

  • 数据库审计
    专门审计数据库操作的
    谁查了什么、改了什么，都记录下来
    防止数据泄露和篡改
    金融、政府等对数据安全要求高的行业常用

  • 漏扫设备
    漏洞扫描设备
    定期扫内网的漏洞
    生成报告，督促修复

  还有很多很多...
    安全设备种类非常多
    不同场景用不同的设备
```

安全设备很多，
但不是买得越多越好。
适合自己的才是最好的。
买了一堆设备不会用、
策略没配好，
跟没买一样。

## 11.10 本章总结

这一章我们讲了防火墙和入侵检测系统，
内容不少。
总结一下：

**防火墙：**
- 网络边界的保安，访问控制
- 发展历程：包过滤 → 状态检测 →
  应用层 → 下一代防火墙（NGFW）
- 核心功能：访问控制、NAT、VPN、
  应用识别、内容过滤、IPS...
- 部署模式：路由模式、透明模式、混合模式
- 局限性：防不住应用层攻击、防不住内部攻击、
  防不住0day、配置错了等于白搭

**IDS 入侵检测系统：**
- 监控网络/主机，发现攻击，发出告警
- 分类：NIDS（网络）、HIDS（主机）
- 检测原理：特征检测（误用）、异常检测
- 旁路部署，只检测不阻断
- 优点：不影响业务，发现攻击
- 缺点：不能拦，有误报漏报

**IPS 入侵防御系统：**
- IDS + 阻断功能
- 串接部署，主动拦截攻击
- 对性能和准确性要求高
- 现在NGFW基本都集成了IPS

**WAF Web应用防火墙：**
- 专门防护Web应用的
- 防SQL注入、XSS、文件上传等Web攻击
- 部署方式：反向代理、透明桥、旁路、云WAF
- 局限性：误报漏报、可绕过、逻辑漏洞防不住

**其他安全设备：**
- UTM、抗DDoS、上网行为管理、
  堡垒机、数据库审计、漏扫设备...

最后送你一句话：

> **安全设备是工具，
> 工具好不好用，
> 关键看用的人。**
