## 3.1 先讲一个故事

### 3.1.1 "打电话和寄明信片"

讲传输层之前，
先给你讲个生活中的例子，
帮你理解TCP和UDP的区别。

想象一下，
你有一个重要的消息要告诉远方的朋友。
你有两种方式：

**方式一：打电话。**
你拨电话，
朋友接了，
你说一句，他"嗯"一声表示听到了，
你再说下一句。
如果哪一句他没听清，
他会让你再说一遍。
你们一句一句地聊，
确保每一句话对方都收到了、听清楚了。
聊完了，
说"再见"，挂电话。

**方式二：寄明信片。**
你把消息写在明信片上，
往邮筒里一扔，
就完事了。
明信片能不能寄到？
不知道。
路上丢了怎么办？
不知道。
对方收到了吗？
不知道。
顺序对不对？
不知道。
反正你发出去就不管了。

这两种方式，
正好对应传输层的两个协议：
- **TCP就是打电话** —— 可靠、有连接、有确认、会重传
- **UDP就是寄明信片** —— 不可靠、无连接、发完就不管

各有各的用处，
没有谁好谁坏，
看场景。

## 3.2 传输层

好，
我们正式讲**第四层：传输层**。

### 3.2.1 传输层是干嘛的？

网络层（IP层）只管把数据包
从源IP送到目的IP，
但是送到了之后，
交给哪个程序呢？
一台电脑上同时跑着那么多程序——
浏览器、QQ、微信、游戏...
数据包是给谁的？

这就是传输层要解决的问题：
**端到端的通信，
也就是进程到进程的通信。**
用**端口号**来区分不同的程序。

除了分端口，
传输层还提供：
- 可靠传输（TCP）
- 流量控制
- 拥塞控制
- 等等

传输层最主要的两个协议就是：
**TCP**和**UDP**。
我们一个一个讲。

### 3.2.2 端口号

先讲**端口号**。
刚才说了，
端口号是用来区分不同程序的。

端口号是个16位的数字，
范围是 0 - 65535。

端口号分几类：
```
  0 - 1023     周知端口（Well-known Ports）
                 给常用服务用的，固定的
                 比如：
                   80    HTTP
                   443   HTTPS
                   22    SSH
                   21    FTP
                   25    SMTP
                   53    DNS
                   3389  RDP
                   ...

  1024 - 49151  注册端口（Registered Ports）
                 给一些不那么常用的服务用
                 比如 MySQL 3306，Redis 6379...

  49152 - 65535 动态/私有端口
                 客户端程序临时用的
                 你浏览器开一个网页，
                 系统会随机给你分配一个源端口
                 用完就还
```

渗透测试里，
端口扫描扫的就是这个。
一个端口对应一个服务，
找到开着的端口，
就找到了可能的攻击面。

### 3.2.3 TCP协议

先讲TCP，
这个最重要，
内容也最多。

TCP（Transmission Control Protocol，
传输控制协议）
是一个**面向连接的、可靠的、基于字节流的**
传输层协议。

几个关键词：
- **面向连接**：通信之前要先建立连接
  （就像打电话先拨号）
- **可靠传输**：保证数据能送到，
  丢了会重传，顺序错了会重排
- **字节流**：数据是一串流，
  不分包（应用层自己分包）

TCP为什么能做到可靠？
靠的是一系列机制：
```
  • 确认与重传：发出去的包要等确认，
    超时没收到确认就重传
  • 序号与确认号：每个字节都有编号，
    保证顺序不乱
  • 流量控制：发送方不会发太快把接收方撑死
  • 拥塞控制：网络拥塞的时候放慢速度
    避免把网络搞堵
```

这些机制保证了TCP的可靠性。
代价就是速度相对慢一点，
开销大一点。
但对于需要可靠传输的场景，
比如网页、文件传输、邮件...
TCP是必须的。

### 3.2.4 TCP三次握手

TCP最有名的就是**三次握手**了。
什么是三次握手？
就是建立TCP连接的时候，
要来回发三个包，
连接才能建立成功。

为什么要三次？
两次不行吗？四次不可以吗？
这是面试必考题，
也是安全里经常用到的概念。

我用打电话的例子给你讲：

```
  客户端（你）："喂，听得到吗？"   （第一次）
  服务端（朋友）："听得到，你呢？" （第二次）
  客户端（你）："我也听得到，开始聊吧" （第三次）
```

就这样，
三次握手，
连接建立好了，
开始传数据。

用专业点的话说就是：
```
  1. 客户端发 SYN 包
     （SYN=1, seq=x）
     我要建立连接，我的初始序号是x

  2. 服务端回 SYN+ACK 包
     （SYN=1, ACK=1, ack=x+1, seq=y）
     收到了，同意建立连接，
     我的初始序号是y，
     确认你的序号x，下一个要x+1

  3. 客户端回 ACK 包
     （ACK=1, ack=y+1）
     收到了，确认你的序号y，下一个要y+1
     连接建立
```

为什么是三次？
两次行不行？
**不行。**
因为网络是复杂的，
包可能会延迟、会重复。
如果只有两次，
可能出现这种情况：
一个迟到的连接请求包，
突然到了服务端，
服务端以为是新的连接请求，
就答应了，
但客户端根本没想要建立连接，
就浪费了服务端的资源。
三次握手就是为了防止这种情况。

（当然，
真正的原因更复杂一点，
但对于新手来说，
理解到这个程度就够了。）

**SYN洪水攻击**就是利用这个原理的：
攻击者发大量的SYN包，
但不回第三次的ACK，
服务端就会一直等着，
资源都被占满了，
正常用户就连不上了。
这就是DDoS的一种。

### 3.2.5 TCP四次挥手

建连要三次握手，
断开连接呢？
要**四次挥手**。

为什么建连三次，
断连要四次？
因为TCP是全双工的，
两边都能发数据，
所以两边都要单独关。

还是用打电话举例：
```
  你："我说完了，挂了啊？"   （第一次）
  朋友："好，你挂吧。"       （第二次）
  （朋友这边可能还有话要说，
    说完了他再说挂）
  朋友："我也说完了，挂了。"  （第三次）
  你："好，拜拜。"           （第四次）
```

就这么个意思。
两边各关各的，
都关完了，
连接就断了。

专业点说：
```
  1. 客户端发 FIN 包
     我这边说完了，要关了

  2. 服务端回 ACK 包
     收到了，我知道了
     （但我可能还有数据没发完，
       等我发完再说）

  3. 服务端也发完了，发 FIN 包
     我也说完了，关吧

  4. 客户端回 ACK 包
     好的，收到
     （客户端等一会儿，确保对方收到ACK，
       然后真正关闭）
```

就是这么个过程。
了解一下就行，
不用死记硬背。

### 3.2.6 TCP的状态

TCP连接有很多状态，
比如 LISTEN、SYN_SENT、ESTABLISHED、
TIME_WAIT、CLOSE_WAIT...
这些状态不用全记住，
但要知道有这么回事。
你用 `netstat` 命令的时候，
就能看到连接的状态。

以后做安全分析、
看日志、查攻击的时候，
这些状态能帮你判断发生了什么。

### 3.2.7 UDP协议

讲完TCP，
再简单讲讲UDP。

UDP（User Datagram Protocol，
用户数据报协议）
是一个**无连接的、不可靠的**
传输层协议。

什么意思？
就是发完就不管了，
- 不用建立连接
- 不需要确认
- 不保证送到
- 不保证顺序
- 没有流量控制、拥塞控制

听起来很烂是不是？
但UDP有UDP的好处：
**快！非常快！**
开销小，延迟低。

什么时候用UDP？
```
  • DNS查询 —— 就一个请求一个响应，
    丢了大不了重查
  • 视频、语音通话 —— 丢几个包无所谓，
    卡一下比延迟强
  • 游戏 —— 实时性要求高，
    丢点数据没事
  • 广播/多播 —— 一对多发的时候
```

各有各的用处，
按需选择。

UDP的安全问题也不少，
比如UDP洪水攻击、
DNS放大攻击...
都是利用UDP不需要建立连接、
不需要确认的特点。
后面讲DDoS的时候会讲。

### 3.2.8 传输层和安全的关系

传输层跟安全有啥关系？
关系大了：
```
  • 端口扫描 —— 看哪些端口开着，
    就是扫描TCP/UDP端口
  • 防火墙 —— 很多防火墙就是基于端口和IP过滤的
  • DDoS攻击 —— SYN洪水、UDP洪水...
    都是传输层的攻击
  • 会话劫持 —— 劫持TCP连接
  • 端口复用 —— 一个端口跑多个服务
  • ......
```

传输层的知识，
是网络安全的基础中的基础。
一定要搞懂。

## 3.3 会话层、表示层

往上，
**第五层：会话层**，
**第六层：表示层**。

这两层说实话，
实际应用中用得不多，
很多协议都把这两层的功能
合并到应用层里了。
所以我们简单带过，
知道有这么回事就行。

### 3.3.1 会话层

会话层是干嘛的？
**建立、管理、终止会话。**
就是两个应用程序之间的会话，
什么时候开始、什么时候结束、
会话的同步...
大概就干这个的。

实际中，
大多数协议都把会话层的功能
放在应用层自己实现了，
所以这一层存在感比较低。

### 3.3.2 表示层

表示层是干嘛的？
**数据格式转换、加密解密、压缩解压。**
就是把数据转成双方都能看懂的格式。
比如：
- 编码转换
- 加密解密
- 数据压缩

这一层的功能，
现在也大多合并到应用层或者传输层了。
比如HTTPS的加密，
有人说是表示层的，
也有人说是应用层的。
不用太纠结。

## 3.4 应用层

再往上，
就是最顶层——
**第七层：应用层**。

这一层是我们平时接触最多的，
也是跟安全关系最密切的。
我们平时说的Web安全、
邮件安全、DNS安全...
都是应用层的安全。

应用层的协议太多了，
我们讲几个最常用、
跟安全关系最大的。

### 3.4.1 HTTP 与 HTTPS

第一个，
**HTTP（HyperText Transfer Protocol，
超文本传输协议）。**
这个我们每天都在用，
打开网页用的就是HTTP。

HTTP是怎么工作的？
简单说就是：
**请求-响应**模式。
客户端发一个HTTP请求，
服务器回一个HTTP响应。
就完事了。

HTTP请求长什么样？
```
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html
Cookie: xxx=yyy
...
（空行）
（请求体，POST的时候有）
```

HTTP响应长什么样？
```
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234
Server: Nginx
...
（空行）
<html>...</html>
```

HTTP是明文传输的，
所有内容都不加密，
中间人能看到所有内容。
所以现在基本都用**HTTPS**了。

**HTTPS**是什么？
就是 **HTTP + SSL/TLS**，
在HTTP外面加了一层加密。
数据传输是加密的，
中间人就算截到了也看不懂。
安全多了。

HTTPS的原理是什么？
简单说就是：
```
  1. 客户端和服务器建立TCP连接
  2. TLS握手：
     • 协商加密算法
     • 服务器出示证书（证明自己的身份）
     • 交换密钥
  3. 握手完成，后面所有HTTP数据都加密传输
```

HTTPS用到了非对称加密和对称加密：
- 非对称加密用来协商密钥和验证身份
- 对称加密用来加密实际传输的数据
（因为对称加密快）

HTTP和HTTPS是Web安全的基础，
后面讲Web攻击的时候会大量用到。
一定要搞明白。

### 3.4.2 DNS 域名系统

第二个，
**DNS（Domain Name System，
域名系统）。**

DNS是干嘛的？
**把域名翻译成IP地址。**

因为我们记不住那么多IP地址，
但好记的域名（比如baidu.com）
我们记得住。
DNS就是干翻译的活的：
你输入域名，
DNS告诉你对应的IP是多少，
然后你的电脑再去连那个IP。

DNS用的是UDP协议（大部分时候），
端口53。

DNS的工作过程大概是这样的：
```
  你要访问 www.example.com
    ↓
  先查本地缓存，有就直接用
    ↓
  没有就问本地DNS服务器
    ↓
  本地DNS服务器一层一层往上问
  （根域名服务器 → .com顶级域名服务器
   → example.com权威服务器）
    ↓
  最终拿到IP，返回给你
    ↓
  你用这个IP去访问
```

DNS看起来很简单，
但安全问题可不少：
```
  • DNS欺骗/缓存投毒
    给你一个假的IP，
    把你引到钓鱼网站

  • DNS劫持
    运营商或者路由器篡改DNS结果

  • DDoS攻击
    DNS放大攻击，
    用DNS协议做DDoS

  • DNS隧道
    把数据藏在DNS请求里，
    绕过防火墙偷数据
    （因为很多防火墙不拦DNS）

  • 域名劫持
    域名被盗、被篡改
```

DNS是互联网的基础设施，
DNS一出问题，
整个互联网都受影响。
DNS安全非常重要。

### 3.4.3 FTP 文件传输协议

第三个，
**FTP（File Transfer Protocol，
文件传输协议）。**
用来传文件的。

FTP用两个端口：
- 21端口：控制端口（发命令）
- 20端口：数据端口（传数据，主动模式下）

FTP有两种模式：
- 主动模式（Active）：服务器主动连客户端
- 被动模式（Passive）：客户端主动连服务器
  （现在大多用被动模式，因为防火墙的原因）

FTP的问题是什么？
**明文传输。**
用户名密码明文，
文件内容也明文。
中间人都能看到。
所以现在越来越多人用
**SFTP**（SSH文件传输）
或者 **FTPS**（FTP over TLS），
加密传输，安全多了。

FTP也是常见的攻击目标，
弱口令、匿名访问、
漏洞利用...
都很多。

### 3.4.4 SSH 远程登录

第四个，
**SSH（Secure Shell，
安全外壳协议）。**
用来远程登录服务器的，
端口22。

Windows以前远程桌面用RDP（3389），
Linux远程管理基本都用SSH。
现在Windows也自带OpenSSH了。

SSH为什么叫Secure Shell？
因为它是**加密的**，
所有传输的内容都是加密的，
中间人看不到。
比Telnet（明文的远程登录，23端口）
安全多了。
Telnet现在基本没人用了。

SSH支持多种认证方式：
- 密码认证
- 公钥认证（更安全，推荐用）
- 主机认证
- 等等

SSH也是渗透测试中的常客——
弱口令爆破、
密钥泄露、
SSH隧道、
端口转发...
都跟SSH有关。

### 3.4.5 邮件协议：SMTP / POP3 / IMAP

第五个，
**邮件相关的协议。**

发邮件和收邮件用的是不同的协议。

```
  发邮件：SMTP（Simple Mail Transfer Protocol）
    端口：25（明文）、465/587（加密）
    简单邮件传输协议
    用来把邮件从客户端发到服务器，
    或者服务器之间转发邮件

  收邮件：POP3（Post Office Protocol v3）
    端口：110（明文）、995（加密）
    邮局协议第3版
    把邮件从服务器下载到本地
    下载完服务器上就没了（默认）

  收邮件：IMAP（Internet Message Access Protocol）
    端口：143（明文）、993（加密）
    互联网消息访问协议
    比POP3高级，
    邮件在服务器上，
    可以在服务器上管理邮件、文件夹
    多设备同步方便
```

邮件的安全问题也很多：
- 钓鱼邮件
- 邮件诈骗（商务欺诈BEC）
- 邮件炸弹
- 垃圾邮件
- 邮件服务器漏洞
- 等等

邮件是企业里最常用的沟通工具，
也是社工攻击的主要入口。
后面讲社工的时候还会详细讲。

### 3.4.6 其他常用应用层协议

再简单列几个其他常用的，
有个印象就行：

```
  • RDP（Remote Desktop Protocol）
    远程桌面协议，Windows自带
    端口3389
    远程图形化管理服务器用

  • Telnet
    明文远程登录
    端口23
    不安全，基本淘汰了
    但很多老设备还在用

  • SNMP（Simple Network Management Protocol）
    简单网络管理协议
    端口161
    管理网络设备用的
    配置不好也会有安全问题

  • NFS / SMB
    文件共享协议
    NFS是Linux/Unix的
    SMB是Windows的（端口445、139）
    永恒之蓝就是SMB的漏洞

  • MySQL / Oracle / MSSQL / Redis...
    各种数据库协议
    各自有各自的端口
    也是常见的攻击目标
```

协议太多了，
讲不完。
常用的先掌握，

## 3.5 TCP/IP四层模型

讲完OSI七层，
再简单提一下**TCP/IP四层模型**。
这个也很重要，
而且实际中用得更多。

OSI七层是理论模型，
而TCP/IP四层是实际互联网用的模型。
它把OSI的七层合并成了四层：

```
  OSI七层          TCP/IP四层

  应用层
  表示层    ──→   应用层
  会话层
  ─────────────
  传输层    ──→   传输层
  ─────────────
  网络层    ──→   网络层（网际层）
  ─────────────
  数据链路层 ──→
  物理层    ──→   网络接口层（链路层）
```

TCP/IP四层：
1. 应用层
2. 传输层
3. 网络层（网际层）
4. 网络接口层（链路层 / 主机-网络层）

我们平时说的TCP/IP协议栈，
就是这个。
OSI是理论，
TCP/IP是实际。
两个都要知道。

## 3.6 常用网络命令实战

讲了这么多理论，
最后给你讲几个
最常用的网络命令。
这些都是实操中天天要用的，
一定要会。

### 3.6.1 ping

最基础的，
测试网络通不通。

```bash
ping 目标IP或域名

例子：
ping 8.8.8.8
ping www.baidu.com
```

用的是ICMP协议。
ping不通不代表一定不在线，
可能是防火墙禁了ICMP。

### 3.6.2 tracert / traceroute

看数据包经过了哪些路由节点，
路径追踪。

```bash
# Windows
tracert 目标

# Linux
traceroute 目标
```

排查网络问题的时候很有用，
看看是在哪一段断的。

### 3.6.3 ipconfig / ifconfig / ip

看本机的网卡信息、IP地址、MAC地址...

```bash
# Windows
ipconfig
ipconfig /all   # 更详细

# Linux
ifconfig       # 老命令
ip addr        # 新命令
```

### 3.6.4 netstat / ss

看网络连接、端口、路由表...
非常常用。

```bash
# 看所有连接和监听端口
netstat -ano    # Windows
netstat -tulnp  # Linux

# 看监听的端口
netstat -an | findstr LISTENING   # Windows
netstat -tulnp                    # Linux

# 看路由表
netstat -rn
```

ss命令是netstat的替代品，
Linux上新一点的系统都用ss了，
更快。
`ss -tulnp` 跟 netstat -tulnp 类似。

### 3.6.5 nslookup / dig

查DNS，
看域名对应的IP。

```bash
nslookup 域名

# Linux还有dig，更强大
dig 域名
```

### 3.6.6 arp

看ARP缓存表，
IP和MAC的对应关系。

```bash
arp -a
```

这些命令是最基础的，
一定要熟练。
不管是做网络还是做安全，
天天都要用。

## 3.7 本章总结

这一章我们讲了传输层到应用层，
加上上一章的下三层，
OSI七层就全部讲完了。
内容很多，总结一下：

**传输层（第4层）：**
- 端口号：0-65535，区分不同的程序
  周知端口（0-1023）、注册端口、动态端口
- **TCP**：面向连接、可靠、字节流
  - 三次握手建立连接
  - 四次挥手断开连接
  - 确认重传、流量控制、拥塞控制
  - 常见端口：80(HTTP)、443(HTTPS)、22(SSH)、21(FTP)...
- **UDP**：无连接、不可靠、快
  - 常见端口：53(DNS)、67/68(DHCP)...
- SYN洪水等DDoS攻击

**会话层 / 表示层：**
- 实际用得不多，功能大多合并到应用层了
- 简单了解即可

**应用层（第7层）：**
- **HTTP/HTTPS**：网页浏览，请求-响应模式
  HTTPS是HTTP+TLS加密
- **DNS**：域名解析，把域名转IP
  安全问题：DNS欺骗、劫持、放大攻击、DNS隧道
- **FTP/SFTP/FTPS**：文件传输
  FTP明文，不安全
- **SSH**：安全远程登录，加密的，22端口
- **邮件协议**：
  发邮件SMTP，收邮件POP3/IMAP
- **RDP、Telnet、SNMP、SMB、数据库协议...**

**TCP/IP四层模型：**
- 应用层、传输层、网络层、网络接口层
- 实际互联网用的就是这个

**常用网络命令：**
- ping、tracert/traceroute
- ipconfig/ifconfig/ip
- netstat/ss
- nslookup/dig
- arp

最后送你一句话：

> **网络是安全的骨架，
> 骨架搭好了，
> 才能往上建高楼。**
