## 📖 写在前面

上一章我们讲了数据安全与隐私保护，
这一章我们来讲另一个很重要的方向——
**终端安全与主机加固。**

什么是终端？
终端就是我们日常用的电脑、
服务器、手机、平板...
这些端设备。
为什么终端安全重要？
因为很多攻击最后都要落到终端上——
钓鱼邮件骗你点，
点了就在你电脑上运行木马；
Web渗透拿下服务器，
服务器也是终端；
内网横向移动，
也是一台一台终端打过去。

可以说，
**终端是安全的最后一道防线，
也是最容易被突破的防线。**
终端安全做不好，
前面多少层防护都白搭。

这一章我们就来讲讲：
终端安全是什么、
为什么重要、
终端安全要做什么、
主机加固又是怎么回事、
Windows和Linux各怎么加固、
还有什么是EDR...

## 14.1 先讲一个故事

### 14.1.1 "一台未打补丁的服务器"

我有个朋友在一家创业公司做运维。
公司不大，
也就几十台服务器。
朋友平时很忙，
各种事情一大堆，
服务器打补丁这种事，
他总觉得"不着急，
反正有防火墙，
黑客打不进来"。
所以服务器常年不打补丁，
系统还是好几年前的老版本。

我劝过他好几次，
说赶紧打打补丁，
不然出事了麻烦。
他总是说：
"没事没事，
我们又不是什么大公司，
谁会来打我们？
打补丁太麻烦了，
还容易把业务搞挂了。"

结果呢？
公司做了个活动，
官网流量很大，
也被黑客盯上了。
黑客用了一个很老的Struts2漏洞，
直接就把Web服务器拿下来了。
这个漏洞好几年前就有了，
补丁早就出来了，
但他们服务器一直没更。

拿下Web服务器之后，
黑客又利用内网另一台服务器
一个更老的漏洞（永恒之蓝），
横向移动到了数据库服务器，
把整个数据库都拖走了。
公司的核心数据全没了，
用户信息也泄露了。

最后花了好大的功夫才恢复，
公司损失了一大笔钱，
我那个朋友也被问责了。

---

你看，
很多人都有这个心态：
"我又不是什么大目标，
谁会来打我？"
但实际上，
黑客攻击很多时候是广撒网的，
扫描器扫到有漏洞的机器，
就自动打进来。
不管你是大公司还是小公司，
只要有漏洞，
就会被盯上。

**打补丁、做加固，
是终端安全最基础、
也是最重要的事情。**
这一章，
我们就来讲讲怎么把终端和主机
变得更安全。

## 14.2 终端安全概述

### 14.2.1 什么是终端安全？

什么是终端（Endpoint）？
简单说就是**网络的端点设备**。
包括：
- 桌面电脑、笔记本电脑
- 服务器（物理机、虚拟机、云主机）
- 手机、平板等移动设备
- 甚至IoT设备、打印机、摄像头...
只要是连在网络上的端设备，
都算终端。

那什么是终端安全？
就是**保护这些终端设备的安全，
防止它们被入侵、被利用、
数据被窃取，
同时保证终端的可用性。**

终端安全为什么重要？
```
  1. 终端是攻击的最终目标
    不管是钓鱼、Web攻击还是什么，
    最后都是要拿到终端的权限
    终端失守了，基本就完了

  2. 终端数量多、分布广、难管理
    企业里几十台上百台电脑，
    还有各种服务器
    每个都可能出问题
    管理起来很麻烦

  3. 终端是数据的载体
    重要数据很多都存在终端上
    终端丢了、被攻破了，
    数据就泄露了

  4. 终端是内网横向移动的跳板
    拿下一台终端，
    就可以打内网的其他机器
    终端是内网渗透的立足点
```

所以说，
终端安全非常重要。
很多企业重网络轻终端，
防火墙买最好的，
终端却裸奔，
这是非常错误的。

### 14.2.2 终端安全的发展

终端安全也经历了几代发展：

```
  第一代：杀毒软件
    最传统的终端安全
    就是装个杀毒软件，查杀病毒
    特征码检测为主
    只能防已知病毒

  第二代：终端安全管理（TSM/EMM）
    不光杀毒，
    还管终端的安全配置、
    补丁管理、外设管控、
    上网行为管理...
    功能更全了

  第三代：EDR（端点检测与响应）
    Endpoint Detection and Response
    不只是防，
    还能检测高级威胁，
    发现攻击了还能响应和处置
    更智能，更主动
    对付勒索软件、APT攻击更有效

  第四代：XDR（扩展检测与响应）
    Extended Detection and Response
    不光管终端，
    还把网络、邮件、云...
    各种数据整合在一起，
    做关联分析和自动响应
    更全面，更智能
    现在的趋势
```

从杀毒软件到EDR到XDR，
终端安全一直在进化，
越来越智能，
也越来越全面。

## 14.3 终端安全的核心内容

终端安全都要做什么？
内容很多，
我给你梳理一下主要的方面。

### 14.3.1 恶意代码防护

第一个，也是最基础的——
**防病毒、防恶意代码。**

这个大家都熟，
装杀毒软件嘛。
但现在光靠传统杀毒软件不够了，
还要有：
- 实时监控：文件、进程、网络实时监控
- 启发式检测：发现未知病毒
- 行为检测：看行为是不是恶意的
- 云查杀：云端大病毒库
- 勒索软件防护：专门防勒索的功能
- 等等...

现在企业里一般用企业版杀毒软件，
或者EDR，
统一管理，统一策略。

### 14.3.2 补丁管理

第二个非常重要——
**补丁管理**。
前面那个故事也讲了，
很多攻击都是利用已知漏洞，
补丁早就有了，
但就是不打。

补丁管理包括：
- 补丁检测：检测系统和软件有哪些补丁没打
- 补丁分发：自动把补丁推送到终端
- 补丁安装：自动安装补丁
- 补丁测试：重要补丁先测试，
  别打了补丁业务挂了
- 补丁状态监控：看哪些终端打了，哪些没打

为什么补丁管理很重要？
因为**绝大多数攻击，
利用的都是已知的、有补丁的漏洞。**
只要补丁打及时了，
大部分攻击都能防住。

很多企业安全做得不好，
根源就是补丁管理一塌糊涂。

### 14.3.3 基线配置 / 主机加固

第三个，**基线配置**，
也叫主机加固、安全加固。
就是按照安全基线，
把系统的安全配置调好，
关掉不安全的东西，
减少攻击面。

这个我们后面会专门讲。
简单说就是：
- 关掉没用的服务和端口
- 改掉默认配置、默认密码
- 设置安全的密码策略
- 配置访问控制
- 开启审计日志
- 等等...

系统默认安装是为了好用，
不是为了安全。
默认配置往往有很多不安全的地方，
需要我们手动加固。

### 14.3.4 访问控制与权限管理

第四个，**权限管理**。
终端的权限也要管好：
- 用户权限最小化，
  普通用户日常用普通权限，
  不要用管理员权限
- 本地管理员权限严格管控
- 特权操作要有审批、有审计
- 等等...

很多人图方便，
电脑一上来就用管理员账号，
什么都用管理员权限跑。
这样一旦中招，
恶意程序直接就是管理员权限，
想干嘛干嘛，危害大得多。
如果是普通用户权限，
很多事情它干不了，
危害就小很多。

**最小权限原则，
在终端上同样适用。**

### 14.3.5 数据保护

第五个，**终端上的数据保护**。
终端上存了很多重要数据，
也要保护好：
- 全盘加密：电脑丢了也不怕
  （BitLocker、FileVault）
- 数据防泄漏（DLP）：防止数据被拷走、传走
- 移动存储管控：U盘等外设管控
- 文档加密：重要文档加密
- 等等...

尤其是笔记本电脑，
容易丢，
一定要开全盘加密。
不然电脑丢了，
里面的数据就全泄露了。

### 14.3.6 终端审计与监测

第六个，**监控和审计**。
终端上的操作要能监控、能审计：
- 日志收集：系统日志、安全日志、应用日志
- 进程监控：什么进程在运行
- 网络监控：终端连了什么地方
- 文件监控：重要文件有没有被改、被删
- 操作审计：用户干了什么
- 异常检测：发现异常行为及时告警

出了安全事件，
要有日志可查，
能溯源。
平时也要监控，
及时发现攻击。

### 14.3.7 其他方面

还有很多其他方面：
- **准入控制**：终端不安全不让进内网
  （比如没打补丁、没装杀毒软件
  就不让连公司网络）
- **移动设备管理（MDM）**：手机、平板等移动设备的管理
- **虚拟化安全**：虚拟机的安全
- **云主机安全**：云上服务器的安全
- **漏洞扫描**：定期扫终端的漏洞
- 等等...

终端安全是个系统工程，
不是装个杀毒软件就完事了。

## 14.4 EDR —— 端点检测与响应

现在企业终端安全，
越来越多的开始用 **EDR**。
前面提到了，
这里专门讲一下。

### 14.4.1 什么是EDR？

EDR是什么？
Endpoint Detection and Response，
端点检测与响应。

简单说就是：
**装在终端上的安全软件，
不只是防病毒，
还能持续监控终端的行为，
发现高级威胁和异常，
并且能做出响应和处置。**

和传统杀毒软件有什么区别？
```
  传统杀毒软件：
    • 主要靠特征码
    • 只能防已知病毒
    • 功能比较单一
    • 偏防御，检测和响应能力弱

  EDR：
    • 不光有传统杀毒的功能
    • 还有行为分析、威胁狩猎
    • 能发现未知威胁、高级威胁
    • 有丰富的检测和响应能力
    • 可以远程隔离、处置、溯源
    • 有控制台，统一管理所有终端
```

简单说，
EDR比传统杀毒软件更高级、更智能，
对付勒索软件、APT攻击这些
高级威胁更有效。

### 14.4.2 EDR的核心功能

EDR一般有这些核心功能：

```
  1. 终端防护
    传统杀毒的功能都有，
    病毒查杀、实时防护、
    防火墙、入侵防护...

  2. 行为监控与分析
    持续监控终端的各种行为：
      进程创建、文件操作、
      网络连接、注册表修改、
      命令执行...
    然后分析这些行为是不是恶意的

  3. 威胁检测
    用各种方法检测威胁：
      特征匹配、行为分析、
      机器学习、威胁情报匹配...
    发现已知和未知的威胁

  4. 调查与溯源
    出了事件，
    可以在控制台上调查：
      这个病毒怎么来的？
      干了什么？
      影响了哪些文件？
      连了哪些IP？
    完整的攻击链溯源

  5. 响应与处置
    发现威胁了，可以快速处置：
      隔离文件、杀进程、
      隔离终端（断网）、
      回滚操作、
      清除威胁...
    不用一台一台跑过去处理
    控制台远程就能搞定

  6. 威胁狩猎（Threat Hunting）
    主动去寻找潜藏的威胁
    不是等告警，
      而是主动搜
    对付那些潜伏的、隐蔽的攻击

  7. 统一管理
    一个管理控制台，
      管所有终端
    统一下发策略、统一升级、
      统一监控
    管理员不用一台一台管
```

EDR现在已经成了企业终端安全的标配，
尤其是对安全要求比较高的企业。
当然，EDR也不是万能的，
它也有被绕过的可能，
但比传统杀毒软件强多了。

## 14.5 主机加固概述

接下来讲**主机加固**，
也叫系统加固、安全基线配置。
这是终端安全里非常重要的一块，
也是运维和安全工程师的基本功。

### 14.5.1 为什么要主机加固？

系统默认安装好之后，
是偏向于"好用"的——
服务开得多、权限放得宽、
配置比较松。
这样用起来方便，
但安全性就差了。

就像新房子，
开发商给你装的锁是最普通的，
谁都能开。
你住进去肯定要换个好锁，
装个防盗门，
再加个猫眼，
是不是？
主机加固也是一个道理——
**给系统"装个防盗门"，
把不安全的地方都补上，
减少攻击面。**

加固的目的：
- 减少攻击面：关掉没用的服务和端口
- 提高安全性：把不安全的配置改掉
- 降低风险：就算被攻击了，
  危害也小一点
- 符合合规要求：等保、ISO 27001
  都有安全配置的要求

### 14.5.2 加固的原则

主机加固遵循什么原则？
```
  1. 最小化原则
    只装需要的软件，
    只开需要的服务，
    只开需要的端口
    多余的全都关掉、卸掉
    （攻击面越小越安全）

  2. 最小权限原则
    服务用最低权限跑，
    用户用最低权限用
    多一点都不给

  3. 安全性与可用性平衡
    不能为了安全把系统搞的没法用了
    也不能为了好用完全不管安全
    找到一个平衡点

  4. 可审计
    开启日志，
    关键操作要有记录
    出了事能查

  5. 基线化
    做成标准的安全基线，
    所有新机器都按这个来
    统一标准，统一管理
```

### 14.5.3 常见的加固标准

加固不是瞎改，
一般有标准的基线和规范。
常见的：
```
  • CIS Benchmarks
    CIS（互联网安全中心）出的安全基线
    各种系统、软件都有
    非常权威，国际通用

  • 等保要求
    国内等级保护的要求里，
    也有主机安全的部分
    国内企业做等保要按这个来

  • 企业自己的安全基线
    很多企业会根据自己的情况，
    制定自己的安全基线标准

  • 厂商的加固指南
    比如微软、RedHat都有官方的安全加固指南
```

做加固的时候，
最好照着标准来，
不要自己拍脑袋改，
容易改出问题。

## 14.6 Windows 系统加固

接下来，
我们分别讲讲Windows和Linux
大概要加固哪些地方。
先讲Windows。

Windows系统加固常见的点：

```
  1. 账号安全
    • 禁用默认的Administrator账号
      （或者改名，设置强密码）
    • 禁用Guest账号
    • 新建管理员账号，不要用默认的
    • 普通用户日常用普通权限，
      不要用管理员账号
    • 设置强密码策略：
      密码长度、复杂度、过期时间...
    • 账户锁定策略：
      输错几次密码就锁账号
      防暴力破解

  2. 服务与端口
    • 关掉不需要的服务
      （远程注册表、Telnet、
        打印服务...不用的都关）
    • 关掉不需要的端口
    • 禁用不需要的协议
    • 开启Windows防火墙
      配置防火墙规则，只开需要的端口

  3. 系统配置
    • 禁用自动播放
      防止U盘病毒
    • 禁用默认共享（IPC$、Admin$...）
    • 关闭NetBIOS和LLMNR
      防止内网欺骗
    • 启用DEP（数据执行保护）
    • 启用ASLR（地址空间随机化）
    • 禁用不必要的功能
    • 显示文件扩展名
      防止被双扩展名骗（比如 .txt.exe）

  4. 补丁与更新
    • 开启自动更新
    • 及时打系统补丁和安全更新
    • 定期更新软件

  5. 日志与审计
    • 开启安全审计
      登录事件、对象访问、
      策略更改、特权使用...
      该开的审计都打开
    • 设置日志大小和保留时间
    • 日志最好传到远程日志服务器
      防止本地日志被删

  6. 权限控制
    • 严格控制本地管理员组的成员
      不要谁都是管理员
    • 文件权限设置合理
    • 注册表权限设置
    • UAC（用户账户控制）开最高

  7. 网络安全
    • 配置防火墙规则
    • 限制远程桌面（RDP）：
      只允许特定IP访问、
      改默认端口、
      开启网络级别认证、
      不要用弱口令
    • 禁用SMBv1等老协议
    • 等等...

  8. 其他
    • 安装杀毒软件/EDR
    • 启用BitLocker全盘加密
    • 屏幕保护加密码，
      离开自动锁屏
    • 禁用Ctrl+Alt+Del的不安全选项
    • 等等...
```

Windows加固的点非常多，
这里只列了常见的。
实际做的时候照着CIS基线或者等保要求来。

## 14.7 Linux 系统加固

再讲Linux系统加固。
Linux服务器用得非常多，
加固也很重要。

Linux系统加固常见的点：

```
  1. 账号安全
    • 禁用root远程登录
      （SSH不要让root直接登）
    • 禁用不需要的系统账号
    • 设置强密码策略
      （pam_cracklib等模块）
    • 密码定期更换
    • 用普通用户日常操作，
      需要root权限用sudo
    • 限制su到root的用户
    • 锁定多次登录失败的账号
      （pam_tally2、faillock）

  2. SSH服务加固
    SSH是最常被攻击的服务，
    一定要加固好：
    • 改默认端口（22改别的）
    • 禁用root远程登录
    • 禁用密码登录，用密钥登录
      （密钥比密码安全多了）
    • 禁用空密码
    • 限制可以登录的用户
    • 设置空闲超时自动断开
    • 开启SSH日志
    • 用fail2ban之类的工具防暴力破解

  3. 服务与端口
    • 关掉不需要的服务
      （systemctl disable/stop）
    • 只开需要的端口
    • 用netstat/ss看看哪些端口在监听
      不正常的都关掉
    • 安装配置防火墙
      （iptables/firewalld/ufw）
      只允许必要的访问

  4. 系统配置
    • 关闭SELinux？别关！
      开着SELinux能大大提高安全性
      很多人图方便直接关了，非常不好
    • 启用地址空间随机化（ASLR）
    • 禁用不必要的内核模块
    • 配置内核参数（sysctl.conf）
      网络相关的安全参数
      防SYN洪水、防IP欺骗...
    • 设置历史命令记录
      记录更多的命令历史，
      加上时间戳
    • 禁用Ctrl+Alt+Del重启
      防止误操作或者恶意重启

  5. 文件系统与权限
    • 重要文件和目录的权限要对
      比如/etc/passwd、/etc/shadow
    • 设置umask默认权限
    • 去掉不必要的SUID/SGID文件
      SUID文件很危险，
      有漏洞可能被用来提权
    • 重要文件做完整性校验
      （比如tripwire、AIDE）
      被改了能发现
    • /tmp、/var/tmp等目录
      加nosuid、noexec等挂载选项
      防止在临时目录里执行程序

  6. 日志与审计
    • 开启系统日志（syslog/rsyslog）
    • 开启审计服务（auditd）
      记录关键操作
    • 日志传到远程日志服务器
    • 日志定期备份
    • 配置日志轮转（logrotate）
      别把磁盘占满了

  7. 补丁与更新
    • 系统及时打补丁
      （yum update / apt upgrade）
    • 软件及时更新
    • 定期检查漏洞
    • 有漏洞管理流程

  8. 其他
    • 安装杀毒软件或者EDR
      Linux也有病毒和木马，
        不是绝对安全
    • 服务器时间同步
      日志时间要准确，
        不然排查问题很麻烦
    • 监控系统状态
      CPU、内存、磁盘、网络...
    • 定期备份
      （这个虽然不算加固，但很重要）
    • 等等...
```

Linux加固的点也很多，
而且不同的发行版（CentOS、Ubuntu、Debian...）
细节也不太一样。
同样建议照着标准基线来做。

## 14.8 加固的注意事项

最后讲几个主机加固的注意事项：

```
  1. 加固前一定要备份
    改配置之前先备份
    改坏了还能还原
    尤其是生产环境，
      千万要小心

  2. 先测试，再上线
    新的加固策略，
      先在测试环境试
    没问题了再上生产
    别一上来就全量改，
      万一业务挂了就惨了

  3. 做好文档
    改了什么配置，
    为什么改，
    有什么影响，
    都记下来
    以后出问题好查

  4. 不要过度加固
    别为了安全把系统搞的没法用了
    安全是为业务服务的，
      不是阻碍业务的
    找到平衡点

  5. 定期检查和更新
    加固不是一劳永逸的
    系统在变，业务在变，
      漏洞也在不断出来
    基线要定期更新，
      系统要定期检查
    持续加固，持续改进

  6. 自动化
    机器多了手动加固不现实
    用自动化工具批量做
    比如Ansible、SaltStack、Puppet...
    写好加固脚本，
      新机器自动跑一遍
    效率高，还不容易错
```

主机加固是个细活，
也是个长期的活。
做好了能大大提高系统的安全性，
很多攻击根本就进不来。

## 14.9 本章总结

这一章我们讲了终端安全与主机加固。
总结一下：

**终端安全概述：**
- 终端是网络的端点设备（电脑、服务器、手机...）
- 终端是安全的最后一道防线，也是最容易被突破的
- 发展历程：杀毒软件 → 终端管理 → EDR → XDR

**终端安全的核心内容：**
- 恶意代码防护（杀毒软件/EDR）
- 补丁管理（非常重要，大部分攻击利用已知漏洞）
- 基线配置/主机加固（减少攻击面）
- 访问控制与权限管理（最小权限）
- 数据保护（全盘加密、DLP、外设管控）
- 终端审计与监测（日志、监控、异常检测）
- 准入控制、移动设备管理...

**EDR 端点检测与响应：**
- 比传统杀毒更高级，更强
- 行为监控、威胁检测、调查溯源、
  响应处置、威胁狩猎
- 对付高级威胁更有效

**主机加固：**
- 给系统装个"防盗门"，减少攻击面
- 原则：最小化、最小权限、安全可用平衡、
  可审计、基线化
- 标准：CIS基线、等保要求、企业自己的基线

**Windows 加固要点：**
- 账号安全、服务端口、系统配置、
  补丁更新、日志审计、权限控制、
  网络安全、杀毒加密...

**Linux 加固要点：**
- 账号安全、SSH加固、服务端口、
  系统配置（SELinux别关）、文件权限、
  日志审计、补丁更新...

**加固注意事项：**
- 备份、先测试再上线、做好文档、
  不要过度加固、定期检查、自动化

最后送你一句话：

> **安全是个系统工程，
> 终端是这个系统的底座。
> 底座不牢，
> 上面再好看都是空中楼阁。**
