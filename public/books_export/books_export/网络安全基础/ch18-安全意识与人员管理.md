# 第十八章 安全意识与人员管理

> 第18章 | 预计学习时间：5小时 | 难度：⭐⭐

---

## 📖 写在前面

前面我们讲了很多技术层面的东西，
防火墙、IDS、加密、认证...
但你有没有发现，
很多安全事件，
最后都跟人有关系？

- 员工点了钓鱼邮件，导致内网被攻破
- 员工用弱密码，导致账号被撞库
- 员工把公司文件发到个人邮箱，导致泄露
- 员工随便插陌生U盘，导致中毒
- 员工把密码写在便签上，被人看到
- 员工在社交媒体上发内部信息，导致信息泄露
- ......

有统计表明，
**超过80%的安全事件，
都和人的因素有关。**
不是技术不够强，
而是人这个环节太薄弱了。

所以说，
**人是安全体系中最薄弱的一环，
也是最重要的一环。**
技术再牛，
人不行，
一切白搭。

这一章，
我们就来讲讲人的问题——
**安全意识与人员管理。**
怎么提高员工的安全意识，
怎么管理人，
让人这个最薄弱的环节，
变得坚固一点。

坐稳了，开唠。

## 18.1 先讲一个故事

### 18.1.1 "一个U盘攻破整个公司"

我给你讲个真实的故事。

某公司安全做得还不错，
防火墙、IPS、WAF、EDR...
该有的设备都有，
也都配置好了。
安全团队觉得自己防护得挺好，
黑客肯定打不进来。

结果有一次做渗透测试，
红队的人没费多大劲就打进去了。
怎么进去的？
**扔U盘。**

红队的人买了一堆U盘，
在上面印上公司的logo，
还写着"2024年Q2季度财务报表"、
"员工工资明细"之类的字样。
然后把这些U盘扔在公司的
停车场、电梯口、卫生间门口...
这些人来人往的地方。

结果呢？
不到一天，
就有好几个员工把U盘捡起来，
插到自己电脑上看了。
U盘里放了个带木马的文件，
名字叫"工资表.xlsx.exe"
（双扩展名，默认隐藏扩展名的话，
看起来就是个Excel文件）。
员工一点开，
木马就运行了，
红队的人就拿到了员工电脑的权限。

拿到员工电脑之后，
在内网里横向移动，
没过几天，
整个内网就被打穿了，
域控都拿下了。

就这么简单。
花了几十万买的安全设备，
被几个几块钱的U盘攻破了。
你说讽刺不讽刺？

---

这个故事告诉我们什么？
**技术防得住黑客，
防不住"猪队友"。**
安全意识薄弱的员工，
是企业最大的安全漏洞。

防火墙再厚，
设备再多，
架不住内部有人"开门揖盗"啊。
所以，
提高员工的安全意识，
是企业安全建设中
成本最低、效果最好的事情之一。
投入不多，
但收益巨大。

这一章，
我们就来讲讲安全意识和人员管理。

## 18.2 人是最大的安全弱点

### 18.2.1 为什么人是最大的弱点？

为什么说人是最大的安全弱点？
几个原因：

```
  1. 人是会犯错的
    人不是机器，
      总会有疏忽的时候
    再小心的人，
      也有走神、大意的时候
    一次疏忽，
      就可能导致严重的后果

  2. 安全意识不足
    大部分员工不是安全专业的
      不知道什么是危险、
      什么不能做
    不是他们想搞破坏，
      是他们根本不知道这样做有危险
    无知者无畏

  3. 人性的弱点
    好奇心、贪心、恐惧、
      乐于助人、服从权威...
    这些人性的弱点，
      很容易被社会工程学攻击利用
    而这些弱点，
      是技术防不住的

  4. 人是最难管理的
    设备可以强制配置，
      系统可以强制加固
    但人有自己的想法，
      很难强制
    你总不能24小时盯着员工吧

  5. 内部威胁
    除了不小心的，
      还有故意的
    内鬼、不满的员工、
      被收买的员工...
    内部人员作案更难防
```

因为这些原因，
人成了安全体系中
最薄弱、最难防的一环。
但反过来想，
如果把人的安全意识提上来了，
大部分的社工攻击、钓鱼攻击，
就都没用了。
**投资在人身上，
往往是ROI最高的安全投入。**

### 18.2.2 常见的人为安全风险

员工常见的不安全行为有哪些？
我给你列几个最常见的：

```
  密码相关：
    • 弱密码：123456、password、生日...
    • 所有网站用同一个密码
    • 密码写在便签上、记在本子上
    • 把密码告诉别人
    • 长期不换密码

  邮件相关：
    • 随便点陌生邮件的链接和附件
    • 不识别钓鱼邮件
    • 工作邮箱用来注册各种网站
    • 公司敏感信息通过个人邮箱发

  上网和办公相关：
    • 随便连公共WiFi
    • 在公共电脑上登公司账号
    • 随便下载软件，从非正规渠道下
    • 公司电脑装各种乱七八糟的软件
    • 随便插陌生的U盘、移动硬盘
    • 锁屏密码不设，离开电脑不锁屏
    • 把公司文件带回家、传到个人云盘
    • 在社交网络上晒公司内部信息
    • 敏感文件随便放，不加密

  其他：
    • 给陌生人开门、放陌生人进公司
    • 把工牌随便借人
    • 随便扫二维码
    • 相信陌生电话，泄露信息
    • ......
```

这些行为看起来都是小事，
但很多安全事件，
就是从这些小事开始的。
千里之堤，溃于蚁穴。

## 18.3 安全意识培训

怎么提高员工的安全意识？
最基础、最直接的方法就是——
**安全意识培训。**

### 18.3.1 培训的目标

安全意识培训的目标是什么？
```
  1. 让员工知道什么是危险的
    知道哪些行为不安全
    知道常见的攻击手段
    （钓鱼、木马、社工...）

  2. 让员工知道该怎么做
    遇到可疑情况怎么办？
    出了问题找谁？
    日常工作中要注意什么？

  3. 养成良好的安全习惯
    从"知道"到"做到"
    把安全变成习惯
    融入到日常工作中

  4. 营造安全文化
    让大家重视安全
    觉得安全是每个人的事
    不是只有安全部门的事
```

培训不是走形式，
不是上完课就完事了。
要真正让员工听进去、
记下来、用得上。

### 18.3.2 培训的内容

安全意识培训都培训什么？
常见的内容：

```
  基础安全知识：
    • 什么是网络安全？
    • 为什么安全很重要？
    • 每个人的安全责任
    • 公司的安全政策和规定

  密码安全：
    • 什么是强密码？怎么设强密码？
    • 为什么不能共用密码？
    • 密码管理器的使用
    • 多因素认证的重要性

  钓鱼邮件识别：
    • 什么是钓鱼邮件？
    • 钓鱼邮件有什么特征？
    • 怎么识别钓鱼邮件？
    • 收到可疑邮件怎么办？
    • 实际案例分析

  社会工程学防范：
    • 什么是社会工程学？
    • 常见的社工攻击手法
    • 怎么防范社工攻击？
    • 电话诈骗、冒充身份怎么防？

  办公安全：
    • 离开电脑要锁屏
    • 不要随便插U盘
    • 不要随便下载软件
    • 公共WiFi安全
    • 涉密文件怎么处理
    • 打印安全、碎纸
    • 物理安全（门禁、尾随）

  数据安全与隐私保护：
    • 什么数据是敏感的？
    • 敏感数据怎么处理？
    • 怎么发敏感文件？
    • 个人信息保护
    • 数据泄露的危害

  事件报告：
    • 遇到可疑情况怎么办？
    • 出了问题找谁？
    • 怎么报告？
    • 不要隐瞒，早报告早处理

  其他：
    • 常见恶意代码防范
    • 移动设备安全
    • 远程办公安全
    • 法律法规相关
    • 最新的威胁和案例
    ......
```

培训内容要结合员工的岗位和实际工作，
不要讲太多高深的技术，
要通俗易懂，
跟他们的工作相关，
让他们觉得有用。
如果讲的都是跟他们没关系的技术术语，
听了也白听。

### 18.3.3 培训的形式

培训怎么搞？
形式可以很多样，
不要老是上课讲PPT，
容易犯困，效果也不好。

```
  1. 线下培训 / 线上课程
    最传统的形式
    系统讲解知识
    适合新员工入职培训、定期培训

  2. 视频 / 动画
    做成短视频、动画
    生动有趣，容易看进去
    可以发到内部平台，随时看

  3. 案例分享
    讲真实的案例，
      尤其是同行业的
    真实案例比讲道理有冲击力多了
    "你看某某公司就是因为这个出事的"
    员工印象更深刻

  4. 海报 / 宣传册 / 邮件
    平时在办公室贴海报，
      发安全小贴士
    定期发安全提醒邮件
    耳濡目染，慢慢渗透

  5. 知识竞赛 / 答题
    搞个安全知识竞赛，
      答对了有奖励
    用游戏化的方式，
      提高参与度
    比单纯听课效果好

  6. 角色扮演 / 模拟演练
    比如模拟钓鱼电话、
      模拟社工攻击
    让员工亲身体验
    印象更深刻

  7. 钓鱼演练
    这个后面专门讲
    非常有效
```

培训形式要多样化，
反复强调，
不断强化。
不是培训一次就完事了，
要持续做，
安全意识才能真正提上来。

### 18.3.4 不同人群的培训

不同岗位的人，
培训的重点也不一样。
不能所有人都讲一样的内容。

```
  全体员工：
    基础安全意识、
    钓鱼防范、密码安全、
    办公安全、数据安全...
    最通用的内容

  新员工：
    入职必训
    公司的安全制度、
      基础安全知识
    上岗前先培训，
      考核通过才能上岗

  研发人员：
    安全编码、
    漏洞防范、
    代码安全、
    开源组件安全、
    DevSecOps...
    跟开发相关的安全

  运维人员：
    系统加固、漏洞管理、
    应急响应、权限管理...
    运维相关的安全

  财务 / HR：
    重点讲钓鱼邮件、
      诈骗防范
    因为他们经常是诈骗的目标
    掌握的敏感数据也多

  高管 / 领导：
    他们是鱼叉式钓鱼的重点目标
      （鲸鱼攻击）
    而且他们权限大，
      中招了危害大
    要专门培训，
      提高他们的重视程度

  第三方人员：
    外包、供应商、访客...
    他们也要遵守安全规定
    也要有基本的安全培训
```

因材施教，
效果才好。

## 18.4 钓鱼演练

光说不练假把式。
培训了员工会不会识别钓鱼邮件？
光靠考试没用，
要实战检验——
**钓鱼演练**。

### 18.4.1 什么是钓鱼演练？

什么是钓鱼演练？
简单说就是：
**公司自己给自己员工发钓鱼邮件，
看看谁会点、谁会中招，
然后针对性地进行教育和培训。**

就是模拟真实的钓鱼攻击，
来检验员工的安全意识，
同时也是一种培训。
而且是最有效的培训方式之一——
**真正上当过一次，
比听十次课印象都深刻。**

### 18.4.2 钓鱼演练怎么做？

钓鱼演练大致怎么搞？

```
  1. 制定计划
    明确演练的目的、范围、时间
    获得管理层的批准和支持
    （非常重要，不然会出事）

  2. 设计钓鱼场景
    设计仿真的钓鱼邮件
    可以模仿真实的场景：
      • 工资条通知
      • 系统升级通知
      • IT部门密码重置
      • 快递通知
      • 会议通知
      • 中奖通知
      ...
    越真实效果越好

  3. 发送钓鱼邮件
    给目标员工发邮件
    可以分部门、分批发

  4. 统计数据
    统计多少人点了链接，
      多少人输入了账号密码，
      多少人下载了附件
    按部门、按岗位统计
    哪些部门、哪些人是重灾区

  5. 针对性教育
    中招的员工，
      马上进行一对一的安全教育
    告诉他们为什么这是钓鱼邮件，
      哪里有问题，
      以后怎么识别
    不是为了惩罚，是为了教育

  6. 总结复盘
    整体的中招率是多少？
    哪些场景中招率高？
    哪些部门需要加强培训？
    下次演练怎么改进？
    形成报告，向上汇报
```

### 18.4.3 钓鱼演练的注意事项

钓鱼演练虽好，
但搞不好也容易出问题。
几个注意事项：

```
  1. 一定要获得管理层支持
    提前跟领导说清楚，
      获得批准
    不然有人觉得你在搞他，
      投诉你就麻烦了

  2. 以教育为目的，不是为了惩罚
    重点是教育和提高，
      不是为了抓典型、罚人
    要是罚得太狠，
      大家会有抵触情绪
    当然，屡教不改的
      可以适当惩罚

  3. 注意尺度
    不要搞太过分的，
      不要涉及隐私
    不要真的放病毒
    模拟就好，点到为止

  4. 及时反馈和教育
    员工中招了，
      马上告诉他为什么是钓鱼，
      怎么识别
    趁热打铁，效果最好

  5. 定期做
    不是做一次就完事了
    每季度或者每半年来一次
    持续强化
    安全意识是需要不断巩固的
```

很多企业实践下来，
经过几次钓鱼演练之后，
员工的中招率会显著下降。
效果比单纯上课好太多了。
**安全是练出来的，
不是听出来的。**

## 18.5 人员安全管理

除了培训和演练，
人员安全管理也很重要。
从招人到管人到走人，
全生命周期都要有安全管理。

### 18.5.1 入职安全

招人阶段就要考虑安全。

```
  • 背景调查
    重要岗位（尤其是安全、
      运维、财务、高管）
    要做背景调查
    有没有犯罪记录、
      有没有不良记录
    防止招进来的人本身就有问题

  • 入职培训
    新员工入职必须做安全培训
    考核通过才能上岗
    从一开始就建立安全意识

  • 权限开通
    按照最小权限原则，
      开通刚好够用的权限
    不要一上来什么权限都给

  • 签署安全协议
    保密协议、安全责任书...
    明确安全责任和义务
    法律上有个约束
```

### 18.5.2 在职安全

员工在职期间，
持续的安全管理。

```
  • 定期安全培训
    持续培训，不断强化
    新的威胁、新的案例
      及时传达

  • 权限管理
    定期审计权限
    岗位变动了，权限及时调整
    最小权限原则
    多余的权限及时收回

  • 行为监控 / 审计
    重要岗位的操作要审计
    异常行为及时发现
    不是不信任，是制度保障

  • 离职交接与权限回收
    这个非常重要！
    员工离职了，
      所有权限要立刻收回
    系统账号、门禁、邮箱、
      VPN、各种应用...
    全部禁用或删除
    设备、工牌、钥匙都要交回
    很多安全事件都是离职员工搞的
      因为权限没及时收

  • 定期考核
    安全意识和安全行为
      纳入绩效考核
    跟绩效挂钩，大家才重视
```

### 18.5.3 离职安全

离职环节是人员安全管理的
一个关键节点，
也是最容易出问题的地方。

```
  离职前：
    • 提前收回权限
      不要等走了才想起来
    • 安排工作交接
    • 收回所有公司资产
      电脑、手机、工牌、钥匙...
    • 数据清理和检查
      有没有带走公司数据？

  离职时：
    • 签署保密协议和离职承诺
    • 告知保密义务和法律责任
    • 关闭所有账号和权限

  离职后：
    • 继续保密义务
    • 竞业限制（如果有的话）
    • 敏感岗位可以做一些审计
```

很多企业对离职环节不重视，
员工都走了好几个月，
账号还开着，
门禁还能用，
这是非常危险的。
**离职流程一定要规范，
权限一定要及时收回。**

### 18.5.4 第三方人员管理

除了正式员工，
还有很多第三方人员：
外包、供应商、
临时工、访客...
这些人的安全管理也不能忽视。
很多安全事件就是从第三方突破的。

```
  第三方人员的安全管理：
    • 签署保密协议和安全协议
    • 最小权限原则，
      给刚好够用的权限
    • 访问要审批、有记录
    • 现场有人陪同
    • 网络访问要隔离，
      不能直接连内网
    • 离开及时收回权限
    • 定期审计
```

第三方是很多企业安全的盲区，
一定要重视起来。

## 18.6 安全文化

最后讲一下**安全文化**。
比培训和制度更高一层的，
是文化。

### 18.6.1 什么是安全文化？

什么是安全文化？
简单说就是：
**整个公司都重视安全，
每个人都觉得安全是自己的事，
大家都自觉地按安全规范做事，
安全融入到了日常工作的方方面面。**

不是安全部门在推动安全，
而是所有人都主动做安全。
安全不是负担，
是每个人的责任和习惯。

要达到这个程度很难，
但这是安全管理的最高境界。

### 18.6.2 怎么建设安全文化？

安全文化怎么建设？
```
  1. 高层重视和支持
    这是最重要的
    老板都不重视，
      下面肯定更不重视
    安全要上升到公司战略层面
    高层要带头做安全

  2. 全员参与
    安全不是安全部门一个部门的事
    是每个人的事
    让大家都参与进来
    有渠道反馈安全问题
    有奖励机制

  3. 持续教育和培训
    安全意识是需要不断强化的
    持续培训、持续演练
    潜移默化，形成习惯

  4. 制度和流程保障
    把安全融入到业务流程里
    不是额外加的负担
    而是工作的一部分
    用制度来保障

  5. 正向激励
    做得好的要奖励
    发现漏洞、报告隐患的要表扬
    让大家有积极性
    不要只有惩罚没有奖励

  6. 从管理层做起
    领导带头遵守安全规定
    比如强密码、锁屏、开MFA
    上行下效
    领导自己都不遵守，
      下面人肯定也不拿当回事
```

安全文化建设是个长期的过程，
不是一朝一夕的事。
但一旦建起来了，
效果是非常好的。

## 18.7 本章总结

这一章我们讲了安全意识与人员管理。
总结一下：

**人是最大的安全弱点：**
- 超过80%的安全事件和人有关
- 人会犯错、安全意识不足、
  有各种人性弱点、最难管理
- 投资在人身上，ROI最高

**常见人为安全风险：**
- 密码相关：弱密码、共用密码...
- 邮件相关：点钓鱼邮件、乱发文件...
- 办公相关：乱插U盘、不锁屏、公共WiFi...

**安全意识培训：**
- 目标：知危险、会应对、成习惯、建文化
- 内容：密码安全、钓鱼识别、社工防范、
  办公安全、数据安全、事件报告...
- 形式：多样化，培训、视频、案例、
  知识竞赛、演练...
- 不同人群不同重点

**钓鱼演练：**
- 自己发钓鱼邮件，检验员工安全意识
- 最有效的安全培训方式之一
- 注意：获得支持、教育为目的、注意尺度、
  及时反馈、定期做

**人员安全管理：**
- 入职：背景调查、入职培训、最小权限、保密协议
- 在职：定期培训、权限管理、行为审计、绩效考核
- 离职：及时收回权限、交接、资产回收
- 第三方：也要管好，不能是盲区

**安全文化：**
- 全员重视、自觉遵守、融入工作
- 高层支持、全员参与、持续教育、
  制度保障、正向激励、领导带头
- 长期建设，效果最好

最后送你一句话：

> **安全的本质是人。
> 技术是手段，
> 制度是保障，
> 人心才是防线。**
