## 1.1 先讲一个故事

### 1.1.1 "一次点击，损失数千万"

先给你讲一个真实的故事。

某公司，
做外贸的，
生意做得挺大，
一年几个亿的营收。
公司有几十号员工，
都用电脑办公，
有自己的官网、邮箱、业务系统。
老板觉得我们就是个做贸易的，
又不是什么高科技公司，
安全什么的无所谓，
装个杀毒软件就够了。

结果有一天，
公司财务收到一封邮件，
说是银行发来的，
标题是"您的账户有一笔异常交易，请核实"。
财务一看，
银行发的，
还有模有样的，
盖着银行的章，
落款也是银行的名字。
心里咯噔一下，
赶紧点了邮件里的链接，
想看看是什么异常交易。

点进去一看，
是一个跟银行网银长得一模一样的页面，
让输入账号密码验证码。
财务没多想，
就把账号密码都输进去了。

输完之后，
页面提示"系统维护中，请稍后再试"。
财务也没当回事，
关掉页面继续干活了。

结果第二天，
公司账上的两千多万，
全被转走了。
报警，
但钱已经被转到境外，
追回来的可能性微乎其微。

就因为点了一个链接，
输了一次密码，
两千多万没了。
公司差点倒闭。

---

你看，
这就是网络安全。
它不是什么虚无缥缈的东西，
也不是只有大公司、
只有政府才需要关心的事。
它跟我们每个人都息息相关。
- 你的QQ号会不会被盗？
- 你的银行卡会不会被刷？
- 你的隐私照片会不会泄露？
- 你的公司会不会被黑客攻击？

这些都是网络安全的事。

这一章，
我们就从最基础的讲起，
给你建立一个整体的认知。

## 1.2 网络安全是什么？

### 1.2.1 一个通俗的定义

什么是网络安全？
官方的定义有很多，
我给你一个最通俗的：

> **网络安全，就是保护你的电脑、手机、网络、数据，
> 不被坏人偷、不被坏人改、不被坏人搞坏。**

就这么简单。

再专业一点说，
网络安全是指：
保护计算机系统、网络、
程序和数据免受
数字攻击、损坏或未授权访问。

但你不用记这些，
知道大概是什么意思就行。

### 1.2.2 网络安全和信息安全是一回事吗？

经常有人问：
网络安全和信息安全有什么区别？
和 cybersecurity、information security 都是啥关系？

简单说一下：

```
信息安全（Information Security）
  └── 范围最大，所有信息的安全都算
      不管是纸质的还是电子的

  网络安全（Network Security / Cybersecurity）
    └── 主要指网络环境下的信息安全
        电脑、手机、网络、系统...
        我们一般说的网络安全就是这个

  还有很多细分：
    • Web安全 —— 网站、Web应用的安全
    • 移动安全 —— 手机、APP的安全
    • 云安全 —— 云计算、云服务的安全
    • 工控安全 —— 工业控制系统的安全
    • 数据安全 —— 数据的安全
    • 终端安全 —— 电脑、手机等终端的安全
    • ......
```

不用纠结这些概念，
知道大概意思就行。
我们这本书讲的"网络安全"，
就是一般意义上的网络空间安全，
范围比较广，
什么都讲一点。

## 1.3 CIA 三元组 —— 网络安全的核心目标

网络安全到底要保护什么？
说来说去，
核心就是三个目标，
叫 **CIA三元组**。
注意，
不是美国那个CIA啊，
是三个英文单词的首字母。

### 1.3.1 C —— 保密性（Confidentiality）

第一个，
**保密性（Confidentiality）。**

什么意思？
就是：
**该知道的人才能知道，
不该知道的人不能知道。**

用人话说就是：
**别让不该看的人看到。**

比如：
- 你的银行卡密码，只有你自己知道
- 公司的商业机密，只有内部人能看
- 你的聊天记录，别人不能随便看
- 病人的病历，只有医生能看

这些都是保密性。
如果保密没做好，
不该知道的人知道了，
就叫**信息泄露**。

比如前面故事里的，
账号密码被骗子骗走了，
就是保密性被破坏了。

### 1.3.2 I —— 完整性（Integrity）

第二个，
**完整性（Integrity）。**

什么意思？
就是：
**数据是完整的、准确的，
没有被篡改、没有被破坏。**

用人话说就是：
**别让不该改的人乱改。**

比如：
- 你银行里有100万，不能被人改成100块
- 你发的消息，不能被人在中间改了
- 公司的财务数据，不能被人篡改
- 系统文件，不能被病毒改了

这些都是完整性。
如果完整性被破坏了，
数据被改了，
那可能比泄露还严重。
比如你账上的钱被人转走了，
不光是知道了你的密码，
还动了你的钱，
这就是完整性也被破坏了。

### 1.3.3 A —— 可用性（Availability）

第三个，
**可用性（Availability）。**

什么意思？
就是：
**该用的时候能用，
想用的时候用得了。**

用人话说就是：
**别想用的时候用不了。**

比如：
- 你要网购，结果网站崩了，打不开
- 公司OA系统，上班的时候用不了
- 你要打游戏，服务器被攻击了，登不上去
- 医院的系统坏了，看不了病

这些都是可用性的问题。
最典型的就是DDoS攻击，
把网站打瘫痪，
让正常用户访问不了，
就是破坏可用性。

### 1.3.4 三者的关系

CIA这三个，
是网络安全最核心的三个目标。
所有的安全工作，
本质上都是围绕这三个目标展开的。

```
  保密性 —— 不泄露
  完整性 —— 不篡改
  可用性 —— 不瘫痪
```

这三个目标，
有时候是一致的，
有时候也会有矛盾。
比如：
- 你把系统搞的特别安全，谁都进不来，
  保密性上去了，
  但可用性就差了，
  正常用户用着也费劲。
- 你为了方便，怎么方便怎么来，
  可用性上去了，
  但安全性就差了，
  保密性完整性都没保障。

所以安全是一个平衡的艺术，
不是越安全越好，
而是要在安全和便利之间找一个平衡点。
这个我们后面还会讲。

除了CIA三元组，
后来还有人加了其他几个，
比如不可否认性、真实性、可审计性什么的，
但最核心的还是这三个。
先把这三个记住就行。

## 1.4 网络安全的重要性

为什么要学网络安全？
为什么网络安全这么重要？

我给你讲几个方面。

### 1.4.1 对个人的重要性

对我们普通人来说，
网络安全重要吗？
太重要了。

```
你可能遇到的安全问题：
  • 微信、QQ、游戏号被盗
  • 手机电脑中病毒、变肉鸡
  • 银行卡被盗刷
  • 个人隐私泄露（照片、聊天记录、位置...）
  • 接到各种诈骗电话、垃圾短信
  • 被钓鱼、被诈骗
  • 身份证信息被冒用
  • ......
```

这些事，
相信很多人都遇到过，
或者身边有人遇到过。
网络安全跟我们每个人都有关系，
不是说你不是搞IT的就跟你没关系。
只要你用手机、用电脑、上网，
就跟网络安全有关。

学点网络安全知识，
至少能让你自己少上当、少踩坑，
保护好自己的隐私和财产。

### 1.4.2 对企业的重要性

对企业来说，
网络安全就更重要了。

```
企业面临的安全风险：
  • 官网被黑、业务系统瘫痪
  • 数据被偷、客户信息泄露
  • 被勒索病毒加密，付赎金
  • 内部员工泄密
  • 被竞争对手攻击
  • 业务中断，损失惨重
  • 合规问题，被罚款
  • 品牌声誉受损
  • 严重的直接倒闭
```

现在企业越来越数字化，
所有业务都跑在网络上、系统里。
系统一瘫痪，
业务就停了。
数据一泄露，
就可能赔个底朝天。
前面讲的那个财务被骗的故事，
就是典型的例子。

所以现在企业越来越重视安全，
对安全人才的需求也越来越大。
这也是为什么越来越多人想入行安全的原因——
需求大，工资高。

### 1.4.3 对国家的重要性

往大了说，
网络安全对国家也至关重要。

现在是信息时代，
电力、交通、金融、能源、医疗...
关键基础设施都靠网络和系统。
如果这些被攻击了，
那后果不堪设想。
- 电网被攻击，大面积停电
- 金融系统被攻击，钱没了
- 交通系统被攻击，飞机火车跑不了
- 医疗系统被攻击，看不了病

所以网络安全已经上升到了国家安全的层面，
是国家安全的重要组成部分。
没有网络安全，
就没有国家安全。

## 1.5 网络安全都学什么？

很多新手想入行，
但不知道网络安全都学什么，
感觉东西特别多，
不知道从哪下手。

我给你梳理一下，
网络安全大概有哪些方向、哪些内容。
你先有个整体的印象。

### 1.5.1 几个大的方向

网络安全细分方向很多，
但大的方向可以分成这么几类：

```
  1. 攻击方向（红队 / 渗透测试）
     • 就是想办法打进去，找漏洞
     • 渗透测试工程师、红队队员、漏洞研究员
     • 很多新手觉得这个酷，想做这个

  2. 防守方向（蓝队 / 安全运营）
     • 防守、监控、应急响应
     • 安全运营工程师、SOC分析员、应急响应
     • 企业里需求量最大的方向

  3. 管理方向（安全管理 / 合规）
     • 安全体系建设、风险管理、合规
     • 安全经理、CISO、等保测评
     • 偏管理，技术要求相对低一点

  4. 研究方向
     • 漏洞研究、恶意代码分析、密码学...
     • 研究员、算法工程师
     • 技术要求高，门槛高

  5. 其他
     • Web安全、移动安全、云安全、
       工控安全、数据安全、IoT安全...
     • 按领域分的各种方向
```

方向很多，
不用一开始就确定方向，
可以先都了解一下，
看看自己对哪个感兴趣、
适合哪个。

### 1.5.2 需要学什么知识？

那学网络安全都要学些什么呢？
我给你列一下，
你心里有个数。

```
基础部分：
  • 计算机基础 —— 电脑是怎么工作的
  • 网络基础 —— TCP/IP、各种协议
  • 操作系统 —— Windows、Linux
  • 编程基础 —— Python、Shell、C/C++
  • 数据库基础
  • Web基础 —— HTML、JavaScript、PHP/Java

安全部分：
  • 密码学基础
  • 常见攻击手段
  • 常见防护手段
  • 安全工具使用
  • 渗透测试流程
  • 应急响应流程
  • 安全管理与法规
  • ......
```

看起来很多是不是？
别慌，
不用一下子全学会，
慢慢来。
而且也不是所有东西都要精通，
很多东西了解一下、知道怎么回事就行。
先打基础，
再慢慢深入。

## 1.6 网络安全的发展历程

简单聊聊网络安全的发展历史，
让你有个整体的时间概念。
不用记，
当个故事听就行。

### 1.6.1 早期：从计算机病毒开始

最早的安全问题，
大概是从计算机病毒开始的。
上世纪80、90年代，
那时候互联网还不普及，
病毒主要靠软盘、光盘传播。
最有名的比如"小球病毒"、"CIH病毒"。
那时候的病毒，
大多是搞破坏的，
删文件、毁系统，
很多是程序员搞恶作剧。

那时候的安全，
主要就是杀毒软件。

### 1.6.2 网络时代：漏洞与攻击

互联网普及之后，
安全问题就更多了。
2000年前后，
各种网络漏洞开始出现，
蠕虫病毒开始泛滥。
比如有名的"红色代码"、"SQL Slammer"、"冲击波"、"震荡波"...
一个漏洞出来，
全世界的电脑遭殃。

这时候开始有了防火墙、入侵检测，
也开始有了渗透测试、漏洞研究。
Web也开始普及，
SQL注入、XSS这些Web漏洞开始出现。

### 1.6.3 产业化：黑产与APT

大概从2010年前后开始，
网络攻击开始产业化、
组织化了。
出现了黑色产业链——
偷数据、骗钱、勒索、
卖漏洞、卖木马...
形成了完整的产业链，
赚钱效应越来越明显。

同时也出现了APT攻击，
就是高级持续性威胁，
国家级的、有组织的攻击，
目标明确，技术高超，
能潜伏好几年。
比如著名的"震网"病毒，
就是国家级的网络武器。

这时候企业的安全投入越来越大，
安全行业也快速发展。

### 1.6.4 现在：万物互联，安全无处不在

到了现在，
云计算、大数据、物联网、
人工智能、5G...
新技术层出不穷，
攻击面也越来越大。
- 云安全成了热点
- 数据安全越来越受重视
- 工控安全、车联网安全、IoT安全...
  新的方向不断出现
- AI也开始用在攻防两端
- 法律法规也越来越完善

网络安全已经从一个小众领域，
变成了跟每个人、每个企业、
每个国家都息息相关的事情。

## 1.7 怎么学习网络安全？

最后给新手讲一讲，
怎么学习网络安全。
给你一些建议，
少走弯路。

### 1.7.1 先打基础，别急着学"黑客技术"

很多新手一上来就想学黑客技术、
想学渗透、想搞工具，
觉得那个酷。
但基础不牢，地动山摇。

**网络基础、操作系统、编程基础，
这些是地基。**
地基打牢了，
后面学什么都快。
地基没打好，
后面学的都是花架子，
看着厉害，
其实一戳就破。

所以别急，
慢慢来，
先把基础学好。

### 1.7.2 多动手，别光看不练

安全是一门实践性很强的学科，
光看书看视频没用，
必须动手做。
- 搭环境自己练
- 打CTF练手
- 做实验
- 玩靶机

看十遍不如自己做一遍。
动手做了，
你才能真正理解。

### 1.7.3 保持好奇心，多问为什么

学安全要有点黑客精神——
好奇心强，
喜欢钻研，
遇到问题就想搞明白。
- 这个东西是怎么工作的？
- 为什么会有这个漏洞？
- 如果我换一种方法会怎么样？

多问几个为什么，
你才能学得深、学得透。

### 1.7.4 遵纪守法，守住底线

这个我必须放在前面说。
学安全，
**技术越厉害，责任越大。**
你有了攻击的能力，
但绝对不能乱用。

- 没有授权，不要去碰别人的系统
- 不要去搞破坏、偷东西
- 不要做违法的事
- 用技术做有意义的事

技术是把双刃剑，
用对了是保护别人的白帽子，
用错了就是违法犯罪的黑帽子。
一念之差，
天壤之别。

这条底线，
一定要守住。

### 1.7.5 持续学习，永无止境

安全这个行业，
变化特别快。
新漏洞、新技术、新攻击方法，
层出不穷。
今天学的东西，
可能明年就过时了。

所以一定要**保持学习的习惯**，
不断更新自己的知识。
干安全这一行，
就是活到老学到老。

当然，
这也是这行有意思的地方——
永远有新东西可以学，
永远不会无聊。

## 1.8 本章总结

第一章就到这里。
这一章没讲什么技术，
主要是给你建立一个整体的认知。

总结一下：

**什么是网络安全：**
- 保护系统、网络、数据不被偷、不被改、不被搞坏
- 跟每个人都息息相关

**CIA三元组（核心目标）：**
- 保密性（Confidentiality）—— 不泄露
- 完整性（Integrity）—— 不篡改
- 可用性（Availability）—— 不瘫痪

**为什么重要：**
- 对个人：保护隐私和财产
- 对企业：保护业务和数据
- 对国家：国家安全的一部分

**学习方向：**
- 攻击（红队/渗透）
- 防守（蓝队/运营）
