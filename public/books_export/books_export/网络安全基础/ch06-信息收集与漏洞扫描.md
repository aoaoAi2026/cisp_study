## 6.1 先讲一个故事

可能来自电影——
噼里啪啦敲键盘，
屏幕上满是绿色的代码流，
几分钟就黑进了对方系统。

现实中是这样吗？
完全不是。

真实的黑客攻击，
**80%以上的时间都花在信息收集和分析上，
真正动手的时间可能不到20%。**

我给你讲个真实的例子。

有一次，
某公司请了个渗透测试团队，
对他们的系统做渗透测试，
给的信息只有一个公司名和官网域名。

你猜团队第一步干嘛？
不是去扫网站，
不是去找后台，
而是——
**先收集信息。**

他们干了什么？
- 查了公司的域名注册信息（whois）
- 子域名爆破，找到了几十个子域名
- 端口扫描，看看哪些端口开着
- 爬了官网所有页面，看看有什么信息
- 搜索引擎里搜公司的各种信息
- 招聘网站上看公司招什么技术栈的人
- GitHub上搜有没有员工不小心泄露的代码
- 社交平台上找员工信息，准备社工
- 等等...

收集了整整两天的信息，
最后呢？
在一个废弃的测试子站上，
发现了一个没关的phpinfo页面，
泄露了服务器的各种信息；
在GitHub上找到了一个员工的私人仓库，
里面有测试环境的账号密码；
通过招聘信息知道了用的什么技术栈，
针对性地找那个技术栈的漏洞。

最后，
不到一周的时间，
他们就从外网打到了内网核心。
而突破口，
全是信息收集阶段发现的。

## 6.2 信息收集概述

### 6.2.1 什么是信息收集？

什么是信息收集（Information Gathering）？
简单说就是：
**在攻击之前，
尽可能多地收集目标的各种信息。**

信息收集的目的是什么？
- 了解目标的网络架构
- 发现可能的攻击面
- 寻找潜在的漏洞点
- 为后续攻击做准备

信息收集越充分，
后面的攻击就越容易。
很多渗透测试的新手，
一上来就拿着工具瞎扫，
扫半天没结果，
就是因为信息收集没做好。

### 6.2.2 信息收集的分类

信息收集一般分两类：

```
  1. 被动信息收集（Passive）
     不直接和目标交互
     通过第三方的信息源来收集
     比如搜索引擎、公开数据库...
     优点：隐蔽，目标发现不了
     缺点：信息可能不够新、不够全

  2. 主动信息收集（Active）
     直接和目标进行交互
     比如端口扫描、漏洞扫描、目录爆破...
     优点：信息准确、实时
     缺点：容易被目标发现，留下痕迹
```

实际操作中，
一般是**先被动，后主动**。
先通过被动收集摸个大概，
心里有底了，
再做主动收集，
针对性地去扫，
效率更高，也更隐蔽。

### 6.2.3 信息收集的内容

信息收集都收集什么？
太多了：
```
  • 组织信息：
    公司名称、地址、电话、邮箱、
    组织结构、员工信息...

  • 网络信息：
    域名、子域名、IP地址段、
    网络拓扑、开放端口、运行的服务...

  • 系统信息：
    操作系统类型和版本、
    运行的软件和版本、
    系统配置...

  • 应用信息：
    Web应用类型、CMS类型、
    框架、中间件、
    用户名、邮箱、
    备份文件、测试页面...

  • 漏洞信息：
    已知的CVE漏洞、
    配置缺陷、弱口令...
```

只要是对攻击有用的，
什么都收集。
信息多多益善。

## 6.3 被动信息收集

先讲**被动信息收集**。
被动收集的好处是隐蔽，
目标完全发现不了你。
所以能被动收集到的，
就尽量被动收集。

### 6.3.1 搜索引擎

第一个，
**搜索引擎**。
别小看搜索引擎，
很多敏感信息都是通过搜索引擎搜到的。
用得好的话，
能挖到不少好东西。

最常用的就是Google，
各种Google Hacking语法，
可以精准搜索。

比如：
```
  site:xxx.com       只搜某个域名下的内容
  inurl:login        URL里包含login的页面
  intext:后台        页面内容里有"后台"的
  filetype:pdf       只搜PDF文件
  intitle:index of   目录浏览
  "powered by xxx"   用了某个CMS的
  ...等等
```

把这些语法组合起来，
能搜到很多有意思的东西：
- 后台登录页面
- 未公开的测试环境
- 泄露的文档和备份
- 员工的邮箱和联系方式
- 等等

除了Google，
国内的百度、必应也可以用。
还有一些专门的网络空间搜索引擎，
比如Shodan、ZoomEye、Fofa，
这些更厉害——
能搜联网的设备、
某个端口开了什么服务、
某个版本的软件有多少台机器...
非常强大。

### 6.3.2 Whois 查询

**Whois**是什么？
就是域名注册信息查询。
每个域名注册的时候，
都会留下注册人、注册邮箱、
注册商、注册日期、DNS服务器...
这些信息都是公开的，可以查到。

通过Whois信息，
你可以知道：
- 域名是谁注册的
- 注册邮箱是什么
- DNS服务器是哪个
- 域名什么时候过期
- 等等

虽然现在很多人开启了Whois隐私保护，
但还是能查到不少信息。
而且注册邮箱可能还关联着其他账号，
可以继续深挖。

### 6.3.3 子域名收集

**子域名收集**是信息收集中非常重要的一环。
很多时候，
主站防护得很好，
很难打进去；
但某个子域名可能是个测试站，
或者忘了关，
漏洞一大堆，
从子域名突破，
再想办法打到主站，
这是常见的套路。

怎么收集子域名？
方法很多：
```
  1. 搜索引擎
     site:xxx.com
     能搜到不少子域名

  2. DNS区域传输
     如果DNS配置不当，
     允许区域传输，
     那就能直接拿到所有子域名
     （不过现在很少有配置错的了）

  3. 子域名爆破
     拿一个常用的子命名字典（www、mail、
     test、dev、admin、api...）
     一个个去解析，
     解析成功的就是存在的
     常用工具：
       • subfinder
       • amass
       • 子域名挖掘机
       • OneForAll
       等等

  4. SSL证书
     证书里可能有Subject Alternative Name
     能看到相关的域名

  5. 第三方数据源
     比如VirusTotal、SecurityTrails
     这些平台收集了很多域名数据
     可以查

  6. JS文件
     爬网站的JS文件，
     里面可能藏着其他子域名的接口地址
```

子域名收集是个技术活，
方法越多，
收集得越全。
漏掉一个子域名，
可能就漏掉了一个突破口。

### 6.3.4 网络空间搜索引擎

前面提到了，
**网络空间搜索引擎**，
比如Shodan、ZoomEye、Fofa。
这些是什么？
简单说就是：
**它们爬遍了整个互联网，
把所有联网的设备、
开放的端口、运行的服务、
甚至设备的漏洞，
都收集起来，
让你可以搜索。**

比如你可以搜：
- "某个城市 端口22" —— 看看这个城市有多少台开了SSH的机器
- "Apache Struts2" —— 看看有多少台机器用了Struts2
- "port:3389 os:windows" —— 开了3389的Windows机器
- "fid:xxx" —— 某个特定的应用
- 等等

非常强大。
很多时候，
你甚至不用自己扫，
去这些平台搜一下，
目标的开放端口、运行的服务、
甚至漏洞，
都已经给你整理好了。

### 6.3.5 社交媒体与员工信息

被动收集还包括什么？
**社交媒体和员工信息。**

为什么要收集员工信息？
因为人是最大的安全弱点。
拿到了员工的邮箱、姓名、
甚至个人信息，
就可以做社会工程学攻击，
比如钓鱼邮件、撞库...

怎么收集？
- LinkedIn、脉脉等职业社交平台
  可以找到公司的员工列表、职位
- 微博、知乎、GitHub等平台
  看看员工有没有泄露什么信息
- 公司官网的团队介绍、新闻
  也会有员工信息
- 招聘网站
  看公司招什么人，用什么技术栈
  有时候招聘信息里还会泄露系统架构

GitHub尤其要注意，
很多员工会把公司的代码、配置文件、
甚至账号密码，
不小心传到自己的GitHub公开仓库里。
这种事情太常见了。
所以信息收集的时候，
GitHub搜索是必做的一步。

### 6.3.6 其他被动收集方法

还有很多其他的被动收集方法，
比如：
```
  • DNS记录查询
    A记录、MX记录、NS记录、TXT记录...
    能挖到不少信息

  • 网盘搜索
    百度网盘、微云...
    看看有没有公司泄露的内部资料

  • 文库平台
    百度文库、豆丁...
    有时候能找到公司的PPT、文档

  • 漏洞平台
    CNVD、CVE详情...
    看看目标有没有公开的漏洞

  • 证书透明度日志（CT Log）
    可以查到某个域名颁发过哪些证书
    从而发现子域名
```

被动信息收集的方法太多了，
关键在于思路要开阔，
各种公开的信息源都要利用起来。

## 6.4 主动信息收集

被动收集完了，
有了大概的了解，
接下来就可以做**主动信息收集**了。
主动收集就是直接和目标交互，
获取更准确、更全面的信息。

### 6.4.1 主机发现

主动收集的第一步，
**主机发现**——
看看哪些IP是活的，
哪些机器是开着的。

怎么发现？
```
  1. ICMP Ping
     最基础的，ping一下
     通了就是活的
     但很多服务器禁ping，
       所以ping不通不一定不活

  2. ARP扫描
    内网里用，发ARP请求
    看哪些IP有回应
    内网里很准

  3. 端口扫描（半开扫描）
     发个SYN包，
     只要有回应（不管是SYN+ACK还是RST），
     就说明主机是活的
     比ping靠谱，因为很多机器开着端口但禁ping

  4. 常用工具：
     • Nmap（最常用，功能强大）
     • Masscan（高速扫描，扫全网都很快）
     • fping、hping...
```

### 6.4.2 端口扫描

主机发现之后，
下一步就是**端口扫描**——
看看目标主机开了哪些端口，
跑了哪些服务。

端口是什么？
前面网络基础讲过，
这里就不重复了。
每个开放的端口，
都可能是一个攻击入口。

常见的端口和对应的服务，
你得心里有数：
```
  21    FTP
  22    SSH
  23    Telnet
  25    SMTP
  53    DNS
  80    HTTP
  110   POP3
  135   RPC
  139   NetBIOS
  143   IMAP
  443   HTTPS
  445   SMB
  3306  MySQL
  3389  RDP（Windows远程桌面）
  5432  PostgreSQL
  6379  Redis
  8080  HTTP代理/替代端口
  ......还有很多很多
```

端口扫描的原理是什么？
简单说就是：
**给目标的端口发个包，
看有没有回应，
来判断端口开没开。**

常见的扫描方式：
```
  • TCP全连接扫描
    完整的三次握手
    最准确，但慢，而且容易被发现

  • SYN半开扫描
    只发SYN包，收到SYN+ACK就说明开了
    不建立完整连接，比较隐蔽
    最常用的扫描方式

  • UDP扫描
    扫UDP端口的
    比TCP难扫，速度慢
    但很多服务跑在UDP上，
      比如DNS、SNMP...

  • FIN扫描、NULL扫描、XMAS扫描...
    更隐蔽的扫描方式
    利用TCP协议的各种特性
    用来躲避防火墙和IDS的检测
```

端口扫描最常用的工具就是 **Nmap**，
功能强大，几乎是标配。
后面我们会专门讲Nmap。

### 6.4.3 服务与版本识别

光知道开了哪个端口还不够，
还得知道**跑的是什么服务、什么版本**。
因为漏洞都是跟版本相关的——
某个版本有漏洞，
另一个版本就修好了。
知道了版本，
才能知道有没有漏洞可以利用。

怎么识别服务和版本？
```
  1. 端口默认对应
    比如80一般是HTTP，22一般是SSH
    但这不准确，因为人可以改端口

  2.  Banner抓取
    连接上去，看服务返回的Banner信息
    很多服务会返回自己的版本号
    比如SSH连接的时候会显示"OpenSSH 7.2p2..."

  3.  特征匹配
    给服务发各种探测包，
    看返回的特征，
    跟已知的服务特征比对
    Nmap的-sV就是干这个的
    有一个很大的特征库

  4.  Web应用识别
    对于Web服务，
    还要识别用的什么CMS、什么框架、什么中间件
    常用工具：
      • whatweb
      • Wappalyzer（浏览器插件）
      • 御剑指纹识别
      等等
```

### 6.4.4 操作系统识别

还有**操作系统识别**——
判断目标跑的是什么操作系统，
Windows还是Linux，
什么版本。

怎么识别？
```
  1. 端口特征
    比如开了135、139、445、3389，
    大概率是Windows
    开了22，可能是Linux/Unix

  2. TTL值
    ping返回的TTL值
    Windows一般是128
    Linux一般是64
    （当然这个不准，可以改）

  3. TCP/IP栈指纹
    不同操作系统的TCP/IP协议栈实现有细微差别
    比如窗口大小、DF位、TTL、分片...
    通过这些特征来判断操作系统
    Nmap的-O参数就是干这个的
    有个指纹库，准确率还挺高
```

知道了操作系统，
就可以针对性地找对应的漏洞和攻击方法。

### 6.4.5 Web信息收集

如果目标有Web应用，
那Web信息收集也很重要。

收集什么？
```
  1. 目录与文件扫描
     看看网站上有什么目录和文件
     比如后台、备份文件、phpinfo、
       测试页面、上传目录...
     常用工具：
       • dirsearch
       • gobuster
       • 御剑后台扫描
       • DirBuster
     原理就是拿字典爆破，
       一个个请求，
       根据返回状态码判断存在不存在

  2. CMS识别
    看看用的什么CMS（内容管理系统）
    比如WordPress、Drupal、Joomla、
      Discuz、织梦、帝国...
    不同的CMS有不同的漏洞
    识别出来就可以找对应的漏洞

  3. 中间件识别
    看看用的什么Web中间件
    Apache、Nginx、IIS、Tomcat、
      JBoss、WebLogic、WebSphere...
    这些中间件本身也经常有漏洞

  4. 框架识别
    看看用的什么开发框架
    ThinkPHP、Laravel、Spring、Struts2...
    框架漏洞这几年出了不少

  5. 参数爬取
    爬网站的所有链接和参数
    看看有哪些输入点
    后面测注入、XSS用
    工具：Burp Suite爬虫、AWVS爬虫...

  6. 敏感信息收集
    看看页面源代码里有没有注释、
      硬编码的账号密码、
      内部IP、API密钥...
    看看robots.txt里有没有敏感路径
    看看有没有备份文件（.bak、.swp、~...）
    看看有没有.git、.svn等版本控制文件泄露
    这些都是大杀器
```

Web信息收集的内容非常多，
也非常重要，
因为Web是最常见的攻击面。

## 6.5 神器 Nmap

讲主动信息收集，
不能不提 **Nmap**。
Nmap是啥？
Network Mapper，
网络映射器。
可以说是最著名、最常用的
网络安全扫描工具之一。
几乎是安全从业者的标配。

Nmap能干嘛？
- 主机发现
- 端口扫描
- 服务版本识别
- 操作系统识别
- 漏洞扫描（有脚本引擎）
- 网络拓扑探测
- ...功能非常多

### 6.5.1 常用的Nmap命令

给你介绍几个常用的Nmap命令，
新手常用的就那几个。

```bash
# 最简单的扫描，扫一个IP
nmap 192.168.1.1

# 扫一个网段
nmap 192.168.1.0/24

# 指定端口扫描
nmap -p 22,80,443,3306 192.168.1.1

# 扫全端口（1-65535）
nmap -p- 192.168.1.1

# SYN半开扫描（需要root权限）
nmap -sS 192.168.1.1

# 服务版本识别
nmap -sV 192.168.1.1

# 操作系统识别
nmap -O 192.168.1.1

# 综合扫描（常用，什么都干）
nmap -sV -O -sS 192.168.1.1

# 快速扫描
nmap -F 192.168.1.1

# 扫描完把结果存成文件
nmap -oN result.txt 192.168.1.1

# 用脚本扫描漏洞
nmap --script=vuln 192.168.1.1
```

Nmap的功能非常强大，
参数也非常多，
新手不用全记，
把常用的几个记牢就够了。
遇到复杂的需求再查文档。

### 6.5.2 Nmap脚本引擎（NSE）

Nmap还有个很强大的功能——
**NSE（Nmap Scripting Engine）**，
Nmap脚本引擎。
就是可以用Lua写脚本，
扩展Nmap的功能。

Nmap自带了一大堆脚本，
可以用来做：
- 漏洞探测
- 弱口令检测
- 服务枚举
- 后门检测
- 等等

比如：
```bash
# 扫漏洞
nmap --script=vuln 目标

# 扫认证相关的（弱口令之类的）
nmap --script=auth 目标

# 用所有脚本扫
nmap --script=all 目标

# 指定某个脚本
nmap --script=http-enum 目标
```

NSE非常强大，
用好了能省很多事。

## 6.6 漏洞扫描

信息收集得差不多了，
接下来就是**漏洞扫描**了。

### 6.6.1 什么是漏洞扫描？

什么是漏洞扫描？
简单说就是：
**用工具自动检测目标有没有已知的漏洞。**

漏洞扫描器有一个漏洞库，
里面存着各种已知漏洞的检测方法。
扫描的时候，
它一个个去试，
看看目标有没有这些漏洞。
有就报出来。

漏洞扫描的作用：
- 快速发现目标的已知漏洞
- 评估目标的安全状况
- 为漏洞修复和渗透测试提供方向

漏洞扫描是安全评估中最常用的手段之一，
效率高，成本低。

### 6.6.2 漏洞扫描的分类

漏洞扫描怎么分类？

```
  按扫描对象分：
    • 主机漏洞扫描
      扫操作系统、服务的漏洞
      比如Nessus、OpenVAS

    • Web漏洞扫描
      扫Web应用的漏洞
      SQL注入、XSS、文件上传...
      比如AWVS、AppScan、Burp Suite Scanner

    • 数据库漏洞扫描
    • 工控漏洞扫描
    • ...各种专项的

  按扫描方式分：
    • 主动扫描
      主动发包去探测
      最常见

    • 被动扫描
      不主动发包，
      通过监听流量来分析漏洞
      比较少用
```

### 6.6.3 常见漏洞扫描器

常见的漏洞扫描器有哪些？
给你介绍几个有名的。

**主机/综合类：**
```
  • Nessus
    最有名的商业漏洞扫描器之一
    功能强大，漏洞库全
    有免费版（Home），但功能有限
    企业版很贵

  • OpenVAS / Greenbone
    开源的漏洞扫描器
    Nessus的开源分支发展来的
    功能也很强大，免费
    中小企业用得很多

  • X-Scan
    国产的，老牌子了
    图形界面，简单易用
    弱口令扫描很强
    很多老安全人都用过

  • 漏洞扫描平台
    比如绿盟、启明星辰、深信服...
    国内厂商的商业产品
    功能全面，适合企业用
```

**Web应用类：**
```
  • AWVS（Acunetix Web Vulnerability Scanner）
    很有名的Web漏洞扫描器
    扫描速度快，准确率高
    支持登录扫描
    很多公司都在用

  • AppScan
    IBM的，也是老牌Web扫描器
    功能强大
    但比较重，速度慢一点

  • Burp Suite Scanner
    Burp Suite的扫描模块
    做Web渗透测试的基本都用Burp
    扫描功能也很强
    可以配合手动测试用

  • OWASP ZAP
    开源的Web扫描器
    OWASP出品，免费
    功能也不错
    适合预算有限的

  • Xray
    国内的，长亭的
    功能很强，速度快
    社区版免费，专业版收费
    现在用的人越来越多

  • 还有很多其他的...
```

漏洞扫描器很多，
各有各的特点。
实际工作中，
一般是多个扫描器配合着用，
因为每个扫描器的误报漏报都不一样，
多个一起扫，
交叉验证，
结果更全面。

### 6.6.4 漏洞扫描的流程

漏洞扫描大致是个什么流程？
```
  1. 确定扫描目标和范围
     扫什么？哪些IP？哪些域名？
     不能乱扫，扫错了要担责任的

  2. 选择扫描工具和策略
     用什么扫描器？扫哪些类型的漏洞？
     扫描强度多大？会不会影响业务？

  3. 配置扫描器
     配置目标、登录凭证、
     排除项、扫描策略...

  4. 执行扫描
     等着，扫描可能要跑很久

  5. 分析扫描结果
     扫描器扫出来的不一定全对
     有很多误报
     需要人工验证哪些是真的漏洞

  6. 输出报告
     整理漏洞，分级，给出修复建议
     输出扫描报告
```

记住一点：
**扫描器的结果仅供参考，
不能全信。**
扫描器有误报，
也有漏报。
真正有价值的漏洞，
很多是扫描器扫不出来的，
需要人工去发现。
扫描器是辅助，
人才是核心。

### 6.6.5 漏洞验证

扫描器扫出来的漏洞，
一定要**人工验证**，
确认是不是真的存在。
不然拿着误报去跟业务方说，
人家修复半天发现不是漏洞，
就很尴尬，也显得你不专业。

怎么验证？
- 按照漏洞的原理，
  手动去测试一下
- 看返回结果是不是真的符合漏洞特征
- 排除误报的可能

验证漏洞的时候要注意：
- 不要对生产系统造成影响
- 破坏性的漏洞（比如删库、写入）不要真的执行
- 点到为止，证明存在就行

## 6.7 漏洞分级与CVE

讲到漏洞，
就不得不提 **CVE** 和漏洞分级。

### 6.7.1 什么是CVE？

CVE是什么？
Common Vulnerabilities and Exposures，
通用漏洞披露。
简单说就是：
**给每个公开的漏洞分配一个唯一的编号，
方便大家交流和引用。**

CVE编号的格式是：
`CVE-年份-序号`
比如 CVE-2024-12345

有了CVE编号，
大家说漏洞的时候就不用描述半天了，
直接说CVE编号，
大家都知道说的是哪个漏洞。
非常方便。

CVE由MITRE公司维护，
是国际上最通用的漏洞编号体系。
国内也有自己的，
比如CNNVD（国家信息安全漏洞库）、
CNVD（国家信息安全漏洞共享平台）。

### 6.7.2 漏洞严重等级

漏洞怎么分级？
一般按严重程度分：
```
  严重（Critical） / 高危
    可以直接获取服务器权限、
    远程代码执行、SQL注入getshell...
    危害很大，需要立即修复

  高危（High）
    比较严重的漏洞，
    可能导致数据泄露、权限提升...
    需要尽快修复

  中危（Medium）
    有一定危害，
    但利用条件比较苛刻，
    或者危害有限
    安排时间修复

  低危（Low）
    危害比较小的漏洞
    比如信息泄露但信息不重要
    有空就修，不修问题也不大

  信息（Info）
    不算漏洞，就是一些信息
    比如开了什么服务、版本是什么
    了解一下就行
```

具体怎么评分？
国际上有个标准叫 **CVSS**
（Common Vulnerability Scoring System，
通用漏洞评分系统）。
它根据漏洞的可利用性、影响范围、
影响程度等多个维度，
给漏洞打一个0-10分的分数。
分数越高越严重。
现在最新的是CVSS 3.1。

CVSS评分比较客观，
很多企业用它来给漏洞分级，
确定修复优先级。

## 6.8 本章总结

这一章我们讲了信息收集与漏洞扫描，
内容比较多。
总结一下：

**信息收集概述：**
- 攻击的第一步，非常重要
- 分被动收集和主动收集
- 先被动，后主动

**被动信息收集：**
- 搜索引擎（Google Hacking、网络空间搜索引擎）
- Whois查询
- 子域名收集（多种方法、多种工具）
- 社交媒体与员工信息
- 各种公开数据源

**主动信息收集：**
- 主机发现（Ping、ARP、端口扫描）
- 端口扫描（TCP SYN、UDP、各种扫描方式）
- 服务与版本识别
- 操作系统识别
- Web信息收集（目录扫描、CMS识别、中间件、框架...）

**Nmap：**
- 最常用的网络扫描工具
- 主机发现、端口扫描、版本识别、系统识别
- NSE脚本引擎，功能强大

**漏洞扫描：**
- 自动检测已知漏洞
- 主机扫描器：Nessus、OpenVAS...
- Web扫描器：AWVS、AppScan、Burp、Xray...
- 扫描结果需要人工验证，有误差
- 扫描器是辅助，人才是核心

**CVE与漏洞分级：**
- CVE：通用漏洞编号，唯一标识
- 漏洞分级：严重、高危、中危、低危、信息
- CVSS：通用漏洞评分系统，0-10分

最后送你一句话：

> **信息收集是攻击的起点，
> 也是防守的起点。
> 知道敌人会怎么收集你的信息，
> 你才知道怎么去隐藏、怎么去防护。**
