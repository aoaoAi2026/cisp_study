# 第九章 中间件安全加固

## 9.1 中间件安全概述

中间件是连接操作系统和应用程序的桥梁，包括Web应用服务器、消息队列、缓存等。中间件的安全漏洞往往直接导致服务器被接管。

### 9.1.1 常见中间件安全风险

```

┌─────────────────────────────────────────────┐
│         常见中间件及典型漏洞                  │
├─────────────────────────────────────────────┤
│ Tomcat                                      │

│ ├── Manager/Host Manager未授权访问           │
```

## 9.2 Tomcat安全加固

### 9.2.1 版本安全

```bash
# 查看Tomcat版本
./bin/version.sh
# 或访问：http://host:8080/（默认页面会显示版本）

# 安全版本选择原则：
# - 使用稳定版，不用Beta版
# - 关注CVE漏洞，及时升级
# - 至少使用Tomcat 8.5+，推荐9.x或10.x
# - 禁用所有示例应用和管理应用
```

```xml

      <url-pattern>*.old</url-pattern>
    </web-resource-collection>

  <!-- 禁止HTTP方法 -->
  <security-constraint>

      <http-method>OPTIONS</http-method>

  </security-constraint>
```

### 9.2.6 运行安全

```bash
# 1. 使用专用低权限用户运行Tomcat
useradd -r -s /sbin/nologin tomcat
chown -R tomcat:tomcat /opt/tomcat
chmod -R 700 /opt/tomcat/conf
chmod -R 750 /opt/tomcat/bin
chmod -R 750 /opt/tomcat/logs
chmod -R 750 /opt/tomcat/temp
chmod -R 750 /opt/tomcat/work

# 2. 限制文件权限
chmod 600 /opt/tomcat/conf/tomcat-users.xml
chmod 600 /opt/tomcat/conf/server.xml

# 3. JVM安全参数
# bin/setenv.sh
export JAVA_OPTS="$JAVA_OPTS -Djava.security.manager"
export JAVA_OPTS="$JAVA_OPTS -Djava.security.policy=conf/catalina.policy"
export JAVA_OPTS="$JAVA_OPTS -Xms512m -Xmx1024m"
export JAVA_OPTS="$JAVA_OPTS -XX:+DisableExplicitGC"
export JAVA_OPTS="$JAVA_OPTS -Dfile.encoding=UTF-8"
export JAVA_OPTS="$JAVA_OPTS -Duser.timezone=Asia/Shanghai"
# 禁用JMX远程（如不需要）
export JAVA_OPTS="$JAVA_OPTS -Dcom.sun.management.jmxremote=false"

# 4. 防火墙限制
firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="192.168.1.0/24" port port=8080 protocol=tcp accept'
firewall-cmd --reload
```

## 9.4 消息队列安全加固

### 9.4.1 RabbitMQ安全加固

```bash
# 1. 删除默认用户
rabbitmqctl delete_user guest

# 2. 创建管理员用户
rabbitmqctl add_user admin Strong_RabbitMQ_Password!2024
rabbitmqctl set_user_tags admin administrator

# 3. 创建应用用户
rabbitmqctl add_user app_user App_User_Password!2024
rabbitmqctl set_user_tags app_user

# 4. 设置虚拟主机权限
rabbitmqctl add_vhost /app
rabbitmqctl set_permissions -p /app app_user ".*" ".*" ".*"

# 5. 启用插件
# 管理插件（需要时启用，且限制访问IP）
rabbitmq-plugins enable rabbitmq_management

# 6. 配置文件
# /etc/rabbitmq/rabbitmq.conf
cat > /etc/rabbitmq/rabbitmq.conf << 'EOF'
# 监听地址
listeners.tcp.default = 127.0.0.1:5672

# 管理界面端口
management.tcp.port = 15672
management.tcp.ip = 127.0.0.1

# SSL配置

# ssl_options.certfile = /path/to/server_certificate.pem
# ssl_options.keyfile = /path/to/server_key.pem
```

### 9.4.2 Kafka安全加固

```properties
# server.properties

# ===== 认证配置 =====
# SASL认证
listeners=SASL_PLAINTEXT://192.168.1.100:9092
security.inter.broker.protocol=SASL_PLAINTEXT
sasl.mechanism.inter.broker.protocol=PLAIN
sasl.enabled.mechanisms=PLAIN

# kafka_server_jaas.conf

    org.apache.kafka.common.security.plain.PlainLoginModule required
    username="admin"
    password="admin_password"
    user_admin="admin_password"
    user_producer="producer_password"
    user_consumer="consumer_password";
};

# 启用ACL
authorizer.class.name=kafka.security.authorizer.AclAuthorizer
super.users=User:admin

# ===== 网络安全 =====
# 监听内网地址
listeners=PLAINTEXT://192.168.1.100:9092

# ===== SSL/TLS传输加密 =====
# listeners=SSL://192.168.1.100:9093
# ssl.keystore.location=/path/to/kafka.keystore.jks
# ssl.keystore.password=keystore_password
# ssl.key.password=key_password
# ssl.truststore.location=/path/to/kafka.truststore.jks
# ssl.truststore.password=truststore_password
# ssl.client.auth=required

# ===== ZooKeeper安全 =====
zookeeper.connect=192.168.1.100:2181
# zookeeper.set.acl=true
```

```bash
# Kafka ACL示例
# 创建主题权限
kafka-acls --authorizer-properties zookeeper.connect=localhost:2181 \
    --add \
    --allow-principal User:producer \
    --operation Write \
    --topic my-topic

# 消费权限
kafka-acls --authorizer-properties zookeeper.connect=localhost:2181 \
    --add \
    --allow-principal User:consumer \
    --operation Read \
    --topic my-topic \
    --group my-group
```

## 9.5 中间件加固通用原则

```
中间件加固通用原则：

1. 版本管理
   ├── 使用稳定版本，及时升级
   ├── 关注CVE漏洞公告
   ├── 建立版本升级流程
   └── 测试环境验证后再升级

2. 身份认证
   ├── 删除默认账户
   ├── 强密码策略
   ├── 最小权限分配
   ├── 多因素认证（核心系统）
   └── 定期更换密码

3. 网络安全
   ├── 监听内网地址，不暴露公网
   ├── 防火墙限制访问源
   ├── 启用SSL/TLS加密传输
   ├── 修改默认端口（辅助）
   └── VPN/专线访问管理后台

4. 配置加固
   ├── 禁用不必要的功能和组件
   ├── 删除示例和测试代码
   ├── 隐藏版本信息
   ├── 禁用危险的HTTP方法
   ├── 安全响应头
   └── 文件权限最小化

5. 监控审计
   ├── 访问日志开启
   ├── 操作日志记录
   ├── 日志集中收集
   ├── 异常告警
   └── 定期安全审计

6. 运行安全
   ├── 专用低权限用户运行
   ├── 进程资源限制
   ├── 禁止root运行
   ├── 文件系统隔离
   └── 容器化部署（可选）
```
