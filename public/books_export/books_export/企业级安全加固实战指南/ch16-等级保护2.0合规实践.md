# 第十六章 等级保护2.0合规实践

## 16.1 等级保护2.0概述

### 16.1.1 等保2.0背景

等级保护制度是我国网络安全的基本制度，2019年正式发布的等保2.0标准相比等保1.0有了重大变化，覆盖范围更广，要求更严格。

```
等保发展历程：

等保1.0（2007-2019）
├── 主要覆盖：基础信息网络、信息系统
├── 标准体系：GB/T 22239-2008
├── 五个安全级别
├── 技术+管理五大类要求
└── 侧重：防护为主

等保2.0（2019至今）
├── 覆盖扩展：云计算、移动互联、物联网、工业控制、大数据
├── 标准体系：GB/T 22239-2019
├── 一个中心，三重防护
├── 主动防御、动态感知、持续监测
└── 侧重：全方位、全周期、主动防御
```

### 16.1.2 等级划分

```
信息系统安全等级：

第一级（自主保护级）
- 一般信息系统
- 受到破坏后，对公民、法人和其他组织的合法权益造成损害
- 但不损害国家安全、社会秩序和公共利益
- 运营、使用单位自主保护

第二级（指导保护级）
- 一般信息系统
- 受到破坏后，对公民、法人和其他组织的合法权益产生严重损害
- 或对社会秩序和公共利益造成损害，但不损害国家安全
- 国家信息安全监管部门指导

第三级（监督保护级）
- 重要信息系统
- 受到破坏后，对社会秩序和公共利益造成严重损害
- 或对国家安全造成损害
- 国家信息安全监管部门监督检查

第四级（强制保护级）
- 重要信息系统
- 受到破坏后，对社会秩序和公共利益造成特别严重损害
- 或对国家安全造成严重损害
- 国家信息安全监管部门强制监督检查

第五级（专控保护级）
- 特殊重要信息系统
- 受到破坏后，对国家安全造成特别严重损害
- 国家指定的专门部门、专门机构进行专门监督
```

### 16.1.3 等保2.0核心要求

```
等保2.0核心框架：一个中心，三重防护

安全管理中心
├── 系统管理
├── 审计管理
├── 安全管理
└── 集中管控

安全区域边界
├── 边界防护
├── 访问控制
├── 入侵防范
├── 安全审计
└── 可信验证

安全通信网络
├── 网络架构
├── 通信传输
└── 可信验证

安全计算环境
├── 身份鉴别
├── 访问控制
├── 安全审计
├── 入侵防范
├── 恶意代码防范
├── 可信验证
├── 数据完整性
├── 数据保密性
├── 数据备份恢复
└── 剩余信息保护

安全管理中心 + 三重防护 + 管理要求（五个层面）
= 等保2.0完整体系
```

## 16.2 安全通信网络

### 16.2.1 网络架构

```
三级等保网络架构要求：

□ 网络架构设计
  □ 划分不同的网络区域
  □ 重要网段部署在网络边界内部
  □ 重要网段与其他网段有效隔离
  □ 网络设备业务容量满足峰值需求
  □ 核心网络设备、线路冗余
  □ 网络拓扑图与实际一致

□ 网络设备安全
  □ 主要网络设备启用SSH等安全远程管理
  □ 网络设备管理员身份鉴别
  □ 网络设备配置定期备份
  □ 网络设备及时更新补丁

□ VLAN划分
  □ 业务区、管理区、DMZ区VLAN划分
  □ VLAN间通过三层设备访问控制
  □ 禁止VLAN跳跃攻击
```

### 16.2.2 通信传输

```
通信传输安全要求：

□ 传输完整性
  □ 重要数据传输采用校验码技术
  □ 或采用密码技术保证完整性
  □ 检测到数据完整性被破坏后能恢复

□ 传输保密性
  □ 重要数据传输采用加密技术
  □ 加密算法符合国家密码管理规定
  □ 密钥管理安全

□ 通信双方认证
  □ 通信双方建立会话时验证对方身份
  □ 采用密码技术进行身份鉴别
  □ 防止会话劫持

示例配置：
- 全站HTTPS加密传输
- IPSec VPN加密通道
- SSH加密管理
- 数据库连接SSL加密
- API通信TLS加密
```

## 16.3 安全区域边界

### 16.3.1 边界防护

```
边界防护要求：

□ 边界设备部署
  □ 不同安全区域边界部署访问控制设备
  □ 内外网边界部署防火墙
  □ 重要区域边界部署入侵防御系统

□ 访问控制
  □ 防火墙默认拒绝，按需放行
  □ 访问控制规则最小化
  □ 规则源/目的/端口/协议明确
  □ 定期清理无效规则

□ 入侵防范
  □ 边界部署IDS/IPS
  □ 检测并阻断常见网络攻击
  □ 攻击特征库定期更新
  □ 攻击事件告警

□ 恶意代码防范
  □ 边界部署防病毒网关
  □ 检测并阻断恶意代码
  □ 病毒库定期更新

□ 边界完整性检查
  □ 检测非法内连外连
  □ 发现非法外联告警
  □ 无线接入管控
```

  □ 事件日期和时间
  □ 事件类型
  □ 事件源/目的IP
  □ 事件结果（成功/失败）
  □ 操作用户

□ 审计保护
  □ 审计记录定期备份
  □ 审计记录未被篡改
  □ 审计日志保留至少6个月
  □ 审计日志集中存储

□ 审计分析
  □ 定期审计分析
  □ 异常行为发现
  □ 安全事件追溯
```

## 16.4 安全计算环境

### 16.4.1 身份鉴别

```
身份鉴别三级要求：

□ 唯一性
  □ 每个用户有唯一标识
  □ 不共享账号
  □ 匿名/默认账号禁用

□ 复杂度
  □ 口令长度≥8位
  □ 大小写+数字+特殊字符
  □ 口令90天更换
  □ 不能与最近5次重复
  □ 不允许空口令

□ 失败处理
  □ 登录失败次数限制（5次）
  □ 登录失败后锁定账号（30分钟）
  □ 登录失败告警

□ 双因素认证
  □ 重要系统采用两种以上鉴别技术
  □ 口令+令牌/证书/生物特征
  □ 远程管理必须双因素

□ 其他
  □ 登录超时自动退出（10分钟）
  □ 鉴别信息加密存储
  □ 防止重放攻击
```

访问控制三级要求：

  □ 默认权限最小化
  □ 重要资源访问控制

□ 默认账户处理

  □ 修改默认口令
  □ 重命名管理员账户

### 16.4.4 入侵防范与恶意代码防范

```
入侵防范三级要求：

□ 系统入侵防范

  □ 及时安装系统补丁

  ├── 现场勘查

  └── 整改建议

第三阶段：安全整改
  ├── 技术整改
  │   ├── 网络架构调整
  │   ├── 安全设备部署
  │   ├── 系统加固

## 16.8 常见不达标项与整改建议

```
常见不达标项与整改建议：

1. 身份鉴别类
   □ 问题：密码策略不符合要求
   整改：配置密码复杂度、有效期、历史记录
   
   □ 问题：没有登录失败锁定
   整改：配置pam_tally2/fail2ban
   
   □ 问题：双因素认证未启用
   整改：部署动态口令/证书认证

2. 访问控制类
   □ 问题：默认账户未禁用
   整改：删除/禁用默认账户
   
   □ 问题：权限过大
   整改：遵循最小权限，拆分权限

3. 安全审计类
   □ 问题：审计日志不完整
   整改：开启审计功能，补充审计规则
   
   □ 问题：日志保留不足6个月
   整改：扩大日志存储，集中日志系统

4. 入侵防范类
   □ 问题：系统未打补丁
   整改：建立补丁管理机制，定期更新
   
   □ 问题：未部署防病毒
   整改：安装杀毒软件，定期更新病毒库

5. 数据安全类
   □ 问题：敏感数据明文存储
   整改：加密存储敏感数据
   
   □ 问题：无异地备份
   整改：建立异地备份机制

6. 管理类
   □ 问题：制度不健全
   整改：完善安全管理制度体系
   
   □ 问题：人员未培训
   整改：定期安全培训，记录保存

7. 边界防护类
   □ 问题：防火墙规则过于宽松
   整改：最小化原则，清理冗余规则
   
   □ 问题：无入侵检测
   整改：部署IDS/IPS设备
```
