# 第八章 数据库安全加固

## 8.1 数据库安全概述

数据库是企业核心数据的存储地，一旦被攻破，将造成最严重的数据泄露。数据库安全加固是纵深防御中至关重要的一环。

### 8.1.1 数据库安全风险

```
数据库安全威胁全景：
┌─────────────────────────────────────┐
│        数据库攻击面                 │
├─────────────────────────────────────┤
│ 身份认证风险                        │
│ ├── 弱口令/默认口令                 │
│ ├── 暴力破解                        │
│ ├── 凭据泄露                        │

│ 配置风险                            │

├─────────────────────────────────────┤
│ 注入攻击                            │

```

## 8.2 MySQL安全加固

### 8.2.1 安装与初始化加固

```bash
# 安装MySQL（以RHEL/CentOS为例）
yum install mysql-server -y

# 启动MySQL
systemctl start mysqld
systemctl enable mysqld

# 查找初始密码
grep 'temporary password' /var/log/mysqld.log

# 运行安全初始化脚本（必须！）
mysql_secure_installation
```

**mysql_secure_installation 会做以下加固：**
```
1. 设置root密码（强密码）
2. 删除匿名用户
3. 禁止root远程登录
4. 删除test数据库
5. 重新加载权限表
```

### 8.2.3 账户与权限安全

```sql
-- ===== 账户检查 =====

-- 查看所有用户
SELECT user, host, authentication_string, plugin FROM mysql.user;

-- 查看匿名用户
SELECT user, host FROM mysql.user WHERE user = '';

-- 删除匿名用户
DROP USER ''@'localhost';
DROP USER ''@'%';

-- 查看有超级权限的用户
SELECT user, host FROM mysql.user WHERE Super_priv = 'Y';

-- 查看可从任意主机登录的用户
SELECT user, host FROM mysql.user WHERE host = '%';

-- ===== root账户加固 =====

-- 禁止root远程登录
UPDATE mysql.user SET host = 'localhost' WHERE user = 'root';
-- 或直接删除远程root用户
DROP USER 'root'@'%';

-- 修改root密码
ALTER USER 'root'@'localhost' IDENTIFIED BY 'Strong_Password!2024';

-- ===== 创建专用应用账户 =====

-- 创建应用用户（最小权限）
CREATE USER 'app_user'@'192.168.1.%' IDENTIFIED BY 'App_Strong_Pwd!2024';

-- 只授予特定库的读写权限
GRANT SELECT, INSERT, UPDATE, DELETE ON appdb.* TO 'app_user'@'192.168.1.%';

-- 撤销不必要的权限
REVOKE FILE, SUPER, PROCESS ON *.* FROM 'app_user'@'192.168.1.%';

-- 创建只读用户（数据分析等）
CREATE USER 'readonly'@'192.168.1.%' IDENTIFIED BY 'ReadOnly_Pwd!2024';
GRANT SELECT ON appdb.* TO 'readonly'@'192.168.1.%';

-- 创建备份用户（备份专用）
CREATE USER 'backup'@'localhost' IDENTIFIED BY 'Backup_Pwd!2024';
GRANT SELECT, LOCK TABLES, SHOW VIEW, EVENT, TRIGGER ON *.* TO 'backup'@'localhost';

-- ===== 权限审计 =====

-- 查看用户权限
SHOW GRANTS FOR 'app_user'@'192.168.1.%';

-- 查看所有用户权限
SELECT * FROM mysql.user\G
SELECT * FROM mysql.db;
SELECT * FROM mysql.tables_priv;
SELECT * FROM mysql.columns_priv;

-- 刷新权限
FLUSH PRIVILEGES;

-- ===== 危险权限检查 =====

-- FILE权限（可读写文件）
SELECT user, host FROM mysql.user WHERE File_priv = 'Y';

-- SUPER权限（超级管理员）
SELECT user, host FROM mysql.user WHERE Super_priv = 'Y';

-- PROCESS权限（可查看所有进程）
SELECT user, host FROM mysql.user WHERE Process_priv = 'Y';

-- CREATE ROUTINE/ALTER ROUTINE（存储过程）
SELECT user, host FROM mysql.user WHERE Create_routine_priv = 'Y';

-- GRANT OPTION（可授权他人）
SELECT user, host FROM mysql.user WHERE Grant_priv = 'Y';
```

### 8.2.6 数据库对象安全

```sql
-- ===== 删除测试库 =====
DROP DATABASE IF EXISTS test;

-- ===== 移除对test库的默认权限 =====
DELETE FROM mysql.db WHERE Db LIKE 'test%' OR Db LIKE 'test\_%';
FLUSH PRIVILEGES;

-- ===== 存储过程安全 =====

-- 存储过程使用SQL SECURITY DEFINER时要注意权限
DELIMITER //
CREATE PROCEDURE GetUser(IN p_id INT)
SQL SECURITY INVOKER  -- 使用调用者权限，而不是定义者权限
BEGIN
    SELECT id, username, email FROM users WHERE id = p_id;
END //
DELIMITER ;

-- ===== 视图安全 =====
-- 使用视图限制用户能看到的列
CREATE VIEW v_users_public AS
SELECT id, username, email
FROM users;

GRANT SELECT ON appdb.v_users_public TO 'readonly'@'192.168.1.%';

-- ===== 触发器安全 =====
-- 审核触发器的使用
SELECT trigger_name, event_manipulation, trigger_body
FROM information_schema.triggers
WHERE trigger_schema = 'appdb';
```

### 8.2.7 数据加密

```sql
-- ===== 表空间加密（MySQL 5.7+，InnoDB） =====

-- 创建加密表
CREATE TABLE sensitive_data (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    id_card VARCHAR(20),
    phone VARCHAR(20),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB ENCRYPTION='Y';

-- 对已有表启用加密
ALTER TABLE sensitive_data ENCRYPTION='Y';

-- ===== 列级加密 =====
-- 插入时加密
INSERT INTO users (username, password) 
VALUES ('testuser', SHA2('password123', 256));

-- 使用AES加密
INSERT INTO sensitive_data (user_id, id_card)
VALUES (1, AES_ENCRYPT('110101199001011234', 'encryption_key'));

-- 解密查询
SELECT user_id, AES_DECRYPT(id_card, 'encryption_key') as id_card
FROM sensitive_data;

-- ===== 哈希存储（不可逆） =====
-- 密码使用bcrypt或argon2（推荐在应用层实现）
-- MySQL内置函数：SHA2, MD5（不推荐，不够安全）
```

## 8.3 Redis安全加固

### 8.3.1 配置文件加固

```bash
# /etc/redis/redis.conf

# ===== 网络安全 =====
# 绑定地址（只监听本地或内网，不要监听所有接口）
bind 127.0.0.1
# bind 127.0.0.1 192.168.1.100

# 端口（可修改默认端口）
port 6379

# 禁用TCP keepalive检测
# tcp-keepalive 60

# ===== 认证安全 =====
# 设置密码（强密码！）
requirepass Strong_Redis_Password_2024!

# 重命名危险命令
rename-command CONFIG ""
rename-command FLUSHDB ""
rename-command FLUSHALL ""
rename-command KEYS ""
rename-command EVAL ""
rename-command SCRIPT ""
rename-command SHUTDOWN ""
rename-command DEBUG ""
# 或重命名为复杂名称
# rename-command CONFIG my_secret_config_cmd

# ===== 持久化安全 =====
# RDB持久化
save 900 1
save 300 10
save 60 10000
dbfilename dump.rdb
dir /var/lib/redis

# AOF持久化
appendonly yes
appendfilename "appendonly.aof"
appendfsync everysec

# ===== 权限与安全 =====
# 以非root用户运行redis（通过systemd控制）
# 配置文件中无需设置，通过启动用户控制

# 保护模式（当没有绑定地址或没有设置密码时，只允许本地访问）
protected-mode yes

# ===== 内存管理 =====
# 最大内存（根据实际情况设置）
maxmemory 256mb
# 内存淘汰策略
maxmemory-policy allkeys-lru

# ===== 客户端限制 =====
# 最大客户端连接数
maxclients 1000

# 客户端超时（秒）
timeout 300

# ===== 禁用危险功能 =====
# 禁用Lua脚本（如不需要）
# lua-time-limit 0

# 禁止执行特定命令
# 已通过rename-command实现
```

### 8.3.2 操作系统层防护

```bash
# Redis专用用户
useradd -r -s /sbin/nologin redis

# Redis数据目录权限
chown -R redis:redis /var/lib/redis
chmod 700 /var/lib/redis
chmod 600 /var/lib/redis/dump.rdb
chmod 600 /var/lib/redis/appendonly.aof

# 配置文件权限
chown redis:root /etc/redis/redis.conf
chmod 640 /etc/redis/redis.conf

# 防火墙限制访问
# 只允许应用服务器访问Redis
firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="192.168.1.0/24" port port=6379 protocol=tcp accept'
firewall-cmd --reload

# 检查Redis是否以root运行（不应该）
ps aux | grep redis
```

    user: "app_user",
    pwd: "App_User_Password!2024",

    ]
})

db.createUser({
    user: "readonly",
    pwd: "ReadOnly_Password!2024",
    roles: [
        { role: "read", db: "appdb" }
    ]
