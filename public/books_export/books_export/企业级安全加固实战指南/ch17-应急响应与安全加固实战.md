# 第十七章 应急响应与安全加固实战

## 17.1 应急响应概述

应急响应是安全体系的最后一道防线，当安全事件发生时，快速、有序的响应能够最大程度减少损失。本章从实战角度介绍常见安全事件的应急处理与加固方法。

### 17.1.1 应急响应流程

```
PDCERF应急响应六阶段：

1. 准备阶段（Preparation）
   ├── 应急预案编制
   ├── 应急团队组建
   ├── 工具准备
   ├── 演练培训
   └── 备份机制

2. 检测阶段（Detection）
   ├── 异常发现
   ├── 事件确认
   ├── 初步判断
   ├── 影响评估
   └── 事件分级

3. 遏制阶段（Containment）
   ├── 隔离受感染主机
   ├── 切断攻击路径
   ├── 防止扩散
   ├── 保护证据
   └── 最小化影响

4. 根除阶段（Eradication）
   ├── 清除恶意代码
   ├── 修复漏洞
   ├── 删除后门账户
   ├── 清理恶意配置
   └── 系统加固

5. 恢复阶段（Recovery）
   ├── 系统恢复
   ├── 业务验证
   ├── 监控观察
   ├── 用户通知
   └── 恢复正常

6. 总结阶段（Follow-up）
   ├── 事件复盘
   ├── 原因分析
   ├── 改进措施
   ├── 经验总结
   └── 预案更新
```

### 17.1.2 安全事件分级

```
安全事件分级（按严重程度）：

特别重大事件（I级）
├── 核心业务系统全面瘫痪
├── 大量敏感数据泄露
├── 造成重大经济损失
├── 社会影响恶劣
└── 需要启动最高级别响应

重大事件（II级）
├── 重要系统部分瘫痪
├── 部分敏感数据泄露
├── 较大经济损失
└── 需要高级别响应

较大事件（III级）
├── 单个系统受影响
├── 少量数据泄露
├── 一般经济损失
└── 标准响应流程

一般事件（IV级）
├── 非核心系统受影响
├── 无数据泄露
├── 较小影响
└── 日常处理
```

## 17.2 入侵排查实战

### 17.2.1 快速排查清单

```
Linux入侵快速排查清单：

第一分钟：检查系统状态
├── 系统负载：w, uptime, top
├── 网络连接：netstat -tulnp, ss -tulnp
├── 登录用户：who, w, last
├── 进程异常：ps aux, top
└── 系统时间：date, hwclock

第五分钟：检查账户异常
├── 新增账户：cat /etc/passwd
├── 特权账户：awk -F: '$3==0' /etc/passwd
├── 空密码账户：awk -F: '$2==""' /etc/shadow
├── sudo权限：cat /etc/sudoers
└── 最近登录：last, lastb

第十分钟：检查进程与服务
├── 异常进程：ps aux --forest
├── 隐藏进程：ls /proc | sort -n
├── 服务状态：systemctl list-units --type=service
├── 开机自启：systemctl list-unit-files --state=enabled
└── 定时任务：crontab -l, /etc/crontab, /etc/cron.*

第十五分钟：检查文件异常
├── 最近修改文件：find / -mmin -1440 -type f 2>/dev/null
├── SUID文件：find / -perm -4000 -type f 2>/dev/null
├── 可写目录：find /tmp /var/tmp -type f 2>/dev/null
├── Web目录：find /var/www -mmin -1440 2>/dev/null
└── 敏感文件：ls -la /etc/passwd /etc/shadow

第二十分钟：检查日志
├── 安全日志：tail -200 /var/log/secure
├── 认证日志：tail -200 /var/log/auth.log
├── 系统日志：tail -200 /var/log/messages
├── Web日志：tail -200 /var/log/nginx/access.log
└── 错误日志：tail -200 /var/log/nginx/error.log
```

### 17.2.2 常用排查命令

```bash
# ===== 账户排查 =====
# 查看所有可登录用户
cat /etc/passwd | grep -E "/bin/bash|/bin/sh|/bin/zsh"

# 查看UID为0的用户
awk -F: '$3==0 {print $1}' /etc/passwd

# 查看最近登录成功
last -100

# 查看登录失败记录
lastb -100
faillog -a
pam_tally2 --user

# 查看当前登录用户
w
who

# ===== 进程排查 =====
# 查看所有进程树
ps auxf
pstree -p

# 查看高CPU进程
top -bn1 | head -20

# 检查隐藏进程（对比ps和/proc）
ps -eo pid | sort -n > /tmp/ps_pid.txt
ls /proc | grep -E '^[0-9]+$' | sort -n > /tmp/proc_pid.txt
diff /tmp/ps_pid.txt /tmp/proc_pid.txt

# 查看进程打开的文件
lsof -p <pid>

# 查看进程网络连接
ls -la /proc/<pid>/fd
cat /proc/<pid>/net/tcp

# ===== 网络排查 =====
# 查看所有网络连接
netstat -tulnp
ss -tulnp

# 查看异常连接（国外IP等）
netstat -an | grep ESTABLISHED

# 查看端口监听
netstat -lnp

# 查看流量（需要iftop/nethogs）
iftop
nethogs

# ===== 文件排查 =====
# 最近24小时修改的文件
find / -mmin -1440 -type f 2>/dev/null | head -50

# 最近1小时修改的可执行文件
find / -mmin -60 -type f \( -perm -111 -o -name "*.sh" -o -name "*.py" \) 2>/dev/null

# 查找SUID/SGID文件
find / -perm -6000 -type f 2>/dev/null

# 查找隐藏文件
find / -name ".*" -type f 2>/dev/null | head -30

# /tmp目录检查
ls -la /tmp
ls -la /var/tmp
ls -la /dev/shm

# ===== 日志排查 =====
# 查看SSH登录日志
grep "Accepted" /var/log/secure | tail -50
grep "Failed" /var/log/secure | tail -50

# 查看sudo使用记录
grep "sudo" /var/log/secure | tail -50

# 查看Web攻击日志
grep -i "union\|select\|drop\|script\|xss\|../" /var/log/nginx/access.log | tail -50

# 查看系统重启记录
last reboot
```

### 17.2.3 WebShell排查

```bash
# WebShell排查方法

# 1. 基于文件特征
# 查找一句话木马特征

grep -r "base64_decode" /var/www/html/ 2>/dev/null
grep -r "assert\s*(" /var/www/html/ 2>/dev/null
grep -r "system\s*(" /var/www/html/ 2>/dev/null
grep -r "shell_exec" /var/www/html/ 2>/dev/null

grep -r "gzinflate\|str_rot13\|base64" /var/www/html/ 2>/dev/null
grep -r "eval(gzinflate" /var/www/html/ 2>/dev/null

# 查找特殊函数组合
grep -rE "eval.*\$_" /var/www/html/ 2>/dev/null
grep -rE "assert.*\$_" /var/www/html/ 2>/dev/null
grep -rE "call_user_func.*\$_" /var/www/html/ 2>/dev/null

# 2. 基于文件时间
# 最近7天修改的PHP文件
find /var/www/html -name "*.php" -mtime -7 -type f

# 3. 基于日志分析

grep "POST" /var/log/nginx/access.log | grep -v "/login\|/api\|/admin" | head -30

awk '{print $7}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -20

# 查找可疑的User-Agent
grep -i "sqlmap\|nmap\|nikto\|acunetix" /var/log/nginx/access.log

# 4. 工具检测
# 河马WebShell检测
# wget https://www.shellpub.com/download/hm-linux.tgz
# ./hm scan /var/www/html

# Linux WebShell查杀（findweb）
# find /var/www/html -name "*.php" -exec grep -l "eval\|assert\|system" {} \;

# 5. 常见WebShell位置
# - 上传目录：uploads/, files/, /tmp
# - 图片目录：可能藏图片马
# - 缓存目录：cache/, compile/
# - 配置文件目录：config/
# - 备份文件：.bak, .php.bak, .php.~
```

## 17.3 常见安全事件处置

### 17.3.1 Web入侵事件处置

```
Web入侵事件处置流程：

1. 遏制阶段
   ├── 隔离受攻击服务器（断网或防火墙隔离）
   ├── 备份日志和可疑文件（保护证据）
   ├── 暂时下线受影响页面
   └── 通知业务方

2. 溯源分析
   ├── 分析访问日志，找到攻击入口
   ├── 分析WebShell，找到攻击者IP
   ├── 排查所有WebShell（可能有多个）
   ├── 检查数据库是否被篡改/泄露
   ├── 确定入侵时间线
   └── 判断是否有横向移动

3. 清除与修复
   ├── 删除所有WebShell
   ├── 删除攻击者创建的后门账户
   ├── 修复被篡改的文件
   ├── 修复漏洞（注入点、上传漏洞等）
   ├── 更新应用到安全版本
   └── 系统安全加固

4. 恢复阶段
   ├── 从干净备份恢复（确认备份干净）
   ├── 验证业务功能正常
   ├── 加强监控（WAF、日志审计）
   ├── 观察24-48小时无异常
   └── 业务恢复上线

5. 总结改进
   ├── 输出事件报告
   ├── 分析根本原因
   ├── 制定改进措施
   ├── 更新应急预案
   └── 安全培训
```

## 17.4 入侵后系统加固

### 17.4.1 系统清理与加固

```bash
# 入侵后系统加固步骤

# 第一步：备份与取证
# 备份关键日志
cp -r /var/log /backup/intrusion/logs_$(date +%Y%m%d)
# 备份可疑文件
# 内存镜像（如需要）
# dd if=/dev/mem of=/backup/memory.img （慎用，生产环境需评估）

# 第二步：账户清理
# 删除可疑账户
userdel -r suspicious_user
# 锁定可疑账户
usermod -L suspicious_user
# 修改所有用户密码（大范围泄露时）
# 检查并修复sudoers
visudo -c
# 检查所有组
cat /etc/group

# 第三步：SSH安全加固
# 修改SSH配置（见第三章）
# 更换SSH密钥
# 禁用密码登录，使用密钥
# 限制登录IP

# 第四步：后门清理
# 检查并删除定时任务后门
crontab -r  # 或逐条检查
rm -f /etc/cron.d/suspicious
# 检查并删除系统服务后门
systemctl disable malicious.service
# 检查init.d/rc.local
cat /etc/rc.local
# 检查bash后门
cat ~/.bashrc
cat ~/.bash_profile
cat /etc/profile
# 检查SSH后门（authorized_keys）
cat ~/.ssh/authorized_keys

# 第五步：恶意文件清除
# 删除WebShell
# 删除恶意脚本
# 删除挖矿程序
# 删除rootkit
# 使用rkhunter/chkrootkit扫描
rkhunter --check
# 重新安装被篡改的系统文件
# rpm -Vf /usr/bin/ss  # 验证文件完整性

# 第六步：漏洞修复
# 更新系统补丁
yum update -y
# 更新应用到最新版本
# 修复Web漏洞

# 第七步：安全加固
# 防火墙加固
# 安装HIDS/IDS
# 启用审计
# 配置日志集中收集
# 安装防病毒软件

# 第八步：验证观察
# 监控24-48小时
# 检查异常连接
# 检查异常进程
# 验证业务正常
```
