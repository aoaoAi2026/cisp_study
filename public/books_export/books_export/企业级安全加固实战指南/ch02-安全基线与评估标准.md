# 第二章 安全基线与评估标准

## 2.1 安全基线概述

安全基线（Security Baseline）是一套经过验证的最低安全配置标准，是系统必须满足的基本安全要求。它是安全加固的起点，也是安全检查的依据。

### 2.1.1 什么是安全基线

安全基线是信息系统在安全方面应达到的基本要求集合，包含：

- **配置基线**：操作系统、数据库、中间件等的安全配置标准
- **补丁基线**：必须安装的安全补丁最低要求
- **密码基线**：密码长度、复杂度、更换周期等要求
- **权限基线**：用户权限、文件权限的最低安全标准
- **网络基线**：端口开放、防火墙规则的基本要求

**基线的特点：**
- **最低标准**：必须满足的最低安全要求
- **可量化**：每条基线都有明确的检查标准
- **可验证**：通过工具或人工检查确认是否达标
- **可落地**：技术上可行，业务上可接受

### 2.1.2 为什么需要安全基线

**没有基线的问题：**
```
问题场景：某企业100台服务器
├── 运维A配置的服务器：密码12位，90天更换
├── 运维B配置的服务器：密码8位，180天更换
├── 运维C配置的服务器：密码6位，从不更换
├── 结果：安全水平参差不齐，最弱的决定整体安全
```

**建立基线的好处：**
1. **统一标准**：所有系统遵循同一安全标准
2. **提高效率**：有明确的检查清单，不用每次从零开始
3. **便于审计**：对照基线逐项检查，结果清晰
4. **持续改进**：基线可根据威胁变化迭代更新
5. **新人上手快**：有标准可依，减少对个人经验的依赖

### 2.1.3 基线的分级策略

不同安全等级的系统应采用不同级别的基线：

| 基线级别 | 适用场景 | 严格程度 |
|---------|---------|---------|
| Level 1 | 办公系统、测试环境 | 基础，对业务影响小 |
| Level 2 | 一般业务系统 | 中等，兼顾安全与可用 |
| Level 3 | 核心业务系统 | 严格，安全优先 |
| Level 4 | 涉密系统、支付系统 | 非常严格，最大安全 |

**分级实施示例：**
```yaml
密码策略基线：
  Level 1:
    最小长度: 8位
    复杂度: 字母+数字
    更换周期: 180天
  Level 2:
    最小长度: 10位
    复杂度: 大小写+数字+特殊字符
    更换周期: 90天
  Level 3:
    最小长度: 12位
    复杂度: 大小写+数字+特殊字符
    更换周期: 60天
    MFA: 启用
  Level 4:
    最小长度: 16位
    复杂度: 大小写+数字+特殊字符
    更换周期: 30天
    MFA: 强制启用
    硬件令牌: 要求
```

### 2.2.1 CIS Benchmarks

CIS（Center for Internet Security）Benchmarks是全球公认的安全配置基准，由全球安全专家共同维护。

**CIS Benchmarks特点：**
- 覆盖广泛：Windows、Linux、macOS、数据库、中间件、云平台
- 分级建议：Level 1（基础）、Level 2（增强）
- 免费公开：可在官网免费下载
- 定期更新：跟随软件版本和威胁形势更新

### 2.2.2 等保2.0基线要求

等级保护2.0对安全计算环境的基线要求：

**身份鉴别：**
```
- 应对登录的用户进行身份标识和鉴别
- 身份标识具有唯一性
- 身份鉴别信息具有复杂度要求并定期更换
- 应采用口令、密码技术、生物技术等两种或两种以上组合的鉴别技术
- 应具有登录失败处理功能
- 当进行远程管理时，应采取必要措施防止鉴别信息在网络传输过程中被窃听
```

**访问控制：**
```
- 应对登录的用户分配账户和权限
- 应重命名或删除默认账户，修改默认账户的默认口令
- 应及时删除或停用多余的、过期的账户
- 应授予管理用户所需的最小权限
- 应由授权主体配置访问控制策略
- 应严格限制默认账户的访问权限
```

**安全审计：**
```
- 应启用安全审计功能
- 审计覆盖到每个用户
- 应对重要的用户行为和重要安全事件进行审计
- 审计记录应包括事件的日期和时间、用户、事件类型、是否成功
- 应对审计记录进行保护，定期备份
- 应能对审计记录进行分析，生成审计报表
```

**入侵防范：**
```
- 应遵循最小安装的原则
- 应关闭不需要的系统服务、默认共享和高危端口
- 应通过设定终端接入方式或网络地址范围对管理终端进行限制
- 应能发现可能存在的已知漏洞
- 应能够检测到对重要服务器进行入侵的行为
- 应提供重要数据的本地数据备份与恢复功能
```

## 2.3 安全基线检查方法

### 2.3.1 人工检查

人工检查适用于小批量、复杂场景的基线核查：

**Linux系统人工检查清单：**
```bash
# 1. 检查密码策略
cat /etc/login.defs | grep -E "PASS_MAX_DAYS|PASS_MIN_DAYS|PASS_MIN_LEN"

# 2. 检查空密码账户
awk -F: '($2 == "") {print $1}' /etc/shadow

# 3. 检查UID为0的账户
awk -F: '($3 == 0) {print $1}' /etc/passwd

# 4. 检查开放端口
netstat -tlnp
ss -tlnp

# 5. 检查运行的服务
systemctl list-unit-files --state=enabled | grep enabled
chkconfig --list

# 6. 检查SSH配置
grep -E "PermitRootLogin|PasswordAuthentication|Protocol" /etc/ssh/sshd_config

# 7. 检查文件权限
ls -la /etc/passwd /etc/shadow /etc/group /etc/gshadow

# 8. 检查审计日志
rpm -q audit
systemctl status auditd
```

**Windows系统人工检查清单：**
```powershell
# 1. 检查密码策略
net accounts
Get-ADDefaultDomainPasswordPolicy

# 2. 检查管理员组成员
net localgroup administrators
Get-LocalGroupMember -Group "Administrators"

# 3. 检查账户锁定策略
net accounts /lockoutthreshold

# 4. 检查审核策略
auditpol /get /category:*

# 5. 检查防火墙状态
netsh advfirewall show allprofiles state
Get-NetFirewallProfile

# 6. 检查自动更新
wuauclt /detectnow
Get-WindowsUpdateLog

# 7. 检查共享
net share
Get-SmbShare

# 8. 检查已安装补丁
wmic qfe list
Get-HotFix
```

### 2.3.3 编写自定义基线检查脚本

**示例：Linux基线检查脚本**

```bash
#!/bin/bash
# Linux安全基线检查脚本
# 输出格式：[PASS] 或 [FAIL]

echo "=============================================="
echo "       Linux安全基线检查报告"
echo "=============================================="
echo "检查时间: $(date)"
echo "主机名: $(hostname)"
echo "系统版本: $(cat /etc/redhat-release 2>/dev/null || cat /etc/os-release)"
echo "=============================================="

PASS=0
FAIL=0
WARN=0

check_pass() {
    echo "[PASS] $1"
    ((PASS++))
}

check_fail() {
    echo "[FAIL] $1"
    ((FAIL++))
}

check_warn() {
    echo "[WARN] $1"
    ((WARN++))
}

# ===== 账户安全检查 =====
echo ""
echo "【1. 账户安全检查】"
echo "-------------------------"

# 1.1 检查空密码账户
empty_users=$(awk -F: '($2 == "") {print $1}' /etc/shadow)
if [ -z "$empty_users" ]; then
    check_pass "不存在空密码账户"
else
    check_fail "存在空密码账户: $empty_users"
fi

# 1.2 检查除root外UID为0的账户
uid0_users=$(awk -F: '($3 == 0 && $1 != "root") {print $1}' /etc/passwd)
if [ -z "$uid0_users" ]; then
    check_pass "除root外无UID=0的账户"
else
    check_fail "存在其他UID=0的账户: $uid0_users"
fi

# 1.3 检查密码最长使用期限
pass_max_days=$(grep "^PASS_MAX_DAYS" /etc/login.defs | awk '{print $2}')
if [ "$pass_max_days" -le 90 ] 2>/dev/null; then
    check_pass "密码最长使用期限: ${pass_max_days}天 (≤90天)"
else
    check_fail "密码最长使用期限: ${pass_max_days}天 (应≤90天)"
fi

# 1.4 检查密码最小长度
pass_min_len=$(grep "^PASS_MIN_LEN" /etc/login.defs | awk '{print $2}')
if [ "$pass_min_len" -ge 8 ] 2>/dev/null; then
    check_pass "密码最小长度: ${pass_min_len}位 (≥8位)"
else
    check_fail "密码最小长度: ${pass_min_len}位 (应≥8位)"
fi

# ===== 服务与端口检查 =====
echo ""
echo "【2. 服务与端口检查】"
echo "-------------------------"

# 2.1 检查SSH协议版本
ssh_protocol=$(grep "^Protocol" /etc/ssh/sshd_config | awk '{print $2}')
if [ "$ssh_protocol" = "2" ]; then
    check_pass "SSH协议版本: $ssh_protocol (应为2)"
else
    check_warn "SSH协议版本未明确设置"
fi

# 2.2 检查root远程登录
permit_root=$(grep "^PermitRootLogin" /etc/ssh/sshd_config | awk '{print $2}')
if [ "$permit_root" = "no" ]; then
    check_pass "禁止root远程登录"
else
    check_fail "允许root远程登录: $permit_root"
fi

# 2.3 检查危险服务
dangerous_services=("telnet" "rsh" "rlogin" "rexec" "talk" "ntalk")
for svc in "${dangerous_services[@]}"; do
    if systemctl is-enabled $svc &>/dev/null; then
        check_fail "危险服务已启用: $svc"
    fi
done
check_pass "已检查危险服务，未发现启用项"

# ===== 文件权限检查 =====
echo ""
echo "【3. 文件权限检查】"
echo "-------------------------"

# 3.1 /etc/passwd 权限
passwd_perm=$(stat -c "%a" /etc/passwd)
if [ "$passwd_perm" = "644" ]; then
    check_pass "/etc/passwd权限: $passwd_perm"
else
    check_fail "/etc/passwd权限: $passwd_perm (应为644)"
fi

# 3.2 /etc/shadow 权限
shadow_perm=$(stat -c "%a" /etc/shadow)
if [ "$shadow_perm" = "000" ] || [ "$shadow_perm" = "400" ]; then
    check_pass "/etc/shadow权限: $shadow_perm"
else
    check_fail "/etc/shadow权限: $shadow_perm (应≤400)"
fi

# 3.3 检查全局可写文件
world_writable=$(find / -xdev -type f -perm -o+w 2>/dev/null | head -20)
if [ -z "$world_writable" ]; then
    check_pass "未发现全局可写文件"
else
    check_warn "发现全局可写文件（前20个）:"
    echo "$world_writable"
fi

# ===== 日志审计检查 =====
echo ""
echo "【4. 日志审计检查】"
echo "-------------------------"

if systemctl is-active auditd &>/dev/null; then
    check_pass "审计服务(auditd)已启动"
else
    check_fail "审计服务(auditd)未启动"
fi

if systemctl is-enabled auditd &>/dev/null; then
    check_pass "审计服务已设置开机自启"
else
    check_fail "审计服务未设置开机自启"
fi

# ===== 总结 =====
echo ""
echo "=============================================="
echo "              检查结果汇总"
echo "=============================================="
echo "通过 (PASS): $PASS"
echo "失败 (FAIL): $FAIL"
echo "警告 (WARN): $WARN"
echo "=============================================="

TOTAL=$((PASS + FAIL + WARN))
if [ $TOTAL -gt 0 ]; then
    RATE=$(echo "scale=1; $PASS * 100 / $TOTAL" | bc)
    echo "合规率: ${RATE}%"
fi

exit $FAIL
```

**使用方法：**
```bash
chmod +x baseline_check.sh
./baseline_check.sh > baseline_report.txt
cat baseline_report.txt
```

## 2.4 风险评估方法

### 2.4.1 漏洞扫描

**Nessus：**
Nessus是业界领先的漏洞扫描器。

```bash
# 基本扫描命令
nessuscli scan new --targets 192.168.1.0/24 --policy "Basic Network Scan"

# 导出报告
nessuscli report export <scan_id> --format pdf --output report.pdf
```

**OpenVAS (Greenbone)：**
开源漏洞扫描解决方案。

```bash
# 启动OpenVAS
openvas-start

# 命令行扫描
omp -u admin -w password -T <target> -n "Scan Name" -c <config_id>

# 查看报告列表
omp -u admin -w password -G
```

**Nmap脚本扫描：**
```bash
# 使用Nmap的漏洞扫描脚本
nmap --script vuln -p1-1000 192.168.1.100

# 认证漏洞扫描
nmap --script auth -p1-1000 192.168.1.100

# 综合安全扫描
nmap -sV --script=default,vuln,auth -p- 192.168.1.100
```

### 2.4.2 渗透测试

渗透测试通过模拟真实攻击来验证系统安全性：

**渗透测试流程：**
```
1. 信息收集
   ├── 网络扫描
   ├── 端口识别
   ├── 服务指纹
   └── 信息泄露检测

2. 漏洞探测
   ├── 漏洞扫描
   ├── 手动验证
   ├── 漏洞利用测试
   └── 权限提升测试

3. 漏洞利用
   ├── 验证漏洞真实性
   ├── 获取访问权限
   ├── 横向移动测试
   └── 数据窃取模拟

4. 报告输出
   ├── 风险等级评估
   ├── 修复建议
   └── 加固优先级
```

## 2.5 加固效果评估

### 2.5.1 评估指标

**定量指标：**
| 指标 | 计算公式 | 目标值 |
|------|---------|--------|
| 基线合规率 | 合规项 / 总检查项 × 100% | ≥95% |
| 高危漏洞数 | CVSS≥7.0的漏洞数量 | 0 |
| 中危漏洞数 | 4.0≤CVSS<7.0的漏洞数量 | ≤5 |
| 漏洞修复率 | 已修复漏洞 / 总漏洞数 × 100% | ≥90% |
| 平均修复时间 | 总修复时间 / 漏洞数 | ≤7天 |
| 弱口令占比 | 弱口令账户 / 总账户数 × 100% | 0 |

**定性指标：**
- 安全意识水平
- 管理制度完善程度
- 应急响应能力
- 安全工具覆盖率
