# 第二十二章 备份与灾难恢复

## 22.1 备份概述

### 22.1.1 备份的重要性

数据是组织最重要的资产。有效的备份策略可以保护数据免受以下威胁：

- **硬件故障**：硬盘损坏、RAID控制器故障
- **软件错误**：文件系统损坏、数据库崩溃
- **人为错误**：误删除、错误配置
- **恶意攻击**：勒索软件、数据篡改
- **自然灾害**：火灾、洪水、地震

### 22.1.2 备份类型

**按备份内容分类**

1. **完全备份（Full Backup）**
   - 备份所有数据
   - 恢复最简单，耗时最长
   - 占用最多存储空间

2. **增量备份（Incremental Backup）**
   - 只备份自上次备份以来的变化
   - 占用空间最小，备份速度快
   - 恢复时需要多个备份集

3. **差异备份（Differential Backup）**
   - 备份自上次完全备份以来的变化
   - 介于完全备份和增量备份之间

**按备份位置分类**

- **本地备份**：备份存储在本地磁盘
- **远程备份**：备份传输到远程位置
- **云备份**：使用云存储服务

## 22.2 文件系统备份

### 22.2.1 rsync备份

```bash
# 基本rsync备份
rsync -avz /source/ /backup/

# 使用SSH加密传输
rsync -avz -e ssh /source/ user@remote:/backup/

# 删除目标中不存在的文件（镜像）
rsync -avz --delete /source/ /backup/

# 显示进度
rsync -avz --progress /source/ /backup/

# 限制带宽
rsync -avz --bwlimit=1000 /source/ /backup/

# 排除特定文件
rsync -avz --exclude='*.log' --exclude='tmp/' /source/ /backup/

# 保持符号链接
rsync -avz -L /source/ /backup/

# 完整备份脚本
cat > /usr/local/bin/backup-sync.sh << 'EOF'
#!/bin/bash
# rsync备份脚本

SOURCE="/data"
BACKUP="/backup"
LOG="/var/log/backup-sync.log"
REMOTE_USER="backup"
REMOTE_HOST="backup-server.example.com"
REMOTE_DIR="/backup/data"

# 日期标记
DATE=$(date +%Y%m%d)
LOGFILE="$LOG.$DATE"

exec > >(tee -a "$LOGFILE") 2>&1

echo "=== 备份开始: $(date) ==="
echo "源: $SOURCE"
echo "目标: $BACKUP"

# 本地备份
rsync -avz --delete --exclude='*.tmp' --exclude='.cache/' \
    --exclude='proc/' --exclude='sys/' \
    -e "ssh -o StrictHostKeyChecking=no" \
    "$SOURCE/" "$BACKUP/"

# 远程备份（可选）
# rsync -avz --delete \
#     -e "ssh -o StrictHostKeyChecking=no" \
#     "$SOURCE/" "$REMOTE_USER@$REMOTE_HOST:$REMOTE_DIR/"

# 保留最近30天的日志
find $LOG -name "*.log.*" -mtime +30 -delete

echo "=== 备份完成: $(date) ==="
EOF

chmod +x /usr/local/bin/backup-sync.sh

# 添加到crontab
echo "0 2 * * * root /usr/local/bin/backup-sync.sh" >> /etc/crontab
```

### 22.2.2 tar备份

```bash
# 创建压缩备份
tar -czvf backup.tar.gz /data

# 排除特定目录
tar -czvf backup.tar.gz --exclude='/proc' --exclude='/sys' \
    --exclude='/dev' --exclude='/run' /

# 创建增量备份
TARFILE="/backup/system-$(date +%Y%m%d).tar.gz"
SNAPSHOT="/backup/snapshot.file"

tar -czvf "$TARFILE" --listed-incremental="$SNAPSHOT" /data

# 恢复增量备份
tar -czvf /backup/incremental.tar.gz --listed-incremental=/dev/null /data

# 完整备份脚本
cat > /usr/local/bin/backup-tar.sh << 'EOF'
#!/bin/bash
# tar备份脚本

BACKUP_DIR="/backup"
SOURCE_DIRS="/etc /var/www /home /opt"
DATE=$(date +%Y%m%d%H%M)
RETENTION_DAYS=30

# 创建备份目录
mkdir -p "$BACKUP_DIR"

# 创建备份
BACKUP_FILE="$BACKUP_DIR/backup-$DATE.tar.gz"
tar -czvf "$BACKUP_FILE" \
    --exclude='/proc' --exclude='/sys' \
    --exclude='/dev' --exclude='/run' \
    --exclude='/tmp' --exclude='/var/cache' \
    $SOURCE_DIRS

# 创建校验文件
sha256sum "$BACKUP_FILE" > "$BACKUP_FILE.sha256"

# 清理旧备份
find "$BACKUP_DIR" -name "backup-*.tar.gz*" -mtime +$RETENTION_DAYS -delete

echo "备份完成: $BACKUP_FILE"
echo "备份大小: $(du -h "$BACKUP_FILE" | cut -f1)"
EOF

chmod +x /usr/local/bin/backup-tar.sh
```

### 22.2.3 远程备份

```bash
# 使用SSH进行加密远程备份
rsync -avz -e "ssh -i /root/.ssh/backup_key" /data/ user@remote:/backup/

# 使用tar和SSH
tar -czvf - /data | ssh user@remote "cat > /backup/backup-$(date +%Y%m%d).tar.gz"

# 使用ncrypt远程备份
tar -czvf - /data | gzip | ncrypt -k password | ssh user@remote "cat > /backup/backup-$(date +%Y%m%d).tar.gz.gpg"

# 远程备份脚本
cat > /usr/local/bin/remote-backup.sh << 'EOF'
#!/bin/bash
# 远程备份脚本

REMOTE_USER="backup"
REMOTE_HOST="backup.example.com"
REMOTE_DIR="/backup/client1"
SSH_KEY="/root/.ssh/backup_key"
SOURCE_DIR="/data"
DATE=$(date +%Y%m%d)

# 创建临时目录
TMP_DIR=$(mktemp -d)

# 本地打包
tar -czvf "$TMP_DIR/backup-$DATE.tar.gz" -C / "$SOURCE_DIR" 2>/dev/null

# 创建清单
cat > "$TMP_DIR/manifest.txt" << MANIFEST
Backup Date: $(date)
Source: $SOURCE_DIR
Size: $(du -h "$TMP_DIR/backup-$DATE.tar.gz" | cut -f1)
MANIFEST

# 传输文件
scp -i "$SSH_KEY" \
    "$TMP_DIR/backup-$DATE.tar.gz" \
    "$TMP_DIR/manifest.txt" \
    "$REMOTE_USER@$REMOTE_HOST:$REMOTE_DIR/"

# 清理
rm -rf "$TMP_DIR"

echo "远程备份完成: $(date)"
EOF

chmod +x /usr/local/bin/remote-backup.sh
```

## 22.3 数据库备份

### 22.3.1 MySQL/MariaDB备份

```bash
# mysqldump基本备份
mysqldump -u root -p --all-databases > all_databases.sql

# 压缩备份
mysqldump -u root -p --all-databases | gzip > backup.sql.gz

# 备份单个数据库
mysqldump -u root -p database_name > database_name.sql

# 备份特定表
mysqldump -u root -p database_name table1 table2 > tables.sql

# 完整备份脚本
cat > /usr/local/bin/mysql-backup.sh << 'EOF'
#!/bin/bash
# MySQL备份脚本

MYSQL_USER="root"
MYSQL_PASS="password"
BACKUP_DIR="/backup/mysql"
DATE=$(date +%Y%m%d)
RETENTION_DAYS=30

mkdir -p "$BACKUP_DIR"

# 备份所有数据库
mysqldump -u "$MYSQL_USER" -p"$MYSQL_PASS" \
    --all-databases \
    --single-transaction \
    --routines \
    --triggers \
    --events \
    | gzip > "$BACKUP_DIR/mysql-all-$DATE.sql.gz"

# 创建备份清单
cat > "$BACKUP_DIR/manifest-$DATE.txt" << MANIFEST
MySQL Backup Report
==================
Backup Date: $(date)
Backup File: mysql-all-$DATE.sql.gz
File Size: $(du -h "$BACKUP_DIR/mysql-all-$DATE.sql.gz" | cut -f1)
MANIFEST

# 清理旧备份
find "$BACKUP_DIR" -name "*.sql.gz" -mtime +$RETENTION_DAYS -delete
find "$BACKUP_DIR" -name "manifest-*.txt" -mtime +$RETENTION_DAYS -delete

echo "MySQL备份完成: $(date)"
EOF

chmod +x /usr/local/bin/mysql-backup.sh

# XtraBackup在线备份（更高效）
cat > /usr/local/bin/mysql-xtrabackup.sh << 'EOF'
#!/bin/bash
# Percona XtraBackup备份脚本

BACKUP_DIR="/backup/xtrabackup"
DATE=$(date +%Y%m%d%H%M)
FULL_BACKUP="$BACKUP_DIR/full-$DATE"
INCR_BACKUP="$BACKUP_DIR/incr-$DATE"
RETENTION_DAYS=7

mkdir -p "$BACKUP_DIR"/{full,incr,logs}

# 完整备份
xtrabackup --backup \
    --target-dir="$FULL_BACKUP" \
    --user=root --password=password \
    --compress \
    --parallel=4

# 应用日志
xtrabackup --prepare --target-dir="$FULL_BACKUP"

# 增量备份（如果完整备份存在）
if [ -d "$FULL_BACKUP" ]; then
    xtrabackup --backup \
        --target-dir="$INCR_BACKUP" \
        --incremental-basedir="$FULL_BACKUP" \
        --user=root --password=password
fi

# 清理旧备份
find "$BACKUP_DIR" -maxdepth 1 -type d -name "full-*" -mtime +$RETENTION_DAYS -exec rm -rf {} \;
find "$BACKUP_DIR" -maxdepth 1 -type d -name "incr-*" -mtime +$RETENTION_DAYS -exec rm -rf {} \;

echo "XtraBackup备份完成"
EOF

chmod +x /usr/local/bin/mysql-xtrabackup.sh
```

MANIFEST

find "$BACKUP_DIR" -name "*.dump" -mtime +$RETENTION_DAYS -delete
find "$BACKUP_DIR" -name "*.sql.gz" -mtime +$RETENTION_DAYS -delete

## 22.4 灾难恢复计划

### 22.4.1 灾难恢复策略（3-2-1原则）

**3-2-1备份原则**

- **3份数据副本**：主数据 + 2份备份
- **2种不同介质**：磁盘 + 磁带/云
- **1份异地备份**：防止本地灾难

### 22.4.2 恢复测试脚本

```bash
# 恢复测试脚本
cat > /usr/local/bin/backup-test.sh << 'EOF'
#!/bin/bash
# 备份恢复测试脚本

BACKUP_DIR="/backup"
TEST_DIR="/tmp/backup-test"
LOG="/var/log/backup-test.log"

exec > >(tee -a "$LOG") 2>&1

echo "=== 备份恢复测试开始: $(date) ==="

# 创建测试目录
mkdir -p "$TEST_DIR"

# 测试tar备份
LATEST_TAR=$(ls -t $BACKUP_DIR/backup-*.tar.gz 2>/dev/null | head -1)
if [ -n "$LATEST_TAR" ]; then
    echo "测试恢复: $LATEST_TAR"
    tar -tzf "$LATEST_TAR" > /dev/null 2>&1
    if [ $? -eq 0 ]; then
        echo "✓ tar备份完整性正常"
    else
        echo "✗ tar备份损坏!"
    fi
fi

# 测试MySQL备份
LATEST_MYSQL=$(ls -t $BACKUP_DIR/mysql-*.sql.gz 2>/dev/null | head -1)
if [ -n "$LATEST_MYSQL" ]; then
    echo "测试恢复: $LATEST_MYSQL"
    gunzip -t "$LATEST_MYSQL" > /dev/null 2>&1
    if [ $? -eq 0 ]; then
        echo "✓ MySQL备份完整性正常"
    else
        echo "✗ MySQL备份损坏!"
    fi
fi

# 清理
rm -rf "$TEST_DIR"

echo "=== 备份恢复测试完成: $(date) ==="
EOF

chmod +x /usr/local/bin/backup-test.sh

# 添加到crontab（每周测试）
echo "0 3 * * 0 root /usr/local/bin/backup-test.sh" >> /etc/crontab
```

### 22.4.3 灾难恢复计划模板

```markdown
# 灾难恢复计划 (DRP)

## 1. 概述

### 1.1 目的
本灾难恢复计划旨在确保在发生灾难时，组织能够快速恢复关键业务系统。

### 1.2 适用范围
- 生产服务器
- 数据库系统
- 网络基础设施
- 关键业务应用

### 1.3 恢复时间目标 (RTO)
- 核心系统：4小时
- 一般系统：24小时
- 非关键系统：72小时

### 1.4 恢复点目标 (RPO)
- 核心数据：1小时

## 2. 灾难场景与响应

### 2.1 硬件故障
**响应步骤：**
1. 确认故障范围
2. 切换到备用硬件
3. 从最新备份恢复数据
4. 验证系统功能

**响应步骤：**
1. 停止受影响服务
2. 分析错误日志
3. 必要时回滚到之前版本
4. 从备份恢复配置文件

### 2.3 勒索软件攻击
**响应步骤：**
1. 立即隔离受感染系统
2. 不要支付赎金
3. 从离线备份恢复
4. 通知安全团队

### 2.4 自然灾害
**响应步骤：**
1. 激活异地恢复站点
2. 验证备份可用性
3. 在备用位置恢复服务
4. 通知所有相关方

## 22.5 云备份

### 22.5.1 AWS S3备份

```bash
# 安装AWS CLI
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
./aws/install

# 配置AWS CLI
aws configure

# 创建S3备份脚本
cat > /usr/local/bin/s3-backup.sh << 'EOF'
#!/bin/bash
# AWS S3备份脚本

BUCKET="my-backup-bucket"
DATE=$(date +%Y%m%d)
SOURCE_DIR="/data"

# 上传文件
aws s3 cp "$SOURCE_DIR" "s3://$BUCKET/backup/$DATE/" --recursive \
    --storage-class STANDARD_IA \
    --metadata "backup-date=$DATE"

# 设置生命周期规则（自动删除30天前的备份）
aws s3api put-bucket-lifecycle-configuration \
    --bucket "$BUCKET" \
    --lifecycle-configuration '{
        "Rules": [{
            "ID": "BackupExpiration",
            "Prefix": "backup/",
            "Status": "Enabled",
            "Expiration": {"Days": 30}
        }]
    }'

echo "S3备份完成: $(date)"
EOF

chmod +x /usr/local/bin/s3-backup.sh
```

## 22.7 实战案例：企业备份系统部署

### 22.7.1 案例需求

某企业需要建立完整的备份系统：
- 本地备份（每日）
- 远程备份（每小时增量）
- 数据库自动备份
- 云端异地备份
- 定期恢复测试

### 22.7.2 部署步骤

**第一步：建立备份目录结构**

```bash
# 创建备份目录
mkdir -p /backup/{daily,weekly,monthly,mysql,postgres,remote,logs}
chmod 700 /backup

# 设置备份保留策略
DAILY_RETENTION=7
WEEKLY_RETENTION=4
MONTHLY_RETENTION=6
```

**第二步：部署备份脚本**

```bash
# 文件系统每日备份
cat > /usr/local/bin/backup-daily.sh << 'EOF'
#!/bin/bash
# 每日文件系统备份

BACKUP_DIR="/backup/daily"
DATE=$(date +%Y%m%d)
SOURCE_DIRS="/etc /var/www /home /opt"

tar -czvf "$BACKUP_DIR/backup-$DATE.tar.gz" \
    --exclude='/proc' --exclude='/sys' \
    --exclude='/dev' --exclude='/run' \
    $SOURCE_DIRS

# 校验
sha256sum "$BACKUP_DIR/backup-$DATE.tar.gz" > "$BACKUP_DIR/backup-$DATE.sha256"

# 清理旧备份
find "$BACKUP_DIR" -name "backup-*.tar.gz" -mtime +7 -delete
find "$BACKUP_DIR" -name "backup-*.sha256" -mtime +7 -delete

echo "每日备份完成: $(date)"
EOF

chmod +x /usr/local/bin/backup-daily.sh

# 数据库备份
cat > /usr/local/bin/backup-database.sh << 'EOF'
#!/bin/bash
# 数据库备份

BACKUP_DIR_MYSQL="/backup/mysql"
BACKUP_DIR_PG="/backup/postgres"
DATE=$(date +%Y%m%d)

# MySQL备份
mysqldump -u root -p'password' --all-databases \
    --single-transaction --routines --triggers \
    | gzip > "$BACKUP_DIR_MYSQL/mysql-$DATE.sql.gz"

# PostgreSQL备份
pg_dumpall -U postgres | gzip > "$BACKUP_DIR_PG/postgres-$DATE.sql.gz"

# 清理旧备份
find "$BACKUP_DIR_MYSQL" -name "mysql-*.sql.gz" -mtime +7 -delete
find "$BACKUP_DIR_PG" -name "postgres-*.sql.gz" -mtime +7 -delete

echo "数据库备份完成: $(date)"
EOF

chmod +x /usr/local/bin/backup-database.sh

# 远程增量备份
cat > /usr/local/bin/backup-remote.sh << 'EOF'
#!/bin/bash
# 远程增量备份

REMOTE_USER="backup"
REMOTE_HOST="backup-server.example.com"
REMOTE_DIR="/backup/client1"
SSH_KEY="/root/.ssh/backup_key"

rsync -avz --delete \
    -e "ssh -i $SSH_KEY" \
    /backup/daily/ \
    "$REMOTE_USER@$REMOTE_HOST:$REMOTE_DIR/daily/"

rsync -avz --delete \
    -e "ssh -i $SSH_KEY" \
    /backup/mysql/ \
    "$REMOTE_USER@$REMOTE_HOST:$REMOTE_DIR/mysql/"

rsync -avz --delete \
    -e "ssh -i $SSH_KEY" \
    /backup/postgres/ \
    "$REMOTE_USER@$REMOTE_HOST:$REMOTE_DIR/postgres/"

echo "远程备份完成: $(date)"
EOF

chmod +x /usr/local/bin/backup-remote.sh
```

**第三步：配置自动执行**

```bash
# 添加到crontab
cat >> /etc/crontab << 'EOF'
# 每日凌晨2点文件备份
0 2 * * * root /usr/local/bin/backup-daily.sh >> /var/log/backup.log 2>&1

# 每日凌晨3点数据库备份
0 3 * * * root /usr/local/bin/backup-database.sh >> /var/log/backup.log 2>&1

# 每6小时远程备份
0 */6 * * * root /usr/local/bin/backup-remote.sh >> /var/log/remote-backup.log 2>&1

# 每周日凌晨恢复测试
0 4 * * 0 root /usr/local/bin/backup-test.sh
EOF
```

**第四步：验证备份系统**

```
