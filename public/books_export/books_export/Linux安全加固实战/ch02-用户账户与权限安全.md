# 第2章 用户账户与权限安全

## 2.1 用户账户管理

### 2.1.1 用户账户基础概念

Linux系统是一个多用户操作系统，能够同时支持多个用户登录并执行各种任务。每个用户都有一个唯一的用户标识符（UID），系统通过UID来识别和管理用户。用户信息存储在`/etc/passwd`文件中，而密码信息则存储在`/etc/shadow`文件中，只有root用户才有权限读取shadow文件。

在Linux系统中，用户账户不仅仅是为了登录使用，更重要的是用于进程所有权的控制。每个运行中的进程都归属于某个用户，进程的权限由其所属用户的权限决定。这种机制确保了不同用户之间的资源隔离和访问控制。

```
# 查看当前用户信息
$ id
uid=1000(admin) gid=1000(admin) groups=1000(admin),4(adm),24(cdrom),27(sudo)

# 查看所有用户账户
$ cat /etc/passwd | tail -10
nobody:x:65534:65534:nobody:/nonexistent:/usr/sbin/nologin
systemd-timesync:x:100:101:systemd Time Synchronization:/:/usr/sbin/nologin
systemd-network:x:101:102:systemd Network Management:/:/usr/sbin/nologin
admin:x:1000:1000:admin,,,:/home/admin:/bin/bash

# 理解passwd文件中各字段的含义
# 用户名:密码占位符:UID:GID:用户描述:主目录:登录Shell
```

Linux系统将用户账户分为三大类：**root用户**（UID为0）是系统的超级管理员，拥有最高权限，可以执行任何操作；**系统用户**（UID在1-999之间）用于运行系统服务和守护进程，通常不需要登录；**普通用户**（UID从1000开始）供普通用户使用，权限受到限制。

### 2.1.2 useradd命令详解

`useradd`是Linux系统中创建新用户账户的基本命令。这个命令在执行时会进行多项操作：创建用户条目、更新相关数据库、创建用户主目录（如果指定）、设置用户环境等。

```bash
# 基本语法
useradd [选项] 用户名

# 常用选项
# -c: 添加用户描述信息
# -d: 指定用户主目录
# -e: 设置账户过期日期（格式：YYYY-MM-DD）
# -f: 设置密码过期后多少天禁用账户（0表示立即禁用，-1表示永不禁用）
# -g: 指定主组
# -G: 指定附加组（多个组用逗号分隔）
# -m: 创建用户主目录
# -M: 不创建用户主目录
# -p: 指定加密后的密码（不推荐，会留在shell历史中）
# -s: 指定登录Shell
# -u: 指定UID

# 创建基本用户账户
useradd -m -s /bin/bash newuser

# 创建带有完整信息的用户
useradd -m \
    -c "Zhang San - System Administrator" \
    -e 2025-12-31 \
    -f 30 \
    -G sudo,adm,www-data \
    -s /bin/bash \
    zhangsan

# 创建系统用户（不创建主目录，不设置登录Shell）
useradd -r -s /usr/sbin/nologin \
    -c "Database Service Account" \
    dbadmin

# 创建指定UID的用户
useradd -m -u 1500 specialuser

# 查看useradd的默认值配置
useradd -D
# 输出示例：
# GROUP=100
# HOME=/home
# INACTIVE=-1
# EXPIRE=
# SHELL=/bin/sh
# SKEL=/etc/skel
# CREATE_MAIL_SPOOL=yes
```

创建用户时，系统会使用`/etc/default/useradd`文件中定义的默认值，以及`/etc/login.defs`中的配置。用户主目录会从`/etc/skel`目录复制默认文件，这些文件包括`.bashrc`、`.profile`等用户环境配置文件。

```bash
# 修改useradd的默认配置
useradd -D -s /bin/bash
useradd -D -b /export/home
useradd -D -e 2025-12-31

# 直接编辑配置文件修改默认配置
cat /etc/default/useradd
# GROUP=100
# HOME=/home
# INACTIVE=-1
# EXPIRE=
# SHELL=/bin/sh
# SKEL=/etc/skel
# CREATE_MAIL_SPOOL=yes

# 修改默认配置
sed -i 's/SHELL=\/bin\/sh/SHELL=\/bin\/bash/' /etc/default/useradd
```

### 2.1.3 usermod命令详解

`usermod`命令用于修改已存在的用户账户属性。它的选项与`useradd`类似，但增加了锁定/解锁账户、修改用户ID等特殊功能。

```bash
# 基本语法
usermod [选项] 用户名

# 常用选项
# -c: 修改用户描述信息
# -d: 修改用户主目录
# -e: 修改账户过期日期
# -f: 修改密码过期后禁用账户的天数
# -g: 修改用户主组
# -G: 修改附加组（会覆盖原有附加组）
# -a: 与-G一起使用，追加附加组而不是覆盖
# -l: 修改用户名
# -L: 锁定账户（在密码前添加!）
# -U: 解锁账户
# -s: 修改登录Shell
# -u: 修改UID

# 修改用户描述信息
usermod -c "Li Si - Finance Department" lisi

# 修改用户的登录Shell
usermod -s /bin/zsh lisi

# 修改用户主目录
usermod -d /home/new_home -m lisi

# 将用户添加到附加组
usermod -aG docker,nginx lisi

# 修改用户名
usermod -l newname oldname

# 锁定用户账户（禁止登录）
usermod -L lisi

# 解锁用户账户
usermod -U lisi

# 修改用户UID
usermod -u 2000 lisi
# 注意：修改UID后，需要手动更改该用户文件的Ownership
find /home/lisi -uid 1500 -exec chown -h lisi {} \;

# 同时修改多个属性
usermod -c "Modified Description" \
    -s /bin/bash \
    -G sudo,adm \
    -e 2025-12-31 \
    lisi
```

锁定用户账户是一个常用的安全操作。当某个账户需要临时禁用但不能删除时，锁定功能非常有用。`usermod -L`会在密码字段前添加一个"!"符号，使得原密码无法使用。解锁时`usermod -U`会移除这个符号，但如果密码字段原本就有"!"（表示密码已过期），则可能需要额外处理。

### 2.1.4 userdel命令详解

`userdel`命令用于删除用户账户。删除用户时需要谨慎操作，因为相关的文件、邮件和进程所有权可能会受到影响。

```bash
# 基本语法
userdel [选项] 用户名

# 常用选项
# -r: 删除用户主目录及其内容
# -f: 强制删除，即使该用户当前仍登录或有其运行的进程

# 安全删除用户（保留主目录作为备份）
userdel username

# 完全删除用户及其主目录
userdel -r username

# 强制删除正在运行的用户进程和主目录
userdel -rf username

# 删除用户后处理遗留文件
# 如果忘记使用-r参数，可以通过以下方式手动清理
# 查找该用户的所有文件
find / -uid UID -ls 2>/dev/null
# 将文件所有权转移给其他用户
chown -R newowner:newowner /path/to/files
# 或者直接删除
rm -rf /home/username
```

删除用户前，建议先检查该用户是否有正在运行的进程、计划任务、cron作业等。如果不进行这些检查就删除用户，可能导致以下问题：进程变成僵尸进程、邮件无法投递、计划任务失败等。

```bash
# 删除用户前应进行的检查

# 1. 检查用户是否已登录
who | grep username
w

# 2. 检查用户的进程
ps -u username
ps -ef | grep username

# 3. 检查用户的cron任务
crontab -u username -l
cat /var/spool/cron/username

# 4. 检查用户的at任务
atq -u username

# 5. 检查用户是否拥有任何文件
find / -uid UID -ls 2>/dev/null | head -20

# 6. 备份用户数据后再删除
tar -czvf /backup/${username}_home.tar.gz /home/${username}
userdel -r username
```

## 2.2 密码策略配置

### 2.2.1 /etc/login.defs文件详解

`/etc/login.defs`是Linux系统中用户登录和密码策略的基础配置文件。这个文件定义了许多系统级的安全策略，包括密码长度、密码有效期、UID范围、用户HOME目录权限等。

```bash
# 查看login.defs文件内容
cat /etc/login.defs
```

```
# /etc/login.defs - 用户登录管理配置文件

# 指定创建用户时使用的UID范围
UID_MIN                  1000
UID_MAX                 60000
# 系统用户的UID范围
SYS_UID_MIN               201
SYS_UID_MAX               999

# 指定创建组时使用的GID范围
GID_MIN                  1000
GID_MAX                 60000
SYS_GID_MIN               201
SYS_GID_MAX               999

# 密码有效期的设置
PASS_MAX_DAYS   99999    # 密码最大使用天数（90-365为宜）
PASS_MIN_DAYS   0        # 密码最小使用天数（防止频繁改密）
PASS_WARN_AGE  7        # 密码过期前警告天数

# 用户创建时是否创建邮件池
CREATE_HOME     yes      # 是否创建主目录

# 用户主目录默认权限
UMASK           022      # 022表示目录755，文件644

# 密码复杂度检查（某些系统支持）
# PASS_MIN_LEN    8       # 最小密码长度

# 加密方式（MD5、SHA256、SHA512）
ENCRYPT_METHOD SHA512

# 是否使用shadow password
USE_PWQUALITY  yes

# 登录失败锁定设置
FAILLOG_ENAB    yes
LOG_UNKFAIL_ENAB yes
```

配置密码策略时，需要在`/etc/login.defs`中设置以下关键参数：

```bash
# 修改密码策略建议值
# 编辑 /etc/login.defs

PASS_MAX_DAYS   90        # 密码90天后必须更换
PASS_MIN_DAYS   7         # 密码更改后至少7天才能再次更改
PASS_WARN_AGE   14        # 密码过期前14天开始警告
PASS_MIN_LEN    12        # 最小密码长度12位

# 这些设置对新建用户生效，已有用户需要用chage命令修改
```

需要注意的是，`/etc/login.defs`中的设置只对新创建的用户生效。对于已有用户，需要使用`chage`命令来修改其密码策略设置。

```bash
# 使用chage命令修改已有用户的密码策略
# 语法: chage [选项] 用户名

# 查看用户的密码策略
chage -l username
# 输出示例：
# 最近一次密码修改日期                    : 6月 25, 2026
# 密码过期时间                    : 9月 23, 2026
# 密码失效时间                    : 从不
# 账户过期时间                    : 从不
# 两次改变密码之间相距的最小天数      : 7
# 两次改变密码之间相距的最大天数      : 90
# 在密码过期之前警告的天数         : 14

# 设置密码最大有效期
chage -M 90 username

# 设置密码最小有效期（7天内不能修改密码）
chage -m 7 username

# 设置密码过期警告期
chage -W 14 username

# 设置账户过期日期
chage -E 2025-12-31 username

# 强制用户首次登录时修改密码
chage -d 0 username

# 综合设置（最长使用期90天，最短7天，过期前14天警告）
chage -M 90 -m 7 -W 14 username
```

### 2.2.2 /etc/pam.d/目录详解

PAM（Pluggable Authentication Modules，即可插拔认证模块）是Linux系统中处理用户认证的核心框架。PAM使用模块化的设计，允许系统管理员灵活配置各种认证策略，而无需修改应用程序本身。

PAM配置文件位于`/etc/pam.d/`目录下，每个支持的应用程序都有自己独立的配置文件。例如，`/etc/pam.d/sshd`用于SSH服务的认证，`/etc/pam.d/login`用于终端登录的认证。

```bash
# 查看PAM配置文件目录
ls -la /etc/pam.d/
# common-account   common-auth     common-password   common-session
# login            runuser         sshd              sudo
```

PAM配置文件的每一行格式为：

```
模块类型   控制标志   模块路径   模块参数
```

**PAM模块类型**包括：

- **auth**：认证模块，用于验证用户身份（如检查密码、智能卡等）
- **account**：账户模块，用于检查账户属性（如账户是否过期、是否允许当前时间登录等）
- **password**：密码模块，用于更新用户密码和密码策略
- **session**：会话模块，用于管理会话设置（如设置环境变量、挂载文件系统等）

**控制标志**决定当模块成功或失败时的行为：

- **required**：必须成功，如果失败继续检查同类型的其他模块，但最终会失败
- **requisite**：必须成功，如果失败立即返回失败
- **sufficient**：如果成功且之前没有required模块失败，则足以认证通过
- **optional**：模块的结果被忽略，仅供参考
- **include**：包含指定文件中的所有规则

```
# /etc/pam.d/common-auth 文件示例
# 此文件由pam-auth-update配置生成

auth    [default=ignore]      pam_unix.so nullok
auth    optional    pam_permit.so
auth    required    pam_deny.so

# /etc/pam.d/common-account 文件示例
account [default=ignore]      pam_unix.so
account optional    pam_permit.so
account required    pam_deny.so

# /etc/pam.d/common-session 文件示例
session required    pam_limits.so
session [default=ignore]      pam_unix.so
session optional    pam_permit.so
session required    pam_deny.so
```

### 2.2.3 密码复杂度配置

Linux系统通过PAM模块`pam_pwquality.so`（也称为`pam_cracklib.so`）来实现密码复杂度检查。这个模块可以强制要求密码满足最小长度、大小写、数字、特殊字符等要求。

```bash
# Debian/Ubuntu系统安装密码质量检查模块
apt-get install libpam-pwquality

# CentOS/RHEL系统通常已默认安装
yum install pam_pwquality
```

配置密码复杂度的主要参数包括：

```
# 编辑 /etc/security/pwquality.conf 或在PAM配置中指定

# 最小密码长度
minlen = 12

# 密码必须包含的字符种类数（数字、大写、小写、特殊字符）
dcredit = -1    # 至少包含1个数字
ucredit = -1   # 至少包含1个大写字母
lcredit = -1   # 至少包含1个小写字母
ocredit = -1  # 至少包含1个特殊字符

# 密码不能包含用户名
usercheck = 1

# 不能在上次密码的基础上只改几个字符
difok = 3      # 新密码至少要有3个字符与旧密码不同

# 旧密码重复检查
maxrepeat = 0  # 禁止相同字符连续出现
maxclassrepeat = 4  # 同一类字符最多连续出现4次

# 密码检查字典文件
dictcheck = 1

# 检查/etc/shadow中的破解列表
gecoscheck = 1
```

```bash
# 在PAM配置中启用密码复杂度检查
# 编辑 /etc/pam.d/common-password 或 /etc/pam.d/system-auth

# 在password行添加pam_pwquality.so模块
password requisite pam_pwquality.so retry=3 minlen=12 dcredit=-1 ucredit=-1 lcredit=-1 ocredit=-1
password sufficient pam_unix.so sha512 shadow nullok try_first_pass use_authtok
password required pam_deny.so
```

实际配置示例：

```bash
# /etc/security/pwquality.conf 完整配置
[匮乏]
# 密码最小长度为12位
minlen = 12

# 密码必须包含至少1个数字
dcredit = -1

# 密码必须包含至少1个大写字母
ucredit = -1

# 密码必须包含至少1个小写字母
lcredit = -1

# 密码必须包含至少1个特殊字符
ocredit = -1

# 新密码至少3个字符与旧密码不同
difok = 3

# 不能包含用户名
usercheck = 1

# 同一字符不能连续出现超过1次
maxrepeat = 1

# 密码历史（最近5个密码不能重复使用）
# 这需要在pam_unix.so中配合remember参数使用
```

在PAM密码配置中添加密码历史功能：

```bash
# 编辑 /etc/pam.d/common-password 或 /etc/pam.d/system-auth
# 添加 remember=5 参数到 pam_unix.so 行
password [success=2 default=ignore] pam_unix.so sha512 shadow try_first_pass use_authtok remember=5
```

### 2.2.4 账户锁定策略

账户锁定是防止暴力破解的有效手段。当用户连续多次输入错误密码后，系统会自动锁定该账户，一段时间后自动解锁或由管理员手动解锁。

```bash
# 配置PAM实现账户锁定
# 编辑 /etc/pam.d/common-auth 或 /etc/pam.d/system-auth

# 在auth行添加pam_tally2.so模块
auth required pam_tally2.so deny=5 unlock_time=600 onerr=fail file=/var/log/tallylog
```

对于CentOS/RHEL系统，可以使用`pam_faillock`模块：

```bash
# 编辑 /etc/pam.d/system-auth

# 在auth部分的required行之前添加
auth        required      pam_faillock.so preauth silent deny=5 unlock_time=600 fail_interval=300

# 在auth部分的sufficient行之后添加
auth        [default=die] pam_faillock.so authfail deny=5 unlock_time=600

# 在account部分添加
account     required      pam_faillock.so
```

对于Ubuntu系统，可以使用`pam tally2`：

```bash
# 编辑 /etc/pam.d/common-auth

auth required pam_tally2.so onerr=fail file=/var/log/tallylog
auth required pam_tally2.so deny=5 unlock_time=300
```

配置完成后，可以手动锁定和解锁账户：

```bash
# 手动锁定账户
pam_tally2 --user username --set=0
# 或
faillock --user username --set

# 查看账户锁定状态
pam_tally2 --user username
# 或
faillock --user username

# 解锁账户（清除失败记录）
pam_tally2 --user username --reset
# 或
faillock --user username --reset

# 解锁被锁定的账户（使用pam_tally2）
pam_tally2 -r -u username

# 查看所有失败登录尝试
pam_tally2
```

## 2.3 sudo权限管理

### 2.3.1 sudo基础配置

sudo（superuser do）是Linux系统中最重要的权限管理工具之一，它允许普通用户以root或其他用户身份执行命令。正确配置sudo是系统安全的重要组成部分——既要让用户能够完成必要的工作，又不能授予过多的权限。

```bash
# 查看当前用户的sudo权限
sudo -l
# 输出示例：
# [sudo] password for admin:
# User admin may run the following commands on this host:
#     (ALL : ALL) ALL
#     (root) NOPASSWD: /usr/bin/systemctl restart nginx

# 查看特定用户的sudo权限
sudo -l -U username
```

sudo的配置文件是`/etc/sudoers`，不建议直接编辑此文件，而是编辑`/etc/sudoers.d/`目录下的文件。

```bash
# 编辑sudoers配置
# 正确方法：使用visudo命令
visudo

# 语法检查
visudo -c
# 输出：/etc/sudoers: parsed OK

# 查看sudoers.d目录内容
ls -la /etc/sudoers.d/
```

### 2.3.2 sudoers配置详解

sudoers配置文件有复杂的语法规则：

```
user    host=(runas)    commands
用户    主机=(运行身份)   命令
```

```bash
# 基本语法示例

# 允许admin用户以root身份执行所有命令
admin    ALL=(ALL)    ALL

# 允许developers组的所有成员以root身份执行特定命令
%developers    ALL=(root)    /usr/bin/systemctl restart nginx, /usr/bin/systemctl stop nginx

# 允许webadmin用户以www-data身份运行指定命令
webadmin    ALL=(www-data)    /usr/bin/whoami, /usr/bin/ps aux

# 允许任何用户以root身份运行所有命令（不推荐）
ALL    ALL=(ALL)    ALL

# NOPASSWD：不提示输入密码
admin    ALL=(ALL)    NOPASSWD: /usr/bin/systemctl restart nginx

# 限制某些命令不能使用sudo运行
admin    ALL=(ALL)    ALL, !/bin/bash, !/bin/sh
```

实际配置示例：

```bash
# /etc/sudoers.d/admin - 管理员配置

# 允许admin组的所有成员sudo
%admin ALL=(ALL) ALL

# /etc/sudoers.d/webteam - Web团队配置
# 允许webteam组成员以root身份管理nginx服务
%webteam ALL=(root) NOPASSWD: /usr/bin/systemctl restart nginx
%webteam ALL=(root) NOPASSWD: /usr/bin/systemctl stop nginx
%webteam ALL=(root) NOPASSWD: /usr/bin/systemctl status nginx
%webteam ALL=(root) NOPASSWD: /usr/bin/systemctl reload nginx
%webteam ALL=(root) NOPASSWD: /usr/bin/nginx -t
%webteam ALL=(root) NOPASSWD: /usr/bin/nginx -s *

# 允许webteam组成员查看nginx日志
%webteam ALL=(root) NOPASSWD: /usr/bin/tail -f /var/log/nginx/error.log
%webteam ALL=(root) NOPASSWD: /usr/bin/tail -f /var/log/nginx/access.log

# /etc/sudoers.d/dbteam - 数据库团队配置
# 允许dbteam组成员以mysql身份执行备份命令
%dbteam ALL=(mysql) NOPASSWD: /usr/bin/mysqldump --all-databases
%dbteam ALL=(mysql) NOPASSWD: /usr/bin/mysqlcheck --all-databases
%dbteam ALL=(mysql) NOPASSWD: /usr/bin/mysql -e "SHOW DATABASES;"
```

### 2.3.3 sudo安全最佳实践

```bash
# 1. 禁用root的sudo权限（如果已有其他管理员账户）
# 编辑 /etc/sudoers.d/admin
# 移除 root ALL=(ALL) ALL 这一行

# 2. 强制使用密码
# 确保没有 NOPASSWD 配置在生产环境

# 3. 禁止某些危险命令
%developers ALL=(ALL) ALL, !/usr/bin/passwd, !/usr/bin/passwd root, !/usr/sbin/useradd, !/usr/sbin/userdel

# 4. 设置命令限制
# 只允许以只读方式运行特定命令
%auditors ALL=(ALL) NOPASSWD: /usr/bin/less /var/log/*.log, /usr/bin/tail /var/log/*.log

# 5. 设置sudo会话超时
# 默认 sudo 密码缓存为15分钟，可以调整为更短
Defaults    timestamp_timeout=5

# 6. 启用sudo日志记录
Defaults    log_host, log_year, logfile="/var/log/sudo.log"

# 7. 限制可以通过sudo使用shell
# 禁止使用可以获取交互式shell的命令
%wheel ALL=(ALL) ALL, !/usr/bin/su, !/usr/bin/bash, !/usr/bin/zsh, !/bin/sh

# 8. 限制可以sudo的用户组
# 只允许特定组的用户使用sudo
%sudo-group ALL=(ALL) ALL
```

```bash
# 查看sudo日志（需要配置logfile）
cat /var/log/sudo.log

# 查看sudo操作的审计日志
# 在RHEL/CentOS系统中，sudo操作会记录到/var/log/secure
# 在Debian/Ubuntu系统中，sudo操作会记录到/var/log/auth.log
grep sudo /var/log/auth.log | tail -20
```

## 2.4 特殊账户管理

### 2.4.1 root账户安全

root账户是Linux系统中最强大的账户，拥有不受限制的系统权限。保护root账户是系统安全的核心任务之一。

```bash
# 检查root账户状态
# 查看root密码信息
chage -l root

# 查看root账户的登录Shell
grep "^root:" /etc/passwd

# 查看谁可以使用su成为root
cat /etc/pam.d/su
# 检查是否存在允许特定用户组使用su的配置
auth            required        pam_wheel.so use_uid
# 如果这一行被注释掉或不存在，任何人都可以su到root
```

**root账户安全加固措施**：

```bash
# 1. 设置root账户强密码策略
# 编辑 /etc/login.defs
PASS_MAX_DAYS   90
PASS_MIN_DAYS   7
PASS_WARN_AGE   14

# 2. 使用sudo代替root登录
# 编辑 /etc/ssh/sshd_config
PermitRootLogin no

# 3. 限制su命令的使用
# 编辑 /etc/pam.d/su
auth            required        pam_wheel.so use_uid
# 只有wheel组的成员才能使用su成为root

# 创建用户并加入wheel组
usermod -aG wheel username

# 4. 监控root登录
# 在 /etc/profile.d/ 创建监控脚本
cat > /etc/profile.d/root-login-monitor.sh << 'EOF'
if [ "$EUID" -eq 0 ]; then
    logger -p auth.info "Root user executed: $0 $*"
fi
EOF

# 5. 禁止root远程登录SSH
# 确保sshd_config中
PermitRootLogin no
```

### 2.4.2 nobody账户管理

`nobody`是一个特殊的系统账户，主要用于运行没有特权的服务或处理被放弃的文件。当服务以nobody身份运行时，即使该服务被攻击者利用，他们也只能获得非常有限的权限。

```bash
# 查看nobody账户信息
id nobody
# 输出：uid=65534(nobody) gid=65534(nogroup) groups=65534(nogroup)

grep "^nobody:" /etc/passwd
# 输出：nobody:x:65534:65534:nobody:/nonexistent:/usr/sbin/nologin

# 验证nobody账户的Shell
# 通常应该是 /usr/sbin/nologin 或 /bin/false
```

nobody账户应该满足以下安全要求：

```bash
# 1. 确保nobody账户无法登录
# 检查Shell是否为nologin或false
grep "^nobody:" /etc/passwd | cut -d: -f7
# 如果不是，执行：
usermod -s /usr/sbin/nologin nobody

# 2. 确保nobody没有可用的home目录
# nobody的主目录应该是 /nonexistent 或 /var/empty
grep "^nobody:" /etc/passwd | cut -d: -f6
# 如果需要修改：
usermod -d /nonexistent nobody

# 3. 确保nobody不属于任何特权组
# nobody应该只属于nogroup或nobody组
groups nobody

# 4. 检查是否有服务以nobody身份运行
ps aux | grep "^nobody"
```

### 2.4.3 系统服务账户

Linux系统会创建多个用于运行特定服务的系统账户。这些账户通常UID较小，用于运行如apache、mysql、postgres、nginx等服务进程。

```bash
# 查看所有系统账户
cat /etc/passwd | awk -F: '{if ($3 < 1000) print $0}'

# 常见系统账户及其用途：
# daemon: 运行系统守护进程
# bin: 拥有系统二进制文件
# sys: 拥有系统二进制文件
# adm: 系统监控账户
# nobody: 无特权服务账户
# systemd-*: systemd服务账户
# postgres: PostgreSQL数据库
# mysql: MySQL数据库
# apache/nginx: Web服务器

# 检查是否所有系统账户都有不可用的Shell
for user in $(awk -F: '($3 < 1000) {print $1}' /etc/passwd); do
    shell=$(grep "^$user:" /etc/passwd | cut -d: -f7)
    if [ "$shell" != "/usr/sbin/nologin" ] && [ "$shell" != "/bin/false" ]; then
        echo "Warning: $user has shell: $shell"
    fi
done

# 设置不需要的系统账户为不可登录
for user in daemon bin sys games; do
    usermod -s /usr/sbin/nologin $user 2>/dev/null
done
```

## 2.5 SSH密钥认证配置

### 2.5.1 SSH密钥对生成

SSH密钥认证是比密码认证更安全的认证方式。密钥对由公钥和私钥组成，私钥保存在本地，公钥上传到服务器。

```bash
# 生成SSH密钥对（ED25519算法，推荐使用）
ssh-keygen -t ed25519 -a 100 -f ~/.ssh/id_ed25519 -C "user@hostname"

# 生成RSA算法的密钥对（兼容旧客户端）
ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa -C "user@hostname"

# 参数说明：
# -t: 密钥类型（ed25519, rsa, ecdsa）
# -b: 密钥位数（RSA建议4096）
# -a: KDF（密钥派生函数）迭代次数，越高越安全但越慢
# -f: 私钥文件路径
# -C: 密钥注释，通常是邮箱或描述

# 生成过程中会提示输入密码（passphrase）
# 强烈建议设置密码，这会在私钥丢失时提供额外保护
```

```bash
# 密钥生成示例
$ ssh-keygen -t ed25519 -a 100 -f ~/.ssh/id_ed25519 -C "admin@webserver01"
Generating public/private ed25519 key pair.
Enter passphrase (empty for no passphrase):
Enter same passphrase again:
Your identification has been saved in /ssh/id_ed25519.
Your public key has been saved in /ssh/id_ed25519.pub.
The key fingerprint is:
SHA256:abcdefghijklmnopqrstuvwxyz1234567890ABCDEFG admin@webserver01
The key's randomart image is:
+---[ED25519 3072]----+
|                     |
|                     |
|                     |
+----[SHA256]----------+
```

### 2.5.2 公钥上传和配置

```bash
# 方法1：使用ssh-copy-id命令（推荐）
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server

# 方法2：手动复制公钥
# 查看公钥内容
cat ~/.ssh/id_ed25519.pub

# 手动添加到服务器
# 1. 登录服务器
ssh user@server

# 2. 创建.ssh目录并设置权限
mkdir -p ~/.ssh
chmod 700 ~/.ssh

# 3. 添加公钥到authorized_keys
echo "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA... admin@webserver01" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys

# 4. 验证配置
cat ~/.ssh/authorized_keys
```

```bash
# 方法3：一条命令完成（如果没有ssh-copy-id）
cat ~/.ssh/id_ed25519.pub | ssh user@server "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"
```

### 2.5.3 SSH密钥安全配置

```bash
# 编辑 ~/.ssh/config 或 /etc/ssh/sshd_config

# 服务器端配置 /etc/ssh/sshd_config
# 确保以下配置已设置

# 禁用密码认证（必须先确保密钥认证可用）
```


## 2.6 用户账户审计

### 2.6.1 账户审计方法

定期审计用户账户是发现潜在安全问题的重要手段。审计内容包括：检查空密码账户、检查未经授权的账户、审查权限分配、验证密码策略执行情况等。

```bash
# 1. 检查空密码账户
awk -F: '($2 == "") {print $1}' /etc/shadow
# 如果有输出，说明存在空密码账户，需要立即处理

# 2. 检查UID为0的非root账户
awk -F: '($3 == 0) {print $1}' /etc/passwd
# 正常情况应该只有root，如果还有其他账户需要立即检查

# 3. 检查可以登录的非系统账户
grep -E '(/bin/bash|/bin/sh)$' /etc/passwd | awk -F: '{print $1, $7}'
# 确认每个可登录账户都是授权的

# 4. 检查最近创建的用户
tail -20 /etc/passwd

# 5. 检查密码策略执行情况
# 查看密码过期信息
for user in $(cut -d: -f1 /etc/passwd); do
    expiry=$(chage -l $user 2>/dev/null | grep "密码过期" | head -1)
    last_change=$(chage -l $user 2>/dev/null | grep "最近一次密码修改" | head -1)
    echo "$user: $last_change | $expiry"
done

# 6. 检查账户锁定状态
pam_tally2
# 或
faillock --list
```

## 2.7 PAM暴力破解防护

### 2.7.1 fail2ban配置

fail2ban是一个自动化入侵防御工具，它通过监控日志文件并自动更新防火墙规则来阻止恶意IP。

```bash
# 安装fail2ban
# Debian/Ubuntu
apt-get install fail2ban

# CentOS/RHEL
yum install epel-release
yum install fail2ban
```

```bash
# 配置fail2ban
cat > /etc/fail2ban/jail.local << 'EOF'
[DEFAULT]
# IP白名单
ignoreip = 127.0.0.1/8 ::1 192.168.1.0/24
# 禁止时长（秒）
bantime = 86400
# 检测时长（秒）
findtime = 600
# 最大尝试次数
maxretry = 5

[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 3
bantime = 604800
findtime = 3600

[nginx-http-auth]
enabled = true
port = http,https
filter = nginx-http-auth
logpath = /var/log/nginx/error.log
maxretry = 3
bantime = 3600

[php-url-fopen]
enabled = false
port = http,https
filter = php-url-fopen
logpath = /var/log/nginx/access.log
maxretry = 3
EOF
```

```bash
# 启动和检查fail2ban
systemctl enable fail2ban
systemctl start fail2ban

fail2ban-client status

fail2ban-client status sshd
```

## 2.8 实战案例：账户被入侵后的处理

### 2.8.1 发现和确认入侵

当怀疑系统被入侵时，应该按照以下步骤进行处理：

```bash
# 1. 识别可疑账户
# 检查/etc/passwd中是否有未知账户
diff <(cat /etc/passwd) <(sudo cat /etc/passwd.bak 2>/dev/null) || echo "No backup found"

# 检查最近添加的账户
lastlog | grep "Never"
who | head -20

# 2. 检查账户修改时间
stat /etc/passwd
stat /etc/shadow
stat /etc/sudoers
stat /etc/sudoers.d/*

# 3. 检查异常登录
# 查看当前登录用户
who
# 查看登录历史
last | head -30
# 查看失败的登录尝试
lastb | head -30

# 4. 检查特权提升
# 查看哪些用户可以sudo
for user in $(cut -d: -f1 /etc/passwd); do
    if sudo -l -U $user 2>/dev/null | grep -q "may run"; then
        echo "$user has sudo access:"
        sudo -l -U $user 2>/dev/null
    fi
done
```

### 2.8.2 隔离和遏制

```bash
# 1. 立即断开可疑会话
# 查看用户会话
who
# 踢出可疑用户
pkill -KILL -u username

# 2. 锁定可疑账户
usermod -L username
# 或设置账户过期
usermod -e 2020-01-01 username

# 3. 撤销可疑的sudo权限
# 检查并编辑 /etc/sudoers 和 /etc/sudoers.d/
visudo

# 4. 删除未经授权的公钥
# 检查 ~/.ssh/authorized_keys
cat ~/.ssh/authorized_keys
# 移除未知公钥

# 5. 更新SSH配置（禁用可疑密钥）
# 编辑 /etc/ssh/sshd_config
# 添加或修改
AuthorizedKeysFile    .ssh/authorized_keys
```

### 2.8.3 调查和清理

```bash
# 1. 检查账户相关文件的完整性
# 使用rpm/deb包验证
rpm -Va | grep -E '^\..{2,3}T.*\/etc\/passwd|^\..{2,3}T.*\/etc\/shadow|^\..{2,3}T.*\/etc\/sudoers'
# 或
debsums -e | grep -E '/etc/passwd|/etc/shadow|/etc/sudoers'

# 2. 检查计划任务中的恶意代码
cat /etc/crontab
ls -la /etc/cron.d/
ls -la /etc/cron.daily/
ls -la /etc/cron.hourly/
# 检查所有用户的crontab
for user in $(cut -d: -f1 /etc/passwd); do
    crontab -l -u $user 2>/dev/null
done

# 3. 检查SSH authorized_keys中的可疑条目
for user in $(cut -d: -f1 /etc/passwd); do
    authkeys="/home/$user/.ssh/authorized_keys"
    if [ -f "$authkeys" ]; then
        echo "=== $user authorized_keys ==="
        cat "$authkeys"
    fi
done

# 4. 检查SSH配置是否被篡改
diff /etc/ssh/sshd_config /etc/ssh/sshd_config.bak 2>/dev/null || echo "No backup"

# 5. 检查系统日志中的异常
grep -E 'Accepted|Failed' /var/log/auth.log | tail -100
grep -E 'su:|sudo:' /var/log/auth.log | tail -50

# 6. 检查新创建的账户
awk -F: '{print $1, $3, $7}' /etc/passwd | while read user uid shell; do
    if [ "$shell" != "/usr/sbin/nologin" ] && [ "$shell" != "/bin/false" ] && [ $uid -ge 1000 ]; then
        echo "Login user: $user (UID: $uid)"
    fi
done
```

### 2.8.4 系统恢复

```bash
# 1. 重置所有可能泄露的密码
# 重置root密码
passwd root

# 重置所有用户密码（建议）
for user in $(cut -d: -f1 /etc/passwd); do
    if [ "$user" != "root" ] && [ "$user" != "sync" ] && [ "$user" != "shutdown" ] && [ "$user" != "halt" ]; then
        # 生成随机密码并重置
        newpass=$(openssl rand -base64 12)
        echo "$user:$newpass" | chpasswd
        echo "$user: $newpass"
    fi
done

# 2. 重新生成SSH密钥
# 对于系统SSH密钥
ssh-keygen -A

# 对于用户的个人密钥，需要用户重新生成并分发
echo "Please regenerate your SSH keys and update authorized_keys on servers"

# 3. 重新配置fail2ban
systemctl restart fail2ban

# 4. 确保SELinux/AppArmor启用
getenforce  # on CentOS/RHEL
aa-status   # on Ubuntu

# 5. 恢复文件权限
# 根据安全基线恢复权限
chmod 600 /etc/shadow
chmod 644 /etc/passwd
chmod 600 /etc/gshadow
chmod 644 /etc/group
chmod 700 /etc/ssh
chmod 600 /etc/ssh/sshd_config

# 6. 执行完整的安全更新
apt-get update && apt-get upgrade -y  # Debian/Ubuntu
yum update -y                          # CentOS/RHEL

# 7. 通知相关安全团队和用户
# 建立事件报告
```

用户账户与权限安全是Linux系统安全的基石。本章详细介绍了用户账户管理的各个方面，包括用户创建、修改、删除的基本操作，以及如何通过PAM模块配置密码策略和账户锁定策略。

密码策略的正确配置对于防止弱密码和暴力破解至关重要。通过`/etc/login.defs`和`/etc/pam.d/`目录下的配置文件，可以实现密码复杂度要求、历史记录、最小长度等安全控制。结合账户锁定策略，可以有效防止暴力破解攻击。

sudo权限管理需要在工作需要和安全限制之间找到平衡。正确的sudo配置应该遵循最小权限原则，只授予用户完成工作所必需的最小权限集合。

SSH密钥认证比密码认证更安全，因为密钥对提供了更强的加密保护和双因素认证的可能性。在配置SSH认证时，应该禁用密码认证，确保私钥安全，并正确管理authorized_keys文件。

定期进行用户账户审计是发现潜在安全问题的关键。通过检查空密码账户、UID异常、登录历史、sudo使用记录等，可以及时发现和处理安全问题。

## 2.11 PAM认证模块详解

### 2.11.1 PAM架构概述

PAM（Pluggable Authentication Modules，可插拔认证模块）是Linux系统中统一管理认证、授权和会话的框架。它将认证功能从应用程序中分离出来，通过模块化的方式提供灵活的认证机制。

PAM的核心优势在于其模块化设计：应用程序不需要了解具体的认证细节，只需调用PAM接口；管理员可以根据需求组合不同的PAM模块，实现复杂的认证策略；认证策略的修改不需要重新编译应用程序。

PAM配置文件位于`/etc/pam.d/`目录下，每个服务都有独立的配置文件。配置文件采用四列格式：

```
模块类型    控制标志    模块路径    模块参数
```

**模块类型**：
- `auth`：认证模块，验证用户身份（如密码验证）
- `account`：账户模块，检查账户是否可用（如过期检查）
- `password`：密码模块，管理密码更新（如密码复杂度检查）
- `session`：会话模块，管理用户会话（如环境设置）

**控制标志**：
- `required`：必须成功，失败则继续执行其他模块但最终返回失败
- `requisite`：必须成功，失败则立即返回失败
- `sufficient`：成功则立即返回成功，失败则继续执行
- `optional`：可选模块，结果不影响整体认证结果

### 2.11.2 pam_faillock登录失败锁定

pam_faillock模块用于记录登录失败次数并锁定账户，防止暴力破解攻击。

```bash
# 查看当前faillock配置
grep faillock /etc/pam.d/*

# 配置pam_faillock（CentOS 7+）
# 编辑 /etc/pam.d/system-auth 和 /etc/pam.d/password-auth

# 在auth部分添加（注意顺序，在pam_unix.so之前）
auth        required      pam_faillock.so preauth deny=5 unlock_time=900 fail_interval=900
auth        sufficient    pam_unix.so nullok try_first_pass
auth        [default=die] pam_faillock.so authfail deny=5 unlock_time=900 fail_interval=900

# 在account部分添加
account     required      pam_faillock.so

# 参数说明：
# preauth：在认证前检查锁定状态
# deny=5：5次失败后锁定
# unlock_time=900：锁定900秒后自动解锁
# fail_interval=900：失败计数在900秒内累积
```

```bash
# 查看失败记录
faillock --list

# 查看特定用户的失败记录
faillock --user username --list

# 解锁用户
faillock --user username --reset

# 配置永久锁定（需要管理员手动解锁）
auth        required      pam_faillock.so preauth deny=5 unlock_time=0 fail_interval=900
```

### 2.11.3 pam_time时间限制

pam_time模块可以限制用户在特定时间段登录。

```bash
# 配置时间限制
# 编辑 /etc/security/time.conf

# 格式：服务;终端;用户;时间限制
# 示例：

# 只允许工作日8:00-18:00登录
login;*;!*root;Wk0800-1800

# 禁止周末登录
login;*;*;!Wk

# 只允许特定用户在特定时间登录
login;tty1;admin;Wk0900-1700

# 禁止特定用户登录
login;*;baduser;!Al

# 在PAM配置中启用
# 编辑 /etc/pam.d/login
account     required      pam_time.so
```

### 2.11.4 pam_access访问控制

pam_access模块基于IP地址和用户名控制访问权限。

```bash
# 配置访问控制
# 编辑 /etc/security/access.conf

# 格式：权限:用户:来源
# + 表示允许，- 表示拒绝

# 只允许root从特定IP登录
+:root:192.168.1.100
-:root:ALL

# 允许wheel组从内网登录
+:@wheel:192.168.1.
-:ALL:ALL

# 拒绝特定用户
-:baduser:ALL

# 允许所有用户从本地登录
+:ALL:LOCAL

# 在PAM配置中启用
# 编辑 /etc/pam.d/login 或 /etc/pam.d/sshd
account     required      pam_access.so
```

## 2.12 sudo高级配置详解

### 2.12.1 sudoers语法详解

sudoers配置文件使用特定的语法格式，理解这些语法对于正确配置sudo权限至关重要。

```bash
# sudoers基本语法
# 用户/组    主机=(运行身份)    命令

# 示例解析：
# admin ALL=(ALL) ALL
# admin用户可以在所有主机上，以任何身份，执行任何命令

# %wheel ALL=(ALL) ALL
# wheel组成员可以在所有主机上，以任何身份，执行任何命令

# admin ALL=(root) /usr/bin/systemctl restart nginx
# admin用户可以在所有主机上，以root身份，执行systemctl restart nginx

# admin ALL=(root) NOPASSWD: /usr/bin/systemctl status nginx
# admin用户执行systemctl status nginx时不需要输入密码

# 详细语法示例
# 编辑 /etc/sudoers 或使用 visudo

# 定义别名
# 用户别名
User_Alias WEBMASTERS = admin, webadmin, deployer

# 主机别名
Host_Alias WEBSERVERS = web1, web2, web3

# 命令别名
Cmnd_Alias WEB_COMMANDS = /usr/bin/systemctl restart nginx, /usr/bin/systemctl status nginx, /usr/sbin/nginx -t

# 运行身份别名
Runas_Alias WEBRUNAS = root, nginx

# 使用别名配置
WEBMASTERS WEBSERVERS=(WEBRUNAS) WEB_COMMANDS
```

### 2.12.2 sudo安全配置最佳实践

```bash
# 安全的sudo配置示例

# 1. 禁止NOPASSWD（除非有特殊需求）
# 不推荐：admin ALL=(ALL) NOPASSWD: ALL
# 推荐：需要密码确认
admin ALL=(ALL) ALL

# 2. 限制命令范围
# 只允许特定命令
admin ALL=(root) /usr/bin/systemctl restart nginx, /usr/bin/systemctl status nginx

# 3. 使用命令别名组织权限
Cmnd_Alias SERVICES = /usr/bin/systemctl start *, /usr/bin/systemctl stop *, /usr/bin/systemctl restart *
Cmnd_Alias SOFTWARE = /usr/bin/yum, /usr/bin/rpm, /usr/bin/apt
Cmnd_Alias DEBUGGING = /usr/bin/tcpdump, /usr/bin/strace, /usr/bin/lsof

admin ALL=(root) SERVICES, SOFTWARE, DEBUGGING

# 4. 记录sudo操作日志
Defaults logfile="/var/log/sudo.log"
Defaults log_host, log_year, syslog=local2

# 5. 限制sudo执行环境
Defaults env_reset
Defaults secure_path="/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin"

# 6. 设置超时时间
Defaults timestamp_timeout=5  # 5分钟后需要重新输入密码

# 7. 防止sudo提权
# 禁止使用sudo执行某些危险命令
admin ALL=(root) ALL, !/usr/bin/su, !/usr/bin/bash, !/usr/bin/sh
```

### 2.12.3 sudo日志审计配置

```bash
# 配置sudo详细日志
# 编辑 /etc/sudoers

# 启用日志记录
Defaults logfile="/var/log/sudo.log"
Defaults log_host        # 记录主机名
Defaults log_year        # 记录年份
Defaults log_input       # 记录输入（谨慎使用，可能记录敏感信息）
Defaults log_output      # 记录输出（谨慎使用）

# 发送到syslog
Defaults syslog=auth
Defaults syslog_goodpri=notice
Defaults syslog_badpri=alert

# 配置rsyslog处理sudo日志
# 编辑 /etc/rsyslog.conf 或创建 /etc/rsyslog.d/sudo.conf
local2.notice    /var/log/sudo.log

# 重启rsyslog
systemctl restart rsyslog

# 分析sudo日志脚本
cat > /usr/local/bin/sudo_audit.sh << 'EOF'
#!/bin/bash
# sudo日志分析脚本

LOG_FILE="/var/log/sudo.log"

echo "=== sudo使用统计 ==="
echo "最活跃的用户:"
awk '{print $1}' $LOG_FILE | sort | uniq -c | sort -rn | head -10

echo ""
echo "最常执行的命令:"
grep "COMMAND" $LOG_FILE | awk -F'COMMAND=' '{print $2}' | sort | uniq -c | sort -rn | head -10

echo ""
echo "sudo执行失败:"
grep -i "error\|failed" $LOG_FILE | tail -20

echo ""
echo "危险命令执行:"
grep "COMMAND" $LOG_FILE | grep -E "su|passwd|userdel|rm|chmod|chown" | tail -20
EOF

chmod +x /usr/local/bin/sudo_audit.sh
```

## 2.13 用户账户安全检查脚本

```bash
#!/bin/bash
# 用户账户安全检查脚本
# 保存为 /usr/local/bin/user_security_check.sh

echo "========================================"
echo "Linux用户账户安全检查报告"
echo "检查时间: $(date)"
echo "========================================"

# 1. 检查空密码账户
echo ""
echo "[1] 检查空密码账户..."
EMPTY_PASS=$(awk -F: '($2 == "" || $2 == "!") {print $1}' /etc/shadow 2>/dev/null)
if [ -n "$EMPTY_PASS" ]; then
    echo "[警告] 发现空密码账户:"
    echo "$EMPTY_PASS"
else
    echo "[正常] 无空密码账户"
fi

# 2. 检查UID为0的非root账户
echo ""
echo "[2] 检查UID为0的账户..."
UID_ZERO=$(awk -F: '($3 == 0 && $1 != "root") {print $1}' /etc/passwd)
if [ -n "$UID_ZERO" ]; then
    echo "[危险] 发现UID为0的非root账户:"
    echo "$UID_ZERO"
else
    echo "[正常] 只有root账户UID为0"
fi

# 3. 检查无密码或密码过期的账户
echo ""
echo "[3] 检查密码状态..."
for user in $(cut -d: -f1 /etc/passwd | grep -v nologin | grep -v false); do
    PASS_STATUS=$(passwd -S $user 2>/dev/null | awk '{print $2}')
    if [ "$PASS_STATUS" = "NP" ] || [ "$PASS_STATUS" = "L" ]; then
        echo "[警告] $user: 密码状态异常 ($PASS_STATUS)"
    fi
done

# 4. 检查可登录账户
echo ""
echo "[4] 检查可登录账户..."
LOGIN_USERS=$(grep -E '/bin/bash|/bin/sh' /etc/passwd | awk -F: '{print $1}')
echo "可登录账户列表:"
echo "$LOGIN_USERS"

# 5. 检查sudo权限分配
echo ""
echo "[5] 检查sudo权限..."
if [ -f /etc/sudoers ]; then
    echo "sudoers配置摘要:"
    grep -v "^#" /etc/sudoers | grep -v "^$" | grep -E "^[a-zA-Z]|^%"
fi

# 6. 检查最近登录
echo ""
echo "[6] 最近登录记录..."
last -n 10

# 7. 检查失败登录
echo ""
echo "[7] 失败登录统计..."
if [ -f /var/log/auth.log ]; then
    grep "Failed password" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -rn | head -10
elif [ -f /var/log/secure ]; then
    grep "Failed password" /var/log/secure | awk '{print $(NF-3)}' | sort | uniq -c | sort -rn | head -10
fi

# 8. 检查账户锁定状态
echo ""
echo "[8] 账户锁定状态..."
if command -v faillock &> /dev/null; then
    faillock --list 2>/dev/null | head -20
elif command -v pam_tally2 &> /dev/null; then
    pam_tally2 2>/dev/null | head -20
fi

echo ""
echo "========================================"
echo "检查完成"
echo "========================================"
```
