## 15.1 开源漏洞扫描工具

### 15.1.1 OpenVAS部署与使用

OpenVAS（Open Vulnerability Assessment System）是开源漏洞扫描领域的标杆工具，其前身为Nessus的开源版本。OpenVAS提供了全面的漏洞扫描能力，支持数千种漏洞检测。

OpenVAS架构由多个组件组成：OpenVAS Scanner负责执行漏洞扫描，OpenVAS Manager管理扫描结果和配置，Greenbone Security Assistant提供Web界面。理解这一架构有助于更好地配置和使用该工具。

**OpenVAS安装步骤：**

```bash
# 在Debian/Ubuntu系统上安装
apt-get update
apt-get install -y openvas openvas-scanner greenbone-security-assistant
gvm-libs gvm-tools

# 安装数据库
apt-get install -y postgresql postgresql-contrib

# 初始化OpenVAS
openvas-setup

# 检查安装状态
openvas-check-setup
```

**首次配置OpenVAS：**

```bash
# 启动PostgreSQL
systemctl start postgresql
systemctl enable postgresql

# 创建OpenVAS用户
su - postgres -c "createuser -DRS gvm"

# 创建数据库
su - postgres -c "psql -c \"ALTER USER gvm WITH PASSWORD 'your_password';\""
su - postgres -c "psql -c \"CREATE DATABASE gvm OWNER gvm;\""

# 同步漏洞数据
greenbone-nvt-sync
greenbone-scapdata-sync
greenbone-certdata-sync

# 启动服务
systemctl start openvas-manager
systemctl start openvas-scanner
systemctl start gsad

# 访问Web界面
# https://localhost:9392
```

**使用OpenVAS进行扫描：**

1. 登录Greenbone Security Assistant Web界面
2. 创建扫描目标：
   - 点击"Configuration" → "Targets"
   - 添加目标主机IP地址或CIDR范围
   - 配置端口列表（默认使用"All IANA assigned TCP"）
   - 设置凭证（SSH、RDP等）
3. 创建扫描任务：
   - 点击"Scans" → "Tasks"
   - 选择扫描模板（如"Full and fast"）
   - 关联目标
4. 启动扫描并监控进度
5. 生成报告

**命令行操作OpenVAS：**

```bash
# 使用gvm-cli进行扫描
gvm-cli socket --xml '<get_version/>'

# 创建目标
gvm-cli socket --xml '
<create_target>
  <name>Internal Network Scan</name>
  <hosts>192.168.1.0/24</hosts>
  <port_list id="33d0cd82-518c-11e3-b91f-4061868a56a2"/>
</create_target>
'

# 启动扫描
gvm-cli socket --xml '
<create_task>
  <name>Weekly Network Scan</name>
  <target id="[TARGET_ID]"/>
  <scan_config id="[CONFIG_ID]"/>
</create_task>
'

# 查看扫描结果
gvm-cli socket --xml '<get_reports/>'
```

### 15.2.3 Lynis自动化集成

**定期自动审计脚本：**

```bash
#!/bin/bash
# Lynis自动化审计脚本
# /usr/local/bin/lynis-daily.sh

set -euo pipefail

readonly LOG_DIR="/var/log/lynis"
readonly REPORT_DIR="/var/reports/lynis"
readonly PREVIOUS_INDEX_FILE="${LOG_DIR}/.previous_hardening_index"

mkdir -p "$LOG_DIR" "$REPORT_DIR"

# 运行Lynis审计
echo "开始Lynis系统审计..."
lynis audit system \
    --report-file "${REPORT_DIR}/lynis_report_$(date +%Y%m%d_%H%M%S).log" \
    --log-file "${LOG_DIR}/lynis.log"

# 提取关键指标
hardening_index=$(lynis show report | grep "^hardening_index" | cut -d= -f2)
warnings_count=$(lynis show report | grep -c "^\[WARNING\]")
suggestions_count=$(lynis show report | grep -c "^\[SUGGESTION\]")

echo "硬加固指数: ${hardening_index}"
echo "警告数量: ${warnings_count}"
echo "建议数量: ${suggestions_count}"

# 与上次比较
if [[ -f "$PREVIOUS_INDEX_FILE" ]]; then
    previous_index=$(cat "$PREVIOUS_INDEX_FILE")
    index_diff=$((hardening_index - previous_index))
    
    if [[ $index_diff -gt 0 ]]; then
        echo "安全状况改善: +${index_diff}"
    elif [[ $index_diff -lt 0 ]]; then
        echo "安全状况下降: ${index_diff}"
    else
        echo "安全状况无变化"
    fi
fi

# 保存当前指数
echo "$hardening_index" > "$PREVIOUS_INDEX_FILE"

# 发送通知（如有严重警告）
critical_warnings=$(lynis show report | grep "^\[WARNING\]" | grep -i -E "critical|high|medium" || true)
if [[ -n "$critical_warnings" ]]; then
    echo "发现关键警告，需要立即处理:"
    echo "$critical_warnings" | mail -s "[安全警告] Lynis审计发现关键问题" admin@example.com
fi
```

配置定时任务：

```bash
# /etc/cron.d/lynis
# 每天凌晨2点运行Lynis审计
0 2 * * * root /usr/local/bin/lynis-daily.sh
```

## 15.3 Rkhunter检测

### 15.3.1 Rkhunter概述与安装

Rkhunter（Rootkit Hunter）是一款专门用于检测rootkit、隐藏进程、后门程序和本地漏洞利用的开源工具。它通过以下方式进行检测：检查文件和进程的MD5哈希值、比对已知rootkit特征、检查权限异常、检测隐藏文件等。

**Rkhunter安装：**

```bash
# Debian/Ubuntu
apt-get install rkhunter

# RHEL/CentOS
yum install epel-release
yum install rkhunter

# 源码安装
cd /opt
wget https://sourceforge.net/projects/rkhunter/files/rkhunter/1.4.6/rkhunter-1.4.6.tar.gz
tar -xzf rkhunter-1.4.6.tar.gz
cd rkhunter-1.4.6
./installer.sh --install
```

### 15.3.2 Rkhunter配置与使用

**基础配置：**

```bash
# 编辑配置文件
vim /etc/rkhunter.conf

# 关键配置项：
# 邮件通知
MAIL-ON-WARNING=admin@example.com

# 日志文件
LOGFILE=/var/log/rkhunter.log

# 白名单（已知安全的文件）
ALLOWHIDDENFILE=/etc/.git
ALLOWHIDDENDIR=/etc/.git

# 允许可疑端口
ALLOWPORT=45678

# 启用特定测试
ENABLE_TESTS=ALL
DISABLE_TESTS=hidden_ports
```

**首次运行前更新文件属性数据库：**

```bash
# 更新rkhunter数据库
rkhunter --update
rkhunter --propupd

# 查看数据库更新状态
cat /var/lib/rkhunter/db/rkhunter.dat | head -20
```

**执行扫描：**

```bash
# 完整扫描
rkhunter --checkall

# 跳过特定测试
rkhunter --check --skip-keypress --disable tests

# 仅检查特定项
rkhunter --check --tests "os_specifics file_properties hidden_procs"

# 查看详细输出
rkhunter --checkall --verbose

# 检查后自动生成报告
rkhunter --checkall --report-warnings-only
```

**扫描结果解读：**

```bash
# 查看最近扫描的日志
tail -100 /var/log/rkhunter.log

# 查找警告
grep -i warning /var/log/rkhunter.log
grep -i error /var/log/rkhunter.log

# 查看特定检查结果
rkhunter --check --list-tests
rkhunter --check --test "test_name"
```

**常见警告处理：**

```
警告1: "Possible hidden files"
原因: 系统中存在隐藏文件或目录
处理: 检查这些文件是否是正常的系统文件
  ls -la /proc/*/ 2>/dev/null | grep "^d"
  find / -name ".*" -type f 2>/dev/null | head -50

警告2: "Suspicious shared memory segments"
原因: 进程使用异常的共享内存段
处理: 检查可疑进程
  ipcs -m
  ps aux | grep <pid>

警告3: "Hidden processes"
原因: 进程被rootkit隐藏
处理: 使用其他工具交叉验证
  unhide proc sys
  chkrootkit
```

### 15.3.3 Rkhunter自动化扫描

```bash
#!/bin/bash
# rkhunter-daily.sh
# /etc/cron.daily/rkhunter-daily

LOGFILE="/var/log/rkhunter/rkhunter_daily.log"
LOCKFILE="/var/lock/subsys/rkhunter"

# 防止重复运行
if [[ -f "$LOCKFILE" ]]; then
    echo "Rkhunter正在运行或上次运行未完成"
    exit 0
fi

touch "$LOCKFILE"
trap "rm -f $LOCKFILE" EXIT

# 更新数据库
/usr/bin/rkhunter --update --quiet

# 运行检查
/usr/bin/rkhunter --checkall --skip-keypress --report-warnings-only 2>&1 | tee "$LOGFILE"

# 检查结果并发送报告
WARNINGS=$(grep -c "Warning" "$LOGFILE" 2>/dev/null || echo "0")

if [[ "$WARNINGS" -gt 0 ]]; then
    SUBJECT="[安全警告] Rkhunter发现${WARNINGS}个警告"
    cat "$LOGFILE" | mail -s "$SUBJECT" admin@example.com
fi

# 清理旧日志（保留30天）
find /var/log/rkhunter/ -name "*.log" -mtime +30 -delete
```

## 15.4 ClamAV病毒扫描

### 15.4.1 ClamAV安装与配置

ClamAV是开源的杀毒引擎，被广泛应用于邮件网关、文件服务器和终端保护。它支持Linux平台，提供命令行界面和库文件，可集成到各种安全解决方案中。

**ClamAV安装：**

```bash
# Debian/Ubuntu
apt-get update
apt-get install clamav clamav-daemon clamav-freshclam

# RHEL/CentOS
yum install epel-release
yum install clamav clamav-update

# 停止freshclam服务以进行手动更新
systemctl stop clamav-freshclam
```

**配置freshclam（病毒库更新）：**

```bash
# 编辑配置文件
vim /etc/clamav/freshclam.conf

# 关键配置：
# 代理设置（如需要）
# HTTPProxyServer proxy.example.com
# HTTPProxyPort 8080

# 每天更新次数
MaxAttempts 3

# 接收者
NotifyClamd /etc/clamav/clamd.conf

# 启动更新
systemctl start clamav-freshclam
systemctl enable clamav-freshclam

# 手动更新
freshclam -v
```

**配置clamd：**

```bash
# /etc/clamav/clamd.conf
LocalSocket /var/run/clamav/clamd.sock
LocalSocketGroup clamav
LocalSocketMode 660
FixStaleSocket true
User clamav
ScanMail true
ScanArchive true
MaxFileSize 100M
MaxScanSize 150M
ConcurrentDatabaseReload no
SelfCheck 1800
```

### 15.4.2 ClamAV扫描操作

**基础扫描命令：**

```bash
# 更新病毒库
freshclam

# 扫描指定目录
clamscan /home
clamscan /home --recursive -l /var/log/clamscan.log

# 扫描并删除感染文件
clamscan /home --remove

# 移动感染文件到隔离区
clamscan /home --move=/var quarantine

# 快速扫描系统关键目录
clamscan /etc /var /usr /bin /sbin --recursive -i

# 全面系统扫描（排除/proc和/sys）
clamscan / --recursive -i --exclude-dir="^/proc" --exclude-dir="^/sys" --exclude-dir="^/dev"

# 扫描后仅报告结果摘要
clamscan /home --report=/var/log/clamscan_report.txt
```

**使用clamdscan（后台扫描）：**

```bash
# 启动clamd服务
systemctl start clamav-daemon
systemctl enable clamav-daemon

# 使用clamdscan（更快，使用后台进程）
clamdscan /home
clamdscan --fdpass /home --infected --remove --multiscan

# 参数说明：
# --fdpass: 使用文件描述符权限
# --infected: 仅报告感染文件
# --remove: 删除感染文件
# --multiscan: 使用多个线程
```

**扫描脚本示例：**

```bash
#!/bin/bash
# /usr/local/bin/full-system-av-scan.sh

set -euo pipefail

LOG_DIR="/var/log/clamav"
REPORT_DIR="${LOG_DIR}/reports"
QUARANTINE_DIR="/var/clamav/quarantine"

mkdir -p "$LOG_DIR" "$REPORT_DIR" "$QUARANTINE_DIR"

TIMESTAMP=$(date +%Y%m%d_%H%M%S)
LOG_FILE="${LOG_DIR}/full_scan_${TIMESTAMP}.log"

echo "开始全面系统扫描 - ${TIMESTAMP}"

# 更新病毒库
echo "更新病毒库..."
freshclam >> "$LOG_FILE" 2>&1

# 扫描系统
echo "扫描系统关键目录..."
clamscan / \
    --recursive \
    --log="${LOG_FILE}" \
    --move="${QUARANTINE_DIR}" \
    --exclude-dir="^/proc" \
    --exclude-dir="^/sys" \
    --exclude-dir="^/dev" \
    --exclude-dir="^/var/lib/docker" \
    --exclude-dir="^/var/cache" \
    --exclude-dir="^/tmp" \
    --exclude-dir="^/run" \
    --max-dir-recursion=20

# 检查结果
INFECTED=$(grep -c "Infected" "$LOG_FILE" || echo "0")
echo ""
echo "扫描完成"
echo "感染文件数: $INFECTED"

# 发送报告
if [[ "$INFECTED" -gt 0 ]]; then
    SUBJECT="[安全警告] ClamAV发现${INFECTED}个感染文件"
    mail -s "$SUBJECT" -A "$LOG_FILE" admin@example.com << EOF
发现${INFECTED}个可能的恶意文件。

详细信息请查看附件日志。
EOF
fi

# 清理旧日志
find "$LOG_DIR" -name "*.log" -mtime +30 -delete
```

## 15.5 Chkrootkit检测

### 15.5.1 Chkrootkit概述

Chkrootkit是另一个广泛使用的rootkit检测工具，它可以检测约70种已知的rootkit，以及一些可疑的检测项，如测试隐藏进程、网络嗅探器等。与Rkhunter配合使用可以提供更全面的检测覆盖。

**安装Chkrootkit：**

```bash
# Debian/Ubuntu
apt-get install chkrootkit

# RHEL/CentOS
yum install chkrootkit

# 源码安装
cd /opt
wget ftp://ftp.chkrootkit.org/pub/seg/pac/chkrootkit.tar.gz
tar -xzf chkrootkit.tar.gz
cd chkrootkit-*
make sense
```

### 15.5.2 Chkrootkit使用

**基础扫描：**

```bash
# 运行完整检查
chkrootkit

# 静默模式
chkrootkit -q

# 详细输出
chkrootkit -v

# 检查特定项目
chkrootkit -x "strings" "bindshell" "rinse"

# 更新本地rootkit签名
chkrootkit -L | head -20
```

**常用检查项目：**

```bash
# 检查被绑定的可疑端口
chkrootkit -x port

# 检查wtmp和lastlog被篡改
chkrootkit -x wtmpx lastlog

# 检查sniffer
chkrootkit -x sniffer

# 检查promiscuous模式
chkrootkit -x promis

# 检查隐藏进程
chkrootkit -x hidden

# 检查rootkit
chkrootkit -x chkdirs chkproc
```

**扫描结果解读：**

```bash
# 运行并保存结果
chkrootkit > /var/log/chkrootkit.log 2>&1

# 分析结果
```

### 15.7.3 漏洞优先级评估

漏洞优先级评估需要综合考虑多个因素：

**CVSS评分系统：**

```
CVSS v3.1 计算公式：
Base Score = 
  Roundup(Minimum(Impact + Exploitability, 10))

Impact = 6.42 × ISC
  其中 ISC = (1 - (1-ConfImpact)×(1-IntegImpact)×(1-AvailImpact))

Exploitability = 8.22 × AttackVector × AttackComplexity × PrivilegesRequired × UserInteraction

AttackVector:
  Network (N) = 0.85
  Adjacent (A) = 0.62
  Local (L) = 0.55
  Physical (P) = 0.2
```

**漏洞优先级计算脚本：**

```bash
#!/bin/bash
# vulnerability-prioritizer.sh

# 输入：CVE列表文件，每行一个CVE ID
# 输出：按优先级排序的漏洞列表

CVE_LIST="${1:-cves.txt}"
OUTPUT_FILE=" prioritized_cves_$(date +%Y%m%d).txt"

echo "漏洞优先级评估报告" > "$OUTPUT_FILE"
echo "生成时间: $(date)" >> "$OUTPUT_FILE"
echo "======================================" >> "$OUTPUT_FILE"

while read cve; do
    # 获取CVE信息
    response=$(curl -s "https://services.nvd.nist.gov/rest/json/cve/1.0/${cve}" 2>/dev/null)
    
    if [[ -z "$response" ]]; then
        echo "无法获取 $cve 信息"
        continue
    fi
    
    # 提取CVSS评分
    cvss=$(echo "$response" | jq -r '.result.CVE_Items[0].impact.baseMetricV31.cvssData.baseScore' 2>/dev/null)
    severity=$(echo "$response" | jq -r '.result.CVE_Items[0].impact.baseMetricV31.cvssData.baseSeverity' 2>/dev/null)
    
    # 获取描述
    description=$(echo "$response" | jq -r '.result.CVE_Items[0].cve.description.description_data[0].value' 2>/dev/null)
    
    # 计算优先级
    case "$severity" in
        CRITICAL)
            priority=1
            ;;
        HIGH)
            priority=2
            ;;
        MEDIUM)
            priority=3
            ;;
        LOW)
            priority=4
            ;;
        *)
            priority=5
            ;;
    esac
    
    echo "$priority|$cve|$cvss|$severity|$description" >> "$OUTPUT_FILE"
    
done < "$CVE_LIST"

# 按优先级排序
sort -t'|' -k1 -n "$OUTPUT_FILE" -o "$OUTPUT_FILE"

echo "优先级| CVE | CVSS | 严重性 | 描述" | tee -a "$OUTPUT_FILE"
cat "$OUTPUT_FILE" | sort -t'|' -k1 -n | while IFS='|' read -r pri cve cvss sev desc; do
    echo "$pri| $cve | $cvss | $sev | ${desc:0:80}..."
done | tee -a "$OUTPUT_FILE"
```

### 15.8.2 修复实施步骤

**第一步：环境准备与备份**

```bash
#!/bin/bash
# prepare-fix-environment.sh

TARGET_SERVER="${1:-localhost}"
FIX_TYPE="${2:-package}"

echo "准备修复环境..."

# 创建备份目录
BACKUP_DIR="/var/backups/pre-fix-$(date +%Y%m%d_%H%M%S)"
mkdir -p "$BACKUP_DIR"

case "$FIX_TYPE" in
    package)
        # 备份当前安装的包列表
        dpkg --get-selections > "${BACKUP_DIR}/packages.list"
        # 或
        rpm -qa > "${BACKUP_DIR}/packages.list"
        ;;
    kernel)
        # 备份内核配置
        cp -a /boot/config-$(uname -r) "${BACKUP_DIR}/"
        lsmod > "${BACKUP_DIR}/modules.list"
        ;;
    config)
        # 备份配置文件
        cp -a /etc "${BACKUP_DIR}/etc.backup"
        ;;
esac

# 创建恢复脚本
cat > "${BACKUP_DIR}/rollback.sh" << 'EOF'
#!/bin/bash
# 自动回滚脚本
BACKUP_DIR=$(dirname "$0")

echo "开始回滚..."

case "$FIX_TYPE" in
    package)
        dpkg --set-selections < "${BACKUP_DIR}/packages.list"
        apt-get dselect-upgrade
        ;;
    kernel)
        # 恢复到特定内核版本
        grub2-set-default "Previous Linux versions"
        reboot
        ;;
    config)
        rm -rf /etc
        cp -a "${BACKUP_DIR}/etc.backup" /etc
        ;;
esac

echo "回滚完成"
EOF
chmod +x "${BACKUP_DIR}/rollback.sh"

echo "环境准备完成，备份保存到: $BACKUP_DIR"
echo "如需回滚，执行: ${BACKUP_DIR}/rollback.sh"
```

```bash
#!/bin/bash
# execute-fix.sh
# 生产环境修复执行脚本

set -euo pipefail

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*"
}

error_exit() {
    log "ERROR: $*" >&2
    exit 1
}

check_prerequisites() {
    # 检查root权限
    [[ $EUID -eq 0 ]] || error_exit "需要root权限"

}

main() {

    # 记录开始时间
    START_TIME=$(date +%s)
    
    # 执行修复
    log "更新软件包..."
    apt-get update -qq
    apt-get upgrade -y -qq
    
    # 记录结束时间

    log "修复完成，耗时: ${DURATION}秒"
```

```yaml
          assert:
            that:
              - "current_time.stdout >= maintenance_window_start"
              - "current_time.stdout <= maintenance_window_end"
            fail_msg: "当前不在维护窗口时间内"
            success_msg: "维护窗口验证通过"
      when: not force_patch|default(false)
    
    - name: 记录修复前状态
      shell: |
        echo "=== 修复前状态 ===" > /var/log/security-patch-{{ ansible_date_time.date }}.log
        dpkg -l | grep -E "{{ security_packages|join('|') }}" >> /var/log/security-patch-{{ ansible_date_time.date }}.log
        uname -r >> /var/log/security-patch-{{ ansible_date_time.date }}.log
      changed_when: false
      register: pre_patch_state
    
    # 更新软件源
    - name: 更新APT缓存
      apt:
        update_cache: yes
        cache_valid_time: 3600
      register: apt_update
    
    # 识别需要更新的包
    - name: 获取可更新的安全包
      shell: |
        apt list --upgradable 2>/dev/null | grep -E "security|openssl|openssh|linux" || true
      register: security_packages_to_update
      changed_when: false
      failed_when: false
    
    - name: 显示待更新包
      debug:
        msg: "需要更新的安全包:\n{{ security_packages_to_update.stdout }}"
    
    # 升级安全补丁
    - name: 升级安全补丁（不升级内核）
      apt:
        name: "{{ security_packages }}"
        state: latest
        only_upgrade: yes
      register: security_upgrade
      notify: 记录升级后状态
    
    # 处理内核升级（如需要）
    - name: 检查内核是否可以升级
      stat:
        path: /boot/vmlinuz-{{ new_kernel_version | default('latest') }}
      register: new_kernel_exists
      when: upgrade_kernel|default(false)
      failed_when: false
    
    - name: 升级Linux内核
      apt:
        name: 
          - "linux-image-{{ kernel_package_suffix | default('generic') }}"
          - "linux-headers-{{ kernel_package_suffix | default('generic') }}"
        state: latest
      register: kernel_upgrade
      when: upgrade_kernel|default(false) and new_kernel_exists.stat.exists
      notify: 内核升级后处理
    
    # 清理
    - name: 清理缓存
      apt:
        autoremove: yes
        purge: no
      register: apt_cleanup
    
    # 验证修复
    - name: 验证关键服务状态
      service:
        name: "{{ item }}"
        state: started
        enabled: yes
      loop:
        - ssh
        - nginx
        - cron
    
    - name: 验证SSH服务
      command: systemctl is-active ssh
      register: ssh_status
      failed_when: ssh_status.rc != 0
      changed_when: false
    
    # 检查是否需要重启
    - name: 检查是否需要重启
      command: needs-restarting -r
      register: needs_restart
      failed_when: false
      changed_when: false
    
    - name: 设置重启标记
      set_fact:
        reboot_required: true
      when: needs_restart.rc == 1
    
    - name: 记录修复结果
      lineinfile:
        path: /var/log/security-patch-{{ ansible_date_time.date }}.log
        line: "{{ item }}"
      loop:
        - "修复完成时间: {{ ansible_date_time.iso8601 }}"
        - "安全包更新: {{ '成功' if security_upgrade.changed else '无需更新' }}"
        - "内核更新: {{ '成功' if kernel_upgrade.changed else '无需更新' }}"
        - "需要重启: {{ reboot_required }}"
    
    # 发送通知
    - name: 发送修复报告
      mail:
        host: smtp.example.com
        port: 587
        username: "{{ mail_user }}"
        password: "{{ mail_password }}"
        to: "{{ notification_email }}"
        subject: "[安全修复报告] {{ inventory_hostname }} - {{ ansible_date_time.date }}"
        body: |
          服务器: {{ inventory_hostname }}
          修复时间: {{ ansible_date_time.iso8601 }}
          
          修复摘要:
          {% if security_upgrade.changed %}
          - 已更新 {{ security_upgrade.results|length }} 个安全包
          {% endif %}
          {% if kernel_upgrade is defined and kernel_upgrade.changed %}
          - 已升级内核
          {% endif %}
          - 需要重启: {{ reboot_required }}
          
          详细日志: /var/log/security-patch-{{ ansible_date_time.date }}.log
      when: send_notification|default(true)
    
  handlers:
    - name: 记录升级后状态
      shell: |
        echo "=== 修复后状态 ===" >> /var/log/security-patch-{{ ansible_date_time.date }}.log
        dpkg -l | grep -E "{{ security_packages|join('|') }}" >> /var/log/security-patch-{{ ansible_date_time.date }}.log
      changed_when: false
    
    - name: 内核升级后处理
      block:
        - name: 更新GRUB
          command: update-grub
          changed_when: false
        
        - name: 设置重启标记
          set_fact:
            reboot_required: true
      when: true
    
    - name: 安全重启
      reboot:
        reboot_timeout: 600
        pre_reboot_delay: 30
        post_reboot_delay: 60
        test_command: uptime
      when: reboot_required and auto_reboot|default(false)
```

**批量执行脚本：**

```bash
#!/bin/bash
# execute-batch-patch.sh

set -euo pipefail

INVENTORY="${1:-inventory/production.ini}"
PLAYBOOK="security-patch-playbook.yml"
LOG_DIR="/var/log/ansible-patch"
LIMIT="${2:-}"

mkdir -p "$LOG_DIR"

TIMESTAMP=$(date +%Y%m%d_%H%M%S)
LOG_FILE="${LOG_DIR}/patch-${TIMESTAMP}.log"

echo "开始批量安全修复..."
echo "时间: $(date)"
echo "清单: $INVENTORY"

# 先在测试服务器验证
echo "=== 在测试服务器验证 ==="
ansible-playbook -i "${INVENTORY}" \
    --limit "test-server" \
    --tags "pre-check" \
    "$PLAYBOOK" \
    --check --diff \
    2>&1 | tee "${LOG_FILE}.check"

# 执行批量修复（分批进行）
echo "=== 执行批量修复 ==="

if [[ -n "$LIMIT" ]]; then
    # 只修复指定服务器
    ansible-playbook -i "${INVENTORY}" \
        --limit "$LIMIT" \
        "$PLAYBOOK" \
        --extra-vars "auto_reboot=false send_notification=true" \
        2>&1 | tee -a "$LOG_FILE"
else
    # 分批修复，每批10台
    ansible-playbook -i "${INVENTORY}" \
        --limit "server[01:10]" \
        "$PLAYBOOK" \
        --extra-vars "auto_reboot=false send_notification=true" \
        2>&1 | tee -a "${LOG_FILE}.batch1"
    
    # 等待5分钟
    sleep 300
    
    ansible-playbook -i "${INVENTORY}" \
        --limit "server[11:30]" \
        "$PLAYBOOK" \
        --extra-vars "auto_reboot=false send_notification=true" \
        2>&1 | tee -a "${LOG_FILE}.batch2"
    
    # 等待5分钟
    sleep 300
    
    ansible-playbook -i "${INVENTORY}" \
        --limit "server[31:50]" \
        "$PLAYBOOK" \
        --extra-vars "auto_reboot=false send_notification=true" \
        2>&1 | tee -a "${LOG_FILE}.batch3"
fi

# 生成汇总报告
echo ""
echo "=== 修复汇总 ==="
grep -E "(PLAY RECAP|task:)" "${LOG_FILE}"* | tail -100

echo ""
echo "详细日志: $LOG_FILE*"
echo "修复完成: $(date)"
```
