# 第二十三章 等级保护合规实践

## 23.1 等级保护概述

### 23.1.1 等级保护制度

等级保护（等保）是中国网络安全的基本制度，对信息和信息系统按照重要程度进行分级保护。

**等级划分**

| 等级 | 名称 | 适用系统 | 保护重点 |
|------|------|---------|---------|
| 第一级 | 自主保护级 | 小型私营企业系统 | 基本防护 |
| 第二级 | 指导保护级 | 县级重要系统 | 指导性防护 |
| 第三级 | 监督保护级 | 市级重要系统、地市级重要系统 | 强制性防护 |
| 第四级 | 强制保护级 | 省级重要系统 | 专门化防护 |
| 第五级 | 专控保护级 | 国家级重要系统 | 最高防护 |

### 23.1.2 等级保护2.0新特点

等保2.0（GB/T 22239-2019）相比1.0版本：

- **扩大保护范围**：从传统信息系统扩展到云计算、移动互联、物联网、工业控制系统
- **强化主动防御**：从被动防护向主动防护转变
- **突出可信验证**：强化可信计算技术应用
- **强化供应链安全**：强调产品和服务的供应链安全

## 23.3 安全区域边界

### 23.3.1 边界防护

```bash
# 防火墙配置
# 等保三级要求配置边界防火墙

# iptables基础配置
*filter
:INPUT DROP [0:0]
:FORWARD DROP [0:0]
:OUTPUT ACCEPT [0:0]

# 允许loopback
-A INPUT -i lo -j ACCEPT

# 允许已建立连接
-A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# SSH管理
-A INPUT -p tcp -s 10.0.0.0/8 --dport 22 -j ACCEPT

# HTTP/HTTPS
-A INPUT -p tcp --dport 80 -j ACCEPT
-A INPUT -p tcp --dport 443 -j ACCEPT

# DNS
-A INPUT -p udp --dport 53 -j ACCEPT

# ICMP（限制）
-A INPUT -p icmp --icmp-type echo-request -m limit --limit 1/s -j ACCEPT

# 记录日志
-A INPUT -m limit --limit 5/min -j LOG --log-prefix "IPT_INPUT_DROP: "

COMMIT

# nftables配置示例
nft add table ip filter
nft add chain ip filter input { type filter hook input priority 0 \; policy drop \; }

nft add rule ip filter input iif lo accept
nft add rule ip filter input ct state established,related accept
nft add rule ip filter input tcp dport ssh accept
nft add rule ip filter input tcp dport { http, https } accept
```

### 23.3.4 恶意代码防范

```bash
# 安装ClamAV
yum install clamav clamav-update -y

# 更新病毒库
freshclam

# 创建定时扫描脚本
cat > /usr/local/bin/malware-scan.sh << 'EOF'
#!/bin/bash
# 恶意代码扫描脚本

SCAN_DIR="/var/www"
LOG_DIR="/var/log/clamav"
QUARANTINE="/var/lib/clamav/quarantine"
DATE=$(date +%Y%m%d)

mkdir -p "$LOG_DIR" "$QUARANTINE"

# 全盘扫描
clamscan -r / --move="$QUARANTINE" --log="$LOG_DIR/scan-$DATE.log" \
    --exclude-dir=/proc --exclude-dir=/sys --exclude-dir=/dev

# 扫描特定目录
clamscan -r "$SCAN_DIR" --remove --log="$LOG_DIR/www-scan-$DATE.log"

# 发送报告
if [ -f "$LOG_DIR/scan-$DATE.log" ]; then
    INFECTED=$(grep -c "FOUND" "$LOG_DIR/scan-$DATE.log")
    if [ "$INFECTED" -gt 0 ]; then
        echo "发现 $INFECTED 个恶意文件" | mail -s "ClamAV扫描告警" admin@example.com
    fi
fi
EOF

chmod +x /usr/local/bin/malware-scan.sh

# 添加到crontab（每日扫描）
echo "0 3 * * * root /usr/local/bin/malware-scan.sh" >> /etc/crontab
```

### 23.4.3 安全审计

```bash
# 审计配置
# 参考第4章

# 启用审计服务
systemctl enable auditd
systemctl start auditd

# 配置审计规则
cat > /etc/audit/rules.d/security.rules << 'EOF'
# 记录所有sudo命令
-a always,exit -F arch=b64 -S execve -F path=/usr/bin/sudo -F auid>=1000 -F auid!=4294967295 -k privilegedsudo

-w /etc/shadow -p wa -k shadow_changes
-w /etc/sudoers -p wa -k sudoers_changes
-w /etc/ssh/sshd_config -p wa -k sshd_config_changes

# 记录网络配置
-w /etc/sysconfig/network -p wa -k network_changes
-w /etc/sysconfig/network-scripts/ -p wa -k network_scripts_changes

# 记录用户认证
-w /var/log/faillog -p wa -k failed_logins
-w /var/log/lastlog -p wa -k login_records
EOF

# 重载审计规则
auditctl -R /etc/audit/rules.d/security.rules

# 审计日志分析
cat > /usr/local/bin/audit-analysis.sh << 'EOF'
#!/bin/bash
# 审计日志分析脚本

ALERT_EMAIL="security@example.com"
LOG_DIR="/var/log/audit"

# 报告生成

ausearch -k sudoers_changes --format text > /tmp/sudoers_changes.txt
ausearch -k sshd_config_changes --format text > /tmp/sshd_changes.txt

# 检查异常
if [ -s /tmp/passwd_changes.txt ]; then
    echo "检测到/etc/passwd修改!" | mail -s "审计告警" $ALERT_EMAIL

if [ -s /tmp/sudoers_changes.txt ]; then
    echo "检测到sudoers配置修改!" | mail -s "审计告警" $ALERT_EMAIL
fi

# 生成日报告
aureport --summary > /var/log/audit/daily-summary-$(date +%Y%m%d).txt
EOF

chmod +x /usr/local/bin/audit-analysis.sh

# 添加到crontab
```

## 23.7 实战案例：等保三级安全加固

### 23.7.1 案例背景

某企业业务系统定级为等保三级，需要进行全面的安全加固。

### 23.7.2 安全加固实施

**第一步：安全评估**

```bash
# 创建安全评估脚本
cat > /usr/local/bin/security-assessment.sh << 'EOF'
#!/bin/bash
# 等保三级安全评估脚本

REPORT="/var/log/security-assessment-$(date +%Y%m%d).html"

echo "<html><head><title>等保三级安全评估报告</title></head><body>" > "$REPORT"
echo "<h1>等保三级安全评估报告</h1>" >> "$REPORT"
echo "<p>评估时间: $(date)</p>" >> "$REPORT"
echo "<p>服务器: $(hostname)</p>" >> "$REPORT"

# 1. 网络安全评估
echo "<h2>1. 网络安全评估</h2>" >> "$REPORT"
echo "<pre>" >> "$REPORT"
echo "防火墙状态:" >> "$REPORT"
iptables -L -n --line-numbers >> "$REPORT" 2>&1
echo "网络连接:" >> "$REPORT"
ss -tulnp >> "$REPORT" 2>&1
echo "</pre>" >> "$REPORT"

# 2. 系统安全评估
echo "<h2>2. 系统安全评估</h2>" >> "$REPORT"
echo "<pre>" >> "$REPORT"
echo "用户账户:" >> "$REPORT"
awk -F: '{print $1":"$3":"$7}' /etc/passwd >> "$REPORT"
echo "SUID文件数量:" >> "$REPORT"
find / -perm -4000 -type f 2>/dev/null | wc -l >> "$REPORT"
echo "密码策略:" >> "$REPORT"
grep pam_pwquality /etc/pam.d/* >> "$REPORT" 2>&1
echo "</pre>" >> "$REPORT"

# 3. 应用安全评估
echo "<h2>3. 应用安全评估</h2>" >> "$REPORT"
echo "<pre>" >> "$REPORT"
echo "运行服务:" >> "$REPORT"
systemctl list-units --type=service --state=running >> "$REPORT" 2>&1
echo "SSH配置:" >> "$REPORT"
grep -E "Protocol|PermitRootLogin|PasswordAuthentication" /etc/ssh/sshd_config >> "$REPORT" 2>&1
echo "</pre>" >> "$REPORT"

echo "</body></html>" >> "$REPORT"

echo "评估报告已生成: $REPORT"
EOF

chmod +x /usr/local/bin/security-assessment.sh
```

**第二步：网络安全加固**

```bash
# 部署防火墙
cat > /etc/systemd/system/firewall.service << 'EOF'
[Unit]
Description=Custom Firewall Rules
After=network.target

[Service]
Type=oneshot
ExecStart=/usr/local/bin/firewall-setup.sh
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target
EOF

cat > /usr/local/bin/firewall-setup.sh << 'EOF'
#!/bin/bash
# 等保三级防火墙配置

# 清除现有规则
iptables -F
iptables -X
iptables -t nat -F

# 设置默认策略
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

# 允许loopback
iptables -A INPUT -i lo -j ACCEPT
iptables -A OUTPUT -o lo -j ACCEPT

# 允许已建立连接
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# SSH管理（限定管理IP）
iptables -A INPUT -p tcp -s 10.0.0.0/8 --dport 22 -j ACCEPT

# HTTP/HTTPS
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# DNS
iptables -A INPUT -p udp --dport 53 -j ACCEPT
iptables -A INPUT -p tcp --dport 53 -j ACCEPT

# ICMP限制
iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 1/s -j ACCEPT

# 记录日志
iptables -A INPUT -j LOG --log-prefix "IPT_INPUT_DROP: "

# 保存规则
iptables-save > /etc/sysconfig/iptables
EOF

chmod +x /usr/local/bin/firewall-setup.sh
systemctl enable firewall.service
systemctl start firewall.service
```

```bash
# 请求限制
limit_req_zone $binary_remote_addr zone=req_limit:10m rate=10r/s;
limit_conn_zone $binary_remote_addr zone=conn_limit:10m;
EOF

# MySQL安全配置
cat > /etc/my.cnf.d/security.cnf << 'EOF'
[mysqld]
# 禁用危险函数
# secure-file-priv=/var/lib/mysql
# local-infile=0

# SSL配置

ssl-cert=/etc/pki/mysql/server.crt
ssl-key=/etc/pki/mysql/server.key
require-secure-transport=ON

# 日志
log_error=/var/log/mysql/error.log
general_log=0
EOF

systemctl restart nginx
systemctl restart mysqld
```
