# 第12章 邮件服务安全加固

## 12.1 邮件服务安全概述

邮件服务是企业沟通和信息传递的重要基础设施，同时也面临多种安全威胁：垃圾邮件泛滥、钓鱼攻击、邮件伪造、数据泄露、邮件病毒、服务器被入侵用于发送垃圾邮件等。邮件系统的安全性直接关系到企业的信息安全和声誉。

常见的邮件服务软件包括Postfix、Sendmail、Dovecot、Courier等。Postfix是最流行的MTA（邮件传输代理），以其安全性和易配置著称；Dovecot是最常用的IMAP/POP3服务器；Sendmail是历史悠久的MTA，现在使用较少。

邮件服务安全加固的要点包括：服务访问控制、TLS加密传输、用户认证、反垃圾邮件措施、发件人验证（SPF/DKIM/DMARC）、邮件审计和日志分析。本章将详细介绍这些安全措施的配置。

## 12.2 Postfix安全配置

### 12.2.1 Postfix安装与基础配置

```bash
# 安装Postfix
yum install -y postfix postfix-systemmail          # CentOS/RHEL
apt-get install -y postfix postfix-mysql postfix-ldap  # Debian/Ubuntu

# 切换默认MTA
alternatives --set mta /usr/sbin/postfix

# 启动Postfix
systemctl enable postfix
systemctl start postfix

# 检查配置
postfix check

# 查看Postfix版本
postconf -d mail_version
```

### 12.2.2 Postfix基础安全配置

```bash
# /etc/postfix/main.cf

# 基础配置
myhostname = mail.example.com
mydomain = example.com
myorigin = $mydomain
inet_interfaces = localhost
inet_protocols = ipv4

# 邮件存储
home_mailbox = Maildir/
mailbox_command = /usr/bin/procmail

# 安全相关配置
# 禁用VRFY命令（防止枚举用户）
disable_vrfy_command = yes

# 限制邮件大小（10MB）
message_size_limit = 10485760

# 限制邮箱大小（1GB）
mailbox_size_limit = 1073741824

# 禁止显示Postfix版本
smtpd_banner = $myhostname ESMTP

# SASL认证配置
smtpd_sasl_auth_enable = yes
smtpd_sasl_type = dovecot
smtpd_sasl_path = private/auth
smtpd_sasl_security_options = noanonymous
smtpd_sasl_local_domain = $mydomain
broken_sasl_auth_clients = yes

# TLS配置
smtpd_use_tls = yes
smtpd_tls_cert_file = /etc/pki/tls/certs/mail.example.com.crt
smtpd_tls_key_file = /etc/pki/tls/private/mail.example.com.key
smtpd_tls_security_level = may
smtpd_tls_auth_only = yes
smtpd_tls_received_header = yes
smtpd_tls_session_cache_timeout = 3600s
tls_random_source = dev:/dev/urandom

# 访问控制
smtpd_client_restrictions =
    permit_mynetworks,
    permit_sasl_authenticated,
    reject_rbl_client zen.spamhaus.org,
    reject_rbl_client bl.spamcop.net,
    reject_unknown_client_hostname,
    permit

smtpd_helo_restrictions =
    reject_invalid_helo_hostname,
    reject_non_fqdn_helo_hostname,
    reject_unknown_helo_hostname,
    permit

smtpd_sender_restrictions =
    reject_non_fqdn_sender,
    reject_unknown_sender_domain,
    reject_sender_login_mismatch,
    permit_sasl_authenticated,
    permit

smtpd_recipient_restrictions =
    reject_non_fqdn_recipient,
    reject_unknown_recipient_domain,
    reject_unlisted_recipient,
    permit_mynetworks,
    permit_sasl_authenticated,
    reject

smtpd_data_restrictions =
    reject_unauth_pipelining,
    permit
```

### 12.2.4 Postfix加密配置

```bash
# 1. 生成SSL证书
mkdir -p /etc/pki/tls/private
chmod 700 /etc/pki/tls/private

openssl req -x509 -newkey rsa:2048 -days 365 \
    -keyout /etc/pki/tls/private/mail.example.com.key \
    -out /etc/pki/tls/certs/mail.example.com.crt \
    -nodes -subj "/C=CN/ST=Beijing/L=Beijing/O=Example/CN=mail.example.com"

# /etc/postfix/main.cf

# 推荐使用STARTTLS（端口587）

smtpd_tls_key_file = /etc/pki/tls/private/mail.example.com.key

# 强制TLS对于某些域名
# /etc/postfix/tls_per_site
# mail.gmail.com     MUST
# mail.yahoo.com     MAY

tls_per_site_must_sites = hash:/etc/postfix/tls_per_site
tls_per_site_maybe_sites = hash:/etc/postfix/tls_per_site

# 3. 创建tls_per_site
# /etc/postfix/tls_per_site
# gmail-smtp-in.l.google.com   MUST
# smtp.gmail.com                MUST

# 4. 测试TLS连接
openssl s_client -connect localhost:25 -starttls smtp
```

## 12.5 SPF/DKIM/DMARC配置

### 12.5.1 SPF记录配置

SPF（Sender Policy Framework）用于声明哪些邮件服务器可以代表您的域名发送邮件：

```bash
# DNS中添加SPF记录
# 类型：TXT
# 值：
v=spf1 mx ip4:203.0.113.10 include:_spf.example.com ~all

# 解释：
# v=spf1        SPF版本
# mx            允许MX记录中的邮件服务器发送邮件
# ip4:203.0.113.10  允许特定IP发送
# include:_spf.example.com  包含其他域名的SPF记录
# ~all          软失败（建议生产环境使用）
# -all          硬失败（严格模式）
# ?all          中性（不做限制）

# 更严格的配置示例
v=spf1 mx a:mail.example.com ip4:203.0.113.10/32 -all

# 使用第三方邮件服务（如SendGrid）
v=spf1 include:sendgrid.net ~all

# 多域名配置
# 在每个域名的DNS中添加相应的SPF记录
```

### 12.5.2 DKIM配置

DKIM（DomainKeys Identified Mail）使用公钥加密技术为邮件添加数字签名：

```bash
# 1. 安装OpenDKIM
yum install -y opendkim opendkim-tools   # CentOS/RHEL
apt-get install -y opendkim opendkim-tools  # Debian/Ubuntu

# 2. 生成DKIM密钥
mkdir -p /etc/opendkim/keys
chown opendkim:opendkim /etc/opendkim/keys
chmod 700 /etc/opendkim/keys

# 生成1024位密钥（推荐）
opendkim-genkey -D /etc/opendkim/keys/ -d example.com -s default -b 2048

# 重命名并设置权限
mv /etc/opendkim/keys/default.private /etc/opendkim/keys/default
chown opendkim:opendkim /etc/opendkim/keys/default
chmod 600 /etc/opendkim/keys/default
chmod 644 /etc/opendkim/keys/default.txt

# 3. 配置OpenDKIM
# /etc/opendkim/opendkim.conf

AutoRestart         yes
AutoRestartRate     10/1M
Background          yes
Canonicalization    relaxed/simple
DNSTimeout          5
KeyTable            refile:/etc/opendkim/key.table
LogWhy              yes
OversignHeaders     From
SignatureAlgorithm   rsa-sha256
SigningTable        refile:/etc/opendkim/signing.table
Socket              inet:8891@localhost
Syslog              yes
SyslogSuccess       yes
UMask              022
UserID              opendkim:opendkim

# 4. 创建密钥表
# /etc/opendkim/key.table
default._domainkey.example.com example.com:default:/etc/opendkim/keys/default

# 5. 创建签名表
# /etc/opendkim/signing.table
example.com default._domainkey.example.com
*@example.com default._domainkey.example.com

# 6. 创建信任密钥表
# /etc/opendkim/trusted.hosts
127.0.0.1
::1
localhost
*.example.com

# 7. 配置Postfix使用OpenDKIM
# /etc/postfix/main.cf
smtpd_milters = inet:localhost:8891
non_smtpd_milters = inet:localhost:8891
milter_default_action = accept
milter_protocol = 6

# 8. 重启服务
systemctl restart opendkim
systemctl enable opendkim
systemctl restart postfix

# 9. 在DNS中添加TXT记录
# 查看公钥
cat /etc/opendkim/keys/default.txt

# DNS记录名称：default._domainkey.example.com
# DNS记录值：v=DKIM1; k=rsa; p=MIGfMA0GCSqGSIb3DQEBAQUAA...
```

### 12.5.3 DMARC配置

DMARC（Domain-based Message Authentication, Reporting and Conformance）在SPF和DKIM基础上提供报告机制：

```bash
# DNS中添加DMARC记录
# 类型：TXT
# 域名：_dmarc.example.com
# 值：
v=DMARC1; p=quarantine; rua=mailto:dmarc-reports@example.com; ruf=mailto:dmarc-reports@example.com; pct=100

# 参数说明：
# v=DMARC1         版本
# p=               策略：none（监控）、quarantine（隔离可疑邮件）、reject（拒绝）
# rua              聚合报告接收邮箱
# ruf              详细故障报告接收邮箱（可选）
# pct               적용百分比（100表示全部）
# sp=              子域策略
# adkim=           DKIM对齐模式（r=relaxed, s=strict）
# aspf=            SPF对齐模式（r=relaxed, s=strict）

# 建议配置顺序：
# 第一阶段：p=none（监控）
# 第二阶段：p=quarantine（隔离）
# 第三阶段：p=reject（拒绝）
```

## 12.6 邮件加密（PGP、S/MIME）

### 12.6.1 GPG安装和基本使用

```bash
# 安装GPG
yum install -y gnupg2          # CentOS/RHEL
apt-get install -y gnupg2      # Debian/Ubuntu

# 生成密钥
gpg --full-generate-key
# 选择：
# 算法：RSA and RSA
# 密钥长度：4096
# 过期时间：2年
# 用户ID：真实姓名和邮箱
# 密码：强密码

# 列出密钥
gpg --list-keys
gpg --list-secret-keys

# 导出公钥
gpg --armor --export user@example.com > public_key.asc

# 导出私钥（谨慎保管）
gpg --armor --export-secret-keys user@example.com > private_key.asc

# 导入公钥
gpg --import colleague_public_key.asc

# 信任导入的公钥
gpg --edit-key colleague@example.com
# 在编辑模式输入：trust
# 选择信任级别：5（完全信任）
```

### 12.6.2 GPG加密和解密

```bash
# 加密文件（使用接收者公钥）
gpg --encrypt --sign --armor --recipient user@example.com document.txt

# 等效于：
gpg --encrypt --recipient user@example.com document.txt
# --sign 添加签名
# --armor 输出ASCII格式
# -e -s -a -r user@example.com document.txt

# 解密文件
gpg --decrypt encrypted_message.asc

# 解密并验证签名
gpg --verify signed_message.sig

# 加密目录（先打包）
tar czf - directory/ | gpg --encrypt --recipient user@example.com | gpg --armor > directory.tar.gz.gpg
```

### 12.6.3 S/MIME配置

```bash
# 安装certutil（用于管理证书）
yum install -y nss-tools   # CentOS/RHEL
apt-get install -y libnss3  # Debian/Ubuntu

# 生成自签名S/MIME证书
# 需要先安装openssl

# 1. 生成私钥和证书签名请求
openssl req -new -newkey rsa:2048 -nodes \
    -keyout email.key -out email.csr \
    -subj "/C=CN/ST=Beijing/L=Beijing/O=Example/OU=IT/CN=user@example.com"

# 2. 使用OpenSSL生成自签名证书
openssl x509 -req -days 365 \
    -in email.csr -signkey email.key \
    -out email.crt

# 3. 转换为PKCS12格式（大多数邮件客户端使用）
openssl pkcs12 -export \
    -in email.crt -inkey email.key \
    -out email.p12 \
    -name "User Email Certificate"

# 4. 导出公钥供他人使用
openssl pkcs12 -in email.p12 -nokeys -out email_pub.crt

# Thunderbird导入S/MIME证书：
# 账户设置 -> 安全 -> 管理证书 -> 导入

# Outlook导入S/MIME证书：
# 文件 -> 选项 -> 信任中心 -> 电子邮件安全 -> 导入/导出
```

## 12.7 Postfix反垃圾邮件配置

### 12.7.1 安装和配置SpamAssassin

```bash

# 语言和编码

loadplugin Mail::SpamAssassin::Plugin::Razor2

# Razor配置

# 启动SpamAssassin
systemctl enable spamassassin
systemctl start spamassassin
```

### 12.7.2 配置Postfix集成SpamAssassin

```bash
# /etc/postfix/master.cf

# 添加spamassassin过滤器
spamassassin unix -     n       n       -       -       user=spamd argv=/usr/bin/spamc -f -e /usr/sbin/sendmail -oi -f ${sender} ${recipient}

# 修改Postfix主配置
# /etc/postfix/main.cf

content_filter = spamassassin

# 在main.cf中添加
smtp      inet  n       -       n       -       -       smtpd
    -o content_filter=spamassassin

# 或使用smtpd代理
spamassassin unix - n n - - pipe
    user=spamd argv=/usr/bin/spamc -s 50000000 --max-size=50000000 -e /usr/sbin/sendmail -oi -f ${sender} ${recipient}

# 重启Postfix
systemctl restart postfix
```

### 12.11.4 系统加固

```bash
#!/bin/bash
# secure_mail.sh - 邮件服务器安全加固脚本

echo "Starting mail server security hardening..."

# 1. 重置所有邮件账户密码
# 生成随机密码
NEW_PASS=$(openssl rand 12 | base64 | tr -d '/+=' | head -c 16)

# 如果使用系统用户
for user in $(grep -E "imappop" /etc/passwd | cut -d: -f1); do
    echo "$user:$NEW_PASS" | chpasswd
    echo "Password reset for $user"
done

# 如果使用虚拟用户（MySQL）
mysql -u postfix -p'PostfixDBP@ss!' postfix << 'EOF'
UPDATE virtual_users SET password='$NEW_PASS' WHERE email='user@example.com';
EOF

# 2. 修复Postfix配置
cat > /etc/postfix/main.cf << 'EOF'
# 网络配置
inet_interfaces = 127.0.0.1, 192.168.1.10
inet_protocols = ipv4

# 禁用Open Relay
mynetworks = 127.0.0.0/8, 192.168.1.0/24
relay_domains = 
smtpd_relay_restrictions = permit_mynetworks, permit_sasl_authenticated, reject

# 启用SASL认证
smtpd_sasl_auth_enable = yes
smtpd_sasl_type = dovecot
smtpd_sasl_path = private/auth
smtpd_sasl_local_domain = $mydomain
broken_sasl_auth_clients = yes
smtpd_sasl_security_options = noanonymous
smtpd_sasl_authenticated_header = no

# TLS配置
smtp_use_tls = yes
smtpd_use_tls = yes
smtpd_tls_cert_file = /etc/pki/tls/certs/mail.example.com.crt
smtpd_tls_key_file = /etc/pki/tls/private/mail.example.com.key
smtpd_tls_security_level = may
smtpd_tls_auth_only = yes
smtpd_tls_received_header = yes
tls_random_source = dev:/dev/urandom

# 安全设置
disable_vrfy_command = yes
smtpd_helo_required = yes
strict_rfc821_envelopes = yes
masquerade_domains = example.com

# RBL黑名单
smtpd_client_restrictions =
    permit_mynetworks,
    permit_sasl_authenticated,
    reject_rbl_client zen.spamhaus.org,
    reject_rbl_client bl.spamcop.net,
    reject_unknown_client_hostname,
    permit

# 发送限制
smtp_destination_rate_delay = 1s
default_destination_rate_delay = 1s
smtpd_client_connection_count_limit = 10
smtpd_client_connection_rate_limit = 30
message_size_limit = 10485760
mailbox_size_limit = 1073741824

# 日志
maillog_file = /var/log/maillog
log_level = info
EOF

# 3. 配置Dovecot SSL
cat > /etc/dovecot/conf.d/10-ssl.conf << 'EOF'
ssl = required
ssl_cert = </etc/pki/tls/certs/mail.example.com.crt
ssl_key = </etc/pki/tls/private/mail.example.com.key
ssl_min_protocol = TLSv1.2
ssl_prefer_server_ciphers = yes
ssl_protocols = !SSLv2 !SSLv3 !TLSv1.0 !TLSv1.1
EOF

# 4. 配置防火墙
iptables -F
iptables -A INPUT -i lo -j ACCEPT
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 允许SMTP
iptables -A INPUT -p tcp -s 192.168.1.0/24 --dport 25 -j ACCEPT
iptables -A INPUT -p tcp -s 192.168.1.0/24 --dport 465 -j ACCEPT
iptables -A INPUT -p tcp -s 192.168.1.0/24 --dport 587 -j ACCEPT

# 允许IMAP/POP3
iptables -A INPUT -p tcp -s 192.168.1.0/24 --dport 143 -j ACCEPT
iptables -A INPUT -p tcp -s 192.168.1.0/24 --dport 993 -j ACCEPT
iptables -A INPUT -p tcp -s 192.168.1.0/24 --dport 995 -j ACCEPT

# 拒绝其他所有访问
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

# 保存规则
iptables-save > /etc/iptables/rules.v4

# 5. 启用SpamAssassin
systemctl enable spamassassin
systemctl start spamassassin

# 6. 启用OpenDKIM
systemctl enable opendkim
systemctl start opendkim

# 7. 配置fail2ban防止暴力破解
cat > /etc/fail2ban/jail.d/mail.conf << 'EOF'
[postfix]
enabled = true
port = smtp,465,587
filter = postfix
logpath = /var/log/maillog
maxretry = 5
bantime = 3600

[dovecot]
enabled = true
port = imap,993,pop3,995
filter = dovecot
logpath = /var/log/maillog
maxretry = 5
bantime = 3600
EOF

systemctl enable fail2ban
systemctl restart fail2ban

# 8. 重启服务
systemctl restart postfix
systemctl restart dovecot

# 9. 验证配置
echo ""
echo "=== Verification ==="
echo "1. Testing SMTP connection..."
timeout 5 openssl s_client -connect localhost:25 -starttls smtp </dev/null 2>&1 | grep -E "Protocol|Cipher"

echo ""
echo "2. Testing SASL authentication..."
echo "AUTH LOGIN" | openssl s_client -connect localhost:25 -starttls smtp 2>/dev/null | head -5

echo ""
echo "3. Checking mail queue..."
mailq

echo ""
echo "=== Security Hardening Completed ==="
```

```bash
smtpd_helo_restrictions = permit_mynetworks, reject_invalid_helo_hostname, reject_non_fqdn_helo_hostname
smtpd_sender_restrictions = permit_mynetworks, reject_non_fqdn_sender, reject_sender_login_mismatch
smtpd_recipient_restrictions = permit_mynetworks, permit_sasl_authenticated, reject_unauth_destination, reject_non_fqdn_recipient

# 速率限制
smtpd_client_connection_count_limit = 10
smtpd_client_connection_rate_limit = 30
smtpd_client_message_rate_limit = 100

# 防垃圾邮件
header_checks = regexp:/etc/postfix/header_checks
mime_header_checks = regexp:/etc/postfix/mime_header_checks

# 禁用VRFY命令

# 隐藏版本信息
smtpd_banner = $myhostname ESMTP
```
