# 第二十章 代理服务器安全

## 20.1 代理服务器概述

### 20.1.1 代理服务器的作用

代理服务器作为客户端和服务器之间的中介，在网络安全中扮演重要角色：

- **访问控制**：控制对特定网站的访问
- **内容过滤**：过滤恶意流量和不良内容
- **隐私保护**：隐藏用户真实IP地址
- **缓存加速**：缓存常用内容提高访问速度
- **日志审计**：记录网络访问行为

### 20.1.2 代理服务器类型

1. **正向代理（Forward Proxy）**
   - 代理客户端访问外部服务器
   - 典型应用：翻墙、访问控制

2. **反向代理（Reverse Proxy）**
   - 代理服务器响应客户端请求
   - 典型应用：负载均衡、隐藏后端服务器

3. **透明代理（Transparent Proxy）**
   - 无需客户端配置的代理
   - 典型应用：企业网络监控

4. **SOCKS代理**
   - 更通用的代理协议
   - 支持TCP和UDP流量

## 20.2 Squid代理服务器安全配置

### 20.2.1 Squid基础配置

```bash
# 安装Squid
yum install squid -y    # CentOS
apt install squid -y    # Ubuntu

# 备份原始配置
cp /etc/squid/squid.conf /etc/squid/squid.conf.bak

# 主配置文件
cat > /etc/squid/squid.conf << 'EOF'
# 基础配置
http_port 3128
visible_hostname proxy.example.com

# 日志配置
logformat squid %ts.%03tu %6tr %>a %Ss/%03Hs %<st %rm %ru %un %Sh/%<A %mt
access_log /var/log/squid/access.log squid
cache_log /var/log/squid/cache.log
cache_store_log /var/log/squid/store.log

# 内存和缓存配置
cache_mem 256 MB
maximum_object_size_in_memory 512 KB
cache_dir ufs /var/spool/squid 10000 16 256
maximum_object_size 4096 KB
cache_swap_low 90
cache_swap_high 95

# DNS配置
dns_nameservers 8.8.8.8 8.8.4.4
dns_timeout 30 seconds

# 安全配置
# 隐藏版本信息
httpd_suppress_version_string on

# 禁用所有代理到本地连接
acl to_localhost dst 127.0.0.0/8 0.0.0.0/8
http_access deny to_localhost

# 禁止修改以下头部
request_header_access Allow allow all
request_header_access Authorization allow all
request_header_access WWW-Authenticate allow all
request_header_access Proxy-Authorization allow all
request_header_access Proxy-Authenticate allow all
request_header_access Cache-Control allow all
request_header_access Content-Encoding allow all
request_header_access Content-Length allow all
request_header_access Content-Type allow all
request_header_access Date allow all
request_header_access Expect allow all
request_header_access Host allow all
request_header_access If-Modified-Since allow all
request_header_access Last-Modified allow all
request_header_access Location allow all
request_header_access Pragma allow all
request_header_access Accept allow all
request_header_access Accept-Charset allow all
request_header_access Accept-Encoding allow all
request_header_access Accept-Language allow all
request_header_access Content-Language allow all
request_header_access Mime-Version allow all
request_header_access Retry-After allow all
request_header_access Title allow all
request_header_access Connection allow all
request_header_access Proxy-Connection allow all
request_header_access User-Agent allow all
request_header_access Cookie allow all
request_header_access All deny all

# 访问控制
acl manager proto cache_object
acl localhost src 127.0.0.1/32 ::1
acl localnet src 10.0.0.0/8
acl localnet src 172.16.0.0/12
acl localnet src 192.168.0.0/16

# SSL端口
acl SSL_ports port 443 8443
acl Safe_ports port 80
acl Safe_ports port 443
acl Safe_ports port 8443

# CONNECT方法限制
acl CONNECT method CONNECT

# 访问规则
http_access allow manager localhost
http_access deny manager
http_access deny !Safe_ports
http_access deny CONNECT !SSL_ports
http_access allow localnet
http_access deny all
EOF

# 测试配置
squid -k parse

# 启动服务
systemctl enable squid
systemctl start squid

# 检查状态
systemctl status squid
```

### 20.2.2 Squid访问控制配置

```bash
# 扩展访问控制

# 编辑 /etc/squid/squid.conf

# 定义IP黑名单
acl blocked_ips src "/etc/squid/blocked_ips.txt"

# 定义域名黑名单
acl blocked_domains dstdomain "/etc/squid/blocked_domains.txt"

# 定义时间限制
acl work_hours time M T W H F A1 09:00-18:00

# 定义用户认证
auth_param basic program /usr/lib64/squid/basic_ncsa_auth /etc/squid/passwd
auth_param basic children 5
auth_param basic realm "Proxy Authentication"
acl authenticated proxy_auth REQUIRED

# 定义下载类型限制
acl download_ext urlpath_regex -i \.(exe|msi|dmg|zip|rar|tar|gz|mp3|mp4|avi|mkv)$

# 应用访问规则
http_access deny blocked_ips
http_access deny blocked_domains
http_access allow authenticated work_hours
http_access deny download_ext

# 创建黑名单文件
cat > /etc/squid/blocked_ips.txt << 'EOF'
192.0.2.100
198.51.100.0/24
EOF

cat > /etc/squid/blocked_domains.txt << 'EOF'
.example-malicious.com
.facebook.com
.twitter.com
EOF

# 重新加载配置
squid -k reconfigure
```


### 20.2.4 Squid日志分析与监控

```bash
# Squid日志分析脚本
cat > /usr/local/bin/squid_log_analysis.sh << 'EOF'
#!/bin/bash
# Squid访问日志分析

LOG="/var/log/squid/access.log"
REPORT="/var/www/html/squid_report.html"

echo "<html><head><title>Squid日志分析报告</title></head><body>" > $REPORT
echo "<h1>Squid访问日志分析报告</h1>" >> $REPORT
echo "<p>生成时间: $(date)</p>" >> $REPORT

# 访问量统计
echo "<h2>访问量统计</h2>" >> $REPORT
echo "<pre>" >> $REPORT
echo "总访问次数: $(wc -l < $LOG)" >> $REPORT
echo "GET请求: $(grep -c 'GET' $LOG)" >> $REPORT
echo "POST请求: $(grep -c 'POST' $LOG)" >> $REPORT
echo "</pre>" >> $REPORT

# Top访问IP
echo "<h2>Top 20 访问IP</h2>" >> $REPORT
echo "<pre>" >> $REPORT
awk '{print $3}' $LOG | cut -d/ -f1 | sort | uniq -c | sort -rn | head -20 >> $REPORT
echo "</pre>" >> $REPORT

# 热门URL
echo "<h2>Top 20 热门URL</h2>" >> $REPORT
echo "<pre>" >> $REPORT
awk '{print $7}' $LOG | sort | uniq -c | sort -rn | head -20 >> $REPORT
echo "</pre>" >> $REPORT

# 错误统计
echo "<h2>HTTP状态码统计</h2>" >> $REPORT
echo "<pre>" >> $REPORT
awk '{print $5}' $LOG | cut -d/ -f1 | sort | uniq -c | sort -rn >> $REPORT
echo "</pre>" >> $REPORT

# 带宽统计
echo "<h2>带宽使用统计</h2>" >> $REPORT
echo "<pre>" >> $REPORT
awk '{sum+=$4} END {print "总流量: " sum/1024/1024 " MB"}' $LOG >> $REPORT
echo "</pre>" >> $REPORT

echo "</body></html>" >> $REPORT

echo "报告已生成: $REPORT"
EOF

chmod +x /usr/local/bin/squid_log_analysis.sh

# 添加到crontab每日执行
echo "0 0 * * * /usr/local/bin/squid_log_analysis.sh" >> /etc/crontab
```

## 20.4 反向代理安全配置

### 20.4.1 Nginx反向代理安全配置

```nginx
# /etc/nginx/conf.d/reverse-proxy.conf

server {
    listen 80;
    server_name api.example.com;
    
    # 返回安全头部
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    
    # 隐藏版本信息
    server_tokens off;
    proxy_hide_header X-Powered-By;
    proxy_hide_header Server;
    
    location / {
        # 反向代理到后端
        proxy_pass http://backend:8080;
        
        # 设置代理头
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
        
        # 缓冲设置
        proxy_buffering on;
        proxy_buffer_size 4k;
        proxy_buffers 8 4k;
        proxy_busy_buffers_size 8k;
        
        # 安全限制
        proxy_ssl_verify on;
        proxy_ssl_protocols TLSv1.2 TLSv1.3;
    }
}

# WebSocket反向代理
server {
    listen 80;
    server_name ws.example.com;
    
    location / {
        proxy_pass http://websocket_backend:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400;
    }
}
```

### 20.4.2 HAProxy反向代理安全配置

```ini
# /etc/haproxy/haproxy.cfg

frontend reverse_proxy
    bind *:80
    bind *:443 ssl crt /etc/ssl/certs/server.pem
    
    # 默认后端
    default_backend app_backend
    
    # ACL规则
    acl api path_beg /api/
    acl admin path_beg /admin/
    
    # API版本控制
    acl v1 path_beg /v1/
    acl v2 path_beg /v2/
    
    use_backend api_v2_backend if v2
    use_backend api_v1_backend if v1

backend app_backend
    mode http
    balance roundrobin
    option httpchk GET /health
    http-check expect status 200
    
    server app1 10.0.1.10:8080 check weight 100
    server app2 10.0.1.11:8080 check weight 100
    server app3 10.0.1.12:8080 check weight 80 backup
    
    # XFF头处理
    option forwardfor
    http-request set-header X-Forwarded-Host %[hdr(host)]
    http-request set-header X-Forwarded-Proto https if { ssl_fc }
    
    # 安全头部
    http-response add-header X-Frame-Options "SAMEORIGIN"
    http-response add-header X-Content-Type-Options "nosniff"
    http-response add-header X-XSS-Protection "1; mode=block"
    
    # 限流
    stick-table type ip size 100k expire 30s
    http-request track-sc0 src
    http-request deny deny_status 429 if { src_get_gpc0(dev) gt 100 }
```

## 20.6 实战案例：企业代理服务器部署

### 20.6.1 案例需求

某企业需要部署代理服务器：
- 员工上网行为管理
- 恶意网站过滤
- 带宽控制
- 日志审计
- HTTPS过滤

### 20.6.2 部署步骤

**第一步：安装Squid和辅助组件**

```bash
yum install squid squidclamav clamav clamav-update -y

# 更新病毒库
freshclam

# 配置SELinux
setsebool -P squid_use_ftp 1
setsebool -P httpd_can_network_connect 1

# 启动服务
systemctl enable squid
systemctl start squid
```

**第二步：配置访问控制**

```bash
cat > /etc/squid/squid.conf << 'EOF'
# 基础配置
http_port 3128
visible_hostname proxy.example.com

# 日志
logformat squid %ts.%03tu %6tr %>a %Ss/%03Hs %<st %rm %ru %un %Sh/%<A %mt
access_log /var/log/squid/access.log squid
cache_log /var/log/squid/cache.log

# 缓存配置
cache_mem 512 MB
maximum_object_size_in_memory 1 MB
cache_dir ufs /var/spool/squid 20000 16 256

# DNS
dns_nameservers 8.8.8.8

# 安全配置
httpd_suppress_version_string on
header_access X-Forwarded-For deny all
header_access Server deny all

# 访问控制定义
acl manager proto cache_object
acl localhost src 127.0.0.1/32 ::1
acl localnet src 10.0.0.0/8

acl SSL_ports port 443
acl CONNECT method CONNECT
acl Safe_ports port 80 443

#工作时间
acl work_hours time M T W H F 09:00-18:00

#用户认证
auth_param basic program /usr/lib64/squid/basic_ncsa_auth /etc/squid/passwd
auth_param basic children 10
auth_param basic realm "Enterprise Proxy - Login Required"
acl authenticated proxy_auth REQUIRED

#网站分类（使用SquidGuard或自定义列表）
acl blocked_sites dstdomain "/etc/squid/blocked_sites.txt"
acl adult_sites dstdomain "/etc/squid/adult_sites.txt"
acl streaming_sites dstdomain "/etc/squid/streaming_sites.txt"
acl download_ext urlpath_regex -i \.(exe|msi|torrent)$

#带宽限制
delay_pools 2
delay_class 1 1
delay_parameters 1 2000000/2000000  # 2MB/s全局限制
delay_access 1 allow all

delay_class 2 2
delay_parameters 2 500000/500000 500000/500000
delay_access 2 allow authenticated

#访问规则
http_access deny blocked_sites
http_access deny adult_sites
http_access allow authenticated work_hours
http_access allow authenticated !streaming_sites !download_ext
http_access deny all

#带宽应用
http_access allow authenticated
EOF

# 创建黑名单
cat > /etc/squid/blocked_sites.txt << 'EOF'
.malware-site.com
.phishing-site.com
EOF

cat > /etc/squid/adult_sites.txt << 'EOF'
.adult-site.com
EOF

cat > /etc/squid/streaming_sites.txt << 'EOF'
.youtube.com
.netflix.com
EOF

# 创建用户
htpasswd -cm /etc/squid/passwd user1
htpasswd -m /etc/squid/passwd user2

# 测试并重载
squid -k parse
squid -k reconfigure
```

**第三步：配置HTTPS过滤**

```bash
# 生成CA证书
mkdir -p /etc/squid/ssl
cd /etc/squid/ssl

openssl genrsa -out squid.key 2048
openssl req -new -key squid.key -out squid.csr -subj "/C=CN/ST=Beijing/L=Beijing/O=Example/OU=IT/CN=Squid-CA"
openssl x509 -req -in squid.csr -signkey squid.key -out squid.crt -days 3650

# 转换证书为DER格式（用于Windows客户端）
openssl x509 -in squid.crt -outform DER -out squid.der

# 在squid.conf添加HTTPS过滤
cat >> /etc/squid/squid.conf << 'EOF'

# HTTPS过滤配置
https_port 3129 intercept ssl-bump \
    cert=/etc/squid/ssl/squid.crt \
    key=/etc/squid/ssl/squid.key \
    defaultsite=proxy.example.com \
    vhost

acl step1 at_step SslBump1
acl step2 at_step SslBump2
acl step3 at_step SslBump3

ssl_bump peek step1
ssl_bump bump all
ssl_bump stare step2
ssl_bump terminate step3

# 对特定站点不进行过滤
acl no_filter dstdomain "/etc/squid/no_filter.txt"
ssl_bump splice no_filter
EOF

# 重新加载
squid -k reconfigure
```
