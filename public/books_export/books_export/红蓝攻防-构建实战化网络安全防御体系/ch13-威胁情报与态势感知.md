## 13.1 先讲威胁情报

先讲威胁情报。

什么是威胁情报（Threat Intelligence）？
官方定义一大堆，
我给你用人话解释一下。

### 13.1.1 什么是威胁情报？

简单说，
**威胁情报就是关于"坏人"的信息。**

坏人是谁？
他们用什么手法？
他们有什么工具？
他们最近在搞什么？
他们的IP、域名、木马哈希是什么？
他们针对哪些行业、哪些公司？
他们的战术、技术、流程（TTPs）是什么？

这些信息，
就是威胁情报。

```
威胁情报可以是：
  • 一个恶意IP
  • 一个恶意域名
  • 一个木马的哈希值
  • 一个钓鱼邮件的样本
  • 一种攻击手法
  • 一个攻击组织的画像
  • 一次即将发生的攻击预警
  • ......

  只要是跟威胁有关的、有价值的信息，
  都可以叫威胁情报。
```

威胁情报的核心价值是什么？
四个字：**知己知彼。**
- 知己：知道自己有啥、有啥漏洞（资产、漏洞）
- 知彼：知道敌人是谁、有啥招、最近在干啥（威胁情报）

知己知彼，
才能百战不殆。

### 13.1.2 威胁情报的类型

威胁情报有很多种，
按照不同的维度可以分成不同的类型。
我给你讲几种常见的分法。

**按类型分：**

```
第一种：IOC（指标型情报）
  • 最基础的情报
  • 就是各种"指标"：
    - 恶意IP
    - 恶意域名
    - 恶意URL
    - 文件哈希
    - 等等
  • 特点：具体、明确、好自动化匹配
  • 缺点：时效性短，很容易失效
    （IP换一个就没用了）

第二种：TTPs（战术技术流程型情报）
  • 比IOC高级一点
  • 讲的是攻击者的手法
    - 用什么工具？
    - 怎么钓鱼？
    - 怎么横向移动？
    - 怎么留后门？
  • 特点：更抽象、更能反映攻击者的行为模式
  • 时效性更长，手法不会天天变

第三种：战略型情报
  • 更宏观的
  • 比如：
    - 哪个攻击组织最近活跃？
    - 他们针对什么行业？
    - 他们的目的是什么？
    - 最近有什么新的攻击趋势？
  • 特点：宏观、偏战略
  • 用来指导安全建设方向
```

三种情报，
从具体到抽象，
从战术到战略，
各有用处。

**按来源分：**

```
内部情报：
  • 你们自己内部产生的情报
  • 比如：你们遇到的攻击、你们发现的漏洞
  • 特点：最贴合你们的实际情况，价值最高
  • 很多人忽略了自己内部的情报，光想着买外部的

外部情报：
  • 从外面买来的或者免费的
  • 比如：厂商的威胁情报、开源情报、社区情报
  • 特点：量大、覆盖面广
  • 缺点：不一定贴合你的场景，需要筛选
```

记住，
**最好的威胁情报，是你自己的情报。**
你们自己遇到的攻击、
自己总结的手法，
才是最有用的。
外部情报是补充，
不是全部。

### 13.1.3 威胁情报怎么用？

很多人买了威胁情报，
就只是接进SIEM里匹配一下，
匹配上了就告警。
这其实是最低级的用法。

威胁情报的用法多着呢，
我给你讲几个常见的应用场景。

```
用法一：检测/告警
  • 把IOC跟你的日志、流量匹配
  • 匹配上了就告警
  • 比如：发现有机器在连恶意IP，马上告警
  • 这是最基础、最常见的用法

用法二：威胁狩猎
  • 有了新的情报（比如新的攻击手法、新的IOC）
  • 主动去你的环境里搜
  • 看看有没有已经被攻击的痕迹
  • 这个比被动检测更主动

用法三：防护加固
  • 根据情报，提前做防护
  • 比如：
    - 情报说最近有个新漏洞在被利用
    - 你赶紧去查你们有没有，有就赶紧补
  • 再比如：
    - 情报说某个APT组织最近针对你们行业
    - 你提前针对他们的手法做检测、做防护
  • 把防护做在攻击前面，这才是情报的高级用法

用法四：应急响应
  • 出事了，用情报辅助判断
  • 比如：
    - 攻击者用的木马，情报里说这是某某组织的
    - 那你就知道大概是谁干的、他们的手法是什么
    - 排查起来就有方向了
  • 再比如：
    - 情报里说这个攻击者还会留什么后门
    - 你就可以重点去查那些点

用法五：风险评估/战略决策
  • 根据威胁情报，判断你们面临的威胁
  • 比如：
    - 最近针对你们行业的攻击变多了
    - 那你们就可以多投点资源在安全上
  • 再比如：
    - 某种攻击手法特别火，你们还防不住
    - 那就优先把这块补上
  • 指导安全建设的方向和优先级
```

看到了吧？
威胁情报不只是用来匹配告警的。
它可以用在安全的方方面面，
从检测、到狩猎、到防护、到应急、到战略。

关键是你会不会用。
很多人买了情报，
只用了10%的价值，
剩下90%都浪费了。

### 13.1.4 威胁情报的常见误区

最后讲讲威胁情报的几个常见误区，
很多人都踩过坑。

```
误区一：情报越多越好
  • 不对。
  • 情报不是越多越好，是越有用越好
  • 一堆跟你没关系的情报，
    不仅没用，还会干扰你
  • 关键是质量，不是数量
  • 跟你有关的、准确的、及时的，才是好情报

误区二：买了情报就万事大吉
  • 不对。
  • 情报只是原材料，怎么用才是关键
  • 买了情报不用，或者用得不好
    等于白买
  • 情报是给人用的，不是摆着看的

误区三：只看外部情报，不看内部情报
  • 不对。
  • 最好的情报是你自己的情报
  • 你们自己遇到的攻击、自己总结的经验
    才是最贴合你们的
  • 外部情报是补充，别搞反了

误区四：只看IOC，不看TTPs
  • 不对。
  • IOC时效性短，很容易失效
  • TTPs才是更稳定、更有价值的
  • 光靠IOC匹配，很多攻击发现不了
  • 要研究TTPs，针对行为做检测
```

威胁情报不是万能的，
但用好了，
能给你的安全体系加成很多。
别神话它，也别小看它。

## 13.2 再讲态势感知

讲完威胁情报，
再讲另一个更高大上的概念：
态势感知。

什么是态势感知（Situation Awareness，简称SA）？
这个词更玄乎，
一百个人有一百种解释。
厂商吹得更是天花乱坠。

我给你用人话讲讲，
到底什么是态势感知。

### 13.2.1 什么是态势感知？

"态势感知"这个词，
最早是军队里的。
什么意思呢？
就是：
**你对当前整个战场形势的了解和判断。**

- 敌人在哪？有多少？
- 我们的部队在哪？状态怎么样？
- 地形怎么样？天气怎么样？
- 敌人想干嘛？接下来会怎么打？
- 我们该怎么应对？

把这些都搞清楚了，
心里有数了，
这就是态势感知。

放到网络安全里，
也是一个意思：
**你对当前整个网络安全状况的了解和判断。**

- 现在有没有攻击？
- 攻击从哪来？在打什么？
- 我们哪些资产有风险？
- 攻击者到哪一步了？
- 整体的安全态势怎么样？
- 接下来可能会发生什么？
- 我们该怎么应对？

就是这么个意思。
说白了，
就是"心里有数"。
对整个安全状况心里有数，
这就是态势感知。

### 13.2.2 态势感知的三个层次

态势感知不是一个单一的东西，
它分三个层次。

```
第一层：感知（Perception）
  • 知道"发生了什么"
  • 比如：
    - 现在有多少告警？
    - 有哪些攻击？
    - 哪些资产受影响了？
  • 这是最基础的一层
  • 就是把数据收集起来、展示出来
  • 很多所谓的"态势感知"，就停留在这一层
  • 做个大屏，花花绿绿的，看起来挺厉害
  • 但其实只是个展示，没什么用

第二层：理解（Comprehension）
  • 不仅知道发生了什么，
    还知道"意味着什么"
  • 比如：
    - 这些告警，是不是同一次攻击？
    - 攻击到哪一步了？
    - 影响有多大？严重不严重？
    - 哪些是真的攻击，哪些是误报？
  • 这一层就需要分析和理解了
  • 不是光展示数据就行的
  • 要把零散的信息串起来，形成整体的判断

第三层：预测（Projection）
  • 不仅知道现在怎么样，
    还能预测"接下来会怎样"
  • 比如：
    - 攻击者接下来可能会做什么？
    - 可能会打哪些系统？
    - 风险会怎么发展？
    - 我们该怎么提前应对？
  • 这是最高级的一层
  • 能做到预测，才是真的态势感知
  • 但也最难
```

三个层次，
从低到高。
很多人以为做个大屏就是态势感知了，
其实那只是最底层的"感知"，
甚至连感知都算不上，
只是个可视化展示。

真正的态势感知，
是要能理解、能判断、能预测的。
不是摆摆数据就完事了。

### 13.2.3 态势感知的常见误区

说到态势感知，
误区就更多了。
我给你讲几个最常见的。

```
误区一：态势感知 = 大屏
  • 大错特错。
  • 大屏只是展示方式，不是态势感知本身
  • 没有分析能力，大屏就是个花架子
  • 领导来了看一看，平时啥用没有
  • 很多公司的态势感知就是这样
    买个大屏，摆着看，实际啥用没有

误区二：态势感知 = SIEM + 可视化
  • 也不对。
  • SIEM是工具，态势感知是能力
  • 有SIEM不代表你有态势感知
  • 就像给你一把枪，你不一定会打
  • 态势感知是在SIEM基础上，
    通过分析、运营、人员能力，
    形成的对整体安全态势的把握能力

误区三：态势感知是个产品，买了就有了
  • 大错特错。
  • 态势感知不是买个产品就能有的
  • 它是一种能力，需要建设、需要运营、需要积累
  • 买产品只是第一步，
    后面还有大量的工作要做
  • 很多人买了态势感知产品，
    以为就有态势感知能力了
    其实差远了

误区四：态势感知无所不能
  • 也不对。
  • 态势感知不是万能的
  • 它不能替你挡攻击，也不能替你应急
  • 它只是帮你更清楚地了解状况
  • 最终决策和行动，还是要靠人
  • 别神话它
```

态势感知这个词被炒得太火了，
很多人被带偏了。
记住，
**态势感知是一种能力，
不是一个产品。
它的核心是让人心里有数，
不是做大屏给领导看。**

### 13.2.4 怎么建设态势感知能力？

那态势感知能力怎么建设？
我给你一个简单的思路。

```
第一步：先把基础打好
  • 资产搞清楚
  • 日志接齐全
  • 检测规则写好
  • 告警能正常处置
  • 这些基础的东西都没有
    谈什么态势感知？都是空中楼阁

第二步：做好集中化
  • 所有安全数据集中到一起
  • 日志、流量、终端、威胁情报...
  • 有了集中的数据，才能做整体分析
  • 数据分散在各个地方，形不成态势

第三步：从"看到"到"看懂"
  • 先把数据展示出来（第一层：感知）
  • 然后做关联分析，把零散的告警串起来
  • 搞清楚哪些是真攻击、攻击到哪一步了
  • 慢慢做到能理解整体状况（第二层：理解）
  • 这一步需要大量的运营和积累

第四步：从"看懂"到"预判"
  • 理解了现在的状况
  • 再慢慢积累经验
  • 分析攻击者的行为模式
  • 尝试预测接下来可能会发生什么
  • 做到提前准备、提前应对（第三层：预测）
  • 这是最高境界，很难，但可以慢慢来
```

态势感知不是一蹴而就的，
是一步一步建设出来的。
别想着一步到位，
也别想着买个产品就万事大吉。
慢慢来，
从基础开始，
一步一步往上走。

## 13.3 威胁情报和态势感知的关系

讲到这，
你可能会问：
威胁情报和态势感知，
是什么关系？

简单说：
**威胁情报是态势感知的"输入"之一，
态势感知是威胁情报的"应用场景"之一。**

```
它们的关系：

  威胁情报 ──→ 提供"敌人"的信息 ──→ 态势感知
  资产数据  ──→ 提供"自己"的信息 ──→ 态势感知
  日志流量  ──→ 提供"正在发生什么" ──→ 态势感知
  漏洞数据  ──→ 提供"弱点"的信息 ──→ 态势感知
  ......

  态势感知把所有这些信息整合起来，
  形成对整体安全状况的判断。
```

威胁情报是态势感知的重要原料，
但不是全部。
态势感知需要方方面面的信息，
威胁情报只是其中一块。

反过来，
态势感知也是威胁情报的重要应用场景。
情报好不好用，
能不能发挥价值，
在态势感知里体现得最明显。

## 13.4 给防守方的建议

最后，给防守方几点建议。

### 建议一：先务实，再谈高大上

很多人一上来就想搞威胁情报、搞态势感知，
觉得这些高大上。
但其实基础的东西都没做好。

别着急，
先把基础打牢：
- 资产摸清了吗？
- 日志接全了吗？
- 检测做好了吗？
- 应急能顶得住吗？

这些基础的东西做好了，
再往上搞威胁情报、搞态势感知，
水到渠成。
基础都没有，
搞出来的都是花架子，
中看不中用。

### 建议二：威胁情报，先从自己的情报做起

想搞威胁情报，
别一上来就买一堆商业情报。
先把自己内部的情报建设好。

- 你们遇到过哪些攻击？
- 都是什么手法？
- 攻击者用了什么工具？
- 从哪进来的？
- 有哪些IOC？

把这些整理出来，
就是最好的威胁情报，
因为它最贴合你们的实际情况。
然后再用外部情报做补充。

内部情报 + 外部情报，
结合起来用，
效果才最好。

### 建议三：态势感知，别搞成大屏工程

很多公司搞态势感知，
最后就搞成了一个大屏工程，
钱花了不少，
实际没啥用。

别这样。
态势感知的核心是**能力**，
不是大屏。
- 能不能快速发现攻击？
- 能不能搞清楚攻击影响？
- 能不能判断整体态势？
- 能不能辅助决策？

这些才是关键。
大屏只是展示手段，
有了能力，
怎么做展示都行。
没有能力，
再好看的大屏也是摆设。

### 建议四：最终还是要落地到实战

不管是威胁情报还是态势感知，
最终都要落地到实战。
- 能不能帮助发现攻击？
- 能不能帮助更快响应？
- 能不能帮助更好地防护？
- 能不能降低风险、减少损失？

能解决实际问题，
才是真的有用。
不然说的再好听，
都是虚的。

别搞那些花里胡哨的概念，
实实在在解决问题，
比什么都强。

## 13.5 本章总结

这一章我们讲了威胁情报和态势感知，总结一下：

**威胁情报：**
- 是什么：关于"坏人"的信息
- 类型：IOC（指标型）、TTPs（战术技术型）、战略型
- 来源：内部情报 + 外部情报（内部的最有价值）
- 用法：检测告警、威胁狩猎、防护加固、应急响应、战略决策
- 常见误区：不是越多越好、不是买了就有用、不能只看IOC

**态势感知：**
- 是什么：对整体安全状况的了解和判断（心里有数）
- 三个层次：感知（看到）→ 理解（看懂）→ 预测（预判）
- 常见误区：不是大屏、不是产品、不是万能的
- 怎么建：先打基础 → 集中化 → 从看到看懂 → 从看懂到预判

**两者的关系：**
- 威胁情报是态势感知的重要输入
- 态势感知是威胁情报的重要应用场景

> 知己是基础，
> 知彼是加持。
> 心里有数，
> 才能从容应对。**
