# 第七章 攻击者视角：权限维持 —— 打进来了，就得想办法常来

> 第7章 | 预计学习时间：4小时 | 难度：⭐⭐⭐⭐

---

## 📖 写在前面

上一章我们讲了内网渗透，
这一章我们讲权限维持。

什么是权限维持？
简单说就是：
**攻击者打进来之后，
想办法留住访问权限，
就算系统重启了、管理员发现了入口，
还能再进来。**

用大白话说就是——
**留后门。**

打进来不容易，
总不能打进来一次，
被管理员发现了，
删了WebShell、改了密码，
就进不来了。
那多亏啊。

所以攻击者一定会想办法留后门，
而且会留好多个，
这个被发现了还有那个，
那个被清了还有下一个。

这一章我们就站在攻击者的角度，
聊聊权限维持那些事：
- 为什么要做权限维持？
- 常见的权限维持手法有哪些？
- 无文件攻击、白利用又是啥？
- 攻击者怎么把自己藏起来？
- Rootkit、Bootkit这些高级玩意儿是啥？
- 防守方该怎么发现和防御？

坐稳了，开唠。

---

## 7.1 先从一个故事讲起

### 7.1.1 "删不干净"的后门

先给你讲个真实的故事。

有一家公司，我们叫它C公司吧。
有一天，C公司安全团队发现，
他们的一台Web服务器被入侵了，
上面有个WebShell。

安全团队很"专业"地处理了：
- 把WebShell删了
- 把漏洞补了
- 把管理员密码改了
- 把系统扫了一遍病毒，没扫出来
- 然后说："搞定了，没问题了。"

结果呢？
过了一个星期，
又发现被入侵了。
而且这次，
攻击者拿到了更高的权限，
还跳到了内网的好几台机器上。

安全团队纳闷了：
"不对啊，漏洞不是补了吗？
密码不是改了吗？
WebShell不是删了吗？
怎么又进来了？"

他们请了个专家来帮忙查。
专家查了整整一天，
找出了好几个后门：

```
第一个后门：WebShell
  • 就是最开始发现的那个，已经被删了
  • 但这只是攻击者留的第一个，也是最明显的一个

第二个后门：隐藏账号
  • 攻击者在系统里加了一个隐藏的管理员账号
  • 名字跟系统默认账号很像，不仔细看看不出来
  • 用这个账号，随时能远程登录

第三个后门：计划任务
  • 攻击者建了个计划任务
  • 每隔一小时，就去请求一个C2地址
  • 只要执行了，就能把Shell弹回来
  • 就算系统重启了也不怕

第四个后门：注册表启动项
  • 攻击者在注册表里加了个启动项
  • 开机自动运行一个程序
  • 这个程序是白利用，用的是系统自带的工具
  • 杀毒软件根本不查

第五个后门：WMI事件订阅
  • 这个更隐蔽
  • 用WMI的事件订阅功能
  • 触发条件满足了就执行恶意代码
  • 一般人根本想不到去查这里

第六个后门：服务
  • 攻击者装了一个系统服务
  • 名字起得跟真的系统服务似的
  • 不仔细看根本发现不了
  • 服务是自动启动的，开机就运行
```

找出六个后门！
安全团队都看傻了：
"我的天，这也太阴险了！"

专家说：
"这才哪到哪啊，
  我还没查内核层呢，
  要是有Rootkit，那更难找。"

---

这个故事告诉我们什么？

**攻击者打进来之后，一定会留后门，而且会留好多个。**
- 明的暗的
- 系统层的应用层的
- 容易发现的不容易发现的
- 一层一层的

你清掉一两个没用，
人家还有好几个。
除非你全部找出来，全部清干净，
不然人家随时能回来。

这就是权限维持的厉害之处。
打进来只是第一步，
留住权限才是关键。

这一章，我们就来聊聊，
攻击者都是怎么留后门的，
以及防守方该怎么防、怎么找。

## 7.2 为什么要做权限维持？

先说说，
攻击者费那么大劲留后门，图啥？

### 7.2.1 怕丢了"阵地"

最直接的原因：
**怕好不容易打进来的入口，没了。**

比如：
- WebShell被管理员发现删了
- 漏洞被补了
- 密码被改了
- 系统重装了
- 机器重启了

入口没了，
就等于阵地丢了。
再想打进来，又得从头再来。
那多费劲啊。

所以得留后门，
就算这个入口没了，
还有别的入口能进来。

### 7.2.2 长期潜伏

第二个原因：
**攻击者想要长期潜伏，
不是打进来一次就走。**

很多攻击，尤其是APT攻击，
不是一两天的事，
可能是几个月、甚至几年。
攻击者要长期潜伏在里面，
慢慢偷数据、慢慢搞事情。

那总不能每次进来都重新打一遍吧？
太费劲了，也太容易被发现了。

所以得留后门，
想什么时候来就什么时候来，
跟回自己家似的。

### 7.2.3 狡兔三窟

第三个原因：
**多留几个，保险。**

一个后门被发现了，
还有第二个。
第二个被清了，
还有第三个。
第三个第四个都没了，
还有第五第六个。

只要有一个没被发现，
攻击者就能再回来。
防守方要全部找出来，
太难了。

这就是所谓的"狡兔三窟"——
多留几个入口，
总有一个能用。

### 7.2.4 隐藏自己

第四个原因：
**把自己藏起来，
不让管理员发现。**

最开始的入口，
比如WebShell，
其实是比较容易被发现的。
攻击者不会长期用这个。
他们会用WebShell打进来，
然后装上更隐蔽的后门，
再把WebShell删掉。

这样，
管理员就算查到了WebShell，
删掉了，
以为没事了，
其实更隐蔽的后门早就装上了。

## 7.3 常见的权限维持手法

权限维持的手法非常多，
我给你分分类，
从简单到复杂，
从明显到隐蔽，
一个一个说。

### 7.3.1 账号类后门

最简单最直接的，
就是加账号。

```
常见手法：
  1. 直接加管理员账号
     • net user hacker password /add
     • net localgroup administrators hacker /add
     • 简单粗暴，但也最容易被发现

  2. 隐藏账号
     • 账号名后面加个$，比如hacker$
     • 用net user看不到，要在注册表里才能看到
     • 稍微隐蔽一点

  3. 克隆账号
     • 把一个普通账号克隆成管理员权限
     • 看上去是普通用户，实际是管理员
     • 更隐蔽

  4. 篡改已有账号
     • 不改账号，改密码
     • 或者给某个已有账号加权限
     • 更难发现

  5. 后门账号
     • 利用系统的漏洞或者特性
     • 创建更隐蔽的账号
     • 一般人根本找不到
```

账号类后门，
优点是简单、直接、好用，
缺点是也比较容易被发现。
现在的管理员，
基本都会查新增账号。

所以攻击者一般不会只留账号后门，
还会留其他的。

### 7.3.2 启动项/计划任务/服务类后门

第二类，
就是利用系统的自动启动功能。
系统一开机、一登录，
恶意程序就自动运行了。

```
常见手法：
  1. 注册表启动项
     • Run、RunOnce这些键
     • 把恶意程序加进去
     • 开机/登录自动运行
     • 最常见，但也比较容易查到

  2. 开始菜单启动组
     • 开始菜单→程序→启动
     • 放个快捷方式在里面
     • 登录就自动运行
     • 很老的手法了

  3. 计划任务
     • 建个计划任务
     • 定时执行、开机执行、登录执行
     • 触发条件可以很复杂
     • 比启动项隐蔽一点

  4. 系统服务
     • 装一个系统服务
     • 服务一般是系统权限运行的
     • 开机自动启动
     • 名字起得像系统服务，不仔细看看不出来
     • 比较隐蔽

  5. 驱动/内核模块
     • 装一个恶意驱动
     • 内核级的，权限高
     • 也更难发现
     • Rootkit常用手法

  6. 其他自启动方式
     • Winlogon、Screensaver、DLL劫持...
     • 各种各样的自启动点
     • 多到你想不到
```

这类后门非常多，
Windows光自启动的方式就有几十上百种。
防守方要全部检查到，
非常难。

### 7.3.3 WebShell/后门文件类

第三类，
就是留文件。

```
常见手法：
  1. WebShell
     • 最常见的
     • 一句话木马、大马、小马
     • 藏在网站目录里
     • 改个名字、伪装成正常文件
     • 找起来挺费劲的

  2. 一句话后门
     • 非常短小的后门
     • 可以插到正常文件里
     • 比如插到图片里、插到正常的网页代码里
     • 非常隐蔽

  3. 内存马
     • 不落盘，直接在内存里
     • 没有文件，所以杀毒软件查不到
     • 重启了就没了，但只要服务不重启就一直在
     • 现在越来越流行

  4. 后门程序
     • 直接放一个木马程序
     • 或者把后门代码插到正常程序里
     • 运行了就中招
     • 免杀做得好的话，杀毒软件查不出来
```

文件类后门，
以前很常用，
现在因为有EDR、有杀毒软件，
越来越不好藏了。
所以攻击者越来越喜欢用"无文件"的方式。

### 7.3.4 白利用/无文件攻击

第四类，
也是现在最流行的——
白利用和无文件攻击。

什么叫白利用？
就是**用系统自带的、正常的工具，来干攻击的事。**
这些工具本身是合法的、是白名单的，
杀毒软件不会查它们。
但攻击者拿它们来干坏事。

什么叫无文件攻击？
就是**恶意代码不写到磁盘上，直接在内存里跑。**
没有文件，所以基于文件的杀毒软件查不到。

这俩现在是攻击者的最爱，
也是防守方最头疼的。

```
常见的白利用工具：
  1. PowerShell
     • Windows自带的，功能强大
     • 可以直接执行命令、下载执行、横向移动...
     • 攻击者最爱用的工具之一
     • 因为是系统自带的，杀毒软件不会随便杀

  2. wscript/cscript
     • Windows脚本宿主
     • 可以执行VBScript、JScript
     • 也是系统自带的

  3. mshta
     • 执行HTA文件
     • 也是系统自带的
     • 经常被用来做钓鱼和下载执行

  4. regsvr32
     • 注册DLL的工具
     • 可以远程加载执行
     • 又叫"squiblydoo"攻击
     • 很隐蔽

  5. WMI (Windows Management Instrumentation)
     • Windows的管理功能
     • 可以远程执行、查询信息、横向移动
     • 非常强大
     • 也是系统自带的

  6. PsExec/WMIC/winrm
     • 远程管理工具
     • 正常的运维工具
     • 被攻击者拿来横向移动

  7. certutil
     • 证书工具
     • 可以用来下载文件
     • 攻击者经常用它来下载木马

  ......还有很多很多
```

这些工具，
全是Windows系统自带的，
全是合法的、正常的。
但攻击者拿它们来干攻击的事。
你总不能把这些工具都禁用了吧？
那系统也没法用了。

这就是白利用的厉害之处——
用你自己的工具，打你自己。
杀毒软件还没法查，
因为程序本身是正常的。

那怎么防？
只能看行为——
看这些工具在干什么。
正常情况下管理员不会用PowerShell去下载奇怪的东西，
也不会用WMI去执行乱七八糟的命令。
行为异常了，就告警。

这也是为什么现在EDR越来越重要——
光看文件不够了，得看行为。

### 7.3.5 更高级的后门

上面说的，
还都是应用层、用户层的。
还有更高级的，
内核层、甚至硬件层的后门。

```
更高级的后门：
  1. Rootkit
     • 内核级的后门
     • 可以隐藏文件、隐藏进程、隐藏网络连接
     • 你在系统里根本看不到它
     • 非常难发现，非常难清除

  2. Bootkit
     • 比Rootkit更底层
     • 在系统启动之前就运行了
     • 感染MBR、UEFI
     • 就算你重装系统都没用
     • 非常非常难搞

  3. 固件后门
     • 感染BIOS、硬盘固件、网卡固件...
     • 比Bootkit还底层
     • 基本上很难发现，也很难清除
     • 相当于给硬件开了个后门

  4. 硬件后门
     • 直接在硬件里做手脚
     • 比如在设备里装个窃听装置
     • 或者芯片本身就有后门
     • 这个就更难防了
```

当然，这些高级后门，
一般的攻击者也不会用，
成本太高了。
一般都是APT组织、国家级的攻击才会用。
普通企业遇到的概率不大。

但你得知道有这些东西，
万一哪天遇到了呢？

## 7.4 攻击者怎么藏自己？

留后门只是第一步，
还得把自己藏好，
不能让管理员发现了。

那攻击者都是怎么藏的？

### 7.4.1 隐藏文件

最简单的，藏文件。
- 文件属性设为隐藏
- 改文件名，伪装成系统文件
- 藏在系统目录里，跟正常文件混在一起
- 插到正常文件里（图片、文档、正常的程序）
- 甚至不用文件，直接在内存里（无文件）

### 7.4.2 隐藏进程

然后是藏进程。
- 注入到正常进程里（比如explorer.exe、svchost.exe）
- 进程名起得跟系统进程一样，不仔细看看不出来
- 用Rootkit隐藏进程，任务管理器里根本看不到

### 7.4.3 隐藏网络连接

还有藏网络连接。
- 用正常的协议通信（HTTP、HTTPS、DNS...）
- 伪装成正常的业务流量
- 心跳间隔很长，藏在正常流量里
- 用隧道技术，把攻击流量藏在正常协议里
- 用Rootkit隐藏网络连接，netstat看不到

### 7.4.4 擦痕迹

最后，擦痕迹。
- 删日志（系统日志、安全日志、Web日志...）
- 清登录记录
- 改文件时间戳，伪装成老文件
- 清历史命令
- 清浏览器记录

能擦的都擦干净，
让你找不到证据。

## 7.5 给防守方的建议

说了这么多攻击手法，
最后给防守方几点建议。

### 建议一：账号管理要严

首先，账号要管好。
- 定期检查账号，看看有没有新增的异常账号
- 定期检查管理员组，有没有不该在里面的人
- 权限最小化，别什么人都给管理员
- 重要账号用多因素认证（MFA）
- 监控账号的异常登录、异常行为

账号是最基础的，
也是最重要的。
账号管好，
能拦住很多攻击。

### 建议二：监控自启动项

第二，
要监控系统的自启动项。
- 注册表启动项
- 计划任务
- 系统服务
- 驱动
- 等等等等

这些都是攻击者最爱留后门的地方。
定期检查，
发现异常的自启动项，
立刻查清楚。

当然，
手动检查不现实，
得靠工具——
EDR、HIDS、主机审计工具，
都能监控这些。

### 建议三：行为检测，不能光靠特征

第三，
**不能光靠特征码检测，
行为检测更重要。**

为什么？
因为现在攻击者越来越喜欢用白利用、无文件攻击。
- 没有恶意文件
- 用的都是系统自带的工具
- 特征码检测根本查不出来

那怎么发现？
看行为。
比如：
- PowerShell执行了奇怪的命令
- 正常的管理员不会用的操作
- 异常的网络连接
- 异常的进程创建
- 异常的注册表修改
- ...

行为异常了，
就告警、就调查。

这也是为什么EDR现在这么火——
传统杀毒软件靠特征码，
已经不够用了。

### 建议四：定期做全面检查

第四，
定期做全面的安全检查。
- 查账号
- 查启动项
- 查服务
- 查进程
- 查网络连接
- 查文件
- 查日志

别等出事了才查，
平时定期查，
主动去找问题。

当然，
也可以用威胁狩猎的方式，
主动去找攻击者留下的痕迹。
说不定就找出几个没被发现的后门呢。

### 建议五：假设已经被攻破，纵深防御

最后，
还是那句话：
**假设攻击者已经打进来了。**

不要觉得有防火墙、有杀毒软件就万事大吉。
要假设：
- 攻击者已经进来了
- 攻击者已经留了后门
- 攻击者就在内网里潜伏着

在这个基础上，
去构建你的防御：
- 就算进来了，也让他干不了太多事（最小权限）
- 就算留了后门，也能被发现（行为检测）
- 就算横向移动，也走不远（网络分段）
- 就算偷数据，也偷不到核心的（数据防护）

纵深防御，
一层又一层，
就算某一层被突破了，
还有下一层。

## 7.6 本章总结

这一章我们站在攻击者的角度，
聊了权限维持那些事，总结一下：

**为什么要做权限维持：**
1. 怕丢了"阵地"
2. 长期潜伏的需要
3. 狡兔三窟，多留几个保险
4. 把自己藏得更深

**常见的权限维持手法：**
1. 账号类后门（加账号、隐藏账号、克隆账号）
2. 启动项/计划任务/服务类后门（几十上百种自启动方式）
3. WebShell/后门文件类（WebShell、内存马、后门程序）
4. 白利用/无文件攻击（用系统自带工具干坏事，无文件不落盘）
5. 更高级的后门（Rootkit、Bootkit、固件后门、硬件后门）

**攻击者怎么藏自己：**
- 隐藏文件
- 隐藏进程
- 隐藏网络连接
- 擦痕迹、删日志

**给防守方的建议：**
1. 账号管理要严
2. 监控自启动项
3. 行为检测比特征检测更重要
4. 定期做全面检查，主动狩猎
5. 假设已经被攻破，纵深防御

最后送你一句话：

> **打进来只是开始，
> 能藏得住、留得下，
> 才是真的高手。**
