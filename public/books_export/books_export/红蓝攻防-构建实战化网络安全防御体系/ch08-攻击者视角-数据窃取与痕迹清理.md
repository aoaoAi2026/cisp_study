前面几章我们讲了外网打点、内网渗透、权限维持。

**数据窃取和清理。**

攻击者费了半天，攻击者图啥？
总不会是来旅游的吧？
当然不是。
攻击者冒着风险打进来，
总是有目的的。
有的是为了偷数据，
有的是为了搞破坏，
有的是为了敲诈勒索。

但不管目的是什么，
干完活干完的最后，
攻击者一定会做两件事：
- **第一件：把想拿的东西拿走（数据、权限...拿到手
- **第二件：把痕迹擦干净，让你发现不了，或者至少很难查到

这一章我们就站在攻击者的角度，
聊聊这两件事：
- 攻击者都偷什么数据？
- 怎么偷？
- 怎么把数据带出去？
- 怎么擦痕迹？
- 怎么逃避溯源？

## 8.1 先从一个故事讲起

### 8.1.1 "神不知鬼不觉"的数据泄露

先给你讲个真实的故事。

有一家公司，做游戏的，我们叫它D公司吧。
D公司发展得很好，
用户好几千万，
数据也很值钱——用户信息、充值记录、游戏数据...

有一天，
暗网上突然出现了一个帖子，
说有D公司的用户数据在卖，
几千万条用户信息，
包括用户名、密码、手机号、邮箱...
要价好几十万。

D公司一开始还不信：
"不可能，我们安全做得很好，
怎么可能数据泄露呢？"
结果对方放出了一部分样本，
一比对，还真是他们的用户数据。

D公司慌了，赶紧查。
查来查去，
查了好几个月，
才查出来个大概：

**攻击者是半年前进来的，
从一个老系统的漏洞打进来的，
进来之后，
在内网里横向移动，
最后摸到了数据库服务器，
然后把用户数据一点点往外传。

传了多久？
传了好几个月。
每天传一点，
慢慢传，
神不知鬼不觉。

而D公司呢？
D公司啥也不知道。
直到数据在暗网上卖了，
他们才知道出事了。

为什么没发现？
因为：
- 数据库的审计没开
- 数据出去了多少，不知道
- 异常的数据流量，也没监测
- 日志也没人看
- 等发现的时候，
数据早被人偷走了。

而且，攻击者擦痕迹也擦得很干净
- 日志删了
- 后门留了好几个，到最后都没找全
- 攻击路径也没完全查清楚
- 最后也没查出来是谁干的

就知道数据丢了，
其他啥也不知道。

---

这个故事是不是听着很气人？
但我告诉你，
这不是个例。
很多数据泄露事件，
都是这样的：
- 攻击者进来了，不知道
- 数据被偷走了，不知道
- 直到数据被卖了、被用来干坏事了，才知道
- 查又查不出来，擦屁股了

这就是数据窃取和痕迹清理的厉害之处。
攻击者不仅要偷东西，
还要偷得悄无声息，
偷完还要把屁股擦干净，
让你查都查不到。

这一章，我们就来聊聊，
攻击者是怎么偷数据、怎么擦痕迹的。
了解了这些，
你才知道该怎么防、怎么查。

## 8.2 攻击者都偷什么数据？

首先第一个问题：
攻击者打进内网了，
他都会偷什么数据？

很多人以为，
攻击者就是偷数据库里的用户数据。
其实不止。
攻击者感兴趣的东西多着呢。

我给你列一下，
攻击者最感兴趣的几类数据。

### 第一类：用户数据

最常见的，就是用户数据。
- 用户名、密码
- 手机号、邮箱
- 身份证号、住址
- 交易记录、消费记录
- 等等

这些数据值多少钱？
在黑产里很值钱的。
可以用来诈骗、
用来撞库、
用来发垃圾邮件、
用途多了去了。

尤其是有用户数据的公司，
比如互联网公司、游戏公司、电商公司、
都是攻击者的重点目标。

### 第二类：商业机密

第二类，商业机密。
- 公司的财务数据、
- 客户名单、
- 产品 roadmap、
- 技术文档、
- 源码、
- 合同、
- 招投标信息、
- 等等

这些数据，
竞争对手感兴趣，
商业间谍感兴趣。
值老钱了。

比如：
- 偷了你的招投标底价，
那竞标对手就能针对性报价，
你说值多少钱？

### 第三类：凭据/凭证

第三类，账号密码、密钥、凭证。
- 管理员账号密码
- 数据库密码
- 各种系统的密钥
- API Key、Access Key
- VPN账号
- 等等

偷这些干啥？
为了更大的破坏。
有了这些，
攻击者就能登更多系统、
拿更多权限、
偷更多东西。

很多时候，
攻击者偷数据，
不是为了卖，
是为了继续攻击。

### 第四类：知识产权

第四类，知识产权。
- 源代码
- 专利
- 技术文档
- 设计图纸
- 等等

这些东西，
对竞争对手来说，
太值钱了。
花几百万、几千万研发的东西，
偷走了直接就能用，
省了多少时间多少钱。

### 第五类：基础设施信息

第五类，基础设施信息。
- 网络拓扑
- 系统架构
- 账号体系
- 安全设备配置
- 等等

偷这些干啥？
为了后续攻击做准备。
知道了你家底，
下次打起来就更容易了。

---

当然，
不同的攻击者，
目标不一样。
- 小黑客，可能就想偷点用户数据卖钱
- 商业间谍，目标就是商业机密、知识产权
- APT组织，可能什么都要，长期潜伏，慢慢偷
- 勒索软件，可能不偷数据，直接加密了勒索

但不管是什么样的攻击者，
打进内网之后，
一定会收集信息、找值钱的东西。
这一步，是一定会做的。

---

## 8.3 数据窃取的方法

找到了值钱的数据，
接下来就是怎么偷走了。

怎么偷？
专业点说叫数据外泄（Exfiltration）。

数据外泄，是攻击者很重要的一步。
辛辛苦苦打进来，
最后得把东西带出去，
才算完。

那数据怎么带出去？
方法多了去了。
我给你列几种常见的。

### 8.3.1 直接下载/打包下载

最简单最直接的，
就是直接下载。

```
常见手法：
  1. 直接从Web下载
     • 找到了数据库，直接导出
     • 找到了文件服务器，直接下载
     • 简单粗暴

  2. 打包压缩
     • 先把文件打包、压缩
     • 有的还加密
     • 一次性传走
     • 不容易被发现

  3. 分片传输
     • 大文件，分成好多小份
     • 一点一点传
     • 藏在正常流量里
     • 不容易被检测到
```

这种方法最简单，
但也最容易被发现——
流量太大了，
很容易引起怀疑。
所以攻击者一般会慢点，
或者用更隐蔽的方法。

### 8.3.2 DNS隧道

第二种，DNS隧道。

什么是DNS隧道？
简单说就是：
把要传的数据，
藏在DNS查询请求里，
一点点带出去。

为什么用DNS？
因为DNS协议太常见了，
一般企业都会放通。
而且DNS流量通常不大，
也不太会被人注意。

```
怎么传？
  比如要传 "password123"，
  就把它编码一下，
  放到DNS查询的域名里，
  比如：
  cGFzc3dvcmQxMjM.attacker.com
  然后去解析这个域名，
  攻击者的DNS服务器收到这个查询，
  就拿到数据了。

  一点一点传，
  传大文件也能传，
  就是慢点。
  但是隐蔽啊。
```

DNS隧道，
是攻击者很喜欢用的一种方法。
因为一般企业，
很少会监测DNS流量的内容。
域名解析，
太正常了，
谁会想到里面藏着数据呢？

### 8.3.3 HTTP/HTTPS隧道

第三种，
用HTTP/HTTPS隧道。

就是把数据藏在正常的HTTP/HTTPS流量里。
比如：
- 伪装成正常的网页访问
- 伪装成正常的API调用
- 伪装成正常的文件下载
- 等等

为什么用这个？
因为HTTP/HTTPS流量太大了，
混在里面，
根本看不出来。
尤其是HTTPS还是加密的，
你不解密，
根本看不到内容。

```
常见手法：
  • 把数据藏在HTTP请求头里
  • 把数据藏在Cookie里
  • 把数据藏在请求体里
  • 把数据藏在图片/文件里（隐写术）
  • ......
```

这种方法，
隐蔽性非常强，
检测难度大。
尤其是现在HTTPS普及了，
流量都是加密的，
你不解密，
根本不知道传的是什么。

### 8.3.4 其他隧道

除了DNS和HTTP，
还有各种其他隧道。

```
其他隧道：
  • ICMP隧道 —— 用ping包传数据
  • SSH隧道 —— 用SSH传
  • 云存储 —— 传到网盘、云存储，然后再下载
  • 邮件 —— 用邮件把数据发出去
  • 社交平台 —— 发到微博、论坛这些
  • ......
```

各种各样的隧道，
只有你想不到，
没有攻击者做不到。

只要是能出网的协议，
攻击者都能拿来当隧道用。

### 8.3.5 物理带出

最后一种，
物理带出。

什么意思？
就是人直接把数据拷贝走。
- 拷到U盘里带走
- 拷到手机里带走
- 打印出来带走
- 甚至直接把硬盘拆走

如果攻击者能物理接触到机器，
那偷数据就太容易了。
尤其是内鬼偷数据，
最喜欢这么干。

当然，
远程攻击者一般用不上这个，
但近源渗透、内鬼，
就很常用。

---

数据外泄的方法太多了，
防不胜防。
你封了一个协议，
人家还有一百个协议等着你。

所以，数据防泄露（DLP）
是个很难的问题。
不是买个DLP产品就能搞定的。

---

## 8.4 痕迹清理：擦屁股

数据偷完了，
接下来干啥？
擦屁股。
专业点说叫痕迹清理，
或者叫反取证。

什么意思？
就是把攻击者留下的痕迹，
都擦掉，
让你发现不了，
或者很难查到。

为什么要擦？
- 不想被发现
- 不想被溯源
- 想长期潜伏，还想再来
- 不想被抓住

擦痕迹，
是攻击者的必修课。

那攻击者都擦啥？
怎么擦？

### 8.4.1 删日志

最常见的，
删日志。

```
删什么日志？
  • 系统日志
  • 安全日志
  • Web日志
  • 应用日志
  • 数据库日志
  • 防火墙日志
  • ......能删的都删

怎么删？
  • 直接把日志文件删了
  • 只删跟自己有关的那部分
  • 改日志，把自己的记录抹掉
  • 清日志配置，让日志不记录
  • 等等
```

日志一删，
你就没证据了。
想查都没法查。

当然，
如果日志是存在远程服务器上的，
攻击者删不了，
那他就没办法了。
所以，日志集中存储，
很重要。

### 8.4.2 清文件

第二，清理文件。

- 上传的木马、工具，删掉
- WebShell，删掉（或者藏得更深）
- 用过的工具，删掉
- 临时文件，删掉
- 能不留痕迹就不留

当然，
有的攻击者，
不仅删自己的文件，
还会把文件的时间戳改了，
伪装成老文件，
让你看不出来。

### 8.4.3 隐藏进程和网络连接

第三，
隐藏自己的进程和网络连接。

怎么隐藏？
- 注入到正常进程里，任务管理器看不到
- 用Rootkit隐藏，根本看不到
- 网络连接藏起来，netstat看不到
- 等等

你在系统里查，
啥异常都没发现，
但攻击者其实就在那。

### 8.4.4 伪造/误导

更高明的攻击者，
还会伪造痕迹，
误导溯源。

```
常见手法：
  • 伪造IP，把攻击来源改成别的地方
  • 留一些假的线索，把调查引向错误的方向
  • 伪装成其他攻击组织的手法
  • 故意留下一些假证据
  • 等等
```

让你查半天，
查出来的都是假的，
根本找不到真实的攻击者。

### 8.4.5 留后门，准备下次再来

最后，
擦完了，
也别忘了留后门。
擦干净了，
下次还想来，
还能再来。

---

痕迹清理，
是攻击者的基本功。
擦得干净不干净，
直接决定了会不会被发现，
以及被发现了能不能查到什么程度。

厉害的攻击者，
干完活走了，
你可能几个月、甚至几年都发现不了。
等发现了，
啥证据都没了，
查都没法查。

---

## 8.5 为什么溯源为什么这么难？

说了这么多，
你可能会问：
为什么溯源这么难？
为什么攻击者打进来，
偷了东西，
还能全身而退？

我给你总结几个原因。

### 原因一：日志不全

最常见的原因，
日志不全。

很多单位：
- 核心系统的日志有，边缘系统的没有
- 服务器日志有，终端日志没有
- 网络设备日志有，安全设备的不全
- 日志只存在本地，攻击者进来就删了
- 日志存的时间太短，几天就覆盖了

日志不全，
你拿什么溯源？
巧妇难为无米之炊啊。

### 原因二：监测能力不足

第二个原因，
监测能力不足。

- 攻击发生了，发现不了
- 数据被偷了，不知道
- 等发现了，攻击者早走了
- 黄花菜都凉了

发现得太晚，
攻击者都跑了，
你再想查，
难了。

### 原因三：攻击者太狡猾

第三个原因，
攻击者太狡猾了。

- 擦痕迹
- 用隧道
- 伪造线索
- 用跳板、用代理
- 一层又一层

你想追，
追着追着就断了。

溯源真实IP，
太难了。

### 原因四：溯源能力不够

第四个原因，
自己的溯源能力不够。

- 不会分析日志
- 不会分析流量
- 不会分析样本
- 不会做画像
- 不知道从哪查起

溯源是个技术活，
不是谁都能干的。
能力不够，
就算有日志有流量，
你也查不出东西来。

## 8.6 给防守方的建议

说了这么多攻击手法，
最后给防守方几点建议。

### 建议一：日志要全，要集中存

首先，
日志一定要全，
一定要集中存储。

- 什么日志都要接：
  • 系统日志
  • 安全日志
  • Web日志
  • 应用日志
  • 数据库日志
  • 网络设备日志
  • 安全设备日志
  • 终端日志
  • ......

能接的都接上。
而且，
别存在本地，
存到远程的日志服务器上，
攻击者访问不到的地方。
最好是只写不能删的，
写一次读多次（WORM）。
就算打进来了，
日志也删不掉。

日志是溯源的基础。
没有日志，
啥都白搭。

### 建议二：全流量留存

第二，
有条件的话，
全流量存下来。

为什么？
因为日志可以删，
流量删不了。
攻击者就算把日志全删了，
流量已经存下来了，
一样能查。

全流量分析，
是溯源的一大利器。
- 攻击是怎么进来的？看流量
- 攻击者干了啥？看流量
- 数据有没有被偷？看流量
- 偷了多少？看流量
- 传到哪去了？看流量

全都能查。

当然，
全流量存储成本不低，
但真出事了，
你就知道它有多重要了。

### 建议三：数据防泄漏（DLP）

第三，
做数据防泄漏。

- 重要数据，
  要知道在哪、谁能访问、
  谁访问了、
  有没有被下载了
- 异常的数据访问要告警
- 异常的数据外发要拦
- 核心数据要加密
- 重要数据要脱敏

不能等数据被偷了才后悔，
平时就要做好防护。

### 建议四：提高溯源能力

第四，
提高自己的溯源能力。

- 会看日志
- 会分析流量
- 会分析恶意样本
- 会做攻击时间线
- 会画攻击路径
- 会做攻击者画像

这些能力，
平时就要练。
别等出事了才临时抱佛脚。

怎么练？
- 多做应急响应
- 多做红蓝对抗
- 多复盘
- 多学习

能力是练出来的，
不是看出来的。

### 建议五：假设数据已经泄露，做纵深防御

最后，
还是那句话：
假设数据已经被偷了。

不要觉得攻击者偷不到数据，
要假设他已经偷到了，
在这个基础上，
去做防护，
做检测，
做响应。

就算数据被偷了，
能不能早点发现，
能不能少偷点，
能不能追回来点。

纵深防御，
一层一层的，
每一层都能拦住一部分。
就算某一层被突破了，
还有下一层。

## 8.7 本章总结

这一章我们站在攻击者的角度，
聊了数据窃取和痕迹清理，总结一下：

**攻击者都偷什么数据：**
1. 用户数据（账号密码、个人信息...）
2. 商业机密（财务、客户、招投标...）
3. 凭据/凭证（账号、密钥、API Key...）
4. 知识产权（源码、专利、图纸...）
5. 基础设施信息（拓扑、架构、配置...）

**数据外泄的常见方法：**
1. 直接下载/打包下载（简单直接，容易被发现）
2. DNS隧道（隐蔽，藏在DNS查询里）
3. HTTP/HTTPS隧道（藏在正常Web流量里，加密的更难查）
4. 其他隧道（ICMP、SSH、云存储、邮件...）
5. 物理带出（U盘、手机、内鬼）

**痕迹清理：**
- 删日志（最常见）
- 清文件（上传的工具、木马）
- 隐藏进程和网络连接
- 伪造/误导（假线索、假证据）
- 留后门（准备下次再来）

**为什么溯源难：**
1. 日志不全
2. 监测能力不足，发现太晚
3. 攻击者太狡猾（擦痕迹、用跳板）
4. 自己的溯源能力不够

**给防守方的建议：**
1. 日志要全，要集中存储
2. 全流量留存，以备溯源
3. 数据防泄漏（DLP）
4. 提高自己的溯源能力
5. 假设数据已经泄露，做纵深防御

最后送你一句话：

> **攻击者费尽心思打进来，
> 不是为了参观的。
> 打进来只是开始，
> 偷走东西才是目的。**
