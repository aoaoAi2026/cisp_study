## 14.1 先从一个故事讲起

### 14.1.1 "纸面上的强大"

先给你讲个真实的故事。

有一家国企，我们叫它H公司吧。
H公司很重视安全，
安全预算不少，
买了一堆安全设备，
什么防火墙、WAF、IDS/IPS、SIEM、EDR、DLP...
市面上有的，基本都买了。
各种规章制度也挺全的，
等级保护也过了，
看起来安全做得挺好的。

领导也觉得：
"我们安全做得不错，
该有的都有了，
应该没什么问题。"

直到有一年，
上级要求搞红蓝对抗实战演练。
H公司也报名参加了。

结果呢？
**红队只用了两天，就打穿了整个内网，
拿到了核心系统的最高权限。
而蓝队呢？
全程几乎啥也没发现，
最后还是红队告诉他们被打穿了，
他们才知道出事了。**

尴尬吧？
更尴尬的是，
红队是怎么进来的？
说出来你可能不信——
**从一台没人管的测试服务器进来的。**
那台服务器上有个老漏洞，
一直没修，
因为是测试环境，
没人管，
也不在资产清单里。

红队从这台测试服务器进来，
然后用哈希传递（Pass-the-Hash）横向移动，
没几天就打到了核心区。
而蓝队呢？
- 那台测试服务器根本就不在监控范围内
- 横向移动的行为也没检测出来
- SIEM告警倒是有，但全是误报，没人看
- EDR？哦，那台服务器没装EDR

全程就跟透明的一样。

演练结束之后复盘，
H公司的人才发现，
原来他们的安全，
都是纸面上的。
- 设备买了，但很多没调好
- 规则写了，但误报太多没人看
- 制度有了，但很多没落地
- 资产清了，但很多漏了
- 人也有，但能力跟不上

看起来什么都有，
实际上不堪一击。

## 14.2 红蓝对抗的目的和价值

首先，
我们先搞清楚：
为什么要搞红蓝对抗？
它的价值在哪里？

很多人觉得红蓝对抗就是走个形式，
上面要求搞，就搞一下，
搞完就完了。
不对。
红蓝对抗的价值大着呢。

```
红蓝对抗的价值：

1. 检验防御能力
   • 你的防御到底行不行？
   • 设备好不好用？
   • 检测准不准？
   • 响应快不快？
   • 不打一场，你永远不知道
   • 红蓝对抗就是最好的"实战检验"

2. 发现真实问题
   • 平时你发现不了的问题
     一演练全出来了
   • 比如：
     - 资产不清的问题
     - 检测盲区的问题
     - 响应流程的问题
     - 人员能力的问题
   • 这些问题，靠自查是查不出来的
   • 只有真打一场，才能暴露出来

3. 锻炼队伍
   • 平时都是纸上谈兵
   • 真出事了，慌不慌？
   • 会不会处理？
   • 流程顺不顺？
   • 练得多了，才会从容
   • 红蓝对抗就是最好的练兵

4. 验证投入效果
   • 安全投了那么多钱
   • 到底值不值？
   • 买的设备好不好用？
   • 建的体系有没有用？
   • 红蓝对抗一打就知道
   • 也能给老板一个交代

5. 持续改进
   • 每次演练，都会发现一堆问题
   • 把这些问题一个个解决
   • 防御能力就一点点提升了
   • 打一次，进一步
   • 持续迭代，持续进化
```

简单说，
红蓝对抗就是安全的"试金石"，
也是安全能力的"磨刀石"。
既能检验你行不行，
也能帮你变得更行。

实战化防御体系，
没有红蓝对抗，
就是闭门造车。
你永远不知道自己的真实水平。

## 14.3 红蓝对抗的组织形式

红蓝对抗怎么组织？
有哪些形式？

我给你讲几种常见的。

### 14.3.1 按目标范围分

```
第一种：专项演练
  • 针对某个特定的目标或者场景
  • 比如：
    - Web应用攻防演练
    - 钓鱼演练
    - 内网横向演练
    - 应急响应演练
  • 目标明确，针对性强
  • 适合练某个专项能力

第二种：综合演练
  • 全范围的，从外网到内网
  • 什么都可以打，什么都可以防
  • 就跟真实的攻击一样
  • 最考验整体能力
  • 也最接近实战
```

### 14.3.2 按对抗程度分

```
第一种：通告式演练
  • 蓝队知道要演练
  • 也知道大概时间
  • 双方都有准备
  • 优点：好组织，风险可控
  • 缺点：不够真实，蓝队有准备
    测出来的不是真实水平

第二种：半通告式
  • 蓝队知道有演练，但不知道具体时间
  • 也不知道红队会怎么打
  • 相对真实一点
  • 也比较好组织

第三种：盲演（不通知）
  • 蓝队完全不知道
  • 就跟真的攻击一样
  • 最真实，最能检验真实水平
  • 缺点：不好组织，风险大
    万一出点什么事，说不清
  • 一般大公司、成熟的团队才会搞
```

越接近真实，
效果越好，
但组织难度和风险也越大。
根据自己的情况选。
刚开始搞的话，
可以先从通告式的开始，
慢慢过渡到盲演。

### 14.3.3 按红队来源分

```
第一种：内部红队
  • 自己公司的人扮演红队
  • 优点：了解内部情况，成本低，方便
  • 缺点：太熟了，可能手下留情
    也可能思维固化，跳不出来

第二种：外部红队
  • 请外面的专业团队来打
  • 优点：专业、客观、不留情面
    能发现很多内部发现不了的问题
  • 缺点：贵，而且不了解内部情况
    需要时间熟悉

第三种：内外结合
  • 内部红队 + 外部红队
  • 或者外部红队打，内部红队陪练
  • 优势互补，效果最好
  • 当然成本也最高
```

各有优缺点，
根据自己的情况选。
一般建议：
平时可以内部红队常练，
定期请外部红队来打一次，
检验真实水平。

## 14.4 红蓝对抗的一般流程

一次完整的红蓝对抗，
一般是什么流程？
我给你梳理一下。

```
红蓝对抗的完整流程：

  1. 准备阶段
     ├─ 确定目标和范围
     ├─ 制定规则
     ├─ 组建队伍（红队、蓝队、紫队）
     ├─ 前期准备（工具、环境、权限）
     └─ 摸底和踩点（红队）

  2. 执行阶段
     ├─ 红队攻击
     ├─ 蓝队防守/监测/响应
     ├─ 紫队裁判/协调
     └─ 过程记录

  3. 结束阶段
     ├─ 红队汇报攻击路径和成果
     ├─ 蓝队汇报检测和响应情况
     ├─ 成绩评定
     └─ 现场复盘

  4. 整改阶段（最重要！）
     ├─ 问题整理
     ├─ 制定整改计划
     ├─ 落实整改
     └─ 验证整改效果
```

四个阶段，
最重要的是哪个？
**整改阶段。**

很多单位搞红蓝对抗，
前面热热闹闹的，
打完了，评个奖，就完事了。
发现的问题，
也没人整改，
下次再打，还是那些问题。
那搞这个有啥意义？
劳民伤财。

记住，
**红蓝对抗的目的不是输赢，
是发现问题、解决问题。**
不整改的红蓝对抗，
不如不搞。

## 14.5 红队怎么打？

红队怎么攻击？
前面攻击视角篇我们讲了很多，
这里就不细讲了，
简单回顾一下红队的一般思路。

```
红队攻击的一般路径：

  1. 侦察踩点
     • 收集信息：域名、IP、员工信息...
     • 找突破口

  2. 初始访问
     • 从外网打进来
     • 比如：Web漏洞、钓鱼、弱口令...

  3. 权限维持
     • 留后门，防止掉线

  4. 信息收集
     • 在内网里收集信息
     • 摸清楚网络结构、账号体系...

  5. 横向移动
     • 一台一台打过去
     • 逐步深入

  6. 权限提升
     • 拿到更高的权限

  7. 目标达成
     • 拿到预设的目标
     • 比如：核心系统权限、特定数据...
```

红队的思路，
跟真实的攻击者差不多，
只不过是有授权的、有规则的。

一支好的红队，
不光技术要好，
还要懂思路、懂方法、懂实战。
不是光会用工具就行的。

## 14.6 蓝队怎么防？

蓝队怎么防守？
这是大家最关心的。

很多人以为蓝队就是盯着监控大屏，
有告警就处理。
不对。
蓝队的工作多着呢。

```
蓝队在对抗中要做什么：

  1. 监测检测
     • 盯着各种监控
     • 发现攻击行为
     • 这是最基础的

  2. 分析研判
     • 告警是真是假？
     • 攻击到哪一步了？
     • 影响有多大？
     • 是不是红队干的？（演练的时候）

  3. 响应处置
     • 发现攻击了，赶紧处理
     • 封IP、隔离主机、杀进程...
     • 该怎么应急就怎么应急

  4. 溯源反制
     • 攻击是从哪来的？
     • 红队用了什么手法？
     • 有没有其他入口？
     • 能不能反制？（比如找到红队的服务器）

  5. 加固防御
     • 被打了之后，赶紧补漏洞
     • 加规则、加防护
     • 防止红队继续深入
```

蓝队不是被动挨打的，
是要主动出击的。
发现攻击、分析攻击、处置攻击、溯源攻击，
还要边打边补，
越打越强。

一支好的蓝队，
应该是越打越难打，
而不是被一路打穿。

## 14.7 紫队是干嘛的？

前面几次提到紫队，
紫队是干嘛的？

简单说，
紫队就是**裁判+教练+协调员**。

```
紫队的职责：

  1. 裁判
     • 判定攻击有没有效
     • 判定防守有没有效
     • 判定得分
     • 维护规则，解决争议

  2. 协调
     • 协调红队和蓝队
     • 有问题找紫队
     • 控制演练节奏
     • 确保演练顺利进行

  3. 教练
     • 打完了，复盘的时候
     • 紫队来点评
     • 告诉双方哪里做得好，哪里做得不好
     • 帮助双方提升

  4. 保障
     • 防止演练出意外
     • 有问题及时叫停
     • 确保业务不受大的影响
```

紫队很重要，
一场红蓝对抗能不能顺利、有没有效果，
紫队的水平很关键。
既要懂攻击，也要懂防守，
还要能中立客观。

一般来说，
紫队可以是内部的，
也可以是第三方的。
如果是内部红队打内部蓝队，
那紫队最好是第三方的，
比较公正。

## 14.8 怎么才能不走过场？

很多单位搞红蓝对抗，
最后都搞成了走过场、摆样子。
- 红队放水
- 蓝队演戏
- 皆大欢喜
- 啥也没学到

怎么才能避免走过场，
真正练出东西？

几个关键点。

### 1. 领导重视，真打真练

首先，
领导得真重视，
不是嘴上说说。
要明确：
**这次演练就是来找问题的，
问题找得越多越好，
不怕暴露问题，
就怕发现不了问题。**

如果领导的态度是"别搞出事来，差不多就行"，
那最后肯定是走过场。
大家都懂的。

所以，
领导的态度很重要。
要鼓励真打真练，
不怕暴露问题。

### 2. 规则明确，奖惩分明

第二，
规则要明确，
奖惩要分明。

- 红队什么能做，什么不能做
- 蓝队什么算分，什么不算
- 怎么判定输赢
- 发现问题了怎么算
- 打好了有什么奖励
- 放水了有什么惩罚

都要说清楚。
规则清楚了，
大家才会认真玩。

### 3. 红队要真打，别放水

第三，
红队要真打，
别放水。

很多内部红队，
跟蓝队熟，
不好意思下狠手，
打着打着就放水了。
那还有啥意思？

所以，
红队要有职业精神，
该怎么打就怎么打，
不用留情面。
打得越狠，
蓝队进步越快。

如果是外部红队，
就好很多，
人家是来干活的，
不会跟你客气。

### 4. 蓝队要真防，别演戏

第四，
蓝队也要真防，
别演戏。

别觉得反正就是演练，
无所谓。
就当成真的攻击来打，
该怎么监测就怎么监测，
该怎么响应就怎么响应。

演得越真，
收获越大。

### 5. 重点是整改，不是输赢

最重要的一点：
**红蓝对抗的目的不是输赢，
是发现问题、解决问题。**

别把输赢看得太重。
- 红队赢了，不代表红队厉害，
  只说明蓝队还有很多要改进的地方。
- 蓝队赢了，也不代表蓝队就完美了，
  可能只是红队没找到突破口而已。

输赢不重要，
重要的是：
- 发现了多少问题？
- 这些问题能不能改？
- 改了之后能力有没有提升？

把注意力放在整改上，
放在能力提升上，
而不是输赢上。

每一次演练，
只要能发现问题、解决问题，
就是成功的。

## 14.9 怎么通过红蓝对抗持续提升？

一次红蓝对抗，
只能发现一次的问题。
怎么通过红蓝对抗持续提升防御能力？

几个要点。

### 1. 常态化，不要搞运动式

首先，
红蓝对抗要常态化，
不要一年搞一次，
搞完就完事了。

- 小的演练，可以经常搞
  比如每个月搞个专项演练
- 大的综合演练，
  每个季度或者每半年搞一次
- 有条件的，可以搞持续性的
  红队常年在，随时打

练得越频繁，
进步越快。
一年搞一次，
每次都跟第一次一样，
没啥长进。

### 2. 闭环管理，发现一个改一个

第二，
**发现的问题一定要闭环。**

每次演练发现的问题，
- 整理出来
- 定好责任人
- 定好整改时间
- 一条一条去改
- 改完了验证
- 验证过了才关闭

不能发现了就发现了，
啥也不改。
那下次再打，还是这些问题。

每一个问题，
都要落地。
这样才能打一次、进一步。

### 3. 从"治标"到"治本"

第三，
整改不能只治标，
要治本。

什么意思？
比如红队通过一个漏洞打进来了，
你不能只把那个漏洞修了就完事了。
还要想：
- 为什么会有这个漏洞？
- 是开发没写好？还是测试没测出来？
- 为什么上线前没发现？
- 为什么漏洞扫描没扫出来？
- 其他系统有没有类似的问题？
- 怎么从流程上避免以后再出这种问题？

挖到根上，
从根源上解决。
不然修了一个，
还有千千万万个。

### 4. 攻防相长，互相促进

最后，
红蓝对抗是一个攻防相长的过程。

- 红队越打越厉害，
  蓝队就越防越强。
- 蓝队越防越强，
  红队就得研究新的打法。
- 你追我赶，
  一起进步。

最好的状态是：
红队和蓝队，
像两个高手切磋，
越打越强，
而不是一边倒。

## 14.10 给防守方的建议

最后，给防守方（蓝队）几点建议。

### 建议一：别怕输，输了才知道哪里不行

很多蓝队怕输，
觉得被打穿了很没面子。
其实大可不必。

被打穿了不是坏事，
反而是好事。
- 现在演练的时候被打穿，
  总比真的被攻击的时候被打穿强吧？
- 现在发现问题，
  总比出事了再后悔强吧？

输不可怕，
可怕的是输了还不知道为什么输，
也不去改。

摆正心态，
把每一次被打穿，
都当成一次学习和进步的机会。

### 建议二：别光盯着告警，要主动狩猎

很多蓝队在对抗中，
就只会盯着告警看。
告警来了就处理，
没有告警就觉得没事。
这是不对的。

前面讲过威胁狩猎，
演练的时候更要主动狩猎。
- 主动去找攻击的痕迹
- 主动去查异常
- 不能等告警

等告警的话，
很多攻击你发现不了。
主动出击，
才能更早发现。

### 建议三：边打边总结，边打边提升

红蓝对抗不是打完就完了，
打的过程中也要学习、也要改进。

- 红队用了一种新手法，
  赶紧学，赶紧补检测
- 被攻破了一个点，
  赶紧补上，别让红队再从这进
- 发现自己哪方面不行，
  打完赶紧补

越打越强，
才是好蓝队。

### 建议四：复盘的时候多问几个"为什么"

复盘的时候，
别停留在表面。
多问几个为什么。

- 为什么没检测到？
  因为没接日志？
  还是没写规则？
  还是规则写得不好？
  还是误报太多被淹没了？
- 为什么响应这么慢？
  因为流程不顺？
  还是人不够？
  还是能力不行？
- 为什么会有这个漏洞？
  为什么没防住？

挖到根上，
才能从根本上解决问题。

## 14.11 本章总结

这一章我们讲了红蓝对抗实战演练，总结一下：

**红蓝对抗的价值：**
1. 检验防御能力（到底行不行，打了才知道）
2. 发现真实问题（自查查不出来的问题）
3. 锻炼队伍（练得多了才会从容）
4. 验证投入效果（钱花得值不值）
5. 持续改进（打一次，进一步）

**组织形式：**
- 按范围：专项演练、综合演练
- 按对抗程度：通告式、半通告式、盲演
- 按红队来源：内部红队、外部红队、内外结合

**一般流程：**
准备 → 执行 → 结束 → 整改（最重要！）

**红蓝紫三队：**
- 红队：攻击方，想尽办法打进去
- 蓝队：防守方，监测、检测、响应、溯源
- 紫队：裁判+教练+协调员

**怎么不走过场：**
1. 领导重视，真打真练
2. 规则明确，奖惩分明
3. 红队要真打，别放水
4. 蓝队要真防，别演戏
5. 重点是整改，不是输赢

**怎么持续提升：**
1. 常态化，不要搞运动式
2. 闭环管理，发现一个改一个
3. 从"治标"到"治本"
4. 攻防相长，互相促进

最后送你一句话：

> **平时多流汗，战时少流血。
> 演练的时候被打穿不可怕，
> 真出事的时候扛不住，
> 才是真的可怕。**
