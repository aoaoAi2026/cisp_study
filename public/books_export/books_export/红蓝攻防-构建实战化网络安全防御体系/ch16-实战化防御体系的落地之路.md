## 16.1 先从一个问题讲起

### 16.1.1 "我该从哪开始？"

经常有人问我：
"老师，我们公司想做安全建设，
你能不能给我一个方案？
告诉我第一步干啥，第二步干啥，
一步步怎么建？"

每次有人这么问，
我都会先反问一句：
"你们现在是什么情况？
多大规模？
什么行业？
现在安全做到什么程度了？
有没有团队？有没有预算？
最担心的风险是什么？"

为什么这么问？
因为**没有放之四海而皆准的方案。**
- 几十人的小公司，
  和几万人的大公司，
  建设路径肯定不一样。
- 互联网公司，
  和制造业、金融业，
  重点也不一样。
- 已经有一些基础的，
  和从零开始的，
  起点也不一样。

没有标准答案。
但，
有通用的思路和方法。
有大致的路线和阶段。

这一章，
我就给你讲一个通用的建设思路，
一个分阶段的路线图。
你可以根据自己的实际情况，
去调整、去裁剪。

不管你是几十人的小公司，
还是上万人的大集团，
这个思路都适用——
只是节奏快慢、投入多少的区别。

## 16.2 建设实战化防御体系的几个原则

在讲路线图之前，
先讲几个基本原则。
这些原则，
我在前面各章也零散提到过，
这里集中再强调一下。

做安全建设，
这几个原则要守住，
不然很容易走偏。

### 原则一：实战导向，结果说话

第一个原则：
**实战导向，结果说话。**

什么意思？
就是别搞花架子，
别搞形式主义，
别为了合规而合规，
别为了买设备而买设备。
一切围绕实战——
真出事了能不能扛得住？
能扛住，就是好的。
扛不住，说得再好听也没用。

怎么检验？
红蓝对抗。
是骡子是马，拉出来遛遛。
用实战来检验，
用结果来说话。

### 原则二：顶层设计，分步实施

第二个原则：
**顶层设计，分步实施。**

什么意思？
- 先要有个整体的规划和蓝图，
  知道最终要建成什么样。
- 然后一步一步来，
  分阶段实施，
  先做什么，后做什么，
  心里有数。

不能走一步看一步，
东一榔头西一棒子，
最后建出来的东西四不像。
也不能想着一步到位，
一上来就想搞个大而全的，
结果战线拉得太长，
哪哪都做不好。

先有蓝图，
然后分步走，
小步快跑，
持续迭代。

### 原则三：基础先行，由下往上

第三个原则：
**基础先行，由下往上。**

什么意思？
就是先打基础，
再搞上层建筑。

- 先把资产摸清，
  再搞监测。
- 先把日志接全，
  再搞SIEM。
- 先把检测做好，
  再搞威胁狩猎。
- 先把基础打牢，
  再搞高大上的东西。

很多人一上来就想搞态势感知、搞AI、搞威胁狩猎，
结果基础的东西一塌糊涂。
建出来的都是空中楼阁，
中看不中用。

地基打牢了，
楼才能盖得高。
这个道理，
放在安全建设里也一样。

### 原则四：持续运营，持续改进

第四个原则：
**持续运营，持续改进。**

安全建设不是一劳永逸的，
不是建完了就完事了。
它是一个持续的过程，
需要一直运营，一直改进。

- 新的漏洞天天出，
- 新的攻击手法天天有，
- 业务也在天天变。

你不运营、不改进，
用不了多久，
就跟不上了。

安全建设，
永远在路上。
没有终点，
只有一个又一个新的起点。

### 原则五：人是核心，技术是工具

第五个原则：
**人是核心，技术是工具。**

再强调一遍：
- 设备是人用的，
- 体系是人建的，
- 运营是人做的。

没有人，
再好的技术、再好的设备，
都是摆设。

建设安全体系，
先把人建起来，
把能力建起来。
不然花再多钱，
也是打水漂。

---

这五个原则，
你要是能守住，
安全建设就不会出大问题。
就算走得慢一点，
也是在正确的方向上。

## 16.4 第一阶段：基础建设期

第一个阶段：基础建设期。
从零到一，打基础。

这个阶段，
适合安全建设刚起步的公司，
或者安全做得很差、
基本上没什么东西的公司。

### 16.4.1 这个阶段的目标

这个阶段的目标是什么？
**把最基础的东西建起来，
把最明显的窟窿补上。**

不求有多强，
但求不要裸奔。
最起码的防护要有，
最基础的检测要有，
最基本的响应能力要有。

### 16.4.2 这个阶段要做什么

```
这个阶段重点做几件事：

  1. 摸清家底
     • 把资产搞清楚
     • 有多少服务器？多少应用？
     • 哪些是重要的？
     • 先搞个七七八八
     • 不用追求100%准确，先有个大概

  2. 补上最基本的防护
     • 边界防火墙要有吧？
     • Web应用前面加个WAF不过分吧？
     • 服务器装个杀毒软件/EDR不难吧？
     • 弱口令先清一遍
     • 高危漏洞先修一修
     • 把最明显的窟窿先补上

  3. 把日志接起来
     • 重要系统的日志先接上
     • 防火墙、WAF、服务器、核心应用...
     • 先集中存起来
     • 不用一开始就买很贵的SIEM
     • 先用开源的或者便宜的也行
     • 先有再说

  4. 搞点基础的检测
     • 最常见的攻击得能发现吧？
     • 端口扫描、暴力破解、SQL注入...
     • 写几条最基础的规则
     • 先能发现一些常见攻击

  5. 基本的应急能力
     • 出事了有人管
     • 有基本的流程
     • 知道该找谁
     • 能做基本的处理
     • 不用很专业，先有

  6. 有专人负责安全
     • 别全是兼职
     • 至少有一两个人专门干安全
     • 有人盯，有人管
```

就这么几件事，
都是最基础的。
但就这几件事，
很多公司都没做到。

做到这些，
至少能从"完全裸奔"，
变成"有基本的防护"。
大部分 script kiddie（脚本小子）的攻击，
就能挡一挡了。

### 16.4.3 这个阶段的常见坑

```
常见的坑：

坑一：一上来就买一堆设备
  • 觉得安全就是买设备
  • 买一堆回来，没人会用，也没人运营
  • 全是摆设
  • 先把人配上，再买设备
  • 不然买了也是浪费

坑二：追求大而全
  • 什么都想做
  • 结果什么都做不好
  • 这个阶段，先把最基础的做好
  • 别贪多

坑三：没重点，胡子眉毛一把抓
  • 不知道先做什么后做什么
  • 东一榔头西一棒子
  • 先抓高风险的、最容易出问题的
  • 先把最明显的窟窿补上
```

这个阶段，
核心就是一个字：**稳**。
踏踏实实把基础打好，
别着急，
别搞花架子。

## 16.5 第二阶段：体系雏形期

第二个阶段：体系雏形期。
初步形成体系。

经过第一阶段，
基础的东西有了，
接下来就开始搭体系的架子了。

### 16.5.1 这个阶段的目标

这个阶段的目标：
**初步建成安全防御体系，
各项能力都有，
虽然不一定很强，
但至少都覆盖到了。**

从"有基本防护"，
变成"有完整体系"。

### 16.5.2 这个阶段要做什么

```
这个阶段重点做几件事：

  1. 资产与风险管理体系化
     • 资产清单搞准确
     • 资产分级
     • 漏洞管理体系建起来
     • 有流程、有SLA、有闭环
     • 风险管理有章法

  2. 监测体系完善
     • 日志接得更全
     • 检测规则多起来
     • 覆盖更多攻击场景
     • SIEM用起来
     • 误报慢慢降下来
     • 7x24小时监测有了

  3. 应急响应体系化
     • 应急流程完善
     • 有预案、有演练
     • 不同类型的事件怎么处理
     • 都有章法
     • 不是出事了再拍脑袋

  4. 安全运营团队成型
     • 有专门的安全运营团队
     • 分工明确
     • 不是一两个人啥都干了
     • 有做运营的、有做渗透的、有做开发的...

  5. 安全制度和流程
     • 各种安全制度建起来
     • 各种流程跑起来
     • 安全管理有章可循

  6. 初步的威胁情报
     • 买点威胁情报，或者用免费的
     • 接进来，匹配匹配
     • 先搞起来再说
```

这个阶段下来，
安全体系的架子就搭起来了，
各项能力都有了，
虽然还不一定很强，
但至少样样都有，
不会有明显的短板。

### 16.5.3 这个阶段的常见坑

```
常见的坑：

坑一：体系建起来了，但没人运营
  • 架子搭起来了，流程有了，制度有了
  • 但没人去跑、去运营
  • 最后变成一纸空文
  • 体系不是写出来的，是跑出来的

坑二：重建设，轻运营
  • 建设的时候热火朝天
  • 建完了就没人管了
  • 设备放那落灰
  • 规则写了就再也不改了
  • 很快就失效了
  • 运营比建设更重要

坑三：各干各的，不成体系
  • 资产是资产，监测是监测，应急是应急
  • 各干各的，互不通气
  • 形不成合力
  • 体系要打通，要联动
  • 不是一堆零散的东西堆在一起
```

这个阶段，
核心是**体系化**。
不再是零散的点，
而是形成一个整体，
各项能力协同工作。

## 16.6 第三阶段：能力成熟期

第三个阶段：能力成熟期。
各项能力完善，
从"有"到"强"。

体系架子有了，
接下来就是把各项能力做强、做深。

### 16.6.1 这个阶段的目标

这个阶段的目标：
**各项安全能力都比较成熟，
能应对大部分常见攻击，
检测比较准，
响应比较快，
运营比较顺。**

从"样样都有"，
变成"样样都行"。

### 16.6.2 这个阶段要做什么

```
这个阶段重点做几件事：

  1. 检测能力深化
     • 检测规则越来越准
     • 误报越来越少
     • 覆盖的攻击场景越来越多
     • 从特征检测到行为检测
     • ATT&CK各个阶段都有检测能力
     • 检测能力上一个台阶

  2. 威胁狩猎做起来
     • 从被动检测，到主动狩猎
     • 有专门的人做威胁狩猎
     • 能发现一些检测发现不了的攻击
     • 狩猎反过来促进检测能力提升

  3. 威胁情报深化应用
     • 威胁情报不只是匹配IOC
     • TTPs也用起来
     • 情报驱动的检测、狩猎、防护
     • 情报真正发挥价值

  4. 态势感知能力
     • 从零散的告警，到整体态势的把握
     • 能看懂当前的安全状况
     • 心里有数
     • 不是光做大屏，是真的有感知能力

  5. 自动化和效率提升
     • 很多重复的工作自动化
     • 告警分诊自动化
     • 简单的响应自动化（SOAR）
     • 把人从重复劳动中解放出来
     • 去做更有价值的事

  6. 红蓝对抗常态化
     • 定期搞红蓝对抗
     • 用实战检验能力
     • 打一次，进一步
     • 攻防相长

  7. 安全运营成熟
     • 安全运营流程顺畅
     • 有量化指标
     • 用数据说话
     • 持续改进有章法
```

这个阶段下来，
安全能力就比较强了，
一般的攻击根本打不进来，
就算打进来了，
也能很快发现、很快处理。
大部分公司，
能做到这个阶段，
就已经很厉害了。

### 16.6.3 这个阶段的常见坑

```
常见的坑：

坑一：进入瓶颈期，不知道怎么提升
  • 该做的都做了，该买的都买了
  • 但感觉也就那样了
  • 不知道再往哪个方向努力
  • 这时候需要实战来驱动
  • 多搞红蓝对抗，多找问题
  • 问题就是改进的方向

坑二：沉迷技术，忘了业务
  • 技术越搞越深
  • 但跟业务脱节
  • 安全团队自己玩得嗨
  • 业务部门觉得你们瞎折腾
  • 安全要服务于业务
  • 不能脱离业务

坑三：为了指标而指标
  • 有了KPI之后
  • 为了指标好看而做
  • 反而偏离了初衷
  • 指标是手段，不是目的
  • 别忘了初心
```

这个阶段，
核心是**深化和提升**。
从"有"到"好"，
从"能用"到"好用"。
靠的是持续的运营和打磨。

## 16.7 第四阶段：实战淬炼期

第四个阶段：实战淬炼期。
用实战来检验，用实战来提升。

到了这个阶段，
各项能力都比较成熟了，
接下来怎么进一步提升？
靠实战。

### 16.7.1 这个阶段的目标

这个阶段的目标：
**通过持续的实战演练，
不断发现问题，不断改进，
让防御体系越来越接近实战，
越来越能打。**

从"纸面强大"，
变成"实战能打"。

### 16.7.2 这个阶段要做什么

```
这个阶段重点做几件事：

  1. 红蓝对抗常态化、多样化
     • 不是一年搞一次
     • 而是经常搞
     • 专项演练、综合演练、盲演...
     • 各种形式都有
     • 持续锤炼

  2. 紫队协作，攻防相长
     • 红队攻，蓝队防，紫队协调
     • 打完一起复盘
     • 红队告诉蓝队哪里不行
     • 蓝队改进之后，红队再研究新打法
     • 你追我赶，一起进步

  3. 实战化的检测和响应
     • 检测规则是不是真的有效？
       用实战来检验
     • 响应流程是不是真的顺畅？
       用实战来检验
     • 哪里不行改哪里
     • 越来越贴近实战

  4. 漏洞管理实战化
     • 不是光看CVSS评分
     • 而是看实际可不可利用
     • 看在你们环境里风险有多大
     • 更真实地评估风险

  5. 威胁情报实战化
     • 情报有没有用？
       看实战中能不能发挥作用
     • 不断优化情报的应用方式
     • 让情报真正助力实战
```

这个阶段，
实战是核心驱动力。
所有的能力，
都要在实战中检验，
在实战中改进，
在实战中成长。

### 16.7.3 这个阶段的常见坑

```
常见的坑：

坑一：为了输赢，偏离目的
  • 红蓝对抗变成了争输赢
  • 红队为了赢不择手段
  • 蓝队为了不输各种耍赖
  • 忘了初衷是发现问题、提升能力
  • 输赢不重要，成长才重要

坑二：只打不改，打了白打
  • 红蓝对抗打了一场又一场
  • 发现的问题还是那些问题
  • 整改跟不上
  • 打了也白打
  • 闭环整改才是关键

坑三：红队水平不行，练不出东西
  • 红队太弱
  • 打过来打过去就那几招
  • 蓝队也得不到锻炼
  • 好的红队很重要
  • 要么培养，要么外聘
```

这个阶段，
核心是**实战驱动**。
用实战来发现问题，
用实战来检验能力，
用实战来驱动成长。

## 16.8 第五阶段：持续进化期

第五个阶段：持续进化期。
持续改进，越变越强。

到了这个阶段，
安全能力已经很强了，
也经过了实战的检验。
接下来呢？
接下来就是持续进化，
没有终点。

### 16.8.1 这个阶段的目标

这个阶段的目标：
**建立持续改进的机制，
让安全体系能够自我进化、
自我完善，
跟着威胁的变化、
跟着业务的变化，
一直保持强大。**

从"很强"，
变成"一直很强"。

### 16.8.2 这个阶段要做什么

```
这个阶段重点做几件事：

  1. 数据驱动的安全运营
     • 用数据说话
     • 用数据发现问题
     • 用数据衡量效果
     • 用数据指导方向
     • 量化运营，精确运营

  2. 智能化和自动化
     • 更多的工作交给机器
     • AI辅助分析、辅助决策
     • 自动化响应越来越多
     • 人只做最有价值的事

  3. 威胁情报驱动的防御
     • 情报不仅仅是匹配
     • 而是驱动整个防御体系
     • 情报来了，检测自动更新
     • 防护自动调整
     • 情报闭环

  4. 自适应安全
     • 防御体系能够根据威胁的变化
       自动调整
     • 攻击来了，自动加强防护
     • 风险高了，自动提升检测级别
     • 自适应，自调整

  5. 安全与业务深度融合
     • 安全不再是安全部门的事
     • 而是融入到业务的每个环节
     • 开发安全、运维安全、业务安全...
     • 全员安全，全链路安全

  6. 持续改进的文化
     • 安全不是一个项目
     • 而是一种文化
     • 每个人都有安全意识
     • 每个团队都对安全负责
     • 持续改进，永不止步
```

这个阶段，
是最高境界，
也是很多公司追求的目标。
能不能到，
要看公司的重视程度、
投入程度、
以及安全团队的能力。

但没关系，
路虽远，行则将至。
一步步往前走，
总会越来越近。

### 16.8.3 这个阶段的常见坑

```
常见的坑：

坑一：自大自满，觉得自己天下无敌
  • 觉得自己安全做得很好了
  • 谁也打不进来
  • 然后就放松了
  • 然后就出事了
  • 永远不要觉得自己足够安全
  • 永远保持敬畏

坑二：技术堆砌，忘了本质
  • 各种高大上的技术都往上堆
  • AI、大数据、机器学习...
  • 结果越搞越复杂，越来越难用
  • 效果还不一定好
  • 技术是为目标服务的
  • 别为了技术而技术

坑三：脱离业务，闭门造车
  • 安全越做越深
  • 但跟业务越来越远
  • 业务部门越来越反感
  • 最后安全推不动
  • 安全要融入业务
  • 要赋能业务，而不是阻碍业务
```

这个阶段，
核心是**进化**。
不是靠一次建设，
而是靠持续的进化能力。
能够跟着威胁、跟着业务，
一直变化，一直成长。

## 16.9 给不同角色的建议

最后，
给不同角色的人，
各几句心里话。

### 给安全负责人的话

如果你是安全负责人，
或者CIO、CTO：

1. **争取老板的支持很重要。**
   安全是个花钱的事，
   也是个不容易出成绩的事。
   没有老板的支持，
   很难做成事。
   怎么争取支持？
   用业务语言跟老板沟通，
   讲风险、讲价值、讲投入产出。
   别光讲技术术语，老板听不懂。

2. **要懂业务，要跟业务部门搞好关系。**
   安全不是孤立的，
   是为业务服务的。
   跟业务部门搞好关系，
   安全才能推得动。
   别总站在业务的对立面，
   要做业务的伙伴，
   帮业务安全地把事干成。

3. **带队伍比自己技术强更重要。**
   到了一定级别，
   你的个人技术就不是最重要的了。
   能带出一支能打硬仗的队伍，
   比你自己能打重要得多。
   培养人、锻炼人、给大家成长的空间。

4. **要有定力，不要被各种概念带着跑。**
   今天这个火，明天那个热，
   别跟着瞎起哄。
   知道自己要什么，
   知道自己的节奏，
   一步一个脚印，
   比什么都强。

### 给安全工程师/分析师的话

如果你是一线的安全工程师、分析师：

1. **打好基础，练好基本功。**
   别总想着搞什么高大上的东西，
   基本功打牢了，
   比什么都强。
   协议懂不懂？
   系统熟不熟？
   日志会不会看？
   攻击手法清不清楚？
   这些都是基本功。

2. **多实战，少纸上谈兵。**
   安全是个实战的活，
   光看书、光听课没用。
   多动手，多实践，
   多处理事件，多做渗透，
   多参加红蓝对抗。
   实战能力是练出来的，
   不是看出来的。

3. **保持学习，保持好奇心。**
   这个行业变化太快了，
   不学习很快就落伍了。
   保持好奇心，
   对新技术、新攻击手法保持敏感，
   一直学习，一直进步。

4. **养成记录和总结的习惯。**
   做过的事，
   学到的东西，
   都记下来。
   定期复盘，定期总结。
   成长会快很多。
   不然做了很多事，
   最后都忘了，
   跟没做过一样。

### 给老板/决策者的话

如果你是老板，或者公司的决策者：

1. **安全是投资，不是成本。**
   别把安全当成本中心，
   觉得花了钱也看不见效果。
   安全是投资，
   投的是风险的降低，
   投的是业务的保障。
   真出事了，
   损失的可能比你投的多得多。

2. **给安全足够的重视和资源。**
   嘴上重视没用，
   要落实在资源上。
   该给人给人，
   该给钱给钱，
   该授权授权。
   又要马儿跑，又要马儿不吃草，
   那是不可能的。

3. **安全是一把手工程。**
   安全推不推得动，
   关键在一把手。
   你真重视，
   下面就不敢不重视。
   你只是嘴上说说，
   下面也就跟你演戏。

4. **要有耐心，安全不是一蹴而就的。**
   别指望投一笔钱，
   下个月就安全了。
   安全是个长期的事，
   需要持续投入，
   持续建设。
   给团队时间，
   给安全成长的时间。
