# 第六章 攻击者视角：内网渗透 —— 进去之后横着走

> 第6章 | 预计学习时间：5小时 | 难度：⭐⭐⭐⭐

---

## 📖 写在前面

上一章我们讲了外网打点，
这一章我们讲内网渗透。

什么是内网渗透？
简单说就是：
攻击者打进来之后，
在内网里继续搞事情，
从一台机器打到另一台机器，
从普通权限弄到管理员权限，
最后拿下整个内网。

很多人以为，
边界被突破了就完了。
其实不是的。
边界被突破，
只是攻击的开始。

真正的大戏，在内网里。

而且，很多单位的内网，
跟不设防差不多——
一台机器被攻破了，
整个内网横着走。
想登哪台登哪台，
想看啥看啥。

这一章我们就站在攻击者的角度，
聊聊内网渗透那些事：
- 打进来之后，攻击者第一件事干啥？
- 攻击者怎么在内网里收集信息？
- 横向移动都有哪些手法？
- 提权都有哪些套路？
- 域环境怎么打？
- 为什么内网总是这么难防？
- 防守方该怎么防？

坐稳了，开唠。

## 6.1 先从一个故事讲起

### 6.1.1 从一台Web服务器到整个域控

先给你讲个真实的故事。

有一年护网，我认识的一个红队，
去打一家事业单位。

怎么打进去的？
说出来你可能不信——
一个很老的运维系统，
有个文件上传漏洞，
而且还是默认密码。
就这么着，
红队拿到了一台Web服务器的权限。

拿到WebShell之后呢？
按说这只是个普通的Web服务器，
权限也不高，
上面也没啥重要东西。

但是，红队的操作是这样的：

**第一步：信息收集。**
- 先看这台机器是什么系统、什么权限
- 看了看IP地址、网段
- 看了看这台机器上存的配置文件
- 从配置文件里找到了数据库的账号密码
- 还找到了几个其他系统的地址

**第二步：抓密码。**
- 上传了个mimikatz
- 一跑，抓到了好几个登录过这台机器的用户的密码哈希
- 其中有一个，是域用户
- 而且这个域用户，权限还不小

**第三步：探内网。**
- 用这台机器当跳板，
- 扫了扫内网的网段
- 发现内网机器真不少
- 而且很多机器都开着3389、445这些端口

**第四步：横向移动。**
- 用抓到的域用户密码哈希，
- 去试内网里的其他机器
- 一试，好多机器都能登上去！
- 为什么？因为很多服务器用的是同一个域管理员账号运维

**第五步：找域控。**
- 横向移动的过程中，
- 找到了域控制器（DC）
- 用那个权限不小的域用户一试，
- 能登上去！

**第六步：拿域控。**
- 登上域控之后，
- 把域里所有用户的哈希都导出来了
- 整个域，就这么拿下了。

从打进去到拿下整个域控，
用了多久？
**不到一天。**

而蓝队那边呢？
直到一周之后护网结束，
都没发现。
因为这台Web服务器的日志没接SIEM，
内网流量也没怎么监控，
人家在内网里横着走，
根本没人察觉。

---

这个故事是不是听着很夸张？
但我告诉你，
这不是个例，
这是很多单位内网的真实写照。

**边界像铜墙铁壁，内网像不设防的城市。**
这是很多单位的通病。

为什么会这样？
因为很多人觉得：
"内网是安全的，
  外面有防火墙挡着，
  内网不用搞那么严。"

大错特错！
攻击者总有办法打进来，
钓鱼、漏洞、第三方、内鬼...
总有一款适合他。

真正的考验，
是在内网。
内网能不能防住，
才是真本事。

这一章，我们就站在攻击者的角度，
看看内网渗透是怎么玩的。
了解了攻击，
你才知道该怎么防。

## 6.2 拿到Shell之后，攻击者先干啥？

很多人以为，
攻击者拿到Shell之后，
就开始乱搞了。
其实不是的。

真正有经验的攻击者，
拿到Shell之后，
第一件事不是乱搞，
是**稳一稳、看一看、藏一藏**。

先确保这个Shell不会丢，
再看看环境安全不安全，
然后再开始干活。

### 6.2.1 第一步：判断环境

拿到Shell之后，
攻击者最先要搞清楚的是：
- 我是谁？（当前用户权限）
- 我在哪？（这台机器的信息）
- 这是什么地方？（网络环境）

具体会看什么呢？

```
系统信息：
  • 是什么操作系统？Windows还是Linux？
  • 系统版本是多少？
  • 打了哪些补丁？
  • 主机名是什么？
  • 是什么角色？（Web服务器？数据库？普通PC？）

用户信息：
  • 当前用户是谁？
  • 是什么权限？普通用户还是管理员？
  • 这台机器上有哪些用户？
  • 有没有登录过其他用户？

网络信息：
  • IP地址是多少？
  • 属于哪个网段？
  • 网关是什么？
  • DNS是什么？
  • 能不能上网？
  • 能访问哪些网段？
  • 防火墙规则是什么样的？

进程和服务：
  • 跑了哪些进程？
  • 装了哪些服务？
  • 有没有杀毒软件？有没有EDR？
  • 有没有监控？

文件和信息：
  • 有哪些有意思的文件？
  • 配置文件里有没有密码？
  • 有没有数据库连接信息？
  • 有没有其他系统的地址？
```

信息收集得越全，
后面的路就越好走。

### 6.2.2 第二步：判断风险

信息收集完了，
接下来要判断：
我现在安全吗？
会不会被发现？

看什么呢？

```
安全软件检查：
  • 有没有杀毒软件？（360、火绒、卡巴斯基...）
  • 有没有EDR/HIDS？
  • 有没有主机审计？
  • 有没有流量监控？

日志检查：
  • 这台机器的日志开了吗？
  • 日志是存在本地还是远程？
  • 会不会马上被发现？

网络监测：
  • 我现在的操作会不会触发告警？
  • 执行命令会不会被监控？
  • 传文件会不会被检测？
```

判断完风险，
再决定下一步怎么干。
如果安全软件多、监控严，
那就小心点、慢点来、藏好点。
如果没啥防护，
那就可以放开手脚干了。

### 6.2.3 第三步：稳定和隐藏

环境摸清楚了，
风险也判断了，
接下来就是：
**把Shell稳住、把自己藏好。**

具体干啥呢？

```
稳定Shell：
  • 找个更稳定的Shell方式
  • 不要用容易被发现的WebShell
  • 移植CS、MSF这些远控
  • 确保就算WebShell被删了，我还能进来

权限维持：
  • 加个隐藏账号？
  • 写个注册表？
  • 整个计划任务？
  • 确保就算重启了，我还能进来

隐藏自己：
  • 把上传的文件藏起来
  • 把进程藏好
  • 把痕迹擦一擦
  • 尽量别留下太多日志
```

稳了、藏好了，
后面就可以安心干活了。
就算有一天这个入口被发现了，
还有别的入口能进来。

## 6.3 内网信息收集：摸清家底

稳下来之后，
攻击者就开始内网信息收集了。
要搞清楚：
这个内网长什么样？
有哪些机器？
哪些是重要的？
哪些好打？

内网信息收集，
是横向移动的基础。
信息越多，
找到突破口的概率就越大。

### 6.3.1 网络拓扑探测

首先要搞清楚的，
是网络结构。

```
攻击者会看：
  • 我在哪个网段？
  • 这个网段有多少台机器活着？
  • 这些机器都开了什么端口？
  • 有哪些网段是互通的？
  • 哪些网段访问不了？
  • 核心网段在哪？
  • 域控在哪？数据库在哪？核心业务在哪？
```

怎么探测？
- 扫描器扫（但动静大，容易被发现）
- 看本机的ARP表、路由表
- 看本机的网络连接
- 翻配置文件、文档
- 用DNS解析、域查询
- 等等等等

高手甚至能做到"无声探测"——
不主动扫描，
就从本机的信息和配置里，
把整个内网结构猜个八九不离十。

### 6.3.2 资产和服务识别

知道有哪些机器了，
接下来要搞清楚：
每台机器是干啥的？
跑了什么服务？
有没有漏洞？

```
攻击者会识别：
  • 操作系统类型和版本
  • 开了哪些服务？（Web、数据库、文件共享...）
  • 服务的版本是多少？
  • 有没有已知漏洞？
  • 是不是重要系统？
```

怎么识别？
- 端口扫描 + 服务识别
- Banner抓取
- 漏洞扫描（但动静大）
- 弱口令探测
- 等等

识别完了，
就知道哪些机器好打、
哪些机器价值高，
心里就有谱了。

### 6.3.3 域信息收集

如果目标是域环境，
那域信息收集就非常重要了。

```
攻击者会收集：
  • 域名是什么？
  • 域控有几台？分别在哪？
  • 域结构是怎样的？（单域？林？父子域？）
  • 有哪些域用户？哪些是高权限的？
  • 有哪些域组？
  • 有哪些域机器？
  • 组策略是怎样的？
  • 有没有域管理员登录过哪些机器？
```

这些信息收集全了，
域渗透就成功了一半。

在域环境里，
很多信息是普通域用户就能查的。
所以只要拿到一个域用户权限，
就能查到大量域信息。
这也是域渗透相对容易的原因之一。

## 6.4 横向移动：一路打穿内网

信息收集完了，
心里有谱了，
接下来就是横向移动。

什么是横向移动？
就是从你已经控制的这台机器，
去控制内网里其他的机器。
一台变两台，两台变四台，
越打越多，
最后控制整个内网。

横向移动是内网渗透的核心，
也是最体现技术的部分。

### 6.4.1 横向移动的前提

横向移动靠什么？
说白了，就靠三样东西：
- **凭证**（账号密码、哈希、票据...）
- **漏洞**（系统漏洞、服务漏洞...）
- **配置错误**（权限配置错了、没打补丁...）

这三样有一样，
就能横向移动。

其中，**凭证是最常用的。**
为什么？
因为很多单位，
很多服务器用的是同一个管理员账号，
或者密码都差不多。
抓到一个密码，
能登一大片机器。

这就是为什么攻击者最喜欢抓密码了。
抓到一个管理员密码，
等于拿下了半个内网。

### 6.4.2 常见的横向移动手法

横向移动的手法非常多，
我给你列几种最常见的：

```
1. 远程桌面（RDP）
   • 拿到账号密码，直接远程桌面连过去
   • 最简单、最直接
   • 但动静也大，容易被发现

2. SMB/PsExec
   • 利用Windows的SMB协议
   • 用psexec之类的工具，远程执行命令
   • 很常用，也很方便

3. WMI/WinRM
   • Windows的远程管理功能
   • 正常的管理功能，被攻击者拿来干坏事
   • 比较隐蔽，因为是系统自带的
   • 这就是所谓的"白利用"

4. SSH
   • Linux环境下最常用
   • 有账号密码或者密钥，直接登
   • 简单粗暴

5. 哈希传递（Pass-the-Hash）
   • 不用知道明文密码，有密码哈希就行
   • 用哈希直接登录
   • 非常经典的内网渗透手法
   • 抓哈希比抓明文容易多了

6. 票据传递（Pass-the-Ticket）
   • 域环境下的Kerberos票据
   • 拿到票据就能冒充别人
   • 黄金票据、白银票据，都是这个路数

7. 漏洞利用
   • 利用系统或服务的漏洞
   • 比如永恒之蓝（MS17-010）
   • 有漏洞的话，直接打过去就能拿权限
   • 效率高，但容易被检测到

8. 应用/服务漏洞
   • Web服务漏洞、数据库漏洞...
   • 从一个应用打进去，拿下服务器

9. 钓鱼/社工
   • 内网里再发钓鱼邮件
   • 伪装成IT部门、领导
   • 骗更多人的账号密码

......还有很多很多
```

手法多得很，
这个不行用那个，
总有一款适合你。

### 6.4.3 横向移动的思路

横向移动不是瞎打，
是有思路、有策略的。

一般的思路是：

```
第一步：抓凭证
  • 在当前机器上抓密码、抓哈希
  • 能抓多少抓多少
  • 有了凭证，横向移动就好办了

第二步：找目标
  • 找价值高的机器（域控、数据库、核心业务...）
  • 找好打的机器（有漏洞、弱口令...）
  • 先挑软柿子捏

第三步：打过去
  • 用各种手法打过去
  • 拿到新机器的权限

第四步：在新机器上继续
  • 到了新机器，继续抓凭证
  • 继续收集信息
  • 继续找新的目标
  • 继续打

第五步：一步步扩大
  • 一台变两台，两台变四台
  • 像滚雪球一样，越滚越大
  • 最后打到核心目标
```

就这么一步步的，
从最边缘的一台机器，
打到最核心的域控、数据库。
整个内网就拿下了。

## 6.5 提权：从小兵到将军

打进去一台机器，
如果只是普通用户权限，
很多事干不了。
所以攻击者会想办法提权——
从普通用户，
提升到管理员权限、系统权限。

### 6.5.1 为什么要提权？

提权有啥用？
用处太大了：

```
• 能抓密码了（mimikatz需要高权限）
• 能装后门了（装服务、改注册表...）
• 能看更多东西了（所有用户的文件）
• 能关杀毒软件了
• 能干更多"坏"事了
```

权限越高，
能干的事就越多，
内网渗透就越容易。

### 6.5.2 常见的提权手法

提权手法也很多，
我给你列几个常见的：

```
1. 系统漏洞提权
   • 利用操作系统的漏洞
   • 比如Windows的各种提权漏洞
   • Linux的内核漏洞
   • 有未打补丁的漏洞，直接提权

2. 服务配置错误提权
   • 服务的权限配置错了
   • 普通用户能修改服务的配置
   • 服务是以系统权限运行的
   • 改一改，就能提权

3. 计划任务提权
   • 计划任务是以高权限运行的
   • 如果任务执行的程序，普通用户能改
   • 那就能提权

4. 注册表提权
   • 注册表的权限配置错了
   • 普通用户能改某些注册表项
   • 改完之后就能提权

5. 总是以高权限运行的程序
   • 有些程序总是以管理员权限跑
   • 如果这些程序有漏洞、或者能被利用
   • 就能借它们的权限提权

6. 弱权限的文件/文件夹
   • 系统目录、程序目录权限配置太松
   • 普通用户能改系统文件
   • 改了之后就能提权

7. 数据库提权
   • 数据库是高权限运行的
   • 拿到数据库权限，利用数据库的功能提权
   • 比如MySQL的UDF提权、SQL Server的xp_cmdshell

8. 第三方软件/驱动提权
   • 装了某些有漏洞的第三方软件、驱动
   • 利用这些漏洞提权
```

提权的手法非常多，
就看目标系统有什么问题。
系统越老、补丁越少，
提权就越容易。

### 6.5.3 域环境的提权

如果是域环境，
提权的目标就更高了——
从普通域用户，
提升到域管理员权限。

域提权的手法也很多：
- 利用域管的登录会话（域管登过这台机器，抓哈希）
- Kerberoasting攻击
- AS-REP Roasting
- 组策略漏洞
- 域信任关系利用
- 等等等等

域环境的提权，
比单机提权复杂得多，
但也有意思得多。
一旦拿到域管权限，
整个域就是你的了。

## 6.6 为什么内网总是这么"好打"？

说了这么多内网渗透的手法，
你可能会问：
为什么内网这么好打？
为什么很多单位的内网，
跟不设防一样？

我给你总结几个原因。

### 原因一："内网安全"的错误观念

最根本的原因，
是很多人有个错误的观念：
**"内网是安全的"。**

觉得外面有防火墙挡着，
内网就没事了。
所以：
- 内网机器不用打补丁
- 内网服务不用加固
- 密码可以简单点
- 不用那么多监控
- 不用那么多权限控制

结果呢？
攻击者一旦打进来，
就跟进了自己家一样，
想干啥干啥。

**记住一句话：**
**没有什么内网是绝对安全的。**
攻击者总有办法进来。
你得假设攻击者已经进来了，
在内网也得设防。

### 原因二：信任泛滥

第二个原因，信任泛滥。

什么意思？
就是内网里，
机器和机器之间太信任了，
人和机器之间太信任了。

比如：
- 所有服务器都用同一个管理员账号
- 所有用户都能访问所有服务器
- 普通用户也能访问数据库
- 域管理员到处登录
- 等等

信任越多，
攻击者横向移动就越容易。
抓到一个密码，
就能登一大片。
拿到一台机器，
就能打穿整个内网。

**信任是安全的敌人。**
最小权限原则，
说起来容易，
做起来难。

### 原因三：内网监控缺失

第三个原因，
内网的监控太弱了。

很多单位：
- 边界流量监控得挺严，内网流量没人看
- 服务器日志接了SIEM，终端日志没接
- 边界有IDS/IPS，内网啥也没有
- 终端杀毒都不一定装全了
- 更别说什么EDR、内网流量分析了

攻击者在内网里，
横着走、竖着走，
干什么都没人知道。
等发现的时候，
人家早就打完收工了。

### 原因四：补丁跟不上

第四个原因，
内网机器的补丁跟不上。

为什么？
- 觉得内网安全，不用打补丁
- 怕打补丁影响业务，不敢打
- 机器太多，打不过来
- 老系统，打不了补丁

结果呢？
内网里一堆没打补丁的机器，
一堆已知漏洞，
攻击者一扫一大片。
永恒之蓝都出来多少年了，
现在还能在内网里扫到一大堆。

---

## 6.7 给防守方的建议

聊了这么多攻击手法，
最后给防守方几点建议。

### 建议一：网络分段，别让内网横着走

首先，
把网络分段做好。
- 办公网、业务网、生产网，分开
- 不同的业务系统，分开
- 重要系统单独划区
- 网段之间有访问控制
- 不是想访问就能访问的

这样，
就算攻击者打进来了，
也只能在一个小网段里折腾，
想打到核心系统，
还得再突破好几层。

总比一进来就能横着走强。

### 建议二：最小权限，别什么人都能访问什么

第二个，最小权限原则。
- 用户的权限，给够能用的就行，别给太多
- 服务的权限，也是最小化
- 不要什么都用管理员账号
- 不同系统用不同的账号密码
- 域管理员不要随便登录普通服务器

这样，
就算攻击者拿到了一个账号，
也干不了太多事，
也跳不到太多机器。

横向移动的难度，就大大增加了。

### 建议三：内网也要有监控

第三个，
内网的监控要跟上。
不要只监控边界，
内网也得监控。

- 内网流量也要监测
- 横向移动的检测要做
- 异常登录、异常访问要告警
- 终端EDR要装，要有检测能力
- 重要服务器的日志要接全
- 域控的监控要加强

别让攻击者在内网里横着走还没人知道。

早一点发现，
损失就小一点。

### 建议四：补丁管理，不能只做表面

第四个，
补丁管理要落到实处。
- 内网的机器也要打补丁
- 重要的高危补丁，优先打
- 不能打补丁的，用其他方式缓解
- 老系统，该升级升级，该淘汰淘汰

补丁是成本最低的防护，
把补丁打好了，
能防住一大半攻击。

### 建议五：假设被攻破，做纵深防御

最后一个，
**假设攻击者已经打进来了。**

不要觉得有边界防护就万事大吉。
要假设：
- 边界会被突破
- 有一台机器会被拿下
- 攻击者会在内网里横向移动

在这个假设的基础上，
去构建你的内网防御。
- 就算进来了，也让他走不远
- 就算拿到一台机器，也让他拿不到核心数据
- 就算他横向移动，也能很快发现

这就是纵深防御。
一层一层的，
每一层都能拦住一部分攻击。
就算突破了一层，
还有下一层等着。

## 6.8 本章总结

这一章我们站在攻击者的角度，
聊了内网渗透那些事，总结一下：

**拿到Shell之后：**
- 第一步：判断环境（我是谁、我在哪、这是什么地方）
- 第二步：判断风险（安全不安全、会不会被发现）
- 第三步：稳定和隐藏（稳住Shell、藏好自己）

**内网信息收集：**
- 网络拓扑探测（有哪些网段、哪些机器）
- 资产和服务识别（每台机器干啥的、有啥漏洞）
- 域信息收集（域控、域用户、域组...）

**横向移动：**
- 前提：凭证、漏洞、配置错误
- 常见手法：RDP、SMB/PsExec、WMI、SSH、哈希传递、票据传递、漏洞利用...
- 思路：抓凭证→找目标→打过去→新机器上继续→滚雪球

**提权：**
- 为什么要提权：权限高了能干更多事
- 常见手法：系统漏洞、配置错误、服务/计划任务/注册表...
- 域提权：各种域攻击手法

**为什么内网好打：**
1. "内网安全"的错误观念
2. 信任泛滥（同一账号、权限太大）
3. 内网监控缺失
4. 补丁跟不上

**给防守方的建议：**
1. 网络分段，别让内网横着走
2. 最小权限，限制横向移动
3. 内网也要有监控，早发现
4. 补丁管理要落到实处
5. 假设被攻破，做纵深防御

最后送你一句话：

> **最好的内网防御，
> 是让攻击者就算打进来了，
> 也寸步难行。**
