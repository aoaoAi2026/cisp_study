## 4.1 先从一个故事讲起

### 4.1.1 一场"失败的红蓝对抗

先给你讲个真实的故事。

有一家公司，我们叫它B公司吧。
B公司的安全负责人，听完一次安全峰会，
听了红蓝对抗的分享，热血沸腾。
回来就跟老板说：
"老板，我们也搞一次红蓝对抗吧，
检验一下我们的真实防御能力。"

老板觉得挺好："行啊，搞吧。预算批了。"

于是安全负责人就找了一家安全公司，
谈好价格，定了时间，两周后开始。

然后呢？然后就等着开始了。

红队来了5个人，
到了现场，
找了个会议室，坐下，开干。

蓝队呢？
蓝队就是公司自己的安全团队，
正常上班，正常值班，
也没做啥准备。

然后呢？
然后就是红队一顿猛打，
三天就打穿了整个内网。

最后出了份报告，
列了一堆问题，就走了。

蓝队呢？
蓝队表示很委屈：
"这也太突然了，我们都没准备好，
而且他们知道我们的网络结构，
打我们当然容易了..."

红队走了之后呢？
报告放那，
也没人跟进整改，
也没人复盘总结。
过了半年，啥改进的没改，
该咋样还咋样。

钱花了几十万，
最后啥效果没有。
就得到了一份报告，
看完就扔那了。

---

你说，这场红蓝对抗，
是成功还是失败？

显然是失败的。
钱花了，时间花了，
最后啥改进也没有，
那不就是白花了吗？

问题出在哪？
出在组织和流程上。
- 前期准备没做好
- 目标没定清楚
- 蓝队也没进入状态
- 后期也没跟进改进

一场好的红蓝对抗，
可不是"红队来打、打完就走"这么简单。
它是一个系统工程，
从前期准备、到中期对抗、到后期改进，
每一步都很重要。

这一章，我们就来聊聊，
一场正规的红蓝对抗，
到底应该怎么组织、怎么跑。

## 4.2 红蓝对抗的类型

首先，红蓝对抗不是只有一种。
不同的目的、不同的规模、不同的玩法，
可以分成很多种类型。

我给你梳理一下常见的几种。

### 4.2.1 按目标分：能力检验型 vs 能力提升型

按目标来分，有两种：

**第一种：检验型。**
就是来摸摸底，看看现在的防御到底行不行。
- 目标：检验当前安全防护能力
- 重点：发现问题、暴露短板
- 结果：一份问题清单、风险评估

这种就相当于体检，
看看你身上有啥毛病。

**第二种：提升型。**
不只是检验，还要在对抗过程中，
还要锻炼队伍、提升能力。
- 目标：锻炼蓝队的检测、响应、溯源能力
- 重点：过程中的攻防互动
- 结果：不仅有问题清单，还有能力提升

这种就相当于实战演习，
不仅要打出出汗、治治病，
还要锻炼部队战斗力。

这两种没有好坏之分，
看你的目的是什么。
- 第一次搞不清楚自己水平，先搞检验型
- 已经有一定基础了，想练队伍，搞提升型

### 4.2.2 按通知方式分：黑盒 vs 白盒 vs 灰盒

按红队掌握的信息多少来分，有三种：

**黑盒对抗：
- 红队完全不知道目标内部信息
- 跟真实黑客一样，从零开始收集信息
- 最接近真实攻击
- 但效率低，时间成本高

**白盒对抗：
- 红队知道目标的全部信息
- 网络拓扑、系统架构、源码都给
- 效率高，能深入发现问题
- 但不够真实

**灰盒对抗：
- 给一部分信息，不给全部
- 比如给个外网资产清单，不给内部
- 介于两者之间
- 性价比最高

现在实际中，
灰盒用得比较多。
完全黑盒太费时间，
而且很多时候红队是内部的，
本来就知道一些信息，没必要装不知道。
灰盒比较均衡。

### 4.2.3 按对抗激烈程度分：普通攻防演练 vs 护网级对抗

按激烈程度来分：

**普通红蓝演练：**
- 规模小，时间短，几天到几周
- 红队可以沟通，有问题随时叫停
- 重点是发现问题，不是真的"打个你死我活
- 适合企业内部演练

**护网级对抗：
- 规模大，时间长，半个月到一个月
- 红队蓝队完全对立，真刀真枪干
- 有评分、有排名
- 就跟真实攻击几乎一样
- 适合国家级、行业级的护网行动

两种不一样，
护网级的强度高、压力大，
对蓝队的考验也大。

### 4.2.4 按组织方式分：内部红队 vs 外部红队

按谁来当红队来分：

**内部红队：**
- 自己公司的人当红队
- 优点：了解业务、成本低
- 缺点：太熟了，可能下不去手；也容易被蓝队摸透

**外部红队：**
- 请第三方安全公司的红队
- 优点：专业、客观、经验丰富
- 缺点：贵、不了解业务、需要时间熟悉

一般来说，
- 预算有限、想常搞，用内部红队
- 预算够、想真实检验，用外部红队

当然，现在很多大公司，
既有内部红队，也会定期请外部红队。
内部的平时练，外部的来检验。

## 4.3 红蓝对抗的铁三角：红队、蓝队、紫队

一场正规的红蓝对抗，
一般有三方：红队、蓝队、紫队。
我们叫它"红蓝紫铁三角"。

分别干啥的？怎么组织架构是怎样的？

### 4.3.1 红队：进攻方

红队是进攻方，
负责模拟真实黑客进行攻击。

**红队的目标：
- 在规则范围内，尽可能达成攻击目标
- 拿到最高权限、核心数据
- 尽可能不被发现、尽可能久地潜伏
- 尽可能多地暴露防守的问题

**红队的典型角色：

```
红队队长（红队负责人）
  • 负责整体攻击方案
  • 分工和协调
  • 跟紫队沟通
  • 把握节奏

外网打点手
  • 负责外网信息收集、漏洞挖掘、外网打点
  • 拿初始访问权限

内网渗透手
  • 打进去之后，负责内网渗透
  • 横向移动、提权、域渗透

钓鱼/社工手
  • 负责钓鱼邮件、社会工程学
  • 从人的层面突破

开发/免杀工程师
  • 负责木马免杀
  • 工具开发
  • 绕过检测

近源渗透手（可选）
  • 负责物理渗透、近源攻击
  • 比如混进大楼、丢U盘、WiFi攻击
```

根据规模可多可少，
少的话两三个人也能干，
大的话十几个人。
分工细。

### 4.3.2 蓝队：防守方

蓝队是防守方，
负责监测、检测、响应、溯源。

**蓝队的目标：
- 尽快发现攻击
- 及时响应、遏制攻击
- 溯源分析攻击路径
- 修复漏洞、加固系统
- 把损失降到最低

**蓝队的典型角色：

```
蓝队队长（蓝队负责人）
  • 负责整体防守指挥
  • 协调资源、决策
  • 跟紫队沟通

监测分析组（SOC分析师）
  • 盯着监控、看告警
  • 研判告警、分析是不是真攻击
  • 发现异常、上报

应急响应组
  • 真出事了上
  • 遏制、根除、恢复
  • 现场处置

溯源分析组
  • 负责攻击溯源
  • 搞清楚攻击从哪来、怎么进来的、干了啥
  • 攻击者画像

保障组
  • 负责设备、系统正常运行
  • 日志、流量正常
  • 后勤保障
```

跟红队一样，
蓝队也是可大可小。
小的话几个人，
大的话几十上百人都有。

### 4.3.3 紫队：裁判+教练

紫队是介于红队和蓝队中间的，
既是裁判，也是教练。

**紫队的目标：**
- 保证对抗公平、有序进行
- 制定规则、监督过程
- 评估效果
- 推动双方改进

紫队，很多人不理解，
觉得有啥用？
我给你说说紫队干啥的活儿：

```
1. 前期准备
   • 定目标、定范围、定规则
   • 定时间、定方案
   • 两边沟通协调

2. 过程监督
   • 监督红队有没有违规
   • 监督蓝队有没有作弊
   • 有争议了仲裁
   • 出问题了叫停

3. 效果评估
   • 评估红队的攻击成果
   • 评估蓝队的防守效果
   • 打多少分、排名次
   • 出评估报告

4. 推动改进
   • 组织复盘
   • 推动整改
   • 跟踪改进效果
   • 帮助双方提升
```

紫队很重要的角色。
没有紫队，
红蓝双方很容易吵起来。
- 红队说我赢了，蓝队说你违规了
- 蓝队说我们防住了，红队说你根本没发现我

这时候就需要紫队来当裁判。

而且，很多时候对抗完了就完了，
没人跟进改进。
紫队可以跟进改进、推动落地。

当然，也不是所有对抗都有紫队。
小的对抗，可能就安全负责人兼了。
但正规的、大型的对抗，
紫队是必须有的。

## 4.4 红蓝对抗的全流程

一场完整的红蓝对抗，
一般分五个阶段：

```
第一阶段：前期准备
第二阶段：对抗实施
第三阶段：收尾总结
第四阶段：整改加固
第五阶段：复测验证
```

一个一个说。

### 4.4.1 第一阶段：前期准备

前期准备非常重要。
准备工作做得好不好，
直接决定了对抗的效果。

前期准备都干啥？

```
1. 明确目标和范围
   • 这次对抗的目标是什么？
   • 检验能力？还是锻炼队伍？
   • 哪些系统在范围内？哪些不在？
   • 哪些事能做，哪些不能做？
   • 时间多久？

2. 定规则
   • 红队哪些攻击手法
   • 哪些工具能做、哪些不能做
   • 什么情况要叫停
   • 怎么算成功、怎么算失败
   • 评分标准是什么？

3. 签合同/授权书
   • 白纸黑字写清楚
   • 别到时候出事了说不清楚
   • 法律风险要规避

4. 信息收集/环境确认
   • 给红队哪些信息？（黑盒/白盒/灰盒）
   • 哪些系统是重点保护目标？
   • 网络拓扑、资产清单确认

5. 蓝队准备
   • 蓝队进入状态
   • 设备检查、日志检查
   • 人员到位、值班安排
   • 应急预案准备

6. 启动会
   • 各方到一块
   • 目标、规则、注意事项都说清楚
   • 有问题提前问
   • 然后宣布：对抗开始
```

前期准备越充分，
对抗效果就越顺利。
很多对抗搞不好，
就是因为前期没准备好，
中间各种扯皮、各种意外。

### 4.4.2 第二阶段：对抗实施

准备工作做完了，
就进入对抗实施阶段。
就是真刀真枪开干了。

这个阶段一般是这样的：

```
1. 红队进攻
   • 红队按照计划开始攻击
   • 从外网打点、钓鱼、社工...
   • 想办法打进去
   • 内网横向移动
   • 扩大战果

2. 蓝队防守
   • 蓝队正常监测、分析
   • 发现异常、分析、上报
   • 该响应、处置
   • 溯源分析

3. 过程管理
   • 紫队全程监督
   • 记录攻击事件
   • 有争议仲裁
   • 紧急情况叫停

4. 每日沟通
   • 一般每天会有个沟通会
   • 紫队通报一下进展
   • 有什么问题及时沟通
   • （当然不会告诉蓝队红队在哪，那
   就是说整体进度之类的）

5. 结束
   • 时间到了，或者红队达成目标了
   • 对抗结束
   • 红队提交成果
   • 蓝队提交防守报告
```

这个阶段是最精彩的，
攻防博弈，斗智斗勇。

这个阶段的时间，
短的几天，长的几周，
护网级的可能一个月。

### 4.4.3 第三阶段：收尾总结

对抗结束了，
不是就完了，
还有很多收尾工作要做。

```
1. 红队提交报告
   • 攻击路径
   • 发现的问题
   • 拿到了哪些成果
   • 改进建议

2. 蓝队提交报告
   • 发现了哪些攻击
   • 哪些防住了，哪些没防住
   • 响应过程
   • 溯源结果
   • 改进计划

3. 紫队评估
   • 整体评估
   • 攻防双方表现打分
   • 整体安全水平评估
   • 综合报告

4. 复盘会
   • 红队讲攻击思路、怎么打的
   • 蓝队讲怎么防的、怎么发现的
   • 一起复盘整个过程
   • 哪里做得好，哪里做得不好
   • 以后怎么改

5. 现场恢复
   • 红队留下的后门、木马都清掉
   • 系统恢复到对抗前的状态
   • 账号、权限都收回来
```

很多人对抗一结束就散了，
这是不对的。
收尾工作很重要，
尤其是复盘会。
攻防双方面对面复盘，
收获才最大。

### 4.4.4 第四阶段：整改加固

总结完了，
发现了一堆问题。
那接下来就是整改了。

```
1. 问题梳理
   • 把所有问题列出来
   • 分类、分级
   • 哪些高危、哪些中危、哪些低危

2. 制定整改计划
   • 每个问题怎么改
   • 谁来改
   • 什么时候改完
   • 验收标准是什么

3. 整改实施
   • 各个部门按照计划整改
   • 安全部门跟进
   • 定期汇报进度

4. 过程监督
   • 紫队或者安全部门跟进
   • 有困难协调资源
   • 推动整改进度
```

这一步是最关键的，
也是最容易被忽略的。
很多公司对抗搞完了，
报告出了，
然后放那了，
没人整改，
那钱就白花了。

发现问题不整改，
等于白搞。

### 4.4.5 第五阶段：复测验证

整改完了，
是不是真的改好了吗？
得测一测。

```
1. 整改完成后，安排复测
   • 红队再测一遍
   • 看看问题是不是真的修好了
   • 有没有新的问题

2. 验证通过了，就闭环了
   没通过，回去继续改

3. 总结归档
   • 所有资料归档
   • 经验教训总结
   • 下次对抗的基础

4. 常态化
   • 定期搞红蓝对抗
   • 每次都有提升
   • 形成良性循环
```

到这一步，
一场完整的红蓝对抗才算真的结束。
从准备到复测，
形成一个完整的闭环。

---

五个阶段，
一环扣一环，
哪个环节没做好，
效果都会打折扣。

---

## 4.5 红蓝对抗的几个关键原则

最后，聊聊红蓝对抗要搞得好，
有几个关键原则要把握。

### 原则一：目标先行，规则前置

啥意思？
就是先把目标和规则定清楚，再开始。

很多对抗开始之前，
一定要先说好：
- 这次对抗的目标是什么？
- 范围是什么？
- 哪些能干、哪些不能干？
- 怎么算赢、怎么算输？

别稀里糊涂就开始了，
打到一半扯皮。

目标和规则说清楚了，
大家都能少很多麻烦。

### 原则二：对事不对人

这个非常重要。

红蓝对抗，是检验安全能力，
不是考哪个不行。
红队赢了，不代表红队厉害、蓝队菜。
红队赢了，说明我们整体安全还有问题。
大家一起改进，就好了。

千万别搞成了对立，
- 红队想方设法阴蓝队
- 蓝队想方设法耍赖
- 最后变成了斗气

那就变味了。

一定要强调：
**我们是一伙的，目标是一致的，
都是为了提升整体安全能力。**

红队是来帮忙找问题的，
不是来砸场子的。
蓝队是来检验的，
不是来考试的。

心态摆正了，
对抗才有意义。

### 原则三：保密第一

安全第一，
这个是底线。

什么意思？
- 不能影响业务正常运行
- 不能搞破坏
- 不能偷真实数据
- 不能搞出事

红队操作要小心再小心
别为了赢，把人家业务搞挂了，
那就麻烦大了。

所以，对影响业务的操作，
一定要提前报备、一定要谨慎。
真出了问题，
及时叫停，
安全第一。

### 原则四：重在改进，不是输赢

很多人搞红蓝对抗，
特别看重输赢。
红队想赢，蓝队也想赢。
这很正常。

但要记住：
**输赢不是目的，改进才是目的。**

就算红队全赢了，
把问题都暴露出来了，
然后都改了，
那这场对抗就是成功的。

反过来，
就算蓝队赢了，
但其实是红队没好好打，
没发现什么问题，
那这场对抗就是失败的。

输赢不重要，
有没有真东西才重要。

## 4.6 本章总结

这一章我们讲了红蓝对抗的组织与流程，总结一下：

**红蓝对抗的类型：
- 按目标分：检验型 vs 提升型
- 按信息多少分：黑盒 vs 白盒 vs 灰盒
- 按激烈程度分：普通演练 vs 护网级
- 按红队来源分：内部红队 vs 外部红队

**红蓝紫铁三角：**
- 🔴 红队：进攻方，模拟真实攻击
- 🔵 蓝队：防守方，监测、检测、响应、溯源
- 🟣 紫队：裁判+教练，定规则、监督、评估、推动改进

**对抗全流程五个阶段：**
1. 前期准备：定目标、定规则、签授权、做准备、开启动会
2. 对抗实施：红队攻、蓝队防、紫队监督
3. 收尾总结：双方出报告、开复盘会、现场恢复
4. 整改加固：问题梳理、整改计划、实施整改
5. 复测验证：复测整改效果、闭环、归档、常态化

**四个关键原则：**
1. 目标先行，规则前置
2. 对事不对人
3. 安全第一
4. 重在改进，不在输赢

最后送你一句话：

> **一场好的红蓝对抗，
> 不是红队赢了，也不是蓝队赢了，
> 而是整个公司的安全能力赢了。**
