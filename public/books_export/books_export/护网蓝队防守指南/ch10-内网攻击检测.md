# 第十章 内网攻击检测 —— 严防死守内部

> 第10章 | 预计学习时间：5.5小时 | 难度：⭐⭐⭐⭐

---

## 📖 写在前面

很多人觉得，安全防护就是防边界。
防火墙建高点，WAF配好点，就安全了。

大错特错。

现在的攻击，都是这样的：
- 边界突破 → 拿下一台机器 → 内网横向移动 → 打域控 → 偷数据

边界只是第一道防线。
真正的战斗，在内网。

很多单位，边界搞得严严实实，内网却像个筛子。
- 一台机器被攻破了，整个内网横着走
- 想登哪台登哪台，想访问啥访问啥
- 域控跟纸糊的一样，一捅就破

这怎么行？

所以，内网攻击检测非常重要。

这一章，我们就来聊聊：
- 内网攻击都有哪些手法？
- 怎么检测横向移动？
- 怎么检测域渗透？
- 怎么检测权限提升？
- 内网安全怎么搞？

坐稳了，开唠。

## 10.1 先从一个故事讲起

### 10.1.1 从一台Web服务器到域控

先给你讲个真实的故事。

有一年护网，我认识的一个蓝队队长，姓张。

张队长他们单位，边界防护做得还可以：
- 下一代防火墙
- IDS/IPS
- WAF
- 邮件网关

但是内网呢？基本没啥防护。
- 内网没有分段
- 服务器之间随便访问
- 管理员账号到处用
- 也没有内网流量监测

护网开始后，前三天没事。

第四天，出事了。

凌晨，张队长收到WAF告警：有一个Web应用被攻击了，疑似上传了WebShell。

张队长赶紧带人查。
一查，坏了——Web服务器确实被上传了WebShell。

"还好发现得早，赶紧删了，补漏洞，应该没事吧？"张队长当时是这么想的。

结果呢？
他们只处理了这台Web服务器，没查内网。

两天后，更大的事来了——
域控被登录了，而且是用一个普通用户账号登录的。

这不对啊，普通用户怎么能登录域控？

赶紧查。
这一查，吓出一身冷汗。

攻击者早在两天前进了Web服务器之后，就没闲着：
- 当天就提权了
- 然后抓了密码哈希
- 然后用哈希传递，登录了同网段的另一台服务器
- 然后又从那台服务器，抓了更多密码
- 然后一台一台横向移动
- 最后，找到了一个域管理员的账号
- 用域管理员账号，登录了域控

两天时间，从一台Web服务器，打到了域控。

而张队长他们，直到人家登录了域控才发现。

中间那么多横向移动的动作，一个告警都没有。
为啥？
因为内网没有监测啊。

人家在内网里逛了两天，跟逛自己家后花园一样。

---

这个故事告诉我们什么？

**边界被突破是迟早的事，内网防护才是关键。**

别把所有宝都压在边界上。
边界总有被突破的一天。
内网安全才是真正的考验。

内网的检测和防护，做不好的话，边界再强也没用。

## 10.2 内网攻击都有啥手法？

要检测内网攻击，首先得知道内网攻击都有哪些手法。

我给你梳理一下，从拿到一台机器权限之后，攻击者一般会干啥。

### 10.2.1 内网信息收集

攻击者刚进来，第一步肯定是摸情况。
我在哪？我是谁？这网络里有啥？

**常见的信息收集手法：**

```
本机信息收集：
  • 我是谁？（当前用户名、权限）
  • 这台机器是啥？（操作系统、IP、 hostname）
  • 这台机器在哪个域？（域信息）
  • 这台机器上有啥？（文件、密码、配置...）
  • 网络配置是什么？（IP段、网关、DNS）

域信息收集：
  • 域内有多少用户？
  • 域内有多少机器？
  • 域控是哪台？
  • 有哪些组？（管理员组、域管理员组...）
  • 有哪些OU？

网络探测：
  • 扫网段，看哪些机器在线
  • 扫端口，看哪些端口开着
  • 看哪些机器有漏洞
  • 找重要的机器（域控、数据库、文件服务器...）
```

这些是攻击者进来之后的"常规操作"。
先把内网摸个底朝天，然后再动手。

### 10.2.2 权限提升

拿到一台机器了，但可能只是普通用户权限。
不够啊，得想办法弄到管理员权限。

**常见的提权手法：**

```
系统漏洞提权：
  • Windows提权漏洞（MS14-068、MS16-032、CVE-2021-40449...）
  • Linux提权漏洞（脏牛、SUDO漏洞...）
  • 数据库提权（UDF提权、MOF提权...）

错误配置提权：
  • 服务路径有空格/没引号
  • 服务权限配置错误
  • 计划任务权限过高
  • 注册表权限配置错误
  • AlwaysInstallElevated
  • 等等...

第三方软件/服务提权：
  • 有漏洞的第三方服务
  • 以管理员权限运行的软件有漏洞
  • 数据库提权（MySQL、MSSQL...）
```

提权是内网渗透的关键一步。
拿到管理员权限，后面的操作就方便多了。

### 10.2.3 凭据窃取

有了管理员权限，下一步就是偷密码、偷哈希、偷票据。

为啥？
因为偷到了凭据，就能去登录其他机器了。

**常见的凭据窃取手法：**

```
抓密码哈希：
  • Mimikatz 抓内存里的密码和哈希
  • 导出SAM和SYSTEM，拿到本地账号哈希
  • Procdump + mimikatz
  • 各种抓密码的工具

哈希传递（Pass-the-Hash, PtH）：
  • 不用明文密码，用哈希就能登录
  • 横着走的神器

票据传递（Pass-the-Ticket, PtT）：
  • 偷到Kerberos票据，就能冒充别人
  • 黄金票据、白银票据
  • 域渗透常用手法

其他：
  • 浏览器里保存的密码
  • 各种客户端里保存的密码（FTP、邮箱、远程桌面...）
  • 配置文件里的明文密码
```

凭据窃取是横向移动的基础。
偷到的凭据越多，能打的机器就越多。

### 10.2.4 横向移动

偷到了凭据，就开始横向移动了。
从这台机器，跳到那台机器，一台一台打过去。

**常见的横向移动手法：**

```
远程登录：
  • 远程桌面（RDP）
  • SSH
  • Telnet

远程执行：
  • PsExec
  • WMI
  • WinRM
  • Schtasks（计划任务）
  • SC（服务控制）
  • DCOM

哈希传递：
  • 用偷来的哈希，直接登录其他机器
  • 不用明文密码

其他：
  • IPC$连接
  • 共享文件夹
  • 组策略（GPO）投毒
  • DCSync
```

横向移动是内网渗透的核心。
攻击者就是这样，一台打一台，最后打遍整个内网。

### 10.2.5 域渗透

如果内网有域，那域控就是最终目标。
拿到域控，就等于拿下了整个域。

**常见的域渗透手法：**

```
信息收集：
  • 域内用户、机器、组、OU
  • 域控定位
  • 域管理员定位

权限获取：
  • 找域管理员登录过的机器，偷他的哈希/票据
  • 利用域管理员的进程
  • 钓鱼钓域管理员

权限维持：
  • 黄金票据（Golden Ticket）
  • 白银票据（Silver Ticket）
  • 域管理员后门账号
  • DCSync权限
  • AdminSDHolder
  • 等等...

域控攻击：
  • DCSync（复制域内所有哈希）
  • DCShadow（伪造域控）
  • 直接登录域控
```

拿下域控，就是内网渗透的终极目标。
拿到域控，整个域就都是你的了。

## 10.3 怎么检测内网攻击？

讲完了攻击手法，再说说怎么检测。

### 10.3.1 检测信息收集行为

攻击者进来，第一步是信息收集。
如果能在这个阶段就发现，那损失最小。

**怎么检测？**

```
端口扫描/主机探测：
  • 短时间内，一个IP扫很多其他IP的端口
  • 同一个源IP，大量的SYN包
  • 大量的ICMP包（ping扫描）
  • 检测工具：NIDS、全流量分析、EDR

异常的信息收集命令：
  • 执行了常见的信息收集命令
    - net user / domain
    - net group "domain admins" /domain
    - net view /domain
    - whoami /all
    - ipconfig /all
    - systeminfo
    - tasklist /v
    - netstat -ano
  • 检测工具：EDR、HIDS、命令行审计

异常的查询行为：
  • 大量的LDAP查询（查询域信息）
  • 大量的DNS查询（查主机名）
  • 异常的SMB连接（扫共享）
  • 检测工具：域控日志、流量分析
```

发现这些行为，就要警惕了：
是不是有人在内网里摸情况？

### 10.3.2 检测权限提升

接下来是提权。
普通用户突然变成管理员了，这肯定有问题。

**怎么检测？**

```
新的管理员账号/权限变化：
  • 突然多出了管理员账号
  • 普通用户被加入了管理员组
  • 事件ID：4720（新建用户）、4732（加入组）
  • 检测工具：Windows事件日志、EDR

异常的进程创建：
  • 普通用户的进程，创建了高权限的子进程
  • 进程的完整性级别突然升高
  • 检测工具：EDR、系统日志

提权漏洞的利用痕迹：
  • 某些提权漏洞有特定的特征
  • 系统日志里有错误/崩溃信息
  • 检测工具：EDR、漏洞检测

异常的服务/计划任务：
  • 突然多了可疑的服务
  • 突然多了可疑的计划任务
  • 服务指向的文件很可疑
  • 事件ID：4697（服务安装）、4698（计划任务创建）
  • 检测工具：Windows事件日志、EDR
```

提权是个关键节点。
在提权的时候就发现，就能把攻击者按住。

### 10.3.3 检测凭据窃取

提权之后，攻击者肯定会偷凭据。
Mimikatz一跑，密码哈希全到手。

**怎么检测？**

```
Mimikatz等工具的特征：
  • 进程名是 mimikatz 或者常见的变体
  • 进程访问了 lsass.exe（Mimikatz要读lsass内存）
  • 检测工具：EDR、HIDS

异常的内存读取行为：
  • 非系统进程，读取 lsass.exe 的内存
  • 这是抓密码的典型特征
  • 检测工具：EDR

异常的注册表操作：
  • 读取 SAM、SYSTEM 这些注册表项
  • （用来导出密码哈希）
  • 检测工具：EDR、注册表监控

凭据被使用的痕迹：
  • 偷到的哈希，被用来远程登录其他机器
  • 异常的登录行为
  • 检测工具：登录日志、流量分析
```

凭据窃取是横向移动的前奏。
发现有人在偷凭据，就要赶紧响。

### 10.3.4 检测横向移动

横向移动是内网攻击的核心。
如果能在横向移动的早期发现，就能把影响降到最小。

**怎么检测？**

```
异常的远程登录：
  • 一个账号，短时间内登录很多台机器
  • 平时不登录的机器，突然有人登录了
  • 非正常上班时间的登录
  • 事件ID：4624（登录成功）、4625（登录失败）
  • 检测工具：Windows事件日志、SIEM

异常的远程执行：
  • PsExec、WMI、WinRM 这些远程执行工具被使用
  • 服务创建事件（PsExec会创建服务）
  • WMI活动日志
  • WinRM日志
  • 检测工具：Windows事件日志、EDR

异常的SMB连接：
  • 一台机器，短时间内连接很多其他机器的445端口
  • 异常的IPC$连接
  • 异常的文件共享访问
  • 检测工具：流量分析、NIDS

哈希传递的检测：
  • 登录类型是3（网络登录），但没有对应的明文认证
  • 异常的NTLM认证
  • 事件ID：4624，登录类型3
  • 检测工具：域控日志、SIEM
```

横向移动的检测很重要，也很难。
因为攻击者用的都是"合法"的手段——
用正常的远程桌面、正常的WMI、正常的管理员账号。
看起来跟正常的运维操作一模一样。

怎么区分是正常运维还是攻击？
这就需要基线了——
你得知道正常情况下是什么样的。
跟正常的不一样，就是异常。

### 10.3.5 检测域渗透

如果内网有域，那域渗透检测就是重中之重。
域控要是被拿下了，基本就全完了。

**怎么检测？**

```
异常的域管理员登录：
  • 域管理员账号，登录了不该登录的机器
  • 域管理员账号，从异常的IP登录
  • 非工作时间的域管理员登录
  • 检测工具：域控日志、SIEM

异常的域控访问：
  • 不是管理员的账号，访问了域控的敏感资源
  • 大量的LDAP查询（在摸域信息）
  • 检测工具：域控日志、流量分析

DCSync检测：
  • 非域控的机器，请求了目录复制权限
  • （DCSync就是利用这个复制哈希）
  • 事件ID：4662（对象访问），属性有复制权限的GUID
  • 检测工具：域控日志

黄金票据/白银票据：
  • 票证授予时间异常
  • 票证的用户名和RID不匹配
  • 检测工具：域控日志、Kerberos日志

域内权限变化：
  • 有人被加入了域管理员组
  • 有新的域管理员账号
  • AdminSDHolder被修改
  • 事件ID：4728、4732、4756...
  • 检测工具：域控日志
```

域渗透检测是蓝队的高阶技能。
需要对域、对Kerberos、对活动目录，有比较深入的理解。

但这也是最值钱的技能。
域防护做好了，内网就稳了一大半。

## 10.4 内网安全防护体系

讲完了检测，再说说防护。
光检测不防护，等于只报警不抓人。

内网安全防护怎么搞？
给你一个"内网纵深防御"的思路。

### 10.4.1 第一层：网络分段

最外面一层，网络分段。

啥叫网络分段？
就是把一个大内网，分成很多小的网段。
不同的业务、不同的级别，放在不同的网段里。
网段之间有访问控制，不是谁都能访问谁。

```
常见的分段方式：
  • 按业务分：生产区、办公区、测试区、DMZ区
  • 按级别分：核心区、重要区、一般区
  • 按部门分：财务部、人事部、研发部...

分段之后：
  • DMZ区的服务器被攻破了，攻击者只能在DMZ里折腾
  • 想进核心区？没那么容易
  • 大大增加了横向移动的难度
```

网络分段是内网防护的基础。
不做分段，内网就是一片平原，攻击者骑着马就能冲过去。

### 10.4.2 第二层：身份与访问控制

接下来是身份安全。
现在都讲"零信任"，核心就是身份。

```
怎么做：
  • 最小权限原则：每个账号只给必要的权限
  • 管理员账号越少越好：别人人都是管理员
  • 多因素认证（MFA）：重要系统、管理员账号，都要开MFA
  • 定期清理账号：离职的、不用的，都删掉
  • 密码策略：强密码、定期改
  • 特权账号管理（PAM）：管理员账号统一管理、审批、审计
```

很多内网渗透，最后都是偷到了管理员的密码，然后一路打到底。
如果管理员账号管得严，攻击者就难多了。

### 10.4.3 第三层：主机安全

每一台主机，都是一个防线。

```
主机安全怎么做：
  • 系统加固：关端口、关服务、最小权限
  • 补丁管理：及时打补丁，特别是高危漏洞
  • EDR/HIDS：装终端检测与响应产品
  • 应用白名单：只允许运行指定的程序，其他都不让跑
  • 禁用不必要的功能：比如WMI、WinRM、PSRemoting，不用就关了
  • 本地管理员密码不一样：每台机器的本地管理员密码都不同（LAPS）
```

每台主机都做好安全，攻击者就算进来了，也很难提权、很难横向移动。

### 10.4.4 第四层：数据安全

最核心的，还是数据。
就算攻击者打进内网了，也让他拿不到数据。

```
数据安全怎么做：
  • 数据分类分级：哪些是核心数据，哪些是一般数据
  • 加密存储：敏感数据加密存
  • 数据访问控制：谁能访问什么数据，都控制好
  • 数据防泄露（DLP）：防止数据被拷出去、传出去
  • 数据库审计：谁查了什么数据，都记下来
  • 备份：数据被删了还能恢复
```

数据是核心资产。
数据安全做好了，就算内网被打穿了，损失也不会太大。

## 10.5 内网检测的难点与应对

最后，聊聊内网检测的几个难点，以及怎么应对。

### 难点一：正常和攻击不好区分

很多内网攻击手法，用的都是"正常"的工具和协议。
- 远程桌面，正常运维也用，攻击者也用
- WMI，正常管理也用，攻击者也用
- PowerShell，正常脚本也用，攻击者也用

怎么区分是正常操作还是攻击？

**应对方法：**
- 建立正常基线：知道正常情况下是什么样的
- 看上下文：这个操作，是不是这个人该干的？时间、地点、方式对不对？
- 关联分析：单独一个操作可能正常，但和其他异常行为凑到一起，就不正常了
- 行为分析：每个人的操作习惯是不一样的，不符合习惯的就是可疑的

### 难点二：内网流量太大，存不起

内网的东西向流量，比南北向流量大得多。
全流量存储？成本太高了，存不起。

**应对方法：**
- 重点区域全流量：核心区、重要服务区，全流量存
- 其他区域存元数据：NetFlow/IPFIX，只存摘要，不存完整包
- 按需抓包：发现可疑的，再去抓那个会话的包
- 分级存储：近期的存细粒度的，久一点的存粗粒度的

平衡成本和效果，找一个最优点。

### 难点三：告警太多，看不过来

内网里机器多、流量大，告警也多。
一天几万条告警，根本看不过来。

**应对方法：**
- 告警分级：高危的立刻看，中危的尽快看，低危的有空看
- 告警聚合：同一次攻击的告警，聚合成一个事件
- 关联分析：把相关的告警串起来，形成完整的攻击链
- 优先级排序：严重的、离核心资产近的，优先处理

把精力放在最重要的告警上。

### 难点四：攻击者用合法工具

现在的攻击者，越来越喜欢用"合法工具"。
- 用系统自带的PowerShell，不用自己写木马
- 用PsExec、WMI这些正常的管理工具
- 用白名单里的程序加载恶意代码（"白利用"）

这样杀毒软件、EDR都很难检测，因为都是正常的程序。

**应对方法：**
- 行为检测：不看是什么程序，看它在干什么
- 命令行审计：PowerShell、cmd 的命令都记下来，找异常的
- 减少攻击面：不用的功能就关了，比如禁用不必要的PowerShell功能
- 应用白名单：只允许运行指定的程序，其他都不让跑

从"黑名单"思维，转向"白名单"思维。

## 10.6 本章总结

这一章我们讲了内网攻击检测，内容很多，给你总结一下：

**内网攻击的常见手法：**
1. 信息收集（本机、域、网络）
2. 权限提升（漏洞提权、配置错误提权）
3. 凭据窃取（抓哈希、抓票据、哈希传递）
4. 横向移动（远程桌面、PsExec、WMI、哈希传递...）
5. 域渗透（信息收集、权限获取、域控攻击、权限维持）

**怎么检测内网攻击：**
- 检测信息收集：端口扫描、异常命令、异常查询
- 检测提权：权限变化、异常进程、可疑服务/计划任务
- 检测凭据窃取：Mimikatz特征、内存读取、注册表操作
- 检测横向移动：异常登录、异常远程执行、异常SMB连接、哈希传递
- 检测域渗透：异常域管理员登录、DCSync、黄金票据、权限变化

**内网纵深防御体系：**
1. 网络分段：把大内网分成小网段，层层设防
2. 身份与访问控制：最小权限、MFA、特权账号管理
3. 主机安全：加固、补丁、EDR、应用白名单
4. 数据安全：加密、访问控制、DLP、备份

**内网检测的难点：**
- 正常和攻击不好区分 → 建立基线、关联分析、行为分析
- 内网流量太大存不起 → 重点区域全流量、其他存元数据
- 告警太多看不过来 → 分级、聚合、关联、排优先级
- 攻击者用合法工具 → 行为检测、命令行审计、减少攻击面、白名单

记住一句话：

> **边界被突破是迟早的，内网才是真正的战场。**

别把所有希望都寄托在边界上。
内网防护做好了，就算边界被突破了，攻击者也翻不了天。
