这一章，我就给你画一张蓝队的作战地图。看完你就知道：

- 每个阶段要干啥？
- 怎么系统性地做防守？
- 护网来了怎么安排？

脑子里有了这张地图，干活就不慌了。

## 2.1 先从一个故事讲起

### 2.1.1 小明的困惑

小明刚毕业，进了一家公司的安全团队当蓝队工程师。

第一天上班，队长扔给他一个账号："这是SIEM的账号，你先看看告警，熟悉熟悉。"

小明打开SIEM一看，好家伙——

一天好几千条告警，各种颜色、各种级别、各种类型，看的他眼花缭乱。

"这么多告警，我先看哪个？"
"哪些是真的，哪些是误报？"
"真攻击的话，我该怎么处理？"

小明硬着头皮看了一天，看了几百条告警，大部分都是误报，真正有威胁的没几条。

晚上下班，小明觉得很挫败："我这一天都干啥了？看了一堆没用的东西？"

队长看出了他的困惑，说："别急，新人都这样。你现在是'只见树木，不见森林'。等你脑子里有了一张完整的防守地图，就知道该怎么干了。"

小明问："什么是防守地图？"

队长说："走，我给你讲讲。"

## 2.2 蓝队的经典防御框架

打仗讲究"兵法"，防守也讲究"方法论"。蓝队有几个经典的防御框架，你得知道。

### 2.2.1 最经典的：P2DR模型

这个模型最经典，也最实用。

P2DR是四个英文单词的缩写：

```
Policy（安全策略）
Protection（防护）
Detection（检测）
Response（响应）
```

四个部分组成一个闭环：

```
          ┌─────────────┐
          │   Policy    │  安全策略：制定规则
          └──────┬──────┘
                 │
          ┌──────▼──────┐
          │ Protection  │  防护：建防御工事
          └──────┬──────┘
                 │
          ┌──────▼──────┐
          │  Detection  │  检测：发现攻击
          └──────┬──────┘
                 │
          ┌──────▼──────┐
          │  Response   │  响应：处置攻击
          └──────┬──────┘
                 │
                 └─────── 反馈到策略，形成闭环
```

我给你翻译翻译，用人话说：

**Policy（安全策略）**：就是"规矩"。先定下来，我们要防什么？怎么防？什么级别？什么允许？什么不允许？比如：
- 外部不能直接访问数据库
- 服务器必须开防火墙
- 密码必须三个月改一次
- 敏感数据必须加密

没有策略，就没有方向。你都不知道要防啥，怎么防？

**Protection（防护）**：就是"建防线"。按照策略，把防御工事建起来。比如：
- 装防火墙
- 装IDS/IPS
- 服务器加固
- 打补丁
- 部署EDR
- 上WAF

这就是我们常说的"事前"阶段——攻击没来之前，先把防线建好。

**Detection（检测）**：就是"盯着"。防线建好了，得有人盯着，看看有没有人打进来。比如：
- 7×24小时监控
- 日志分析
- 流量分析
- 告警研判
- 威胁狩猎

这是"事中"阶段——攻击发生的时候，得能发现。

**Response（响应）**：就是"动手"。真发现攻击了，赶紧处理。比如：
- 封IP
- 断网
- 杀进程
- 删木马
- 溯源分析
- 漏洞修复

这是"事后"阶段——攻击发生之后，赶紧处置，减少损失。

然后呢？然后把这次攻击的经验教训，反馈到策略里，改进策略、加固防护，形成一个闭环。

一次又一次，越防越好。

这就是P2DR模型，简单、实用、好记。

### 2.2.2 最全面的：PPDRR模型

在P2DR的基础上，又发展出了PPDRR模型，多了两个P和一个R：

```
Policy（安全策略）
Protection（防护）
Detection（检测）
Response（响应）
Recovery（恢复）
```

多了个Recovery（恢复）——攻击完了之后，怎么把系统恢复正常？

比如：
- 数据恢复
- 系统重建
- 业务恢复
- 服务上线

也挺有道理的。响应完了不是就完事了，还得把系统恢复到正常状态，业务得接着跑啊。

### 2.2.3 最有名的：ATT&CK防御视角

说到攻击，大家都知道MITRE ATT&CK框架。

那防御呢？MITRE也有防御的框架，叫**ATT&CK for Defense**，或者叫**Mitigations（缓解措施）**。

简单说就是，ATT&CK把攻击手法都列出来了，然后对应的，每个攻击手法都有对应的防御方法。

比如：
- 攻击手法：钓鱼邮件
- 防御方法：邮件网关、安全意识培训、沙箱检测

- 攻击手法：哈希传递（PTH）
- 防御方法：禁用NTLM、启用Windows Defender Credential Guard、限制管理员登录

你把ATT&CK的攻击手法挨个看一遍，对应的防御方法也看一遍，就知道该怎么防了。

这个很实用，强烈建议去看看。

### 2.2.4 最接地气的：纵深防御

这个不是什么模型，是一种思想，也是我最推崇的。

啥叫纵深防御？

简单说就是：**别把鸡蛋放在一个篮子里。**

别指望一道防线就能防住所有攻击。要建多层防线，一层一层的，攻破了一层还有下一层。

就像古代的城池：
- 最外面是护城河
- 然后是城墙
- 然后是瓮城
- 然后是内城
- 然后是皇宫
- 最后还有御林军

一层一层，攻破一层还有一层。

网络安全也是一样：
- 第一层：边界防火墙
- 第二层：IDS/IPS
- 第三层：WAF
- 第四层：终端杀毒/EDR
- 第五层：身份认证（IAM）
- 第六层：数据加密
- 第七层：备份恢复

红队攻破了防火墙？没关系，还有WAF。
攻破了WAF？没关系，还有EDR。
攻破了EDR？没关系，还有数据加密。
数据加密也被绕过去了？没关系，还有备份，大不了恢复。

一层一层，总有一层能拦住。

这就是纵深防御的思想。

别迷信某一个产品、某一种技术。没有银弹。

## 2.3 蓝队工作的五个阶段

讲完了框架，我们来聊聊实战。

在真实的护网或者红蓝对抗中，蓝队的工作一般分五个阶段：

```
1. 准备阶段  →  护网前一个月到一周
2. 监测阶段  →  护网期间，7×24小时
3. 检测阶段  →  发现告警后，分析研判
4. 响应阶段  →  确认攻击后，应急处置
5. 恢复阶段  →  处置完后，恢复加固
```

一个阶段一个阶段来，我给你详细讲。

### 2.3.1 第一阶段：准备阶段——不打无准备之仗

这个阶段是护网开始之前，大概提前一到两个月就开始了。

这个阶段的核心目标是：**把该补的漏洞补上，该加固的加固好，该准备的准备好，尽最大可能减少攻击面。**

具体干啥呢？

**第一件事：资产梳理**

先搞清楚，我们有多少资产？
- 多少台服务器？
- 多少个网络设备？
- 多少个业务系统？
- 多少个域名？
- 多少个IP段？
- 哪些是重要的？哪些是一般的？
- 哪些暴露在公网？哪些在内网？

资产都搞不清楚，怎么防？

很多公司被打进来，就是因为资产没摸清——有一台测试服务器暴露在公网上，没人管，漏洞一大堆，红队就是从这台机器打进来的。

**第二件事：漏洞扫描与修复**

资产摸清了，然后扫漏洞。

全扫一遍：
- 系统漏洞
- Web漏洞
- 配置错误
- 弱口令

扫出来之后，按优先级修复：
- 高危的，赶紧补
- 中危的，排期补
- 低危的，记着以后补

漏洞越少，攻击面越小，红队越难打进来。

**第三件事：安全加固**

光补漏洞还不够，还得加固。

怎么加固？
- 服务器加固（关端口、关服务、最小权限）
- 网络加固（访问控制、VLAN划分）
- 应用加固（安全配置、WAF规则）
- 账号加固（禁用默认账号、改默认密码、强密码策略）
- 日志加固（开日志、集中存储、防篡改）

加固做得好，能防住80%的攻击。

**第四件事：安全产品检查**

家里那些安全设备，都检查一遍，确保都能用、都好用。

- 防火墙：规则对不对？有没有失效的规则？
- IDS/IPS：特征库更新了没？策略调好了没？
- WAF：规则优化了没？误报多不多？
- EDR：Agent都装了没？版本对不对？
- SIEM：日志都接进来了没？规则写了没？
- 态势感知：数据对不对？有没有漏的？

别等护网开始了才发现，某个设备早就挂了，或者日志没接进来。那可就凉了。

**第五件事：应急预案与演练**

写应急预案，真出事了怎么办？

- 发现WebShell怎么办？
- 发现服务器被入侵怎么办？
- 发现勒索病毒怎么办？
- 发现数据泄露怎么办？
- 谁负责什么？谁来决策？怎么上报？

写完了还要演练一下，看看预案好不好用，有没有问题。

不然真出事了，手忙脚乱的，不知道该干啥。

**第六件事：安全意识培训**

很多时候，红队不是从技术打进来的，是从人打进来的。

一封钓鱼邮件，员工一点，就中招了。

所以护网前，一定要给全体员工做安全意识培训：
- 什么是钓鱼邮件？怎么识别？
- 陌生邮件别乱点
- 陌生链接别乱开
- 陌生文件别乱下载
- 密码要保管好，别告诉别人
- 遇到可疑情况找谁报告

人是最薄弱的环节，也是最重要的防线。

准备阶段做扎实了，护网就能轻松很多。

准备做得好，护网没烦恼。

### 2.3.2 第二阶段：监测阶段——千里眼顺风耳

护网开始了，进入监测阶段。

这个阶段的核心目标是：**7×24小时盯着，有攻击第一时间发现。**

怎么监测？

**第一件事：监控大屏**

这个大家最熟悉了，就是那个花花绿绿的大屏。

大屏上显示啥？
- 实时告警
- 攻击趋势
- 攻击来源
- 攻击类型
- 安全状态

大屏的作用是宏观态势感知，告诉你"大概有没有事"。

但别光看大屏，大屏只能看个大概，具体还得看告警。

**第二件事：告警监控**

盯着SIEM、IDS、WAF、EDR这些安全设备的告警。

告警来了，先分个级：
- 高危告警：必须立刻看
- 中危告警：尽快看
- 低危告警：没事了再看

优先级要搞清楚，别在低危告警上浪费太多时间，高危告警反而漏了。

**第三件事：日志分析**

光靠告警还不够，很多攻击是没有告警的。

你得主动去看日志：
- 关键服务器的登录日志
- 重要系统的操作日志
- 防火墙的访问日志
- Web服务器的访问日志

有时候，攻击的痕迹就藏在日志里，你不主动找，它不会自己跳出来。

**第四件事：流量分析**

有条件的话，全流量分析也很重要。

很多攻击，日志里看不到，但流量里能看到。
比如：
- 异常的外联流量
- 异常的大流量（可能是数据外泄）
- 异常的端口连接
- C2通信流量

流量是不会骗人的。

**第五件事：威胁情报**

多关注威胁情报：
- 最近有什么新漏洞？
- 最近有什么新攻击手法？
- 最近有没有针对我们这个行业的攻击？
- 有没有IOC（指标）可以用？

知己知彼，百战不殆。

监测阶段，拼的就是一个"早"字——早发现，早处置，损失就小。晚发现一步，可能就被打穿了。

### 2.3.3 第三阶段：检测阶段——分析研判

告警来了，不能上来就当真。很多告警都是误报。

这个阶段的核心目标是：**搞清楚这到底是不是真攻击？攻击到哪一步了？严不严重？**

怎么分析？

**第一步：告警研判**

先看这个告警是真的还是误报。

怎么判断？
- 看告警的规则是什么？
- 看触发的条件是什么？
- 看原始日志/原始数据包是什么？
- 看这个IP/域名/文件是啥？
- 有没有其他相关的告警？

有些告警一看就是误报，比如：
- 内部正常的业务访问
- 已知的正常行为
- 扫描器自己吓自己

有些告警就很可疑，需要深入分析。

**第二步：关联分析**

一个告警说明不了什么，但好几个告警凑到一起，就说明问题了。

比如：
- 一个IP在扫端口
- 同时这个IP在尝试Web漏洞
- 同时这个IP还在尝试暴力破解

那这肯定不是善茬，多半是红队在打点。

所以要把相关的告警、日志、流量都串起来看，才能看到完整的攻击链路。

**第三步：深入分析**

如果觉得可疑，那就深入查。

怎么查？
- 查这个IP还干了啥？
- 查被攻击的机器有没有被入侵？
- 查有没有后续的行为？
- 查有没有其他机器也被攻击了？

顺藤摸瓜，搞清楚攻击的全貌。

**第四步：定性定级**

最后，给这个事件定性定级：

- 是误报？还是真实攻击？
- 攻击到哪一步了？（只是扫描？还是已经打进来了？还是已经横向移动了？）
- 严重程度怎么样？（低危？中危？高危？紧急？）
- 需不需要启动应急响应？

该上报的上报，该处置的处置。

检测阶段，拼的是"准"——判断要准，别把误报当真的，也别把真攻击当误报。

漏报和误报，都是要命的。

### 2.3.4 第四阶段：响应阶段——兵来将挡

确认是真实攻击了，那就别愣着了，赶紧上！

这个阶段的核心目标是：**尽快把攻击拦住，把损失降到最低。**

怎么响应？经典的PDCERF流程了解一下：

```
Preparation（准备）→ Detection（检测）→ Containment（遏制）
      → Eradication（根除）→ Recovery（恢复）→ Follow-up（跟踪）
```

我给你讲人话版的：

**第一步：遏制——先把攻击拦住**

先别管别的，先把攻击按住，别让它继续扩散。

怎么遏制？
- 封IP（把攻击IP封了）
- 断网（被入侵的机器先断网，别让红队继续操作）
- 隔离（把被入侵的机器从网络里隔离开）
- 停用账号（被偷的账号先禁用）

先止住血再说，别的后面慢慢说。

**第二步：根除——把攻击者赶出去**

止住血了，然后开始清理。

清理啥？
- 删WebShell
- 杀木马进程
- 删后门账号
- 清计划任务
- 清启动项
- 清注册表
- 补漏洞

把攻击者留下的所有东西都清干净，把漏洞补上，别让人家再进来。

**第三步：溯源——搞清楚怎么回事**

清理完了，还没完。得搞清楚：
- 攻击从哪来的？
- 怎么打进来的？
- 进来之后干了啥？
- 偷了什么东西？
- 还有没有其他的入口？
- 还有没有其他机器被黑了？

搞清楚来龙去脉，才能防止下次再被同样的方式打进来。

**第四步：上报——该说的得说**

该上报的上报，该通知的通知。

- 给领导汇报情况
- 给业务部门通报
- 给监管部门报备（如果需要的话）
- 给客户说明情况（如果影响到客户了）

别瞒，瞒不住的。早说比晚说好。

响应阶段，拼的是"快"——处置要快，越快损失越小。

但快不等于乱，忙中容易出错。按流程来，一步一步来。

### 2.3.5 第五阶段：恢复阶段——吃一堑长一智

攻击处置完了，系统也恢复了，这事就完了？

没有。远远没有。

这个阶段的核心目标是：**从攻击中学习，把防线建得更牢，防止下次再被同样的方式打进来。**

具体干啥？

**第一件事：系统恢复**

先把系统恢复正常。
- 业务恢复上线
- 数据恢复
- 网络恢复
- 服务恢复

确认一切正常了，大家才能松口气。

**第二件事：复盘总结**

这个最重要。

开复盘会，大家坐下来好好聊聊：
- 这次攻击是怎么发生的？
- 我们为什么没防住？
- 哪个环节出了问题？
- 哪些地方做得好？
- 哪些地方做得不好？
- 以后怎么改进？

别光骂"蓝队怎么没防住"，没用。

要把问题找出来，改进掉，下次不再犯，这才是复盘的意义。

**第三件事：改进加固**

复盘完了，有一堆改进项，那就赶紧落地。

- 该补的漏洞补上
- 该加的规则加上
- 该优化的流程优化
- 该培训的培训
- 该买的设备申请

别复盘完了就完了，该干啥干啥，一点改进都没有。那复盘还有啥意义？

**第四件事：报告编写**

最后，写一份完整的报告。

报告里写啥？
- 事件概述
- 攻击时间线
- 攻击手法
- 影响范围
- 处置过程
- 原因分析
- 改进建议

报告是给领导看的，也是给以后的自己看的。

写好报告，也是蓝队的基本功。

## 2.4 护网蓝队的作战节奏

讲完了五个阶段，我再给你讲讲护网期间蓝队的作战节奏。

护网一般14天，这14天不是平均用力的，是有节奏的。

### 2.4.1 第一周：试探期——稳住别慌

护网前几天，红队一般在干啥？

在侦察、在打点、在试探。

所以第一周，告警会特别多：
- 各种扫描
- 各种探测
- 各种尝试
- 各种钓鱼

这时候蓝队要干啥？

**稳住，别慌。**

- 正常监测，正常处置
- 该封IP封IP，该拦的拦
- 注意观察，收集情报——红队大概是什么水平？常用什么手法？
- 别自己先乱了阵脚

第一周，一般不会有太猛烈的攻击，都是试探。

但也别大意，有些厉害的红队，第一天就能打进来。

### 2.4.2 第二周：攻坚期——严防死守

到了第二周，红队试探得差不多了，开始攻坚了。

这时候，真家伙就上来了：
- 真刀真枪的攻击
- 0day漏洞
- 各种社工
- 内网横向移动
- 拿域控、偷数据

这一周是最关键的一周，也是最累的一周。

蓝队要干啥？

**绷紧弦，严防死守。**

- 全员上岗，7×24小时连轴转
- 重点盯着关键系统、核心资产
- 告警响应要快，别拖
- 发现一点苗头赶紧查，别等事情闹大了
- 多做主动检测，别光等着告警

这一周扛过去了，基本就赢了。

### 2.4.3 最后两天：疯狂期——小心反扑

护网最后两天，红队也急了，啥招都使出来了。

这时候最容易出事，因为：
- 大家都累了，精神松懈了
- 红队也急了，各种手段都用上了
- 越是最后越容易出问题

蓝队要干啥？

**坚持住，别松懈。**

- 最后两天，谁都不能请假
- 全员保持高度警惕
- 重点区域重点盯防
- 别以为快结束了就没事了

行百里者半九十。最后两天，才是真正考验人的时候。

### 2.4.4 护网结束：恢复期——总结复盘

护网结束了，大家都松了一口气。

但还没完。

- 赶紧写报告
- 赶紧复盘总结
- 赶紧把发现的问题整改掉
- 该加固的加固，该补的补上

别护网一结束就全忘了，该咋样咋样。那护网还有啥意义？

护网的目的不是为了"赢"，是为了"提升安全能力"。

## 2.5 蓝队的思维方式

最后，我想聊聊蓝队的思维方式。

技术可以学，工具可以用，但思维方式，才是决定你能不能成为一个优秀蓝队的关键。

### 2.5.1 攻击者思维

啥叫攻击者思维？

就是站在攻击者的角度想问题：
- 如果我是红队，我会怎么打进来？
- 如果我是黑客，我会先攻哪里？
- 如果我想偷数据，我会怎么偷？

别总站在防守的角度想"我有什么什么设备"，要站在攻击的角度想"我怎么攻破这些设备"。

你能想到的攻击路径越多，你就能防得越全面。

所以，优秀的蓝队，都懂攻击。
不懂攻击的蓝队，不是好蓝队。

### 2.5.2 底线思维

啥叫底线思维？

就是做最坏的打算，往最好的方向努力。

别想着"我们不会被打进来的"，要想着"万一被打进来了怎么办？"

- 万一防火墙被绕过了怎么办？
- 万一Web服务器被拿下了怎么办？
- 万一域控被偷了怎么办？
- 万一数据泄露了怎么办？

把最坏的情况都想到了，预案都做好了，真出事了就不慌。

有预案和没预案，区别大了。

### 2.5.3 闭环思维

啥叫闭环思维？

就是一件事要有始有终，不能干一半就完了。

发现了漏洞 → 要跟进修复 → 修复完了要验证 → 验证完了要记录
发现了攻击 → 要应急处置 → 处置完了要溯源 → 溯源完了要加固 → 加固完了要复盘

形成一个闭环，一件事才算真正完了。

很多公司安全做不好，就是因为没闭环——
漏洞扫出来了，没人跟进修复；
攻击发现了，处置完就完了，也不加固；
复盘完了，改进项没人管。

这样的话，永远在同一个地方摔跤。

### 2.5.4 细节思维

啥叫细节思维？

就是关注细节。

很多攻击，就藏在一个不起眼的细节里：
- 一条奇怪的日志
- 一个异常的进程
- 一个陌生的账号
- 一次不正常的登录

你注意到了，就能发现攻击。
你没注意到，就错过了。

蓝队的工作，很多时候就是在跟细节打交道。

细心，细心，再细心。

重要的事情说三遍。

## 2.6 本章总结

这一章我们聊了蓝队的作战地图，内容挺多的，给你总结一下：

**防御框架：**
- P2DR模型：策略→防护→检测→响应，形成闭环
- 纵深防御：多层防线，别把鸡蛋放在一个篮子里
- ATT&CK防御视角：每个攻击手法都有对应的防御方法

**五个工作阶段：**
1. 准备阶段：资产梳理、漏洞修复、安全加固、产品检查、预案演练、意识培训
2. 监测阶段：大屏监控、告警监控、日志分析、流量分析、威胁情报
3. 检测阶段：告警研判、关联分析、深入分析、定性定级
4. 响应阶段：遏制→根除→溯源→上报，越快越好
5. 恢复阶段：系统恢复、复盘总结、改进加固、报告编写

**护网作战节奏：**
- 第一周：试探期，稳住别慌
- 第二周：攻坚期，严防死守
- 最后两天：疯狂期，小心反扑
- 结束后：恢复期，总结复盘

**蓝队思维方式：**
- 攻击者思维：站在攻击者角度想问题
- 底线思维：做最坏的打算，往最好的方向努力
- 闭环思维：一件事要有始有终，形成闭环
- 细节思维：关注细节，细节决定成败

脑子里有了这张作战地图，以后干活就不慌了。

知道自己在哪个阶段，该干啥事，按部就班来就行。
