## 16.1 先从一个故事讲起

### 16.1.1 一个蓝队老兵的十年

先给你讲个真实的故事。

我有个朋友，叫阿强。
阿强干红队蓝队这行，快十年了。

十年前，阿强刚毕业，进了一家公司的安全部门。
那时候安全部门人少，就两三个人，啥都干。
- 管防火墙
- 搞运维
- 修电脑
- 甚至有时候还帮人装系统

那时候大家对安全也不重视。
领导觉得，安全就是个成本中心，不产生价值。
有一次公司要砍预算，第一个想到的就是砍安全的。

阿强那时候也迷茫：
"我干这个，有前途吗？"

但阿强没放弃。
没人重视，他就自己找事干。
- 研究防火墙配置，把规则优化了一遍又一遍
- 自己写脚本，监控服务器安全
- 晚上下班了，自己看书、学技术
- 公司没人懂，他就去外面参加安全会议，跟同行交流

慢慢的，阿强水平上来了。

有一年，公司被黑客攻击了，网站被挂了黑页。
大家都慌了，不知道怎么办。
阿强站出来，花了一个通宵，把问题查清楚了，把漏洞补了，把系统恢复了。

那一次，领导第一次意识到：
"原来安全还挺重要的。"

后来，公司越来越重视安全，安全部门也从两三个人，发展到了十几个人。
阿强也成了安全部门的负责人。

现在的阿强，是业内有名的蓝队专家。
带团队、建体系、做架构、搞研究。
手下十几号人，管着公司所有的安全业务。

有人问阿强：
"你干了十年蓝队，最大的感悟是什么？"

阿强说：
"第一，这条路不好走，但值得走。
 第二，永远不要停止学习。
 第三，安全的价值，是靠自己挣来的，不是别人给的。
 第四，坚持下去，时间会给你答案。"

---

阿强的故事，也是很多蓝队人的故事。
从迷茫到坚定，从小白到专家，从不受重视到独当一面。

蓝队这条路，没有捷径。
但只要你坚持走下去，一定会有收获。

这最后一章，我就跟你聊聊，蓝队这条路该怎么走。

## 16.2 蓝队的职业发展路径

先说说，蓝队有哪些发展方向？

很多人以为，蓝队就是"看监控的"、"搞运维的"。
其实不是。
蓝队的发展路径很宽，方向很多。

### 16.2.1 技术路线

如果你喜欢技术，想在技术上深耕，那可以走技术路线。

```
初级蓝队工程师
  ↓
中级蓝队工程师
  ↓
高级蓝队工程师
  ↓
资深蓝队工程师 / 安全研究员
  ↓
安全专家 / 首席安全官（技术方向）
```

每个阶段干什么？

**初级（0-2年）：**
- 看监控、处理告警
- 用工具做漏洞扫描、病毒查杀
- 基础的应急响应
- 写简单的脚本
- 熟悉各类安全设备和工具

**中级（2-5年）：**
- 能独立处理常见的安全事件
- 会做渗透测试、漏洞挖掘
- 能做安全架构设计、安全加固
- 能写工具、写自动化脚本
- 带新人、做培训

**高级（5-10年）：**
- 复杂事件的分析和溯源
- 安全体系建设
- 安全研究（漏洞研究、威胁研究...）
- 安全产品/平台的设计和选型
- 团队技术负责人

**资深/专家（10年以上）：**
- 领域内的顶尖水平
- 行业影响力
- 制定安全战略
- 解决疑难杂症
- 带团队、做布道

技术路线，就是一条道走到黑，在技术上越来越深。
适合喜欢技术、享受解决问题的人。

### 16.2.2 管理路线

如果你喜欢带人、喜欢做管理，那可以走管理路线。

```
蓝队工程师
  ↓
蓝队组长 / 安全主管
  ↓
安全经理
  ↓
安全总监
  ↓
首席安全官（CSO） / 信息安全负责人
```

每个阶段干什么？

**组长/主管：**
- 带一个小团队（3-10人）
- 做项目、做计划
- 分配任务、把控进度
- 团队建设、人员培养

**经理：**
- 管一个部门（十几到几十人）
- 做部门规划、预算
- 跨部门沟通协调
- 向上汇报、向下管理

**总监：**
- 管整个安全体系
- 制定安全战略
- 跟高管层对接
- 对公司整体安全负责

**CSO：**
- 公司最高安全负责人
- 参与公司战略决策
- 对董事会/CEO负责
- 管理公司所有安全相关事务

管理路线，就是从自己干，到带着别人干。
适合沟通能力强、有领导力、喜欢做管理的人。

### 16.2.3 专家路线

还有一种，就是专家路线。
不做管理，也不做纯技术，而是做某个领域的专家。

比如：
```
应急响应专家
威胁情报专家
合规审计专家
安全架构师
安全顾问
数据安全专家
云安全专家
工业控制安全专家
......
```

每个领域都可以深耕，成为那个领域的大牛。
专家路线，越老越吃香。
适合喜欢钻研、在某个领域特别感兴趣的人。

### 16.2.4 其他方向

除了上面说的，还有很多其他方向：

```
安全产品经理  —— 做安全产品
安全运营      —— 管安全运营体系
安全培训      —— 做安全意识、安全培训
安全销售      —— 卖安全产品和服务
安全咨询      —— 给客户做安全咨询
......
```

路子宽得很。
别把自己局限在"蓝队就是看监控的"这个框框里。

## 16.3 怎么从入门到精通？

说了方向，再说说方法。
怎么才能从一个蓝队小白，成长为蓝队大牛？

### 16.3.1 打基础

首先，基础要打牢。

蓝队需要哪些基础？

```
网络基础：
  • TCP/IP协议
  • 路由交换
  • 常见网络设备（防火墙、WAF、IDS/IPS...）
  • 网络抓包和分析（Wireshark）

系统基础：
  • Windows系统管理和安全
  • Linux系统管理和安全
  • 常见服务（Web、数据库、中间件...）
  • 系统日志分析

Web安全：
  • 常见Web漏洞（SQL注入、XSS、文件上传、命令执行...）
  • Web应用防火墙（WAF）
  • Web日志分析
  • WebShell检测和处理

工具使用：
  • 漏洞扫描工具（Nessus、AWVS...）
  • 入侵检测/防御系统（IDS/IPS）
  • SIEM平台
  • 杀毒软件、EDR/HIDS
  • 应急响应工具

安全知识：
  • 常见攻击手法
  • 常见木马和病毒
  • 威胁情报
  • 安全标准和框架（等保2.0、ISO27001...）
```

基础不牢，地动山摇。
别上来就想学高大上的东西，基础打牢了，后面才快。

### 16.3.2 多实战

光看书、听课没用，得多实战。

怎么实战？

```
1. 工作中练
   日常工作就是最好的练习。
   每处理一个告警，就认真对待。
   每做一次应急响应，就总结总结。
   每发现一个问题，就深挖一下。

2. 搭建环境自己练
   自己搭个靶场、搭个实验环境。
   自己攻击自己，自己防御自己。
   把各种攻击手法都试一遍，看看日志是什么样的、告警是什么样的。

3. 参加护网/攻防演练
   有机会参加护网、攻防演练，一定要去。
   真刀真枪干一场，比你看十本书都有用。
   虽然累，但成长最快。

4. 打CTF
   有兴趣可以打打CTF，尤其是防守型的、取证型的。
   锻炼分析能力、动手能力。

5. 做实验、写工具
   自己动手写点小工具、小脚本。
   自己设计点实验，验证自己的想法。
```

纸上得来终觉浅，绝知此事要躬行。
实战是成长最快的方式。

### 16.3.3 多总结

光干活不总结，成长也慢。

怎么总结？

```
1. 写笔记
   学了什么、做了什么、遇到了什么问题、怎么解决的，都记下来。
   好记性不如烂笔头。

2. 写博客
   把自己的经验、研究，写成文章。
   写的过程，就是梳理的过程。
   还能跟同行交流，收获更多。

3. 做复盘
   每次事件处理完，都复盘一下。
   哪里做得好？哪里做得不好？
   下次怎么改进？

4. 建知识库
   把常用的命令、工具、方法、案例，整理成知识库。
   既方便自己查，也方便分享给团队。
```

总结是把经验变成能力的过程。
干了十年，不等于有十年经验。
可能只是一年的经验，重复了十年。

差别就在总结。

### 16.3.4 多交流

别一个人闷头干，多跟同行交流。

怎么交流？

```
1. 加社群
   安全圈有很多社群、微信群、QQ群。
   加几个，平时聊聊天，看看大家都在聊什么。

2. 参加会议
   有机会多参加安全会议、沙龙。
   听听大牛的分享，认识认识同行。

3. 关注大佬
   Twitter、微博、微信公众号，关注一些安全大佬。
   看看他们都在研究什么、分享什么。

4. 加入团队
   有条件的话，加入一个好的团队。
   跟厉害的人一起干活，成长最快。
```

一个人走得快，一群人走得远。
交流能让你少走很多弯路。

### 16.3.5 保持好奇和热情

最后，也是最重要的——保持好奇和热情。

安全这行，技术更新太快了。
- 新的漏洞不断出来
- 新的攻击手法不断出现
- 新的安全技术不断发展

如果你没有好奇心，没有热情，很快就会跟不上。

怎么保持？
- 对新技术保持好奇，多去了解了解
- 找到自己真正感兴趣的方向
- 给自己设定目标，一个个去实现
- 享受解决问题的成就感
- 偶尔跳出舒适区，挑战一下自己

兴趣是最好的老师。
真的喜欢这行，你就不会觉得累。

## 16.4 常见的瓶颈和突破方法

成长的路上，总会遇到瓶颈。
很正常，别慌。

说说几个常见的瓶颈，以及怎么突破。

### 16.4.1 瓶颈一：感觉天天在打杂，学不到东西

很多刚入行的朋友会有这个感觉：
"天天就是看监控、处理告警、杀病毒，感觉学不到东西，像个打杂的。"

怎么破？

```
1. 把简单的事做到极致
   看监控，能不能把每个告警都搞清楚？为什么会告警？原理是什么？
   杀病毒，能不能把每个病毒都分析清楚？是什么类型？怎么进来的？怎么彻底清除？
   简单的事，做到极致，就不简单了。

2. 主动找事做
   没人让你做什么，你就自己找事做。
   - SIEM规则能不能优化一下？
   - 安全策略能不能改进一下？
   - 能不能写个自动化脚本，提高效率？
   - 能不能做个安全加固，减少告警？
   主动找事做，成长就快了。

3. 下班时间自己学
   工作时间没机会学，那就下班时间学。
   自己搭环境、做实验、看资料。
   差距是在八小时之外拉开的。
```

别觉得打杂就没价值。
同样是打杂，有的人打了一年还是打杂，有的人打了一年就成长起来了。
差别在人。

### 16.4.2 瓶颈二：技术太多，学不过来

安全这行，技术太多了。
学了这个，还有那个。
感觉永远学不完，很焦虑。

怎么破？

```
1. 先广再深
   先什么都了解一点，有个大概的印象。
   然后找到自己感兴趣的方向，深入下去。
   什么都懂一点，但要有一两个自己擅长的。

2. 按需学习
   不用什么都学，用到什么学什么。
   工作中遇到什么问题，就去学什么。
   这样学了就能用，印象深刻，也有成就感。

3. 关注核心，忽略噪音
   很多新技术、新概念，其实都是包装，核心的东西就那么多。
   把基础打牢了，很多新技术一看就懂。
   别被各种概念带偏了。

4. 接受不完美
   没有人什么都懂。
   大牛也不是什么都会。
   接受自己有不懂的地方，很正常。
   知道自己不知道，并且知道怎么去查，就够了。
```

学无止境，但也不用焦虑。
一步一步来，每天进步一点点，就够了。

### 16.4.3 瓶颈三：干了好几年，感觉没成长

有些人，干了三五年，感觉自己还是那样。
技术也没怎么涨，职位也没怎么升。
迷茫，不知道该往哪走。

怎么破？

```
1. 找找原因
   为什么没成长？
   - 是公司环境不行，学不到东西？
   - 是自己没努力，天天混日子？
   - 是方向不对，不适合自己？
   先找到原因，再想办法。

2. 换个环境
   如果是公司的问题，那可以考虑换个环境。
   去一个更有挑战、更能学到东西的地方。
   有时候，换个环境，成长会快很多。

3. 逼自己一把
   如果是自己的问题，那就逼自己一把。
   给自己定个目标，比如半年内学会什么、拿到什么证书、做出什么成果。
   人不逼自己一把，不知道自己的潜力有多大。

4. 换个方向
   如果觉得这条路不适合自己，那也可以考虑换个方向。
   蓝队不适合，可以试试红队、试试产品、试试咨询、试试管理。
   条条大路通罗马。
```

遇到瓶颈不可怕，可怕的是在瓶颈里待着不行动。
想清楚，然后行动。

### 16.4.4 瓶颈四：干安全没前途，想转行

有时候会听到有人说：
"干安全又累又不挣钱，还背锅，没前途，不想干了。"

怎么看？

```
我的看法是：
  1. 安全这行，确实不轻松。
     要学的东西多，责任大，还经常背锅。
     但哪行轻松呢？
     干开发也累，干产品也累，干运营也累。
     没有哪一行是轻轻松松就能挣大钱的。

  2. 安全这行，还是有前途的。
     现在越来越重视安全，需求越来越大。
     真正厉害的人，从来不愁找不到好工作，也不愁挣不到钱。
     关键是你是不是真的厉害。

  3. 要不要转行，看你自己。
     如果你真的不喜欢这行，干着很痛苦，那早点转行也挺好。
     去做自己喜欢的事，比什么都强。
     但如果你只是因为遇到点困难就想逃，那我劝你再坚持坚持。
     哪行都有困难，逃是逃不掉的。
```

路是自己选的。
选了，就好好走。
不想走了，就换一条。
没什么对不对，自己不后悔就行。

## 16.5 未来的方向

最后，聊聊未来。
蓝队未来的发展方向是什么？

### 16.5.1 自动化和智能化

这个是大趋势。

- 简单的告警处理，自动化
- 常见的应急响应，自动化
- 威胁检测，用AI、机器学习
- 态势感知，智能分析

以后，很多重复性的工作，都会被自动化替代。

那蓝队是不是就失业了？
当然不是。
- 低级的、重复性的工作会减少
- 但高级的、需要分析和判断的工作会越来越多
- 你要做的，是驾驭这些工具和系统，而不是被它们替代

**怎么应对？**
- 学会用自动化工具
- 学会写脚本、写工具
- 提高自己的分析能力和判断能力
- 往高处走，做机器做不了的事

### 16.5.2 云安全

现在越来越多的企业上云。
云安全成了刚需。

云安全跟传统安全不一样：
- 云的架构不一样
- 责任模型不一样
- 工具和方法也不一样

未来，懂云安全的蓝队会越来越吃香。

**怎么应对？**
- 学习主流云平台（阿里云、腾讯云、AWS、Azure...）
- 学习云原生安全（容器、K8s、微服务...）
- 学习云安全工具和产品
- 有机会多做云安全项目

### 16.5.3 数据安全

数据越来越重要，数据安全也越来越受重视。
- 《数据安全法》、《个人信息保护法》都出来了
- 企业对数据安全的投入越来越大
- 数据安全人才缺口很大

数据安全也是蓝队的一个重要方向。

**怎么应对？**
- 学习数据安全相关的法律法规
- 学习数据分类分级、数据防泄露（DLP）
- 学习数据脱敏、数据加密
- 学习数据库安全、大数据安全

### 16.5.4 攻防对抗升级

攻和防的对抗，会越来越激烈。
- 攻击手法越来越高明
- 攻击工具越来越先进
- APT攻击越来越多
- 0day漏洞越来越值钱

对应的，防守的要求也越来越高。
- 不能只靠边界防护
- 要纵深防御、要主动防御
- 要威胁狩猎、要溯源反制

未来的蓝队，会越来越专业，越来越强。

**怎么应对？**
- 持续学习，跟上技术发展
- 多实战，积累经验
- 培养攻防思维，站在攻击者角度思考
- 往高阶走，做威胁狩猎、溯源分析、反制

### 16.5.5 安全运营体系化

以后的安全，不是靠几个设备、几个人就行了。
要靠体系。

- 管理制度
- 技术体系
- 运营流程
- 人员能力
- 企业文化

整套体系建起来，才能真正做好安全。

这也是蓝队的方向——
从"救火队"变成"体系建设者"。

**怎么应对？**
- 不仅懂技术，还要懂管理、懂流程、懂业务
- 学习安全框架和标准（等保2.0、ISO27001、NIST...）
- 培养体系化思维
- 多站在公司角度、业务角度想问题

## 16.6 写在最后

好了，16章的内容，到这里就全部结束了。

最后，我想再跟你说几句话。

### 第一句：安全是一场马拉松，不是百米冲刺。

别想着一口吃成个胖子。
慢慢来，持续学习，持续进步。
每天进步一点点，一年下来，你会吓一跳。

### 第二句：保持热爱，保持好奇。

干安全这行，兴趣很重要。
真的喜欢，你才能走得远。
保持好奇心，对新技术、新事物保持敏感。

### 第三句：别怕犯错，别怕出事。

谁都是从犯错过来的。
犯错不可怕，可怕的是犯了错还不长记性。
每一次出事，都是成长的机会。
就看你能不能抓住。

### 第四句：安全的价值，是靠自己挣来的。

别抱怨领导不重视、别抱怨公司不投入。
你做出成绩了，让大家看到安全的价值了，
自然就重视了，自然就有投入了。
价值是干出来的，不是喊出来的。

### 第五句：守住底线，不忘初心。

安全这行，接触的东西比较敏感。
- 别去做违法的事
- 别去碰不该碰的东西
- 守住底线，对得起自己的良心

技术是把双刃剑，看你怎么用。
用好了，保护别人；用坏了，害人害己。
