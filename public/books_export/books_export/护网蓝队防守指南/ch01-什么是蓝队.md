## 📖 写在前面

很多人第一次听到"蓝队"这俩字，脑子里都会冒出几个问题：

"蓝队？是不是就是修电脑的？"
"蓝队和运维有啥不一样？"
"护网蓝队又是个啥？是不是就是天天盯着大屏看？"

别着急，这一章我们就慢慢聊，用大白话把这些问题都讲明白。

看完你就知道蓝队到底是干啥的，和红队、运维、安全工程师有啥区别，以及——为什么说蓝队是网络安全里最考验综合能力的方向之一。

坐稳了，我们开唠。

## 1.1 蓝队到底是个啥东西？

### 1.1.1 先从一个真实的故事讲起

先给你讲个真实发生的故事。

有这么一家大公司，老板姓刘，我们就叫他刘总吧。

刘总很重视安全，公司防火墙买的是最好的，什么下一代防火墙、入侵检测、态势感知、EDR一个不少，安全团队有二十多个人，7×24小时值班，监控大屏24小时亮着。

刘总心里美滋滋的：钱没白花，我们公司固若金汤啊！

直到有一天——

护网行动开始了。

第一天，平安无事。安全团队的人该上班上班，该喝茶喝茶。大屏上偶尔蹦出几个告警，都是老熟人了——什么"扫描探测"、"暴力破解"，都是常有的事，大家都见怪不怪了，随手就给屏蔽了。

第二天，还是没啥事。大家觉得：就这？护网也不过如此嘛。

第三天，出事了。

凌晨三点，值班的小李突然发现，有一台Web服务器的日志有点不对劲——有个IP在尝试各种奇怪的路径，看起来像是在扫备份文件。

小李眉头一皱，觉得事情不简单。赶紧查了一下，发现这个IP在过去三天一直在搞事。

"不对啊，这个IP昨天就来过，当时以为是普通扫描，就给拉黑了。怎么又来了？而且换了个IP又来了？

小李越想越不对劲，赶紧往上报告给队长。

队长过来一看，脸色变了："坏了，这不是普通扫描，这是红队在打点！

赶紧查！

于是全公司紧急动员，查日志、查流量、查进程......查了半天，发现——

人家早就打进来了。

而且不是刚打进来不久，刚拿到一台Web服务器的权限，正在内网摸鱼呢。

还好发现得早，赶紧把入口封了，把机器断网了，把漏洞补上了，总算没让红队继续往里打。

刘总吓出一身冷汗："还好还好，发现得早，不然就完了。"

后来护网结束后，刘总问安全队长："这次多亏你们发现得早，你们是怎么发现的？"

队长说："也没啥，就是值班的小伙子细心，多看了一眼日志。不然等他们再晚发现，等他们把内网打穿了就完了。"

刘总听完，沉默了半天，说："以前我觉得安全就是买设备、堆产品。现在才知道，设备是个人啊设备再好，没人盯着、没人会用、没人懂，等于白搭。"

---

这群"盯着大屏、查日志、发现攻击、拦住攻击"的人，就是我们要说的——**蓝队**。

### 1.1.2 给蓝队下个定义

说了半天故事，那到底啥是蓝队？

用一句话说：

> **蓝队，就是站在防守者的角度，负责发现攻击检测、应急响应、溯源分析，保护一个组织的网络和系统不被黑客打进来的团队。**

我再翻译翻译，说得更直白点：

蓝队就是"网络保安" + "网络警察" + "网络医生"的结合体。

- 像保安一样，7×24小时盯着大门，发现可疑人员（可疑流量）来了就拦住；
- 像警察一样，真出事了（被攻击了），赶紧出警（应急响应），查案子（溯源分析），把坏人赶出去；
- 像医生一样，平时给系统做体检（安全检查），生病了给治病（漏洞修复），告诉客户怎么预防生病（安全加固）。

红队是"职业假黑客"，蓝队就是"职业假保安"。

红队负责"打"，蓝队负责"防"。

再打个比方，就像打仗：

红队是攻城的，想尽办法要打进城；蓝队是守城的，想尽办法不让人打进来。

红队打得进算红队赢，蓝队守得住算蓝队赢。

### 1.1.3 再用一个生活中的例子帮你理解

我再给你举个生活中的例子，帮你更直观地理解蓝队是干啥的。

比如你家小区的保安：

- 保安的工作是什么？
  - 盯着大门，陌生人来了问一声"你找谁？
  - 可疑人员（鬼鬼祟祟的，盘问一下
  - 真有小偷偷东西，赶紧上去抓
  - 抓完小偷，还要调监控、查一下小偷是怎么进来的
  - 平时还要给物业提建议：这里要加装个摄像头，那里要加高围墙

蓝队干的也是这个事儿：
- 盯着网络边界（防火墙、IDS、IPS）
- 发现可疑流量告警，拉出来分析一下
- 真有攻击打进来了，赶紧拦、赶紧断、赶紧封
- 攻击完了，溯源分析，搞清楚攻击者是谁、怎么进来的、干了啥
- 平时给公司提安全建议：这里要加固、那里要加设备、这里要改配置

这个小区保安，就相当于"蓝队"。

保安不是"蓝队就是网络世界的保安。

保安看起来简单，真的不简单。

### 1.1.4 红蓝紫三方是怎么回事

经常听人说"红蓝对抗"、"红蓝紫三方"，那红蓝紫到底是什么关系？

我给你打个比方，你就懂了。就像拍电影打仗：

```
红队 = 进攻的军队（攻击方）
  他们的任务是：想尽一切办法打进去
  目标可能是：拿下目标系统、拿到核心数据、获取域控权限
  越隐蔽越好，越快越好，不被发现最好

蓝队 = 防守的军队（防御方）
  他们的任务是：发现攻击、拦住攻击
  最好还能溯源——找到攻击者是谁、从哪来的、用了什么手法
  如果发现不了，那就输了

紫队 = 导演部/裁判（组织方）
  他们既不攻也不守
  负责制定比赛规则、判定谁得分谁失分
  最后组织大家复盘总结：哪里做得好、哪里需要改进
```

再换个比喻，像踢足球：
- 红队 = 前锋，负责进球（打进去、拿分）
- 蓝队 = 后卫+守门员，负责守门（拦住、发现）
- 紫队 = 裁判，负责吹哨、计分（组织、评判）

这三个角色缺了哪个都不行：
- 没有红队，蓝队不知道自己防不防得住
- 没有蓝队，红队就是自己跟自己玩
- 没有紫队，乱成一锅粥，谁赢谁输都说不清

## 1.2 蓝队、运维、安全工程师，到底啥区别？

很多人搞不清这三者的关系，觉得"不都是搞安全的吗？"

还真不一样。我给你掰开揉碎了讲。

### 1.2.1 先说说运维工程师

这个最好理解，就是我们常说的"运维"。

运维的主要工作是什么？

- 保证系统正常运行
- 服务器别挂了赶紧修好
- 业务上线、部署、扩容、监控
- 数据库备份、系统升级、环境配置

他们的特点是：

**第一，核心目标是"可用性"。** 系统要稳定运行、业务能正常访问就行。**第二，主要关心的是"系统能不能跑"。** 跑不跑得动、卡不卡、稳不稳。

**第三，安全是副业。** 运维也管安全，但主要是"顺便管"，比如装个防火墙、打个补丁，但这都是基础安全加固，不是主业。

打个比方：运维就像是物业公司，小区物业管物业是小区的"管家，负责小区的水电煤全，水电、电梯、路灯、绿化......全得管。安全也管，但不是主业。

### 1.2.2 再说说安全工程师

那安全工程师又是啥？

安全工程师，简单说就是：

> 专门搞安全的，负责安全建设、安全加固、漏洞修复、安全产品部署、安全策略制定。

安全工程师的特点：

**第一，核心目标是"安全性"。** 系统要安全、数据要安全、网络要安全。**第二，主要做的是"建设"和"加固"。** 买安全设备、配安全策略、做安全评估、补安全加固。

**第三，偏向"事前"。** 主要是防患于未然，在攻击没来之前，先把 defenses 建起来、加固好。

打个比方：安全工程师就像是小区的"安全顾问。帮小区设计安防系统：哪装摄像头、哪装防盗门、围墙要多高、门禁怎么弄......设计完了，装好了，就完事了。至于平时谁在干嘛？平时主要是设计和建设。

### 1.2.3 最后说说蓝队

那蓝队又有啥不一样呢？

如果说安全工程师是"建防线的"，那蓝队就是"守防线的"。

安全工程师把防火墙买来了、配好了、建好了防线，然后呢？

然后蓝队来守。

蓝队是7×24小时盯着的，攻击来了要能发现、发现了要能响应、响应了要能处置、处置完了要能溯源、溯源完了要能加固。

再打个比方：

**安全工程师** = 给小区装摄像头、装防盗门、修围墙的人

**蓝队** = 坐在监控室里，7×24小时盯着监控大屏的保安

一个建，一个守。

蓝队的特点：

**第一，更接近实战。** 安全工程师可能只是建设，但真打起来了，是蓝队在一线顶着。

**第二，7×24小时值班。** 攻击不会因为攻击不会下班，蓝队也不能下班。黑客凌晨3点攻击你也得爬起来干活。

**第三，讲究"事中"和"事后"。** 安全工程师主要是"事前"建设，蓝队是"事中"检测和"事中和事后。

**第四，综合性强。** 蓝队什么都得懂点：网络要懂、系统要懂、Web要懂、攻击要懂、工具要懂、分析要懂、应急要懂...比安全工程师懂的还杂多了。

**第五，压力大。** 真出事了，第一个找的就是蓝队。没发现攻击？蓝队背锅。发现了拦不住？蓝队背锅。拦住了没溯源清楚？蓝队背锅。

### 1.2.4 一张图帮你记住区别

我给你画个简单的对比表：

```
┌─────────────┬────────────────┬────────────────┬────────────────┐
│             │  运维工程师     │  安全工程师   │   蓝队          │
├─────────────┼────────────────┼────────────────┼────────────────┤
│ 核心目标    │ 系统可用稳定   │ 安全建设加固    │ 检测响应溯源     │
├─────────────┼────────────────┼────────────────┼────────────────┤
│ 工作时间    │ 朝九晚五     │ 朝九晚五      │ 7×24小时值班  │
├─────────────┼────────────────┼────────────────┼────────────────┤
│ 侧重点      │ 事前+事后（建设） │ 事前（建设）  │ 事中+事后（防守）│
├─────────────┼────────────────┼────────────────┼────────────────┤
│ 技能要求    │ 操作系统      │ 安全产品+漏洞     │ 啥都得懂点    │
├─────────────┼────────────────┼────────────────┼────────────────┤
│ 压力来源    │ 系统挂了  │ 漏洞没补上     │ 被打进来了  │
├─────────────┼────────────────┼────────────────┼────────────────┤
│ 日常工作    │ 部署运维    │ 安全评估加固   │ 监控分析响应   │
└─────────────┴────────────────┴────────────────┴────────────────┘
```

简单记：
- 运维 = 管系统能不能跑
- 安全工程师 = 建安全防线
- 蓝队 = 守安全防线，真打起来了顶上

## 1.3 护网蓝队又是啥？

前面讲了蓝队是什么，那经常听到的"护网"、"护网蓝队"又是个啥？

### 1.3.1 护网到底是啥

"护网"是简称，全称一般是**网络安全实战攻防演习。

有的地方叫"护网行动"，有的叫"网安演练"，有的叫"攻防对抗赛"，名字不一样，本质上都是一回事。

简单说就是：

> 官方组织一大批单位，一些当攻方（红队），一些当守方（蓝队），在规定的时间里，红队想尽办法打进去，蓝队想尽办法防住，最后看谁得分高、谁排名靠前。

就像是网络安全版的"军事演习"，只不过打的是网络战，不是真枪真炮。

### 1.3.2 护网蓝队的日常

很多人以为护网蓝队就是"盯着大屏看"，那就大错特错了。

护网期间蓝队的日常是这样的：

**护网前一个月：**
- 资产梳理：把家里有啥东西全列出来
- 漏洞扫描：扫漏洞，该补的补
- 安全加固：该加固的加固
- 安全检查：各种安全产品检查一遍，确保都能用
- 应急预案：写预案，真出事了怎么办
- 压力测试：自己先自测一下，看看扛不扛得住

**护网期间：**
- 7×24小时值班，人歇屏不歇
- 监控大屏，盯着告警
- 告警分析，真的假的？误报还是真攻击？
- 发现攻击，赶紧拦、赶紧封IP、断网、杀进程
- 应急响应，被打进来了赶紧处理
- 溯源分析，找到攻击从哪来的，怎么进来的
- 每日复盘，今天发现了啥攻击，有啥新情况

**护网后：**
- 写报告，总结这次护网的情况
- 复盘，哪里做得好，哪里做得不好
- 整改，该补的漏洞补上，该加的设备加上
- 能力提升，下次护网要改进的地方

你看，蓝队不是光护网那几天忙，护网前一个月就开始忙了，护网后还得忙好一阵。

### 1.3.3 护网蓝队的真实感受

说了这么多理论，我给你讲个真实的护网蓝队的故事，让你感受一下护网蓝队是啥样的。

小明是个蓝队工程师，在一家国企的安全团队工作。今年护网，他们单位是防守目标。

护网前一个月，小明就开始忙了：

资产梳理：他们单位有多少服务器？多少网络设备？多少业务系统？哪些是重要的？哪些是一般的？全得理清楚。

漏洞扫描：扫出来几百个漏洞，高危的赶紧补，中危的挑着补，低危的记着以后补。

安全加固：服务器加固、网络加固、系统加固、应用加固......能加固的全加固一遍。

应急预案：写了好几十页的应急预案，什么情况该怎么办，谁负责什么，全写清楚。

压力测试：自己找了个安全公司，先做了一次演练，看看自己防不防得住。

忙了一个月，总算准备得差不多了。

护网终于开始了。

**护网第1天**

早上8点，护网正式开始。

小明和队友们坐在监控室里，盯着大屏，手心都出汗了。

第一天，告警特别多。各种扫描、各种探测、各种尝试。

"没事，都是常规操作，该封IP封IP，该拉黑拉黑。

第一天，平安无事。

**护网第3天**

凌晨2点，小明值班。

突然，SIEM（安全信息和事件管理）响了一个高危告警："检测到异常的WebShell上传行为"。

小明一下子精神了。

赶紧查！

查日志、查流量、查进程...查了半个小时，终于搞清楚了：
- 攻击来源：一个境外IP
- 攻击方式：文件上传漏洞
- 已经上传了一个WebShell
- 还没来得及进一步动作

还好发现得早！

赶紧处理：
- 封IP
- 删WebShell
- 补漏洞
- 查这台机器有没有被进一步利用
- 查其他机器有没有类似情况

忙活到天亮，总算搞定了。

第一次交锋，蓝队胜。

**护网第7天**

中午，大家正在吃午饭。

突然，值班的小李喊了一声："不好！内网有异常！"

大家赶紧放下筷子跑过去。

有一台内网服务器，有异常的远程登录记录，从一台Web服务器登录过来的。

坏了，Web服务器被打进来了，而且已经向内网渗透了！

赶紧查！

查了一下，发现红队已经拿到了一台Web服务器的权限，然后用这台机器当跳板，正在扫内网。

还好发现得及时，红队刚进来，还没怎么动呢。

赶紧处置：
- 把Web服务器断网
- 把被登录的那台机器也断网
- 查日志、查进程、查木马
- 清理干净
- 补漏洞
- 全内网扫一遍有没有其他被入侵的

又忙了一天一夜，总算搞定了。

第二次交锋，还是蓝队胜。虽然有点惊险。

**护网第12天**

最后冲刺阶段了，红队也急了，各种手段都用上了。

钓鱼邮件、暴力破解、0day漏洞......什么都来。

蓝队也绷紧了弦，全员上岗，7×24小时连轴转。

小明已经连续3天每天只睡3个小时了。

眼睛都红了，但不敢松懈。

最后两天，是最关键的时候。

**护网第14天**

护网结束。

最终结果：他们单位成功防守住了大部分攻击，虽然有两次被红队打进来了，但都及时发现并处置了，没有造成大的损失。

蓝队排名，中等偏上，还不错。

小明终于可以好好睡一觉了。

这就是护网蓝队的样子：有紧张、有兴奋、有挫败感、有成就感。十几天里，你可能天天熬夜，也可能几天没动静但神经紧绷，最后成功防守住的那一刻，什么都值了。

### 1.3.4 护网蓝队和普通蓝队有啥不一样

都是蓝队，护网蓝队和平时做蓝队还不太一样：

```
普通蓝队日常：
  • 日常安全监控
  • 日常应急响应
  • 安全加固
  • 漏洞管理
  • 节奏相对平缓

护网蓝队：
  • 超高强度，7×24小时连轴转
  • 攻击密度大，每天都有新情况
  • 精神高度紧张，一刻也不能松懈
  • 时间短，就那么十几天
  • 压力山大，被打进来就要背锅
```

打个比方：
- **普通蓝队日常**像是**平时小区保安**，每天正常巡逻，偶尔有点小事处理一下
- **护网蓝队**像是**演习期间的保安**，全员上岗、高度戒备、随时可能有"敌人"打过来

护网更考验蓝队的"应急能力"和"抗压能力"——十几天高强度工作，你得扛得住，还得在关键时刻顶得上。

就像考试一样，平时学得再好，考试的时候发挥不出来，也白搭。

## 1.4 蓝队的价值在哪里？

很多人觉得蓝队就是"花钱的部门"，不像红队还能"打比赛拿奖"，蓝队好像没什么存在感。

那蓝队的价值到底在哪里？

### 1.4.1 蓝队是最后一道防线

我给你算一笔账：

一家公司，买防火墙花了100万，买EDR花了50万，买SIEM花了80万，态势感知100万......加起来好几百万。

这些设备买回来了，然后呢？

然后没人会用、没人盯着、没人分析、没人响应......

那等于白买。

安全产品是死的，人是活的。

再好的安全设备，也得有人会用才行。

防火墙买了，规则谁来配？配得不对，等于没买。
IDS装了，告警谁来看？看了不分析，等于白看。
EDR部署了，发现木马谁来处置？处置不及时，等于白部署。

蓝队，就是让这些安全产品"活起来"的人。

没有蓝队，再贵的安全产品也只是摆设。

### 1.4.2 蓝队是"止损"的

红队的价值是"发现问题"，蓝队的价值是"解决问题"。

红队告诉你"你这里有漏洞"，然后呢？

然后蓝队来补：漏洞补上、攻击拦住、事件处置掉、防止下次再被打进来。

红队是"攻"，蓝队是"防"。

红队打进来一次，你损失可能是几百万、几千万，甚至更多。

数据泄露了，罚款、赔偿、声誉损失......

蓝队拦住一次攻击，就可能帮你省了几百万。

这就是蓝队的价值。

### 1.4.3 蓝队能力是真的"安全能力"

很多公司安全建设误区：以为买了一堆安全产品，就安全了。

大错特错。

安全产品是买不来的。

安全是一种能力，不是一堆产品。

真正的安全能力是什么？
- 发现攻击的能力
- 分析攻击的能力
- 响应攻击的能力
- 溯源攻击的能力
- 从攻击中学习改进的能力

这些能力，都是蓝队的能力。

没有蓝队，就没有真正的安全能力。

## 1.5 蓝队需要什么技能？

很多新手问我："我想做蓝队，需要学什么？"

我给你详细列一下，你可以对照着看自己还差哪些。

### 1.5.1 技术技能

首先是**网络知识**，这个是基础中的基础。

TCP/IP协议、HTTP协议、路由交换、防火墙、入侵检测、VPN......这些得懂。毕竟蓝队天天跟网络打交道，网络都不懂，怎么分析流量、怎么查攻击？

然后是**操作系统**。

Windows和Linux都得熟悉，命令行、PowerShell、bash、各种系统命令、系统机制、启动项、服务、计划任务、注册表......

为啥？因为攻击最终都是落在系统上的，你不懂系统，怎么查入侵、怎么查木马、怎么查后门？

再有就是**Web安全**。

SQL注入、XSS、文件上传、命令注入、WebShell......这些得懂。你得知道红队是怎么打进来的，才能知道怎么防、怎么查。

然后是**攻击手法**。

你得懂攻击是怎么打的，才能知道怎么防。提权、横向移动、域渗透、免杀......红队的手法，你都得了解。不懂攻击，怎么防攻击？

还有**日志分析**。

Windows日志、Linux日志、Web日志、防火墙日志、IDS日志......各种日志你都得会看。攻击来了，首先是查日志。

还有**流量分析**。

全流量分析、PCAP分析、Wireshark、Tcpdump......流量分析是蓝队的基本功。

还有**应急响应**。

入侵了怎么处理？怎么遏制？怎么根除？怎么恢复？怎么取证？这些都得会。

还有**各种安全产品**。

防火墙、IDS/IPS、WAF、EDR、SIEM、态势感知、堡垒机、数据库审计......常见的安全产品你得会用、懂原理。

最后是**编程/脚本能力**。

不用你写多厉害的程序，但至少Python、Shell脚本得会写。能写个小工具批量处理、能写个脚本自动化分析，这都是一个合格蓝队的基本要求。

### 1.5.2 非技术技能

别小看非技术技能，很多时候这些比技术还重要。

**第一个是细心。**

蓝队最需要的品质是什么？就是细心。

一个攻击可能就藏在一条不起眼的告警里，你没注意到，就放过了，就出事了。

很多时候，攻击不是没告警，是告警了你没发现、没当回事、没分析出来。

细心，是蓝队的第一品质。

**第二个是耐心。**

查日志、查流量、查进程......一查就是几个小时、甚至几天。没有耐心是干不了蓝队的。

一条一条日志看，一个一个包分析。枯燥吗？枯燥。但这就是蓝队的日常。

**第三个是逻辑思维能力。**

蓝队分析攻击，就像警察破案。

从一条线索，顺藤摸瓜，一步步还原整个攻击过程。

你得有很强的逻辑推理能力，能把零散的线索串起来，还原出整个攻击链路。

像侦探一样。

**第四个是抗压能力。**

护网期间，7×24小时连轴转，每天睡不了几个小时，精神高度紧张。

真出事了，领导盯着你、老板盯着你、大家都盯着你，等着你出结果。

压力大不大？大。

没有抗压能力，干不了蓝队。

**第五个是沟通能力。**

出事了，你得跟各个部门沟通：跟运维沟通、跟开发沟通、跟业务部门沟通、跟领导汇报...

你得把技术问题讲清楚，让非技术的人也能听懂。还得协调各方一起处置。

沟通能力很重要。

**第六个是学习能力。**

安全这行，技术更新太快了。今天出个新漏洞，明天出个新攻击手法，后天出个新工具......

你得不断学习，跟上节奏。不然哪天红队用个新手法打进来，你看不懂、不会防，那就麻烦了。

活到老，学到老。这句话放在安全行业，再合适不过。

最后，也是最重要的——**责任心**。

蓝队的工作，很多时候是"良心活"。

你认真查了，没查到问题，那是真没攻击。
你没认真查，没查到问题，那可能是攻击你没发现。

没人知道你有没有认真查。

全靠责任心。

### 1.5.3 不同阶段需要掌握的内容

我给你列个学习路线图，不同阶段该学什么：

**入门阶段（0-1年）：**
- 网络基础知识（TCP/IP、HTTP、路由交换）
- Windows/Linux系统基础
- Web安全基础
- 常见安全产品原理（防火墙、IDS、WAF）
- 日志分析基础
- 应急响应基础流程

**进阶阶段（1-3年）：**
- 深入理解各种攻击手法
- 流量分析、日志分析
- 应急响应实战能力
- 溯源分析能力
- 各种安全产品熟练使用
- Python脚本编写

**高级阶段（3年+）：**
- 威胁狩猎（Threat Hunting）
- 红蓝对抗
- 安全体系建设
- 安全运营
- 团队管理
- 复杂攻击的溯源分析

## 1.6 蓝队的常见误区

最后，我再说说很多人对蓝队的几个常见误解。

### 1.6.1 误区一：蓝队就是"看大屏的"

很多人觉得蓝队就是坐在监控室里，盯着大屏看，很轻松。

大错特错。

看大屏只是蓝队工作的很小一部分。

蓝队还要：
- 告警分析（这个告警是真的还是误报？
- 攻击研判（真攻击的话，攻击到哪一步了？）
- 应急响应（被打进来了赶紧处理）
- 溯源分析（攻击从哪来的？怎么进来的？干了啥？）
- 安全加固（补漏洞、加固系统）
- 漏洞管理（扫漏洞、跟进修复）
- 安全产品运维（防火墙、IDS、EDR...）
- 写报告、写预案、做培训......

看大屏？那是最基础的。

### 1.6.2 误区二：蓝队技术含量低，不如红队

很多人觉得红队技术含量高，蓝队技术含量低。

也不对。

红队讲究的是"深度"，在某一个点上挖得很深。

蓝队讲究的是"广度"，什么都得懂点。

网络、系统、Web、攻击、防御、产品、管理......啥都得会点。

红队可能一门精，蓝队门门通。

而且，蓝队要懂红队的所有手法，不然怎么防？

蓝队的技术含量，一点也不比红队低。

只是方向不同而已。

### 1.6.3 误区三：蓝队就是"背锅的"

很多人觉得蓝队就是背锅侠：
- 被打进来了，蓝队的锅
- 没发现攻击，蓝队的锅
- 发现了拦不住，还是蓝队的锅

听起来好像是这么回事。

但你换个角度想：

- 系统有漏洞，是开发的锅？还是运维的锅？还是蓝队的锅？
- 员工点了钓鱼邮件，是员工的锅？还是HR的锅？还是蓝队的锅？
- 老板不重视安全，不给预算，是老板的锅？还是蓝队的锅？

安全是整个公司的事，不是蓝队一个团队的事。

蓝队不是背锅的，蓝队是最后一道防线。

前面的防线都破了，才轮到蓝队。

所以，一个公司安全做得好不好，不是看蓝队厉不厉害，是看整个公司的安全意识和安全体系。

蓝队很重要，但蓝队不是万能的。
