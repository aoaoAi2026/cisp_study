## 6.1 密码破解概述

### 6.1.1 什么是密码破解

```
密码破解 = 把密文（哈希）还原成明文的过程

比如：
  明文：123456
  MD5哈希：e10adc3949ba59abbe56e057f20f883e
  
  密码破解就是给你这个哈希，
  让你反推出明文是 123456。
```

### 6.1.2 密码破解的方法

```
密码破解主要有这几种方法：

┌──────────────────────────────────┐
│  1. 字典攻击（Dictionary Attack） │
│  拿字典里的词一个个去试           │
│  成功率：取决于字典好不好          │
│  速度：快                         │
└──────────────────────────────────┘

┌──────────────────────────────────┐
│  2. 暴力破解（Brute Force）       │
│  所有组合一个个试（0-9 a-z A-Z...）│
│  成功率：100%（只要时间够）        │
│  速度：慢到离谱（位数多了根本跑不完）│
└──────────────────────────────────┘

┌──────────────────────────────────┐
│  3. 彩虹表（Rainbow Table）       │
│  预先算好一大堆哈希，存起来查表     │
│  速度：超快                       │
│  缺点：占空间太大（几百GB很正常）  │
└──────────────────────────────────┘

┌──────────────────────────────────┐
│  4. 掩码攻击（Mask Attack）       │
│  知道密码的一部分，猜剩下的        │
│  比如知道前三位是abc，后四位是数字 │
│  速度：比暴力快很多                │
└──────────────────────────────────┘

┌──────────────────────────────────┐
│  5. 规则攻击（Rule-based Attack） │
│  对字典里的词做变形               │
│  比如 password → Password → passw0rd → P@ssw0rd...
│  成功率：高                       │
│  速度：中等                       │
└──────────────────────────────────┘
```

### 6.1.3 常见的哈希类型

先来认一认各种哈希长啥样，别到时候认不出来。

| 哈希类型 | 长度 | 示例 |
|----------|------|------|
| MD5 | 32位 | `e10adc3949ba59abbe56e057f20f883e` |
| NTLM | 32位 | `31d6cfe0d16ae931b73c59d7e0c089c0` |
| SHA1 | 40位 | `7c4a8d09ca3762af61e59520943dc26494f8941b` |
| SHA256 | 64位 | `8d969eef6ecad3c29a3a629280e686cf0c3f5d5a86aff3ca12020c923adc6c92` |
| SHA512 | 128位 | 很长一串... |
| MySQL | 16位（老版）| `6f8c114b58f2ce9e` |
| MySQL5 | 40位 | 和SHA1差不多 |
| bcrypt | 60位左右 | `$2a$05$...` 开头 |
| WordPress | 34位 | `$P$` 开头 |
| WPA/WPA2 | - | 握手包，不是哈希 |
| ZIP/RAR | - | 加密压缩包 |

> 💡 **小技巧**：认不出来是什么哈希？
> 用 `hash-identifier` 工具扫一下就知道了。
> Kali里自带的，直接 `hash-identifier` 然后粘贴哈希值。

## 6.2 Hashcat——密码破解界的扛把子

Hashcat是目前最强大的密码恢复工具，
支持几百种哈希类型，
用GPU跑的话速度飞起。

### 6.2.1 Hashcat vs John the Ripper

两个都是常用的密码破解工具，
简单对比一下：

| 对比项 | Hashcat | John the Ripper |
|--------|---------|-----------------|
| 速度 | GPU加速，很快 | CPU为主，也有GPU版本 |
| 支持的哈希 | 非常多（几百种） | 也很多 |
| 易用性 | 参数多，但用熟了很顺手 | 相对简单一点 |
| 推荐度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

新手建议先学Hashcat，因为：
- 速度快，省时间
- 功能全，啥都能破
- 社区活跃，更新快

### 6.2.2 Hashcat基本用法

```bash
# 基本格式：
hashcat -m 哈希类型 -a 攻击模式 哈希文件 字典文件

# 参数说明：
# -m  指定哈希类型（比如 0=MD5, 1000=NTLM）
# -a  指定攻击模式（0=字典，3=掩码，6=字典+掩码...）
# -o  输出文件（把结果存哪）
# --show  显示已经破解出来的
```

### 6.2.3 攻击模式（-a参数）

```
-a 0   字典攻击（Straight）
       拿字典一个个跑

-a 1   组合攻击（Combination）
       两个字典拼起来
       字典1的每个词 + 字典2的每个词

-a 3   掩码攻击（Brute-force）
       按掩码规则生成密码去试

-a 6   字典+掩码（Hybrid Wordlist + Mask）
       字典的词 + 掩码的后缀

-a 7   掩码+字典（Hybrid Mask + Wordlist）
       掩码的前缀 + 字典的词
```

### 6.2.4 常用哈希类型（-m参数）

```
常见的-m值：

0     = MD5
10    = md5($pass.$salt)
20    = md5($salt.$pass)
100   = SHA1
300   = MySQL4.1/MySQL5
500   = MD5 Crypt
900   = MD4
1000  = NTLM  ← 重点！Windows哈希
1100  = MSSQL
1400  = SHA256
1700  = SHA512
1800  = sha512crypt $6$  ← Linux shadow里的那种
2500  = WPA/WPA2  ← WiFi密码
3200  = bcrypt $2*$, Blowfish (Unix)
400   = phpass   ← WordPress, Drupal...
133   = WPA/WPA2 PMKID

...还有几百种，用 hashcat --help | grep 关键词 来搜
```

> 💡 **查哈希类型**：
> 不知道哈希类型对应哪个编号？
> 去 https://hashcat.net/wiki/doku.php?id=example_hashes
> 上面有所有哈希类型的编号和示例。

### 6.2.5 实战1：破解MD5

先来个最简单的，破解MD5。

```bash
# 第一步：创建一个哈希文件
echo "e10adc3949ba59abbe56e057f20f883e" > md5.hash

# 第二步：用字典跑
hashcat -m 0 -a 0 md5.hash /usr/share/wordlists/rockyou.txt

# -m 0 表示MD5
# -a 0 表示字典攻击
```

跑出来了，结果会显示：
```
e10adc3949ba59abbe56e057f20f883e:123456
```

后面的 `123456` 就是明文密码。

```bash
# 查看已经破解的结果
hashcat -m 0 md5.hash --show

# 也可以直接看potfile
cat ~/.hashcat/hashcat.potfile
```

### 6.2.6 实战2：破解NTLM哈希

Windows密码哈希就是NTLM的，
这个最常用了。

```bash
# 哈希文件里存着抓到的哈希
# 格式可以是 用户名:RID:LM:NTLM::: 这种
# 也可以直接存NTLM哈希值

# 直接跑NTLM
hashcat -m 1000 -a 0 ntlm.hash rockyou.txt

# 如果是那种完整的hashdump格式（用户名:RID:LM:NTLM:::）
# 直接用-m 1000也能跑，hashcat会自动识别
```

> 💡 **小知识**：
> NTLM哈希跑起来巨快，
> 一张普通的显卡每秒能跑几十亿次，
> 弱密码分分钟就跑出来了。

### 6.2.7 实战3：掩码攻击

如果你知道密码的一部分，
比如知道密码是8位，前3位是abc，后5位是数字，
那就可以用掩码攻击，比暴力快多了。

#### 掩码符号说明

```
?l = 小写字母 a-z
?u = 大写字母 A-Z
?d = 数字 0-9
?s = 特殊字符 !@#$%^&*()...
?a = ?l?u?d?s 全部
?b = 0x00 - 0xff
```

#### 掩码例子

```
8位全数字：
  ?d?d?d?d?d?d?d?d

6位小写字母：
  ?l?l?l?l?l?l

前3位小写，后4位数字：
  ?l?l?l?d?d?d?d

第一位大写，后面5位小写加数字：
  ?u?l?l?l?l?l?d

第1位大写，第2-5小写，第6-8数字，第9位特殊符号：
  ?u?l?l?l?l?d?d?d?s
```

#### 实际跑一下

```bash
# 假设密码是6位数字
hashcat -m 0 -a 3 md5.hash ?d?d?d?d?d?d

# 假设密码是abc开头，后面4位数字
hashcat -m 0 -a 3 md5.hash abc?d?d?d?d

# 指定字符集（比如我只想用0-5这几个数字）
hashcat -m 0 -a 3 -1 012345 md5.hash ?1?1?1?1?1?1
# -1 自定义第一个字符集是 012345
# ?1 用第一个自定义字符集

# 可以自定义4个字符集：-1, -2, -3, -4
```

### 6.2.8 实战4：规则攻击

规则攻击就是对字典里的每个词做各种变形，
这样能大大提高破解率。

```
比如字典里有个词 password：
  加数字 → password1, password123, password2024
  改大小写 → Password, PASSWORD, pASSWORD
  替换字符 → p@ssword, passw0rd, P@ssw0rd
  翻转 → drowssap
  加特殊符号 → password!
  ......
```

Hashcat自带了很多规则，在：
```
/usr/share/hashcat/rules/
```

常用的规则：
- `best64.rule` —— 最常用的64个规则，推荐
- `d3ad0ne.rule` —— 效果不错，速度还行
- `dive.rule` —— 规则很多，慢但效果好
- `rockyou-30000.rule` —— 3万条规则
- `generated2.rule` —— 很多很多规则

```bash
# 用规则跑
hashcat -m 0 -a 0 md5.hash rockyou.txt -r /usr/share/hashcat/rules/best64.rule

# 多个规则一起用
hashcat -m 0 -a 0 md5.hash rockyou.txt -r rule1.rule -r rule2.rule
```

> 💡 **建议**：
> 先用 best64.rule 快速跑一遍，
> 没跑出来再用更大的规则，
> 比如 dive.rule 或者 rockyou-30000.rule。

### 6.2.9 实战5：破解Linux shadow

Linux的影子文件（/etc/shadow）里的密码哈希，
现在一般是sha512crypt（$6$开头的那种）。

```
shadow里的一行长这样：
root:$6$abcdefgh$iJKlmnoPqrsTuvWxYz...:19000:0:99999:7:::
   ↑        ↑
  用户名   密码哈希（$6$开头是sha512crypt）
```

破解方法：

```bash
# 第一步：把哈希单独存到文件里
# 把$6$开头到下一个冒号之前的内容复制出来
echo '$6$abcdefgh$iJKlmnoPqrsTuvWxYz...' > sha512.hash

# 第二步：跑hashcat
hashcat -m 1800 -a 0 sha512.hash rockyou.txt

# -m 1800 就是 sha512crypt $6$ 的意思
```

> ⚠️ **注意**：sha512crypt跑起来比MD5慢很多很多，
> 因为它计算的时候迭代了很多次，就是为了防破解。
> 弱密码可能要跑很久，强密码基本跑不出来。

### 6.2.10 实战6：破解WiFi密码（WPA/WPA2）

抓了WiFi的握手包之后，
就可以用hashcat跑密码了。

```bash
# 首先你得有握手包，比如叫 wifi.hccapx
# （.cap格式的要先转换成hccapx格式，可以用在线工具或者cap2hccapx）

# 然后跑字典
hashcat -m 2500 -a 0 wifi.hccapx rockyou.txt

# -m 2500 就是 WPA/WPA2
```

> 💡 **WiFi密码破解速度**：
> WPA2跑起来比NTLM慢不少，
> 因为算法比较复杂。
> 一般显卡每秒大概跑几千到几万次，
> 8位纯数字要跑挺久的。
> 所以字典好才是王道。

### 6.2.11 实战7：破解ZIP/RAR压缩包

加密的压缩包也能跑。

```bash
# ZIP的话先转换成hashcat能识别的格式
# 用zip2john工具（John the Ripper里的）
zip2john test.zip > zip.hash

# 然后跑
hashcat -m 13600 -a 0 zip.hash rockyou.txt
# -m 13600 = WinZip

# RAR的话用rar2john
rar2john test.rar > rar.hash

# 然后跑
hashcat -m 13000 -a 0 rar.hash rockyou.txt
# -m 13000 = RAR3-hp
```

### 6.2.12 Hashcat常用技巧

```
技巧1：查看进度
  按 s 键显示状态
  按 p 键暂停/继续
  按 q 键退出

技巧2：恢复中断的任务
  hashcat 跑的时候如果中断了，
  下次用 --restore 恢复
  hashcat --restore

技巧3：指定显卡
  多卡的话可以指定用哪张卡
  hashcat -d 1,2 ...  用第1和第2张卡
  hashcat -D 1 ...    用第1个设备类型（1=CPU,2=GPU,3=...）

技巧4：看看GPU温度
  按 s 能看到温度、风扇转速、利用率
  别把显卡烧了...

技巧5：优化速度
  -w 参数调工作负载
  -w 1  低负载（慢，凉快）
  -w 2  推荐（默认）
  -w 3  高负载（快，可能卡）
  -w 4  疯狂（最快，电脑基本动不了）
```

## 6.3 Hydra——在线爆破神器

Hashcat是离线破解（有哈希就行，不用连目标），
Hydra是在线爆破——一个个去试登录。

### 6.3.1 Hydra能破啥

Hydra支持超多协议：
- SSH
- FTP
- Telnet
- MySQL / MSSQL / PostgreSQL / Oracle...
- HTTP基本认证 / 表单登录
- SMB
- RDP
- VNC
- IMAP / POP3 / SMTP
- ......
  基本上你能想到的都支持。

### 6.3.2 Hydra基本用法

```bash
# 基本格式：
hydra -l 用户名 -P 密码字典 目标IP 协议

# 常用参数：
# -l  指定用户名（单个）
# -L  指定用户名字典（多个）
# -p  指定密码（单个）
# -P  指定密码字典
# -t  线程数（默认16，别设太大，会被封）
# -f  找到一个就停止
# -vV 显示详细过程
# -o  输出文件
# -s  指定端口
```

### 6.3.3 实战1：爆破SSH

```bash
# 单个用户名，字典爆破密码
hydra -l root -P rockyou.txt 192.168.1.100 ssh -t 4 -vV

# 用户名和密码都是字典
hydra -L users.txt -P rockyou.txt 192.168.1.100 ssh -t 4

# 指定端口
hydra -l root -P rockyou.txt 192.168.1.100 ssh -s 2222 -t 4

# 找到就停
hydra -l root -P rockyou.txt 192.168.1.100 ssh -f -o result.txt
```

### 6.3.4 实战2：爆破FTP

```bash
hydra -l admin -P rockyou.txt 192.168.1.100 ftp -t 4
```

### 6.3.5 实战3：爆破MySQL

```bash
hydra -l root -P rockyou.txt 192.168.1.100 mysql -t 4
```

### 6.3.6 实战4：爆破HTTP表单登录

这个稍微复杂一点，因为每个网站的登录表单都不一样。

首先你得知道：
- 登录的URL
- 提交方式（POST还是GET）
- 提交的参数是什么
- 怎么判断登录成功还是失败

用Burp抓个包，看看提交的是什么。

比如抓的包是这样的：
```
POST /login.php HTTP/1.1
Host: target.com

username=admin&password=123456&submit=Login
```

然后返回的内容里，
登录失败会显示 "Login failed"，
登录成功会显示 "Welcome"。

那Hydra就可以这么写：

```bash
hydra -l admin -P rockyou.txt target.com http-post-form \
  "/login.php:username=^USER^&password=^PASS^&submit=Login:F=Login failed"

# 格式：
# "URL:参数:失败的特征字符串"
# ^USER^ 会被替换成用户名
# ^PASS^ 会被替换成密码
# F=xxx 表示响应里有xxx就是失败

# 如果是成功的特征，用 S=
# "url:参数:S=Welcome"
# 表示响应里有Welcome就是成功
```

> 💡 **小提示**：
> HTTP表单爆破要注意CSRF token，
> 很多网站登录表单里有个随机的token，
> 每次都不一样，
> 这种Hydra就搞不定了，
> 需要用Burp的Intruder或者写脚本。

### 6.3.7 实战5：爆破SMB（Windows共享）

```bash
hydra -l administrator -P rockyou.txt 192.168.1.100 smb -t 4
```

### 6.3.8 实战6：爆破RDP（远程桌面）

```bash
hydra -l administrator -P rockyou.txt 192.168.1.100 rdp -t 4
```

### 6.3.9 Hydra使用注意事项

```
⚠️  注意事项：

1. 别把线程设太高
   设太大容易把目标打死，也容易被封
   一般4-8个线程就够了

2. 会留下日志
   在线爆破每一次尝试都会留下日志，
   对方很容易发现

3. 小心被封IP
   很多系统有登录失败次数限制，
   失败几次就锁账号或者封IP

4. 可能违法
   没授权的话，随便爆破别人的服务是违法的！
   只能在自己的实验环境或者授权的环境里用
```

## 6.4 字典——破解成功的关键

```
密码破解成功的关键，80%取决于你的字典好不好。
工具再强，字典里没有，那也白搭。
```

### 6.4.1 Kali里自带的字典

Kali已经给我们准备了一些字典，
在 `/usr/share/wordlists/` 目录下。

```bash
ls -la /usr/share/wordlists/

# 里面有：
# rockyou.txt → 最经典的字典，必用！
# dirb/ → 目录扫描写典
# dirbuster/ → 同上
# fern-wifi/ → WiFi字典
# wfuzz/ → fuzz字典
# ......
```

rockyou.txt是最有名的密码字典之一，
来自RockYou公司2009年的泄露事件，
有1400多万个真实密码。

```bash
# 如果rockyou.txt是压缩的，先解压
sudo gunzip /usr/share/wordlists/rockyou.txt.gz

# 看看有多少行
wc -l /usr/share/wordlists/rockyou.txt
# 应该有一千四百多万行
```

### 6.4.2 密码的常见规律

根据统计，人们设置密码有很多规律：

```
常见规律：

1. 纯数字
   • 123456
   • 12345678
   • 111111
   • 000000
   • 生日（19900101）
   • 手机号
   • QQ号

2. 键盘顺序
   • qwerty
   • 123456789
   • asdfgh
   • zxcvbn

3. 常见单词
   • password
   • admin
   • letmein
   • welcome
   • iloveyou
   • monkey
   • dragon

4. 单词+数字
   • password1
   • admin123
   • welcome123
   • 公司名+123

5. 单词+数字+特殊符号
   • Password123!
   • P@ssw0rd
   • Admin@123

6. 首字母大写
   • Password
   • Admin
   • Welcome

7. 替换字符（leet speak）
   • a → @, 4
   • e → 3
   • i → 1, !
   • o → 0
   • s → $, 5
   • t → 7
   比如：password → p@ssw0rd
```

### 6.4.3 生成针对性字典

通用字典命中率有限，
如果能针对目标生成专属字典，
命中率会高很多。

#### 方法1：用CUPP生成

CUPP是个专门生成个人专属字典的工具，
你输入目标的姓名、生日、宠物名、公司名...
它自动生成各种组合。

```bash
# 安装
sudo apt install cupp

# 启动
cupp -i

# 然后它会问你一堆问题：
# First Name? → 名
# Surname? → 姓
# Nickname? → 昵称
# Birthdate? → 生日
# Wife/Husband Name? → 配偶名字
# ...
# 答完就生成字典了
```

#### 方法2：用cupp的3.0版本

GitHub上搜 `cupp` 有更新的版本，
功能更多。

#### 方法3：从网站爬信息生成字典

用cewl工具，
爬目标网站的内容，
从网页里提取单词生成字典。

```bash
# 基本用法
cewl http://目标网站.com -w output.txt

# 指定爬的深度
cewl http://目标网站.com -d 3 -w output.txt

# 包含邮箱
cewl http://目标网站.com -e -w output.txt

# 包含数字
cewl http://目标网站.com -n -w output.txt
```

这个适合生成针对某个公司的字典，
比如公司名、产品名、Slogan这些，
很可能就是他们的密码。

### 6.4.4 字典整理

字典不是越大越好，
整理过的小字典可能比乱七八糟的大字典命中率还高。

```
常用的字典整理命令：

# 去重
sort -u dict.txt > dict_unique.txt

# 按长度筛选（比如只要6-12位的）
awk 'length>=6 && length<=12' dict.txt > dict_6-12.txt

# 只要纯数字的
grep -E '^[0-9]+$' dict.txt > dict_num.txt

# 只要小写字母的
grep -E '^[a-z]+$' dict.txt > dict_lower.txt

# 两个字典合并
cat dict1.txt dict2.txt | sort -u > dict_merged.txt

# 统计行数（看看有多少个密码）
wc -l dict.txt
```

### 6.4.5 哪里能找到好字典

```
推荐几个不错的字典资源：

1. SecLists
   GitHub上的 danielmiessler/SecLists
   超级全的字典集合，
   有密码字典、目录字典、用户名字典...
   强烈推荐！

2. Probable-Wordlists
   GitHub上的 berzerk0/Probable-Wordlists
   按出现概率排序的字典，
   越靠前的越可能是密码
   （因为是从各种泄露数据里统计出来的）

3. RockYou2024
   新版的RockYou字典，更大。

4. CrackStation字典
   crackstation.net 上的字典，
   有15GB的版本，很全。
```

## 6.5 John the Ripper——老牌破解工具

John the Ripper（简称John）也是个很经典的密码破解工具，
虽然速度可能不如Hashcat，
但它有很多好用的小工具，
比如各种格式转换的工具。

### 6.5.1 John基本用法

```bash
# 基本用法
john --format=格式 哈希文件

# 例子：破解MD5
john --format=raw-md5 md5.hash

# 用字典
john --format=raw-md5 --wordlist=rockyou.txt md5.hash

# 查看结果
john --show md5.hash
```

### 6.5.2 John的辅助工具

John自带了一堆辅助工具，都在 `/usr/sbin/` 或者 `/usr/bin/` 里：

```
常用的：

zip2john → 把ZIP转换成John能读的格式
rar2john → 把RAR转换成John能读的格式
ssh2john → 把SSH私钥转换成John能读的格式
pdf2john → 把PDF转换成John能读的格式
office2john → 把Office文档转换成John能读的格式
keepass2john → 把KeePass数据库转换成John能读的格式
bitlocker2john → 把BitLocker转换成John能读的格式
......
```

这些工具超级实用，
碰到各种加密格式，
先找有没有对应的 `xxx2john` 工具。

### 6.5.3 实战：破解Word文档密码

```bash
# 第一步：转换成John格式
office2john test.docx > doc.hash

# 第二步：跑John
john --wordlist=rockyou.txt doc.hash

# 第三步：看结果
john --show doc.hash
```

也可以用Hashcat跑，
office2john转出来的哈希，
看看是什么格式，
对应Hashcat的-m参数，
用Hashcat跑更快。

## 6.7 密码破解的成功率

很多新手会问：
"为什么我的密码跑不出来？"

答案很简单：**字典里没有。**

```
密码破解成功率参考：

• 弱密码（123456, admin, password...）
  → 90%+ 能跑出来

• 普通密码（姓名+生日，公司名+123...）
  → 有针对性字典的话30%-50%

• 强密码（8位以上，大小写+数字+特殊符号，随机）
  → 基本跑不出来
```

所以说，字典才是王道。
与其花时间在那傻跑，
不如花时间把字典弄好点。

### 提高成功率的建议

```
1. 先小字典，后大字典
   先用小而精的字典（比如top10000）快速跑一遍
   没出再用大字典

2. 先用规则，再暴力
   字典+规则的效果比纯暴力好太多了

3. 针对目标生成字典
   知道目标的姓名、生日、公司名...
   针对性生成字典，命中率高很多

4. 收集泄露数据
   各种网站泄露的数据库，
   里面的密码都是真实的用户在用的，
   拿来做字典命中率很高
```

## 6.8 防御密码破解

讲了这么多攻击，也得讲讲怎么防。

```
防御密码破解的方法：

1. 设置强密码
   • 至少8位，越长越好
   • 大小写+数字+特殊符号
   • 不要用常见单词
   • 不要用生日、手机号、姓名
   • 每个网站用不同的密码

2. 用加盐哈希（Salt）
   不要直接存明文密码的哈希
   要加盐再哈希，每个用户盐不一样
   这样彩虹表就没用了

3. 用慢哈希算法
   不要用MD5、SHA1这种快的
   要用 bcrypt、scrypt、Argon2 这种慢的
   让破解者跑不动

4. 限制登录尝试次数
   登录失败几次就锁账号或者验证码
   防止在线爆破

5. 多因素认证（MFA）
   密码+短信/令牌/生物识别
   就算密码泄露了也登不进去

6. 定期改密码？
   这个有争议，现在很多安全专家建议
   除非怀疑泄露了否则不用定期改
   因为定期改反而导致用户设弱密码
```
