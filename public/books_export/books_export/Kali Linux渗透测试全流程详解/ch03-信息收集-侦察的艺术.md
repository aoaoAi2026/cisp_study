# 第三章 信息收集——侦察的艺术

> 第3章 | 预计学习时间：5小时 | 难度：⭐⭐⭐

## 📖 写在前面

上一章我们把Kali环境搭好了，
从这一章开始，
我们正式进入渗透测试的流程。

渗透测试的第一步是什么？
是拿工具一顿乱扫吗？
是直接找漏洞打吗？

都不是。
渗透测试的第一步，
是**信息收集**。
也叫侦察（Reconnaissance）。

很多新手不重视信息收集，
觉得"不就是查查域名IP嘛，有啥难的"，
上来就想直接打漏洞。
结果呢？
打半天打不进去，
不是因为技术不行，
而是因为信息没收集够，
不知道从哪下手。

有经验的渗透测试工程师，
会花大量的时间在信息收集上。
收集的信息越全，
后面的工作越容易。
甚至有时候，
信息收集做好了，
漏洞自己就冒出来了。

所以说，
信息收集是渗透测试的基础，
也是最重要的一步之一。
基础打牢了，
后面才能走得远。

这一章我们就来好好聊聊信息收集，
看看一个目标摆在你面前，
你都能挖出哪些信息来。

坐稳了，开唠。

## 3.1 先讲一个故事

### 3.1.1 "信息收集救了命"

先给你讲个真实的故事。

有个做渗透测试的朋友，我们叫他老张吧。
老张有一次接了个活，
给一家公司做渗透测试。
目标是他们的官网，
授权范围也只有官网。

老张一开始以为挺简单的，
不就是个官网嘛，
扫扫漏洞，看看有没有注入啥的。
结果呢？
扫了半天，
官网做得还挺扎实，
常见的漏洞都没有，
WAF也挺厉害，
各种绕过都不好使。

眼看时间要到了，
还没打进去，
老张有点急了。

这时候老张想，
既然官网打不进去，
那我多收集点信息，
看看有没有别的入口。
反正信息收集是授权范围之内的。

于是老张开始疯狂收集信息：
- 子域名，一个一个挖
- 域名注册信息，查
- 公司的员工信息，搜
- 邮箱格式，找
- 泄露的密码，查
- 各种网络空间搜索引擎，扫

这不收不知道，一收下一跳。

老张发现了什么？
- 这家公司有个测试站，
  子域名是test.xxx.com，
  没人管，漏洞一大堆
- 还有个老系统，
  是外包做的，交完活就没人维护了
  有个SQL注入漏洞，直接拖库
- 而且从泄露的信息里，
  还找到了几个员工的邮箱和密码，
  虽然是老密码，
  但有一个还能用，
  能登后台

最后，
老张从那个测试站的SQL注入打进去，
拿到了服务器权限，
又用泄露的账号登了后台，
还从老系统里找到了内网的线索。
圆满完成任务。

客户都惊了：
"我们以为你们就打打官网，
没想到你们找出这么多问题..."

---

这个故事告诉我们什么？
**信息收集太重要了。**
- 官网打不进去，不代表别的地方打不进去
- 信息收集做得好，
  可能找到更弱的入口
- 信息越全，
  思路越多，
  打进去的概率越大

新手和老手的区别，
很多时候不是在漏洞利用上，
而是在信息收集上。
新手扫完端口就没思路了，
老手能从各种角度挖出各种信息，
找到各种突破口。

所以，
不要轻视信息收集。
这是一门艺术，
也是渗透测试最基础、最重要的基本功。

## 3.2 信息收集概述

在正式讲方法之前，
先讲几个基本概念。

### 3.2.1 信息收集的分类

信息收集分两大类：
**被动信息收集**和**主动信息收集**。

```
被动信息收集：
  • 什么意思？
    不直接跟目标系统接触，
    通过第三方的信息来收集。
  • 特点：
    - 目标不会发现你
    - 不会留下痕迹
    - 比较安全，不会触发告警
  • 比如：
    - 查Whois信息
    - 搜索引擎
    - 网络空间搜索引擎
    - 社交媒体
    - 公开的泄露数据
    - 等等

主动信息收集：
  • 什么意思？
    直接跟目标系统交互，
    去探测、去扫描。
  • 特点：
    - 会直接接触目标
    - 可能会留下痕迹
    - 可能会触发WAF、IDS告警
    - 但信息更准确、更直接
  • 比如：
    - 端口扫描
    - 服务识别
    - 漏洞扫描
    - 目录扫描
    - 等等
```

一般来说，
渗透测试的顺序是：
**先被动，后主动。**
- 先偷偷摸摸收集各种公开信息，
  别惊动目标。
- 收集得差不多了，
  再主动去扫、去探。

当然，
授权测试的话不用那么小心翼翼，
但习惯要养好。

### 3.2.2 我们要收集什么信息？

信息收集都收集啥？
可以收集的东西太多了，
我给你列一下常见的。

```
网络信息：
  • 域名、子域名
  • IP地址、IP段
  • 端口、开放的服务
  • 网站技术栈（什么语言、什么框架、什么中间件）
  • WAF/防火墙信息
  • 网络拓扑（能拿到多少算多少）

组织信息：
  • 公司结构、部门
  • 员工姓名、邮箱、电话
  • 员工社交账号
  • 办公地址、联系方式
  • 合作伙伴、供应商

系统信息：
  • 操作系统类型和版本
  • 中间件类型和版本
  • 数据库类型和版本
  • 应用程序版本
  • 漏洞信息

人员信息：
  • 员工信息（社工用）
  • 常用密码（泄露库）
  • 邮箱格式
  • 账号命名规则

其他信息：
  • 历史漏洞
  • 泄露的数据
  • 代码仓库（GitHub、码云...）
  • 文档（PDF、PPT、docx...）
  • ......
```

能收集的信息太多了，
只有你想不到，
没有你找不到的。

收集得越多越细，
后面就越好打。

## 3.3 被动信息收集

我们先讲被动信息收集。
不用碰目标，
先能拿多少拿多少。

### 3.3.1 Whois 查询

最基础的，
Whois 查询。
什么是Whois？
就是查域名的注册信息。
- 谁注册的？
- 什么时候注册的？
- 注册商是谁？
- DNS服务器是什么？
- 联系邮箱、电话是多少？
- 等等

这些信息都是公开的，
一查就能查到。

```
怎么查：
  1. 命令行查：
     whois 域名
     比如：whois baidu.com

  2. 网站查：
     站长工具、爱站网、阿里云Whois查询...
     很多网站都能查
```

从Whois信息里能得到什么？
- 注册人的名字（可能是管理员的名字）
- 联系邮箱（可以用来社工、钓鱼）
- 联系电话
- DNS服务器（可以试试域传送）
- 注册时间、过期时间

虽然现在很多域名做了隐私保护，
查不到具体信息了，
但还是可以看看，
万一没保护呢？

### 3.3.2 子域名收集

比Whois更重要的，
是**子域名收集**。

什么是子域名？
比如baidu.com是主域名，
www.baidu.com、tieba.baidu.com、map.baidu.com
这些就是子域名。

为什么子域名这么重要？
因为：
- 主站一般防护最好，难打
- 子域名可能有各种测试站、边缘系统、老系统
- 这些系统往往没人管、漏洞多，
  是很好的突破口

就像前面故事里讲的，
主站打不进去，
从一个没人管的子域名打进去了。

子域名收集是信息收集里非常重要的一环。
怎么收集子域名？
方法多了去了。

```
常见的子域名收集方法：

  方法一：搜索引擎
    • 百度/Google搜 site:域名
    • 比如 site:baidu.com
    • 能搜出很多收录的子域名

  方法二：DNS历史记录
    • 用一些DNS查询的网站
    • 比如：
      - DNSdumpster
      - VirusTotal
      - SecurityTrails
      - 等等
    • 能查到很多历史解析的子域名

  方法三：证书透明度（CT）
    • SSL证书里会包含域名信息
    • 可以通过证书透明度日志查子域名
    • 比如crt.sh这个网站
    • 输入域名，能查到很多子域名
    • 这个方法很准，推荐

  方法四：子域名爆破
    • 用工具跑字典，猜子域名
    • 比如 sublist3r、amass、fierce...
    • 字典够大的话，能跑出来很多
    • 这个是主动的了，因为要发请求
    • 不过一般也不会太引起注意

  方法五：第三方数据源
    • 有很多网站收集了大量子域名
    • 比如FOFA、Shodan、ZoomEye
    • 搜一下就能出来很多

  方法六：爬虫爬
    • 爬主站，从页面里找子域名的链接
    • 也能找到一些

  方法七：泄露的文件/文档
    • 从公开的PDF、PPT、Word文档里
    • 也经常能找到子域名
```

方法很多，
而且要多种方法结合着用。
不同的方法能找到不同的子域名，
多管齐下，
才能收集得比较全。

推荐几个好用的工具：
- **Sublist3r**：子域名收集工具，Kali里有
- **Amass**：更强大的信息收集工具，子域名收集只是其中一个功能
- **OneForAll**：国产的子域名收集工具，很好用
- **subfinder**：快速子域名发现工具

工具只是辅助，
关键是思路要开阔，
能想到从各种角度去挖。

### 3.3.3 DNS信息收集

除了子域名，
DNS相关的信息也很多。

```
常见的DNS信息收集：

  1. 查看各种记录
     • A记录：域名对应的IP
     • MX记录：邮件服务器
     • NS记录：DNS服务器
     • CNAME记录：别名
     • TXT记录：文本记录（经常藏东西）
     • 等等

     怎么查？
     • dig 域名
     • nslookup 域名
     • 或者在线网站查

  2. 域传送漏洞
     • 如果DNS服务器配置不好
       可能存在域传送漏洞
     • 直接能把整个域的所有记录都拉下来
       子域名、内部IP啥的全出来了
     • 虽然现在很少见了，
       但还是可以试试
     • 万一呢？

     怎么试？
     • dig @DNS服务器 域名 axfr
     • 能成功的话，赚大了

  3. 反向查询
     • 知道IP，反查域名
     • 看看这个IP上还绑了哪些网站
     • 有时候能有意外收获

  4. DNS历史记录
     • 这个域名历史上解析到过哪些IP？
     • 能找到一些现在不用了的IP
     • 可能还没关呢
```

DNS里藏着很多信息，
就看你会不会挖。

### 3.3.4 搜索引擎的妙用

别小看搜索引擎，
Google、百度这些，
可是信息收集的神器。
会用的人，
能用搜索引擎挖出一大堆信息。

```
搜索引擎常用技巧：

  site:域名       只搜这个域名下的页面
  inurl:关键词    URL里有关键词的
  intitle:关键词  标题里有关键词的
  intext:关键词   正文里有关键词的
  filetype:pdf   只搜PDF文件
  "关键词"       精确匹配
  -关键词        排除某个词
  ......
```

举几个例子：
- `site:xxx.com inurl:login` —— 找后台登录页
- `site:xxx.com filetype:pdf` —— 找PDF文档，里面可能有敏感信息
- `site:github.com xxx.com` —— 搜GitHub上有没有泄露的代码
- `site:xxx.com 密码` —— 搜有没有不小心泄露的密码
- `site:xxx.com inurl:php?id=` —— 找可能有SQL注入的页面

组合起来用，
能搜到很多意想不到的东西。

国外的话推荐Google，
国内的话百度、必应都可以用。
多换几个引擎搜，
结果不一样。

### 3.3.5 网络空间搜索引擎

除了普通的搜索引擎，
还有专门搜网络设备的搜索引擎，
叫**网络空间搜索引擎**。
比如：
- **FOFA**（国产，比较适合国内）
- **Shodan**（国外的，很有名）
- **ZoomEye**（知道创宇的，国产）
- **Quake**（360的）

这些搜索引擎干啥的？
它们不停地扫描全网，
把各种设备、网站、服务都收录下来，
你可以通过各种条件去搜。

```
能搜什么：
  • 某个IP段有哪些设备
  • 某个公司有哪些公网系统
  • 某种组件/中间件的所有实例
  • 有某种漏洞的主机
  • 某个城市的摄像头
  • ......

  只有你想不到，没有你搜不到的
```

举几个例子：
- `domain="xxx.com"` —— 搜这个域名相关的所有资产
- `ip="1.1.1.1/24"` —— 搜这个C段的所有资产
- `app="Tomcat"` —— 搜所有Tomcat的服务器
- `city="北京"` —— 搜北京的设备
- `body="后台管理"` —— 搜页面里有"后台管理"的网站

这些东西用好了，
能帮你快速找到目标的所有资产，
包括很多你不知道的边缘系统。
做信息收集一定要会用。

推荐先玩玩FOFA，
国内的资产比较全，
中文界面也友好。

### 3.3.6 员工信息与社工情报

还有一类信息很重要——
**人员信息**。
也就是目标公司的员工信息。

这些信息有啥用？
用来做社会工程学攻击啊。
- 钓鱼邮件
- 暴力破解账号
- 社工库查密码
- 等等

怎么收集员工信息？

```
常见方法：

  1. 领英/脉脉
     • 职场社交平台
     • 搜公司名，能找到很多员工
     • 姓名、职位、工作经历...都有
     • 非常好用

  2. 公司官网
     • 关于我们、团队介绍
     • 新闻里可能也有人名
     • 招聘页面也有很多信息

  3. 微博/知乎/各种社交平台
     • 搜公司名，可能找到员工
     • 从员工的发言里能挖到很多信息

  4. 邮箱格式
     • 搞清楚公司的邮箱命名规则
       比如 姓.名@公司.com 还是 姓名全拼@公司.com
     • 有了名字和邮箱格式，
       就能生成一堆邮箱账号
     • 用来暴力破解、钓鱼都很好用

  5. 社工库/泄露库
     • 看看这些邮箱/手机号
       有没有在历史上泄露过
     • 如果有泄露的密码，
       说不定现在还能用
     • 这个就不用我多说了
```

人员信息收集好了，
社会工程学攻击就成功了一半。
很多时候，
人才是最大的漏洞。

### 3.3.7 GitHub等代码仓库

还有一个很重要的地方——
代码托管平台，
比如GitHub、码云、GitLab等等。

很多公司、很多员工，
会不小心把代码传到公开仓库里，
而代码里面，
经常会有各种敏感信息：
- 数据库密码
- API Key
- 内网地址
- 账号密码
- 各种配置文件
- 甚至整个项目的源代码

运气好的话，
直接就能从代码里找到后台地址、
数据库密码、
甚至直接拿到源码去审计漏洞。

```
怎么搜：
  1. 直接搜公司名、产品名
  2. 搜域名
  3. 搜员工的名字/GitHub账号
  4. 搜关键词：password、secret、key、token...
  5. 结合各种过滤条件
```

很多大公司都在GitHub上翻过车，
更别说小公司了。
这个地方一定要搜一搜，
经常有意想不到的收获。

### 3.3.8 其他被动信息收集方法

还有很多其他方法，
我就不一一细讲了，
给你列一下，
有个印象就行。

```
其他方法：
  • 网盘搜索：搜各种网盘，
    看看有没有泄露的文件
  • 文库：百度文库、豆丁...
    可能有内部文档
  • 招聘网站：从招聘信息里
    能看出公司用什么技术栈
  • 新闻/媒体：看看有没有相关的新闻
  • 论坛/贴吧：员工可能会在上面吐槽
  • ......
```

总之，
只要是公开的信息，
都可以想办法收集。
思路要开阔，
不要局限于某几种方法。

## 3.4 主动信息收集

被动信息收集做得差不多了，
接下来就是主动信息收集了。
直接去探、去扫，
看看目标到底开了哪些端口、
跑了哪些服务、
有哪些漏洞。

### 3.4.1 端口扫描之王——Nmap

讲主动信息收集，
第一个必须讲**Nmap**。
这是端口扫描的神器，
渗透测试必备，
必学工具，没有之一。

Nmap能干啥？
- 主机发现（哪些IP是活的）
- 端口扫描（开了哪些端口）
- 服务识别（跑的什么服务、什么版本）
- 操作系统识别（是什么系统）
- 漏洞扫描（简单的漏洞探测）
- 脚本扩展（NSE脚本，各种功能）

功能非常强大，
而且用法很多，
学深了很有东西。

这里我们讲最常用的几种用法，
先入门，
更多的用法自己慢慢研究。

```
常用的Nmap命令：

  1. 最基础的端口扫描
     nmap 目标IP

     会扫1000个常用端口，
     看看哪些开着。

  2. 指定端口扫描
     nmap -p 80,443,3306 目标IP
     nmap -p 1-65535 目标IP     （全端口扫描）
     nmap -p- 目标IP            （也是全端口）

  3. 服务版本识别
     nmap -sV 目标IP

     不光扫端口，
     还识别端口上跑的是什么服务、什么版本。
     这个很重要，知道了版本才能找对应漏洞。

  4. 操作系统识别
     nmap -O 目标IP

     识别目标是什么操作系统。

  5. 全功能扫描
     nmap -A 目标IP

     相当于 -sV -O 还有其他一堆功能全开，
     全面扫描，信息最全。
     就是慢一点，动静大一点。

  6. 主机发现（扫C段）
     nmap -sP 192.168.1.0/24
     或者 nmap -sn 192.168.1.0/24

     看看这个网段里哪些主机是活的。

  7. 脚本扫描
     nmap --script=脚本名 目标IP

     Nmap有NSE脚本引擎，
     有几百个脚本可以用。
     比如：
     --script=vuln  扫常见漏洞
     --script=http-enum  枚举Web路径
     等等很多很多

  8. 加快速度
     nmap -T4 目标IP

     -T是速度，0到5，越大越快
     T4比较常用，速度快也不容易漏
     T5最快，但可能漏报
```

Nmap的用法太多了，
讲一章都讲不完。
新手先把上面这几个常用的掌握好，
够用了。
后面慢慢深入。

记住一句话：
**渗透测试工程师可以什么都不会，
但不能不会Nmap。**
虽然有点夸张，
但也说明了Nmap的重要性。

### 3.4.2 服务识别与指纹识别

扫出来端口和服务了，
接下来要干啥？
要搞清楚：
- 这是什么服务？
- 什么版本？
- 运行在什么系统上？
- 用的什么框架？
- 等等

这些信息，
叫**指纹**。
把指纹搞清楚了，
才能找对应的漏洞。

怎么识别？

```
服务/系统指纹识别：

  1. Nmap -sV -O
     • 刚才讲过了
     • 能识别服务版本和操作系统

  2. 看Banner
     • 很多服务连上去就会返回Banner
     • 比如SSH、FTP、SMTP...
     • 里面有版本信息
     • 直接nc或者telnet连上去看看

  3. Web指纹识别
     • 如果是Web服务，
       要识别的东西就多了：
       - 是什么Web服务器？（Nginx/Apache/IIS...）
       - 什么编程语言？（PHP/Java/Python...）
       - 什么框架？（ThinkPHP/SpringBoot/Django...）
       - 什么CMS？（WordPress/Discuz/织梦...）
       - 什么数据库？

     怎么识别？
     • 看HTTP头（Server、X-Powered-By...）
     • 看页面特征（favicon.ico、特定路径、特定参数）
     • 看Cookie
     • 看返回的错误页面
     • 用工具识别

     常用工具：
     • whatweb （Kali自带，很好用）
     • wappalyzer （浏览器插件）
     • 御剑指纹识别
     • ......

  4. CMS识别
     • 如果网站是用CMS做的，
       识别出来是什么CMS很重要
     • 因为CMS的漏洞都是公开的，
       知道了CMS，直接找对应版本的漏洞就行了
```

指纹识别是个技术活，
做得越细，
后面找漏洞就越容易。
看到一个网站，
扫一眼就能说出它用了什么技术栈，
那才算入门了。

### 3.4.3 目录/文件扫描

如果目标是个Web应用，
那目录扫描也是少不了的。
什么是目录扫描？
就是用工具跑字典，
看看网站上有哪些目录、哪些文件。
比如：
- 后台地址（/admin、/login、/manage...）
- 备份文件（.bak、.zip、.rar...）
- 配置文件
- 数据库文件
- 测试文件
- phpinfo()页面
- 等等

这些东西如果被人找到了，
可能会泄露很多敏感信息，
甚至直接getshell。

```
常用的目录扫描工具：

  • dirb：Kali自带，简单好用
  • dirbuster：图形界面的，Java写的
  • gobuster：Go语言写的，很快
  • 御剑：国产的，很多人用
  • dirsearch：Python写的，很强大，推荐

用法都差不多：
  1. 输入目标URL
  2. 选择字典
  3. 开始扫
  4. 看结果，哪些路径存在
```

目录扫描的关键是字典，
字典越好，
扫出来的东西越多。
网上有很多开源的字典，
可以找来用，
也可以自己积累。

### 3.4.4 漏洞扫描

信息收集的最后一步，
就是漏洞扫描了。
用漏洞扫描器去扫一下，
看看有没有已知的漏洞。

常用的漏洞扫描器：
- **Nessus**：商业软件，很强大，收费的
- **OpenVAS**：开源的，免费，功能也挺全
- **AWVS**：Web漏洞扫描神器，打网站必备
- **AppScan**：IBM的，也是Web扫描
- **Nikto**：轻量的Web扫描器，Kali自带
- **Xray**：国产的，很好用，有免费版

漏洞扫描器能帮你快速发现
大部分已知的、常见的漏洞，
省很多事。
但别完全依赖扫描器，
它不是万能的：
- 逻辑漏洞扫不出来
- 0day扫不出来
- 误报、漏报都很正常
- 需要人工去验证

扫描器是帮手，
不是替代你的。
核心还是要靠人。

## 3.5 信息整理与分析

信息收集了一大堆，
别就那么放着，
要整理、要分析。
不然收集了那么多，
最后都浪费了。

怎么整理？

```
整理方法：

  1. 分类整理
     • 域名/子域名
     • IP/端口/服务
     • 人员信息
     • 漏洞信息
     • ......
     分门别类放好

  2. 画攻击面地图
     • 把目标的所有资产都列出来
     • 哪些是公网的，哪些是内网的
     • 每个资产上有哪些服务、哪些漏洞
     • 哪些重要，哪些不重要
     • 心里要有张图

  3. 找突破口
     • 整理完了，分析一下
     • 从哪里打最容易？
     • 哪些地方可能有漏洞？
     • 优先级怎么排？
     • 先打哪个，后打哪个？

  4. 制定攻击路线
     • 大概的攻击思路是什么？
     • 第一步打哪，第二步打哪
     • 这条路走不通了，备选方案是什么？
```

信息收集不是目的，
目的是为了后面的攻击做准备。
收集完信息，
要做到心里有数，
知道下一步该干啥。

## 3.6 给新手的几个建议

最后，给信息收集阶段的新手几个建议。

### 建议一：耐心点，别着急

很多新手收集信息的时候没耐心，
随便扫两下就想去打漏洞了。
别急，
信息收集越充分，
后面越顺利。
磨刀不误砍柴工。

经验丰富的渗透测试工程师，
可能会花一半甚至更多的时间在信息收集上。
别觉得这是浪费时间。

### 建议二：思路要开阔

别只会用一两种方法，
思路要开阔。
- 除了扫端口，还可以搜GitHub
- 除了搜子域名，还可以搜员工信息
- 除了主动扫，还可以被动收集
- 各种方法都试试

思路越开阔，
收集到的信息越多，
突破口也就越多。

### 建议三：记录下来

收集到的信息，
都记下来。
别靠脑子记，
记不住的。
用笔记、用表格、用文档，
怎么方便怎么来。
整理得清清楚楚的，
后面用的时候一查就有。

### 建议四：养成信息收集的习惯

信息收集不是只有渗透测试的时候才做，
平时也要养成习惯。
看到一个网站，
下意识地想想：
这个站用的什么技术栈？
有哪些子域名？
可能有哪些漏洞？

慢慢的，
信息收集就成了你的本能，
水平自然就上去了。

## 3.7 本章总结

这一章我们讲了信息收集，
内容比较多，总结一下：

**信息收集的分类：**
- 被动收集：不接触目标，通过第三方收集（安全、隐蔽）
- 主动收集：直接探测目标（信息更直接，但会留痕迹）
- 一般顺序：先被动，后主动

**被动信息收集：**
1. Whois查询 —— 域名注册信息
2. 子域名收集 —— 非常重要，找边缘系统的入口
3. DNS信息收集 —— 各种记录、域传送、反向查询
4. 搜索引擎 —— Google Hacking，各种语法组合
5. 网络空间搜索引擎 —— FOFA、Shodan、ZoomEye
6. 员工信息/社工情报 —— 为社会工程学做准备
7. GitHub等代码仓库 —— 找泄露的代码、密码
8. 还有很多很多...

**主动信息收集：**
1. **Nmap** —— 端口扫描之王，必学
   - 主机发现、端口扫描、服务识别、系统识别、脚本扫描
2. 服务/指纹识别 —— 知道是什么版本，才能找漏洞
   - whatweb、wappalyzer...
3. 目录/文件扫描 —— 找后台、备份、敏感文件
   - dirsearch、gobuster、御剑...
4. 漏洞扫描 —— 用扫描器快速发现已知漏洞
   - Nessus、AWVS、OpenVAS、Xray...

**信息整理：**
- 分类整理，画攻击面地图
- 找突破口，制定攻击路线
- 信息收集是为了更好地攻击

**给新手的建议：**
1. 耐心点，别着急
2. 思路要开阔
3. 记录下来
4. 养成信息收集的习惯

最后送你一句话：

> **优秀的渗透测试工程师，
> 首先是一个优秀的情报收集专家。
> 信息收集做得好，
> 渗透测试就成功了一半。**
