## 1.1 渗透测试概述

### 1.1.1 什么是渗透测试

渗透测试（Penetration Testing）是一种受控的安全评估方法，通过模拟真实攻击者的技术和方法，对计算机系统、网络或应用程序进行安全测试，以发现潜在的安全漏洞和弱点。

**1. 渗透测试的目标**

- 发现安全漏洞和弱点
- 评估现有安全防护措施的有效性
- 验证安全策略的执行情况
- 提供安全改进建议

**2. 渗透测试与漏洞扫描的区别**

| 对比项 | 渗透测试 | 漏洞扫描 |
|--------|----------|----------|
| 测试方式 | 人工+工具 | 自动工具 |
| 测试深度 | 模拟真实攻击 | 检测已知漏洞 |
| 测试时间 | 较长（数天至数周） | 较短（数小时） |
| 发现内容 | 漏洞链、业务逻辑问题 | CVE已知漏洞 |
| 结果输出 | 详细攻击路径 | 漏洞清单 |

**3. 渗透测试类型**

```
按信息掌握程度分类：
├── 黑盒测试（Black Box）
│   └── 不了解目标内部信息，完全模拟外部攻击者
├── 白盒测试（White Box）
│   └── 完全了解系统内部信息，代码、架构等
└── 灰盒测试（Gray Box）
    └── 部分了解系统信息，如普通用户权限

按测试目标分类：
├── 网络渗透测试
├── Web应用渗透测试
├── 移动应用渗透测试
├── 无线网络渗透测试
├── 社会工程渗透测试
└── 物理安全渗透测试
```

### 1.1.2 渗透测试流程

**1. 渗透测试标准框架**

```
渗透测试执行标准（PTES）：
├── 前期交互（Pre-engagement Interactions）
├── 情报收集（Intelligence Gathering）
├── 威胁建模（Threat Modeling）
├── 漏洞分析（Vulnerability Analysis）
├── 漏洞利用（Exploitation）
├── 后渗透（Post Exploitation）
└── 报告（Reporting）
```

**2. 各阶段详解**

```
阶段1: 前期交互
├── 确定渗透测试范围
├── 定义测试目标
├── 获取正式授权
├── 建立沟通机制
└── 确定规则和时间表

阶段2: 情报收集
├── 开源情报（OSINT）
├── 被动信息收集
├── 主动信息收集
└── 指纹识别

阶段3: 威胁建模
├── 确定攻击入口点
├── 识别资产和弱点
├── 制定攻击方案
└── 确定优先级

阶段4: 漏洞分析
├── 漏洞扫描
├── 手工测试
├── 漏洞验证
└── 漏洞分析

阶段5: 漏洞利用
├── 漏洞利用开发
├── 绕过安全防护
├── 获取初始访问
└── 维持访问

阶段6: 后渗透
├── 权限提升
├── 横向移动
├── 数据收集
└── 持久化控制

阶段7: 报告编写
├── 执行摘要
├── 技术细节
├── 风险评估
└── 修复建议
```

### 1.1.3 渗透测试法律与道德

**1. 授权的重要性**

渗透测试必须获得**书面授权**，授权文档应包含：

- 测试目标范围
- 测试方法限制
- 时间安排
- 数据处理要求
- 应急联系方式

**2. 渗透测试注意事项**

```
禁止测试内容：
├── 未授权的DDoS攻击
├── 可能导致系统崩溃的攻击
├── 社工攻击（除非明确授权）
└── 物理安全测试（除非明确授权）

必须遵守：
├── 不超过授权范围
├── 不获取或泄露敏感数据
├── 不破坏系统正常运行
├── 及时报告高危漏洞
└── 保护测试数据安全
```

## 1.2 Kali Linux简介

### 1.2.1 Kali Linux概述

Kali Linux是Offensive Security公司开发的专为渗透测试和安全审计设计的Linux发行版，基于Debian系统，预装了数百款安全工具。

**1. Kali Linux发展历史**

```
Kali Linux历史：
├── 前身：BackTrack（2006-2012）
├── Kali Linux 1.0（2013.3）
├── Kali Linux 2.0（2015.8）
├── Kali Linux 2017.1（滚动更新模式）
└── 持续更新中...

版本特点：
├── 滚动更新：始终保持最新工具
├── 多平台支持：x86、ARM、容器、云
├── 定制化：支持Kali Linux NetHunter
└── 取证模式：支持数字取证
```

**2. Kali Linux特点**

```
1. 丰富的工具集
   - 预装600+安全工具
   - 涵盖渗透测试全流程
   - 持续更新的工具库

3. 定制化能力强

4. 社区支持
```

```
└── 网络模式选择桥接或NAT

VirtualBox：
├── 推荐分配4GB+内存
├── 分配4+处理器
```

### 1.3.2 VMware虚拟机安装

**1. 创建虚拟机步骤**

```
1. 下载Kali Linux镜像
   官网：https://www.kali.org/get-kali/
   下载：kali-linux-2024.x-installer-amd64.iso

2. VMware创建虚拟机
   - 文件 → 新建虚拟机
   - 选择"典型"
   - 选择下载的ISO镜像
   - 选择Linux → Debian 10.x 64位
   - 设置虚拟机名称和位置
   - 分配磁盘空间50GB
   - 自定义硬件（内存4GB、处理器4核）
   - 完成创建

3. 启动虚拟机安装
   - 启动虚拟机
   - 选择"Graphical install"
   - 选择语言、位置、键盘布局
   - 配置网络（主机名、域名）
   - 设置Root密码
   - 选择分区方案（建议使用整个磁盘）
   - 完成安装
```

**2. VMware Tools安装**

```bash
# 安装Open VM Tools（推荐）
apt update
apt install open-vm-tools-desktop fuse

# 或安装完整VMware Tools
# 虚拟机菜单 → 安装VMware Tools
mount /dev/cdrom /mnt
cd /mnt
tar xzf VMwareTools-*.tar.gz -C /tmp
cd /tmp/vmware-tools-distrib
./vmware-install.pl
```

### 1.3.3 VirtualBox虚拟机安装

**1. 创建虚拟机步骤**

```
1. 新建虚拟机

   - 类型：Linux

   - 内存：4096MB
```

### 1.4.2 DVWA搭建

**1. DVWA简介**

DVWA（Damn Vulnerable Web Application）是一个用PHP和MySQL开发的Web应用，故意包含了各种常见的Web漏洞。

**2. 安装步骤**

```bash
# 安装LAMP/LEMP
apt update
apt install apache2 php php-mysql php-gd mariadb-server

# 下载DVWA
cd /var/www/html
git clone https://github.com/digininja/DVWA.git

# 配置数据库
mysql -u root -p
CREATE DATABASE dvwa;
CREATE USER 'dvwa'@'localhost' IDENTIFIED BY 'p@ssw0rd';
GRANT ALL PRIVILEGES ON dvwa.* TO 'dvwa'@'localhost';
FLUSH PRIVILEGES;
EXIT;

# 配置DVWA
cd /var/www/html/DVWA
cp config/config.inc.php.dist config/config.inc.php
vim config/config.inc.php

# 设置目录权限
chmod 755 /var/www/html/DVWA/hackable/uploads/
chmod 666 config/config.inc.php

# 重启服务
systemctl restart apache2
systemctl restart mariadb

# 访问DVWA
# 浏览器访问 http://localhost/DVWA
```

**3. DVWA安全级别设置**

```
DVWA安全级别：

Low（低）：
- 无任何防护
- 所有漏洞可直接利用
- 适合初学者

Medium（中）：
- 有简单防护
- 可学习绕过技术
- 适合进阶学习

High（高）：
- 有较完整防护
- 需要更深入的绕过
- 适合深入学习

Impossible（不可能）：
- 完整防护代码示例
- 无漏洞可利用
- 用来对比学习
```
