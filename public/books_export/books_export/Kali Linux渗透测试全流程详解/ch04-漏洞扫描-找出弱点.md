## 4.2 漏洞扫描概述

先讲几个基本概念。

### 4.2.1 什么是漏洞扫描？

什么是漏洞扫描？
官方一点说：
漏洞扫描是指基于漏洞数据库，
通过扫描等手段，
对指定的远程或者本地计算机系统的
安全脆弱性进行检测，
发现可利用漏洞的一种安全检测行为。

用人话说就是：
**拿着一个漏洞库，
去目标系统上比对，
看看有没有已知的漏洞。**

就像体检一样，
拿着一个检查表，
一项一项查，
看看你有没有什么毛病。

### 4.2.2 漏洞扫描的分类

漏洞扫描有很多种分类方式，
我给你讲几种常见的。

```
按扫描对象分：

  1. 主机漏洞扫描
     • 扫操作系统、系统服务的漏洞
     • 比如Windows漏洞、Linux漏洞
     • 各种服务的漏洞（SSH、FTP、SMB...）
     • 代表工具：Nessus、OpenVAS、Nexpose

  2. Web应用漏洞扫描
     • 专门扫Web应用的漏洞
     • 比如SQL注入、XSS、CSRF、文件上传...
     • 代表工具：AWVS、AppScan、Xray、OWASP ZAP

  3. 数据库漏洞扫描
     • 专门扫数据库的
     • 比如MySQL、Oracle、SQL Server的漏洞
     • 弱口令、配置错误、权限问题...

  4. 其他
     • 无线网络漏洞扫描
     • 物联网设备漏洞扫描
     • 云安全扫描
     • ......
```

```
按扫描方式分：

  1. 主动扫描
     • 主动向目标发包，探测漏洞
     • 大部分扫描器都是这种
     • 优点：准确、信息全
     • 缺点：动静大，容易被发现

  2. 被动扫描
     • 不主动发包，通过监听流量来分析
     • 比如你代理抓包，它在旁边分析
     • 优点：隐蔽，不会触发告警
     • 缺点：只能看到你访问过的页面
```

实际工作中，
我们最常用的是两种：
**主机漏洞扫描**和**Web漏洞扫描**。
一个打系统和服务，
一个打Web应用。
这两个是基本功，
一定要会用。

### 4.2.3 漏洞扫描能干嘛？不能干嘛？

漏洞扫描器很强大，
但它不是万能的。
我们来客观地看一看。

```
漏洞扫描器能干嘛：
  ✓ 快速发现已知漏洞
  ✓ 覆盖范围广，能扫很多地方
  ✓ 有详细的报告和修复建议
  ✓ 提高效率，节省人力
  ✓ 适合做大面积的普查

漏洞扫描器不能干嘛：
  ✗ 发现0day漏洞（人家公开的才知道）
  ✗ 发现逻辑漏洞（业务逻辑问题它不懂）
  ✗ 完全准确（会有误报和漏报）
  ✗ 代替人工（最终还是要人去验证和利用）
  ✗ 利用漏洞（大部分扫描器只检测不利用）
```

简单说，
漏洞扫描器是个好帮手，
能帮你干很多体力活，
但核心的判断和利用，
还是要靠人。

别神化它，
也别小瞧它，
把它当成你工具箱里的一把锤子，
该用的时候用，
但别什么都指望它。

## 4.4 漏洞扫描的一般流程

讲完工具，
讲讲漏洞扫描的一般流程。
不是说拿个扫描器点开扫就完事了，
要有章法。

```
漏洞扫描的一般流程：

  1. 确定范围
     • 扫哪些IP？哪些端口？哪些URL？
     • 哪些能扫，哪些不能扫？
     • 先把范围定清楚，别瞎扫

  2. 选择工具
     • 根据目标类型选工具
     • 扫主机用Nessus/OpenVAS
     • 扫Web用AWVS/Xray/ZAP

  3. 配置扫描
     • 配置扫描策略
     • 扫哪些漏洞类型？
     • 速度多快？
     • 要不要登录扫描？
     • 等等

  4. 开始扫描
     • 启动扫描，等它跑完
     • 这个过程可能几分钟到几小时不等
     • 可以去喝杯茶

  5. 分析结果
     • 扫完了，看结果
     • 哪些是高危？哪些中危低危？
     • 哪些可能是误报？

  6. 漏洞验证
     • 关键的一步！
     • 人工去验证漏洞是不是真的存在
     • 能不能利用？
     • 危害有多大？
     • 去伪存真

  7. 整理报告
     • 把确认的漏洞整理出来
     • 写清楚漏洞详情、风险、修复建议
     • 形成最终的报告
```

七步走，
其中**漏洞验证**是最关键的。
没有经过验证的扫描结果，
是不能直接写到报告里的。
不然会像故事里的小李一样，
闹笑话。

## 4.5 扫描结果怎么看？

扫描完了，
会出来一大堆结果。
怎么看？
怎么分析？

### 4.5.1 漏洞分级

一般扫描器会把漏洞分成几个等级：
- **高危（High/Critical）**：很严重，能直接getshell或者拖库的那种
- **中危（Medium）**：有一定危害，但没那么严重
- **低危（Low）**：危害比较小，或者很难利用
- **信息泄露（Informational）**：就是泄露了一些信息，不算漏洞

当然，
这个分级只是扫描器的判断，
不一定准确。
同一个漏洞，
在不同的环境下，
危害可能天差地别。
还是要人工判断。

### 4.5.2 怎么看待误报和漏报？

扫描器有两个常见的问题：
**误报**和**漏报**。

```
误报（False Positive）：
  • 扫描器说有漏洞，实际上没有
  • 就是"报错了"
  • 误报太多会浪费你的时间
  • 也会让报告显得不专业

漏报（False Negative）：
  • 实际上有漏洞，扫描器没扫出来
  • 就是"漏掉了"
  • 这个更危险，因为你以为没问题，实际上有
```

为什么会有误报漏报？
- 漏洞本身就很复杂，很难100%准确判断
- 不同的环境、不同的配置，结果不一样
- 扫描器的规则是人写的，不可能完美
- 有些漏洞本来就很难自动化检测

所以，
**不能不信扫描器，
也不能全信扫描器。**
它给你提供线索，
你去验证、去判断。
这才是正确的态度。

### 4.5.3 怎么验证漏洞？

怎么验证漏洞是真是假？
这个是硬功夫，
得靠经验，
也得靠对漏洞原理的理解。

我给你讲几个通用的思路：

```
验证漏洞的一般思路：

  1. 看原理
     • 这个漏洞到底是什么原理？
     • 产生的原因是什么？
     • 成功利用应该是什么样的？
     • 先搞懂原理，你才知道怎么验证

  2. 手工复现
     • 按照漏洞的原理，手工去试
     • 构造Payload，发请求
     • 看看有没有预期的结果
     • 能复现，就是真的
     • 复现不了，可能是误报

  3. 看返回
     • 看返回的页面、响应码、响应内容
     • 跟正常情况有没有区别？
     • 区别是不是漏洞导致的？
     • 别看到不一样就说是漏洞
       可能是正常的业务差异

  4. 评估危害
     • 就算真的有漏洞，
       也要评估一下实际危害有多大
     • 是不是需要特定条件？
     • 是不是很难利用？
     • 不能一概而论都算高危

  5. 查资料
     • 遇到不确定的，去查资料
     • 看这个漏洞的详细说明
     • 看别人的验证方法
     • 别自己瞎猜
```

漏洞验证能力，
是渗透测试工程师的核心能力之一。
这个能力不是一天两天能练成的，
需要你一个个漏洞去研究，
一次次去验证，
经验多了，
自然就准了。

新手一开始可能分不清真假，
没关系，
多做多练，
慢慢就有感觉了。

## 4.6 提高扫描效果的几个技巧

最后，
给你几个提高漏洞扫描效果的技巧。

### 技巧一：先做好信息收集

扫描不是上来就扫，
信息收集做得好，
扫描效果才好。
- 知道目标有哪些端口、哪些服务
- 知道目标是什么系统、什么版本
- 知道是什么Web应用、什么框架
- 有针对性地去扫，
  比盲目的全扫效果好得多

### 技巧二：选对扫描策略

不是所有扫描都要开全功能。
- 快速摸底的时候，
  就用快速扫描策略
- 要深入检测的时候，
  再开全扫、深度扫描
- 不同的场景用不同的策略
- 不然又慢又不准

### 技巧三：登录状态扫描

很多网站，
不登录只能看到首页，
登录了才能看到更多功能。
- 如果有测试账号，
  一定要配置登录扫描
- 不然好多页面扫不到，
  漏报会很多
- 可以用Cookie，
  也可以用录制登录的方式

### 技巧四：结合手工测试

扫描器是死的，
人是活的。
- 扫描器扫完了，
  一定要手工再测一测
- 尤其是业务逻辑、权限这些
  扫描器根本搞不定
- 手工 + 工具，
  效果才最好

### 技巧五：多个工具交叉验证

一个扫描器可能不准，
那我用两个、三个呢？
- 多个工具一起扫
- 都扫出来的，大概率是真的
- 只有一个扫出来的，可能是误报
- 交叉验证，准确率会高很多

当然，
也不是工具越多越好，
两三个主流的就够了。

## 4.7 给新手的几个提醒

最后，给新手几个提醒，
都是过来人的经验。

### 提醒一：不要迷信扫描器

开头就说了，
再强调一遍。
扫描器是工具，不是神。
它能帮你提高效率，
但不能代替你的思考。
永远不要把扫描器的结果直接当最终结果。
一定要验证。

### 提醒二：不要只看数量，要看质量

很多新手觉得，
扫出来的漏洞越多越厉害。
不对。
- 扫出100个误报，
  不如1个确认的高危有价值。
- 质量比数量重要。
- 把每个漏洞搞透，
  比扫一堆假漏洞强多了。

### 提醒三：理解漏洞原理比会用工具重要

会用十个扫描器，
不如把一个漏洞的原理搞透。
- 原理懂了，
  你才能自己写POC、自己写规则
- 原理懂了，
  你才能判断是不是误报
- 原理懂了，
  你才能真正利用漏洞

工具只是载体，
原理才是根本。

### 提醒四：合法使用，别乱扫

最后再强调一遍法律问题。
**漏洞扫描也是攻击行为，
没授权不能乱扫。**

- 你自己的机器，随便扫
- 授权了的目标，按授权范围扫
- 别人家的系统，别手贱去扫
- 哪怕你只是"看看"，也不行

别拿自己的前途开玩笑。

## 4.8 本章总结

这一章我们讲了漏洞扫描，总结一下：

**漏洞扫描是什么：**
- 用工具检测目标有没有已知漏洞
- 是渗透测试中快速发现问题的方法

**漏洞扫描的分类：**
- 主机漏洞扫描：扫系统和服务（Nessus、OpenVAS）
- Web漏洞扫描：扫Web应用（AWVS、Xray、ZAP）
- 主动扫描 vs 被动扫描

**常见工具：**
- 主机类：Nessus、OpenVAS、Nmap脚本
- Web类：AWVS、AppScan、OWASP ZAP、Xray、Nikto

**扫描流程：**
确定范围 → 选工具 → 配置 → 扫描 → 分析结果 → **漏洞验证** → 整理报告

**正确看待扫描结果：**
- 有分级：高危、中危、低危、信息
- 有误报和漏报，很正常
- 关键是人工验证，去伪存真
- 不能不信，也不能全信

**提高效果的技巧：**
1. 先做好信息收集
2. 选对扫描策略
3. 登录状态扫描
4. 结合手工测试
5. 多个工具交叉验证

最后送你一句话：

> **扫描器是放大镜，
> 帮你找到问题，
> 但判断问题、解决问题，
> 还是要靠人。**
