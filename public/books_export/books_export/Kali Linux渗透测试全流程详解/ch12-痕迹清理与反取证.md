## 12.1 先讲一个故事

### 12.1.1 "一只U盘引发的血案"

我听过一个真实的案例。
有个攻击者，
打进了一家公司的内网，
在里面待了好几个月，
偷了很多机密数据。
一直没被发现。

最后他是怎么暴露的呢？
说出来你可能不信——
因为一个U盘。

他有一次操作的时候，
往服务器里上传了一个工具，
工具的元数据里（文件属性），
有他的电脑名、用户名。
他没注意，
用完了也没删干净。

后来管理员排查的时候，
发现了这个可疑文件，
看了看文件属性，
看到了那个用户名和电脑名。
好巧不巧，
那个攻击者的用户名就是他自己的真名拼音，
电脑名也很有辨识度。
顺着一查，
就查到了。

你看，
辛辛苦苦几个月，
栽在一个小小的文件元数据上。
就因为没把痕迹清理干净。

---

当然，
这是个真实的犯罪案例，
那个人最后被抓了。
举这个例子不是教你怎么干坏事，
而是告诉你：
**痕迹清理非常重要。
一个小细节没注意，
就可能全盘皆输。**

（还是那句话，
这些技术用在授权测试和合法的学习研究上。
干违法的事，迟早被抓。）

## 12.2 你留下了哪些痕迹？

在讲怎么清理之前，
先想想——
你入侵一台机器，
都会留下哪些痕迹？

很多新手以为，
删了上传的文件就完了。
太天真了。
你留下的痕迹多着呢。

我给你数数：

```
常见的痕迹：

  1. 日志
     • 系统日志
     • 安全日志（登录、权限变更...）
     • 应用程序日志
     • Web访问日志
     • 防火墙日志
     • 数据库日志
     • 中间件日志
     • ......各种日志

  2. 文件
     • 你上传的工具、木马、后门
     • 你创建的文件、修改的文件
     • 你下载的数据
     • 临时文件
     • 回收站里的文件

  3. 登录记录
     • 登录了哪些机器
     • 什么时候登录的
     • 从哪登录的
     • 远程桌面的连接记录
     • SSH登录记录
     • 等等

  4. 命令历史
     • 你执行过的命令
     • （cmd的历史、PowerShell的历史、bash的history...）

  5. 进程/服务/计划任务
     • 你创建的服务
     • 你添加的计划任务
     • 你启动的进程
     • 你安装的驱动

  6. 注册表修改
     • 你改的注册表项
     • 你添加的注册表项
     • （Windows上尤其多）

  7. 网络连接记录
     • 防火墙、网关、流量设备上的日志
     • 这个你在目标机器上清不掉

  8. 其他
     • 最近打开的文件记录
     • 剪切板内容
     • 内存里的东西
     • 等等
```

你看，
痕迹无处不在。
想完全清理干净，
几乎是不可能的。
尤其是网络设备上的日志，
你在目标机器上根本碰不到。

但是，
**清理得越干净，
被发现的概率就越低。**
能清多少清多少。

## 12.3 Windows 痕迹清理

先讲Windows的痕迹清理。

### 12.3.1 清理日志

最基本的——
**清日志。**

Windows有三大日志：
- 系统日志（System）
- 安全日志（Security）
- 应用程序日志（Application）

还有其他各种应用的日志。

怎么清？

```cmd
:: 方法一：用wevtutil（推荐）
:: 清系统日志
wevtutil cl System
:: 清安全日志
wevtutil cl Security
:: 清应用程序日志
wevtutil cl Application
:: cl 是 clear 的意思

:: 方法二：用PowerShell
Clear-EventLog -LogName System,Security,Application

:: 方法三：Meterpreter里直接清
clearev
:: 一条命令，三大日志全清
```

简单吧？
一条命令的事。
Meterpreter里更简单，
`clearev` 就搞定了。

但是注意，
**清日志本身也会留下日志。**
比如安全日志里会记录"日志被清除"这个事件。
管理员一看日志被清了，
肯定知道有人来过。
所以清日志是把双刃剑——
你把证据毁了，
但也告诉人家"我来过了"。

高级一点的做法，
是**只删掉跟你相关的日志条目**，
而不是把整个日志都清了。
这样管理员不容易发现。
不过这个操作起来比较麻烦，
一般的工具也没这个功能。
新手知道有这么回事就行。

### 12.3.2 清理文件

第二，
**清理你上传的文件、创建的文件。**

- 上传的工具、木马，用完删掉
- 收集的资料、导出的数据，带走之后删掉
- 临时文件，清掉
- 回收站也清空

```cmd
:: 删除文件
del /f /q 文件名

:: 删除目录
rd /s /q 目录名

:: 清空回收站
rd /s /q C:\$Recycle.Bin
:: 或者其他盘的
```

注意：
**普通删除是可以恢复的。**
只要没被覆盖，
用数据恢复软件就能恢复。
所以如果不想被恢复，
要用**安全删除**的工具，
反复擦写，让它恢复不了。
比如Cipher、SDelete之类的。

不过一般情况下，
普通删除也就够了，
很少有人会去做数据恢复。
真要碰到做取证的，
那普通删除肯定不够看。

### 12.3.3 清理命令历史

第三，
**清理命令历史。**

你执行过的命令，
很多地方都会有记录。

```cmd
:: cmd的命令历史
:: 这个一般关掉窗口就没了
:: 但有些情况会保存

:: PowerShell的命令历史
:: PowerShell默认会保存历史记录
:: 路径一般在：
:: %APPDATA%\Microsoft\Windows\PowerShell\PSReadLine\ConsoleHost_history.txt
:: 要清的话把这个文件删了
del %APPDATA%\Microsoft\Windows\PowerShell\PSReadLine\ConsoleHost_history.txt

:: 或者执行命令清空
[Microsoft.PowerShell.PSConsoleReadLine]::ClearHistory()
Remove-Item (Get-PSReadlineOption).HistorySavePath
```

PowerShell的历史很多人不知道，
经常忘清。
要注意。

### 12.3.4 清理远程桌面 / 运行历史

第四，
**各种使用痕迹。**

比如远程桌面连接过的记录，
运行里打开过的程序，
最近访问的文件...
这些都是痕迹。

```cmd
:: 远程桌面连接历史
:: 在注册表里
:: HKEY_CURRENT_USER\Software\Microsoft\Terminal Server Client\Default
:: 把里面的项删掉
reg delete "HKEY_CURRENT_USER\Software\Microsoft\Terminal Server Client\Default" /va /f

:: 运行（Win+R）的历史
:: HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\RunMRU
reg delete "HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\RunMRU" /va /f

:: 最近打开的文件
:: 用户目录下的Recent
del /f /q %APPDATA%\Microsoft\Windows\Recent\*.*

:: 还有开始菜单的最近程序之类的
:: 也可以清一清
```

这些都是小细节，
但往往细节决定成败。

### 12.3.5 撤销你做的修改

第五，
**把你改过的东西改回去。**

- 你改的注册表，改回去
- 你加的用户，删掉
- 你创建的服务，删掉
- 你加的计划任务，删掉
- 你改的系统配置，恢复原样
- 替换的系统文件，换回来

总之，
**你动过的东西，
尽量恢复到你来之前的样子。**
别留下各种尾巴。

要做到这点，
你操作之前最好记一下，
原来是什么样的。
不然改完了忘了原来是什么，
想恢复都恢复不了。

### 12.3.6 Meterpreter 的 clearev

最后提一下Meterpreter的 `clearev` 命令。
这个命令会：
- 清空系统日志
- 清空安全日志
- 清空应用程序日志

一键清日志，
非常方便。
但也就清个日志，
其他的痕迹还是要自己清。

## 12.4 Linux 痕迹清理

讲完Windows的，
再讲讲Linux的。
Linux的痕迹清理相对简单一点。

### 12.4.1 清理日志

Linux的日志一般都在 `/var/log/` 下面。

常见的日志文件：
```
/var/log/messages     系统日志
/var/log/syslog       系统日志（Debian/Ubuntu系）
/var/log/auth.log     认证日志（SSH登录等）
/var/log/secure       安全日志（CentOS/RedHat系）
/var/log/lastlog      最近登录记录
/var/log/wtmp         登录记录（last命令读的就是这个）
/var/log/btmp         失败登录记录
/var/log/faillog      失败登录
/var/log/cron         定时任务日志
/var/log/apache2/     Apache日志
/var/log/nginx/       Nginx日志
......还有很多
```

怎么清？
最简单的，
直接把日志文件清空。

```bash
# 清空一个文件（比删掉好，因为删掉的话服务可能不会自动重建）
> /var/log/auth.log
> /var/log/syslog
> /var/log/lastlog

# 或者用 truncate
truncate -s 0 /var/log/auth.log

# 多个一起清
for log in /var/log/auth.log /var/log/syslog /var/log/messages; do
    > $log
done
```

注意，
**别把日志文件删了**，
清空内容就行。
因为有些服务写日志的时候，
如果文件被删了，
它不会自动重建，
日志就写不进去了。
管理员一看日志没了，
一下就发现了。
清空内容就好。

### 12.4.2 清理命令历史（history）

Linux里最容易忘清的就是——
**命令历史（history）。**

你执行过的所有命令，
默认都会存在用户目录下的 `.bash_history` 文件里。
管理员一翻history，
你干了啥一目了然。

所以走之前一定要清history！

怎么清？

```bash
# 方法一：清空history文件
> ~/.bash_history

# 方法二：history -c 清空当前历史，再写回去
history -c
history -w

# 方法三：rm直接删
rm -f ~/.bash_history

# 还可以在操作之前就不让它记录
# 比如：
unset HISTORY HISTFILE HISTSAVE HISTZONE HISTLOG
export HISTSIZE=0
export HISTFILESIZE=0
# 这样你执行的命令就不会被记录了
```

强烈建议操作之前先把history关了，
省得最后忘清。
这个很多人容易忘，
一定要注意。

### 12.4.3 清理登录记录

Linux里有几个命令可以看登录记录：
```
last       看最近的登录记录（读的是/var/log/wtmp）
lastb      看失败的登录记录（读的是/var/log/btmp）
lastlog    看所有用户最后一次登录（读的是/var/log/lastlog）
who / w    看当前在线用户
```

这些记录要不要清？
如果你登录过这台机器，
那这些记录里就会有你的痕迹。
要清的话也能清，
但清了之后管理员可能会发现异常。

```bash
# wtmp 和 btmp
# 直接清空文件
> /var/log/wtmp
> /var/log/btmp

# lastlog
> /var/log/lastlog

# 当前登录的utmp
# 就是w和who看的那个
# 文件在 /var/run/utmp
# 一般你退出登录就没了
```

还是那句话，
清日志等于告诉人家你来过。
清还是不清，
要看情况。
有时候全清了反而可疑。

### 12.4.4 清理文件

跟Windows一样，
你上传的工具、创建的文件，
走之前都删掉。

```bash
# 删除文件
rm -f 文件名

# 删除目录
rm -rf 目录名

# 安全删除（反复擦写，防恢复）
# 用 shred 命令
shred -u 文件名
# -u 是删完之后把文件也删了
```

shred比普通rm安全，
反复覆盖，数据恢复不了。
但需要时间，
大文件的话比较慢。

### 12.4.5 撤销修改

同样，
你改过的东西要改回去。
- 加的用户删掉
- 加的cron任务删掉
- 加的服务删掉
- 改的配置文件改回去
- SSH公钥删了
- 等等

恢复原样。

## 12.5 反取证概述

最后简单聊聊**反取证（Anti-Forensics）**。

什么是反取证？
简单说就是：
**对抗取证分析，
让调查者难以取证、
或者取不到有效证据。**

痕迹清理是反取证的一部分，
但反取证不止痕迹清理。

### 12.5.1 反取证的几个方向

反取证大致有几个方向：

```
  1. 数据销毁
     • 就是我们前面讲的痕迹清理
     • 销毁证据、删除日志、擦除文件
     • 让调查者找不到证据

  2. 数据加密
     • 把重要的数据、工具、后门都加密
     • 就算被拿到了也看不了
     • 比如加密的压缩包、加密的Payload

  3. 数据混淆 / 伪装
     • 把文件伪装成正常文件
     • 把代码混淆得看不懂
     • 假旗行动（故意留下假证据，误导调查方向）

  4. 无文件 / 内存操作
     • 尽量不在硬盘上留东西
     • 都在内存里操作
     • 重启就没了
     • 取证的时候找不到

  5. 反调试 / 反分析
     • 让你的程序难以被调试、难以被分析
     • 加壳、混淆、虚拟机保护...
     • 增加分析的难度

  6. 其他
     • 时间戳伪造
     • 身份隐藏
     • 跳板/代理
     • 等等
```

反取证是个很深的话题，
这里只是给你个概念，
知道有这么回事就行。

### 12.5.2 痕迹清理的局限性

最后必须说一句：
**痕迹清理不是万能的。**

```
你清不掉的东西：
  • 网络设备上的日志（防火墙、路由器、网关、IDS/IPS...）
  • 集中式日志系统（如果日志都传到日志服务器了，
    你在目标机器上清了也没用）
  • 监控软件/EDR的数据
  • 流量记录
  • 云平台的操作日志
  • 物理监控（摄像头什么的）
  • ......
```

现在很多企业都有集中日志系统，
日志实时传到专门的日志服务器上。
你在目标机器上清日志，
根本没用，
人家日志服务器上都有备份。

还有EDR、NDR这些安全设备，
你做的很多操作都会被记录下来。
想完全不被发现，
越来越难了。

但是，
**该清还是得清。**
能清多少清多少，
至少增加调查的难度。

## 12.6 痕迹清理的原则

最后给几个原则。

### 原则一：尽量少留痕迹，而不是事后清理

最好的痕迹清理，
是**一开始就少留痕迹**。
而不是做完了再去擦。
- 操作之前先关history
- 尽量少创建文件，多用内存操作
- 尽量用系统自带的工具（"自带的不算文件"）
- 能用现有工具就不上传新工具
- 少改配置，少动系统

从源头上减少痕迹，
比事后清理有效得多。

### 原则二：注意时间戳

文件的时间戳（创建时间、修改时间、访问时间）
也是痕迹。
你改了某个文件，
它的修改时间就会变成你改的那个时间，
很容易看出来。

高级一点的做法，
会把修改过的文件的时间戳改回去，
改成跟原来差不多的时间。
这样不容易被发现。

### 原则三：别清太干净，反而可疑

有时候，
日志全清了、
所有痕迹都没了，
反而更可疑。
管理员一看就知道：
"哦，被入侵了，日志都被清了。"

有时候，
**只清理关键的痕迹，
保留大部分正常的东西，
反而更不容易被发现。**

当然，
这个度怎么把握，
就看经验了。

### 原则四：合法使用，别干坏事

最后还是那句话，
**这些技术只能用在合法的授权测试中。**
- 渗透测试项目，测试完了要跟客户说明
- 做了什么修改、留了什么后门，都要列清楚
- 该恢复的恢复，该清理的清理
- 别给客户留隐患

用技术做正确的事。

## 12.7 本章总结

这一章我们讲了痕迹清理与反取证，
内容相对轻松一点。
总结一下：

**你留下的痕迹：**
- 各种日志（系统、安全、应用、Web、网络...）
- 文件（上传的、创建的、修改的）
- 登录记录
- 命令历史
- 进程、服务、计划任务
- 注册表修改
- 网络连接记录
- 等等...

**Windows 痕迹清理：**
- 清日志（wevtutil / PowerShell / clearev）
- 删文件
- 清命令历史（PowerShell历史别忘）
- 清远程桌面、运行、最近文件等使用痕迹
- 撤销所有修改（注册表、用户、服务、任务...）

**Linux 痕迹清理：**
- 清日志（/var/log/ 下的各种日志，清空别删除）
- 清history（很重要！很多人忘）
- 清登录记录（wtmp、btmp、lastlog...）
- 删文件（shred安全删除）
- 撤销修改（用户、cron、服务、配置、SSH公钥...）

**反取证：**
- 数据销毁、加密、混淆、无文件、反调试...
- 痕迹清理只是反取证的一部分

**痕迹清理的局限性：**
- 网络设备日志、集中日志系统清不掉
- 监控、EDR的数据清不掉
- 完全不被发现很难，只能增加难度

**原则：**
- 尽量少留痕迹，而不是事后清理
- 注意时间戳等细节
- 别清太干净，反而可疑
- 合法使用

最后送你一句话：

> **高手的痕迹，
> 不是完全没有，
> 而是藏在千千万万正常的痕迹里，
> 让人发现不了。**
