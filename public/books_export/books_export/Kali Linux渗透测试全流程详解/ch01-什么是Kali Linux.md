## 1.2 法律与道德红线

在正式讲技术之前，
我们先把最重要的事说清楚：
**法律和道德红线。**

这不是套话，
这是真的能决定你前途的事。
一定要认真看。

### 1.2.1 法律红线

哪些事绝对不能干？

```
绝对不能碰的红线：

  1. 未经授权，入侵别人的系统
     • 不管是政府、企业、还是个人的电脑
     • 没经过人家同意，你就别碰
     • 哪怕你只是"看看"，也是违法的
     • 更别说拿数据、搞破坏了

  2. 未经授权，测试别人的系统
     • 别觉得"我只是测测有没有漏洞，不搞破坏"
     • 不搞破坏也不行
     • 没授权就是不行
     • 测漏洞也得人家同意了才能测

  3. 偷数据、卖数据
     • 这个更严重
     • 偷数据、卖数据、利用数据搞诈骗
     • 都是重罪
     • 千万别碰

  4. 搞破坏
     • 删数据、挂网页、勒索、搞瘫痪
     • 这些都是严重的犯罪
     • 后果很严重

  5. 做黑产
     • 卖木马、卖漏洞、帮人攻击
     • 这些都是黑产
     • 进去了就别想轻易出来
```

中国有几部法律你得知道：
- **《网络安全法》**
- **《数据安全法》**
- **《个人信息保护法》**
- **《刑法》第285条、286条、287条**

这些法律都对网络犯罪有明确的规定，
轻则拘留罚款，
重则判刑坐牢。
别拿自己的前途开玩笑。

### 1.2.2 什么是合法的？

那什么是合法的？
我们学了渗透测试，
能用来干啥？

```
合法的事情：

  1. 学习和研究
     • 自己搭环境学习
     • 在自己的机器上做实验
     • 研究技术原理
     • 这些都是没问题的

  2. 授权的渗透测试
     • 公司请你来做渗透测试
     • 签了合同、有书面授权
     • 在授权范围内做测试
     • 这是合法的，也是正当职业

  3. 漏洞挖掘（SRC）
     • 很多公司有SRC（安全应急响应中心）
     • 鼓励白帽子找漏洞
     • 找到漏洞还能拿奖金
     • 这也是合法的，而且还能赚钱

  4. CTF比赛
     • Capture The Flag，夺旗赛
     • 合法的网络安全技术比赛
     • 专门给你练手的环境
     • 放心打，没问题

  5. 自己的系统
     • 拿自己的电脑、自己的服务器做测试
     • 随便你怎么折腾
     • 只要不影响别人就行
```

看到了吧，
合法的、能做的事情多着呢。
学好了，
找个正经的安全工作，
或者挖漏洞赚奖金，
不比干违法的事强？
光明正大的赚钱，
不香吗？

### 1.2.3 道德底线

除了法律，
还有道德底线。

```
道德底线：

  1. 不做损人利己的事

     • 偷数据、卖数据、搞破坏
```

## 1.3 Kali Linux是什么？

讲完了法律道德，
我们正式开始。

Kali Linux是什么？
简单说，
**Kali Linux是一个专门做渗透测试和安全研究的Linux发行版。**

什么叫发行版？
就是基于Linux内核做的操作系统，
比如Ubuntu、CentOS、Debian，
这些都是Linux发行版。
Kali也是其中一个，
只不过它是专门为安全行业做的。

### 1.3.1 Kali的前世今生

Kali不是凭空冒出来的，
它有个"前身"，
叫**BackTrack**（简称BT）。

BackTrack当年可是赫赫有名，
是最流行的渗透测试系统。
很多老黑客都是用着BT长大的。

后来BackTrack不更新了，
原来的团队又做了一个新的，
就是Kali Linux。
2013年发布第一个版本，
一直更新到现在。

```
发展历史：
  • 2006年：BackTrack 1发布（基于Slackware）
  • 2010年：BackTrack 5（基于Ubuntu）
  • 2013年：Kali Linux 1.0发布（基于Debian）
  • 2019年：Kali 2019.4开始，有了默认的Xfce桌面
  • 至今：持续更新，越来越强大
```

现在的Kali Linux，
由Offensive Security公司维护。
这个公司也是做安全培训的，
很有名的OSCP认证就是他们家的。

### 1.3.2 Kali里面有什么？

Kali最厉害的地方是什么？
就是它**预装了几百个安全工具**，
不用你一个个去找、去装，
装完系统直接就能用。

有多少个工具？
**600多个。**
涵盖了渗透测试的方方面面。

```
Kali里的工具分类（部分）：

  1. 信息收集
     • Nmap、Whois、Dig、theHarvester...
     • 收集目标的各种信息

  2. 漏洞评估
     • Nessus、OpenVAS、Nikto...
     • 扫描漏洞、评估风险

  3. Web应用程序
     • Burp Suite、SQLMap、OWASP ZAP...
     • 专门打Web应用的

  4. 数据库评估
     • sqlmap、SQLNinja...
     • 数据库安全测试

  5. 密码攻击
     • Hydra、Hashcat、John the Ripper...
     • 各种密码破解

  6. 无线攻击
     • Aircrack-ng、Kismet、Fern...
     • 无线网络渗透

  7. 逆向工程
     • IDA、Ghidra、radare2...
     • 逆向分析、破解

  8. 取证工具
     • Autopsy、Volatility、Foremost...
     • 数字取证、事件响应

  9. 社会工程学
     • Social Engineering Toolkit（SET）
     • 社工攻击

  10. 漏洞利用
      • Metasploit、searchsploit...
      • 漏洞利用框架

  11. 后渗透
      • Mimikatz、Empire、PowerSploit...
      • 打进来之后的操作

  12. 报告工具
      • Dradis、KeepNote、pipal...
      • 写报告、整理结果

  ......还有很多很多
```

看到了吧，
从信息收集到漏洞扫描，
从Web渗透到密码破解，
从无线攻击到逆向工程，
从社工到后渗透，
应有尽有。

你不用再到处找工具了，
装个Kali，
全都给你准备好了。
这就是为什么大家都用Kali做渗透测试。

不用折腾环境，
装完就能用，
专心学技术就行。

## 1.4 什么是渗透测试？

讲完了Kali，
再讲讲什么是渗透测试。
你可能经常听到这个词，
但不一定清楚到底是啥。

### 1.4.1 渗透测试的定义

什么是渗透测试？
官方一点的定义：

> 渗透测试（Penetration Testing），
> 是通过模拟恶意攻击者的攻击方法，
> 对计算机系统、网络或应用程序
> 进行安全评估的过程。

用人话说就是：
**经过授权，
模拟黑客的手法，
去打客户的系统，
看看能不能打进去，
能打到什么程度，
然后告诉客户哪里有问题，
该怎么修。**

就这么个事。

渗透测试的目的**不是搞破坏**，
而是**发现问题、帮助客户解决问题**。
是为了让系统更安全。

### 1.4.2 渗透测试的类型

渗透测试有不同的分类方法，
根据测试者掌握的信息多少，
一般分三种：

```
三种类型：

  1. 黑盒测试
     • 测试者完全不了解系统内部结构
     • 就跟真实的黑客一样
       从外面一点点摸
     • 最接近真实攻击场景
     • 对测试者水平要求最高

  2. 白盒测试
     • 测试者完全了解系统信息
     • 源代码、架构、文档，都给你
     • 能更深入地发现问题
     • 效率也更高

  3. 灰盒测试
     • 介于黑白之间
     • 给一部分信息，
       比如普通用户的账号
     • 既有效率，又有一定的真实度
     • 实际项目中最常见
```

实际工作中，
三种都有。
很多时候是灰盒测试，
兼顾效率和深度。

### 1.4.3 渗透测试的一般流程

渗透测试是怎么个流程？
是不是拿到目标就一顿乱扫、乱打？
不是的。
正规的渗透测试，
有一套标准的流程。

```
渗透测试的一般流程：

  1. 前期交互
     • 跟客户沟通，确定测试范围
     • 签合同、签授权书
     • 定好规则：哪些能测，哪些不能测
     • 时间、方式、联系人...
     • 这一步很重要，没授权不能动手

  2. 信息收集（侦察）
     • 收集目标的各种信息
     • 域名、IP、端口、服务、员工信息...
     • 信息越多，后面越容易打

  3. 漏洞分析
     • 找漏洞
     • 扫端口、扫服务、扫漏洞
     • 研究哪些地方可以打

  4. 漏洞利用
     • 真刀真枪地干
     • 利用漏洞打进去
     • 获取权限

  5. 后渗透
     • 打进来之后干嘛？
     • 提权、横向移动、拿更多权限
     • 偷数据、留后门...
     • 看你能深入到什么程度

  6. 痕迹清理（可选）
     • 有的客户要求测完了把痕迹清了
     • 有的不需要
     • 看约定

  7. 报告编写
     • 最后写报告
     • 发现了哪些漏洞
     • 怎么利用的
     • 风险有多大
     • 怎么修复
     • 报告是最终交付物，很重要
```

七步走，
从前期准备到最后交付，
一个完整的流程。
我们这本书，
基本上也是按这个流程来讲的。

## 1.5 学习Kali的正确姿势

最后，
聊聊怎么学习Kali、怎么学渗透测试。
给新手几个建议，
让你少走点弯路。

### 1.5.1 别做"脚本小子"

第一个建议：
**别做"脚本小子"（Script Kiddie）。**

什么是脚本小子？
就是只会用别人写好的工具、脚本，
点两下鼠标，
看着结果出来，
就觉得自己很厉害了。
但你问他原理，
他啥也不知道。
工具一失效，
就啥也不会了。

别做这样的人。
- 用工具可以，但要懂原理
- 知道这个工具是干嘛的
- 知道它背后的原理是什么
- 知道为什么能用，为什么不能用
- 工具不好用的时候，自己能想办法

原理搞懂了，
工具只是顺手的事。
原理不懂，
用再多工具也是白搭。

### 1.5.2 打好基础

第二个建议：
**打好基础。**

很多人一上来就想学怎么黑网站、怎么拿服务器，
觉得那个才酷。
基础的东西不想学，
觉得没意思。

不对。
基础不牢，地动山摇。
- 网络基础懂不懂？TCP/IP、HTTP协议...
- 操作系统懂不懂？Windows、Linux...
- 编程基础有没有？Python、Shell...
- 数据库懂不懂？SQL...

这些基础打好了，
学渗透测试才能学得深、走得远。
基础不行，
你就只能停留在用工具的层面，
永远成不了高手。

别急，
慢慢来，
基础打牢了，
后面进步会很快。

### 1.5.3 多动手，多实践

第三个建议：
**多动手，多实践。**

安全是个实战的活，
光看书、看视频没用，
得自己动手做。
- 工具要自己敲一遍
- 漏洞要自己复现一遍
- 实验要自己做一遍

看十遍，
不如自己动手做一遍。
做一遍，
你就懂了。

怎么练手？
- 自己搭靶机环境（DVWA、WebGoat、VulnHub...）
- 打CTF比赛
- 在线实验平台
- 挖SRC（有经验了再去）

多练，
练得多了，
水平自然就上去了。

### 1.5.4 保持好奇心，持续学习

第四个建议：
**保持好奇心，持续学习。**

安全这个行业，
技术更新太快了。
- 新的漏洞天天出
- 新的攻击手法天天有
- 新的工具不断涌现

你不学习，
很快就落伍了。

保持好奇心，
对新技术、新漏洞保持敏感，
一直学习，
一直进步。

这个行业，
活到老，学到老。
但也正因为这样，
才有意思，不是吗？

## 1.6 本章总结

第一章就讲到这里，
总结一下：

**法律与道德红线（最重要！）：**
- 未经授权，绝对不能碰别人的系统
- 可以做的：学习研究、授权测试、SRC漏洞挖掘、CTF比赛
- 技术是把双刃剑，用对地方才是好事

**Kali Linux是什么：**
- 专门做渗透测试和安全研究的Linux发行版
- 前身是BackTrack，现在由Offensive Security维护
- 预装了600+安全工具，开箱即用
- 信息收集、漏洞扫描、Web渗透、密码破解、无线攻击、逆向取证...应有尽有

**渗透测试是什么：**
- 经过授权，模拟攻击，发现安全问题
- 目的是帮助客户修复漏洞、提升安全
- 三种类型：黑盒、白盒、灰盒
- 七步流程：前期交互→信息收集→漏洞分析→漏洞利用→后渗透→痕迹清理→报告

**学习建议：**
1. 别做脚本小子，要懂原理
2. 打好基础（网络、系统、编程、数据库）
3. 多动手，多实践
4. 保持好奇心，持续学习

最后再强调一遍：

> **守住法律底线，
> 用好技术，
> 做对社会有用的安全从业者。**
