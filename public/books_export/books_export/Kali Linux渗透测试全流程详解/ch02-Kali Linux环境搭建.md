## 2.1 先问个问题：怎么用Kali？

在开始装之前，
先问你一个问题：
**你打算怎么用Kali？**
装在什么地方？

这个问题很重要，
不同的用法，
安装方式不一样。

我给你列几种常见的选择，
你可以根据自己的情况选。

### 2.1.1 虚拟机安装（推荐新手）

第一种，也是最推荐新手的——
**装在虚拟机里。**

什么是虚拟机？
就是用VMware、VirtualBox这些软件，
在你现有的系统（Windows/Mac）里，
虚拟出一台电脑，
然后把Kali装在这个虚拟的电脑里。

```
优点：
  ✓ 安全，不会弄坏你原来的系统
  ✓ 想删就删，想装就装，随便折腾
  ✓ 可以随时快照，搞坏了一键还原
  ✓ 跟宿主机文件共享方便
  ✓ 一台电脑可以同时跑好几个系统

缺点：
  ✗ 性能比物理机差一点
  ✗ 有些硬件（比如无线网卡）不好用
  ✗ 有些特殊场景用不了
```

**新手强烈推荐这种方式。**
反正就是学习用，
装在虚拟机里，
随便折腾，
搞坏了也不怕，
恢复快照就行。
最省心。

### 2.1.2 物理机安装（双系统）

第二种，
装在物理机上，
跟你原来的系统组成双系统。

就是电脑开机的时候，
让你选进Windows还是进Kali。

```
优点：
  ✓ 性能好，直接跑在硬件上
  ✓ 所有硬件都能用（无线网卡、显卡...）
  ✓ 适合做无线渗透、性能要求高的场景

缺点：
  ✗ 安装有点麻烦，新手容易搞坏分区
  ✗ 两个系统切换要重启，不方便
  ✗ 搞坏了修复起来麻烦
```

如果你对Linux比较熟了，
而且经常做无线渗透之类需要直接操作硬件的测试，
可以考虑双系统。
新手的话，
还是先玩虚拟机吧。

### 2.1.3 U盘Live系统

第三种，
装在U盘上，
做成Live USB。
就是插U盘就能用Kali，
拔了U盘还是你原来的系统。

```
优点：
  ✓ 便携，插到哪台电脑都能用
  ✓ 不碰本地硬盘，安全
  ✓ 应急的时候好用

缺点：
  ✗ U盘速度慢，用着不爽
  ✗ 数据不保存（默认是Live模式，重启就没了）
  ✗ 容易丢，丢了数据就没了
```

适合应急或者临时用，
日常学习不太方便。

### 2.1.4 云服务器上装

第四种，
买个云服务器，
把Kali装在云上面。

```
优点：
  ✓ 随时随地能访问
  ✓ 公网IP，做一些测试方便
  ✓ 不占本地资源

缺点：
  ✗ 要花钱
  ✗ 没有图形界面（或者很卡）
  ✗ 有些场景用不了
```

适合有特殊需求的，
新手不太推荐。

---

**总结一下：**
- 新手学习 → 虚拟机（VMware/VirtualBox）
- 需要性能、需要硬件 → 物理机/双系统
- 应急、便携 → U盘Live
- 需要公网 → 云服务器

对于绝大多数人，
尤其是新手，
**虚拟机安装是最佳选择。**

后面我们就以VMware虚拟机为例，
讲怎么装Kali。
VirtualBox大同小异，
思路是一样的。

## 2.2 下载Kali Linux

第一步，
先下载Kali的镜像文件。

去哪里下？
当然是官网，
别从乱七八糟的地方下，
小心有病毒。

官网地址：`https://www.kali.org/get-kali/`

打开官网，
你会看到好几个版本，
选哪个呢？

### 2.2.1 选哪个版本？

Kali提供了好几种下载选项，
我给你讲讲区别。

```
常见版本：

  1. Kali Linux Installer
     • 标准安装版
     • 就是一个系统安装镜像
     • 需要你一步步安装
     • 可以自定义配置
     • 适合装物理机或者想自定义的

  2. Kali Linux Live
     • Live版镜像
     • 可以直接U盘启动
     • 也可以安装
     • 适合应急、临时用

  3. Kali Linux VMware / VirtualBox / Hyper-V
     • 预装好的虚拟机镜像
     • 下载下来直接就能用
     • 不用你一步步装
     • 最省心，最推荐新手
     • 选跟你虚拟机软件对应的就行

  4. Kali Linux ARM
     • ARM架构的
     • 装在树莓派、手机这些ARM设备上的
     • 普通电脑不用管
```

**新手最简单的方式：**
直接下载预装好的虚拟机镜像，
导入就能用，
省得自己装了。
强烈推荐。

如果你想体验一下安装过程，
或者想自定义配置，
那就下载Installer版，
自己一步步装。

### 2.2.2 选32位还是64位？

当然是64位。
现在的电脑基本都是64位的，
直接下64位的就行。
32位的现在很少用了。

## 2.3 虚拟机安装（以VMware为例）

下载完了，
我们开始装。
我分两种情况讲：
- 第一种：下载预编译的虚拟机镜像，直接导入（最简单）
- 第二种：用Installer镜像，一步步安装（适合想体验安装过程的）

你可以选一种来做。

### 2.3.1 最简单：导入预编译镜像

如果你下载的是VMware版的预编译镜像，
那就太简单了。

```
步骤：

  1. 解压下载的压缩包
     • 下载下来一般是个.7z或者.zip的压缩包
     • 解压出来，得到一个文件夹
     • 里面有.vmx和.vmdk这些文件

  2. 打开VMware
     • 点"打开虚拟机"
     • 找到刚才解压出来的.vmx文件
     • 打开它

  3. 开机
     • 点"开启此虚拟机"
     • 等它启动
     • 就这么简单！

  4. 默认账号密码
     • 用户名：kali
     • 密码：kali
     • 进去之后自己改密码
```

搞定了，
就这么简单。
不用你分区、不用你配置，
下载完导入就能用。
新手首选。

### 2.3.2 体验安装：用Installer镜像一步步装

如果你下载的是Installer安装镜像，
那就需要一步步装。
我给你讲讲大概的步骤。

```
安装步骤：

  1. 新建虚拟机
     • 打开VMware，新建虚拟机
     • 选"典型"就行
     • 下一步，选择"安装程序光盘映像文件(iso)"
     • 浏览，找到你下载的Kali iso文件
     • 下一步

  2. 配置虚拟机
     • 给虚拟机起个名字
     • 选个存放位置（别放C盘，占空间）
     • 磁盘大小，建议至少60G
       （Kali工具多，占地方）
     • 选"将虚拟磁盘拆分成多个文件"
     • 下一步，完成

  3. 调整配置（可选但推荐）
     • 内存：建议至少2G，4G以上更好
     • 处理器：核心数给2个以上
     • 网络适配器：选NAT模式
       （新手用NAT最省心，自动上网）
     • 这些配置后面也能改

  4. 开始安装
     • 点"开启此虚拟机"
     • 进去之后选"Graphical install"
       （图形界面安装）
     • 选语言：中文简体
     • 选地区：中国
     • 键盘布局：汉语
     • 等它加载组件

  5. 配置系统
     • 主机名：随便起，比如kali
     • 域名：可以空着
     • 设置root密码：自己设一个，别忘了
     • 时区：上海

  6. 磁盘分区
     • 新手最简单：选"使用整个磁盘"
     • 然后一路下一步
     • 最后选"是"，写入磁盘
     • 等它分区

  7. 选择软件
     • 选择要安装的工具集
     • 默认的就行，全选也可以
     • 桌面环境选Xfce就行（默认的，轻量好用）
     • 下一步，等它装
     • 这一步比较慢，耐心等

  8. 安装完成
     • 装完了，选"是"，装GRUB引导
     • 选/dev/sda
     • 安装完成，点继续，重启
     • 就装好了！

  9. 登录
     • 用户名root，密码你刚才设的
     • 进去就可以用了
```

整个安装过程大概半小时到一小时，
看你电脑性能。
跟着提示一步步来就行，
没什么难度。

## 2.4 装完了，先配置这几件事

系统装完了，
先别急着玩工具，
先做几件事，
把系统配置得顺手一点。

### 2.4.1 改root密码（如果你用预编译镜像）

预编译的镜像默认账号是kali/kali，
而且默认不是root。
Kali早期版本默认是root，
现在改了，默认是普通用户。

但做渗透测试，
很多时候需要root权限。
怎么开root？

```
设置root密码：

  1. 打开终端
  2. 输入：sudo passwd root
  3. 输入当前用户密码（kali）
  4. 输入两次新的root密码
  5. 搞定

然后切换到root：
  su root
  输入刚才设的密码
  就变成root了
```

平时建议用普通用户，
需要的时候再切root。
当然，
你要是觉得麻烦，
直接用root登录也行，
自己的机器，随便折腾。

### 2.4.2 配置网络

装完先看看能不能上网。
一般NAT模式的话，
默认就能上网，
不用怎么配置。

如果上不了网，
看看这几个地方：
- 虚拟机网络是不是NAT模式？
- 宿主机能不能上网？
- 虚拟机里网卡有没有启用？

一般都没啥问题。

### 2.4.3 更新系统和工具

装完第一件事——
**更新系统和工具。**
因为你下载的镜像可能不是最新的，
工具都不是最新版本。

```
更新命令：

  1. 先更新软件源：
     apt update

  2. 再更新系统和软件：
     apt full-upgrade -y

  3. 等它更完，可能比较慢，耐心等
```

这一步一定要做。
渗透测试工具更新很快，
用老版本可能很多功能没有，
或者有bug。
保持最新，总是没错的。

### 2.4.4 安装VMware Tools（增强工具）

如果你是虚拟机安装，
一定要装VMware Tools（或者叫open-vm-tools）。
装了之后：
- 鼠标可以无缝切换
- 可以自适应屏幕大小
- 可以和宿主机复制粘贴文件
- 等等，方便很多

```
安装open-vm-tools：

  apt install -y open-vm-tools-desktop

  装完重启一下就好了
```

一般装完Kali默认就带了，
但如果没有的话，
手动装一下。

### 2.4.5 配置中文输入法

如果你习惯用中文，
可以装个中文输入法。

```
装fcitx输入法：

  apt install -y fcitx fcitx-googlepinyin

  装完重启一下
  然后右上角找到输入法配置
  把Google拼音加进去
  就可以用了
```

或者你喜欢用别的输入法也可以，
自己装就行。

### 2.4.6 设置快照（虚拟机用户必做！）

最后一件事，
也是最重要的一件事——
**拍个快照。**

什么是快照？
就是把当前系统的状态保存下来，
以后系统搞坏了，
直接恢复到这个状态，
跟新的一样。

```
VMware里拍快照：
  • 虚拟机 → 快照 → 拍摄快照
  • 起个名字，比如"刚装好的纯净系统"
  • 拍一下，几秒钟就好

以后搞坏了：
  • 虚拟机 → 快照 → 恢复到快照
  • 一键还原，美滋滋
```

这个功能对新手太重要了，
随便折腾，
搞坏了恢复就行，
不用怕。
一定要记得拍！

## 2.5 Kali的基本使用

环境搭好了，
简单讲讲Kali怎么用。
如果你之前用过Linux，
那你可以跳过这一节。
如果是第一次用Linux，
简单了解一下。

### 2.5.1 桌面环境

Kali默认的桌面环境是**Xfce**，
轻量、快速，
做渗透测试足够了。

桌面左边有个侧边栏，
是常用的工具快捷方式。
左上角是应用程序菜单，
所有工具都在里面，
分门别类放好了。

你可以点开看看，
感受一下有多少工具。
上一章说的600多个工具，
都在里面了。

### 2.5.2 终端是你的好朋友

做渗透测试，
**终端（命令行）是你最好的朋友。**
大部分工具都是命令行的，
用命令行操作才是常态。

别害怕命令行，
用多了就熟了，
而且比图形界面高效得多。

怎么开终端？
- 点侧边栏的终端图标
- 或者右键桌面 → 打开终端
- 或者快捷键 Ctrl+Alt+T

常用的几个基础命令，
新手先记住这几个：

```
常用基础命令：

  ls        列出当前目录的文件
  cd 目录   切换目录
  pwd       显示当前在哪
  cat 文件  查看文件内容
  vi/nano   编辑文件（新手用nano简单点）
  cp        复制文件
  mv        移动/重命名文件
  rm        删除文件（删了就没了，小心！）
  mkdir     新建文件夹
  ifconfig  看网卡信息（或者ip a）
  whoami    看当前用户是谁
  clear     清屏
  history   看历史命令
```

不用死记硬背，
用多了自然就记住了。
先有个印象就行。

### 2.5.3 工具在哪找？

这么多工具，
在哪找呢？
有几种方式：

```
找工具的方式：

  1. 从应用程序菜单找
     • 左上角点开，按分类找
     • 信息收集、漏洞分析、Web应用...
     • 分类很清楚

  2. 命令行直接输名字
     • 知道工具名字的话，
       直接在终端里输名字就行
     • 比如nmap、sqlmap、msfconsole...
     • 最快，老手都这么干

  3. 用kali-menu
     • 终端里输kali-menu
     • 可以看到所有工具的列表
     • 也能搜索

  4. 上网搜
     • 不知道用啥工具的时候
     • 搜一下"Kali 做XX用什么工具"
     • 就知道了
```

慢慢来，
用多了工具就熟了。
不用一开始就想把所有工具都搞明白，
用到哪个学哪个就行。

## 2.6 常用的几类工具

最后，
简单给你介绍几类最常用的工具，
让你有个大概印象。
后面章节会详细讲。

### 2.6.1 信息收集类

最基础也是最常用的一类。
- **Nmap**：端口扫描神器，必学
- **theHarvester**：收集邮箱、子域名
- **whois**：查域名注册信息
- **dig/nslookup**：查DNS
- **Wappalyzer**：识别网站用的技术

### 2.6.2 Web渗透类

做Web渗透用的，
也是用得最多的。
- **Burp Suite**：Web渗透神器，必学（抓包、改包、扫描...）
- **SQLMap**：SQL注入神器
- **Nikto**：Web漏洞扫描器
- **Dirbuster/Gobuster**：扫目录、扫后台
- **OWASP ZAP**：另一个Web扫描工具

### 2.6.3 漏洞利用类

找到漏洞之后利用的。
- **Metasploit**：漏洞利用框架，必学
- **Searchsploit**：查漏洞POC/EXP
- **BeEF**：浏览器攻击框架

### 2.6.4 密码攻击类

搞密码的。
- **Hydra**：在线密码破解
- **Hashcat**：哈希破解（GPU加速，巨快）
- **John the Ripper**：另一个密码破解工具
- **Crunch/cupp**：生成密码字典

### 2.6.5 无线攻击类

搞无线网络的。
- **Aircrack-ng**：无线破解套件
- **Wireshark**：抓包分析神器
- **Kismet**：无线嗅探
- **Fern WiFi Cracker**：图形化的无线破解工具

### 2.6.6 后渗透类

打进来之后用的。
- **Mimikatz**：抓密码神器（Windows）
- **PowerSploit**：PowerShell攻击框架
- **Empire**：后渗透框架

### 2.6.7 取证类

取证、应急响应用的。
- **Wireshark**：流量分析
- **Volatility**：内存取证
- **Autopsy**：磁盘取证
- **Foremost**：文件恢复

---

工具很多，
不用急，
我们一本本书慢慢讲。
先混个脸熟就行。

## 2.7 本章总结

这一章我们讲了Kali Linux环境的搭建，
总结一下：

**安装方式选择：**
- 新手推荐 → 虚拟机安装（VMware/VirtualBox）
- 需要性能/硬件 → 物理机/双系统
- 应急便携 → U盘Live
- 需要公网 → 云服务器

**下载：**
- 官网下载，推荐预编译虚拟机镜像，最省心

**安装后配置：**
1. 设置root密码
2. 确认网络通
3. 更新系统和工具（apt update && apt full-upgrade）
4. 装VMware Tools
5. 配置中文输入法（可选）
6. **一定要拍快照！**

**基础使用：**
- 桌面环境Xfce，简单易用
- 终端是核心，多练命令行
- 工具按分类放在菜单里，也可以直接输命令

常用工具分类：
信息收集、Web渗透、漏洞利用、密码攻击、无线攻击、后渗透、取证...
