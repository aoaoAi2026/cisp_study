# 第十一章 权限维持 —— 怎么留住你的Shell

> 第11章 | 预计学习时间：5小时 | 难度：⭐⭐⭐⭐

---

## 📖 写在前面

上一章我们讲了横向移动，
打进一台机器，
然后一台一台打过去，
最终拿下整个内网。

但是有个问题：
**你打进去了，
但万一管理员重启了机器，
或者杀软把你的木马删了，
或者漏洞被补了，
你不就掉了吗？**

辛辛苦苦打进去的权限，
总不能说没就没了。
怎么办？

这就要讲到——
**权限维持（Persistence），
也叫持久化。**

什么是权限维持？
简单说就是：
**在目标系统上留下后门，
就算重启、就算漏洞被补了，
你依然能再进来。**

用人话说就是：
**留个后门，下次还能来。**

权限维持是后渗透里非常重要的一环。
毕竟，
打进去不容易，
总不能进去一次就再也进不去了。
留好后门，
想什么时候来什么时候来。

这一章我们就来讲讲常见的权限维持手法。
Windows的、Linux的，
各种后门，各种姿势。
内容也比较多，
坐稳了，开唠。

## 11.1 先讲一个故事

### 11.1.1 "进去了，又出来了"

我刚入行的时候，
有一次做一个项目，
打了好几天，
终于用一个Web漏洞拿到了一台服务器的权限。
当时特别兴奋，
截了图，看了文件，
感觉差不多了，
就把Shell关了，去写报告了。

结果第二天，
我想再上去看看，
发现连不上了。
怎么回事？
一打听，
人家管理员晚上重启了服务器，
Web服务重启了，
我的Shell当然就没了。

我当时那个悔啊，
早知道留个后门了。
又花了两天，
才重新打进去。
打进去第一件事就是——
先做三个不同的持久化，
生怕再掉了。

从那之后我就明白了一个道理：
**打进去之后第一件事，
先做持久化，
把屁股坐稳了再说。**

不然你辛辛苦苦打进去，
人家重启一下，
你就白忙活了。

---

当然，
还是那句话，
这些技术只能用在授权测试中。
未经授权留后门是违法的，
而且性质很严重。
学习研究可以，
别乱用。

这一章，
我们就来讲讲常见的持久化手法。
都是很经典的方法，
了解了这些，
你不仅会做持久化，
也知道该怎么去防范、去检测。

## 11.2 Windows 持久化手法

先讲Windows的，
因为Windows最常见，
持久化的手法也最多。

Windows的持久化，
说白了就是：
**想办法让你的木马/后门，
在系统启动的时候自动运行，
或者在特定条件下被触发运行。**

方法太多了，
我给你讲几种最常见、最经典的。

### 11.2.1 注册表 Run 键 —— 最经典的开机自启

第一个，
也是最经典的——
**注册表 Run 键。**

什么意思呢？
Windows注册表里有几个特殊的键，
系统启动、用户登录的时候，
会自动执行这个键下面的程序。
最常用的就是：

```
HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run
HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Run

还有RunOnce、RunServices等等
```

你把你的后门程序的路径，
写进这个注册表里，
下次开机/登录的时候，
它就会自动运行。

简单吧？
就一行命令的事。

比如：
```cmd
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v "MyProgram" /t REG_SZ /d "C:\path\to\backdoor.exe" /f
```

解释一下：
- `reg add`：添加注册表项
- `HKCU\...\Run`：就是Run键的路径，HKCU是当前用户
- `/v "MyProgram"`：键的名字，随便起，别太显眼
- `/t REG_SZ`：类型是字符串
- `/d "..."`：值，就是你的后门程序的路径
- `/f`：强制覆盖

这样下次用户登录的时候，
后门就会自动跑起来。

这个方法的特点：
- 简单，上手快
- 太经典了，杀软和EDR都盯着呢
- 容易被发现，容易被删
- 适合练手，实战中得想办法隐藏

Meterpreter里也有个模块专门干这个：
`run persistence`
一条命令就给你做好注册表持久化，
很方便。

### 11.2.2 计划任务

第二个，
**计划任务（Scheduled Tasks）。**

Windows有计划任务功能，
可以设置定时执行的任务，
也可以设置开机启动、登录启动的任务。

把你的后门做成一个计划任务，
设置成开机自动运行，
或者每隔一段时间运行一次，
也是很经典的持久化手法。

怎么创建计划任务？
用 `schtasks` 命令：

```cmd
:: 创建一个开机启动的计划任务
schtasks /create /tn "Windows Update Service" /tr C:\path\backdoor.exe /sc onstart /ru SYSTEM

解释：
  /tn 任务名       任务的名字，起个伪装性强的
  /tr 路径          要运行的程序
  /sc onstart      触发方式：开机启动
  /ru SYSTEM       以SYSTEM权限运行
```

还可以设置成用户登录时运行、
每天定时运行、
空闲时运行...
各种触发方式。

计划任务的好处：
- Windows自带功能，比较"正常"
- 可以指定运行权限（SYSTEM都行）
- 可以设置各种触发条件
- 比Run键稍微隐蔽一点

当然，
计划任务也是防守方重点检查的地方，
别起个太显眼的名字。

Meterpreter里也有计划任务的持久化模块，
直接 `run persistence` 就能选。

### 11.2.3 服务后门

第三个，
**服务（Service）。**

把你的后门注册成一个系统服务，
设置成自动启动，
开机就会自动运行。
而且服务一般都是SYSTEM权限，
权限还高。

怎么创建服务？
用 `sc` 命令：

```cmd
sc create "服务名" binPath= "C:\path\backdoor.exe" start= auto type= own type= interact
sc start "服务名"
```

服务的好处：
- 开机自启，权限高
- 比较隐蔽（系统里服务多了去了）
- 不容易被普通用户发现

但是也有问题：
- 服务对程序有要求，不是所有程序都能当服务跑
  （普通的exe可能不行，得是服务型的程序）
- 服务异常退出的话系统会有日志
- 也是防守方检查的重点

当然，
做个服务类型的后门也不难，
Metasploit生成Payload的时候可以选服务类型的。

### 11.2.4 启动文件夹

第四个，
**启动文件夹。**

这个更简单，
Windows有个"启动"文件夹，
放在里面的程序，
用户登录的时候会自动运行。

路径一般是：
```
当前用户的：
C:\Users\用户名\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\

所有用户的：
C:\ProgramData\Microsoft\Windows\Start Menu\Programs\StartUp\
```

把你的后门快捷方式或者程序，
往这个文件夹里一扔，
下次登录就自动运行了。

简单吧？
简单是简单，
但也太明显了，
随便一看就能看到。
一般不用这个，
太low了。

### 11.2.5 WMI 事件订阅

第五个，
**WMI 事件订阅（WMI Event Subscription）。**

这个就高级一点了。
什么意思呢？
WMI可以订阅事件，
比如系统启动的时候、
用户登录的时候、
某个进程创建的时候...
当事件发生的时候，
执行指定的脚本或者程序。

把后门和WMI事件订阅绑在一起，
比如系统启动就触发，
开机自动运行后门。

这个方法的好处：
- 比较隐蔽，一般人不会查WMI事件
- 没有文件也能做（无文件，存在WMI仓库里）
- 杀软和EDR相对关注得少一点
- 权限也可以很高

当然，
设置起来稍微麻烦一点，
不过现在都有现成的工具和脚本，
一条命令就能搞定。
Metasploit里也有对应的模块。

这种是相对高级一点的持久化手法，
实战中用得也比较多。

### 11.2.6 映像劫持（IFEO）

第六个，
**映像劫持（IFEO，Image File Execution Options）。**

这个也很有意思。
什么是映像劫持？
Windows注册表里有个地方叫IFEO，
本来是给程序员调试程序用的。
你可以设置某个程序启动的时候，
先运行另一个程序（调试器）。

如果把某个常用程序（比如notepad.exe、calc.exe）
的调试器设置成你的后门，
那每次用户打开记事本的时候，
你的后门就会先运行。
这样用户一用电脑，
你的后门就起来了。

听起来很猥琐是不是？
但确实是个有效的持久化方法。
而且不容易被发现，
因为一般人不会去查IFEO。

当然，
这个方法也有缺点：
- 得用户触发那个程序才会运行
- 不是开机自动运行的
- 但正常用户每天都要用各种程序，
  触发概率还是很高的

也是一种思路。

### 11.2.7 粘滞键后门

第七个，
**粘滞键后门（Sticky Keys）。**

这个是非常经典的、
很老但很实用的一个后门。

什么是粘滞键？
就是按五次Shift键弹出的那个功能。
它的程序是 `C:\Windows\System32\sethc.exe`。

这个后门的原理是：
在登录界面（还没输密码登录的时候），
按五次Shift，
就会弹出粘滞键。
如果我们把 sethc.exe 替换成 cmd.exe，
那在登录界面按五次Shift，
就会弹出一个cmd窗口，
而且是SYSTEM权限的！

这样就算不知道密码，
也能在登录界面拿到System权限的Shell。
非常猥琐，也非常实用。

怎么做？
简单，替换文件就行：
```cmd
copy c:\windows\system32\cmd.exe c:\windows\system32\sethc.exe
```
（当然要先备份原文件，而且要改权限，因为系统文件受保护）

除了粘滞键，
还有轻松访问中心、放大镜、屏幕键盘...
这些登录界面就能调用的工具，
都可以替换成cmd，
做成后门。

这个后门的特点：
- 非常经典，老黑客都知道
- 不需要登录，在登录界面就能用
- 权限高（SYSTEM）
- 但是要替换系统文件，动静有点大
- 现在很多防守方也会检查这个

不过依然是个很实用的方法。

### 11.2.8 后门账号

第八个，
**后门账号。**

这个最直接了——
直接在系统里加一个管理员账号，
想用的时候直接登进来。
不管是远程桌面还是SMB还是什么，
有账号就能进。

怎么加？
简单：
```cmd
net user 用户名 密码 /add
net localgroup administrators 用户名 /add
```

再加个隐藏属性什么的，
普通用户看不到。

这个方法的特点：
- 简单粗暴，直接有效
- 有了账号想怎么登怎么登
- 但是也很容易被发现
  （查用户列表就能看到）
- 适合短期用，长期的话容易被发现

当然，
也有更隐蔽的账号后门，
比如克隆账号（把管理员账号复制一个）、
注册表里藏账号...
隐蔽性更高。

### 11.2.9 WebShell 后门

第九个，
**WebShell后门。**

如果目标是Web服务器，
那留个WebShell也是很好的持久化方式。
- 找个偏僻的目录，藏一个一句话木马
- 或者给正常文件插一句后门代码
- 或者弄个不死马，删了自动生成

WebShell的好处：
- 通过80/443端口访问，不容易被怀疑
- 只要Web服务在，后门就在
- 重启也不怕
- 比较隐蔽

当然，
WebShell也容易被Web查杀工具扫到，
要做免杀、做隐藏。

### 11.2.10 更多高级的持久化手法

Windows的持久化手法太多了，
我讲的这些只是最常见的。
还有很多更高级的：
```
  • 注册表其他的各种键
  • 驱动级后门
  • Rootkit
  • 内存马（不落地，只在内存里）
  • 各种COM劫持
  • 快捷方式劫持
  • 文件关联
  • 组策略
  • DLL劫持
  • ......
```

只有你想不到，
没有做不到的。
Windows的机制太复杂了，
能用来做持久化的地方太多了。
防守方防不胜防，
攻击者花样百出。

## 11.3 Linux 持久化手法

讲完Windows的，
再讲讲Linux的。
Linux的持久化手法也不少，
但相对Windows来说少一点。

### 11.3.1 计划任务 —— Cron

第一个，
也是最经典的——
**Cron 定时任务。**

Linux的cron就相当于Windows的计划任务。
可以设置定时执行的命令/脚本。

把你的后门加到crontab里，
比如每分钟执行一次，
或者开机执行一次。
就算被杀了，
过一分钟又起来了。

怎么加？
```bash
# 编辑当前用户的crontab
crontab -e

# 加一行，比如每分钟执行一次
* * * * * /path/to/backdoor.sh

# 或者写进/etc/crontab
# 或者放到/etc/cron.d/、/etc/cron.hourly/等目录
```

cron的好处：
- 简单，Linux自带
- 稳定可靠
- 可以设置各种定时方式

当然，
cron也是防守方重点检查的地方。
别写太明显的命令。

### 11.3.2 开机自启服务（Systemd / init.d）

第二个，
**开机自启服务。**

Linux的服务，
设置成开机自启，
开机就自动运行你的后门。
而且一般服务都是root权限。

Systemd的系统（新一点的Linux）：
```bash
# 写一个.service文件
# 放到 /etc/systemd/system/
# 然后 enable 开机自启
systemctl enable myservice
```

老一点的系统用init.d：
```bash
# 脚本放到 /etc/init.d/
# 然后加开机启动
update-rc.d myscript defaults
```

跟Windows的服务差不多一个意思。
也是经典的持久化手法。

### 11.3.3 启动脚本 / rc.local

第三个，
**rc.local 或者各种启动脚本。**

Linux有很多启动脚本，
系统启动的时候会执行。
比如：
- `/etc/rc.local`
- `/etc/rc.d/` 下面的各种脚本
- `/etc/profile`、`/etc/bashrc`
- 用户目录下的 `.bashrc`、`.bash_profile`、`.profile`

把你的后门命令，
加到这些脚本里，
开机或者用户登录的时候就会执行。

比如加到用户的 `.bashrc` 里，
用户一登录Shell，
后门就运行了。
很隐蔽，
一般人不会去查自己的bashrc。

当然，
如果是root的bashrc，
那权限更高。

### 11.3.4 SSH 后门 / SSH 密钥

第四个，
**SSH相关的后门。**

Linux服务器SSH肯定是开的，
SSH的后门也有很多种。

```
  1. SSH公钥后门
     • 把你的公钥加到目标的 ~/.ssh/authorized_keys 里
     • 下次你用对应的私钥直接就能登
     • 不需要密码
     • 非常好用，也很隐蔽
     • 推荐这个，干净优雅

  2. 后门账号
     • 加一个root权限的用户
     • 直接密码登录
     • 简单粗暴但容易被发现

  3. SSH软链接后门
     • 利用ssh的PAM配置什么的
     • 更隐蔽的后门

  4. SSHD后门
     • 替换sshd程序
     • 留个万能密码什么的
     • 更高级一点
```

最推荐的就是**SSH公钥后门**，
干净、稳定、隐蔽，
用着也方便。
只要有root权限，
往root的authorized_keys里加个公钥，
以后随时都能登。

### 11.3.5 SUID Shell / 后门文件

第五个，
**SUID 后门。**

Linux的SUID权限前面提过，
设置了SUID的程序，
运行的时候以文件所有者的权限运行。

如果你复制一份bash，
设置成SUID，所有者是root，
那普通用户运行这个bash，
直接就能得到root权限的Shell。

```bash
cp /bin/bash /tmp/.hiddenbash
chmod u+s /tmp/.hiddenbash
```

以后想用的时候，
执行 `/tmp/.hiddenbash -p`，
就是root了。

隐蔽吧？
藏个文件在犄角旮旯，
名字起得隐蔽点，
一般人发现不了。

### 11.3.6 其他Linux持久化手法

还有很多，比如：
```
  • WebShell（跟Windows一样）
  • 内核模块 Rootkit
  • 库文件劫持（LD_PRELOAD等）
  • 各种钩子
  • ......
```

Linux的持久化手法虽然没Windows多，
但也不少。
核心思路都一样——
**想办法让你的后门自动运行，
或者留个随时能调用的入口。**

## 11.4 无文件持久化 & 内存马

现在越来越流行的一个概念——
**无文件（Fileless）持久化。**

什么是无文件？
就是**不在硬盘上留下文件**，
后门藏在注册表里、WMI里、内存里、
BIOS里、固件里...
硬盘上找不到恶意文件，
查杀工具就很难发现。

为什么要无文件？
因为现在杀软越来越强，
文件落地很容易被查杀。
不落地，
杀软就没那么好查了。

```
常见的无文件持久化：

  1. 注册表/ WMI 持久化
     • 后门代码存在注册表里、WMI仓库里
     • 执行的时候从注册表里读出来执行
     • 硬盘上没有恶意文件

  2. 内存马 / 进程注入
     • 后门只存在于内存里
     • 注入到正常进程里
     • 硬盘上找不到
     • 但重启就没了，所以要配合持久化

  3. 固件后门 / BIOS后门
     • 藏在BIOS、固件里
     • 重装系统都没用
     • 非常高级，也非常难发现

  4. 引导区后门
     • 藏在MBR、VBR里
     • 也是重装系统都没用
     • 非常底层
```

无文件是现在攻击的一个趋势，
越来越多的高级攻击都用无文件的方式。
防守方也越来越头疼。

不过对于新手来说，
先把基础的持久化学好，
再去研究这些高级的。

## 11.5 做好持久化的几个原则

最后，
给大家讲几个做好持久化的原则。

### 原则一：多做几个，鸡蛋别放一个篮子里

别只做一个持久化，
多做几个不同的。
- 注册表一个
- 计划任务一个
- 后门账号一个
- WebShell再来一个

这个被发现了还有那个，
东方不亮西方亮。
不然只留一个，
被删了你就进不去了。

### 原则二：注意隐蔽

持久化最重要的就是别被发现。
- 文件名、服务名、任务名，
  起得像正常的系统程序一样
- 别叫什么backdoor、hacker、test...
  太明显了
- 路径也藏深一点，
  别放桌面、别放根目录
- 能无文件就无文件，
  能少留文件就少留文件

越隐蔽，
存活时间越长。

### 原则三：权限够高

尽量做高权限的持久化。
- 最好是SYSTEM/root权限的
- 普通用户权限的用起来不方便

当然，
没高权限的时候，
普通用户的也得做，
有总比没有强。

### 原则四：合法使用，别乱用

最后还是那句话，
**没有授权，
绝对不要留后门。**

留后门的性质很严重，
比单纯的入侵性质更恶劣。
学习研究可以，
做授权测试的时候，
测试完了也要把后门清干净。
别给人家留隐患。

技术是把双刃剑，
用对地方。

## 11.6 本章总结

这一章我们讲了权限维持（持久化），
总结一下：

**Windows 常见持久化手法：**
- 注册表 Run 键 —— 最经典，简单粗暴
- 计划任务 —— 灵活，权限高
- 服务后门 —— 开机自启，SYSTEM权限
- 启动文件夹 —— 最简单但最明显
- WMI事件订阅 —— 隐蔽，可无文件
- 映像劫持（IFEO）—— 猥琐流
- 粘滞键后门 —— 登录界面就能用
- 后门账号 —— 直接有效
- WebShell —— Web服务器专用
- 还有很多高级的...

**Linux 常见持久化手法：**
- Cron定时任务 —— 经典好用
- 开机自启服务（Systemd / init.d）
- 启动脚本 / rc.local / bashrc
- SSH公钥后门 —— 推荐！干净优雅
- SUID后门
- 内核模块Rootkit...
- WebShell

**无文件持久化：**
- 注册表/WMI中存代码
- 内存马/进程注入
- 固件/BIOS后门
- 引导区后门

**做好持久化的原则：**
- 多做几个，别只留一个
- 注意隐蔽
- 权限尽量高
- 合法使用，别乱用

最后送你一句话：

> **打进去容易，
> 坐稳了难。
> 持久化做不好，
> 都是白忙活。**
