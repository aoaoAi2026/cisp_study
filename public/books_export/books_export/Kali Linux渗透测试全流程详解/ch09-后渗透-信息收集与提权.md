## 9.2 Windows 后渗透信息收集

先讲Windows的，
因为Windows在内网里最多见。

当你拿到一台Windows机器的Shell之后，
不管是cmd、powershell还是Meterpreter，
第一步要做的就是——
**收集信息**。
搞清楚这台机器的情况。

收集哪些信息呢？
我给你分门别类讲一讲。

### 9.2.1 系统基本信息

先看看这台机器是什么系统、
什么配置、什么权限。

```cmd
:: 系统信息
systeminfo
:: 或者更简洁的
ver

:: 主机名
hostname

:: 看当前用户是谁
whoami
whoami /all    :: 看当前用户的所有信息、权限组

:: 看环境变量
set

:: 看网络配置
ipconfig /all

:: 看路由表
route print

:: 看ARP表
arp -a

:: 看开放的端口
netstat -ano
```

这些命令能让你快速了解：
- 这是什么系统（Win7/Win10/Server 2008/2016...）
- 打了哪些补丁
- 主机名是什么
- 当前用户是谁、权限怎么样
- 网卡IP、DNS、网关
- 网络连接情况
- 开了哪些端口

### 9.2.2 用户与权限信息

接下来看看用户和权限相关的。

```cmd
:: 列出所有本地用户
net user

:: 看某个用户的详细信息
net user 用户名

:: 列出所有本地组
net localgroup

:: 看管理员组有哪些人
net localgroup administrators

:: 看当前登录的用户
query user
quser

:: 看远程桌面登录的用户
qwinsta
```

```cmd
:: 如果是域环境的话（加入域的机器）
:: 看域名
net config workstation

:: 看域用户
net user /domain

:: 看域管理员组
net group "Domain Admins" /domain

:: 看域控
nltest /dclist:域名
```

这些信息非常重要，
尤其是域环境的信息。
如果这台机器在域里，
那你的目标就更大了——
整个域都是目标。

### 9.2.3 进程、服务、计划任务

看看这台机器上跑了什么东西，
可能有什么有用的软件、
有什么可以利用的服务。

```cmd
:: 列出所有进程
tasklist
tasklist /v    :: 带详细信息，包括启动用户
:: 或者更详细的
wmic process get name,executablepath,processid

:: 列出所有服务
net start
:: 或者
sc query state= all

:: 列出计划任务
schtasks /query /fo LIST /v
:: 或者
at   :: 老系统用
```

看看有没有什么数据库、
中间件、运维工具、
杀软、监控软件...
这些都很重要。
- 有数据库？可能有敏感数据
- 有杀软？要注意免杀和绕过
- 有运维工具？可能有密码，或者可以利用
- 有可疑服务？可能有别的后门

### 9.2.4 软件与文件

看看装了什么软件，
有什么文件。

```cmd
:: 看安装的软件（注册表里的）
wmic product get name,version

:: 看Program Files目录
dir "C:\Program Files"
dir "C:\Program Files (x86)"

:: 看用户目录里有什么
dir C:\Users\
dir C:\Users\用户名\Desktop
dir C:\Users\用户名\Documents
dir C:\Users\用户名\Downloads

:: 找敏感文件
:: 比如找配置文件、密码文件、文档...
:: 这个要自己找，看经验
```

重点关注：
- 桌面的文件、文档
- 浏览器保存的密码
- 各种客户端的配置（FTP、数据库、SSH...）
- 运维相关的脚本、工具
- 机密文档、数据

### 9.2.5 凭证收集（找密码）

后渗透最重要的事情之一——
**找密码**。
密码是横向移动的钥匙。

密码可能在哪里？

```
  1. 系统里的密码哈希
     • SAM文件里的本地用户哈希
     • 域缓存凭据
     • 内存里的明文密码（mimikatz）

  2. 浏览器保存的密码
     • Chrome、Firefox、Edge...
     • 很多人把密码存在浏览器里
     • 非常有价值

  3. 各种客户端
     • FTP客户端（FileZilla、Xshell...）
     • 数据库客户端（Navicat、DBeaver...）
     • 远程桌面（RDP保存的密码）
     • SSH客户端（PuTTY、Xshell...）
     • 邮箱客户端
     • 等等

  4. 配置文件里的密码
     • Web应用的配置文件
     • 数据库连接配置
     • 中间件配置
     • 各种脚本里的密码

  5.  sticky notes、文本文件
     • 很多人把密码记在记事本里、桌面的txt里
     • 别笑，真的很多

  6. WiFi密码
     • 本机连过的WiFi密码
     • 有时候也有用
```

怎么抓？
最经典的就是**mimikatz**，
神器中的神器。
抓系统密码、抓内存里的明文、
各种姿势的抓。
Meterpreter里直接 `load mimikatz` 就能用。

除了mimikatz，
还有各种专门的工具，
比如找浏览器密码的、
找各种客户端密码的，
Metasploit里也有很多post模块，
直接 `run post/windows/gather/credentials/xxx` 就能用。

密码是内网渗透的硬通货，
越多越好。

### 9.2.6 网络与内网信息

既然要横向移动，
那就要搞清楚内网的情况。

```cmd
:: 网卡信息（前面讲过了）
ipconfig /all
:: 看看有几块网卡、几个网段
:: 有的机器是双网卡，一边外网一边内网，那你就赚到了

:: 域信息（前面也讲了）
:: 如果在域里，域的信息很重要

:: 看网络共享
net view
net view /domain

:: 看本机的共享
net share

:: 看会话连接
net session

:: 看当前的连接
netstat -ano
```

```cmd
:: 扫内网（怎么扫后面讲）
:: 先看看ARP表里有哪些邻居
arp -a

:: 看看缓存的DNS记录
ipconfig /displaydns
```

先搞清楚：
- 这台机器在哪个网段？
- 还有哪些网段？
- 域名叫什么？域控是谁？
- 内网大概有多少机器？
- 有没有什么明显的服务器（文件服务器、数据库服务器...）？

心里有个大概的谱，
再决定下一步怎么走。

## 9.3 Linux 后渗透信息收集

讲完Windows的，
再简单讲讲Linux的。
Linux的信息收集思路差不多，
就是命令不一样。

### 9.3.1 系统基本信息

```bash
# 系统信息
uname -a
cat /etc/issue
cat /etc/*release
hostname

# 当前用户
whoami
id

# 环境变量
env
```

### 9.3.2 用户信息

```bash
# 看所有用户
cat /etc/passwd

# 看有哪些用户能登录
cat /etc/passwd | grep /bin/bash

# 看shadow（需要root权限）
cat /etc/shadow

# 看登录过的用户
last
lastlog

# 当前在线用户
w
who
```

### 9.3.3 网络信息

```bash
# 网卡信息
ifconfig
ip a

# 路由
route
ip route

# 端口
netstat -tulnp
ss -tulnp

# ARP
arp -a
ip neigh

# DNS
cat /etc/resolv.conf
```

### 9.3.4 进程、服务、定时任务

```bash
# 进程
ps aux
ps -ef

# 服务
service --status-all
systemctl list-units --type=service  # 新系统

# 定时任务
crontab -l
cat /etc/crontab
ls -la /etc/cron.*
```

### 9.3.5 找密码、找敏感信息

```bash
# 历史命令（非常重要！很多密码在历史记录里）
history
cat ~/.bash_history
cat /root/.bash_history

# 找配置文件里的密码
# grep 搜 password、passwd、pwd、secret、key...
grep -r "password" /var/www/
grep -r "passwd" /home/

# 看SSH密钥
ls -la ~/.ssh/
cat ~/.ssh/id_rsa

# 看各种配置文件
# Web应用、数据库、中间件...
```

Linux里，
**历史命令**是个好东西，
很多管理员习惯在命令行里敲密码，
全记在history里了，
一翻一个准。

还有SSH私钥，
如果能拿到root的私钥，
那很多机器都能直接登。

## 9.4 提权概述

信息收集做得差不多了，
接下来很重要的一步——
**提权**。

什么是提权？
简单说就是：
**从低权限用户，
想办法变成高权限用户。**

为什么要提权？
```
  • 权限低，很多操作做不了
  • 抓不了密码哈希
  • 装不了后门
  • 看不了很多文件
  • 很多后渗透操作需要高权限
  • ......
```

总之，
权限越高越好。
目标是拿到最高权限：
- Windows：SYSTEM / Administrator
- Linux：root

### 9.4.1 提权的两种大方向

提权一般分两大方向：

```
  1. 纵向提权（垂直提权）
     • 从普通用户提到管理员/root
     • 也就是权限级别往上升
     • 这是最常见的

  2. 横向提权
     • 从一个普通用户，变成另一个普通用户
     • 权限级别没变，但身份变了
     • 比如从www-data用户变成另一个用户
     • 那个用户可能有更多权限、更多资源
     • 有时候也很有用
```

我们一般说提权，
指的是纵向提权。

### 9.4.2 提权的常见方法

提权的方法很多，
我给你分分类。

```
Windows提权常见方法：

  1. 系统内核漏洞
     • 比如MS16-032、CVE-2021-40449之类的
     • 系统没打补丁，有本地提权漏洞
     • 利用漏洞直接提权到SYSTEM
     • 最直接粗暴的方法

  2. 服务利用
     • 系统服务配置有问题
     • 比如：
       - 服务路径没加引号，路径里有空格
       - 服务的二进制文件权限设置不当，普通用户能改
       - 注册表服务项权限有问题
     • 利用这些配置缺陷提权

  3. 计划任务
     • 计划任务运行的权限高
     • 任务执行的脚本/程序我们能修改
     • 替换成我们的payload，等任务执行
     • 就能拿到高权限

  4. 注册表相关
     • 注册表的一些项权限设置不当
     • 可以利用来提权

  5. 第三方软件/驱动
     • 安装的第三方软件有漏洞
     • 或者驱动有漏洞
     • 利用这些提权

  6. 令牌窃取 / 烂土豆
     • 从服务账号（比如IIS、服务权限）
       偷令牌，或者用Rotten Potato系列提权
     • 很经典的手法

  7. AlwaysInstallElevated
     • 注册表开了这个的话，普通用户装MSI
       也是以SYSTEM权限运行的
     • 直接就能提权

  ......还有很多很多
```

```
Linux提权常见方法：

  1. 内核漏洞
     • 经典的脏牛（Dirty Cow）、CVE-2021-4034...
     • 内核漏洞提权，直接变root

  2. SUID / SGID
     • 某些程序设置了SUID位
       运行的时候以文件所有者的权限运行
     • 如果这些程序有漏洞、或者可以被利用
       就能提权
     • 最经典的，比如nmap、vim、find、bash...
       这些有SUID的话，很容易提权

  3. sudo
     • sudo配置不当
     • 比如普通用户能以root身份运行某个命令
     • 那个命令如果可以利用，就能提权
     • 比如 sudo vim 就能提权

  4. 计划任务（Cron）
     • root的定时任务
     • 执行的脚本/程序我们能改
     • 改了等它执行，就能拿到root

  5. 通配符注入
     • cron里用了通配符（比如 *）
     • 配合一些命令的参数，可以注入
     • 也算一种提权手法

  6. 环境变量劫持
     • 路径没写全，用相对路径
     • 我们改PATH环境变量
       放一个同名的恶意程序在前面
     • 高权限程序运行的时候就会执行我们的

  7. NFS共享
     • NFS配置不当
     • no_root_squash之类的
     • 可以利用来提权

  8. 密码复用
     • 找到其他用户的密码
     • 正好root也用这个密码
     • su过去就完了

  ......还有很多
```

提权的方法太多了，
一本书都讲不完。
这里只是给你个大概的概念，
知道有哪些方向。

### 9.4.3 提权的一般思路

拿到Shell之后，
提权怎么个思路？

```
一般步骤：

  1. 先看看当前权限
     whoami / id
     已经是最高权限了？那不用提了

  2. 收集系统信息
     什么系统？什么版本？打了哪些补丁？
     内核版本是多少？

  3. 找可能的提权点
     • 看看系统信息，有什么已知的内核漏洞
     • 看看SUID/SGID文件（Linux）
     • 看看服务配置（Windows）
     • 看看计划任务
     • 看看注册表
     • 看看有没有能利用的第三方软件
     • 等等

  4. 选择合适的提权方法
     找到可能的提权点，选一个最靠谱的

  5. 上传提权工具/payload，执行
     注意免杀，注意不被发现

  6. 提权成功，验证权限
     看看是不是已经变成最高权限了
```

简单说就是：
**收集信息 → 找弱点 → 利用弱点提权**。

## 9.6 提权实战思路举例

光说理论太干了，
我给你举几个简单的例子，
让你有个直观的感受。

（只是举例，
实际环境可能不一样，
但思路是一样的。）

### 例1：Windows 服务路径未加引号

这是个经典的Windows提权手法。

什么意思呢？
Windows服务的二进制路径，
如果路径里有空格，
又没加引号的话，
就可能被利用。

比如服务路径是：
`C:\Program Files\My Service\service.exe`

如果没加引号，
Windows解析的时候会依次尝试：
```
C:\Program.exe
C:\Program Files\My.exe
C:\Program Files\My Service\service.exe
```

如果前面的目录普通用户有权限写文件，
那我们放一个恶意的Program.exe或者My.exe在那，
服务重启的时候，
就会以SYSTEM权限运行我们的程序，
直接提权。

这就叫**服务路径未引用（Unquoted Service Path）**。
很经典的提权手法。

PowerUp里有个函数 `Get-UnquotedService` 就是找这个的。

### 例2：Linux SUID 提权

Linux里SUID也是经典的提权方式。

什么是SUID？
简单说，
设置了SUID位的程序，
不管是谁运行，
都以文件所有者的权限运行。

如果某个程序的所有者是root，
又设置了SUID位，
而这个程序又可以被利用来执行命令或者读文件，
那我们就能用它来提权。

怎么找SUID文件？
```bash
find / -perm -u=s -type f 2>/dev/null
```

找到之后，
去GTFOBins这个网站查一下，
看看这个程序能不能用来提权。
GTFOBins是个好东西，
收集了各种Unix/Linux工具的提权姿势，
非常实用。

比如nmap有SUID的话，
可以用交互模式执行shell，
直接提权。
比如find有SUID的话，
也可以执行命令。
很多工具都能利用。

### 例3：计划任务提权

这个Windows和Linux都有。

思路很简单：
- 有个计划任务/定时任务，
- 运行的权限很高（比如SYSTEM/root），
- 任务执行的脚本或者程序，我们有权限修改，
- 那我们就把它替换成我们的恶意程序，
- 等任务一执行，我们就拿到高权限了。

就这么简单。
这种情况在实际环境中还挺常见的，
很多开发者图省事，
脚本权限随便设，
就留下了提权的口子。

## 9.7 提权的注意事项

最后提几个提权的注意事项。

```
  1. 先收集信息，再动手
     别上来就瞎试各种exp
     先把信息收集全了
     挑最靠谱的方法试
     不然容易把系统搞崩，或者被发现

  2. 注意系统稳定性
     有些内核漏洞的exp不太稳定
     可能会蓝屏、panic
     实战中要注意，别把目标机器搞挂了
     尤其是重要的服务器

  3. 注意免杀
     提权的exp、工具
     可能会被杀软、EDR检测到
     要注意免杀和绕过
     不然工具一上传就被删了

  4. 留好退路
     提权之前先把Shell稳住
     别提权没成功，把原来的Shell也搞没了
     那就尴尬了

  5. 多准备几种方法
     一个方法不行，还有下一个
     东方不亮西方亮
     提权的方法多着呢
```

## 9.8 本章总结

这一章我们讲了后渗透的第一步：
信息收集和提权。

总结一下：

**后渗透信息收集：**
- 系统基本信息（系统版本、主机名、用户、网络...）
- 用户与权限（本地用户、管理员组、域信息...）
- 进程、服务、计划任务
- 软件与文件
- 凭证收集（密码、哈希、各种客户端凭据...）—— 很重要！
- 网络与内网信息

**Windows信息收集命令：**
systeminfo、whoami、ipconfig、net user、tasklist、netstat...

**Linux信息收集命令：**
uname、id、ifconfig、ps、crontab、history...

**提权概述：**
- 纵向提权 vs 横向提权
- 目标是拿到最高权限（SYSTEM/root）

**Windows提权常见方法：**
- 内核漏洞
- 服务利用（路径未引用、权限不当...）
- 计划任务
- 注册表
- 令牌窃取/烂土豆
- AlwaysInstallElevated
- 第三方软件/驱动

**Linux提权常见方法：**
- 内核漏洞
- SUID/SGID
- sudo配置不当
- 计划任务（cron）
- 环境变量劫持
- NFS
- 密码复用

**提权辅助工具：**
- Windows：WinPEAS、PowerUp、Watson、Seatbelt...
- Linux：LinPEAS、LinEnum、Linux Exploit Suggester...

**提权一般思路：**
收集信息 → 找弱点 → 利用弱点提权

最后送你一句话：

> **打进去只是开始，
> 能走多远，
> 取决于你的信息收集有多细。**
