# 第十章 内网渗透 —— 横向移动

> 第10章 | 预计学习时间：6.5小时 | 难度：⭐⭐⭐⭐⭐

---

## 📖 写在前面

上一章我们讲了后渗透的信息收集和提权。
拿到一台机器的高权限之后，
接下来该干嘛？

当然是——
**横向移动。**

什么是横向移动？
简单说就是：
**从你已经控制的这台机器，
去攻击内网里的其他机器，
扩大你的控制范围。**

用人话说就是：
**从一台打到一片。**

横向移动是内网渗透的核心，
也是最有意思的部分。
外网打进一台机器只是个开始，
内网里能走多远，
才是真本事。

这一章我们就来讲讲横向移动。
从内网信息收集，
到各种横向移动的手法，
再到域环境的渗透思路。
内容比较多，
也比较硬核。

坐稳了，开唠。

## 10.1 先讲一个故事

### 10.1.1 "一台服务器，拿下整个内网"

我有个朋友，
做渗透测试的。
有一次他做一个项目，
从外网打进去一台Web服务器，
权限还不高，就是个普通的IIS用户。

一般人到这里可能就觉得差不多了，
交个报告完事。
但我这个朋友比较轴，
他就从这台Web服务器开始，
慢慢摸。

首先，
他从Web配置文件里找到了数据库的密码，
登进去数据库，
从数据库里又找到了一堆后台账号密码。
然后，
他用这些密码试了试服务器的本地管理员，
竟然有一台对上了——
密码复用。

拿到这台管理员权限的机器之后，
他抓了哈希，
然后用哈希传递（Pass-the-Hash）
扫整个内网的445端口，
看看哪些机器能用这个哈希登进去。
一扫，扫出十几台。

他一台一台登上去，
每台都抓密码哈希、抓明文密码，
收集到的凭据越来越多。
有的机器是域管理员登录过的，
抓到了域管理员的凭据。

最后，
他用域管理员的凭据，
直接登录了域控。
整个域，几百台机器，
全拿下了。

从一台最普通的Web服务器，
到整个域控，
用了不到两天。
靠的就是横向移动。

---

你看，
内网渗透就是这样，
往往只要打进一台，
就能像滚雪球一样，
越滚越大，
最后把整个内网都拿下来。

因为内网里防护通常比较弱，
而且密码复用、弱口令的情况特别多。
只要你有耐心，
慢慢摸，
一般都能摸到核心区域。

这一章，
我们就来讲讲横向移动的各种手法。

## 10.2 横向移动前的准备

在开始横向移动之前，
你得先做好准备工作。

### 10.2.1 收集凭据

横向移动，
最关键的是什么？
**凭据。**

不管你用什么方法横向移动，
最终都要登录到另一台机器上。
登录就要有凭据——
要么是用户名密码，
要么是哈希，
要么是票据，
要么是密钥。

所以第一步，
就是尽可能多地收集凭据。

从哪里收集？
上一章讲过了：
```
  • 系统内存里的密码（mimikatz）
  • SAM里的哈希
  • 浏览器保存的密码
  • 各种客户端的密码
  • 配置文件里的密码
  • 历史命令里的密码
  • 共享里的文档
  • 等等
```

凭据越多，
横向移动的成功率就越高。
很多企业内网，
密码复用率高得惊人，
一个管理员密码可能能用在几十台上百台机器上。

所以记住：
**凭据是横向移动的硬通货。**

### 10.2.2 内网信息收集

除了凭据，
你还得知道内网有什么。
有哪些机器？
哪些开了什么端口？
哪些是服务器？
哪些是域控？
哪些是重要的机器？

也就是——
**内网的信息收集和探测。**

你已经控制的这台机器，
就是你的"跳板机"。
从这台机器出发，
去探测整个内网。

怎么探测？
```
  1. 先看本机的信息
     • ipconfig / ifconfig —— 有几个网卡、几个网段
     • 域信息
     • ARP表
     • 网络连接
     • 共享

  2. 存活主机探测
     • 扫网段，看哪些IP是活的
     • 怎么扫？可以用工具，也可以用系统自带的
     • 比如：
       - for循环ping
       - 用nmap、masscan（得想办法传上去）
       - 用PowerShell脚本
       - 等等

  3. 端口扫描
     • 存活的主机，扫扫端口
     • 看看开了哪些服务
     • 445（SMB）、3389（RDP）、22（SSH）、
       3306（MySQL）、1433（MSSQL）、5985（WinRM）...
     • 开的端口决定了用什么方法打

  4. 域信息收集（如果是域环境）
     • 域有多大？
     • 有多少用户、多少机器？
     • 域控是谁？
     • 有哪些组？
     • 哪些是管理员？
     • 等等
```

内网信息收集的工具也很多，
比如 Empire、Cobalt Strike 里面都有内置的，
也有各种PowerShell脚本、C#工具。
Meterpreter里也有很多辅助模块。

先搞清楚内网的大概情况，
再决定下一步怎么打。

### 10.2.3 准备工具

要横向移动，
工具肯定少不了。
常用的工具得准备好，
想办法传到跳板机上。

```
常用的内网工具：
  • mimikatz —— 抓密码神器
  • psexec / paexec —— 横向移动经典工具
  • wmic / wmiexec —— WMI执行命令
  • powershell —— 自带的，各种脚本都能用
  • cscript —— 执行vbs脚本
  • net、sc、schtasks... —— 系统自带命令
  • nc / netcat —— 瑞士军刀
  • portproxy —— 端口转发
  • 各种扫描脚本
  • 各种提权工具
  • ......
```

当然，
传工具的时候要注意免杀，
别一上传就被杀了。

## 10.3 常见的横向移动手法

准备工作做好了，
接下来就是真刀真枪的横向移动了。

横向移动的手法有很多，
我给你讲几种最常见、最经典的。

### 10.3.1 IPC$ + 计划任务（AT / Schtasks）

这个是最老牌、最经典的Windows横向移动手法了。

什么是IPC$？
IPC$是Windows的命名管道共享，
是进程间通信的一种方式。
简单说，
就是可以通过网络，
和目标机器建立一个IPC连接。

建立连接之后能干嘛？
可以查看共享、可以上传下载文件、
可以用计划任务执行命令...

经典的流程是这样的：

```
  1. 建立IPC$连接
     net use \\目标IP\ipc$ "密码" /user:"用户名"

  2. 上传木马/Payload
     copy payload.exe \\目标IP\c$
     （如果有admin$或者c$共享的话）

  3. 创建计划任务执行
     schtasks /create /s 目标IP /tn 任务名 /tr c:\payload.exe /sc once /st 时间
     （用at命令也可以，老系统用at）

  4. 等任务执行，拿到Shell
```

这个是非常经典的手法，
从Windows 2000时代就有了，
一直到现在还能用。
当然，
前提是：
- 目标开了445端口（SMB）
- 你有目标的账号密码（本地管理员或者域管理员更好）
- 目标的admin$ / c$ 共享能访问
- 防火墙没拦

经典是经典，
但现在用的少了，
因为动静有点大，
而且很多环境限制了。
但学习还是要知道的。

### 10.3.2 PsExec

第二个，**PsExec**。
这个可以说是横向移动的代名词了，
太有名了。

PsExec是Sysinternals工具集里的一个工具，
可以在远程Windows系统上执行命令，
还能交互式运行。
功能非常强大。

PsExec的原理其实也是基于SMB和服务，
它会：
1. 通过SMB连接目标
2. 把psexec的服务程序上传到目标
3. 创建并启动一个服务
4. 通过这个服务执行命令
5. 执行完再把服务删掉

用法也很简单：
```
psexec \\目标IP -u 用户名 -p 密码 cmd.exe
```

执行之后，
就会弹一个目标机器的cmd Shell回来，
就跟在本地操作一样。

Metasploit里也有psexec的模块，
`exploit/windows/smb/psexec`，
用哈希或者密码都能打，
直接返回Meterpreter会话。
非常方便。

PsExec虽然好用，
但也有缺点：
- 目标会留下日志（系统日志、安全日志）
- 会创建服务，动静有点大
- 很多杀软、EDR都盯着psexec
- 现在很多企业环境里禁用了

但不管怎么说，
PsExec都是横向移动最经典的手法，
必须得知道。

### 10.3.3 Pass-the-Hash（哈希传递）

第三个，
**Pass-the-Hash，简称PtH，哈希传递。**

这个可以说是内网渗透中最有名的技术之一了。

什么是哈希传递？
简单说就是：
**不用破解密码明文，
直接用密码哈希就能登录。**

用人话说就是：
**我不需要知道你的密码是什么，
只要拿到你的密码哈希，
我就能用你的身份登录。**

这就厉害了。
我们抓到的NTLM哈希，
不用费劲去破解，
直接就能用来横向移动。
这也是为什么内网里抓哈希这么重要。

Pass-the-Hash的原理是什么呢？
简单说，
Windows的某些认证协议（比如NTLM认证），
在认证的时候，
传的就是密码哈希，
而不是明文。
所以只要你有哈希，
就能通过认证，
跟有密码是一样的效果。

哪些地方能用Pass-the-Hash？
- SMB（445端口）—— 最常用
- WMI
- WinRM
- RDP（受限模式下也行）
- 等等

经典的工具：
```
  • mimikatz —— 可以把哈希注入到当前进程
    然后用这个身份去访问别的机器
  • psexec —— Metasploit的psexec模块支持哈希
  • wmiexec.py（Impacket工具集里的）—— Linux下用，支持哈希
  • CrackMapExec —— 内网神器，支持Pass-the-Hash
  • 还有很多工具都支持
```

Pass-the-Hash有什么条件？
- 你得有目标用户的NTLM哈希
- 目标得用NTLM认证
- 目标机器得开着对应的服务（SMB、WMI等）
- 你有访问权限（管理员权限最好）

可以说，
Pass-the-Hash是内网渗透的核心技术之一。
抓到一个管理员哈希，
可能就能横扫几十台上百台机器。

当然，
微软也推出了一些防护措施，
比如：
- LSA保护
- Restricted Admin Mode
- Credential Guard
- 等等

这些能一定程度上防Pass-the-Hash，
但实际环境中，
能用Pass-the-Hash的情况还是非常多。

### 10.3.4 WMI 执行命令

第四个，**WMI**。

WMI是什么？
Windows Management Instrumentation，
Windows管理规范。
简单说，
就是Windows自带的一个管理接口，
可以通过它来管理远程的Windows机器。
查询信息、执行命令、创建进程...
啥都能干。

WMI横向移动有什么好处？
- Windows自带，不需要传工具
- 很多环境没禁用WMI
- 相对来说没PsExec那么扎眼
- 135端口 + 动态端口（或者445也可以）

怎么用？
最经典的就是用系统自带的 `wmic` 命令：

```cmd
:: 连接目标，执行命令
wmic /node:目标IP /user:用户名 /password:密码 process call create "cmd.exe /c 命令"
```

就能在目标上执行命令了。

当然，
用wmic执行命令是没有回显的，
执行完了看不到结果。
想看到结果怎么办？
可以把结果写到文件里，
然后读出来，
或者用别的方式带回来。

除了wmic，
还有很多更好用的WMI工具，
比如：
```
  • wmiexec.py（Impacket）
    - Linux下用的，有回显
    - 支持密码，也支持哈希
    - 非常好用

  • Invoke-WmiCommand（PowerShell）
    - PowerShell脚本，有回显

  • wmic的各种变种工具
```

WMI是个很好用的横向移动通道，
一定要掌握。

### 10.3.5 WinRM / PowerShell Remoting

第五个，**WinRM**。

WinRM是什么？
Windows Remote Management，
Windows远程管理。
是微软后来推的一个远程管理协议。
基于Web Services-Management（WS-Management），
走HTTP/SOAP协议。
默认端口是5985（HTTP）和5986（HTTPS）。

WinRM有什么好处？
- 微软官方的，Windows Server 2012以后默认开启
- PowerShell远程管理就是基于WinRM的
- 功能强大，用起来方便
- 现在服务器上开的越来越多了

怎么用？
最直接的就是PowerShell远程：

```powershell
# 建立会话
Enter-PSSession -ComputerName 目标IP -Credential 凭据

# 执行命令
Invoke-Command -ComputerName 目标IP -ScriptBlock { 命令 } -Credential 凭据
```

用起来很方便，
就跟在本地用PowerShell一样。

工具方面：
```
  • evil-winrm —— 非常火的WinRM Shell工具
    - 支持密码、哈希、证书
    - 功能很强大
    - 现在做渗透测试的基本都用这个

  • CrackMapExec —— 也支持WinRM
  • Metasploit里也有对应的模块
```

WinRM现在越来越常见了，
尤其是新的Windows服务器，
很多都开着WinRM。
看到5985/5986端口开着，
就可以试试WinRM。

### 10.3.6 RDP 远程桌面

第六个，**RDP**，远程桌面。

这个大家都熟悉，
Windows自带的远程桌面连接。
3389端口。

有了账号密码，
直接远程桌面登上去，
图形界面操作，
爽得很。
而且很多管理员都用RDP，
登进去还能看看人家桌面上有啥，
经常能找到好东西。

但是RDP也有问题：
- 目标如果有人在用，你登上去人家会发现（被挤下来）
- 动静比较大
- 有些机器限制了哪些用户能登RDP

不过，
也有办法，
比如可以开启RDP的受限管理员模式，
可以用哈希登录（Pass-the-Hash也能用在RDP上）。

工具的话，
Windows自带的mstsc就不说了，
Linux下有xfreerdp、rdesktop这些。
还有Impacket里的rdp_check之类的工具。

RDP虽然老，
但依然是很重要的横向移动手段。
尤其是碰到运维人员的机器，
用RDP登进去，
收获往往很大。

### 10.3.7 SSH

第七个，**SSH**。
这个主要是Linux之间的横向移动，
当然Windows现在也有OpenSSH了。

SSH横向移动就更简单了：
- 拿到账号密码，直接ssh登过去
- 拿到私钥，也能登过去
- 还能用SSH隧道做端口转发

SSH也是内网里很常见的，
尤其是Linux服务器多的环境。
方法简单粗暴，
但非常有效。

### 10.3.8 数据库、中间件、各种服务

除了系统层面的，
还有各种服务也能用来横向移动。
比如：

```
  • 数据库
    - MySQL、MSSQL、Oracle、Redis、MongoDB...
    - 有账号密码直接登
    - 很多数据库还能执行系统命令
    - 登进去不仅能拿数据，还能拿服务器权限

  • 中间件
    - Tomcat、WebLogic、JBoss...
    - 有弱口令或者漏洞的话也能打

  • 运维管理系统
    - 堡垒机、监控、运维平台...
    - 拿下来就是核心节点

  • 共享文件夹
    - 很多共享里有敏感文件、脚本、密码
    - 也能用来传播木马

  • 等等...
```

内网里各种服务多了去了，
只要有凭据、有漏洞，
都能成为横向移动的跳板。

## 10.4 端口转发与代理

讲横向移动，
就不得不讲**端口转发**和**代理**。

为什么需要？
因为你控制的那台机器在内网里，
你要打内网里的其他机器，
你的攻击机（Kali）在外网，
直接访问不到内网的机器啊？
怎么办？

这时候就需要——
**把你控制的那台机器做成跳板，
通过它来访问内网的其他机器。**

怎么实现？
端口转发和代理。

### 10.4.1 端口转发

什么是端口转发？
简单说就是：
**把目标机器的某个端口，
转发到你的机器上，
或者把你的端口转发到目标机器上。**

比如：
你想访问内网另一台机器的3389端口，
但是你直接连不上。
你可以在你控制的跳板机上，
把那台内网机器的3389端口，
转发到你能访问的地方。
然后你访问转发后的端口，
就等于访问目标的3389了。

端口转发的工具有很多：
```
  • netsh（Windows自带）
    - portproxy
    - Windows自带，不用传工具
    - 很好用

  • nc / netcat
    - 瑞士军刀，也能做端口转发

  • lcx / portmap
    - 经典的端口转发工具
    - 内网端口转发、反弹式端口转发

  • frp、nps
    - 现代的内网穿透工具
    - 功能很强大

  • SSH 端口转发
    - Linux下常用
    - ssh -L、ssh -R、ssh -D

  • Meterpreter portfwd
    - Metasploit自带端口转发功能
    - 很方便
```

端口转发适合访问少数几个端口的情况，
比如就想连个RDP、连个数据库，
用端口转发就够了。

### 10.4.2 代理 / Socks隧道

如果要访问的端口很多，
或者想用自己的工具（比如Nmap、SQLMap）
直接扫内网，
端口转发就不够用了。
这时候就需要——
**代理**。

最常用的就是Socks代理。
在跳板机上架一个Socks代理，
然后你本地的所有工具都走这个代理，
这样就相当于你的Kali也进了内网，
想扫什么扫什么，想用什么工具用什么工具。

怎么搭Socks代理？
```
  • Meterpreter 的 autoroute + socks4/5 模块
    - Metasploit自带
    - 有Meterpreter会话就能用
    - 很方便

  • EarthWorm / EW
    - 轻量级的Socks代理工具
    - 很好用，速度也快
    - 经典工具

  • frp、nps
    - 也支持Socks代理

  • SSH -D
    - SSH动态端口转发
    - Linux下很方便

  • 还有很多其他的
    - ssocks、reGeorg...
    - 各种Webshell代理、内存代理
```

有了Socks代理，
配合Proxychains（Linux下）或者Proxifier（Windows下），
各种工具都能直接用在内网了。
Nmap扫内网、
SQLMap打内网的Web、
Hydra爆破内网服务...
都不在话下。

代理是内网渗透的必备技能，
一定要掌握。

## 10.5 域环境渗透思路

讲横向移动，
就不能不讲**域环境**。

什么是域？
简单说就是：
Windows的一种网络管理模式，
把很多计算机加入到一个域里，
由域控制器（Domain Controller，简称DC）统一管理。
域里有域用户、域管理员，
一个域账号可以在域里的所有机器上登录（有权限的话）。

企业内网里，
域环境非常非常常见。
尤其是规模大一点的企业，
基本都是域环境。

域环境的渗透，
目标很明确——
**拿下域控（DC）。**
拿下了域控，
整个域就都是你的了，
想干嘛干嘛。

### 10.5.1 域渗透的一般思路

域渗透一般是个什么思路？

```
大致流程：

  1. 先打进域内一台机器
     • 不管什么方法，先搞一台域内机器的权限
     • 普通用户权限都行

  2. 信息收集
     • 收集域的信息
     • 域名、域控、域用户、域组、域机器...
     • 越详细越好

  3. 找域管理员的凭据
     • 抓密码、抓哈希、抓票据
     • 找域管理员登录过的机器
     • 想办法弄到域管理员的凭据

  4. 横向移动，逐步扩大
     • 用收集到的凭据打其他机器
     • 收集更多的凭据
     • 越滚越大
     • 从普通用户 -> 本地管理员 -> 域管理员

  5. 拿下域控
     • 有了域管理员权限，直接登录域控
     • 或者用域管理员的哈希/票据去打域控
     • 拿下域控，整个域就到手了
```

这是最经典的域渗透思路，
俗称"偷域管理员凭据"的思路。
因为域管理员权限太大了，
只要拿到域管理员的凭据，
整个域就沦陷了。

### 10.5.2 怎么找域管理员？

域渗透的关键是拿到域管理员的凭据。
怎么拿？

```
常见的方法：

  1. 找域管理员登录过的机器
     • 域管理员经常会登录一些服务器运维
     • 这些机器的内存里可能留有域管理员的凭据
     • 你拿下了这些机器，就能抓到域管理员的密码/哈希
     • 怎么找哪些机器有域管理员登录过？
       有工具可以查，比如 psloggedon、net session...
       或者用BloodHound

  2. 钓鱼
     • 给域管理员发钓鱼邮件
     • 钓管理员的密码或者Shell

  3. Kerberoast攻击
     • 针对Kerberos服务票据的攻击
     • 请求服务票据，然后离线破解
     • 破解出服务账号的密码
     • 如果服务账号权限高，也能利用

  4. AS-REP Roasting
     • 针对不需要预身份认证的用户
     • 能拿到这些用户的AS-REP包
     • 离线破解密码

  5. 委派攻击
     • 约束委派、非约束委派
     • 利用配置不当的委派
       冒充其他用户访问服务

  6. 漏洞利用
     • 域控本身有漏洞
     • 比如MS14-068、Zerologon之类的
     • 直接打域控漏洞拿权限

  还有很多其他的攻击手法...
```

域渗透的攻击手法非常多，
深不见底。
从最基础的抓密码横向，
到各种Kerberos攻击（黄金票据、白银票据、Pass-the-Ticket...），
再到各种域内漏洞利用。
东西特别多。

### 10.5.3 BloodHound —— 域渗透可视化神器

说到域渗透，
就不得不提 **BloodHound**。
这是个神器。

BloodHound是什么？
简单说就是：
**用图的方式，
把域里的各种关系（用户、机器、组、权限...）
可视化地展示出来，
帮你找到从当前位置到域管理员的最短路径。**

用人话说就是：
**帮你找攻击路径的神器。**

域里用户、机器、组、权限，
关系错综复杂，
靠人根本理不清。
BloodHound把这些关系都画成图，
一点，
就能告诉你怎么走最快能拿到域管理员权限。
非常直观。

现在做域渗透，
基本人手一个BloodHound。
上传收集器跑一下，
数据导入BloodHound，
攻击路径一目了然。
大大提高了域渗透的效率。

有机会一定要试试，
真的很好用。

## 10.6 给新手的建议

最后给新手几个建议。

### 建议一：多收集凭据，多试

横向移动，
凭据是核心。
- 密码越多越好
- 哈希越多越好
- 各种凭据都收集

收集到之后，
别闲着，
都拿去试试，
看看哪些机器能登。
很多时候，
一个意想不到的密码，
可能就能打开一扇大门。

### 建议二：思路要灵活

横向移动没有固定的套路，
别死磕一种方法。
这个方法不行，
就换另一个。
SMB不行试试WMI，
WMI不行试试WinRM，
再不行试试RDP、SSH、数据库...
办法总比困难多。

### 建议三：注意隐蔽，动静别太大

实战中，
要注意隐蔽。
- 别上来就全端口大扫网段
- 别上来就所有机器挨个爆破
- 动静太大容易被发现
- 慢慢来，先摸清楚情况，再精准打击

### 建议四：保护好你的跳板机

你的跳板机就是你的根据地，
别丢了。
- 做个持久化（下一章讲）
- 别在跳板机上乱搞把它搞崩了
- 留好后路

## 10.7 本章总结

这一章我们讲了横向移动，
内容很多，也很核心。
总结一下：

**横向移动前的准备：**
- 收集凭据（密码、哈希、票据...）—— 核心！
- 内网信息收集（存活主机、端口、服务、域信息...）
- 准备工具

**常见的横向移动手法：**
- IPC$ + 计划任务 —— 老派经典
- PsExec —— 横向移动代名词
- Pass-the-Hash（哈希传递）—— 内网神器
- WMI —— 自带的，低调好用
- WinRM / PowerShell Remoting —— 越来越常见
- RDP 远程桌面 —— 图形界面，信息量大
- SSH —— Linux环境常用
- 数据库、中间件、各种服务 —— 都能成为跳板

**端口转发与代理：**
- 端口转发 —— 转发单个端口
- Socks代理 —— 把Kali放进内网，各种工具都能用
- 内网渗透必备技能

**域环境渗透：**
- 目标：拿下域控（DC）
- 核心思路：拿到域管理员凭据
- 方法：找域管理员登录过的机器、Kerberoast、委派攻击、漏洞利用...
- BloodHound —— 域渗透可视化神器

最后送你一句话：

> **内网渗透，
> 就像滚雪球。
> 只要你能进去第一台，
> 后面就会越滚越大。**
