## 9.1 应急响应基础

### 9.1.1 什么是应急响应

```
应急响应 = 出了安全事件之后的处理过程

就像生病了去医院：
  1. 挂号 → 先确认是什么病
  2. 检查 → 做各种化验、拍片
  3. 诊断 → 确定是什么病、严重程度
  4. 治疗 → 吃药、打针、手术
  5. 康复 → 出院、复查、注意事项

安全应急响应也是类似的流程。
```

### 9.1.2 应急响应的PDCERF模型

国际上比较通用的应急响应模型叫**PDCERF**：

```
P - Preparation     准备阶段
D - Detection       检测阶段
C - Containment     遏制阶段
E - Eradication     根除阶段
R - Recovery        恢复阶段
F - Follow-up       总结阶段
```

我们一个一个来看：

#### 1. Preparation（准备阶段）

```
出事之前就要做好准备：
  ✓ 应急预案（出事了谁负责、怎么处理）
  ✓ 工具准备（取证工具、查杀工具）
  ✓ 日志备份（万一被删了还有备份）
  ✓ 人员培训（知道出事了找谁）
  ✓ 保险（网络安全保险，出事了能赔点）
```

#### 2. Detection（检测阶段）

```
发现安全事件：
  ✓ 告警（IDS/IPS/WAF/EDR告警）
  ✓ 用户报告（电脑慢了、文件被加密了）
  ✓ 日志异常（有人半夜登录、大量失败登录）
  ✓ 流量异常（往外传大量数据）
  ✓ 第三方通知（监管、合作方、黑客自己发勒索信）

发现之后先确认：
  是不是真的被入侵了？
  还是误报？还是正常业务？
```

#### 3. Containment（遏制阶段）

```
确认被入侵了，先别忙着查，
先把攻击遏制住，不让影响扩大！

常见的遏制措施：
  • 断网（拔网线、禁用网卡）
  • 隔离（把被入侵的机器从网络里隔离开）
  • 改密码（把可能泄露的密码都改了）
  • 封禁IP（在防火墙/路由器上封掉黑客的IP）
  • 禁用账号（把可疑的账号先禁用）

⚠️ 注意：
  断网之前最好先抓内存、导日志，
  不然一关机/断网，很多证据就没了！
```

#### 4. Eradication（根除阶段）

```
把黑客彻底清理出去：
  • 删掉黑客留下的后门、木马
  • 删掉黑客新建的账号
  • 修复被利用的漏洞
  • 打补丁
  • 清理恶意文件

⚠️ 注意：
  一定要清理干净！
  不然你以为黑客走了，
  其实他留了好几个后门，
  你前脚刚清理完，他后脚又进来了。
```

#### 5. Recovery（恢复阶段）

```
把系统恢复正常：
  • 从备份恢复（干净的备份！）
  • 验证系统正常
  • 逐步恢复业务
  • 加强监控（防止再次被黑）

恢复顺序一般是：
  先恢复最重要的业务，
  再恢复次要的。
```

#### 6. Follow-up（总结阶段）

```
事情处理完了，要复盘：
  • 写应急响应报告
  • 分析根本原因（为什么会被入侵？）
  • 总结经验教训
  • 改进安全措施
  • 更新应急预案
  • 该处罚的处罚，该奖励的奖励

一句话：吃一堑，长一智。
```

### 9.1.3 应急响应的原则

```
应急响应几个重要原则：

1. 最小影响
   尽量不影响正常业务
   （但安全事件严重的话，该断网还得断网）

2. 保护证据
   处理过程中注意保护证据
   以后可能要报警、打官司
   证据不能乱改

3. 快速响应
   越快处理，损失越小
   拖得越久，黑客搞的破坏越大

4. 做好记录
   每一步操作都记下来
   时间、操作人、做了什么、结果是什么
   后面写报告要用

5. 及时上报
   该告诉领导的告诉领导
   该报警的报警
   该通知监管的通知监管
   别自己捂着，最后捂出大事
```

## 9.2 Windows入侵排查

发现Windows机器可能被入侵了，
从哪查起？

### 9.2.1 先看看账号

```bash
# 1. 看有哪些用户
net user

# 2. 看管理员组有哪些人
net localgroup administrators

# 3. 看最近登录的用户
net user 用户名 | findstr /B /C:"上次登录"

# 4. 看隐藏用户（注册表）
# 正常用户在这：
# HKEY_LOCAL_MACHINE\SAM\SAM\Domains\Account\Users\
# 用 reg query 查
reg query "HKLM\SAM\SAM\Domains\Account\Users\Names"
```

重点关注：
- 陌生的用户名（尤其是带$的隐藏用户）
- 不在你认知里的管理员
- 很久没用的账号突然登录了

### 9.2.2 看进程和服务

```bash
# 1. 看进程列表
tasklist
# 或者更详细的
tasklist /v

# 2. 看进程的路径（用wmic）
wmic process get name,processid,executablepath,commandline

# 3. 看启动项
# 方法1：msconfig（图形界面）
# 方法2：注册表
reg query "HKLM\Software\Microsoft\Windows\CurrentVersion\Run"
reg query "HKCU\Software\Microsoft\Windows\CurrentVersion\Run"

# 4. 看计划任务
schtasks /query /fo LIST /v

# 5. 看服务
net start
# 或者
wmic service get name,displayname,pathname,startmode
# 找路径里有可疑的、名字奇怪的服务
```

重点关注：
- 名字奇怪的进程（随机字母、和系统进程很像但不对的）
- 路径在临时目录、用户目录下的服务
- 可疑的计划任务（名字奇怪的、定时执行奇怪命令的）

### 9.2.3 看网络连接

```bash
# 1. 看所有网络连接
netstat -ano

# 解释一下：
# Proto → 协议（TCP/UDP）
# Local Address → 本地地址:端口
# Foreign Address → 远程地址:端口
# State → 状态（ESTABLISHED是已经建立连接）
# PID → 进程ID

# 2. 看哪个进程建立的连接
# 用刚才的PID去tasklist里找是哪个进程
tasklist | findstr "PID号"

# 3. 看路由表
route print
```

重点关注：
- 陌生的外部IP（尤其是境外IP）
- 异常的端口连接
- 大量对外连接（可能是在挖矿或者传数据）

### 9.2.4 看日志

Windows的日志存在事件查看器里。

```bash
# 命令行看事件日志（也可以用图形界面的eventvwr.msc）

# 1. 看系统日志
wevtutil qe System /c:100 /rd:true /f:text
# /c:100 看最近100条
# /rd:true 倒序（最新的在前）

# 2. 看安全日志（登录、权限相关）
wevtutil qe Security /c:100 /rd:true /f:text

# 3. 看应用程序日志
wevtutil qe Application /c:100 /rd:true /f:text
```

常用的事件ID：

| 事件ID | 说明 |
|--------|------|
| 4624 | 登录成功 |
| 4625 | 登录失败 |
| 4634 | 注销 |
| 4672 | 管理员登录（特殊权限登录） |
| 4720 | 创建用户 |
| 4724 | 改密码 |
| 4728 | 添加用户到组 |
| 4732 | 本地组添加成员 |
| 7045 | 安装服务 |
| 104 | 清除日志 |

> 💡 **重点看**：
> 有没有大量4625（登录失败）→ 可能在爆破
> 有没有陌生的4624（登录成功）→ 黑客进来了
> 有没有4720（创建用户）→ 黑客建了后门账号
> 有没有104（清除日志）→ 黑客擦屁股了

### 9.2.5 看文件

```bash
# 1. 看最近修改的文件
# 找最近几天修改过的exe、bat、ps1等
# （用搜索功能或者forfiles命令）
forfiles /p C:\ /s /m *.exe /d -7 /c "cmd /c echo @path @fdate @ftime"
# /d -7 表示7天内修改的

# 2. 看临时目录
dir C:\Windows\Temp\
dir C:\Users\用户名\AppData\Local\Temp\
# 病毒、木马喜欢藏在临时目录

# 3. 看启动目录
dir "C:\ProgramData\Microsoft\Windows\Start Menu\Programs\StartUp"
dir "C:\Users\用户名\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup"

# 4. 看回收站、最近打开的文件
# 最近打开的文件：
# C:\Users\用户名\Recent\
```

### 9.2.6 排查快速检查清单

```
Windows排查快速清单：

□ 用户检查
  □ net user 列出所有用户
  □ net localgroup administrators 看管理员
  □ 注册表看隐藏用户

□ 进程检查
  □ tasklist /v 看进程
  □ wmic process 看进程路径和命令行
  □ 有没有可疑进程

□ 启动项检查
  □ 注册表Run键
  □ 启动目录
  □ 计划任务
  □ 服务

□ 网络检查
  □ netstat -ano 看连接
  □ 有没有可疑外连

□ 日志检查
  □ 安全日志：登录成功/失败
  □ 系统日志：服务安装
  □ 有没有日志被清除

□ 文件检查
  □ 临时目录
  □ 最近修改的文件
  □ 可疑的exe/bat/ps1
```

## 9.3 Linux入侵排查

Linux被入侵了怎么查？

### 9.3.1 先看账号

```bash
# 1. 看所有用户
cat /etc/passwd

# 2. 看有哪些用户UID是0（root权限）
awk -F: '$3==0{print $1}' /etc/passwd
# 正常应该只有root一个

# 3. 看最近登录的用户
last
lastlog

# 4. 看当前登录的用户
who
w
# 看看有没有陌生的IP登录进来

# 5. 看sudo权限
sudo -l
# 或者看sudoers文件
cat /etc/sudoers
```

重点关注：
- UID为0的非root用户
- 陌生的用户名
- 不该有sudo权限的用户有了sudo权限

### 9.3.2 看进程和启动项

```bash
# 1. 看进程列表
ps aux
# 或者
ps -ef

# 2. 看进程的详细信息
# 看进程的可执行文件路径
ls -la /proc/进程号/exe
# 看进程启动的命令行
cat /proc/进程号/cmdline | tr '\0' ' '

# 3. 看启动项
# 开机自启服务
systemctl list-unit-files --type=service | grep enabled
# 或者老系统用
chkconfig --list | grep on

# 4. 看crontab（计划任务）
crontab -l
ls -la /etc/cron.d/
ls -la /etc/cron.daily/
ls -la /etc/cron.hourly/
cat /etc/crontab
# 所有用户的crontab
ls -la /var/spool/cron/crontabs/

# 5. 看rc.local
cat /etc/rc.local
cat /etc/rc.d/rc.local
```

重点关注：
- 名字奇怪的进程（随机字符串、伪装成系统进程的）
- 路径在/tmp、/dev/shm、用户目录下的进程
- 可疑的计划任务
- 陌生的系统服务

### 9.3.3 看网络连接

```bash
# 1. 看所有网络连接
netstat -tulnp
# 或者（推荐）
ss -tulnp

# 2. 看所有连接（包括已建立的）
netstat -an
ss -an

# 3. 看哪个进程在连接
lsof -i :端口号
# 比如 lsof -i :4444 看4444端口是什么进程在用

# 4. 看路由表
route -n
ip route show

# 5. 抓包看流量
tcpdump -i eth0 -w capture.pcap
# 抓一段时间的包，然后分析
```

重点关注：
- 可疑的对外连接（特别是高位端口）
- 监听的可疑端口
- 大量向外发数据的连接（可能在偷数据/挖矿）

### 9.3.4 看历史命令

```bash
# 1. 看当前用户的历史命令
history
cat ~/.bash_history

# 2. 看其他用户的
cat /home/用户名/.bash_history
cat /root/.bash_history

# 3. 看看有没有人删了历史记录
ls -la ~/.bash_history
# 大小为0或者时间不对就可疑
# 或者被软链接到/dev/null了
```

> ⚠️ **注意**：
> 黑客进来第一件事往往就是清历史记录，
> 所以.bash_history空了或者没了，
> 那很可能是被入侵了。

### 9.3.5 看文件

```bash
# 1. 看最近修改的文件
# 最近1天修改过的文件
find / -mtime -1 -type f 2>/dev/null

# 最近7天修改过的
find / -mtime -7 -type f 2>/dev/null

# 2. 看可疑目录
ls -la /tmp/
ls -la /dev/shm/
ls -la /var/tmp/
# 这些地方是黑客喜欢藏东西的地方

# 3. 看SUID文件（可能被留了后门）
find / -perm -4000 -type f 2>/dev/null
# 看看有没有新增的SUID文件

# 4. 看可执行文件
find /tmp /dev/shm /var/tmp -type f -executable 2>/dev/null
# 临时目录里的可执行文件都看看

# 5. 看.ssh目录（公钥后门？）
ls -la ~/.ssh/
cat ~/.ssh/authorized_keys
# 有没有陌生的公钥
```

### 9.3.6 看日志

```bash
# 主要日志文件都在 /var/log/ 目录下

# 1. 认证日志（登录相关，重点！）
# Debian/Ubuntu：
cat /var/log/auth.log
# CentOS/RHEL：
cat /var/log/secure

# 重点看：
#   - 大量Failed password → 暴力破解
#   - Accepted password → 登录成功（看看有没有陌生IP）
#   - sudo: ... → 谁用sudo执行了什么

# 2. 系统日志
cat /var/log/syslog     # Debian/Ubuntu
cat /var/log/messages   # CentOS/RHEL

# 3. 内核日志
dmesg
cat /var/log/kern.log

# 4. 历史日志（压缩包）
ls -la /var/log/*.gz
zcat /var/log/auth.log.2.gz
```

> 💡 **小技巧**：
> 日志很多，用grep过滤：
> ```bash
> # 只看登录成功的
> grep "Accepted" /var/log/auth.log
> 
> # 只看登录失败的
> grep "Failed password" /var/log/auth.log
> 
> # 按IP统计登录失败次数
> grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort | uniq -c | sort -rn
> ```

### 9.3.7 排查快速检查清单

```
Linux排查快速清单：

□ 用户检查
  □ /etc/passwd 看所有用户
  □ UID=0的用户
  □ last/lastlog 看登录记录
  □ who 看当前在线用户
  □ sudo权限

□ 进程检查
  □ ps aux 看进程
  □ /proc/xxx 看进程详情
  □ 有没有可疑进程

□ 启动项检查
  □ systemctl 看服务
  □ crontab 计划任务
  □ rc.local
  □ init.d脚本

□ 网络检查
  □ netstat / ss 看连接
  □ lsof 看端口对应进程
  □ 可疑外连

□ 日志检查
  □ auth.log/secure 登录日志
  □ syslog/messages 系统日志
  □ bash_history 历史命令

□ 文件检查
  □ /tmp /dev/shm 临时目录
  □ 最近修改的文件
  □ SUID文件
  □ .ssh/authorized_keys
```

## 9.4 内存取证——Volatility

内存取证就是从内存镜像里分析数据，
很多黑客会把东西藏在内存里，
不落地到硬盘，
这时候就需要内存取证了。

### 9.4.1 什么是内存取证

```
内存取证 = 把内存里的数据dump出来，然后分析

内存里有什么？
  • 所有运行的进程
  • 网络连接
  • 打开的文件
  • 密码（明文！比如mimikatz抓的就是内存里的）
  • 注入的代码
  • 没有存到硬盘的文件
  • ......

很多时候硬盘上找不到的东西，
内存里能找到。
```

### 9.4.2 怎么抓内存镜像

首先得把内存dump下来。

```
Windows下抓内存：
  • DumpIt（推荐，简单好用）
  • FTK Imager
  • WinPmem
  • .....

Linux下抓内存：
  • LiME
  • fmem
  • /dev/mem（老系统）
  • .....
```

一般应急响应的时候，
如果怀疑机器被入侵了，
**先抓内存镜像**，再做其他操作，
不然一重启内存里的数据就没了。

### 9.4.3 Volatility介绍

Volatility是最流行的内存取证框架，
功能超级强大。

Kali里有Volatility 2，
也可以自己装Volatility 3。

```bash
# 看看装了没
volatility -h
# 或者
vol -h
```

### 9.4.4 Volatility基本用法

假设你有一个内存镜像叫 `memdump.raw`。

```bash
# 第一步：先判断是什么系统的镜像
volatility -f memdump.raw imageinfo
# 会列出可能的系统配置
# 记下 Suggested Profile(s) 那一行
# 比如 Win7SP1x64

# 后面的命令都要加 --profile=配置名
```

#### 常用插件

```bash
# 1. 看进程列表
volatility -f memdump.raw --profile=Win7SP1x64 pslist
# 或者 pstree（树形显示，能看出父子进程关系）
volatility -f memdump.raw --profile=Win7SP1x64 pstree
# 或者 psscan（能找出被隐藏的进程）
volatility -f memdump.raw --profile=Win7SP1x64 psscan

# 2. 看网络连接
volatility -f memdump.raw --profile=Win7SP1x64 netscan
# 或者 connscan（老系统）
volatility -f memdump.raw --profile=Win7SP1x64 connscan

# 3. 看加载的驱动
volatility -f memdump.raw --profile=Win7SP1x64 modules

# 4. 看DLL（某个进程加载了哪些DLL）
volatility -f memdump.raw --profile=Win7SP1x64 dlllist -p 进程ID

# 5. 看打开的文件
volatility -f memdump.raw --profile=Win7SP1x64 filescan
# 太多的话可以grep过滤

# 6. 看注册表
volatility -f memdump.raw --profile=Win7SP1x64 hivelist
# 列出注册表蜂巢
# 然后可以用 printkey 看具体键值

# 7. 提权相关
volatility -f memdump.raw --profile=Win7SP1x64 getsids -p 进程ID
# 看进程的权限令牌

# 8. 命令行参数
volatility -f memdump.raw --profile=Win7SP1x64 cmdline
# 看每个进程是用什么命令启动的

# 9. 恶意代码检测
volatility -f memdump.raw --profile=Win7SP1x64 malfind
# 自动检测可疑的注入代码
# （不是100%准，但是能帮你筛选）

# 10. 抓密码哈希（神器！）
volatility -f memdump.raw --profile=Win7SP1x64 hashdump
# 直接从内存里抓SAM哈希
# 不用等进系统，不用提权

# 11. 抓明文密码（mimikatz插件）
volatility -f memdump.raw --profile=Win7SP1x64 mimikatz
# 直接从内存里抓明文密码！
# （前提是系统里有密码缓存在内存里）
```

### 9.4.5 内存取证的一般思路

```
拿到内存镜像后，一般按这个顺序查：

1. 先确定系统版本
   imageinfo

2. 看进程
   pslist / pstree / psscan
   → 找可疑进程

3. 看网络连接
   netscan / connscan
   → 找可疑外连

4. 看启动项/服务
   （从注册表里找）

5. 看可疑注入
   malfind
   dlllist

6. 看密码
   hashdump
   mimikatz

7. 深入分析
   把可疑进程dump出来
   procdump -p 进程ID -D 输出目录
   然后拿去做逆向、查杀毒
```

## 9.5 其他取证工具

### 9.5.1 磁盘取证

```
常用的磁盘取证工具：

1. FTK Imager
   → 镜像制作、预览
   → 可以只读方式挂载镜像
   → 不会修改原始证据

2. EnCase
   → 专业的取证软件
   → 功能强大
   → 商业软件，很贵

3. Autopsy / Sleuth Kit
   → 开源免费
   → 有图形界面
   → 功能也很全
   → 新手推荐

4. dd
   → 最原始的镜像制作工具
   → 直接按字节复制
   → Linux自带
```

### 9.5.2 日志分析工具

```
日志太多了看不过来？用工具：

1. ELK Stack（Elasticsearch + Logstash + Kibana）
   → 企业级日志分析平台
   → 存得多、搜得快、可视化好
   → 大型企业必备

2. Splunk
   → 商业日志分析平台
   → 功能强大，价格也贵

3. grep / awk / sed
   → Linux三驾马车
   → 简单日志分析直接用命令行就行
   → 灵活、快速、不用装软件

4. LogParser
   → 微软的日志分析工具
   → 可以用SQL语句查日志
   → Windows下好用

5. WinLogBeat
   → 把Windows日志发到ELK
```

### 9.5.3 恶意代码分析

```
发现可疑文件了怎么办？

1. 在线沙箱（最省事）
   • VirusTotal → 查几十款杀毒引擎
   • 微步沙箱
   • 哈勃分析系统
   • Joe Sandbox
   • Any.Run
   把文件传上去，自动分析行为

2. 静态分析
   • 查字符串
   • 查PE头、导入表
   • 查壳
   • 反编译（IDA Pro, Ghidra）

3. 动态分析
   • 装在虚拟机里跑
   • 看它做了什么（改注册表、连网、改文件...）
   • 抓包看通信
   • Procmon + Wireshark + Regshot 三件套
```

## 9.6 应急响应报告怎么写

事情处理完了，得写报告。

### 9.6.1 报告结构

```
一份完整的应急响应报告一般包含：

1. 事件概述
   • 什么时候、怎么发现的
   • 影响范围有多大
   • 事件等级（一般/较大/重大/特别重大）

2. 处置过程
   • 时间线（几点几分发生了什么）
   • 做了哪些操作
   • 谁做的
   • 结果是什么

3. 根因分析
   • 为什么会被入侵
   • 是哪个漏洞被利用了
   • 黑客是怎么进来的
   • 哪个环节出了问题

4. 影响评估
   • 哪些系统受影响
   • 数据有没有泄露
   • 业务中断了多久
   • 损失多少钱
   • 要不要报警、要不要上报监管

5. 整改建议
   • 短期（立刻要做的）
   • 中期（1-3个月）
   • 长期（半年以上）
   • 技术层面
   • 管理层面
   • 人员层面

6. 附录
   • 证据清单
   • 日志截图
   • 木马样本哈希
   • ......
```

### 9.6.2 写报告的注意事项

```
写报告注意几点：

1. 客观准确
   是什么就写什么，别猜测
   不确定的就写"疑似""可能"
   不要夸大也不要缩小

2. 时间线清晰
   按时间顺序写，
   几点几分做了什么，一目了然

3. 有凭有据
   每个结论都要有证据支撑
   截图、日志、文件...

4. 语言专业
   别用口语化的表达
   但也别堆砌术语
   让不懂技术的领导也能看懂

5. 整改建议要具体
   别写"加强安全管理"这种空话
   要写"升级XX系统到XX版本"
   "在XX位置部署XX设备"
   "XX日前完成XX工作"
```
