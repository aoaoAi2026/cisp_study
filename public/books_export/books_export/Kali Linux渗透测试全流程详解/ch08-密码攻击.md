## 8.1 先聊一聊密码

在讲工具之前，
先聊一聊密码本身。

### 8.1.1 为什么密码攻击这么有效？

为什么密码攻击这么有效？
原因很简单——
**人太懒了。**

很多人设置密码的时候，
为了好记，
就用很简单的密码。
比如：
```
123456
password
admin
qwerty
111111
123123
000000
生日
手机号
姓名+生日
公司名+123
......
```

还有很多人，
**所有网站用同一个密码。**
一个地方泄露了，
所有地方都完蛋。

所以在渗透测试中，
密码攻击经常能有惊喜。
尤其是内网环境，
服务器、数据库、交换机、路由器...
很多都是弱口令。

### 8.1.2 密码攻击的几种类型

密码攻击有几种常见的类型：

```
  1. 暴力破解（Brute Force）
     • 所有可能的组合都试一遍
     • 纯暴力，数字+字母+符号，各种组合
     • 缺点：太慢了，密码长一点就破不了
     • 现在纯暴力基本只用来破短密码

  2. 字典攻击（Dictionary Attack）
     • 用一个密码字典，一个一个试
     • 字典里是常用的密码、可能的密码
     • 比暴力破解高效得多
     • 也是最常用的方法
     • 只要密码在字典里，就能破出来
     • 密码不在字典里，就破不出来

  3. 社工密码学
     • 根据收集到的信息生成密码字典
     • 比如：姓名、生日、手机号、公司名、
       宠物名、纪念日...
     • 把这些信息组合起来
     • 成功率比普通字典高很多
     • 针对性强

  4. 彩虹表
     • 提前算好哈希和密码的对应表
     • 拿到哈希直接查表
     • 速度快，但需要很大的存储空间
     • 现在用的少了，加盐之后彩虹表基本废了
```

现在最常用的就是**字典攻击**。
字典的质量，
直接决定了破解的成功率。
所以一个好的字典非常重要。

### 8.1.3 弱密码 Top 10

给你看看全球最常见的弱密码Top 10，
年年排名都差不多：

```
1. 123456
2. 123456789
3. qwerty
4. password
5. 12345
6. 12345678
7. 111111
8. 1234567
9. dragon
10. 123123
```

这些密码，
基本上任何字典里都有。
如果你还用这些密码，
赶紧改了吧。

## 8.2 字典 —— 密码攻击的弹药库

字典是密码攻击的核心。
没有好字典，
再强的工具也白搭。

### 8.2.1 Kali 自带的字典

Kali里自带了一些字典，
在 `/usr/share/wordlists/` 目录下。

```
常用的几个：

  rockyou.txt
    • 最有名的字典之一
    • 来自RockYou公司的数据泄露
    • 1400多万个密码
    • 非常常用，很多人都用这个
    • Kali里是压缩的，要解压用：
      gunzip /usr/share/wordlists/rockyou.txt.gz

  dirb的小字典
    • /usr/share/dirb/wordlists/
    • 有几个小字典，也挺好用

  seclists（如果装了的话）
    • 很大的一个字典集合
    • 各种用途的字典都有
    • 推荐装一个
```

如果你刚入门，
rockyou.txt基本够用了。
字典不在大，
在于合适。

### 8.2.2 字典生成工具

有时候通用字典不够用，
需要针对性的字典。
这时候就需要字典生成工具了。

Kali里常用的有两个：
**Crunch** 和 **Cupp**。

#### Crunch —— 按规则生成字典

Crunch是一个按照规则生成字典的工具。
你可以指定长度、字符集、模式，
它给你生成对应的字典。

```
基础用法：

  crunch 最小长度 最大长度 字符集 [选项]

例子：

  # 生成所有4位数字的字典
  crunch 4 4 0123456789 -o 4digit.txt

  # 生成6位小写字母的字典
  crunch 6 6 abcdefghijklmnopqrstuvwxyz -o 6lower.txt

  # 生成8位，小写+数字的字典（会很大！）
  crunch 8 8 abcdefghijklmnopqrstuvwxyz0123456789 -o 8mix.txt

  # 生成指定前缀的，比如 abc 开头，后面3位数字
  crunch 6 6 0123456789 -t abc%%%
  （%代表数字，@代表小写字母，,代表大写，^代表符号）
```

Crunch适合知道密码的格式的时候用，
比如知道是6位数字，
或者知道前缀是什么，
用Crunch生成就很方便。

但要注意，
全组合的话字典会非常大，
8位混合字符的话有几十亿种可能，
根本用不完。

#### Cupp —— 社工字典生成神器

Cupp是一个社工密码生成工具。
什么意思呢？
就是你输入目标的一些信息，
比如：
- 姓名
- 生日
- 昵称
- 公司名
- 宠物名
- 喜欢的运动/球队
- 等等

Cupp会自动把这些信息各种组合，
生成可能的密码字典。

为什么这个好用？
因为很多人设置密码的时候，
都是用自己的信息组合的。
比如：
- 姓名+生日
- 昵称+123
- 公司名+年份
- 宠物名+特殊符号
- 等等

针对某个人生成的社工字典，
命中率比通用字典高得多。

用法也很简单：
```
cupp -i
```
然后它会一问一答让你输入各种信息，
输完就生成字典了。

做社工、做针对性攻击的时候，
Cupp非常好用。

### 8.2.3 社工密码学

说到社工字典，
就不得不提**社工密码学**。
简单说就是：
**研究人的密码设置习惯，
根据已知信息猜密码。**

这是个很有意思的领域，
也是最考验人的地方。
真正的高手，
不用暴力破解，
靠猜就能猜中很多人的密码。

常见的密码习惯：
```
  • 姓名拼音 + 生日 / 手机号后几位
  • 公司名 / 部门名 + 123 / 年份
  • 喜欢的明星 / 球队 + 数字
  • 宠物名 / 孩子名
  • 键盘上的连续字符（qwerty、1qaz2wsx...）
  • 纪念日、节日
  • 密码123、password、admin这类简单词
  • 同一套密码改改特殊符号、加个后缀
  • ......
```

如果是企业内部，
密码往往还有规律：
- 公司名+年份
- 部门+编号
- 欢迎语+123
- 等等

了解了这些规律，
再加上信息收集到的内容，
生成的字典命中率会很高。

## 8.3 Hydra —— 网络服务爆破神器

字典准备好了，
接下来就是爆破工具了。

第一个，
**Hydra（九头蛇）**，
最经典的网络服务密码爆破工具。
支持的协议超级多，
基本上你能想到的网络服务它都能爆破。

### 8.3.1 Hydra 能爆破什么？

Hydra支持的协议非常多，
列几个常用的：

```
  • SSH
  • FTP
  • Telnet
  • MySQL / PostgreSQL / Oracle / MSSQL...
  • SMB (Windows共享)
  • RDP (远程桌面)
  • HTTP Basic / HTTP Form
  • POP3 / IMAP / SMTP (邮件)
  • VNC
  • Redis
  • MongoDB
  • ......还有很多很多
```

基本上，
只要是有用户名密码登录的网络服务，
Hydra基本都支持。

### 8.3.2 Hydra 基础用法

Hydra的基本命令格式：

```bash
hydra -L 用户字典 -P 密码字典 目标IP 服务
```

或者单个用户名/密码：
```bash
hydra -l 用户名 -p 密码 目标IP 服务
```

来举几个常用的例子。

#### 例1：爆破SSH

```bash
hydra -L users.txt -P pass.txt 192.168.1.100 ssh
```

解释：
- `-L users.txt`：用户名字典文件
- `-P pass.txt`：密码字典文件
- `192.168.1.100`：目标IP
- `ssh`：爆破SSH服务

爆破成功了会显示出来。

#### 例2：爆破FTP

```bash
hydra -L users.txt -P pass.txt ftp://192.168.1.100
```

#### 例3：爆破MySQL

```bash
hydra -L users.txt -P pass.txt 192.168.1.100 mysql
```

#### 例4：爆破SMB（Windows共享）

```bash
hydra -L users.txt -P pass.txt 192.168.1.100 smb
```

#### 例5：爆破RDP（远程桌面）

```bash
hydra -L users.txt -P pass.txt 192.168.1.100 rdp
```

#### 例6：爆破HTTP表单登录

这个稍微复杂一点，
因为HTTP表单每个网站都不一样。
Hydra支持爆破HTTP POST表单登录，
但需要你知道登录的参数。

```bash
hydra -L users.txt -P pass.txt 目标IP http-post-form "/login.php:user=^USER^&pass=^PASS^:F=失败"

解释：
  http-post-form    HTTP POST表单
  /login.php        登录提交的URL
  user=^USER^       用户名字段，^USER^会被替换成字典里的用户名
  pass=^PASS^       密码字段，^PASS^会被替换成字典里的密码
  :F=失败           如果页面里有"失败"这个词，说明登录失败
                    （用来判断成功还是失败）
```

这个用的时候要先抓包，
看看登录请求是怎么发的，
成功和失败的响应有什么区别，
然后再写对应的参数。
稍微麻烦一点，
但也不难。

### 8.3.3 常用参数

再介绍几个Hydra常用的参数：

```
  -t 线程数      线程数，默认16，可以调大一点，速度更快
                 但别太大，容易封IP或者把服务搞崩
  -vV            显示详细过程，每个尝试都显示
  -f             找到一个就停止（爆破成功就停）
  -o 文件名      把结果保存到文件
  -s 端口        指定端口，如果不是默认端口的话
  -e nsr         额外尝试：空密码(n)、用户名即密码(s)、反向(r)
                 挺好用的，很多账号的密码就是用户名本身
```

比如：
```bash
hydra -L users.txt -P pass.txt -t 30 -f -e nsr 192.168.1.100 ssh
```

### 8.3.4 使用Hydra的注意事项

用Hydra有几个要注意的：

```
  1. 不要乱搞
     • 没有授权的情况下，不要去爆破别人的服务
     • 违法的！
     • 只在授权测试或者自己的靶机上用

  2. 控制速度
     • 线程不要开太大
     • 一来容易把目标服务搞崩
     • 二来容易触发防护、被封IP
     • 三来有的服务有登录失败次数限制，
       试多了账号会被锁

  3. 字典大小要合适
     • 字典太大的话，爆破要很久
     • 先拿小字典试试，不行再用大的
     • 优先用针对性的字典，命中率高

  4. 注意账号锁定
     • 很多系统登录失败几次就锁账号
     • 爆破前先搞清楚策略
     • 别把人账号全锁了，那就麻烦了
```

## 8.4 Hashcat —— 哈希破解神器

密码攻击另一个大方向就是**哈希破解**。

什么意思呢？
就是我们拿到了密码的哈希值（比如NTLM哈希、MD5哈希），
但没有明文，
这时候就要把哈希还原成明文密码。

为什么要破解哈希？
- 系统里存的都是哈希，不是明文
- 拿到哈希之后，想知道密码是什么
- 有的地方可以用哈希直接登录（Pass-the-Hash），但有的不行
- 而且密码明文可以用来试其他地方（密码复用）

哈希破解最强的工具就是 **Hashcat**。
它支持的哈希类型超级多，
而且支持GPU加速，
速度飞快。

### 8.4.1 常见的哈希类型

先看看常见的哈希类型有哪些。

```
常见的哈希类型：

  0     MD5
  100   SHA1
  1400  SHA2-256
  1700  SHA2-512
  3200  bcrypt（PHP、WordPress常用，比较慢）
  500   md5crypt / Unix MD5
  1800  sha512crypt / Unix SHA512
  1000  NTLM（Windows的密码哈希，非常常见）
  1100  MSSQL
  12    ASP.NET
  400   Wordpress（MD5加盐，其实是phpass）
  ......还有几百种
```

Hashcat基本上什么哈希都能破，
只要你知道是什么类型的。

### 8.4.2 Hashcat 基础用法

Hashcat的基本用法：

```bash
hashcat -m 哈希类型 -a 攻击模式 哈希文件 字典文件
```

解释：
- `-m`：指定哈希类型，数字，比如0就是MD5，1000就是NTLM
- `-a`：攻击模式
  - 0 = 字典攻击（Straight）—— 最常用
  - 1 = 组合攻击（Combination）—— 两个字典组合
  - 3 = 掩码攻击（Brute-force / Mask）—— 按规则暴力
  - 6 / 7 = 混合模式
- 哈希文件：保存哈希的文件，每行一个哈希
- 字典文件：密码字典

来举几个例子。

#### 例1：字典攻击破MD5

```bash
hashcat -m 0 -a 0 md5_hashes.txt rockyou.txt
```

- `-m 0`：MD5类型
- `-a 0`：字典攻击模式
- `md5_hashes.txt`：要破解的MD5哈希文件
- `rockyou.txt`：字典文件

破出来的结果会显示在屏幕上，
也可以用 `--show` 查看：
```bash
hashcat -m 0 md5_hashes.txt --show
```

#### 例2：破NTLM哈希

```bash
hashcat -m 1000 -a 0 ntlm_hashes.txt rockyou.txt
```

NTLM是Windows的密码哈希，
内网渗透中经常抓到。
NTLM破解速度很快，
GPU好的话每秒能跑几十亿次。

#### 例3：掩码攻击（知道密码格式的时候）

比如知道密码是8位数字，
或者是3个字母加4个数字...
这时候用掩码攻击比字典快。

```bash
# 6位纯数字
hashcat -m 0 -a 3 md5_hash.txt ?d?d?d?d?d?d

# 8位小写字母+数字
hashcat -m 0 -a 3 md5_hash.txt ?a?a?a?a?a?a?a?a
  （?a 代表所有可打印字符，太大了一般不用）

# 前3位小写字母，后4位数字
hashcat -m 0 -a 3 md5_hash.txt ?l?l?l?d?d?d?d
```

掩码字符：
```
?l = 小写字母 (a-z)
?u = 大写字母 (A-Z)
?d = 数字 (0-9)
?s = 特殊符号
?a = 所有字符（上面四个加起来）
?b = 0x00 - 0xff
```

知道密码格式的时候，
掩码攻击非常高效。

### 8.4.3 规则 —— 让字典更强大

Hashcat还有一个很牛的功能——
**规则（Rules）**。

什么意思呢？
就是拿一个基础字典，
然后按照规则对每个密码做各种变形，
比如：
- 首字母大写
- 后面加123
- 加个特殊符号
- 反转
- leet替换（a变@，o变0...）
- 等等

一个字典加个好规则，
能变出几十上百倍的密码量，
命中率大大提升。

用法：
```bash
hashcat -m 0 -a 0 md5_hashes.txt rockyou.txt -r rules/best64.rule
```

`-r` 指定规则文件。
Hashcat自带了一些规则，
也可以自己写。
`best64.rule` 是最常用的一个规则集，
效果不错。

规则是Hashcat的精髓之一，
用好规则，
能让你破解成功率大大提升。

### 8.4.4 速度 —— GPU vs CPU

Hashcat是支持GPU加速的，
也就是用显卡来跑，
速度比CPU快几十上百倍。

所以，
如果你有个好显卡（尤其是NVIDIA的），
破解速度会非常快。
MD5这种，
一块普通的显卡每秒就能跑几十亿次。

当然，
没有显卡也能用CPU跑，
就是慢一点。

### 8.4.5 哪些哈希好破，哪些难破？

不是所有哈希破解难度都一样，
差别非常大。

```
好破的（速度快，容易出结果）：
  • MD5、SHA1、NTLM、MySQL...
  • 速度快，GPU每秒几亿几十亿次
  • 字典够大的话，常见密码基本都能跑出来

难破的（慢，很难出结果）：
  • bcrypt
  • scrypt
  • Argon2
  • 加盐迭代次数多的哈希
  • 这些是专门设计成慢的，用来存密码的
  • 每秒可能只能跑几次、几十次
  • 字典大了根本跑不完
  • 这种的基本只能靠弱密码碰运气
```

所以现在推荐用bcrypt、Argon2这些慢哈希来存密码，
就是为了防破解。
如果网站用的是这些，
就算数据库泄露了，
密码也很难被破解。

## 8.5 John the Ripper —— 老牌密码破解工具

除了Hashcat，
还有一个老牌的密码破解工具——
**John the Ripper**，
简称John。

John也是经典中的经典，
功能也很强大。
它支持的格式也很多，
而且有很多自动检测的功能，
使用也比较方便。

John的特点是：
- 使用简单，很多时候不用指定哈希类型，它自动识别
- 有很多好用的脚本，比如unshadow、zip2john、rar2john...
- 社区强大，各种补丁、各种格式支持

不过速度方面，
尤其是GPU加速，
Hashcat更胜一筹。
所以现在哈希破解一般都用Hashcat了。
但John还是有它的用处，
比如处理一些特殊格式的时候，
John更方便。

简单提一下，
知道有这么个工具就行。
常用的用法：

```bash
# 破解密码（自动识别类型）
john 哈希文件

# 指定字典
john --wordlist=rockyou.txt 哈希文件

# 显示破解结果
john --show 哈希文件
```

还有各种辅助工具，
比如：
- `unshadow`：把passwd和shadow文件合并，用来破Linux密码
- `zip2john`：提取zip压缩包的哈希，然后破解zip密码
- `rar2john`：rar的
- `office2john`：Office文档的
- `pdf2john`：PDF的
- 等等

想破zip、rar、Office、PDF这些文件的密码，
John的这些小工具非常好用。
先提取哈希，
然后用Hashcat或者John去破。

## 8.6 无线网络密码破解

说到密码攻击，
很多人第一个想到的就是——
**蹭网/破WiFi密码**。

这个也算是密码攻击的一种吧。
Kali里也有对应的工具，
最有名的就是 **Aircrack-ng**。

### 8.6.1 WiFi密码破解的原理

WiFi（WPA/WPA2）密码怎么破？
简单说一下原理：

```
WPA/WPA2破解的大致流程：

  1. 开启网卡的监听模式
     让网卡能抓周围的WiFi数据包

  2. 抓握手包（WPA Handshake）
     当有设备连接WiFi的时候，
     会有一个四次握手的过程
     把这个握手包抓下来
     包里就包含了能用来破解密码的信息

  3. 如果没人连接，就把人踢下线，让他重连
     （叫Deauth攻击，取消认证攻击）
     一般人掉线了会自动重连
     这时候就能抓到握手包了

  4. 拿到握手包之后，用字典跑
     用Aircrack-ng或者Hashcat
     跑字典，一个一个试
     试对了就得到密码了
```

所以WPA/WPA2的破解，
本质上也是字典攻击。
密码在字典里就能破，
不在就破不了。
WPA3出来之后，
这种方法就不好使了。

### 8.6.2 工具和步骤

工具就是**Aircrack-ng**套件，
Kali里自带的。
步骤也不复杂，
但需要一块支持监听和注入的无线网卡。
不是所有网卡都支持，
买的时候要注意。

```
大致步骤（了解一下就行）：

  1. 看看无线网卡叫什么
     iwconfig

  2. 开启监听模式
     airmon-ng start wlan0
     （网卡名根据实际情况改）

  3. 扫周围的WiFi
     airodump-ng wlan0mon
     找到目标WiFi的BSSID和信道

  4. 抓握手包
     airodump-ng -c 信道 -w 文件名 --bssid 目标BSSID wlan0mon

  5. 开另一个终端，发Deauth包踢人
     aireplay-ng -0 10 -a 目标BSSID wlan0mon
     （10是发10个包，可以自己调）

  6. 等抓到手包之后，开始破
     aircrack-ng -w 字典文件 抓到的cap文件
```

大概就是这么个流程。
还是那句话，
学习研究可以，
别去蹭别人的网，
违法的。

## 8.7 给新手的几点建议

最后给新手几个建议。

### 建议一：字典是核心，社工是王道

密码攻击，
字典的质量比什么都重要。
- 通用字典是基础
- 针对目标的社工字典，命中率高得多
- 平时多收集、多整理自己的字典

而且，
很多时候不用爆破，
靠社工思路猜一猜，
可能就中了。
尤其是企业内网，
密码往往有规律。

### 建议二：控制速度，注意影响

爆破的时候，
不要光顾着快，
线程开最大、字典最大，
一顿猛扫。
- 容易把目标服务搞崩
- 容易触发防护、被封IP
- 容易锁账号
- 动静太大，容易被发现

悠着点，
先小范围试试，
再决定要不要大规模扫。

### 建议三：合法使用，别碰红线

这个说了好多遍了，
但还是要再说一遍：
**没有授权，绝对不要对任何系统进行密码攻击。**

不管是爆破、破解还是什么，
都是违法的。
轻则拘留罚款，
重则判刑。
学习研究可以，
用自己的靶机、自己的授权环境。

技术是中性的，
怎么用取决于人。
用对地方是安全研究员，
用错地方就是黑客罪犯。

## 8.8 本章总结

这一章我们讲了密码攻击，
内容不少，总结一下：

**密码攻击的类型：**
- 暴力破解、字典攻击、社工密码学、彩虹表
- 最常用的是字典攻击

**字典：**
- 常用字典：rockyou.txt
- 字典生成：Crunch（按规则生成）、Cupp（社工字典）
- 社工密码学：根据目标信息猜密码，命中率高

**Hydra：**
- 网络服务密码爆破神器
- 支持SSH、FTP、MySQL、SMB、RDP、HTTP...各种协议
- 基本用法：hydra -L 用户字典 -P 密码字典 目标 服务
- 注意控制速度、注意账号锁定

**Hashcat：**
- 哈希破解神器，支持GPU加速
- 字典攻击、掩码攻击、规则攻击
- 常用哈希类型：MD5(0)、NTLM(1000)、SHA1(100)...
- 规则让字典威力倍增

**John the Ripper：**
- 老牌密码破解工具
- 各种辅助工具（zip2john、rar2john、office2john...）很好用

**无线网络破解：**
- Aircrack-ng套件
- 抓WPA握手包 + 字典攻击
- 需要支持监听的无线网卡

最后送你一句话：

> **密码是人设置的，
> 所以最大的漏洞永远是人。**
