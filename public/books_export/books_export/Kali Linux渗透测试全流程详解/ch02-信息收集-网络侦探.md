> **"信息收集做得好，渗透测试没烦恼。"**

```
两个新手去面试渗透测试工程师：
```

## 2.1 信息收集到底是干啥的？

### 2.1.1 用大白话解释

想象一下，你要去一栋大楼里"执行任务"：

```
莽撞型选手（新手）：
  直接冲到大门前，咣咣敲门
  → 门不开就走了
  → 完全不知道侧门、后门、窗户在哪

聪明型选手（老司机）：
  先在大楼周围转三天：
  ✓ 大楼有几个入口？正门、侧门、后门？
  ✓ 保安什么时候换班？
  ✓ 哪些窗户没关？
  ✓ 员工从哪进？门禁卡长啥样？
  ✓ 地下室有没有通风管道能钻？
  ✓ ......
  收集完信息，挑最容易的地方进
```

信息收集，就是在正式攻击前，**尽可能多地了解目标**。

### 2.1.2 信息收集分两类

```
信息收集两大流派：

┌──────────────────────────────────┐
│  被动收集（Passive）              │
│  不直接碰目标，通过第三方获取信息  │
│  就像侦探蹲点观察，不打草惊蛇      │
│                                  │
│  优点：不会被发现                 │
│  缺点：信息可能不全               │
│                                  │
│  举例：                           │
│  • Google搜索                    │
│  • Whois查询                     │
│  • 查子域名                       │
│  • 刷微博、LinkedIn找员工信息     │
└──────────────────────────────────┘

┌──────────────────────────────────┐
│  主动收集（Active）               │
│  直接和目标打交道，发数据包过去    │
│  就像侦探直接敲门试探              │
│                                  │
│  优点：信息更准确                  │
│  缺点：可能被对方发现              │
│                                  │
│  举例：                           │
│  • 端口扫描（nmap）               │
│  • 目录扫描                       │
│  • 漏洞扫描                       │
│  • Ping探测                       │
└──────────────────────────────────┘
```

> 💡 **小建议**：实际操作的时候，先被动收集，
> 把能拿到的信息都拿到手，再开始主动扫描。
> 这样能最大限度地避免被发现。

### 2.1.3 我们要收集哪些信息

```
目标信息清单：

1. 网络资产
   ✓ 域名（主域名、子域名）
   ✓ IP地址、IP段
   ✓ 开放的端口和服务
   ✓ 用的什么Web服务器、什么框架

2. 人员信息
   ✓ 有哪些员工
   ✓ 邮箱地址
   ✓ 职位、部门
   ✓ （后面社工可能用得上）

3. 组织信息
   ✓ 公司是干啥的
   ✓ 有哪些分支机构
   ✓ 用的什么供应商

4. 敏感信息
   ✓ 泄露的密码
   ✓ 备份文件
   ✓ 配置文件
   ✓ 源代码
```

## 2.2 被动信息收集——Google Hacking

### 2.2.1 Google还能这么玩？

你可能天天用Google，但你知道Google有很多高级搜索技巧吗？

学会了这些技巧，你能用Google搜到很多你想不到的东西。

这就是**Google Hacking**，也叫**Google Dorks**。

### 2.2.2 常用搜索语法一个一个学

别着急，我们一个一个来，每个都举例子，保证你能看懂。

#### 语法1：site —— 限定在某个网站搜

```
用法：site:域名 关键词

例子：
  site:baidu.com 招聘
  → 只在百度的网站里搜"招聘"相关的

  site:gov.cn 密码
  → 在政府网站里搜"密码"

  site:github.com password
  → 在GitHub里搜密码（能搜到好多人不小心传上去的密码...）
```

实战练习：
> 打开Google，搜 `site:testphp.vulnweb.com`
> 看看能搜到这个测试网站的哪些页面

#### 语法2：inurl —— URL里包含关键词

```
用法：inurl:关键词

例子：
  inurl:admin
  → URL里面有admin的页面
  （大概率是后台登录页）

  inurl:login.php
  → 找PHP写的登录页面

  inurl:php?id=
  → URL里有 id= 的，可能有SQL注入
```

实战练习：
> 搜 `inurl:admin.php site:testphp.vulnweb.com`
> 看看能不能找到后台

#### 语法3：intitle —— 网页标题里有什么

```
用法：intitle:关键词

例子：
  intitle:后台登录
  → 网页标题是"后台登录"的

  intitle:"index of"
  → 目录浏览漏洞（能看到服务器上的文件列表）

  intitle:phpinfo
  → 找phpinfo页面（泄露PHP配置信息）
```

> 💡 **小知识**：如果关键词有空格，要用引号括起来
> 比如：`intitle:"后台管理系统"`

#### 语法4：filetype —— 搜指定类型的文件

```
用法：filetype:文件扩展名 关键词

例子：
  filetype:pdf 网络安全
  → 找PDF格式的网络安全文档

  filetype:xls 工资
  → 找Excel文件，里面可能有工资表...

  filetype:sql site:github.com
  → 在GitHub找SQL文件

  filetype:bak site:test.com
  → 找备份文件
```

#### 语法5：双引号 "" —— 精确匹配

```
用法："关键词"

例子：
  "默认密码"
  → 页面里完整出现"默认密码"这四个字的

  "username" "password"
  → 同时有username和password的页面
```

#### 语法6：减号 —— 排除不想看的

```
用法：-不想看的关键词

例子：
  site:baidu.com -新闻
  → 在百度搜，但是排除"新闻"相关的

  inurl:login -广告
  → 找登录页，但排除带"广告"的
```

### 2.2.3 组合拳——一起用才厉害

单个语法没那么厉害，组合起来才是真的强！

给你看几个经典的**Google Dorks**：

```
1. 找后台登录页
   inurl:admin | inurl:login | inurl:manage
   intitle:后台 | intitle:管理 | intitle:登录
   
2. 找敏感文件
   site:目标.com filetype:bak
   site:目标.com filetype:sql
   site:目标.com filetype:inc
   site:目标.com filetype:conf

3. 找泄露的密码
   site:github.com password smtp
   site:github.com "db_password"
   site:github.com "root:"

4. 找目录遍历
   intitle:"index of" /admin
   intitle:"index of" /backup
   intitle:"index of" /config

5. 找SQL注入点
   inurl:.php?id=
   inurl:.asp?id=
   inurl:.jsp?id=

6. 找phpinfo
   inurl:phpinfo.php
   intitle:phpinfo()
```

### 2.2.4 实战：用Google Dorks找靶机敏感信息

拿我们第一章搭的DVWA靶机做练习？

```
练习1：找后台

练习2：找可能有SQL注入的页面

练习3：找敏感文件
```

### 2.2.5 Shodan——黑客版的Google

如果说Google是搜网页的，那Shodan就是搜设备的。

```
Shodan能搜到啥？
  ✓ 连在公网上的服务器
  ✓ 网络摄像头（很多没密码...）
  ✓ 路由器
  ✓ 打印机
  ✓ 工控设备（想想就可怕）
  ✓ 任何连上网的设备
```

#### Shodan怎么用

打开官网：https://www.shodan.io

```
常用搜索语法：

1. 按产品搜
   product:nginx         → Nginx服务器
   product:apache        → Apache服务器
   product:MySQL         → MySQL数据库

2. 按端口搜
   port:22              → SSH服务
```

## 2.3 被动信息收集——域名信息收集

### 2.3.1 Whois查询——域名是谁的

买域名的时候，你得填注册人、邮箱、电话这些信息。
Whois就是用来查这些信息的。

#### 命令行查Whois

```bash
# 用法：whois 域名

# 试试查百度
kali@kali:~$ whois baidu.com

# 你会看到一大堆信息，重点看这几个：
Domain Name: BAIDU.COM          ← 域名
Registrar: MarkMonitor Inc.     ← 注册商
Registrant Organization: Beijing Baidu Netcom Science Technology Co., Ltd.  ← 注册人/公司
Registrant State/Province: Beijing   ← 省份
Registrant Country: CN               ← 国家
Name Server: ns1.baidu.com           ← DNS服务器
Creation Date: 1999-10-11            ← 注册时间
Expiration Date: 2026-10-11          ← 到期时间
```

#### 怎么利用Whois信息

```
Whois信息的用处：

1. 找注册人信息
   → 知道了注册人名字，后面社工可能用得上
   → 注册人邮箱，可能是个突破口

2. 找DNS服务器
   → 可以试试DNS域传送漏洞

3. 看域名年龄
   → 老域名一般信息更多
   → 新域名可能没什么历史信息

4. 找同注册人域名
   → 这个注册人还注册了哪些域名
   → 可能找到目标的其他资产
```

#### 在线Whois查询工具

除了命令行，也可以用网站查：
- https://whois.chinaz.com （站长工具）
- https://whois.aliyun.com （阿里云）
- https://www.whois.com （国外的）

### 2.3.2 子域名收集——找到更多目标

主站可能防护得很严，但是子域名呢？
```
baidu.com （主站，防护很严）
  ├── www.baidu.com      ← 百度首页
  ├── map.baidu.com      ← 百度地图
  ├── tieba.baidu.com    ← 百度贴吧
  ├── pan.baidu.com      ← 百度网盘
  ├── fanyi.baidu.com    ← 百度翻译
  └── ......
```

**子域名越多，攻击面越大**，说不定哪个子系统就有漏洞呢？

#### 方法1：搜索引擎找子域名

```
用site语法：
  site:baidu.com
  
然后你会看到各种各样的子域名，
把它们都收集起来。
```

#### 方法2：证书透明度（CT日志）

网站部署HTTPS证书的时候，会被记录到证书透明度日志里。
通过查这些日志，能找到很多子域名。

推荐几个在线工具：
- https://crt.sh （最常用的）
- https://censys.io （也能查）

```
crt.sh怎么用：
  打开 https://crt.sh
  输入 %.baidu.com （%是通配符）
  点搜索
  哗哗哗，一大堆子域名就出来了！
```

#### 方法3：子域名爆破

猜！用字典去猜，比如：
- www, mail, ftp, admin, test, dev, api, ...
- 把字典里的词一个个加到主域名前面，看能不能解析

Kali里有好多子域名爆破工具，我们介绍几个常用的。

##### 工具1：Sublist3r

```bash
# 安装（Kali里可能已经有了）
kali@kali:~$ sudo apt install sublist3r

# 使用方法
kali@kali:~$ sublist3r -d baidu.com -o baidu_subs.txt
# -d 指定主域名
# -o 结果保存到文件

# 它会从很多个地方收集子域名，
# 包括搜索引擎、DNS记录、证书透明度等等
```

##### 工具2：Amass（强烈推荐！）

Amass是OWASP的项目，功能超级强大。

```bash
# 安装
kali@kali:~$ sudo apt install amass

# 被动收集（不主动扫，从各数据源收集）
kali@kali:~$ amass enum -passive -d baidu.com -o baidu_passive.txt

# 主动收集（会实际去解析验证）
kali@kali:~$ amass enum -active -d baidu.com -o baidu_active.txt

# 结果会很多，几百个甚至几千个
```

##### 工具3：dnsenum

```bash
# 经典的DNS枚举工具
kali@kali:~$ dnsenum baidu.com

# 它会做这些事：
# 1. 查各种DNS记录（A、MX、NS、TXT...）
# 2. 子域名爆破（自带字典）
# 3. 尝试DNS域传送
# 4. 反向查找
```

#### 方法4：第三方聚合网站

有些网站专门收集子域名，可以直接去查：
- https://phpinfo.me/domain/ （在线子域名查询）
- https://dnsdumpster.com/ （DNSdumpster）
- https://subdomainfinder.c99.nl/ （国外的）

### 2.3.3 DNS记录查询——了解网络结构

DNS里藏着很多信息，我们来看看怎么查。

#### 常见DNS记录类型

| 记录类型 | 干啥的 | 例子 |
|----------|--------|------|
| A记录 | 域名 → IPv4 | baidu.com → 110.242.68.3 |
| AAAA记录 | 域名 → IPv6 | baidu.com → 240e:ff:b0... |
| CNAME | 域名 → 另一个域名 | www.baidu.com → www.a.shifen.com |
| MX记录 | 邮件服务器 | baidu.com → mx.baidu.com |
| NS记录 | DNS服务器 | baidu.com → ns1.baidu.com |
| TXT记录 | 随便写点啥 | SPF记录、验证信息 |
| PTR记录 | IP → 域名（反向） | 1.1.1.1 → one.one.one.one |

#### 查询DNS记录

```bash
# 方法1：nslookup
kali@kali:~$ nslookup baidu.com
Server:         192.168.1.1
Address:        192.168.1.1#53
Name:   baidu.com
Address: 110.242.68.3
Name:   baidu.com
Address: 220.181.38.148

# 方法2：dig（更强大，推荐！）
kali@kali:~$ dig baidu.com

# 查询MX记录
kali@kali:~$ dig baidu.com MX

# 查询所有记录
kali@kali:~$ dig baidu.com ANY

# 查询NS记录
kali@kali:~$ dig baidu.com NS

# 追踪DNS解析路径（看它是怎么一步步查到的）
kali@kali:~$ dig +trace baidu.com
```

### 2.3.4 DNS域传送漏洞——送你一整套子域名

DNS服务器为了同步数据，有个功能叫"域传送"（Zone Transfer）。

```
正常情况下：
  只有备DNS服务器能从主DNS同步数据

配置错误的情况下：
  任何人都能请求域传送
  → 直接拿到这个域的所有DNS记录
  → 所有子域名一次性到手！
```

#### 怎么测有没有域传送漏洞

```bash
# 第一步：先查NS记录（看DNS服务器是谁）
kali@kali:~$ dig baidu.com NS
# 结果里会列出来DNS服务器

# 第二步：对每个NS服务器测试域传送
kali@kali:~$ dig @ns1.baidu.com baidu.com AXFR
# AXFR就是域传送的意思

# 如果成功了，你会看到一大堆记录
# 如果失败了，会提示 Transfer failed 或者 connection refused
```

> 💡 **小知识**：现在这个漏洞已经很少见了，
> 但是万一碰到了，那就是大丰收！

### 2.4.1 为什么要收集人员信息

人是最薄弱的环节，很多时候从人身上突破比从技术上突破容易多了。

```
知道了员工信息能干嘛？
  1. 猜密码（姓名+生日、姓名拼音+123）
  2. 发钓鱼邮件（冒充领导、冒充IT）
  3. 电话社工（骗密码）
  4. ......
```

```bash
# 用法
kali@kali:~$ theHarvester -d 目标公司.com -b all

# -d 指定域名
# -b 指定数据源（all就是全部）

# 可选的数据源：
```

```
1. LinkedIn（领英）

2. 公司官网

   经常会有邮箱

3. 搜索引擎
   "公司名.com" email
   site:公司域名 email
   site:公司域名 邮箱
```

### 2.4.3 组织架构信息

```
怎么了解对方公司架构？

1. 公司官网

3. 年报/招股书
   上市公司会披露很多信息
   组织架构、业务系统、供应商...

4. 新闻报道
   公司最近的动态
   上了什么新项目
   用了什么新技术
```

## 2.5 主动信息收集——端口扫描详解

第一章我们初步学了Nmap，这一章我们深入讲讲。

### 2.5.1 先搞懂端口扫描原理

端口扫描的原理其实很简单：

```
我给目标的某个端口发个包过去：
  → 如果对方回复了 → 这个端口是开的
  → 如果对方没回复 → 这个端口是关的
```

就像你敲别人家的门：
- 有人开门 → 这个房间有人（端口开着）
- 没人应 → 没人（端口关了）

### 2.5.2 各种扫描方式的区别

Nmap支持好多种扫描方式，它们有啥区别呢？

#### 1. TCP连接扫描（-sT）—— 最老实的方式

```
原理：
  完整的TCP三次握手
  我SYN → 你SYN+ACK → 我ACK → 连接建立 → 我断开

优点：
  准确，不容易误报
  不需要root权限

缺点：
  会留下完整的连接日志，容易被发现
  速度慢
```

#### 2. SYN半开扫描（-sS）—— 最常用的方式

```
原理：
  只发SYN包，收到SYN+ACK就说明端口开着
  然后不回ACK，直接RST断开
  （连接没建立完整，所以叫"半开"扫描）

优点：
  速度快
  日志少，比较隐蔽
  准确性高

缺点：
  需要root权限

★ 这是Nmap默认的扫描方式（root用户下）
```

#### 3. UDP扫描（-sU）—— 扫UDP端口

```
TCP是有连接的，UDP是无连接的。
UDP扫描比较麻烦，因为UDP端口关了也不一定回包。

特点：
  速度慢（因为UDP不回复的话要等超时）
  容易漏报
  但有些服务只跑在UDP上，比如：
    DNS(53)、SNMP(161)、DHCP(67/68)
```

#### 4. 其他扫描方式（了解就行）

```
-sA  ACK扫描  → 判断端口有没有被过滤
-sW  窗口扫描 → 类似ACK扫描
-sN  Null扫描 → 没有任何标志位
-sF  FIN扫描  → 发FIN包
-sX  Xmas扫描 → 一堆标志位一起发

这些都是用来躲避IDS/IPS的，
新手先把-sS搞明白就行。
```

### 2.5.3 Nmap脚本引擎（NSE）—— 自动化神器

Nmap不只是能扫端口，它还有个强大的脚本引擎，
能自动做很多事情，比如：

```
✓ 扫漏洞
✓ 暴力破解
✓ 指纹识别
✓ 漏洞利用
✓ ......
```

#### 常用脚本分类

```
脚本都放在 /usr/share/nmap/scripts/ 目录下

常用分类：
  auth     → 认证相关（暴力破解等）
  brute    → 暴力破解
  default  → 默认脚本（扫描时加 -sC 就会跑这些）
  discovery → 信息发现
  dos      → 拒绝服务
  exploit  → 漏洞利用
  external → 调用外部服务
  fuzzer   → Fuzz测试
  intrusive → 侵入式的（可能会搞崩目标）
  malware  → 恶意代码检测
  safe     → 安全的（不会搞崩目标）
  vuln     → 漏洞扫描
```

#### 脚本使用方法

```bash
# 使用默认脚本（相当于 -sC）
kali@kali:~$ nmap -sC 目标IP

# 指定某个脚本
kali@kali:~$ nmap --script=http-title 目标IP
# 扫网页标题

# 指定一类脚本
kali@kali:~$ nmap --script=vuln 目标IP
# 跑所有漏洞相关的脚本

# 多个脚本用逗号分开
kali@kali:~$ nmap --script=http-title,http-headers 目标IP

# 查看脚本帮助
kali@kali:~$ nmap --script-help=http-title
```

#### 实战：用Nmap扫Web漏洞

```bash
# 扫HTTP相关的漏洞
kali@kali:~$ nmap --script=http-vuln* -p80 目标IP

# 扫SQL注入
kali@kali:~$ nmap --script=http-sql-injection -p80 目标IP

# 扫XSS
kali@kali:~$ nmap --script=http-xss -p80 目标IP
```

> ⚠️ **注意**：NSE脚本有的比较"猛"，可能会把目标搞崩，
> 用的时候小心点，特别是 intrusive 类的脚本。

### 2.5.4 扫描结果保存

扫描结果要保存下来，后面方便分析。

```bash
# 保存成普通文本
kali@kali:~$ nmap -oN result.txt 目标IP

# 保存成XML（方便导入其他工具）
kali@kali:~$ nmap -oX result.xml 目标IP

# 保存成Grep格式（方便grep搜索）
kali@kali:~$ nmap -oG result.gnmap 目标IP

# 三种格式都保存（oA = all）
kali@kali:~$ nmap -oA result 目标IP
# 会生成 result.nmap, result.xml, result.gnmap 三个文件
```

### 2.5.5 大规模扫描神器：Masscan

如果要扫的IP很多，Nmap可能有点慢。
Masscan号称"全世界最快的端口扫描器"，
据说5分钟就能扫完整个互联网。

```bash
# 安装
kali@kali:~$ sudo apt install masscan

# 用法：扫一个网段的80端口
kali@kali:~$ sudo masscan 192.168.1.0/24 -p80
# 速度：默认100包/秒，用--rate调速度

# 扫多个端口
kali@kali:~$ sudo masscan 192.168.1.0/24 -p22,80,443,3306

# 全端口扫描
kali@kali:~$ sudo masscan 192.168.1.0/24 -p1-65535 --rate=1000

# 保存结果
kali@kali:~$ sudo masscan 192.168.1.0/24 -p80 -oX result.xml
```

> 💡 **最佳实践**：
> 先用Masscan快速扫出哪些IP哪些端口开着，
> 然后再用Nmap对这些目标做详细扫描（服务版本、脚本等）。
> 这样又快又准。

## 2.6 主动信息收集——Web指纹识别

### 2.6.1 啥是指纹识别

看到一个网站，你得先搞清楚：
- 用的什么Web服务器？（Apache？Nginx？IIS？）
- 用的什么语言？（PHP？Java？Python？）
- 用的什么框架？（WordPress？Drupal？ThinkPHP？）
- 用的什么CMS？（什么版本？）

这些信息就是网站的"指纹"。

**知道了指纹有啥用？**
```
知道了CMS是WordPress 5.0
  → 去搜WordPress 5.0有什么漏洞
  → 找到对应的Exp
  → 直接打过去
```

### 2.6.2 手动识别指纹

#### 1. 看HTTP响应头

```bash
# 用curl看响应头
kali@kali:~$ curl -I http://目标网站.com

HTTP/1.1 200 OK
Server: nginx/1.18.0           ← Nginx服务器，版本1.18.0
Date: Tue, 01 Jan 2024 12:00:00 GMT
Content-Type: text/html; charset=UTF-8
Connection: keep-alive
X-Powered-By: PHP/7.4.3        ← PHP 7.4.3
Set-Cookie: PHPSESSID=abc123... ← PHP的Session
```

#### 2. 看页面源代码

```
右键 → 查看网页源代码
找这些特征：
  <!-- Powered by xxx -->
  <meta name="generator" content="WordPress 5.0">
  wp-content/  → WordPress
  /skin/       → 可能是织梦CMS
  /plus/       → 织梦CMS
  /zb_users/   → Z-Blog
  /template/   → 可能是各种CMS
```

#### 3. 看URL特征

```
WordPress特征：
  /wp-login.php
  /wp-admin/
  /wp-content/

Drupal特征：
  /user/login
  ?q=node/

Joomla特征：
  /administrator/
  /templates/

ThinkPHP特征：
  /index.php?s=/
  /index.php/模块/控制器/方法
```

### 2.6.3 自动指纹识别工具

#### 工具1：WhatWeb

```bash
# 基本用法
kali@kali:~$ whatweb http://目标.com

# 你会看到一堆信息，比如：
http://目标.com [200 OK] 
Apache[2.4.6], 
PHP[7.1.33], 
WordPress[5.4.2],
...

# 深度扫描（加 -a 3）
kali@kali:~$ whatweb -a 3 http://目标.com
# -a 1 最轻量，-a 4 最深度
# 数字越大越详细，但越慢

# 批量扫描
kali@kali:~$ whatweb -i targets.txt -o result.html
# 从文件读目标，输出HTML报告
```

#### 工具2：Wappalyzer

Wappalyzer是个浏览器插件，装上去打开网站自动显示指纹。

```
Firefox/Chrome扩展商店搜 Wappalyzer
装上就行，打开任何网站都会显示它用了什么技术。
```

#### 工具3：御剑指纹识别

国产工具，对国内CMS识别得比较准。
（Kali里没自带，需要自己下载）

## 2.7 主动信息收集——目录扫描

### 2.7.1 啥是目录扫描

网站有很多页面和目录，有的没有链接，你直接访问不到。
但如果文件存在，直接输入URL还是能打开的。

目录扫描就是：**拿个字典，一个个试，看哪些文件/目录存在**。

```
比如字典里有这些词：
  admin
  login
  backup
  config.php
  phpinfo.php
  robots.txt
  ......

工具一个个去访问：
  http://目标.com/admin   → 200 OK  → 存在！
  http://目标.com/login   → 200 OK  → 存在！
  http://目标.com/backup  → 404     → 不存在
  ......
```

### 2.7.2 目录扫描工具

#### 工具1：dirb

Kali自带的老牌目录扫描工具。

```bash
# 基本用法
kali@kali:~$ dirb http://目标.com/

# 指定字典
kali@kali:~$ dirb http://目标.com/ /usr/share/wordlists/dirb/common.txt

# 不显示403的（只看找到的）
kali@kali:~$ dirb http://目标.com/ -N 403

# 递归扫描（找到目录继续往下扫）
kali@kali:~$ dirb http://目标.com/ -r
```

#### 工具2：Gobuster

Go语言写的，速度很快，也很常用。

```bash
# 安装
kali@kali:~$ sudo apt install gobuster

# 目录扫描模式（dir模式）
kali@kali:~$ gobuster dir -u http://目标.com/ -w 字典路径

# 子域名模式（dns模式）
kali@kali:~$ gobuster dns -d 目标.com -w 字典路径

# 常用参数：
  -u   指定URL
  -w   指定字典
  -t   线程数（默认10，大点速度快）
  -x   指定文件后缀，比如 -x php,txt,html
  -o   结果保存到文件

# 例子：扫PHP文件，用50线程
kali@kali:~$ gobuster dir -u http://目标.com/ \
  -w /usr/share/wordlists/dirb/common.txt \
  -x php -t 50 -o result.txt
```

#### 工具3：dirsearch

Python写的，功能很强大，推荐！

```bash
# 安装
kali@kali:~$ sudo apt install dirsearch

# 或者从GitHub下载
git clone https://github.com/maurosoria/dirsearch.git

# 基本用法
kali@kali:~$ dirsearch -u http://目标.com/

# 常用参数：
  -u  URL
  -w  字典
  -e  扩展名（php,asp,jsp...）
  -t  线程
  -x  排除的状态码（-x 404,403）
  -r  递归
  --random-agents  随机User-Agent

# 例子
kali@kali:~$ dirsearch -u http://目标.com/ -e php,html -t 100
```

### 2.7.3 字典从哪来

Kali自带了一些字典：

```bash
# dirb的字典
/usr/share/wordlists/dirb/
  ├── common.txt    常用的，适合快速扫
  ├── big.txt       大字典，扫得全但慢
  └── ...

# dirbuster的字典
/usr/share/wordlists/dirbuster/

# rockyou（密码字典，超级大）
# 需要先解压
kali@kali:~$ sudo gunzip /usr/share/wordlists/rockyou.txt.gz
```

> 💡 **小建议**：
> 先用小字典快速扫一遍，
> 没找到东西再用大字典慢慢扫。
> 字典不是越大越好，适合的才是最好的。

```bash
# 启动
```

## 2.9 信息收集实战演练

说了这么多，我们来实际走一遍流程。

### 2.9.1 目标：testphp.vulnweb.com

Acunetix官方的测试靶站，允许我们测试。

### 2.9.2 第一步：被动信息收集

```bash
# 1. Whois查询
whois testphp.vulnweb.com

# 2. 子域名收集（这是测试站，可能子域名不多）
sublist3r -d testphp.vulnweb.com -o subs.txt

# 3. Google Dorks
打开Google搜：
  site:testphp.vulnweb.com
  site:testphp.vulnweb.com inurl:admin
  site:testphp.vulnweb.com filetype:sql

# 4. 查DNS记录
dig testphp.vulnweb.com ANY
```

### 2.9.3 第二步：主动信息收集

```bash
# 1. 端口扫描
nmap -sV -sC -p- -T4 testphp.vulnweb.com -oA nmap_result

# 2. Web指纹识别
whatweb -a 3 http://testphp.vulnweb.com

# 3. 目录扫描
gobuster dir -u http://testphp.vulnweb.com/ \
  -w /usr/share/wordlists/dirb/common.txt \
  -x php -t 50 -o dir_result.txt
```

把收集到的信息整理成表格：

```
目标信息汇总表：

└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ 发现的目录/文件                      │
```

```
   → 可以试试
```

## 2.10 新手常见问题

### Q1：信息收集要做到什么程度才算完？

A：理论上信息是无限的，永远收集不完。
一般来说，达到这几点就差不多了：
- ✓ 子域名都找得差不多了
- ✓ 主要资产的端口和服务都扫了
- ✓ 知道目标用的什么技术栈
- ✓ 有几个明确的攻击切入点

然后就可以进入下一步了，边打边继续收集。

### Q2：会不会扫着扫着就被封了？

A：有可能。所以：
- 不要用太快的速度（默认T3/T4就行）
- 不要一下子扫全端口（先扫常用端口）
- 可以用代理或者VPN
- 只在授权范围内扫

### Q3：工具太多了，记不住怎么办？

A：不用记，常用的就那几个：
- 子域名：amass, sublist3r
- 端口：nmap, masscan
- 目录：gobuster, dirsearch
- 指纹：whatweb, wappalyzer

其他的需要的时候再查就行。

### Q4：为什么我扫出来的结果和别人不一样？

A：可能原因：
1. 字典不一样
2. 扫描速度不一样（太快可能漏报）
3. 目标那边做了防护（WAF、封IP）
4. 目标环境变化了

很正常，多换几个工具试试。
