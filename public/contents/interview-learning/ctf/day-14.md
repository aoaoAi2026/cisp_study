## 一、F12开发者工具复习

### 1.1 核心功能

```
F12开发者工具核心模块：

1. Elements（元素）：
   - 查看和修改HTML结构
   - 修改CSS样式
   - 实时预览

2. Console（控制台）：
   - 执行JavaScript代码
   - 查看日志输出
   - 使用fetch发送请求

3. Sources（源代码）：
   - 查看网页源代码
   - 设置断点调试
   - 查看资源文件

4. Network（网络）：
   - 查看所有网络请求
   - 分析请求和响应
   - 查看资源加载时间

5. Application（应用）：
   - 查看存储数据
   - 管理Cookie
   - 查看本地存储
```

### 1.2 常用操作

```
常用操作：
  - Ctrl+Shift+I：打开开发者工具
  - Ctrl+U：查看页面源代码
  - Ctrl+R：刷新页面
  - Ctrl+Shift+R：强制刷新
  - F12：打开开发者工具

Network面板操作：
  - 勾选"Preserve log"保留日志
  - 右键请求→Copy as fetch
  - 查看请求头和响应头
```

### 1.3 实战技巧

```
实战技巧：
  1. 使用fetch修改请求方法
  2. 修改Cookie绕过权限
  3. 追踪302跳转获取Flag
  4. 修改User-Agent伪装浏览器
  5. 使用Console测试Payload
```

## 二、Burp Suite核心模块复习

### 2.1 核心模块

```
Burp Suite核心模块：

1. Proxy（代理）：
   - 抓包工具
   - 拦截请求和响应
   - 修改请求内容

2. Repeater（重放器）：
   - 修改请求后重新发送
   - 测试不同参数
   - 调试漏洞

3. Intruder（入侵者）：
   - 自动化爆破工具
   - 暴力破解密码
   - 测试参数值

4. Decoder（解码器）：
   - 编码解码工具
   - Base64、URL、Hex等
   - 支持多种编码

5. Spider（爬虫）：
   - 自动爬取网站
   - 发现隐藏页面
   - 构建站点地图

6. Scanner（扫描器）：
   - 自动扫描漏洞
   - 检测SQL注入、XSS等
```

### 2.2 快捷键汇总

```
常用快捷键：
  - Ctrl+R：发送到Repeater
  - Ctrl+I：发送到Intruder
  - Ctrl+S：发送请求（Repeater）
  - Ctrl+Space：切换拦截状态
  - Ctrl+E：清空请求列表（Proxy）
```

### 2.3 操作流程

```
标准操作流程：
  1. Proxy抓包
  2. 发送到Repeater
  3. 修改参数测试
  4. 如果需要爆破，发送到Intruder
  5. 使用Decoder解码内容
```

## 三、HTTP协议知识点复习

### 3.1 请求方法

```
常见请求方法：
  - GET：获取资源
  - POST：提交数据
  - PUT：更新资源
  - DELETE：删除资源
  - HEAD：获取响应头
  - OPTIONS：查询支持的方法

CTF考点：
  - 修改请求方法
  - POST参数注入
  - GET参数注入
```

### 3.2 状态码

```
常见状态码：
  - 200 OK：成功
  - 301 Moved Permanently：永久重定向
  - 302 Found：临时重定向
  - 400 Bad Request：请求错误
  - 401 Unauthorized：未认证
  - 403 Forbidden：禁止访问
  - 404 Not Found：资源不存在
  - 500 Internal Server Error：服务器错误

CTF考点：
  - 302跳转追踪
  - 403绕过
  - 404目录扫描
```

### 3.3 请求头

```
常见请求头：
  - Host：目标域名
  - User-Agent：客户端标识
  - Referer：来源页面
  - Cookie：会话状态
  - Content-Type：内容类型
  - X-Forwarded-For：真实IP

CTF考点：
  - User-Agent伪装
  - Referer伪造
  - Cookie修改
  - X-Forwarded-For伪造
```

### 4.3 答案对照

```
测试1答案：
  - 1xx：信息响应
  - 2xx：成功
  - 3xx：重定向
  - 4xx：客户端错误
  - 5xx：服务器错误

测试2答案：
  | 对比项 | GET | POST |
  |-------|-----|------|
  | 参数位置 | URL中 | 请求体中 |
  | 长度限制 | 有 | 无 |
  | 安全性 | 低 | 高 |
  | 缓存 | 可缓存 | 不可缓存 |

测试3答案：
  - Secure：只允许HTTPS传输
  - HttpOnly：禁止JavaScript读取
  - SameSite：限制跨站发送

测试4答案：
  1. Proxy：抓包工具
  2. Repeater：修改重放
  3. Intruder：自动化爆破
  4. Decoder：编码解码

测试5答案：
  - 判断列数：ORDER BY n
  - 联合查询：UNION SELECT 1,2,3
  - 查询数据库名：UNION SELECT 1,database(),3
  - 查询表名：UNION SELECT 1,group_concat(table_name),3 FROM information_schema.tables
```
