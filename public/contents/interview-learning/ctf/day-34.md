## 一、音频隐写术基础概念

### 1.1 什么是音频隐写？

```
【通俗易懂的解释】

想象你在听收音机：
  - 广播里播放着音乐
  - 但你不知道的是，音乐里藏着秘密信息
  - 只有知道方法的人才能提取出来

这就像"看不见的信"：
  - 表面是一首正常的歌曲
  - 实际上里面隐藏着数据
  - 听觉上完全察觉不到

音频隐写的原理：
  - 人耳有听觉限制
  - 有些频率人耳听不到（超声波）
  - 有些细微变化人耳分辨不出
  - 把信息藏在这些"盲区"里

为什么音频适合隐写？
  - 音频文件普遍存在
  - 不容易引起怀疑
  - 可以隐藏大量数据
  - 隐蔽性强
```

### 1.2 音频隐写的常见方法

```
【音频隐写技术分类】

方法1：LSB隐写（最低位隐写）
  原理：
    - 音频数据由采样点组成
    - 每个采样点有16位或24位
    - 最低位的变化人耳几乎听不出
    - 把信息藏在最低位

  比喻：
    - 就像在书的页码上做微小标记
    - 读书的人不会注意
    - 但知道的人能读出信息

  特点：
    - 容量大，可以藏很多数据
    - 不影响音质
    - 但容易被检测

方法2：频谱图隐写
  原理：
    - 音频可以转换为频谱图
    - 频谱图是一张"图片"
    - 可以在图片上写字或画图
    - 转回音频后信息被隐藏

  比喻：
    - 就像在声波的"照片"上写字
    - 看频谱图就能看到信息
    - 但听音频完全不知道

  特点：
    - 非常直观
    - 信息明显（看频谱图）
    - CTF中最常见

方法3：回声隐写
  原理：
    - 在音频中添加微弱回声
    - 回声的延迟编码信息
    - 人耳听不出差异

  比喻：
    - 就像在山谷里喊话
    - 回声的快慢代表不同意思
    - 只有知道规则的人能解码

方法4：相位编码
  原理：
    - 修改音频的相位信息
    - 相位变化不影响听感
    - 可以隐藏数据

方法5：扩频隐写
  原理：
    - 把信息分散到整个频谱
    - 能量很低，难以检测
    - 抗干扰能力强
```

### 1.3 音频文件格式详解

```
【常见音频格式】

1. WAV格式
  特点：
    - 无损格式
    - 音质最好
    - 文件较大
    - 最适合隐写

  文件结构：
    RIFF头（4字节）：52 49 46 46（"RIFF"）
    文件大小（4字节）
    WAVE标识（4字节）：57 41 56 45（"WAVE"）
    fmt 子块：格式信息
    data 子块：音频数据

  CTF中的重要性：
    - 大多数音频隐写题用WAV
    - 可以直接分析原始数据
    - LSB隐写最常见

2. MP3格式
  特点：
    - 有损压缩格式
    - 文件小
    - 不适合LSB隐写
    - 但可以在ID3标签藏信息

  文件头：
    ID3标签：49 44 33（"ID3"）
    音频帧：FF FB 或 FF FA

  隐写方式：
    - ID3标签中隐藏信息
    - 歌词、封面图片中藏数据

3. FLAC格式
  特点：
    - 无损压缩
    - 音质好，文件适中
    - 可以隐藏信息

4. OGG格式
  特点：
    - 开源格式
    - 常见于游戏
    - 文件头：4F 67 67 53（"OggS"）
```

## 三、频谱图分析原理与实战

### 3.1 频谱图原理详解

```
【频谱图是如何生成的？】

音频的本质：
  - 声音是振动
  - 振动产生波形
  - 波形包含不同频率

频谱图的生成过程：
  1. 音频采样
     - 把连续的声音变成数字
     - 采样率：44100Hz = 每秒44100个采样点

  2. 分帧
     - 把音频分成小段（帧）
     - 每帧约20-50毫秒

  3. 窗函数
     - 给每帧加窗函数
     - 减少频谱泄漏

  4. FFT变换
     - 快速傅里叶变换
     - 把时域信号变成频域信号
     - 这就是"魔法"所在

  5. 计算能量
     - 计算每个频率的能量
     - 转换成颜色深浅

  6. 生成图片
     - 横轴：时间
     - 纵轴：频率
     - 颜色：能量强度

【生活化比喻】

想象你在听交响乐：
  - 时域波形：就像看到所有乐器一起演奏
  - 频谱图：就像把每个乐器的声音分开
  - 你能看到哪个时间哪个频率有声音

频谱图就像"音乐的X光片"：
  - 表面看是音乐
  - X光片（频谱图）能看到内部结构
  - 隐藏的信息就像X光片上的异常
```

### 3.2 频谱图隐写检测方法

```
【如何发现频谱图隐写？】

步骤1：打开音频文件
  - 用Audacity打开
  - 观察波形是否正常

步骤2：切换到频谱图视图
  - 点击轨道下拉菜单
  - 选择"频谱图"

步骤3：观察频谱图
  查看是否有：
    - 明显的文字
    - 二维码图案
    - 异常的线条或图案
    - 不自然的矩形区域

步骤4：调整参数
  - 尝试不同的FFT大小
  - 放大查看细节
  - 调整颜色范围

步骤5：截图分析
  - 如果看到二维码，截图保存
  - 用二维码扫描工具识别
  - 如果是文字，直接读取

【常见频谱图隐写类型】

类型1：直接文字
  特征：
    - 频谱图上直接显示文字
    - 通常是flag
    - 一眼就能看到

  解题：
    - 直接读取文字内容

类型2：二维码
  特征：
    - 频谱图上有黑白方块
    - 像二维码的图案

  解题：
    - 截图保存
    - 用二维码扫描工具识别
    - 可能需要调整对比度

类型3：摩斯密码
  特征：
    - 频谱图上有规律的长短线条
    - 像摩斯密码的点划

  解题：
    - 记录长短线条
    - 翻译成摩斯密码
    - 解码得到flag

类型4：高低频分离
  特征：
    - 高频和低频分别有信息
    - 需要分别查看

  解题：
    - 使用高通/低通滤波器
    - 分离不同频段
    - 分别分析
```

## 四、常见音频隐写技术

### 4.1 LSB隐写详解

```
【LSB隐写原理】

什么是LSB？
  LSB = Least Significant Bit = 最低有效位

音频数据结构：
  - 每个采样点16位（或24位）
  - 例如：1011001011010110
  - 最高位影响大，最低位影响小
  - 最低位从0变1，声音几乎没变化

LSB隐写原理：
  原始数据：1011001011010110
  隐藏信息：1
  修改后：  1011001011010111
  只改了最后一位，人耳听不出区别

【生活化比喻】

想象一本书：
  - 每页有很多字
  - 你只改每个字的最后一笔
  - 比如把"大"改成"太"
  - 读者几乎看不出区别
  - 但知道的人能读出隐藏信息

【LSB隐写检测】

方法1：使用工具
  工具：StegSolve、SilentEye、OpenStego
  操作：
    - 打开音频文件
    - 选择LSB提取
    - 查看提取的数据

方法2：使用Python脚本
  代码示例：
    import wave

    def extract_lsb(wav_file):
        with wave.open(wav_file, 'rb') as w:
            frames = w.readframes(w.getnframes())
            bits = []
            for byte in frames:
                bits.append(byte & 1)
            # 每8位组成一个字节
            chars = []
            for i in range(0, len(bits), 8):
                byte = bits[i:i+8]
                chars.append(chr(int(''.join(map(str, byte)), 2)))
            return ''.join(chars)

方法3：使用Audacity
  操作：
    - 打开音频
    - 放大查看波形
    - 观察是否有规律的变化
    - LSB隐写可能产生微小抖动
```

### 4.3 音频波形隐写

```
【波形隐写原理】

什么是波形？
  - 音频的振幅随时间变化
  - 形成波形图
  - 波形可以编码信息

波形隐写方法：

方法1：波形高度编码
  原理：
    - 波形的高度代表振幅
    - 可以用高度编码二进制
    - 高振幅 = 1，低振幅 = 0

  检测方法：
    - 用Audacity查看波形
    - 观察振幅是否有规律变化
    - 记录高低变化，转成二进制

方法2：波形方向编码
  原理：
    - 波形有上下两部分
    - 上半部分代表1
    - 下半部分代表0

  检测方法：
    - 查看波形方向
    - 记录上下变化

方法3：静音间隔编码
  原理：
    - 在音频中插入静音
    - 静音的长短代表不同信息
    - 类似摩斯密码

  检测方法：
    - 查看波形中的静音区域
    - 测量静音时长
    - 解码信息

【实战案例】

题目：wave_secret.wav
描述：波形中藏着秘密

解题步骤：
  1. 用Audacity打开
  2. 观察波形图
  3. 发现波形高度有规律变化

1. LSB提取脚本
```python
import wave

def extract_lsb(wav_file, output_file):
    """
    从WAV文件提取LSB隐写数据
    """
    with wave.open(wav_file, 'rb') as w:
        # 获取音频参数
        n_channels = w.getnchannels()
        sample_width = w.getsampwidth()
        framerate = w.getframerate()
        n_frames = w.getnframes()
        
        # 读取所有帧
        frames = w.readframes(n_frames)
        
        # 提取LSB
        bits = []
        for i in range(0, len(frames), sample_width):
            # 获取采样值
            sample = int.from_bytes(
                frames[i:i+sample_width], 
                byteorder='little', 
                signed=True
            )
            # 提取最低位
            bits.append(sample & 1)
        
        # 每8位组成一个字节
        bytes_data = []
        for i in range(0, len(bits), 8):
            byte_bits = bits[i:i+8]
            if len(byte_bits) == 8:
                byte_value = int(''.join(map(str, byte_bits)), 2)
                bytes_data.append(byte_value)
        
        # 写入文件
        with open(output_file, 'wb') as f:
            f.write(bytes(bytes_data))
        
        print(f"提取完成，保存到 {output_file}")

# 使用方法

import wave
import numpy as np

def split_channels(wav_file):
    """
    分离立体声音频的左右声道
    """
    with wave.open(wav_file, 'rb') as w:
        n_channels = w.getnchannels()
        
        if n_channels != 2:
            print("不是立体声音频")
            return
        
        sample_width = w.getsampwidth()
        framerate = w.getframerate()
        n_frames = w.getnframes()
        frames = w.readframes(n_frames)
        
        # 转换为numpy数组
        audio_data = np.frombuffer(frames, dtype=np.int16)
        
        # 分离左右声道
        left_channel = audio_data[0::2]
        right_channel = audio_data[1::2]
        
        # 保存左声道
        with wave.open('left.wav', 'wb') as left:
            left.setnchannels(1)
            left.setsampwidth(sample_width)
            left.setframerate(framerate)
            left.writeframes(left_channel.tobytes())
        
        # 保存右声道
        with wave.open('right.wav', 'wb') as right:
            right.setnchannels(1)
            right.setsampwidth(sample_width)
            right.setframerate(framerate)
            right.writeframes(right_channel.tobytes())
        
        print("声道分离完成")
        print("左声道保存到 left.wav")
        print("右声道保存到 right.wav")
