## 三、Web题目实战（第一道）

### 3.1 Web题目解题策略

```
【Web题目解题策略】

题目顺序策略：
  建议：
    1. 先做熟悉的题目类型
    2. 后做不熟悉的题目类型
    3. 遇到难题果断跳过
  
  原因：
    - 先做熟悉的题目建立信心
    - 避免在难题上浪费时间
    - 保证基本分数

时间分配策略：
  Web题目总时间：120分钟（4题×30分钟）
  
  每题时间分配：
    - 观察题目：5分钟
    - 分析漏洞：10分钟
    - 构造Payload：10分钟
    - 获取Flag：5分钟
  
  超时处理：
    - 超过30分钟立即跳过
    - 标记为红牌题
    - 最后有时间再回来做

解题流程策略：
  标准流程：
    1. 观察题目结构
    2. 找到漏洞点
    3. 测试漏洞类型
    4. 构造Payload
    5. 绕过限制（如有）
    6. 获取Flag
    7. 提交验证
```

### 3.2 第一道Web题目解题流程

```
【第一道Web题目解题流程】

步骤1：观察题目（5分钟）
  动作：
    1. 打开题目页面
    2. 阅读题目描述
    3. 观察页面结构
    4. 找到输入点
  
  观察：
    - URL结构
    - 页面元素
    - 输入框位置
    - 题目提示

步骤2：判断题目类型（3分钟）
  动作：
    1. 根据URL参数判断类型
    2. 根据页面功能判断类型
  
  类型判断：
    ?id=参数 → 可能是SQL注入
    ?file=参数 → 可能是文件包含
    ?cmd=参数 → 可能是命令执行
    上传功能 → 可能是文件上传
    输入框 → 可能是XSS

步骤3：测试漏洞（10分钟）
  动作：
    根据题目类型测试：
    
    SQL注入测试：
      ?id=1' → 看是否报错
      ?id=1 and 1=1 → 看是否正常
      ?id=1 and 1=2 → 看是否异常
    
    文件包含测试：
      ?file=../../../etc/passwd → 看是否能读取
    
    命令执行测试：
      ?cmd=ls → 看是否有输出
    
    XSS测试：
      输入<script>alert(1)</script> → 看是否有弹窗

步骤4：构造Payload（10分钟）
  动作：
    根据测试结果构造Payload
    
    SQL注入Payload：
      1. 判断闭合方式
      2. 判断列数
      3. 构造联合查询
      4. 查询数据
    
    文件包含Payload：
      1. 尝试伪协议
      2. 绕过限制
    
    命令执行Payload：
      1. 使用连接符
      2. 绕过过滤

步骤5：获取Flag（2分钟）
  动作：
    1. 执行Payload
    2. 查看结果
    3. 提取Flag
    4. 提交验证
```

### 7.3 紧急情况处理

```
【紧急情况处理】

情况1：工具突然无法使用
  处理：
    1. 不要慌张
    2. 尝试重启工具
    3. 尝试使用替代工具
    4. 如果无法解决，跳过该题
  
  替代方案：
    Burp无法使用 → 使用浏览器F12
    StegSolve无法使用 → 使用其他隐写工具
    CyberChef无法使用 → 使用其他解码工具

情况2：网络突然断开
  处理：
    1. 检查网络连接
    2. 尝试重新连接
    3. 如果无法连接，暂停比赛
    4. 等待网络恢复后继续
  
  注意：
    - 网络断开不影响已下载的文件
    - 可以继续分析已下载的文件

情况3：忘记Payload
  处理：
    1. 不要慌张
    2. 查看Payload卡片
    3. 如果卡片没有，百度搜索语法
    4. 根据语法构造Payload
  
  注意：
    - 允许查看卡片和百度搜索语法
    - 禁止搜索题目WriteUp

情况4：时间严重不足
  处理：
    1. 不要慌张
    2. 优先做最熟悉的题目
    3. 每道题严格控制时间
    4. 超时果断跳过
  
  策略：
    - 确保做对≥4题
    - 不要追求全部做对
    - 保证基本分数

情况5：心态崩溃
  处理：
    1. 深呼吸，冷静下来
    2. 喝杯水，休息1分钟
    3. 回顾解题流程
    4. 从熟悉的题目开始
    5. 如果实在无法继续，暂停比赛
  
  注意：
    - 心态崩溃时不要强行解题
    - 先调整心态再继续
```

     - 尝试SQL注入测试
