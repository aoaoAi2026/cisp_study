# Day 5：终端安全与EDR

> 🎯 面试目标：掌握终端安全从传统AV到EDR/XDR的演进路径，以及EDR的核心能力和实战运营

## 知识速览
### 核心概念
- **传统防病毒（AV）**：基于特征库的恶意软件检测，依赖已知病毒签名。缺点：无法检测无文件攻击、内存攻击、零日漏洞利用、脚本攻击。
- **EDR（端点检测与响应）**：不只是检测恶意软件，而是持续监控终端行为（进程创建、网络连接、文件操作、注册表修改等），通过行为分析发现威胁，并提供调查和响应能力。
- **EDR核心能力**：①持续行为监控与记录（Telemetry）②威胁检测（行为分析+威胁情报+IOC匹配）③攻击链路可视化④远程响应（隔离主机、终止进程、删除文件、阻断网络）⑤威胁狩猎（Threat Hunting）。
- **XDR（扩展检测与响应）**：在EDR基础上扩展到网络、邮件、云、身份等多个维度，实现跨数据源的统一检测和响应。相比EDR，XDR的检测面更广、关联能力更强。

### 必问考点
| 面试官可能怎么问 | 你应该怎么答 |
|------------------|--------------|
| EDR和传统杀毒软件的本质区别是什么？ | 传统AV是"拦"——阻止已知恶意程序执行，依赖签名；EDR是"看+查+应"——持续记录终端行为、通过行为分析发现未知威胁、提供调查和响应能力。打个比方：AV是门禁（检查证件），EDR是监控摄像头+安防中心（全程录像+异常行为分析+快速响应） |
| EDR能检测哪些传统AV检测不了的攻击？ | ①无文件攻击（PowerShell内存加载）②Living-off-the-land攻击（利用系统自带工具）③凭证窃取（Mimikatz）④横向移动（PsExec、WMI）⑤供应链攻击（合法签名恶意软件）⑥零日漏洞利用（行为异常而非签名匹配） |
| 如何评估一款EDR产品的好坏？ | ①检测能力：覆盖率（MITRE ATT&CK评测结果）、误报率②性能影响：Agent对终端CPU/内存的消耗③响应能力：支持的响应动作丰富度④易用性：告警可读性、调查体验⑤扩展性：是否支持API集成⑥云原生架构vs本地部署 |
| EDR部署后为什么还有终端被攻破？ | 可能原因：①告警疲劳导致重要告警被忽略②EDR规则配置过于宽松③攻击者使用了EDR未覆盖的TTP④EDR Agent自身被绕过/卸载⑤响应延迟——检测到了但处置不及时⑥缺乏对告警的深度调查，只看了表面现象 |

### 技术细节
EDR告警调查流程（典型场景）：
```
1. 收到告警：主机A检测到可疑PowerShell执行
2. 查看进程树：
   WINWORD.EXE → cmd.exe → powershell.exe -enc <base64>
   → 判断：Word宏启动PowerShell，高度可疑
3. 查看网络连接：
   powershell.exe → 外网IP:443 (C2通信嫌疑)
4. 查看文件操作：
   powershell.exe写入 C:\Users\xxx\AppData\Local\Temp\payload.dll
5. 查看持久化：
   注册表 Run键新增启动项
6. 结论：钓鱼文档→宏执行→下载Payload→持久化
7. 响应：隔离主机→阻断外联IP→删除恶意文件→清除持久化→重装系统
```

## 常见陷阱
- ⚠️ **陷阱："装了EDR就不用管终端安全了"**——EDR是工具，不是万能药。需要安全团队具备威胁狩猎能力和调查分析能力，能看懂进程树、网络连接和攻击链路。没有专业分析师，EDR就是个昂贵的日志记录器。
- ⚠️ **陷阱："EDR告警一定是真的"**——EDR也会产生误报，尤其是正常运维操作（远程管理工具、补丁推送、脚本部署）经常触发告警。需要建立白名单机制和持续调优。
- ⚠️ **陷阱："EDR Agent可以替代补丁管理"**——EDR的虚拟补丁能力是补充而非替代。实际的安全漏洞修复还是要靠及时打补丁，EDR只能作为临时缓解措施。
