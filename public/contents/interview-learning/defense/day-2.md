# Day 2：防火墙技术与策略

> 🎯 面试目标：掌握防火墙的类型、工作原理、策略设计方法以及常见面试深度追问

## 知识速览
### 核心概念
- **包过滤防火墙**：工作在网络层/传输层，根据源IP、目的IP、端口、协议类型决定放行或拒绝数据包。特点：速度快、透明性好，但无法检测应用层攻击。
- **状态检测防火墙**：维护TCP连接状态表，不仅检查单个数据包，还判断该包是否属于一个已建立的合法连接。相比包过滤，能防御IP欺骗和部分拒绝服务攻击。
- **应用层代理防火墙**：在应用层代理客户端与服务器的通信，可以深入检查应用协议内容（如HTTP头、FTP命令）。安全性最高但性能开销大。
- **下一代防火墙（NGFW）**：集成了传统防火墙、IPS、应用识别、用户身份感知、威胁情报等功能于一体。Gartner定义NGFW需具备：标准防火墙功能、应用识别、集成IPS、支持外部情报。

### 必问考点
| 面试官可能怎么问 | 你应该怎么答 |
|------------------|--------------|
| 状态检测防火墙和包过滤防火墙的本质区别是什么？ | 包过滤只看单个数据包的头部信息（五元组），无状态；状态检测维护连接状态表，能识别数据包是否属于已建立的会话，可防御ACK扫描和会话劫持等攻击 |
| 防火墙应该部署在网络的什么位置？ | ①边界防火墙部署在网络出口，作为第一道防线；②内部防火墙部署在不同安全域之间（如办公网与服务器网段）；③主机防火墙部署在每个终端/服务器上；④云环境中使用安全组和VPC防火墙实现微分段 |
| 如何设计一套有效的防火墙策略？ | 遵循"默认拒绝"原则：白名单模式，只放行业务需要的流量。策略设计步骤：①资产梳理→②流量需求分析→③规则编写（源/目的/端口/应用/时间/用户）→④测试验证→⑤定期审计清理冗余规则 |
| 下一代防火墙相比传统防火墙多了哪些关键能力？ | 应用识别（能识别3000+应用，不限端口）、用户身份关联（与AD/LDAP集成）、集成IPS引擎、威胁情报联动、SSL解密检测、内容过滤（URL/文件类型）、高级威胁沙箱 |

### 技术细节
典型防火墙策略示例（iptables）：
```bash
# 默认策略：拒绝所有
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

# 允许已建立的连接
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 允许SSH（管理）
iptables -A INPUT -p tcp --dport 22 -s 10.0.0.0/8 -j ACCEPT

# 允许Web服务
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# 限制ICMP频率（防ping洪水）
iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 1/s -j ACCEPT
```

防火墙规则审计检查清单：
- [ ] 是否存在 any/any 规则？
- [ ] 是否有过期或不再使用的规则？
- [ ] 高危端口（22,3389,1433,3306）是否对公网开放？
- [ ] 入站和出站规则是否都做了限制？

## 常见陷阱
- ⚠️ **陷阱："防火墙是万能的"**——防火墙无法防御应用层攻击（SQL注入、XSS），无法防御已授权的加密流量中的恶意内容，也无法防御内部威胁。需要与其他安全设备协同。
- ⚠️ **陷阱："规则越多越安全"**——过多的防火墙规则会导致管理混乱、性能下降，容易产生规则冲突和安全漏洞。应该定期清理和优化，保持规则精简有效。
- ⚠️ **陷阱："出站流量不需要控制"**——大量APT攻击利用出站流量回传数据（C2通信、DNS隧道），出站过滤同样重要。至少要限制对外部高危端口的访问。
