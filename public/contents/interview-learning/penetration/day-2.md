# Day 2：信息收集与侦察技术

> 🎯 面试目标：能完整说出信息收集的层次和方法论，并举例说明每种技术的实际产出

## 知识速览

### 核心概念
- **被动信息收集**：不直接与目标交互，通过搜索引擎、DNS记录、证书透明度日志、Shodan、Censys等公开渠道收集信息，目标无法感知
- **主动信息收集**：直接与目标系统交互（端口扫描、目录爆破、指纹识别），会产生日志记录，但能获取更精确的信息
- **攻击面测绘**：绘制目标的完整数字资产地图——域名、IP段、子域名、开放端口、Web应用、API接口、第三方服务，回答"目标暴露了什么"

### 必问考点

| 面试官可能怎么问 | 你应该怎么答 |
|------------------|--------------|
| 给你一个公司名称，你如何开始信息收集？ | 第一步被动收集：天眼查/企查查查股权和关联公司→搜索引擎查泄露信息→Shodan/Fofa搜IP资产→证书透明度查子域名→GitHub搜敏感信息泄露；第二步才是主动扫描：子域名爆破→端口扫描→指纹识别→目录扫描 |
| DNS 信息收集能拿到什么关键信息？ | ①A记录/MX记录暴露IP段和邮件服务器；②区域传送漏洞可获取完整DNS记录列表；③泛解析判断是否有CDN/WAF；④历史DNS解析记录可发现已下线的服务；⑤SPF/DMARC记录泄露邮件安全策略和内网信息 |
| 你怎么绕过 CDN 找到真实 IP？ | ①历史DNS解析记录（SecurityTrails等）；②全球多地Ping对比；③邮件服务器IP（发一封找回密码邮件看邮件头）；④子域名可能未挂CDN；⑤SSL证书关联搜索（crt.sh）；⑥网站RSS/邮件订阅源泄露；⑦phpinfo等敏感文件泄露 |

### 技术细节

信息收集工具链速查：
```bash
# 子域名枚举
subfinder -d target.com
amass enum -d target.com

# 端口扫描
nmap -sS -sV -p- --min-rate 10000 target.com
masscan -p1-65535 --rate=10000 target_ip

# 指纹识别
whatweb target.com
wappalyzer  # 浏览器插件

# 目录爆破
gobuster dir -u https://target.com -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt

# GitHub敏感信息
# 搜索关键词：password, api_key, secret, jdbc, connectionString + 公司域名
```

## 常见陷阱
- ⚠️ 面试时说信息收集"随便扫一下就行"——信息收集占渗透测试60%以上的时间投入，收集得越全，后续利用越顺利
- ⚠️ 忘记区分被动和主动——面试官可能会追问"这样做会不会被对方发现"，要能判断每种技术是否产生日志
