# Day 3：网络安全监测基础

> 🎯 面试目标：掌握网络安全监测的核心方法论、常见监测数据源，以及如何建立有效的监测体系

## 知识速览

### 核心概念
- **安全监测三要素**：数据采集（Collect）→ 数据分析（Analyze）→ 告警响应（Respond），缺一环则监测失效
- **监测数据源**：网络流量（NetFlow/pcap）、系统日志（syslog/EventLog）、应用日志（access.log/error.log）、安全设备日志（WAF/IDS/EDR）、威胁情报（IOC feeds）
- **检测策略**：基于签名（Signature-based）匹配已知攻击模式、基于异常（Anomaly-based）检测偏离基线的行为、基于行为（Behavior-based）分析用户/实体行为
- **监测覆盖率**：你有多少资产纳入了监控范围？这是面试中必须能回答的问题

### 必问考点
| 面试官可能怎么问 | 你应该怎么答 |
|------------------|--------------|
| 你们公司的安全监测体系是怎么建的？ | 从三个层面回答：①数据采集层——覆盖了哪些日志源（网络设备、安全设备、服务器、应用、终端）；②分析层——用什么平台（SIEM/大数据平台）做关联分析，用了哪些检测规则；③响应层——告警分级标准、处置SOP、人员值班安排 |
| 如果让你从零搭建安全监测，你第一步做什么？ | 第一步是资产梳理——搞清楚有哪些资产需要监控、它们的网络位置、业务重要性。没有资产清单，监测就是盲人摸象。第二步确定日志源优先级（边界设备>核心服务器>终端），第三步部署日志采集和集中存储 |
| 什么是检测覆盖率？如何评估？ | 检测覆盖率 = 已监控资产数 / 总资产数。评估方法：①按资产类型（网络设备/服务器/应用）分别统计；②按攻击链阶段（侦察/武器化/投递/利用/安装/C2/行动）检查每阶段是否有检测手段；③用红队测试验证实际检测效果 |

### 技术细节

**安全监测的Kill Chain视角——每个阶段如何检测：**
```
侦察阶段 → 检测：端口扫描告警、大量404探测
武器化阶段 → 检测：威胁情报匹配（恶意域名/IP）
投递阶段 → 检测：邮件安全网关、恶意附件检测
利用阶段 → 检测：WAF告警、IDS攻击特征匹配
安装阶段 → 检测：EDR检测异常进程创建、Webshell写入
C2阶段   → 检测：异常外连检测（DNS隧道、HTTPS心跳）
行动阶段 → 检测：数据渗出检测（大流量外传）、横向移动检测
```

## 常见陷阱
- ⚠️ "我们装了IDS/WAF，所以安全监测已经覆盖了"——大错！设备只是工具，没有人去研判告警、没有人去优化规则，设备等于摆设。监测=工具+人+流程
- ⚠️ 面试时说"我们监控了所有东西"——面试官会追问具体覆盖率数字。诚实地说明覆盖了多少资产、哪些是盲区，反而更显专业
