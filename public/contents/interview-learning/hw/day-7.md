# Day 7：安全设备运维基础

> 🎯 面试目标：掌握防火墙/WAF/IDS/堡垒机的运维要点、变更管理和高可用设计面试考点

## 知识速览

### 核心概念
- **防火墙三大表**：连接表(Connection Table——记录每条会话状态)→NAT表(地址转换映射)→路由表(Routing Table——包转发决策)。FULL/PIX/ASA的show conn count是运维最常用命令
- **WAF工作模式**：正向代理(客户端→WAF→后端)、反向代理(客户端→WAF→后端，透明给客户端)、透明桥(二层透明部署，不影响网络拓扑)。API防护场景推荐反向代理
- **堡垒机四A功能**：Authentication(身份认证+MFA)、Authorization(权限控制→能访问哪些资产)、Audit(全程录像审计→回放操作)、Accounting(操作日志统计)
- **网络设备HA架构**：主备模式(Active-Standby→VRRP/HSRP)→主主模式(Active-Active→负载分担)→集群模式(N+1冗余)。防火墙HA需同步连接表，否则主备切换时所有会话断开
- **设备变更管理流程**：变更申请(业务需求+影响评估+回滚方案)→变更审批(技术+安全)→变更窗口(维护窗口执行)→变更验证(配置备份+功能测试+安全扫描)→变更记录(CMDB更新)

### 必问考点
| 面试官可能怎么问 | 你应该怎么答 |
|------------------|--------------|
| 防火墙策略优化的方法和常见问题？ | 优化方法：1)策略命中率分析——长期未命中(超过90天)的规则可能是无效规则，应考虑清理 2)过度宽松规则(ANY→ANY)——替换为具体的源目IP+端口 3)隐性规则冲突——前面any规则会遮蔽后面的精确规则 4)时间策略——非工作时间限制高风险协议的访问(如RDP/SSH)。工具：Algosec/Tufin/FireMon防火墙策略管理。面试强调：不要只看'有没有防火墙'，要看'规则是否最小必要'。 |
| 堡垒机录屏审计的技术原理？如何确保操作不可篡改？ | 录屏：通过堡垒机内置的RDP/SSH Proxy截获和重放会话(类似中间人)→对图形协议(GUI)导出为视频流→字符协议(CLI)记录为文本日志+命令时间戳。防篡改：1)实时推送日志到远程Syslog/ELK 2)日志签名(数字签名确保完整性) 3)WORM存储(Write Once Read Many，不可修改) 4)多副本分散存储(操作者不可同时访问所有副本)。审计回放：支持命令搜索 ('rm -rf'→定位到具体时间点播放)。 |
| WAF误报和漏报的平衡策略？ | 策略层次：1)默认使用'宽松模式'(Block模式下只阻断高置信度攻击如明确SQL注入特征→记录可疑攻击但不阻断) 2)对核心业务逐一开启'严格模式'(经过充分测试确认无误报) 3)建立白名单机制——对已知的正常业务URL参数做例外(如某API的查询参数确实含'select') 4)持续监控分析——每天Review WAF告警，将漏报的攻击特征添加为自定义规则 5)A/B测试新规则——先在Log-only模式运行一周→确认无误报→再切换为Block模式。 |
| 如何设计一个安全设备的高可用架构？ | 四个要点：1)链路冗余——双上联到不同核心交换机，端口聚合(LACP) 2)设备冗余——主备模式(心跳线监控+抢占模式) 3)会话同步——防火墙需同步连接表(新增/老化/超时)、WAF需同步Session信息 4)健康检查——不只是ping，要检测应用端口(如HTTP 200+TLS握手成功)。关键面试点：主备切换时的'会话表冲击'——大量老会话因备机没有状态信息而被丢弃→DNS/NTP等长连接可能断开，需要通过会话同步(Session Sync)减少断连。 |
| 如何做一次防火墙规则的安全审计？ | 审计六步：1)导出当前配置(NAT+ACL+安全策略) 2)识别ANY规则→逐条评估是否可收紧 3)交叉对比允许列表和实际业务需求→标红不必要的允许 4)检查高危端口(3389/22/445/135-139)是否暴露到公网 5)验证规则变更审批记录(最近半年每个变更都有对应的变更单) 6)输出风险清单+整改建议。面试展示：会使用自动化工具(如Nmap+NSE firewalk脚本探测真实开放的端口)而不是只看配置文件。 |

### 技术细节
**防火墙日常运维检查命令(Cisco ASA/Palo Alto)**：
```bash
# Cisco ASA
show conn count                     # 当前连接数
show cpu usage                      # CPU使用率
show memory                         # 内存使用
show failover state                 # HA状态
show logging | include %ASA         # 最新日志

# Palo Alto
show session info                   # 会话信息
show system resources               # 系统资源
show high-availability state        # HA状态
show running security-policy        # 安全策略
```
**自动化变更工具**：Ansible(network modules玩转多厂商设备)+ Git(配置版本控制)+ Jenkins(变更审批→自动下发→验证)。CI/CD for Network = NetDevOps。

## 常见陷阱
- ⚠️ 配置修改后不备份——很多故障因为'我改了配置但忘了备份→恢复不了之前的状态'
- ⚠️ 防火墙策略累积灾难——因业务申请不断添加临时的宽松规则，几年后变成'筛子'，需定期审计清理
- ⚠️ 只关注规则不通→忽视规则太宽——'ANY→ANY'是最常见的'通了但不安全'的反模式
