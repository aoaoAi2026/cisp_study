# Day 1：护网行动全景介绍

> 🎯 面试目标：能够清晰阐述护网行动的组织背景、红蓝紫三队分工、蓝队核心职责，以及值守的标准流程

## 知识速览

### 核心概念
- **护网行动（HW）**：由公安部、网信办等国家主管部门牵头的国家级网络安全实战演习，红队模拟攻击、蓝队负责防守、紫队担任裁判，旨在检验关键信息基础设施的防御能力
- **红蓝紫三队**：红队（攻击方）模拟APT组织突破防线；蓝队（防守方）监控、研判、处置、溯源；紫队（裁判方）定规则、监督流程、判定得失分
- **蓝队四项基本功**：眼（实时监控）→ 脑（告警研判）→ 手（事件处置）→ 笔（报告记录），形成完整的值守闭环
- **黄金15分钟法则**：从告警产生到完成研判处置，护网期间要求不超过15分钟，超时即失分

### 必问考点
| 面试官可能怎么问 | 你应该怎么答 |
|------------------|--------------|
| 请用一句话概括护网行动是什么？ | 护网行动是由公安部牵头的国家级网络安全实战演习，红队模拟攻击、蓝队负责防守、紫队担任裁判，旨在检验关键信息基础设施的防御能力，发现安全短板 |
| 蓝队和红队的核心区别是什么？ | 红队的目标是"攻破"——找到漏洞并利用；蓝队的目标是"守住"——监控、检测、响应、加固。蓝队不需要比红队更懂攻击，但必须比红队更懂防御的薄弱环节在哪里。理解攻击是为了更好地防御 |
| 蓝队值守时发现告警的第一步应该做什么？ | 第一步永远是「核查确认」——判断是真攻击还是误报。不要立即处置！先查：IP是否内网资产、时间是否合理、告警来源设备是否可信、是否有同类告警。确认后再按SOP分级处置 |
| 护网期间蓝队最常见的失误有哪些？ | ①盲目封IP（可能封掉正常业务）；②忽视低危告警（攻击者在试探）；③不填工单（事后无法复盘）；④只看不记（没有留下证据链）；⑤值班交接遗漏（断层导致漏报） |

### 技术细节

**蓝队值守标准流程（白班示例）：**
```
08:50 接班签到 → 与夜班口头交接未闭环事项
09:00-09:30 全量告警扫描（按红→橙→黄优先级处理）
09:30-12:00 持续监控（实时盯告警大屏+SIEM，每2小时态势汇总）
12:00-13:00 午休（必须找人替班，席位不能空）
13:00-17:30 持续监控（下午班，2次态势汇总）
17:30-17:50 撰写值班日报
17:50-18:00 交班（日报发组长+夜班，确保无未处理告警）
```

**值守铁律：**
- 7×24小时席位不空——上厕所找替班，吃饭轮流去
- 告警不超过15分钟响应——慢一秒就可能失分
- 任何操作都要留痕——截图+文字说明，不能事后补
- 不确定的事立即升级——不要自己猜，不要侥幸

## 常见陷阱
- ⚠️ 面试官问"护网就是抓黑客吗"——错误！80%时间看告警+判误报，15%写报告，只有5%在真实处置攻击。但正是那95%的日常积累，决定了5%关键时刻的成败
- ⚠️ 面试时只谈红蓝两队而忽略紫队——紫队是裁判和协调方，面试中体现你对护网全貌的理解，提到紫队说明你不是"新手视角"
