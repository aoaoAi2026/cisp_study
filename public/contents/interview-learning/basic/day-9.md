# Day 9：恶意代码分析入门

> 🎯 面试目标：掌握恶意代码的分类、传播方式和基本分析方法，能回答安全事件中的恶意代码面试题

## 知识速览

### 核心概念
- **恶意代码分类**：病毒（寄生感染）、蠕虫（自主传播）、木马（伪装欺骗）、勒索软件（加密勒索）、Rootkit（隐藏踪迹）、Botnet（远程控制）、间谍软件（信息窃取）。面试关键：能区分病毒和蠕虫——病毒需要宿主程序/文件，蠕虫可自主传播
- **分析环境**：静态分析（不执行代码，查看字符串、导入表、PE结构）vs 动态分析（在沙箱中执行，监控行为）。生产环境严禁直接运行可疑样本，必须使用隔离的虚拟机或沙箱
- **APT攻击链（Cyber Kill Chain）**：侦察→武器化→投递→利用→安装→C2通信→目标行动。每个阶段都有对应的检测和阻断机会，面试常结合真实案例考察

### 必问考点
| 面试官可能怎么问 | 你应该怎么答 |
|------------------|--------------|
| 发现一台主机疑似中了勒索软件，你怎么处理？ | 第一步立即断网隔离（拔网线，不是关机！关机可能破坏加密密钥线索），第二步确认感染范围和加密文件类型，第三步检查备份可用性，第四步提取样本和日志交分析团队，第五步评估是否支付赎金（通常不建议）。同时排查横向移动痕迹 |
| 如何判断一个文件是否为恶意代码？ | 多层判断：1)查杀毒引擎（VirusTotal多引擎扫描）2)静态分析看导入表是否有敏感API（如VirtualAlloc、WriteProcessMemory、URLDownloadToFile）3)动态分析看沙箱行为（文件操作、网络连接、进程注入、注册表修改）4)查看数字签名是否有效 5)检查熵值（高熵可能意味加壳） |
| APT和普通网络攻击有什么区别？ | APT是有组织、有目的、长期潜伏的高级持续性威胁。区别：1)目标明确（特定行业/机构）2)手法高级（0day、定制工具）3)长期潜伏（数月甚至数年）4)背后有国家或组织支持。普通攻击多为广撒网式，追求快速获利 |

### 技术细节

**恶意代码静态分析关键检查点：**
```python
# 可疑API导入检测（静态分析思路）
suspicious_apis = [
    # 进程操作
    "CreateRemoteThread", "WriteProcessMemory", "VirtualAllocEx",
    # 文件操作
    "CreateFile", "WriteFile", "DeleteFile", "MoveFile",
    # 注册表操作
    "RegSetValueEx", "RegCreateKeyEx",
    # 网络操作
    "URLDownloadToFile", "WinHttpOpen", "InternetConnect",
    # 反调试/反分析
    "IsDebuggerPresent", "CheckRemoteDebuggerPresent", "NtQueryInformationProcess",
    # 加密相关（勒索软件常用）
    "CryptEncrypt", "CryptDecrypt", "CryptAcquireContext",
    # 服务操作（持久化）
    "CreateService", "StartService"
]
```

**勒索软件应急处置流程：**
```
[发现告警]
    │
    ▼
[立即断网隔离] ← 最关键一步，防止横向传播
    │
    ▼
[确认感染范围] → 检查文件服务器、共享目录、其他终端
    │
    ▼
[保护证据] → 内存dump、磁盘镜像、勒索信截图
    │
    ▼
[检查备份] → 离线备份是否可用？云端版本是否可回滚？
    │
    ▼
[分析样本] → 沙箱分析，确认勒索软件家族（有的已有解密工具）
    │
    ▼
[恢复/重建] → 从干净备份恢复，或重装系统
    │
    ▼
[加固防范] → 补丁管理、终端防护、用户培训
```

**Kill Chain各阶段防御措施：**
| 阶段 | 攻击行为 | 防御措施 |
|------|---------|---------|
| 侦察 | 端口扫描、信息收集 | 减少信息暴露、部署欺骗技术 |
| 武器化 | 制作恶意payload | 无法直接防御 |
| 投递 | 钓鱼邮件、恶意附件 | 邮件网关、沙箱检测 |
| 利用 | 漏洞利用 | 补丁管理、虚拟补丁(WAF/IPS) |
| 安装 | 安装后门、持久化 | EDR、应用白名单 |
| C2通信 | 连接控制服务器 | 流量分析、DNS安全 |
| 目标行动 | 数据窃取、破坏 | DLP、UEBA异常检测 |

## 常见陷阱
- ⚠️ "我们用杀毒软件所以不会被感染"——杀毒软件基于签名和启发式检测，对0day和高度定制化的APT恶意代码检测率有限。需要多层防护
- ⚠️ 发现恶意代码后第一反应是"赶紧删掉"——应该先保留证据、确认感染路径和范围，否则可能破坏线索导致无法追溯攻击来源
