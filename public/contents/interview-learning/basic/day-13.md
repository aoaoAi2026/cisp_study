# Day 13：社会工程学防御

> 🎯 面试目标：掌握社工攻击向量(钓鱼/伪装/尾随/诱饵)及企业防御体系设计

## 知识速览

### 核心概念
- **社会工程学攻击三角**：信任建立(Authority/Scarcity/Urgency) + 信息收集(OSINT) + 漏洞利用(人性弱点) = 攻击成功
- **鱼叉式钓鱼(Spear Phishing)**：针对特定个人的定制化钓鱼，利用已知的个人信息增加可信度，成功率远高于广撒网式
- **伪装攻击(Pretexting)**：攻击者编造身份和场景(如IT支持/HR/合作方)获取敏感信息，依赖信息不对称
- **BEC商务邮件诈骗**：冒充高管/供应商发送紧急汇款请求，FBI统计全球年损失超260亿美元
- **SIM Swap攻击**：社工电信运营商客服将目标手机号转移到攻击者SIM卡→绕过短信MFA→接管账号

### 必问考点
| 面试官可能怎么问 | 你应该怎么答 |
|------------------|--------------|
| 如何设计有效的企业反钓鱼培训？ | 不能只是PPT培训。有效方案：1)定期模拟钓鱼测试(PhishMe/KnowBe4) 2)即时反馈——点错链接后立即弹出教育页面 3)正向激励——点击举报按钮而非不点击(培养'我看到可疑'的主动意识) 4)按部门定制场景(财务部→假发票、IT部→假工单) 5)重复训练间隔2-4周(遗忘曲线)。面试强调：既要讲方案也要讲效果度量(click rate下降曲线)。 |
| 社工攻击在APT攻击链中一般处于哪个阶段？ | 典型位置：侦察阶段(LinkedIn信息收集)→武器化阶段(针对目标兴趣定制邮件)→投递阶段(发送钓鱼邮件)→利用阶段(诱导点击/下载)。可以说：'如果把APT比作一场战役，社工就是情报侦察部队+特战渗透小组的结合。' |
| MFA能完全防御社工攻击吗？怎么绕过？ | 不能。常见绕过方法：1)MFA疲劳攻击(不停推送认证请求直到受害者误点) 2)实时钓鱼代理(Modlishka/Evilginx)劫持Session Token 3)SIM Swap绕过短信MFA。强调：MFA是重要防线但不是银弹，FIDO2/WebAuthn的防钓鱼特性更强。 |
| 如何识别一封高度定制化的鱼叉钓鱼邮件？ | 三看分析法：一看邮件头(SPF/DKIM/DMARC验证结果、Reply-To是否不一致)；二看链接(悬停显示真实URL、是否用IDN同形异义字如'paypaI.com')；三看上下文(语气是否异常、请求是否不符合正常流程、时间压力是否不合理)。 |
| BEC攻击最有效的技术防御手段是什么？ | 技术层面：1)DMARC策略设为reject(防域名伪造) 2)邮件网关AI检测(异常发件行为分析) 3)外部邮件标记(在主题加[External]) 4)大额转账双人确认流程。但最重要的是流程防御：任何付款变更必须通过预先登记的备用渠道(电话/当面)确认。 |

### 技术细节
**钓鱼邮件分析实战技巧**：
```bash
# 获取邮件头信息
cat suspicious.eml | grep -E "From:|Reply-To:|Return-Path:|Received:"

# 检查SPF/DKIM/DMARC
dig TXT example.com          # SPF记录
dig TXT _dmarc.example.com   # DMARC策略

# 提取链接进行分析
urlscan.io 提交可疑链接安全分析
any.run 沙箱分析恶意附件行为
```
**企业防御架构**：邮件网关(Mimecast/Proofpoint) → AI异常检测(Abnormal Security) → 端点EDR → MFA+FIDO2 → DMARC enforce → 安全意识培训闭环。SOC需建立：收到钓鱼报告→15分钟内拉黑发件人/域名→全网搜索已送达的同批次邮件→收回(Recall/Purge)。

## 常见陷阱
- ⚠️ 认为技术人不会上当——研究表明技术背景和防社工能力没有显著正相关，自负反而是弱点
- ⚠️ 只做年度的安全意识培训——效果持续约4-6周即衰减，需高频短训+模拟测试
- ⚠️ 忽视物理社工——尾随(TAILGATING)、冒充维修人员、USB掉落攻击(BadUSB)等物理手段同样致命
