# Day 1：网络安全概述

> 🎯 面试目标：能够清晰阐述网络安全的基本概念、威胁类型、安全属性，以及CIA三元组在面试中的标准表述

## 知识速览

### 核心概念
- **CIA三元组**：机密性（防止未授权访问）、完整性（防止未授权篡改）、可用性（保证授权用户正常访问），是信息安全最基础的框架
- **AAA模型**：认证（Authentication）确认身份、授权（Authorization）赋予权限、审计（Accounting）记录行为，三个环节缺一不可
- **纵深防御（Defense in Depth）**：多层安全控制策略叠加，单一控制失效不会导致整体失陷，面试中必提的安全设计原则

### 必问考点
| 面试官可能怎么问 | 你应该怎么答 |
|------------------|--------------|
| 请解释一下CIA三元组 | 机密性确保数据不被未授权访问，完整性确保数据不被篡改，可用性确保数据随时可访问。三者有时存在此消彼长的关系，需要根据业务需求权衡 |
| 什么是纵深防御？举个例子 | 纵深防御是在网络边界、内网、主机、应用、数据等多个层面部署安全控制。比如边界防火墙+内网IDS+主机HIDS+WAF+数据加密，即使防火墙被绕过，后续防线仍可发挥作用 |
| 网络安全的主要威胁有哪些？ | 从来源分：外部攻击者、内部威胁、APT组织。从手段分：恶意代码、DDoS、社会工程学、供应链攻击、0day漏洞利用。面试时优先提APT和社会工程学这两个热点 |

### 技术细节

**CIA在实际安全方案中的体现：**
- 机密性 → 加密（AES/RSA）、访问控制列表（ACL）、数据分级
- 完整性 → 数字签名、哈希校验（SHA-256）、WAL日志
- 可用性 → 负载均衡、冗余部署、DDoS防护

**纵深防御的经典7层模型（对照OSI）：**
```
第7层 数据安全：加密、DLP、备份
第6层 应用安全：WAF、代码审计、RASP
第5层 主机安全：HIDS、杀毒软件、主机加固
第4层 网络安全：IDS/IPS、NGFW、网络分段
第3层 边界安全：防火墙、DMZ、VPN
第2层 物理安全：门禁、监控、机柜锁
第1层 策略与流程：安全制度、应急预案、安全意识培训
```

## 常见陷阱
- ⚠️ 面试官问"网络安全和信息安全有什么区别"——网络安全是信息安全的子集，网络安全侧重网络传输层面的防护，信息安全范围更大，涵盖存储、处理、传输全过程
- ⚠️ CIA三元组不能只背三个词，一定要补充"三者之间的平衡关系"。比如追求高可用可能导致机密性下降（多副本增加泄露面）
