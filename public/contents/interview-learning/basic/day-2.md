# Day 2：OSI模型与TCP/IP协议栈

> 🎯 面试目标：熟练对比OSI与TCP/IP模型，掌握各层核心协议和典型攻击方式，能回答协议层面的安全面试题

## 知识速览

### 核心概念
- **OSI七层模型**：物理层→数据链路层→网络层→传输层→会话层→表示层→应用层，面试中重点区分各层职责和代表性协议
- **TCP三次握手**：SYN→SYN+ACK→ACK，面试必问，必须能画出时序图并说明SYN Flood攻击原理
- **协议栈安全**：每个层次都有对应的攻击方式和防御措施，如ARP欺骗（链路层）、IP欺骗（网络层）、SYN Flood（传输层）、SQL注入（应用层）

### 必问考点
| 面试官可能怎么问 | 你应该怎么答 |
|------------------|--------------|
| TCP和UDP的区别，分别适用于什么场景？ | TCP面向连接、可靠传输、有流量控制和拥塞控制，适合HTTP/HTTPS、SSH、邮件；UDP无连接、不可靠但低延迟，适合视频流、DNS、VoIP。面试可以补充：UDP反射放大攻击是DDoS的常见手法 |
| 从浏览器输入URL到页面显示，涉及哪些协议？ | DNS解析域名（UDP 53）→TCP三次握手→TLS握手（HTTPS）→HTTP请求→服务器处理→响应返回→浏览器渲染。每个步骤都能深入追问，要能讲出每层协议 |
| ARP协议有什么安全问题？ | ARP没有认证机制，攻击者可发送伪造ARP响应实现中间人攻击。防御措施：静态ARP表、DAI动态ARP检测、端口安全、802.1X认证 |

### 技术细节

**OSI vs TCP/IP 对比表：**
```
OSI (7层)              TCP/IP (4层)       典型协议与安全威胁
─────────────────────────────────────────────────────────
应用层                 应用层             HTTP/DNS/FTP | SQL注入、XSS、DNS劫持
表示层
会话层
─────────────────────────────────────────────────────────
传输层                 传输层             TCP/UDP | SYN Flood、端口扫描
─────────────────────────────────────────────────────────
网络层                 网络层             IP/ICMP/ARP | IP欺骗、Smurf攻击
─────────────────────────────────────────────────────────
数据链路层             网络接口层          Ethernet/WiFi | ARP欺骗、MAC泛洪
物理层
```

**TCP三次握手与安全：**
```
客户端                    服务器
  │───SYN(seq=x)───────→│   // SYN Flood攻击点：发送大量SYN不完成握手
  │←──SYN+ACK(seq=y,ack=x+1)──│
  │───ACK(seq=x+1,ack=y+1)──→│  // 连接建立
  │                       │
  │←════ 数据传输 ═══════→│
  │                       │
  │───FIN──────────────→│
  │←───FIN+ACK──────────│   // TCP Reset攻击点：伪造RST包中断连接
```

**防御SYN Flood：**
- SYN Cookie：服务器不立即分配资源，用算法验证客户端真实性
- SYN Proxy：代理设备先完成三次握手，再转发给服务器
- 增加SYN队列长度、减少SYN超时时间

## 常见陷阱
- ⚠️ 面试官问"ping用的是什么协议"——ping用的是ICMP协议（网络层），不是应用层协议。ICMP封装在IP包中，没有端口号的概念
- ⚠️ "防火墙可以防御应用层攻击吗"——传统包过滤防火墙不能，需要WAF或NGFW。面试官可能借此引出下一代防火墙的概念
