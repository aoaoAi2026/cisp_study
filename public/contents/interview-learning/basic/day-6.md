# Day 6：密码学基础

> 🎯 面试目标：掌握对称/非对称加密、哈希函数、数字签名、PKI的核心概念和面试标准答案

## 知识速览

### 核心概念
- **对称加密 vs 非对称加密**：对称加密用同一密钥加解密（AES/DES），速度快适合大数据量；非对称加密用公私钥对（RSA/ECC），速度慢但解决密钥分发问题。实际系统中两者结合使用——用非对称加密传输对称密钥（即TLS握手原理）
- **哈希函数特性**：单向不可逆、固定长度输出、雪崩效应、抗碰撞性。MD5和SHA-1已被破解，面试必须强调使用SHA-256及以上。常见应用：密码存储（加盐哈希）、文件完整性校验、数字签名
- **数字签名与PKI**：用私钥签名、公钥验签，实现不可否认性和完整性验证。PKI体系基于CA信任链，核心是证书的签发、吊销（CRL/OCSP）和信任传递

### 必问考点
| 面试官可能怎么问 | 你应该怎么答 |
|------------------|--------------|
| HTTPS的加密过程是怎样的？ | 浏览器发起请求→服务器返回证书（含公钥）→浏览器验证证书链→生成对称密钥用服务器公钥加密传输→后续通信使用对称密钥。核心是RSA/ECDHE做密钥交换，AES做数据加密。ECDHE提供前向安全性 |
| 密码应该如何安全存储？ | 绝不能明文存储或仅用MD5。标准做法：使用bcrypt/scrypt/Argon2等慢哈希算法，每个密码独立随机加盐（Salt），推荐迭代次数≥10万次。可以补充：Pepper（服务器端密钥）作为额外安全层 |
| 什么是中间人攻击？HTTPS如何防御？ | 攻击者在通信双方之间拦截并可能篡改数据。HTTPS通过CA证书体系验证服务器身份，确保公钥的真实性。但前提是CA体系可信，如果CA被攻陷或用户安装了恶意根证书，HTTPS仍可能被中间人 |

### 技术细节

**主流加密算法对比：**
```
类型        算法      密钥长度    安全性          典型应用
────────────────────────────────────────────────────
对称        AES       128/256    安全          数据加密、TLS
对称        DES       56         已破解         不再使用
对称        3DES      168        强度有限       遗留系统
非对称      RSA       2048-4096  安全          密钥交换、签名
非对称      ECC       256       安全（性能优）  移动设备、TLS 1.3
非对称      SM2       256       国密标准       国内合规场景
哈希        SHA-256   256       安全          签名、区块链
哈希        SHA-1     160       已破解         不再使用
哈希        SM3       256       国密标准       国内合规场景
```

**TLS 1.3握手过程（简化版）：**
```
客户端                                 服务器
  │                                       │
  │──ClientHello(支持的密码套件+密钥共享)──→│
  │                                       │
  │←─ServerHello(选定密码套件+密钥共享+证书)─│
  │                                       │
  │──────[计算会话密钥]───────[计算会话密钥]──│
  │                                       │
  │←═══════ 加密数据传输(AES-GCM) ═══════→│
  
  关键：TLS 1.3仅需1-RTT，且移除了不安全的密码套件，默认支持PFS
```

**密码哈希最佳实践（代码示例）：**
```python
import hashlib
import os

def hash_password(password: str) -> str:
    # 生成16字节随机盐
    salt = os.urandom(16)
    # 使用PBKDF2，迭代100,000次
    key = hashlib.pbkdf2_hmac('sha256', password.encode(), salt, 100000)
    # 存储格式：算法$迭代次数$salt$hash
    return f"pbkdf2_sha256$100000${salt.hex()}${key.hex()}"

def verify_password(password: str, stored: str) -> bool:
    algo, iterations, salt_hex, hash_hex = stored.split('$')
    salt = bytes.fromhex(salt_hex)
    key = hashlib.pbkdf2_hmac('sha256', password.encode(), salt, int(iterations))
    return key.hex() == hash_hex
```

## 常见陷阱
- ⚠️ "MD5加密"的说法——MD5是哈希算法不是加密算法，哈希不可逆，加密可逆。面试官用词"加密"可能是在试探你的基础概念
- ⚠️ "我们用了HTTPS所以数据传输绝对安全"——HTTPS保护传输层安全，但不保护端点安全。如果客户端被植入恶意证书或服务器被攻陷，HTTPS也无法保护数据
