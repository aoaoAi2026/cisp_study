# IEC 62443 工控安全标准体系解读

---

## 一、IEC 62443 标准体系概览

```
IEC 62443 "工业通信网络 - 网络和系统安全"

四大部分，14个子标准：

Part 1: General (通用)
├── 1-1: 术语、概念和模型
├── 1-2: 主术语和缩略语
├── 1-3: 系统安全符合性度量
└── 1-4: IACS安全生命周期和用例

Part 2: Policies & Procedures (策略与规程)
├── 2-1: IACS安全管理体系要求
├── 2-3: IACS环境中的补丁管理
├── 2-4: IACS服务提供商安全要求
└── 2-5: IACS资产管理

Part 3: System (系统级) ★核心★
├── 3-1: 安全技术应用于IACS
├── 3-2: 区域和管道(Zone & Conduit)安全级别
└── 3-3: 系统安全要求和安全级别

Part 4: Component (组件级)
├── 4-1: 产品开发生命周期安全要求
└── 4-2: IACS组件技术安全要求
```

## 二、安全等级 (Security Level, SL)

### 2.1 四级定义

```
SL 1: 防止偶然/巧合的违规
  防御能力：抵抗基本无意的违规和普通用户的偶然操作
  → 对应威胁：脚本小子、误操作

SL 2: 防止蓄意的简单手段违规
  防御能力：抵抗使用简单手段、低资源、通用技能的故意违规
  → 对应威胁：黑客组织、普通恶意软件

SL 3: 防止蓄意的复杂手段违规
  防御能力：抵抗使用复杂手段、中等资源、IACS特定技能的故意违规
  → 对应威胁：APT组织、国家支持的攻击

SL 4: 防止蓄意的尖端手段违规
  防御能力：抵抗使用尖端手段、大量资源、IACS特定高级技能的故意违规
  → 对应威胁：国家级资源、定制化ICS恶意软件

评估逻辑：
  1. 评估每个区域的威胁等级（SL-T = 攻击者能力级别）
  2. 确定需要达到的目标SL (SL-Target)
  3. 计算当前已达到的SL (SL-Achieved)
  4. 如果 SL-Achieved < SL-Target → 需要整改
```

## 三、Zone & Conduit 模型

### 3.1 核心概念

```
Zone (区域): 具有相同安全要求的资产集合
  例: Zone-SCADA, Zone-PLC, Zone-SIS

Conduit (管道): 连接Zone的通信通道
  例: Conduit-L3→L2 (SCADA到PLC的通信)

管道安全要求取决于所连接的最高安全级别Zone：
  连接 Zone-SL1 和 Zone-SL2 → 管道安全级别 = SL2
  
管道安全能力示例：
  SL1: 无特殊要求
  SL2: VLAN隔离 + 访问控制列表(ACL)
  SL3: 工控协议DPI + 加密隧道
  SL4: 物理隔离 + 单向网关(Data Diode)
```

## 四、IEC 62443 与其它标准映射

### 4.1 与中国等保2.0的关系

| IEC 62443 | 等保2.0 (GB/T 22239扩展) | 说明 |
|-----------|------------------------|------|
| Part 2-1 (安全管理) | 安全管理制度+管理机构 | 总体安全体系建设 |
| Part 3-1 (技术安全) | 安全计算环境+安全区域边界 | 工控扩展要求 |
| Part 3-3 (系统安全要求) | 工控系统安全扩展要求 | 7个FR(基础要求)→ 等保项 |
| Part 4-2 (组件安全) | 安全通用要求-设备 | 控制器/PLC安全 |
| Zone & Conduit | 安全区域边界 | 工业DMZ+单向隔离 |
| SL 1-4 | 等保1-4级 | 安全等级对标 |

### 4.2 与ISO 27001 / NIST CSF映射

```
IEC 62443 Part 2-1 ≈ ISO 27001 附录A控制项（但专门针对IACS）

NIST CSF五大功能对照：
  Identify   → 资产管理、风险评估 (62443 Part 2-1)
  Protect    → 访问控制、安全配置 (62443 Part 3-3)
  Detect     → 持续监控、异常检测 (62443 Part 3-3)
  Respond    → 事件响应、恢复计划 (62443 Part 2-1)
  Recover    → 恢复计划、灾难恢复 (62443 Part 2-1)
```

## 五、合规路径

```
Phase 1: 差距分析 (Gap Analysis)
  1. 识别所有Zone/Conduit
  2. 评估当前安全级别(SL-A)
  3. 确定目标安全级别(SL-T)

Phase 2: 风险管理
  1. 创建工控专项风险登记表
  2. 评估每个Zone的威胁/脆弱性
  3. 计算剩余风险

Phase 3: 安全方案设计
  1. 选择补偿控制措施
  2. 设计网络架构(Zone/Conduit)
  3. 编写安全用例(Security Case)

Phase 4: 实施与验证
  1. 部署安全措施
  2. SL验证测试
  3. FAT(工厂验收测试) / SAT(现场验收测试)

Phase 5: 持续运维
  1. 补丁管理流程
  2. 定期安全审计
  3. 事件响应演练
```
