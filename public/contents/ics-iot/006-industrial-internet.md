### 1.2 工业数据安全分类

```
工业数据分类（工信部《工业数据分类分级指南》）：

一级数据（工业重要数据）：
  - 关键生产工艺参数
  - 核心设备运行数据
  - 产品质量控制数据
  - 安全环保监测数据
  → 加密存储+传输加密+访问审批+不出厂区

二级数据（工业敏感数据）：
  - 能耗数据
  - 产量统计
  - 设备故障记录
  → 用户认证+权限控制+传输加密

三级数据（工业一般数据）：
  - 公开的产能公告
  - 脱敏后的统计报表
  → 基本访问控制
```

## 二、关键信息基础设施保护

### 2.1 法规要求

```
《关键信息基础设施安全保护条例》(2021年9月1日施行)

CII (Critical Information Infrastructure) 定义：
  一旦遭到破坏/丧失功能/数据泄露，
  可能严重危害国家安全、国计民生、公共利益的设施和系统。

关键行业（《条例》第二条）：
  1. 公共通信和信息服务
  2. 能源（电力/石油/天然气/煤炭）
  3. 交通（铁路/公路/水运/民航）
  4. 水利
  5. 金融
  6. 公共服务（供水/供气/供热）
  7. 电子政务
  8. 国防科技工业

运营者(CIIO)核心义务：
  1. 设立专门安全管理机构
  2. 安全检测评估（≥1次/年）
  3. 网络安全监测和信息报送
  4. 网络安全事件应急预案（≥1次/年演练）
  5. 人员安全管理（关键岗位人员背景审查）
  6. 采购安全审查（网络产品和服务安全审查）
  7. 数据境内存储+数据出境安全评估
```

### 2.2 五阶段保护框架

```
关键信息基础设施保护五阶段：

阶段1: 识别 (Identify)
├── CII资产识别与认定
├── 业务影响分析 (BIA)
├── 风险评估 (RVA)
└── 供应链依赖关系图

阶段2: 防护 (Protect)
├── 纵深防御体系
├── 身份与访问管理 (IAM+PAM)
├── 安全配置基线
├── 补丁与漏洞管理
└── 供应链安全管理

阶段3: 检测 (Detect)
├── 态势感知平台
├── 威胁检测 (IDS/IPS/EDR)
├── 异常行为分析 (UEBA)
├── 欺骗防御 (蜜罐/Honeytoken)
└── 安全日志集中管理(SIEM)

阶段4: 响应 (Respond)
├── 应急响应预案
├── 事件分级与上报
├── 应急指挥系统
├── 协同响应流程
└── 事后复盘与改进

阶段5: 恢复 (Recover)
├── 灾备/容灾体系
├── 业务连续性计划(BCP)
├── 数据恢复验证
└── 恢复后安全审查
```

## 四、等保2.0 工控扩展要求

```
等保2.0 (GB/T 22239-2019) 工业控制系统安全扩展要求：

第三级（等保三级）关键工控扩展要求：

物理安全扩展：
  - 室外控制设备物理防护（防破坏/防盗/防雷/防水/防尘）
  - 无线通信物理边界管控

网络安全扩展：
  - 工业控制系统与企业网之间部署工业防火墙/网闸
  - 工业无线网络与有线网络边界隔离
  - 工控协议深度包检测（针对Modbus/DNP3/S7comm等）
  - 拨号接入审批（远程运维Modem）

主机安全扩展：
  - 工程站/操作员站应用白名单
  - 移动介质(USB)管控
  - 外联检测（禁止工控主机非法外联）

应用安全扩展：
  - 工控软件/固件完整性校验

数据安全：
  - 组态数据/控制逻辑完整性校验
  - 关键数据异地实时备份
```

| 考点 | 频次 | 核心记忆点 |

### 高分考点速查表

| 考察维度 | 关键结论 | 常见干扰项 |
|:---|:---|:---|
| CIIO 定义 | 关键信息基础设施运营者，承担安全主体责任 | "CIIO 只需通过等保即可" ❌ |
| 安全审查 | 采购网络产品和服务可能影响国家安全的需申报 | "所有采购都需安全审查" ❌ |
