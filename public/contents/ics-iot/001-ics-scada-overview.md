# 工业控制系统(ICS/SCADA)安全全景

---

## 一、ICS/SCADA 系统架构

### 1.1 Purdue 模型 (IEC 62443参考架构)

```
Purdue Enterprise Reference Architecture:

Level 5: Enterprise Network (企业网络)
         ERP, CRM, Email, Internet
              ↑ DMZ (防火墙+IPS+DMZ服务器)
Level 4: Site Operations (厂级运营)
         Historian(历史数据库), MES, 报表服务器
              ↑
Level 3: Supervisory Control (监控层/SCADA)
         HMI人机界面, SCADA服务器, 工程师站, OPC服务器
              ↑
Level 2: Local Control (本地控制)
         PLC/DCS控制器, RTU远程终端单元
              ↑
Level 1: Sensors & Actuators (传感器与执行器)
         I/O模块, 变送器, 执行机构
              ↑
Level 0: Physical Process (物理过程)
         电机/阀门/传感器/物理设备

关键安全原则：
  1. 严格控制L4→L3/L3→L2边界
  2. OT流量不路由到IT网络（单向隔离）
  3. L3及以下不使用通用IT协议（RDP/SSH需堡垒机）
```

### 1.2 OT环境特殊要求

```
IT安全 vs OT安全：
┌──────────────┬───────────────────┬───────────────────┐
│ 维度          │ IT               │ OT                │
├──────────────┼───────────────────┼───────────────────┤
│ 首要目标      │ 机密性(CIA倒序)   │ 可用性(AIC正序)    │
│ 补丁管理      │ 定期(月度)        │ 计划停机(年度/大修) │
│ 生命周期      │ 3-5年             │ 15-20年            │
│ 实时性要求    │ 容忍秒级延迟       │ 毫秒/微秒级         │
│ 反病毒        │ 实时扫描全套       │ 仅文件白名单+环境感知 │
│ 重启影响      │ 可接受            │ 生产中断(安全/经济)  │
│ 协议          │ TCP/IP标准        │ Modbus/DNP3/Profinet│
└──────────────┴───────────────────┴───────────────────┘
```

## 二、工控威胁态势

### 2.1 重大工控安全事件

| 年份 | 事件 | 攻击方式 | 影响 |
|------|------|---------|------|
| 2010 | **Stuxnet (震网)** | 5个0day+PLC代码篡改 | 伊朗核设施~1000台离心机损毁 |
| 2014 | **Havex** | 工控供应商网站水坑攻击 | 欧洲能源行业2000+站点被扫描 |
| 2015 | **BlackEnergy3** | 鱼叉邮件+SCADA直接操作 | 乌克兰大停电(23万居民,6小时) |

| 2017 | **TRITON (Trisis)** | 施耐德Triconex安全仪表系统(SIS)攻击 | 沙特石化厂,险些引发物理灾难 |

| 2022 | **INCONTROLLER (Pipedream)** | 施耐德/欧姆龙PLC专用攻击工具 | 美国关键基础设施被针对性攻击 |

### 2.2 TRITON深度分析

```
TRITON (Trisis / HatMan) - 2017年8月：

攻击目标：施耐德Triconex安全仪表系统(SIS)
SIS = Safety Instrumented System: 工业安全的最后一道防线
（当检测到温度/压力/液位等异常时，Shutdown整个工厂防止爆炸/泄漏）

攻击链：
  1. 攻击者进入工程站（SIS Engineering Workstation）
  2. 投递TRITON恶意软件，伪装成Triconex TriStation通信协议
  3. 替换SIS控制器中的固件（植入后门）
  4. 远程操作SIS，可能导致：
     a. 禁用安全保护（工厂在无保护下运行→可能物理爆炸）
     b. 虚假触发Shutdown（经济损失）

关键教训：
  - SIS必须与DCS物理隔离（独立网络/独立工程站）
  - SIS工程站必须严格锁定（禁止安装非授权软件）
  - 定期固件完整性校验
```

## 三、工控安全防护框架

### 3.1 纵深防御

```
Layer 0-1: 物理安全
  ├── 机柜上锁/门禁/视频监控
  ├── 物理访问日志
  └── 未授权USB端口禁用

Layer 2-3: 网络安全
  ├── 工业防火墙(DPI支持工控协议)
  ├── 单向隔离网闸(Data Diode): IT→OT单向
  ├── OT VLAN隔离(不同生产区域独立VLAN)
  └── OT入侵检测(Modbus/DNP3/Profinet DPI)

Layer 3-4: 主机与应用安全
  ├── 应用白名单（仅允许经过签名的可执行文件）
  ├── 工程站/USB设备管控
  ├── 补丁管理(计划停机窗口)
  └── HMI/SCADA系统加固

Layer 5: 监控与响应
  ├── OT SOC/工控安全态势感知
  ├── OT SIEM (工控协议日志解析)
  ├── 物理过程行为基线告警
  └── ICS-CERT威胁情报汇聚
```
