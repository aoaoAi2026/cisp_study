## Playbook 1：钓鱼邮件自动处置

```
触发：邮件网关告警/用户上报
输入：发件人/主题/邮件ID/附件Hash/URL

流程：
① 提取所有IOC（发件人/域名/URL/附件Hash）
② 并行查询威胁情报（VT × URL/附件Hash）
③ 情报判断：
   >3个引擎判定恶意 → 继续
   <2个 → 人工确认
④ 搜索影响范围：
   查询邮件网关 → 同一发件人发了谁
   查询SIEM → 谁点击了链接/下载了附件
⑤ 自动处置（恶意确认后）：
   全量删除邮件（所有收件箱）
   发件人域名加入黑名单
   恶意URL加入防火墙/WAF黑名单
   重置受影响用户密码
⑥ 通知用户 + 创建Case + 记录IOC

MTTR：从4小时 → <3分钟
```

## Playbook 2：EDR 恶意告警自动取证

```
触发：CrowdStrike/SentinelOne/Defender严重告警
输入：主机名/进程ID/告警类型

流程：
① 远程获取进程详情（进程名/命令行/PID/用户）
② 获取进程树（父子关系全链路）
③ 网络连接查询：
   查询SIEM/NetFlow → 该主机的出站连接
   查询DNS → 解析的域名
④ 文件提取+Hash计算：
   进程可执行文件 → Hash → VT查询
⑤ 关联分析：
   同一父进程是否在其他主机也执行？
   同一IP是否连接了其他主机？
⑥ 自动决策：
   VT命中 → 自动隔离主机 + 通知Tier2
   VT未命中 + 行为可疑 → 创建Case通知Tier1
   确定误报 → 标记+关闭
```

## Playbook 3：暴力破解自动封禁

```
触发：SIEM检测到SSH/RDP/WinRM暴力破解
输入：源IP/目标主机/协议/尝试次数

流程：
① 确认非内部运维IP：
   查询CMDB → 源IP是否属于内部运维网段？
   内部IP → 忽略/人工确认
② 威胁情报查询：
   查VT/AbuseIPDB → 源IP是否已知恶意？
③ 自动封禁（满足任一条件）：
   非内部IP + 失败次数>10
   已知恶意IP
   内部测试环境IP
④ 防火墙/安全组添加封禁规则（有效期24小时）
⑤ 通知SOC + 创建Case

回滚：24小时后自动解封（或SOC手动提前解封）
误封处理：允许SOC一键解封 + 加入白名单
```

## Playbook 4：DGA 域名检测与阻断

```
触发：DNS分析检测异常域名
输入：域名/源IP/时间窗

流程：
① 域名特征分析：
   长度>25字符
   熵值>3.5
   低频查询（<10次/24h）
② 查询域名注册信息（WHOIS）：
   昨天刚注册 → 高可疑
   注册商为小众注册商 → 中可疑
③ 查询DNS解析记录：
   解析到VPS/云IP → 高可疑
④ 自动阻断：
   域名 → DNS黑名单
   解析IP → 防火墙黑名单
⑤ 搜索所有查询过该域名的内网主机
   → 关联EDR检查这些主机的进程/连接
```

## Playbook 5-10 摘要

```
PB5: 可疑进程自动隔离
  EDR检测到可疑进程 → 自动挂起进程 →
  提取样本+日志 → 创建Case → 通知Tier2

PB6: 数据外泄自动拦截
  NetFlow/DLP检测大流量出站 → 
  自动添加防火墙规则阻断出站IP → 
  源主机自动隔离 → Case创建

PB7: 漏洞扫描自动派单
  漏洞扫描完成 → 按资产Owner自动创建Jira工单 →
  设定SLA → 到期自动提醒/升级

PB8: 新IOC自动回溯搜索
  威胁情报新增IOC → 
  自动在SIEM回溯搜索历史 → 
  命中→创建Case

PB9: VPN异常登录自动锁定
  异地登录/异常时间/新设备 → 
  自动禁用VPN账号 → 强制重新MFA认证

PB10: 蜜罐告警自动富化
  蜜罐告警 → 攻击源IP自动查询多个情报源 →
  富化信息(Hash/域名/ASN/GeoIP) → 创建Case
```
