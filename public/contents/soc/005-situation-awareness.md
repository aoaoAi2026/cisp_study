# 安全态势感知平台建设

---

## 一、态势感知三层模型

```
安全态势感知 = Situational Awareness

三层体系架构：

┌─────────────────────────────────────────┐
│  呈现层 (Presentation)                   │
│  态势大屏 / 仪表盘 / 报表 / 告警          │
├─────────────────────────────────────────┤
│  分析层 (Analysis)                       │
│  关联分析 / 攻击链检测 / 风险评分 / ML    │
├─────────────────────────────────────────┤
│  数据层 (Data)                           │
│  日志/流量/资产/漏洞/威胁情报/配置基线    │
└─────────────────────────────────────────┘

关键指标（态势要素）：
  资产态势   — 资产总数/在线率/风险分布
  漏洞态势   — 漏洞总数/严重漏洞/修复率
  威胁态势   — 攻击尝试/成功入侵/SOC告警
  事件态势   — 事件数量/级别/MTTD/MTTR
```

## 二、态势感知核心指标

### 2.1 资产态势

```
资产清点 → 安全始于可见性

资产维度：
  IT资产: 服务器/虚拟机/容器/K8s Pod/数据库/中间件
  OT资产: PLC/SCADA/RTU/HMI
  网络资产: 路由器/交换机/防火墙/VPN/打印机
  云资产: AWS EC2/Azure VM/阿里云 ECS/腾讯云 CVM

资产关键属性：
  资产ID → 唯一标识
  IP/主机名 → 网络标识
  操作系统/版本 → 漏洞关联
  业务系统 → 业务影响评估
  Owner → 责任归属
  等保级别 → 合规要求

常见资产发现手段：
  ├── Nmap/Masscan 主动扫描(仅非生产窗口)
  ├── 对接CMDB(ITSM/ServiceNow)
  ├── Agent采集(EDR/HIDS信息)
  ├── 云API同步
  ├── DHCP/DNS日志解析
  └── 网络流量被动识别(NetFlow/sFlow)
```

### 2.2 漏洞态势

```
漏洞生命周期可视化：
  发现 → 评估 → 分发 → 修复 → 验证 → 关闭

关键指标：
  漏洞总数: 各严重度分布(Critical/High/Medium/Low)
  修复率:  已修复/总漏洞
  修复SLA达标率: 按时修复/应修复
  暴露面: 互联网可访问的高危漏洞资产数
  0-day响应: 从发现到临时缓解的时长

1-Day漏洞快速响应(Log4Shell级别)：
  T+0h: 发现漏洞通告
  T+1h: 态势感知平台搜索受影响资产(组件/版本)
  T+2h: 输出受影响资产清单
  T+4h: 短期缓解措施完成(WAF规则/临时下线)
  T+24h: 正式修复方案发布
  T+72h: 核心资产全量修复完成
```

## 三、告警降噪策略

```
安全运营的"告警疲劳"：
  日均告警: 10,000+ → 可处理 <200 → 98%是噪音

降噪五层策略：

1. 白名单 (Explicit Allow)
   运维IP/自动化工具/已知测试
   → 不生成告警

2. 阈值优化 (Threshold Tuning)
   单个SSH失败 ≠ 暴力破解
   5次/2分钟 = 暴力破解
   → 调整告警触发阈值

3. 资产上下文 (Asset Context)
   非重要资产的低危告警 → 降级
   核心资产的低危行为 → 也要关注
   → 根据资产重要度动态调整

4. 告警合并 (Alert Aggregation)
   同一攻击源的100条告警 → 合并为1个事件
   → 减少告警数量

5. 动态基线 (Dynamic Baseline)
   月底运维窗口大量RDP访问 → 正常
   平时的RDP访问 → 异常
   → 学习正常行为模式
```

## 四、安全大屏设计

### 4.1 大屏布局

```
安全态势大屏推荐布局 (3列 x 2行)：

┌──────────────┬──────────────┬──────────────┐
│  总体态势     │  威胁监控     │  事件处理     │
│  今日攻击次数  │  实时攻击地图  │  待处理告警    │
│  风险评分     │  Top威胁IP   │  MTTD/MTTR   │
│  资产覆盖率    │  Top攻击类型  │  事件趋势     │
├──────────────┼──────────────┼──────────────┤
│  资产态势     │  漏洞态势     │  合规态势     │
│  资产总数     │  漏洞分布     │  密评进度     │
│  在线率      │  修复率      │  等保达标     │
│  风险资产分布  │  到期未修复   │  合规差距     │
└──────────────┴──────────────┴──────────────┘
```

### 4.2 Grafana/Kibana 实现

```
Grafana — 时间序列指标大屏：
  - 攻击趋势(线图)
  - 漏洞分布(饼图)
  - 告警TOP10(柱状图)
  - 世界地图攻击源(Geomap)

Kibana — 日志分析大屏：
  - Elastic Dashboard + Canvas
  - Lens可视化
  - 告警规则告警
  - 钻取分析(点击告警 → 查看原始日志)
```
