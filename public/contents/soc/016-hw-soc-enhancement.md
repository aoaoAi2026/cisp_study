## 一、护网期间 vs 平日 SOC

```
┌────────────┬──────────────────┬──────────────────┐
│ 维度        │ 平日 SOC          │ 护网 SOC          │
├────────────┼──────────────────┼──────────────────┤
│ 告警优先级  │ 按资产重要性      │ 所有外网告警+1级   │
│ EDR策略     │ 检测为主          │ 阻断模式全开       │
│ 值守        │ 工作日 8-18       │ 7x24 双人值守     │
│ 规则集      │ 常规检测规则       │ +护网专用规则      │
│ 处置速度    │ SLA按级别         │ 可见即可处置      │
│ 容忍度      │ 允许误报          │ 宁杀错不放过       │
└────────────┴──────────────────┴──────────────────┘
```

## 二、告警优先级动态调整

```
护网告警升级规则：
  ✦ 所有来自外网的告警 → 在原等级基础上+1级
    例：平日P3(异常登录) → 护网P2
  ✦ 任何尝试内网扫描 → 直接P2
  ✦ 任何Web攻击尝试 → 直接P2
  ✦ EDR检测到可疑行为 → 直接P1
  ✦ 任何成功登录告警(非VPN) → 直接P1

临时降级白名单：
  仅允许已知的护网安全产品IP(扫描器/蜜罐流量)
  其他白名单规则全部暂停(护网结束时恢复)
```

## 三、EDR 策略强化

```
护网前必须完成的EDR调整：

Windows:
  □ 策略从"仅检测"→"检测+阻断"
  □ PowerShell约束语言模式 → ConstrainedLanguage
  □ 启用Attack Surface Reduction(ASR)规则(全部)
  □ 启用Controlled Folder Access(防勒索)
  □ 启用Credential Guard(防凭据窃取)

Linux:
  □ auditd规则从基础→增强(新增execve/网络连接监控)
  □ SELinux从Permissive→Enforcing(护网前1周)
  □ 内核模块加载限制(kernel.modules_disabled=1)

网络层:
  □ 边界防火墙策略收紧(默认拒绝出站 → 白名单放行)
  □ DNS出站仅允许企业DNS服务器
  □ 非标准端口出站全部阻断
```

## 四、7x24 双人值守

```
护网排班：
  每班8小时，双人值守（一主一备）
  主值：告警处理+事件响应
  备值：告警审核+规则优化+情报刷新

  班次：
    早班 08:00-16:00  (2人)
    中班 16:00-00:00  (2人)
    夜班 00:00-08:00  (2人)

交接班流程：
  ① 交班人填写交接班记录（当班处理的事件/发现的异常）
  ② 接班人审阅并确认
  ③ 双方签字
  ④ 交班人至少等15分钟确保接班人上手

值守工具：
  ✦ 告警监控大屏(SIEM Dashboard)
  ✦ 即时通讯群(企业微信/钉钉/Slack)
  ✦ 应急电话(值班长24小时开机)
```

## 五、护网专用检测规则

```
护网专项Sigma规则集：

① 护网红队常用TTP检测
  - Cobalt Strike Beacon特征
  - Metasploit Meterpreter流量
  - 自定义DNS隧道检测

② 横向移动检测强化
  - 所有PsExec/WMI/WinRM远程执行
  - 非域控主机的DCSync请求
  - 异常的Kerberos票据请求(TGS/TGT)

③ 外联检测
  - 所有DNS查询的非域名（DGA检测）
  - HTTPS出站到非常见CDN/云服务
  - ICMP大包（可能的ICMP隧道）

④ 数据外泄检测
  - 大流量出站(>100MB/hour)
  - 压缩包(.zip/.7z/.rar)大量出站
  - 数据库dump文件传输
```

## 六、护网报表体系

```
护网报表频率：小时报 + 日报 + 终报

小时报模板：
  【护网 X时-X时 安全态势】
  当前状态: 正常/关注/告警
  本时段攻击尝试: XX次
  本时段成功拦截: XX次
  当前待处理告警: X条
  人员状态: 全员在岗/XX请假

日报模板：
  【护网 Day X 日报】
  今日攻击趋势(时序图)
  Top 攻击源(IP+国家)
  Top 攻击类型
  处置事件汇总(共X起)
  待办事项(明日重点关注)

终报(护网结束后)：
  全程攻击趋势总览
  红队TTPs提取
  蓝队能力评估
  发现的安全短板
  整改优先级建议
```

## 七、护网后规则回滚

```
护网结束 → 1周内完成：

□ 告警优先级规则 → 恢复正常
□ EDR策略 → 回滚到检测模式
□ 白名单恢复
□ 护网专用规则 → 抽取出有效部分融入日常规则
□ 值守模式 → 恢复正常排班
□ 经验沉淀：护网期间的TTPs → 编写长期检测规则

保留：
  ✓ 护网期间证明有效的规则
  ✓ 新发现的攻击模式
  ✓ 改进的响应流程
```
