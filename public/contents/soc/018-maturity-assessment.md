# 安全运营成熟度评估与持续改进

---

## 一、SOC-CMM 成熟度模型

```
SOC-CMM (SOC Capability Maturity Model) 五级：

L1: Initial (初始级)
  特征：无正式SOC，告警全靠"碰巧看见"
  检测：被动依赖工具默认规则
  响应：无标准化流程，凭个人经验
  
L2: Managed (已管理级)
  特征：有SOC团队，有基础SIEM
  检测：有关键资产告警，但覆盖率<60%
  响应：有基本SOP但未严格执行

L3: Defined (已定义级)
  特征：SOC流程标准化，有SOAR基础
  检测：ATT&CK Top 30覆盖，告警降噪有效
  响应：SOP+部分自动化，MTTD<1小时

L4: Quantitative (量化管理级)
  特征：SOC运营指标驱动，自动化>60%
  检测：ATT&CK Top 50+覆盖，UEBA+NDR融合
  响应：MTTD<30min, 自动化率>60%

L5: Optimizing (优化级)
  特征：持续自适应安全，AI驱动运营
  检测：自学习检测模型，近零误报
  响应：MTTD<5min, 自动化率>85%
```

## 二、五域评估法

```
对每个域打分(1-5)，取平均→整体成熟度

域1: 人员 (People)
  □ SOC团队规模与梯队
  □ 技能培训计划与认证
  □ 7x24值守安排
  评分: __/5

域2: 流程 (Process)
  □ 告警处理SOP
  □ 事件响应流程
  □ 漏洞管理流程
  □ 变更管理流程
  评分: __/5

域3: 技术 (Technology)
  □ SIEM平台能力
  □ EDR/NDR/UEBA覆盖
  □ SOAR自动化
  □ 日志覆盖率
  评分: __/5

域4: 度量 (Metrics)
  □ MTTD/MTTR/KPI体系
  □ 运营日报/周报/月报
  □ 持续改进追踪
  评分: __/5

域5: 治理 (Governance)
  □ 管理层支持
  □ 预算保障
  □ 合规对接
  □ 外部协作
  评分: __/5

总体成熟度 = Σ各域评分 / 5 = __/5
```

## 三、年度改进路线图

```
从 L2 → L3 (6-12个月)：
  重点：流程标准化 + 告警降噪
  投资：SIEM规则优化 + Tier1-2培训
  预期：MTTD从2h→1h, 可处理告警从500→200

从 L3 → L4 (12-18个月)：
  重点：SOAR自动化 + UEBA + ATT&CK覆盖
  投资：SOAR平台 + UEBA部署
  预期：MTTD<30min, 自动化率>60%

从 L4 → L5 (18-24个月+)：
  重点：AI驱动 + 自适应安全
  投资：AI安全分析平台 + 红紫篮对抗常态化
  预期：MTTD<5min, 自动化率>85%
```

## 四、MSSP 外包成熟度评估

```
选择MSSP的安全运营外包模式：

共享SOC：
  ✦ 成本最低
  ✦ 告警通知 → 客户自己处置
  ✦ 适用：小型企业，预算有限

混合SOC：
  ✦ 7x24监控外包 + 复杂事件自建
  ✦ 优势：关键数据不出境/不出公司
  ✦ 适用：中型企业

专属SOC：
  ✦ MSSP为你建专属SOC
  ✦ 白牌交付(贴上客户品牌)
  ✦ 适用：大型企业

评估MSSP的关键问题：
  ① 你的一线分析师有多少经验？
  ② 你们的误报率控制到多少？
  ③ 告警到通知的延迟是多少？
  ④ 你们用什么SIEM/SOAR平台？
  ⑤ 护网期间能否提供增强服务？
```

## 五、ROI 量化

```
SOC投资回报量化方法：

直接成本避免：
  一次勒索攻击平均损失：￥200万-2000万
  一次数据泄露平均损失：￥500万-5000万
  → SOC如果能每年阻止1-2次重大事件 → ROI > 500%

效率提升：
  自动化每天节省分析师 4小时×分析员数
  例：5人团队 × 4h × 250工作日 = 5000小时/年

合规价值：
  避免因安全事件导致监管处罚
  满足等保/密评/关基检查要求
```
