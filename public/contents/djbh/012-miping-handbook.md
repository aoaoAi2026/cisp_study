# 密评实操手册：评估流程 / 关键检查项 / 整改

---

## 一、密评全流程

```
密评五步:

Step 1: 密码应用方案评审(建设前)
  审查密码应用方案是否满足 GB/T 39786

Step 2: 现场测评
  测评师现场检查: 产品/配置/密钥管理

Step 3: 测评报告
  输出: 符合/部分符合/不符合

Step 4: 整改(如不通过)
  按照不符合项逐项整改

Step 5: 复测
  整改完成后复测 → 通过 → 获取报告
```

---

## 二、关键检查项

```
检查1: 身份鉴别
  ✓ 登录是否使用国密数字证书? (SM2)
  ✗ 仅用密码 → 不符合

检查2: 通信加密
  ✓ 是否使用国密TLCP? (不是TLS)
  ✗ 使用国际TLS → 不符合

检查3: 数据加密
  ✓ 数据库敏感字段是否SM4加密?
  ✗ 明文存储 → 不符合

检查4: 数据完整性
  ✓ 日志是否有HMAC-SM3校验?
  ✗ 无校验 → 不符合

检查5: 密钥管理
  ✓ 密钥是否HSM/KMS管理?
  ✗ 硬编码/明文存储 → 严重不符合
```

---

## 三、典型整改方案

```
场景: 政务网站密评不通过

问题: HTTPS使用国际TLS(RSA+AES-GCM)
整改:
  ① Nginx → 铜锁版(支持TLCP)
  ② CFCA申请SM2双证书
  ③ 配置国密密码套件: ECC-SM2-SM4-GCM-SM3
  ④ 测试国密浏览器可访问 → 密评通过

场景: 数据库密评不通过
问题: 敏感字段明文存储
整改:
  ① 应用层增加SM4加密/解密
  ② 密钥存储在KMS
  ③ 加密字段: 身份证/手机号/银行卡
```
