# 安全管理制度体系建设模板（8 大类文件清单）

---

## 一、8大类文件全景

```
等保三级管理制度文件 = 8 大类 + 至少 20+ 具体文件

1. 安全策略 (1-2份)
2. 安全管理制度 (3-5份)
3. 安全操作规范 (3-5份)
4. 人员安全管理制度 (4-6份)
5. 安全建设管理制度 (3-4份)
6. 安全运维管理制度 (4-6份)
7. 应急管理制度 (2-3份)
8. 监督检查制度 (2-3份)
```

---

## 二、逐类模板

### 1. 安全总纲

```markdown
# XX公司网络安全管理总纲

## 第一章 总则
第一条 为保障公司信息系统的安全稳定运行，依据《网络安全法》
《等级保护条例》等法律法规，制定本总纲。

第二条 本制度适用于公司所有信息系统及全体人员...

## 第二章 组织与职责
第四条 公司设立网络安全领导小组，由分管副总任组长...

## 第三章 安全管理原则
第七条 遵循"谁主管谁负责，谁运行谁负责"原则
第八条 实行三员分立（系统管理员/安全管理员/审计管理员）
```

### 2. 密码管理制度

```markdown
# XX公司密码安全管理制度

## 一、密码策略
1. 最小长度: ≥12个字符
2. 复杂度: 大写+小写+数字+特殊字符中至少3类
3. 最长使用期限: 90天
4. 最短使用期限: 1天(防止立即改回)
5. 强制密码历史: 记住最近10个密码

## 二、账户锁定策略
1. 连续5次失败登录→锁定30分钟
2. 管理员手动解锁或等待超时

## 三、特权账户管理
1. root/Administrator密码由堡垒机托管, 定期轮换
2. 使用堡垒机运维, 不直接使用特权账户
```

### 3. 备份恢复制度

```markdown
# XX公司数据备份与恢复管理制度

## 一、备份策略
1. 核心数据库: 每日全量备份, 保留30天
2. 文件服务器: 每日增量, 每周全量, 保留90天
3. 异地备份: 每日同步到异地灾备中心

## 二、恢复测试
1. 每季度进行一次恢复演练
2. 演练范围: 随机抽取1个数据库+1个文件系统
3. 演练记录: 时间/恢复目标/恢复结果/问题

## 三、备份介质管理
1. 备份介质异地存放
2. 介质定期检查可读性(每半年)
3. 退役介质安全擦除或物理销毁
```

### 4. 应急管理制度

```markdown
# XX公司网络安全事件应急管理制度

## 一、事件分级
P1 (特别重大): 核心系统中断/大量数据泄露
P2 (重大): 重要系统中断/非核心数据泄露
P3 (较大): 一般系统受攻击/可疑行为
P4 (一般): 扫描/探测/低危告警

## 二、响应流程
1. 发现→上报(10分钟内)
2. 研判→定级(30分钟内)
3. 启动应急响应(1小时内)
4. 处置→恢复(视情况)

## 三、应急组织
应急响应组长: CSO/安全负责人
技术组: 安全+运维
业务组: 受影响部门负责人
联络组: 法务+PR(如需对外通报)
```

### 5. 人员管理制度

```markdown
# XX公司人员安全管理制度

## 一、录用审查
1. 关键岗位人员: 背景审查+学历验证+从业资格
2. 签署保密协议

## 二、安全培训
1. 入职培训: 安全制度+操作规范(1天)
2. 年度培训: ≥1次/年
3. 专项培训: 钓鱼识别/护网动员

## 三、离岗管理
1. 收回门禁卡/系统账户/邮箱/VPN
2. 签署离职保密承诺书
3. 离岗审计(检查最近1月操作记录)
```

### 6. 变更管理制度

```markdown
# XX公司变更管理制度

## 一、变更分类
标准变更: 低风险, 预授权(补丁更新)
普通变更: 需审批(配置修改)
紧急变更: 先执行后补流程(安全事件响应)

## 二、变更流程
申请→审批→测试→实施→验证→归档

## 三、禁止变更窗口
业务高峰期/护网期间/重大活动期间
```

### 7. 供应商管理制度

```markdown
# XX公司供应商安全管理制度

## 一、供应商准入
1. 安全资质审查(等保证书/安全认证)
2. 签署保密协议+安全责任书

## 二、供应商访问
1. 远程访问→堡垒机+VPN
2. 物理访问→登记+陪同
3. 访问权限最小化+定期审计

## 三、供应商退出
1. 回收所有访问权限
2. 清除供应商系统内的测试数据
3. 确认数据无残留
```

### 8. 审计管理制度

```markdown
# XX公司安全审计管理制度

## 一、审计范围
系统操作/数据库访问/网络设备配置/安全策略变更

## 二、审计频率
自动审计: 实时
定期审计: ≥1次/月

## 三、审计记录保护
1. 审计记录不可删除/不可修改
2. 审计记录保留≥6个月
3. 审计进程不被终止
```
