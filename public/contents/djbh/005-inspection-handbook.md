## 一、测评前准备（T-7天）

### 硬件环境

```
□ 安全产品全部运行(绿灯, 无告警灯)
□ 服务器/网络设备正常运行
□ 日志系统正常(确认日志在生成)
□ 测评用测试机准备(安装测评工具权限)
□ 机房整洁/温湿度正常/消防在位
```

### 文档准备

```
□ 管理制度文件装订成册(8大类)
  - 网络安全管理总纲
  - 安全管理制度
  - 安全操作规范
  - 人员管理制度
  - 应急管理制度
  - 运维管理制度
  - 外包管理制度
  - 安全检查制度

□ 定级报告 + 专家评审意见
□ 备案证明 复印件
□ 系统拓扑图(最新版, 标注安全产品)
□ 数据流图
□ 安全设施清单(型号+位置)
□ 上次测评报告(如有)
```

### 人员

```
□ 测评期间人员安排确认:
  - 至少1名系统管理员在岗
  - 至少1名安全管理员在岗
  - 网络管理员/数据库管理员联系方式
  - 测评陪同人员(全程跟随)
```

## 二、测评日配合

### Day 1-2: 文档审查 + 访谈

```
测评师会审查:
  ✦ 管理制度文件是否完整
  ✦ 定级报告格式是否规范
  ✦ 网络拓扑图是否与实际一致

配合要点:
  ✓ 所有文件准备好, 测评师要看什么立即提供
  ✓ 文件放在测评师手边(不要翻半天)
  ✓ 对缺失的文件不要隐瞒 → 诚实说明正在补
```

### Day 2-4: 技术检测

```
测评师会执行:
  ✦ 漏洞扫描(通常在夜间或业务低峰期)
  ✦ 基线核查(登录服务器检查配置)
  ✦ 渗透测试(在授权范围内)
  ✦ 弱口令检测

配合要点:
  ✓ 登录凭据事先准备好(测试账户)
  ✓ 测评师操作必须有陪同人员在场
  ✓ 记录测评师发现的所有问题(便于后续整改)
```

### Day 5: 结果确认

```
测评师输出初步问题清单 → 逐项确认

配合要点:
  ✓ 对问题有异议 → 当场沟通 + 提供证据
  ✓ 确认的问题 → 签字确认
  ✓ 记录所有扣分项 → 回去整改
```

## 三、人员安排

```
核心人员:

测评总协调人(1人):
  - 负责与测评师沟通
  - 调配资源
  - 每日进度汇报

系统管理员(1-2人):
  - 陪同登录服务器
  - 演示安全配置
  - 回答技术问题

安全管理员(1人):
  - 展示安全产品(WAF/堡垒机等)
  - 解释安全策略

网络管理员(1人):
  - 提供网络拓扑
  - 配合网络设备检查

文档负责人(1人):
  - 管理所有文档
  - 随时提供测评师需要的文件

轮岗: 测评2周 → 核心人员不够用 → 确保AB角
```

## 四、访谈问题

```
测评师常见访谈问题 + 参考回答:

Q: "密码策略是什么？"
A: "密码长度≥12位，包含大小写+数字+特殊字符,
    90天强制更换，5次失败锁定30分钟。
    这是当前系统的配置，您可以登录验证。"

Q: "备份恢复测试做过吗？"
A: "做过。这是最近一次(2026年5月)的恢复测试记录，
    从备份服务器恢复了一个测试数据库，耗时45分钟，数据完整。
    测试记录和报告在这里。"

Q: "安全事件响应流程是什么？"
A: "发现安全事件 → 上报安全管理员 → 
    判断事件等级 → P1事件立即启动应急响应组 →
    30分钟内通知管理层 → 1小时内采取遏制措施。
    这是我们的事件响应预案文件。"

Q: "日志保留多久？"
A: "热数据30天(SSD)，温数据90天(HDD)，
    冷数据归档365天(S3)。这是日志管理系统的截图。"

Q: "怎么给新员工分配系统权限？"
A: "HR发送入职通知 → IT创建基础账户(默认最小权限) →
    部门主管提交权限申请表 → 安全管理员审批 →
    按需分配额外权限。最小权限原则。"
```

## 五、测评后跟进

```
测评结束后:

Week 1:
  □ 获取测评报告
  □ 分析扣分项
  □ 制定整改计划
  □ 分配整改责任人

Week 2-4:
  □ 逐项整改扣分项
  □ 自评 → 确认整改完成

Month 2:
  □ 如需要: 申请复测
  □ 获取最终报告

持续:
  □ 年度自评
  □ 年度测评
  □ 变更管理
```
