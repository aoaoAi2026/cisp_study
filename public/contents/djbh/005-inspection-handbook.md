# 等保测评迎检配合手册（甲方侧）

## 1. 时间节点安排

| 阶段 | 时间 | 主要工作 |
|------|------|---------|
| 准备阶段 | 测评前 4 周 | 内部自查、补齐制度、准备资料 |
| 预评估 | 测评前 2 周 | 内部预评估、修复明显高风险问题 |
| 现场测评 | 1 – 2 周 | 文档审查、访谈、配置核查、工具测试、渗透测试 |
| 整改期 | 测评后 2 – 4 周 | 对不符合项整改、准备复测 |
| 复测与完成 | 整改后 1 周 | 复测、出具正式报告、完成备案更新 |

## 2. 资料清单（提前打包准备）

## 3. 访谈准备（按岗位整理）

### 3.1 系统管理员（Linux / Windows）

- 账号管理策略：命名规范、禁用策略、口令策略
- 远程运维：是否通过堡垒机、是否启用 MFA、源 IP 限制
- 补丁管理：周期、方式、紧急补丁流程
- 备份：备份频率、保留时间、恢复验证
- 日志：开启哪些日志、保留时长、是否集中收集

### 3.2 安全管理员

- 安全设备清单与策略
- 漏洞管理流程
- 事件处置流程与经验
- 与外包 / 上级的沟通机制

### 3.3 审计管理员

- 角色是否与系统管理员 / 安全管理员分离
- 审计日志范围、周期、集中收集、访问控制
- 是否对审计操作本身有记录

### 3.4 网络管理员

- 拓扑、VLAN、防火墙策略
- 边界防护、入侵检测、IP/MAC 绑定
- 网络变更流程

### 3.5 开发 / 应用负责人

- 上线前安全测试流程
- 代码是否经过安全评审
- 数据分类分级与加密策略
- 应用日志、异常处理

### 3.6 业务负责人

- 系统功能、用户规模、业务数据敏感度
- 第三方合作情况
- 业务连续性与容灾计划

## 4. 现场配合要点

1. **指定总协调人**：负责总体沟通、资源协调、问题响应（一般为信息安全负责人 / 信息中心主任）
2. **每日例会**：当天问题汇总、明日计划、临时问题解决方案
3. **证据留存**：所有访谈、核查过程在内部记录在案；输出文件加盖公章 / 签字确认
4. **现场问题处置 SLA**：
   - P0 问题：立即处理
   - P1 问题：24 小时内给出处理方案
   - P2 / P3：纳入整改计划，明确时间节点
5. **避免口头承诺**：所有承诺与计划都落到书面备忘录

## 5. 测评报告反馈

- 对测评机构出具的《不符合项清单》逐项确认并给出整改计划
- 对报告中的错误或误判，提供证据进行沟通
- 整改完成后组织复测
- 最终报告加盖公章后提交公安网安备案更新

## 6. 测评后长效机制

- **年度计划**：每年一次内部自查 + 一次等保测评
- **安全预算**：每年固定安全运营 / 设备维保 / 培训预算
- **KPI**：漏洞修复率、事件响应时长、制度执行率
- **重大变更**：业务系统重大调整、架构变更时重新定级与测评
