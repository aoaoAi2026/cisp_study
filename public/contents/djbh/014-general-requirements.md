## 二、安全物理环境

| 编号 | 控制点 | 三级要求摘要 | 常见扣分项 |
|------|--------|------------|-----------|
| 8.1.1.1 | 物理位置选择 | 机房场地避免地下室/顶层/水管道附近 | 机房设在负一楼 |
| 8.1.1.2 | 物理访问控制 | 机房出入口电子门禁+专人值守 | 刷卡不记录/尾随进门 |

| 8.1.1.7 | 防静电 | 防静电地板+接地 | 无防静电接地 |

## 三、安全通信网络

| 编号 | 控制点 | 三级要求摘要 | 常见扣分项 |
|------|--------|------------|-----------|
| 8.1.2.1 | 网络架构 | 划分安全区域/重要区域部署防火墙 | VLAN间无ACL/防火墙策略宽松 |
| 8.1.2.2 | 通信传输 | 远程通信采用密码技术加密 | 远程运维明文Telnet/管理口无加密 |
| 8.1.2.3 | 可信验证 | 基于可信根对通信设备进行可信验证 | ★此项为等保2.0新增，多数系统无 |

**关键解读**：

```
"通信传输采用密码技术保证完整性、保密性"

三级要求解读（非密评，仅等保）：
  ✓ Web管理界面 → HTTPS (TLS 1.2+)
  ✓ 远程运维SSH → SSHv2
  ✓ VPN → IPsec/SSL VPN
  ✓ 数据库连接 → TLS
  不强制要求国密（等保三级），但推荐
  密评要求时才强制国密（SM2/SM3/SM4）

常见误区：
  "等保三级必须用国密SSL" — 错！
  等保三级：推荐用国密，但TLS也可
  密评（密码应用评估）：必须用国密
```

## 四、安全区域边界

| 编号 | 控制点 | 三级要求摘要 | 常见扣分项 |
|------|--------|------------|-----------|
| 8.1.3.1 | 边界防护 | 区域边界防火墙/网闸 | 无DMZ或防护设备 |
| 8.1.3.2 | 访问控制 | 基于会话状态/应用协议粒度的访问控制 | 防火墙规则过于宽松 |
| 8.1.3.3 | 入侵防范 | 主动阻断网络攻击 | 无IPS/未更新规则库 |
| 8.1.3.4 | 恶意代码防范 | 边界恶意代码检测 | 无防病毒网关/未更新 |
| 8.1.3.5 | 安全审计 | 边界设备日志审计 | 无日志/日志保留<6个月 |

**区域边界配置示例**：

```
互联网 ── [DDoS清洗] ── [边界防火墙] ── [IPS] ──
                                            ├── [DMZ: Web/Nginx]
                                    [核心防火墙] ──
                                            ├── [应用区: App/API]
                                            ├── [数据区: DB/Cache]
                                            └── [管理区: 堡垒机/监控]
```

## 五、安全计算环境（核心）

| 编号 | 控制点 | 三级要求摘要 | 常见扣分项 |
|------|--------|------------|-----------|
| 8.1.4.1 | 身份鉴别 | 唯一标识+密码复杂度+登录失败处理 | 共享账号/弱口令/无锁定策略 |
| 8.1.4.2 | 访问控制 | 最小权限+默认账户禁用+访问控制策略 | 未禁用Guest/Administrator改名 |
| 8.1.4.3 | 安全审计 | 全量审计+保护审计记录+审计进程保护 | 审计未开启/日志未集中存储 |
| 8.1.4.4 | 入侵防范 | 最小安装+补丁更新+安全加固 | 默认组件未卸载/补丁滞后 |
| 8.1.4.5 | 恶意代码防范 | 防病毒软件+定期查杀 | 防病毒未安装/病毒库过期 |
| 8.1.4.6 | 资源控制 | 限制终端登录方式+超时退出 | 无超时锁定/无登录限制 |
| 8.1.4.7 | 数据完整性 | 重要数据校验 | 无完整性校验机制 |
| 8.1.4.8 | 数据保密性 | 重要数据加密存储 | 数据库明文存储敏感字段 |
| 8.1.4.9 | 数据备份恢复 | 本地+异地备份+恢复测试 | 无备份/备份从不测试恢复 |
| 8.1.4.10 | 剩余信息保护 | 释放存储空间前清除信息 | 无文件/内存清除机制 |

**身份鉴别最强扣分项**：

```bash
# 测评工具检查清单
# 1. 密码策略检查
net accounts   # Windows密码策略
cat /etc/login.defs | grep PASS   # Linux密码策略

# 2. 登录失败处理
# Windows: gpedit.msc → 安全设置 → 账户锁定策略
# Linux: /etc/pam.d/system-auth → pam_faillock.so

# 3. 超时退出
echo $TMOUT   # Shell超时
grep ClientAliveInterval /etc/ssh/sshd_config  # SSH超时

# 4. 禁止Root远程登录
grep PermitRootLogin /etc/ssh/sshd_config
# → 应该是 PermitRootLogin no
```

## 六、安全管理中心

| 编号 | 控制点 | 三级要求摘要 | 常见扣分项 |
|------|--------|------------|-----------|
| 8.1.5.1 | 系统管理 | 系统管理员权限分离 | 无审计管理员/安全管理员角色 |
| 8.1.5.2 | 审计管理 | 审计记录集中管理 | 审计分散/无综合分析 |
| 8.1.5.3 | 安全管理 | 安全策略集中管控 | 策略分散/无统一管理 |
| 8.1.5.4 | 集中管控 | 统一监控+运维管控+安全事件处置 | ★等保2.0新增，多数系统短板 |

**三员分立原则**：

```
系统管理员：负责系统运维（安装/配置/升级）
安全管理员：负责安全策略（权限分配/策略配置）
审计管理员：负责安全审计（日志检查/违规审计）

三员必须由不同人担任（至少分开账户）
高权限操作需三员中的两员在场（二人规则）
```

## 八、测评技巧

```
测评三方法：
  1. 访谈 — 询问IT/安全人员实际操作流程
     例："你如何给新员工分配系统权限？"
  
  2. 检查 — 查看文档+配置+日志
     例：检查防火墙策略是否最小化
     例：检查审计日志是否开启并留存6个月以上
  
  3. 测试 — 使用工具进行技术测试
     例：漏洞扫描工具检测系统漏洞
     例：尝试绕过访问控制

常见测评高风险（直接扣分超标→不通过）：
  ✗ 互联网可直接访问核心数据库
  ✗ 默认口令未修改（admin/admin123）
  ✗ 无任何审计日志
  ✗ 无备份机制
  ✗ 远程运维无加密（明文Telnet）
```
