# 等保测评常见高风险问题 Top 20 与整改建议

## 1. 身份鉴别类

1. **操作系统/数据库/应用系统未启用双因素认证**
   - 整改：VPN、运维堡垒机、服务器、核心数据库启用 MFA（动态口令 / 短信 / 硬件 Key）
   - 参考要求：GB/T 22239-2019 S4.1.3.1 / S4.4.3.1

2. **存在默认 / 弱口令 / 空口令账号**
   - 整改：全量扫描（Hydra / Ncrack / 自研工具）+ 统一密码策略（复杂度 ≥ 8、定期更换、禁用历史密码）

3. **存在共享账号 / 多人共用同一账号**
   - 整改：实行一人一号；通过堡垒机做操作审计与个人身份绑定

4. **登录失败无锁定策略**
   - 整改：Windows / Linux / SSH / 应用登录失败 5 次后锁定 15 分钟

## 2. 访问控制类

5. **管理后台暴露在公网**
   - 整改：管理后台只允许内网访问或通过 VPN + 白名单 IP 访问

6. **账号权限过大，不符合最小权限原则**
   - 整改：根据岗位 / 角色做 RBAC；定期（季度）权限评审

7. **数据库访问账号权限过高（DBA 权限给业务用户）**
   - 整改：按业务创建专用用户，授予 SELECT/INSERT 最小权限；存储过程 / UDF 禁用

## 3. 安全审计类

8. **无集中日志收集与审计平台**
   - 整改：部署 SIEM / 日志平台，收集主机 / 网络 / 数据库 / 应用日志 ≥ 6 个月

9. **重要主机 / 数据库未开启审计**
   - 整改：开启 Linux auditd、Windows 安全日志、MySQL / Oracle / SQL Server 审计

10. **日志未包含关键安全事件（登录、权限变更、敏感操作）**
   - 整改：完善业务应用日志，记录 Who / When / Where / What / Result

11. **安全审计员角色未分离**
   - 整改：设置独立的审计管理员角色，与系统管理员、安全管理员三权分离

## 4. 入侵防范与边界防护

12. **无入侵检测/防护设备（IDS/IPS/APT 检测）**
   - 整改：在关键边界部署 NGFW + IPS / WAF + 主机 EDR

13. **未定期进行漏洞扫描与渗透测试**
   - 整改：至少每季度一次漏洞扫描；二级系统每年至少一次渗透测试；三级系统每半年一次

14. **公网开放非必要端口（Redis / MongoDB / 数据库 / Docker API）**
   - 整改：端口收敛 + 源 IP 白名单；Redis 加密码、禁用 CONFIG 命令

## 5. 恶意代码与补丁

15. **服务器未部署主机防病毒 / EDR**
   - 整改：部署 EDR（360 天擎 / 奇安信天擎 / 火绒企业版 等）

16. **未建立补丁管理机制，存在大量历史高危补丁未打**
   - 整改：建立月度补丁窗口 + SLA（严重 7 天、高危 30 天）；使用 WSUS / Ansible / Salt

## 6. 数据安全与备份

17. **敏感数据明文存储（密码明文 / 身份证号明文 / 手机号明文）**
   - 整改：密码使用 bcrypt / argon2；身份证、银行卡号、手机号脱敏存储，使用加密字段或 TDE

18. **无异地数据备份 / 备份未定期恢复验证**
   - 整改：本地 + 异地备份（3-2-1 原则）；每季度至少一次恢复演练并留存记录

19. **重要数据传输通道未加密（HTTP / Telnet / FTP）**
   - 整改：全站 HTTPS（HSTS）、禁用 Telnet/FTP 改用 SSH/SFTP/SCP、数据库连接启用 SSL

## 7. 安全管理类

20. **无信息安全管理制度 / 无应急响应预案 / 无年度安全培训**
   - 整改：按《安全管理制度体系建设模板》补齐制度；组织应急演练（每半年至少一次）；全员安全培训（至少每年一次）

---

> 等保不是终点，而是信息安全的基线。通过持续整改与运营，才能形成真正的防御能力。
