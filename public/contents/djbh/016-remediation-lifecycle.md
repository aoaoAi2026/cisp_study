```
整改项目生命周期：

  差距分析 → 方案设计 → 采购招标 → 实施部署 → 验收测试 → 测评配合 → 持续运维
    2周        2-4周      4-8周      4-12周     1-2周     2-4周      长期
```

## 二、预算编制

### 2.1 等保三级典型预算构成

```
等保三级整改预算参考（中小型系统 10-50台服务器）：

软件产品：
  ├── 堡垒机                    ~5-15万
  ├── 日志审计系统              ~5-20万
  ├── 漏洞扫描系统（如无法复用）  ~5-10万
  ├── 数据库审计系统             ~10-30万
  ├── WAF（Web应用防火墙）       ~5-20万
  ├── EDR/杀毒软件（按端点）     ~500-2000元/端点
  └── 备份系统                   ~3-15万

硬件设备：
  ├── 下一代防火墙（NGFW）       ~5-30万
  ├── 堡垒机硬件（如需）          ~3-10万
  └── 服务器（用于安全产品）      ~3-10万

安全服务：
  ├── 等保测评费（三级）          ~8-15万
  ├── 渗透测试服务               ~3-10万
  ├── 安全加固服务               ~5-15万
  └── 安全咨询（方案设计）        ~3-10万

总预算范围（中小型）：30-100万元
总预算范围（大型/关基）  ：100-500万元
```

### 2.2 常见预算陷阱

```
1. 只算产品不算服务
   → 买了设备但没人会用 → 白买
  
2. 测评费预算遗漏
   → 测评费用通常8-15万（三级），需单独预算

3. 到期续费遗漏
   → 安全产品授权/特征库更新需年度续费（约首次30-50%）

4. 二次测评费用
   → 整改不通过→复测费用（约首次50-70%）

5. 硬件服务器遗漏
   → 安全产品需要独立的服务器/虚拟机运行
```

## 三、供应商选型

### 3.1 供应商评估矩阵

```
供应商评分卡：

┌────────────┬──────┬──────┬──────┬──────┐
│ 评估维度     │ 权重 │ A公司 │ B公司 │ C公司 │
├────────────┼──────┼──────┼──────┼──────┤
│ 等保合规资质 │ 25%  │  8   │  9   │  7   │
│ 产品功能完整性│ 20%  │  8   │  8   │  9   │
│ 项目实施经验  │ 15%  │  7   │  9   │  6   │
│ 售后服务能力  │ 15%  │  6   │  8   │  7   │
│ 价格竞争力    │ 15%  │  8   │  6   │  9   │
│ 国产化适配    │ 10%  │  7   │  8   │  8   │
├────────────┼──────┼──────┼──────┼──────┤
│ 加权总分     │ 100% │ 7.45 │ 8.05 │ 7.55 │
└────────────┴──────┴──────┴──────┴──────┘
```

### 3.2 供应商常见套路

```
1. 低价竞标 → 实施时加价
   方案只说"部署堡垒机"，不说明是集群还是单机
   实施时说"单机不够，需要加购" → 加价
   → 方案必须明确：产品型号/数量/授权方式/维保年限

2. 功能演示 ≠ 实际产品
   演示用的是旗舰版/最新版 → 报价是标准版
   → 合同注明：交付产品版本号+功能清单

3. 技术能力不匹配
   销售说得天花乱坠 → 实施工程师不懂Linux
   → 要求配备项目经理+方案中明确驻场天数
```

### 4.2 常见延误原因

```
1. 设备到货延迟
   → 提前确认货期，合同约定到货时间+违约金

2. 网络变更窗口受限
   → 提前与业务方确认停机窗口
   → 分多次小变更而非一次性大变更

3. 操作系统版本太老
   → 安全产品Agent不支持Windows Server 2008等老旧系统
   → 提前确认产品兼容性清单

4. 第三方系统不配合
   → 日志采集中需要应用系统改造
   → 提前沟通确认接口规范

5. 沟通协调耗时
   → 安全部 ↔ 网络部 ↔ 系统部 ↔ 应用部
   → 项目启动会明确各部门接口人
```

## 五、验收标准

### 5.1 验收清单

```
等保整改项目验收Checklist：

□ 安全产品：
  □ 堡垒机：双因素登录正常、操作审计录像可回放
  □ 日志审计：30+条日志源接入、范式化正确
  □ 漏洞扫描：完成全量扫描、高危漏洞已修复
  □ 数据库审计：核心数据库访问日志正常采集
  □ WAF：策略生效、误拦截率<5%

□ 安全加固：
  □ Windows服务器：组策略+补丁+杀毒+审计
  □ Linux服务器：SSH加固+auditd开启+最小服务
  □ 网络设备：ACL策略+Telnet关闭+SNMPv3

□ 管理制度：
  □ 信息安全管理制度文件齐全（8大类）
  □ 日志有定期审查记录
  □ 备份有恢复测试记录

□ 培训与知识转移：
  □ 管理员培训完成
  □ 运维手册交付
  □ 应急联系方式上墙
```

## 六、测评配合

### 6.1 测评前准备清单

```
测评前1周必须完成：

□ 所有安全产品正常运行（无告警灯红）
□ 管理制度文件装订成册
□ 网络拓扑图更新（标注安全产品位置）
□ 人员到位（至少1名安全管理员+1名系统管理员）
□ 机房整洁（线缆规整/温湿度正常/消防器材在位）
□ 管理账户权限复核（三员分离）
□ 漏洞扫描0高危
```

### 6.2 测评当日配合要点

```
1. 专人陪同
   测评师每个操作都需要有人陪同（防误操作+及时解答）

2. 实时响应
   测评师问"这个服务器密码策略是什么"
   → 立即登录展示，而非只说"应该是..."

3. 记录问题
   测评过程中记录所有被扣分项
   → 便于测评后快速整改

4. 不要争论
   测评师说某处不符合 → 不要当场争论
   → 记录后私下沟通确认/提供证据
```
