## 一、云计算等保定级

```
GB/T 22239-2019 附录 C — 云计算安全扩展要求

云服务模式与责任共担：

IaaS (基础设施即服务): 云厂商负责物理/虚拟化安全
                     租户负责OS以上安全
PaaS (平台即服务): 云厂商负责OS/中间件安全
                   租户负责应用/数据安全
SaaS (软件即服务): 云厂商负责绝大部分安全
                   租户负责数据/用户管理安全

定级方法：
  云平台：单独定级（IaaS/PaaS平台通常≥三级）
  云租户系统：单独定级（利用云平台的安全能力作为基础）
```

## 二、云计算扩展要求核心条款

### 2.1 基础设施位置

```
"云计算基础设施应位于中国境内"

→ 选择云厂商：必须选择提供中国境内数据中心的厂商
  ✓ 阿里云（中国Region）
  ✓ 腾讯云（中国Region）
  ✓ 华为云（中国Region）
  ✓ AWS中国（北京/宁夏）
  ✓ Azure中国（世纪互联运营）
  ✗ 直接使用海外Region
```

### 2.2 虚拟化安全

```bash
# 关键检查项：

# 1. 虚拟机逃逸风险
#    同级租户→逃逸到宿主机→攻击其他租户
#    防御：云厂商侧虚拟化安全（租户无法控制）
#    租户可要求云厂商提供等保证明

# 2. 镜像安全
#    检查云市场镜像的来源和签名
#    自建镜像：安全加固后再制作模板

# 3. 快照安全
#    快照可能含敏感数据
#    快照加密存储、快照共享范围控制
```

### 2.3 多租户隔离

```
等保要求：
  "应保证不同租户间虚拟资源隔离"

检查要点：
  ✓ 不同租户的虚拟机不可直接通信（除非配置了安全组）
  ✓ 使用VPC网络隔离（不同租户不同VPC）
  ✓ 共享存储需访问控制（不同租户不可访问对方数据）
  ✓ 使用KMS加密+租户独享密钥

云平台验证方式：
  甲方：查看云平台的等保测评报告
  测评机构：抽样测试跨租户访问
```

### 2.4 数据迁移安全

```
等保要求：
  "应保证云租户数据的可迁移性，数据迁移过程应保证完整性"

实践方法：
  ✓ 数据库定时备份导出（mysqldump/pg_dump）
  ✓ 对象存储跨账号/跨云复制
  ✓ 应用容器化（K8s可跨云迁移）
  ✓ 避免厂商锁定（Vendor Lock-in）
     → 不使用云厂商专有数据库格式
     → 使用MySQL/PostgreSQL等开源标准数据库
```

## 三、云等保合规方案

### 3.1 阿里云等保三级方案

```
阿里云等保三级合规组件矩阵：

安全产品（阿里云原生产品）：
  ├── 云安全中心（安骑士 + 态势感知）— 主机安全+漏洞管理
  ├── 云防火墙 — 边界防护+IPS
  ├── Web应用防火墙(WAF) — Web防护
  ├── 堡垒机 — 运维审计
  ├── 日志服务(SLS) + ActionTrail — 日志审计
  ├── 数据库审计 — DB审计
  ├── 数据安全中心(DSC) — 数据识别+脱敏
  ├── 密钥管理服务(KMS) — 加密密钥管理
  └── 云监控 — 统一监控

阿里云已通过等保三级的服务（可直接复用）：
  ECS/RDS/SLB/VPC/OSS/Redis/MongoDB等
  → 可降低租户自建等保的物理机房要求
```

### 3.2 腾讯云等保三级方案

```
腾讯云安全产品矩阵：
  ├── 主机安全（云镜） — EDR+漏洞管理
  ├── 云防火墙 — 边界防护
  ├── Web应用防火墙 — Web防护
  ├── 堡垒机（BH） — 运维审计
  ├── 日志服务(CLS) + 操作审计 — 日志审计
  ├── 数据安全审计 — DB审计
  ├── 密钥管理系统(KMS) — 加密
  ├── 安全运营中心(SOC) — 统一安全运营
  └── 云监控 — 统一监控
```

### 3.3 混合云等保方案

```
自建IDC + 公有云 混合部署等保：

网络：
  IDC ↔ 云 VPC 通过 专线/VPN 互联
  → 等保测评范围：IDC侧 + 云侧 + 互联通道

安全产品：
  IDC侧：传统硬件安全产品
  云侧：云原生安全产品
  → 需统一安全管理平台

日志：
  IDC日志 + 云上日志 → 集中到云上SIEM/本地SIEM

测评：
  IDC + 云 统一纳入测评范围
  测评机构对IDC和云分别现场测评
```

## 四、云等保与云密评衔接

```
云等保 + 云密评 = 合规双保险

等保三级（云上）：
  → 要求使用密码技术
  → 不强制国密算法（TLS也可以）
  → 重点在"用了密码"

密评（密码应用安全性评估）：
  → 要求使用国密算法（SM2/SM3/SM4）
  → 重点在"密码用得对不对，够不够安全"
  → 对于关基/政务/等保三级以上系统强制要求

云上等保+密评实践：
  1. 先通过等保三级（使用TLS/国际密码）
  2. 再通过密评（升级为TLCP/SM4/SM3/SM2）
  3. 或一步到位：云等保+云密评同步完成

阿里云/腾讯云/华为云 已推出国密密评方案：
  - 云上国密SSL证书服务
  - 支持TLCP的负载均衡
  - 国密HSM（硬件密码机）
```
