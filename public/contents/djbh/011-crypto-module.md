# 等保 2.0 密码模块要求与产品选型指南

---

## 一、密码模块安全等级

```
GM/T 0028-2014 对标 FIPS 140-2:

安全一级(SL1):
  ✦ 基本密码算法实现 ✓
  ✦ 无物理防护
  → 软件密码模块

安全二级(SL2):
  ✦ + 防篡改封条/涂层
  ✦ + 基于角色的认证
  → 硬件模块基础版

安全三级(SL3):
  ✦ + 主动防篡改(检测到物理攻击→清零密钥)
  ✦ + 基于身份的认证
  → HSM/金融密码机

安全四级(SL4):
  ✦ + 环境故障保护(温度/电压异常→清零)
  ✦ + 穿透密封
  → 军事/最高密级
```

---

## 二、产品选型要点

| 场景 | 推荐产品 | 安全等级 |
|------|---------|---------|
| 政务Web | 国密SSL网关(TLCP) | SL2 |
| 金融支付 | 金融数据密码机(HSM) | SL3 |
| 电子签章 | 签名验签服务器 | SL2-3 |
| 密钥管理 | KMS/密码服务平台 | SL3 |
| 邮件加密 | S/MIME国密网关 | SL2 |

---

## 三、选型检查

```
□ 是否有国家密码管理局认证证书?
□ 证书是否在有效期内?
□ 支持的密码算法: SM2/SM3/SM4
□ 密钥管理: 支持安全生成/存储/轮换
□ 性能: 签名验签 TPS 满足需求?
□ 双机热备/集群支持?
□ API/SDK: 支持标准接口(PKCS#11/JCE)?
```
