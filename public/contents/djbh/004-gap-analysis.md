## 一、方法论

```
差距分析 = 逐项对照 GB/T 22239 控制点 → 评估当前状态 → 输出差距

检查方法:
  ① 文档审查 — 看管理制度文件是否齐全
  ② 现场检查 — 看机房/门禁/消防/温湿度
  ③ 配置核查 — 登录系统检查安全配置
  ④ 工具测试 — 漏洞扫描/基线核查/弱口令检测
  ⑤ 人员访谈 — 询问管理员实际操作流程

判定标准:
  ✓ 符合   — 完全满足
  △ 部分符合 — 部分满足（50-80%）
  ✗ 不符合  — 不满足（<50%）
  N/A — 不适用
```

## 三、报告模板

```markdown
# 等保三级差距分析报告

## 1. 项目概述
- 系统名称: XXX 核心业务系统
- 目标等级: 等保三级 (S3A3G3)
- 分析日期: 2026-06-15
- 分析人: 张三

## 2. 分析方法
- 文档审查: 管理制度文件 5 类
- 现场检查: 机房 + 办公区域
- 配置核查: 服务器/数据库/网络设备
- 工具测试: OpenVAS 漏洞扫描 + 基线核查
- 人员访谈: 3 人

## 3. 差距分析结果

### 统计汇总
┌──────────┬────┬──────┬──────┬─────┐
│          │ 符合│ 部分  │ 不符合│ N/A │
├──────────┼────┼──────┼──────┼─────┤
│ 物理环境  │ 3  │ 3    │ 2    │ 0   │
│ 通信网络  │ 0  │ 1    │ 2    │ 0   │
│ 区域边界  │ 1  │ 1    │ 3    │ 0   │
│ 计算环境  │ 0  │ 5    │ 5    │ 0   │
│ 管理中心  │ 0  │ 2    │ 2    │ 0   │

总体合规率: 7/37 = 18.9% → 严重不达标

## 4. 高风险项 (必须整改，否则不通过)
✗ 审计日志未开启
✗ 弱口令(root/123456, admin/admin123)
✗ 无防病毒软件
✗ 数据库明文存储敏感信息
✗ 未部署堡垒机
✗ 数据未加密传输(Telnet明文)
```
