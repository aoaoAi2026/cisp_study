# 分保与等保的区别与联系：涉密信息系统分级保护解析

## 1. 基本概念与适用对象

在国内信息安全合规体系中，最容易混淆的两个体系是：**分级保护（简称"分保"）** 和 **等级保护（简称"等保"）**。它们均由国家顶层制度要求推进，但法律依据、适用对象和主管单位完全不同。

| 项目 | 等级保护（等保） | 分级保护（分保） |
|------|-----------------|-----------------|
| 制度起源 | 《网络安全法》第二十一条 | 《保守国家秘密法》第二十九条 |
| 适用对象 | 通用信息系统 / 关键信息基础设施 / 云/物/工控等 | **处理、存储、传输国家秘密**的涉密信息系统 |
| 主管单位 | 公安部网安局（等保 110） | 国家保密行政管理部门及地方保密局 |
| 分级 | 一 ~ 五级（实际常用二/三级） | 内部（非密）、秘密、机密、绝密三级 |
| 主要标准 | GB/T 22239-2019 等 | BMB17、BMB20、BMB19、BMB22、BMB23 等 |
| 测评机构 | 等保测评机构（省级以上公安备案） | 国家保密行政管理部门授权的涉密信息系统测评机构 |
| 结论形式 | 等级测评报告（通过/基本符合/不符合） | 涉密信息系统测评结论（通过/不通过） |

## 2. 密级划分原则

### 2.1 国家秘密的三个密级

《保守国家秘密法》规定国家秘密的密级分为三级：

| 密级 | 定义 | 损害范围 | 保密期限 |
|------|------|---------|---------|
| 秘密 | 一般国家秘密 | 使国家的安全和利益遭受**损害** | ≤ 10 年 |
| 机密 | 重要国家秘密 | 使国家的安全和利益遭受**严重**损害 | ≤ 20 年 |
| 绝密 | 最重要国家秘密 | 使国家的安全和利益遭受**特别严重**损害 | ≤ 30 年 |

### 2.2 涉密信息系统定级原则

- **秘密级**：存储、处理、传输秘密级国家秘密的系统（对应 BMB17 的内部要求）
- **机密级**：存储、处理、传输机密级国家秘密的系统（对应 BMB20 基本要求）
- **绝密级**：存储、处理、传输绝密级国家秘密的系统（对应 BMB23 基本要求）

> 一个涉密信息系统**按最高密级**定级，不得降级处理。例如，系统只处理一条机密级信息，也须按机密级建设。

## 3. 分保核心标准体系

| 标准编号 | 标准名称 | 作用 |
|----------|----------|------|
| BMB 17-2006 | 涉及国家秘密的信息系统分级保护技术要求 | 技术要求总纲（安全物理环境、网络、主机、应用、数据等） |
| BMB 20-2007 | 涉及国家秘密的信息系统分级保护管理规范 | 管理要求（制度、人员、保密责任、审批流程） |
| BMB 19-2006 | 涉及国家秘密的信息系统测评指南 | 分保测评方法 |
| BMB 22-2007 | 涉及国家秘密的信息系统安全保密测评中心 | 测评机构规范 |
| BMB 23-2008 | 涉及国家秘密的信息系统分级保护方案设计指南 | 方案设计方法 |

## 4. 分保 vs 等保：关键差异对照

### 4.1 技术差异要点

```
                           等保（三级）                  分保（机密级）
物理隔离要求            逻辑隔离 + 边界防护                  物理隔离（禁接入互联网）
身份鉴别              双因素认证（口令+Token/生物）        双因素+审计/管理三员分立
访问控制              基于角色访问控制（RBAC）             强制访问控制（MAC）
审计日志保存          ≥ 6 个月                              ≥ 6 个月 + 审计不可篡改
网络接入              允许外网访问（受保护）                严禁直接接入公网，必须物理断开
数据外带              DLP / 加密 U 盘                       信息设备外带必须审批+登记
介质管理              登记管理                               统一编号、统一标识、统一配发、统一回收
运维                  普通运维 + 堡垒机审计                  涉密人员资质 + 全程录像审计
边界防护              NGFW/IPS                              安全保密产品需国密+保密局认可
密码应用              商密 SM2/SM3/SM4                     SM1 算法（受控分发）+ SM2/SM4
```

### 4.2 管理差异要点

| 要点 | 等保 | 分保 |
|------|------|------|
| 责任主体 | 等保三级单位法人为第一责任人 | **定密责任人 + 保密委** |
| 人员管理 | 安全培训 + 背景审查（三级） | **涉密人员三级审批 + 脱密期管理** |
| 三员（审计/系统/安全管理） | 要求角色分离 | **强制三员分立，职责互斥** |
| 外包/外聘 | 签署保密协议即可 | 外包人员不得接触涉密信息 |
| 系统上线审批 | 内部审批 + 安全测试报告 | **保密行政管理部门审批方可启用** |
| 废弃设备 | 常规销毁 | **由专用设备粉碎 / 消磁 / 登记** |

## 5. "等保 + 分保"并跑场景

现实中，部分单位（如政府机关、科研院所、大型央企）同时运营两套系统：一套互联网服务系统（等保），一套内部涉密系统（分保）。在这种场景下必须注意以下原则：

### 5.1 严格物理隔离

涉密系统与非涉密系统之间**必须物理隔离**，禁止在同一台服务器、同一台交换机、甚至同一机房物理位置混布。使用虚拟机或 VLAN 等方式"伪隔离"属于严重高风险。

### 5.2 跨系统数据交换

非涉密系统 → 涉密系统：必须通过**单向导入设备（光闸）**实现单向数据导入，并进行病毒查杀和格式清洗。

涉密系统 → 非涉密系统：必须通过**涉密信息单向导出控制设备**，并由单位保密委员会审批，经脱密处理后方可导出。

```
非涉密数据        →  单向导入设备（光闸/导入盘）  →  涉密系统
涉密系统         →  审批流程 + 脱密/脱敏          →  非涉密环境（严禁直接反向操作）
```

### 5.3 合规文件与流程清单

1. 《定密责任书》《保密责任书》（全员签署）
2. 《涉密信息系统分级保护方案》（按 BMB23 编写）
3. 《涉密信息系统使用人员登记表》
4. 《涉密载体（纸质/电子）台账》
5. 《保密教育培训记录》（年度 ≥ 15 学时）
6. 《涉密设备外带审批单》
7. 《涉密系统审计日志保存与查看记录》
8. 《定密与解密审批表》

## 6. 常见违法违规情形

根据《保守国家秘密法》第五章及相关处分条例，以下情形直接导致问责：

1. 涉密计算机连接互联网
2. 未按规定配备保密管理机构/人员
3. 使用非涉密移动存储介质在涉密计算机与互联网计算机之间交叉使用
4. 未按规定对涉密人员进行审查、教育、脱密期管理
5. 未经审批擅自让外部人员接触涉密信息
6. 涉密系统未通过分保测评即投入使用

## 7. 实施建议

```
Step 1  梳理信息资产，明确哪些系统处理、存储、传输国家秘密
Step 2  定密责任人牵头定密，形成《国家秘密事项一览表》
Step 3  按 BMB17/20/23 设计涉密系统技术方案
Step 4  完成物理隔离、三员、审计、密码、介质等关键建设
Step 5  引入经保密行政管理部门授权的测评机构开展分保测评
Step 6  通过后上线，每年定期复测，重大变更重新评估
```

> **记忆口诀**：等保管"一般"，分保管"秘密"；等保守国门，分保守核心；两套制度并行不冲突，但绝不能相互替代。处理国家秘密的系统 **必须** 同时满足分保，不可以用等保替代。
