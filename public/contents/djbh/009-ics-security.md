# 工业控制系统安全（等保 2.0 工控扩展要求）实战指南

## 1. 工业控制系统（ICS）概述

工业控制系统（Industrial Control System, ICS）是对工业生产过程中使用的各类控制与监测系统的统称，常见子类包括：

| 缩写 | 英文全称 | 典型场景 |
|------|---------|----------|
| SCADA | Supervisory Control And Data Acquisition | 电力、供水、油气、铁路等广域远程监控 |
| DCS | Distributed Control System | 石化、化工、核电、大型工业流程 |
| PLC | Programmable Logic Controller | 各类自动化生产车间、流水线 |
| RTU | Remote Terminal Unit | 远程现场终端，采集/控制 |
| SIS | Safety Instrumented System | 安全仪表系统（紧急停车、保护联锁） |
| MES | Manufacturing Execution System | 制造执行系统（生产调度、质量、追溯） |
| HMI | Human Machine Interface | 人机界面、操作员站 |

等保 2.0 将工控系统视为关键信息基础设施的核心组成，GB/T 22239-2019 **附录 D 工业控制系统扩展要求** 专门列出工控特有的安全控制点。

## 2. 等保 2.0 工控扩展要求要点

### 2.1 五大技术控制点

| 控制点 | 扩展要求摘要 | 说明 |
|--------|-------------|------|
| 安全物理环境（S1） | 工业控制环境需物理位置独立，具备电磁防护、防干扰、防破坏 | 与 IT 机房物理隔离 |
| 安全通信网络（S2） | 工业协议白名单、流量审计，禁止工业协议直接上公网 | Modbus / S7 / DNP3 等 |
| 安全区域边界（S3） | 工控系统与信息网络之间须部署**工业防火墙**；单向导入设备 | 禁止 TCP/IP 双向穿透 |
| 安全计算环境（S4） | 设备访问控制、操作审计、控制指令安全、禁止随意升级 | PLC/RTU 固件安全 |
| 安全管理中心（S5） | 工控安全监测平台、统一日志审计、漏洞与补丁统一管理 | |

### 2.2 典型工控纵深防御架构

```
互联网 / 外网
      │
  企业防火墙（IT 侧）
      │
  办公 / 管理网络（IT 域）
      │
  工控边界防火墙 + 工控网闸（仅允许已知工业协议）
      │
  生产控制网络（OT 域）
      │
  操作站 / HMI / SCADA / 安全工程师站
      │
  控制器域（PLC / DCS / SIS / 传感器）
```

关键原则：**IT/OT 分层隔离 + 最小协议白名单 + 单向导入**

## 3. 典型工控安全风险 Top 10

1. **工控系统直连公网**（通过 Shodan 可扫描到数万个暴露的 Modbus/S7 端口）
2. **使用 Windows XP/2003/7 等已停服操作系统**（无法打补丁）
3. **工业协议无鉴权**（Modbus TCP 默认无口令，指令即可写入）
4. **使用默认口令 / 弱口令**（Siemens、Rockwell、Schneider 等厂商默认密码公开）
5. **无补丁管理流程**（PLC/DCS 固件长期不升级）
6. **无防病毒 / EDR**（误杀工业软件，不敢部署）
7. **USB 随意使用**（U盘病毒在 HMI 间横向）
8. **远程维护未受控**（TeamViewer / 向日葵随意开放）
9. **安全事件无审计**（PLC 启停、工艺参数修改无日志）
10. **IT/OT 未分离**（同网段、同交换机、同防火墙）

## 4. 工控安全落地清单（三级系统）

### 4.1 网络与边界

```bash
# 工业防火墙（推荐）
#   - 工业协议深度解析：Modbus/TCP、S7、EtherNet/IP、DNP3、OPC UA
#   - 白名单模式：仅允许已知主站 → 从站读写指令
#   - 支持 Modbus 功能码、寄存器地址级细粒度控制

# 网络架构改造建议
生产控制网（OT）  ──┐
                     │  工业防火墙 + 网闸
  IT 办公网        ──┘
  DMZ / 对外服务  ── 企业防火墙 ── 互联网
```

### 4.2 设备与终端

| 措施 | 说明 |
|------|------|
| 主机白名单 | 使用应用白名单机制（软件限制策略 / AppLocker / 专用工业 EDR），替代传统杀毒 |
| 禁用 USB | 物理关闭或通过策略禁用 U 盘，必要时使用合规加密 U 盘 + 审批记录 |
| 禁用远程维护软件 | 远程维护必须通过堡垒机 + 审批 + 录像 |
| 设备版本锁定 | 不得随意升级操作系统 / 驱动，升级前须在测试环境验证 |
| 基线安全配置 | CIS Benchmark 工业版 + 关闭不需要的服务、端口、共享 |

### 4.3 账号与审计

```
三员角色模型（工控场景）：
  系统管理员  ── 操作站与 PLC 系统维护
  安全管理员  ── 安全策略、白名单、边界配置
  审计管理员  ── 独立审计，对前两类操作进行独立核查
```

工控审计要求（与 IT 侧不同的特殊点）：

- 操作员登录/退出、启停控制操作、参数修改、程序下载上传均须留痕
- 审计日志应 ≥ 6 个月，不可被操作人员修改
- 应集中采集 SCADA/HMI/PLC/工业防火墙的日志

## 5. 工控渗透测试与等保测评注意事项

工控环境与 IT 环境有本质区别：

| 项目 | IT 环境 | 工控环境 |
|------|---------|---------|
| 漏洞扫描 | 常规 Nmap + Nessus 扫描 | **只允许授权扫描**，禁止爆破，避免影响生产 |
| 渗透测试 | 可以 DoS 测试（临时环境） | **禁止 DoS / 溢出测试**，可能造成实际停机 |
| 工具选型 | Nmap、SQLMap、Burp | Modbus-Scan、S7-Scan、Wireshark 工业协议分析、Shodan（仅侦查） |
| 审批流程 | 书面授权 | **必须由生产/安全部门双审批**，在停产窗口或测试环境执行 |

## 6. 典型工具链

```bash
# 工控协议扫描与识别
nmap --script broadcast-s7-info.nse -sU -p 102 192.168.0.0/24    # Siemens S7
nmap --script modbus-discover.nse -p 502 192.168.0.0/24          # Modbus TCP

# 工业协议分析（Wireshark 过滤器）
modbus || s7comm || enip || dnps || opcua

# Shodan 搜索暴露的工控设备（仅用于教育与授权测试，严禁未授权访问）
#   port:502  product:modbus
#   port:102  product:siemens
#   port:44818 product:allen-bradley
```

## 7. 合规交付物清单（等保 2.0 工控扩展）

1. 《工业控制系统资产清单》（含 PLC/DCS/SCADA/HMI 型号、版本、网络位置）
2. 《工业控制系统网络拓扑图》（含边界、分区、安全设备）
3. 《工业协议白名单》（协议、端口、功能码、主从站）
4. 《工控安全管理制度与操作规范》
5. 《工控安全检测与审计报告》
6. 《工控漏洞与补丁管理记录》
7. 《IT/OT 边界审计记录》
8. 《应急预案与工业控制安全演练记录》

> 工控安全的核心原则：**先可用性，后保密性**。与 IT 系统"保密性优先"的传统认知正好相反——在工控场景下，停机事故的损失远大于数据泄露。任何等保 2.0 工控整改方案，必须在确保生产可用性的前提下，循序渐进地提升安全性。
