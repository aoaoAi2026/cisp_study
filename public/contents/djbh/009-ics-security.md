# 等保2.0 移动互联安全扩展要求

---

## 一、适用范围

```
移动互联扩展要求适用于:
  ✓ 企业APP + 后端API
  ✓ 微信公众号/小程序
  ✓ 移动OA/移动审批
  ✓ 移动设备管理系统(MDM)

不适用于:
  ✗ 纯短信/电话服务
```

---

## 二、核心控制点

### 1. 无线接入安全

```
要求: 无线接入需认证+加密

实现:
  ✓ 企业WiFi → WPA2-Enterprise + 802.1X
  ✓ APP通信 → HTTPS(HSTS) + 证书绑定
  ✓ 移动设备准入 → MDM + 证书
```

### 2. 移动终端管控

```
要求:
  ✦ 终端设备需注册+绑定
  ✦ 遗失/被盗后可远程擦除
  ✦ 应用安装白名单
  ✦ 禁止越狱/Root设备接入

实现方案:
  □ 部署MDM(Microsoft Intune/AirWatch/自研)
  □ 设备合规性检测(越狱/Root/OS版本)
  □ 远程锁定+擦除功能
```

### 3. 移动应用安全

```
要求:
  ✦ APP需签名发布
  ✦ 代码混淆+加固
  ✦ 本地数据加密存储
  ✦ 防止逆向/重打包

实现:
  □ 应用加固(360/梆梆/爱加密)
  □ SharedPreferences/Keychain加密
  □ 证书固定(Certificate Pinning)
  □ 反调试/反Hook检测
```

### 4. 数据安全

```
要求:
  ✦ 传输加密(TLS 1.2+)
  ✦ 本地敏感数据加密
  ✦ 禁止缓存敏感数据
  ✦ 离线数据保护

实现:
  □ HTTPS + HSTS
  □ AES加密本地数据库
  □ 内存中不保存密码明文
```
