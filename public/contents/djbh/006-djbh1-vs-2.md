# 等保 1.0 vs 等保 2.0 核心差异对比

---

## 一、法律依据变化

| 等保1.0 | 等保2.0 |
|---------|---------|
| 《信息安全等级保护管理办法》(2007) | 《网络安全法》(2017第21条) |
| 部门规章 | **法律** — 等保成为法定义务 |

---

## 二、标准体系

| 维度 | 等保1.0 | 等保2.0 |
|------|---------|---------|
| 核心标准 | GB/T 22239-2008 | **GB/T 22239-2019** |
| 标准数量 | 约30个 | 约50+个 |
| 名称变化 | 信息系统安全等级保护 | **网络安全**等级保护 |
| 结构变化 | 按信息系统定级 | 按**等级保护对象**定级 |

---

## 三、保护对象扩展

```
等保1.0: 信息系统
   ↓ 扩展
等保2.0: 
  ✓ 基础信息网络
  ✓ 云计算平台
  ✓ 大数据平台
  ✓ 物联网(IoT)
  ✓ 工业控制系统(ICS)
  ✓ 移动互联系统
```

---

## 四、标准结构对比

```
等保1.0 (GB/T 22239-2008):
  S类(业务信息安全) + A类(系统服务安全) + G类(通用)
  控制点: 约300个

等保2.0 (GB/T 22239-2019):
  安全通用要求 + 安全扩展要求
  ✓ 云计算扩展 (附录C)
  ✓ 移动互联扩展 (附录D)  
  ✓ 物联网扩展 (附录E)
  ✓ 工业控制扩展 (附录F)
  控制点: 211个(三级通用) + 扩展

结构更清晰: 一个要求 → 四级对照
  第一级 | 第二级 | 第三级 | 第四级
  (一目了然各级别差异)
```

---

## 五、新增要求

| 新增项 | 说明 |
|--------|------|
| **可信验证** | 基于可信根验证(TPM/TCM) |
| **安全管理中心** | 等保2.0核心要求, 集中管控 |
| **个人信息保护** | 对接《个人信息保护法》 |
| **密码技术** | 强调采用密码技术保护 |
| **云计算/移动/物联/工控** | 4个扩展要求 |
| **攻击回溯** | 能追踪攻击源 |

---

## 六、测评变化

| 维度 | 等保1.0 | 等保2.0 |
|------|---------|---------|
| 测评方法 | 访谈/检查/测试 | 不变, 但更强调工具检测 |
| 高风险项 | 无明确定义 | **清晰的高风险项清单** |
| 一票否决 | 无 | **高风险项不通过=测评不通过** |
| 测评频率 | 三级≥1次/年 | 三级≥1次/年(不变) |

---

## 七、企业升级指南

```
等保1.0 → 等保2.0 升级清单:

□ 重新定级(考虑新扩展要求)
□ 补齐安全管理中心(堡垒机+日志审计+漏洞扫描+集中管控)
□ 密码技术应用(原1.0无强制要求 → 2.0强调)
□ 可信验证(如有条件)
□ 云计算/工控/移动扩展(如适用)
□ 管理制度更新(增加个人信息保护等)
□ 重新测评(按2019版标准)
```
