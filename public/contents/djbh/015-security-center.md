# 等保安全管理中心建设指南

---

## 一、等保2.0 "安全管理中心" 要求

```
GB/T 22239-2019 8.1.5 安全管理中心（三级要求）：

8.1.5.1 系统管理
  → 划分三员(系统管理员/安全管理员/审计管理员)
  → 系统管理员通过堡垒机运维

8.1.5.2 审计管理
  → 审计记录集中存储、保护
  → 日志留存≥6个月
  → 审计记录不可删除/不可篡改

8.1.5.3 安全管理
  → 安全策略集中管理
  → 安全事件告警统一展示

8.1.5.4 集中管控
  → 划分管理区域
  → 运维操作通过安全通道（堡垒机）
  → 统一监控（服务器/网络/安全设备）
  → 安全事件集中分析与响应
```

## 二、集中管控产品矩阵

### 2.1 "4+1+N" 典型方案

```
等保三级"4+1+N" 安全管理中心方案：

4个必备安全产品：
  ① 堡垒机 (运维审计)
     → 齐治/安恒明御/Teleport/麒麟
     → 功能：双因素认证 + 操作审计 + 命令过滤
  ② 日志审计系统
     → 日志易/Splunk/ELK/Wazuh
     → 功能：全量日志采集+关联分析+合规报表
  ③ 漏洞扫描系统
     → 绿盟RSAS/安恒明鉴/Nessus/OpenVAS
     → 功能：定期扫描+漏洞管理+修复跟踪
  ④ 数据库审计系统
     → 安恒DB Audit/启明星辰/Imperva
     → 功能：SQL审计+敏感数据访问告警

1个安全管理平台：
  → 态势感知/SIEM/SOC平台
  → 奇安信天眼/深信服SIP/安恒AiLPHA

N个其他安全产品（按需）：
  → WAF/IPS/EDR/沙箱/蜜罐/流量分析/终端管控
```

## 三、堡垒机选型与部署

### 3.1 等保三级堡垒机要求

```
堡垒机必须实现：
  ✓ 双因素认证（密码+OTP/手机验证）
  ✓ 协议代理（SSH/RDP/VNC/数据库协议）
  ✓ 操作审计录像（全量录像+可回放检索）
  ✓ 高危命令阻断（rm -rf /、DROP TABLE等）
  ✓ 三员分立（系统管理员/审计员/安全管理员使用不同账户）
  ✓ 审计日志不可删除不可修改
  ✓ 自动改密（特权账号密码定期轮换）

生产环境配置要点：
  - 堡垒机管理IP接入管理VLAN
  - 被管资产IP可达（路由+防火墙策略）
  - 堡垒机自身纳入监控
  - 会话录像存储≥6个月
  - 高可用部署（主备/集群）
```

### 3.2 免费/开源方案

```bash
# Teleport (开源, Go语言)
# 支持SSH/K8s/数据库/Web应用代理
curl https://goteleport.com/static/install.sh | bash -s 16.0.0
teleport start --roles=auth,proxy,node

# Apache Guacamole (支持RDP/SSH/VNC/Telnet, 纯Web访问)
docker run -d \
  -p 8080:8080 \
  -v /data/guacamole:/config \
  oznu/guacamole

# JumpServer (国产开源堡垒机)
# https://github.com/jumpserver/jumpserver
# 支持SSH/RDP/VNC/数据库/Web Terminal
# 内置审计录像+命令过滤
```

## 四、日志审计系统

### 4.1 日志采集要求

```
等保三级日志要求：

日志源覆盖：
  必采：
  ✓ 服务器操作系统日志（Windows Event Log / Linux syslog）
  ✓ 网络设备日志（防火墙/路由器/交换机）
  ✓ 安全设备日志（IDS/IPS/WAF/防病毒）
  ✓ 数据库日志（MySQL/PostgreSQL/Oracle审计日志）
  ✓ 应用日志（Web服务器/中间件）
  ✓ 堡垒机操作日志

日志留存：
  ≥6个月（热数据30天/归档≥6个月）
  
日志保护：
  ✓ 日志不可删除（普通用户无删除权限）
  ✓ 日志不可篡改（附加完整性校验/数字签名）
  ✓ 日志进程保护（杀不死/守护进程监控）
```

### 4.2 Syslog 集中配置

```bash
# Linux → Rsyslog 发送到日志服务器
# /etc/rsyslog.d/50-remote.conf
*.* @192.168.100.10:514   # UDP
# 或 TCP（推荐）
*.* @@192.168.100.10:514  # TCP
# 生产环境推荐 TLS 加密
$DefaultNetstreamDriver gtls
$ActionSendStreamDriverMode 1
```

```xml
<!-- Windows → Winlogbeat 发送到 ELK -->
<!-- winlogbeat.yml -->
winlogbeat.event_logs:
  - name: Security
  - name: System
  - name: Application
output.elasticsearch:
  hosts: ["https://192.168.100.10:9200"]
```

### 4.3 日志范式化

```
不同设备日志 → 统一格式（字段映射）：

源IP统一: src_ip → source.ip
目标IP统一: dst_ip → destination.ip  
操作用户统一: user → user.name
事件类型统一: 登录成功→authentication.success
```

## 五、安全策略一体化

### 5.1 统一管控平台要求

```
属于等保2.0新增，测评重点：

要求：
  ✓ 安全策略（防火墙规则/ACL/WAF规则）集中配置和下发
  ✓ 安全告警统一展示（所有安全设备的告警汇集一处）
  ✓ 安全事件工单流转（告警→工单→处理→闭环）
  ✓ 统一大屏展示（资产/漏洞/威胁/事件态势）

实现方案：
  小型：ELK + 手工配置
  中型：Wazuh + TheHive 工单
  大型：商业SOC平台（奇安信天眼/深信服SIP/安恒AiLPHA）
```
