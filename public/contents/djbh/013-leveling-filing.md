## 一、定级流程全景

```
等保定级五步法：

  自主定级 → 专家评审 → 主管部门核准 → 公安机关备案 → 备案通过

法律依据：
  GB/T 22240-2020《信息安全技术 网络安全等级保护定级指南》
  《网络安全等级保护条例》(征求意见稿)
```

## 二、定级要素

### 2.1 两个定级要素

```
定级 = f(受侵害客体, 受侵害程度)

受侵害客体：
  ① 公民、法人和其他组织的合法权益
  ② 社会秩序、公共利益
  ③ 国家安全

受侵害程度：
  一般损害 / 严重损害 / 特别严重损害

等级判定矩阵：
┌───────────────┬──────────┬──────────┬──────────┐
│ 受侵害客体     │ 一般损害  │ 严重损害  │ 特别严重  │
├───────────────┼──────────┼──────────┼──────────┤
│ 个人/组织权益  │ 第一级    │ 第二级    │ 第三级    │
│ 社会秩序/公益  │ 第二级    │ 第三级    │ 第四级    │
│ 国家安全      │ 第三级    │ 第四级    │ 第五级    │
└───────────────┴──────────┴──────────┴──────────┘
```

### 2.2 业务信息安全 vs 系统服务安全

```
定级需区分两个维度，取较高者：

业务信息安全等级 (S)：
  信息被破坏后对客体的侵害程度
  例：数据泄露 → 影响用户隐私 → S=3

系统服务安全等级 (A)：
  系统服务中断后对客体的侵害程度
  例：支付系统中断 → 无法交易 → A=3

最终等级 = max(S, A)
  若 S=3, A=2 → 第三级
  若 S=2, A=4 → 第四级
```

## 三、定级报告标准模板

### 3.1 报告结构

```markdown
# 信息系统安全等级保护定级报告

## 一、信息系统基本情况
- 系统名称：XX公司核心业务系统
- 承载业务：在线交易处理
- 责任单位：XX公司信息技术部
- 系统概述：(200字描述系统功能、用户规模、数据量)

## 二、定级依据
- 《网络安全法》第二十一条
- GB/T 22240-2020 信息安全技术 网络安全等级保护定级指南

## 三、业务信息安全等级确定

| 业务信息类别 | 描述 | 受侵害客体 | 受侵害程度 | 定级 |
|-------------|------|-----------|-----------|------|
| 用户个人信息 | 百万级用户姓名/手机/地址 | 公民合法权益 | 严重损害 | 3级 |
| 交易数据 | 日均万笔交易明细 | 社会秩序 | 严重损害 | 3级 |

业务信息安全等级：S = 3

## 四、系统服务安全等级确定

| 服务类型 | 描述 | 中断影响 | 受侵害客体 | 受侵害程度 | 定级 |
|---------|------|---------|-----------|-----------|------|
| 在线交易 | 24h不间断 | 服务中断超2h/日 | 社会公共利益 | 严重损害 | 3级 |

系统服务安全等级：A = 3

## 五、安全保护等级确定
根据 S=3, A=3 → 安全保护等级为第三级 (S3A3G3)

## 六、相关方签字
定级责任人：    日期：
单位负责人：    日期：
```

### 3.2 定级报告常见被退回原因

```
Top 5 被退回原因：

1. 受侵害客体分析不充分
   ✗ "系统被破坏可能影响国家安全" — 定性过高但无论证
   ✓ 需提供被监管机构认定的证明（如关基认定文件）

2. 业务描述过于笼统
   ✗ "本系统为公司核心系统"
   ✓ 应具体：承载什么业务？多少用户？什么数据？

3. 定级与描述不一致
   前面说"影响公民个人权益"，后面定"四级"
   → 逻辑矛盾

4. 缺少安全保护等级评定过程
   直接给出结论，没有 S/A 分维度分析

5. 格式不规范
   缺少公章/签字/日期
```

## 四、专家评审

### 4.1 评审会组织要点

```
需要组织专家评审的系统：
  ✓ 第三级以上系统
  ✓ 定级为第二级但涉及重要数据的系统（部分省市要求）

专家要求：
  - ≥3 名专家（通常5人，来自测评机构/高校/行业）
  - 至少1名高级职称
  - 专家不得来自系统开发/运维方（利益冲突回避）

评审会材料清单：
  1. 定级报告（草案）
  2. 系统拓扑图
  3. 业务流程图
  4. 数据流图
  5. 系统功能说明文档
  6. 安全需求分析

评审要点：
  ✓ 定级是否准确（S/A等级是否合理）
  ✓ 是否遗漏受侵害客体
  ✓ 受侵害程度分析是否充分
  ✓ 是否满足定级要素要求
```

### 4.2 评审意见模板

```
专家评审意见书（参考格式）：

经专家组对《XX系统安全保护等级定级报告》进行评审，形成意见如下：

1. 定级报告格式规范，内容完整，符合GB/T 22240-2020要求。
2. 系统业务信息安全等级确定为 X 级，系统服务安全等级确定为 Y 级，
   依据充分，定级结果可信。
3. 建议（整改意见，如有）

评审结论：☑ 通过  □ 修改后通过  □ 不通过

专家签字：         日期：
```

## 五、公安机关备案

### 5.1 备案流程

```
第三级以上系统备案流程：

  ① 完成自主定级 + 专家评审
  ② 准备备案材料
  ③ 运营单位所在地 → 市级公安机关网安部门
  ④ 公安机关审核（5-10个工作日）
  ⑤ 审核通过 → 发放备案证明
  ⑥ 每年12月31日前完成年度自查报告

备案材料清单：
  1. 信息系统安全等级保护备案表（共4页16项）
  2. 定级报告（含专家评审意见）
  3. 系统拓扑结构及说明
  4. 安全设施清单
  5. 网络安全管理制度清单
  6. 营业执照复印件（首次备案）
  7. 等保证书（测评后补交）
```

### 5.2 备案表核心填写要点

```
《信息系统安全等级保护备案表》关键栏目：

第三页：信息系统基本情况
  - "安全保护等级" — 与定级报告一致
  - "业务信息安全等级" — S值
  - "系统服务安全等级" — A值
  - "承载业务情况" — 不超过200字
  - "服务范围" — 全国/省内/市内/单位内部

第二页：安全责任单位
  - 安全责任人 = 法人代表或分管领导（必须是真实姓名）
  - 24小时应急电话（必须是能打通的值班电话）

第一页：备案单位信息
  - 统一社会信用代码
  - 单位类型（党政机关/事业单位/企业/其他）
```

## 六、备案常见问题与对策

```
Q: 备案后系统变更有哪些需要重新备案？
A: 以下情况需重新定级备案：
   - 安全保护等级变更
   - 系统功能/数据发生重大变化（影响定级要素）
   - 系统拆分/合并
   - 系统迁移到云计算环境（可能改变责任主体）

Q: 二级系统需要备案吗？
A: 二级及以下系统——自主保护，向公安机关告知性备案
   三级及以上——强制性备案
   具体要求以当地公安机关为准

Q: 备案被退回了怎么办？
A: 分析退回原因 → 修改材料 → 重新提交
   常见原因：材料不齐全/格式不规范/定级不合理
   
Q: 已备案但测评还没做，合规吗？
A: 备案后需尽快完成测评。
   部分省市公安机关会定期抽查未测评的备案系统。
```
