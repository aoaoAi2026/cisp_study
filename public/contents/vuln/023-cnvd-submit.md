## 2. CNVD 提交流程

```
发现漏洞 → 验证漏洞 → 编写报告 → 加密邮件提交 → CNVD审核 → 编号分配 → 通知厂商 → 公开
										├─ 验证通过 → 漏洞编号
├─ 验证不通过 → 退回
```

### 2.1 提交要求

```
1. 漏洞必须未公开（首次发现）
2. 附带完整POC/复现步骤
3. 影响范围明确（版本/产品名）
4. 漏洞类型明确（SQL注入/RCE/XSS等）
5. 使用加密邮件发送：vulreport@cnvd.org.cn
```

## 三、漏洞描述
ThinkPHP 在解析URL时对控制器名称过滤不严，攻击者可通过构造特殊URL调用任意类和方法，导致远程代码执行。

### 环境要求
- PHP 7.0+
- ThinkPHP 5.0.23
- http://target.com/

### Payload
GET /index.php?s=index/think\app/invokefunction&function=call_user_func_array&vars[0]=system&vars[1][]=whoami HTTP/1.1
Host: target.com

### 具体步骤
1. 部署 ThinkPHP 5.0.23 默认安装
2. 访问上述URL
3. 页面返回 `whoami` 命令执行结果
4. 确认远程代码执行

## 五、漏洞分析

### 代码分析
漏洞位于 library/think/App.php 第147行：

// 漏洞代码
public static function run() {
	$dispatch = self::routeCheck($request);
	// $dispatch中的controller来自用户输入
	// 未经过滤传递给exec()
}

### 修复建议
1. 升级到 ThinkPHP 5.0.24+
2. 对controller名称进行白名单限制
3. 禁用反斜杠路由解析（临时缓解）

## 六、修复方案
```php
// App.php 增加白名单过滤
$allowed_controllers = ['index', 'user', 'admin'];
if (!in_array($dispatch['controller'], $allowed_controllers)) {
	throw new HttpException(404);
}
```

## 4. 漏洞等级评定

### CNVD 评级标准

| 等级 | 条件 |
|------|------|
| **超危** | 远程利用+无需认证+直接系统权限 |
| **高危** | 远程利用+需简单认证+系统命令执行 |
| **中危** | 需认证+信息泄露+非核心功能 |
| **低危** | 需特定条件+难以利用+影响小 |

### 常见漏洞评级速查

| 漏洞类型 | 无需认证 | 需要认证 |
|---------|---------|---------|
| 命令执行(RCE) | 超危 | 高危 |
| SQL注入 | 高危 | 中危 |
| 文件上传(WebShell) | 高危 | 中危 |
| SSRF | 高危 | 中危 |
| XSS | 中危 | 低危 |
| 信息泄露 | 中危 | 低危 |
| CSRF | 低危 | 低危 |

## 5. 护网中漏洞的处理

护网期间发现的漏洞：

```
立即处置：
1. 如果是自己的漏洞 → 紧急修复（不打CNVD防止红队利用）
2. 如果是红队利用的0day → 紧急加固 + 护网结束后再提交
3. 发现的公共漏洞 → 立即修复 + 正常流程提交
```

### 知识巧记口诀

> **CNVD提交口诀**：
> 发现漏洞先验证，影响版本要精确；
> 复现步骤写清楚，PoC截图做证明；
> 修复建议不能少，提交之后跟进度；
> 厂商修复再公开，道德底线要坚守。

> **平台对比记忆**：CNVD(国家级/共享)、CNNVD(国家级/收录)、CVE(国际级/编号)、SRC(厂商级/奖励)。

### 考试陷阱提醒

| 陷阱 | 正确认知 |
|------|----------|
| ❌ 发现漏洞直接公开披露 | ✅ 应先通知厂商修复，遵循负责任的披露原则，否则可能承担法律责任 |
| ❌ 提交CNVD和CVE是一样的 | ✅ CNVD面向中国、中文提交；CVE面向国际、英文提交；两套编号体系独立 |
| ❌ 漏洞提交后不用跟进 | ✅ 需跟踪审核进度、配合补充信息、确认修复后配合公开披露 |

> 💡 **一句话总结**：CNVD漏洞提交是安全从业者的重要能力——既体现技术水平又涉及职业道德，CISP考试考查漏洞管理全生命周期和负责任披露原则。
