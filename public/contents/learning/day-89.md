## 一、网络安全法核心要点

### 1.1 颁布与实施

| 项目 | 时间 |
|------|------|
| 通过 | 2016年11月7日 |
| 施行 | 2017年6月1日 |

### 1.2 核心制度（必考！）

| 制度 | 核心要求 | 处罚 |
|------|---------|------|
| **网络安全等级保护** | 网络运营者必须落实等保要求 | 最高100万罚款 |
| **实名制** | 网络接入、信息发布、即时通讯需实名 | 可暂停服务 |
| **关键信息基础设施CII保护** | 重要行业CII数据境内存储；出境需安全评估 | 高额罚款 |
| **个人信息保护** | 禁止非法收集和出售 | 10倍违法所得罚款 |
| **安全监测与应急** | 制定应急预案，定期演练 | — |

### 1.3 关键信息基础设施CII

**涉及行业**：金融、能源、交通、水利、公共服务、电子政务等

**特殊要求**：
- 安全审查制度
- 数据和运营信息境内存储
- 向境外提供数据需安全评估

### 1.4 网络运营者义务一览

1. 制定内部安全管理制度
2. 确定网络安全负责人
3. 采取防范计算机病毒和网络攻击的技术措施
4. 监测、记录网络运行状态和网络安全事件
5. 留存网络日志不少于**6个月**
6. 数据分类、备份和加密

## 二、等级保护 2.0

### 2.1 等保1.0 vs 2.0

| 对比 | 等保1.0 | 等保2.0 |
|------|---------|---------|
| 颁布时间 | 2007年 | 2019年 |
| 保护对象 | 信息系统 | 网络（含云/物/移/工控/大数据） |
| 标准结构 | GB/T 22239-2008 | GB/T 22239-2019 |
| 安全要求 | 技术要求+管理要求 | 安全通用要求+安全扩展要求 |

### 2.2 等级划分（高频！）

| 等级 | 描述 | 定级依据 | 保护强度要求 |
|------|------|---------|-------------|
| **第一级** | 自主保护级 | 对公民法人造成一般损害 | 用户自主保护 |
| **第二级** | 指导保护级 | 对公民法人造成严重损害/对社会造成一般损害 | 主管部门指导 |
| **第三级** | 监督保护级 | 对社会造成严重损害/对国家造成一般损害 | **每年至少1次测评** ⭐ |
| **第四级** | 强制保护级 | 对国家造成严重损害 | 每半年至少1次测评 |
| **第五级** | 专控保护级 | 对国家造成特别严重损害 | 特殊保护 |

> **记忆重点**：三级是"监督保护级"，最常见的企业系统等级，**每年至少1次安全测评**。这是考试的高频考点！

### 2.3 等保流程"五步走"

```
定级 → 备案 → 安全建设整改 → 等级测评 → 监督检查
```

## 三、GDPR与数据隐私

### 3.1 GDPR核心要素

| 要素 | 内容 |
|------|------|
| 全称 | 通用数据保护条例（General Data Protection Regulation） |
| 生效 | 2018年5月25日 |
| 适用范围 | 处理欧盟居民数据的任何组织（不管在哪） |
| 罚款 | 最高2000万欧元或全球年营业额的4%（取更高者） |

### 3.2 数据主体权利

| 权利 | 说明 |
|------|------|
| 知情权 | 数据处理前需告知 |
| 访问权 | 可查阅自己的数据 |
| 更正权 | 可要求修改不准确数据 |
| 删除权（被遗忘权） | 可要求删除数据 |
| 限制处理权 | 可限制数据处理 |
| 数据可携带权 | 可获取数据副本并转移 |
| 反对权 | 可反对特定数据处理 |

### 3.3 GDPR六大处理原则

1. 合法、公正、透明
2. 目的限制
3. 数据最小化
4. 准确性
5. 存储限制
6. 完整性和保密性

### 3.4 中国个人信息保护法（PIPL）

- 施行：2021年11月1日
- 核心：告知-同意原则
- 数据出境：安全评估/标准合同/认证
- 罚款：最高5000万元或上年营收5%

## 四、应急响应与PDCERF

### 4.1 PDCERF模型（必考！）

| 阶段 | 英文 | 核心工作 |
|------|------|---------|
| P | **准备**（Preparation） | 建立应急团队、制定预案、培训演练 |
| D | **检测**（Detection） | 发现安全事件、确认事件真实性 |
| C | **遏制**（Containment） | 阻止事态扩大、隔离受影响系统 |
| E | **根除**（Eradication） | 清除攻击痕迹、修补漏洞、删除后门 |
| R | **恢复**（Recovery） | 恢复系统正常运行、验证安全性 |
| F | **跟踪/总结**（Follow-up） | 事件复盘、改进措施、更新预案 |

### 4.2 应急响应团队角色

| 角色 | 职责 |
|------|------|
| 总指挥 | 决策、对外沟通 |
| 技术负责人 | 技术分析和处置 |
| IT运维 | 系统恢复和备份还原 |
| 法务/合规 | 法律问题、证据保全 |
| 公关 | 媒体和客户沟通 |

### 4.3 数字取证原则

| 原则 | 说明 |
|------|------|
| 保护现场 | 不破坏原始证据 |
| 记录一切 | 拍照、记录每一步操作 |
| 链式保管 | 证据流转全程可追溯 |
| 法律效力 | 确保证据在法庭上可用 |

## 五、BCP/DRP业务连续性

### 5.1 BCP vs DRP（高频混淆！）

| 对比 | BCP（业务连续性计划） | DRP（灾难恢复计划） |
|------|----------------------|---------------------|
| 范围 | **更大**，涵盖整个业务 | 较小，聚焦IT系统恢复 |
| 关注点 | 业务持续运营 | IT系统和数据恢复 |
| 包含关系 | **BCP包含DRP** | DRP是BCP的IT部分 |
| 对象 | 整个组织 | IT部门和系统 |

> **核心记忆**：BCP > DRP，BCP包含DRP。BCP关注业务不中断，DRP关注IT能恢复。

### 5.2 RPO和RTO再回顾

| 指标 | 含义 | 核心问题 |
|------|------|---------|
| RPO | 恢复点目标 | 最多能丢多少数据？（多长时间的数据） |
| RTO | 恢复时间目标 | 系统恢复最长要多久？ |

### 5.3 备份策略

| 类型 | 方法 | 优点 | 缺点 |
|------|------|------|------|
| 完全备份 | 备份所有数据 | 恢复简单 | 耗时长、空间大 |
| 增量备份 | 仅备份上次备份后的变化 | 速度快、空间小 | 恢复需多个备份集 |
| 差异备份 | 仅备份上次完全备份后的变化 | 恢复较快 | 备份逐渐变大 |

### 5.4 容灾级别

| 级别 | 描述 | RTO/RPO |
|------|------|---------|
| 冷站 | 有空场地无设备 | RTO：数天到数周 |
| 温站 | 有设备需配置 | RTO：数小时到数天 |
| 热站 | 设备就绪可随时切换 | RTO：分钟到小时 |
| 双活 | 两个站点同时运行 | RTO：秒级 |

### 5.5 ISO 27001 ISMS

| 要素 | 说明 |
|------|------|
| 核心模型 | PDCA |
| 控制域 | 14个控制域，114项控制措施 |
| 版本 | ISO/IEC 27001:2013（最新2022版） |
| 核心文件 | SoA（适用性声明） |

## 六、专项选择题练习（15题）

**1. 《中华人民共和国网络安全法》正式施行的时间是？**

A. 2016年11月7日
B. 2017年6月1日
C. 2018年5月25日
D. 2019年12月1日

<details>
<summary>答案与解析</summary>

**B** —— 网络安全法于2016年11月7日通过，2017年6月1日正式施行。2018年5月25日是GDPR生效日期。

</details>

**2. 等保2.0中，第三级系统要求的等级测评频率是？**

A. 每半年至少1次
B. 每年至少1次
C. 每两年至少1次
D. 没有明确要求

<details>
<summary>答案与解析</summary>

**B** —— 三级（监督保护级）要求**每年至少1次**等级测评。四级要求每半年1次。三级是最常见的系统等级，也是常考点。

</details>

**3. PDCERF应急响应模型中，"C"代表？**

A. 沟通（Communication）
B. 遏制（Containment）
C. 创建（Creation）
D. 分类（Classification）

<details>
<summary>答案与解析</summary>

**B** —— PDCERF：Preparation(准备) → Detection(检测) → **Containment(遏制)** → Eradication(根除) → Recovery(恢复) → Follow-up(总结)。C=遏制，关键是隔离阻止蔓延。

</details>

**4. 以下关于BCP和DRP的描述，哪项正确？**

A. DRP包含BCP
B. BCP包含DRP
C. 两者完全独立无关
D. 两者完全相同

<details>
<summary>答案与解析</summary>

**B** —— BCP（业务连续性计划）范围大于DRP（灾难恢复计划），BCP包含DRP。DRP是BCP中IT系统和数据恢复的部分。

</details>

**5. GDPR中，被遗忘权赋予数据主体的权利是？**

A. 有权修改自己的数据
B. 有权要求删除自己的数据
C. 有权访问自己的数据
D. 有权导出自己的数据

<details>
<summary>答案与解析</summary>

**B** —— 被遗忘权（Right to be Forgotten）= 删除权（Right to Erasure），数据主体可以要求删除其个人数据。CISP考试中GDPR和数据权利是重点。

</details>

**6. 网络安全法中规定网络日志留存时间不少于？**

A. 3个月
B. 6个月
C. 12个月
D. 24个月

<details>
<summary>答案与解析</summary>

**B** —— 《网络安全法》明确规定：网络运营者应留存网络日志不少于**6个月**。这是一个精确的数字考点。

</details>

**7. 应急响应中，保存证据最重要的原则是？**

A. 尽快恢复系统
B. 保持证据的原始性，建立完整链式保管
C. 立即删除恶意文件
D. 直接通知媒体

<details>
<summary>答案与解析</summary>

**B** —— 数字取证中，保持证据的原始性（不可更改）和链式保管（Chain of Custody）是确保证据法律效力的关键。A虽然重要但不能牺牲证据。

</details>

**8. 等保2.0与1.0相比，新增的保护对象不包括？**

A. 云计算平台
B. 物联网
C. 办公文件柜
D. 工业控制系统

<details>
<summary>答案与解析</summary>

**C** —— 等保2.0扩展了保护对象，包括云计算、物联网、移动互联、工控系统和大数据。办公文件柜属于物理安全范畴，不是2.0新增的IT保护对象。

</details>

**9. 灾难恢复中，"热站"的特点是什么？**

A. 只有空地没有设备
B. 有设备需配置后才能使用
C. IT设备就绪，可随时切换
D. 不需要任何准备

<details>
<summary>答案与解析</summary>

**C** —— 热站（Hot Site）：设备已部署、数据近实时同步，切换时间最短（分钟到小时级）。冷站只有场地，温站有设备需配置。RTO：冷站>温站>热站。

</details>

**10. 以下哪项不是GDPR的合法数据处理基础？**

A. 数据主体同意
B. 履行合同所必需
C. 控制者单方面认为是合理的
D. 法定义务

<details>
<summary>答案与解析</summary>

**C** —— GDPR有6种合法处理基础：同意、合同必要、法定义务、重大利益、公共利益、合法利益。不存在"控制者认为合理"这一基础。

</details>

**11. 完整的备份策略中，恢复时需要最多备份集的是？**

A. 完全备份
B. 增量备份
C. 差异备份
D. 镜像备份

<details>
<summary>答案与解析</summary>

**B** —— 增量备份只备份上次备份后的变化，恢复时需要**完全备份+所有增量备份**，需要的备份集最多。差异备份只需完全备份+最后一次差异备份。

</details>

**12. PDCERF中，根除阶段（Eradication）的主要任务是？**

A. 发现安全事件
B. 彻底清除攻击痕迹和后门，修补漏洞
C. 准备应急团队
D. 写总结报告

<details>
<summary>答案与解析</summary>

**B** —— E=根除，核心工作是：清除恶意软件/后门、修补被利用的漏洞、重建受感染的系统、重置被泄露的凭据。

</details>

**13. 个人信息保护法对违法行为的最高罚款是？**

A. 100万
B. 1000万
C. 5000万或上年营收5%
D. 1亿

<details>
<summary>答案与解析</summary>

</details>

**14. ISO 27001 ISMS的核心管理模型是？**

A. SLDC
B. ITIL
C. PDCA

**15. 等保流程的五个步骤，正确的顺序是？**

A. 备案 → 定级 → 建设整改 → 等级测评 → 监督检查
B. 定级 → 备案 → 安全建设整改 → 等级测评 → 监督检查
C. 建设整改 → 定级 → 备案 → 等级测评 → 监督检查
D. 定级 → 等级测评 → 备案 → 建设整改 → 监督检查

<details>
<summary>答案与解析</summary>

**B** —— 等保五步：定级 → 备案 → 安全建设整改 → 等级测评 → 监督检查。必须先定级再备案，建设整改后才能测评。

</details>

## 七、高频易错点总结

| 易混淆概念 | 快速记忆法 |
|-----------|-----------|
| **BCP vs DRP** | BCP > DRP，BCP包含DRP |
| **RPO vs RTO** | P=Point(丢多少)，T=Time(等多久) |
| **PDCERF六阶段** | 准备→检测→遏制→根除→恢复→总结 |
| **等保三级测评频率** | **每年至少1次** |
| **日志留存时间** | **至少6个月**（网络安全法） |
| **GDPR被遗忘权** | 删除权，可要求删除个人数据 |
| **备份策略对比** | 完全=全备&快恢复；增量=快备&慢恢复；差异=折中 |
