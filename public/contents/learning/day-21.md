## 二、AAA 框架回顾

```
Authentication (认证)
├── 你是谁？
├── 三因素：Knowledge / Possession / Inherence
├── 技术：密码、MFA、生物识别、证书
└── 关键原则：多因素 > 单因素

Authorization (授权)
├── 你能做什么？
├── 模型：DAC / MAC / RBAC / ABAC
├── 原则：最小权限 + 职责分离
└── 决策：基于身份/角色/属性/规则

Accounting (审计)
├── 你做了什么？
├── 日志六要素：何时/谁/IP/做什么/资源/结果
├── 保护：不可篡改、WORM、异地备份
└── 合规：网络日志留存≥6个月
```

## 三、访问控制模型对比表

| 模型 | 决策依据 | 灵活性 | 安全级别 | 典型实现 |
|------|----------|--------|----------|----------|
| **DAC** (自主) | 资源拥有者决定 | ⭐⭐⭐ 高 | ⭐ 低 | Linux chmod |
| **MAC** (强制) | 系统安全标签 | ⭐ 低 | ⭐⭐⭐ 高 | SELinux |
| **RBAC** (角色) | 用户角色 | ⭐⭐ 中 | ⭐⭐ 中 | AD 组策略 |
| **ABAC** (属性) | 多维属性组合 | ⭐⭐⭐ 高 | ⭐⭐⭐ 高 | 零信任架构 |
| **Rule-Based** | 预设规则 | ⭐⭐ 中 | ⭐⭐ 中 | 防火墙 ACL |

### BLP vs Biba（必考！）

| 模型 | 保护目标 | 读规则 | 写规则 |
|------|----------|--------|--------|
| **Bell-LaPadula** (BLP) | **机密性** | No Read Up (不上读) | No Write Down (不下写) |
| **Biba** | **完整性** | No Read Down (不下读) | No Write Up (不上写) |

**口诀**：
- BLP：**低不能读高，高不能写低** → 仿泄密
- Biba：**高不能读低，低不能写高** → 防污染

| 方案 | 安全性 | 便利性 | 防钓鱼 | 离线可用 | 成本 |
|------|--------|--------|--------|----------|------|
| SMS OTP | ⚠️ 低 | ✅ | ❌ | — | ⭐ |

| 攻击 | 描述 | 防御 |
|------|------|------|

## 六、SSO 与联邦协议速查

### 三大协议定位

```
SAML 2.0    → 认证 + SSO (企业应用，XML)
OAuth 2.0   → 授权 (API 授权，JSON)
OpenID Connect → 认证 + SSO (现代应用，JSON+JWT)
                = OAuth 2.0 + 身份层
```

### OAuth 2.0 四种 Grant Type

| Grant Type | 安全 | 场景 |
|------------|------|------|
| Authorization Code + PKCE | ✅✅ | Web/SPA |
| Client Credentials | ✅ | 服务间 |
| Password | ⚠️ | CLI/遗留 |
| Implicit | ❌ 已弃用 | — |

### SAML vs OIDC 选型

```
SAML → 企业遗留应用 (Salesforce, Workday, SAP)
OIDC → 现代 Web/移动应用
两者共存 → 选择支持两套协议的 IdP (Okta, Azure AD)
```

## 八、易混淆辨析与学习建议

### TOP 5 混淆辨析

1. **认证 vs 授权**
   - 认证 (AuthN) = 你是谁
   - 授权 (AuthZ) = 你能做什么
   - 考试陷阱：把授权场景当认证问

2. **SAML vs OAuth vs OIDC**
   - SAML = 认证 SSO (XML)
   - OAuth = 授权 (JSON)
   - OIDC = OAuth + 认证层 = 现代 SSO

3. **BLP vs Biba**
   - BLP = 机密性 (Confidentiality)
   - Biba = 完整性 (Integrity)
   - 口诀：BLP→Confidential, Biba→Integrity

4. **TOTP vs HOTP**
   - TOTP = 基于时间 (30秒刷新)
   - HOTP = 基于计数器 (按一次换一次)

5. **FAR vs FRR**
   - FAR = 假阳性 (坏人放进来)
   - FRR = 假阴性 (好人挡出去)
