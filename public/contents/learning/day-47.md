## 一、Wi-Fi安全协议演进

### 1.1 安全协议时间线

```
┌─────────┬──────────┬──────────────┬──────────────────────┐
│  协议    │   年份    │   加密/认证   │       安全性          │
├─────────┼──────────┼──────────────┼──────────────────────┤
│ WEP     │   1999   │ RC4 + CRC-32 │ ☠ 分钟级破解          │
│ WPA     │   2003   │ TKIP (RC4)   │ ⚠️ 已知弱点           │
│ WPA2    │   2004   │ CCMP (AES)   │ ⚠️ KRACK攻击(2017)   │
│ WPA3    │   2018   │ SAE + GCMP   │ ✅ 当前推荐           │
└─────────┴──────────┴──────────────┴──────────────────────┘
```

### 1.2 WEP为何不安全

```
WEP致命缺陷：

① 短IV（24位=1670万个值）
   → 流量大的AP数小时内IV循环重复
   → IV重复 → RC4密钥流复用 → 可恢复明文

② RC4弱点
   → FMS攻击(2001)：利用弱IV，100万包可破解
   → PTW攻击(2007)：仅需4万包
   → 当前：商用工具几秒到几分钟破解

③ CRC-32完整性检查
   → 线性的，可预测修改
   → 攻击者可以伪造数据包的CRC

④ 静态密钥
   → 所有客户端使用相同密钥
   → 无密钥管理机制

结论：WEP = 完全不安全，等同于开放网络！
```

## 二、WPA3深度解析

### 2.1 WPA2 vs WPA3

```
┌──────────────────┬─────────────────────┬─────────────────────┐
│      特性         │       WPA2          │       WPA3          │
├──────────────────┼─────────────────────┼─────────────────────┤
│ 个人模式认证      │ PSK (预共享密钥)     │ SAE (对等同步认证)   │
│ 握手             │ 4-Way Handshake     │ SAE + 4-Way HS     │
│ 前向保密          │ ❌ (PSK模式)         │ ✅ 内置              │
│ 离线字典攻击      │ ✅ 易受攻击           │ ❌ 抗离线字典        │
│ 加密             │ CCMP-128 (AES)      │ GCMP-256 (AES-256) │
│ 管理帧保护(PMF)   │ 可选                │ 强制                │
│ Wi-Fi Easy Connect│ ❌                  │ ✅ (DPP, QR扫码)    │
│ 公共网络保护      │ ❌ (明文)            │ ✅ (OWE加密)         │
│ 企业模式         │ 802.1X              │ 802.1X + 192位      │
└──────────────────┴─────────────────────┴─────────────────────┘
```

### 2.2 SAE握手详解

```
WPA3 SAE (Simultaneous Authentication of Equals)：

  核心：基于Dragonfly密钥交换（密码认证密钥交换PAKE）

  Client                          AP
    │                              │
    │── Commit (标量+元素) ──────→│  ← 双方生成临时密钥对
    │←── Commit (标量+元素) ────│
    │                              │
    │── Confirm (验证令牌) ──────→│  ← 双方验证对方知道密码
    │←── Confirm (验证令牌) ────│
    │                              │
    │  PMK 已安全建立              │
    │  ← 4-Way Handshake →        │

关键安全特性：
  ✅ 抗离线字典攻击：每次尝试需要与AP交互
  ✅ 前向保密：即使密码泄露，历史流量仍安全
  ✅ 完美隐匿：不发送密码或哈希
```

### 2.3 WPA3新技术

```
① OWE (Opportunistic Wireless Encryption)：
  开放网络自动加密（无密码Wi-Fi也加密！）
  使用DH密钥交换建立连接

② DPP (Device Provisioning Protocol)：
  通过QR码/NFC轻松连接（替代WPS）
  基于公钥的零触摸配置

③ 192位安全模式 (WPA3-Enterprise)：
  ECDH + ECDSA ≥ 384位曲线
  AES-256-GCMP
  SHA-384 HMAC
  适用于政府/军事高安全环境
```

## 三、802.1X企业认证

### 3.1 802.1X架构

```
802.1X 三要素：

  ┌──────────┐      ┌──────────┐      ┌──────────┐
  │ Supplicant│ ←──→ │Authenticator│ ←──→ │  Auth    │
  │  (客户端)  │ EAPoL│ (交换机/AP)  │ RADIUS│ Server   │
  │          │      │          │      │(RADIUS)  │
  └──────────┘      └──────────┘      └──────────┘

流程：
1. 客户端连接到交换机/AP → 端口未授权（仅放行EAPoL）
2. 客户端通过EAPoL发送认证信息
3. 交换机/AP将EAP封装到RADIUS转发给认证服务器
4. 认证服务器验证 → 授权/拒绝
5. 授权后交换机/AP开放端口 → 正常网络访问

EAP方法选项：
  PEAP-MSCHAPv2：常见企业方案
  EAP-TLS：证书认证（最安全）
  EAP-TTLS：隧道认证
  EAP-SIM/AKA：移动网络认证
```

### 3.2 EAP-TLS vs PEAP

| 特性 | EAP-TLS | PEAP-MSCHAPv2 |
|------|---------|---------------|
| 客户端认证 | 客户端证书 | 用户名+密码 |
| 服务器认证 | 服务器证书 | 服务器证书 |
| 安全性 | ✅ 最安全 | ⚠️ MSCHAPv2已知弱点 |
| 部署难度 | 高（需要客户端证书PKI） | 中 |
| 推荐度 | 强烈推荐（高安全环境） | 一般（需配合强密码策略） |

## 四、常见Wi-Fi攻击与防御

### 4.1 攻击全景

```
┌──────────────────┬────────────────────────────────────┐
│     攻击类型      │              描述                  │
├──────────────────┼────────────────────────────────────┤
│ Rogue AP         │ 部署假冒AP，诱骗用户连接              │
│ Evil Twin        │ 仿冒合法AP的SSID和MAC               │
│ Deauthentication │ 发送伪造Deauth帧断开客户端           │
│ KRACK (WPA2)     │ 重用安装的PTK恢复密钥                │
│ PMKID攻击        │ 从WPA2 RSN IE中提取PMKID哈希        │
│ WPS PIN暴力      │ 11,000种可能(PIN分两半验证)          │
│ 信标洪水          │ 伪造数千SSID淹没无线环境             │
│ 客户端探测响应伪造│ 响应客户端探查→冒充目标AP            │
└──────────────────┴────────────────────────────────────┘
```

### 4.2 Evil Twin攻击流程

```
Evil Twin 完整攻击链：

① 侦察：扫描目标区域的合法SSID
② 部署：在同一信道广播相同SSID的AP（信号更强）
③ 解除认证：向客户端发送Deauth帧迫使其断开合法AP
④ 客户端重连：客户端自动连接到信号更强的假冒AP
⑤ 捕获：截获用户流量，展示仿冒Portal页面
⑥ 窃取凭据：用户输入Wi-Fi密码/邮箱密码

防御：
  ✅ WPA3 (SAE认证AP，双向验证)
  ✅ 802.1X (服务器证书验证)
  ✅ WIPS (检测异常AP)
  ✅ 用户意识培训
```

### 4.3 Deauth攻击

```
Deauthentication (解除认证) 攻击：

Wi-Fi管理帧（包括Deauth）是明文的、不需认证的！
→ 攻击者可以伪造AP的MAC地址发送Deauth帧

影响：
  · 强制客户端掉线 → 抓取WPA握手包 → 离线破解
  · 强迫客户端连接到Evil Twin
  · DoS攻击

防御：
  ✅ 802.11w：管理帧保护 (PMF)
     → WPA3强制执行
     → WPA2可选但推荐启用
  ✅ WIPS监控大量Deauth帧
```

## 五、无线入侵检测(WIPS)

### 5.1 WIPS能力

```
WIPS (Wireless Intrusion Prevention System)：

① 检测Rogue AP：
   → 分析信标帧、MAC地址WPS信息
   → 有线侧+无线侧关联比对

② 检测攻击行为：
   → Deauth洪水
   → Evil Twin
   → MAC欺骗
   → WEP/WPA破解尝试
   → 错误配置

③ 自动防御：
   → 发送Deauth到假冒AP的客户端
   → 无线层面隔离恶意设备

④ 合规定位：
   → 无线电测向/三角定位找到物理设备
```

## 六、蓝牙安全

### 6.1 蓝牙攻击

```
┌───────────────┬───────────────────────────────────┐
│    攻击        │              描述                  │
├───────────────┼───────────────────────────────────┤
│ Bluejacking   │ 向蓝牙设备发送未请求消息             │
│ Bluesnarfing  │ 未授权访问设备数据(联系人/短信)      │
│ Bluebugging   │ 完全控制蓝牙设备                     │
│ BlueBorne     │ 无需配对的远程代码执行 (CVE-2017)   │
│ KNOB          │ 攻击密钥协商，降级加密为1字节密钥    │
│ BIAS          │ 蓝牙假冒攻击，绕过安全连接            │
│ BLURtooth     │ 跨传输攻击，CTKD密钥覆写            │
└───────────────┴───────────────────────────────────┘

Bluetooth版本安全：
  BR/EDR (经典蓝牙): 多个已知攻击
  BLE 4.2+: LE Secure Connections 更安全
  Bluetooth 5.2+: 进一步增强
```

### 6.2 蓝牙安全最佳实践

```
✅ 默认关闭蓝牙可发现模式
✅ 使用蓝牙 4.2+ 版本
✅ 配对在安全环境进行
✅ 及时更新蓝牙固件
✅ 不要接受未请求的配对请求
✅ 对高安全环境考虑禁用蓝牙
✅ 使用LE Secure Connections配对方法
✅ 实施蓝牙设备白名单
```

## 八、CISP考试速查

### 关键考点

| 考点 | 记忆要点 |
|------|---------|
| WEP不安全原因 | "RC4+短IV(24位)+静态密钥=分钟级破解" |
| WPA3核心改进 | "SAE抗离线暴力 + 前向保密 + PMF强制" |
| 802.1X三组件 | "Supplicant + Authenticator + Auth Server" |
| KRACK攻击 | "WPA2的4次握手重安装密钥漏洞(2017)" |
| Evil Twin防御 | "WPA3 SAE双向认证" |

### 常见陷阱

1. **"隐藏SSID可以防黑客"** → 客户端探测时会广播SSID，很容易发现
2. **"MAC过滤是一种安全措施"** → MAC地址可以轻松伪造
3. **"WPA2很安全不需要升级"** → KRACK攻击已证明其弱点
4. **"WPA3的企业模式和个人模式相同"** → 企业模式多802.1X+强加密
