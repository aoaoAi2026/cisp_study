## 一、信息安全基础高频考点 ⭐⭐⭐⭐

### 必背三句话

```
1. CIA三要素：机密性(Confidentiality)、完整性(Integrity)、可用性(Availability)
   口诀：CIA不是情报局，C是加密I是签名A是备份

2. 纵深防御：多层防护，没有单一安全控制能防御所有威胁
   口诀：城门→城墙→护城河→卫兵，层层设防

3. 最小权限：完成工作所需的最小权限集合
   口诀：够用就好，不多不少
```

### ISO 27001 PDCA速记

```
P - Plan（策划） → "定规矩"：范围、风险评估、安全策略
D - Do（实施）   → "按规矩做"：实施控制措施、培训
C - Check（检查）→ "查效果"：内部审计、管理评审、监控
A - Act（改进）  → "改不足"：纠正措施、预防措施、持续改进

口诀：皮蛋叉烧（PDCA）→ 定做查改
```

### 安全控制分类

```
管理控制 → "人管"：策略、流程、培训
技术控制 → "技防"：防火墙、加密、IDS
物理控制 → "物防"：门禁、CCTV、防火

预防性控制 → 事前（灭火器培训）
检测性控制 → 事中（烟雾报警器）
纠正性控制 → 事后（灭火系统）

口诀：人技物、前中后
```

## 三、访问控制速记卡片 ⭐⭐⭐⭐⭐

### BLP vs Biba（最高频对比）

```
BLP保护机密性：              Biba保护完整性：
                             
  高级别  ←不可读            低完整性 ←不可写  
    ↓不可写                   ↑不可读
  低级别                     高完整性

BLP：不上读、不下写          Biba：不下读、不上写
    （低不读高、高不写低）       （高不读低、低不写高）

考场区分秘诀：
  BLP → "上"字在口诀中排第一项 → 先讲"不XX读" → BLP = 不上读
  Biba → "下"字在口诀中排第一项 → 先讲"不XX读" → Biba = 不下读
```

### 四种访问控制模型对比

```
DAC → 所有者定权限 → 灵活不安全 → 个人文件共享
MAC → 系统定权限   → 严格安全   → 军队/政府
RBAC→ 角色定权限   → 平衡实用   → 企业ERP
ABAC→ 属性定权限   → 动态精细   → 零信任/云

口诀：D自定、M强制、R角色、A属性（按字母记）
```

### 认证协议速记

```
Kerberos → 企业内部 → 对称加密 → TGT票据 → "门票系统"
RADIUS   → 网络接入 → UDP → 共享密钥 → "路由器认证"
SAML 2.0 → 跨组织SSO → XML断言 → 联邦身份 → "联盟护照"
OAuth 2.0→ 第三方授权 → Access Token → "授权委托书"
OIDC     → SSO 2.0 → JWT → OAuth + 身份 → "新一代护照"
```

## 四、应急响应关键数字 ⭐⭐⭐⭐

### PICERL六步法

```
P - 准备      → 应急预案、团队、工具（平战结合）
I - 识别      → 检测确认、定级通报
C - 遏制      → 隔离、断网、禁用 ← 止损第一！
E - 根除      → 删除恶意软件、修复漏洞、改密码
R - 恢复      → 从备份还原、验证完整性
L - 总结      → 复盘、更新预案、追究责任

口诀：PICERL = "批挨拆俄如勒"
     准备→发现→按住→根除→恢复→总结
```

### 核心数字指标

```
RTO → 恢复时间目标 → "能停多久"
RPO → 恢复点目标   → "能丢多少"
MTBF→ 平均故障间隔 → 可靠性指标
MTTR→ 平均修复时间 → 可维护性指标
可用性 = MTBF / (MTBF + MTTR)

灾备站点：
  冷站 → 只有房子 → 恢复需数天-数周 → 成本最低
  温站 → 有部分家伙 → 恢复需数小时 → 成本中等
  热站 → 全副武装 → 恢复需分钟 → 成本最高
```

## 五、漏洞攻防一句话总结 ⭐⭐⭐⭐

```
SQL注入     → 用户输入拼进SQL里 → 参数化查询是王道
反射型XSS   → 参数里的恶意脚本 → 输出编码 + CSP
存储型XSS   → 数据库里的毒脚本 → 入库过滤 + 出库编码
DOM型XSS    → 前端JS自己挖坑    → 不用innerHTML
CSRF        → 借别人身份发请求 → Token + SameSite
SSRF        → 让服务器替我做内网请求 → 白名单 + 禁内网
命令注入    → 把系统命令塞进输入 → 不用exec，用安全API
文件上传    → 假冒的图片实为木马 → 魔术数字 + 隔离 + 改名
XXE         → XML里藏外部实体 → 禁用DTD和外部实体
DDoS        → 人多欺负人少 → 流量清洗 + CDN + 限速
```

## 六、密码学参数对照法 ⭐⭐⭐⭐⭐

### 一键速查表

```
算法     类型      关键长度/输出      分组     安全基础
──────────────────────────────────────────────────
DES      对称      56位密钥          64位     Feistel → 已弃用
3DES     对称      112/168位         64位     三重DES → 逐渐淘汰
AES      对称      128/192/256密钥   128位    SPN → 当前主流
RC4      流密码    可变              无       流密码 → 已弃用
RSA      非对称    ≥2048位           N/A      大整数分解
ECC      非对称    256位(~=128安全)   N/A      椭圆曲线
SHA-256  哈希      256位输出        512b输入   MD结构
SHA-512  哈希      512位输出       1024b输入   MD结构
MD5      哈希      128位输出        512b输入   已破解！
HMAC     MAC       =哈希输出长度    N/A       带密钥哈希
```

### 最易错参数记忆

```
AES分组长度：只有128位！（不是可变！不是256！）
AES密钥长度：128/192/256位（这才是可变的）

对称算法（DES/AES）：分组长度 = 密钥长度？错！
  DES：分组64位，密钥56位（不同！）
  AES：分组128位，密钥128/192/256位（也可不同！）

RSA安全强度：2048位密钥 ≈ 112位安全强度
ECC安全强度：256位密钥 ≈ 128位安全强度（比RSA 2048更强！）
```

### 加密模式一句话

```
ECB → 最简单的，也是最危险的（相同明文=相同密文）
CBC → 链式加密，前块影响后块
CTR → 计数器模式，可并行加密
GCM → 目前最佳，加密+认证一体

不需要IV的模式：只有ECB！（考试必考点）
```

## 七、网络安全协议对比 ⭐⭐⭐⭐

### OSI层与安全协议

```
7 应用层  → DNS → DNSSEC → 防劫持投毒
            HTTP → HTTPS  → TLS加密
            FTP  → SFTP/FTPS → 安全传输
            SMTP → SMTPS/STARTTLS → 邮件加密

6 表示层  → TLS/SSL → 1.3最新 → 握手1-RTT

5 会话层  → SOCKS → 通用代理

4 传输层  → TCP → TLS over TCP
            UDP → DTLS → 保护UDP

3 网络层  → IP → IPsec(ESP) → VPN加密隧道
            ICMP → 禁止或限速 → 防隧道防Flood

2 链路层  → 以太网 → 802.1X → 端口接入控制
            Wi-Fi → WPA3 → SAE防暴力破解
```

### TLS 1.3 vs TLS 1.2 速记

```
TLS 1.3 的改进：更快、更安全、更简单
  快：1-RTT握手（TLS 1.2需2-RTT）
  安全：砍掉所有弱算法（RSA密钥交换、CBC、RC4、3DES、SHA-1）
  简单：加密套件从几十个精简到5个
  新功能：0-RTT（快速重连，但有重放风险）
```

### 防火墙演进

```
第1代：包过滤 → 检查IP/端口 → 最早的
第2代：应用代理 → 应用层代理 → 速度慢
第3代：状态检测 → 跟踪连接状态 → 主流
第4代：深度包检测 → 应用层识别 → DPI
NGFW：下一代 → 集成IPS+应用识别+威胁情报
WAF：Web应用防火墙 → HTTP层防护 → 防SQLi/XSS
```

## 十、风险管理公式速算 ⭐⭐⭐

```
核心公式（必须会手算！）：

  SLE = AV × EF
  (单次损失 = 资产价值 × 损失比例)

  ALE = SLE × ARO
  (年度损失 = 单次损失 × 年发生次数)

  残余风险 = 总风险 - 控制抵消
  Risk  = Likelihood × Impact

例题速算：
  资产100万，每次损失60%，每2年发生1次
  SLE = 100万 × 60% = 60万
  ARO = 0.5次/年
  ALE = 60万 × 0.5 = 30万/年
```

### 风险处置四策略

```
规避 → 不做了           → 停用有漏洞的应用
转移 → 甩锅给保险公司    → 买网络安全保险
缓解 → 加控制措施       → 部署WAF/IDS
接受 → 不管了(知情的)   → 低风险成本可控

❌ "忽略"——不是合法策略！（考试挖坑）
```

## 十一、数据隐私核心概念 ⭐⭐⭐⭐

```
个人信息处理合法性基础（个保法第13条）：
  ① 个人同意
  ② 合同必需
  ③ 法定职责
  ④ 公共卫生
  ⑤ 已公开信息合理处理
  ⑥ 其他法律规定

敏感个人信息（需单独同意）：
  生物识别 → 指纹/人脸
  医疗健康 → 病历/体检
  金融账户 → 卡号/余额
  行踪轨迹 → GPS数据
  宗教信仰/特定身份
  未满14周岁 → 需监护人同意

数据出境安全评估触发条件：
  ▶ 重要数据出境
  ▶ CII运营者个人信息出境
  ▶ 处理100万+人信息的处理者 → 个人信息出境
  ▶ 累计10万+人或1万+人敏感信息出境

脱敏技术对比：
  静态脱敏(SDM) → 修改存储数据 → 测试/开发环境
  动态脱敏(DDM) → 查询时实时改写 → 生产环境
  匿名化        → 不可逆 → 不再属于个人信息
  去标识化      → 条件可逆 → 仍属于个人信息
```
