## 一、什么是信息安全

### 1.1 核心定义

信息安全（Information Security）是指通过技术手段与管理措施，保护信息及其相关系统免受 **未经授权的访问、使用、披露、破坏、修改和中断**。其本质是确保信息在整个生命周期——创建、存储、传输、使用、归档、销毁——始终处于可控状态。

> **💡 核心理解**：信息安全 ≠ 网络安全。网络安全侧重网络层面的攻防，信息安全涵盖网络、系统、应用、数据、物理、人员、管理等多个维度。

### 1.2 信息安全 vs 网络安全 vs 数据安全

```
┌─────────────────────────────────────┐
│           信息安全 (最广)             │
│  ┌─────────────────────────────┐    │
│  │       网络安全 (网络层面)     │    │
│  │  ┌─────────────────────┐    │    │
│  │  │ 数据安全 (数据本身)   │    │    │
│  │  └─────────────────────┘    │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```

| 维度 | 信息安全 | 网络安全 | 数据安全 |
| :--- | :--- | :--- | :--- |
| **范围** | 最广，涵盖所有 | 网络层面的攻防 | 数据全生命周期 |
| **关注点** | 信息资产的 CIA | 网络边界、传输安全 | 数据分类、加密、脱敏 |
| **典型措施** | ISMS、风险评估 | 防火墙、IDS/IPS、VPN | DLP、加密、脱敏 |
| **代表法规** | 网络安全法 | 网络安全法 | 数据安全法 |

### 1.3 为什么信息安全如此重要

| 维度 | 重要性说明 | 现实案例 |
| :--- | :--- | :--- |
| **国家安全** | 关键基础设施依赖信息系统 | 震网病毒破坏伊朗核设施 |
| **企业生存** | 数据泄露可导致破产 | Equifax 1.43 亿用户信息泄露 |
| **个人权利** | 信息被滥用危害隐私 | 电信诈骗、精准营销骚扰 |
| **社会稳定** | 大规模攻击影响民生 | WannaCry 导致医院停诊 |

## 二、CIA 三要素深度解析

CIA 三要素（CIA Triad）是信息安全最核心的理论模型，也是 CISP 考试的 **必考内容**，几乎贯穿全部科目。

### 2.1 机密性（Confidentiality）

**定义**：确保信息仅被授权主体访问，防止未经授权的信息披露。

**常见的破坏方式**：
- 网络嗅探（明文传输被捕获）
- 权限配置错误（敏感目录可被任意访问）
- 内部人员泄露（离职带走客户数据）
- 物理偷窥（肩窥、拾获存储介质）
- 社会工程学（假冒身份骗取信息）

**保护措施**：

| 措施类别 | 具体手段 | 应用场景 |
| :--- | :--- | :--- |
| **加密** | AES、RSA、TLS | 数据传输、存储加密 |
| **访问控制** | ACL、RBAC、MAC | 文件/系统权限管理 |
| **身份认证** | 密码、MFA、生物识别 | 登录验证 |
| **数据脱敏** | 掩码、替换、泛化 | 开发测试环境 |
| **DLP** | 数据防泄露系统 | 邮件/外设管控 |

> **🔑 高分考点**：机密性 ≠ 完全不可见。授权用户仍然可以访问，关键是"未授权"的不能访问。

### 2.2 完整性（Integrity）

**定义**：确保信息在存储和传输过程中不被未授权篡改，保证信息的准确性和一致性。

**常见的破坏方式**：
- 中间人攻击（篡改传输中的报文）
- SQL 注入（篡改数据库记录）
- 恶意软件（修改系统文件）
- 存储介质损坏（比特翻转）
- 人为误操作（误删除/修改）

**保护措施**：

| 措施类别 | 具体手段 | 原理 |
| :--- | :--- | :--- |
| **哈希校验** | MD5、SHA-256 | 对比前后哈希值，确认未修改 |
| **数字签名** | RSA 签名、ECDSA | 身份认证 + 完整性校验 |
| **版本控制** | Git、SVN | 记录变更历史，可回滚 |
| **WAL 日志** | 预写日志 | 数据库崩溃恢复 |
| **RAID / 纠删码** | RAID 5/6 | 冗余保护防止数据损坏 |

> **⚠️ 考试陷阱**：MD5 已不安全（存在碰撞攻击），CISP 考试中如问完整性校验，**优先选 SHA-256 及以上**。

### 2.3 可用性（Availability）

**定义**：确保授权用户在需要时能够及时、可靠地访问信息及相关系统。

**常见的破坏方式**：
- DDoS 攻击（流量洪水耗尽带宽）
- 勒索软件（加密文件导致无法使用）
- 硬件故障（服务器/磁盘损坏）
- 电力中断（机房断电）
- 配置错误（误删生产数据）

**保护措施与关键指标**：

| 指标 | 含义 | 典型值 |
| :--- | :--- | :--- |
| **RTO**（恢复时间目标） | 从故障到恢复的最大允许时间 | 核心系统 < 4h |
| **RPO**（恢复点目标） | 允许丢失的最大数据量（按时间） | 核心系统 < 15min |
| **SLA**（服务等级协议） | 可用性承诺 | 99.9% → 年宕机 < 8.76h |

**可用性分级**：
```
99%     → 年宕机 3.65 天
99.9%   → 年宕机 8.76 小时    （三个九）
99.99%  → 年宕机 52.56 分钟   （四个九）
99.999% → 年宕机 5.26 分钟    （五个九，电信级）
```

> **💡 知识巧记**："机密靠加密，完整性看哈希，可用性要冗余——三者缺一不可，场景不同权重不同。"

### 2.4 CIA 三要素的平衡

CIA 三者之间存在**制约关系**，需要根据业务场景权衡：

| 场景 | 优先级 | 说明 |
| :--- | :--- | :--- |
| **银行交易系统** | 完整性 > 机密性 > 可用性 | 数据绝不能被篡改 |
| **新闻门户网站** | 可用性 > 完整性 > 机密性 | 被篡改可恢复，但不能打不开 |
| **军事通信系统** | 机密性 > 完整性 > 可用性 | 泄密后果不可挽回 |
| **医疗急救系统** | 可用性 > 机密性 > 完整性 | 救命优先于隐私 |

## 三、其他安全属性扩展

除了 CIA 三要素，信息安全还包含以下重要属性（常作为多选题干扰项）：

| 属性 | 英文 | 含义 | 示例 |
| :--- | :--- | :--- | :--- |
| **真实性** | Authenticity | 主体或数据的身份真实可信 | 数字证书验证网站身份 |
| **不可否认性** | Non-Repudiation | 行为发生后无法抵赖 | 数字签名、审计日志 |
| **可控性** | Controllability | 对信息和系统的支配能力 | 权限管理、数据分类 |
| **可审计性** | Accountability | 操作行为可追溯追责 | 日志记录、审计追踪 |
| **隐私性** | Privacy | 个人信息自主控制 | 个人信息保护法 |

> **🔑 高分考点**：牢记 CIA 是"三要素"（必选），其他是"扩展属性"。考题可能混淆这两组概念。

## 四、安全威胁系统分类

### 4.1 按来源分类

| 来源 | 占比（行业估算） | 特征 | 典型案例 |
| :--- | :--- | :--- | :--- |
| **内部威胁** | ~30-40% | 员工疏忽、恶意员工、离职报复 | 员工误发邮件泄露数据 |
| **外部威胁** | ~60-70% | 黑客、APT、网络犯罪组织 | DDoS 攻击、勒索软件 |
| **第三方威胁** | 与外部重叠 | 供应链攻击、外包泄露 | SolarWinds 供应链攻击 |

### 4.2 按攻击方式分类

| 分类 | 特点 | 代表攻击 | 防御思路 |
| :--- | :--- | :--- | :--- |
| **被动攻击** | 不改变系统状态，难以检测 | 窃听、流量分析、侧信道 | 加密、屏蔽、流量填充 |
| **主动攻击** | 改变系统状态或数据 | 篡改、伪造、重放、DoS | 完整性校验、访问控制 |
| **社会工程** | 针对"人"而非"技术" | 钓鱼、假冒、尾随 | 安全培训、制度约束 |

### 4.3 常见威胁 Top 10

```
1. 恶意软件（病毒/蠕虫/木马/勒索软件）
2. 网络钓鱼（Phishing/Spear Phishing）
3. 中间人攻击（MITM）
4. 拒绝服务攻击（DoS/DDoS）
5. SQL 注入
6. 跨站脚本（XSS）
7. 零日漏洞利用
8. 密码攻击（暴力破解/撞库/凭据填充）
9. 内部人员威胁
10. 供应链攻击
```

## 五、攻击动机与攻击者画像

### 5.1 攻击者类型

| 类型 | 动机 | 技术能力 | 目标 |
| :--- | :--- | :--- | :--- |
| **脚本小子** | 炫耀、好奇 | 低（使用现成工具） | 无特定目标 |
| **黑客活动分子** | 政治/社会主张 | 中 | 政府、企业网站 |
| **网络犯罪组织** | 经济收益 | 中-高 | 企业数据、金融系统 |
| **APT 组织** | 政治/军事/商业情报 | 极高 | 政府、军工、关键设施 |
| **内部人员** | 报复、利益、疏忽 | 不定 | 所在组织 |
| **国家行为体** | 国家安全 | 顶极 | 敌对国家基础设施 |

### 5.2 攻击生命周期（Cyber Kill Chain）

```
侦查 → 武器化 → 投递 → 利用 → 安装 → 命令与控制 → 目标行动
```

> **💡 知识巧记**："先踩点，后制器，投送到目标，利用漏洞进，安装后门驻，C2 来遥控，最后达目标"——在任意阶段阻断即可破坏攻击链。

## 六、纵深防御（Defense in Depth）

### 6.1 核心理念

纵深防御是信息安全最经典的设计原则：**不依赖单一安全措施，通过多层安全控制的叠加，即使某一层被突破，后续层次仍能提供保护**。

```
┌─────────────────────────────────────────────────────────┐
│                    纵深防御层次模型                       │
├─────────────────────────────────────────────────────────┤
│  第一层：物理安全（门禁、监控、机柜锁）                    │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 第二层：网络安全（防火墙、IDS/IPS、网段隔离）      │    │
│  │ ┌───────────────────────────────────────────┐   │    │
│  │ │ 第三层：主机安全（HIDS、防病毒、加固）       │   │    │
│  │ │ ┌───────────────────────────────────┐     │   │    │
│  │ │ │ 第四层：应用安全（WAF、认证、鉴权） │     │   │    │
│  │ │ │ ┌───────────────────────────┐     │     │   │    │
│  │ │ │ │ 第五层：数据安全（加密、DLP）│     │     │   │    │
│  │ │ │ └───────────────────────────┘     │     │   │    │
│  │ │ └───────────────────────────────────┘     │   │    │
│  │ └───────────────────────────────────────────┘   │    │
│  └─────────────────────────────────────────────────┘    │
│  第六层：安全管理（制度、培训、审计）                      │
└─────────────────────────────────────────────────────────┘
```

### 6.2 纵深防御 vs 单一防护

| 对比维度 | 单一防护 | 纵深防御 |
| :--- | :--- | :--- |
| 安全性 | 单点故障就会沦陷 | 多重保护，层层阻击 |
| 成本 | 看似低（买一个设备） | 较高（多层投入） |
| 管理复杂度 | 简单 | 复杂，需协调联动 |
| 实战表现 | 不堪一击 | 大幅提高攻击成本 |

> **🔑 高分考点**：纵深防御 ≠ 买很多安全设备堆叠。真正的纵深防御是**从物理到应用、从预防到响应、从技术到管理的完整覆盖**。

## 七、信息安全治理框架

### 7.1 主流框架对比

| 框架 | 全称 | 侧重点 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **ISO 27001** | 信息安全管理体系 | 管理体系、PDCA 循环 | 通用，认证导向 |
| **NIST CSF** | 网络安全框架 | 识别-保护-检测-响应-恢复 | 美国及国际通用 |
| **等保 2.0** | 网络安全等级保护 | 合规、定级、测评 | 中国强制要求 |
| **COBIT** | 信息及相关技术控制目标 | IT 治理 | 大型企业 IT 治理 |

### 7.2 PDCA 循环（ISO 27001 核心）

```
Plan（规划）→ Do（实施）→ Check（检查）→ Act（改进）
    ↑                                              ↓
    └──────────────────────────────────────────────┘
                   （持续改进循环）
```

> **💡 知识巧记**："PDCA 四个方面——先规划建体系，再落实去执行，定期检查找差距，持续改进保安全。"

## 八、安全控制措施全景

### 8.1 三大控制类别

| 类别 | 说明 | 典型措施 |
| :--- | :--- | :--- |
| **管理控制**（Administrative） | 制度、流程、人员 | 安全策略、培训、审计、BCP |
| **技术控制**（Technical） | 软硬件技术手段 | 防火墙、加密、IDS/IPS、IAM |
| **物理控制**（Physical） | 物理环境安全 | 门禁、监控、机柜锁、UPS |

### 8.2 控制功能分类

| 功能 | 说明 | 示例 |
| :--- | :--- | :--- |
| **预防性** | 事前阻止 | 防火墙 ACL、身份认证 |
| **检测性** | 事中发现 | IDS、日志监控、异常检测 |
| **纠正性** | 事后恢复 | 备份恢复、补丁修复 |
| **威慑性** | 心理震慑 | 法律告警、监控标识 |

## 九、常见安全误区与纠正

| 误区 | 正确理解 |
| :--- | :--- |
| "装了防火墙就安全了" | 防火墙只是边界防护，内部威胁、应用层攻击不可忽视 |
| "加密了数据就万无一失" | 加密保证机密性，但不保证完整性和可用性 |
| "我们小公司不会被攻击" | 自动化攻击工具无差别扫描，小型组织常因防护弱而首当其冲 |
| "安全是 IT 部门的事" | 安全是全员责任，70% 以上的泄露涉及人为因素 |
| "合规 = 安全" | 合规是基线要求，真正安全需要超越合规的持续改进 |
