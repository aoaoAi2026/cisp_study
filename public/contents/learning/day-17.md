## 一、生物识别技术概述

### 1.1 什么是生物识别

```
生物识别 (Biometrics) =
通过人体固有的生理特征或行为特征来进行身份认证

分类：

生理特征（静态，不变的）
├── 指纹 (Fingerprint)
├── 人脸 (Face)
├── 虹膜 (Iris)
├── 视网膜 (Retina)
├── 掌静脉 (Palm Vein)
├── 手形 (Hand Geometry)
└── DNA

行为特征（动态，可变的）
├── 声纹 (Voiceprint)
├── 签名动态 (Signature Dynamics)
├── 击键模式 (Keystroke Dynamics)
├── 步态 (Gait)
└── 鼠标/触控行为模式
```

### 1.2 生物识别的独特属性

```
生物识别与其他认证因素的比较：

             密码        智能卡        生物识别
易丢失       ✅不会       ❌会          ✅不会
易遗忘       ❌会        ✅不会         ✅不会
易共享       ❌容易      ❌容易         ✅很难
易复制       ❌容易      ⚠️可能        ⚠️可能（伪造）
可更换       ✅          ✅            ❌不可更换！
需要记忆     ❌          ✅             ✅不需要
用户接受度   ✅          ✅             ⚠️隐私顾虑
```

> **最关键的差异**：密码可以被改变，智能卡可以补办，但**指纹、虹膜等生物特征一旦泄露，终身无法更换**。这也是生物特征安全保护的核心挑战。

## 二、核心评价指标：FAR 与 FRR

### 2.1 评价指标定义

```
生物识别系统的四大核心指标：

FAR (False Acceptance Rate) 误识率 → 假阳性
└── 非法用户被误认为合法用户通过
└── "坏人被当成好人放进来"
→ 越低越好（安全）

FRR (False Rejection Rate) 拒真率 → 假阴性
└── 合法用户被误拒绝
└── "好人被当成坏人挡在外面"
→ 越低越好（可用性）

CER (Crossover Error Rate) 交叉错误率
└── FAR = FRR 时的值
→ 越低越好（系统综合性能指标）

FTE (Failure to Enroll) 注册失败率
└── 无法成功采集和注册生物模板的比例
→ 越低越好
```

### 2.2 FAR 与 FRR 的权衡

```
FAR ↔ FRR 的跷跷板关系：

         FAR (安全)
          │
    高    │  ⚠️  严格模式
          │        └── FAR低（安全）
          │            FRR高（用户烦）
          │
          ├──────────── CER ← 最优平衡点
          │
    低    │  ⚠️  宽松模式
          │        └── FAR高（不安全）
          │            FRR低（用户爽）
          │
          └──────────────────→ FRR (可用性)
                    高

不同场景的倾向：
├── 军事/核设施：宁可 FRR 高，也不能 FAR 高（宁可不出不进，不可放错）
├── 手机解锁：宁可 FAR 略高，也不能 FRR 高（用户讨厌反复认证失败）
└── 支付场景：FAR 和 FRR 都要极低（钱和体验都重要）
```

### 2.3 各技术的 CER 对比

| 技术 | CER 近似值 | 说明 |
|------|-----------|------|
| 虹膜识别 | < 0.0001% | 最准确 |
| 视网膜扫描 | < 0.0001% | 极准确但侵入性强 |
| 指纹识别 | 0.1% - 2% | 受手指状态影响大 |
| 3D 人脸识别 | 0.01% - 0.1% | 结构光/ToF 方案 |
| 2D 人脸识别 | 0.1% - 5% | 受光照角度影响大 |
| 声纹识别 | 1% - 5% | 受环境噪音和身体状况影响 |
| 签名动态 | 2% - 5% | 行为特征不够稳定 |

## 三、指纹识别

### 3.1 三种采集技术

```
指纹采集技术对比：

1. 光学识别
├── 原理：光照射手指 → CCD 采集反射图像
├── 优点：成本低、耐用
├── 缺点：受手指干湿影响，易被照片/硅胶指模欺骗
└── 常见于：老式考勤机

2. 电容识别（硅芯片）
├── 原理：手指脊谷与传感器电容板距离不同 → 电容值不同
├── 优点：精度高，活体检测能力强，体积小
├── 缺点：成本较高，静电放电风险
└── 常见于：智能手机、笔记本

3. 超声波识别
├── 原理：超声波发射 → 手指脊谷反射差异 → 3D 指纹模型
├── 优点：不怕污渍、水渍，可穿透玻璃/屏幕，防假体
├── 缺点：成本最高，识别稍慢
└── 常见于：高端手机(屏下超声波)
```

### 3.2 指纹识别的安全挑战

```
指纹识别的安全漏洞：

1. 复制攻击 (Spoofing)
├── 硅胶指模
├── 明胶指模（可用3D打印制造）
├── 木胶指模（家居材料）
└── 照片（对光学式）

2. 残留指纹提取
├── 门把手、玻璃杯、手机屏幕
├── 用指纹粉 + 胶带提取
└── 转印到指纹膜上

3. 数据库泄露
├── 指纹模板被黑客窃取
├── → 终身无法更换！

防御措施：
├── 活体检测（电容→脉搏，超声波→真皮层）
├── 多模态：指纹 + 人脸/虹膜
├── 指纹模板加密存储（Secure Enclave/TEE）
└── Match-on-Chip（指纹模板永不离开芯片）
```

## 四、人脸识别

### 4.1 从 2D 到 3D

```
人脸识别技术的三代进化：

第1代：2D 照片对比
├── 对比照片平面特征（眼距、鼻梁等）
├── 问题：照片/视频可欺骗（Print Attack）
└── 已基本淘汰

第2代：2D + 活体检测
├── 要求用户配合动作：眨眼、点头、张嘴
├── 检测 3D 深度信息（光影分析）
├── 问题：高清视频回放仍可欺骗
└── 手机 App 常见（支付宝、微信刷脸）

第3代：3D 结构光/ToF + AI
├── 投射数万个不可见光点 → 构建 3D 面部深度图
├── 红外摄像头 → 不受环境光照影响
├── AI 神经网络 + 专用芯片
├── FAR 低至百万分之一
└── Apple Face ID 是典型代表
```

### 4.2 人脸识别的攻击与防御

```
已知的攻击方式：

Print Attack（照片攻击）
├── 打印高清照片 → 2D 系统被欺骗
└── 防御：活体检测 → 要求动作交互

Replay Attack（视频回放攻击）
├── 播放预录的高清视频
└── 防御：3D 深度检测 → 视频是平面的

3D Mask Attack（3D 面具攻击）
├── 高精度 3D 打印面部模具
├── 成本高（几万到几十万美元）
└── 防御：红外活体 → 真皮层反射 vs 材料反射

Deepfake Attack（深度伪造攻击）
├── AI 生成实时换脸视频
├── 可通过视频通话欺骗远程人脸认证
└── 防御：微表情分析、光照异常检测、AI 真假判别
```

### 4.3 隐私与伦理考量

```
人脸识别的监管现状：

中国：
├── 《个人信息保护法》→ 人脸信息属于敏感个人信息
├── 公共场所安装人脸采集设备需设置显著标识
├── 收集人脸信息需取得单独同意
└── 目的限于维护公共安全

欧盟 (GDPR)：
├── 生物识别数据属于特殊类别数据
├── 原则上禁止处理，除非有明确例外
├── 公共空间实时人脸识别被严格限制
└── 各国细则差异大（法国严格，英国相对宽松）
```

## 五、虹膜与视网膜识别

### 5.1 虹膜 vs 视网膜

```
虹膜识别 (Iris Recognition)
├── 虹膜 = 瞳孔周围的有色环形区域
├── 特征：条理、斑点、冠状、隐窝等
├── 每个人独一无二（包括双胞胎！）
├── 约 200+ 个独立特征点（指纹约 40-60 个）
├── 红外摄像头采集（非侵入式，像拍照）
├── FAR 极低：约 1/1,200,000
└── 应用：机场通关、国防设施、银行金库

视网膜扫描 (Retinal Scan)
├── 视网膜 = 眼球后壁的血管模式
├── 特征：毛细血管的网络结构
├── 需要用户贴近设备，注视特定光源
├── 侵入性强，用户接受度低
├── 精度极高但实际部署少
└── 主要限于极高安全场景
```

### 5.2 虹膜识别的优势与局限

```
优势：
├── ✅ 极高的唯一性（超越指纹）
├── ✅ 非接触式采集（卫生）
├── ✅ 难以伪造（需要活体眼球的瞳孔反射）
├── ✅ 长期稳定（不随年龄大幅变化）
└── ✅ 戴眼镜/隐形眼镜不影响

局限：
├── ❌ 设备成本高（红外专用摄像头）
├── ❌ 采集距离有限（通常 10-50cm）
├── ❌ 某些眼部疾病可能影响识别
├── ❌ 用户接受度中等（部分人觉得"扫眼睛"不舒服）
└── ❌ 强光环境下可能受影响
```

## 六、声纹与行为特征识别

### 6.1 声纹识别

```
声纹识别（Voice/Speaker Recognition）：
识别的是"谁在说话"，而非"说了什么"

声纹提取的声学特征：
├── 声道形状（物理特征，像指纹）
├── 发声习惯（行为特征，后天习得）
├── 语速、节奏模式
├── 音调、共振峰
└── 发音清晰度

两类模式：
├── 文本相关 (Text-Dependent)：用户需说出特定短语
└── 文本无关 (Text-Independent)：任意说话内容

安全挑战：
├── 录音重放攻击
├── AI 语音合成攻击 (Deepfake Audio)
├── 环境噪音干扰
├── 感冒/喉咙痛 → 声音变化 → FRR 升高
└── 防御：Challenge-Response（随机短语）+ 活体检测
```

### 6.2 行为特征识别

```
行为生物特征的核心特征：

1. 击键动力学 (Keystroke Dynamics)
   ├── 按键间隔时间
   ├── 按键持续时间 (Dwell Time)
   ├── 打字节奏模式
   └── 可以持续验证（不是登录时验证一次）

2. 鼠标/触控行为
   ├── 光标移动轨迹
   ├── 点击方式（位置、力度、间隔）
   └── 滑动速度和加速度

3. 步态识别
   ├── 走路姿态（通过视频或地面传感器）
   ├── 每一步的幅度、频率、角度
   └── 远距离识别（适合公共安防）

优势：
├── 持续认证（不是"一次性"）：登录后仍可检测异常
├── 无感知：不需要用户特意配合
└── 难以模仿：行为模式是长期养成习惯

局限：
├── FRR 较高：行为特征受情绪、疲劳、受伤影响
├── 不能单独使用：作为辅助持续认证手段
└── 隐私争议：用户被持续监控
```

## 七、生物特征的安全保护

### 7.1 生物模板的安全存储

```
生物模板的存储方案：

方案一：本地存储 + 安全硬件
├── Apple Secure Enclave / Android TEE
├── 生物特征不离开设备
├── 不传云端 → 无集中泄露风险
└── 最佳实践 ✅

方案二：加密存储
├── 生物模板经 AES 加密后存储
├── 解密密钥与生物传感器绑定(Hardware-Bound)
└── 优于明文存储

方案三：可撤销生物模板 (Cancelable Biometrics)
├── 对生物特征应用可逆变换
├── 存储变换后的模板，而非原始生物特征
├── 一旦泄露 → 更改变换函数 → 生成新模板
└── 类似"生物特征的密码" ← 研究前沿

禁止的做法：
├── ❌ 将生物特征明文存储在服务器
├── ❌ 将原始指纹/人脸图像跨网络传输
└── ❌ 使用不安全的硬件存放密钥
```

### 7.2 Match-on-Device 原则

```
Match-on-Device (MOD) / Match-on-Chip：

传统模式（不安全）：
应用程序 → 采集指纹 → 传到服务器比对
         └── 指纹传输中可能截获

MOD 模式（安全）：
应用程序 → 请求认证
    ↓
安全芯片 → 采集指纹 → 本地比对 → 只返回 Yes/No
    └── 生物特征永不离开芯片！

密钥：私钥仅在安全芯片中，由生物特征"解锁"
原则：生物特征不应是"密码"（可传输的字符串）
     而应是"钥匙"（解锁本地安全存储的工具）
```

### 7.3 生物特征的法律保护

```
生物特征信息的法律定位：

《个人信息保护法》→ 第28条：生物识别信息 = 敏感个人信息

处理敏感个人信息的条件：
├── 必须具有特定的目的和充分的必要性
├── 必须取得单独同意
├── 必须告知处理敏感个人信息的必要性
├── 必须进行个人信息保护影响评估 (PIA)
└── 不满 14 岁未成年人的生物信息需要监护人同意

企业实践要点：
├── 明确告知生物信息收集目的
├── 提供非生物认证替代方案（不能强制！）
├── 生物特征本地化存储优先
└── 员工离职后立即删除生物信息
```

## 八、CISP 考试要点速查

### 核心指标速查

| 指标 | 全称 | 含义 | 目标 |
|------|------|------|------|
| FAR | False Acceptance Rate | 坏人被放进来 | 越低越好 |
| FRR | False Rejection Rate | 好人被挡掉 | 越低越好 |
| CER | Crossover Error Rate | FAR=FRR的交点 | 越低越好 |
| FTE | Failure to Enroll | 注册失败 | 越低越好 |

### 技术对比速查

| 技术 | CER | 侵入性 | 可伪造性 | 适用场景 |
|------|-----|--------|---------|----------|
| 虹膜 | 极低 | 低 | 极低 | 国防/高安全 |
| 3D人脸 | 很低 | 无 | 低 | 手机/支付 |
| 指纹 | 低 | 低 | 中 | 手机/考勤 |
| 声纹 | 中 | 无 | 高 | 电话客服 |
| 签名 | 高 | 无 | 中 | 金融柜台 |

### 常见考试陷阱

1. ~~FAR 越低越好，FRR 无关紧要~~ → FAR 和 FRR 是跷跷板，需要根据场景平衡
2. ~~指纹可以被完全伪造~~ → 现代电容/超声波传感器具有活体检测能力，但仍有风险
3. ~~生物特征可以像密码一样更换~~ → 生物特征**终身无法更换**，泄露后果严重
4. ~~Match-on-Server 更安全~~ → Match-on-Device 更安全（生物特征不离开本地）
