## 一、数据安全法的核心框架

### 1.1 立法定位

```
《数据安全法》的定位：
◉ 数据安全领域的基础性法律
◉ 建立了数据安全管理制度体系
◉ 不是隐私法 — 关注的是数据"安全"，而非数据"隐私"
◉ 适用于所有数据类型 — 包括个人数据和非个人数据
```

### 1.2 核心制度体系

```
数据安全法的五大核心制度：

一、数据分类分级制度 (第21条)
├── 国家层面：重要数据目录（国家核心数据）
├── 行业层面：行业数据分类分级指南
└── 组织层面：组织内部数据分类分级

二、数据安全风险评估制度 (第22条)
├── 重要数据处理者定期开展评估
├── 将评估报告报送主管部门
└── 类似等保的"安全评估"机制

三、数据安全审查制度 (第24条)
└── 国家数据安全工作协调机制 → 数据安全审查

四、数据出口管制与对等措施 (第25条)
└── 属于管制物项的数据 → 依法出口管制

五、数据跨境传输管理 (第36条)
├── 重要数据出境 → 安全评估
└── 境外司法调取数据 → 审批
```

### 1.3 数据处理活动全生命周期管控

```
数据安全法管哪几个环节？

收集 → 存储 → 使用 → 加工 → 传输 → 提供 → 公开

每个环节的安全义务：
├── 收集：合法、正当、必要
├── 存储：分类分级存放、加密、备份
├── 使用：权限管控、最小必要、审计留痕
├── 加工：脱敏后再利用
├── 传输：加密传输、完整性校验
├── 提供：安全评估、合同约束、脱敏
└── 公开：去标识化处理
```

## 二、数据分类分级深度解析

### 2.1 三级分类体系

```
核心数据 (CISP 常考的"最高级别")
├── 定义：关系国家安全、国民经济命脉、重大公共利益
├── 特征：一旦破坏 → 国家安全遭受特别严重损害
├── 管控：最高安全级别，最严格保护措施
└── 示例：国家机密级数据、国家核心基础设施运行数据

重要数据
├── 定义：一旦破坏 → 危害国家安全、经济运行、社会稳定
├── 特征：跨部门影响，可能引发公共安全事件
├── 管控：重点保护、定期安全评估
└── 示例：基因数据、重要地理信息、重要行业关键数据

一般数据
├── 定义：核心数据和重要数据之外的数据
├── 特征：主要影响企业自身或个体
├── 管控：常规安全管理措施
└── 示例：企业内部非敏感数据、已公开数据
```

### 2.2 "重要数据"的具体范围

根据《数据安全法》和相关标准，重要数据包括但不限于：

| 领域 | 重要数据示例 |
|------|-------------|
| **遗传资源** | 人口基因数据、遗传资源信息 |
| **地理信息** | 优于国家公开发布精度的地理信息 |
| **经济运行** | 未公布的宏观经济数据 |
| **重要行业** | 能源、交通、水利、金融等关键行业运行数据 |
| **食品药品** | 药品临床数据、食品安全监测数据 |
| **重要设备** | 核心技术参数、网络安全漏洞信息 |

### 2.3 分类分级实操流程

```
组织数据分类分级步骤：

步骤一：数据资产摸底
├── 梳理所有数据资产
├── 建立数据资产清单
└── 确定数据拥有者

步骤二：数据分类
├── 按业务属性分类（客户数据、员工数据、业务数据...）
├── 按数据性质分类（结构化、非结构化...）
└── 按管理责任分类（部门A、部门B...）

步骤三：数据分级
├── 按敏感度分级（一般 → 重要 → 核心）
├── 按影响程度定级（个人权益 → 公共利益 → 国家安全）
└── 按法规要求对标（数据安全法 + 行业规范）

步骤四：分类分级结果落地
├── 数据标记/标签
├── 不同级别 → 不同保护策略
└── 分类分级结果动态更新
```

## 三、个人信息保护法核心机制

### 3.1 个人信息的法律定义

```
《个人信息保护法》第4条：

个人信息 = 以电子或其他方式记录的
    与已识别或可识别的自然人有关的各种信息
    （不包括匿名化处理后的信息）

关键点：
├── "已识别"：直接关联具体个人（姓名、身份证号）
├── "可识别"：结合其他信息能关联到个人（IP + 行为）
└── "匿名化"后不再属于个人信息 ← 法律意义上的豁免
```

### 3.2 七项个人权利（考点密集区）

```
个人在个人信息处理活动中的七项权利：

1. 知情权 → 知道谁在处理、为什么处理 ← "告知"义务的对应面
2. 决定权 → 有权限制或拒绝处理
3. 查阅复制权 → 查阅复制自己的信息 ← 第45条
4. 更正权 → 信息有误时要求改正 ← 第46条
5. 删除权 → 在特定情形下要求删除 ← 第47条（"被遗忘权"）
6. 可携带权 → 转移到指定处理者 ← 第45条
7. 解释说明权 → 要求说明处理规则

删除权的触发条件：
├── 处理目的已实现/无法实现/不再必要
├── 处理者停止提供产品/服务
├── 保存期限已满
├── 个人撤回同意
└── 处理者违法或违约处理
```

### 3.3 敏感个人信息保护

**定义**（第28条）：一旦泄露或非法使用，容易导致自然人的人格尊严受到侵害或人身、财产安全受到危害的个人信息。

```
特殊保护规则：

1. 处理条件更严格
   ├── 必须具有特定目的和充分必要性
   └── 不能"顺便"处理敏感信息

2. 单独同意（不得与其他同意捆绑）
   ├── 不能隐藏在冗长的隐私政策中
   └── 需要独立的、明确的确认动作

3. 额外的告知义务
   └── 必须告知"为什么要处理你的敏感信息（必要性）"

4. 必须进行 PIA
   └── 处理敏感信息必须进行个人信息保护影响评估

5. 未成年人特殊保护
   └── 不满 14 周岁的个人信息 = 敏感个人信息
```

### 3.4 自动化决策规则

```
《个人信息保护法》第24条：

自动化决策（包括用户画像、算法推荐）的合规要求：

├── 透明性：保证决策的透明度和结果公平公正
├── 禁止歧视：不得实行不合理的差别待遇
├── 选择权：同时提供不针对个人特征的选项
│           或提供便捷的拒绝方式
├── 重大影响：做出对个人权益有重大影响的决定时
│   个人有权要求说明，并有权拒绝仅通过自动化决策做出决定
└── 商业营销：通过自动化决策进行营销的
    须同时提供不针对个人特征的选项
```

## 四、数据处理活动的合规要求

### 4.1 "告知-同意"机制详解

```
告知的内容要求（第17条）：

┌──────────────────────────────────────┐
│ · 个人信息处理者的名称和联系方式      │
│ · 处理目的、处理方式                  │
│ · 处理的个人信息种类、保存期限        │
│ · 个人行使权利的方式和程序            │
│ · 其他应当告知的事项                  │
└──────────────────────────────────────┘

同意的有效条件：
├── 自愿：没有强迫或不当影响
├── 明确：明确的意思表示
├── 知情：在充分告知的前提下
└── 可撤回：撤回同意不影响此前已进行的处理

同意的特殊形式：
├── 单独同意：敏感信息处理、跨境提供、公开等情形
├── 书面同意：法律法规规定的情形
└── 重新同意：处理目的、方式、种类变更时
```

### 4.2 无需同意的例外情形（第13条）

```
以下情况无需取得个人同意：

├── 为订立、履行个人作为一方当事人的合同所必需
├── 为履行法定职责或法定义务所必需
├── 为应对突发公共卫生事件所必需
├── 为公共利益实施新闻报道、舆论监督等
├── 在合理范围内处理个人自行公开的信息
└── 法律、行政法规规定的其他情形
```

### 4.3 个人信息保护影响评估 (PIA)

```
必须进行 PIA 的五种情形（第55条）：

1. 处理敏感个人信息
2. 利用个人信息进行自动化决策
3. 委托处理、对外提供或公开个人信息
4. 向境外提供个人信息
5. 其他对个人权益有重大影响的活动

PIA 报告的三项核心内容：
├── 处理目的和处理方式
├── 对个人权益的影响和安全风险
└── 所采取的保护措施和有效性

保存期限：PIA 报告和处理情况记录至少保存 3 年
```

## 五、数据安全技术措施体系

### 5.1 技术防护全景图

```
数据安全技术措施金字塔：

          ┌──────────────┐
          │   加密技术    │  ← 最核心
          │ (存储/传输加密)│
          ├──────────────┤
          │   脱敏技术    │  ← 数据共享
          ├──────────────┤
          │  访问控制    │  ← 权限管理
          ├──────────────┤
          │  审计日志    │  ← 可追溯
          ├──────────────┤
          │  DLP 防泄漏   │  ← 监控防护
          ├──────────────┤
          │  备份恢复    │  ← 底线保障
          └──────────────┘
```

### 5.2 数据存储保护

| 保护措施 | 技术方案 | 适用场景 |
|----------|----------|----------|
| **传输加密** | TLS 1.3、IPSec VPN | 网络传输、API 调用 |
| **存储加密** | AES-256、SM4（国密） | 数据库、文件系统、备份 |
| **应用层加密** | 字段级加密、列级加密 | 敏感字段（身份证、手机号） |
| **数据库加密** | TDE（透明数据加密） | SQL Server、Oracle、MySQL |
| **密钥管理** | HSM、KMS | 加密密钥的安全存储和使用 |

### 5.3 访问控制矩阵

| 数据级别 | 访问控制策略 |
|----------|-------------|
| **核心数据** | 双人审批 + 审批留痕 + 最小访问窗口 + 禁止导出 |
| **重要数据** | 角色审批 + 操作审计 + 脱敏显示 + 导出审批 |
| **一般数据** | 岗位授权 + 操作日志 + 按需访问 |

### 5.4 DLP 数据防泄漏

```
DLP 的三道防线：

网络层 DLP
├── 监控 HTTP/HTTPS/FTP/SMTP 流量
├── 检测向外传输的敏感数据
└── 阻断或告警

存储层 DLP
├── 扫描文件服务器和数据库
├── 发现未授权的敏感数据存储
└── 报告和分类

终端层 DLP
├── USB 端口管控
├── 剪贴板/截屏控制
└── 应用程序控制
```

## 六、个人信息脱敏技术实践

### 6.1 脱敏技术分类

| 技术 | 方法 | 示例 | 可逆性 |
|------|------|------|--------|
| **掩码** (Masking) | 部分字符替换为* | 138****5678 | ❌ 不可逆 |
| **替换** (Substitution) | 真实值→虚假值 | 张三→用户001 | ✅ 可逆（查映射表） |
| **泛化** (Generalization) | 精确值→范围值 | 35岁→30-40岁 | ❌ 不可逆 |
| **加密** (Encryption) | 密文存储，授权解密 | AES-256 | ✅ 可逆（有密钥） |
| **哈希** (Hashing) | 单向散列 | SHA-256 | ❌ 不可逆 |
| **扰乱** (Shuffling) | 同列值重排 | 工资横向打乱 | ⚠️ 统计特性保留 |
| **差分隐私** (Differential Privacy) | 加噪声 | 查询结果+随机偏差 | ❌ |

### 6.2 数据脱敏规则

```
脱敏黄金法则：

1. ❌ 不可逆优先：能用掩码、哈希的不用加密（减少密钥管理风险）
2. ✅ 业务需求导向：需要还原的场景用可逆脱敏（加密/替换表）
3. 🎯 字段级控制：同一张表的字段按敏感度差异化脱敏
4. 🔗 关联风险防范：不同表中脱敏后数据可能被关联还原
5. 📊 统计可用性：泛化和扰乱后保持统计特性
```

## 七、跨境数据流动管理

### 7.1 中国数据出境规则全景

```
数据出境的三种合法路径：

路径一：数据出境安全评估（强制申报情形）
├── 重要数据出境
├── CII 运营者收集的个人信息出境
├── 处理 100 万人以上个人信息 → 个人信息出境
└── 1 年内累计出境 > 10 万人个人信息 or > 1 万人敏感信息

          →
路径二：标准合同备案
├── 非上述强制情形
├── 与境外接收方签订《个人信息出境标准合同》
└── 在省级网信办备案

          →
路径三：个人信息保护认证
└── 通过国家网信部门认可的专业机构认证且签订认证协议
```

### 7.2 境外执法调取数据

```
《数据安全法》第36条：
非经中华人民共和国主管机关批准，境内的组织、个人
不得向外国司法或者执法机构提供存储于境内的数据。

《个人信息保护法》第41条：
非经主管机关批准，不得向外国司法/执法机构提供个人信息。

→ 两个法律互相呼应，堵住"长臂管辖"的缺口
→ 未经批准擅自向外国政府提供数据 → 违法
```

## 八、CISP 考试要点速查

### 核心数字

| 数字 | 含义 |
|------|------|
| 3级 | 数据分类分级：一般→重要→核心 |
| 7项 | 个人信息权利数量 |
| 14岁 | 不满14周岁=敏感个人信息 |
| 100万 | 处理100万+个人信息→出境需安全评估 |
| 3年 | PIA 报告最低保存年限 |
| 5000万/5% | 个保法最高处罚 |

### 关键定义辨析

| 概念 | 核心特征 |
|------|----------|
| 一般数据 | 不影响国家安全 |
| 重要数据 | 影响国家安全和经济运行 |
| 核心数据 | 关系国家命脉、最高级别保护 |
| 敏感个人信息 | 易导致人格尊严侵害（需单独同意） |
| 匿名化 | 处理后无法识别个人 → 不再属于个人信息 |
| 去标识化 | 仍可能通过其他信息恢复识别 → 仍属个人信息 |

### 常见考试陷阱

1. ~~数据只有两级分类~~ → **三级**（一般/重要/核心）
2. ~~所有数据都可以自由出境~~ → 重要数据和 CII 个人信息须**安全评估**
3. ~~敏感信息只需要告知就可以处理~~ → 需要**单独同意 + 告知必要性**
4. ~~PIA 保存 2 年~~ → 至少 **3 年**
5. ~~工作单位名称属于敏感个人信息~~ → 不属于敏感信息（属于普通个人信息）

- [ ] 记住个人七项权利

- [ ] 知道数据出境的三条路径

> 💡 **本章小结**：数据安全法和个人信息保护法是 CISP 法规考试的两大新热点。核心记忆点：数据分三级（一般/重要/核心）、个人信息有七项权利、敏感信息需单独同意、数据出境有三条路径。技术层面要掌握脱敏技术分类（掩码/替换/泛化/加密/哈希）及其可逆性判断。
