> **📘 文档定位**：CISP 考试必考重点 | 难度：⭐⭐⭐ | 预计阅读：25 分钟

> 网络安全等级保护制度是中国网络安全的一项基本制度，也是《网络安全法》第21条明确的法律义务。从等保 1.0 到等保 2.0，保护对象从信息系统扩展到云计算、物联网、工控系统等新兴领域。本章将深入解析等级保护的核心要求、定级流程和实施路径。

## 一、什么是等级保护

### 1.1 基本概念

```
网络安全等级保护 =
国家对网络安全保护对象按其重要程度进行分级
    → 确定不同级别的安全保护能力要求
    → 运营者按要求建设、落实、测评
    → 监管部门监督检查
```

**法律依据**：《网络安全法》第21条——"国家实行网络安全等级保护制度"。

### 1.2 等保的三个核心维度

```
等级保护制度 = 标准体系 + 测评体系 + 监管体系

标准体系 (怎么做)
├── GB/T 22239-2019  《信息安全技术 网络安全等级保护基本要求》
├── GB/T 28448-2019  《信息安全技术 网络安全等级保护测评要求》
├── GB/T 25070-2019  《信息安全技术 网络安全等级保护安全设计技术要求》
└── GB/T 22240-2020  《信息安全技术 网络安全等级保护定级指南》

测评体系 (怎么验证)
├── 定级 → 备案 → 建设整改 → 等级测评 → 监督检查
└── 三级及以上系统每年至少测评一次 ← 考点！

监管体系 (怎么监管)
├── 公安机关备案
├── 网信部门统筹协调
└── 行业主管部门监督管理
```

### 1.3 从等保 1.0 到等保 2.0

| 维度 | 等保 1.0 (2008-2019) | 等保 2.0 (2019 至今) |
|------|---------------------|---------------------|
| **保护对象** | 信息系统 | 信息系统 + 云计算 + 物联网 + 工控 + 移动互联 + 大数据 |
| **标准名称** | "信息安全等级保护" | **"网络安全等级保护"** |
| **安全要求** | 技术为主 | **技术 + 管理** 并重 |
| **防护理念** | 被动防御 | 主动防御、**可信计算** |
| **覆盖阶段** | 建设阶段 | 全生命周期（规划→设计→建设→运维→废弃） |
| **核心标准** | GB 17859-1999 | GB/T 22239-2019 |

> **关键变化**：等保 2.0 新增了对云计算、物联网、移动互联、工控系统、大数据等**新型应用**的安全扩展要求。

## 二、五个安全保护等级

### 2.1 五级分级体系

| 等级 | 定级名称 | 保护能力 | 侵害客体 | 监管强度 | 测评周期 |
|------|----------|----------|----------|----------|----------|
| **第1级** | 自主保护级 | 基本保护 | 公民、法人合法权益 | 备案 | 无强制 |
| **第2级** | 指导保护级 | 较高保护 | 社会秩序、公共利益 | 备案 + 指导 | **建议**2年 |
| **第3级** | 监督保护级 | 完备保护 | **国家安全** | 监督 + 检查 | **每年至少1次** |
| **第4级** | 强制保护级 | 最高保护 | 国家安全(**严重**) | 强制监督 | **每半年至少1次** |
| **第5级** | 专控保护级 | 特别保护 | 国家安全(**特别严重**) | 专门管控 | 专门管控 |

### 2.2 侵害客体与损害程度矩阵

```
定级由两个维度决定：
    × 侵害客体 (谁受损)
    × 侵害程度 (多严重)

侵害客体：
├── 公民、法人和其他组织的合法权益
├── 社会秩序和公共利益
└── 国家安全

侵害程度：
├── 一般损害
├── 严重损害
└── 特别严重损害

定级矩阵：

    侵害程度 →  一般损害    严重损害    特别严重损害
侵害客体 ↓      ─────────────────────────────
公民合法权益       1级        2级          2级
社会秩序/公共利益    2级        3级          4级
国家安全            3级        4级          5级
```

### 2.3 各等级典型应用举例

| 等级 | 典型系统 |
|------|----------|
| 1级 | 小微企业官网、内部 OA（无敏感数据） |
| 2级 | 一般企业网站、电商平台（非关键行业） |
| **3级** | **政务系统、银行核心系统、大型互联网平台**、CISP 考试最常见！ |
| 4级 | 国家电网调度系统、央行清算系统 |
| 5级 | 核设施控制系统、国防通信系统 |

> **CISP 相关系统通常要求三级保护**：实际工作中大多数安全管理系统要求达到等保三级。

## 三、等保 2.0 的核心变化

### 3.1 标准名称变化

```
旧：GB/T 22239-2008 《信息安全技术 信息系统安全等级保护基本要求》
新：GB/T 22239-2019 《信息安全技术 网络安全等级保护基本要求》
                        ^^^^^^^^              ^^^^^^^^
                   加了"网络"二字       "信息系统"→"网络安全"
```

### 3.2 通用安全要求 + 安全扩展要求

```
GB/T 22239-2019 的结构：

通用安全要求（所有保护对象适用）
├── 安全物理环境     ← 等保1.0 中的物理安全
├── 安全通信网络     ← 等保1.0 中的网络安全
├── 安全区域边界     ← 等保1.0 中的网络安全
├── 安全计算环境     ← 等保1.0 中的主机+应用+数据安全
├── 安全管理中心     ← 等保2.0 新增！
└── 安全管理制度/机构/人员/建设/运维 ← 管理要求

安全扩展要求（针对特定应用场景）：
├── 云计算安全扩展要求
├── 移动互联安全扩展要求
├── 物联网安全扩展要求
└── 工业控制系统安全扩展要求
```

### 3.3 可信计算技术的引入

等保 2.0 引入了基于**可信计算**的主动防御理念：

```
可信验证的要求链：
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 可信根(硬件)│ → │ 可信平台控制│ → │ 可信软件基│
│ TPM/TCM   │    │   模块     │    │   (TSB)   │
└──────────┘    └──────────┘    └──────────┘

三级要求：关键设备进行可信验证
四级要求：全部设备进行可信验证 + 动态关联感知
```

## 四、定级流程详解

### 4.1 定级的六个步骤

```
步骤一：确定定级对象
├── 信息系统
├── 云计算平台
├── 物联网系统
├── 工控系统
└── 移动互联应用

步骤二：初步确定等级
├── 确定业务信息安全等级 (S)
│   └── 考虑机密性被破坏后的影响
├── 确定系统服务安全等级 (A)
│   └── 考虑可用性被破坏后的影响
└── 取 L = MAX(S, A)

步骤三：专家评审
步骤四：主管部门审批
步骤五：公安机关备案
步骤六：级别调整（如需要）
```

### 4.2 定级要素

```
业务信息安全等级 (S)
├── 业务信息被破坏影响的客体
├── 业务信息被破坏影响的程度
└── S 取值：1, 2, 3, 4, 5

系统服务安全等级 (A)
├── 系统服务被破坏影响的客体
├── 系统服务被破坏影响的程度
└── A 取值：1, 2, 3, 4, 5

安全保护等级 L = MAX(S, A)
```

**定级示例**：
```
某银行核心交易系统：
- 数据泄露影响国家安全 → S = 3（严重损害国家安全）
- 服务中断影响国家安全 → A = 3（严重损害国家安全）
- L = MAX(3, 3) = 3 级

某小微企业内部OA：
- 数据泄露影响企业合法权益 → S = 1（一般损害企业）
- 服务中断影响企业合法权益 → A = 1（一般损害企业）
- L = MAX(1, 1) = 1 级
```

## 五、"一个中心、三重防护"技术体系

### 5.1 整体架构图

```
         等保 2.0 技术防护架构

              ┌─────────────┐
              │  安全管理中心 │  ← "一个中心"
              │  · 系统管理  │
              │  · 安全管理  │
              │  · 审计管理  │
              └──────┬──────┘
                     │
        ┌────────────┼────────────┐
        │            │            │
   ┌────▼────┐  ┌────▼────┐  ┌────▼────┐
   │安全通信网络│  │安全区域边界│  │安全计算环境│
   │(网络通信) │  │(网络边界) │  │(主机/应用) │
   └─────────┘  └─────────┘  └─────────┘
        "三重防护"
```

### 5.2 安全管理中心（三级要求）

```
安全管理中心 → 集中管控

系统管理
├── 对系统管理员进行身份鉴别
├── 管理员操作进行权限控制
└── 通过管理口进行运维管理

安全管理
├── 安全策略集中管理
├── 安全事件集中告警
└── 安全产品集中管控

审计管理
├── 审计策略集中管理
├── 审计记录集中存储和分析
└── 审计日志不可修改、不可删除
```

### 5.3 三重防护三级核心要求

**安全通信网络**：
```
通信传输
├── 采用密码技术保证传输机密性和完整性 ← 加密！
└── 采用可信验证技术

通信设备
├── 网络设备冗余部署
└── 带宽满足业务高峰需求

网络架构
├── 重要网络区域与非重要网络区域隔离
├── 提供关键设备硬件冗余
└── 保证网络带宽
```

**安全区域边界**：
```
边界防护
├── 保证跨越边界的访问和数据流通过边界设备
└── 非授权无线设备能够被发现和阻断

访问控制
├── 网络边界/区域之间 → 访问控制策略
└── 默认除允许以外的所有通信 → 拒绝

入侵防范
├── 主动阻断来自外部/内部的网络攻击
└── 检测新型网络攻击行为 ← 等保2.0新增！

安全审计
└── 边界设备的审计记录发送到审计中心

恶意代码防范
└── 在关键网络节点检测和清除恶意代码
```

**安全计算环境**：
```
身份鉴别
├── 双因素认证（口令 + 令牌/生物/证书）
├── 登录失败处理（锁定/告警）
└── 远程管理加密

访问控制
├── 三权分立：系统管理员/安全管理员/审计管理员
├── 主体访问客体 → 最小权限原则
└── 默认拒绝

安全审计
├── 审计范围全覆盖
├── 审计记录保护（不可修改）
└── 审计进程保护（不可中断）

入侵防范
├── 最小安装原则（关闭无用端口和服务）
├── 漏洞扫描并及时修复
└── 检测对重要节点的入侵行为

恶意代码防范
├── 安装防恶意代码软件
└── 定期更新和查杀

可信验证
└── 基于可信根对系统引导、内核、应用进行度量验证

数据完整性/机密性
├── 重要数据传输和存储完整性校验
└── 重要数据存储加密

数据备份恢复
├── 提供异地实时备份功能
└── 提供重要数据处理系统热冗余
```

## 六、等保定级因素与分值计算

### 6.1 业务信息安全等级计算

```
S = 业务信息安全等级取值

依据：业务信息被破坏时对客体的侵害程度

考虑因素：
├── 信息的敏感程度（公开/内部/秘密/机密/绝密）
├── 信息的重要性（关键业务/辅助业务）
├── 数据量级（涉及的用户/记录数量）
└── 泄密后果的扩散性

分值映射：
├── 侵害权益：S=1
├── 侵害社会秩序（一般）：S=2
├── 侵害社会秩序（严重）：S=3
└── 侵害国家安全：S=3/4/5
```

### 6.2 系统服务安全等级计算

```
A = 系统服务安全等级取值

依据：系统服务被破坏时对客体的侵害程度

考虑因素：
├── 服务范围（全国/区域/局部）
├── 用户规模（千万/百万/万）
├── 服务连续性要求（7×24/5×8）
├── 恢复时间目标 (RTO)
└── 实时性要求（实时/准实时/批量）

分值映射：
├── 侵害权益：A=1
├── 侵害社会秩序/公共利益（一般）：A=2
├── 侵害社会秩序/公共利益（严重）：A=3
└── 侵害国家安全：A=3/4/5
```

## 七、实施路径与测评标准

### 7.1 等保 2.0 实施五步法

```
第一步：定级         第二步：备案         第三步：建设整改
┌──────────┐       ┌──────────┐       ┌──────────┐
│·确定定级对象│  →    │·公安机关备案│  →    │·差距分析   │
│·初步定级   │       │·提交定级报告│       │·安全建设   │
│·专家评审   │       │·审核和确认  │       │·整改实施   │
└──────────┘       └──────────┘       └──────────┘
                                                ↓
第四步：等级测评                                  │
┌──────────┐                                     │
│·测评机构测评│  ←──────────────────────────────┘
│·获取测评报告│      第五步：监督检查
│·整改复核   │      ┌──────────┐
└──────────┘      │·公安机关检查│
                  │·行业主管部门│
                  │·年度检查    │
                  └──────────┘
```

### 7.2 测评结论

| 结论 | 符合度 | 处理方式 |
|------|--------|----------|
| **符合** | ≥85% 且无高风险项 | 通过 |
| **基本符合** | ≥65% 且无高风险项 | 整改后通过 |
| **不符合** | <65% 或有高风险项 | 重新建设整改后测评 |

### 7.3 测评频次

```
二级系统：建议每两年测评一次
三级系统：每年至少测评一次    ← 强制要求
四级系统：每半年至少测评一次   ← 强制要求
五级系统：依据特殊安全需求
```

## 八、CISP 考试要点速查

### 核心数字记忆

| 数字 | 含义 |
|------|------|
| 5 | 保护等级数（1~5级） |
| 6+3 | 等保2.0 技术类要求：物理+通信+边界+计算+管理中心+制度→6大类, 管理类3大类 |
| 85% | 测评"符合"的最低阈值 |
| 65% | 测评"基本符合"的阈值 |
| 1年 | 三级系统强制测评周期 |

### 关键标准号

| 标准号 | 名称 | 用途 |
|--------|------|------|
| GB/T 22239-2019 | 基本要求 | "怎么建设" |
| GB/T 28448-2019 | 测评要求 | "怎么测评" |
| GB/T 25070-2019 | 设计技术要求 | "怎么设计" |
| GB/T 22240-2020 | 定级指南 | "怎么定级" |

### 常见考试陷阱

1. ~~等保分为三级~~ → 分为**五级**（1自主~5专控）
2. ~~等保只保护信息系统~~ → 等保2.0 保护对象扩展至**云、物联网、工控、移动互联**
3. ~~三级系统两年测评一次~~ → 三级系统**每年至少测评一次**
4. ~~等保只需技术措施~~ → 等保2.0 要求**技术 + 管理**并重
5. ~~"一个中心"只做审计~~ → 安全管理中心包含**系统管理+安全管理+审计管理**三方面
