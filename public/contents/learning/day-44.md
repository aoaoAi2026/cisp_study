## 一、防火墙概述与演进

### 1.1 防火墙定义

```
防火墙 = 在不同网络区域之间实施访问控制的设备或系统

核心功能：
  ① 访问控制：根据策略允许/拒绝流量
  ② 网络隔离：划分不同安全区域
  ③ 日志审计：记录所有通过的流量
  ④ NAT转换：隐藏内部网络结构
  ⑤ VPN终结：安全隧道连接点
```

### 1.2 五代防火墙演进

```
┌──────────┬───────────────┬──────────────────────────┐
│   代际    │     技术      │         关键能力          │
├──────────┼───────────────┼──────────────────────────┤
│ 第一代    │ 包过滤 (ACL)  │ 基于IP/端口/协议的允许拒绝 │
│ 第二代    │ 状态检测       │ 记录连接状态，动态规则      │
│ 第三代    │ 应用代理       │ 深度解析应用层协议         │
│ 第四代    │ UTM           │ 集成IDS/IPS/AV/反垃圾邮件  │
│ 第五代    │ NGFW          │ 应用识别+用户身份+威胁情报  │
└──────────┴───────────────┴──────────────────────────┘
```

## 二、包过滤防火墙

### 2.1 工作原理

```
包过滤防火墙检查每个IP包的头部信息：

┌────────────────────────────────────────────┐
│  检查字段：                                  │
│  · 源IP地址                                 │
│  · 目标IP地址                               │
│  · 源端口                                   │
│  · 目标端口                                 │
│  · 协议类型（TCP/UDP/ICMP）                 │
│  · TCP标志位（SYN/ACK/FIN/RST）             │
│  · 网络接口（入/出方向）                     │
└────────────────────────────────────────────┘

ACL处理流程：
  匹配第一条规则 → 执行动作(允许/拒绝) → 停止匹配
  不匹配任何规则 → 执行默认策略

关键设计原则：
  ✅ 默认拒绝 (Default Deny)："没明确允许的就是禁止的"
  ❌ 默认允许："没明确禁止的就是允许的"（不安全！）
```

### 2.2 ACL规则示例

```
Cisco ACL示例：

! 允许内部网络访问Web服务
access-list 101 permit tcp 192.168.1.0 0.0.0.255 any eq 80
access-list 101 permit tcp 192.168.1.0 0.0.0.255 any eq 443

! 允许已建立连接的回程流量
access-list 101 permit tcp any 192.168.1.0 0.0.0.255 established

! 拒绝来自已知恶意IP的流量
access-list 101 deny ip 10.0.0.0 0.255.255.255 any

! 默认拒绝所有
access-list 101 deny ip any any log

! 应用到接口
interface GigabitEthernet0/0
ip access-group 101 in
```

### 2.3 优缺点

| 优点 | 缺点 |
|------|------|
| 速度快（仅检查头部） | 无状态（不知道连接上下文） |
| 实现简单 | 无法识别应用层协议 |
| 资源消耗低 | 容易配置错误 |
| 透明性好 | 不能防御应用层攻击 |

## 三、状态检测防火墙

### 3.1 状态表机制

```
状态检测防火墙维护连接状态表：

连接建立时：
  Client→Server: SYN →
    防火墙记录：(SrcIP, DstIP, SrcPort, DstPort, TCP=SYN_SENT)

  Server→Client: SYN+ACK ←
    防火墙更新：(SrcIP, DstIP, SrcPort, DstPort, TCP=ESTABLISHED)
    
回程流量：
  Server→Client: 数据包 ←
    防火墙查状态表 → "匹配已建立连接" → 直接放行
    （不需要额外规则！）

优势：
  ✅ 自动处理回程流量（减少ACL规则）
  ✅ 防御欺骗攻击（非预期状态的包被丢弃）
  ✅ 防御端口扫描（可以识别异常连接模式）
```

### 3.2 状态检测安全能力

```
额外安全能力：

① TCP序列号检查：
   跟踪每个连接的序列号
   拒绝窗口外的包 → 防TCP劫持

② 协议异常检测：
   识别违反RFC的协议行为
   如：未完成握手的FIN包

③ 连接速率限制：
   同一源IP的连接数/新连接速率
   → 防DoS

④ 应用层协议助手 (ALG)：
   FTP ALG：动态开放数据端口
   SIP ALG：解析SIP报文开放媒体端口
   H.323 ALG：视频会议协议
```

### 3.3 状态检测 vs 包过滤

| 维度 | 包过滤 | 状态检测 |
|------|--------|---------|
| 工作层 | 网络层/传输层 | 传输层 |
| 是否记录状态 | ❌ 无状态 | ✅ 有状态 |
| 回程流量 | 需显式规则 | 状态表自动处理 |
| 防欺骗 | 弱 | ✅ 较强 |
| 性能 | 极快 | 较快 |
| OSI层 | 3-4层 | 3-4层 |

## 四、应用代理防火墙

### 4.1 代理原理

```
应用代理防火墙（应用层网关）结构：

客户端 ←──→ [代理客户端] ←──[安全检查]──→ [代理服务端] ←──→ 真实服务器

特点：
  · 客户端不直接与服务器通信
  · 代理完全解析应用层协议
  · 可以过滤应用层内容（URL、命令、文件类型）

示例：HTTP代理
  ① 客户端发送HTTP GET /admin → 代理
  ② 代理解析HTTP请求 → 检查URL是否允许
  ③ 代理重新构造请求 → 发送给服务器
  ④ 代理检查服务器响应内容 → 过滤敏感信息
  ⑤ 代理返回过滤后的响应给客户端
```

### 4.2 两种代理模式

| 代理类型 | 说明 | 适用场景 |
|----------|------|----------|
| **显式代理** | 客户端需配置代理地址 | 企业Web代理 |
| **透明代理** | 客户端无感知，网络自动转发 | Web内容过滤 |
| **正向代理** | 代理客户端访问外部资源 | 上网行为管理 |
| **反向代理** | 代理外部客户端访问内部服务器 | 服务器负载均衡/WAF |

### 4.3 优缺点

| 优点 | 缺点 |
|------|------|
| 深度协议理解（可过滤内容） | 性能开销大（需两次连接） |
| 隔离客户端和服务器 | 每种协议需要专门的代理 |
| 强大的日志和审计能力 | 不支持所有协议 |
| 应用层攻击检测 | 可能引入延迟 |

## 五、下一代防火墙（NGFW）

### 5.1 NGFW与UTM的区别

```
UTM (统一威胁管理):
  多个安全功能的简单集成（AV+IPS+FW+反垃圾邮件）
  每个功能独立运行
  统一管理界面
  通常是软件层面的集成

NGFW (下一代防火墙/Gartner定义):
  ✅ 标准防火墙功能（状态检测）
  ✅ 集成IPS
  ✅ 应用识别与控制（不分端口）
  ✅ 用户身份感知
  ✅ 集成威胁情报
  ✅ 基于应用和用户的安全策略
  ✅ 单次扫描架构（非多次独立扫描）
```

### 5.2 应用识别技术

```
NGFW不论端口号识别应用：

识别方法（多维度）：
  ① 签名匹配：应用协议特征库
  ② 行为分析：连接模式、包大小分布
  ③ 协议解析：TLS SNI、HTTP Host头
  ④ 统计特征：包间隔、流持续时间
  ⑤ 启发式：已知应用模式的偏差

示例：
  端口80上的流量不一定是HTTP（可能是SSH隧道）
  端口443上的流量不一定是HTTPS（可能是OpenVPN）
  端口8080上的流量可能是QQ、微信等
```

### 5.3 主流NGFW品牌

| 品牌 | 特点 | 测评参考 |
|------|------|---------|
| Palo Alto | App-ID技术领先 | Gartner领导者 |
| Fortinet (FortiGate) | 自研ASIC芯片，性能优秀 | Gartner领导者 |
| Check Point | 以色列安全老牌 | Gartner远见者 |
| Cisco Firepower | 思科生态整合 | Gartner挑战者 |
| 华为USG | 国产化场景 | 国内首选 |
| 深信服AF | 中国NGFW | 国内性价比 |

## 六、防火墙架构设计

### 6.1 安全区域规划

```
经典三层架构：

      Internet (Untrust)
           │
      ┌────▼────┐
      │  防火墙   │
      └─┬──┬──┬─┘
        │  │  │
    ┌───▼──▼──▼───┐
    │    DMZ区域   │  ← Web、Mail、DNS (中等信任)
    ├─────────────┤
    │ 内部服务器区  │  ← DB、App (高安全)
    ├─────────────┤
    │  办公网区域  │  ← 员工终端 (中等信任)
    └─────────────┘

安全策略：
  Untrust → DMZ：仅80,443
  Untrust → 内部：全部拒绝
  DMZ → 内部：特定数据库端口
  内部 → Internet：HTTP/HTTPS（通过代理）
```

### 6.2 高可用设计

```
防火墙HA模式：

① Active-Passive：
  主防火墙工作，备用待机
  故障时自动切换（VRRP/HSRP）
  
② Active-Active：
  两台防火墙同时工作
  流量分担（需会话同步）
  
③ 堆叠/集群：
  多台设备堆叠为逻辑单设备
  统一管理，性能叠加

关键考虑：
  · 会话同步：主备切换时不中断已有连接
  · 配置同步：主备配置一致性
  · 心跳链路：专用链路避免误判
```

## 七、防火墙策略管理

### 7.1 策略优化原则

```
策略优化七原则：

① 最常用规则放前面（减少匹配次数）
② 避免重复规则（合并相近规则）
③ 定期审查未命中规则（可能是无效规则）
④ 拒绝日志要开启（便于审计和排障）
⑤ 变更管理流程（申请→审批→变更→验证）
⑥ 定期策略审计（至少每季度）
⑦ 自动化策略分析（使用Tufin/Algosec等工具）

常见策略错误：
  ❌ any any allow（允许一切）
  ❌ 防火墙放行 ICMP any any（放大攻击利用）
  ❌ 双向allow（内网到外网和外网到内网都放）
  ❌ 过期服务规则（已废弃的服务端口仍开放）
```

### 7.2 默认拒绝原则

```
正确的策略设计：

  ① 最后一条规则：deny all + log
  ② 仅开放业务必需的端口/协议
  ③ 开放前评估：最小权限原则
  ④ 临时规则必须设过期时间

安全基线示例：
  入站：拒绝所有
  出站：仅允许 HTTP(80), HTTPS(443), DNS(53)
  管理：仅允许特定管理IP访问SSH(22)/HTTPS(443)
```

## 八、WAF与数据库防火墙

### 8.1 WAF（Web应用防火墙）

```
WAF特点：
  · 工作在应用层（OSI第7层）
  · 专门保护Web应用
  · 签名检测 + 行为分析 + ML

检测能力：
  ✅ SQL注入
  ✅ XSS
  ✅ CSRF
  ✅ 文件包含
  ✅ 命令注入
  ✅ 敏感信息泄露
  ✅ OWASP Top 10全覆盖

部署模式：
  ① 反向代理：WAF部署在Web服务器前
  ② 透明桥接：二层部署，不需要改网络配置
  ③ 插件模式：Apache/Nginx模块

开源WAF：
  · ModSecurity (+ OWASP CRS规则)
  · Coraza (Go实现，ModSecurity v3兼容)
  · NAXSI (Nginx模块)
```

### 8.2 数据库防火墙

```
数据库防火墙：
  · 分析SQL语句，阻止恶意查询
  · 数据脱敏
  · 异常访问行为告警
  · 敏感数据访问控制

防护示例：
  SELECT * FROM users
   → 数据库防火墙: 不允许SELECT *，必须指定列
  
  ' OR '1'='1
   → 数据库防火墙: 检测到SQL注入特征 → 阻断
```

## 九、CISP考试速查

### 关键考点

| 考点 | 记忆要点 |
|------|---------|
| 防火墙核心原则 | "默认拒绝，最小权限" |
| 状态检测优势 | "状态表自动处理回程+防欺骗" |
| NGFW关键能力 | "应用识别（不论端口）+用户感知" |
| WAF vs 网络防火墙 | "WAF工作在应用层(第7层)" |
| DMZ区域 | "对互联网开放、对内限制的非军事区" |
| 包过滤限制 | "仅检查IP头，无状态" |

### 常见混淆

1. **"防火墙=100%安全"** → 防火墙只是纵深防御的一部分
2. **"有防火墙就不需要WAF"** → 网络防火墙和WAF保护不同层
3. **"NGFW和UTM是同一个东西"** → NGFW统一单次扫描架构，UTM是功能堆叠
4. **"状态检测防火墙检查应用层数据"** → 状态检测在传输层，不解析应用数据
