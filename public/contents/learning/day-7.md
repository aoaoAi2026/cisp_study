## 二、Day 1 回顾：信息安全概述

### 核心知识点

**CIA 三要素（CISP 考试基石）**：

| 要素 | 含义 | 破坏示例 | 保护手段 |
|------|------|----------|----------|
| **机密性** Confidentiality | 信息不泄露给未授权者 | 数据泄露、窃听 | 加密、访问控制 |
| **完整性** Integrity | 信息不被未授权篡改 | 数据被篡改、中间人攻击 | 哈希校验、数字签名 |
| **可用性** Availability | 授权用户能及时访问 | DDoS 攻击、系统宕机 | 冗余、备份、容灾 |

**纵深防御 (Defense in Depth)**：
- 核心理念：多层安全控制叠加，不依赖单一防线
- 每一层独立发挥作用，一层被突破不影响其他层
- 类比：城堡的护城河 + 城墙 + 守卫 + 密室

**安全威胁分类**：
- 人为威胁（有意/无意）
- 技术威胁（软硬件缺陷）
- 环境威胁（自然灾害、物理破坏）

## 三、Day 2 回顾：信息安全发展历程

### 核心知识点

**信息安全发展的三个阶段**：
```
通信保密阶段 (1940s-1970s) → 关注机密性
     ↓ 互联网出现
信息安全阶段 (1980s-1990s) → 关注 CIA 三要素
     ↓ 大数据/云/物联网
信息安全保障阶段 (2000s-至今) → 关注整体治理，涵盖
  ├── 安全管理
  ├── 安全技术
  ├── 安全运营
  └── 合规与审计
```

**关键里程碑**：

| 年代 | 事件 | 意义 |
|------|------|------|
| 1977 | DES 成为美国标准 | 密码学产业化 |
| 1983 | TCSEC (橙皮书) 发布 | 第一个安全评估标准 |
| 1997 | ISO 27001 前身发布 | 信息安全管理体系标准化 |
| 2001 | AES 取代 DES | 对称加密新标准 |
| 2017 | 《网络安全法》施行 | 中国网络安全法治元年 |
| 2021 | 《数据安全法》《个人信息保护法》 | 数据保护法治体系建立 |

**零信任架构 (Zero Trust)**：
- 核心理念：**永不信任，始终验证**
- 不再区分"内网可信/外网不可信"
- 每次访问都要验证身份和授权
- Google BeyondCorp 是零信任的经典实践

## 四、Day 3 回顾：网络安全基本概念

### 核心知识点

**OSI 七层模型与安全威胁**：

```
应用层 (7)  ← HTTP 攻击、SQL 注入、XSS
表示层 (6)  ← SSL 剥离攻击
会话层 (5)  ← 会话劫持
传输层 (4)  ← SYN Flood、端口扫描
网络层 (3)  ← IP 欺骗、路由攻击、ICMP 攻击
数据链路层 (2) ← ARP 欺骗、MAC Flood
物理层 (1)  ← 线缆窃听、物理破坏
```

**TCP/IP 协议栈的安全**：
- **IP 协议**：无认证，源地址可伪造 → IP 欺骗
- **TCP 协议**：三次握手漏洞 → SYN Flood 攻击
- **UDP 协议**：无连接无状态 → 放大攻击 (NTP/DNS Amplification)
- **ARP 协议**：无认证 → ARP 欺骗/中间人攻击
- **DNS 协议**：UDP 无认证 → DNS 欺骗/缓存投毒

**攻击面 (Attack Surface)**：
- 网络攻击面：开放端口、服务
- 软件攻击面：API、输入点
- 人为攻击面：员工、承包商

## 五、Day 4 回顾：恶意软件分类与防范

### 核心知识点

**恶意软件分类速查**：

| 类型 | 英文 | 核心特征 | 传播方式 |
|------|------|----------|----------|
| 病毒 | Virus | 寄生在合法程序中，需要宿主 | 文件共享、U盘 |
| 蠕虫 | Worm | 独立传播，无需宿主 | 网络漏洞、邮件 |
| 木马 | Trojan | 伪装成合法软件 | 下载、社工 |
| 勒索软件 | Ransomware | 加密文件索要赎金 | 邮件附件、漏洞 |
| 间谍软件 | Spyware | 秘密收集用户信息 | 捆绑安装 |
| Rootkit | Rootkit | 隐藏自身存在 | 漏洞提权后安装 |
| 无文件恶意软件 | Fileless | 只在内存中运行 | PowerShell、WMI |
| 逻辑炸弹 | Logic Bomb | 特定条件触发破坏 | 内部人员植入 |

**APT (高级持续性威胁)**：
- **A**dvanced：技术先进，使用 0-day 漏洞
- **P**ersistent：长期潜伏，目标明确的长期作战
- **T**hreat：通常由国家支持，有资金有技术

APT 攻击链 (Kill Chain)：
```
侦察 → 武器化 → 投递 → 利用 → 安装 → C2通信 → 目标达成
```

## 六、Day 5 回顾：社会工程学攻击

### 核心知识点

**社会学攻击的六大原理 (Cialdini)**：
```
权威性 → 冒充领导/警察
互惠性 → 先给好处后索要
承诺一致性 → 从回答简单问题开始
社会认同 → "其他人都已经做了"
稀缺性 → "仅剩最后名额"
喜好性 → 建立共同爱好拉近关系
```

**五大攻击手法**：

| 手法 | Medium | 特点 |
|------|--------|------|
| Phishing | 邮件 | 批量发送，广撒网 |
| Spear Phishing | 邮件 | 定制化，目标明确 |
| Vishing | 电话 | 语音操控，实时互动 |
| Smishing | 短信 | 短小精悍，移动端 |
| Baiting | 物理介质 | USB陷阱，实物诱惑 |

**防御核心**：安全意识培训 > 技术手段
- 技术防线（邮件网关、MFA）
- 人员防线（培训 + 模拟钓鱼测试）
- 流程防线（资金转账独立验证）

## 七、Day 6 回顾：密码学基础概念

### 核心知识点

**三大加密类型对比**：

```
对称加密 (Symmetric)
├── 代表：AES、SM4
├── 密钥：同一密钥加解密
├── 优势：速度快
└── 劣势：密钥分发困难

非对称加密 (Asymmetric)
├── 代表：RSA、ECC、SM2
├── 密钥：公钥加密 + 私钥解密
├── 优势：密钥分发方便
└── 劣势：速度慢 100-1000 倍

哈希函数 (Hash)
├── 代表：SHA-256、SM3
├── 特点：不可逆、定长输出、抗碰撞
└── 应用：完整性校验、密码存储、数字签名
```

**密钥使用方向速查（必背！）**：

| 操作 | 使用哪个密钥 |
|------|-------------|
| 加密消息给 Bob | Bob 的**公钥** |
| Bob 解密消息 | Bob 的**私钥** |
| Bob 进行签名 | Bob 的**私钥** |
| 验证 Bob 的签名 | Bob 的**公钥** |

**已禁用的算法** (CISP 常考的"不能用"清单)：
- ❌ MD5：碰撞已找到
- ❌ SHA-1：碰撞已实现
- ❌ DES：56位密钥太短
- ❌ RC4：密钥流偏差
- ❌ SSL 2.0/3.0：协议级漏洞

**推荐使用的算法**：
- ✅ AES-256-GCM（对称加密）
- ✅ RSA-2048+ / ECC P-256（非对称加密）
- ✅ SHA-256 / SHA-3（哈希）
- ✅ TLS 1.3（传输安全）
- ✅ Argon2id（密码存储）
