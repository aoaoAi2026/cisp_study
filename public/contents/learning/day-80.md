## 一、信息安全基础易混概念

### 🔴 易错点1：CIA三要素的英文全称

```
❌ 常犯错误：
   C = Confidence（信心）       → 错误！
   C = Confidentiality（机密性） → 正确！

❌ 常犯错误：
   将"可用性"理解为可靠性（Reliability）
   可用性 = Availability → 授权用户在需要时可以访问
   可靠性 = 系统持续正常运行的属性
```

**记忆法**：CIA不是美国中央情报局，是"**C**onfidentiality **I**ntegrity **A**vailability"

---

### 🔴 易错点2：威胁 vs 脆弱性 vs 风险

| 概念 | 定义 | 例句 |
|------|------|------|
| 威胁（Threat） | 可能造成损害的潜在因素 | 黑客、地震、管理员误操作 |
| 脆弱性（Vulnerability） | 系统/控制手段中的弱点 | 未打补丁、弱密码、开放端口 |
| 风险（Risk） | 威胁利用脆弱性造成损害的可能性 | 勒索软件通过未修复漏洞（脆弱性）加密数据 |
| 暴露（Exposure） | 资产处于可能受损害的状态 | 数据库暴露在公网上 |

**口诀**：威胁是"谁干的"，脆弱性是"哪里有缝"，风险是"缝被利用的概率和后果"

---

### 🔴 易错点3：主动攻击 vs 被动攻击

| 类型 | 特征 | 示例 |
|------|------|------|
| **被动攻击** | 不修改数据，难以检测 | 窃听、流量分析、电磁截获 |
| **主动攻击** | 修改数据/系统，较易被检测 | 篡改、伪造、重放、DoS、恶意代码 |

**易错提醒**：流量分析不修改任何数据，属于**被动**攻击，常被误认为主动攻击。

---

### 🔴 易错点4：安全策略层次

```
策略层次（从高到低）：
  Policy（策略）    → "做什么"（战略）     ← 最高管理层签发
  Standard（标准）  → "做到什么程度"（战术）
  Guideline（指南） → "建议怎么做"（推荐）
  Procedure（规程） → "具体怎么做"（步骤） ← 最详细

⚠️ 注意：不是 Policy → Procedure → Standard → Guideline！
```

---

### 🔴 易错点5：纵深防御（Defense in Depth）≠ 仅多层防火墙

纵深防御的真谛是多维度、多层级的防护组合：
```
组织维度：管理控制 + 技术控制 + 物理控制
网络维度：边界 + 内网 + 主机 + 应用 + 数据
时间维度：事前（预防）+ 事中（检测）+ 事后（响应）
```

## 三、访问控制模型对比（最高频易错）

### 🔴 易错点10：BLP vs Biba（CISP最爱出对比题）

```
┌─────────────────────────────────────────────┐
│                                              │
│   BLP模型（机密性保护）         Biba模型（完整性保护）   │
│   ┌──────────┐              ┌──────────┐     │
│   │ 高密级    │              │ 高完整性  │     │
│   │  ↓ 不下写 │              │  ↓ 不下读 │     │
│   │ 低密级    │              │ 低完整性  │     │
│   │  ↑ 不上读 │              │  ↑ 不上写 │     │
│   └──────────┘              └──────────┘     │
│                                              │
│   规则：不上读、不下写        规则：不下读、不上写     │
│   场景：军事/政府            场景：金融/医疗        │
│   口诀：BLP→BLP→把低的PASS   口诀：Biba→Biba→把低的       │
│   （低不能读高，高不能写低）   （高不能读低，低不能写高）    │
│                                              │
│   ⚠️ 两者规则正好相反！考试最爱挖坑！              │
└─────────────────────────────────────────────┘
```

**终极记忆法**：
- BLP（机密性）= 守口如瓶：不**上**读（嘴严，不能往上打听），不**下**写（不能往下泄露）
- Biba（完整性）= 洁身自好：不**下**读（不能沾染低级数据），不**上**写（低质量不能污染高质量）

### 🔴 易错点11：DAC vs MAC vs RBAC vs ABAC

| 模型 | 权限决定者 | 灵活性 | 安全性 | 典型场景 |
|------|-----------|--------|--------|---------|
| DAC | 数据所有者 | 高 | 低 | 个人电脑、共享文件夹 |
| MAC | 系统策略/安全标签 | 低 | 高 | 军事、情报机构 |
| RBAC | 角色分配 | 中 | 中 | 企业ERP、OA |
| ABAC | 属性规则引擎 | 很高 | 很高 | 零信任、云环境 |

## 四、应急响应与灾备指标（易混淆）

### 🔴 易错点13：RTO vs RPO（每年必考）

```
时间线 ──────────────────────────────────────►

  ←──── RPO ────→
  (允许数据丢失)
       ↓
    最后一次        灾难发生      ←──── RTO ────→
    备份点           时刻         (允许恢复时间)
       │              │                    │
       ├──────────────┼────────────────────┤
                      ↑                    ↑
                   系统中断             恢复正常运行


RTO：Recovery Time Objective（恢复时间目标）
     → "你能停多久？" → 时间越长，容忍度越高 → 成本越低

RPO：Recovery Point Objective（恢复点目标）
     → "你能丢多少？" → 越接近零，要求越高 → 成本越高
```

---

### 🔴 易错点14：冷站/温站/热站对比

| 站点类型 | 设备 | 数据 | 恢复时间 | 成本 | 适用RTO |
|---------|------|------|---------|------|---------|
| 冷站 Cold | 只有场地 | 无 | 数天-数周 | 低 | RTO > 1周 |
| 温站 Warm | 部分设备 | 近实时 | 数小时-1天 | 中 | RTO 数小时 |
| 热站 Hot | 完全冗余 | 准实时 | 分钟级 | 高 | RTO 分钟级 |

> **易错**：排错恢复时间和成本的顺序。恢复速度：冷<温<热；成本：冷<温<热。

---

### 🔴 易错点15：PICERL vs NIST 800-61 响应阶段

```
PICERL（6步）              NIST（4步）              映射关系
─────────────────       ────────────────       ──────────
P：Preparation          P：Preparation          → 1:1映射
I：Identification       D：Detection&Analysis   → I ≈ D
C：Containment          C：Containment          → 拆分为3步
E：Eradication          E：Eradication
R：Recovery             R：Recovery
L：Lessons Learned      A：Post-Incident        → L ≈ A

⚠️ 考试问"应急响应NIST有几个阶段"，答4个，不是6个！
```

## 五、密码学参数与特性（最易混淆）

### 🔴 易错点16：各算法核心参数一览

| 算法 | 类型 | 密钥/输出长度 | 分组长度 | 安全基础 |
|------|------|-------------|---------|---------|
| DES | 对称 | 56位 | 64位 | Feistel网络 |
| 3DES | 对称 | 112/168位 | 64位 | 三重DES |
| AES | 对称 | 128/192/256位 | **128位** | SPN结构 |
| RSA | 非对称 | ≥2048位 | N/A | 大整数分解 |
| ECC | 非对称 | 256位(~128位安全) | N/A | 椭圆曲线 |
| SHA-256 | 哈希 | **256位输出** | 512位输入块 | MD结构 |
| MD5 | 哈希 | 128位 | 512位 | 已破解 |
| HMAC | MAC | =哈希输出长度 | N/A | 带密钥哈希 |

> **易错提醒**：AES分组固定128位！密钥可以是128/192/256位。不要记混。

---

### 🔴 易错点17：数字签名 ≠ 加密

```
数字签名的功能：
   ✅ 完整性（Integrity）     → 数据未被篡改
   ✅ 认证性（Authentication）→ 确实是签名者所签
   ✅ 不可否认（Non-repudiation）→ 不能抵赖
   ❌ 机密性（Confidentiality）→ 不含加密功能！

数字签名过程：
  签名：Hash(消息) → 用发送者私钥加密摘要 → 数字签名
  验证：用发送者公钥解密签名 → 对比Hash(消息)

⚠️ 注意："用私钥加密"只是签名步骤，不是加密消息！
   加密消息 = 用接收者公钥加密
   签名 = 用发送者私钥加密摘要
```

---

### 🔴 易错点18：加密模式对比

| 模式 | 需要IV | 并行加密 | 并行解密 | 错误传播 | 安全性 |
|------|--------|---------|---------|---------|--------|
| ECB | ❌ 不需要 | ✅ | ✅ | 无 | 最差（不推荐） |
| CBC | ✅ | ❌ | ✅ | 1块 | 一般 |
| CTR | ✅ (Nonce) | ✅ | ✅ | 无 | 好（注意Nonce不可重用） |
| GCM | ✅ (Nonce) | ✅ | ✅ | 无 | 最好（自带认证） |

> **唯一不需要IV的模式是ECB**，也恰好是最不安全的模式！

---

### 🔴 易错点19：AH vs ESP —— IPsec核心混淆点

```
AH（Authentication Header）：
  ✅ 数据完整性验证
  ✅ 数据来源认证
  ✅ 抗重放保护
  ❌ 不提供加密（数据仍明文传输）
  ❌ 不能穿越NAT（因为哈希覆盖了IP头字段）

ESP（Encapsulating Security Payload）：
  ✅ 数据加密（机密性）
  ✅ 可选数据完整性验证和认证
  ✅ 抗重放保护
  ✅ 可以穿越NAT

现代IPsec部署中AH已基本淘汰，推荐使用ESP。
```

## 六、网络安全协议对比

### 🔴 易错点20：TLS 1.2 vs TLS 1.3

| 特性 | TLS 1.2 | TLS 1.3 |
|------|---------|---------|
| 握手RTT | 2-RTT | **1-RTT**（0-RTT重连） |
| 密钥交换 | RSA或DH | 仅DH/ECDH，移除RSA密钥交换 |
| 对称加密 | 支持CBC | 仅AEAD（GCM/CCM/ChaCha20-Poly1305） |
| 已删除的算法 | - | RC4、3DES、CBC模式、SHA-1、静态RSA |

> **记忆**：TLS 1.3精简了握手、砍掉了所有不安全算法。

---

### 🔴 易错点21：IDS vs IPS 部署方式

```
IDS（入侵检测）：
  ┌─────────┐     镜像流量      ┌─────┐
  │  交换机   │ ──────────────► │ IDS │
  └────┬────┘                  └─────┘
       │ 正常流量（不经过IDS）
       ▼
  ┌─────────┐  → 旁路部署，不阻断流量
  │  路由器   │  → 只能检测和报警
  └─────────┘

IPS（入侵防御）：
  ┌─────────┐    在线流量    ┌─────┐    在线流量    ┌─────────┐
  │  交换机   │ ───────────► │ IPS │ ───────────► │  路由器   │
  └─────────┘               └─────┘               └─────────┘
  → 在线部署（串联在流量路径中）
  → 可以检测 + 主动阻断

⚠️ IPS可能成为单点故障和性能瓶颈！
```

## 七、应用安全攻击与防御

### 🔴 易错点22：XSS三种类型精准区分

| 类型 | 存储位置 | 传播方式 | 危害范围 |
|------|---------|---------|---------|
| 反射型 | URL参数 | 诱骗点击链接 | 单个用户 |
| 存储型 | 服务器(DB/文件) | 所有访问者自动触发 | **所有用户** |
| DOM型 | 仅客户端 | JS处理URL/输入 | 单个用户 |

> **区分关键**：恶意脚本存在哪里？→ 服务器=存储型，反射型不存储，DOM型不经过服务器。

### 🔴 易错点23：CSRF vs SSRF

```
CSRF（跨站请求伪造）：           SSRF（服务器端请求伪造）：
  ┌──────────┐                    ┌──────────┐
  │ 攻击者站点  │                    │   攻击者   │
  └────┬─────┘                    └────┬─────┘
       │ 诱导用户点击                   │ 构造恶意请求
       ▼                              ▼
  ┌──────────┐                    ┌──────────┐
  │ 受害者浏览器 │                    │  目标服务器  │ ← 这里发起请求
  └────┬─────┘                    └────┬─────┘
       │ 借用用户Cookie                  │ 服务器向内部发起请求
       ▼                              ▼
  ┌──────────┐                    ┌──────────┐
  │  目标网站   │                    │  内网服务   │
  └──────────┘                    └──────────┘

核心区别：
  CSRF：利用用户浏览器发起（受害者权限）
  SSRF：利用服务器发起请求（服务器权限）
```

## 八、风险管理与量化计算

### 🔴 易错点24：SLE和ALE的计算

```
    资产价值 (AV)
         │
         ├── × 暴露因子 (EF) ──► 单次损失预期 (SLE)
         │                              │
         │                              ├── × 年发生率 (ARO) ──► 年度损失预期 (ALE)
         │                              │
         │                              └── ÷ 年发生率 (ARO) ──► 反推SLE

例题：某系统价值200万元，预计每次安全事件损失40%，每年发生概率25%，求ALE。

  SLE = AV × EF = 200万 × 0.4 = 80万
  ARO = 0.25（次/年）
  ALE = SLE × ARO = 80万 × 0.25 = 20万/年

⚠️ 注意：ARO是"平均每年发生几次"，不是概率百分比！0.25次/年表示4年1次。
```

---

### 🔴 易错点25：四种风险处置策略

| 策略 | 英文 | 含义 | 例子 |
|------|------|------|------|
| 规避 | Avoidance | 不做了 | 不用有漏洞的软件 |
| 转移 | Transfer | 买保险/外包 | 网络安全保险 |
| 缓解 | Mitigation | 加控制措施 | 部署防火墙 |
| 接受 | Acceptance | 承担残余风险 | 低风险成本可控 |

> **"忽略"不是合法的风险处置策略！**

## 九、数据隐私核心概念

### 🔴 易错点26：匿名化 vs 去标识化 vs 假名化

| 技术 | 能否识别个人 | 可逆性 | 是否仍属个人信息 |
|------|------------|--------|---------------|
| 匿名化 | ❌ 不可逆转，无法识别 | 不可逆 | ❌ 不再属于 |
| 去标识化 | ⚠️ 借助额外信息可识别 | 条件可逆 | ✅ 仍属于 |
| 假名化 | ⚠️ 借助密钥可映射 | 可逆 | ✅ 仍属于 |

> **关键**：匿名化是不可逆的，信息不再是"个人信息"。去标识化仍属于个人信息范畴。

---

### 🔴 易错点27：敏感个人信息类别

```
《个保法》规定的敏感个人信息（需单独同意）：

  生物识别          宗教信仰          特定身份
  医疗健康          金融账户          行踪轨迹
  不满14周岁的未成年人个人信息

⚠️ 常见误区：不是所有的个人信息都是敏感个人信息！
   邮箱地址、工作单位、教育经历通常是"个人信息"但不一定是"敏感个人信息"。
```

### 🔴 易错点28：数据出境安全评估触发条件

```
必须申报数据出境安全评估的情形（三选一）：
1. 数据处理者向境外提供重要数据
2. CII运营者向境外提供个人信息

4. 自上年度1月1日起累计向境外提供10万人以上个人信息 或 1万人以上敏感个人信息
```

7. AH协议提供数据加密功能。→ **❌ 错误，AH不加密，ESP加密**

## 🎯 错题复习方法论

1. **分类记录**：将错题按知识域分类（法律/密码学/网络等），而非按题目顺序
2. **深挖根源**：每道错题问自己"为什么错"——是没记住？是概念混淆？是审题不清？
3. **间隔复习**：错题当天复习一遍，2天后复习一遍，7天后复习一遍
4. **建立对照表**：将易混淆概念成对列出（如BLP vs Biba），刻意区分
5. **自问自答**：用费曼学习法——假装教别人，能讲出来才是真懂了
