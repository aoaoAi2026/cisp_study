## 一、CII 的定义与识别

### 1.1 法律定义

《关键信息基础设施安全保护条例》第2条：

> 关键信息基础设施 (CII) 是指公共通信和信息服务、能源、交通、水利、金融、公共服务、电子政务、国防科技工业等重要行业和领域的，以及其他一旦遭到破坏、丧失功能或者数据泄露，可能**严重危害国家安全、国计民生、公共利益**的重要网络设施、信息系统等。

### 1.2 识别的三个条件

```
CII 认定 = 行业 × 功能 × 后果

条件一：属于重点行业/领域
├── 公共通信和信息服务
├── 能源、交通、水利
├── 金融、公共服务
├── 电子政务（一站式政务平台）
└── 国防科技工业

条件二：提供重要网络设施或信息系统
├── 网络设施：骨干网、核心路由器、根服务器
└── 信息系统：业务系统、控制系统、数据中心

条件三：后果严重
└── 破坏 → 严重危害国家安全 or 国计民生 or 公共利益

三者必须同时满足！
```

### 1.3 CII 与等保的关系

```
CII 与等级保护的层级关系：

    网络安全等级保护（等保）
    ┌──────────────────────┐
    │   所有网络系统       │
    │  ┌────────────────┐  │
    │  │  关键信息基础    │  │
    │  │   设施 CII      │  │  ← "重点中的重点"
    │  │  (三级及以上)   │  │
    │  └────────────────┘  │
    └──────────────────────┘

特殊要求：
├── CII 必须在等保基础上满足额外的 CII 保护要求
├── CII 一般要求等保三级或以上
└── CII 运营者有额外报告义务和审查义务
```

## 二、CII 的重点行业领域

### 2.1 八大重点行业

| 行业 | CII 典型设施 | 安全威胁 |
|------|-------------|----------|
| **公共通信** | 骨干网、域名系统(DNS)、根服务器 | 大规模断网、DNS 劫持 |
| **信息服务** | 云平台、大型数据中心 | 数据大规模泄露 |
| **能源** | 电力调度系统、油气管道控制 | 大面积停电、能源供应中断 |
| **交通** | 民航调度、铁路信号、城市轨道交通控制 | 交通事故、运输瘫痪 |
| **水利** | 大坝监控、水利调度 | 洪水失控、水污染 |
| **金融** | 银行核心交易、证券交易、支付清算 | 金融市场动荡 |
| **公共服务** | 医疗卫生、社保、供水供气 | 社会秩序混乱 |
| **电子政务** | 一体化政务平台、国家基础数据库 | 政务服务瘫痪 |

### 2.2 CII 识别程序

```
CII 认定流程图：

行业主管部门
    │
    ├─① 制定本行业 CII 认定规则
    │
    ├─② 组织本行业 CII 识别认定
    │
    ├─③ CII 认定结果通知运营者
    │
    └─④ 通报公安机关

运营者
    │
    └─ 认为本单位网络可能被认定为 CII的
       → 可主动报请认定
```

## 三、CII 运营者的安全保护义务

### 3.1 八项核心义务

```
CII 运营者的八大安全保护义务：

1. 安全管理机构和负责人
   ├── 设立专门安全管理机构
   ├── 指定安全管理负责人
   ├── 关键岗位人员 → 安全背景审查
   └── 安全管理机构人员 → 年度培训

2. 安全保护措施
   └── 安全保护措施与 CII → 三同步

3. 网络安全检测评估（年检）
   └── 每年至少进行一次安全检测评估
       → 发现问题及时整改
       → 评估情况报送保护工作部门

4. 重要数据境内存储
   ├── 运营中收集和产生的个人信息和重要数据
   │   应当在境内存储
   └── 确需出境的 → 安全评估

5. 安全审查
   └── 采购网络产品和服务可能影响国家安全的
       → 申报网络安全审查

6. 安全事件报告
   ├── 发生重大安全事件
   │   发现重大安全威胁和隐患
   └── → 立即向保护工作部门和公安机关报告

7. 应急演练
   └── 定期开展应急演练

8. 安全教育培训
   └── 定期开展从业人员网络安全教育培训
```

### 3.2 与普通等保义务的差异

| 维度 | 普通等保 | CII 特殊要求 |
|------|----------|-------------|
| **安全机构** | 网络安全负责人 | 专门安全管理机构 |
| **人员审查** | 无 | **安全背景审查** |
| **检测评估** | 三级每年一次 | 每年至少一次 + 报行业主管 |
| **数据存储** | 无强制 | **境内存储** |
| **安全审查** | 不需要 | **产品和服务采购前审查** |
| **事件报告** | 向网信/公安 | 向保护工作部门 + 公安 |

## 四、"三同步"原则

### 4.1 核心理念

```
"三同步"原则 (同步规划、同步建设、同步使用)：

    ──────────── 时间线 ────────────→

    规划阶段         建设阶段          运维阶段
    ┌──────┐        ┌──────┐         ┌──────┐
    │ 业务  │        │ 施工  │         │ 上线  │
    │ 规划  │        │ 开发  │         │ 运维  │
    └──┬───┘        └──┬───┘         └──┬───┘
       │               │                │
    ┌──▼───┐        ┌──▼───┐         ┌──▼───┐
    │ 安全  │  同步   │ 安全  │   同步  │ 安全  │
    │ 规划  │  进行   │ 建设  │   进行  │ 运行  │
    └──────┘        └──────┘         └──────┘
       │               │                │
   ·威胁建模       ·代码审计        ·安全监控
   ·安全需求       ·安全测试        ·漏洞管理
   ·架构评审       ·集成测试        ·事件响应
```

### 4.2 三同步的具体要求

| 阶段 | 安全任务 | 具体要求 |
|------|----------|----------|
| **同步规划** | 安全需求分析 | 业务需求分析同步进行安全需求分析；威胁建模 |
| **同步规划** | 安全架构设计 | 整体架构中包含安全架构；纵深防御设计 |
| **同步建设** | 安全产品部署 | 业务系统部署同步部署安全产品 |
| **同步建设** | 安全测试 | 功能测试中包含安全测试；代码安全审计 |
| **同步使用** | 安全运维同步 | 系统运行同步安全监控和运维 |
| **同步使用** | 持续改进 | 定期评估、持续优化安全措施 |

### 5.2 CII 冗余设计

```
冗余是 CII 可用性的生命线：

N+1 冗余
├── 关键设备至少 N+1（1台备份）
└── 适用于：交换机、防火墙、服务器

N+N 冗余
├── 关键设备完全双活
└── 适用于：核心路由器、电力、制冷

异地灾备
├── "两地三中心"模式
│   主中心 + 同城灾备 + 异地灾备
└── 适用于：金融核心交易系统

双链路
├── 通信双链路（不同运营商）
├── 电力双路供应
└── 网络双平面设计
```

## 六、CII 供应链安全

### 6.1 供应链风险全景

```
CII 供应链风险矩阵：

┌──────────┬──────────┬──────────┬──────────┐
│  风险类型  │ 硬件层面  │ 软件层面  │ 服务层面  │
├──────────┼──────────┼──────────┼──────────┤
│ 后门植入  │ 芯片后门  │ 开源组件  │ 远程维护  │
│ 供应链阻断 │ 设备禁运  │ 许可吊销  │ 服务中断  │
│ 数据泄露  │ 设备冗余  │ 数据上云  │ 第三方处理│
│ 产品漏洞  │ 固件漏洞  │ 0-day漏洞 │ 配置缺陷  │
└──────────┴──────────┴──────────┴──────────┘
```

### 6.2 网络安全审查触发条件

```
《网络安全审查办法》第2条：

强制审查的情形：
├── CII 运营者采购网络产品和服务
│   影响或可能影响国家安全的 → 必须申报审查
│
└── 掌握超过 100 万用户个人信息的网络平台运营者
    赴国外上市 → 必须申报审查

审查内容：
├── 产品和服务投入使用后带来的安全风险
│   · 关键设备安全性、开放性、透明性、来源多样性
│   · 供应链安全
├── 产品和服务提供者受外国政府影响的可能性
└── 核心数据、重要数据或大量个人信息泄露的风险
```

### 7.2 CII 事件分级

| 级别 | 判定标准 | 响应要求 |
|------|----------|----------|
| **特别重大** | 全国范围影响，国家安全严重威胁 | 国家层面应急响应 |
| **重大** | 跨省影响，多个 CII 受影响 | 国家级协调 + 省级响应 |
| **较大** | 省内影响，单个 CII 受影响 | 省级响应 |
| **一般** | CII 局部受影响 | 运营者自身应急 |

## 八、CISP 考试要点速查

### 核心概念

| 概念 | 要点 |
|------|------|
| CII 定义 | 八大行业 + 严重危害国家安全/国计民生/公共利益 |
| 三同步 | 同步规划、同步建设、**同步使用** |
| 专门机构 | CII 运营者须设立专门安全管理机构 |
| 年检 | 每年至少一次安全检测评估 |
| 境内存储 | CII 运营者个人信息和重要数据须在境内存储 |
| 审查 | 采购网络产品和服务 + 影响国家安全 → 申报审查 |

### CII 八义务速记口诀

> "机构建，三同步，年检测，数据存，要审查，事件报，勤演练，常培训"

### 常见考试陷阱

1. ~~所有网络系统都是 CII~~ → 只有八大重点行业的关键设施才是
2. ~~CII 只需满足等保要求~~ → CII 在等保基础上有额外强化要求
3. ~~CII 无需三同步~~ → 安全措施必须**同步规划、建设、使用**
4. ~~CII 数据可以自由出境~~ → 个人信息和重要数据须**境内存储**
5. ~~CII 采购只需考虑价格~~ → 可能影响国家安全的须**申报审查**

> 💡 **本章小结**：CII 是《网络安全法》的核心保护对象，也是 CISP 法规考试的重中之重。八大行业的范围、"三同步"原则、专门安全管理机构、年度检测评估和境内数据存储是五个最常考的维度。记住：CII 保护 = 等保（基础） + 行业特殊要求（强化），安全措施必须与业务同步推进。
