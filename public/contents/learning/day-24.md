## 一、安全监控技术全景

### 1.1 技术演进路线

```
安全监控技术演进：

1990s ──── IDS (入侵检测系统)
    → 只检测不阻断，旁路部署

2000s ──── IPS (入侵防御系统)
    → 检测 + 阻断，串联部署

2000s ──── SIEM (安全信息和事件管理)
    → 日志集中 + 关联分析 + 合规报表

2010s ──── EDR (端点检测和响应)
    → 深入端点的实时行为分析

2020s ──── XDR (扩展检测和响应)
    → 跨端点/网络/云的统一检测

2020s ──── SOAR + 态势感知
    → 自动化响应 + 全局风险可视化
```

### 1.2 安全监控金字塔

```
          态势感知 (全局可视化+决策支持)
                │
      ┌─────────┴─────────┐
      │                   │
    SIEM               SOAR
 (集中分析与报表)    (编排与自动化)
      │                   │
  ┌───┴────┐         ┌────┴────┐
  │        │         │         │
 IDS/IPS  EDR      NDR      DLP
(网络检测) (端点)  (流量分析) (数据防泄漏)
```

### 1.3 部署位置对比

| 技术 | 部署方式 | 数据源 | 实时阻断 |
|------|----------|--------|----------|
| IDS | 旁路 (镜像流量) | 网络流量 | ❌ |
| IPS | 串联 (inline) | 网络流量 | ✅ |
| SIEM | 独立平台 | 日志集中 | ❌ |
| EDR | 端点 Agent | 进程/文件/网络 | ⚠️ 部分 |
| NDR/NTA | 旁路/串联 | 全流量/NetFlow | ⚠️ 部分 |

## 二、IDS vs IPS 深度辨析

### 2.1 核心区别

```
IDS (Intrusion Detection System)
┌──────────────────────────────────────┐
│ 部署模式：旁路 (SPAN/TAP)              │
│                                      │
│  网络流量 ───────── 网络 ─────────→    │
│                │                     │
│                ├→ 镜像流量 (副本)      │
│                   ↓                 │
│              ┌────────┐             │
│              │  IDS   │ → 告警       │
│              └────────┘             │
│                                      │
│  原始流量不受影响 → 只检测不阻断       │
└──────────────────────────────────────┘

IPS (Intrusion Prevention System)
┌──────────────────────────────────────┐
│ 部署模式：串联 (Inline)               │
│                                      │
│  网络流量 ───→ ┌────────┐ ───→ 网络   │
│              │  IPS   │             │
│              └────────┘             │
│                                      │
│  流量经过 IPS → 检测 + 阻断           │
└──────────────────────────────────────┘
```

### 2.2 详细对比

| 维度 | IDS | IPS |
|------|-----|-----|
| **部署** | 旁路 (不影响网络流量) | 串联 (流量必须经过) |
| **能力** | 检测 + 告警 | 检测 + 告警 + **阻断** |
| **网络影响** | 无 (仅复制流量) | 有 → 单点故障风险 |
| **延迟** | 零延迟 | 有处理延迟 (~ms级) |
| **误报影响** | 仅噪音告警 | 可能阻断正常业务 |
| **检测范围** | 全覆盖（能看到所有旁路流量） | 仅限于经过它的流量 |
| **CISP高频考点** | "IDS只能检测不能阻断" | "IPS是唯一可在线阻断的" |

### 2.3 检测方法

```
IDS/IPS 的四种检测方法：

1. 基于签名 (Signature-Based)
   ├── 匹配已知攻击特征
   ├── 优点：误报低、准确
   ├── 缺点：无法检测未知攻击(0-day)
   └── 类似：杀毒软件的病毒库

2. 基于异常 (Anomaly-Based)
   ├── 建立正常行为基线 → 偏离基线 = 告警
   ├── 优点：可检测未知攻击
   ├── 缺点：误报率高、基线建立难
   └── 需要长期学习

3. 基于协议 (Protocol-Based)
   ├── 检查是否符合协议规范
   ├── 不符合 → 可能是攻击
   └── 例：HTTP GET 请求不应有 SQL 语句

4. 基于信誉 (Reputation-Based)
   ├── 检查 IP/域名在威胁情报黑名单中
   └── 例：已知 C2 服务器的 IP → 立即阻断

现代 IDS/IPS 通常混合使用以上四种方法
```

### 2.4 部署位置

```
IDS/IPS 在企业网络中的部署位置：

    Internet
        │
   ┌────▼────┐
   │ 边界IPS  │ ← 最外层防线 (外部攻击)
   └────┬────┘
        │
   ┌────▼────┐
   │ 边界防火墙│
   └────┬────┘
        │
   ┌────▼────┐
   │ 边界IDS  │ ← 旁路监听 (检测是否绕过边界IPS)
   └────┬────┘
        │
   ┌────▼────┐
   │ 核心交换机│
   └────┬────┘
        │
   ┌────▼────┐
   │内部网段IPS│ ← 内网防护 (横向移动检测)
   └─────────┘
```

## 三、SIEM 平台详解

### 3.1 SIEM 的双重基因

```
SIEM = SIM + SEM

SIM (Security Information Management)
├── 日志长期存储
├── 历史趋势分析
├── 合规报表生成
└── 侧重：管理、合规、审计

SEM (Security Event Management)
├── 实时日志关联分析
├── 实时告警
├── 仪表盘展示
└── 侧重：实时监控、事件响应

现代 SIEM = SIM + SEM 融合
```

### 3.2 SIEM 的五大核心功能

```
1. 日志聚合 (Aggregation)
   └── 从数百个源收集日志 → 统一存储

2. 标准化 (Normalization)
   └── 不同格式 → 统一字段结构
   例：Windows 的 "登录成功" = Linux 的 "Accepted password"

3. 关联分析 (Correlation)
   └── 单条日志看不出攻击 → 多条关联后看出全貌
   例：
   防火墙：拒绝 1.2.3.4 (低危)
   IDS：1.2.3.4 端口扫描 (中危)
   服务器：1.2.3.4 SSH 登录失败 100次 (高危)
   → 关联：暴力破解攻击！

4. 告警生成 (Alerting)
   └── 关联规则触发 → 通知分析师

5. 报表和仪表盘 (Reporting & Dashboard)
   └── 合规报表、趋势图、实时监控大屏
```

### 3.3 SIEM 关联规则示例

```
SIEM 关联规则（伪代码）：

规则：暴力破解检测
  WHEN
    (同一源IP在5分钟内)
    AND (登录失败次数 > 10)
    AND (目标为同一主机)
    AND (端口为 22/3389)
  THEN
    生成高危告警："疑似暴力破解"

规则：横向移动检测
  WHEN
    (用户A在主机1上登录)
    AND (同一用户在5分钟内从主机1登录到主机2)
    AND (使用远程服务创建/计划任务/WMI)
  THEN
    生成高危告警："疑似横向移动"
```

## 四、EDR 与 XDR

### 4.1 EDR 的能力

```
EDR (Endpoint Detection and Response)：

传统杀毒软件的局限：
├── 只检测已知恶意软件（签名匹配）
├── 无行为分析
└── 无溯源能力

EDR 的增强：
├── 进程行为分析 → 即使无签名也能检测
├── 完整的进程树可见性
├── 文件/注册表/网络连接监控
├── 远程隔离主机、结束进程
├── 事件时间线重建 → 溯源
└── 威胁狩猎 → 主动搜索

EDR 的典型告警：
├── PowerShell 执行编码的命令 (base64 -enc)
├── 进程注入其他进程
├── 创建计划任务/启动项
├── LSASS 内存访问 (凭据窃取)
└── 异常父子进程关系 (Word.exe → cmd.exe)
```

### 4.2 XDR 的愿景

```
XDR (Extended Detection and Response) =

EDR + NDR + Cloud + Identity + Email → 统一检测和响应

核心价值：打破数据孤岛

传统模式 (数据孤岛)：
EDR只看端点 → EDR告警
NDR只看网络 → NDR告警
云安全只看云 → 云告警
Email安全只看邮件 → 邮件告警
↓
各管各的，无法关联

XDR 模式 (统一视角)：
┌──────────────┐
│     XDR      │
│   统一分析引擎 │
└──┬──┬──┬──┬──┘
   │  │  │  │
 端点 网络 云 邮件
   │  │  │  │
   └──┴──┴──┘
     统一告警

例：钓鱼邮件 → 用户点击 → 下载恶意软件 → C2通信
XDR 能看到完整攻击链，而不是孤立的四个告警
```

## 五、态势感知平台

### 5.1 概念

```
安全态势感知 (Security Situational Awareness)：

= 通过全局可视化，实时感知组织的整体安全状态

与传统 SIEM 的区别：
├── SIEM：关注"发生了什么事"
├── 态势感知：回答"我们现在的安全状况如何"
│   ├── 整体风险评分是多少？
│   ├── 当前有多少活跃威胁？
│   ├── 防御覆盖面的盲区在哪？
│   └── 与昨天/上周相比趋势如何？

态势感知的三个层次：

感知层 (Perception)
├── 数据采集：从全量资产收集数据
└── 实时展示：资产、漏洞、威胁、事件一张图

理解层 (Comprehension)
├── 关联分析：告警关联为事件，事件关联为攻击
├── 攻击链还原
└── 影响评估

预测层 (Projection)
├── 趋势预测：威胁演进方向
├── 风险预判：可能被攻击的薄弱点
└── 决策支持：应该优先修哪个漏洞？
```

## 六、安全告警机制设计

### 6.1 告警分级

```
告警严重程度的分级标准：

Critical (危急) — 立即响应 (≤15分钟)
├── 正在发生的入侵 (Active Intrusion)
├── 数据正在被窃取
├── 勒索软件正在加密
└── 核心系统被控制 (域控/核心数据库)

High (高危) — 1小时内响应
├── 成功的恶意代码感染
├── 账户凭证疑似泄露
├── C2 通信确认
└── 关键系统漏洞被利用

Medium (中危) — 24小时内处理
├── 暴力破解尝试 (尚未成功)
├── 端口扫描
├── 可疑文件下载
└── 非关键系统漏洞

Low (低危) — 周内处理
├── 信息泄露类 (目录列表、版本号暴露)
├── 合规类轻微不符合
└── 已阻止的攻击尝试
```

### 6.2 告警规则优化原则

```
减少告警噪音的方法：

1. 使用阈值，而非单条触发
   ❌ 一次登录失败 → 告警
   ✅ 同一IP 5分钟10次登录失败 → 告警

2. 结合上下文（资产价值）
   ❌ 任何主机上的XSS尝试 → 高危
   ✅ 核心交易系统上的XSS尝试 → 高危
      测试环境上的XSS尝试 → 低危

3. 时间窗口优化
   ❌ 全天24小时相同规则
   ✅ 工作时间宽松，非工作时间严格
      凌晨3点的大量数据导出 → 极度可疑

4. 告警富化（减少调查时间）
   ├── 自动附加：资产信息、用户信息、GeoIP
   ├── 自动关联：同一IP的历史告警
   └── 自动调查：查询威胁情报、VirusTotal
```

## 七、CISP 考试要点速查

### IDS vs IPS

| 对比 | IDS | IPS |
|------|-----|-----|
| 部署 | 旁路 | **串联** |
| 阻断 | ❌ 不阻断 | ✅ **可阻断** |
| 延迟 | 0 | 有 |
| 单点故障 | 无 | 有风险 |

### 检测方法

| 方法 | 优点 | 缺点 |
|------|------|------|
| 签名 | 误报低 | 无法检测未知 |
| 异常 | 检测未知 | 误报高 |
| 协议 | 精准 | 协议变种困难 |
| 信誉 | 快速 | 依赖情报时效 |

### 常见考试陷阱

1. ~~IDS 可以阻断攻击~~ → IDS 只能**检测告警**，IPS 才能阻断
2. ~~IPS 旁路部署~~ → IPS 是**串联/inline**部署
3. ~~SIEM 就是日志存储~~ → SIEM 的核心是**关联分析**
4. ~~EDR 只杀毒~~ → EDR 有行为分析、溯源、响应能力
