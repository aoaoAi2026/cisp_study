## 一、密码学核心概念总览

### 1.1 密码学三大功能

| 功能 | 实现技术 | 用途 |
|------|---------|------|
| 机密性 | 加密（对称/非对称） | 保护数据不被看到 |
| 完整性 | 哈希函数、MAC | 检测数据是否被篡改 |
| 不可否认性 | 数字签名 | 证明发送者身份 |

### 1.2 密码体制分类

```
密码体制
├── 对称密码（私钥密码）
│   ├── 流密码：RC4
│   └── 分组密码：DES、3DES、AES、SM4
└── 非对称密码（公钥密码）
    ├── RSA（大数分解）
    ├── ECC（椭圆曲线）
    ├── SM2（国密）
    └── Diffie-Hellman（密钥交换）
```

## 二、对称 vs 非对称加密深度对比

| 对比维度 | 对称加密 | 非对称加密 |
|---------|---------|-----------|
| **密钥数量** | 1把（共享密钥） | 2把（公钥+私钥） |
| **加解密速度** | 快（毫秒级） | 慢（秒级，差异约1000倍） |
| **密钥分发** | 困难（密钥需安全传输） | 简单（公钥可公开） |
| **典型算法** | AES、DES、SM4 | RSA、ECC、SM2 |
| **密钥长度** | 128/256位（AES） | 2048/4096位（RSA） |
| **应用场景** | 数据加密、文件加密 | 数字签名、密钥交换 |
| **安全性基础** | 混淆与扩散 | 数学难题（大数分解等） |
| **密钥管理难度** | 高（N个用户需O(n²)个密钥） | 低（N个用户只需N对密钥） |

### 2.1 AES（高级加密标准）重点

- **分组长度**：128位（固定）
- **密钥长度**：128位（AES-128）、192位（AES-192）、256位（AES-256）
- **轮数**：10轮（128位）、12轮（192位）、14轮（256位）
- **当前地位**：全球范围内最广泛使用的对称加密标准
- **国密对应**：SM4

### 2.2 RSA 重点

- **数学基础**：大整数因数分解难题
- **当前推荐密钥长度**：2048位或以上
- **用途**：数字签名、密钥交换、小额数据加密
- **国密对应**：SM2（基于椭圆曲线，比RSA更高效）

### 2.3 混合加密（考试常考！）

实践中通常结合使用：
1. 用**对称加密**（AES）加密大量数据（快）
2. 用**非对称加密**（RSA）加密对称密钥（解决密钥分发）

这就是 HTTPS/TLS 的工作原理！

## 三、哈希函数与数字签名

### 3.1 哈希函数特性

| 特性 | 含义 |
|------|------|
| 定长输出 | 无论输入多大，输出固定长度 |
| 单向性 | 从哈希值不能推导出原文 |
| 抗碰撞性 | 很难找到两个不同输入产生相同哈希值 |
| 雪崩效应 | 输入微变导致输出剧变 |

**常见哈希算法**：
| 算法 | 输出长度 | 安全性 |
|------|---------|--------|
| MD5 | 128位 | ❌ 已破解，不可用于安全 |
| SHA-1 | 160位 | ❌ 已不建议使用 |
| SHA-256 | 256位 | ✅ 当前主流 |
| SHA-3 | 可变 | ✅ 最新标准 |
| SM3 | 256位 | ✅ 国密标准 |

### 3.2 数字签名流程（必考！）

```
发送方                              接收方
  │                                   │
  │ ① 对原文计算哈希值               │
  │ ② 用发送方私钥加密哈希值         │
  │ ③ 原文 + 签名一并发送 ──────────→ │
  │                                   │ ④ 用发送方公钥解密签名（得哈希值A）
  │                                   │ ⑤ 对收到的原文重新计算哈希值B
  │                                   │ ⑥ 比较 A == B？
  │                                   │    相等 → 验证通过
```

> **关键记忆**：私钥签名（加密），公钥验证（解密）——与加密方向相反！

### 3.3 数字证书与PKI

- **数字证书** = 用户公钥 + CA的数字签名
- **CA（证书颁发机构）**：可信第三方，用自己私钥签发证书
- **PKI（公钥基础设施）**：管理数字证书的完整体系
- **CRL（证书吊销列表）**：已吊销证书的清单

## 四、访问控制模型

### 4.1 三大经典模型对比（高频！）

| 特性 | DAC（自主访问控制） | MAC（强制访问控制） | RBAC（基于角色） |
|------|-------------------|-------------------|-----------------|
| **决策者** | 数据所有者 | 系统/安全策略 | 管理员（角色定义） |
| **访问依据** | ACL | 安全标签 | 角色 |
| **灵活性** | 高 | 低 | 中 |
| **安全性** | 低（所有者可能犯错） | 高 | 中高 |
| **典型场景** | 文件系统权限 | 军事/政府系统 | 企业应用 |

### 4.2 MAC安全标签模型

**Bell-LaPadula模型**（机密性）：不读上、不写下
- 简单安全规则：低级别不能读高级别（No Read Up）
- 星规则：高级别不能写低级别（No Write Down）

**Biba模型**（完整性）：不读下、不写上
- 低完整性不能读高完整性（No Read Down）
- 高完整性不能写低完整性（No Write Up）

> **记忆技巧**：Bell-LaPadula 是"不读上不写下"（为机密性），Biba 反之"不读下不写上"（为完整性）

### 4.3 身份认证三要素

| 要素 | 含义 | 示例 |
|------|------|------|
| 你知道什么 | 知识因素 | 密码、PIN码 |
| 你拥有什么 | 持有因素 | 智能卡、USB Key |
| 你是什么 | 生物因素 | 指纹、虹膜、人脸 |

**多因素认证**：使用两类或以上 = 强认证

### 4.4 常见认证协议

| 协议 | 特点 | 用途 |
|------|------|------|
| Kerberos | 票据+TGT，需要可信KDC | 企业内网认证 |
| RADIUS | AAA集中认证 | 网络设备认证 |
| LDAP | 目录服务 | 用户信息查询 |
| OAuth 2.0 | 授权框架 | 第三方登录 |
| SAML | XML断言 | 单点登录(SSO) |

## 五、专项选择题练习（15题）

**1. 以下哪种加密算法属于对称加密？**

A. RSA
B. ECC
C. AES
D. SM2

<details>
<summary>答案与解析</summary>

**C** —— AES是对称加密。RSA、ECC、SM2都是非对称加密（公私钥成对）。

</details>

**2. 数字签名使用发送方___对哈希值加密？**

A. 公钥
B. 私钥
C. 会话密钥
D. 对称密钥

<details>
<summary>答案与解析</summary>

**B** —— 数字签名 = 发送方用**私钥**加密哈希值，接收方用发送方公钥解密验证。私钥签名、公钥验证。

</details>

---

**3. AES-256的密钥长度和分组长度分别是？**

A. 256位 / 256位
B. 256位 / 128位
C. 128位 / 256位
D. 128位 / 128位

<details>
<summary>答案与解析</summary>

**B** —— AES分组长度固定128位，密钥长度可选128/192/256位。分组长度≠密钥长度。

</details>

**4. 在HTTPS通信中，实际数据用___加密，该密钥的交换用___完成？**

A. AES / RSA
B. RSA / AES
C. DES / MD5
D. MD5 / SHA-256

<details>
<summary>答案与解析</summary>

**A** —— 混合加密：AES对称加密数据（快），RSA非对称加密AES密钥（安全交换）。这是考"混合加密"的核心场景。

</details>

**5. Bell-LaPadula模型主要保护的信息安全属性是？**

A. 可用性
B. 完整性
C. 机密性
D. 不可否认性

<details>
<summary>答案与解析</summary>

**C** —— Bell-LaPadula是为军事系统设计的，目标是保护机密性，"不读上不写下"防止信息从高级别流向低级别。

</details>

**6. Biba模型的核心规则是？**

A. 不读上、不写下
B. 不读下、不写上
C. 不读上、不写上
D. 不读下、不写下

<details>
<summary>答案与解析</summary>

**B** —— Biba保护完整性，规则与Bell-LaPadula相反：不读下（低完整性不能读高完整性数据，防止被污染），不写上（高完整性不能写低完整性对象）。

</details>

**10. Kerberos认证中，发放服务票据的核心组件是？**

A. AS（认证服务器）
B. TGS（票据授予服务器）
C. DNS服务器
D. Web服务器

<details>
<summary>答案与解析</summary>

**B** —— Kerberos流程：用户先向AS认证获得TGT，然后用TGT向TGS请求特定服务的票据，最后用服务票据访问服务。KDC = AS + TGS。

</details>

**12. 数字证书中包含的核心信息是？**

**15. ECC（椭圆曲线加密）相比RSA的优势是？**

A. 比RSA更老
B. 相同安全强度下密钥更短、运算更快
C. 不需要密钥分发
D. 是哈希算法

<details>

</details>

## 六、高频易错点总结

| 易混淆概念 | 快速记忆法 |
|-----------|-----------|
| **对称 vs 非对称** | 对称=1把钥匙，快但分发难；非对称=2把钥匙，慢但分发易 |
| **AES-256** | 分组固定128位，密钥256位（不要搞反）|
| **数字签名** | 私钥签名，公钥验证（与加密方向相反！）|
| **Bell-LaPadula vs Biba** | BL=机密性=不读上不写下；Biba=完整性=不读下不写上 |
| **DAC vs MAC vs RBAC** | DAC=所有者定，MAC=系统定(标签)，RBAC=角色定 |
| **RPO vs RTO** | P=Point数据丢多少(时间点)，T=Time恢复要多久 |
| **HTTPS混合加密** | 数据用AES（对称快），AES密钥用RSA交换（非对称安全）|
