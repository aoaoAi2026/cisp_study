## 一、非对称加密核心原理

### 为什么需要非对称加密

```
对称加密的困境：
Alice 和 Bob 从未见过面 → 怎么安全共享密钥？

非对称加密的解决方案：
├── 每个人有一对密钥：公钥 (公开) + 私钥 (保密)
├── 公钥加密 → 只有对应私钥能解密
└── 公钥可以公开分发 → 不需要安全信道

     Bob的公钥(公开)              Bob的私钥(保密)
         │                            │
明文 → [加密] → 密文 → [不安全信道] → [解密] → 明文

    任何人都能加密 → 只有Bob能解密

数学基础：
├── RSA：大整数分解的困难性
├── ECC：椭圆曲线离散对数问题
└── DH：有限域上离散对数问题
```

### 对称 vs 非对称 对比

| 维度 | 对称加密 | 非对称加密 |
|------|----------|-----------|
| 密钥数量 | 1个 (共享) | 2个 (公钥+私钥) |
| 速度 | ⚡ 快 (GB/s) | 🐢 慢 (100-1000x) |
| 密钥分发 | ❌ 困难 | ✅ 容易 |
| 典型用途 | 数据加密 | 密钥交换、数字签名 |
| 代表算法 | AES, SM4 | RSA, ECC, SM2 |

## 二、RSA 深入

### RSA 密钥生成

```
RSA 密钥生成的步骤：

1. 选择两个大素数 p 和 q (通常 1024-2048 位)
2. 计算 n = p × q  (模数，公钥的一部分)
3. 计算 φ(n) = (p-1)(q-1)
4. 选择 e：1 < e < φ(n)，且 gcd(e, φ(n)) = 1
   常用 e = 65537 (0x10001)
5. 计算 d：d × e ≡ 1 (mod φ(n))
   d 是 e 的模逆元

公钥 = (n, e)
私钥 = (n, d)

加密：C = M^e mod n
解密：M = C^d mod n

安全性依赖：
├── 已知 n = p × q → 很难找到 p 和 q (大整数分解)
└── 量子计算机 (Shor算法) 可以高效分解 → 未来的威胁
```

### RSA 安全性

```
RSA 密钥长度与安全强度：

RSA-1024  ≈ 80位安全  (⚠️ 已不够)
RSA-2048  ≈ 112位安全 (✅ 当前标准)
RSA-3072  ≈ 128位安全 (✅ 更高安全)
RSA-4096  ≈ 140位安全

密钥长度增长 vs 安全收益递减：
2048→4096 长度×2，安全只+~28位，性能降10×
→ ECC 是更好的替代方案
```

## 三、椭圆曲线密码 (ECC)

### ECC 的优势

```
为什么 ECC 比 RSA 更好？

等价安全强度下：

RSA-3072 位 ≈ ECC-256 位  (都是 ~128位安全)
RSA-7680 位 ≈ ECC-384 位  (都是 ~192位安全)

ECC 的优势：
├── 密钥更短 (256 vs 3072)
├── 签名更快
├── 带宽占用少 (证书/握手包更小)
├── 更适合移动/嵌入式设备
└── 同样的安全 → 更好的性能

TLS 1.3 支持的 ECC 曲线：
├── secp256r1 (NIST P-256) ← 最常用
├── secp384r1 (NIST P-384)
├── X25519 (Curve25519) ← 现代推荐，更安全
└── X448
```

### 常用的 ECC 标准

| 标准 | 曲线 | 用途 | 推荐 |
|------|------|------|------|
| NIST P-256 | secp256r1 | ECDSA/ECDH | ✅ |
| Curve25519 | X25519 | ECDH 密钥交换 | ✅✅ 推荐 |
| Ed25519 | Edwards | 数字签名 | ✅✅ 推荐 |
| SM2 | 国密曲线 | 国密签名/加密 | ✅ 国内合规 |
| secp256k1 | — | 比特币 | ✅ |

## 四、Diffie-Hellman 密钥交换

### DH 原理

```
Diffie-Hellman 密钥交换：

公开参数：大素数 p，生成元 g

Alice                         Bob
选择私钥 a                     选择私钥 b
计算 A = g^a mod p            计算 B = g^b mod p
    │        ── A ──→               │
    │        ←── B ──               │
共享密钥 = B^a mod p           共享密钥 = A^b mod p
         = g^(ab) mod p              = g^(ab) mod p
                                        ↑ 相同！

安全性：已知 g, p, A, B → 无法高效计算 g^(ab) mod p
        (计算 Diffie-Hellman 问题，依赖离散对数难题)

DH 协议家族：
├── DH：经典 Diffie-Hellman (基于模p)
├── ECDH：基于椭圆曲线的 DH (更高效)
└── ECDHE：临时密钥 (Ephemeral) → 前向安全性

前向安全性 (Forward Secrecy)：
├── ECDHE：每次会话生成临时密钥 → 会话结束丢弃
├── 即使长期私钥泄露 → 历史会话无法解密
└── TLS 1.3 要求所有加密套件都提供前向安全性
```

## 五、公钥与私钥使用方向（必背！）

```
数字签名 vs 数据加密 → 密钥使用方向截然不同！

加密 (提供机密性)：
To: Bob → 使用 Bob的公钥 加密 → Bob用 自己的私钥 解密
           "用别人的公钥加密，让别人用私钥打开"

签名 (提供不可否认性)：
From: Alice → 使用 Alice的私钥 签名 → 别人用 Alice的公钥 验证
               "用自己的私钥签名，让别人用公钥确认"

完整记忆表：
┌──────────────┬──────────────────┬─────────────────┐
│     场景      │    使用的密钥      │      理由        │
├──────────────┼──────────────────┼─────────────────┤
│ 加密发送给Bob │  Bob的公钥        │  只有Bob能解密   │
│ Bob解密消息  │  Bob的私钥        │  私钥解公钥密    │
│ Alice签名    │  Alice的私钥      │  证明是我签的    │
│ 验证Alice签名 │  Alice的公钥      │  公钥验私钥签    │
│ 密钥交换     │  对方公钥+自己私钥 │  DH/ECDH计算    │
└──────────────┴──────────────────┴─────────────────┘
```

## 六、CISP 考试要点速查

### 核心对比

| 维度 | RSA | ECC |
|------|-----|-----|
| 数学基础 | 大整数分解 | 椭圆曲线离散对数 |
| 等效密钥长度 | 3072位≈128位安全 | 256位≈128位安全 |
| 性能 | 慢 | 快 |
| 密钥大小 | 大 | 小 |
| 量子威胁 | Shor算法可破 | Shor算法可破 |

### DH/ECDH 速查

| 协议 | 特点 |
|------|------|
| DH | 经典，基于大素数模运算 |
| ECDH | 基于椭圆曲线，更高效 |
| ECDHE | Ephemeral (临时) → 前向安全性 |

### 常见考试陷阱

1. ~~公钥用于解密~~ → 公钥用于**加密**和**验签**，私钥用于**解密**和**签名**
2. ~~RSA-1024 仍然安全~~ → 1024位已不够，至少需 2048位
3. ~~DH 就是加密算法~~ → DH 是**密钥交换**协议，不直接加密数据
4. ~~ECC 和 RSA 使用相同的数学原理~~ → 完全不同的数学难题

> 💡 **本章小结**：非对称加密的考试核心是"签名用私钥、验签用公钥、加密用对端公钥"。ECC 256位 ≈ RSA 3072位 安全是经典数字题。DH 是密钥交换协议而非加密算法。前向安全性 (ECDHE) 是现代 TLS 的必备特性。
