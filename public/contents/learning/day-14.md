## 五、个人权利体系

```
个人信息保护法赋予个人的七项权利：

┌──────────────────────────────────────┐
│                                      │
│  1. 知情权  → 知道谁在用什么方式处理什么信息
│  2. 决定权  → 可以同意，也可以不同意
│  3. 查阅权  → 查自己的信息长什么样
│  4. 复制权  → 带走自己的信息副本
│  5. 更正权  → 错了？帮我改过来
│  6. 删除权  → "被遗忘权"，请删除
│  7. 携带权  → 转移到其他服务商
│                                      │
└──────────────────────────────────────┘

处理敏感个人信息（生物/医疗/金融/行踪/未成年人）：
├── 必须具有特定目的 + 充分必要性
├── 必须取得单独同意
├── 必须告知必要性
└── 必须进行保护影响评估 (PIA)
```

## 六、核心制度总览

### 等级保护 vs CII 保护

| 维度 | 等级保护 | CII 保护 |
|------|----------|----------|
| 适用对象 | 所有网络系统 | 八大行业的关键设施 |
| 法律基础 | 网安法第21条 | 网安法第31-39条 |
| 保护框架 | 定级→备案→建设→测评→检查 | 识别→专管机构→三同步→年检→审查 |
| 强制等级 | 二级及以上 | 一般要求三级以上 |
| 数据存储 | 无特殊要求 | 境内存储 |
| 安全审查 | 不需要 | 采购前须审查 |

### 等保定级速查

```
侵害客体 × 侵害程度 = 定级

侵害程度：一般  →  严重   →  特别严重
          ↓         ↓           ↓
个人权益  1级       2级         2级
公共秩序  2级       3级 ←最常见的 4级
国家安全  3级       4级         5级

定级公式：L = MAX(S, A)
S = 业务信息安全等级（机密性被破坏）
A = 系统服务安全等级（可用性被破坏）
```

## 八、易混淆点辨析

### 混淆 1：网安法 vs 数安法 vs 个保法

```
最容易混淆的关系：

网络安全法 → 管"网络系统"的安全
              包含：等保、CII、网络运营者义务
              
数据安全法 → 管"数据"的安全
              包含：分类分级、安全审查、出境管理
              
个人信息保护法 → 管"个人信息"的权益保护
              包含：同意机制、七项权利、敏感保护

三者关系：
网安法(2017) ← 最早，奠定基础
    │
    ├→ 数安法(2021) ← 数据领域专项补充
    │       └→ 个保法(2021) ← 个人信息专项补充
    └→ 三法交叉：数据出境、安全审查、行政处罚
```

### 混淆 2：等保 vs CII

```
等保是对所有网络系统的分级保护，CII 是重点中的重点。

二者关系：CII ⊂ 等保（CII 是等保保护对象的一个子集）

├── CII 的基础是等保三级
├── CII 在等保基础上叠加特殊要求
│   ├── 专门安全管理机构
│   ├── 境内数据存储
│   ├── 安全审查
│   └── 年度评估报送
└── 等保是通用标准，CII 是行业强化
```

### 混淆 3：第一道防线 vs 第二道防线

```
第一道防线 = 业务部门（"做"的人）
├── 是风险的主人
├── 执行安全控制
└── 不是"被检查"的客体，而是"自我管理"的主体

第二道防线 = 安全/合规部门（"管"的人）
├── 制定标准和策略
├── 指导和监督
└── 不是"代替业务做安全"，而是"帮业务做好安全"

关键区别：
├── 一道防线的安全考核 → 业务负责人的考核
├── 二道防线的安全考核 → 是否建立了有效的管理框架
└── 三道防线的审计 → 独立评估前两道防线的有效性
```
