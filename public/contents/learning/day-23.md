## 一、为什么日志是安全基石

### 1.1 日志的三重价值

```
日志 = 安全的眼睛和记忆

追溯 (Accountability)
├── "谁在什么时间从哪台机器做了什么"
├── 安全事件溯源
└── 合规审计（等保要求：日志留存 ≥ 6个月）

检测 (Detection)
├── 从海量日志中发现异常模式
├── SIEM 告警 → 实时检测
└── 威胁狩猎 → 主动发现

取证 (Forensics)
├── 安全事件发生后的证据链
├── 还原攻击全过程
└── 法律证据（需要防篡改 + 时间戳可信）
```

### 1.2 日志管理的核心原则

```
日志管理五项原则：

1. 全面采集 (Collect Everything)
   └── 所有产生日志的系统都应纳入采集范围

2. 集中管理 (Centralized Management)
   └── 分散的日志价值低 → 集中后才能关联分析

3. 结构化存储 (Structured Storage)
   └── 方便查询、检索和分析

4. 安全保护 (Protection)
   ├── 防篡改：WORM / Hash链
   ├── 防删除：权限分离
   └── 防泄露：加密传输和存储

5. 生命周期管理 (Lifecycle Management)
   ├── 热数据：实时检索 (7-30天)
   ├── 温数据：快速回溯 (30天-6个月)
   ├── 冷数据：归档存储 (6个月-数年)
   └── 到期自动销毁
```

## 二、日志来源分类

### 2.1 主要日志类型

```
安全运营关注的七类日志：

1. 系统日志 (Syslog/Event Log)
   ├── Windows Event Log (安全、系统、应用)
   ├── Linux Syslog (/var/log/*)
   ├── 内容：登录/登出、服务启停、系统错误
   └── Event ID 4624 (登录成功) / 4625 (登录失败)

2. 网络设备日志
   ├── 防火墙：拒绝的连接、策略变更
   ├── 路由器/交换机：路由变更、端口up/down
   ├── VPN 网关：登录/登出、流量
   └── Syslog 格式

3. 应用日志
   ├── Web 服务器：Nginx/Apache access_log、error_log
   ├── 数据库：MySQL/PostgreSQL 慢查询、错误日志
   ├── 中间件：应用错误、访问日志
   └── 格式因应用而异（需要解析）

4. 安全设备日志
   ├── IDS/IPS：入侵检测告警签名
   ├── WAF：Web 攻击拦截记录
   ├── DLP：数据泄露检测
   └── EDR：端点检测和响应

5. 认证日志
   ├── AD/LDAP：域控登录/认证日志
   ├── SSO/IdP：单点登录记录
   └── IAM：权限变更审计

6. 云平台日志
   ├── AWS CloudTrail：API 调用审计
   ├── Azure Activity Log：管理操作
   └── GCP Audit Logs

7. 数据库审计日志
   ├── SQL 操作记录
   ├── DBA 操作审计
   └── 必须满足合规要求
```

### 2.2 日志格式标准化

```
常见日志格式：

Syslog (RFC 5424)
<PRI>VERSION TIMESTAMP HOSTNAME APP-NAME PROCID MSGID STRUCTURED-DATA MSG

示例：
<34>1 2024-01-15T10:30:00Z myhost sshd 12345 - - Failed password for root from 1.2.3.4 port 22 ssh2

CEF (Common Event Format) — 安全设备常用
CEF:Version|Device Vendor|Device Product|Device Version|Signature ID|Name|Severity|Extension

示例：
CEF:0|Cisco|ASA|9.8|106100|Deny IP Traffic|High|src=1.2.3.4 dst=10.0.0.1

JSON — 现代应用/云平台首选
{
  "timestamp": "2024-01-15T10:30:00Z",
  "event": "login_failed",
  "user": "root",
  "source_ip": "1.2.3.4"
}
```

## 三、日志采集架构

### 3.1 采集架构模式

```
三种日志采集架构：

模式一：Agent 代理模式
┌──────┐  ┌──────┐  ┌──────┐
│服务器1│  │服务器2│  │服务器3│
│Agent │  │Agent │  │Agent │
└──┬───┘  └──┬───┘  └──┬───┘
   │         │         │
   └─────────┼─────────┘
             │  (加密传输)
        ┌────▼────┐
        │ 日志收集器 │ → 缓冲队列 → 日志存储
        └─────────┘
推荐：Filebeat / Fluentd / NXLog

模式二：Syslog 直发模式
┌──────┐  ┌──────┐      ┌──────────┐
│网络设备│──│安全设备│─────→│ Syslog服务器│
└──────┘  └──────┘      └─────┬────┘
                               │
                          ┌────▼────┐
                          │ 日志解析器 │
                          └─────────┘
适合：网络设备和安全设备

模式三：API Pull 模式
┌──────────┐
│  云平台    │
│CloudTrail │──────→ API 拉取 → 日志处理器
│  (S3)     │
└──────────┘
适合：云平台审计日志
```

### 3.2 日志采集管道

```
日志采集处理管道 (Pipeline)：

1. 采集 (Ingestion)
   ├── Agent：Filebeat/Fluentd
   ├── Syslog：514/601 TCP|UDP
   └── API：HTTP REST

2. 缓冲 (Buffer)
   ├── Kafka / Redis Stream
   └── 削峰填谷、防止日志风暴

3. 解析 (Parsing)
   ├── Grok：提取结构化字段 (Logstash)
   ├── 正则匹配
   └── 字段映射 (IP→地理位置、UserID→用户名)

4. 丰富 (Enrichment)
   ├── GeoIP：IP→地理位置
   ├── 威胁情报：IP/域名→已知恶意？
   └── 资产管理：资产名称、所有者、重要级别

5. 索引 (Indexing)
   ├── Elasticsearch
   └── 按时间分区、按类型分区

6. 可视化 (Visualization)
   └── Kibana / Grafana
```

## 四、日志存储与保护

### 4.1 日志分层存储

```
日志分层存储架构：

热存储 (Hot) — SSD
├── 保留：7-30天
├── 数据：最近数据，可实时检索
├── 性能：高
└── 用途：实时监控、活跃调查

温存储 (Warm) — HDD
├── 保留：30天 - 6个月
├── 性能：中
└── 用途：近期历史回溯

冷存储 (Cold) — 归档 (S3/磁带)
├── 保留：6个月 - 数年
├── 性能：低（小时级恢复）
└── 用途：合规归档、长期取证

冷冻存储 (Frozen) — 云对象存储
├── 保留：按法规要求
├── 查询前需先解冻
└── 成本极低
```

### 4.2 日志完整性保护

```
日志防篡改策略：

1. Hash 链
   每条日志包含上一条日志的哈希
   → 任何修改都会破坏链
   → BitCoin 同款技术

   Log1 → Hash1
   Log2 → Hash(Hash1 + Log2)
   Log3 → Hash(Hash2 + Log3)
   ...

2. 数字签名
   日志服务器用私钥对日志块签名
   → 审计时用公钥验证

3. WORM 存储
   Write Once Read Many
   → 写入后不可修改和删除

4. 权限分离
   ├── 安全管理岗：配置日志策略
   ├── 审计管理岗：查看日志
   └── 系统管理员：无权修改日志
```

## 五、关键日志类型详解

### 5.1 Windows 安全事件

```
Windows Event ID 速查：

登录相关 (Security Log)：
├── 4624：成功登录 → Type 2(交互) / 3(网络) / 10(远程交互)
├── 4625：登录失败 → 状态码 0xC000006A(错误密码)
├── 4634：注销
├── 4648：使用显式凭据尝试登录 (RunAs)
├── 4771：Kerberos 预认证失败
└── 4768：TGT 授予 (Kerberos)

账户管理：
├── 4720：创建用户
├── 4722：启用用户
├── 4725：禁用用户
├── 4726：删除用户
├── 4732：添加到安全组 (如 Domain Admins)
└── 4738：更改用户属性

特权使用：
├── 4672：特殊权限分配给新登录
└── 4673：特权服务被调用

攻击检测关键 Event ID：
├── 4625 + 后续 4624 → 暴力破解成功
├── 4732 + 添加 Domain Admins → 提权
├── 4720 + 4722 → 创建新用户并启用
└── 异常登录时间 + Type=10 → 可疑远程访问
```

### 5.2 Linux Syslog

```
Linux 日志位置速查：

├── /var/log/auth.log  (认证日志：ssh登录/sudo)
├── /var/log/syslog    (系统日志)
├── /var/log/messages  (通用系统消息)
├── /var/log/apache2/  (Apache 访问和错误日志)
├── /var/log/boot.log  (启动日志)
├── /var/log/cron      (Cron 定时任务)
└── /var/log/audit/    (auditd 审计日志)

SSH 暴力破解检测：
auth.log 中大量重复：
Failed password for root from 1.2.3.4 port 22 ssh2
→ IP: 1.2.3.4 短时大量失败 → 触发告警
```

### 5.3 Web 服务器日志

```
Nginx/Apache Access Log 关键字段：

192.168.1.1 - - [15/Jan/2024:10:30:00 +0800]
"GET /login.php?user=admin' OR '1'='1 HTTP/1.1"
200 1234 "http://example.com" "Mozilla/5.0"


攻击检测模式：
├── SQL 注入：URL中包含 ' OR / UNION SELECT / ;DROP
├── XSS：URL中包含 <script> / javascript:
├── 目录遍历：URL中包含 ../../../
├── 文件包含：URL中包含 /etc/passwd
├── Web Shell：POST 请求中 eval() / base64_decode
└── 扫描行为：同一 IP 短时间内大量 404 → 目录扫描
```

## 六、日志分析实战

### 6.1 日志分析基本方法论

```
安全事件日志分析方法：

1. 异常检测
   ├── 统计基线偏离（如某服务器平时100次登录，今天10000次）
   ├── 时间异常（凌晨3点的大量活动）
   └── 地理位置异常（同一用户30分钟内出现在两个国家）

2. 关联分析
   ├── 多条低危事件串起来 → 高危
   │   例：端口扫描 → 漏洞利用 → 提权 → 横向移动
   └── 不同数据源互验
       例：VPN日志(登录) + 服务器日志(文件访问)

3. 攻击链还原
   侦察 → 武器化 → 投递 → 利用 → 安装 → C2 → 行动
   在日志中找到每个阶段的证据

4. IOC 搜索
   └── 在历史日志中搜索已知恶意 IOC (IP/域名/哈希)
```

### 6.2 日志级别的经验法则

```
Syslog 严重级别：

0: Emergency  (系统不可用)      ← 立即告警
1: Alert      (必须立即处理)     ← 立即告警
2: Critical   (严重)            ← 立即告警
3: Error      (错误)            ← 高优先级
4: Warning    (警告)            ← 中优先级
5: Notice     (注意)            ← 关注
6: Informational (信息)         ← 记录
7: Debug      (调试)            ← 主要开发用

安全监控关注：0-4 级别
响应时间要求：1-3 级别需在 15 分钟内响应
```

## 七、CISP 考试要点速查

### 核心数字和概念

| 概念 | 要点 |
|------|------|
| 日志留存 | ≥ 6个月 (网安法)，推荐12个月 |
| 日志格式 | Syslog (RFC 5424) / CEF / JSON |
| 集中管理 | SIEM 或日志平台 |
| 防篡改 | Hash链 / WORM / 数字签名 |
| 权限分离 | 系统管理员不能删除日志 |

### 关键 Event ID (Windows)

| Event ID | 含义 |
|----------|------|
| 4624 | 成功登录 |
| 4625 | 登录失败 |
| 4720 | 创建用户 |
| 4732 | 添加到安全组 |
| 4672 | 特权分配 |

### 常见考试陷阱

1. ~~日志保留3个月就够了~~ → 网安法要求**不少于6个月**
2. ~~系统管理员可以管理审计日志~~ → 审计日志应与管理员**权限分离**
3. ~~日志只要存了就算安全~~ → 需要**防篡改**保护
4. ~~看单条日志就能判断攻击~~ → 需要多条跨源**关联分析**
