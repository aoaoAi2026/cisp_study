## 一、网络分段基础概念

### 1.1 为什么需要分段

```
不合理的扁平网络：

  所有设备在同一广播域/子网
  ┌─────┬─────┬─────┬─────┬─────┬─────┐
  │  PC  │ 服务器│ 打印机│ 摄像头│  IoT  │  访客 │
  └──┬───┴──┬───┴──┬───┴──┬───┴──┬───┴──┬──┘
     └──────┴──────┴──────┴──────┴──────┘

问题：
  · 任何设备被攻破 → 可到达所有其他设备
  · 广播风暴影响全局
  · 无法差异化安全策略
  · 无法实现信任分区


分段后的网络：

  ┌────────┐  ┌────────┐  ┌──────────┐  ┌────────┐
  │ 办公网  │  │ 服务器网 │  │  访客网络  │  │ 管理网  │
  │ VLAN10 │  │ VLAN20 │  │  VLAN30  │  │ VLAN40 │
  └───┬────┘  └───┬────┘  └────┬─────┘  └───┬────┘
      └───────────┴────────────┴─────────────┘
                     │
               [防火墙/ACL]
              访问控制跨VLAN
```

### 1.2 分段层次

```
不同层次的分段技术：

  物理分段：
    不同交换机/路由器 → 完全物理隔离
    最高安全，最不灵活，最昂贵

  逻辑分段：
    VLAN → 二层逻辑隔离
    子网 → 三层逻辑隔离

  微分段：
    主机级策略 → 工作负载间控制
    SDP → 软件定义边界
    零信任 → 身份驱动的访问控制
```

## 二、VLAN深度解析

### 2.1 VLAN基础知识

```
VLAN = Virtual LAN → 将一个物理交换机逻辑上划分为多个广播域

┌─────────────────────────────────────────┐
│          VLAN编号范围                     │
├─────────┬───────────────────────────────┤
│ 0       │ 不使用（表示无VLAN）            │
│ 1       │ 默认VLAN（Cisco）              │
│ 2-1001  │ 标准VLAN                       │
│ 1002-1005│ Token Ring/FDDI保留           │
│ 1006-4094│ 扩展VLAN (需VTPv3)           │
│ 4095     │ 保留                           │
└─────────┴───────────────────────────────┘

VLAN标记协议：
  · ISL (Inter-Switch Link) — Cisco私有，已淘汰
  · IEEE 802.1Q — 业界标准，4字节Tag
  
802.1Q帧结构：
  ┌────────┬──────┬────────┬───────────┬──────┐
  │ DestMAC│ SrcMAC│ 802.1Q │ Type/Size │ Data │
  │ 6字节  │ 6字节│ 4字节  │ 2字节     │      │
  └────────┴──────┴──┬──┬──┴───────────┴──────┘
                     │  │
            TPID(0x8100) │ PCP(3位) DEI(1位) VID(12位)
```

### 2.2 端口类型

```
Access端口 (接入端口)：
  · 属于单个VLAN
  · 帧不带Tag进出
  · 用于连接终端设备（PC、打印机）
  · switchport mode access
  · switchport access vlan 10

Trunk端口 (中继端口)：
  · 承载多个VLAN的流量
  · 帧带802.1Q Tag
  · 用于交换机之间的连接
  · switchport mode trunk
  · switchport trunk allowed vlan 10,20,30

Native VLAN：
  · Trunk端口上未打Tag的帧所属的VLAN
  · 默认VLAN 1（Cisco）→ 安全风险！应更改
  · switchport trunk native vlan 999
```

## 三、VLAN攻击与防御

### 3.1 VLAN跳跃攻击

```
① Switch Spoofing (交换机欺骗)：
  攻击者伪装自己是交换机
  与目标交换机建立Trunk连接
  → 获得所有VLAN的访问权限

  条件：端口启用DTP(Dynamic Trunking Protocol)
  防御：关闭DTP (switchport nonegotiate)
       设置端口为固定Access模式

② Double Tagging (双标签攻击)：
  攻击者 接入VLAN10
  发送帧：802.1Q(VLAN10) + 802.1Q(VLAN20) + 数据
  ↓
  交换机A：移除外层Tag (VLAN10) → 转发
  ↓
  交换机B：收到 802.1Q(VLAN20) + 数据 (在Native VLAN中)
  → 将帧发送到VLAN20！

  条件：攻击者接入VLAN = Native VLAN (默认VLAN1)
  防御：将Native VLAN改为未使用的VLAN
       所有Trunk端口打Tag (包括Native VLAN)
```

### 3.2 MAC地址泛洪攻击

```
攻击MAC地址表：

  交换机的CAM表空间有限（如8000条）
  
  攻击：发送大量伪造源MAC地址的帧
  → CAM表填满
  → 交换机退化为Hub模式
  → 所有帧广播到所有端口
  → 攻击者可以嗅探所有VLAN内流量

  防御：
  ✅ Port Security
     · 限制每端口MAC地址数量
     · 违规动作：shutdown / restrict
     switchport port-security maximum 2
     switchport port-security violation shutdown
     switchport port-security mac-address sticky
```

### 3.3 VLAN安全最佳实践

```
✅ 关闭所有未使用的端口，放入隔离VLAN
✅ 将Native VLAN改为未使用的VLAN（不是VLAN 1）
✅ 所有Trunk端口上强制所有VLAN打Tag
✅ 关闭DTP (switchport nonegotiate)
✅ 关闭VTP或使用VTPv3+密码
✅ 限定Trunk端口允许的VLAN (switchport trunk allowed vlan)
✅ 部署Port Security
✅ 使用PVLAN隔离同一VLAN内的客户端
✅ 管理VLAN独立且限制访问
```

## 四、微分段 (Micro-Segmentation)

### 4.1 微分段概念

```
微分段：将安全策略下推到单个工作负载级别

传统分段                        微分段
┌──────────┐                   ┌──────────┐
│  VLAN 10  │                   │  VLAN 10  │
│           │                   │  ┌─┐ ┌─┐  │
│  ┌─┐ ┌─┐  │                  │  │A│ │B│  │ ← A与B之间也受控制
│  │A│ │B│  │                  │  └─┘ └─┘  │
│  └─┘ └─┘  │                  │   ↕ ACL  │
│           │                  │  ┌─┐     │
│ 流量不受限  │                  │  │C│     │
└──────────┘                   └──┴─┴─────┘

微分段允许：
  · DB服务器仅允许App服务器IP访问3306端口
  · Web-1不能直接访问Web-2
  · 即使在同一VLAN内
```

### 4.2 微分段实现技术

```
① 主机防火墙：
   iptables, Windows Firewall, firewalld
   便宜但管理复杂

② 虚拟化平台原生微分段：
   VMware NSX Distributed Firewall
   策略随VM迁移

③ 容器网络策略：
   Kubernetes NetworkPolicy
   Calico, Cilium

④ 软件定义微分段：
   Illumio, Guardicore, Edgewise
   基于标签/身份的策略
```

## 五、软件定义网络(SDN)与安全

### 5.1 SDN安全优势

```
传统网络                            SDN
┌──────────────┐              ┌──────────────┐
│ 逐台配置      │              │ 集中控制器     │
│ 手动规则      │              │ 一致性策略下发  │
│ 变更困难      │              │ 自动化编排     │
│ 策略漂移      │              │ 声明式意图     │

SDN安全优势：
  ✅ 策略一致性（单点定义，全网生效）
  ✅ 动态响应（自动隔离受感染端点）
  ✅ 全网可见性（流量矩阵、异常检测）
  ✅ 策略仿真（测试变更影响）
  ✅ 微分段原生支持
```

### 5.2 流量镜像与监控

```
SDN提供的安全监控能力：

传统网络：需要SPAN/TAP硬件 → 有限覆盖
SDN网络：软件定义镜像 → 任意流量可复制到分析平台

应用：
  · 全流量捕获（100%可见性）
  · 按需取证（触发式流量捕获）
  · 影子IT发现
  · 数据泄露检测
```

## 六、零信任网络架构

### 6.1 零信任原则

```
零信任核心理念：
"Never Trust, Always Verify" — 永不信任，始终验证

与传统边界模型对比：

传统：                       零信任：
┌─────────────┐             ┌─────────────┐
│  内网 = 信任  │             │  无隐式信任    │
│  外网 = 不信任│             │  每次请求验证  │
└─────────────┘             │  最小权限      │
                             │  假定已攻破    │
                             └─────────────┘

NIST SP 800-207 零信任架构：

① 所有数据源和计算服务都视为资源
② 所有通信都是安全的，不受网络位置影响
③ 按会话授予对资源的访问权限
④ 策略必须是动态的
⑤ 企业持续监控所有资产
⑥ 认证和授权在允许访问前严格执行
⑦ 收集尽可能多的信息以改进安全姿态
```

### 6.2 SDP (软件定义边界)

```
SDP (Software Defined Perimeter)：

三层架构：
  ┌──────┐    ┌──────────┐    ┌──────┐
  │ Client│ ←→ │Controller │ ←→ │Gateway│
  │      │    │(认证+策略)  │    │(代理) │
  └──────┘    └──────────┘    └──┬───┘
                                  │
                            ┌─────▼─────┐
                            │  内部应用   │
                            │ (从未暴露) │
                            └───────────┘

流程：
1. 客户端先向Controller认证
2. Controller下发授权的Gateway列表
3. 客户端建立到Gateway的mTLS连接
4. Gateway代理访问内部应用

"先认证，再连接" → 类似于"黑云"效果
未被授权的用户根本看不到/连不到资源
```

## 七、网络访问控制(NAC)

### 7.1 NAC核心功能

```
NAC (Network Access Control)：

  ┌────────────────────────────────────┐
  │  接入设备 ──→ NAC策略服务器         │
  │      │                              │
  │      ├── 谁在连接？ (802.1X认证)    │
  │      ├── 设备是什么？ (MAC/Profiling)│
  │      ├── 设备安全吗？ (健康检查)     │
  │      └── 应该放哪里？ (VLAN分配)    │
  └────────────────────────────────────┘

安全健康检查内容：
  ✅ 防病毒软件是否运行且更新？
  ✅ 操作系统补丁是否最新？
  ✅ 防火墙是否启用？
  ✅ 是否有特定注册表项？
  ✅ 磁盘加密是否启用？
  ✅ 是否有禁止的软件？

不合规设备的处置：
  · 隔离VLAN（仅能访问修复服务器）
  · 强制门户页面（提示修复步骤）
  · 直接拒绝接入
```

### 7.2 主流NAC方案

| 方案 | 类型 | 特点 |
|------|------|------|
| Cisco ISE | 商业 | 思科生态深度整合 |
| Aruba ClearPass | 商业 | 多厂商支持好 |
| Forescout | 商业 | 无代理设备发现 |
| PacketFence | 开源 | 功能全面 |
| FreeRADIUS + 脚本 | 自建 | 灵活但维护成本高 |

## 八、CISP考试速查

### 关键考点

| 考点 | 记忆要点 |
|------|---------|
| VLAN范围 | "标准1-1001，扩展1006-4094" |
| 802.1Q | "4字节Tag，12位VID(0-4095)" |
| VLAN跳跃防御 | "关闭DTP + 改Native VLAN + 所有Trunk打Tag" |
| 微分段 | "工作负载级策略，同VLAN内也隔离" |
| 零信任原则 | "永不信任、始终验证、最小权限、假定已攻破" |
| SDP架构 | "Controller + Gateway + Client = 先认证再连接" |

### 常见陷阱

1. **"同一个VLAN的设备可以直接互相访问"** → 传统网络中是，但微分段可以进一步隔离
2. **"只要分VLAN就是网络分段"** → 分段应该是多层防御的组合
3. **"零信任=微分段"** → 微分段是实现零信任的技术手段之一
4. **"VLAN Tag是加密的"** → 802.1Q Tag是明文，可被篡改
