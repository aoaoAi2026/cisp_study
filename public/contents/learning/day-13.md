## 一、合规管理的核心理念

### 1.1 合规的定义

```
合规 (Compliance) =

确保组织的活动和行为
符合：
├── 外部要求：法律法规、行业标准、合同义务
├── 内部要求：企业制度、安全策略、操作流程
└── 最佳实践：国际标准、行业惯例

合规 ≠ 安全
├── 合规是"满足最低要求" → "及格了"
├── 安全是"持续降低风险" → "更安全了"
├── "合规了不一定安全，安全了通常能合规"
└── 合规是起点，不是终点
```

### 1.2 合规管理的 PDCA 循环

```
          Plan（策划）
     · 识别合规义务
     · 制定安全策略
     · 设定安全目标
          │
    ┌─────▼─────┐
    │            │
Act │  持续改进   │ Do
    │  的循环     │（实施）
    │            │
    └─────┬─────┘
          │
    · 纠正不符合项
    · 改进管理体系    Check（检查）
    · 优化控制措施  · 监控和测量
                  · 内部审计
                  · 管理评审
```

### 1.3 合规管理的基本要素

```
信息安全合规管理体系的构成：

制度层 (Policy)
├── 信息安全方针（顶层）
├── 安全管理制度
├── 安全操作流程
└── 安全表单和记录

组织层 (Organization)
├── 网络安全与信息化领导小组
├── 信息安全管理部门
├── 内部审计部门
└── 全员安全意识

技术层 (Technology)
├── 安全产品部署
├── 安全配置基线
└── 技术合规检查工具

监督层 (Supervision)
├── 内部审计
├── 外部审计/测评
├── 管理评审
└── 持续监控
```

## 二、三道防线模型

### 2.1 三道防线架构

```
          三道防线模型
                             董事会/审计委员会
                                    │
    ┌───────────────────────────────┼──────────────────────────────┐
    │                               │                              │
    ▼                               ▼                              ▼
┌─────────┐                   ┌─────────┐                   ┌─────────┐
│第一道防线│                   │第二道防线│                   │第三道防线│
│业务部门  │                   │合规/安管部│                   │内部审计  │
│"做正确的事"│                 │"保障合规" │                   │"独立监督"│
└────┬────┘                   └────┬────┘                   └────┬────┘
     │                             │                             │
├──执行安全控制              ├──制定安全策略               ├──独立评估
├──日常安全运营              ├──合规监控和检查              ├──内控有效性
├──风险自评估                ├──安全培训和意识              ├──向审计委员会报告
└──事件初步处置              └──监管报送                    └──建议改进
```

### 2.2 三道防线的职责详述

| 防线 | 角色 | 核心职责 |
|------|------|----------|
| **第一道防线** (业务部门) | 风险的主人 | 执行控制措施；识别和报告风险；遵守制度和流程 |
| **第二道防线** (安全/合规部) | 风险的管理者 | 制定策略和标准；监控合规状态；风险评估和培训 |
| **第三道防线** (内部审计) | 独立的监督者 | 独立评估控制有效性；向审计委员会报告；提出改进建议 |

### 2.3 三道防线的协作关系

```
有效的三道防线 = 各司其职 + 信息共享 + 权责清晰

反面教材：混淆防线
├── 安全部门既制定策略又自查 → 既当裁判又当运动员
├── 内部审计由 CIO 管理 → 缺乏独立性
└── 业务部门完全依赖安全部门 → 第一道防线"真空"

最佳实践：
├── 安全部门向 CIO/CSO 汇报（第二道防线）
├── 内部审计向审计委员会汇报（第三道防线独立于管理层）
└── 各防线定期向管理层汇报风险状况
```

## 三、ISO 27001 信息安全管理体系

### 3.1 标准体系概览

```
ISO 27000 系列标准体系：

ISO 27000  概述和术语
    │
    ├── 要求类标准 ──┐
    │               │
    │   ISO 27001   │  信息安全管理体系 — 要求
    │   (认证依据)   │  ← 可以做认证！
    │               │
    ├── 指南类标准 ──┤
    │               │
    │   ISO 27002   │  信息安全控制措施实施指南
    │   ISO 27003   │  ISMS 实施指南
    │   ISO 27005   │  信息安全风险管理指南
    │               │
    └── 行业扩展 ────┘
        ISO 27017   云服务安全
        ISO 27018   云中 PII 保护
        ISO 27701   隐私信息管理
```

### 3.2 ISO 27001 核心结构

```
ISO 27001:2022 标准结构：

第4章 组织环境
├── 理解组织及其环境
├── 理解利益相关方的需求和期望
└── 确定 ISMS 范围

第5章 领导力
├── 领导力和承诺
├── 信息安全方针
└── 组织的角色、职责和权限

第6章 策划
├── 应对风险和机会的措施
├── 信息安全目标和实现计划
└── 变更策划

第7章 支持
├── 资源、能力、意识
├── 沟通
└── 文件化信息

第8章 运行
├── 运行策划和控制
├── 信息安全风险评估
└── 信息安全风险处置

第9章 绩效评价
├── 监视、测量、分析和评价
├── 内部审核
└── 管理评审

第10章 改进
└── 持续改进、不符合和纠正措施
```

### 3.3 ISO 27002 控制措施域 (Annex A)

```
ISO 27001:2022 附录 A 的 4 大控制主题：

1. 组织控制 (5. Organizational Controls)
   信息安全策略、资产管理、人力资源安全、供应商关系...

2. 人员控制 (6. People Controls)
   安全教育与培训、违规处理...

3. 物理控制 (7. Physical Controls)
   物理安全边界、设备安全、安全区域...

4. 技术控制 (8. Technological Controls)
   访问控制、密码学、安全编码、网络安全、监控...

共 93 项控制措施（2022版精简自 2013版的114项）
```

### 3.4 中国等级保护 vs ISO 27001 对比

| 维度 | 等保 2.0 | ISO 27001 |
|------|----------|-----------|
| **性质** | 中国法定要求 | 国际自愿性标准 |
| **适用范围** | 中国境内网络运营者 | 全球任何组织 |
| **强制性** | 法定强制 | 自愿认证 |
| **保护对象** | 信息系统/云/物联网等 | 信息资产（广义） |
| **核心方法** | 定级→建设→测评 | 风险评估→控制→认证 |
| **认证标志** | 等保测评报告 | ISO 27001 证书 |
| **测评/认证机构** | 等保测评机构（公安部认定） | 认证机构（CNAS 认可） |

### 3.5 ISO 27001 认证流程

```
1. 差距分析 (Gap Analysis)
   └── 对比现状与标准要求 → 找出差距项

2. 体系建设 (Implementation)
   ├── 编写方针、程序、策略
   ├── 实施控制措施
   └── 培训、意识提升

3. 运行 (Operation)
   ├── ISMS 试运行（至少3个月）
   ├── 内部审计
   └── 管理评审

4. 认证 (Certification)
   ├── 第一阶段审核：文件审查
   ├── 第二阶段审核：现场审核
   └── 获得证书（有效期 3 年）

5. 维持 (Surveillance)
   ├── 每年监督审核
   └── 三年后重新认证
```

## 四、合规差距分析实务

### 4.1 差距分析方法论

```
合规差距分析五步法：

步骤一：确定基准
├── 选择参考标准（等保/ISO 27001/GDPR...）
├── 选择版本号
└── 列出现有合规义务清单

步骤二：现状调研
├── 文档审查（策略、制度、流程）
├── 技术检查（配置、工具、日志）
├── 人员访谈（意识、执行情况）
└── 历史事件分析

步骤三：差距评估
├── 逐条对照标准要求
├── 评估符合程度（完全/部分/不符合/不适用）
└── 识别差距项和风险等级

步骤四：制定整改计划
├── 差距项优先级排序（高风险→低风险）
├── 整改措施、责任人、时间表
└── 资源需求评估

步骤五：跟踪验证
├── 整改实施跟踪
├── 整改效果验证
└── 输出合规报告
```

### 4.2 差距评分矩阵

| 控制项状态 | 评分 | 含义 |
|------------|------|------|
| ✅ 完全符合 | 1.0 | 控制措施已实施且有效运行 |
| ⚠️ 部分符合 | 0.5 | 有措施但不完整/部分有效 |
| ❌ 不符合 | 0.0 | 未实施或完全失效 |
| N/A 不适用 | — | 该控制项不适用于本组织 |

```
总体合规度 = Σ(各项评分) / 适用项总数 × 100%

目标值：
├── 认证准备期：≥ 60%
├── 认证审核前：≥ 85%
└── 持续维持：≥ 90%
```

## 五、安全审计方法

### 5.1 审计类型

| 审计类型 | 执行者 | 目的 | 频率 |
|----------|--------|------|------|
| **内部审计** (第一方) | 内部审计部门 | 自查自纠 | 每年一次 |
| **外部审计** (第二方) | 客户/合作方 | 供应链合规 | 按合同要求 |
| **认证审计** (第三方) | 认证机构 | 获得认证 | 初次+年度监督 |

### 5.2 审计三角

```
审计的三个核心问题：

    证据 ─────────→ 标准
    (实际做了什么)    (应该做什么)
         │              │
         └──────┬───────┘
                ↓
             发现
          (找到了什么)
                │
                ▼
    ┌────────────────────────┐
    │ 符合 (Conformance)      │ → 证据满足标准要求
    ├────────────────────────┤
    │ 不符合 (Non-conformity) │ → 证据不满足标准要求
    ├────────────────────────┤
    │ 观察项 (Observation)    │ → 潜在风险，尚未不符合
    └────────────────────────┘
```

### 5.3 审计方法

```
审计收集证据的六种方法：

1. 文档审查
   └── 审阅：策略、制度、操作流程、日志记录

2. 人员访谈
   └── 验证：意识、理解、执行情况

3. 现场观察
   └── 确认：物理环境、操作习惯

4. 技术测试
   └── 验证：配置基线、漏洞状态

5. 穿行测试 (Walk-through)
   └── 跟踪一个具体操作从开始到结束的全过程

6. 抽样检查
   └── 随机抽取样本验证 → 建立统计置信度
```

### 5.4 审计报告要素

```
一份合格的审计报告包含：

├── 审计目的和范围
├── 审计标准
├── 审计方法和时间
├── 总体评价
├── 发现项清单
│   ├── 严重不符合项（立即整改）
│   ├── 一般不符合项（限期整改）
│   └── 观察项（持续关注）
├── 整改建议
└── 上次审计问题关闭情况
```

## 七、持续合规的关键挑战

### 7.1 合规管理的常见误区

```
1. "认证通过 = 安全"
   └── 认证只是起点，安全是持续的过程

2. "有制度 = 在合规"
   └── 制度要被执行、被验证、被审计

3. "上了工具 = 合规了"
   └── 工具是手段，不是目的

4. "合规是安全部门的事"
   └── 合规需要全员参与，业务部门是第一道防线

5. "等保、ISO 27001 选一个就行"
   └── 等保是法定要求、ISO 是国际认证，不矛盾
      可以"等保 + ISO"双体系融合
```

### 7.2 等保与 ISO 27001 的融合路径

```
双体系融合的关键策略：

1. 统一管理手册
   └── 一份管理体系手册，同时满足等保和 ISO 27001 要求

2. 共用风险评估
   └── 等保定级 + ISO 27001 风险评估 → 统一风险清单

3. 映射控制措施
   └── 等保三级控制点 ↔ ISO 27001 附录 A 控制项
       做一份映射表

4. 合并审计检查
   └── 内部审计同时检查等保和 ISO 合规状态

5. 统一文档体系
   └── 四级文档架构内，每份文档标注"等保√ / ISO√"
```

## 八、CISP 考试要点速查

### 核心概念

| 概念 | 一句话 |
|------|--------|
| 三道防线 | 业务做、安全管、审计查 |
| PDCA | 策划→实施→检查→改进 |
| ISO 27001 | 信息安全管理体系（可认证） |
| ISO 27002 | 控制措施实施指南（不认证） |
| 差距分析 | 对比现状与标准要求 |
| 不符合项 | 证据不满足标准要求 |

### 关键对照

| 对比项 | 等保 2.0 | ISO 27001 |
|--------|----------|-----------|
| 性质 | 强制 | 自愿 |
| 核心要求 | GB/T 22239 | ISO 27001 |
| 实施步骤 | 定级→建设→测评 | 风险评估→控制→认证 |
| 证书内容 | 测评报告 | 认证证书 |

### 常见考试陷阱

1. ~~ISO 27002 可以做认证~~ → 只有 **ISO 27001** 可以做认证
2. ~~内审应隶属于安全部门~~ → 内审应**独立于**安全部门（向审计委员会汇报）
3. ~~差距分析 = 合规审计~~ → 差距分析是自查，审计是独立检查
4. ~~制度越多越合规~~ → 制度要被执行和验证才有效
5. ~~合规管理的目标是"不出事"~~ → 目标是"持续改进"，不是零风险
