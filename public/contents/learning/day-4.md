## 一、恶意软件总览与分类框架

### 1.1 什么是恶意软件

恶意软件（Malware = Malicious + Software）是指**任何以破坏、干扰、窃取或未经授权访问计算机系统为目的的软件**。它可以是一个可执行程序、脚本、宏代码，甚至是可以执行恶意逻辑的合法工具。

### 1.2 核心分类对比表

| 类别 | 自我复制 | 需要宿主 | 主要目的 | 代表案例 |
| :--- | :--- | :--- | :--- | :--- |
| **病毒** | ✅ 是 | ✅ 需要 | 破坏系统、传播 | CIH、宏病毒 |
| **蠕虫** | ✅ 是（独立） | ❌ 不需要 | 快速传播、占用带宽 | Conficker、CodeRed |
| **木马** | ❌ 否 | ❌ 不需要 | 后门、窃密 | Zeus、DarkComet |
| **勒索软件** | 部分可以 | ❌ 不需要 | 加密文件索要赎金 | WannaCry、LockBit |
| **间谍软件** | ❌ 否 | ❌ 不需要 | 窃取信息、监控 | FinFisher、Pegasus |
| **Rootkit** | ❌ 否 | ✅ 需要 | 隐藏自身和攻击活动 | Sony BMG Rootkit |
| **逻辑炸弹** | ❌ 否 | ❌ 不需要 | 条件触发破坏 | 员工离职"定时炸弹" |

> **🔑 高分考点**：病毒 vs 蠕虫是必考题。核心区别——病毒需要**宿主文件**，蠕虫是**独立程序**。

## 二、病毒（Virus）详解

### 2.1 核心特征

| 特征 | 说明 |
| :--- | :--- |
| **依附性** | 必须附着在合法程序或文件中 |
| **潜伏性** | 可长期潜伏，等待触发条件 |
| **感染性** | 能够感染其他文件 |
| **破坏性** | 可能删除文件、格式化磁盘、破坏引导区 |

### 2.2 病毒分类

| 类型 | 感染对象 | 特点 |
| :--- | :--- | :--- |
| **文件型病毒** | .exe、.dll、.com 等可执行文件 | 运行时感染其他文件 |
| **宏病毒** | Office 文档（Word/Excel） | 利用 VBA 宏功能 |
| **引导区病毒** | 磁盘主引导记录（MBR） | 开机时先于系统加载 |
| **脚本病毒** | .vbs、.js、.bat 等脚本 | 通过邮件或网页传播 |
| **多态病毒** | 可改变自身代码 | 逃避免疫特征检测 |

> **⚠️ 考试陷阱**：宏病毒虽然寄宿在 Office 文件，但它**利用的是应用程序的宏功能**，而非系统漏洞。禁用宏是最经济的防护手段。

## 三、蠕虫（Worm）详解

### 3.1 与病毒的关键区别

```
病毒：需要用户执行被感染的文件 → 被动传播
蠕虫：自动扫描网络，主动传播 → 无需人工干预
```

### 3.2 里程碑蠕虫事件

| 蠕虫 | 年份 | 传播方式 | 影响 |
| :--- | :--- | :--- | :--- |
| **Morris Worm** | 1988 | Unix sendmail + finger 漏洞 | 感染互联网 10% 主机，催生 CERT |
| **CodeRed** | 2001 | IIS 缓冲区溢出 | 数小时感染 35 万服务器 |
| **SQL Slammer** | 2003 | SQL Server 解析漏洞 | 10 分钟感染 7.5 万主机 |
| **Conficker** | 2008 | MS08-067 漏洞 + 密码猜测 | 感染超千万台计算机 |

> **💡 知识巧记**：" Morris 开先河，CodeRed 半小时，Slammer 十分钟，Conficker 千万台——传播越来越快，影响越来越大。"

## 四、木马（Trojan）详解

### 4.1 核心特征

木马的特点是**伪装**——它看起来是合法有用的软件，但暗藏恶意功能。名称来源于特洛伊木马。

| 特征 | 说明 |
| :--- | :--- |
| **欺骗性** | 伪装成正常软件（游戏、工具、破解补丁） |
| **非复制性** | 不能自我复制，需要用户主动下载安装 |
| **后门性** | 通常建立远程控制通道 |
| **多功能性** | 兼具窃密、破坏、代理等多种功能 |

### 4.2 常见木马类型

| 类型 | 功能 | 示例 |
| :--- | :--- | :--- |
| **远程控制木马（RAT）** | 远程操控受害主机 | DarkComet、NanoCore |
| **窃密木马** | 窃取账号密码、浏览器信息 | Zeus、SpyEye |
| **下载者（Downloader）** | 下载并安装更多恶意软件 | Emotet 早期版本 |
| **银行木马** | 针对性窃取网上银行信息 | Zeus、Dridex |
| **后门（Backdoor）** | 绕过认证，提供持续访问 | 各种 Webshell |

## 五、勒索软件（Ransomware）深度分析

### 5.1 攻击流程

```
入侵 → 横向移动 → 权限提升 → 加密文件 → 索要赎金 → (付款/恢复)
  ↓        ↓          ↓          ↓          ↓
漏洞利用  内网扩散   获取高权限  高强度加密   比特币等加密货币
```

### 5.2 重大勒索软件事件

| 名称 | 年份 | 传播方式 | 特点 |
| :--- | :--- | :--- | :--- |
| **WannaCry** | 2017 | EternalBlue 漏洞（永恒之蓝） | 全球爆发，150 国家受影响 |
| **NotPetya** | 2017 | 伪装成勒索，实为破坏性攻击 | 不可恢复，目的为破坏 |
| **Ryuk** | 2018+ | 定向攻击大型组织 | 赎金额度极高（数百万美元） |
| **Colonial Pipeline** | 2021 | 关键基础设施攻击 | 导致美国东海岸燃油中断 |

### 5.3 勒索软件即服务（RaaS）

现代勒索软件已形成完整产业链：

```
开发团队（提供勒索软件平台）
    ↕
加盟者（发起攻击）→ 得手后按比例分成
    ↕
赎金处理（加密货币洗钱）
```

> **🔑 高分考点**：RaaS 模式降低了攻击门槛，是近年来勒索攻击激增的核心原因。

### 5.4 防御策略

| 策略 | 具体措施 |
| :--- | :--- |
| **3-2-1 备份原则** | 3 份数据、2 种介质、1 份离线 |
| **漏洞管理** | 及时修补关键漏洞（如 EternalBlue） |
| **最小权限** | 限制用户和管理员权限范围 |
| **网络隔离** | 网段划分、RDP 限制 |
| **安全意识** | 识别钓鱼邮件和恶意附件 |

## 六、其他恶意软件类型

### 6.1 间谍软件（Spyware）

| 子类型 | 功能 |
| :--- | :--- |
| **键盘记录器（Keylogger）** | 记录所有键盘输入 |
| **屏幕截图** | 定期截屏回传 |
| **浏览器劫持** | 修改主页、注入广告、追踪浏览 |
| **信息窃取** | 收集文件、邮件、聊天记录 |

### 6.2 Rootkit

Rootkit 是最隐蔽的恶意软件类型，通过修改操作系统内核或驱动程序来隐藏自身：

| 类型 | 运行位置 | 特点 |
| :--- | :--- | :--- |
| **用户态 Rootkit** | 应用层 | 替换系统命令（如隐藏进程） |
| **内核态 Rootkit** | 内核层 | 修改内核，极难检测 |
| **固件 Rootkit** | BIOS/UEFI/硬件 | 重装系统也无法清除 |

### 6.3 无文件恶意软件（Fileless Malware）

不写入磁盘，直接在内存中运行，利用 PowerShell、WMI 等合法工具执行恶意逻辑：

```
攻击者 → 钓鱼邮件/漏洞利用 → PowerShell 内存加载恶意代码 → C2 通信
（全程不落盘，传统杀毒软件难以检测）
```

## 七、恶意软件传播方式

### 7.1 传播向量分析

| 传播方式 | 占比估算 | 技术要求 | 防护难度 |
| :--- | :--- | :--- | :--- |
| **电子邮件附件** | ~50% | 低 | 中（需用户配合） |
| **漏洞利用** | ~25% | 高 | 高（需及时补丁） |
| **恶意网站/水坑攻击** | ~10% | 中 | 中（需要 URL 过滤） |
| **USB/可移动设备** | ~5% | 低 | 中（需设备管控） |
| **供应链投毒** | ~5% | 高 | 极高（需供应商管理） |
| **社会工程即时通信** | ~5% | 低 | 中（需安全意识） |

> **💡 知识巧记**："邮件钓鱼是老大，漏洞利用排第二，U 盘插入要小心，供应链攻击最难防。"

## 八、恶意软件分析与检测技术

### 8.1 分析方法

| 方法 | 说明 | 优缺点 |
| :--- | :--- | :--- |
| **静态分析** | 不运行，分析文件结构和代码 | 安全但容易被混淆绕过 |
| **动态分析** | 在沙箱中运行，观察行为 | 有效但可能被反虚拟化检测 |
| **逆向工程** | 反汇编/反编译，逐条分析 | 最深入但耗时、门槛高 |

### 8.2 检测技术演进

```
特征码检测 → 启发式分析 → 行为检测 → AI/ML 检测 → EDR/XDR
（传统杀毒） （模式匹配）  （行为基线）  （算法模型）  （终端检测响应）
```

| 代际 | 技术 | 能力 |
| :--- | :--- | :--- |
| 第一代 | 特征码匹配 | 检测已知恶意软件 |
| 第二代 | 启发式分析 | 检测已知变种 |
| 第三代 | 行为检测 | 检测未知恶意行为 |
| 第四代 | AI/ML | 大数据驱动的预测 |
| 第五代 | EDR/XDR | 终端检测+响应+溯源 |

## 九、多层防御体系

### 9.1 纵深防御矩阵

| 防御层 | 措施 | 工具/技术 |
| :--- | :--- | :--- |
| **边界防御** | 邮件过滤、Web 过滤 | 邮件网关、代理服务器 |
| **网络防御** | 网络流量检测、NIDS | IDS、IPS、防火墙 |
| **终端防御** | 防病毒、HIDS、应用程序白名单 | EDR、杀毒软件 |
| **数据防御** | 备份、访问控制、加密 | 备份系统、DLP |
| **人员防御** | 安全意识培训、钓鱼演练 | 培训平台、模拟钓鱼 |

### 9.2 感染后应急响应

```
1. 隔离 → 断开网络连接，防止横向扩散
2. 识别 → 确定恶意软件类型和感染范围
3. 遏制 → 关闭受影响系统，阻止进一步传播
4. 清除 → 删除恶意软件，修复系统
5. 恢复 → 从干净备份恢复数据
6. 复盘 → 分析入侵路径，改进防御措施
```
