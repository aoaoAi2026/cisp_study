## 一、第一阶段：通信安全（1940s-1960s）

### 1.1 时代背景

二战期间，军事通信的保密需求催生了现代密码学。这个阶段的核心特征是 **"通信保密"**——确保信息在传输过程中不被敌方截获和破译。

| 维度 | 特征 |
| :--- | :--- |
| 安全焦点 | 通信保密（Communication Security） |
| 核心技术 | 密码学、加密机 |
| 主导力量 | 军方、情报机构 |
| 威胁来源 | 敌对国家情报机关 |

### 1.2 里程碑事件

```
1945  ── 二战结束，Enigma 密码机破译改变战争走向
1949  ── 香农发表《通信的数学理论》，奠定信息论基础
1949  ── 香农发表《保密系统的通信理论》，现代密码学诞生
1950s ── 美国 NSA（国家安全局）成立
1960s ── ARPANET 项目启动，互联网前身
```

> **🔑 高分考点**：1949 年香农《保密系统的通信理论》标志着**现代密码学的诞生**，将密码学从艺术变成科学。

### 1.3 核心思想

这一阶段的保护理念是 **"只有授权的人才能看到信息"**。通过加密技术确保即使信息被截获，也无法被破译。安全措施主要围绕加密算法的研发和密钥管理。

## 二、第二阶段：计算机安全（1970s-1980s）

### 2.1 时代背景

随着计算机的普及和大型机/小型机的出现，安全问题从通信层面扩展到计算机系统层面。多用户操作系统带来了**访问控制**和**资源隔离**的新需求。

| 维度 | 特征 |
| :--- | :--- |
| 安全焦点 | 计算机系统安全、访问控制 |
| 核心技术 | 操作系统安全、数据库安全 |
| 主导力量 | 政府、计算机厂商、标准化组织 |
| 标志性概念 | TCSEC 橘皮书、Bell-LaPadula 模型 |

### 2.2 关键里程碑

| 年份 | 事件 | 意义 |
| :--- | :--- | :--- |
| 1971 | ARPANET 首次远程连接 | 网络安全的原点 |
| 1973 | Bell-LaPadula 模型提出 | 形式化访问控制模型的开端 |
| 1976 | Diffie-Hellman 密钥交换 | 公钥密码学诞生 |
| 1977 | DES 成为美国联邦标准 | 第一个公开的加密标准 |
| 1978 | RSA 算法公开 | 实用公钥加密体系 |
| 1983 | TCSEC（橘皮书）发布 | 第一个计算机安全评估标准 |
| 1988 | Morris 蠕虫事件 | 第一个引起广泛关注的网络攻击 |

### 2.3 TCSEC 橘皮书详解

TCSEC（Trusted Computer System Evaluation Criteria）是美国国防部发布的**第一个计算机安全评估标准**，按安全性从低到高分为四个等级：

```
D ── 最小保护（Minimal Protection）
  └── 不满足任何更高等级

C ── 自主保护（Discretionary Protection）
  ├── C1：自主安全保护
  └── C2：受控访问保护（商用系统常见）

B ── 强制保护（Mandatory Protection）
  ├── B1：标记安全保护
  ├── B2：结构化保护
  └── B3：安全域

A ── 验证保护（Verified Protection）
  └── A1：验证设计（理论上最高）
```

> **💡 知识巧记**："D 无保护，C 自决断，B 强制标，A 要验证——等级越高越难达到。"

## 三、第三阶段：信息系统安全（1990s）

### 3.1 时代背景

90 年代，互联网开始商业化，安全从"单机保护"扩展到**信息系统整体安全**。人们认识到安全不仅仅是技术问题，更是**管理和技术相结合**的体系问题。

### 3.2 重要进展

| 年份 | 事件 | 意义 |
| :--- | :--- | :--- |
| 1991 | 欧洲发布 ITSEC | 首次区分功能性+保障性 |
| 1991 | PGP（Pretty Good Privacy）发布 | 邮件加密的民主化 |
| 1993 | 美国 NIST 提出 RBAC 模型 | 基于角色的访问控制 |
| 1995 | BS 7799 发布 | 信息安全管理体系前身 |
| 1996 | ISO 13335 发布 | IT 安全管理指南 |
| 1999 | 通用准则 CC（ISO 15408）发布 | 国际统一的安全评估标准 |

### 3.3 ITSEC vs TCSEC

| 对比维度 | TCSEC（美国） | ITSEC（欧洲） |
| :--- | :--- | :--- |
| 发布年份 | 1983/1985 | 1991 |
| 评估对象 | 仅操作系统 | 操作系统+应用+网络设备 |
| 安全属性 | 仅机密性 | 机密性+完整性+可用性 |
| 评估维度 | 单一功能级 | 功能级（F）+ 保障级（E）分离 |

## 四、第四阶段：网络安全（2000s-2010s）

### 4.1 时代特征

互联网全面普及，电子商务、社交网络、移动互联网蓬勃发展。攻击从"炫技"演变为**有组织的犯罪活动**。安全防护从边界防护转向**纵深防御+检测响应**。

| 维度 | 特征 |
| :--- | :--- |
| 安全焦点 | 网络攻防、数据保护 |
| 威胁特征 | 有组织犯罪、APT、国家级攻击 |
| 核心技术 | 防火墙、IDS/IPS、SIEM、DLP |
| 法律法规 | 各国开始信息安全立法 |

### 4.2 重大事件

```
2001  ── 9·11 事件推动信息安全战略地位提升
2001  ── CodeRed 蠕虫，数小时感染 35 万服务器
2003  ── SQL Slammer 蠕虫，10 分钟内感染 7.5 万主机
2005  ── ISO 27001（原 BS 7799-2）正式发布
2007  ── 爱沙尼亚遭受大规模网络攻击（"第一次网络战争"）
2008  ── Conficker 蠕虫，感染超千万台计算机
2010  ── Stuxnet 震网蠕虫，首次国家级"网络武器"
2013  ── 斯诺登棱镜门事件
2014  ── 索尼影业攻击事件
2015  ── OPM（美国人事管理局）数据泄露，2150 万人受影响
2017  ── WannaCry 勒索软件全球爆发
2017  ── Equifax 1.43 亿用户数据泄露
```

### 4.3 安全防护体系演进

```
单点防护 → 纵深防御 → 自适应安全 → 主动威胁狩猎
（防火墙）  （多层叠加） （持续监控）   （威胁情报驱动）

2000s              2010s             2015s             2020s
```

## 五、第五阶段：信息化安全（2020s至今）

### 5.1 核心趋势

| 趋势 | 说明 | 安全挑战 |
| :--- | :--- | :--- |
| **云原生** | 容器、Serverless、多云混合 | 配置错误、共享责任模型、API 安全 |
| **AI/ML 双刃剑** | AI 驱动安全检测，也驱动攻击 | 深度伪造、AI 生成钓鱼、对抗样本 |
| **零信任** | "永不信任，始终验证" | 实施复杂度高，传统架构改造难 |
| **隐私计算** | 联邦学习、同态加密 | 计算效率低，标准化不足 |
| **量子威胁** | 量子计算机威胁现有密码体系 | 后量子密码学迁移 |
| **供应链安全** | 开源组件、第三方服务 | 依赖漏洞传递、SolarWinds 模式攻击 |

### 5.2 零信任架构（Zero Trust Architecture）

```
┌────────────────────────────────────────────┐
│            零信任核心三原则                  │
├────────────────────────────────────────────┤
│  ① 永不信任，始终验证（Never Trust, Always  │
│     Verify）                                │
│  ② 最小权限原则（Least Privilege）          │
│  ③ 假设已被入侵（Assume Breach）            │
└────────────────────────────────────────────┘
```

> **🔑 高分考点**：零信任不是一种产品，而是一种**安全理念和架构设计原则**。其关键组件包括：微隔离（Micro-Segmentation）、新一代身份管理（IAM）、持续监控与分析。

## 六、安全评估标准演进史

### 6.1 国际标准演进路线

```
1983 TCSEC（橘皮书）—— 美国
   ↓
1991 ITSEC —— 欧洲
   ↓
1999 CC（通用准则，ISO 15408）—— 国际统一
   ↓
2005 ISO 27001 —— 信息安全管理体系
   ↓
2014 NIST CSF —— 网络安全框架
```

### 6.2 通用准则（CC）评估保证级别

| 级别 | EAL | 说明 |
| :--- | :--- | :--- |
| EAL1 | 功能测试 | 基本测试 |
| EAL2 | 结构测试 | 开发者测试+配置管理 |
| EAL3 | 系统测试和检查 | 安全工程方法 |
| EAL4 | 系统设计、测试和评审 | 商用产品常见级别 |
| EAL5 | 半形式化设计和测试 | 高安全保障 |
| EAL6 | 半形式化验证设计和测试 | 极高风险场景 |
| EAL7 | 形式化验证设计和测试 | 最高安全保障（理论级别） |

> **💡 知识巧记**："TCSEC 看机密，ITSEC 看功能+保障，CC 统一评估 EAL，ISO 27001 管体系——标准越来越全面。"

## 七、中国信息安全发展里程碑

| 年份 | 事件 | 意义 |
| :--- | :--- | :--- |
| 1994 | 中国全功能接入互联网 | 网络安全起点 |
| 1999 | GB 17859 计算机信息系统安全等级划分准则 | 中国 TCSEC |
| 2007 | 等保 1.0（公通字[2007]43号） | 等级保护制度建立 |
| 2014 | 中央网络安全和信息化领导小组成立 | 网信事业顶层设计 |
| 2017 | 《网络安全法》正式施行 | 第一部基础性法律 |
| 2019 | 等保 2.0（GB/T 22239-2019） | 扩展至云计算、物联网 |
| 2021 | 《数据安全法》《个人信息保护法》 | 数据治理法律体系 |

## 八、标志性安全事件年鉴

### 8.1 按类型分类的重大事件

| 事件类型 | 代表性事件 | 年份 | 影响与教训 |
| :--- | :--- | :--- | :--- |
| **蠕虫爆发** | Morris Worm | 1988 | 第一个引起关注的网络攻击 |
| **国家级 APT** | Stuxnet | 2010 | 网络武器破坏物理设施 |
| **勒索软件** | WannaCry | 2017 | 利用 NSA 泄露的永恒之蓝 |
| **数据泄露** | Equifax | 2017 | 漏洞未及时修补的代价 |
| **供应链攻击** | SolarWinds | 2020 | 软件供应链安全警钟 |
| **日志漏洞** | Log4j (Log4Shell) | 2021 | 开源组件安全的脆弱性 |

### 8.2 每次事件的共同教训

1. **漏洞管理**不及时是大多数事件的直接原因
2. **纵深防御**的缺失让攻击者轻易扩大战果
3. **安全监控**不足导致事件长时间未被发现
4. **应急响应**能力决定损失大小

## 九、发展趋势与未来挑战

### 9.1 五个关键趋势

```
1. AI 驱动的自适应安全 → 从人工研判到智能自动化
2. 安全左移（Shift Left）→ 从运维阶段提前到开发和设计阶段
3. 安全即服务（SECaaS）→ 云端交付安全能力
4. 隐私增强技术（PETs）→ 平衡数据利用与隐私保护
5. 后量子密码学（PQC）→ 应对量子计算威胁
```

### 9.2 给安全从业者的启示

| 启示 | 说明 |
| :--- | :--- |
| **技术是工具，不是答案** | 任何技术都需要配套的流程和人员 |
| **安全是持续过程** | 没有"一劳永逸"的安全方案 |
| **理解攻击者视角** | 防守需要了解攻击的思维和方法 |
| **基础不牢，地动山摇** | 防火墙、补丁、备份永远是第一道防线 |

### 🎯 考试陷阱提醒

| 陷阱 | 正确理解 |
| :--- | :--- |
| "TCSEC 是第一个国际标准" | TCSEC 是美国的，ITSEC 是欧洲的，CC 才是国际标准 |
