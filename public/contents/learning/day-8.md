```
    └── 跨境提供规则

    └── 关键信息基础设施识别、保护、监管
```

## 二、《网络安全法》深度解读

### 2.1 立法目的与适用范围

**立法目的（第1条）**：
1. 保障网络安全
2. 维护网络空间主权和国家安全
3. 保护社会公共利益
4. 保护公民、法人合法权益
5. 促进经济社会信息化健康发展

**适用范围**：在中华人民共和国境内建设、运营、维护和使用网络，以及网络安全的监督管理。

### 2.2 网络运营者的安全义务（CISP 考试重点）

```
网络安全法规定的核心义务体系：

├── 安全保护义务（第21条）
│   ├── 制定内部安全管理制度和操作规程
│   ├── 确定网络安全负责人
│   ├── 采取防范计算机病毒和网络攻击的技术措施
│   ├── 监测、记录网络运行状态和网络安全事件
│   ├── 留存网络日志不少于六个月 ← 考点！
│   └── 采取数据分类、重要数据备份和加密等措施
│
├── 个人信息保护义务（第40-44条）
│   ├── 建立健全用户信息保护制度
│   ├── 收集使用个人信息须经用户同意
│   ├── 不得泄露、篡改、毁损个人信息
│   └── 发生信息泄露须立即采取补救措施并告知用户
│
├── 安全事件报告义务（第25条）
│   └── 发生网络安全事件时立即启动应急预案
│       → 采取补救措施
│       → 向有关主管部门报告
│
├── 技术支持和协助义务（第28条）
│   └── 为公安机关、国安机关依法维护国家安全
│       和侦查犯罪的活动提供技术支持和协助
│
└── 实名制义务（第24条）
    └── 为用户提供信息发布、即时通讯等服务时
        → 要求用户提供真实身份信息
```

### 2.3 关键条款速记

| 条款 | 内容 | 关键词 |
|------|------|--------|
| 第21条 | 网络安全等级保护制度 | 等级保护 |
| 第24条 | 网络实名制 | 真实身份 |
| 第25条 | 安全事件应急预案和报告 | 应急预案 |
| 第31条 | CII 安全保护义务 | 关键基础设施 |
| 第37条 | 数据本地化存储 | 境内存储 |
| 第40-44条 | 个人信息保护 | 明示收集+用户同意 |
| 第47条 | 违法信息处置 | 停止传输+消除+报告 |

### 2.4 日志留存要求的深层含义

> "采取监测、记录网络运行状态、网络安全事件的技术措施，并**按照规定留存相关的网络日志不少于六个月**。" ——《网络安全法》第21条

**为什么是六个月？**
- 安全事件溯源的最小周期
- APT 攻击的潜伏期通常为 3-6 个月
- 配合执法机关的侦查周期
- 实际落地中建议至少保存 **12 个月**

## 三、《数据安全法》要点

### 3.1 数据分类分级制度

```
数据分类分级的三个层次：

第一级：一般数据
├── 定义：不涉及国家安全、公共利益、个人权益的数据
├── 管控：常规安全管理
└── 示例：公开的企业宣传资料

第二级：重要数据
├── 定义：一旦遭到篡改、破坏、泄露可能危害国家安全、
│         经济运行、社会稳定、公共健康和安全的数据
├── 管控：重点保护，风险评估
└── 示例：基因数据、地理信息、重要行业运行数据

第三级：核心数据
├── 定义：关系国家安全、国民经济命脉、重要民生、
│         重大公共利益的数据
├── 管控：最高级别安全保护
└── 示例：国家机密级数据、核设施数据
```

### 3.2 数据安全保护义务

| 义务 | 具体要求 |
|------|----------|
| 建立健全全流程数据安全管理制度 | 覆盖收集、存储、使用、加工、传输、提供、公开等环节 |
| 组织开展数据安全教育培训 | 全员覆盖，定期开展 |
| 采取相应技术措施 | 数据加密、脱敏、访问控制、审计 |
| 数据安全风险评估报告 | 定期开展，形成报告 |
| 数据安全事件应急处置 | 发生泄露立即处置 + 通知用户 + 报告主管部门 |

### 3.3 数据出境安全管理

```
数据出境的三种评估路径：

1. 安全评估（强制情形）
   ├── 重要数据出境
   ├── CII 运营者收集的个人信息出境
   ├── 处理100万人以上个人信息者 → 个人信息出境
   └── 1年内累计出境10万人以上个人信息或1万人以上敏感信息

2. 标准合同备案
   └── 非上述情形，可与境外接收方签订标准合同并备案

3. 保护认证
   └── 通过国家网信部门认可的专业机构认证
```

## 四、《个人信息保护法》框架

### 4.1 "告知-同意"核心机制

```
个人信息处理的基本原则（第5-9条）：

┌──────────────────────────────────────┐
│  1. 合法、正当、必要、诚信原则        │
│  2. 明确、合理的目的                 │
│  3. 最小必要（限于实现目的的最小范围） │
│  4. 公开、透明                       │
│  5. 质量保证（准确、及时更新）        │
│  6. 安全保障（防止泄露、篡改、丢失）   │
│  7. 最短时间保存                     │
└──────────────────────────────────────┘

告知（透明度）                      同意（自主性）
┌────────────────┐              ┌────────────────┐
│ · 处理者的名称  │     ⇄       │ · 自愿、明确    │
│ · 处理目的      │  (告知是同意  │ · 单独同意(敏感)│
│ · 处理方式      │    的前提)    │ · 可撤回        │
│ · 信息种类      │              │ · 非捆绑        │
│ · 保存期限      │              │ · 不满14岁的    │
│ · 行使权利的方式│              │   监护人同意    │
└────────────────┘              └────────────────┘
```

### 4.2 个人权利清单

| 权利 | 含义 | 行使场景 |
|------|------|----------|
| **知情权** | 知道谁在处理、为什么处理、怎么处理 | 收集信息前必须告知 |
| **决定权** | 有权限制或拒绝处理 | "不同意"按钮应同等显著 |
| **查阅复制权** | 查到自己的信息并能带走 | 数据可携带 |
| **更正权** | 信息有误可以要求改正 | 个人信息不准确 |
| **删除权** | "被遗忘权"——要求删除 | 目的已实现、撤回同意 |
| **可携带权** | 转移到指定对象 | 更换服务商 |

### 4.3 敏感个人信息

**定义**：一旦泄露或非法使用，容易导致自然人的人格尊严受到侵害或人身、财产安全受到危害的个人信息。

**范围**：

```
敏感个人信息类型（第28条）：

├── 生物识别信息 (人脸、指纹、声纹、虹膜)
├── 宗教信仰信息
├── 特定身份信息 (民族、政治观点)
├── 医疗健康信息
├── 金融账户信息
├── 行踪轨迹信息
└── 不满十四周岁未成年人的个人信息

特殊规则：
├── 只有具有特定目的和充分必要性方可处理
├── 必须取得个人的单独同意 ← 不能与其他同意捆绑
├── 必须告知处理敏感信息的必要性
└── 必须进行个人信息保护影响评估
```

### 4.4 个人信息保护影响评估 (PIA)

```
必须进行 PIA 的情形（第55条）：

1. 处理敏感个人信息
2. 利用个人信息进行自动化决策
3. 委托处理、提供、公开个人信息
4. 向境外提供个人信息
5. 其他对个人权益有重大影响的活动

PIA 报告内容：
├── 处理目的、方式
├── 对个人权益的影响及风险
└── 保护措施及其有效性

保存期限：至少 3 年
```

## 五、行政法规与部门规章

### 5.1 《关键信息基础设施安全保护条例》

**CII 的定义**（第2条）：公共通信和信息服务、能源、交通、水利、金融、公共服务、电子政务、国防科技工业等重要行业和领域，以及其他一旦遭到破坏、丧失功能或数据泄露，可能严重危害国家安全、国计民生、公共利益的重要网络设施、信息系统等。

**CII 运营者的安全保护义务**：

```
核心义务体系：

├── 安全管理机构和负责人
│   └── 设立专门安全管理机构，指定安全管理负责人
│
├── 安全保护措施（三同步原则）
│   └── 安全保护措施与 CII 同步规划、同步建设、同步使用
│
├── 网络安全检测评估
│   └── 每年至少进行一次 ← 考点！
│
├── 重要数据境内存储
│   └── 运营中收集和产生的个人信息和重要数据应当在境内存储
│
├── 安全审查
│   └── 网络产品和服务可能影响国家安全的，应当申报审查
│
└── 应急演练
    └── 定期开展应急演练
```

### 5.2 《网络安全审查办法》

| 维度 | 内容 |
|------|------|
| **审查对象** | CII 运营者采购网络产品和服务；掌握100万+用户个人信息的平台赴国外上市 |
| **审查机构** | 网络安全审查办公室（设在中央网信办） |
| **审查内容** | 产品和服务的安全性、可控性、供应链安全、数据安全风险、被外国政府利用的可能 |
| **审查时限** | 初步审查 30 个工作日（特殊情况可延长 +15 天）→ 特别审查程序 90 个工作日 |

### 5.3 重要时间节点汇总

| 法律/法规 | 发布时间 | 施行时间 |
|-----------|----------|----------|
| 网络安全法 | 2016.11.7 | **2017.6.1** |
| 密码法 | 2019.10.26 | **2020.1.1** |

## 六、法律责任与违规后果

### 6.1 处罚力度对照表

| 法律 | 最高罚款 | 责任人处罚 |
|------|----------|-----------|
| 网络安全法 | 100万元 | 1-10万元 |
| 数据安全法 | **1000万元** | 5-20万元 |
| 个人信息保护法 | **5000万元或上年营业额5%** | 10-100万元 + 禁业 |
| GDPR (对比) | 2000万欧元或上年全球营业额4% | — |

### 6.2 三类法律责任

```
行政处罚（最常用）：
├── 警告、通报批评
├── 责令暂停相关业务、停业整顿
├── 吊销相关业务许可证或营业执照
├── 罚款（单位 + 直接负责人）
└── 对直接责任人禁业（个保法特有）

民事责任：
├── 侵害个人信息权益 → 损害赔偿
└── 举证责任倒置！← 个保法第69条
    （个人信息处理者须证明自己没有过错）

刑事责任：
└── 刑法第285条 × 网络安全法
    ├── 第285条：非法侵入计算机信息系统罪
    ├── 第286条：破坏计算机信息系统罪
    └── 第287条：利用计算机实施犯罪
```

### 6.3 个保法的"天价罚款"机制

```
阶梯式处罚：
├── 情节一般：100万元以下
├── 情节严重：5000万元或上年营业额5%以下 ← 取较高者
│
"情节严重" 的情形：
├── 违法处理大量个人信息
├── 违法处理敏感个人信息
├── 多次违法拒不改正
├── 造成严重后果
└── 使用非法手段

后果还包括：
├── 责令暂停相关业务
├── 吊销营业执照
├── 直接责任人 10-100 万元罚款
└── 直接责任人禁业一定期限
```

## 七、国际法规对比

| 维度 | 中国 | 欧盟 (GDPR) | 美国 |
|------|------|------------|------|
| **基础法律** | 三部专门法律 | 一部统一法规 | 行业分散立法 (HIPAA/GLBA/CCPA) |
| **适用范围** | 域内 + 域外处理中国数据 | 域内 + 域外处理欧盟数据 | 各州/行业不同 |
| **最高罚款** | 5000万或5% | 2000万欧元或4% | CCPA: $7500/次故意违规 |
| **数据跨境** | 安全评估 + 标准合同 | 充分性认定 + 标准条款 | 相对宽松 |
| **个人信息权利** | 7项权利 | 8项权利 | 州法差异大 |

### 常见考试陷阱

1. ~~网络安全法 2016.6.1 施行~~ → **2017.6.1** 施行（2016.11.7 通过）
2. ~~个保法最高罚款 1000 万~~ → **5000万或上年营业额5%**（取较高者）
3. ~~网络日志留存至少 3 个月~~ → **不少于 6 个月**
4. ~~所有数据都可以出境~~ → 重要数据和 CII 个人信息需安全评估
5. ~~敏感信息与非敏感信息处理规则相同~~ → 敏感信息需**单独同意**
