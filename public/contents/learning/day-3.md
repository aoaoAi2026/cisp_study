## 一、OSI 七层模型详解

### 1.1 为什么要分层

计算机网络采用分层的设计思想，核心优势是**解耦合**——每层只关心自己的职责，不关心其他层的实现细节。当某一层技术发生变化时，不影响其他层。

> **💡 关键理解**：分层的本质是"封装+抽象"。下层为上层提供服务，上层调用下层接口，层层递进。

### 1.2 七层模型逐层详解

| 层级 | 名称 | 数据单元 | 功能描述 | 典型协议/设备 | 安全关注点 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **7** | 应用层 | 消息 | 为用户应用提供网络服务接口 | HTTP、FTP、SMTP、DNS | Web 漏洞、API 安全、注入攻击 |
| **6** | 表示层 | 消息 | 数据格式转换、加密解密、压缩 | TLS/SSL、JPEG、ASCII | 弱加密算法、编码绕过 |
| **5** | 会话层 | 消息 | 建立、管理、终止会话 | NetBIOS、RPC、SIP | 会话劫持、会话固定 |
| **4** | 传输层 | 段/数据报 | 端到端可靠传输、流控、差错控制 | TCP、UDP | SYN Flood、端口扫描 |
| **3** | 网络层 | 数据包 | 路由选择、逻辑寻址、跨网段通信 | IP、ICMP、OSPF、BGP | IP 欺骗、路由攻击、DDoS |
| **2** | 数据链路层 | 帧 | 相邻节点可靠传输、MAC 寻址 | 以太网、WiFi、ARP、VLAN | ARP 欺骗、MAC 泛洪 |
| **1** | 物理层 | 比特 | 比特流传输、物理介质、电气特性 | 网线、光纤、无线电 | 物理访问、电磁泄露、搭线窃听 |

> **💡 知识巧记**："应表会传网链物——应用表示会话传输，网络链路跟物理。七层从上到下记，底层为上层服务。"

### 1.3 各层数据单元名称（必考）

```
应用层/表示层/会话层 → 消息（Message）
传输层（TCP）       → 段（Segment）
传输层（UDP）       → 数据报（Datagram）
网络层              → 数据包（Packet）
数据链路层           → 帧（Frame）
物理层              → 比特（Bit）
```

> **🔑 高分考点**：不同层的数据单元名称是常考选择题。——"消息段包帧比"（从上到下）。

## 二、TCP/IP 四层模型与对比

### 2.1 TCP/IP 四层模型

| 层级 | 典型协议 | 核心功能 | 对应 OSI 层 |
| :--- | :--- | :--- | :--- |
| **应用层** | HTTP、HTTPS、DNS、SSH、FTP、SMTP | 应用程序网络服务接口 | 5 + 6 + 7（应用+表示+会话） |
| **传输层** | TCP、UDP | 端到端通信、可靠性/实时性 | 4（传输层） |
| **网络层** | IP、ICMP、ARP、IGMP | 路由、寻址、逻辑地址 | 3（网络层） |
| **网络接口层** | 以太网、Wi-Fi、PPP | 物理传输、帧封装、MAC 寻址 | 1 + 2（物理+数据链路） |

### 2.2 OSI 七层 vs TCP/IP 四层

| 对比维度 | OSI 七层 | TCP/IP 四层 |
| :--- | :--- | :--- |
| **层数** | 7 | 4 |
| **上层差异** | 应用层/表示层/会话层分离 | 合并为一个"应用层" |
| **下层差异** | 数据链路层/物理层分离 | 合并为"网络接口层" |
| **定位** | 理论参考模型 | 实际工程模型 |
| **出现时间** | 先定义模型，后开发协议 | 先有协议，后总结模型 |
| **协议独立性** | 与具体协议无关 | 深度绑定 TCP/IP 协议族 |

> **🔑 高分考点**：OSI 七层是"理论参考模型"，TCP/IP 四层是"事实标准"。OSI 协议未普及，但七层模型的思想被广泛采用。

### 2.3 TLS 在模型中的位置

TLS（Transport Layer Security）是一个特殊的存在——它位于传输层和应用层之间，被称为"第 4.5 层"或"中间层"：

```
应用层（HTTP）
     ↕
  TLS 层（加密、认证、完整性）  ← 位于传输层之上、应用层之下
     ↕
传输层（TCP）
```

## 三、数据封装与解封装流程

### 3.1 发送端封装（自上而下逐层加头）

```
应用层数据
  → [TCP Header | 应用数据]                 ← 传输层添加 TCP 头
  → [IP Header | TCP Header | 数据]         ← 网络层添加 IP 头
  → [帧头 | IP Header | TCP Header | 数据 | 帧尾]  ← 链路层添加帧头帧尾
  → 物理层比特流
```

### 3.2 接收端解封装（自下而上逐层去头）

```
物理层收到比特流
  → 链路层去帧头帧尾，检查 CRC → [IP Header | TCP Header | 数据]
  → 网络层去 IP 头，根据协议号转发 → [TCP Header | 数据]
  → 传输层去 TCP 头，根据端口号→应用 → 应用数据
```

> **🔑 高分考点**：封装/解封装过程中，**每层只解析本层头部信息**，不查看上层数据内容。这是分层设计的核心优势。

## 四、关键网络协议速查

### 4.1 应用层协议

| 协议 | 端口 | 传输层 | 功能 | 安全版本 |
| :--- | :--- | :--- | :--- | :--- |
| HTTP | 80 | TCP | 超文本传输 | HTTPS (443/TCP) |
| DNS | 53 | TCP/UDP | 域名解析 | DNSSEC、DoH、DoT |
| FTP | 20/21 | TCP | 文件传输 | SFTP (22)、FTPS |
| SMTP | 25 | TCP | 邮件发送 | SMTPS (465) |
| SSH | 22 | TCP | 安全远程登录 | 自身即安全协议 |
| Telnet | 23 | TCP | 远程登录（明文，不安全） | 已废弃，用 SSH 替代 |
| DHCP | 67/68 | UDP | 动态 IP 分配 | DHCP Snooping |
| SNMP | 161/162 | UDP | 网络设备管理 | SNMPv3 |

> **💡 知识巧记**："Web 看 80/443，邮件 25 和 110，22 是安全壳，53 域名解，67/68 自动分 IP。"

### 4.2 网络层关键协议

| 协议 | 功能 | 安全风险 |
| :--- | :--- | :--- |
| **IP** | 逻辑寻址和路由 | IP 欺骗、源路由攻击 |
| **ICMP** | 错误报告和诊断（ping/traceroute） | ICMP Flood、Smurf 攻击 |
| **ARP** | IP 地址→MAC 地址映射 | ARP 欺骗（中间人攻击） |
| **IGMP** | 组播管理 | IGMP Flood |

## 五、各层常见攻击面全景

### 5.1 按 OSI 层分类的攻击矩阵

| 层级 | 典型攻击 | 攻击原理 | 防御措施 |
| :--- | :--- | :--- | :--- |
| **7. 应用层** | SQL 注入、XSS、CSRF | 恶意输入绕过程序逻辑 | WAF、参数化查询、输入验证 |
| **6. 表示层** | SSL Stripping、弱加密 | 降级攻击、利用弱密码套件 | 强制 HTTPS、禁用旧协议 |
| **5. 会话层** | 会话劫持、会话固定 | 窃取/固定 Session ID | 安全 Cookie、Session 重生成 |
| **4. 传输层** | SYN Flood、TCP 劫持 | 三次握手缺陷、序列号预测 | SYN Cookie、TCP 加密 |
| **3. 网络层** | IP 欺骗、Smurf、路由投毒 | 伪造源 IP、路由协议漏洞 | 入口过滤、RPF 检查、路由认证 |
| **2. 链路层** | ARP 欺骗、MAC 泛洪、VLAN 跳跃 | 伪造 ARP 应答、CAM 表溢出 | DAI、端口安全、PVLAN |
| **1. 物理层** | 搭线窃听、电磁泄露、设备破坏 | 物理接触、电磁辐射 | 物理隔离、屏蔽、监控 |

### 5.2 攻击分类：主动 vs 被动

| 分类 | 特征 | 代表攻击 | 检测难度 |
| :--- | :--- | :--- | :--- |
| **被动攻击** | 只监听不修改 | 网络嗅探、流量分析 | 很难检测 |
| **主动攻击** | 修改数据或系统状态 | DDoS、篡改、注入 | 可检测（有行为痕迹） |

## 六、网络攻击技术分类

### 6.1 拒绝服务攻击（DoS/DDoS）

| 类型 | 原理 | 防御思路 |
| :--- | :--- | :--- |
| **SYN Flood** | 大量 SYN 不完成握手 | SYN Cookie、连接限制 |
| **UDP Flood** | 大量 UDP 包耗尽带宽 | 流量清洗、速率限制 |
| **HTTP Flood** | 大量 HTTP 请求耗尽应用资源 | WAF、CC 防护、验证码 |
| **DNS Amplification** | 伪造源 IP 放大 DNS 查询 | 源地址验证、递归限制 |
| **NTP Amplification** | 同理利用 NTP 放大 | 禁用 monlist、访问控制 |

> **🔑 高分考点**：SYN Flood 利用了 TCP 三次握手的**半连接队列**溢出。防御核心是 SYN Cookie（不分配完整资源直到握手完成）。

### 6.2 中间人攻击（MITM）

```
正常通信：  A ────────────→ B
MITM攻击：  A ──→ [攻击者] ──→ B
            ←── 双方以为在直接通信 ──→
```

实现中间人的常见手段：ARP 欺骗、DNS 劫持、SSL Stripping、伪造 Wi-Fi 热点。

## 八、防火墙深度解析

### 8.1 防火墙演进

| 类型 | 工作原理 | OSI 层级 | 优缺点 |
| :--- | :--- | :--- | :--- |
| **包过滤防火墙** | 基于 IP/端口/协议五元组过滤 | 3-4 层 | 快但粗粒度，无状态 |
| **状态检测防火墙** | 维护连接状态表，动态允许返回流量 | 3-4 层 | 兼顾速度和安全，主流方案 |
| **应用代理防火墙** | 代理内外连接，深度检查应用数据 | 7 层 | 安全但慢，破坏端到端 |
| **下一代防火墙（NGFW）** | 状态检测 + 应用识别 + IPS + 威胁情报 | 3-7 层 | 综合防护，业界主流 |
| **WAF（Web 应用防火墙）** | 专用于 HTTP/HTTPS 流量检查 | 7 层 | SQL 注入、XSS 等 Web 攻击 |

### 8.2 防火墙默认策略

> **⚠️ 考试陷阱**：防火墙默认策略应该是**"默认拒绝"（Default Deny）**，即"没有明确允许的就是拒绝"，而非"默认允许，逐条禁止"。

### 8.3 DMZ 隔离区

```
         Internet
            │
         [防火墙]
         /      \
      DMZ      内网
   (Web/Mail)   (数据库/文件)
```

DMZ 是位于外网和内网之间的**缓冲区**，放置需要对外提供服务的服务器（如 Web、邮件），即使被攻破也无法直接访问内网核心资产。

> **🔑 高分考点**：DMZ 中的服务器一旦被攻破，**不能直接访问内网**，这是 DMZ 设计的核心价值。

## 九、IDS/IPS 技术比较

### 9.1 IDS vs IPS

| 对比维度 | IDS（入侵检测系统） | IPS（入侵防御系统） |
| :--- | :--- | :--- |
| **部署模式** | 旁路（镜像流量） | 串行（Inline） |
| **工作方式** | 检测+告警 | 检测+阻断 |
| **响应能力** | 被动（只报告不阻断） | 主动（实时阻断） |
| **网络影响** | 不影响网络性能 | 可能引入延迟 |
| **误报后果** | 只产生告警噪音 | 可能阻断正常业务 |
| **典型场景** | 合规审计、安全分析 | 实时攻击防护 |

### 9.2 检测技术

| 技术 | 原理 | 优缺点 |
| :--- | :--- | :--- |
| **特征匹配** | 匹配已知攻击签名 | 准确但只能检测已知攻击 |
| **异常检测** | 偏离正常基线即为异常 | 能检测未知攻击，误报率高 |
| **协议分析** | 检查协议合规性 | 发现异常协议行为 |

> **🔑 高分考点**：IDS 部署在**旁路**（不影响流量），IPS 部署在**串行**（流量必经之路）。这是两者最本质的区别。

## 十、高分考点与知识巧记

| 序号 | 考点 | 考试频率 | 难度 | 关键答案 |

| 7 | DMZ 的核心作用 | ⭐⭐⭐ | 低 | 内外网隔离缓冲区 |
| 8 | 常用端口号 | ⭐⭐⭐⭐⭐ | 低 | HTTP 80/HTTPS 443/DNS 53/SSH 22 |

### 💡 知识巧记口诀

> "七层模型要记牢：物数网传会表应。四层 TCP/IP：网络接口连网络，传送到应用。端口必须背：Web 80 和 443，DNS 53，25 发邮件，22 是安全壳。"

### 🎯 考试陷阱提醒

| 陷阱 | 正确理解 |
| :--- | :--- |
| "OSI 七层模型是实际使用的协议栈" | OSI 是理论模型，TCP/IP 四层才是实际使用的 |
| "防火墙默认允许所有流量，逐条封禁" | 安全最佳实践是**默认拒绝**，按需开放 |
| "TLS 工作在应用层" | TLS 工作在**传输层之上、应用层之下**（第 4.5 层） |
| "HTTP 端口是 8080" | HTTP 默认端口是 **80**，8080 只是常见替代端口 |

## 学习建议

1. **背诵 OSI 七层顺序**和每层的数据单元名称（基本每套题都有）
2. **牢记常用端口号**：HTTP(80)、HTTPS(443)、DNS(53)、SSH(22)、FTP(21)、SMTP(25)
3. **理解 IDS 和 IPS 的本质区别**：旁路 vs 串行，告警 vs 阻断
4. **掌握防火墙演变**：包过滤→状态检测→应用代理→NGFW
5. **阅读进阶参考**：`public/contents/knowledge/001-tcp-ip-deep.md` 深入了解 TCP/IP
