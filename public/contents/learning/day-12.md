# 网络安全审查制度：制度框架与企业合规实践

> **📘 文档定位**：CISP 考试法规域重点 | 难度：⭐⭐ | 预计阅读：20 分钟
>
> 网络安全审查制度是中国网络安全治理的"守门人"——它从供应链安全和数据安全两个维度，对可能影响国家安全的产品、服务和上市行为进行审查。本章帮你掌握审查制度的核心框架、触发条件、审查流程和企业应对策略。

## 一、制度背景与法律依据

### 1.1 立法演进

```
网络安全审查制度演进：

2017.6  《网络安全法》第35条
         └── 明确 CII 运营者采购网络产品和服务须审查

2020.6  首部《网络安全审查办法》施行

2021.7  "滴滴事件" — 审查办法修订的催化剂

2022.2  修订版《网络安全审查办法》施行
         └── 新增：超 100 万用户个人信息平台赴国外上市须审查

2022.7  滴滴被罚 80.26 亿元 — 史上最大数据安全处罚
```

### 1.2 法律依据

| 法律/法规 | 条款 | 核心要求 |
|-----------|------|----------|
| 《网络安全法》 | 第35条 | CII 采购网络产品和服务→安全审查 |
| 《数据安全法》 | 第24条 | 国家建立数据安全审查制度 |
| 《个人信息保护法》 | 第40条 | CII + 大量个人信息处理者的境外传输→安全评估 |
| 《网络安全审查办法》 | 全文 | 审查的具体程序和标准 |

### 1.3 审查机构

```
网络安全审查的组织架构：

中央网络安全和信息化委员会
    │
    ├── 批准网络安全审查结论
    │
    └── 网络安全审查办公室（设在中央网信办）
          │
          ├── 组织审查工作
          ├── 受理申报
          └── 发出审查结论
                │
                ├── 会同 13 个部委联合审查
                │   （发改委、工信部、公安部、国安部、
                │     财政部、商务部、人民银行等）
                │
                └── 可委托第三方技术机构测试评估
```

## 二、审查对象和触发条件

### 2.1 两种触发情形

```
情形一：CII 运营者采购网络产品和服务
├── 可能影响国家安全的 → 必须主动申报审查
├── 审查的是：采购行为本身 → 产品和服务投入使用后的安全风险
│
情形二：掌握 >100 万用户个人信息 → 赴国外上市
├── 网络平台运营者 → 必须主动申报审查
├── 审查的是：上市是否导致数据失控
└── 关键数字：100 万用户个人信息的处理量
```

### 2.2 什么是"网络产品和服务"

```
被审查的"网络产品和服务"包括：

├── 核心网络设备
│   路由器、交换机、服务器、存储设备
├── 高性能计算设备
│   超算中心、GPU 集群
├── 大型数据库和数据处理系统
│   金融交易系统、政务平台
├── 云计算服务
│   IaaS、PaaS、SaaS
├── 网络安全产品
│   防火墙、IDS/IPS、态势感知平台
├── 大型专用软件
│   工控系统、调度系统

不包括：普通办公软件、消费级产品
```

### 2.3 "掌握 100 万用户个人信息"的判断标准

```
"掌握"的含义：
├── 注册用户数 ≥ 100 万
├── 即使仅在境内运营，只要赴海外上市 → 触发审查
├── "个人信息"范围：依《个人信息保护法》第4条

"赴国外上市"：
├── 美国 (纽交所/纳斯达克)、港交所、新加坡等
├── 不包含 A 股 (上交所/深交所/北交所)
└── 包括直接 IPO、SPAC 合并、借壳上市等
```

## 三、审查流程与时限

### 3.1 审查流程图

```
网络安全审查流程：

运营者自主申报
    │
    ├─ 或
    │
网络安全审查办公室依职权启动审查
    │
    ├─ 或
    │
社会举报 → 审查办公室启动
    │
    ▼
┌──────────────────────┐
│  ① 材料接收与初审     │  10 个工作日
│     材料不全→退回补正  │
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  ② 初步审查          │  30 个工作日
│     一般情形→出结论   │  (可延长 15 个工作日)
│     复杂情形→特别审查  │
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  ③ 特别审查程序       │  90 个工作日
│     情况复杂的        │  (可延长)
│     → 中央网信委批准  │
└──────────────────────┘
```

### 3.2 时限汇总

| 阶段 | 时限 | 说明 |
|------|------|------|
| 材料初审 | **10 个工作日** | 材料不齐全可退回补正 |
| 初步审查 | **30 个工作日** | 可延长 15 个工作日 |
| 特别审查 | **90 个工作日** | 情况特别复杂的 |
| 总时长(一般) | 约 30 个工作日 | 不含补正时间 |
| 总时长(复杂) | 约 120+ 个工作日 | 不含补正和延长 |

> **注意**：审查期间，企业可以正常经营，但上市进程通常暂停。

### 3.3 审查结论

```
三种审查结论：

1. 通过审查
   └── 不影响国家安全 → 可继续采购或上市

2. 有条件通过
   └── 存在一定风险但不严重 → 附加限制条件后通过
       ├── 数据必须境内存储
       ├── 审计权限保留
       └── 后续持续报告义务

3. 不通过审查
   └── 危害国家安全 → 禁止采购或停止上市

未申报的后果：
├── 事后发现 → 可依法叫停
├── 已经上市 → 可要求停止使用/退市
└── 罚款 + 责任人追责
```

## 四、审查内容与标准

### 4.1 审查维度

```
网络安全审查的五大维度：

维度一：供应链安全
├── 产品和服务提供者的国别背景
├── 产品关键组件的来源可控性
├── 是否存在断供/禁运风险
└── 开放性和透明度（源代码开放程度）

维度二：非法控制风险
├── 是否存在后门/木马
├── 是否可以被远程控制
├── 产品固件/更新机制是否安全
└── 是否有隐藏的数据回传功能

维度三：数据安全风险
├── 核心数据、重要数据是否会被境外获取
├── 大量个人信息是否可能被泄露
├── 数据存储位置是否可控
└── 数据加密和访问控制能力

维度四：被外国政府利用的可能性
├── 产品服务提供者受外国法律管辖
│   (如美国 CLOUD Act → 可要求提供数据)
├── 供应商与外国政府、军队的关系
└── 是否有强制交付数据的法律义务

维度五：技术依赖评估
├── 技术是否自主可控
├── 替代品的可获得性
└── 技术断供的影响程度
```

### 4.2 风险等级判断矩阵

| 供应链可控 | 数据可控 | 无外国管制风险 | 审查结论倾向 |
|-----------|----------|---------------|-------------|
| ✅ 是 | ✅ 是 | ✅ 是 | 通过 |
| ⚠️ 部分 | ✅ 是 | ✅ 是 | 有条件通过 |
| ✅ 是 | ⚠️ 部分 | ✅ 是 | 有条件通过 |
| ❌ 否 | — | — | 不通过 |
| — | ❌ 否 | — | 不通过 |
| — | — | ❌ 否 | 不通过 |

## 五、企业合规应对策略

### 5.1 合规自查清单

```
企业应建立的安全审查合规体系：

前期准备
├── □ 是否属于 CII 运营者？查阅行业主管认定
├── □ 是否处理超过 100 万用户个人信息？
├── □ 是否有赴海外上市计划？
├── □ 采购的产品/服务是否涉及关键领域？

供应商管理
├── □ 供应商国别背景调查
├── □ 产品关键技术组件来源审查
├── □ 供应商合规承诺书
├── □ 产品安全检测报告

数据管控
├── □ 重要数据和个人信息是否境内存储？
├── □ 跨境传输是否已完成安全评估？
├── □ 数据处理全链路审计能力

申报准备
├── □ 产品/服务详细说明
├── □ 安全风险自评估报告
├── □ 供应商的安全合规证明
└── □ 数据安全影响评估报告
```

### 5.2 上市前合规要点

```
赴海外上市企业的专项合规：

1. 数据合规先行
   ├── 完成数据分类分级
   ├── 重要数据和个人信息境内存储
   ├── 完成数据出境安全评估（如适用）
   └── 建立数据安全管理制度

2. 网络安全审查申报
   ├── 提前 3-6 个月启动准备
   ├── 组织材料和证据
   └── 积极配合审查

3. 业务连续性预案
   ├── 考虑审查不通过的情况
   ├── 准备替代上市地点（如 A 股）
   └── 确保核心业务不因审查而中断
```

## 六、典型案例分析

### 6.1 案例一：滴滴事件（2021-2022）

```
时间线：
2021.6.30  滴滴在美国纽交所挂牌上市（未申报审查）
2021.7.2   网络安全审查办公室宣布对滴滴实施网络安全审查
2021.7.4   网信办通知下架"滴滴出行"App
2021.7.9   网信办下架 25 款滴滴系 App
2021.7.16  七部门联合进驻滴滴开展网络安全审查
2021.12.3  滴滴宣布启动纽交所退市
2022.7.21  网信办开出 80.26 亿元罚单

教训：
├── 掌握大量个人信息→ 赴海外上市 → 必须申报
├── 未经审查直接上市的后果极其严重
├── 不仅罚款，还涉及下架、停新、退市
└── 个人 + 公司双重责任（董事长、总裁各罚 100 万）
```

### 6.2 案例二：CII 采购安全审查

某能源央企采购境外厂商的 SCADA 控制系统，涉及国家电网调度。该采购属于 CII 运营者的网络产品采购，主动向网络安全审查办公室申报审查。审查发现：
- 产品固件在境外更新 → 存在远程控制风险
- 系统日志自动回传境外服务器 → 数据出境风险

**审查结论**：有条件通过——要求固件更新须在境内完成验证，日志服务器部署在境内。

## 七、与其他审查制度的区别

| 制度 | 适用对象 | 触发条件 | 审查机构 | 法律依据 |
|------|----------|----------|----------|----------|
| **网络安全审查** | CII 运营者采购/巨头赴海外上市 | 主动申报/依职权 | 网信办+13部委 | 网安法第35条 |
| **数据安全审查** | 所有数据处理者 | 依职权 | 数据安全工作协调机制 | 数安法第24条 |
| **外商投资安全审查** | 外国投资者 | 投资关键领域 | 发改委+商务部 | 外商投资法 |
| **反垄断审查** | 经营者集中 | 达到营业额门槛 | 市场监管总局 | 反垄断法 |

> **关键区别**：网络安全审查关注的是**国家安全**，而非市场公平竞争。审查标准是"是否危害国家安全"，而非"是否影响市场竞争"。

## 八、CISP 考试要点速查

### 核心数字

| 数字 | 含义 |
|------|------|
| 100万 | 掌握100万+用户信息→海外上市须审查 |
| 10/30/90 | 审查时限：10（初审）、30（初步审查）、90（特别审查） |
| 13部委 | 联合审查的部委数量 |
| 80.26亿 | 滴滴事件的代表性处罚金额 |

### 关键区别

| 对比项 | 采购审查 | 上市审查 |
|--------|----------|----------|
| 适用主体 | CII 运营者 | 掌握100万+用户信息的平台 |
| 触发行为 | 采购网络产品和服务 | 赴国外上市 |
| 共同点 | 可能影响国家安全 → 须审查 |

### 常见考试陷阱

1. ~~所有企业赴海外上市都需要审查~~ → 只有"掌握100万+用户信息"的平台才需要
2. ~~网络安全审查 = 数据安全审查~~ → 两个不同的制度，审查主体和范围不同
3. ~~审查时限是 30 天~~ → 初步审查 30 个工作日，特别审查 90 个工作日
4. ~~上市审查只看数据安全~~ → 还看供应链安全和被外国政府利用的可能

- [ ] 掌握网络安全审查的两种触发情形
- [ ] 记住审查的三个时限（10/30/90 个工作日）
- [ ] 理解五大审查维度
- [ ] 能区分网络安全审查与其他审查制度
- [ ] 知道 100 万用户门槛的含义

> 💡 **本章小结**：网络安全审查制度是 CISP 考试的热点法规考点。重点记住"两种触发情形"（CII 采购 + 100万用户赴海外上市）和"三个时限"（10/30/90 工作日）。滴滴事件是审查制度修订的直接催化剂，也是考试中典型的情景分析素材。
