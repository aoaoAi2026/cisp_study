## 一、HD 钱包原理

### 1.1 BIP32/BIP39/BIP44

```
BIP 标准体系：

BIP39 (助记词):
  128-256位熵 → 助记词(12/24个单词)
  助记词 → PBKDF2(Salt: "mnemonic" + passphrase) → 512位种子

BIP32 (HD钱包-分层确定性钱包):
  种子 → 主私钥 → 子私钥(可无限衍生)
  衍生路径：m / purpose' / coin_type' / account' / change / address_index

BIP44 (多币种):
  固定路径格式：
    m / 44' / coin_type' / account' / change / index
    
    Bitcoin:  m/44'/0'/0'/0/0
    Ethereum: m/44'/60'/0'/0/0
    Solana:   m/44'/501'/0'/0'

安全含义：
  1. 一个助记词 = 所有币种所有地址的根密钥
  2. 助记词泄露 = 所有资产被盗
  3. 助记词必须物理安全 + 加密存储
```

## 二、钱包类型与安全等级

### 2.1 冷热钱包对比

```
热钱包 (Hot Wallet):
  MetaMask, Rabby, Phantom — 联网
  适用：日常小额交易(< $10K)
  风险：浏览器扩展漏洞/XSS/钓鱼网站

暖钱包 (Warm Wallet):
  多签钱包 (Gnosis Safe), MPC钱包 — 需多人/多因素确认
  适用：DAO资金/协议金库/中额($10K-$1M)
  风险：签名阈值配置/社会工程攻击

冷钱包 (Cold Wallet):
  硬件钱包 (Ledger/Trezor/Keystone/OneKey)
  适用：大额/长期存储 (>$1M)
  风险：硬件供应链攻击/物理盗窃/固件漏洞

深度冷钱包 (Deep Cold Storage):
  在离线机器上生成助记词 → 
  钢板/钛板雕刻助记词 → 
  多份分别保存在不同的安全地点(银行保险柜)
  适用：超长期/超大额/机构级
```

### 2.2 硬件钱包

```
主流硬件钱包：

Ledger Nano X/S+:
  + BOLOS安全操作系统
  + 安全元件芯片(CC EAL5+)
  - 2023年 Ledger Connect Kit 供应链攻击(前端被投毒)
  - 2023年 Ledger Recover 服务引发隐私争议

Trezor Model T/Safe 3:
  + 开源固件
  + Shamir备份(20中x阈值恢复)
  - 旧型号Trezor One物理提取漏洞(需物理接触设备)

OneKey (国产):
  + 支持国密
  + 开源
  + 性价比高

Keystone:
  + 完全隔空(QR码传输，无USB/蓝牙)
  + 三个安全芯片
  + 国密支持

安全使用原则：
  1. 从官方渠道购买(防供应链攻击)
  2. 验证包装完整性(防篡改)
  3. 固件更新从官方源
  4. PIN码+助记词(不存数字照片！)
  5. 助记词物理存储(钢板/钛板)
```

## 三、常见钱包攻击

### 3.1 Approve 钓鱼

```
攻击方式：
  用户在钓鱼网站 → MetaMask弹出 → 
  "是否授权该合约使用您的USDC?" 
  用户不仔细看就点"确认" → 
  攻击者合约获得了无限的USDC approve权限

检测与防御：
  - MetaMask → 检查 Approve 的具体合约地址和金额
  - 使用 revoke.cash 或 Etherscan Token Approvals 定期检查+撤销
  - 不要让Approve是unlimited(无限额)，只用当次需要的量
  - 使用 Rabby Wallet(自动检测高危Approve并警告)
```

### 3.2 假空投/签名骗取

```
攻击链：
  1. 用户收到空投(不知名Token)
  2. 尝试在钓鱼网站"领取"
  3. 网站请求签名(不是交易，而是恶意签名消息)
  4. 签名类型：eth_sign(可导致full control) 或 permit(预授权)
  5. 攻击者使用签名转移资产

防御：
  - 不领取不认识来源的空投
  - 检查签名内容(不要盲签)
  - MetaMask设置 → 关闭"eth_sign"功能
  - 使用独立的空投钱包(与主资产钱包隔离)
```

### 3.3 剪贴板劫持

```
攻击方式：
  恶意软件监控剪贴板 → 
  检测到粘贴的内容是"0x..."地址 → 
  替换为攻击者的地址

防御：
  1. 发送前核对地址最后4位和全部字符
  2. 使用硬件钱包的屏幕确认(大额交易)
  3. 使用ENS域名(ether.eth)替代二维码地址
  4. 不要复制粘贴地址，使用白名单(地址簿)
```

## 四、MPC 与多签钱包

### 4.1 Gnosis Safe (多签)

```
Gnosis Safe = 最成熟的多签方案

配置示例：
  3/5多签 → 任何操作需要3个签名者中的任意3个签署
  (CEO, CTO, CFO, 安全官, 外部审计) → 5人中任意3人

安全优势：
  ✓ 单个人被攻击/私钥泄露 → 资产安全(仍需其他2人签名)
  ✓ 可设置权限分离(如日常操作2/3，大额操作4/5)
  ✓ 支持交易模拟(Tenderly Simulation)
```

### 4.2 MPC 钱包

```
MPC (Multi-Party Computation) 钱包：

原理：
  私钥从未作为一个整体存在！
  私钥 = shard1 + shard2 + shard3 (三个分片，分布在三个地方)
  签名时：三方MPC协议 → 输出完整签名
  任何单个分片被窃取 = 无法签名

代表产品：
  Fireblocks, Copper, Cobo MPC
  → 主要面向机构客户

vs 多签：
  MPC: 链上任何地址都支持(对公链是透明的，像是普通EOA)
  多签: 需要部署独立的智能合约地址(有Gas成本)
```

## 五、Checklist

- [ ] 大额资产(> $100K)使用硬件钱包/多签
- [ ] 助记词物理备份(防火防水防磁) → 钢板/钛板
- [ ] 助记词分片存储(2/3或3/5异地)
- [ ] 定期检查+撤销无用Approve
- [ ] 使用不同的钱包隔离风险(交易/DeFi/HODL)
- [ ] 交易前确认(用硬件钱包屏幕验证地址)
- [ ] 不要对任何"空投领取"网站签名
- [ ] 使用地址簿/白名单(不用剪贴板)
- [ ] 机构：使用MPC+多签双重保护
- [ ] 定期安全演练(助记词恢复测试、换设备测试)

### 考试陷阱提醒

| 陷阱 | 正确理解 |
|:---|:---|
| "硬件钱包绝对安全" | ❌ 仍需验证物理设备未被篡改、固件为正版、助记词不泄露 |
| "MPC 就是多签" | ❌ MPC 是密码学私钥分片计算，多签是链上合约级，机制完全不同 |
| "助记词存在云盘加密就行" | ❌ 任何触网的存储都是风险，必须物理介质离线存储 |

---

> **钱包安全的本质是"私钥永远不触网"——从生成到签名，私钥的整个生命周期都应保持离线。**
