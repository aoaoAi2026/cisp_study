# 《密码法》与商用密码管理条例深度解读

---

## 一、密码法核心制度

### 1.1 密码分类管理 (《密码法》第六条)

```
三类密码：

核心密码 (最高密级):
  - 保护国家绝密/机密/秘密级信息
  - 由中央密码管理部门统一管理
  - 严格知悉范围，专门传递渠道
  → 普通企业不涉及

普通密码:
  - 保护国家机密/秘密级信息
  - 由中央密码管理部门统一管理
  → 涉密单位使用

商用密码:
  - 保护不属于国家秘密的信息
  - 公民/法人/其他组织可使用商用密码保护网络与信息安全
  - 由国务院密码管理部门（国家密码管理局）主管
  → 本指南聚焦商用密码合规
```

### 1.2 商用密码管理要点

```
商用密码使用自由原则（《密码法》第二十一条）：
  "国家鼓励商用密码技术的研究开发、学术交流、成果转化和推广应用"
  企业可自由使用商用密码进行非涉密信息保护

需要行政许可的场景：
  1. 商用密码产品：部分产品需要认证
  2. 商用密码服务：密码检测认证机构需许可
  3. 进出口管控：特定密码产品出口需许可
  4. 外商投资负面清单（商密领域有限制）

不需要许可的场景：
  - 使用取得证书的商用密码产品
  - 使用开源密码库(GmSSL/OpenSSL/BouncyCastle)
  - 自行开发但集成已认证密码模块
```

## 二、新版《商用密码管理条例》(2023修订)

### 2.1 关键变化

```
2023年7月1日施行的《商用密码管理条例》(修订)：

五大核心变化：
  1. 建立商用密码应用安全性评估制度（密评）
  2. 明确关键信息基础设施必须通过密评
  3. 商用密码检测认证体系完善
  4. 电子认证服务纳入商密管理
  5. 进出口管制更精细化（不再"一刀切"）

企业关注要点：
  ├── CII运营者（关基单位）→ 必须使用商用密码进行保护 → 必须通过密评
  ├── 等保三级以上系统 → 必须使用密码技术 → 需通过密评
  ├── 政务信息系统 → 《国家政务信息化项目建设管理办法》要求密码保障
  └── 其他自愿使用 → 推荐通过密评（提升安全可信度）
```

## 三、密码合规体系联动

### 3.1 等保 + 密评 + 关基 = 三位一体

```
三者的关系：

    等保2.0 (网络安全基础)
         │
         ├── 要求使用密码技术保护
         │   (安全计算环境 / 安全通信网络)
         │
         ▼
    密评 (密码应用安全性评估)
         │
         ├── 验证密码技术是否合规+正确+有效
         │
         ▼
    关基保护 (关键信息基础设施)
         │
         └── 法律强制要求（《密码法》第27条 +
             《关键信息基础设施安全保护条例》第41条）

结论：
  等保2.0 → 要求用密码
  密评 → 验证密码用得对不对
  关基 → 不用密码或用错了 → 违法
```

### 3.2 哪些系统必须通过密评

```
必须过密评的场景（截止2026）：
  ✅ 关键信息基础设施（CII）
  ✅ 等保三级以上网络和信息系统
  ✅ 政务信息系统（《国家政务信息化项目建设管理办法》）

等保二级：推荐通过密评（非强制）
等保一级：无要求
```

## 四、密码合规落地路径

```
Phase 1: 盘点与差距分析
├── 梳理系统清单（哪些是等保三级/关基/政务）
├── 梳理当前密码使用现状
│   ├── 哪些系统用了密码？用的是什么？（国产/国际）
│   ├── 数据传输加密 → TLS/国密TLCP？
│   ├── 数据存储加密 → AES/SM4？
│   ├── 身份认证 → 数字证书？
│   └── 完整性保护 → 数字签名/HMAC/SM2/SM3？
└── 差距分析（对标GB/T 39786）

Phase 2: 方案设计
├── 密码应用方案（含密评要点）
├── 选择密码产品（需取得商密认证证书）
├── 部署国密证书体系（SM2/SM3/SM4）
└── 方案专家评审

Phase 3: 建设实施
├── 部署国密SSL网关/HTTPS
├── 部署签名验签服务器
├── 部署密钥管理系统(KMS)
├── 数据加密改造
└── 系统集成与联调

Phase 4: 密评
├── 方案评估（建设前）
├── 系统评估（建设后）
└── 年度评估（持续）
```
