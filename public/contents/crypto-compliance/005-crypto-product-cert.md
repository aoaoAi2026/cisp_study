## 一、商用密码产品认证

### 1.1 认证目录

```
商用密码产品认证目录（第一批, 2020年5月）：

26类产品需要获得国密局认证证书：

密码算法类：
  1. 密码芯片
  2. 密码模块
  3. 密码卡 (PCIe密码卡)

密码设备类：
  4. 服务器密码机
  5. 签名验签服务器
  6. 时间戳服务器
  7. 安全认证网关(SSL VPN / TLCP网关)
  8. IPSec VPN 安全网关
  9. 金融数据密码机
  10. 电子签章系统
  11. 动态口令系统/令牌

密码系统类：
  12. 密钥管理系统(KMS)
  13. 证书认证系统(CA)
  14. 电子认证服务系统

软件类：
  15. 密码软件(密码库/密码模块软件)
  16. 数字证书认证系统软件
  ...

关键要点：
  1. 等保/密评/关基 → 必须使用有认证证书的商密产品
  2. 可以使用认证产品(推荐)或自研密码模块(需单独过认证)
```

### 1.2 认证流程

```
商用密码产品认证流程：

Step 1: 确定产品范围
  → 明确产品属于哪一类（26类目录）

Step 2: 选择检测机构
  → 具备商用密码产品检测资质的检测机构
  → 如：国家商密检测中心、各省检测分中心

Step 3: 型式试验/检测
  → 检测机构按照GM/T标准进行检测
  → 测试项：
    - 功能测试（算法实现是否正确）
    - 性能测试（吞吐率/并发数/延迟）
    - 安全性测试（侧信道/故障注入攻击抗性）
    - 环境适应性（温/湿度/电磁兼容）

Step 4: 提交认证申请
  → 向国家密码管理局提交
  → 附检测报告+产品文档+企业资质

Step 5: 获得认证证书
  → 有效期：5年
  → 证书编号：GM/T XXXX-XXXX
  → 列入密码产品认证目录

Step 6: 年度监督审查
  → 获证后≥1次/年监督检查
  → 产品变更可能触发重新认证
```

## 二、密码模块安全等级 (GM/T 0028)

### 2.1 四级模型

```
GM/T 0028-2014 《密码模块安全技术要求》
对标 FIPS 140-2 (美国) / ISO/IEC 19790 (国际)

四级安全等级：

安全级别1 (SL1)：
  - 基本安全要求
  - 使用经过批准的密码算法
  - 对物理安全无特殊要求（普通PC即可）
  - 适用：软件密码模块

安全级别2 (SL2)：
  - SL1 + 物理安全机制
  - 防篡改封条/涂层 (Tamper Evidence)
  - 基于角色的身份认证
  - 适用：硬件安全模块基础版

安全级别3 (SL3)：
  - SL2 + 主动防篡改
  - 检测到物理攻击 → 自动清零密钥
  - 基于身份的身份认证
  - 关键安全参数(密钥)只以加密形式进出
  - 适用：HSM/金融密码机(PCI HSM标准)

安全级别4 (SL4)：
  - SL3 + 最严格物理保护
  - 环境故障保护(EFP)：温度/电压异常→立即清除密钥
  - 穿透密封防护
  - 适用：军事/最高密级场景
```

### 2.2 十一大安全域

```
GM/T 0028 要求覆盖11个安全域：

  1. 密码模块规格
  2. 密码模块接口(逻辑接口+物理接口)
  3. 角色/服务与认证
  4. 软件/固件安全
  5. 运行环境(操作系统要求)
  6. 物理安全(逐级递升)
  7. 非侵入式安全(侧信道/故障注入防护)
  8. 敏感安全参数管理(密钥全生命周期)
  9. 自测试(上电自检+条件自检)
  10. 生命周期保障(设计/开发/交付/运维)
  11. 其他攻击缓解
```

### 3.2 开源与白盒方案

```
如果你需要不依赖于特定硬件的密码能力：
  方案1：GmSSL (开源国密库)
  方案2：铜锁/Tongsuo (蚂蚁开源，国密TLS)
  方案3：使用认证过的密码卡 + 自研上层应用
  
注意：纯软件密码模块(无认证证书) → 
  可用于密评，但需通过单独的软件密码模块检测认证
  或：调用已认证的密码机API(密钥不由软件直接管理)
```
