## 📖 核心知识讲解

### 一、Windows在企业安全中的位置

虽然服务器大多是Linux，但在企业环境中：

| 资产类型 | 操作系统 | 安全关注点 |
|:---|:---|:---|
| 员工办公电脑 | **Windows** ✅ | 钓鱼攻击入口、权限滥用 |
| 域控制器(DC) | **Windows Server** ✅ | 整个内网的身份认证中心，被控=全网沦陷 |
| 文件服务器 | Windows Server | 数据泄露 |
| Web服务器 | Linux | Web攻击 |
| 数据库 | Linux | 数据窃取 |

> 🎯 **核心认知**：企业中Windows端点是攻击者进入内网的"入口"，域控制器是攻击者最终的目标。蓝队必须能分析Windows日志。

### 二、事件查看器 —— Windows的"黑匣子"

Windows把系统中发生的所有重要事情都记录在"事件日志"中，就像飞机的黑匣子。

**打开方式：**
1. 按 `Win + R`，输入 `eventvwr.msc`，回车
2. 或者在搜索框直接搜"事件查看器"

**三大核心日志：**

| 日志类型 | 记录内容 | 蓝队重点 |
|:---|:---|:---|
| **系统日志** | 系统启动/关机、驱动加载、服务启停 | 发现异常服务、系统崩溃 |
| **安全日志** | 登录/注销、权限使用、策略变更 | ⭐ 最重要！登录分析就靠它 |
| **应用程序日志** | 软件运行日志、安装卸载 | 应用异常排查 |

### 三、Windows事件ID速查表（蓝队必背）

每个事件都有一个ID编号。记住这些数字，你就知道发生了什么：

| 事件ID | 含义 | 蓝队解读 |
|:---|:---|:---|
| **4624** | 登录成功 ✅ | 谁、从哪个IP、在什么时候成功登录了 |
| **4625** | 登录失败 ❌ | 暴力破解的特征！短时间内大量4625=有人在猜密码 |
| **4672** | 分配了特殊权限 | 有人获得了管理员权限，需要关注 |
| **4719** | 系统审核策略更改 | 有人修改了安全策略（可能是攻击者关掉日志） |
| **4720** | 创建了用户账户 | 突然多了一个用户？后门账号！ |
| **4726** | 删除了用户账户 | 攻击者删完账号跑路 |
| **4740** | 账户被锁定 | 密码错太多被锁定了 |
| **4776** | 域控制器验证凭据 | 域环境中很重要，可追踪横向移动 |
| **7036** | 服务状态改变 | 看到陌生服务启动了？可能是后门 |
| **7045** | 新服务安装 | 更直接：有人在装新服务 |

> 🎯 **记忆技巧**：
> - 4624/4625 = 成对记忆，成功和失败
> - 4720/4726 = 成对记忆，创建和删除
> - 4开头 = 安全相关事件
> - 7开头 = 系统相关事件

### 四、深入解析事件ID 4624 —— 蓝队的"身份证"

一次登录成功会记录非常详细的信息。打开一条4624事件，你重点关注这些字段：

| 字段 | 含义 | 实战示例 |
|:---|:---|:---|
| **登录类型(Logon Type)** | 登录方式 | `10`=远程桌面，`3`=网络登录，`2`=控制台登录 |
| **账户名** | 哪个用户登录了 | 看到admin/super的奇怪登录就要警惕 |
| **源网络地址** | 从哪个IP登录的 | 公司IP范围外的IP=可疑 |
| **工作站名** | 从哪台机器登录的 | 陌生计算机名=可疑 |
| **进程名** | 通过什么程序登录的 | 不是正常程序=可疑 |

**登录类型代码含义：**

| 代码 | 含义 | 蓝队警觉度 |
|:---|:---|:---|
| 2 | 本地控制台（人坐在电脑前） | 🟢 低 |
| 3 | 网络登录（访问共享文件夹） | 🟡 中 |
| 4 | 批处理（计划任务） | 🟡 中 |
| 5 | 服务登录 | 🟡 中 |
| 7 | 解锁屏幕 | 🟢 低 |
| 10 | 远程桌面(RDP) ⚠️ | 🔴 高！攻击者最爱 |
| 11 | 缓存凭据登录 | 🔴 高 |

### 五、实战分析：如何识别暴力破解

暴力破解最明显的特征就是：**短时间内出现大量4625事件**

**分析步骤：**

```
1. 打开事件查看器 → Windows日志 → 安全
2. 右侧点击"筛选当前日志"
3. 事件ID输入：4625
4. 时间范围：最近24小时
5. 观察结果：
   - 同一个账户名，不同的来源IP → 可能不是暴破
   - 同一个来源IP，不同的账户名 → 典型暴破特征！
   - 同一IP，同一账户，密集4625 → 针对单个账户的暴破
```

> ⚠️ **实战判断标准**：1分钟内如果同一个源IP出现了5次以上4625，基本可以判定为暴力破解。

### 六、事件ID 4720 —— 突然冒出的用户就是后门

如果你看到一个管理员组的新用户突然出现，这就是红色警报：

```powershell
# 用PowerShell快速检查最近的用户创建
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4720} -MaxEvents 10 | 
    Format-List TimeCreated, Message
```

### 七、日志被清除的痕迹

攻击者入侵后第一件事往往是清除日志。如何发现？

- **事件ID 1102**：安全日志被清除！
- **事件ID 104**：系统日志被清除！

看到这两个ID，说明有人（大概率是攻击者）在销毁证据。

## 🔧 实操任务

### 任务1：探索事件查看器（15分钟）

1. 按 `Win + R` → `eventvwr.msc` → 回车
2. 依次点击查看：
   - Windows日志 → 系统（看看你的电脑开机多久了）

3. 双击任意一条事件，看看记录了哪些信息

### 任务2：筛选登录事件（20分钟）

1. 在安全日志中，右侧点击"筛选当前日志"
2. 事件ID处输入 `4624,4625`（逗号表示"或"）
3. 观察最近的登录成功和失败记录
4. 找到一条4624事件，打开查看：
   - 登录类型是多少？
   - 源网络地址是什么？
   - 是什么进程发起的登录？

### 任务3：模拟暴力破解检测（15分钟）

1. 故意输错几次Windows锁屏密码
2. 回到事件查看器，刷新安全日志
3. 查找事件ID 4625
4. 观察这些4625事件有什么共同特征

### 任务4：PowerShell快速查询（10分钟）

打开PowerShell（管理员模式），依次执行：

```powershell
# 查看最近10次登录成功
Get-WinEvent -LogName Security | Where-Object {$_.Id -eq 4624} | Select-Object -First 10

# 查看最近10次登录失败
Get-WinEvent -LogName Security | Where-Object {$_.Id -eq 4625} | Select-Object -First 10

# 统计今天的登录失败次数
(Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625; StartTime=(Get-Date).Date}).Count
```

## 🎯 蓝队面试高频题（Day 6 主题）

**Q1：Windows安全日志中，事件ID 4624 和 4625 分别代表什么？**

> - **4624**：登录成功。真正需要关注的是4624中的Logon Type字段：2-交互式登录（有人坐在电脑前）、3-网络登录（访问共享文件）、10-远程桌面登录（RDP）。若是凌晨出现Type 10登录→高度可疑。
> - **4625**：登录失败。如果短时间内出现大量4625→暴力破解攻击。需要关注Failure Reason字段：%%2313=未知用户名、%%2304=错误的密码。

**Q2：Windows蓝队应急排查的5个核心检查点是什么？**

> 1. **进程**（tasklist / tasklist /svc）：有没有可疑进程名（如svch0st.exe假装svchost.exe）
> 2. **服务**（services.msc 或 sc query）：有没有新增的可疑服务
> 3. **启动项**（msconfig 或注册表 Run键）：有没有随开机自动启动的恶意程序
> 4. **计划任务**（taskschd.msc 或 schtasks）：有没有被植入的持久化定时任务
> 5. **网络连接**（netstat -ano）：有没有外连到可疑IP
> 
> 口诀：**进服启计网**（进程→服务→启动项→计划任务→网络）。念三遍记住。

**Q3：如何判断一个Windows账户是不是被攻击者创建的？**

> 检查手段：
> 1. `net user` → 看有没有不认识的用户名
> 2. `net localgroup administrators` → 看管理员组有没有陌生账户
> 3. 事件查看器→安全日志→事件ID 4720（用户账户创建）→看创建时间和创建者
> 4. `lusrmgr.msc`（本地用户和组）→看用户"隶属于"哪些组
> 5. 对比已知运维人员名单→不在名单上的就是可疑账户

## 📖 深度阅读：Windows vs Linux 蓝队视角对比

很多新人只学Linux或只学Windows，但实际工作中两者都要懂。以下是蓝队视角的核心差异：

| 维度 | Windows | Linux |
|:---|:---|:---|
| **进程管理** | 任务管理器/tasklist | ps/top/htop |
| **日志位置** | 事件查看器（EVTX二进制格式） | /var/log/*（纯文本） |
| **操作方式** | 主要图形界面+PowerShell | 主要命令行+Shell脚本 |
| **用户管理** | lusrmgr.msc / net user | /etc/passwd /etc/shadow |
| **防火墙** | Windows Defender防火墙 | iptables/nftables |
| **软件安装** | .exe / .msi 安装包 | apt/yum/dnf 包管理器 |
| **计划任务** | Task Scheduler（taskschd.msc） | cron/systemd timers |
| **攻击面** | RDP(3389)、SMB(445)、WinRM(5985) | SSH(22)、Web端口(80/443/8080) |

> **蓝队实战**：内网通常Windows服务器做域控/文件服务器/数据库、Linux做Web服务器/中间件。一个完整的护网场景里两者都会涉及。先精通一个（建议Linux，更开放透明），再补另一个。

## ⚠️ 新手常见误区纠正

1. **误区**："Windows很安全，装个杀毒软件就行了"
   - **真相**：Windows是全球最大目标——不是因为不安全，而是因为用得最多。杀毒软件只防已知恶意软件，0day攻击、社会工程学攻击、配置缺陷它都防不了。安全是系统工程，杀毒软件只是其中一环。
   
2. **误区**："Linux比Windows安全"
   - **真相**：安全性不取决于操作系统，取决于管理员。配置不当的Linux（root弱口令+SSH暴露公网+无防火墙）比配置完善的Windows脆弱得多。各有各的弱点——Linux弱在SSH暴破和Web漏洞，Windows弱在RDP和SMB。

3. **误区**："蓝队学Windows的主要是学会用事件查看器就行"
   - **真相**：事件查看器只是入门。你还要会PowerShell脚本自动化分析、用SysInternals工具套件（Process Explorer/Autoruns/TCPView）、用Windows日志转发搭建集中日志平台。Day 6是起点，不是终点。

## 🎓 Windows蓝队利器：SysInternals工具套件

Windows自带工具够用，但微软官方出品的 SysInternals 套件才是蓝队真正的"瑞士军刀"。以下是蓝队最常用的5个：

**1. Process Explorer（procexp.exe）——任务管理器的"爸爸"**
- 以树状图显示所有进程的父子关系（谁启动了谁）
- 右键进程→"Check VirusTotal"→直接在线查毒
- 双击进程→查看TCP/IP连接→知道每个进程连了哪些IP
- **蓝队用它**：找出伪装成系统进程的恶意程序（svchost.exe的父进程必须是services.exe，否则是假的）

**2. Autoruns（autoruns.exe）——"你到底有多少开机启动项？"**
- 比msconfig强大100倍的启动项查看工具
- 列出所有开机自动运行的位置：注册表Run键、服务、计划任务、浏览器插件、驱动...
- **蓝队用它**：找出攻击者设置的所有持久化手段（恶意软件最喜欢藏在 Autoruns 中列出的某个角落）

**3. TCPView（tcpview.exe）——"你的电脑在和谁聊天？"**
- 实时显示所有TCP/UDP连接+对应的进程名
- 比 `netstat -ano` 直观100倍，带图形界面
- **蓝队用它**：一眼看出有没有不认识的进程在和陌生IP通信

**4. Process Monitor（procmon.exe）——"一切操作的录像机"**
- 实时记录系统中所有进程的文件读写、注册表修改、网络连接操作
- 输出量极大（每秒几万条），必须配合过滤
- **蓝队用它**：追踪某个可疑进程到底在干什么——读了什么文件、写入了什么、连了哪里

**5. Strings（strings.exe）——"从乱码中提取可读内容"**
- 从任意二进制文件中提取可读字符串（ASCII/Unicode）
- **蓝队用它**：分析可疑exe/dll文件 → 提取里面的URL、IP地址、命令、加密密钥等线索

### 面试高频题与真实案例

## 💡 面试高频题：Windows事件日志

**Q: Windows安全日志中，事件ID 4624和4625分别代表什么？**
A: 4624=登录成功，4625=登录失败。蓝队值守时重点关注4625事件——短时间内大量4625（同一账户或同一源IP）=暴力破解攻击。登录类型字段（Logon Type）也很关键：2=交互式登录，3=网络登录，10=远程交互登录（RDP）。类型10的4625集中出现=RDP爆破。

**Q: 事件ID 4672和4648分别代表什么？**
A: 4672=特权账户登录（如管理员），任何提权操作都需要关注。4648=使用显式凭据尝试登录（如runas命令），攻击者常用此手法切换账户。同时出现4672+4648=高度可疑的提权行为。

**Q: 哪些事件ID与「持久化」相关？**
A: ①4698=计划任务创建（攻击者创建定期执行的恶意任务）；②704=服务安装（后门注册为系统服务）；③4657=注册表修改（修改启动项）；④1107=安全策略更改。蓝队需定期导出以上事件做基线对比。

**Q: Windows事件日志有哪些容易被攻击者清除的痕迹？**
A: ①事件1102=审计日志被清除（攻击者试图抹除痕迹）。如果系统日志突然出现"空洞"（一大段时间无日志），很可能是被手动清除了。②攻击者也可能停止Windows Event Log服务以阻止新日志生成——此时需要依靠网络设备的流量日志做补充。

**Q: 蓝队如何快速从海量Windows日志中定位关键事件？**
A: ①使用wevtutil命令行工具导出+筛选；②关注事件ID白名单（4624,4625,4672,4688,4698,4720,1102等）；③重点关注非工作时间的事件；④建立基线后，用比对工具（如Event Log Explorer）发现增量异常。


## 📊 真实案例：通过Windows日志发现内网横向移动

某次护网中，蓝队发现一台普通PC的Windows安全日志中出现大量4625（登录失败）事件，且Logon Type=3（网络登录），目标账户涵盖了近50个不同用户名。

分析过程：
1. 定位源IP——发现是一个内网Linux服务器（192.168.1.50），不属于该PC的正常业务访问范畴
2. 追溯该Linux服务器——发现其/var/log/secure中有大量SSH登录失败记录，确认该服务器已被攻陷
3. 攻击链还原——攻击者先拿下Linux跳板机→从跳板机对内网Windows主机进行SMB暴力破解→意图横向移动
4. 处置——立即隔离该Linux服务器，重置受影响Windows主机的所有账户密码，封禁攻击源IP

结论：单台Windows的4625日志就可能暴露整个内网沦陷的攻击链。蓝队人员要养成"看到一个事件ID，思考它背后的攻击逻辑"的习惯。

## 🔬 进阶专题：本日知识在真实护网中的应用

### 护网场景还原

想象你正在护网值守中，突然收到一条与本日主题相关的告警。以下是标准的思维和处理流程：

**1. 第一反应（0-30秒）**：确认告警来源设备是否正常 → 查看告警级别 → 判断是否为"狼来了"

**2. 快速研判（30秒-2分钟）**：提取关键信息（源IP/目标资产/攻击类型/时间）→ 查威胁情报 → 看同一IP的其他告警

**3. 深度分析（2-10分钟）**：回放原始流量/日志 → 还原攻击payload → 判断攻击是否成功

**4. 处置决策（10-15分钟）**：确认是攻击→定级→封IP/隔离/升级事件；确认是误报→标记→优化规则

**5. 记录闭环（15-20分钟）**：填写工单（研判依据+处置动作+证据截图）→ 标记闭环


### 高手与新手的关键区别

| 维度 | 新手 | 高手 |
| --- | --- | --- |
| 看告警 | 只看这一条告警的内容 | 同时关联其他设备/其他时间的同类告警 |
| 判断依据 | "这个payload看起来像攻击" | "这个payload+这个User-Agent+这个时间点+这个频率=SQLMap自动化扫描" |
| 处置方式 | 封IP→完事 | 封IP+排查漏洞+检查是否已有数据泄露+通知业务方+更新WAF规则 |
| 记录质量 | "已处置" | "10:05收到WAF SQL注入告警→核查源IP 45.x为境外IP+威胁情报标记恶意→查看该IP完整请求链→确认SQLMap自动化扫描→已在WAF永久封禁+通知开发修复参数化查询→附件：攻击流量pcap+威胁情报截图" |

> **记住**：蓝队不是"见招拆招"的被动防守，而是"知己知彼"的主动防御。每天学完一个知识点后，立刻思考"如果我是攻击者我会怎么用"+"如果我是防御者我该怎么挡"——这种双向思维是蓝队核心竞争力的来源。
