## 📖 核心知识讲解

### 一、模拟值守说明

今天是28天速成计划中**最重要的一天**——你需要把前三周学到的所有技能整合起来，模拟一次真实的护网值守。

**模拟值守设置：**
- 时长：4小时（可分段完成，每次1小时）
- 场景：一家中型企业的Web系统护网期间
- 工具：你的虚拟机、笔记、浏览器
- 你需要扮演：蓝队一线值守分析师

### 二、值守工作台准备

在开始值守前，确保以下"工位"就绪：

```
你的值守工位：
□ 终端1：SOC告警面板（你将要处理的告警列表）
□ 终端2：Linux命令行（用于日志分析、情报查询）
□ 终端3：浏览器（打开微步在线、VirusTotal、CVE查询）
□ 终端4：笔记软件（记录研判结果、写工单）
□ 通讯工具：假设有企业微信/钉钉群（用于上报）
```

### 三、值守规则

1. **P0/P1告警必须在15分钟内响应**
2. **每条告警都需要有研判记录**（哪怕只是"确认误报"）
3. **关联告警要合并处理**（同一IP的多个告警视为一次攻击事件）
4. **每小时记录一次值守日志**
5. **发现问题要及时"上报"（在笔记中模拟）**

### 四、告警处理的标准格式

每处理一条告警，按以下格式记录：

```markdown
### 告警编号：ALERT-001
**时间**：2026-06-13 09:15
**设备**：IDS
**类型**：SQL注入
**来源IP**：45.33.32.156
**目标**：192.168.1.100:80

**研判过程**：
1. 情报核查：微步判定为恶意IP(红色标记)
2. 日志验证：在nginx访问日志中找到对应请求
3. 攻击确认：URL包含"UNION SELECT"，确认SQL注入尝试
4. 结果判断：状态码200，后续无异常登录日志 → 攻击尝试中，未完全成功

**等级**：P1 高危
**处置**：封禁IP + 排查注入点 + 记录工单
**状态**：处置中
```

## 🔬 进阶专题：本日知识在真实护网中的应用

### 护网场景还原

想象你正在护网值守中，突然收到一条与本日主题相关的告警。以下是标准的思维和处理流程：

**1. 第一反应（0-30秒）**：确认告警来源设备是否正常 → 查看告警级别 → 判断是否为"狼来了"

**2. 快速研判（30秒-2分钟）**：提取关键信息（源IP/目标资产/攻击类型/时间）→ 查威胁情报 → 看同一IP的其他告警

**3. 深度分析（2-10分钟）**：回放原始流量/日志 → 还原攻击payload → 判断攻击是否成功

**4. 处置决策（10-15分钟）**：确认是攻击→定级→封IP/隔离/升级事件；确认是误报→标记→优化规则

**5. 记录闭环（15-20分钟）**：填写工单（研判依据+处置动作+证据截图）→ 标记闭环


### 高手与新手的关键区别

| 维度 | 新手 | 高手 |
| --- | --- | --- |
| 看告警 | 只看这一条告警的内容 | 同时关联其他设备/其他时间的同类告警 |
| 判断依据 | "这个payload看起来像攻击" | "这个payload+这个User-Agent+这个时间点+这个频率=SQLMap自动化扫描" |
| 处置方式 | 封IP→完事 | 封IP+排查漏洞+检查是否已有数据泄露+通知业务方+更新WAF规则 |
| 记录质量 | "已处置" | "10:05收到WAF SQL注入告警→核查源IP 45.x为境外IP+威胁情报标记恶意→查看该IP完整请求链→确认SQLMap自动化扫描→已在WAF永久封禁+通知开发修复参数化查询→附件：攻击流量pcap+威胁情报截图" |

### 护网场景还原

想象你正在护网值守中，突然收到一条与本日主题相关的告警。以下是标准的思维和处理流程：

**1. 第一反应（0-30秒）**：确认告警来源设备是否正常 → 查看告警级别 → 判断是否为"狼来了"

**2. 快速研判（30秒-2分钟）**：提取关键信息（源IP/目标资产/攻击类型/时间）→ 查威胁情报 → 看同一IP的其他告警

**3. 深度分析（2-10分钟）**：回放原始流量/日志 → 还原攻击payload → 判断攻击是否成功

**4. 处置决策（10-15分钟）**：确认是攻击→定级→封IP/隔离/升级事件；确认是误报→标记→优化规则

**5. 记录闭环（15-20分钟）**：填写工单（研判依据+处置动作+证据截图）→ 标记闭环


### 高手与新手的关键区别

| 维度 | 新手 | 高手 |
| --- | --- | --- |
| 看告警 | 只看这一条告警的内容 | 同时关联其他设备/其他时间的同类告警 |
| 判断依据 | "这个payload看起来像攻击" | "这个payload+这个User-Agent+这个时间点+这个频率=SQLMap自动化扫描" |
| 处置方式 | 封IP→完事 | 封IP+排查漏洞+检查是否已有数据泄露+通知业务方+更新WAF规则 |
| 记录质量 | "已处置" | "10:05收到WAF SQL注入告警→核查源IP 45.x为境外IP+威胁情报标记恶意→查看该IP完整请求链→确认SQLMap自动化扫描→已在WAF永久封禁+通知开发修复参数化查询→附件：攻击流量pcap+威胁情报截图" |
