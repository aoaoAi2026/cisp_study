## 📖 核心知识讲解

### 一、蓝队为什么要学Nmap？

很多人以为Nmap是"黑客工具"，其实它在蓝队手中同样是利器：

| 蓝队场景 | 用Nmap做什么 |
|:---|:---|
| **资产发现** | 扫描公司网段，发现所有在线设备（防止被遗忘的"幽灵资产"） |
| **端口检查** | 检查服务器开放的端口是否符合基线要求 |
| **漏洞排查** | 发现不该开放的服务（如Telnet、默认Redis） |
| **入侵验证** | 收到告警后扫描可疑IP，验证目标是否真的开放了被攻击的服务 |
| **安全巡检** | 定期扫描，和上次的结果对比，发现新开放的可疑端口 |

### 二、端口扫描是什么？（用敲门来理解）

想象一栋大楼里有65535个房间（端口号），你想知道哪些房间有人：

| 扫描方式 | 敲门比喻 | 优缺点 |
|:---|:---|:---|
| **SYN扫描(-sS)** | 敲门听到回音就离开（不推门进去） | 速度快，不容易被记录 |
| **TCP全连接扫描(-sT)** | 敲门→门开→说"不好意思走错了" | 最准确，但会被完整记录 |
| **UDP扫描(-sU)** | 扔个纸团进去看有没有人回 | 速度慢，但能发现UDP服务 |
| **Ping扫描(-sn)** | 只确认这栋楼存不存在 | 最快，只发现存活主机 |

### 三、Nmap基本使用

#### 安装
```bash
# Kali Linux自带，无需安装
# Ubuntu/Debian:
sudo apt install nmap

# Windows:
# 下载地址：https://nmap.org/download.html
```

#### 最基础的用法
```bash
nmap 目标IP或域名

# 示例：扫描baidu.com
nmap baidu.com
```

#### 常用参数速查表

| 参数 | 全称 | 作用 | 什么时候用 |
|:---|:---|:---|:---|
| `-sS` | SYN Stealth Scan | SYN半开放扫描（最常用） | 默认扫描方式 |
| `-sT` | TCP Connect Scan | TCP全连接扫描 | 没root权限时用 |
| `-sU` | UDP Scan | UDP端口扫描 | 排查DNS/DHCP/SNMP |
| `-sV` | Version Detection | 检测服务版本 | 想知道具体软件和版本 |
| `-O` | OS Detection | 操作系统识别 | 判断目标是什么系统 |
| `-p` | Port | 指定端口 | 只扫特定端口 |
| `-p-` | All Ports | 扫全部65535个端口 | 全面排查时用（很慢！） |
| `-F` | Fast | 快速扫描（只扫100个常用端口） | 快速巡检 |
| `-A` | Aggressive | 激进扫描（包含版本+OS+脚本） | 全面信息收集 |
| `-Pn` | No Ping | 跳过ping直接扫描 | 目标禁用ping时用 |
| `-T0~T5` | Timing | 扫描速度(0最慢5最快) | T4为默认，T5可能丢包 |
| `-oN` | Output Normal | 保存结果为文本 | 输出报告 |
| `-oX` | Output XML | 保存结果为XML | 导入其他工具 |

### 四、实战扫描示例

#### 示例1：快速查看目标开放了哪些端口
```bash
nmap -F 192.168.1.1

# 输出示例：
# PORT     STATE    SERVICE
# 22/tcp   open     ssh
# 80/tcp   open     http
# 443/tcp  open     https
# 3306/tcp open     mysql
# 8080/tcp open     http-proxy
```

**蓝队解读：** 这台服务器开了SSH、Web(80/443)、MySQL、还有一个8080上跑着Web服务。如果这是一台只应该跑Web的服务器，那3306（MySQL对外暴露）就是风险点。

#### 示例2：检测服务版本
```bash
nmap -sV -p 22,80,443 192.168.1.1

# 输出示例：
# PORT     STATE  SERVICE    VERSION
# 22/tcp   open   ssh        OpenSSH 6.6.1p1 (protocol 2.0)
# 80/tcp   open   http       Apache httpd 2.2.15
# 443/tcp  open   ssl/http   Apache httpd 2.2.15
```

**蓝队解读：** OpenSSH 6.6.1是2014年的老版本，Apache 2.2.15更老（2010年）。这些版本可能有严重漏洞，需要升级或加固。

#### 示例3：扫描整个网段的存活主机
```bash
nmap -sn 192.168.1.0/24

# 输出示例：
# Host 192.168.1.1 is up.      # 网关
# Host 192.168.1.100 is up.    # 员工电脑
# Host 192.168.1.200 is up.    # 未知设备！
```

**蓝队解读：** 发现了一台不该存在的设备（192.168.1.200），可能是有人私自接入的设备，也可能是攻击者的跳板机。

#### 示例4：组合参数，一次全面扫描
```bash
# 对单个主机进行全面扫描
nmap -sS -sV -O -p- -T4 192.168.1.1

# 参数解释：
# -sS: SYN扫描
# -sV: 检测服务版本
# -O:  识别操作系统
# -p-: 扫全部端口
# -T4: 速度第4档（较快）
```

### 五、Nmap扫描结果的蓝队解读指南

扫描完后，你需要逐条判断：

```
PORT     STATE    SERVICE       VERSION               危险评估
22/tcp   open     ssh           OpenSSH 8.0             🟢 正常
80/tcp   open     http          nginx 1.18.0            🟢 正常
443/tcp  open     ssl/http      nginx 1.18.0            🟢 正常
23/tcp   open     telnet        Linux telnetd           🔴 高危！明文协议
6379/tcp open     redis         Redis 5.0.0             🔴 高危！默认无密码
445/tcp  open     netbios-ssn   Samba smbd              🟡 中危
3306/tcp filtered mysql                                  🟢 关了，安全
```

**判断标准：**
- 🔴 `telnet(23)`、`ftp明文(21)`、无密码`redis(6379)`、无密码`mongodb(27017)` → 立即处理
- 🟡 `SMB(445)`、`RDP(3389)`、`MySQL外网(3306)` → 需要评估是否需要对外开放
- 🟢 Web服务(80/443)、SSH(22) → 正常，但要确保最新版本

### 六、蓝队的Nmap使用准则

> ⚠️ **重要**：作为蓝队，扫描前请确认：
> 1. 只扫描自己负责的资产或获得授权的系统
> 2. 不经授权扫描他人系统可能违法！
> 3. 在生产环境中注意扫描速度，-T5可能影响业务

## 🎯 蓝队面试高频题（Day 12 主题）

**Q1：Nmap -sS 和 -sT 扫描有什么区别？蓝队怎么检测？**

> - `-sS`（SYN扫描/半开扫描）：发SYN→收到SYN+ACK（端口开放）立即发RST断开，不完成三次握手。速度快、隐蔽性较好。也叫"半开扫描"因为不建立完整TCP连接。
> - `-sT`（TCP Connect扫描）：完成完整的三次握手后再断开。慢但更可靠。受限于操作系统的连接数限制。**更易被检测**因为应用层日志会留下完整连接记录。
>
> 蓝队检测：大量SYN包（不完成握手）+ 同一IP连接大量不同端口 → 很可能是SYN扫描。如果防火墙日志/系统日志中出现了完整TCP连接记录且来自同一IP遍及不同端口 → Connect扫描。

**Q2：Nmap扫描结果的"open""closed""filtered"分别代表什么？**

> - **open**：端口开放，有服务在监听 → 攻击者可以进一步探测这个服务的类型和版本
> - **closed**：端口可达但没有服务监听 → 系统返回RST包
> - **filtered**：无法判断 → 被防火墙/IDS拦截了，Nmap收不到任何回应
>
> 蓝队视角：如果你的关键端口在Nmap下显示"filtered"→说明防火墙规则生效了。但如果显示"open"→这个端口已经暴露了。

**Q3：Nmap除了端口扫描还能做什么？**

> - 服务版本探测（-sV）：识别端口背后跑的是什么服务和版本
> - 操作系统识别（-O）：通过TCP/IP栈指纹猜测目标操作系统
> - 漏洞扫描（--script vuln）：用NSE脚本进行漏洞检测
> - 网络发现（-sn）：发现局域网中存活的主机（不扫端口）
> - 脚本扫描（--script）：强大的NSE脚本引擎，可做各种安全检测

## 📖 深度阅读：从Nmap扫描结果反推攻击者意图

作为蓝队，看到扫描流量时要能判断攻击者的**阶段和意图**：

| Nmap参数 | 攻击者意图 | 蓝队应对 |
|:---|:---|:---|
| `-sn`（ping扫描） | 发现存活主机 → 攻击者还在"踩点"阶段 | 不回应ICMP（配置防火墙规则） |
| `-sS -p 1-1000` | 快速侦察常用端口 → 信息收集 | 监控SYN包速率、限制同一IP的SYN频率 |
| `-sV`（版本探测） | 识别服务版本 → 准备精准攻击（找对应版本CVE） | 隐藏Server头、用非标准端口（但安全不能靠混淆） |
| `-sC`（默认脚本） | 综合扫描 → 信息收集+简单漏洞检测 | 告警：存在扫描行为，记录源IP |
| `--script vuln` | 漏洞扫描 → 攻击前最后侦察 | 高危告警！立刻封禁IP并加强监控 |
| `-O`（OS检测） | 识别操作系统 → 可能是定向攻击前奏 | 属于侦察行为，记录+关注后续 |

> **核心判断**：如果只看到 `-sn` 或 `-sS` 扫描→可能只是脚本小子在"广撒网"。但如果看到了 `-sV + --script vuln`→说明这是个认真的攻击者，他在做攻前侦察。这种情况下必须立刻提高警惕。

## ⚠️ 新手常见误区纠正

1. **误区**："学Nmap就是学怎么扫描别人"
   - 补充：蓝队学Nmap目的是①理解攻击者怎么扫你→知道怎么防 ②对自己系统做安全检查→发现暴露的端口和服务。**不要未经授权扫描他人系统——这可能违法。**
   
2. **误区**："防火墙把端口都封了就扫不到我了"

### 面试高频题与实操扩展

## 💡 面试高频题：Nmap扫描工具

**Q: Nmap的SYN扫描(-sS)和TCP Connect扫描(-sT)有什么区别？**
A: SYN扫描（半开扫描）只发送SYN包，收到SYN-ACK后就发送RST断开，不完成三次握手——因此更隐蔽、更快，但需要root权限。TCP Connect扫描完成完整三次握手后正常断开——更慢、更容易被日志记录，但不需要特殊权限。蓝队检测Nmap扫描时：SYN扫描通过流量中的"SYN→SYN-ACK→RST"模式识别；Connect扫描通过系统日志中的连接记录识别。

**Q: Nmap有哪些常用的扫描参数组合？**
A: ①`nmap -sV -p 1-65535 target`（全端口+服务版本检测）；②`nmap -sC -sV target`（默认脚本+版本检测）；③`nmap -O target`（操作系统识别）；④`nmap -sn 192.168.1.0/24`（存活主机探测）；⑤`nmap -sU -p 53,161 target`（UDP扫描）。蓝队要知道这些参数对应的流量特征，以便识别正在被扫描。

**Q: 蓝队如何检测自己正在被Nmap扫描？**
A: ①流量层面：大量SYN包发往不同端口→RST或超时→特征明显；②日志层面：防火墙/IDS日志中短时间内出现大量不同目标端口的拒绝记录；③告警层面：IDS规则（如Snort的"SCAN nmap"规则）会自动触发告警。蓝队可以在实验室用Nmap扫描自己，观察Wireshark和日志中的特征，建立对扫描流量的"直觉"。

**Q: Nmap的脚本引擎(NSE)是什么？蓝队需要了解吗？**
A: NSE（Nmap Scripting Engine）是Nmap的扩展脚本系统，可以执行漏洞检测、暴力破解、信息收集等高级功能。蓝队必须了解的NSE脚本：①http-sql-injection（检测SQL注入）；②http-enum（枚举Web目录）；③smb-vuln-ms17-010（检测永恒之蓝漏洞）；④ssl-heartbleed（检测心脏滴血漏洞）。了解攻击者的工具能帮助你更好地识别攻击行为。

**Q: 使用Nmap扫描他人服务器合法吗？**
A: 未经授权的端口扫描在中国属于违法行为（《网络安全法》第27条）。蓝队人员只能在以下场景使用：①扫描自己搭建的靶场环境（如DVWA/VulHub）；②获得书面授权的渗透测试项目；③内部安全自查（扫描自家资产）。绝不要因为好奇去扫描任何非授权目标。


## 📊 实操扩展：Nmap扫描结果解读模板

蓝队在收到"有人在对我们进行Nmap扫描"的告警后，可以反过来利用扫描结果了解攻击者掌握了哪些信息：

**攻击者视角的信息暴露**：
```
PORT     STATE    SERVICE    VERSION
22/tcp   open     ssh        OpenSSH 7.4
80/tcp   open     http       Apache httpd 2.4.6
443/tcp  open     ssl/http   Apache httpd 2.4.6
3306/tcp open     mysql      MySQL 5.7.38
8080/tcp open     http       Apache Tomcat 9.0.50
```

蓝队看到这个结果后应该立即自查：
1. 3306端口（MySQL）为何暴露在外网？→应该限制仅内网访问
2. OpenSSH 7.4是否存在已知漏洞？→查CVE数据库
3. Tomcat 9.0.50版本是否过旧？→检查是否有安全更新
4. 8080端口的管理后台是否使用了默认密码？→立即修改

**总结**：Nmap扫描不是"坏事"——它是攻击者的"前奏"。聪明的蓝队把每次扫描当作一次免费的"外部安全检查"，根据攻击者能看到的信息来修补自己的暴露面。

## 🎯 实战思维训练

### 蓝队"条件反射"训练

网络安全值守中，很多判断需要在几秒内完成。以下是本日主题相关的"条件反射"训练：

**看到以下现象 → 立即联想到 → 采取动作**：

1. 短时间内同一IP大量不同URL请求 → 目录/漏洞扫描 → 检查是否返回了不该返回的内容
2. WAF告警+同IP的Web日志中有500错误 → 攻击可能在尝试绕过WAF → 查看完整请求体
3. 非工作时间的管理员登录 → 凭据泄露/后门 → 确认是否为合法运维操作
4. 同一文件被频繁POST请求 → Webshell心跳 → 检查文件内容和创建时间
5. 出站流量突增到非标准端口 → 数据渗出/C2通信 → 追踪目标IP并阻断

### "如果是你，你怎么防？"

## 🔬 进阶专题：本日知识在真实护网中的应用

### 护网场景还原

想象你正在护网值守中，突然收到一条与本日主题相关的告警。以下是标准的思维和处理流程：

**1. 第一反应（0-30秒）**：确认告警来源设备是否正常 → 查看告警级别 → 判断是否为"狼来了"

**2. 快速研判（30秒-2分钟）**：提取关键信息（源IP/目标资产/攻击类型/时间）→ 查威胁情报 → 看同一IP的其他告警

**3. 深度分析（2-10分钟）**：回放原始流量/日志 → 还原攻击payload → 判断攻击是否成功

**4. 处置决策（10-15分钟）**：确认是攻击→定级→封IP/隔离/升级事件；确认是误报→标记→优化规则

**5. 记录闭环（15-20分钟）**：填写工单（研判依据+处置动作+证据截图）→ 标记闭环


### 高手与新手的关键区别

| 维度 | 新手 | 高手 |
| --- | --- | --- |
| 看告警 | 只看这一条告警的内容 | 同时关联其他设备/其他时间的同类告警 |
| 判断依据 | "这个payload看起来像攻击" | "这个payload+这个User-Agent+这个时间点+这个频率=SQLMap自动化扫描" |
| 处置方式 | 封IP→完事 | 封IP+排查漏洞+检查是否已有数据泄露+通知业务方+更新WAF规则 |
| 记录质量 | "已处置" | "10:05收到WAF SQL注入告警→核查源IP 45.x为境外IP+威胁情报标记恶意→查看该IP完整请求链→确认SQLMap自动化扫描→已在WAF永久封禁+通知开发修复参数化查询→附件：攻击流量pcap+威胁情报截图" |

### 你今天学的知识在ATT&CK中的映射

花5分钟思考以下问题（有助于建立安全思维框架）：

- 今天的攻击手法在ATT&CK中属于哪个战术？对应的技术编号是什么？

- 用Kill Chain模型分析：这种攻击通常发生在哪几个阶段？

- 如果你负责设计针对这种攻击的防御方案，你会从哪几个层面入手？


> **记住**：蓝队不是"见招拆招"的被动防守，而是"知己知彼"的主动防御。每天学完一个知识点后，立刻思考"如果我是攻击者我会怎么用"+"如果我是防御者我该怎么挡"——这种双向思维是蓝队核心竞争力的来源。
