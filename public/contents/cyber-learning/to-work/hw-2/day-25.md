## 📖 核心知识讲解

### 一、护网三阶段概览

护网行动不是从演习当天开始的。完整的工作分为三个阶段：

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 护网前   │ → │ 护网中   │ → │ 护网后   │
│ 准备阶段  │    │ 实战阶段  │    │ 复盘阶段  │
│ 1-2个月   │    │ 2-4周    │    │ 1-2周    │
└──────────┘    └──────────┘    └──────────┘
```

### 二、护网前 —— 不打无准备之仗

#### 1. 资产梳理（最重要的准备工作）

**为什么重要？** 你都不知道自己有哪些服务器、开了哪些端口，怎么防守？

```
资产梳理清单：
□ 外网资产
  ├── 域名列表（包括子公司、测试环境）
  ├── 公网IP（每个IP上的服务是什么）
  ├── 云端资产（云服务器、OSS、CDN）
  └── 第三方系统（外包、SaaS平台）

□ 内网资产
  ├── 服务器清单（IP、用途、负责人、重要等级）
  ├── 网络设备（交换机、路由器、防火墙）
  ├── 安全设备（IDS/IPS/WAF/日志审计）
  └── 数据库（MySQL/Redis/Oracle/MongoDB）

□ 人员资产
  ├── 账号清单（管理员账号、服务账号）
  ├── 权限梳理（谁有什么权限）
  └── 离职人员账号是否清理
```

**常见问题（护网前就被蓝队发现的问题）：**
- "这台服务器什么时候上线的？没人知道！" → 幽灵资产
- "这个端口怎么开着？早就不用了" → 未清理的测试环境
- "这个管理员账号是谁的？" → 离职未清理

#### 2. 风险排查与加固

```
风险排查重点：
□ 弱口令检查（admin/admin123/password123等）
□ 端口开放检查（不必要的端口关闭）
□ 漏洞扫描（高危漏洞在护网前修复）
□ 基线检查（操作系统/数据库/中间件安全配置）
□ 访问控制（不该访问的能否访问到）
□ 日志审计（确保所有设备的日志都在正常产生和接收）
```

#### 3. 方案和人员准备

```
护网前必须就绪的文档：
□ 护网值守方案（谁在哪值班、什么时间）
□ 应急响应预案（出事怎么处理）
□ 通讯录（所有人联系方式、TL负责人）
□ 日报模板
□ 情报来源（从哪里获取最新威胁信息）
```

### 三、护网中 —— 实战阶段的日常

#### 1. 值班安排

```
典型的值班排班：
  白班：08:00 - 20:00（主班，人员最多）
  夜班：20:00 - 08:00（夜班，人员减少但必须有）
  
  每班配置：
  - 值班组长 1人（决策、协调）
  - 监控研判 2-3人（看告警、做研判）
  - 应急响应 1-2人（出现事件时处理）
  - 日志审计 1-2人（深度日志分析）
```

#### 2. 情报联动

护网期间要实时关注外部情报：

```
情报来源：
□ 护网指挥部下发的通报（重点IP/域名/漏洞）
□ 行业共享群（同行发现的新攻击手法）
□ 威胁情报平台实时监控
□ 监管机构通报

情报联动流程：
收到情报 → 验证 → 确认影响范围 → 添加防护规则 → 通报全组
```

#### 3. 护网值守日报编写

日报是护网期间最重要的文档之一，管理层通过日报了解当前态势。

**日报模板：**

```markdown

### 四、护网后 —— 复盘整改

护网结束不代表工作结束，真正有价值的工作在复盘阶段。

**复盘要回答的问题：**

1. **发现层面**
   - 我们发现了多少攻击？有多少是"应该发现但没发现"的？
   - 哪些攻击手段我们之前没遇到过？

2. **响应层面**
   - 告警到处置的平均时间是多少？有没有改进空间？
   - 有没有因为处置不及时导致失分？

3. **防护层面**
   - 哪些漏洞被攻击者利用成功了？为什么之前没修？
   - 安全设备规则有没有优化空间（误报太多/漏报）？

4. **协作层面**
   - 团队配合有没有问题？信息流转是否顺畅？
   - 和管理层、业务部门的沟通是否到位？

**整改清单示例：**
- 修复高危漏洞5个
- 优化IDS规则12条（减少误报）
- 清理幽灵资产3台
- 更新应急响应流程
- 更新值守方案和排班策略

**Q3：护网中收到上级通报"你单位某IP被攻陷"，你的第一反应是什么？**

> 1. 确认：通报中的IP是不是自己单位的？哪个系统？
> 2. 隔离：立刻网络隔离该IP（交换机VLAN隔离或拔网线）——先止血
> 3. 取证：在隔离前快速截图（网络连接、进程、用户）
> 4. 排查：上机排查入侵痕迹（进程/网络/用户/文件/日志）
> 5. 汇报：向上级汇报排查进展（15分钟内给出初步判断）
> 6. 处置：清除后门、修复漏洞、改密码
> 7. 闭环：形成事件报告，更新防御策略

### 面试高频题

## 💡 面试高频题：护网全流程详解

**Q: 护网前（准备阶段）需要做哪些工作？**
A: ①资产梳理："你无法保护你不知道的东西"——盘点所有内外网资产（IP、域名、端口、服务、数据库、中间件），建立CMDB；②风险自查：漏洞扫描→漏洞修复→弱口令排查→开放端口收敛；③安全加固：操作系统/数据库/中间件基线检查与加固、WAF/IDS/EDR策略优化；④应急预案：制定护网期间的应急处置流程、值班安排、升级标准、联系人清单；⑤演练：组织一次模拟护网演习→发现问题→修正预案。护网前准备越充分，护网中越从容。

**Q: 护网中（值守阶段）的日常工作流程是什么？**
A: ①交班：接收上一班的未闭环事项和重点关注；②集中告警研判：处理夜间积累的告警，逐一研判→处置→闭环；③主动巡检：不是等告警——主动检查设备状态、日志趋势、流量异常；④情报联动：接收最新威胁情报，调整检测规则和封禁策略；⑤日报编写：当日告警总数、处置数量、攻击趋势分析、待办事项；⑥交班：向下一班交代清楚所有未尽事项。关键原则：护网期间"宁杀错不放过"——所有告警必须100%研判。

**Q: 护网日报应该怎么写？**
A: 护网日报模板：①今日态势总览（告警总数/处置数/待处置数/封禁IP数）；②TOP攻击源（排名前5的攻击IP及其攻击类型）；③今日重点事件（筛选1-3个需要关注的攻击事件，说明攻击链和处置结果）；④设备运行情况（各安全设备是否正常运行，有无异常）；⑤情报更新（今日收到的威胁情报和采取的对应措施）；⑥待办事项（未闭环的事件、需要下一班重点关注的事项）；⑦值班人员签名和工作时段。

**Q: 护网后（复盘阶段）应该做什么？**
A: ①数据统计（攻击趋势图、告警类型分布、处置时效分析）；②案例分析（选出护网期间最有代表性的3-5个攻击事件，做深度复盘——攻击者怎么进来的？我们怎么发现的？处置是否及时？哪里可以改进？）；③短板定位（哪些攻击类型我们没有检测能力？哪些设备策略需要优化？哪些流程需要改进？）；④整改计划（根据复盘中发现的短板，制定具体的整改措施和时间表）；⑤技战法沉淀（将本次护网中形成的检测规则、处置SOP、分析技巧沉淀为可复用的技战法文档）。

## 🎯 实战思维训练

### 蓝队"条件反射"训练

网络安全值守中，很多判断需要在几秒内完成。以下是本日主题相关的"条件反射"训练：

**看到以下现象 → 立即联想到 → 采取动作**：

1. 短时间内同一IP大量不同URL请求 → 目录/漏洞扫描 → 检查是否返回了不该返回的内容
2. WAF告警+同IP的Web日志中有500错误 → 攻击可能在尝试绕过WAF → 查看完整请求体
3. 非工作时间的管理员登录 → 凭据泄露/后门 → 确认是否为合法运维操作
4. 同一文件被频繁POST请求 → Webshell心跳 → 检查文件内容和创建时间
5. 出站流量突增到非标准端口 → 数据渗出/C2通信 → 追踪目标IP并阻断

## 🔬 进阶专题：本日知识在真实护网中的应用

### 护网场景还原

想象你正在护网值守中，突然收到一条与本日主题相关的告警。以下是标准的思维和处理流程：

**1. 第一反应（0-30秒）**：确认告警来源设备是否正常 → 查看告警级别 → 判断是否为"狼来了"

**2. 快速研判（30秒-2分钟）**：提取关键信息（源IP/目标资产/攻击类型/时间）→ 查威胁情报 → 看同一IP的其他告警

**3. 深度分析（2-10分钟）**：回放原始流量/日志 → 还原攻击payload → 判断攻击是否成功

**4. 处置决策（10-15分钟）**：确认是攻击→定级→封IP/隔离/升级事件；确认是误报→标记→优化规则

**5. 记录闭环（15-20分钟）**：填写工单（研判依据+处置动作+证据截图）→ 标记闭环


### 高手与新手的关键区别

| 维度 | 新手 | 高手 |
| --- | --- | --- |
| 看告警 | 只看这一条告警的内容 | 同时关联其他设备/其他时间的同类告警 |
| 判断依据 | "这个payload看起来像攻击" | "这个payload+这个User-Agent+这个时间点+这个频率=SQLMap自动化扫描" |
| 处置方式 | 封IP→完事 | 封IP+排查漏洞+检查是否已有数据泄露+通知业务方+更新WAF规则 |
| 记录质量 | "已处置" | "10:05收到WAF SQL注入告警→核查源IP 45.x为境外IP+威胁情报标记恶意→查看该IP完整请求链→确认SQLMap自动化扫描→已在WAF永久封禁+通知开发修复参数化查询→附件：攻击流量pcap+威胁情报截图" |

### 你今天学的知识在ATT&CK中的映射

花5分钟思考以下问题（有助于建立安全思维框架）：

- 今天的攻击手法在ATT&CK中属于哪个战术？对应的技术编号是什么？

- 用Kill Chain模型分析：这种攻击通常发生在哪几个阶段？

- 你的检测手段应该部署在Kill Chain的哪一环？为什么？

- 如果你负责设计针对这种攻击的防御方案，你会从哪几个层面入手？


> **记住**：蓝队不是"见招拆招"的被动防守，而是"知己知彼"的主动防御。每天学完一个知识点后，立刻思考"如果我是攻击者我会怎么用"+"如果我是防御者我该怎么挡"——这种双向思维是蓝队核心竞争力的来源。
