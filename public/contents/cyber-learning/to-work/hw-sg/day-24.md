## 📖 一、护网第一天——最疯狂的一天

护网第一天的攻击通常是最猛烈的。红队会集中火力进行"饱和式攻击"——同时从多个方向、用多种手法攻击你的所有暴露面。

### 📊 首日的典型数据

```
告警量：150-300条/小时（正常日可能只有10-20条）
攻击类型分布：
  扫描探测：      40%（大量端口扫描、目录枚举）
  Web攻击：       30%（SQL注入、XSS、文件上传尝试）
  暴力破解：      15%（SSH/RDP/数据库暴力破解）
  钓鱼投递：      10%（钓鱼邮件集中投递）
  疑似0day/高级：  5%（需要重点关注！）

你的状态：手忙脚乱 + 精神紧绷 + 咖啡续命
```

## 📖 二、首日生存法则——告警分级

### 🎯 告警分级标准

面对200+条/小时的告警，你不可能每条都手动研判。必须分级处理：

```
🔴 紧急（Priority 1）—— 立即手动处置
  特征：
  - 核心业务系统出现异常
  - 已确认入侵（Webshell上传成功、异常登录成功）
  - 数据正在外传
  
  处理：立即启动应急响应 + 通知值班领导
  时限：5分钟内开始处置

🟠 高危（Priority 2）—— 30分钟内处置  
  特征：
  - 高危攻击行为（SQL注入尝试、文件上传尝试）
  - 来自已知恶意IP的访问
  - 非工作时间管理员操作
  
  处理：自动封禁IP + 标记跟踪 + 批量研判
  时限：30分钟内完成研判

🟡 中危（Priority 3）—— 批量处理
  特征：
  - 扫描探测（端口扫描、目录扫描）
  - 低危Web攻击（被WAF拦截的）
  - 常规登录失败
  
  处理：自动封禁高频扫描IP + 定时Review
  时限：4小时一批次

🟢 低危（Priority 4）—— 定时Review
  特征：
  - 信息类告警
  - 已被WAF/防火墙自动拦截的攻击
  - 已知的正常告警噪声
  
  处理：定时批量查看 + 过滤噪声
  时限：每天2次Review
```

## 📖 三、首日上午班（08:00-12:00）任务

```
⏰ 08:00  交班：夜班通报昨晚情况
⏰ 08:15  态势查看：过去12小时告警统计
⏰ 08:30  批量处理：自动封禁高频攻击IP
⏰ 09:00  重点研判：昨晚的高危告警回顾
⏰ 10:00  新告警处置（逐条分级→自动批量+重点手判）
⏰ 11:00  中期统计：上午攻击趋势分析
⏰ 11:30  写上午简报（给领导）
⏰ 12:00  交班：向中班通报上午重点
```

### 📊 上午需要关注的几个数据

```
1. 告警总量 vs 正常基线
   正常：100条/小时 → 今天：300条/小时 → 确认异常

2. 攻击源IP分布
   前10个攻击IP的国家分布 → 判断是有组织攻击还是随机扫描

3. 被攻击最多的目标
   哪些系统被集火 → 这些是红队重点目标 → 加强监控

4. 有没有"突破性"告警
   有没有告警表明攻击者已经成功了？
   → Webshell上传成功? 登录成功? 数据外传?
```

## 📖 五、首日态势日报——怎么写？

### 📄 日报四要素

```
1️⃣ 态势综述（1段话）
   "截至18:00，护网首日整体可控。
    共收到告警XXXX条，自动处置XXXX条，
    重点事件X起已全部处置闭环。"

2️⃣ 重点事件（Top 3-5）
   每件事件包含：
   - 事件描述（什么时间、什么系统、发生了什么）
   - 处置措施（做了什么）
   - 当前状态（已闭环/处置中/待处置）
   - 影响评估（影响什么、影响多大）

3️⃣ 统计数据
   📊 告警总量：XXXX条
   📊 自动处置率：XX%
   📊 事件闭环率：XX%
   📊 封禁IP：XX个
   📊 高危告警：XX条（已处置XX条）

4️⃣ 待办事项
   - 还未闭环的事件
   - 明日需要关注的重点
   - 需要领导协调的事项
```

## 📖 七、夜班值守要点

护网期间的凌晨是最危险的——攻击者知道你累了：

```
🌙 夜班（20:00-08:00）特殊注意事项：

1. 必须双人值守
   绝对不能一个人值夜班！
   疲劳导致漏看告警是夜班最常见的翻车原因

2. 设置强力告警提醒
   高危告警 → 声音+弹窗+手机通知
   不能只靠屏幕上的小红点

3. 定时活跃度检查
   每2小时一次：起身走动、洗把脸、喝杯水
   设置闹钟提醒

4. 高频攻击自动处置
   凌晨大部分攻击是自动化的
   → 自动封禁规则可以开得激进一些
   → 可疑的直接封，天亮再Review

5. 交接班记录要详细
   不能只写"一切正常"
   要写：今晚有什么异常？处理了什么？还有什么待处理？
```

## 💻 八、动手试试：告警分级自动处置

```python
# 护网首日告警自动分级处置系统
class AlertTriage:
    def __init__(self):
        self.alerts = []
        self.blocked_ips = set()
        self.incidents = []
    
    def add_alert(self, severity, src_ip, category, description):
        """添加一条告警"""
        self.alerts.append({
            'severity': severity,
            'src_ip': src_ip,
            'category': category,
            'desc': description,
            'time': '14:30'
        })
    
    def triage(self):
        """告警分级自动处置"""
        urgent, high, medium, low = [], [], [], []
        
        for alert in self.alerts:
            sev = alert['severity']
            if sev == '紧急':
                urgent.append(alert)
                # 紧急告警 → 自动封禁+创建事件单
                self.blocked_ips.add(alert['src_ip'])
                self.incidents.append(f'INC-紧急: {alert["desc"]}')
            elif sev == '高危':
                high.append(alert)
                # 高危告警 → 自动封禁（如果来自外部）
                if not alert['src_ip'].startswith(('10.', '192.168.')):
                    self.blocked_ips.add(alert['src_ip'])
            elif sev == '中危':
                medium.append(alert)
            else:
                low.append(alert)
        
        # 输出分级结果
        print('=== 🚨 护网首日告警分级处置 ===\n')
        
        print(f'🔴 紧急: {len(urgent)}条 — 立即手动处置！')
        for a in urgent[:3]:
            print(f'   [{a["category"]}] {a["src_ip"]} → {a["desc"]}')
        
        print(f'\n🟠 高危: {len(high)}条 — 30分钟内研判')
        for a in high[:3]:
            print(f'   [{a["category"]}] {a["src_ip"]} → {a["desc"]}')
        
        print(f'\n🟡 中危: {len(medium)}条 — 批量处理')
        print(f'\n🟢 低危: {len(low)}条 — 定时Review')
        
        print(f'\n📊 总计: {len(self.alerts)}条告警')
        print(f'🔒 已自动封禁: {len(self.blocked_ips)}个IP')
        print(f'📋 已创建事件单: {len(self.incidents)}个')
        
        # 是否需要升级
        if len(urgent) > 0:
            print(f'\n⚠️  发现紧急告警！建议立即通知值班领导！')

# === 模拟首日告警洪流 ===
triage = AlertTriage()

# 模拟各类告警
triage.add_alert('紧急', '45.33.32.156', 'Webshell', 
    '检测到Webshell文件 /uploads/cmd.jsp — 可能是红队突破成功！')
triage.add_alert('高危', '103.224.182.250', 'SQL注入',
    '对 /api/user 发起SQL注入攻击，被WAF拦截')
triage.add_alert('高危', '185.220.101.34', '暴力破解',
    '对SSH端口(22)发起暴力破解，1分钟内尝试200+次')
triage.add_alert('中危', '192.168.1.200', '内网扫描',
    '内网主机发起大规模端口扫描，扫描了30+台机器')
triage.add_alert('中危', '89.248.165.59', '目录扫描',
    '对Web服务器发起目录枚举扫描')
triage.add_alert('低危', '8.8.8.8', '探测',
    '简单的ping扫描探测')

# 再来一批模拟告警
for i in range(191):
    triage.add_alert('低危', f'{i//50}.{i%256}.{i%128}.{i%255}', 
        '扫描探测', f'自动化扫描探测 #{i}')

triage.triage()
```
