## 📖 三、战前加固——十大必做项

```
☑ 1. 弱口令清零
   检查RDP、SSH、数据库、VPN、管理后台的密码
   强度要求：12位以上+大小写+数字+特殊字符
   工具：Hydra（自己测试自己）

☑ 2. 高危漏洞清零
   CVSS 7.0+ 的漏洞必须全部修复
   尤其关注：Log4j、Struts2、Shiro、Fastjson、Spring

☑ 3. 互联网暴露面收敛
   关掉不必要的端口
   管理后台加IP白名单
   测试环境移出公网

☑ 4. 安全设备策略收紧
   WAF启用严格模式
   防火墙策略改为"默认拒绝"
   IDS/IPS规则库更新到最新

☑ 5. 日志全覆盖
   确认所有关键系统日志接入SIEM
   日志保留≥6个月
   关键告警规则测试通过

☑ 6. EDR/HIDS全终端部署
   服务器+办公电脑+运维终端 100%覆盖
   确认Agent在线

☑ 7. 蜜罐布设
   内网核心区5-10个蜜罐
   名字像真实资产

☑ 8. 备份核验
   核心业务数据备份完整性验证
   备份恢复测试（确保能恢复！）

☑ 9. 应急预案就绪
   各类突发事件SOP准备完毕
   所有人知道自己的应急职责

☑ 10. 第三方排查
   合作伙伴VPN接入核查
   外包人员账号清理
   供应链安全检查
```

## 📖 四、资产清册——最基础也最重要

### 📋 一张好的资产清册长什么样

```
不能只是："我们有50台服务器"
必须有：

| 资产名称 | IP地址 | 操作系统 | 开放端口 | 业务描述 | 安全等级 | 责任人 | 是否公网暴露 |
|----------|--------|----------|----------|----------|----------|--------|-------------|
| WEB-01 | 203.0.113.10 | CentOS 7.9 | 80,443 | 在线商城首页 | 核心 | 张三 | ✅ |
| WEB-02 | 203.0.113.11 | CentOS 7.9 | 80,443 | 在线商城首页 | 核心 | 张三 | ✅ |
| DB-01 | 10.0.0.20 | MySQL 8.0 | 3306 | 交易数据库 | 核心 | 李四 | ❌ |
| HR-SRV | 10.0.0.30 | Win2019 | 3389,80 | HR系统 | 重要 | 王五 | ❌ |
| DEV-TEST | 10.0.0.99 | Ubuntu | 22,8080 | 开发测试 | 一般 | 赵六 | ❌ |
```

### 🔍 资产排查中常见的"惊喜"

```
😱 "这台服务器是什么？谁部署的？"—— 僵尸服务器
😱 "这个端口为什么开着？"—— 前任运维开的测试端口忘了关
😱 "这个域名指向了内网？"—— DNS配置错误导致的暴露
😱 "这个管理后台没有密码？"—— 默认配置没改
😱 "这个API有外网IP能直接访问？"—— 安全组配置遗漏
```

### 🎤 启动会必备内容

```

  - 护网行动是什么
  - 这次护网的时间、规模、目标
  - 我们为什么重要（哪些核心业务必须守住）

  📊 展示护网指挥架构图：
  ┌──────────────────────┐
  │   总指挥（CTO/安全总监） │
  ├──────┬───────┬───────┤
  │ 监测组 │ 处置组 │ 后勤组 │
  │(盯告警)│(应急响应)│(餐饮排班)│
  └──────┴───────┴───────┘

第三部分：纪律要求（5分钟）
  ⚠️ 保密要求：护网期间不得对外透露任何细节
  ⚠️ 值班制度：三班倒，严格交接
  ⚠️ 上报流程：什么情况通知谁（附晋级表）

  📱 护网专用企业微信群
  📞 紧急联系方式（24小时畅通）
  📧 日报提交地址

第五部分：应急启动条件（3分钟）
  什么情况启动什么级别的应急响应
  谁有权启动
```

## 📖 六、后勤保障——护网中最被低估的环节

```
🕐 排班表：
  早班 08:00-16:00 （2人）
  中班 16:00-00:00 （2人）
  晚班 00:00-08:00 （2人，必须双人！）
  机动/备勤 （2人，随时顶班）

🍜 餐饮：
  - 一日三餐 + 夜宵
  - 咖啡/茶/红牛不限量供应
  - 泡面/饼干/水果常备

🏠 休息：
  - 值班室要有沙发/行军床
  - 备勤人员可以休息

🔌 设备：
  - 备机（至少1台）
  - 备件（网线/电源/硬盘）
  - 充电器/充电宝

🧠 心理：
  - 护网是高压力工作
  - 领导定期到一线慰问
  - 排班合理，不要连续夜班超过3天
```

## 💻 七、动手试试：战前检查清单系统

```python
# 护网战前准备跟踪系统
from datetime import datetime, timedelta

class PreHWChecker:
    def __init__(self, hw_start_date):
        self.hw_start_date = datetime.strptime(hw_start_date, '%Y-%m-%d')
        self.today = self.hw_start_date  # 默认今天就是护网开始日
        self.checklist = []
    
    def set_today(self, date_str):
        """设置"今天"的日期，用于计算倒计时"""
        self.today = datetime.strptime(date_str, '%Y-%m-%d')
    
    def add_item(self, category, task, days_before_hw, priority='中'):
        """
        days_before_hw: 需要在护网前多少天完成
        """
        deadline = self.hw_start_date - timedelta(days=days_before_hw)
        self.checklist.append({
            'category': category,
            'task': task,
            'deadline': deadline,
            'days_before': days_before_hw,
            'priority': priority,
            'done': False
        })
    
    def mark_done(self, task_name):
        """标记任务完成"""
        for item in self.checklist:
            if item['task'] == task_name:
                item['done'] = True
                break
    
    def report(self):
        """输出战前检查报告"""
        days_left = (self.hw_start_date - self.today).days
        
        print(f'{"="*55}')
        print(f'🛡️  护网战前检查清单')
        print(f'📅 护网开始: {self.hw_start_date.strftime("%Y-%m-%d")}')
        print(f'📅 今天: {self.today.strftime("%Y-%m-%d")}')
        print(f'⏰ 距护网还有: {days_left} 天')
        print(f'{"="*55}\n')
        
        # 按紧急程度分类
        overdue = []  # 逾期未完成
        urgent = []   # 今天该完成但还没做
        upcoming = [] # 未来需要完成
        done = []     # 已完成
        
        for item in self.checklist:
            if item['done']:
                done.append(item)
            elif self.today > item['deadline']:
                overdue.append(item)
            elif self.today.date() == item['deadline'].date():
                urgent.append(item)
            else:
                upcoming.append(item)
        
        # 输出报告
        if overdue:
            print('🔴【逾期未完成】—— 必须立即处理！')
            for item in overdue:
                days_overdue = (self.today - item['deadline']).days
                print(f'  ❌ [{item["category"]}] {item["task"]}')
                print(f'     应于 {item["deadline"].strftime("%m-%d")} 前完成，已逾期 {days_overdue} 天')
            print()
        
        if urgent:
            print('🟡【今日必须完成】')
            for item in urgent:
                print(f'  ⚡ [{item["category"]}] {item["task"]}')
                print(f'     截止日期: 今天')
            print()
        
        if upcoming:
            print('🟢【待完成（未逾期）】')
            for item in sorted(upcoming, key=lambda i: i['deadline']):
                d_left = (item['deadline'] - self.today).days
                print(f'  ☐ [{item["category"]}] {item["task"]} (还剩 {d_left} 天)')
            print()
        
        if done:
            print(f'✅【已完成】({len(done)}项)')
        
        # 统计
        total = len(self.checklist)
        completed = len(done)
        pct = completed / total * 100
        
        print(f'\n📊 完成进度: {completed}/{total} = {pct:.0f}%')
        print(f'   🔴 逾期: {len(overdue)}项')
        print(f'   🟡 今日: {len(urgent)}项')
        print(f'   🟢 待做: {len(upcoming)}项')

# === 模拟使用 ===
chk = PreHWChecker('2024-06-24')  # 护网6月24日开始
chk.set_today('2024-06-21')  # 今天是6月21日（距护网3天）

# 添加检查项
chk.add_item('资产盘点', '互联网暴露面全部扫描并编制清单', 7, '高')
chk.add_item('资产盘点', '内网资产清单更新核对', 5, '高')
chk.add_item('漏洞修复', '高危漏洞100%修复', 5, '高')
chk.add_item('漏洞修复', '弱口令全部清零', 5, '高')
chk.add_item('安全设备', 'WAF/IDS策略收紧到护网模式', 3, '高')
chk.add_item('安全设备', '蜜罐部署完成并测试', 3, '中')
chk.add_item('日志审计', '全系统日志接入SIEM', 5, '高')
chk.add_item('日志审计', '关键告警规则测试通过', 3, '高')
chk.add_item('人员', '三班倒排班表确认', 2, '高')

# 模拟一些已完成
chk.mark_done('互联网暴露面全部扫描并编制清单')
```
