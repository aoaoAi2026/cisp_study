## 📖 一、什么是一张好的防守蓝图？

想象你要保护一栋大楼：

```
❌ 糟糕的防守：只在门口放了一个保安
   → 小偷翻窗 → 没人管
   → 小偷走后门 → 没人管
   → 小偷走地下车库 → 没人管

✅ 好的防守：多层防线相互覆盖
   ① 外墙监控（所有入口）
   ② 大堂门禁（刷卡才能进）
   ③ 楼层巡逻（保安定期走动）
   ④ 房间门锁（各自上锁）
   ⑤ 保险柜（最值钱的双重加密）

→ 这就是纵深防御（Defense in Depth）！
```

## 📖 二、纵深防御五层架构

```
第一层 ┌────────────────────────────────┐
 🧱    │  网络边界层                        │
       │  防火墙 + WAF + DDoS防护 + VPN     │
       │  "外人别想随便进来"                 │
       └────────────────────────────────┘
                  ↓ 突破
第二层 ┌────────────────────────────────┐
 🔍    │  网络内部层                        │
       │  IDS/IPS + 流量分析 + 蜜罐 + NAC   │
       │  "进来了就在监控下，一举一动都看着"   │
       └────────────────────────────────┘
                  ↓ 突破
第三层 ┌────────────────────────────────┐
 🖥️    │  主机终端层                        │
       │  HIDS + EDR + 杀毒 + 堡垒机         │
       │  "每台机器都有各自的特工保护"         │
       └────────────────────────────────┘
                  ↓ 突破
第四层 ┌────────────────────────────────┐
 🕸️    │  应用数据层                        │
       │  RASP + WAF + 代码审计 + API网关    │
       │  "应用和数据本身就有防护"            │
       └────────────────────────────────┘
                  ↓ 突破
第五层 ┌────────────────────────────────┐
 🗄️    │  数据核心层                        │
       │  DLP + 数据库审计 + 加密 + 备份     │
       │  "就算拿到文件，也读不了、带不走"    │
       └────────────────────────────────┘
```

> 💡 核心思想：**不指望任何一层100%拦住敌人，但每一层都让敌人更难走，拖慢他们的速度，增加被发现的机会。**

## 📖 三、护网防守第一步：收敛暴露面

### 🎯 暴露面是什么？

```
暴露面 = 你在互联网上能被攻击者看到并攻击的所有东西

包括：
  ✅ 网站（www.你的公司.com, mail.你的公司.com...）
  ✅ IP地址的开放端口（22, 3389, 3306...）
  ✅ VPN/远程接入入口
  ✅ 云上的管理后台
  ✅ 第三方系统（外包商、合作伙伴接入接口）
  ✅ 暴露在公网的API接口
  ✅ 员工GitHub上泄露的配置/密码
```

### ✂️ 怎么收敛暴露面？

```
第一步：全面数字清点（你有哪些资产？）
  ✅ 列出所有域名和子域名
  ✅ 扫描所有IP段的开放端口
  ✅ 盘点所有云服务资源
  ✅ 检查所有第三方接入

第二步：关、藏、限
  ❌ 不必要的端口 → 关掉！
  ❌ 管理后台 → 藏到内网或加IP白名单
  ❌ 测试环境 → 不要暴露在公网
  ❌ 旧系统 → 该下线的下线
  ❌ 默认页面 → 删掉或隐藏
  
  收敛前：暴露面 = 50个域名 × 30个端口 = 1500个攻击点
  收敛后：暴露面 = 10个必要域名 × 5个端口 = 50个攻击点
  → 攻击面缩小了97%！
```

## 📖 四、网络分段——把城堡分成多个独立区域

```
❌ 扁平网络（最危险）：
  所有服务器 + 所有办公电脑 + 所有打印机 + 所有摄像头
  全在同一个网段
  → 攻击者进入任意一台 = 看到所有设备
  
✅ 分区分段（安全）：
  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
  │ 办公网   │  │ 生产网   │  │ 数据库网 │  │ 运维管理 │
  │ VLAN 10 │  │ VLAN 20 │  │ VLAN 30 │  │ VLAN 40 │
  │ PC/打印机│  │Web/应用  │  │ DB/存储  │  │堡垒机/跳 │
  └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘
       │            │            │            │
       └────────────┴─────┬──────┴────────────┘
                          │
                    防火墙策略控制
              （默认拒绝 + 最小开放）
```

### 🔥 防火墙策略黄金法则

```
1️⃣ 默认拒绝（Default Deny）
   不要用"允许所有，拒绝已知坏"
   要用"拒绝所有，只放行确认需要的"

2️⃣ 最小权限（Least Privilege）
   办公网不需要访问数据库网 → 拒绝
   测试网不需要访问生产网 → 拒绝
   只开确实需要的端口

3️⃣ 东西向流量控制
   不只管"从外到内"（南北向）
   还要管"横向"（东西向）
   服务器之间也不能随意互相访问
```

## 📖 五、零信任架构——谁也不信

### 🤔 零信任是什么？

```
传统安全模型：信任但验证
  "内网是安全的" → 进来就信了
  → 一旦突破边界 = 全完

零信任模型：从不信任，始终验证
  "内网也不安全" → 每次访问都要验证
  → 即使在内网里，也要证明"你是谁"
```

### 🔑 零信任的基本原则

```
1️⃣ 每次访问都验证身份
   不是"早上登录一次管一整天"
   而是每次访问新资源都要重新验证

2️⃣ 设备也要检查
   不是"你说是张三是张三"
   还要看"你的电脑打了补丁吗？装了EDR吗？"

3️⃣ 最小权限动态授予
   不是"你是开发人员所以能看到所有代码"
   而是"你只给你正在做的这个项目相关的代码权限"

4️⃣ 持续监控和评估
   行为有异常 → 立即撤销权限
   不是你昨天登录了今天就不用再验证
```

## 📖 六、护网防守方案核心要点

### ⚔️ 防守方案 = 人 + 技术 + 流程

```
1️⃣ 人（排班表 & 后勤）
   ✅ 24小时三班倒排班表
   ✅ 每班至少2人（双人值守防遗漏）
   ✅ 技术领导On-call（紧急决策）
   ✅ 餐饮、休息区、咖啡因储备
   ⚠️ 最常见的失败原因：人员疲劳导致漏看告警

2️⃣ 技术（设备 & 联动）
   ✅ 安全设备清单和策略确认
   ✅ 设备联动：WAF→SIEM→SOAR→EDR
   ✅ 日志全覆盖（一个系统都不能少）
   ✅ 备机备件（关键设备坏了能替换）

3️⃣ 流程（SOP & 预案）
   ✅ 告警分级和SLA：紧急30分钟/高危1小时
   ✅ 事件升级流程：什么情况通知谁
   ✅ 日报模板：每天汇报什么
   ✅ 应急预案：Webshell/勒索/数据泄露怎么处置
```

## 💻 七、动手试试：蓝队防御能力评分

```python
# 蓝队防御能力多维度评分系统
class DefenseScorer:
    def __init__(self, company_name):
        self.company_name = company_name
        self.scores = {}
        self.recommendations = []
    
    def assess(self, domain, items, weight=1.0):
        """
        评估一个安全域
        items: [(检查项, 是否完成, 说明), ...]
        weight: 该域在整个防御体系中的权重
        """
        total = len(items)
        done = sum(1 for _, status, *_ in items if status)
        self.scores[domain] = {
            'done': done,
            'total': total,
            'percentage': done / total * 100,
            'weight': weight,
            'items': items
        }
    
    def generate_recommendations(self):
        """针对未完成项生成建议"""
        for domain, data in self.scores.items():
            for item in data['items']:
                name, status, *extra = item
                if not status:
                    desc = extra[0] if extra else '需要优先处理'
                    self.recommendations.append({
                        'domain': domain,
                        'item': name,
                        'desc': desc
                    })
    
    def report(self):
        """输出综合评分报告"""
        print(f'=== 🛡️ {self.company_name} 蓝队防御能力评分 ===\n')
        
        total_weighted = 0
        max_weighted = 0
        
        for domain, data in self.scores.items():
            pct = data['percentage']
            done = data['done']
            total = data['total']
            weight = data['weight']
            
            total_weighted += pct * weight
            max_weighted += 100 * weight
            
            # 可视化条形图
            bar_len = 20
            filled = int(bar_len * pct / 100)
            bar = '█' * filled + '░' * (bar_len - filled)
            
            # 评级
            if pct >= 80: grade = '🟢 优秀'
            elif pct >= 60: grade = '🟡 良好'
            elif pct >= 40: grade = '🟠 需改进'
            else: grade = '🔴 严重不足'
            
            print(f'{domain} (权重×{weight})')
            print(f'  [{bar}] {pct:.0f}% ({done}/{total}) {grade}')
        
        overall = total_weighted / max_weighted * 100
        print(f'\n{"="*50}')
        print(f'综合评分: {overall:.0f}%')
        
        # 优先改进建议
        if self.recommendations:
            print(f'\n⚡ 优先改进项 (共{len(self.recommendations)}项):')
            for i, rec in enumerate(self.recommendations[:5], 1):
                print(f'  {i}. [{rec["domain"]}] {rec["item"]} — {rec["desc"]}')
        
        return overall

# === 对某公司进行防御评估 ===
ds = DefenseScorer('XX证券公司')

ds.assess('网络边界', [
    ('下一代防火墙部署', True),
    ('WAF保护所有Web应用', True),
    ('DDoS防护服务', False, '建议接入云DDoS清洗服务'),
    ('互联网暴露面收敛', True),
    ('VPN多因素认证', False, 'VPN仅靠密码不安全，需加MFA'),
], weight=1.5)

ds.assess('主机终端', [
    ('EDR全终端覆盖', True),
    ('HIDS部署(如Wazuh)', True),
    ('防病毒软件更新', True),
    ('堡垒机/跳板机', False, '运维直接SSH危险，需部署堡垒机'),
    ('操作系统加固基线', True),
], weight=1.3)

ds.assess('日志审计', [
    ('日志集中到SIEM', True),
    ('核心系统日志全部接入', True),
    ('日志保存≥6个月', True),
    ('关键告警规则已配置', False, '部分系统只有日志没有告警规则'),
    ('日志防篡改', True),

ds.assess('应用安全', [
    ('所有Web应用通过WAF', True),
    ('定期代码安全审计', False, '半年没做代码审计了'),
    ('API安全网关', False, 'API缺乏统一安全管控'),
    ('RASP运行时防护', False, '建议部署RASP作为最后一道防线'),

ds.assess('人员流程', [
    ('24小时护网排班表', True),
```

## 📝 九、今日测验

**Q1：纵深防御的核心思想是什么？**
- A. 单一防护
- B. 多层防线相互补充，任何一层被突破仍有后续防护  ✅
- C. 只防外部
- D. 只靠防火墙

> 不指望一层100%防住，但每一层都让攻击者更难走、更容易被发现。

**Q2：护网防守方案中最容易被忽视的环节是？**
- A. 防火墙策略
- B. 第三方/VPN/远程接入的管控  ✅
- C. WAF部署
- D. IDS规则

> 第三方接入和远程办公VPN是最薄弱的入口之一，护网中大量失分源于此。

**Q3：网络分段的主要目的是什么？**
- A. 方便管理
- B. 限制攻击者横向移动范围  ✅
- C. 提高网速
- D. 降低成本

> 分段后攻击者从办公网进来不等于能到数据库区，限制了攻击范围。

**Q4：零信任架构的核心原则是什么？**
- A. 信任内网
- B. 从不信任，始终验证（Never Trust, Always Verify）  ✅
- C. 信任用户
- D. 信任设备

> 零信任假设网络始终处于不安全状态，对所有访问都进行持续验证。

**Q5：防火墙策略的黄金法则是什么？**
- A. 允许所有
- B. 默认拒绝 + 最小权限开放  ✅
- C. 只拒绝已知威胁
- D. 全开方便

> 先全部拒绝，再按需开放——这是最安全的策略。反过来先全开再按需关闭是非常危险的。
