## 📖 一、为什么日志和流量是护网的"眼睛"

想象一下：你是一个商场的保安，要抓小偷。但是商场里**一个监控摄像头都没有**，而且你**连商家卖什么东西都不知道**——这怎么抓？

同样道理，在护网行动中：

- **日志** = **监控录像机**：记录着谁在什么时间做了什么
- **流量** = **安检扫描门**：每个人进出商场带的包裹都得扫描

如果没有日志和流量数据，网络被攻击了都不知道，这就叫"裸奔"。

## 📖 二、日志都从哪里来？

你的企业网络里有无数设备，它们都在默默记录日志，就像每个房间都有一个监控头：

| 设备类型 | 日志内容 | 像什么？ |
|----------|----------|----------|
| 🖥️ Windows/Linux系统 | 谁登录了、谁创建了文件、谁改了配置 | 房间的门禁刷卡记录 |
| 🌐 防火墙 | 谁想进来、谁想出去、被拦住了吗 | 商场大门的保安登记簿 |
| 📧 邮件服务器 | 谁发了邮件、附件是什么、收件人是谁 | 快递收发记录 |
| 🕸️ Web服务器 | 谁访问了哪个页面、传了什么参数 | 每个店铺的顾客登记表 |
| 🗄️ 数据库 | 谁查了什么数据、改了什么记录 | 仓库的出库入库单 |
| 🔑 域控制器 | 谁申请了门禁卡、谁密码输错了 | 保安科的门禁管理系统 |

### 📋 日志收集策略——不能什么都收，也不能不收

**最优策略是这样思考的**：

```
🟢 登录认证日志 —— 必须收！攻击者肯定要登录某个系统
🟢 Web访问日志   —— 必须收！90%的攻击都走Web
🟢 进程创建日志 —— 必须收！能发现可疑程序的执行
🟡 DNS查询日志   —— 建议收！能发现隐蔽的C2通信通道
🟡 网络流量镜像  —— 按需收！存储成本高，选重点区域
🔴 应用调试日志  —— 可以不收！但出问题了会很后悔
```

> 💡 **小白理解：** 你就想一个问题——"攻击者要做什么，得留下什么记录？"你的答案，就是要收集的日志。

## 📖 三、流量分析——看网络里的"水流"

### 🌊 什么是流量分析？

把网络比作城市的水管系统：
- 正常的流量 = 居民家里正常的用水（有规律、量不大）
- 异常的流量 = 半夜有人从水库大量抽水（反常！）

**流量分析就是：持续监控网络上的"水流"，一发现不对劲就报警。**

### 🔍 NTA（网络流量分析）能发现什么？

| 发现什么 | 具体表现 | 像什么？ |
|----------|----------|----------|
| 🎯 C2通信 | 服务器每隔固定时间与外网IP通信 | 卧底定时给总部发暗号 |
| 🕳️ DNS隧道 | DNS查询量暴增+域名超长 | 用无线电暗语传递情报 |
| 📤 数据外传 | 某台机器突然向外发大量数据 | 半夜雇卡车往外搬仓库 |
| 🔄 横向扫描 | 一台机器快速访问几十台内网机器 | 小偷挨个试每家门锁 |

### 📊 最经典的异常模式：C2心跳检测

攻击者控制了你的机器后，机器需要定期联系攻击者的服务器（C2服务器）获取指令。这种通信就像人的心跳，很有规律：

```
正常用户的网络行为：👉 时快时慢、没有规律
C2的通信行为：      👉 每隔30分钟，发一个固定大小的数据包
```

这就是**特征**！流量分析就是找这些特征。

## 📖 四、护网期间日志分析重点——看哪几个关键日志

护网时你不可能把每行日志都看一遍（可能一天上千万行），得**挑重点看**：

### 🔑 第一大重点关注：登录认证日志

```
👀 看什么：
  - 哪个IP登录了？（境外IP登录内网？🚨）
  - 几点登录的？（凌晨3点CEO账号登录？🚨）
  - 登录成功还是失败了？（同一账号1分钟失败100次？🚨）
  - 从哪个城市登录？（10分钟前在北京、现在在伦敦？🚨）
```

### 🌐 第二大重点关注：Web访问日志

```
👀 看什么：
  - 谁在访问 /admin 后台？（不在白名单的IP？🚨）
  - URL参数里有奇怪的东西吗？(比如 ?q=1' OR '1'='1 ← SQL注入)
  - 有没有人拼命报404？（大概率在扫描）
  - 有没有大文件上传？
```

### 🔄 第三大重点关注：进程创建日志（Windows的Sysmon日志）

```
👀 看什么：
  - Word.exe 启动了 cmd.exe？（很可疑！Word不该启动命令行）
  - 谁用 wmic.exe 远程执行了命令？（典型的横向移动）
  - 出现了 mimikatz.exe 等黑客工具？🚨🚨🚨
```

## 💻 六、动手试试：流量异常检测小工具

```python
# 网络流量异常分析——护网必备小脚本
from collections import defaultdict

class TrafficAnalyzer:
    def __init__(self):
        self.flows = []
    
    def add_flow(self, src_ip, dst_ip, dst_port, bytes_sent):
        """记录一条网络流"""
        self.flows.append({
            'src': src_ip, 'dst': dst_ip, 
            'port': dst_port, 'bytes': bytes_sent
        })
    
    def detect_anomalies(self):
        """检测异常流量"""
        alerts = []
        src_bytes = defaultdict(int)
        src_dsts = defaultdict(set)
        
        for flow in self.flows:
            src_bytes[flow['src']] += flow['bytes']
            src_dsts[flow['src']].add(flow['dst'])
        
        for ip, total in src_bytes.items():
            # 条件1：大流量外传（超过100MB）
            if total > 100_000_000:
                alerts.append(
                    f'🔴 [数据外传] {ip} ➜ 发出了 {total/1024/1024:.1f}MB'
                )
            # 条件2：短时间内访问太多目标（横向扫描）
            if len(src_dsts[ip]) > 20:
                alerts.append(
                    f'🟡 [横向扫描] {ip} ➜ 扫描了 {len(src_dsts[ip])} 台主机'
                )
        return alerts

# === 跑一下试试 ===
print('=== 🛡️ 护网流量异常分析 ===')
ta = TrafficAnalyzer()

# 模拟正常流量
ta.add_flow('10.0.0.5', '1.2.3.4', 443, 50_000_000)
ta.add_flow('10.0.0.5', '1.2.3.4', 443, 40_000_000)

# 模拟横向扫描（一台机器访问很多目标）
for i in range(25):
    ta.add_flow('10.0.0.8', f'192.168.1.{i}', 445, 1000)

# 运行检测
for alert in ta.detect_anomalies():
    print(alert)
```

## 🧠 十、专家锦囊

> **赵安全说：** 护网期间日志分析三步法——①快速排查（看高危告警、异常登录、大流量外传）→②深度分析（看时间关联、多源交叉验证）→③溯源追踪（看攻击路径、横向移动链）。不要被海量日志淹没，要有分析框架。

> **钱运维说：** 在护网中，日志可能会被攻击者删除或篡改，但网络流量镜像数据攻击者很难清除。建议护网前一定要部署好全流量留存和NTA分析，这是最后的防线。
