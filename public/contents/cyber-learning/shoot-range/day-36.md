## 29.1 Docker是什么？（大白话版）

### 29.1.1 先讲个故事

在讲Docker之前，老师先给大家讲个真实的故事。📖

以前啊，有个程序员小哥，他在自己电脑上写了一个网站，写得特别好，运行也很流畅。老板一看，说："不错！部署到服务器上去吧！"

于是小哥就开始在服务器上折腾：
- 先装个Apache服务器
- 再装个MySQL数据库
- 再装个PHP，版本还得对，不对就出问题
- 然后还要配置各种参数、权限、环境变量……

折腾了整整一天，终于弄好了，结果一运行——报错了！😱

小哥懵了："我电脑上明明好好的啊！怎么到服务器上就不行了？"

然后又是查日志、改配置、找原因，折腾了好几天，终于弄好了。结果过了一段时间，服务器要升级，一升级，又坏了……

是不是听起来就很崩溃？😂

这种"我电脑上能跑，你电脑上不能跑"的问题，在程序员的世界里太常见了！大家还给它起了个名字，叫——**"环境问题"**。

那怎么解决这个问题呢？Docker就是来干这个的！

### 29.1.2 Docker就像集装箱！

好啦，说了这么多，Docker到底是个啥？

**一句话：Docker就像是集装箱！**📦

啥？集装箱？你没听错！就是港口那种大大的、方方的集装箱！

大家想想，集装箱有什么好处？

🏭 **好处一：标准化包装**
不管里面装的是什么——是衣服、是食品、是机器零件、还是玩具，都用一样的集装箱来装。不用管里面是什么，外面的尺寸都是统一的。

🚢 **好处二：随便搬**
集装箱装好了之后，可以用卡车拉，可以用火车运，可以用船装，甚至可以用飞机运。不管换到什么交通工具上，都不用把东西拿出来重新打包，直接整个箱子搬过去就行。

🔒 **好处三：互相隔离**
一个箱子里装的是海鲜，另一个箱子里装的是衣服，它们互不影响。海鲜的味儿不会跑到衣服那边去，衣服的毛絮也不会掉到海鲜里面。每个箱子都是独立的。

💰 **好处四：节省空间**
集装箱可以一个叠一个，整齐地码在一起，不会乱七八糟的，能节省很多空间。

那Docker呢？Docker也是一样一样的！

📦 **Docker的"集装箱"叫"镜像"（Image）**
你可以把你的应用（比如一个网站、一个数据库、一个漏洞环境）和它需要的所有依赖（比如Apache、MySQL、PHP、各种库文件），都打包到一个"镜像"里面。这个镜像就相当于一个集装箱，里面东西都装好了，齐全得很。

🚀 **镜像搬到哪都能跑**
你在自己电脑上打包好的镜像，可以直接搬到服务器上运行，不用重新安装、重新配置。只要服务器上装了Docker，就能直接跑这个镜像，保证和你电脑上的效果一模一样！再也不会出现"我电脑上能跑你电脑上不能跑"的问题了！

🏠 **运行起来的镜像叫"容器"（Container）**
镜像就像是一个"模板"，或者说"安装包"。你用这个模板"创建"一个运行的实例，就叫"容器"。
打个比方：镜像就像是Word文档的模板，容器就是你用模板新建出来的具体文档。一个模板可以创建很多个文档，对吧？镜像也是一样，一个镜像可以创建很多个容器。

🏪 **仓库（Registry）就是"应用商店"**
大家手机上都有应用商店吧？想装什么APP，去应用商店搜一下，下载下来就能用。Docker也有类似的东西，叫"仓库"。里面存放了各种各样的镜像，你想要什么，去搜一下，拉下来就能用。
最有名的公共仓库就是Docker Hub，相当于Docker界的"苹果商店"或者"应用宝"。

### 29.1.4 为什么要用Docker？

讲了这么多，大家可能会问："老师，Docker这么好，那它具体有什么好处呢？"

问得好！老师给大家总结了四大好处：

✅ **好处一：环境一致性——再也不怕"玄学问题"了**

这个我们前面讲过了，Docker把应用和依赖都打包在一起，不管在什么环境下运行，都是一样的。你在你电脑上复现的漏洞，到我电脑上一样能复现，不会出现"怎么你那能行我这不行"的情况。

对于我们学安全的来说，这点太重要了！因为有时候一个漏洞能不能复现成功，差一点都不行。PHP版本差0.1，可能就利用不了；Apache配置差一点，可能就出问题。有了Docker，这些问题都不存在了！

✅ **好处二：快速部署——几分钟搞定一个环境**

手动搭一个漏洞环境，快的话几十分钟，慢的话几小时甚至几天。用Docker呢？快的话几分钟，慢的话十几分钟，一个完整的环境就跑起来了！

省下来的时间干嘛去？当然是用来研究漏洞、练习技术啦！我们的时间很宝贵的，不能都浪费在搭环境上，对吧？

✅ **好处三：隔离性好——随便折腾不怕坏**

每个Docker容器都是独立的，你在这个容器里随便搞，删文件、改配置、装病毒……都不会影响到你的主机系统，也不会影响到其他容器。

学安全的小伙伴，谁还没把自己系统搞崩过几次啊？😂 用Docker就不怕了，大不了把容器删了，重新来一个呗，几秒钟的事儿！

✅ **好处四：节省资源——比虚拟机轻量多了**

大家想想，我们用VMware跑一个虚拟机，得分配多少资源？内存至少得给1G吧？硬盘至少得给10G吧？跑起来CPU、内存都占不少。

那Docker呢？Docker容器是共享主机系统内核的，不用模拟一整套硬件。一个容器可能只占几十MB的空间，启动只需要几秒钟，资源占用比虚拟机小多了！

你电脑上同时跑10个虚拟机可能就卡得不行了，但跑10个Docker容器，说不定还很流畅！就是这么牛！🐂

## 29.2 Docker安装

好啦，原理讲了这么多，大家是不是已经手痒痒了？别急，我们现在就来安装Docker！

考虑到我们打靶一般都是在Kali Linux里进行的，所以老师重点讲Kali下的安装方法。Windows的也简单提一下，大家有个了解就行。

### 29.2.1 Kali Linux下安装Docker（重点！）

Kali Linux是基于Debian的，所以安装方法和Debian/Ubuntu差不多。老师给大家推荐最简单的方法——一键安装脚本！

#### 方法一：官方一键脚本（推荐！）

这个方法最简单，一条命令就能搞定，而且会自动配置好镜像源，下载速度快！

**第一步：打开终端**

这个不用多说了吧？在Kali里找个终端打开就行。

**第二步：输入安装命令**

把下面这条命令复制进去，回车：

```bash
curl -fsSL https://get.docker.com | bash -s docker --mirror Aliyun
```

> 💡 **小解释：**
> - `curl` 是一个下载工具，用来从网上下载东西
> - `-fsSL` 是curl的参数，不用管，照着写就行
> - `https://get.docker.com` 是Docker官方的安装脚本地址
> - `| bash` 意思是把下载下来的脚本交给bash执行
> - `-s docker --mirror Aliyun` 是给脚本传参数，说要用阿里云的镜像源，这样下载速度快

这条命令会自动帮你做很多事情：
- 检测你的系统版本
- 添加Docker的软件源
- 安装Docker及其依赖
- 配置好阿里云镜像加速

你就耐心等它执行完就行了，可能需要几分钟，取决于你的网络速度。☕

**第三步：启动Docker服务**

安装完成后，我们需要启动Docker服务：

```bash
systemctl start docker
```

> 💡 **小知识：** `systemctl` 是Linux系统里用来管理服务的命令。`start` 就是启动，后面跟服务名。

**第四步：设置开机自启**

为了方便，我们可以让Docker开机就自动启动，省得每次都手动开：

```bash
systemctl enable docker
```

`enable` 就是"启用"的意思，设置成开机自启。

**第五步：验证安装**

安装好了，我们来验证一下，看看是不是装成功了：

```bash
docker --version
```

如果输出了类似这样的东西，说明安装成功啦！🎉

```
Docker version 24.0.5, build 24.0.5-1~kali1
```

版本号可能不一样，没关系，只要有输出就行。

#### 方法二：apt安装（备选）

如果上面的方法不行，你也可以用Kali自带的源来安装，就是版本可能稍微旧一点，但不影响使用。

```bash
apt update
apt install -y docker.io
```

然后同样的，启动服务、设置开机自启、验证安装：

```bash
systemctl start docker
systemctl enable docker
docker --version
```

> ⚠️ **注意：** 如果你用普通用户身份运行Docker命令，可能会报错说权限不够。解决方法有两个：
> 1. 在命令前面加 `sudo`，比如 `sudo docker --version`
> 2. 把当前用户加到docker组里：`sudo usermod -aG docker $USER`，然后注销重新登录一下就好了
>
> 我们打靶一般都是用root用户，所以这个问题可能遇不到，大家知道就行。

### 29.2.2 Windows下安装Docker（简单了解）

Windows下安装Docker也不难，就是有个小要求——你的Windows得是专业版或企业版，因为需要开启Hyper-V（微软的虚拟化技术）。家庭版的话稍微麻烦一点，但也有办法。

因为我们打靶主要在Kali里，所以这里就简单带过，大家知道有这么回事儿就行。

**第一步：下载Docker Desktop**

去Docker官网下载：https://www.docker.com/products/docker-desktop/

选Windows版本下载。

**第二步：安装**

双击安装包，一路下一步。中间可能会问你要不要开启Hyper-V，记得勾上！

**第三步：重启电脑**

安装完需要重启，记得保存好你的东西。

**第四步：启动Docker**

重启后，Docker Desktop会自动启动。等它启动完成，任务栏右下角会出现一个小鲸鱼的图标，就说明可以用了。

**第五步：验证**

打开CMD或PowerShell，输入：

```
docker --version
```

有版本号输出就说明装好了。

> 💡 **家庭版小提示：** 如果你用的是Windows家庭版，也可以装Docker Desktop，只是需要额外开启一些功能，比如WSL 2（Windows Subsystem for Linux）。安装的时候Docker会自动提示你怎么做，跟着提示走就行。

## 29.3 Docker基础命令（新手够用就行）

Docker的命令有很多，但是大家不用怕，不用全背下来，常用的就那么几个，用多了自然就记住了。

今天老师就给大家讲几个最常用的，学会这几个，用Vulhub就完全够用了！

### 29.3.1 镜像相关命令

镜像就是我们说的"安装包"、"模板"，先来看几个和镜像有关的命令。

#### 🔹 docker images —— 查看本地有哪些镜像

```bash
docker images
```

这条命令会列出你本地已经下载了的所有镜像，包括镜像名、版本、ID、大小等等。

就像你打开手机的"已安装应用"列表，看看你都装了哪些APP。

#### 🔹 docker pull —— 拉取（下载）镜像

```bash
docker pull 镜像名
```

比如你想下载一个Ubuntu镜像：

```bash
docker pull ubuntu
```

这条命令会从仓库里把Ubuntu镜像下载到你本地。就像你在应用商店里下载APP。

#### 🔹 docker rmi —— 删除镜像

```bash
docker rmi 镜像名
```

比如你想删掉Ubuntu镜像：

```bash
docker rmi ubuntu
```

`rmi` 是 "remove image" 的缩写，就是删除镜像。就像你卸载手机上的APP。

### 29.3.2 容器相关命令

容器就是镜像运行起来的实例，是"活的"。再来看几个和容器有关的命令。

#### 🔹 docker ps —— 查看正在运行的容器

```bash
docker ps
```

这条命令会列出当前正在运行的所有容器。就像你打开任务管理器，看看哪些程序正在运行。

#### 🔹 docker ps -a —— 查看所有容器（包括停止的）

```bash
docker ps -a
```

`-a` 是 "all" 的意思，显示所有容器，不管是正在运行的，还是已经停止的。

#### 🔹 docker run —— 运行一个容器

```bash
docker run 镜像名
```

比如你想运行一个Ubuntu容器：

```bash
docker run -it ubuntu /bin/bash
```

> 💡 **参数解释：**
> - `-it` 这是两个参数，`-i` 是交互式，`-t` 是分配一个终端。合起来就是让你能和容器交互，就像登录到容器里一样
> - `ubuntu` 是镜像名
> - `/bin/bash` 是容器启动后要执行的命令，也就是启动bash shell

这条命令会创建一个Ubuntu容器，并且进入到容器里面。你就可以在里面敲命令了，就像操作一台新的Linux机器一样。

想退出容器的话，输入 `exit` 回车就行。

#### 🔹 docker start/stop/restart —— 启动/停止/重启容器

```bash
docker start 容器名或容器ID
docker stop 容器名或容器ID
docker restart 容器名或容器ID
```

这三个命令很简单，就是启动、停止、重启容器。就像你启动、关闭、重启一个程序一样。

#### 🔹 docker exec —— 进入正在运行的容器

如果一个容器已经在后台运行了，你想进去看看，怎么办？用 `docker exec`：

```bash
docker exec -it 容器名 /bin/bash
```

比如你的容器名叫 `my-ubuntu`，你想进去：

```bash
docker exec -it my-ubuntu /bin/bash
```

`exec` 是 "execute" 的缩写，就是在容器里执行命令。我们执行的是 `/bin/bash`，所以就进入了容器的终端。

#### 🔹 docker rm —— 删除容器

```bash
docker rm 容器名或容器ID
```

比如你想删掉一个叫 `my-ubuntu` 的容器：

```bash
docker rm my-ubuntu
```

`rm` 是 "remove" 的缩写，就是删除。注意哦，正在运行的容器删不掉，得先停了再删。

### 29.3.3 命令总结小抄

怕大家记不住，老师给大家做了个小抄，贴在这里，忘了就过来看看：

| 命令 | 作用 | 备注 |
|------|------|------|
| `docker --version` | 查看Docker版本 | 验证安装用 |
| `docker images` | 查看本地镜像 | 看看都有啥"安装包" |
| `docker ps` | 查看运行中的容器 | 看看哪些"程序"在跑 |
| `docker ps -a` | 查看所有容器 | 包括停止的 |
| `docker run` | 运行一个新容器 | 从镜像创建容器 |
| `docker start` | 启动一个已停止的容器 | |
| `docker stop` | 停止一个运行中的容器 | |
| `docker restart` | 重启容器 | |
| `docker exec -it 容器名 /bin/bash` | 进入容器 | 就像登录到容器里 |
| `docker rm 容器名` | 删除容器 | 得先停止才能删 |
| `docker rmi 镜像名` | 删除镜像 | 就是删掉"安装包" |

> 💡 **老师的真心话：** 真的不用背！用的时候过来查就行，用多了自然就记住了。老师刚学的时候也是天天查，现在用得多了，想忘都忘不掉。😂

## 29.4 Docker-Compose安装

### 29.4.1 什么是Docker-Compose？

好了，Docker我们搞定了，接下来再认识一个小伙伴——**Docker-Compose**。

大家可能会问："老师，Docker不是已经很厉害了吗？怎么又冒出来一个Compose？"

问得好！那老师先问大家一个问题：如果一个网站，它需要好几个部分配合才能运行——比如要有Web服务器（Nginx）、要有数据库（MySQL）、还要有缓存（Redis）——这三个都是不同的镜像，那怎么办？

难道要一个个去 `docker run` 吗？那也太麻烦了吧！而且它们之间还要通信、还要配置网络，想想就头疼……

别担心！Docker-Compose就是来解决这个问题的！

**一句话：Docker-Compose是用来管理多容器应用的工具！**

打个比方：
- Docker就像是"单个演员"
- Docker-Compose就像是"导演"，把一群演员组织起来，让他们配合着演一出戏

你只需要写一个 `docker-compose.yml` 文件，把你需要的所有容器、它们的配置、它们之间的关系，都写在这个文件里。然后一条命令，就能把所有容器都启动起来，还能自动帮你配置好网络、依赖关系等等。

是不是很方便？😎

### 29.4.2 为什么需要它？

那对于我们用Vulhub来说，为什么需要Docker-Compose呢？

因为很多漏洞环境，它不是一个容器就能搞定的！比如：
- 一个Web漏洞，可能需要Web容器 + 数据库容器
- 一个Redis漏洞，可能需要Redis容器 + 攻击机容器
- 一个复杂的漏洞环境，可能需要三四个甚至更多容器配合

如果没有Docker-Compose，你得一个个去启动这些容器，还要配置它们之间的网络、端口、依赖关系……麻烦死了！

有了Docker-Compose呢？Vulhub已经把 `docker-compose.yml` 文件给你写好了，你只需要进入目录，敲一条 `docker-compose up -d`，所有容器就都帮你启动好了，网络也通了，端口也映射好了，直接就能用！

就是这么爽！👍

### 29.4.3 安装方法

好了，话不多说，我们来安装Docker-Compose！

#### 方法一：下载二进制文件（推荐）

**第一步：下载**

在终端里输入下面这条命令，下载Docker-Compose的二进制文件：

```bash
curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
```

> 💡 **小解释：**
> - `-L` 参数是让curl跟随重定向
> - 后面那个URL是Docker-Compose的下载地址
> - `$(uname -s)` 会自动获取你的系统类型（Linux）
> - `$(uname -m)` 会自动获取你的系统架构（x86_64）
> - `-o /usr/local/bin/docker-compose` 是把下载的文件保存到 `/usr/local/bin/` 目录下，名叫 `docker-compose`

如果GitHub下载慢的话，大家也可以用国内的镜像源，比如：

```bash
curl -L "https://get.daocloud.io/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
```

**第二步：加执行权限**

下载下来的文件默认是没有执行权限的，我们得给它加上：

```bash
chmod +x /usr/local/bin/docker-compose
```

`chmod +x` 就是给文件加上执行权限的意思。

**第三步：验证安装**

好了，我们来验证一下是不是装好了：

```bash
docker-compose --version
```

如果输出了类似这样的内容，就说明安装成功啦！🎉

```
docker-compose version 2.20.0, build xxxxxxxx
```

版本号可能不一样，没关系，有输出就行。

#### 方法二：用pip安装（备选）

如果你装了Python的pip，也可以用pip来安装：

```bash
pip install docker-compose
```

然后同样验证一下：

```bash
docker-compose --version
```

> 💡 **小提示：** 有些系统里可能叫 `docker compose`（中间没有横杠），这是新版本的写法。两种都试试，哪个能用用哪个。

## 29.6 Vulhub使用教程

好啦，说了这么多Vulhub的好话，现在我们来看看它到底怎么用！其实很简单，就五步，跟着老师一步一步来！

### 29.6.1 第一步：下载Vulhub

首先，我们得把Vulhub的代码下载到我们的电脑上。有两种方法：

#### 方法一：git clone（推荐）

如果你装了git的话，直接克隆下来就行，以后更新也方便。

打开终端，找个你想放的目录，然后输入：

```bash
git clone https://github.com/vulhub/vulhub.git
```

这条命令会把Vulhub的整个项目下载到当前目录下，会生成一个叫 `vulhub` 的文件夹。

> 💡 **小提示：** 如果GitHub访问慢的话，也可以用国内的镜像，比如：
> ```bash
> git clone https://gitee.com/pochiachen/vulhub.git
> ```
> 这是Gitee上的镜像，速度可能快一些。

#### 方法二：下载ZIP包

如果你不想装git，也可以直接下载ZIP压缩包。

去GitHub项目主页：https://github.com/vulhub/vulhub

点击那个绿色的 "Code" 按钮，然后选 "Download ZIP"，就能下载了。

下载完之后，解压到你想放的位置就行。

> 💡 **老师建议：** 还是推荐用git的方法，因为以后想更新Vulhub的时候，只要在目录里敲一条 `git pull` 就能更新了，比重新下载ZIP方便多了。

### 29.6.2 第二步：进入对应漏洞目录

下载好之后，你打开vulhub文件夹，会发现里面有好多好多子文件夹，每一个文件夹就是一类漏洞或者一个产品。

比如：
- `thinkphp/` —— ThinkPHP相关的漏洞
- `struts2/` —— Struts2相关的漏洞
- `nginx/` —— Nginx相关的漏洞
- `redis/` —— Redis相关的漏洞
- ……

每个文件夹里面，又有子文件夹，对应具体的某个漏洞。比如 `thinkphp/` 文件夹里可能有：
- `5-rce/` —— ThinkPHP 5.x 远程代码执行漏洞
- `5.0.23-rce/` —— ThinkPHP 5.0.23 远程代码执行漏洞
- `2-rce/` —— ThinkPHP 2.x 远程代码执行漏洞
- ……

每个具体的漏洞目录里，一般都有这几样东西：
- `docker-compose.yml` —— Docker-Compose的配置文件（最重要）
- `README.md` —— 漏洞说明文档，包括漏洞介绍、环境启动方法、利用方法等
- 其他一些配置文件、源码什么的

想玩哪个漏洞，就用 `cd` 命令进入到对应的目录里。

比如我们想玩ThinkPHP 5.x的RCE漏洞，就进入：

```bash
cd vulhub/thinkphp/5-rce/
```

> 💡 **小技巧：** 不知道有哪些漏洞怎么办？两个办法：
> 1. 直接去Vulhub的GitHub主页看，上面有完整的目录列表
> 2. 在vulhub目录里敲 `ls` 命令，看看都有什么文件夹，逐层深入

### 29.6.3 第三步：启动环境

进入到对应目录之后，启动环境就很简单了，一条命令搞定：

```bash
docker-compose up -d
```

> 💡 **参数解释：**
> - `up` 就是启动的意思，会根据 `docker-compose.yml` 文件创建并启动所有需要的容器
> - `-d` 是 "detach" 的意思，就是在后台运行。不加这个的话，终端会被占用，你一按Ctrl+C容器就停了。加上 `-d` 就让它在后台安静地跑，你该干嘛干嘛。

敲完这条命令之后，Docker就会开始干活了：
- 首先会检查本地有没有需要的镜像
- 如果没有，就去仓库下载
- 下载完之后，创建容器
- 配置网络、端口映射什么的
- 最后启动所有容器

这个过程可能需要几分钟，取决于你的网络速度和漏洞环境的复杂程度。大家耐心等等就好 ☕

### 29.6.4 第四步：访问环境

等它启动完成之后，我们怎么访问这个环境呢？

一般来说，环境启动后，会把容器的端口映射到你的主机上，你访问 `http://localhost:端口号` 或者 `http://你的IP:端口号` 就行。

但是具体是哪个端口呢？别急，有两个方法可以查：

#### 方法一：看README.md

每个漏洞目录里都有 `README.md` 文件，里面一般都会写清楚怎么访问、端口是多少、默认账号密码是什么之类的信息。

你可以用 `cat` 命令查看：

```bash
cat README.md
```

或者用你喜欢的编辑器打开看。

#### 方法二：看docker-compose.yml

你也可以直接看 `docker-compose.yml` 文件，里面会写端口映射的配置：

```bash
cat docker-compose.yml
```

找类似这样的内容：

```yaml
ports:
  - "8080:80"
```

冒号前面的是主机的端口，冒号后面的是容器的端口。比如上面这个，就是把容器的80端口映射到主机的8080端口。

所以你访问 `http://localhost:8080` 就行了。

> 💡 **小知识：** 什么是端口映射？
>
> 你可以把容器想象成一个独立的小房子，它有自己的门（端口）。但是这个小房子在一个院子里，外面的人直接看不到。端口映射就像是在院子的围墙上开一个门，这个门对应着小房子的门。你访问围墙的门，就等于访问小房子的门。
>
> 比如 "8080:80"，就是说围墙的8080号门，对应着小房子的80号门。你从8080进去，就到了小房子的80端口。

查好端口之后，打开浏览器，输入地址，就能看到漏洞环境了！接下来，就开始你的表演吧！🎭

### 29.6.5 第五步：打完了，关闭环境

漏洞复现完了，学习完了，这个环境就没用了，怎么办？很简单，关掉就行！

还是在那个漏洞目录里，敲一条命令：

```bash
docker-compose down
```

这条命令会：
- 停止所有容器
- 删除所有容器
- 清理掉相关的网络

干干净净，一点痕迹都不留！就像从来没存在过一样～✨

然后你的硬盘空间也释放出来了，下次想玩再启动就行，几秒钟的事儿。

## 29.8 Vulhub使用注意事项

好啦，Vulhub的基本使用方法大家都学会了。最后，老师再给大家讲几个注意事项，都是新手容易踩的坑，大家记好咯！

### 29.8.1 端口冲突了怎么办？

有时候，你启动环境的时候可能会报错，说端口被占用了。这是怎么回事呢？

很简单，就是你主机上已经有别的程序在用这个端口了。比如8080端口，可能你之前搭的DVWA在用，或者别的什么服务在用。

那怎么办呢？两个办法：

#### 方法一：把占用端口的服务停掉

如果你知道是哪个程序占用了端口，把它停掉就行。

比如你知道是Apache占用了8080端口：

```bash
systemctl stop apache2
```

#### 方法二：修改docker-compose.yml的端口映射

如果那个服务你不想停，那就改Vulhub的端口，换一个没用的端口就行。

比如原来的是 `8080:80`，你可以改成 `8081:80`，或者 `9000:80`，只要前面那个端口没人用就行。

用编辑器打开 `docker-compose.yml`，找到 `ports` 那部分，改一下冒号前面的数字，保存退出，然后重新启动：

```bash
docker-compose up -d
```

就行了。

> 💡 **小技巧：** 怎么查看某个端口被谁占用了？
>
> 用 `netstat` 或者 `ss` 命令：
> ```bash
> netstat -tulnp | grep 8080
> ```
> 或者
> ```bash
> ss -tulnp | grep 8080
> ```
> 就能看到是哪个程序在占用8080端口了。

### 29.8.2 镜像下载慢怎么办？

有时候，Docker下载镜像特别慢，慢得让人抓狂……这是因为Docker的官方仓库在国外，我们访问起来速度慢。

怎么办？配置国内的镜像加速源啊！

国内有很多镜像源，比如阿里云、中科大、网易云等等，配置之后，下载速度能快很多。

怎么配置呢？很简单：

**第一步：创建或修改daemon.json文件**

```bash
mkdir -p /etc/docker
vim /etc/docker/daemon.json
```

**第二步：写入镜像源配置**

在文件里写入：

```json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ]
}
```

这里老师放了两个，一个中科大的，一个网易的，大家也可以加阿里云的，不过阿里云的需要你自己去阿里云注册一个账号，获取专属的加速器地址。

**第三步：重启Docker服务**

```bash
systemctl restart docker
```

好了，配置完成！再下载镜像的时候，速度就快多了！🚀

### 29.8.3 环境启动失败怎么办？

有时候，`docker-compose up -d` 之后，环境并没有正常运行，访问不了，怎么办？

别慌，我们来看看日志，找原因：

```bash
docker-compose logs
```

这条命令会显示所有容器的日志，看看里面有没有报错信息。根据报错信息，一般就能找到问题所在了。

如果只想看某个服务的日志，可以这样：

```bash
docker-compose logs 服务名
```

服务名可以在 `docker-compose.yml` 里找到，一般是 `web`、`db` 之类的。

常见的启动失败原因：
- 端口被占用了（前面讲过）
- 镜像下载不完整（删掉重新pull）
- 配置文件有问题（检查docker-compose.yml）
- 磁盘空间不够了（清理一下空间）

根据日志具体问题具体分析，一般都能解决。

### 29.8.4 重要警告：不要在生产环境用！

最后，老师要严肃地跟大家说一件事：

这些环境就是用来学习、练习、研究漏洞的，它们本身就是不安全的。如果你把它们放到公网上，分分钟就会被黑掉，甚至被用来当肉鸡、挖矿、发动攻击……

所以大家记住：

- ✅ 可以在自己的内网实验环境里用
- ❌ 绝对不要放到公网上
- ❌ 绝对不要在生产环境用

安全第一，学习第二，大家一定要记住哦！

### 本章总结

好啦，今天这一章到这里就差不多了。我们来回顾一下都学了什么：

📦 **Docker是什么？**
- Docker就像集装箱，把应用和依赖打包到一起，一次打包，到处运行
- 镜像（Image）= 模板/安装包
- 容器（Container）= 镜像运行的实例
- 仓库（Registry）= 存放镜像的地方

🛠️ **Docker怎么装？**
- Kali下用一键脚本：`curl -fsSL https://get.docker.com | bash -s docker --mirror Aliyun`
- 启动服务：`systemctl start docker`
- 设置开机自启：`systemctl enable docker`

⌨️ **Docker常用命令**
- `docker images` —— 看镜像
- `docker ps` —— 看运行中的容器
- `docker ps -a` —— 看所有容器
- `docker run` —— 运行容器
- `docker start/stop/restart` —— 启动/停止/重启容器
- `docker exec -it` —— 进入容器
- `docker rm` —— 删除容器
- `docker rmi` —— 删除镜像

🎼 **Docker-Compose是什么？**
- 用来管理多容器应用的工具
- 一个yml文件定义所有容器，一条命令全部启动
- 安装：下载二进制文件，加执行权限

🎯 **Vulhub是什么？**
- 基于Docker的漏洞环境集合
- 几百个漏洞，应有尽有
- 部署简单，环境标准，更新快速

📖 **Vulhub怎么用？**
1. 下载Vulhub：`git clone https://github.com/vulhub/vulhub.git`
2. 进入对应目录：`cd vulhub/xxx/xxx/`
3. 启动环境：`docker-compose up -d`
4. 访问环境：浏览器访问对应端口
5. 关闭环境：`docker-compose down`
