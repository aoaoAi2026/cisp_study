## 26.2 靶机搭建

好了，介绍完了，咱们开始动手搭环境！

### 26.2.1 下载靶机

首先，我们需要下载Jangow 1.0.1的靶机文件。

**官方下载地址（VulnHub官网）：
```
https://www.vulnhub.com/entry/jangow-101,754/
```

> 💡 **小提示：**
>
> 如果VulnHub官网访问慢、打不开，也没关系，可以去网上搜"Jangow 1.0.1 下载"，找一个靠谱的网盘资源也行。
>
> 下载的文件应该是一个.ova格式的文件，大小大概几百MB的样子。

.ova文件是什么呢？简单说就是虚拟机的"打包文件"，就像你把一个文件夹压缩成了一个.zip文件，.ova就是把整个虚拟机（包括硬盘、配置什么的都打包成一个文件了，导入就能用。

### 26.2.2 导入VMware

下载完.ova文件之后，我们需要把它导入到VMware里。

导入步骤很简单，跟着我来：

1. 打开VMware Workstation
2. 点击左上角的 **"文件"** → **"打开"**
3. 找到你下载的Jangow的.ova文件，选中它
4. 点击 **"打开"**
5. 然后会弹出一个导入的窗口，让你给虚拟机起个名字，选个存放位置
6. 名字就默认的就行，位置选一个你找得到的地方
7. 点击 **"导入"**
8. 等它导入完成，进度条走完

就这么简单！导入完成后，你就能在VMware的左侧列表里看到Jangow这个虚拟机了。

> ⚠️ **注意：**
>
> 导入的时候可能会弹出一个"导入失败"或者"格式不兼容"的提示，别慌！这是正常现象。
>
> 如果遇到这种情况，你可以试试：
> - 点"重试"或者"忽略"
> - 或者用VirtualBox导入试试
> - 一般来说都能成功导入的，不用担心

### 26.2.3 网络设置为NAT模式

导入完之后，我们需要设置一下网络模式。

什么是网络模式呢？简单说就是虚拟机怎么上网、怎么和你的电脑通信。VMware有好几种网络模式，我们这里用 **NAT模式** 就可以了。

为什么用NAT模式呢？因为：
- NAT模式下，虚拟机和你的真机在同一个虚拟网络里，可以互相访问
- 虚拟机可以通过你的电脑上网
- 比较安全，不会把靶机暴露到真实网络里

设置方法：

1. 在VMware左侧，右键点击Jangow虚拟机
2. 点击 **"设置"**
3. 在弹出的窗口里，找到 **"网络适配器"**
4. 右边选择 **"NAT模式"**
5. 点击 **"确定"**

搞定！网络模式就设置好了。

> 🤔 **小疑问：什么是NAT模式？**
>
> 用大白话讲，NAT模式就像你家的路由器。你的真机是路由器，虚拟机是你家的手机、电脑。手机电脑通过路由器上网，它们之间可以互相访问，但外面的人直接访问不到你家的手机。
>
> 这样既保证了安全性，又能让虚拟机和真机互相通信。

### 26.2.4 启动靶机

设置好了，万事俱备，只欠启动！

点击VMware里的 **"开启此虚拟机"** 按钮，启动Jangow靶机。

启动过程中，你会看到一堆黑色的屏幕，有很多文字在滚，那是Linux系统在启动。等它滚完，就启动成功了。

启动完成后，你会看到一个登录界面，让你输入用户名和密码。别慌！我们不需要登录进去——我们是从外部"打"进去的，不是从控制台登录。这就像你去别人家偷东西，你不会走正门敲门进去，而是从窗户、从门缝想办法进去。😂

所以，启动完之后，你就让它在那儿开着就行了，不用管它。我们的操作都是在Kali Linux里进行的。

### 26.2.5 找靶机IP

靶机启动了，但是我们怎么知道它的IP地址是多少呢？

就像你要去别人家，你得知道人家的门牌号吧？IP地址就是靶机在网络里的"门牌号"。

找靶机IP的方法有好几种，我给大家介绍两种最常用的。

**方法一：用arp-scan扫描**

arp-scan是Kali里自带的一个小工具，可以扫描同一个局域网里有哪些设备。

使用方法：

1. 打开你的Kali Linux（确保Kali也是NAT模式哦！）
2. 打开终端
3. 输入命令：
```bash
arp-scan -l
```
4. 回车执行

> 💡 小贴士：
> 如果提示没有arp-scan命令的话，可以用apt install arp-scan安装一下。
> 或者你也可以用sudo来执行：sudo arp-scan -l

执行完之后，你会看到一个列表，列出了同一个局域网里所有设备的IP地址和MAC地址。

那怎么确定哪个是靶机呢？

一般来说，列表里会有这几个：
- 一个是你的真机（网关），通常是.1或者.2结尾的
- 一个是你的Kali Linux自己
- 剩下的那个，就是靶机了！

你可以根据MAC地址的厂商信息来判断，比如VMware的虚拟机MAC地址一般是00:0c:29或者00:50:56开头的。

**方法二：用netdiscover扫描**

netdiscover也是一个常用的扫描工具。

使用方法：
```bash
netdiscover -r 192.168.x.0/24
```

这里的192.168.x.0/24要换成你自己的网段。怎么看自己的网段呢？你可以在Kali里输入ip a或者ifconfig看看自己的IP，比如你的IP是192.168.12.100，那网段就是192.168.12.0/24。

netdiscover会自动扫描，扫到的结果和arp-scan差不多。

> 🤔 **生活小例子：**
>
> 找靶机IP就像你去一个小区找朋友家。你不知道他住几单元几号，怎么办呢？你就挨家挨户敲门问："请问张三住这儿吗？"
>
> arp-scan和netdiscover就是干这个的——它们把整个网段的IP都"问"一遍，看看哪些是"活的"，然后给你列出来。

### 26.2.6 验证：ping一下

找到靶机IP之后，我们来验证一下，看看能不能通。

怎么验证呢？用ping命令！

在Kali的终端里输入：
```bash
ping 靶机IP
```

比如你找到的靶机IP是192.168.12.10，你就输入ping 192.168.12.10。

如果能通的话，你会看到类似这样的回复：
```
64 bytes from 192.168.12.10: icmp_seq=1 ttl=64 time=0.5ms
64 bytes from 192.168.12.10: icmp_seq=2 ttl=64 time=0.3ms
...
```

按Ctrl+C可以停止ping。

如果ping不通的话，那你得检查一下：
- 靶机是不是启动了？
- 靶机和Kali是不是都是NAT模式？
- IP地址有没有找错？

一般来说，只要设置对了，都能ping通的。

> 🤔 **ping是什么？**
>
> ping就像你给别人打电话，打通了，对方接了，说"喂我在"，那就说明能联系上。如果打不通，那就是联系不上。

### 26.3.2 第一步：端口扫描（Nmap）

信息收集的第一步，就是**端口扫描**。

什么是端口呢？用大白话讲，你可以把一台服务器想象成一栋大楼，IP地址是大楼的地址，那端口就是大楼的门。一栋大楼可以有很多门——正门、后门、侧门、车库门、消防通道……每个门通往不同的地方。

一台服务器也可以有很多端口，一共65536个端口（0到65535）。每个端口对应一个服务，比如：
- 21端口：FTP服务（文件传输）
- 22端口：SSH服务（远程登录）
- 80端口：HTTP服务（网站）
- 443端口：HTTPS服务（加密的网站）
- 3306端口：MySQL服务（数据库）
- 3389端口：远程桌面（Windows的远程登录）

端口扫描，就是看看这栋大楼有哪些门是开着的，每个门后面是什么服务，这个服务是什么版本的。

**端口扫描神器：Nmap**

说到端口扫描，就不得不提**Nmap**——端口扫描界的"神器"，几乎所有搞安全的人都用它。Nmap的功能非常强大，能扫端口、能扫服务版本、能扫操作系统、还能写脚本扫描漏洞……

我们今天先介绍几个最常用的参数。

**常用命令：**

```bash
nmap -sV -sC -p- 靶机IP
```

我来解释一下各个参数是什么意思：
- `-sV`：探测服务版本（看看每个端口跑的是什么服务，版本号是多少）
- `-sC`：用默认脚本扫描（用一些常用的脚本去扫，看看有没有什么明显的漏洞或者信息）
- `-p-`：扫描所有端口（从1扫到65535，一个都不放过）

> ⚠️ 注意：
> `-p-`扫全端口的话，速度会比较慢，可能需要几分钟甚至更久。如果着急的话，可以先不扫全端口，先扫常用端口。

**快速扫描（只扫常用端口）：
```bash
nmap -sV 靶机IP
```

这个就快多了，一般几秒钟就能出结果。

> 🤔 **生活小例子：**
>
> 端口扫描就像你到了一栋大楼楼下，你围着大楼转一圈，看看有几个门、每个门是干什么的、门上写着什么字。
>
> - `-sV` 就是看看每个门上挂的牌子，写的是"财务部"还是"人力资源部"
> - `-sC` 就是你凑过去听听每个房间里在干什么，有没有什么有用的信息
> - `-p-` 就是你把整栋楼65536个门都检查一遍，一个都不漏

**演示：扫描Jangow**

好了，理论讲完了，咱们来实际扫一下Jangow看看。

在Kali终端里输入：
```bash
nmap -sV -sC 靶机IP
```

（先用快速扫描试试，如果没扫到什么东西，再用全端口扫。）

假设我们扫出来的结果大概是这样的：

```
PORT   STATE SERVICE VERSION
21/tcp open  ftp     vsftpd 3.0.3
80/tcp open  http    Apache httpd 2.4.18 ((Ubuntu))
```

哇！开了两个端口：
- **21端口**：FTP服务，版本是vsftpd 3.0.3
- **80端口**：HTTP服务，Apache 2.4.18，系统是Ubuntu

这信息量就很大了！我们知道了：
1. 靶机开了FTP和HTTP两个服务
2. Web服务器是Apache，版本2.4.18
3. 操作系统是Ubuntu

有了这些信息，我们就可以继续往下走了。

> 💡 **小技巧：**
>
> 如果快速扫描没扫到东西，别急着放弃，试试全端口扫描。有时候管理员可能把服务开在不常见的端口上，比如把网站开在8080、8000、31337这些端口上。全端口扫一遍，更保险。

### 26.3.3 第二步：Web信息收集

扫到80端口开着，有HTTP服务，那我们接下来就去看看这个网站长啥样。

打开你的浏览器，在地址栏输入：
```
http://靶机IP
```

回车，看看网站首页是什么样子。

这一步很重要，就像你到了一个陌生的地方，先四处看看，了解一下环境。

**看什么呢？**

1. **网站首页**
   - 网站是做什么的？
   - 有什么栏目？
   - 有什么功能？
   - 有没有登录框？
   - 有没有搜索框？
   - 有没有可以上传文件的地方？

2. **四处点点
   - 首页上有什么链接？都点进去看看
   - 看看有哪些页面
   - 看看URL长什么样
   - 是静态页面还是动态页面？（URL里有没有.php、.asp、.aspx这些）

3. **查看网页源码**
   - 按Ctrl+U查看网页源代码
   - 看看注释里有没有什么信息
   - 看看有没有什么隐藏的链接
   - 看看用了什么前端框架
   - 看看CSS和JS文件里有没有线索

> 🤔 **为什么要看源码？**
>
> 就像你买东西，不能只看外包装，得看看里面什么样。网页也是一样，表面上看起来干干净净，源码里可能藏着各种信息——开发者的注释、隐藏的目录、密码提示什么的。

举个例子：有的开发者会在源码里写注释，比如"<!-- TODO: 后台地址是/admin，密码是123456 -->"——这不就送分题了吗？😂

虽然现实中这种情况很少，但也不是完全没有。万一呢？万一就被你碰到了呢？

**生活小例子：找东西 🕵️
>
> 你到一个陌生的房间找东西，你会怎么做？
> - 先环顾四周，看看房间布局
> - 然后拉开抽屉看看
> - 翻开书看看里面有没有夹着什么
> - 甚至摸摸床垫下面有没有
>
> Web信息收集也是一样，你要到处看、到处找，不放过任何一个角落。

### 26.3.4 第三步：目录扫描

看完了首页，点了点链接，我们大概知道了网站有哪些页面。但是——**网站可能还有一些页面是没有链接出来的，是"隐藏"的，我们怎么找到它们呢？

这就需要**目录扫描**了。

什么是目录扫描？简单说就是：用工具，把常见的目录名、文件名一个一个去试，看看哪些是存在的。

比如我们试一下：
- /admin 存不存在？
- /login 存不存在？
- /backup 存不存在？
- /phpinfo.php 存不存在？
- /robots.txt 存不存在？

试出来存在的，就是网站有的。

目录扫描工具也有很多，比如：
- dirb
- dirsearch
- gobuster
- wfuzz

这些工具都差不多，选一个你用着顺手的就行。我们这里用**dirb**来演示，因为Kali里默认就带了，不用额外装。

**dirb使用方法：**

```bash
dirb http://靶机IP/
```

就这么简单！dirb会用它自带的字典，去扫网站有哪些目录和文件。

扫出来的结果大概是这样的：

```
---- Scanning URL: http://192.168.12.10/
---- 
+ http://192.168.12.10/index.html (CODE:200|SIZE:1234)
+ http://192.168.12.10/robots.txt (CODE:200|SIZE:567)
+ http://192.168.12.10/site/ (CODE:301|SIZE:0)
+ http://192.168.12.10/wordpress/ (CODE:301|SIZE:0)
+ http://192.168.12.10/admin/ (CODE:403|SIZE:234)
...
```

扫出来之后，每个目录、每个文件，我们都要去看看里面有啥。

比如：
- /site/ 里面是什么？
- /wordpress/ 里面是WordPress吗？
- /admin/ 是后台吗？为什么是403？（403是禁止访问的意思）

> 🤔 **目录扫描就像什么？**
>
> 目录扫描就像你到了一个小区，你想知道这栋楼里有哪些房间。你怎么知道呢？你就挨家挨户敲门，敲敲门，有人答应，说明这屋有人；没人答应，说明这屋可能没人。
>
> dirb这些工具就是干这个的——它们拿着一本"常见房间号列表"，一个一个去敲，敲开的给你记下来。

**每个目录都进去看看，别嫌麻烦！信息收集就是要细，要全。

### 26.3.5 第四步：CMS识别

扫着扫着，你可能会发现——哎，这个网站好像用的是某个现成的CMS（内容管理系统），比如WordPress、Drupal、Joomla什么的。

什么是CMS呢？CMS就是Content Management System，内容管理系统。说白话就是：别人已经写好的网站程序，你下载下来，装上，就能建网站了，不用自己从零开始写代码。

就像你想开个网店，你不用自己写一个购物网站，你用淘宝、用京东、用拼多多……不对，应该是用Shopify、用有赞这些现成的电商系统。😂

那识别CMS有什么用呢？用处可大了！

如果知道了网站用的是什么CMS，什么版本，那我们就可以去搜这个CMS这个版本有没有已知的漏洞。如果有，直接拿来用就行，不用自己辛辛苦苦找漏洞了。

这就叫——**站在巨人的肩膀上**。

**CMS识别工具有哪些呢？**

1. **Wappalyzer**
   > 这是一个浏览器插件，装在Chrome或者Firefox里，打开网站它自动识别，告诉你这个网站用了什么技术栈。非常方便，推荐安装一个。

2. **whatweb**
   > Kali里自带的命令行工具，也能识别CMS。
   > 使用方法：whatweb http://靶机IP

3. **cmseek**
   > 专门用来扫描CMS漏洞的工具，能识别CMS，还能扫一些常见漏洞。

**演示一下whatweb：
```bash
whatweb http://靶机IP
```

输出大概是这样：
```
http://192.168.12.10 [200 OK] Apache[2.4.18], Country[RESERVED][ZZ], HTML5, HTTPServer[Ubuntu Linux][Apache/2.4.18 (Ubuntu)], IP[192.168.12.10], MetaGenerator[WordPress 5.6], WordPress[5.6]
```

哦！识别出来了，是WordPress，版本5.6！

那接下来我们就可以去搜WordPress 5.6有什么漏洞了。

> 💡 **小知识：**
>
> 为什么CMS漏洞利用这么重要？因为现实中，大部分网站都是用CMS建的，自己从零开始写的很少。所以CMS漏洞利用是非常重要的一个攻击手段。
>
> 常见的CMS有：
> - WordPress：博客用的最多
> - Drupal：也挺多的
> - Joomla：也不少
> - ThinkPHP：国内挺多的
> - Dedecms：织梦，国内老站长用的多
> - Discuz：论坛程序

### 26.3.6 第五步：其他信息收集

除了上面说的这些，还有一些地方也要看看，说不定就能找到有用的信息。

**1. robots.txt**

robots.txt是网站给搜索引擎爬虫写的文件，告诉爬虫哪些页面可以爬，哪些不可以爬。

但是——**不能爬的地方，往往就是有东西的地方！** 😏

怎么看呢？直接访问：
```
http://靶机IP/robots.txt
```

看看里面写了什么，有没有Disallow的目录，那些目录都去看看。

**2. phpinfo.php**

phpinfo.php是PHP的信息页面，里面会显示PHP的版本、配置、扩展什么的，信息非常全。

有的网站会不小心把phpinfo.php留在网站上，如果能找到的话，那信息量就大了。

怎么找呢？目录扫描的时候一般会扫到，或者你手动试试：
```
http://靶机IP/phpinfo.php
```

**3. 备份文件、配置文件**

有的开发者会把备份文件放在网站目录下，比如：
- backup.zip
- www.zip
- web.zip
- .bak文件
- config.php.bak

这些如果能找到，那可能直接就拿到源码了，甚至拿到数据库密码了。

**4. 错误页面**

故意访问一个不存在的页面，看看404页面是什么样的。404页面有时候会泄露一些信息，比如网站路径、Web服务器版本什么的。

**5. 网站底部**

看看网站最底部，一般会有"Powered by XXX"，也能看出用的什么程序。

> 🤔 **生活小例子：信息收集就像侦探破案 🔍**
>
> 你是一个侦探，要破一个案子。你怎么破？
> - 你去现场勘察，找线索
> - 你去问目击者，找线索
> - 你去查受害者的社会关系，找线索
> - 你去调监控，找线索
>
> 线索收集得越多，案子越容易破。
>
> 渗透测试也是一样，你就是侦探，靶机就是案子，你要到处找线索，线索越多，越容易找到突破口。

## 26.4 梳理信息，制定攻击思路

信息收集做得差不多了，我们来整理一下收集到的信息，然后制定一下攻击思路。

### 26.4.1 整理收集到的信息

假设我们收集到了这些信息：

**端口和服务：**
- 21端口：FTP，vsftpd 3.0.3
- 80端口：HTTP，Apache 2.4.18，Ubuntu系统

**Web相关：**
- 网站首页是什么样的
- 有哪些页面
- 用了什么CMS（比如WordPress）
- 有哪些目录（/site、/admin、/wordpress等）

**其他：**
- robots.txt里有什么
- 有没有phpinfo
- 有没有备份文件

把这些信息都列出来，摆在桌面上，然后分析。

> 💡 **小建议：**
>
> 新手的话，可以找个本子记下来，或者建个txt文件写下来。好记性不如烂笔头嘛。信息多了你记不住的。

### 26.4.2 可能的攻击面

整理完信息，我们来看看有哪些可能的攻击面：

**1. Web漏洞**
- SQL注入
- 文件上传
- 命令执行
- 文件包含
- XSS
- CSRF
- ...

Web的攻击面是最大的，因为Web最容易出问题。80端口一般是首选突破口。

**2. FTP弱口令**
- 21端口开了FTP，我们可以试试能不能爆破FTP的用户名和密码
- 比如anonymous匿名登录能不能用
- 弱口令比如admin/admin、root/root、test/test这些

**3. CMS已知漏洞**
- 如果识别出了CMS和版本，去搜一下这个版本有没有已知漏洞
- 比如WordPress某个插件有漏洞
- 直接利用现成的漏洞

**4. 其他端口**
- 如果扫到其他端口，比如22端口SSH，也可以试试爆破
- 比如3306端口MySQL，也可以试试

### 26.4.3 先从哪里入手？

攻击面这么多，我们先从哪里入手呢？

**一般来说，先从Web入手**，因为：
1. Web的攻击面最大，漏洞最多
2. Web最容易出问题
3. 80端口一般是开着的

就像你去偷东西，你肯定先从最容易进的地方进，对吧？窗户没关你肯定翻窗户，门没锁你肯定直接推门。

那Web从哪里开始呢？
1. 先看看网站有什么功能，每个功能都试试
2. 看看有没有输入点（搜索框、留言板、上传点）
3. 看看有没有登录框，试试SQL注入
4. 看看CMS有没有已知漏洞
5. 看看后台能不能爆破

> 🤔 **生活小例子：找开锁 🔓
>
> 你忘带钥匙了，要进家门，你会怎么做？
> - 先试试门有没有锁——没锁直接进
> - 再试试窗户有没有关——没关翻进去
> - 再看看有没有备用钥匙——找到就用
> - 实在不行再叫开锁公司
>
> 渗透测试也是一样，从最简单、最有可能的地方先试，由易到难，一个一个来。
