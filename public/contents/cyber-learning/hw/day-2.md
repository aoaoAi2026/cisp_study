## 一、为什么要学法律法规？

### 1.1 用大白话说

你可能觉得法律很枯燥，但你想想：
- 你把别人的系统攻破了 → **违法**
- 你公司不保护用户数据 → **违法**
- 你网站被攻击了不报告 → **违法**

**法律不是束缚你的枷锁，而是保护你的盾牌。** 护网时你做的每一个动作，都要有法律依据。

### 1.2 网络安全"三驾马车"法律体系

```
┌─────────────────────────────────────────────┐
│              网络安全法（母法）                │
│             2017年6月1日施行                   │

│  数据安全法      │   个人信息保护法            │
│  2021年9月1日   │   2021年11月1日            │
├─────────────────┴───────────────────────────┤
│  配套法规：等保2.0、CII保护条例、密码法...     │
```

## 二、《网络安全法》核心要点

### 2.1 网络安全法的四大支柱

| 支柱 | 说人话 |
|:---|:---|
| **网络安全等级保护（等保）** | 你的系统分几级，就要做到相应的安全标准 |
| **关键信息基础设施保护（CII）** | 银行、电力、交通这些"命脉"系统要重点保护 |
| **个人信息保护** | 用户的姓名、手机号、身份证号不是你想拿就拿的 |
| **监测预警与信息通报** | 发现安全事件必须上报，不能藏着掖着 |

### 2.2 你必须记住的关键条款

| 条款 | 内容 | 一句话 |
|:---|:---|:---|
| 第21条 | 实行网络安全等级保护制度 | 必须做等保 |
| 第25条 | 制定应急预案，及时处置安全事件 | 必须有应急响应能力 |
| 第31条 | 关键信息基础设施重点保护 | CII系统是重点中的重点 |
| 第42条 | 网络运营者不得泄露、篡改、毁损个人信息 | 用户数据不是你的 |

> **⚠️ 第21条的硬性要求**：网络日志留存不少于**6个月**！这是护网中溯源分析的法律基础。

## 三、等级保护2.0（等保2.0）

### 3.1 等保是啥？一个比喻

想象你家：
- 一级等保 = 普通公寓门锁（有就行）
- 二级等保 = 加装防盗门和摄像头（基本安全）
- 三级等保 = 门禁系统+监控+保安24小时巡逻（加强安全）
- 四级等保 = 军区级别安全防护（强制防护）
- 五级等保 = 国家最高级别防护

### 3.2 五个等级速查表

| 等级 | 保护强度 | 测评周期 | 典型系统 |
|:---|:---|:---|:---|
| 一级 | 自主保护 | 不需要 | 小型企业内部系统 |
| 二级 | 指导保护 | 每两年一次 | 一般企业业务系统 |
| 三级 | 监督保护 | **每年一次** | 银行、政府门户、电商平台 |
| 四级 | 强制保护 | 每半年一次 | 国家核心系统 |
| 五级 | 专控保护 | 按要求 | 最高级别国家机密系统 |

> **💡 护网单位绝大多数是三级等保！** 记住：三级等保每年测评一次。

### 3.3 等保2.0与1.0的区别

| 对比项 | 等保1.0 | 等保2.0 |
|:---|:---|:---|
| 覆盖范围 | 传统信息系统 | 新增：云计算、物联网、移动互联、工控、大数据 |
| 安全要求 | 10个层面 | 扩展为技术+管理两大类 |
| 理念 | 被动防护 | 主动防御、动态安全 |

### 3.4 等保三级要做哪些事？

**技术层面（五个方面）**：
1. **物理安全**：机房门禁、温湿度控制、UPS不间断电源
2. **网络安全**：防火墙、入侵检测/防御（IDS/IPS）、网络分段
3. **主机安全**：操作系统加固、防病毒、堡垒机
4. **应用安全**：WAF、代码审计、漏洞扫描
5. **数据安全**：加密传输、数据库审计、备份恢复

**管理层面（五个方面）**：
1. 安全管理制度
2. 安全管理机构
3. 人员安全管理
4. 系统建设管理
5. 系统运维管理

## 四、关键信息基础设施保护

### 4.1 CII是什么？

**关键信息基础设施（Critical Information Infrastructure）**，说白了就是国家的"数字命脉"——出了事会严重影响国家安全、国计民生、公共利益的系统。

### 4.2 CII覆盖的行业

| 行业 | 举例 |
|:---|:---|
| ⚡ 能源 | 国家电网、石油管道控制系统 |
| 🚄 交通 | 铁路调度系统、民航管制系统 |
| 💧 水利 | 大坝监控系统 |
| 💰 金融 | 核心交易系统、清算系统 |
| 🏛️ 公共服务 | 社保系统、公积金系统 |
| 🏢 电子政务 | 政府门户网站、政务云平台 |
| 🏭 国防科工 | 军工生产系统 |
| 📡 电信 | 骨干网络、DNS系统 |

### 4.3 CII运营者的特殊义务

- 🔒 安全保护措施要"**同步规划、同步建设、同步使用**"（三同步原则）
- 📋 每年至少进行一次安全检测和风险评估
- 🚨 发生重大安全事件要立即报告
- 👤 核心岗位人员要进行安全背景审查

## 五、《数据安全法》

### 5.1 数据安全法核心制度

| 制度 | 说人话 |
|:---|:---|
| **数据分类分级** | 不是所有数据都一样重要，先分等级 |
| **数据安全审查** | 处理重要数据要过审查关 |
| **数据出境安全评估** | 数据传到国外要评估合不合适 |
| **数据安全风险评估报告** | 定期自查并向主管部门报告 |

### 5.2 数据分三级

| 级别 | 说明 | 举例 |
|:---|:---|:---|
| 一般数据 | 不涉及敏感信息 | 已公开的公司简介 |
| 重要数据 | 一旦泄露可能危害国家安全/公共利益 | 关键基础设施运行数据、基因数据 |
| 核心数据 | 一旦泄露将严重危害国家安全 | 国家最高机密数据 |

> **💡 护网重点**：重要数据和核心数据是攻击者的主要目标。

## 六、《个人信息保护法》

### 6.1 核心原则：告知-同意

处理个人信息前，必须：
1. **明确告诉用户**：你是谁、要收集什么信息、用来干什么
2. **获得用户同意**：不能默认打勾，必须用户主动确认

### 6.2 个人信息保护的关键规则

| 规则 | 解释 |
|:---|:---|
| **目的限制** | 收集信息时必须说明用途，不能"顺便"用 |
| **最小必要** | 只收集业务必需的信息，不能多拿 |
| **公开透明** | 处理规则要公开，不能说一套做一套 |
| **安全保障** | 必须采取技术措施保护信息安全 |

### 6.3 违反个保法的代价

- 💰 罚款上限：**5000万元** 或 **上一年度营业额5%**（哪个高罚哪个）
- 🚫 责令暂停相关业务、停业整顿
- 📜 直接负责人禁业限制

```
          ┌──────────────┐
          │  网络安全法   │  ← 母法，基础框架
          │  2017.6.1    │
          └──────┬───────┘
        ┌────────┼────────┐
        ▼                 ▼
┌──────────────┐  ┌──────────────┐
│  数据安全法   │  │ 个人信息保护法 │
│  2021.9.1    │  │  2021.11.1   │
│              │  │              │
│ 管"数据"怎么  │  │ 管"个人信息"  │
│ 分级分类保护  │  │ 怎么收集使用  │
└──────────────┘  └──────────────┘
```

**简单记忆**：
- 《网络安全法》→ 管网络的
- 《数据安全法》→ 管数据的
- 《个人信息保护法》→ 管个人信息的

三者叠加 = 网络空间安全法律体系的"铁三角"。

## 八、合规自查怎么做？

### 8.1 等保合规检查清单生成

```python
# 等保合规检查清单自动生成
class ComplianceChecklist:
    def __init__(self, level):
        self.level = level
        self.items = []
    
    def add_check(self, category, item, required=True):
        self.items.append({
            'category': category,
            'item': item,
            'required': required,
            'status': '待检查'
        })
    
    def generate(self):
        print(f'=== 等保{self.level}级合规检查清单 ===')
        total = 0
        checked = 0
        for cat in ['物理安全', '网络安全', '主机安全', '应用安全', '数据安全']:
            print(f'\n📋 【{cat}】')
            cat_items = [i for i in self.items if i['category'] == cat]
            for item in cat_items:
                tag = "🔴 强制" if item['required'] else "🟡 建议"
                print(f'  □ {item["item"]} ({tag})')
            print(f'  → 共 {len(cat_items)} 项')
        if not self.items:
            print('  (请添加检查项)')

# 创建一个三级等保检查清单
cl = ComplianceChecklist('三')
cl.add_check('网络安全', '网络边界防护设备部署（防火墙）')
cl.add_check('网络安全', '入侵检测/防御系统（IDS/IPS）')
cl.add_check('网络安全', '网络区域隔离（VLAN划分）')
cl.add_check('主机安全', '操作系统安全加固')
cl.add_check('主机安全', '防病毒软件安装并更新')
cl.add_check('主机安全', '堡垒机部署（运维审计）')
cl.add_check('应用安全', 'Web应用防火墙（WAF）')
cl.add_check('应用安全', '应用安全漏洞扫描')
cl.add_check('数据安全', '数据加密传输（HTTPS/TLS）')
cl.add_check('数据安全', '数据库审计日志')
cl.add_check('数据安全', '数据备份和恢复机制')
cl.generate()
```

## 九、护网中涉及的法律问题

### 9.1 红队的法律边界

红队在演习中的活动必须在**授权范围**内：
- ✅ 授权的目标IP、域名范围内可以攻击
- ❌ 不能攻击未授权目标
- ❌ 不能造成实际业务中断
- ❌ 不能窃取真实数据
- ❌ 演习结束后必须清理所有后门和痕迹

### 9.2 蓝队的法律义务

- 📝 发现攻击要记录和报告
- 🔒 保护用户数据不泄露
- 📊 日志要保留不少于6个月
- 🚨 重大安全事件要上报

## 十、实战演练与能力检验

### 10.1 护网与等保的关系

等保是"日常建设"，护网是"实战考试"：
- 日常等保做得好 → 护网表现就不会差
- 护网暴露的问题 → 反向推动等保完善

**两者是良性互动的关系，不是两张皮。**
