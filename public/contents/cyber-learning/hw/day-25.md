## 📖 核心知识讲解

### 一、护网三阶段概览

护网行动不是从演习当天开始的。完整的工作分为三个阶段：

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 护网前   │ → │ 护网中   │ → │ 护网后   │
│ 准备阶段  │    │ 实战阶段  │    │ 复盘阶段  │
│ 1-2个月   │    │ 2-4周    │    │ 1-2周    │
└──────────┘    └──────────┘    └──────────┘
```

### 二、护网前 —— 不打无准备之仗

#### 1. 资产梳理（最重要的准备工作）

**为什么重要？** 你都不知道自己有哪些服务器、开了哪些端口，怎么防守？

```
资产梳理清单：
□ 外网资产
  ├── 域名列表（包括子公司、测试环境）
  ├── 公网IP（每个IP上的服务是什么）
  ├── 云端资产（云服务器、OSS、CDN）
  └── 第三方系统（外包、SaaS平台）

□ 内网资产
  ├── 服务器清单（IP、用途、负责人、重要等级）
  ├── 网络设备（交换机、路由器、防火墙）
  ├── 安全设备（IDS/IPS/WAF/日志审计）
  └── 数据库（MySQL/Redis/Oracle/MongoDB）

□ 人员资产
  ├── 账号清单（管理员账号、服务账号）
  ├── 权限梳理（谁有什么权限）
  └── 离职人员账号是否清理
```

**常见问题（护网前就被蓝队发现的问题）：**
- "这台服务器什么时候上线的？没人知道！" → 幽灵资产
- "这个端口怎么开着？早就不用了" → 未清理的测试环境
- "这个管理员账号是谁的？" → 离职未清理

#### 2. 风险排查与加固

```
风险排查重点：
□ 弱口令检查（admin/admin123/password123等）
□ 端口开放检查（不必要的端口关闭）
□ 漏洞扫描（高危漏洞在护网前修复）
□ 基线检查（操作系统/数据库/中间件安全配置）
□ 访问控制（不该访问的能否访问到）
□ 日志审计（确保所有设备的日志都在正常产生和接收）
```

#### 3. 方案和人员准备

```
护网前必须就绪的文档：
□ 护网值守方案（谁在哪值班、什么时间）
□ 应急响应预案（出事怎么处理）
□ 通讯录（所有人联系方式、TL负责人）
□ 日报模板
□ 情报来源（从哪里获取最新威胁信息）
```

### 三、护网中 —— 实战阶段的日常

#### 1. 值班安排

```
典型的值班排班：
  白班：08:00 - 20:00（主班，人员最多）
  夜班：20:00 - 08:00（夜班，人员减少但必须有）
  
  每班配置：
  - 值班组长 1人（决策、协调）
  - 监控研判 2-3人（看告警、做研判）
  - 应急响应 1-2人（出现事件时处理）
  - 日志审计 1-2人（深度日志分析）
```

#### 2. 情报联动

护网期间要实时关注外部情报：

```
情报来源：
□ 护网指挥部下发的通报（重点IP/域名/漏洞）
□ 行业共享群（同行发现的新攻击手法）
□ 威胁情报平台实时监控
□ 监管机构通报

情报联动流程：
收到情报 → 验证 → 确认影响范围 → 添加防护规则 → 通报全组
```

#### 3. 护网值守日报编写

日报是护网期间最重要的文档之一，管理层通过日报了解当前态势。

**日报模板：**

```markdown
# 护网值守日报
**日期**：2026年6月12日
**班次**：白班（08:00-20:00）
**值班人员**：张三（组长）、李四、王五

## 一、整体态势
- 今日告警总数：326条
- 已处置：320条（处置率 98.2%）
- 未处置：6条（均为P3/P4低危）
- 当前整体态势：🟢 平稳可控

## 二、攻击统计
| 攻击类型 | 数量 | 主要来源 |
|:---|:---|:---|
| 端口扫描 | 156 | 境外扫描平台 |
| 目录扫描 | 89 | 多个IP |
| SQL注入 | 32 | 已被WAF拦截 |
| 暴力破解 | 28 | 集中在SSH |
| Webshell通信 | 5 | 需重点排查 |
| 其他 | 16 | - |

## 三、重点事件
### 事件1：Webshell告警
- 时间：14:30
- 来源：IDS设备
- 目标：Web服务器 192.168.1.100
- 研判：确认为蚁剑webshell通信流量
- 处置：已隔离服务器，正在应急排查
- 状态：🔴 处置中

### 事件2：暴力破解成功
- 时间：02:15（夜班记录）
- 来源：45.33.32.156
- 目标：SSH服务
- 研判：暴破成功后登录了admin账号
- 处置：已封IP，修改密码，排查操作记录
- 状态：🟢 已闭环

## 四、情报接收
- 今日接收0day预警1条（XXX产品漏洞），已确认我方不受影响
- 指挥部下发恶意IP清单123个，已全部加入黑名单

## 五、待办事项
1. 事件1的应急排查报告需在明早8点前完成
2. 夜班同事需关注该Web服务器是否有后续异常

## 六、交班备注
- 重点关注目标：192.168.1.100（Web服务器）
- 特殊注意：23:00-03:00是攻击高峰时段
- 当前已封禁IP清单见附件
```

### 四、护网后 —— 复盘整改

护网结束不代表工作结束，真正有价值的工作在复盘阶段。

**复盘要回答的问题：**

1. **发现层面**
   - 我们发现了多少攻击？有多少是"应该发现但没发现"的？
   - 哪些攻击手段我们之前没遇到过？

2. **响应层面**
   - 告警到处置的平均时间是多少？有没有改进空间？
   - 有没有因为处置不及时导致失分？

3. **防护层面**
   - 哪些漏洞被攻击者利用成功了？为什么之前没修？
   - 安全设备规则有没有优化空间（误报太多/漏报）？

4. **协作层面**
   - 团队配合有没有问题？信息流转是否顺畅？
   - 和管理层、业务部门的沟通是否到位？

**整改清单示例：**
- 修复高危漏洞5个
- 优化IDS规则12条（减少误报）
- 清理幽灵资产3台
- 更新应急响应流程
- 更新值守方案和排班策略

## 📝 今日小结

护网不是一天两天的"突击战"，而是贯穿几个月的大工程。今天你了解了从准备到实战到复盘的全流程。作为初级蓝队分析师，你接触最多的是护网中的"值守"环节——看告警、写日报、交班。但理解全流程会让你知道自己的工作在整体中处于什么位置，为什么要做这些事。

**记住今天的核心**：
- 护网前：资产梳理 + 风险排查 + 加固
- 护网中：24小时值守 + 告警研判 + 日报 + 情报联动
- 护网后：复盘 + 整改 + 更新流程
