## 一、纵深防御的核心思想

### 1.1 用城堡比喻纵深防御

想象你要保护一座城堡里的国王（核心数据）：

```
🏰 城堡防御体系

第一层 护城河 + 吊桥
  → 不能让敌人随便过（网络边界防火墙）
  
第二层 城墙 + 瞭望塔
  → 发现敌情、射箭阻止（IDS/IPS、流量分析）
  
第三层 城门守卫 + 巡逻兵
  → 检查每个进出的人（HIDS、EDR、堡垒机）
  
第四层 宫殿护卫
  → 保护国王身边的人（WAF、RASP）
  
第五层 国王的贴身侍卫
  → 最后一道防线（数据加密、DLP、审计）
```

### 1.2 纵深防御的三个核心原则

| 原则 | 大白话 | 反例 |
|:---|:---|:---|
| **多层互补** | 每层有不同作用，一层被破还有下一层 | 只靠防火墙 → 被打穿就完了 |
| **适度冗余** | 重要区域多设几道防线 | 核心系统只做了网络层防护 |
| **纵深而非堆砌** | 各层要有关联和协同，不是简单叠加 | 买了一堆安全设备但各管各的 |

## 二、五层纵深防御架构

### 2.1 五层全貌图

```
┌─────────────────────────────────────────────────────────────┐
│ 第五层 数据层   │ DLP · 数据库审计 · 加密 · 备份          │
├─────────────────────────────────────────────────────────────┤
│ 第四层 应用层   │ WAF · RASP · 代码审计 · API安全         │
├─────────────────────────────────────────────────────────────┤
│ 第三层 主机层   │ HIDS · EDR · 防病毒 · 堡垒机 · 加固     │
├─────────────────────────────────────────────────────────────┤
│ 第二层 网络内部 │ IDS/IPS · 流量分析 · 蜜罐 · NAC         │
├─────────────────────────────────────────────────────────────┤
│ 第一层 网络边界 │ 防火墙 · WAF(边界) · DDoS防护 · VPN     │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 逐层详解

#### 第一层：网络边界 —— "护城河"

| 设备/技术 | 做什么 | 护网中的策略 |
|:---|:---|:---|
| **防火墙** | 控制进出流量，按IP/端口放行或拒绝 | 默认拒绝+最小开放 |
| **边界WAF** | 过滤Web攻击（SQL注入、XSS等） | 启用最高防护级别 |
| **DDoS防护** | 清洗大流量攻击 | 配置清洗阈值 |
| **VPN** | 安全远程接入 | 多因素认证+设备健康检查 |

> **💡 护网边界策略核心**：默认拒绝一切，只放行业务必需的。把所有不必要的端口都关了！

#### 第二层：网络内部 —— "瞭望塔和巡逻兵"

| 设备/技术 | 做什么 | 为什么重要 |
|:---|:---|:---|
| **IDS（入侵检测）** | 旁路监听，发现可疑流量就告警 | 不改变网络架构，只看不拦 |
| **IPS（入侵防御）** | 串联部署，发现攻击就阻断 | 能直接拦截攻击但可能误杀 |
| **NTA（流量分析）** | 深度分析流量行为模式 | 能发现C2通信、数据外传等隐蔽威胁 |
| **蜜罐** | 伪装成核心资产引诱攻击者 | **告警误报率极低**，任何访问都高度可疑 |
| **NAC（网络准入）** | 控制谁能接入网络 | 防止未授权设备接入内网 |

#### 第三层：主机层 —— "城门守卫"

| 设备/技术 | 做什么 | 区别于杀毒软件 |
|:---|:---|:---|
| **EDR** | 终端检测与响应，持续监控进程/网络/文件 | 行为分析，能发现未知威胁 |
| **HIDS** | 主机入侵检测，监控文件完整性 | 检测文件篡改和后门 |
| **防病毒** | 基于特征库查杀已知恶意软件 | 特征匹配，对新型威胁无效 |
| **堡垒机** | 运维审计，所有操作可追溯 | 防止运维人员"乱来"和账号共享 |
| **系统加固** | 关不必要的服务、打补丁、最小权限 | 减少攻击面 |

#### 第四层：应用层 —— "宫殿护卫"

| 技术 | 做什么 | 应用场景 |
|:---|:---|:---|
| **WAF（应用层）** | 深入HTTP/HTTPS协议解析 | 防SQL注入、XSS、命令注入 |
| **RASP** | 运行时应用自我保护，嵌入应用内部 | 最后一公里的应用层防护 |
| **代码审计** | 检查源代码中是否有安全漏洞 | 开发阶段就消除隐患 |
| **API安全网关** | 保护API接口不被滥用 | 限流、认证、注入防护 |

#### 第五层：数据层 —— "贴身侍卫"

| 技术 | 做什么 | 说明 |
|:---|:---|:---|
| **DLP（数据防泄漏）** | 监控和阻止敏感数据外泄 | 即使攻击者拿到了数据也能拦截 |
| **数据库审计** | 记录所有数据库操作 | 谁在什么时候查了什么数据 |
| **加密** | 数据存储加密和传输加密 | 数据被偷了也读不了 |
| **备份恢复** | 定期备份+恢复演练 | 被勒索了还能恢复 |
| **数据水印** | 给数据打上隐形标记 | 数据泄露后能追溯来源 |

## 三、安全设备全家福

### 3.1 各设备一句话介绍

| 设备 | 一句话 | 类比 |
|:---|:---|:---|
| 🔥 防火墙 | 控制谁能进、谁能出 | 小区门卫 |
| 🛡️ WAF | 检查Web请求是否恶意 | 机场安检X光机 |
| 👁️ IDS | 旁听网络流量，发现可疑行为报警 | 监控摄像头+保安 |
| 🚫 IPS | 串联在链路上，发现攻击就拦截 | 持枪门卫 |
| 📊 SIEM | 汇集所有设备日志，找出关联的攻击行为 | 指挥中心大屏 |
| 🚀 SOAR | 自动化编排处置流程 | 自动化作战系统 |
| 💻 EDR | 监控每台电脑的异常行为 | 给每台电脑配个监视器 |
| 🍯 蜜罐 | 伪装成诱人的目标，等着攻击者上钩 | 陷阱/诱饵 |
| 🔑 堡垒机 | 运维操作统一入口，全程录像 | 操作间+摄像头 |

### 3.2 设备联动架构

```
            防火墙日志 ──┐
            WAF告警 ────┤
            IDS告警 ────┼──→ SIEM ──→ 关联分析 ──→ 告警
            EDR告警 ────┤       │
            蜜罐告警 ────┘       ▼
                            SOAR ──→ 自动封禁IP
                                  ──→ 隔离终端
                                  ──→ 创建工单
                                  ──→ 通知分析师
```

> **核心思路**：SIEM是大脑（汇集分析），SOAR是双手（自动执行）。

## 四、安全运营流程（SOP）

### 4.1 告警处置标准流程

```
告警产生
  ↓
告警分级（紧急/高危/中危/低危）
  ↓
告警分析（是真攻击还是误报？）
  ↓ 真攻击
事件定级（什么级别的事件？）
  ↓
事件处置（遏制→根除→恢复）
  ↓
事件关闭（确认处置完成）
  ↓
复盘总结（为什么要改进？）
```

### 4.2 告警分级标准

| 级别 | 定义 | 响应SLA | 例子 |
|:---|:---|:---|:---|
| 🔴 紧急 | 核心业务受影响/确认已入侵 | 15分钟内响应 | 域控被拿下、数据在泄出 |
| 🟠 高危 | 疑似入侵/高危攻击行为 | 30分钟内响应 | 发现Webshell、暴力破解成功 |
| 🟡 中危 | 扫描探测/可疑行为 | 2小时内处置 | 端口扫描、SQL注入尝试 |
| 🟢 低危 | 信息类告警 | 定时Review | 登录成功通知、配置变更 |

### 4.3 交接班流程

```
交班方需准备：
  ✅ 当前未闭环的高危/紧急事件清单
  ✅ 重点监控对象（昨日可疑IP/行为）
  ✅ 安全设备运行状态
  ✅ 待办事项列表
  
接班方需确认：
  ✅ 理解当前安全态势
  ✅ 知道紧急事件处置进展
  ✅ 知道今天重点关注什么
  ✅ 安全设备和监控正常运行
```

## 五、护网蓝队防守关键点

### 5.1 护网前必须做的五件事

| 序号 | 事项 | 重要性 | 不做的后果 |
|:---|:---|:---|:---|
| 1 | **漏洞修复** | 🔴🔴🔴🔴🔴 | 攻击者利用已知漏洞长驱直入 |
| 2 | **资产盘点** | 🔴🔴🔴🔴🔴 | 被攻击了都不知道哪台机器 |
| 3 | **暴露面收敛** | 🔴🔴🔴🔴 | 攻击面太大防不胜防 |
| 4 | **基线加固** | 🔴🔴🔴🔴 | 默认配置是攻击者的最爱 |
| 5 | **权限管控** | 🔴🔴🔴 | 一个普通用户就能看到核心数据 |

### 5.2 护网中每天必做的四件事

1. 📋 **查看交接记录**，了解夜间发生了什么
2. 📊 **清理未闭环告警**，先处理高危/紧急
3. 🔍 **深度研判3-5个可疑事件**
4. 📝 **撰写每日态势日报**

## 六、蓝队防御能力评分

### 6.1 防御能力自评系统

```python
# 蓝队防御能力多维度评分
class DefenseScorer:
    def __init__(self):
        self.scores = {}
    
    def assess(self, domain, items):
        """评估某个防御域的完成情况
        items: [(项目名, 是否已完成), ...]
        """
        total = len(items)
        done = sum(1 for _, status in items if status)
        self.scores[domain] = (done, total)
    
    def report(self):
        print('\n' + '='*50)
        print('      蓝队防御能力评分报告')
        print('='*50)
        grand_done = 0
        grand_total = 0
        for domain, (done, total) in self.scores.items():
            pct = done / total * 100 if total > 0 else 0
            grand_done += done
            grand_total += total
            bar = '█' * done + '░' * (total - done)
            emoji = '✅' if pct >= 80 else ('⚠️' if pct >= 50 else '❌')
            print(f'{emoji} {domain:10s} [{bar}] {pct:.0f}% ({done}/{total})')
        print(f'\n📊 总评分: {grand_done}/{grand_total} = {grand_done/grand_total*100:.0f}%')

# 给你的环境打分
ds = DefenseScorer()
ds.assess('网络边界', [
    ('防火墙部署', True),
    ('WAF部署', True),
    ('DDoS防护', False),       # ← 没做！
    ('VPN安全接入', True),
])
ds.assess('网络内部', [
    ('IDS/IPS部署', True),
    ('流量分析NTA', False),     # ← 没做！
    ('蜜罐部署', True),
    ('网络分段', True),
])
ds.assess('主机安全', [
    ('HIDS部署', True),
    ('EDR部署', True),
    ('防病毒', True),
    ('堡垒机', False),          # ← 没做！
    ('系统加固', True),
])
ds.assess('日志审计', [
    ('日志集中收集', True),
    ('SIEM平台', True),
    ('告警规则配置', True),
    ('日志保留6个月+', False),  # ← 不满足合规要求！
])
ds.report()
```
