## 📖 核心知识讲解

### 一、SOC是什么？（用"指挥中心"来比喻）

SOC（Security Operations Center，安全运营中心）是蓝队的"作战指挥大厅"。

想象一个大型商场：

```
没有SOC：
  几十个保安各自巡逻 → 有人发现小偷 → 用对讲机喊人 → 混乱

有SOC：
  所有监控画面汇聚到一个大屏幕上
  有专人盯着屏幕 → 发现异常 → 立即按流程处理
  所有事件都有记录 → 事后可以复盘
```

SOC就是这样——把企业所有的安全设备（防火墙、IDS、WAF、终端安全）的告警集中到一个平台，统一监控、分析、处置。

### 二、SOC平台的三大核心模块

```
┌────────── SOC平台 ──────────┐
│                              │
│  ┌──────┐  ┌──────┐  ┌────┐ │
│  │事件   │  │日志   │  │工单 │ │
│  │管理   │  │管理   │  │管理 │ │
│  └──────┘  └──────┘  └────┘ │
│     ↓          ↓         ↓   │
│  告警汇聚   日志检索   流程处置 │
│  态势展示   关联分析   记录闭环 │
└──────────────────────────────┘
```

#### 模块1：事件管理（告警汇聚+态势大屏）
- 所有安全设备的告警汇入SOC
- 大屏幕上实时显示：今天多少告警、攻击来源地图、设备状态
- 分析师在这里发现和认领告警

#### 模块2：日志管理（日志检索+关联分析）
- 所有服务器、网络设备的日志统一存储
- 可以搜索：某IP在最近24小时的所有活动
- 高级的能自动关联，比如"这条Web攻击告警 + 同一IP在服务器上登录成功 = 高危！"

#### 模块3：工单管理（流程处置+记录闭环）
- 每一条告警处理后都要建工单
- 记录：谁处理的、怎么处理的、结果是什么
- 保证每条告警都有"闭环"

### 三、告警分级——哪些告警优先处理？

SOC平台每秒可能收到上百条告警，分析师不可能全看，所以需要自动分级：

| 等级 | 名称 | 定义 | 响应要求 | 举个例子 |
|:---|:---|:---|:---|:---|
| **P0** | 严重 | 正在发生的大规模攻击或已被攻破 | 立即处理（5分钟内） | 核心服务器被植入后门 |
| **P1** | 高危 | 针对核心资产的定向攻击 | 15分钟内处理 | 对数据库的SQL注入且告警多次 |
| **P2** | 中危 | 扫描探测或低风险攻击 | 2小时内处理 | 端口扫描、单次登录失败 |
| **P3** | 低危 | 信息类、可能误报 | 当天处理 | 某IP访问了一个404页面 |
| **P4** | 信息 | 仅供记录参考 | 定期清理 | 设备上线通知 |

**分级依据（三个因素综合判断）：**
1. **攻击类型**：SQL注入 > 扫描探测
2. **目标资产**：核心数据库 > 测试服务器
3. **攻击进展**：已经成功 > 只是尝试

### 四、工单处理的标准流程

从收到告警到处理完毕，一个完整的工单生命周期：

```
收到告警
    ↓
[1. 认领]  分析师认领这条告警，开始处理
    ↓
[2. 研判]  判断是真攻击还是误报？攻击成功了吗？
    ↓
[3. 处置]  真实攻击：封IP、排查系统
           误报：标注原因
    ↓
[4. 记录]  把研判过程和处置措施写进工单
    ↓
[5. 闭环]  确认处理完毕，关闭工单
```

**工单记录必须包含的要素：**
- 告警时间、来源IP、告警类型
- 研判结论：真攻击/误报/测试？依据是什么？
- 处置措施：封IP/通知业务/加固系统？
- 处置结果：已解决/转交/待跟进？

### 五、模拟告警研判实战

以下是10条模拟告警，请按告警研判思路逐一分析：

```
告警1：IDS检测到 45.33.32.156 对Web服务器发起SQL注入攻击
        查情报 → 恶意IP | 目标页面返回200 | 凌晨3点活动
        你的研判：→ P1 高危，真实攻击，需要立即处置

告警2：IDS检测到 192.168.1.100 访问了 www.baidu.com 的404页面
        查情报 → 内网IP | 工作时间 | 单一事件
        你的研判：→ P4 信息，正常行为

告警3：IDS检测到 10.0.0.88 在5分钟内产生了200次登录失败
        查日志 → 200次4625 | 全部针对admin账号
        你的研判：→ P1 高危，暴力破解，需立即查成功没

告警4：IPS检测到内网主机 192.168.1.50 向境外IP 103.xxx.xxx.xxx 发送数据
        查情报 → 目的IP关联恶意C2 | 外发数据约为45MB
        你的研判：→ P0 严重，可能已被攻破，正在外传数据

告警5：WAF拦截了某IP的XSS攻击尝试，攻击未成功
        查源IP → 威胁情报显示为安全 | 流量为安全测试团队的IP
        你的研判：→ P3 低危，内部安全测试，需和白帽子团队确认
```

## 📝 今日小结

SOC是蓝队分析师每天面对的核心工作平台。今天你理解了告警从"产生"到"闭环"的全流程，也知道了告警怎么分级。记住：不是每个告警都是一个"大事件"，大部分告警经过研判后你会发现是误报或低风险——这正是SOC分析师的核心价值：从"噪音"中识别真正的"信号"。

**记住今天的核心**：
- SOC = 安全指挥中心，三大模块：事件/日志/工单
- P0-P4分级 = 严重→信息，按紧急程度处理
- 工单闭环 = 认领→研判→处置→记录→关闭
- 分析师的核心价值 = 从大量告警中识别出真正的威胁
