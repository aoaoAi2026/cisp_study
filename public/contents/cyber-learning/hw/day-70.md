## 📖 核心知识讲解



### 一、OWASP Top 10 2021完整列表


| 排名 | 漏洞类型 | 英文 | 一句话理解 |
|--|----|--|-----|
| A1 | 访问控制失效 | Broken Access Control | 没有正确限制用户"能看什么、能做什么" |
| A2 | 加密机制失效 | Cryptographic Failures | 敏感数据没加密或加密方式有缺陷 |
| A3 | 注入 | Injection | 用户输入被当作代码执行了 |
| A4 | 不安全设计 | Insecure Design | 架构设计阶段就埋下的安全问题 |
| A5 | 安全配置错误 | Security Misconfiguration | 默认配置没改、多余功能没关 |
| A6 | 脆弱和过时组件 | Vulnerable Components | 用了有已知漏洞的老版本组件 |
| A7 | 身份认证失效 | Identification Failures | 登录、会话管理方面的问题 |
| A8 | 软件和数据完整性失效 | Software Integrity Failures | 没有验证软件来源和数据的完整性 |
| A9 | 日志记录和监控失效 | Security Logging Failures | 被打了很久都没发现 |
| A10 | 服务端请求伪造 | SSRF | 服务器被利用去请求不该请求的内网资源 |




### 二、每个漏洞的检测与防御




**A1 访问控制失效 — 检测：**

- 越权测试（更换用户ID/角色访问他人数据）

- 目录/接口暴露检查（admin路径是否可匿名访问）

- JWT/Cookie篡改测试

**防御：** 服务端强制验权、最小权限原则、拒绝优先



**A2 加密机制失效 — 检测：**

- 检查HTTPS配置（TLS版本、证书有效性）

- 检查敏感数据是否明文传输（密码、Token、身份证号）

- 检查Cookie的安全属性（Secure、HttpOnly、SameSite）

**防御：** 全站HTTPS、敏感数据加密存储、HSTS头



**A3 注入 — 检测：**

- SQL注入：`'`、`"`、`OR 1=1`、时间盲注

- 命令注入：`; id`、`| whoami`、`` `ls` ``

- LDAP/XML/XPath注入等

**防御：** 参数化查询、输入白名单校验、转义特殊字符



**A4 不安全设计 — 检测：**

- 威胁建模中发现的设计缺陷

- 业务逻辑漏洞（如：无限次尝试优惠码、负数金额）

**防御：** 安全设计评审(SDL)、威胁建模、安全需求文档



**A5 安全配置错误 — 检测：**

- 默认账号密码未修改（admin/admin）

- 目录浏览未关闭（看到所有文件列表）

- 错误信息泄露详细堆栈

- 不必要的HTTP方法开启（PUT/DELETE/TRACE）

**防御：** 基线核查、自动化配置检查、定期加固



**A6 脆弱和过时组件 — 检测：**

- 版本扫描（获取中间件、框架、库的版本号，查CVE）

- 依赖项安全检查（npm audit, OWASP Dependency Check）

**防御：** 定期更新组件、漏洞公告监控、SBOM管理



**A7 身份认证失效 — 检测：**

- 弱密码策略（是否允许123456）

- 暴力破解防护（是否有验证码/锁定机制）

- Session管理（Token是否可预测、登出是否真的失效）

**防御：** 多因素认证、强密码策略、安全的Session管理



**A8 软件和数据完整性失效 — 检测：**

- CDN/第三方脚本是否被篡改（Subresource Integrity）

- 反序列化数据是否可被篡改

**防御：** SRI校验、数字签名、完整性校验机制



**A9 日志和监控失效 — 检测：**

- 关键操作是否有日志（登录、修改、删除）

- 日志是否包含足够信息用于追溯

- 是否有实时告警机制

**防御：** 全量日志、集中日志管理、自动化告警和响应



**A10 SSRF — 检测：**

- URL参数是否可被操控访问内网

- 请求云元数据服务(169.254.169.254)

**防御：** URL白名单、禁用危险协议(file/dict/gopher)、内网访问控制




### 三、OWASP Top 10的蓝队视角


红队利用OWASP Top 10找漏洞打进去，蓝队用OWASP Top 10做"防御检查清单"：



1. **A1-A3**：应用层最直接的攻击 → WAF/IDS/代码审计覆盖

2. **A4-A6**：配置和架构层面的问题 → 基线核查+资产管理覆盖

3. **A7-A8**：认证和完整性 → 身份安全+供应链安全覆盖

4. **A9-A10**：检测能力本身和特殊攻击 → SOC+SIEM+纵深防御覆盖

## 💡 知识卡片：蓝队日常学习技巧

### 如何高效记忆安全知识
1. **不要死记硬背** — 理解"为什么"比记住"是什么"重要100倍
2. **用类比法** — 把技术概念类比成生活中的事物。比如：防火墙像门卫，IDS像监控摄像头，SIEM像中控室的监控大屏
3. **动手验证** — 看100篇文档不如自己搭一次环境。漏洞复现是最好的学习方式
4. **画图** — 画攻击路径图、网络拓扑图、数据流图。画图的过程就是理解的过程
5. **讲给别人听** — 如果你能用大白话把XXE漏洞讲给一个非技术人员听，说明你真的懂了

### 蓝队成长的核心能力
| 能力类型 | 具体表现 | 如何训练 |
|----------|----------|----------|
| 分析思维 | 看到异常现象能推导出攻击路径 | 多看攻防案例，画攻击链 |
| 工具熟练 | CTF/靶场/安全产品快速上手 | 每个工具用熟而非"会用" |
| 沟通能力 | 能把技术问题讲清楚给非技术同事 | 写报告、做分享 |
| 抗压能力 | 应急时冷静，不被海量告警淹没 | 模拟值班+复盘 |
| 持续学习 | 新技术新漏洞出来了能快速跟进 | 关注推特/公众号/RSS |

### 今日学习三步法
1. **先看目标**（5分钟） — 今天我到底要学什么？答得上来一个核心问题就行
2. **主线学习**（90分钟） — 按本文档主线走，该动手就动手，别光看
3. **输出检验**（15分钟） — 把今天学的东西用自己话写下来，画个图更好

> 🔑 **最重要的心态**：安全学习不是速成的。每天进步1%，一年后你会是现在的37倍（1.01^365≈37.8）。别焦虑，坚持写，坚持做。

## 🚀 蓝队进阶之路：从"能检测"到"能防御"

蓝队能力成长一般经历三个阶段：

**第一阶段：能识别** — 看到告警知道大概是什么攻击
**第二阶段：能分析** — 能判断攻击的严重程度、攻击者的意图、攻击到了哪一步
**第三阶段：能防御** — 不仅知道怎么检测，还知道怎么从根本上防御这类攻击

问问自己：对于今天学的这类攻击/技术，你目前在哪个阶段？
如果还在第一阶段，不要焦虑——每一个高手都是从第一阶段走过来的。
目标很清晰：把这个主题学透到能给别人讲清楚的程度。

### 如何加速成长
1. **刻意练习**：不是泛泛地看，而是针对一个漏洞反复演练检测和防御
2. **输出倒逼输入**：尝试写一篇关于今天主题的公众号文章/内部分享（哪怕不发布）
3. **横向对比**：同类漏洞/攻击手法之间的异同点（这帮你建立知识网络）
4. **从攻击学防御**：理解攻击者为什么会用这种手法，就能猜到他们下一步会做什么

坚持每天写，60天后你会有一份属于自己的蓝队知识库。
这比什么教程都更有价值——因为是你自己的理解。
- 持续性胜过强度
- 每天进步一点点
- 120天后你会感谢现在努力的自己

## 🔬 OWASP Top 10实战案例对照

回顾你学过的Day 31-69，每一个漏洞类型都可以在OWASP Top 10中找到位置：

| 之前学的 | OWASP分类 | 检测重心 |
|----------|-----------|----------|
| XSS跨站脚本 | A3 注入 | WAF规则+输出编码检查 |
| SQL注入 | A3 注入 | WAF规则+数据库审计 |
| CSRF | A1 访问控制 | Token校验+Referer检查 |
| SSRF | A10 SSRF | 出站流量监控+URL白名单 |
| 文件上传 | A5 安全配置 | 文件类型检测+上传目录权限 |
| 命令注入 | A3 注入 | WAF规则+进程监控 |
| 弱口令 | A7 身份认证 | 登录日志+爆破检测 |
| 信息泄露 | A5/A4 | 错误页面定制+目录浏览关闭 |
| 逻辑漏洞 | A1/A4 | 业务行为分析+流程审计 |

**顶级蓝队的思考方式：**
不是"我看到一个XSS"，而是"我看到一个A3类型的攻击，它利用了输出编码的缺失，攻击者的下一步可能是窃取Cookie或重定向到钓鱼页面，我正在攻击链的初始阶段，需要检查攻击是否成功"。

> ⭐ 把知识从"散点"变成"网络"，才是从熟练工到专家的关键跨越。
