## 模拟值守第二天：告警研判实战

今天进入"实战"模式——模拟真实的护网告警场景，训练你的研判能力。

### 场景一：凌晨发现大量SQL注入告警

**场景描述**：凌晨3点，WAF突然弹出30+条SQL注入告警。
源IP来自3个不同的国家（美国、俄罗斯、乌克兰），
目标是你公司的官网搜索接口。

**研判流程**：
1. 先在ELK中搜索这些IP的所有历史请求——发现它们之前还有端口扫描行为
2. 查看Payload：都是经典的 ' OR 1=1--、UNION SELECT 1,2,3——
   这些是SQLMap的典型Payload（工具化的自动化攻击）
3. 查看响应内容：所有响应都是正常的搜索结果页，没有数据库报错
4. 判断：自动化扫描工具在广撒网，但注入没有成功
5. 处置：标记为P2，封禁3个IP，在日报中记录。不需要启动应急。

**核心教训**：告警多不等于危害大，关键看攻击有没有成功。

### 场景二：发现Webshell上传成功

**场景描述**：下午，WAF告警显示某IP成功上传了一个JSP文件到Tomcat服务器。
源IP：国内某IP。

**研判流程**：
1. 这是P0告警！立即点开详情查看
2. WAF日志显示：PUT /manager/shell.jsp/ HTTP/1.1 201 Created
3. 201表示文件上传成功——攻击成功！
4. 1分钟后，同一IP GET /manager/shell.jsp?cmd=whoami HTTP/1.1 200
   返回了 root ——攻击者已经可以执行命令了
5. P0处置：立即在防火墙上封禁该IP + SSH登录服务器检查 + 通知组长

**核心教训**：PUT + 201 + 后续GET = 典型的Webshell上传-执行模式。
看到这个模式，不要犹豫，立即处置。

### 场景三：内网横向移动的告警

**场景描述**：IDS告警：内网IP 192.168.1.50 在扫描同网段其他服务器的
445(SMB)和3389(RDP)端口。这是内网横向移动的典型前置行为！

**研判流程**：
1. P0！内网的异常扫描比外网扫描严重100倍——说明边界已被突破
2. 检查192.168.1.50的最近日志——发现1小时前有一个来自境外IP的Weblogic漏洞利用成功
3. 检查192.168.1.50的进程——发现了可疑的Java进程 + 反弹Shell连接
4. 立即隔离192.168.1.50（断网不关机）+ 通知组长 + 启动应急响应
5. 排查其他服务器是否也已被入侵（检查同网段的SSH登录日志）

**核心教训**：内网异常告警 = 最严重的告警。不要纠结是不是误报，先隔离再说。

### 告警研判的ABC原则

总结一个告警研判框架——ABC原则：

**A（Asset，资产）**：受影响的资产重要吗？是核心业务服务器还是测试机？
**B（Behavior，行为）**：攻击者做了什么？只是在探测还是已经得手？
**C（Context，上下文）**：同一攻击者的历史行为是什么？之前有没有相关告警？

每次收到告警，用ABC原则快速过一遍，判断准确率能提升到80%以上。

## 蓝队面试自测

**Q1: 描述一次完整的Web攻击检测流程**
A: 收到告警 -> ELK日志确认 -> 分析Payload -> 判断类型 -> 确认是否成功 -> 处置 -> 写报告。

**Q2: IDS和IPS的区别？**
A: IDS只检测不阻断（摄像头），IPS检测并阻断（门禁）。

**Q3: ATT&CK框架对蓝队的价值？**
A: 告诉我们攻击者"会做什么"和"怎么做"，蓝队据此在对应战术阶段设置检测点。

**Q4: 告警太多怎么处理？**
A: 分级处理——P0立即、P1今日内、P2周汇总，然后优化规则减少误报。

**Q5: WAF和IPS的区别？**
A: WAF专用于Web应用层（HTTP/HTTPS），IPS用于网络层（TCP/IP）。WAF理解SQL注入和XSS，IPS理解端口扫描和DDoS。

## 蓝队面试题库

**Q1: TCP三次握手是什么？为什么理解它对蓝队很重要？**

A: SYN -> SYN-ACK -> ACK。理解三次握手就能理解端口扫描（大量SYN到不同端口）、
DDoS（SYN Flood耗尽半连接队列）、C2通信（异常的TCP连接模式）。

**Q2: 你怎么理解纵深防御？**

A: 纵深防御 = 多层防护。边界防火墙是第一层、WAF是第二层、主机安全是第三层、
数据库审计是第四层。攻击者突破一层还有下一层——这就是纵深防御的价值。

**Q3: 护网期间发现一台服务器被黑了，你的第一步是什么？**

A: 隔离（断网不关机保留证据）> 通知组长 > 排查入侵路径 > 清除后门 >
修复漏洞 > 恢复上线 > 复盘报告。注意：千万不要关机！内存证据会丢失。

**Q4: 怎样区分自动化扫描和定向攻击？**

A: 自动化扫描特征：Payload是字典式（SQLMap的常见Payload）、
扫描速度快（<1秒一个请求）、同时扫描多个不相关目标。
定向攻击：Payload有针对性（针对特定应用）、
攻击节奏有策略（先探测再利用）、
关注特定高价值目标。

**Q5: 日志分析中最常用的Linux命令？**

A: grep（搜索关键字）、awk（字段提取）、sed（文本替换）、
sort + uniq -c（统计）、tail -f（实时监控）。
组合示例：cat access.log | grep "UNION SELECT" | awk '{print $1}' | sort | uniq -c | sort -nr

### 等保合规的常见落地难点

1. **日志保存6个月**：日志量巨大，存储成本高。建议分级存储：热数据(最近7天SSD) → 温数据(30天普通硬盘) → 冷数据(6个月对象存储)
2. **最小权限原则**：运维习惯用root，推行困难。建议先做权限审计，再逐步收紧
3. **定期演练落实**：应急预案写好就束之高阁。建议每季度做一次桌面演练
4. **供应商管理**：第三方系统也在等保范围内，但对方不配合。建议合同约定安全条款

某金融企业在护网期间成功抵御了所有红队攻击，经验总结：

## 工具进阶技巧

### Shell命令效率技巧

统计攻击IP排行榜：
```
cat access.log | grep -E "(UNION SELECT|<script>|\.\./etc/)" | awk '{print $1}' | sort | uniq -c | sort -nr | head -20
```

实时监控新出现的攻击：
```
tail -f access.log | grep --line-buffered -E "(SQL injection|XSS|command injection)"
```

批量封禁IP：
```
for ip in $(cat blocklist.txt); do
  iptables -A INPUT -s $ip -j DROP
done
```

### Python安全脚本速查

快速端口扫描：
```python
import socket
for port in range(1, 1025):
    s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    s.settimeout(0.1)
    if s.connect_ex(('127.0.0.1', port)) == 0:
        print(f"Port {port}: OPEN")
    s.close()
```

检查IP是否为恶意：
```python
import requests
```
