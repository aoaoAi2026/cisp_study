## 一、为什么要学攻击链？

### 1.1 用大白话解释

想象你想偷一家银行：
1. 先踩点观察（**侦察**）—— 看看有几个保安、摄像头在哪
2. 准备工具（**武器化**）—— 找开锁工具、伪装身份
3. 找到入口（**投递**）—— 混进去或者撬门
4. 突破防线（**利用**）—— 打开保险库
5. 安插内应（**安装**）—— 放个内应方便下次进入
6. 保持联系（**C2**）—— 和内应保持沟通
7. 卷钱跑路（**目标达成**）—— 把钱拿走

网络攻击的逻辑一模一样！这个七步法就是 **Cyber Kill Chain**。

### 1.2 为什么蓝队要学红队技术？

> "最好的防守是理解攻击。不知道攻击者怎么打，你就不知道该在哪设防。"

护网中大量失分是因为：
- 不知道攻击者从哪进来的（没监控侦察阶段）
- 不知道攻击者在哪个阶段（没做攻击链分析）
- 不知道下一步会做什么（没有预判能力）

## 二、Cyber Kill Chain 七步攻击法

### 2.1 全貌速览

```
🔍 阶段1：侦察 (Reconnaissance)
  ↓  攻击者：收集目标信息（IP、域名、员工邮箱、技术栈）
  
⚔️  阶段2：武器化 (Weaponization)
  ↓  攻击者：准备攻击工具（制作钓鱼文档、编写exploit）
  
📧 阶段3：投递 (Delivery)
  ↓  攻击者：发送攻击载荷（钓鱼邮件、注入恶意链接）
  
💥 阶段4：利用 (Exploitation)
  ↓  攻击者：触发漏洞获取初始访问权限
  
💾 阶段5：安装 (Installation)
  ↓  攻击者：安装后门、木马，确保持续访问
  
📡 阶段6：命令控制 (C2 - Command & Control)
  ↓  攻击者：建立远程控制通道
  
🎯 阶段7：目标达成 (Actions on Objectives)
  ↓  攻击者：数据窃取、勒索加密、破坏系统
```

### 2.2 每个阶段的详细分析

#### 阶段1：侦察（Reconnaissance）—— "踩点"

| 攻击者做什么 | 具体手段 | 你能在哪发现 |
|:---|:---|:---|
| 收集域名和IP | whois查询、DNS枚举 | 大量DNS查询日志 |
| 扫描开放端口 | Nmap、Masscan扫描 | 防火墙日志中大量探测 |
| 抓取员工信息 | LinkedIn爬取、官网人员列表 | 外部信息无法直接检测 |
| 寻找子域名 | 子域名爆破、证书透明度日志 | DNS查询异常 |

#### 阶段2：武器化（Weaponization）—— "磨刀"

| 攻击者做什么 | 具体手段 | 你能在哪发现 |
|:---|:---|:---|
| 制作恶意文档 | 带宏病毒的Word/Excel文件 | 无法直接检测 |
| 编写exploit | 针对已知漏洞写利用代码 | 无法直接检测 |
| 搭建C2服务器 | 配置CobaltStrike等工具 | 通过威胁情报获取C2 IP |

**这个阶段攻击者在自己的环境里操作，你很难直接检测。但你可以在后面的阶段抓住他。**

#### 阶段3：投递（Delivery）—— "开门"

| 攻击者做什么 | 具体手段 | 你能在哪发现 |
|:---|:---|:---|
| 发送钓鱼邮件 | 伪造发件人、带有恶意链接 | 邮件网关日志 |
| 水坑攻击 | 在目标常访问的网站植入恶意代码 | Web访问日志 |
| USB投递 | 在停车场丢带病毒的U盘 | 终端安全告警 |

#### 阶段4：利用（Exploitation）—— "破门而入"

| 攻击者做什么 | 具体手段 | 你能在哪发现 |
|:---|:---|:---|
| SQL注入 | `' OR '1'='1` 等手法 | WAF日志 |
| 文件上传漏洞 | 上传Webshell | WAF/Web服务器日志 |
| 远程命令执行 | Struts2、Log4j等 | 系统进程日志 |
| 弱口令爆破 | hydra、medusa暴力破解 | 登录认证日志 |

#### 阶段5：安装（Installation）—— "钉钉子"

| 攻击者做什么 | 具体手段 | 你能在哪发现 |
|:---|:---|:---|
| 安装Webshell | 上传小马/大马 | 文件监控、Web目录异常文件 |
| 添加计划任务 | `schtasks /create` | Windows安全日志（事件ID 4698） |
| 写注册表启动项 | `HKLM\...\Run` | Sysmon日志 |
| 创建隐藏用户 | `net user hacker$ /add` | Windows安全日志 |

#### 阶段6：C2通信 — "保持联系"

| 攻击者做什么 | 具体手段 | 你能在哪发现 |
|:---|:---|:---|
| HTTP/HTTPS回连 | Beacon心跳包 | 出站流量异常 |
| DNS隧道 | 超长域名查询 | DNS查询日志（高频率+长域名） |
| ICMP隧道 | 隐藏数据在ping包中 | 异常ICMP流量 |

#### 阶段7：目标达成 — "收网"

| 攻击者做什么 | 具体手段 | 你能在哪发现 |
|:---|:---|:---|
| 数据窃取 | 压缩后通过C2通道外传 | 大量数据外传流量 |
| 勒索加密 | 运行加密程序 | 大量文件修改操作 |
| 销毁日志 | `wevtutil cl Security` | 日志清理事件 |
| 横向移动到域控 | Pass-the-Hash、DCSync | 异常域认证事件 |

## 三、MITRE ATT&CK 框架

### 3.1 ATT&CK 是什么？

ATT&CK（Adversarial Tactics, Techniques, and Common Knowledge）是全球公认的**攻击者行为知识库**，由 MITRE 公司维护。

**一句话**：ATT&CK 就是一本**攻击者手法大全**，告诉你攻击者在每个阶段会用哪些具体技术。

### 3.2 14个战术类别速查

| 战术ID | 战术名称 | 对应 Kill Chain 阶段 |
|:---|:---|:---|
| TA0001 | 初始访问 (Initial Access) | 阶段3 投递 |
| TA0002 | 执行 (Execution) | 阶段4 利用 |
| TA0003 | 持久化 (Persistence) | 阶段5 安装 |
| TA0004 | 权限提升 (Privilege Escalation) | 阶段5 安装 |
| TA0005 | 防御规避 (Defense Evasion) | 贯穿全程 |
| TA0006 | 凭据访问 (Credential Access) | 阶段5 安装 |
| TA0007 | 发现 (Discovery) | 阶段1 侦察（内网版） |
| TA0008 | **横向移动** (Lateral Movement) | 阶段7 |
| TA0009 | 收集 (Collection) | 阶段7 |
| TA0010 | 数据渗出 (Exfiltration) | 阶段7 |
| TA0011 | 命令与控制 (C2) | 阶段6 |

### 3.3 ATT&CK 检测覆盖度热力图

```python
# ATT&CK战术检测覆盖度可视化
class AttackCoverage:
    def __init__(self):
        self.tactics = {
            'TA0001': '初始访问', 'TA0002': '执行', 'TA0003': '持久化',
            'TA0004': '权限提升', 'TA0005': '防御规避', 'TA0006': '凭据访问',
            'TA0007': '发现', 'TA0008': '横向移动', 'TA0009': '收集',
            'TA0010': '数据渗出', 'TA0011': '命令控制'
        }
        self.coverage = {}
    
    def has_detection(self, tactic_id, techniques):
        """记录该战术下有多少技术有检测能力"""
        self.coverage[tactic_id] = techniques
    
    def heatmap(self):
        print('=== ATT&CK 检测覆盖热力图 ===')
        max_tech = 10
        for tid, tname in self.tactics.items():
            detected = self.coverage.get(tid, 0)
            bar = '█' * min(detected, max_tech) + '░' * (max_tech - min(detected, max_tech))
            pct = min(detected, max_tech) * 10
            emoji = '🟢' if pct >= 70 else ('🟡' if pct >= 40 else '🔴')
            print(f'{emoji} {tname:8s} [{bar}] {pct}%')

# 假设你的检测能力覆盖情况
ac = AttackCoverage()
ac.has_detection('TA0001', 3)   # 初始访问覆盖少——要注意！
ac.has_detection('TA0002', 7)   # 执行检测较好
ac.has_detection('TA0003', 5)   # 持久化检测一般
ac.has_detection('TA0008', 6)   # 横向移动
ac.has_detection('TA0011', 4)   # C2检测偏弱——要注意！
ac.heatmap()
```

## 四、红队常用攻击手法

### 4.1 信息收集阶段

| 技术 | 工具 | 做什么 |
|:---|:---|:---|
| 端口扫描 | Nmap, Masscan | 发现开放的服务 |
| 子域名枚举 | Subfinder, Amass | 找到更多攻击面 |
| 目录爆破 | Dirbuster, Gobuster | 发现隐藏的管理后台 |
| 社工信息收集 | theHarvester, Maltego | 收集员工邮箱和社交信息 |

### 4.2 漏洞利用阶段

| 技术 | 攻击目标 | 常见程度 |
|:---|:---|:---|
| SQL注入 | 数据库 | ⭐⭐⭐⭐⭐ 非常常见 |
| XSS | 用户浏览器 | ⭐⭐⭐⭐ |
| 文件上传 | Web服务器 | ⭐⭐⭐⭐⭐ |
| 命令注入 | 操作系统 | ⭐⭐⭐ |
| 反序列化 | Java/.NET应用 | ⭐⭐⭐ |

### 4.3 权限维持阶段

| 技术 | Windows实现 | Linux实现 |
|:---|:---|:---|
| 计划任务 | `schtasks /create` | `crontab -e` |
| 启动项 | 注册表Run键 | `/etc/rc.local` |
| WebShell | ASP/PHP/JSP小马 | 同左 |
| SSH后门 | 用SSH隧道 | 添加SSH公钥 |

### 4.4 横向移动阶段

| 技术 | 大白话解释 | 危险程度 |
|:---|:---|:---|
| **Pass-the-Hash** | 偷了NTLM哈希直接登录，不用密码 | 🔴 极高 |
| **Pass-the-Ticket** | 伪造Kerberos票据冒充任意用户 | 🔴 极高 |
| **WMI远程执行** | 用Windows管理工具远程执行命令 | 🔴 高 |
| **PsExec** | 微软官方远程执行工具，被攻击者滥用 | 🟡 中高 |
| **RDP登录** | 直接远程桌面登录 | 🟡 中 |

## 五、攻击链每个阶段的检测机会

### 5.1 最佳检测窗口

```
攻击链阶段      检测难度      检测手段
─────────────────────────────────────────
侦察            ★★★★☆ 难    DNS日志、FW日志（大量连接）
武器化          ★★★★★ 极难  几乎无法从外部检测
投递            ★★★☆☆ 中等  邮件网关、终端安全
利用            ★★☆☆☆ 较易  WAF、IDS、系统日志
安装            ★★☆☆☆ 较易  EDR、文件监控、进程监控
C2通信          ★★★☆☆ 中等  NTA流量分析、DNS日志
目标达成        ★★☆☆☆ 较易  数据外传检测、文件操作审计
```

### 5.2 打破攻击链的策略

**攻击链越靠后，检测越容易但损失越大——所以要在前面就打断它！**

| 打断位置 | 怎么做 |
|:---|:---|
| 侦察→武器化 | 收敛暴露面、隐藏敏感信息 |
| 武器化→投递 | 邮件安全网关、安全意识培训 |
| 投递→利用 | WAF、网络分段、及时打补丁 |
| 利用→安装 | EDR、最小权限、应用白名单 |
| 安装→C2 | 出站流量监控、DNS安全 |
| C2→目标达成 | 数据防泄漏(DLP)、异常行为检测 |

## 六、从攻击视角到防御策略

### 6.1 心态转变

| 初级思维 | 高级思维 |
|:---|:---|
| "我们把门锁好了" | "锁好了，但万一有人翻窗进来呢？" |
| "这个告警可能是误报" | "先假设它为真，排除后再放过" |
| "我们没被攻击过" | "可能被攻击了但还不知道" |

### 6.2 攻击链驱动的防御清单

- [ ] 侦察阶段：我的互联网暴露面信息被收集了多少？
- [ ] 投递阶段：钓鱼邮件能绕过我的邮件网关吗？
- [ ] 利用阶段：我的WAF能挡住常见Web攻击吗？
- [ ] 安装阶段：有异常进程/文件/注册表修改的检测吗？
- [ ] C2阶段：DNS隧道和加密C2能被发现吗？
- [ ] 横向移动：内网异常登录有告警吗？
- [ ] 目标达成：大量数据外传能被检测吗？

## 七、PTES 渗透测试标准

PTES（Penetration Testing Execution Standard）是渗透测试执行的国际标准，分七个阶段：

1. **前期交互**：和客户确定测试范围、规则
2. **信息收集**：全网搜集目标信息
3. **威胁建模**：分析可能的攻击路径
4. **漏洞分析**：发现可利用的漏洞
5. **漏洞利用**：实际利用漏洞获取权限
6. **后渗透利用**：权限提升、横向移动、数据收集
7. **报告编写**：输出专业测试报告

## 八、护网中的攻击链实战

### 8.1 护网红队典型攻击路径

```
1. 信息收集 → 发现目标Web系统存在旧版本框架
2. 武器化    → 准备针对该版本漏洞的exploit
3. 投递      → 通过漏洞或钓鱼获取初始权限
4. 利用      → 成功获取Web服务器Shell
5. 安装      → 上传CobaltStrike Beacon
6. C2通信    → 建立隐蔽的HTTPS回连通道
7. 横向移动  → 导出Mimikatz偷凭据→PtH打域控
8. 目标达成  → 拿到域控权限，完成攻击
```

### 8.2 蓝队如何沿着攻击链反向溯源

如果发现第6阶段（C2通信），你应该：
1. **向下追**：谁在发C2请求？→ 找到感染的机器 → 找到后门
2. **向上查**：怎么进来的？→ 查Web日志 → 找到漏洞利用痕迹 → 定位攻击入口
3. **横向查**：还去了哪？→ 查内网登录日志 → 追踪横向移动路径
4. **动机分析**：偷了什么？→ 查数据外传流量 → 评估损失

## 九、实战演练与能力检验

### 9.1 动手实验：攻击链映射
