## 📖 核心知识讲解



### 一、什么是安全基线？


**安全基线 = 系统最基础的安全配置要求。**

类比：你买了一套房子，基线的要求就是"门窗要有锁、煤气要关好、监控要开着"。

没有达到基线的系统，就好像门开着没锁的房子——不用什么高深的技术就能进去。




### 二、Linux安全基线检查清单


| 检查类别 | 检查项 | 检查命令 | 合规要求 |
|----|---|----|----|
| 账号安全 | 是否存在空口令账号 | awk -F: '($2==""){print $1}' /etc/shadow | 不允许存在 |
| 账号安全 | root是否允许远程登录 | grep PermitRootLogin /etc/ssh/sshd_config | 应设为no或prohibit-password |
| 口令策略 | 密码复杂度要求 | grep pam_pwquality /etc/pam.d/system-auth | 至少8位，含大小写+数字+特殊字符 |
| 口令策略 | 密码过期时间 | grep PASS_MAX_DAYS /etc/login.defs | 不超过90天 |
| 登录策略 | 登录失败锁定 | grep pam_tally2 /etc/pam.d/sshd | 连续失败5次锁定30分钟 |
| 访问控制 | umask设置 | umask | 应为027或更严格 |
| 访问控制 | 关键文件权限 | ls -la /etc/passwd /etc/shadow | passwd 644, shadow 000或400 |
| 审计日志 | auditd是否运行 | systemctl status auditd | 应运行中 |
| 审计日志 | 日志保存时间 | grep max_log_file /etc/audit/auditd.conf | 符合策略要求 |
| 服务管理 | 不必要的服务 | systemctl list-unit-files --state=enabled | 关闭telnet/tftp等不安全服务 |
| 补丁管理 | 系统更新状态 | yum check-update / apt list --upgradable | 安全更新应及时安装 |




### 三、Windows安全基线检查清单


| 检查类别 | 检查项 | 检查方式 | 合规要求 |
|----|---|----|----|
| 账号安全 | Guest账号状态 | net user Guest | 应禁用 |
| 账号安全 | Administrator重命名 | net user | 应重命名默认管理员 |
| 口令策略 | 密码复杂度 | secpol.msc → 账户策略 → 密码策略 | 启用复杂度要求 |
| 口令策略 | 密码最小长度 | 同上 | 至少8位 |
| 登录策略 | 账户锁定阈值 | secpol.msc → 账户锁定策略 | 5次无效登录锁定 |
| 访问控制 | 共享文件夹 | net share | 默认共享(C$/ADMIN$)应关闭 |
| 访问控制 | UAC状态 | 控制面板 → 用户账户 | 应启用 |
| 审计日志 | 审核策略 | secpol.msc → 本地策略 → 审核策略 | 审核登录/对象访问/策略变更 |
| 审计日志 | 事件日志大小 | eventvwr.msc → Windows日志属性 | 安全日志至少200MB |
| 服务管理 | 不必要的服务 | services.msc | 关闭Print Spooler/Remote Registry等 |
| 防火墙 | 防火墙状态 | wf.msc | 所有配置文件都应开启 |
| 补丁管理 | 更新状态 | 控制面板 → Windows Update | 自动更新应开启 |




### 四、合规整改的"五步走"


1. **评估**：用自动化工具+手工检查，得出当前基线的合规率

2. **风险分级**：不合规项目按"高危/中危/低危"排序

3. **整改计划**：高危立即修，中危排期修，低危逐步修

4. **验证**：整改后重新检查，确认已修复

5. **持续监控**：用自动化工具定期扫描，防止配置漂移






### 五、常用基线核查工具


| 工具 | 平台 | 特点 |
|--|--|--|
| CIS-CAT | Win/Linux | CIS官方工具，对标CIS Benchmark标准 |
| Lynis | Linux | 开源安全审计工具，全面且易用 |
| OpenSCAP | Linux | 基于SCAP标准的合规扫描 |
| Microsoft Baseline Analyzer | Windows | 微软官方基线分析工具 |
| Ansible/Salt/puppet | Win/Linux | 自动化运维工具，可批量执行基线检查 |
| 自写脚本 | Win/Linux | 最灵活，可定制化 |

- 防火墙状态检查

- auditd运行状态检查

## 📝 今日小结


安全基线看起来"琐碎"——改个密码策略、关个服务、调个权限——但就是这些"琐碎"的事情组成了系统安全的基础。

很多重大安全事件的根源，就是基线没做好：一个默认密码没改、一个不需要的服务没关、一个补丁没打。

把基线做成自动化是蓝队的基本功，不要每次都手工检查。

## 💡 知识卡片：蓝队日常学习技巧

### 如何高效记忆安全知识
1. **不要死记硬背** — 理解"为什么"比记住"是什么"重要100倍
2. **用类比法** — 把技术概念类比成生活中的事物。比如：防火墙像门卫，IDS像监控摄像头，SIEM像中控室的监控大屏
3. **动手验证** — 看100篇文档不如自己搭一次环境。漏洞复现是最好的学习方式
4. **画图** — 画攻击路径图、网络拓扑图、数据流图。画图的过程就是理解的过程
5. **讲给别人听** — 如果你能用大白话把XXE漏洞讲给一个非技术人员听，说明你真的懂了

### 蓝队成长的核心能力
| 能力类型 | 具体表现 | 如何训练 |
|----------|----------|----------|
| 分析思维 | 看到异常现象能推导出攻击路径 | 多看攻防案例，画攻击链 |
| 工具熟练 | CTF/靶场/安全产品快速上手 | 每个工具用熟而非"会用" |
| 沟通能力 | 能把技术问题讲清楚给非技术同事 | 写报告、做分享 |
| 抗压能力 | 应急时冷静，不被海量告警淹没 | 模拟值班+复盘 |
| 持续学习 | 新技术新漏洞出来了能快速跟进 | 关注推特/公众号/RSS |

### 今日学习三步法
1. **先看目标**（5分钟） — 今天我到底要学什么？答得上来一个核心问题就行
2. **主线学习**（90分钟） — 按本文档主线走，该动手就动手，别光看
3. **输出检验**（15分钟） — 把今天学的东西用自己话写下来，画个图更好

> 🔑 **最重要的心态**：安全学习不是速成的。每天进步1%，一年后你会是现在的37倍（1.01^365≈37.8）。别焦虑，坚持写，坚持做。

## 🚀 蓝队进阶之路：从"能检测"到"能防御"

蓝队能力成长一般经历三个阶段：

**第一阶段：能识别** — 看到告警知道大概是什么攻击
**第二阶段：能分析** — 能判断攻击的严重程度、攻击者的意图、攻击到了哪一步
**第三阶段：能防御** — 不仅知道怎么检测，还知道怎么从根本上防御这类攻击

问问自己：对于今天学的这类攻击/技术，你目前在哪个阶段？
如果还在第一阶段，不要焦虑——每一个高手都是从第一阶段走过来的。
目标很清晰：把这个主题学透到能给别人讲清楚的程度。

### 如何加速成长
1. **刻意练习**：不是泛泛地看，而是针对一个漏洞反复演练检测和防御
2. **输出倒逼输入**：尝试写一篇关于今天主题的公众号文章/内部分享（哪怕不发布）
3. **横向对比**：同类漏洞/攻击手法之间的异同点（这帮你建立知识网络）
4. **从攻击学防御**：理解攻击者为什么会用这种手法，就能猜到他们下一步会做什么

坚持每天写，60天后你会有一份属于自己的蓝队知识库。
这比什么教程都更有价值——因为是你自己的理解。
- 持续性胜过强度
- 每天进步一点点
- 120天后你会感谢现在努力的自己

## 🔬 Linux/Windows基线检查速查卡

**Linux一键安全检查脚本：**
```bash
#!/bin/bash
echo "===== 账号安全检查 ====="
echo "空口令账号："; awk -F: '$2==""{print $1}' /etc/shadow
echo "可登录账号："; grep -E ":/bin/(bash|sh)$" /etc/passwd
echo "===== SSH安全检查 ====="
grep -E "^(PermitRootLogin|PasswordAuthentication|Port)" /etc/ssh/sshd_config
echo "===== 密码策略检查 ====="
grep -E "^(PASS_MAX_DAYS|PASS_MIN_DAYS|PASS_MIN_LEN)" /etc/login.defs
echo "===== 防火墙检查 ====="
iptables -L -n 2>/dev/null || echo "iptables未配置"
echo "===== 审计检查 ====="
systemctl is-active auditd 2>/dev/null || echo "auditd未运行"
echo "===== 开放端口检查 ====="
ss -tlnp
```

**Windows一键安全检查（PowerShell）：**
```powershell
Write-Host "===== 账号安全检查 ====="
Get-LocalUser | Where-Object Enabled -eq $true
Write-Host "===== 密码策略检查 ====="
net accounts
Write-Host "===== 防火墙检查 ====="
Get-NetFirewallProfile | Select Name, Enabled
Write-Host "===== 共享检查 ====="
Get-SmbShare
Write-Host "===== 服务检查 ====="
Get-Service | Where-Object StartType -eq "Automatic" | Select Name
```

> 💡 把这些脚本加入你的cron/计划任务，每周自动跑一次，把结果发到你的邮箱。这就是"自动化基线监控"的雏形。

## 🚀 蓝队进阶之路：从"能检测"到"能防御"

蓝队能力成长一般经历三个阶段：

**第一阶段：能识别** — 看到告警知道大概是什么攻击
**第二阶段：能分析** — 能判断攻击的严重程度、攻击者的意图、攻击到了哪一步
**第三阶段：能防御** — 不仅知道怎么检测，还知道怎么从根本上防御这类攻击

问问自己：对于今天学的这类攻击/技术，你目前在哪个阶段？
如果还在第一阶段，不要焦虑——每一个高手都是从第一阶段走过来的。
目标很清晰：把这个主题学透到能给别人讲清楚的程度。

### 如何加速成长
1. **刻意练习**：不是泛泛地看，而是针对一个漏洞反复演练检测和防御
2. **输出倒逼输入**：尝试写一篇关于今天主题的公众号文章/内部分享（哪怕不发布）
3. **横向对比**：同类漏洞/攻击手法之间的异同点（这帮你建立知识网络）
4. **从攻击学防御**：理解攻击者为什么会用这种手法，就能猜到他们下一步会做什么

坚持每天写，60天后你会有一份属于自己的蓝队知识库。
这比什么教程都更有价值——因为是你自己的理解。
- 持续性胜过强度
- 每天进步一点点
- 120天后你会感谢现在努力的自己---

## 💎 蓝队实战精华：从日志中发现攻击者

今天的主题在实际工作中可能不会每天遇到，但一旦遇到就是大事。
作为蓝队，你要培养一种"直觉"——当你看着日志的时候，哪些是正常的、哪些是异常的。

### 培养"安全直觉"的方法
1. **先看100个小时的正常日志**。是的，就是纯看。知道"正常"长什么样，才能识别"异常"。
2. **搭建实验环境复现攻击**。看到了攻击的真实面貌，下次在日志中遇到就不会错过。
3. **关注"不和谐"**。比如：平时这个时间没人登录，今天凌晨3点有登录→可疑。
4. **关注"新东西"**。新进程、新服务、新计划任务、新网络连接→第一反应应该是怀疑。
5. **关注"量变"**。平时的日志量是100条/小时，突然变成1000条/小时→出事了。

### 分析未知攻击的思路
当你遇到一条看不懂的告警时：
1. 先确认基本事实：时间、来源IP、目标IP、涉及进程/服务
2. 搜索关键信息：把告警中的关键字放到Google/安全社区搜索
3. 找同类对比：搜索今天的主题+case study
4. 搭建复现：在隔离环境中复现看到的异常行为
5. 总结提炼：把这次的经验变成一条新的检测规则

> 🌟 记住：蓝队不需要比所有攻击者都聪明，你只需要比"昨天的自己"更聪明。
