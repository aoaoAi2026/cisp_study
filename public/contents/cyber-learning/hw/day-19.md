## 📖 核心知识讲解

### 一、探测类攻击总览

攻击者在动手之前，通常会先"踩点"——扫描你的系统，了解开放了什么端口、运行了什么服务、有哪些页面。所谓"知己知彼，百战不殆"。

```
攻击者的探测流程：
  第一步：网络扫描  → 哪些IP活着？
  第二步：端口扫描  → 开了哪些端口？
  第三步：服务识别  → 跑的是什么软件？什么版本？
  第四步：漏洞扫描  → 有没有已知漏洞？
  第五步：目录扫描  → 网站有哪些隐藏页面？
```

> 🎯 **蓝队要点**：扫描本身不造成直接伤害，但扫描是攻击的"前奏"。发现扫描行为后要提高警惕，因为真正的攻击可能紧随其后。

### 二、端口扫描识别

#### 流量特征（如何在Wireshark中识别）

**Nmap SYN扫描的特征：**
```
同一源IP → 在极短时间内 → 向大量不同目标端口 → 发送SYN包

tcp.flags.syn == 1 && tcp.flags.ack == 0
```

**在Wireshark中看到的样子：**
```
15:00:00.001  192.168.1.100 → 目标IP:22   [SYN]
15:00:00.002  192.168.1.100 → 目标IP:23   [SYN]  
15:00:00.003  192.168.1.100 → 目标IP:25   [SYN]
15:00:00.004  192.168.1.100 → 目标IP:80   [SYN]
...（连续数千个SYN包，间隔极短）
```

**Nmap的特征签名：**
- 默认SYN扫描时，源端口固定，Window Size通常是1024
- Nmap扫描结束后，通常不会完成三次握手（收到SYN+ACK后发RST）
- 不常用的端口被探测（如23/25/110等老旧服务端口）

#### 日志特征
```
# 防火墙日志中的扫描痕迹
Jun 12 15:00:00 kernel: DROP IN=eth0 SRC=192.168.1.100 DST=目标IP PROTO=TCP DPT=22
Jun 12 15:00:01 kernel: DROP IN=eth0 SRC=192.168.1.100 DST=目标IP PROTO=TCP DPT=23
Jun 12 15:00:02 kernel: DROP IN=eth0 SRC=192.168.1.100 DST=目标IP PROTO=TCP DPT=25
...
# 特征：同一源IP、密集TCP连接、不同目的端口
```

#### 常见扫描工具识别

| 工具 | 特征 | 日志/Wireshark中怎么辨认 |
|:---|:---|:---|
| **Nmap** | 有序扫描、TCP Window=1024 | 端口顺序递增（21,22,23,25...） |
| **Masscan** | 极快、随机端口顺序 | 端口是乱序的，每秒数千个SYN包 |
| **Zmap** | 只发SYN不等回复，单包扫描 | 几乎看不到回包 |
| **Rustscan** | 很快，Nmap风格 | 速度快但没有明显指纹 |

### 三、目录/文件扫描识别

#### Web日志中的目录扫描特征

```bash
# 目录扫描在Web访问日志中的经典特征：
192.168.1.100 - - [12/Jun/2026:15:00:00] "GET /admin HTTP/1.1" 404
192.168.1.100 - - [12/Jun/2026:15:00:01] "GET /wp-admin HTTP/1.1" 404
192.168.1.100 - - [12/Jun/2026:15:00:01] "GET /phpmyadmin HTTP/1.1" 404
192.168.1.100 - - [12/Jun/2026:15:00:02] "GET /backup.zip HTTP/1.1" 404
192.168.1.100 - - [12/Jun/2026:15:00:02] "GET /.git/config HTTP/1.1" 404
192.168.1.100 - - [12/Jun/2026:15:00:03] "GET /.env HTTP/1.1" 404

# 三大特征：
# 1. 同一IP密集请求
# 2. 大部分返回404（因为目录不存在）
# 3. 请求的都是敏感路径（admin/backup/.git/.env等）
```

#### 快速发现目录扫描的命令
```bash
# 统计产生最多404的IP
grep " 404 " access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head

# 查看某IP访问了哪些路径（判断是否在扫目录）
grep "可疑IP" access.log | awk '{print $7}' | sort -u

# 如果在扫目录，你会看到：
# /admin, /wp-admin, /phpmyadmin, /backup, /.git, /.env, /test, /tmp...
# 这些都不是正常用户会访问的路径
```

#### 常用扫描工具的User-Agent特征
```bash
# 目录扫描器的User-Agent常包含以下关键字：
DirBuster       → "DirBuster"
Gobuster        → "gobuster"
Dirb            → 通常没有独特UA，但会改
sqlmap          → "sqlmap"
Nikto           → "Nikto"
Nessus          → "Nessus"
AWVS            → "acunetix"
```

### 四、资产测绘/指纹识别

攻击者还会探测你的Web服务器类型、版本、框架：

```bash
# 资产测绘在日志中的表现：

# 1. 探测不存在的路径（看404页面的特征）
GET /wp-content/plugins/test.php → 确认是否为WordPress

# 2. 探测默认文件（判断服务器类型）
GET /phpinfo.php → 判断PHP版本
GET /manager/html → 判断是否为Tomcat
GET /console → 判断是否为Spring Boot

# 3. 探测特定header（识别框架和CDN）
# 通过HTTP响应中的Server/X-Powered-By/Src-Cookie判断
```

### 五、蓝队处置建议

| 扫描类型 | 威胁等级 | 推荐处置 |
|:---|:---|:---|
| 端口扫描（来自境外IP） | 🟡 P2 | 封禁IP，加入黑名单 |
| 端口扫描（来自境内扫描平台） | 🟢 P4 | 记录即可（很多是安全公司的常规扫描） |
| 目录扫描（大量404） | 🟡 P2 | 封禁IP，同时检查是否扫描到了敏感信息 |
| 目录扫描+发现敏感文件(200) | 🔴 P1 | 立即处置，排查信息泄露 |
| 资产测绘（探测框架版本） | 🟡 P3 | 记录，可能是后续定向攻击的前奏 |

## 📝 今日小结

扫描探测是攻击者的"侦察兵"。今天你学会了从流量和日志中识别它们——端口扫描看密集SYN，目录扫描看密集404。扫描本身不致命，但每一个严重的入侵，都是从一次扫描开始的。发现扫描后，你的雷达就要调高灵敏度了。

**记住今天的核心**：
- 端口扫描 = 同一IP + 密集SYN + 不同端口
- 目录扫描 = 同一IP + 密集404 + 敏感路径
- Nmap特征 = 端口有序、Window=1024、SYN→收到SYN+ACK→RST
- 扫描 = 攻击前的侦察，值得警惕
