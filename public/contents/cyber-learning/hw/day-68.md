## 📖 核心知识讲解



### 一、Burp Suite是什么？


Burp Suite是Web安全测试领域的"瑞士军刀"，渗透测试人员人手一个。

它本质是一个**拦截代理**：你在浏览器和目标服务器之间架了一个"收费站"，经过的所有流量你都能看、能改、能重放、能爆破。




### 二、核心模块详解


| 模块 | 功能 | 典型使用场景 | 快捷键 |
|--|--|------|---|
| Proxy | 拦截/查看/修改HTTP(S)流量 | 抓包分析、手动修改请求 | Intercept On/Off |
| Repeater | 手动重放单个请求 | 修改参数反复测试 | Ctrl+R发送到Repeater |
| Intruder | 自动化批量攻击（爆破） | 密码爆破、ID枚举、参数fuzz | Ctrl+I发送到Intruder |
| Scanner | 自动漏洞扫描 | 快速发现SQL注入/XSS等 | 右键→Scan |
| Decoder | 编解码工具 | URL/Base64/Hex编解码 | Ctrl+Shift+D |
| Comparer | 请求/响应对比 | 对比两次请求结果的差异 | 右键→Send to Comparer |
| Sequencer | Token随机性分析 | 分析Session/CSRF Token可预测性 | 右键→Send to Sequencer |




### 三、Proxy代理设置（基础但重要）


**步骤：**

1. Burp默认监听 `127.0.0.1:8080`

2. 浏览器设置代理为 `127.0.0.1:8080`（推荐用SwitchyOmega插件）

3. 安装Burp的CA证书（访问 `http://burpsuite` 下载），才能抓HTTPS

4. Intercept is on → 请求被拦截，可以修改后Forward或Drop






### 四、Intruder爆破模块（重点）


Intruder能帮你自动化地发送大量请求，每次替换不同的参数值。



**攻击类型（Attack Type）：**

- **Sniper（狙击手）**：一个位置，逐个试 → 适合单参数爆破

- **Battering Ram（攻城锤）**：多个位置，用同一个值 → 适合用户名=密码的场景

- **Pitchfork（草叉）**：多个位置，各用各的字典，一一对应 → 适合用户名+密码组合

- **Cluster Bomb（集束炸弹）**：多个位置，笛卡尔积 → 适合全面爆破（请求量大！）






### 五、常用Burp Extensions（扩展）


| 扩展名 | 功能 | 安装方式 |
|---|--|----|
| HaE (Highlighter and Extractor) | 高亮敏感信息，自动提取邮箱/IP/路径 | BApp Store |
| Autorize | 自动越权测试（低权限Cookie访问高权限接口） | BApp Store |
| Turbo Intruder | 比Intruder快100倍的高速爆破 | BApp Store |
| Logger++ | 增强的请求历史记录和分析 | BApp Store |
| J2EEScan | J2EE应用漏洞扫描增强 | BApp Store |
| ActiveScan++ | 增强主动扫描能力 | BApp Store |
| JSON Beautifier | JSON格式化显示 | BApp Store |
| XSS Validator | XSS验证工具 | BApp Store |




### 六、蓝队如何使用Burp Suite


蓝队用Burp主要不是为了攻击，而是：



1. **分析攻击Payload**：攻击者发的恶意请求长什么样？用Repeater放一遍观察响应

2. **验证漏洞**：安全扫描器报了个漏洞，用Burp手动验证一下是不是误报

3. **分析WAF拦截规则**：用Intruder发各种变形Payload，看哪些被拦哪些过了

4. **抓包分析恶意流量**：把pcap里的HTTP流量导入Burp分析

5. **测试自己的检测规则**：自己构造攻击请求，看SIEM/WAF能不能检测到






### 七、Burp高级技巧


**技巧1：Scope（范围）过滤**

- 在Target → Scope中设置只关注特定域名/IP的流量

- 避免被浏览器其他请求（如更新检查、统计请求）干扰



**技巧2：Match and Replace（自动替换）**

- Proxy → Options → Match and Replace

- 可以自动替换请求/响应中的内容，如：自动加一个Header、替换User-Agent等



**技巧3：Session Handling Rules（会话处理）**

- 用在需要自动登录的场景：检测到被踢出后自动发登录请求

- 配合Macro使用：录制登录流程→当Session失效时自动执行



**技巧4：上游代理链**

- Burp → 公司代理/其他工具 → 目标

- User options → Upstream Proxy Servers

## 🔬 BurpSuite蓝队实战场景

**场景1：分析钓鱼邮件的恶意链接**
- 在沙箱中打开可疑邮件→不点击链接
- 配好Burp代理后，手动输入或复制链接访问
- 观察请求链：初始URL→跳转链→最终落地页
- 分析落地页的HTML表单和JavaScript
- 记录所有外联域名/IP→加黑名单

**场景2：验证安全设备告警**
- SOC收到告警：某IP对业务系统发起SQL注入攻击
- 在Burp中打开告警附带的pcap或原始请求
- 用Repeater重放攻击请求→确认是否为真实攻击
- 如果是误报→分析为什么告警了（调整规则）
- 如果是真实攻击→确认攻击是否成功（看响应）

**场景3：测试WAF规则有效性**
- 写好一条新的WAF规则后→用Burp的Intruder测试
- 准备被拦截的Payload（应该被拦）+ 应放行的正常请求（不该被拦）
- 批量发送→分析WAF的拦截率和误报率
- 调整规则→再测试→直到拦截率100%且误报率0%

> 💡 Burp Suite不仅仅是一个渗透测试工具，它也是蓝队分析攻击、验证防御效果的神器。

## 🛡️ Burp Suite 常见问题排查

**问题1：抓不到HTTPS包**
- 确认已安装Burp CA证书到浏览器
- Firefox用户：证书安装到Firefox自身的证书存储（设置→隐私与安全→证书）
- Chrome用户：设置→隐私与安全→安全→管理证书→导入
- 确认代理设置正确：127.0.0.1:8080

**问题2：抓不到localhost的包**
- Chrome/Edge在localhost可能绕过代理
- 解决方法：使用 `http://127.0.0.1` 替代 `http://localhost`
- 或使用 Burp 内置浏览器（专业版功能）

**问题3：Intruder速度太慢**
- 社区版有限速（每秒约1个请求）
- 专业版可以在Resource Pool中设置并发线程数
- 或用Turbo Intruder扩展（免费且极快）

**问题4：请求返回"Connection Refused"**
- 检查目标是否在线
- 检查Burp的Intercept是否卡住了请求
- 检查Upstream Proxy设置

## 📖 蓝队Burp技巧：批量分析攻击流量

如果你有一批攻击请求需要分析，可以这样用Burp：
1. 把所有请求整理成HTTP格式的文本文件
2. 在Burp中通过 "Paste from clipboard" 或使用扩展导入
3. 在Proxy History或Repeater中逐个分析
4. 用Filter功能按关键词筛选（如搜索"SELECT"、"eval"、"cmd.exe"）
5. 把攻击特征总结到你的SIEM检测规则中
