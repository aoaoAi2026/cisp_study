## 📖 核心知识讲解

### 一、模拟值守说明

今天是28天速成计划中**最重要的一天**——你需要把前三周学到的所有技能整合起来，模拟一次真实的护网值守。

**模拟值守设置：**
- 时长：4小时（可分段完成，每次1小时）
- 场景：一家中型企业的Web系统护网期间
- 工具：你的虚拟机、笔记、浏览器
- 你需要扮演：蓝队一线值守分析师

### 二、值守工作台准备

在开始值守前，确保以下"工位"就绪：

```
你的值守工位：
□ 终端1：SOC告警面板（你将要处理的告警列表）
□ 终端2：Linux命令行（用于日志分析、情报查询）
□ 终端3：浏览器（打开微步在线、VirusTotal、CVE查询）
□ 终端4：笔记软件（记录研判结果、写工单）
□ 通讯工具：假设有企业微信/钉钉群（用于上报）
```

### 三、值守规则

1. **P0/P1告警必须在15分钟内响应**
2. **每条告警都需要有研判记录**（哪怕只是"确认误报"）
3. **关联告警要合并处理**（同一IP的多个告警视为一次攻击事件）
4. **每小时记录一次值守日志**
5. **发现问题要及时"上报"（在笔记中模拟）**

### 四、告警处理的标准格式

每处理一条告警，按以下格式记录：

```markdown
### 告警编号：ALERT-001
**时间**：2026-06-13 09:15
**设备**：IDS
**类型**：SQL注入
**来源IP**：45.33.32.156
**目标**：192.168.1.100:80

**研判过程**：
1. 情报核查：微步判定为恶意IP(红色标记)
2. 日志验证：在nginx访问日志中找到对应请求
3. 攻击确认：URL包含"UNION SELECT"，确认SQL注入尝试
4. 结果判断：状态码200，后续无异常登录日志 → 攻击尝试中，未完全成功

**等级**：P1 高危
**处置**：封禁IP + 排查注入点 + 记录工单
**状态**：处置中
```

## 🔧 实操任务

### 任务：逐条处理告警（4小时，可分段）

针对以上20条告警，逐条按以下流程处理：

1. **阅读告警** → 理解告警类型和基本判断
2. **情报核验** → 如果源IP数据明确给出，直接用；没给出的假设用微步查询
3. **研判分析** → 判断是否为真实攻击、攻击成功与否
4. **关联分析** → 这条告警和之前的是否有关联？（重点！）
5. **定级(P0-P4)** → 给出告警等级
6. **处置建议** → 写出具体处置措施
7. **记录工单** → 按照上面标准格式写研判记录

## 📝 今日小结

恭喜你完成了模拟值守演练！从今天的模拟中你会发现，值守不是一条一条警报警独立地看，而是要把所有的"点"连成"线"——当ALERT-001（SQL注入）和ALERT-008（文件上传）来自同一个IP时，这些就不再是孤立的告警，而是一个攻击事件的不同阶段。这种"关联分析"的能力，就是初级分析师和高级分析师的本质区别。

**总结今天的核心经验**：
- 同一IP的多条告警要合并看，串成攻击链
- 时间线是最好的朋友——按时间排列告警，攻击过程自然浮现
- 溯源到webshell后，一定要排查攻击者还做了什么
- 值守不是"看完告警=下班"，每条告警都要有记录有闭环
