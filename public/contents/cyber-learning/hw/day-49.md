## 等保2.0是什么？

等保全称"网络安全等级保护"，是中国的一项网络安全基本制度。
简单理解：国家规定不同重要程度的系统必须达到相应的安全保护水平。

等保2.0于2019年12月1日正式实施，相比等保1.0有以下关键变化：

| 变化维度 | 等保1.0 | 等保2.0 |
|:-|:-|:-|
| 保护对象 | 信息系统 | 信息系统+云计算+物联网+工控+移动互联 |
| 安全要求 | 安全+管理 | 安全+管理+可信验证 |
| 定级方式 | 自主定级 | 自主定级+专家评审+主管部门核准 |
| 测评周期 | 三级每年一次 | 三级每年一次（更严格） |
| 覆盖范围 | 传统IT | 传统IT+新技术（云大物移） |

对蓝队来说，等保2.0意味着：你的日常安全工作有了国家标准作为依据。
你做的安全配置、日志审计、应急响应，等保2.0里都有明确要求。

## 等保五个等级

等保将信息系统分为五个安全保护等级，数字越大越重要：

**第一级（自主保护级）**：一般信息系统，损害后对公民/组织有轻微损害。
   举例：企业内部论坛
   蓝队关注度：低

**第二级（指导保护级）**：一般重要系统，损害后对公民/组织有严重损害。
   举例：企业内部OA系统、邮件系统
   蓝队关注度：中（很多企业定为二级）

**第三级（监督保护级）**：重要系统，损害后对社会/国家有严重损害。
   举例：政府网站、金融系统、电商平台
   蓝队关注度：高（护网的主要目标都是三级系统）

**第四级（强制保护级）**：特别重要系统，损害后对国家有特别严重损害。
   举例：国家关键基础设施
   蓝队关注度：极高

**第五级（专控保护级）**：极端重要系统，损害后对国家有毁灭性损害。
   举例：绝密级国家秘密系统
   蓝队关注度：最高（但很少接触）

> 护网行动主要针对三级系统。你在护网中防守的资产至少是等保三级。

## 等保2.0的核心安全要求（八大类）

等保2.0将安全要求分为八大类（5个安全类+3个管理类）：

### 安全物理环境

机房安全：防盗、防火、防水、温湿度控制、电力供应。
蓝队关注点：物理安全出问题，所有网络安全措施都白搭。

### 安全通信网络

网络架构安全、通信传输安全、可信验证。
蓝队关注点：网络分段、加密传输、通信完整性。
这直接影响你能否检测到横向移动攻击。

### 安全区域边界

边界防护、访问控制、入侵防范、恶意代码防范。
蓝队关注点：防火墙策略、WAF部署、IDS/IPS。
这是蓝队的第一道防线——你在护网中大量工作都在这一层。

### 安全计算环境

身份鉴别、访问控制、安全审计、入侵防范、数据安全。
蓝队关注点：密码策略、权限管理、日志审计、漏洞扫描。
这是蓝队日常工作量最大的一层——"安全计算环境"="每天的安全运维"。

### 安全管理中心

系统管理、审计管理、安全管理、集中管控。
蓝队关注点：SOC建设、SIEM平台、告警集中管理。
这对应你在ELK和SOC中学到的知识。

### 安全管理制度

安全策略、管理制度、操作规程。

### 安全管理机构

岗位设置、人员配备、授权审批。

### 安全管理人员

人员录用、安全培训、离岗管理。

## 蓝队面试自测

**Q1: 描述一次完整的Web攻击检测流程**
A: 收到告警 -> ELK日志确认 -> 分析Payload -> 判断类型 -> 确认是否成功 -> 处置 -> 写报告。

**Q2: IDS和IPS的区别？**
A: IDS只检测不阻断（摄像头），IPS检测并阻断（门禁）。

**Q3: ATT&CK框架对蓝队的价值？**
A: 告诉我们攻击者"会做什么"和"怎么做"，蓝队据此在对应战术阶段设置检测点。

**Q4: 告警太多怎么处理？**
A: 分级处理——P0立即、P1今日内、P2周汇总，然后优化规则减少误报。

**Q5: WAF和IPS的区别？**
A: WAF专用于Web应用层（HTTP/HTTPS），IPS用于网络层（TCP/IP）。WAF理解SQL注入和XSS，IPS理解端口扫描和DDoS。

## 蓝队日常演练场景

### 场景：收到被攻破的内网告警

你是值班人员，凌晨2点收到告警：内网服务器192.168.1.50正向外部IP 45.x.x.x:4444发送数据。

处置步骤：
1. **立即确认**（30秒）：查看ELK中192.168.1.50的最近1小时日志
2. **分析判断**（2分钟）：发现该服务器在过去30分钟内有异常进程启动和端口扫描行为
3. **紧急处置**（5分钟）：隔离服务器（断网不关机）、通知组长、封禁45.x.x.x
4. **深入排查**（30分钟）：检查入侵路径（Weblogic漏洞？弱口令SSH？）
5. **写报告**（15分钟）：应急响应报告提交给安全负责人

### 场景：收到等保测评通知

公司下周要进行等保2.0二级测评，你作为蓝队需要准备：
1. 整理资产台账（所有服务器、网络设备、安全设备）
2. 检查安全基线（是否按要求配置了密码策略、访问控制、日志审计）
3. 准备制度文档（安全管理制度、应急预案、运维规范）
4. 检查安全设备运行状态（防火墙、WAF、IDS是否正常运行）

## 等保2.0常见误区


### 误区一：等保就是装几个设备

错。等保是"管理+技术"的综合要求，不仅要有防火墙、WAF等安全设备，
还要有完善的安全管理制度、操作规程和应急预案。
只装设备不建制度，测评很难通过。


### 误区二：等保二级很简单、不需要认真做

错。等保二级虽然要求比三级低，但同样需要：
- 密码策略（复杂度+定期更换）
- 访问控制（最小权限原则）
- 日志审计（至少保存6个月）
- 应急预案（能真正用起来）


### 误区三：等保测评过了就万事大吉

错。等保不是一次性工作，是持续的。测评通过只是起点，
日常的安全运维、安全事件处置、定期的安全自查是常态。
等保要求"持续改进"，不是"通过就行"。

## 实战思维训练营

### 蓝队的日常：从告警到处置的完整心智模型

每天早上打开监控平台，你的流程应该是：
1. 扫一眼大盘——有无P0/P1未处理告警
2. 浏览昨晚的告警汇总——凌晨的攻击最需要关注
3. 处理积压告警——从高危到低危逐条研判
4. 写日报——记录今天的重要发现
5. 持续监控——新告警随时可能弹出

这个流程看似简单，但做到位需要长期训练。

### 5秒判断法实战演练

以下每条告警请在5秒内做出判断：

1. WAF: request contains ' UNION SELECT ' -> [  ]
2. IDS: SYN packets to 100+ ports -> [  ]
3. Server log: /etc/passwd in URL -> [  ]
4. Firewall: outbound to 45.x.x.x:4444 -> [  ]
5. Auth log: 500 failed SSH attempts -> [  ]

答案：1-SQL注入攻击、2-端口扫描、3-文件包含攻击、4-疑似C2通信、5-暴力破解

### 攻击者心理画像

攻击者通常分为三类：
- **脚本小子**：用现成工具扫描，技术含量低但数量多，占告警的80%
- **APT组织**：目标明确、手段高级、长期潜伏，最难检测
- **红队/渗透测试**：合法的"攻击"，需要白名单排除

蓝队的精力分配：70%对付脚本小子，25%追踪可疑高级威胁，5%处理误报。

### Nmap常用命令速查

| 命令 | 用途 | 场景 |
|:---|:---|:---|
| nmap -F target | 快速扫描常用100端口 | 初步资产发现 |

### Wireshark过滤速查

| 过滤器 | 效果 |
|:---|:---|
| ip.addr == 192.168.1.1 | 只看某IP的流量 |
| tcp.port == 80 | 只看80端口 |
| http.request | 只看HTTP请求 |
| dns | 只看DNS |
| tcp.flags.syn == 1 && tcp.flags.ack == 0 | 只看SYN包 |
| !(arp || dns || icmp) | 排除噪音协议 |
| http contains "password" | 搜索HTTP中包含password |
| tcp.stream eq 5 | Follow TCP Stream #5 |

### BurpSuite速查

| 模块 | 用途 | 蓝队场景 |
|:---|:---|:---|
| Proxy | 拦截HTTP请求 | 分析攻击流量 |
| Repeater | 重放修改请求 | 验证攻击是否成功 |
| Intruder | 批量测试 | 测试WAF规则有效性 |
| Decoder | 编解码 | 解码攻击Payload |
| Comparer | 对比请求 | 对比正常和攻击请求差异 |

## 蓝队面试题库

**Q1: TCP三次握手是什么？为什么理解它对蓝队很重要？**

A: SYN -> SYN-ACK -> ACK。理解三次握手就能理解端口扫描（大量SYN到不同端口）、
DDoS（SYN Flood耗尽半连接队列）、C2通信（异常的TCP连接模式）。

**Q2: 你怎么理解纵深防御？**

A: 纵深防御 = 多层防护。边界防火墙是第一层、WAF是第二层、主机安全是第三层、
数据库审计是第四层。攻击者突破一层还有下一层——这就是纵深防御的价值。

**Q3: 护网期间发现一台服务器被黑了，你的第一步是什么？**

A: 隔离（断网不关机保留证据）> 通知组长 > 排查入侵路径 > 清除后门 >
修复漏洞 > 恢复上线 > 复盘报告。注意：千万不要关机！内存证据会丢失。

**Q4: 怎样区分自动化扫描和定向攻击？**

A: 自动化扫描特征：Payload是字典式（SQLMap的常见Payload）、
扫描速度快（<1秒一个请求）、同时扫描多个不相关目标。
定向攻击：Payload有针对性（针对特定应用）、
攻击节奏有策略（先探测再利用）、
关注特定高价值目标。

**Q5: 日志分析中最常用的Linux命令？**

A: grep（搜索关键字）、awk（字段提取）、sed（文本替换）、
sort + uniq -c（统计）、tail -f（实时监控）。
组合示例：cat access.log | grep "UNION SELECT" | awk '{print $1}' | sort | uniq -c | sort -nr

## 深度专题补充

### 等保合规的常见落地难点

1. **日志保存6个月**：日志量巨大，存储成本高。建议分级存储：热数据(最近7天SSD) → 温数据(30天普通硬盘) → 冷数据(6个月对象存储)
2. **最小权限原则**：运维习惯用root，推行困难。建议先做权限审计，再逐步收紧
3. **定期演练落实**：应急预案写好就束之高阁。建议每季度做一次桌面演练
4. **供应商管理**：第三方系统也在等保范围内，但对方不配合。建议合同约定安全条款

### 护网前的最后检查清单

- [ ] 资产台账已更新（确认没有幽灵资产）
- [ ] 高危漏洞已修复（或已制定修复计划+临时WAF规则）
- [ ] 弱口令已全部整改
- [ ] 安全设备正常运行（防火墙/WAF/IDS/ELK）
- [ ] 日志集中且至少保存30天

## 蓝队实战：经典案例分析

### 案例：一次成功的护网防守

某金融企业在护网期间成功抵御了所有红队攻击，经验总结：

**防守策略**：
1. 提前1个月做资产梳理（发现了12个遗忘的测试服务器，及时下线或加固）
2. 对所有对外服务做了漏洞扫描和修复
3. WAF规则从默认升级为自定义规则（针对性防御）
4. ELK大盘实时监控全网流量
5. 7x24小时双人值守（一人看监控，一人做分析）

**关键经验**：
- 资产梳理是护网的第一道防线
- WAF不能只用默认规则，需要根据业务定制
- 双人值守避免了单人疲劳导致的漏报
- 情报共享（与其他防守单位互通攻击IP）事半功倍

**教训**（尽管防守成功，仍有改进空间）：
- 每天日报花了太多时间（2小时），应该自动化日报生成
- 第一天的值守效率低（手忙脚乱），应该提前做模拟演练
- 有些告警处置慢了（因为需要层层上报），应该赋予一线分析师更多权限

## 工具进阶技巧

### Shell命令效率技巧

统计攻击IP排行榜：
```
cat access.log | grep -E "(UNION SELECT|<script>|\.\./etc/)" | awk '{print $1}' | sort | uniq -c | sort -nr | head -20
```

实时监控新出现的攻击：
```
tail -f access.log | grep --line-buffered -E "(SQL injection|XSS|command injection)"
```

批量封禁IP：
```
for ip in $(cat blocklist.txt); do
  iptables -A INPUT -s $ip -j DROP
done
```

### Python安全脚本速查

快速端口扫描：
```python
import socket
for port in range(1, 1025):
    s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    s.settimeout(0.1)
    if s.connect_ex(('127.0.0.1', port)) == 0:
        print(f"Port {port}: OPEN")
    s.close()
```

检查IP是否为恶意：
```python
import requests
# 使用AbuseIPDB等免费威胁情报API
resp = requests.get(f"https://api.abuseipdb.com/api/v2/check?ipAddress={ip}", 
    headers={"Key": "YOUR_API_KEY", "Accept": "application/json"})
if resp.json()["data"]["abuseConfidenceScore"] > 80:
    print(f"{ip} is malicious!")
```
