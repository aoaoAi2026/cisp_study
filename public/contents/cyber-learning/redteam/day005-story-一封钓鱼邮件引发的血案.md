## 🎯 目标背景

### 1.1 目标公司

这次的目标是一家 **互联网创业公司**。

| 项目 | 详情 |
|------|------|
| 公司规模 | 300人左右 |
| 业务类型 | 互联网金融 |
| 服务器数量 | 约100台 |
| 域环境 | 有，单域 |
| 安全防护 | 有WAF、有EDR、有专业安全工程师 |
| 外网攻击难度 | 较高（WAF挺严，漏洞不多） |

这家公司外网防护做得还不错。
我们扫了一周，只找到几个低危漏洞，没什么像样的突破口。

怎么办？
技术上行不通，那就从人身上下手。

> 💡 **社工攻击是什么？**（详见第66章：社会工程学基础）
> 社会工程学（Social Engineering），简称社工。
> 简单说就是利用人的心理弱点，来获取信息或者权限。
> 比如：钓鱼、伪装客服、打电话骗人、尾随进门...
> 这是一门很大的学问，后面第66-68章会详细讲。

### 1.2 为什么选择钓鱼？

为什么选择钓鱼，而不是继续啃技术漏洞？
原因有三个：

```
1. 外网确实没什么好打的
   - WAF规则很严，常见漏洞都打不进去
   - 业务系统用的是自研的，找不到通用漏洞
   - 0day又太贵，不值得

2. 互联网公司员工警惕性相对低？
   - 别看他们是搞互联网的
   - 其实很多人安全意识并不高
   - 特别是产品、运营、市场这些非技术岗
   - 技术岗也不一定就安全，开发人员往往更容易被"技术内容"钓到

3. 钓鱼成本低、见效快
   - 成本：几个域名、一台服务器、几天时间
   - 成功率：只要有10%的人上钩，就赚了
   - 一旦有一个高权限的人上钩，直接就打穿了
```

所以，我们决定：**搞一波鱼叉式钓鱼**。

> 什么是鱼叉式钓鱼（Spear Phishing）？
> 就是针对特定目标的定制化钓鱼，不是广撒网。
> 跟普通的群发钓鱼不一样，鱼叉式钓鱼是"量身定制"的，针对性非常强，成功率也高很多。

## 🔍 第一步：锁定目标人物

### 2.1 先找高价值目标

钓鱼不能瞎钓，得找高价值的目标。
什么是高价值目标？
就是权限高、能接触到核心系统的人。

我们先从情报收集入手：

```
第一步：先搞清楚公司的组织架构
- 从官网、招聘网站、LinkedIn上找
- 大概搞清楚有哪些部门
- 每个部门大概多少人

第二步：找出关键部门和关键人物
- IT部门：系统管理员、运维工程师 → 权限最高！
- 财务部门：财务经理、出纳 → 数据敏感
- 人事部门：HR经理 → 员工信息
- 高管：CEO、CTO、CFO → 权限高，邮箱值钱
- 开发团队：核心开发 → 代码仓库权限

第三步：按价值排序
1. 运维工程师 / 系统管理员 → 🔴 最高价值（有服务器权限）
2. 安全工程师 → 🔴 高价值（但是警惕性也高，风险大）
3. CTO / 技术总监 → 🟠 高价值
4. 财务经理 / 出纳 → 🟠 高价值
5. 核心开发工程师 → 🟠 高价值
6. HR经理 → 🟡 中高价值
7. 高管秘书/助理 → 🟡 中价值（接触信息多）
8. 普通员工 → 🟢 低价值（但是量大，也可能有惊喜）
```

最后，我们锁定了3个重点目标：

```
🎯 重点目标1：张工，运维工程师
- 职位：高级运维工程师
- 权限：服务器root权限、数据库权限、VPN权限
- 价值：🔴 极高
- 难度：🟠 中高（技术人，警惕性比普通人高）

🎯 重点目标2：李经理，财务经理
- 职位：财务经理
- 权限：财务系统权限、银行U盾、各种敏感数据
- 价值：🔴 极高
- 难度：🟡 中等（非技术岗，警惕性一般）

🎯 重点目标3：王总，CTO
- 职位：首席技术官
- 权限：几乎所有系统的最高权限
- 价值：🔴 极高
- 难度：🟠 中高（见多识广，但是忙，容易疏忽）
```

三个目标，我们准备了三种不同的钓鱼方案。
一个一个试，哪个成了算哪个。

> 💡 **情报收集怎么搞？**（详见第12章：信息收集 + 第66章：社会工程学）
> 怎么找到这些人的邮箱、职位、联系方式？
> 这就是情报收集的工作了。
> 第12章和第66章会详细讲各种信息收集的方法。

### 2.2 给张工做社工画像

我们先从张工开始——虽然他是技术人，警惕性高，但是他的权限也最高。
只要钓到他，基本就等于拿下内网了。

我们花了两天时间，给张工做了一个详细的社工画像：

```
👤 张工 社工画像

【基本信息】
- 姓名：张XX
- 性别：男
- 年龄：32岁
- 职位：高级运维工程师
- 工号：IT0023
- 入职时间：3年
- 部门：IT运维部

【联系方式】
- 邮箱：zhangxx@company.com
- 手机：138****1234（从某个泄露的简历里找到的）
- 微信：zhangxx_it（从他的技术博客签名里找到的）
- GitHub：github.com/zhangxx（能看到他的代码、邮箱、常用技术栈）

【技术背景】
- 工作经验：8年运维经验
- 擅长：Linux运维、Docker、K8s、MySQL、Redis...
- 常用工具：Ansible、Prometheus、Grafana、ELK...
- 等级：高级工程师，技术不错

【个人信息】
- 婚姻状况：已婚，孩子2岁（从朋友圈看到的）
- 老家：湖南
- 爱好：
  * 摄影（朋友圈经常发照片，用的是索尼A7M4）
  * 打篮球（每周都打）
  * 玩游戏（Steam账号找到了，主要玩FPS游戏）
  * 逛技术论坛（V2EX、掘金、知乎...）

【工作习惯】
- 上班时间：9:30 - 18:30，有时候加班
- 习惯用MacBook Pro
- 浏览器Chrome，书签栏里有很多运维相关的网站
- 常用邮箱就是公司邮箱
- 密码习惯：从历史泄露记录看，喜欢用 Name+生日+符号 的格式

【性格特点】
- 技术人，比较直接
- 对新技术感兴趣，喜欢折腾
- 做事认真，但是有时候粗心
- 警惕性比普通人高，但也不是特别高
- 看到跟技术相关的东西，容易点进去看

【最可能上钩的钓鱼类型】
1. 技术文章 / 工具分享 → 兴趣相关，最容易点
2. 系统告警 / 安全通知 → 工作相关，不得不看
3. 招聘 / 内推 → 运维人员跳槽频繁，容易感兴趣
4. 摄影相关 → 兴趣爱好
5. 游戏相关 → 兴趣爱好
```

画像做好了。
接下来就是设计钓鱼方案了。

> 🤔 你可能会问：这些信息都是哪来的？
> 说出来你可能不信，90%都是公开的：
> - 他的GitHub → 技术栈、邮箱、常用用户名
> - 他的技术博客 → 联系方式、工作经历
> - 他的朋友圈/微博/抖音 → 个人生活、兴趣爱好
> - 招聘网站的简历（如果他更新过的话）→ 详细工作经历
> - 各种社工库 → 历史密码、泄露的账号
> - 公司官网、新闻 → 职位、部门
>
> **一个有心人的话，拼凑出一个人的完整画像并不难。**

## 🎣 第二步：设计钓鱼方案

### 3.1 选择钓鱼主题

针对张工的特点，我们设计了三个备选钓鱼主题：

```
方案A：技术分享类（推荐度：⭐⭐⭐⭐⭐）
- 主题：《2024最新K8s安全最佳实践白皮书》
- 理由：
  * 张工作为运维，K8s肯定是他关心的
  * "白皮书""最佳实践"这种词，技术人很吃这套
  * 下载资料需要登录，顺理成章地要他的账号密码
- 优点：兴趣相关，点击欲望强
- 缺点：如果他不感兴趣，就不会点

方案B：安全告警类（推荐度：⭐⭐⭐⭐）
- 主题：【安全告警】您的服务器存在高危漏洞，请立即处理
- 理由：
  * 运维人员对安全告警非常敏感
  * 看到"高危漏洞""立即处理"，大概率会点进去看
  * 顺理成章地要求登录验证身份
- 优点：工作相关，不得不看
- 缺点：警惕性高的人可能会怀疑，容易被识破

方案C：内推招聘类（推荐度：⭐⭐⭐）
- 主题：【内推】字节跳动运维专家岗位，年薪80-120万
- 理由：
  * 运维跳槽率高，高薪岗位很有诱惑力
  * 内推邮件很常见，不容易引起怀疑
  * 投递简历需要登录/注册，可以钓密码
- 优点：诱惑力大
- 缺点：太像招聘广告，可能直接被忽略
```

经过讨论，我们选了**方案A + 方案B的结合版**。

什么意思？
就是主题是"安全告警"，内容是"K8s集群有高危漏洞需要处理"，
既跟他的工作相关（不得不看），又跟他的技术栈相关（他懂，不会觉得假）。

### 3.2 设计钓鱼邮件

主题定好了，接下来就是写邮件了。

写钓鱼邮件可是个技术活。
写得太假，一眼就被识破了。
写得太真，跟真的一模一样，那鱼儿就上钩了。

我们来来回回改了七八版，最终版是这样的：

```
📧 钓鱼邮件最终版：

发件人：安全运营中心 <security@company-security.com>
收件人：zhangxx@company.com
主题：【安全告警-紧急】您管理的K8s集群存在CVE-2024-XXXX高危漏洞，请立即处理

张工：

您好！

安全中心今日监测到，您负责的生产环境K8s集群（cluster-prod-01）
存在CVE-2024-XXXX高危漏洞（CVSS评分：9.8）。

该漏洞可导致未授权远程代码执行，攻击者可以直接接管整个K8s集群。
目前已有多家企业因此漏洞被入侵，风险极高。

请您在今天18:00前完成漏洞修复，具体修复步骤请参考内部知识库：
👉 https://kb.company-security.com/k8s-cve-2024-xxxx

登录说明：
- 使用您的公司邮箱账号和域密码登录
- 登录后可查看完整的漏洞详情和修复脚本
- 如遇登录问题，请联系安全中心：分机 8008

⚠️ 重要提醒：
1. 此漏洞为0day级别，请勿外传相关信息
2. 请务必在今日内完成修复，逾期未修复将触发安全合规通报
3. 修复过程中有任何问题，直接回复此邮件即可

辛苦了！

安全运营中心
XX科技有限公司
```

这封邮件的设计有几个"小心机"：

```
1. 发件人名字叫"安全运营中心"，看起来很正规
2. 域名是company-security.com，跟公司域名很像，不仔细看看不出来
3. 主题有"【安全告警-紧急】"，营造紧迫感
4. 提到了具体的集群名字（cluster-prod-01），真实感强
5. 提到了具体的CVE编号，很专业，技术人一看就觉得是真的
6. CVSS评分9.8，强调高危，增加紧迫感
7. "0day级别，请勿外传"，激发好奇心，也让他不敢随便问别人
8. "逾期未修复将触发安全合规通报"，利用人的恐惧心理
9. 落款是"安全运营中心"，还有分机号，看起来很正规
```

> 💡 **钓鱼邮件的心理学**
> 钓鱼邮件之所以能成功，利用的都是人的心理弱点：
> - **恐惧**："账号要被锁了！""要通报批评了！"
> - **好奇**："这是什么？""新工具？"
> - **贪婪**："涨工资了？""中大奖了？"
> - **信任**："领导发的""IT部门发的"
> - **紧迫感**："只有24小时！""马上就要过期了！"
> - **从众**："大家都在看""所有人都要做"
>
> 一封好的钓鱼邮件，往往会同时利用好几个心理弱点。

### 3.3 搭建钓鱼网站

光有邮件还不够，还得有个钓鱼网站。
就是邮件里那个"知识库"的链接。
点进去之后，是一个跟公司内部知识库长得一模一样的登录页面。

我们做了什么？
- 爬了公司真实的内部知识库登录页（从外网能访问到登录页）
- 一比一复刻，一模一样
- 只是把表单提交的地址改了，改成我们的服务器
- 用户输入账号密码之后，我们就能收到

```
🌐 钓鱼网站细节：

1. 页面外观
   - 跟真的登录页一模一样，包括logo、配色、布局、文字
   - 甚至连"忘记密码""联系管理员"这些链接都有
   - 点忘记密码会跳转到真的网站（更有迷惑性）

2. 域名
   - kb.company-security.com
   - 跟公司域名很像，粗看看不出来
   - 申请了SSL证书，HTTPS，有小锁头，看起来更正规

3. 功能
   - 输入账号密码 → 提交到我们的服务器
   - 提交之后，显示"系统繁忙，请稍后重试"
   - 然后自动跳转到真的知识库登录页
   - 用户以为是自己输错了，或者系统卡了
   - 一般会再输一遍，然后就登进去了（其实是跳转到真的网站了）
   - 根本发现不了自己被骗了

4. 后台
   - 我们有个简单的后台，能看到谁上钩了
   - 时间、IP、账号、密码，一目了然
   - 还能统计转化率什么的
```

搭建这个钓鱼网站花了我们大半天时间。
主要是页面仿得要像，不能一眼就被识破。

> - Gophish：开源钓鱼平台，管理钓鱼邮件和网站

> - Modlishka：反向代理钓鱼
> - ...还有很多

## 🚀 第三步：实施钓鱼

### 4.1 选择发送时机

什么时候发钓鱼邮件也是有讲究的。
发的时机不对，效果会差很多。

```
最佳钓鱼时间：
- 上午 9:30 - 10:30 → 刚上班，还没进入状态，容易点
- 下午 14:00 - 15:00 → 刚睡醒，迷迷糊糊的，也容易点
- 周五下午 → 心思已经不在工作上了，警惕性低

最差钓鱼时间：
- 周一早上 → 刚上班，一堆事，可能直接忽略
- 下班前一小时 → 急着下班，没心思看
- 深夜/凌晨 → 太假了，哪有半夜发工作邮件的
```

我们选在了**周二上午10点整**发送。

为什么选周二？
- 周一太忙了，一堆事，可能顾不上看
- 周二是一周中最"正常"的工作日
- 上午10点，刚处理完紧急的事，有空看邮件

而且，运维一般上午都会检查一遍系统、看看告警。

### 6.2 为什么钓鱼这么容易成功？

你可能会觉得：张工作为运维，怎么这么容易就上钩了？
是不是太蠢了？

其实不是。
张工一点都不蠢，技术还挺好的。
但是为什么还是中招了？

原因有几个：

```
1. 钓鱼邮件做得太真了
   - 域名很像
   - 内容专业（提到了具体的CVE、具体的集群名）
   - 发件人是"安全运营中心"，看起来很正规
   - 页面做得一模一样，根本看不出区别

2. 利用了人的心理弱点
   - 紧迫感："今天18:00前必须处理"
   - 恐惧感："逾期要通报批评"
   - 好奇心："0day漏洞，看看是什么"
   - 职业习惯：运维看到告警就想点进去处理

3. 密码复用
   - 一个密码走天下
   - 钓到一个，等于钓到所有

4. 安全意识还是不够
   - 虽然是技术人，但是安全意识不一定强
   - 很多技术人都觉得"这种事不会发生在我身上"
   - 结果往往就发生在他们身上
```

> 💡 **最安全的人是什么样的？**
> 不是技术最好的人，而是警惕性最高的人。
> 技术再好，只要有一次疏忽，就可能中招。
> 而钓鱼攻击，只要成功一次就够了。

### 6.3 怎么防钓鱼？

说了这么多攻击的方法，那怎么防御呢？

```
🛡️ 钓鱼防御指南：

【技术层面】
1. 邮件网关过滤
   - SPF、DKIM、DMARC验证
   - 反钓鱼、反垃圾邮件
   - 可疑链接自动重写/跳转提示

2. 网站防护
   - 重要系统启用双因素认证（2FA）
   - 登录异常检测（异地、新设备、新IP...）
   - 登录失败次数限制

3. 内网防护
   - 最小权限原则
   - 网络隔离（办公网、服务器区、核心区...）
   - 特权账号管理
   - 行为监测和异常检测

【人员层面】
1. 安全意识培训
   - 定期培训，告诉大家什么是钓鱼
   - 怎么识别钓鱼邮件、钓鱼网站
   - 遇到可疑情况怎么办

2. 钓鱼演练
   - 定期自己给自己的员工发钓鱼邮件
   - 测测大家的警惕性
   - 中招的人重点培训

3. 企业文化
   - 营造安全的企业文化
   - 鼓励员工报告可疑情况
   - 不要因为中招就惩罚（不然以后没人敢报告了）

【个人层面】
1. 收到邮件先别急着点
   - 看发件人对不对
   - 看域名对不对
   - 看内容合不合理
   - 有疑问直接打电话/当面问

2. 不要随便输入密码
   - 输入密码前先看域名对不对
   - 重要的账号用密码管理器，自动填充（假网站填不进去）
   - 开启双因素认证

3. 不同网站用不同密码
   - 不要一个密码走天下
   - 用密码管理器（1Password、Bitwarden、Keepass...）
```

> 防钓鱼，技术是基础，意识是关键。
> 再厉害的技术，也挡不住一个傻乎乎去点的人。

## 📚 案例讲解

### 案例1：五种常见的钓鱼类型

**1. 广撒网钓鱼（Phishing）**
- 就是大规模群发，能钓到谁算谁
- 比如："您的亚马逊账号异常""您中了大奖"
- 特点：量大，但是命中率低（一般0.1%-1%）
- 成本低，广种薄收

**2. 鱼叉式钓鱼（Spear Phishing）** ✅ 本章案例就是这种
- 针对特定目标的定制化钓鱼
- 事先收集目标的信息，量身定制邮件
- 特点：命中率高（20%-80%都有可能），针对性强
- 成本高，但是效果好

**3. 鲸钓（Whaling）**
- 专门针对高管、CEO、CFO这些"大鱼"的钓鱼
- 因为高管权限高、信息值钱，所以叫"钓鲸鱼"
- 要求更高，伪装得更像
- 商业邮件诈骗（BEC）很多都是这种

**4. 水坑攻击（Watering Hole）**
- 不是直接发邮件给目标
- 而是先攻陷目标常去的网站
- 然后在网站上挂马或者放钓鱼页面
- 目标访问的时候就中招了
- 就像在猎物常去的水坑里下毒一样，所以叫水坑攻击

**5. 语音钓鱼（Vishing，Voice Phishing）**
- 不是用邮件，而是用电话
- 假装成客服、警察、银行工作人员...
- 打电话骗你账号密码、验证码
- 也就是我们常说的"电信诈骗"

> 💡 **更多钓鱼类型？**（详见第66-68章）
> 钓鱼的类型太多了，这只是最常见的几种。
> 后面第66-68章会系统地讲社会工程学和钓鱼攻击。

### 案例2：钓鱼邮件的十个"小心机"

**一封"好"的钓鱼邮件，往往有这些特点：**

```
1. 紧迫感
   "24小时内必须修改密码，否则账号锁定！"
   利用人的恐惧心理，让人来不及思考就点了

2. 好奇心
   "绝密！公司的薪资表泄露了，看看有你吗？"
   利用人的好奇心，忍不住想点进去看

3. 重要性
   "【紧急】【重要】【CEO亲发】..."
   用各种标签强调重要性，让人不能忽视

4. 真实感
   提到你的名字、工号、部门、具体的项目...
   信息越具体，看起来越真

5. 伪装发件人
   发件名字改成"IT部门""安全中心""HR部门"...
   或者干脆伪装成你认识的人（领导、同事）

6. 仿冒域名
   跟真域名很像，粗看看不出来
   比如：company.com → company-security.com
   比如：taobao.com → ta0bao.com（0和o分不清）

7. 利益诱惑
   "您有一笔补贴待领取"
   "点击领取200元购物卡"
   利用人的贪婪心理

8. 从众心理
   "全体员工请在今日内完成XX操作"
   "已经有80%的同事完成了"
   大家都做，那我也做

9. 技术性
   （针对技术人员）
   提到具体的漏洞编号、技术术语
   看起来很专业，技术人容易信

10. 免责/威胁
    "请勿外传""禁止转发"
    "泄露后果自负"
    反而让人更想看，而且不敢问别人
```

### 案例3：怎么识别钓鱼邮件？

**收到邮件的时候，多留个心眼，检查这几点：**

```
1. 看发件人
   - 发件邮箱对不对？
   - 域名是不是官方的？
   - 有没有奇怪的拼写？
   - （注意：显示名可以随便改，要看真实邮箱地址）

2. 看收件人
   - 是不是发给你的？
   - 还是群发的？
   - 收件人里有没有你不认识的人？

3. 看内容
   - 有没有错别字、语法错误？
   - 语气是不是跟平时不一样？
   - 有没有奇怪的要求？（比如要密码、要验证码、要转账）

4. 看链接
   - 鼠标悬停在链接上，看看真实地址是什么
   - 域名对不对？
   - 是不是HTTPS？
   - （注意：短链接要小心，看不到真实地址）

5. 看附件
   - 附件是什么类型的？
   - .exe、.bat、.js、.docm、.xlsm... 这些要小心
   - 陌生的压缩包也要小心
   - 不要随便下载运行

6. 想一下
   - 这件事合理吗？
   - 为什么突然找我？
   - 为什么不用平时的方式通知？
   - （比如：公司一般用钉钉通知，突然用邮件了？）

7. 核实一下
   - 有疑问直接打电话问发件人
   - 或者当面问
   - 不要用邮件里的联系方式，自己找官方的
   - （比如：邮件里说"如有疑问请打8008"，你别直接打，自己查公司的官方电话）
```

> **记住一句话：**
> **凡是让你着急、让你害怕、让你占便宜的，都要多留个心眼。**
> 骗子最擅长的就是操纵你的情绪，让你来不及思考就上当。

### 案例5：一个反钓鱼成功的真实故事

**讲了这么多钓鱼攻击，我们来讲一个反钓鱼的故事。**

某公司的安全团队，为了提高员工的安全意识，搞了一次钓鱼演练。
他们伪装成HR，给全公司发了一封钓鱼邮件：

> "各位同事：
> 公司今年的体检报告出来了，请点击链接查看：
> http://xxx.com/health-report
> 请用工号和密码登录查看。"

结果呢？
- 全公司500人，有127人点了链接
- 有83人输入了账号密码
- 点击率25%，提交率17%

这个结果其实还不错，比平均水平好一点。
但是，有一个人的表现让所有人都意外了——
是公司的保洁阿姨。

保洁阿姨收到邮件之后，没有点链接。
而是拿着手机，找到了IT部门：

> "小伙子，我收到一封邮件，说能看体检报告。
> 但是我不会弄，你帮我看看是不是真的？
> 我怕是骗子。"

IT部门的人一听，都愣住了。
**全公司学历最高、技术最好的一群人，好多都中招了。**
**反而是保洁阿姨，警惕性最高，没有上当。**

为什么？
- 因为保洁阿姨知道自己不懂电脑，所以反而小心
- 而那些技术人员，觉得自己懂，反而大意了
- 越觉得自己不会中招的人，越容易中招

> 🎯 **这个故事告诉我们什么？**
> 1. 安全意识跟学历、跟技术水平，不一定成正比
> 2. 知道自己"不懂"，反而比觉得自己"都懂"更安全
> 3. 保持警惕，永远是最好的防御
> 4. 永远不要觉得"钓鱼这种事不会发生在我身上"

## 📝 本章小结

- 钓鱼攻击是红队的大杀器，技术打不通的时候，社工往往能突破
- 钓鱼不是随便发邮件，是一门结合心理学的技术活
- 社工画像越详细，钓鱼成功率越高（详见第66章：社会工程学）
- 一封好的钓鱼邮件，会利用人的多种心理弱点：恐惧、好奇、贪婪、信任...
- 高仿真的钓鱼网站，跟真的一模一样，很难分辨
- 鱼叉式钓鱼的成功率远高于广撒网（详见第67章：钓鱼攻击实战）
- 密码复用让钓鱼的危害放大N倍，一个密码通杀所有系统
- 技术再高的人，也可能被钓鱼——警惕性比技术更重要
- 防钓鱼：技术是基础，意识是关键，两者缺一不可
- 永远不要觉得"这种事不会发生在我身上"
