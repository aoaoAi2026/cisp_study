## 一、BUUCTF 介绍

BUUCTF 是由北京联合大学（Beijing Union University）信息安全专业团队维护的 CTF 在线训练平台，是国内最早一批 CTF 训练平台之一，也是很多 CTF 爱好者的启蒙平台。

### 1.1 什么是 BUUCTF

BUUCTF 是一个免费的 CTF 训练平台，提供了大量高质量的 CTF 题目。平台的题目涵盖 Web、Pwn、Reverse、Crypto、Misc 等多个方向，难度从新手到大师级都有，非常适合各个阶段的 CTF 学习者。

平台地址：https://buuoj.cn

### 1.2 BUUCTF 的特点

1. **题目数量多**：汇集了数百道 CTF 题目，很多来自真实比赛
2. **难度分级清晰**：从新手到大师，六级难度体系
3. **题目质量高**：经过筛选的优质题目，很多是历年比赛真题
4. **完全免费**：所有题目都可以免费做
5. **Writeup 丰富**：网上有大量 BUUCTF 题目的 Writeup
6. **新人友好**：有专门的新手区，适合入门

### 1.3 平台发展历史

- BUUCTF 最早由北京联合大学的团队创建
- 最初是为了校内训练和比赛而建
- 后来对外开放，逐渐成为国内知名的 CTF 训练平台
- 持续更新题目，保持活跃度

## 二、平台特点和题目分类

### 2.1 平台界面和功能

BUUCTF 平台的主要功能模块：

1. **题目训练**：按分类和难度浏览题目，在线解题
2. **排行榜**：用户积分排名
3. **比赛**：定期举办线上比赛
4. **Writeup**：用户分享的解题思路
5. **个人中心**：做题记录、积分、收藏等

### 2.2 题目分类

BUUCTF 的题目主要分为以下几个方向：

| 分类 | 题目数量 | 入门难度 | 说明 |
|------|---------|---------|------|
| **Web** | 最多 | 中等 | 最受欢迎的方向，题目类型丰富 |
| **Pwn** | 较多 | 高 | 二进制漏洞利用，难度较高 |
| **Reverse** | 较多 | 高 | 逆向工程，需要汇编基础 |
| **Crypto** | 中等 | 中等 | 密码学，数学要求较高 |
| **Misc** | 较多 | 低 | 杂项，入门友好，脑洞大 |

### 2.3 特色功能

1. **在线靶场**：题目环境在线启动，不需要本地搭建
2. **动态 Flag**：每道题的 Flag 是动态生成的，防止直接抄答案
3. **积分系统**：做题获得积分，等级提升有成就感
4. **题目收藏**：可以收藏感兴趣的题目
5. **Writeup 分享**：可以查看和分享解题思路

## 三、题目难度分级

BUUCTF 的题目分为六个难度等级，从易到难排列。

### 3.1 新手（Newbie）

**适合人群**：完全没有基础的纯新手

**题目特点**：
- 非常简单，几乎是送分题
- 主要考察基础概念和工具使用
- 有详细的提示
- 做了能快速建立信心

**代表题型**：
- 查看源代码找 Flag
- 简单的编码解码
- 基础的隐写术
- 最简单的 SQL 注入

**学习建议**：
- 新手可以从这个级别开始
- 每道题都要理解原理，不要只抄答案
- 做完这个级别对 CTF 有基本概念

### 3.2 入门（Easy）

**适合人群**：有一定基础，刚入门 CTF

**题目特点**：
- 考察单一知识点
- 套路比较固定
- 不需要太多技巧
- 是学习基础知识的好素材

**代表题型**：
- 简单的联合注入
- 反射型 XSS
- 基础的文件上传
- 古典密码
- 图片隐写

**学习建议**：
- 这个级别的题目要全部吃透
- 每道题都要理解漏洞原理
- 学会用工具，但也要手工验证

### 3.3 基础（Basic）

**适合人群**：掌握了基础，想要进阶

**题目特点**：
- 考察的知识点更深入
- 需要一定的绕过技巧
- 题目有一些变化，不能直接套模板
- 开始需要综合运用知识

**代表题型**：
- SQL 盲注
- 文件上传绕过
- 命令注入绕过
- 简单的反序列化
- 常见的 Crypto 攻击

**学习建议**：
- 这个级别是关键的过渡期
- 要开始总结题型和绕过技巧
- 培养独立解题的能力
- 不会的题可以先放一放，过段时间再做

### 3.4 进阶（Medium）

**适合人群**：有一定经验，想要提升

**题目特点**：
- 难度明显提升
- 需要综合运用多个知识点
- 有较多的绕过和变形
- 接近真实比赛中等难度

**代表题型**：
- 复杂的 SQL 注入（二次注入、堆叠注入）
- PHP 反序列化 POP 链
- Java 反序列化
- SSRF 高级利用
- 堆漏洞基础利用

**学习建议**：
- 这个级别需要大量练习
- 每道题都要深入研究
- 学会查找资料和阅读文档
- 开始关注新的漏洞和技术

### 3.5 困难（Hard）

**适合人群**：经验丰富，想要挑战

**题目特点**：
- 难度较高
- 思路比较巧妙
- 需要深厚的技术功底
- 相当于比赛中难题的水平

**代表题型**：
- 复杂的框架漏洞利用
- 高级堆利用技术
- 复杂的逆向分析
- 高强度密码学攻击
- 综合渗透场景

**学习建议**：
- 不要怕做不出来
- 认真看 Writeup，理解思路
- 学习新的技术和方法
- 多和别人交流讨论

### 3.6 大师（Master）

**适合人群**：顶尖选手，追求极限

**题目特点**：
- 非常难
- 通常来自高水平比赛的压轴题
- 需要很强的综合能力
- 可能涉及比较前沿的技术

**学习建议**：
- 量力而行，不要勉强
- 即使做不出来，看看 Writeup 也能学到很多
- 这个级别需要长期积累

## 四、Web 题目推荐和解析

BUUCTF 上 Web 题目数量最多，质量也很高。下面精选 10 道经典题目进行解析。

### 4.1 Easy 类型题目

#### 题目 1：[极客大挑战 2019] EasySQL

**难度**：Easy
**考点**：SQL 注入、万能密码

**题目描述**：一个简单的登录页面。

**解题思路**：
1. 打开题目，是一个登录框
2. 尝试输入 `admin' or '1'='1` 作为用户名
3. 密码随便填，点击登录
4. 登录成功，得到 Flag

**知识点**：
- 万能密码的原理
- 单引号闭合 SQL 语句
- OR 恒真条件

#### 题目 2：[ACTF2020 新生赛] Include

**难度**：Easy
**考点**：文件包含、php://filter

**题目描述**：一个文件包含的页面，提示 flag 在 flag.php 中。

**解题思路**：
1. URL 中有 `?file=xxx.php` 参数
2. 尝试直接包含 flag.php，看不到内容（PHP 被解析执行了）
3. 使用 php://filter 伪协议读取源码：
   `?file=php://filter/read=convert.base64-encode/resource=flag.php`
4. 得到 base64 编码的源码，解码后得到 Flag

**知识点**：
- 文件包含漏洞原理
- php://filter 伪协议的使用
- base64 编码读取 PHP 源码

#### 题目 3：[极客大挑战 2019] Upload

**难度**：Easy
**考点**：文件上传、前端验证绕过

**题目描述**：一个文件上传页面。

**解题思路**：
1. 尝试上传 .php 文件，被前端 JS 拦截
2. 禁用 JavaScript，或者修改前端代码
3. 或者用 Burp 抓包，修改文件后缀
4. 上传成功，得到上传路径
5. 访问 Shell，得到 Flag

**知识点**：
- 前端验证不可靠
- 抓包修改请求的方法
- 文件上传漏洞基础

### 4.2 Medium 类型题目

#### 题目 4：[强网杯 2019] 随便注

**难度**：Medium
**考点**：SQL 注入、堆叠注入

**题目描述**：一个查询页面，输入数据进行查询。

**解题思路**：
1. 输入 `1'` 报错，确认存在注入
2. 尝试联合注入，发现 select 被过滤
3. 尝试堆叠注入：`1'; show databases;-- -`
4. 成功执行，说明支持多语句
5. 查看表：`1'; show tables;-- -`
6. 发现有个表叫 `1919810931114514`
7. 查询表结构：`1'; desc `1919810931114514`;-- -`
8. 发现 flag 字段
9. 因为 select 被过滤，用预处理语句绕过：
   `1'; set @sql=concat('se','lect flag from `1919810931114514`'); prepare stmt from @sql; execute stmt;-- -`
10. 得到 Flag

**知识点**：
- 堆叠注入的条件和利用
- 关键字过滤的绕过方法
- MySQL 预处理语句
- 反引号包裹表名

> 💡 **深入理解：BUUCTF的六级难度体系——"从新手到大师的真实成长曲线"**
>
> BUUCTF把题目分为六个等级，这不是随便标的。
> 每个等级代表的不仅仅是"题目有多难"，而是"你需要掌握什么层次的知识"：
>
> ```
> ⭐ 一级（新手入门）—— "我知道这是什么"
>   例题：DVWA low级别的SQL注入
>   你需要的：知道SQL注入是什么、会用基本的注入语句
>
> ⭐⭐ 二级（初级阶段）—— "我能在简单环境用"
>   例题：有简单过滤的注入
>   你需要的：会绕过基本过滤（空格→/**/，or→||等）
>
> ⭐⭐⭐ 三级（中级阶段）—— "我能应对变化"
>   例题：盲注、报错注入、WAF绕过
>   你需要的：掌握多种注入技术，理解数据库特性
>
> ⭐⭐⭐⭐ 四级（高级阶段）—— "我能从零构造攻击"
>   例题：堆叠注入、二次注入、写Webshell
>   你需要的：不只懂漏洞，还要懂数据库管理和编程
>
> ⭐⭐⭐⭐⭐ 五级（专家阶段）—— "我能分析复杂场景"
>   例题：多条链组合利用、特殊环境下的绕过
>   你需要的：深入理解协议/语言底层机制
>
> ⭐⭐⭐⭐⭐⭐ 六级（大师阶段）—— "我能解决未知问题"
>   例题：新型过滤机制、独创利用方法
>   你需要的：不仅会用已知方法，还能创造新方法
> ```
>
> **成长的核心不是"刷了多少题"，而是"你从每个等级学到了什么"。**
> 很多人在三级卡住，原因往往不是"不会做题"，
> 而是"对数据库/编程语言/HTTP协议的底层理解不够深"。
> 到了这个阶段，不要急着刷更多题，而是回头补基础！

#### 题目 5：[网鼎杯 2018] Comment

**难度**：Medium
**考点**：SQL 注入、二次注入、XSS

**题目描述**：一个留言板系统，可以注册、发帖、评论。

**解题思路**：
1. 注册用户，发帖，评论
2. 在评论中发现 XSS？不对，是 SQL 注入
3. 用户名在评论时会被插入数据库，然后显示出来
4. 注册时用户名注入 SQL 代码，在评论时触发二次注入
5. 构造恶意用户名进行注入
6. 或者利用 SQL 写入文件 GetShell
7. 获取 Flag

**知识点**：
- 二次注入的原理
- 数据存储和使用时的上下文差异
- SQL 注入写文件

#### 题目 6：[SUCTF 2019] EasySQL

**难度**：Medium
**考点**：SQL 注入、堆叠注入、修改表结构

**题目描述**：一个查询页面，输入内容查询。

**解题思路**：
1. 测试注入，发现很多关键字被过滤
2. 测试发现可以堆叠注入
3. 但是 select 被过滤，无法直接查询
4. 思路：修改表结构
5. 猜测 SQL 语句：`SELECT $_POST['query'] || flag FROM Flag`
6. 利用 MySQL 的 || 操作符特性
7. 构造 payload：`*,1`
8. 这样 SQL 变成：`SELECT *,1 || flag FROM Flag`
9. 因为 || 是逻辑或，1 为真，所以结果都是 1
10. 但用其他方法：
    方法一：`1;set sql_mode=PIPES_AS_CONCAT;select 1`
    方法二：直接用堆叠查询
11. 最终获取 Flag

**知识点**：
- MySQL sql_mode 的作用
- PIPES_AS_CONCAT 模式
- 堆叠注入的灵活运用
- SQL 语句结构猜测

#### 题目 7：[CISCN2019 华北赛区] HackWorld

**难度**：Medium
**考点**：SQL 注入、布尔盲注、过滤绕过

**题目描述**：一个简单的查询页面，输入 ID 查询。

**解题思路**：
1. 输入 1 返回正常，输入 0 返回错误
2. 输入 `1 and 1=1` 被拦截，空格被过滤
3. 尝试 `1/*fuzz*/and/*fuzz*/1=1`，被拦截
4. 尝试括号绕过：`(1)and(1=1)` 不行
5. 尝试用异或：`0^1` 返回正常，`0^0` 返回错误
6. 确认可以用异或进行布尔盲注
7. 关键字也被过滤了，需要各种绕过
8. 编写脚本进行盲注
9. 逐步获取数据库名、表名、字段名、Flag
10. 过程中需要各种绕过技巧

**知识点**：
- 空格过滤的绕过
- 异或注入
- 布尔盲注脚本编写
- 各种关键字过滤绕过

## 五、其他方向题目简介

### 5.1 Pwn 方向

BUUCTF 上有不少高质量的 Pwn 题目，很多来自真实比赛。

**经典题目**：
- **ciscn_2019_n_1**：简单的栈溢出，入门题
- **pwn1_sctf_2016**：格式化字符串漏洞
- **jarvisoj_level2**：ret2libc 基础
- **ciscn_2019_es_2**：栈迁移
- **not_the_same_3dsctf_2016**：返回到指定函数

**学习建议**：
- 从栈溢出入门题开始
- 循序渐进，不要跳级
- 每道题都要自己写 exploit
- 理解原理比拿到 Flag 更重要

### 5.2 Reverse 方向

Reverse 题目数量也不少，适合逆向入门。

**经典题目**：
- **SimpleRev**：简单的逆向分析，字符串比较
- **内涵的软件**：基础逆向，算法分析
- **不一样的 Flag**：简单的编码算法
- **刮开有奖**：Windows 逆向
- **EasyRE**：入门级逆向

**学习建议**：
- 先学汇编语言基础
- 熟练使用 IDA 或 Ghidra
- 从简单的算法题入门
- 培养耐心和细致的品质

### 5.3 Crypto 方向

Crypto 题目从古典密码到现代密码都有。

**经典题目**：
- **基础题**：凯撒密码、维吉尼亚密码、栅栏密码
- **RSA 相关**：低加密指数、共模攻击、 Wiener 攻击
- **对称密码**：AES、DES 相关题目
- **编码题**：各种奇怪的编码

**学习建议**：
- 从古典密码开始，有趣又简单
- 学习 Python 脚本编写
- RSA 相关的题目要重点掌握
- 数学基础不好也没关系，多练就行

### 5.4 Misc 方向

Misc 题目种类繁多，脑洞很大。

**经典题目**：
- **基础隐写**：图片隐写、音频隐写
- **流量分析**：PCAP 文件分析
- **压缩包**：伪加密、暴力破解
- **脑洞题**：各种奇奇怪怪的题

**学习建议**：
- Misc 题很适合入门，有趣味性
- 多积累工具和经验
- 脑洞要大，思维要发散

## 六、BUUCTF 训练计划

> 💡 **深入理解：不同难度级别需要的不是"更努力"而是"不同的学习方法"**
>
> 很多人在BUUCTF上刷题是这样的：Easy题做完→Basic→Medium→
> 到了Hard题卡住→每天继续刷Hard→还是做不出来→放弃。
>
> 问题出在哪里？**因为你用刷Easy题的方法刷Hard题，就像用背乘法口诀的方法学微积分。**
>
> 不同难度级别对应的其实是**不同的思维模式**：
>
> ```
> Easy/Basic 级别 → 需要的是"识别模式"的能力
>   → 看到输入框+单引号报错 = SQL注入
>   → 看到文件上传 = 试试各种后缀绕过
>   → 这阶段的题"套路性强"，多做几道自然形成条件反射
>   → 学习方法：大量刷同类题目，建立"题感"
>
> Medium 级别 → 需要的是"组合知识"的能力  
>   → 看到select被过滤 + 可以堆叠 = 写预处理语句绕过
>   → 这阶段单一技巧不够，要把多个知识点"串"起来
>   → 学习方法：每道题画"攻击链图"，理清每一步用什么技术
>
> Hard 级别 → 需要的是"创造性思维"的能力
>   → 题目没有明显套路，需要你从零建立攻击思路
>   → 可能需要理解底层机制（协议、编码、序列化格式等）
>   → 学习方法：先别管题目，回去读相关技术的源码/规范文档
>
> Master 级别 → 需要的是"研究能力"
>   → 可能涉及未公开的绕过方法或新兴技术
>   → 不是"会不会做"，而是"能不能研究出来"
>   → 学习方法：追踪安全研究论文、学习漏洞发现方法论
> ```
>
> **卡在某一级的真正原因**：90%的情况不是因为"题太难"，
> 而是因为上一级的基础没打牢。
> 如果你卡在Medium，回去把Basic级别的同类题再做10道！

使用 BUUCTF 进行训练，制定一个合理的计划很重要。

### 6.1 新手入门计划（4周）

**目标**：完成所有新手和入门级题目，掌握 CTF 基础

**每周安排**：

| 周次 | 重点方向 | 目标题目数 | 学习内容 |
|------|---------|-----------|---------|
| 第1周 | Misc + Crypto 基础 | 10-15 道 | 各种编码、古典密码、简单隐写 |
| 第2周 | Web 基础 | 10-15 道 | SQL 注入基础、XSS、文件上传 |
| 第3周 | Web 进阶 + 其他 | 8-12 道 | 文件包含、命令注入、SSRF 基础 |
| 第4周 | 综合 + 复盘 | 5-10 道 | 回顾错题，整理笔记 |

### 6.2 进阶提升计划（8周）

**目标**：完成基础和大部分进阶题目，具备比赛能力

**训练方式**：
1. 每天至少做 1-2 道题
2. 每周做一次总结复盘
3. 每两周参加一次线上比赛
4. 定期回顾做过的题目

### 6.3 刷题技巧

1. **按分类刷**：集中时间刷同一类型的题，容易形成套路
2. **按难度刷**：从简单到难，循序渐进
3. **限时训练**：模拟比赛环境，限时做题
4. **错题回顾**：做错的题要反复看，确保真正理解
5. **写 Writeup**：自己写 Writeup 能加深理解
6. **交流讨论**：和小伙伴一起做题，讨论思路

### 案例二：Web 题目解析

**背景**：小军正在 BUUCTF 上刷 Web 题，遇到了一道很经典的 SQL 注入题。

**题目**：[强网杯 2019] 随便注

**详细解题过程**：

1. **初步探测**
   - 打开题目，是一个查询框
   - 输入 1，返回正常结果
   - 输入 1'，返回 MySQL 报错
   - 确认存在 SQL 注入

2. **判断注入类型**
   - 尝试 `1 order by 2`，返回正常
   - 尝试 `1 order by 3`，返回错误
   - 确认有 2 个字段
   - 尝试联合注入：`0 union select 1,2`
   - 返回："return preg_match('/select|update|delete|drop|insert|where|./i',"
   - 发现 select 等关键字被过滤了

3. **尝试堆叠注入**
   - 输入：`1'; show databases;-- -`
   - 成功返回数据库列表
   - 说明支持堆叠注入（多语句执行）
   - 但是 select 被过滤，不能直接查数据

4. **查询表结构**
   - `1'; show tables;-- -`
   - 发现两个表：`1919810931114514` 和 `words`
   - `1'; desc `1919810931114514`;-- -`
   - 发现这个表有一个 flag 字段
   - 这就是存储 Flag 的表

5. **绕过 select 过滤**
   
   方法一：预处理语句
   ```
   1'; set @sql=concat('se','lect flag from `1919810931114514`');
       prepare stmt from @sql;
       execute stmt;-- -
   ```
   
   方法二：修改表名
   ```
   1'; rename table words to words1;
       rename table `1919810931114514` to words;
       alter table words change flag id varchar(100);-- -
   ```
   然后查询 `1' or 1=1-- -` 就能得到 flag

6. **执行 Payload，得到 Flag**

**解题心得**：
- SQL 注入的形式多种多样
- 堆叠注入在特定条件下非常强大
- 绕过过滤的方法有很多，要灵活思考
- 做题时要发散思维，不要一条路走到黑

### 案例三：Crypto 入门

**背景**：小丽对密码学很感兴趣，在 BUUCTF 上刷 Crypto 题。

**学习 Crypto 的心得**：

1. **古典密码阶段**
   - 凯撒密码：最简单的移位密码
   - 维吉尼亚密码：多表替换密码
   - 栅栏密码：按行排列，按列读取
   - 培根密码：用 A/B 表示字母
   - 这些都很简单，理解原理就能做

2. **编码题阶段**
   - Base64、Base32、Base16
   - URL 编码、HTML 实体编码
   - Unicode 编码
   - 摩尔斯电码
   - 猪圈密码
   - 各种奇怪的编码
   - 经验：多看多记，用 CyberChef 工具

3. **现代密码阶段**
   - RSA：最常见的考点
     - 低加密指数攻击（e 很小）
     - 共模攻击（同一个 n，不同 e）
     - Wiener 攻击（d 很小）
     - 大整数分解（n 不太大或有特殊结构）
   - AES/DES：相对较少
   - 哈希算法：MD5、SHA 碰撞等

4. **工具推荐**
   - CyberChef：各种编码转换，网页版工具
   - Python：写解题脚本必备
   - SageMath：数学计算工具
   - RsaCtfTool：RSA 攻击工具集
   - yafu：大整数分解工具

**Crypto 学习建议**：
- 不要被数学吓到，很多题不需要高深的数学
- 掌握 Python 脚本编写
- 常见的攻击方式要熟练
- 多做题，积累经验和工具
- RSA 是重中之重，一定要掌握
