> **难度等级**：🟠 高等  
> **预计学习时间**：6小时

> **学习目标**：掌握红队作战的完整流程，从战前准备到战后复盘，能够独立策划和执行红队行动

## 71.1 战前准备阶段（人员/工具/情报/演练）

> **核心理念：红队作战 = 特种部队的军事行动**
>
> 把红队作战流程想象成特种部队的一次突袭行动，每一个阶段都对应特种作战的步骤：
>
> | 红队阶段 | 特种作战类比 | 做什么 |
> |---------|-------------|--------|
> | **战前准备** | 战前侦察+武器准备+人员训练 | 收集目标信息、搭建C2、免杀武器 |
> | **情报收集** | 侦察卫星+线人+无人机 | OSINT、资产发现、攻击面分析 |
> | **外部突破** | 找到防线薄弱点渗透进去 | Web打点、钓鱼、漏洞利用 |
> | **内网横向** | 进入敌方阵地后逐屋清扫 | 提权、凭据获取、横向移动 |
> | **目标达成** | 拿下指挥中心/炸毁目标 | 控制域控、获取核心数据 |
> | **战后清理** | 销毁行动痕迹 | 清除日志、删除后门、清理工具 |
>
> **最重要的认知**：红队行动中，**50%的成败取决于战前准备**。
> 很多新手一上来就想攻，结果四处碰壁。老红队花大量时间做准备——到攻击阶段反而轻松了。
> 磨刀不误砍柴工。

### 71.1.1 战前准备的重要性

> "磨刀不误砍柴工"——红队行动的成败，50%取决于战前准备。

很多新手红队一上来就急着想攻击，结果处处碰壁。而成熟的红队会花大量时间做准备，到攻击阶段就能势如破竹。

### 71.1.2 人员准备

#### 团队组建

**一支完整的红队需要以下角色：**

```
红队队长（1人）
├── 职责：统筹全局、制定策略、对外沟通
├── 要求：5年以上经验，多次护网经验
└── 技能：技术全面、管理能力强

侦察组（2-3人）
├── 职责：情报收集、攻击面分析
├── 要求：2年以上OSINT经验
└── 技能：信息收集、数据分析、社工

攻击组（3-5人）
├── 职责：外部突破、漏洞利用
├── 要求：3年以上渗透测试经验
└── 技能：Web渗透、漏洞利用、编程

后渗透组（2-3人）
├── 职责：内网渗透、提权、横向
├── 要求：3年以上内网渗透经验
└── 技能：内网渗透、域渗透、提权

支撑组（1-2人）
├── 职责：C2运维、免杀、工具开发
├── 要求：2年以上安全开发经验
└── 技能：免杀、编程、运维
```

#### 人员分工原则

1. **专精与全面结合**：每个人有专长领域，但也要了解其他领域
2. **冗余备份**：关键岗位至少有2人能胜任
3. **沟通顺畅**：建立高效的内部沟通机制
4. **轮班安排**：护网期间需要轮班，保证24小时有人值守

## 71.2 情报收集与分析（OSINT/社工/技术）

### 71.2.1 情报收集概述

**情报是红队行动的基础。** 对目标了解越多，攻击的成功率就越高。

**情报收集的目标：**
1. 了解目标的全貌（组织、业务、技术）
2. 发现可利用的弱点
3. 规划攻击路径
4. 识别高价值目标

## 71.3 攻击路径规划（多条攻击路径设计）

### 71.3.1 为什么要规划多条路径

> "不要把鸡蛋放在一个篮子里。"

红队行动中，任何一条攻击路径都可能失败：
- 漏洞被修复了
- 钓鱼被发现了
- C2被封了
- 权限被清除了

所以必须准备多条攻击路径，主攻一条，备用多条。

### 71.3.2 攻击路径设计原则

1. **多样性**：不同路径使用不同技术手段
2. **独立性**：路径之间尽量不相互依赖
3. **优先级**：按成功率和价值排序
4. **隐蔽性**：优先选择隐蔽性好的路径
5. **可行性**：基于情报评估可行性

### 71.3.3 常见攻击路径模板

#### 路径一：Web漏洞 → 内网渗透

```
外部Web系统漏洞利用
    ↓
获取WebShell
    ↓
权限提升（内核漏洞/配置不当）
    ↓
内网信息收集
    ↓
凭据获取（Mimikatz抓密码/Hash）
    ↓
横向移动（PtH/WMI/PsExec）
    ↓
域控攻击（DCSync/ZeroLogon/黄金票据）
    ↓
拿下域控 → 核心系统
```

#### 路径二：钓鱼 → 内网渗透

```
钓鱼邮件投递
    ↓
目标点击/提交/执行
    ↓
获取初始Shell/账号
    ↓
权限提升
    ↓
内网横向移动
    ↓
域控攻击
    ↓
核心目标
```

#### 路径三：VPN/边界设备突破

```
VPN弱口令/漏洞利用
    ↓
进入内网
    ↓
内网信息收集
    ↓
横向移动
    ↓
域控
```

#### 路径四：供应链攻击

```
分析供应商列表
    ↓
选择防护较弱的供应商
    ↓
攻陷供应商
    ↓
利用供应商与目标的连接通道
    ↓
进入目标内网
    ↓
渗透核心
```

#### 路径五：社会工程学

```
关键人员识别
    ↓
深度信息收集
    ↓
社会工程学攻击
    ↓
获取凭据/权限
    ↓
进入内网
```

### 71.3.4 攻击路径评估

对每条路径进行评估：

| 评估维度 | 说明 | 权重 |
|----------|------|------|
| **成功率** | 这条路径成功的概率 | 高 |
| **隐蔽性** | 被发现的概率 | 高 |
| **时间成本** | 需要多少时间 | 中 |
| **技术难度** | 实现的技术门槛 | 中 |
| **价值** | 成功后能获得什么 | 高 |
| **风险** | 失败后的影响 | 中 |

**评估示例：**

| 路径 | 成功率 | 隐蔽性 | 时间 | 难度 | 价值 | 风险 | 总评 |
|------|--------|--------|------|------|------|------|------|
| Web漏洞 | 中 | 低 | 中 | 中 | 高 | 中 | ⭐⭐⭐ |
| 钓鱼 | 高 | 中 | 低 | 低 | 高 | 低 | ⭐⭐⭐⭐⭐ |
| VPN突破 | 中 | 中 | 低 | 低 | 中 | 低 | ⭐⭐⭐⭐ |
| 供应链 | 中 | 高 | 高 | 高 | 高 | 低 | ⭐⭐⭐⭐ |
| 社工 | 高 | 高 | 中 | 中 | 高 | 低 | ⭐⭐⭐⭐ |

## 71.4 外部打点阶段（Web系统/边界设备/钓鱼）

### 71.4.1 外部打点概述

**外部打点**是红队行动的第一步，目标是获取目标的初始访问权限（第一个Shell或第一个账号）。

这是最关键的阶段——如果能成功突破边界，后续才有意义。

### 71.4.2 Web系统攻击

#### 目标选择

从情报收集阶段识别的Web系统中，选择优先攻击目标：

**优先攻击：**
- 使用过时CMS的系统
- 有已知漏洞的中间件
- 功能复杂的系统（攻击面大）
- 测试/开发环境（通常防护较弱）
- 老旧系统（缺少维护）

#### 攻击方法

```bash
# 1. 信息收集
#   指纹识别
whatweb https://target.com
wappalyzer分析

#   目录扫描（注意控制速度）
dirsearch -u https://target.com -t 5

#   子页面发现
#   检查robots.txt, sitemap.xml
#   检查/.git/, /.svn/, /backup/

# 2. 漏洞扫描
#   已知漏洞检查
nuclei -u https://target.com

#   CMS专项扫描
#   WordPress: wpscan
#   Drupal: droopescan
#   Joomla: joomscan

# 3. 漏洞利用
#   SQL注入
sqlmap -u "https://target.com/page?id=1" --batch

#   文件上传
#   尝试上传WebShell

#   命令执行
#   利用RCE漏洞

#   文件包含
#   LFI/RFI利用
```

#### Web打点成功的关键

1. **全面收集信息**：不要只看主页，要深入每个子页面
2. **关注老旧系统**：老旧系统通常防护较弱
3. **关注功能点**：搜索、上传、登录等功能容易有漏洞
4. **关注参数**：URL参数、POST参数、Cookie
5. **注意隐蔽**：控制扫描频率，避免触发WAF

### 71.4.3 边界设备攻击

**常见的边界设备：**
- VPN设备（SSL VPN、IPSec VPN）
- 防火墙
- 路由器
- 负载均衡
- 邮件网关

**攻击方法：**

```bash
# 1. VPN攻击
#   弱口令测试
#   已知漏洞利用（如FortiOS、Pulse Secure等）
#   配置不当利用

# 2. 防火墙/路由器
#   默认口令测试
#   管理后台暴露
#   已知漏洞利用

# 3. 邮件网关
#   漏洞利用
#   配置不当
#   信息泄露
```

## 71.9 痕迹清理与权限维持（后门/驻留）

> **OPSEC 核心概念：进得去，更要'走得干净'**
>
> 红队作战中有一个极其重要的概念叫 **OPSEC（Operational Security，操作安全）**。
> 它的核心思想很简单：**做任何事之前，先想清楚会不会留下痕迹**。
>
> | OPSEC原则 | 类比 | 反面教材 |
> |-----------|------|---------|
> | 最小化暴露 | 小偷只打开需要打开的抽屉 | 来了先遍历所有目录，日志爆表 |
> | 混入正常行为 | 间谍说当地口音、穿当地衣服 | 凌晨3点从没人的服务器大量下载文件 |
> | 及时清理 | 凶手消除指纹 | 工具放在桌面上、后门注册表不删 |
> | 通信安全 | 对讲机用暗语 | 明文传输Mimikatz抓到的密码 |
>
> **OPSEC 贯穿所有阶段**，不仅仅在痕迹清理时才想起。从战前准备（选什么IP、用什么域名）到攻击执行（用什么命令、怎么隐藏），每个决定都要考虑：这一步会不会暴露我？

### 71.9.1 权限维持

权限维持在后面第74章会详细介绍，这里简要说明：

**Windows权限维持方法：**
- 注册表Run键

- 启动项
- 映像劫持

**域环境权限维持：**

- AdminSDHolder
- DSRM后门
- 委派后门

### 71.9.2 痕迹清理

**痕迹清理的原则：**
1. **选择性清理**：只清理红队操作产生的痕迹
2. **不要全清**：全清反而可疑
3. **注意顺序**：先清理应用日志，再清理系统日志
4. **留存后门**：清理痕迹但保留后门

**Windows日志清理：**

```bash
# 清理安全日志
wevtutil cl Security

# 清理系统日志
wevtutil cl System

# 清理应用程序日志
wevtutil cl Application

# 清理PowerShell日志
wevtutil cl "Windows PowerShell"

# 清理RDP登录日志
# 删除Default.rdp
del %userprofile%\documents\Default.rdp
# 清理注册表
reg delete "HKCU\Software\Microsoft\Terminal Server Client" /f
```

**Web日志清理：**

```bash
# 清理Apache日志
# /var/log/apache2/access.log
# /var/log/apache2/error.log

# 清理Nginx日志
# /var/log/nginx/access.log
# /var/log/nginx/error.log

# 选择性清理（只删自己的IP）
sed -i '/attacker_ip/d' /var/log/apache2/access.log
```

**注意：痕迹清理要谨慎，不要影响业务。**
