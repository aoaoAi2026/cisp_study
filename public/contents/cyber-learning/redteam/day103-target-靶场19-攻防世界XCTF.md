## 一、攻防世界（XCTF）介绍

攻防世界（XCTF）是由国家创新与发展战略研究会网络空间战略研究中心指导，南京赛宁信息技术有限公司运营的全国性网络安全攻防竞赛训练平台。它不仅是一个 CTF 训练平台，更是 XCTF 全国联赛的官方平台，在国内 CTF 圈具有很高的知名度和影响力。

### 1.1 什么是攻防世界

攻防世界（https://adworld.xctf.org.cn）是国内最具影响力的 CTF 平台之一，提供了丰富的网络安全训练题目和真实的竞赛体验。平台汇集了来自全国各地的优秀题目，涵盖 Web、Pwn、Reverse、Crypto、Misc、Mobile 等多个方向。

与其他 CTF 平台相比，攻防世界的最大特点是与 XCTF 联赛深度绑定，平台上的很多题目都来自真实的 XCTF 联赛，题目质量高、难度梯度完整。

### 1.2 攻防世界的特点

1. **官方背景**：有官方指导和支持，权威性高
2. **题目质量高**：很多题目来自真实比赛，经过精心设计
3. **难度梯度完整**：从新手到高手，各个水平都能找到适合的题
4. **竞赛体系完善**：与 XCTF 联赛结合，有完整的赛事体系
5. **社区活跃**：有大量的 Writeup 和讨论
6. **影响力大**：很多 CTF 战队和选手都在这个平台训练

### 1.3 适合人群

- CTF 入门者（新手区题目友好）
- 有一定基础想要提升的选手
- 准备参加 XCTF 等大型比赛的战队
- 网络安全从业者（技术提升）
- 高校网络安全社团

## 二、XCTF 联赛介绍

XCTF 联赛是全国最高水平的网络安全攻防赛事之一，由一系列分站赛和总决赛组成。

### 2.1 XCTF 联赛体系

XCTF 联赛采用"分站赛 + 总决赛"的模式：

```
XCTF 联赛
  ├── 分站赛（每年多场）
  │    ├── 北京站
  │    ├── 上海站
  │    ├── 广州站
  │    ├── 成都站
  │    ├── 武汉站
  │    └── ...（每年站点不同）
  │
  ├── 国际赛（邀请赛）
  │    └── 邀请国内外顶尖战队参加
  │
  └── 总决赛
       └── 分站赛优胜队伍晋级
```

### 2.2 比赛形式

XCTF 比赛通常采用以下形式：

**CTF 夺旗赛（Jeopardy）**：
- 最常见的比赛形式
- 多道题目，解题得分
- 按积分排名
- 题目类型：Web、Pwn、Reverse、Crypto、Misc 等

**攻防赛（Attack & Defense）**：
- 每支队伍有自己的靶机
- 既要攻击别人的靶机，也要防守自己的靶机
- 更贴近真实攻防对抗
- 通常在总决赛或高级别赛事中使用

**混合赛制**：
- 结合多种比赛形式
- 更全面地考察战队能力

### 2.3 参赛意义

参加 XCTF 联赛的意义：

1. **技术提升**：与全国顶尖选手同台竞技，快速提升技术
2. **人脉积累**：认识更多圈内的朋友和战队
3. **履历加分**：CTF 比赛成绩在求职时很有分量
4. **奖金奖励**：很多比赛有丰厚的奖金
5. **荣誉认可**：获得好名次是对技术能力的肯定
6. **职业发展**：很多安全企业通过比赛发掘人才

## 三、平台分区

攻防世界平台分为几个主要区域，各有不同的定位和特点。

### 3.1 新手区

**定位**：为完全没有基础的新手准备的入门区域

**特点**：
- 题目难度低，入门友好
- 有详细的题目提示
- 覆盖各个方向的基础题型
- 适合零基础快速入门

**题目数量**：数十道入门题

**适合人群**：
- 刚接触 CTF 的纯新手
- 想了解 CTF 是什么的爱好者
- 高校新生

**学习建议**：
- 先把新手区的题都过一遍
- 每道题都要理解原理
- 做完新手区再去高手区挑战

### 3.2 高手区

**定位**：为有一定基础的选手准备的进阶区域

**特点**：
- 题目难度中等偏上
- 很多来自真实比赛的题目
- 题型丰富，质量高
- 有明确的分值和难度区分

**题目数量**：数百道题目

**题目分值**：
- 1 分题：入门级，比较简单
- 2 分题：中等难度，需要一定基础
- 3 分题：较难，需要扎实的技术
- 4 分题：很难，通常是比赛中的难题
- 5 分题：超难，一般是比赛压轴题

**适合人群**：
- 有一定 CTF 基础的选手
- 准备参加比赛的战队成员
- 想要提升技术的安全从业者

### 3.3 社区区

**定位**：用户分享和社区互动区域

**特点**：
- 用户可以上传自己出的题
- 有各种 Writeup 分享
- 可以讨论题目解法
- 有比赛报名和资讯

**功能**：
- 题目分享
- Writeup 发布
- 比赛信息

## 四、Web 方向经典题目解析

### 题目 1：Web_php_include

**难度**：1 分（入门）
**考点**：文件包含、PHP 伪协议

**题目描述**：
一个简单的文件包含页面，提示"试试来看phpinfo？"

**解题思路**：
1. 打开题目，URL 中有 `?page=xxx.php`
2. 尝试包含 phpinfo() 页面
3. 发现可以文件包含
4. 尝试使用 php://filter 读取源码：
   `?page=php://filter/read=convert.base64-encode/resource=index.php`
5. 得到 base64 编码的源码
6. 解码分析源码，找到 flag 的位置
7. 直接包含 flag 文件，或者从源码中找到 flag

**知识点**：
- 文件包含漏洞原理
- php://filter 伪协议
- base64 编码读取 PHP 源码
- 信息收集的重要性

### 题目 2：NewsCenter

**难度**：1 分（入门）
**考点**：SQL 注入、联合注入

**题目描述**：
一个新闻搜索页面，可以搜索新闻。

**解题思路**：
1. 打开题目，有一个搜索框

3. 确认存在 SQL 注入
4. 使用 `order by` 判断字段数：3 个字段
5. 使用联合注入：`0 union select 1,2,3`
6. 发现 2 和 3 的位置有回显
7. 查数据库名、表名、字段名
8. 最终查到 flag

**知识点**：
- SQL 注入基础
- 联合注入步骤
- information_schema 使用

### 题目 3：Web_php_unserialize

**难度**：2 分（中等）
**考点**：PHP 反序列化、__wakeup 绕过

**题目描述**：
一个 PHP 反序列化的题目，给出了源码。

**源码分析**：
```php
class Demo {
    private $file = 'index.php';
    public function __construct($file) {
        $this->file = $file;
    }
    function __destruct() {
        echo @highlight_file($this->file, true);
    }
    function __wakeup() {
        if ($this->file != 'index.php') {
            $this->file = 'index.php';
        }
    }
}
```

**解题思路**：
1. 分析源码，Demo 类的 __destruct 会读取文件
2. 但是 __wakeup 会把 file 重置为 index.php
3. 需要绕过 __wakeup
4. 绕过方法：序列化字符串中对象属性个数比实际多
5. 构造 payload：
   - 正常序列化：`O:4:"Demo":1:{s:10:"Demofile";s:8:"flag.php";}`
   - 修改属性数：`O:4:"Demo":2:{s:10:"Demofile";s:8:"flag.php";}`
6. 注意 private 属性的序列化格式（类名前后有 %00）
7. 把 payload 传给页面，得到 flag

**知识点**：
- PHP 反序列化漏洞
- __wakeup 魔术方法绕过
- private/protected 属性的序列化格式
- 反序列化漏洞的基本利用思路

### 题目 4：Web_python_template_injection

**难度**：2 分（中等）
**考点**：SSTI、Python 模板注入

**题目描述**：
一个 Python Web 应用，输入的内容会被渲染到页面上。

**解题思路**：
1. 测试输入 `{{7*7}}`，页面返回 49
2. 确认存在模板注入（SSTI）
3. 判断模板引擎：Jinja2（Python）
4. 利用 Python 对象继承关系获取敏感信息
5. 构造 payload 执行命令：
   - 获取基类：`{{''.__class__.__mro__[2].__subclasses__()}}`
   - 寻找可以执行命令的类（如 os._wrap_close、Popen 等）
   - 构造 payload 读取 flag
6. 最终 payload 示例：
   `{{''.__class__.__mro__[2].__subclasses__()[59].__init__.func_globals['linecache'].__dict__['os'].popen('cat flag').read()}}`
7. 或者使用更简单的 payload：
   `{% for c in [].__class__.__base__.__subclasses__() %}{% if c.__name__=='catch_warnings' %}{{ c.__init__.__globals__['__builtins__'].eval("__import__('os').popen('cat /flag').read()") }}{% endif %}{% endfor %}`

**知识点**：
- SSTI（服务端模板注入）原理
- Jinja2 模板注入利用
- Python 对象继承关系
- 从模板注入到 RCE 的利用链

### 题目 5：Web_java_xxe

### 题目 6：Web_SSRF

**难度**：3 分（较难）
**考点**：SSRF、内网探测、Redis 攻击

**题目描述**：
一个图片加载功能，可以从 URL 加载图片。

**解题思路**：
1. 测试 SSRF：
   - 输入 `file:///etc/passwd`，看能否读取
   - 输入 `http://127.0.0.1/`，看能否访问
2. 确认存在 SSRF
3. 探测内网：
   - 扫描内网存活主机
   - 扫描端口
4. 发现内网有 Redis 服务（6379 端口）
5. 利用 SSRF 攻击 Redis：
   - 方式一：写 Webshell
   - 方式二：写 SSH 公钥
   - 方式三：写计划任务
6. 使用 Gopher 协议构造 Redis 命令
7. 成功写入 Shell，获取 flag

**知识点**：
- SSRF 漏洞原理和利用
- 内网探测方法
- Redis 未授权访问漏洞
- Gopher 协议的使用
- SSRF 到 RCE 的利用链

### 题目 7：Web_baby_js

**难度**：3 分（较难）
**考点**：Node.js、原型链污染、JavaScript 安全

**题目描述**：
一个 Node.js 编写的 Web 应用。

**解题思路**：
1. 分析源码（可能需要先通过其他方式获取）
2. 发现存在原型链污染的点
3. 原型链污染原理：
   - JavaScript 中对象可以通过 __proto__ 访问原型
   - 如果用户可以控制对象的键名，就可能污染原型
4. 利用原型链污染：
   - 污染 Object.prototype
   - 影响后续的对象操作
   - 可能导致 RCE 或权限绕过
5. 构造 payload 进行原型链污染
6. 结合应用的业务逻辑进行利用
7. 最终读取 flag

**知识点**：
- JavaScript 原型链原理
- 原型链污染漏洞
- Node.js 安全问题
- 从原型链污染到 RCE 的利用

## 六、XCTF 赛事介绍和参与方式

### 6.1 XCTF 赛事体系

XCTF 联赛是国内最具影响力的网络安全赛事之一，每年举办多场分站赛和一场总决赛。

**赛事级别**：
- **国际赛**：邀请全球顶尖战队，最高水平
- **总决赛**：分站赛优胜队伍参加，年度最高荣誉
- **分站赛**：全国各地举办，各战队积分晋级
- **新人赛**：专门面向新人的比赛

**参赛资格**：
- 通常以战队为单位参赛
- 每队 1-4 人
- 高校学生和社会人士都可以参加
- 部分比赛可能有身份限制

### 6.2 如何参加 XCTF 比赛

1. **关注比赛信息**
   - 攻防世界平台会发布比赛通知
   - 关注 XCTF 官方公众号
   - 加入 CTF 社群获取消息

2. **组建战队**
   - 找到志同道合的伙伴
   - 分工合作（Web、Pwn、Reverse 等各方向）
   - 起一个响亮的队名

3. **报名参赛**

## 七、CTF 赛事经验分享

参加 CTF 比赛不仅需要技术，还需要策略和经验。下面分享一些比赛经验。

### 7.1 赛前准备

**技术准备**：
- 熟悉各个方向的基础题型
- 准备好常用工具和脚本
- 整理自己的 Payload 库
- 复习常见的漏洞利用方法

**环境准备**：
- 安装好 Kali Linux 或其他安全发行版
- 配置好常用工具（Burp、IDA、GDB 等）
- 准备好 VPN 客户端（有些比赛需要）
- 确保网络通畅

**身体准备**：
- 比赛前一天休息好
- 准备好食物和饮料
- 长时间比赛注意劳逸结合

### 7.2 比赛策略

**1. 先易后难**
- 比赛开始先扫一遍所有题目
- 把能快速做出来的题先做了
- 拿到基础分再去啃难题

**2. 分工合作**
- 战队成员按方向分工
- 有人做 Web，有人做 Pwn，有人做 Misc
- 遇到跨方向的题可以合作
- 不要所有人挤在一道题上

**3. 信息共享**
- 建立队内沟通渠道（QQ/微信群、语音等）
- 及时分享解题思路和进展
- 共享工具和 Payload
- 有新发现及时通知队友

**4. 时间管理**
- 一道题卡太久就先放一放
- 换道题做做，换换思路
- 最后检查所有已解题的 Flag 提交
- 不要因为一道题耽误太多时间

### 7.3 解题技巧

**Web 题**：
- 先做信息收集（源码、响应头、目录扫描）
- 找输入点，测试各种漏洞
- 注意题目中的提示
- 善用 Burp Suite

**Pwn 题**：
- 先 checksec 看保护机制
- 分析程序逻辑
- 找漏洞点
- 写 exploit 调试

**Reverse 题**：
- 先查壳
- 拖进 IDA/Ghidra 分析
- 找关键函数
- 分析算法，写逆算法脚本

**Crypto 题**：
- 识别加密类型
- 找已知的攻击方式
- 写脚本求解
- 注意数学性质

**Misc 题**：
- 先看文件类型和属性
- 用各种工具试一遍
- 注意题目描述的提示
- 脑洞要大

### 7.4 赛后复盘

比赛结束后复盘很重要：

1. **做出来的题**
   - 回顾解题过程
   - 有没有更优的解法
   - 总结知识点

2. **没做出来的题**
   - 看官方 Writeup
   - 理解解题思路
   - 学习新的技术和方法
   - 记录到自己的知识库

3. **团队总结**
   - 哪些做得好，哪些不足
   - 分工是否合理
   - 下次比赛怎么改进

4. **整理资料**
   - 收集比赛的题目和 Writeup
   - 整理新学到的技术
   - 更新自己的工具库

## 八、从 CTF 到职业发展

> 💡 **深入理解：CTF比赛除了"名次"还能给你什么？——"比赛后的隐藏价值"**
>
> 很多人打完CTF比赛只看排名：第几名？拿没拿奖？
> 然后就没有然后了。这其实浪费了比赛80%的价值。
>
> 一场CTF比赛，真正有价值的东西排名是这样的：
>
> ```
> 第1位价值：赛后复盘（学到最多东西的部分）
>   → 没做出来的题：看到Writeup的那一刻才知道"原来是这样啊！"
>   → 做出来了但方法笨的题：看到别人的解法才知道"居然能这么简洁！"
>   → 赛后复盘不是"看一遍"，而是"重新做一遍+对比+总结"
>
> 第2位价值：暴露的短板（最有指导意义的部分）
>   → Pwn方向全军覆没 → 说明二进制基础为零，该补了
>   → Web题做得慢 → 说明工具/脚本效率不够，要整理Payload库
>   → 这种"暴露"比"排名"值钱多了——它告诉你下一步学什么
>
> 第3位价值：团队协作经验（可迁移到工作中的部分）
>   → 分工不合理导致有方向没人做 → 你学会了管理任务
>   → 沟通不通导致重复劳动 → 你学会了信息同步
>   → 比赛中出现的所有协作问题，工作中都会重演
>
> 第4位价值：心理素质（最有长期价值的部分）
>   → 连做8小时脑力全开 → 你知道了自己的耐力极限
>   → 一道题卡了3小时终于做出来 → 你学会了"坚持"
>   → 排名落后时稳住心态 → 你学会了"抗压"
>   → 这些东西在工作面试、实战渗透中比具体技术更珍贵
>
> 第5位价值：才轮到排名和奖状
>   → 排名证明的是"某一时刻的水平"，不是"你的全部能力"
>   → 很多现在的大神当年也是倒数第一名起步的
> ```
>
> **赛后黄金24小时**：打完比赛的24小时内是学习效果最好的时候。
> 趁着记忆还新鲜，立马做三件事：
> 1. 把没做出来的题每题写一句话：为什么没做出来？
> 2. 把做出来的题的思路整理成笔记
> 3. 列出"下次比赛前要补的3个知识点"

很多人关心：打 CTF 对找工作有帮助吗？CTF 能做什么职业？下面来聊聊这个话题。

### 8.1 CTF 对职业发展的帮助

1. **技术能力提升**
   - 打下扎实的安全技术基础
   - 锻炼问题解决能力
   - 熟悉各种安全工具
   - 培养安全思维

2. **简历加分项**
   - CTF 比赛成绩是安全岗位的重要加分项
   - 很多企业在招聘时会优先考虑有 CTF 经验的人
   - 好的比赛名次很有说服力

3. **人脉资源**
   - 认识很多圈内的朋友
   - 加入战队，拓展人脉
   - 有更多的内推机会

4. **职业方向探索**
   - 通过 CTF 了解各个安全方向

## 九、实战案例

### 案例一：新手区通关

**背景**：小周是一名大二学生，刚接触 CTF，想在攻防世界新手区练练手。

**目标**：用两周时间通关新手区所有题目。

**学习过程**：

**第一周：Web + Misc**

1. **Web 方向**
   - 从最简单的 SQL 注入开始
   - 做了 Web_php_include、NewsCenter 等题
   - 理解了 SQL 注入和文件包含的原理
   - 学会了使用 Burp Suite 抓包

2. **Misc 方向**
   - 做了图片隐写、编码题
   - 学会了 Stegsolve、binwalk 等工具
   - 感觉 Misc 题很有趣，脑洞大
   - 培养了对 CTF 的兴趣

**第二周：Crypto + Reverse + Pwn**

1. **Crypto 方向**
   - 做了古典密码、RSA 入门题
   - 学会了用 Python 写解题脚本
   - 数学不好也没关系，很多题有套路

2. **Reverse 方向**
   - 做了最简单的几道逆向题
   - 学会了 IDA 的基本使用
   - 感觉逆向很难，需要更多时间学习

3. **Pwn 方向**
   - 尝试了几道栈溢出入门题
   - 配置了 GDB + pwndbg 环境
   - 理解了栈溢出的基本原理
   - 写了第一个 exploit，很有成就感

**通关心得**：
- 新手区的题不难，但覆盖面很广
- 每道题都能学到新知识
- 做完新手区对 CTF 有了全面的了解
- 找到了自己感兴趣的方向（Web）
- 准备向高手区进发

### 案例二：高手区挑战

**背景**：小吴已经做完了新手区，开始挑战高手区。

**目标**：在高手区拿到 100 分，具备参加比赛的基础能力。

**挑战过程**：

**第一个月：Web 方向突破**
- 从 1 分题开始，一道一道做
- 遇到不会的题先自己想，实在不会再看 Writeup
- 每道题都做笔记，整理知识点
- 做完了大部分 1 分和 2 分的 Web 题
- 积分达到了 30 多分

**第二个月：全面发展**
- 开始做 Misc 和 Crypto 的题
- 这些方向的题有时候更容易拿分
- 也尝试了几道 Pwn 和 Reverse 的题
- 虽然做得慢，但也有进展
- 积分突破了 60 分

**第三个月：挑战难题**
- 开始做 3 分的 Web 题
- 难度明显提升，经常卡很久
- SSRF、反序列化、SSTI 等知识点逐个攻克
- 参加了几次线上比赛，虽然排名不高，但学到很多
- 积分突破了 100 分

**心得体会**：
- 高手区的题质量很高，每道题都有收获
- 不要追求速度，要真正理解
- 不会的题可以先放一放，过段时间再做
- 多参加比赛，以赛促学
- 坚持很重要，不要因为难就放弃

### 案例三：XCTF 参赛经验

**背景**：某高校战队"0ops"（化名）第一次参加 XCTF 分站赛。

**战队组成**：
- 队长：大三，主攻 Web，有一定比赛经验
- 队员 A：大二，主攻 Pwn
- 队员 B：大二，主攻 Reverse + Crypto
- 队员 C：大一，主攻 Misc + Web

**赛前准备**：
1. **技术准备**
   - 每周一起训练 2-3 次
   - 刷攻防世界和 BUUCTF 的题
   - 整理常用工具和 Payload
   - 模拟比赛训练

2. **环境准备**
   - 每个人准备好自己的工作环境
   - 配置好共享文档和沟通工具
   - 准备好服务器（用于反弹 Shell 等）

3. **战术准备**
   - 分工：每人负责自己擅长的方向
   - 策略：先易后难，先拿基础分
   - 协作：遇到不会的可以求助队友

**比赛过程**：

**第 1 小时：快速解题**
- 大家快速浏览所有题目
- 各自开始做自己方向的简单题
- 第 30 分钟，队员 C 做出了第一道 Misc 签到题
- 第 45 分钟，队长做出了第一道 Web 题
- 1 小时结束，战队已经解出 4 道题，排名中游

**第 2-4 小时：稳步推进**
- 大家开始做中等难度的题
- 队员 A 在做一道 Pwn 题，遇到了困难
- 队员 B 解出了一道 Crypto 题
- 队长和队员 C 合作解出了一道 Web 题
- 排名逐渐上升

**第 5-8 小时：攻坚阶段**
- 简单题都做得差不多了
- 开始攻克难题
- 队员 A 终于做出了那道 Pwn 题，全队振奋
- 队长也解出了一道难的 Web 题
- 排名进入前 30%

**最后阶段**：
- 检查所有已解题的 Flag
- 再试几道可能做出来的题
- 最终排名 20 多名（共 200 多支队伍）

**赛后总结**：
- 第一次参加大型比赛，成绩还不错
- 暴露了很多问题：Pwn 和 Reverse 比较弱
- 需要加强团队协作和沟通
- 下次比赛目标：进入前 20 名

**比赛经验**：
1. 赛前准备很重要
2. 团队协作比单打独斗强
3. 先易后难是正确的策略
4. 心态要好，不要因为一道题影响全局
5. 每次比赛都是学习的机会

**给 CTF 选手的职业建议**：

   - CTF 能打下很好的技术基础
   - 但实战能力也很重要
   - 两者结合才能全面发展

   - CTF 方向很多，不可能样样精通

3. **多做项目和实习**
   - 书本和比赛的知识是不够的
   - 多做真实项目，积累经验
   - 实习是很好的学习机会

4. **持续学习**
   - 安全领域更新很快
