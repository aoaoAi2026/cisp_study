## 💡 文件包含 vs 命令执行：对比理解更高效

既然刚学完命令执行，现在又来总结文件包含，
那不如放在一起对比——这样两个都更清楚。

| 对比维度 | 命令执行 | 文件包含 |
|---------|---------|---------|
| 一句话描述 | 用户输入拼进了系统命令 | 用户输入控制了文件路径 |
| 直接危害 | 执行系统命令（whoami, ls...） | 读文件 / 执行代码 |
| 利用门槛 | 找到危险函数+拼接点 | 找到包含点+可控参数 |
| 经典绕过 | 空格、关键字、编码 | 路径穿越、伪协议、截断 |
| 常见防护 | 过滤特殊字符、禁用函数 | 固定包含路径、关闭远程包含 |
| 到RCE的方法 | 本身就是RCE | 需要伪协议/日志包含等方式过渡 |

**它们的共同点：**
1. 都是"用户输入进入了处理流程"
2. 防御思路相似：输入验证 + 最小权限
3. 都可以通过绕过技巧绕过过滤
4. 在真实渗透中经常组合使用

**它们经常怎么组合？**
- 先通过文件包含读取源码 → 发现命令执行漏洞点
- 先通过命令执行上传文件 → 再用文件包含去包含它
- 先通过文件包含 + 日志写入 → 再触发命令执行

> 这就是为什么我们在进阶篇先讲命令执行，再讲文件包含——
> 理解了一个，另一个就很容易了，因为它们背后的思维模式是一样的。

## 📊 核心概念回顾

### 4.1 什么是文件包含漏洞？

**一句话定义：**
> 用户输入可控，
> 且被拼接到文件包含函数的路径中，
> 导致攻击者可以包含任意文件，
> 读取敏感信息或执行恶意代码。

### 4.2 LFI vs RFI 对比表

| 特性 | LFI（本地文件包含） | RFI（远程文件包含） |
|------|---------------------|---------------------|
| **全称** | Local File Inclusion | Remote File Inclusion |
| **包含范围** | 服务器本地文件 | 远程URL文件 |
| **配置要求** | 无特殊要求 | allow_url_include=On |
| **默认状态** | 常见 | 少见（默认关闭） |
| **直接危害** | 读取敏感文件 | 直接执行任意代码 |
| **间接危害** | 配合上传/日志等GetShell | - |
| **利用难度** | 稍高 | 很低 |

### 4.3 PHP的4个文件包含函数

| 函数 | 出错时行为 | 是否只包含一次 | 常用场景 |
|------|-----------|---------------|----------|
| `include()` | 警告，继续执行 | 否 | 非关键文件 |
| `require()` | 致命错误，停止 | 否 | 关键配置文件 |
| `include_once()` | 警告，继续执行 | 是 | 类库、函数库 |
| `require_once()` | 致命错误，停止 | 是 | 重要类库 |

**从漏洞利用角度：**
四个函数差别不大，都能用来利用漏洞。

### 4.4 漏洞产生的根本原因

1. **用户输入可控**
2. **输入拼接到文件路径**
3. **没有严格的路径限制**
4. **没有白名单校验**

**常见场景：**
- 页面/模板切换
- 多语言切换
- 主题切换
- 文件查看/下载
- ...

## 🔐 PHP伪协议 Cheat Sheet

这是文件包含中最重要的部分，
一定要记牢！

### 5.1 常用伪协议速查表

| 伪协议 | 作用 | 需allow_url_include | 有后缀能否用 | 利用场景 |
|--------|------|---------------------|-------------|----------|
| `php://filter` | 读取文件源码（base64等） | ❌ 不需要 | ✅ 可以 | 读PHP源码、配置文件 |
| `php://input` | 执行POST中的PHP代码 | ✅ 需要 | ❌ 不行 | 直接RCE |
| `data://` | 执行数据URI中的代码 | ✅ 需要 | ❌ 不行 | 直接RCE |
| `phar://` | 访问phar归档文件 | ❌ 不需要 | 视情况 | 配合上传GetShell |
| `zip://` | 访问zip压缩包文件 | ❌ 不需要 | 视情况 | 配合上传GetShell |
| `file://` | 访问本地文件 | ❌ 不需要 | 视情况 | 本地文件包含 |
| `http://` | 远程HTTP文件包含 | ✅ 需要 | ❌ 不行 | RFI直接GetShell |

### 5.2 php://filter 详解

**读取源码（最常用）：**
```
php://filter/convert.base64-encode/resource=文件名
```

**示例：**
```
?file=php://filter/convert.base64-encode/resource=config.php
```

**有后缀的情况也能用：**
```
后台：include($_GET['file'] . ".php");
利用：?file=php://filter/convert.base64-encode/resource=config
结果：php://filter/convert.base64-encode/resource=config.php ✅
```

**其他过滤器：**
| 过滤器 | 作用 |
|--------|------|
| `convert.base64-encode` | Base64编码（最常用） |
| `string.toupper` | 转大写 |
| `string.tolower` | 转小写 |
| `string.strip_tags` | 去除HTML/PHP标签 |

### 5.3 php://input 详解

**用法：**
```
GET:  ?file=php://input
POST: <?php system('whoami'); ?>
```

**条件：**
- `allow_url_include = On`
- 没有后缀（或能截断后缀）

### 5.4 zip:// 详解

**用法：**
```
zip:///path/to/zipfile%23inner_file.php
```

**示例：**
```
?file=zip://upload/test.jpg%23shell.php
```

**步骤：**
1. 把 shell.php 压缩成 zip
2. zip改后缀为.jpg（绕过上传检测）
3. 上传到服务器
4. 用 zip:// 包含 zip 里的 shell.php
5. 代码执行！

## 🚀 LFI 到 RCE 全图谱

这是文件包含漏洞的核心价值：
**从一个读文件的漏洞，
变成代码执行漏洞。**

### 6.1 方法总览（按实用性排序）

| 排名 | 方法 | 难度 | 条件 | 成功率 |
|------|------|------|------|--------|
| 1 | **配合文件上传** | 低 | 有上传点+知道路径 | 很高 |
| 2 | **日志包含** | 中 | 知道日志路径+可读 | 高 |
| 3 | **php://input** | 低 | allow_url_include=On | 中（配置限制） |
| 4 | **session包含** | 中 | 能控制session内容+知道路径 | 中 |
| 5 | **zip:// / phar://** | 中 | 有上传点 | 中 |
| 6 | **data://** | 低 | allow_url_include=On | 低（配置限制） |
| 7 | **/proc/self/environ** | 中 | 老系统+可读 | 低 |
| 8 | **phpinfo+临时文件** | 高 | 有phpinfo+条件竞争 | 低 |

### 6.2 方法详细说明

#### ① 配合文件上传（最简单直接）
- **原理**：上传图片马，然后用LFI包含
- **条件**：有文件上传点 + 知道上传路径
- **步骤**：上传 → 知道路径 → 包含 → GetShell
- **优点**：简单、稳定、成功率高
- **缺点**：需要有上传点

#### ② 日志包含（最经典）
- **原理**：往User-Agent写PHP代码，日志会记录，然后包含日志
- **条件**：知道日志路径 + 日志可读 + 能控制User-Agent
- **步骤**：写日志 → 包含日志 → GetShell
- **优点**：不需要上传点，只要有LFI就行
- **缺点**：日志可能很大、路径不好猜

#### ③ php://input（最爽快）
- **原理**：POST数据当PHP代码执行
- **条件**：allow_url_include=On + 无后缀（或能截断）
- **步骤**：直接用，一步到位
- **优点**：最简单、最快
- **缺点**：配置限制，现在默认关闭

#### ④ session包含（比较常用）
- **原理**：往session里写PHP代码，然后包含session文件
- **条件**：能控制session内容 + 知道session路径
- **步骤**：找注入点 → 写入session → 包含session文件 → GetShell
- **优点**：不需要上传点
- **缺点**：需要能控制session内容，路径不好猜

#### ⑤ zip:// / phar://
- **原理**：上传zip/phar文件，然后用伪协议包含里面的PHP
- **条件**：有上传点 + 知道路径
- **步骤**：构造zip → 上传 → zip://包含 → GetShell
- **优点**：可能绕过文件内容检测
- **缺点**：需要上传点

## ✂️ 截断技巧总结

当include有后缀时（比如 `.php`），
很多方法就用不了了，
这时候需要截断。

### 7.1 截断方法对比

| 方法 | 适用场景 | 条件 | 成功率 |
|------|----------|------|--------|
| **%00截断** | LFI | PHP < 5.3.4 + GPC关闭 | 低（版本老） |
| **路径长度截断** | LFI | Windows/Linux老版本 | 很低 |
| **问号截断** | RFI | allow_url_include=On | 中（需RFI） |
| **php://filter** | LFI（读源码） | 无特殊要求 | 很高（仅读文件） |

### 7.2 有后缀时的利用思路

如果有后缀，
且不能截断，怎么办？

| 目标 | 方法 |
|------|------|
| **读取PHP源码** | php://filter（完美适配后缀） |
| **包含上传的图片马** | 直接包含（图片马就是普通文件，后缀不影响） |
| **包含日志/session** | 直接包含（日志不是php文件，后缀加了也找不到） |
| **执行代码** | 找其他方法（日志包含、session包含等，不受后缀影响？受影响的话需要截断） |

::: tip 注意
日志包含、session包含这些，
如果include有后缀，
其实也是受影响的，
因为会在后面加.php，
导致找不到文件。

这种情况下，
要么能截断，
要么换其他思路。
:::

## 🛡️ 文件包含漏洞防御

讲了这么多攻击方法，
我们也来讲讲怎么防御。
攻防一体才是完整的知识体系。

### 8.1 防御的核心思路

> **不要信任用户输入，
> 严格限制包含的文件。**

### 8.2 防御手段（从强到弱）

#### ① 白名单校验（最推荐）
只允许包含指定的文件，
用in_array检查。

```php
<?php
// 只允许这几个页面
$allow_pages = ['home', 'about', 'contact'];

$page = $_GET['page'];

if (in_array($page, $allow_pages)) {
    include $page . '.php';
} else {
    echo "非法页面";
}
?>
```

**优点：** 最安全，基本不可能绕过
**缺点：** 需要维护白名单列表

#### ② 限制目录（推荐）
用`realpath()`检查真实路径，
限制在指定目录内。

```php
<?php
$base_dir = '/var/www/html/pages/';
$file = $_GET['file'];

// 解析真实路径
$real_path = realpath($base_dir . $file);

// 检查是否在允许的目录内
if (strpos($real_path, $base_dir) === 0) {
    include $real_path;
} else {
    echo "非法路径";
}
?>
```

**优点：** 比较安全，灵活
**缺点：** 实现稍复杂

#### ③ basename() 只取文件名
用`basename()`函数，
只取文件名部分，
去掉路径信息。

```php
<?php
$file = basename($_GET['file']); // 只保留文件名，去掉../等
include "pages/" . $file;
?>
```

**优点：** 简单有效，防止路径穿越
**缺点：** 还是能包含同目录下的任意文件

#### ④ 过滤 ../（不推荐，容易被绕过）
简单过滤 `../` 字符串，
很容易被绕过（双写、编码等）。

```php
// 这种过滤很容易被绕过！
$file = str_replace('../', '', $_GET['file']);
```

**缺点：** 可以用双写、编码等绕过，不安全

#### ⑤ 关闭危险配置（辅助）
- 关闭 `allow_url_include`（必须关！）
- 关闭 `allow_url_fopen`（如果不需要）

```ini
; php.ini
allow_url_include = Off
allow_url_fopen = Off
```

### 8.3 防御层级图

```
第1层：白名单校验          ← 最强，首选
         │
第2层：限制目录 + realpath  ← 很强，次选
         │
第3层：basename()           ← 防路径穿越
         │
第4层：输入过滤             ← 不够可靠
         │
第5层：关闭危险配置         ← 辅助手段
```

## 🔄 进阶篇总复习

进阶篇我们学习了两个模块：
- **命令执行漏洞**（第26-28章）
- **文件包含漏洞**（第29-31章）

这两个漏洞经常一起出现，
也经常互相配合利用。
让我们来做一个总复习，
把它们串起来。

### 9.1 命令执行漏洞回顾

#### 核心概念
- **命令执行**：用户输入被拼接到系统命令中，导致执行任意命令
- **代码执行**：用户输入被当作代码执行（eval等）
- 两者不同，但危害都很大

#### 危险函数
**命令执行：**
- `system()`、`exec()`、`shell_exec()`、`passthru()`
- `popen()`、`proc_open()`、`pcntl_exec()`

**代码执行：**
- `eval()`、`assert()`
- `preg_replace()` /e 修饰符（老版本）
- `create_function()`（老版本）
- 回调函数（`array_map()`、`call_user_func()`等）

#### 特殊符号
| 符号 | 作用 |
|------|------|
| `;` | 分号，依次执行多条命令 |
| `|` | 管道符，前面的输出作为后面的输入 |
| `&` | 后台执行/命令分隔（Windows） |
| `&&` | 前面成功才执行后面 |
| `\|\|` | 前面失败才执行后面 |
| `` ` `` | 反引号，执行命令 |
| `$()` | 命令替换，执行命令 |

#### 绕过技巧
- **空格绕过**：`${IFS}`、`$IFS$9`、`<`、`%09`（Tab）
- **关键字绕过**：拼接、反斜杠、单双引号、$@、通配符
- **无回显（盲注）**：DNSLog外带、HTTP外带、时间盲注
- **写Shell**：重定向、echo写文件、base64解码写

#### 反弹Shell
- Bash、Python、PHP、Perl、Ruby、NC、Java...
- 至少要会3种以上

### 9.2 文件包含漏洞回顾

（前面已经详细回顾过了，这里简单列一下）

- **LFI vs RFI**
- **4个文件包含函数**
- **PHP伪协议**：php://filter、php://input、zip://、phar://、data://
- **LFI→RCE的方法**：上传、日志、session、伪协议...
- **截断技巧**：%00、路径长度、问号

### 9.3 命令执行 vs 文件包含 对比

| 对比项 | 命令执行 | 文件包含 |
|--------|----------|----------|
| **核心** | 执行系统命令 | 包含文件（可读可执行） |
| **直接危害** | 执行命令 | 读文件（LFI）/ 执行代码（RFI） |
| **间接危害** | - | 配合其他漏洞GetShell |
| **利用难度** | 中（很多过滤） | 中（方法多） |
| **常见程度** | 较常见 | 常见 |
| **防御** |  escapeshellarg/escapeshellcmd | 白名单/限制目录 |

### 9.4 组合拳：命令执行 + 文件包含

在实战中，
这两个漏洞经常互相配合：

**场景1：有文件包含，想执行命令**
- 用文件包含GetShell后，就能执行命令了
- 或者php://input直接执行PHP代码，再调用system

**场景2：有命令执行，想读文件**
- 直接用cat、more等命令读文件
- 比文件包含读文件还方便

**场景3：互相辅助**
- 文件包含读到配置文件 → 拿到数据库密码 → 更大危害
- 命令执行反弹Shell → 然后再找文件包含漏洞 → 进一步利用

## 📚 综合案例分析

光说不练假把式，
我们来看几个综合案例，
把命令执行和文件包含的知识串起来。

### 案例1：从文件包含到内网漫游

**目标：** 某企业官网

**信息收集：**
- 网站是PHP写的
- 有一个页面 `?page=home`，看起来像文件包含

**步骤1：确认文件包含漏洞**
```
?page=php://filter/convert.base64-encode/resource=config
```
成功读到config.php的base64内容，
解码后拿到数据库账号密码。

**步骤2：找日志路径**
```
?page=../../../../var/log/apache2/access.log
```
能读到，有戏！

**步骤3：日志包含GetShell**
- 修改User-Agent为一句话木马
- 多次访问确保写入日志
- 包含日志文件，成功GetShell

**步骤4：执行命令，收集信息**
```
whoami → www-data
uname -a → Linux 内核版本
ifconfig/ip addr → 内网IP段
cat /etc/passwd → 系统用户
```

**步骤5：反弹Shell**
- 用Python反弹Shell到自己的VPS
- 获得交互式Shell

**步骤6：提权**
- 检查内核版本，找对应exp
- 或者找SUID文件、计划任务等
- 成功提权到root

**步骤7：内网渗透**
- 收集内网信息
- 横向移动
- 拿下更多机器
- ...

**总结：**
一个文件包含漏洞，
最终拿下整个内网。
这就是红队的魅力！

### 案例2：命令执行绕过WAF

**目标：** 某网站有个ping功能，存在命令执行
**限制：** 有WAF，过滤了很多关键字

**步骤1：确认漏洞**
```
ip=127.0.0.1;id
```
返回了id的结果，确认有命令执行。

**步骤2：发现被过滤**
```
ip=127.0.0.1;cat /etc/passwd
```
返回WAF拦截页面，
cat被过滤了。

**步骤3：绕过空格**
```
ip=127.0.0.1;cat${IFS}/etc/passwd
```
还是被拦截，cat关键字被过滤了。

**步骤4：绕过关键字**
用拼接：
```
ip=127.0.0.1;a=c;b=at;$a$b${IFS}/etc/passwd
```
成功执行！读到了passwd。

**步骤5：找网站配置**
```
ip=127.0.0.1;find${IFS}/var/www${IFS}-name${IFS}"config.php"
```
找到了配置文件路径。

**步骤6：读配置文件**
```
ip=127.0.0.1;tac${IFS}/var/www/html/config.php
```
（用tac代替cat，反向读）
拿到了数据库密码。

**步骤7：进一步利用**
- 连接数据库
- 找后台管理员密码
- 登录后台
- 上传Shell
- ...

### 案例3：文件上传 + 文件包含组合拳

**目标：** 某CMS系统
**发现：**
1. 有文件上传功能，但只能传图片
2. 有模板包含功能，存在LFI

**步骤1：测试上传**
- 上传php文件，被拦
- 上传图片马（图片+PHP代码），成功
- 上传路径：`/upload/2024/01/xxx.jpg`

**步骤2：测试LFI**
```
?template=../../../../etc/passwd
```
能读到，确认有LFI。

**步骤3：组合利用**
```
?template=../../upload/2024/01/xxx.jpg
```
包含成功！
图片马中的PHP代码被执行，
成功GetShell！

**步骤4：提权、内网...**

**总结：**
单个漏洞可能危害有限，
但多个漏洞组合起来，
威力就大了。
这就是红队的思路：
**把一个个小漏洞串起来，
形成完整的攻击链。**

### 案例5：代码审计发现文件包含

**场景：** 拿到某CMS源码，做代码审计

**审计思路：**
1. 全局搜索 `include`、`require`、`include_once`、`require_once`
2. 看参数是否可控
3. 看有没有做过滤
4. 构造Payload，验证漏洞

**示例：**
搜索发现：
```php
// index.php
$tpl = $_GET['tpl'];
include "templates/" . $tpl . "/index.php";
```

**分析：**
- $tpl可控
- 有前缀 `templates/`
- 有后缀 `/index.php`
- 没有过滤，可以路径穿越

**利用：**
```
?tpl=../../../../etc/passwd%00
```
（%00截断后面的/index.php，需要低版本PHP）

或者用php://filter读源码：
```
?tpl=php://filter/convert.base64-encode/resource=../../config
```
（不对，后面还有/index.php，需要调整）

具体怎么利用，
就要看实际情况了。

## 📝 本章总结

这一章内容很多，

### 文件包含模块核心知识点

1. **概念**：LFI vs RFI
2. **函数**：include、require、include_once、require_once
3. **伪协议**：
   - php://filter → 读源码（最重要！）
   - php://input → 执行代码（需配置）
   - zip:// / phar:// → 配合上传
   - data:// → 执行代码（需配置）
4. **LFI→RCE方法**：
   - 配合文件上传（最简单）
   - 日志包含（最经典）
   - session包含
   - php://input
   - zip:// / phar://
   - ...
5. **截断**：%00、路径长度、问号
6. **防御**：白名单 > 限制目录 > basename > 过滤

### 进阶篇整体回顾

**命令执行漏洞：**
- 原理、危险函数、特殊符号
- 绕过技巧（空格、关键字、盲注）
- 反弹Shell、写Shell
- 绕过disable_functions

**文件包含漏洞：**
- 原理、LFI/RFI
- PHP伪协议
- LFI→RCE各种姿势
- 截断技巧
- 防御

**两者关系：**
