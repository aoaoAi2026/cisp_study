## 一、其他常用 Web 靶场汇总

除了前面介绍的 DVWA、WebGoat、bWAPP、Pikachu 之外，还有很多优秀的 Web 安全靶场。不同靶场各有特色，适合不同阶段和不同需求的学习者。

> 💡 **大白话说如何选靶场——"工具箱里选工具，先想清楚我要干什么"**
>
> 面对这么多靶场，最大的困惑就是"我到底该打哪个？" 这里给你一个傻瓜式的决策流程：
>
> **按你的身份来选**：
> - 我是零基础小白 → Pikachu（中文友好）→ DVWA（系统化）→ PortSwigger Academy（跟课程学）
> - 我是PHP开发者 → Pikachu + bWAPP（都是PHP生态，和你的工作最接近）
> - 我是.NET开发者 → WebGoat.NET（专门为.NET平台设计的）
> - 我是Java开发者 → WebGoat + PortSwigger Academy（Java生态的课程最全）
>
> **按你的目标来选**：
> - 我想考证（OSCP/OSWE）→ PortSwigger Academy + Hack The Box
> - 我想提升实战能力 → Hack The Box / TryHackMe（真实环境，非教学场景）
> - 我想做漏洞挖掘 → PortSwigger Academy → Hackazon（模拟电商，业务逻辑漏洞多）
> - 我想学特定类型的漏洞 → 用PortSwigger Academy按漏洞类型专项练
>
> **一句话总结**：PortSwigger Academy 是所有人的"必修课"（免费、系统、高质量），它加上你工作相关的1-2个专项靶场就够了。不需要全部打完。

### 1.1 DVWA（简要回顾）

### 1.1 DVWA（简要回顾）

**全称**：Damn Vulnerable Web Application

**简介**：最经典的 PHP Web 漏洞练习平台，"该死的脆弱 Web 应用"。

**特点**：
- 四大安全级别：Low / Medium / High / Impossible
- 十大漏洞模块，覆盖 OWASP Top 10
- 每个漏洞都有源码查看功能，方便学习
- 轻量级，部署简单

**适合人群**：Web 安全初学者，适合作为入门第二站（Pikachu 之后）。

**部署方式**：Docker 或手动部署，默认端口 80。

### 1.2 Mutillidae II

**简介**：OWASP 出品的另一款 Web 安全教学靶场，全称为 "OWASP Mutillidae II"。

**特点**：

| 特点 | 说明 |
|------|------|
| **漏洞数量多** | 40+ 漏洞场景，160+ 挑战 |
| **OWASP 出品** | 官方权威，内容专业 |
| **教学性强** | 每个漏洞都有详细的提示和视频教程链接 |
| **难度可调** | 0-16 级安全等级 |
| **功能丰富** | 包含博客、论坛、相册等多种功能模块 |

**包含的漏洞模块**：
- 信息泄露与指纹识别
- 认证与会话管理
- 访问控制失效
- 数据验证（XSS、注入等）
- 客户端安全
- 拒绝服务
- 加密问题
- Web 服务安全

**部署方式**：
```bash
docker pull citizenstig/nowasp
docker run -d -p 80:80 --name mutillidae citizenstig/nowasp
```

访问 `http://localhost/mutillidae`，点击 "setup/reset the DB" 初始化数据库。

## 三、在线靶场 vs 本地靶场对比

### 3.1 在线靶场

**代表**：PortSwigger Web Security Academy、Hack The Box、VulnWeb、Gruyere

**优点**：
- 零部署成本，打开浏览器就能用
- 环境统一，不会因为配置问题影响学习
- 持续更新，有最新的漏洞场景
- 有社区和排行榜，学习动力强
- 不需要占用本地资源

**缺点**：
- 需要联网，无法离线学习
- 实验时间有限制
- 环境无法自定义
- 有些平台需要科学上网
- 无法查看源码（部分平台）

**适合场景**：
- 随时随地学习
- 学习最新的漏洞技术
- 验证某个漏洞的利用方式
- 参加竞赛和挑战

### 3.2 本地靶场

**代表**：DVWA、WebGoat、bWAPP、Pikachu、Vulhub

**优点**：
- 可以离线学习
- 环境完全可控
- 可以查看源码，深入理解漏洞
- 可以修改代码，研究防御方法
- 没有时间限制
- 可以自定义漏洞环境

**缺点**：
- 需要部署环境，有一定门槛
- 占用本地资源
- 更新相对较慢
- 需要自己找学习资料

**适合场景**：
- 系统学习漏洞原理
- 查看和分析源代码
- 深入研究漏洞的防御方法
- 自定义实验环境
- 离线学习

### 3.3 选择建议

> 💡 **大白话说在线vs本地——"健身房办卡还是家里买器械"**
>
> 在线靶场和本地靶场的选择，其实和健身一个道理：
>
> **在线靶场 = 去健身房**（PortSwigger Academy、Hack The Box）
> - 好处：器材齐全（最新漏洞）、不用维护、有教练（课程引导）、有动力（排行榜/社区）
> - 坏处：有时间限制（健身房营业时间）、不能随意改装器材、需要网络
> - 适合：碎片时间学习、想学最新技术、需要外部驱动力的人
>
> **本地靶场 = 在家买健身器材**（DVWA、Vulhub）
> - 好处：随时用、随意改（看源码/改代码/自定义环境）、不受限
> - 坏处：要自己搭环境（相当于组装器材）、占用空间（本地资源）、更新慢
> - 适合：深入研究原理、需要反复练习同一个漏洞、离线学习的人
>
> **最佳组合**：本地学原理（DVWA、bWAPP看源码），线上练技巧（PortSwigger Academy），CTF验证综合能力。

**初学者**：
- 先用本地靶场（Pikachu、DVWA）打基础

## 五、实战案例

### 案例 1：PortSwigger XSS 学习 —— 从基础到进阶

**目标**：通过 Web Security Academy 的 XSS 课程，系统掌握 XSS 各种绕过技巧。

**步骤**：

**步骤 1：注册账号**
访问 https://portswigger.net/web-security 注册免费账号。

**步骤 2：学习 XSS 基础**
进入 "Cross-site scripting (XSS)" 主题，先阅读以下文章：
- What is XSS?
- XSS context
- How to prevent XSS

**步骤 3：完成入门实验室**

**实验 1：Reflected XSS into HTML context with nothing encoded**
- 难度：APPRENTICE
- 场景：搜索框直接输出用户输入
- Payload：`<script>alert(1)</script>`
- 直接输入即可成功

**实验 2：Stored XSS into HTML context with nothing encoded**
- 难度：APPRENTICE
- 场景：评论功能存在存储型 XSS
- Payload：`<script>alert(1)</script>`
- 提交评论后刷新页面触发

**步骤 4：学习进阶绕过**

学习以下技巧，每个技巧对应一个或多个实验：
- 标签绕过（用 `<img>`、`<svg>` 替代 `<script>`）
- 事件处理器绕过（onerror、onload、onclick 等）
- 属性注入（在 HTML 属性中注入）
- JavaScript 上下文注入
- 各种编码绕过（HTML 实体、URL 编码、Unicode）
- WAF 绕过

**步骤 5：挑战 DOM 型 XSS**

DOM XSS 是 XSS 中最难的部分，需要分析前端 JavaScript 代码：
- 找到 Source（数据源，如 location.hash）
- 找到 Sink（危险函数，如 innerHTML）
- 构造 Payload 从 Source 到 Sink

### 案例 2：Mutillidae 实战 —— 从信息收集到 GetShell

**场景**：在 Mutillidae II 中模拟一次完整的渗透测试。

**步骤**：

**步骤 1：信息收集**
- 使用 dirsearch 扫描目录
- 发现 phpinfo.php、robots.txt 等敏感文件
- 使用 Wappalyzer 识别技术栈（PHP、Apache、MySQL）

**步骤 2：寻找注入点**
- 在搜索功能中测试 SQL 注入
- 测试 `'` 报错，确认存在注入
- 使用 UNION 查询获取数据库信息

**步骤 3：获取管理员密码**
- 通过 SQL 注入获取 users 表中的用户名和密码哈希
- 破解 MD5 哈希得到明文密码
- 使用管理员账号登录后台

**步骤 4：上传 Shell**
- 后台有文件上传功能
- 测试上传限制，寻找绕过方法
- 上传 WebShell，获取服务器权限

**步骤 5：提权与内网探索**
- 收集系统信息
- 寻找提权漏洞
- 探索内网其他主机

### 案例 3：Hackazon 体验 —— 电商场景的漏洞挖掘

**场景**：在 Hackazon 电商模拟网站中挖掘业务逻辑漏洞。

**步骤**：

**步骤 1：熟悉网站功能**
- 浏览商品、注册账号
- 体验购物车、下单、支付流程
- 了解用户中心、评论、收藏等功能

**步骤 2：挖掘常见漏洞**
- 搜索功能 XSS 测试
- 商品详情 SQL 注入测试
- 用户资料存储型 XSS 测试

**步骤 3：挖掘业务逻辑漏洞**

**支付漏洞**：
- 添加商品到购物车
- 抓包修改价格为 0.01
- 下单成功，以极低价格购买商品

**优惠券漏洞**：
- 尝试重复使用同一张优惠券
- 尝试修改优惠券金额
- 尝试叠加多张优惠券

**越权漏洞**：
- 查看订单时修改订单 ID
- 尝试查看他人订单
- 尝试修改他人收货地址

**步骤 4：API 安全测试**
- 分析前端调用的 API 接口
- 测试 API 的认证和权限控制
- 尝试水平越权访问其他用户数据

### 案例 5：靶场学习效率提升 —— 从入门到进阶的方法论

**5.1 建立知识体系**

不要零散地做题，要建立系统化的知识体系：

```
Web 安全
├── 服务端漏洞
│   ├── SQL 注入
│   ├── 命令注入
│   ├── 代码执行
│   ├── 文件包含
│   ├── 文件上传
│   ├── 路径遍历
│   ├── SSRF
│   ├── XXE
│   └── 反序列化
├── 客户端漏洞
│   ├── XSS（反射/存储/DOM）
│   ├── CSRF
│   ├── 点击劫持
│   └── CORS
├── 认证与会话
│   ├── 暴力破解
│   ├── 会话劫持
│   ├── 密码重置
│   ├── JWT 攻击
│   └── OAuth 漏洞
├── 访问控制
│   ├── 水平越权
│   ├── 垂直越权
│   └── IDOR
└── 其他
    ├── 信息泄露
    ├── 业务逻辑漏洞
    └── 配置错误
```

**5.2 高效做题方法**

1. **先看题再查资料**：先自己思考 15-30 分钟
2. **实在不会看提示**：看 Hint 或 Writeup 的思路部分
3. **自己复现一遍**：看懂答案后关掉，自己再做一遍
4. **总结归纳**：整理到笔记中，包括：
   - 漏洞原理
   - 利用条件
   - Payload 模板
   - 绕过技巧
   - 防御方法

**5.3 时间管理**

- **番茄工作法**：25 分钟专注学习 + 5 分钟休息
- **每天固定时间**：养成习惯，每天至少 1 小时
- **周复习**：每周回顾本周学习内容
- **月总结**：每月做一次全面总结，查漏补缺
