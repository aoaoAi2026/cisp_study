# 护网应急预案与演练方案

---

## 一、应急预案体系

### 1.1 预案矩阵

```
护网五大核心应急预案：

① Web入侵应急预案：
   触发：WAF/IPS告警 + 服务器Webshell检测
   响应：隔离受影响主机 → 取证(Webshell分析/后门排查) →
         入口修复 → 关联主机排查 → 上报

② 勒索病毒应急预案：
   触发：EDR告警 + 文件被加密
   响应：立即断网(拔网线/防火墙隔离) → 
         停止加密扩散 → 判断勒索家族(ID Ransomware) →
         备份恢复(如有) → 病毒清除+加固

③ DDoS攻击应急预案：
   触发：流量监控告警 + 业务不可用
   响应：联系ISP/云厂商清洗 → 启用CDN/高防切换 →
         WAF限流策略 → 攻击IP溯源

④ 数据泄露应急预案：
   触发：DLP告警 + 异常大流量外传
   响应：阻断出站连接 → 范围评估 → 
         通知法务/隐私团队(PIPL/GDPR) → 证据固定

⑤ 钓鱼事件应急预案：
   触发：用户上报 + 邮件网关告警
   响应：全量删除同发件人邮件 → 恶意URL封禁 →
         受影响账号重置密码 → 全员通知
```

### 1.2 指挥链

```
ICS (Incident Command System) 应急指挥结构：

  Incident Commander (IC) — 总指挥(安全负责人/CSO)
       │
   ┌───┼───┐
   ▼   ▼   ▼
  技术组  沟通组  业务组
  (Tier2/3) (PR/法务) (业务Owner)

IC职责：
  ✓ 决策 → 加固还是隔离？通知还是不通知？
  ✓ 资源协调 → 调动所有必要资源
  ✓ 信息统一出口 → 避免多头对外
  ✓ 定时(每30分钟)内部通报进展
```

## 二、应急演练方案

### 2.1 桌面推演 (Tabletop)

```
桌面推演 = 纸上谈兵，讨论"如果发生X, 我们应该怎么做"

演练步骤：
  ① 编写场景脚本（如：某天凌晨3点，监控告警某台DB服务器大量外传数据）
  ② 召集应急团队(IC+技术+沟通+业务)
  ③ 逐个场景推进："你现在发现/知道什么？你下一步做什么？"
  ④ 讨论每个决策和分歧点
  ⑤ 记录过程 → 复盘 → 优化预案

桌面推演评分卡：
┌──────────────┬──────┬──────┬──────┐
│ 评估维度       │ 目标 │ 实际 │ 得分 │
├──────────────┼──────┼──────┼──────┤
│ 检测时间       │ <15min│     │     │
│ 事件升级判定   │ <5min │     │     │
│ 关键决策质量   │ 无遗漏│     │     │
│ 沟通顺畅度     │ 信息及时│    │     │
│ 外部通报(如需)  │ <1h  │     │     │
└──────────────┴──────┴──────┴──────┘
```

### 2.2 实战演练 (Live Fire)

```
实战演练 = 在受控环境中"真的打"

常见形式：
  1. 红蓝对抗 → 红队模拟真实攻击
  2. 钓鱼测试 → 向员工发送仿真钓鱼邮件
  3. DDoS演练 → 在测试环境进行压力测试

护网实战演练注意事项：
  ⚠️ 必须与业务确认时间窗口（非业务高峰期）
  ⚠️ 必须有"急停"机制（一键终止演练）
  ⚠️ 必须有回滚方案（演练后恢复原状）
  ⚠️ 必须提前通知必要的接口人
  ⚠️ 演练 ≠ 惩罚（不应因演练扣绩效）
```

## 三、信息通报流程

```
事件分级与通报：

P1 (严重 — 数据泄露/核心系统沦陷)：
  通报：IC + CSO + CEO + 法务 + PR
  外部：2小时内通报监管部门(网信办/公安)
  每小时更新一次进展

P2 (高 — 重要系统被入侵/勒索病毒)：
  通报：IC + CSO + 受影响业务负责人
  外部：根据法规要求决定是否通报
  每2小时更新

P3 (中 — 钓鱼邮件/WAF告警)：
  通报：IC + 安全团队
  内部工单流转

信息通报模板：
  【安全事件通报】
  事件等级：P1
  事件类型：疑似数据泄露
  影响范围：XX系统，约X万条用户数据
  当前状态：已隔离受影响系统，正在取证分析
  下一步：预计2小时内完成初步分析
  联系人：张三 138xxxx
```
