```
1. Web 入侵应急预案
2. 勒索病毒应急预案
```

## 二、五大预案

### Web 入侵预案

```
触发条件:
  ✦ WAF/IPS 检测到攻击payload
  ✦ EDR 发现 WebShell
  ✦ 网站被篡改
  ✦ SIEM 告警: 异常文件上传

响应流程:
  ① 确认入侵: 是真实攻击还是误报?
     → 检查Web日志 → 确认攻击payload
     → 检查服务器 → 是否有WebShell?

  ② 遏制 (15分钟内):
     → 如确认WebShell → 隔离受感染主机
     → 从备份恢复被篡改文件
     → 封禁攻击源IP (防火墙临时规则)

  ③ 根除 (1小时内):
     → 删除所有WebShell文件
     → 修复漏洞(上传过滤/参数化查询/WAF规则)
     → 检查入侵入口

  ④ 恢复:
     → 从安全备份恢复
     → 验证服务正常

关键命令:
  # 查找 Webshell
  grep -rn "eval(" /var/www/ --include="*.php" | grep -v vendor
  find /var/www/ -name "*.php" -mtime -1
  
  # 日志溯源
  grep -E "POST|PUT" /var/log/nginx/access.log | tail -50
```

### 勒索病毒预案

```
触发条件:
  ✦ 文件后缀异常(.encrypted/.lockbit)
  ✦ 出现勒索信
  ✦ EDR 检测到大量文件修改

响应流程:
  ① 立即断网! (物理拔网线或防火墙deny all)
  ② 确定感染范围 (多少台? 哪些服务器?)
  ③ 识别勒索家族 → 找解密工具
  ④ 检查备份可用性
  ⑤ 从备份恢复 (不支付赎金!)
  ⑥ 溯源入口 → 修复 → 加固

关键命令:
  # 找加密文件
  find / -name "*.lockbit" -o -name "*.encrypted" 2>/dev/null
  
  # 断网隔离
  iptables -P INPUT DROP; iptables -P FORWARD DROP
```

### DDoS 预案

```
触发条件:
  ✦ 流量监控告警(突发流量)
  ✦ 业务不可用
  ✦ CDN/云监控告警

响应流程:
  ① 联系ISP/云厂商 → 启用流量清洗
  ② CDN切换 → 启用高防IP
  ③ WAF限流策略 → 启用CC防护
  ④ 分析攻击类型(SYN Flood? HTTP Flood?)
  ⑤ 记录攻击源 → 留存证据
```

### 数据泄露预案

```
触发条件:
  ✦ DLP 告警(大量文件外传)
  ✦ 异常出站流量(NetFlow)
  ✦ 数据库审计告警(大量导出)

响应:
  ① 阻断出站连接 (防火墙封禁目标IP)
  ② 隔离外泄源主机
  ③ 评估泄露范围(数据/用户/时间)
  ④ 通知管理层 + 法务
  ⑤ 如涉及个人信息 → 通知监管 + 用户
```

### 钓鱼事件预案

```
触发条件:
  ✦ 员工上报可疑邮件
  ✦ 邮件网关告警

响应:
  ① 提取IOC(发件人/URL/附件Hash)
  ② 全量删除同发件人邮件
  ③ 封禁恶意URL (防火墙/DNS)
  ④ 排查点击者 → 检查终端
  ⑤ 重置受影响密码
  ⑥ 全员通知 (提醒不要点类似邮件)
```

## 三、指挥链

```
ICS (Incident Command System) 结构:

  Incident Commander (IC) — CSO/安全负责人
       │              │              │
   技术组          沟通组          业务组
   (Tier2/3)      (PR/法务)       (受影响部门)

IC 职责:
  ✦ 决策: 隔离? 加固? 通知?
  ✦ 资源: 调配所需人员/工具

  ✦ 定时通报: 每30分钟内部通报进展
```

## 四、桌面推演

```
桌面推演 = 假设场景 → 口头推演 → 找问题

推演步骤:
  ① 编写场景脚本
    例: "周五下午3点，监控告警: 某台服务器CPU 100%，
        发现 /tmp/.cache/ 下有未知进程，连接境外IP..."

  ② 召集应急团队 (IC + 技术 + 业务)

  ③ 场景推进:
    "你现在发现了什么?" → 技术组回答
    "你下一步做什么?" → 技术组回答
    "需要通知谁?" → IC决定
    "如果备份也被加密了?" → 引发讨论

  ④ 记录分歧 → 会后统一 → 更新预案

推演评分:
┌──────────────┬──────┬──────┐
│ 指标          │ 目标  │ 得分 │
├──────────────┼──────┼──────┤
│ 发现时间       │ <15m │      │
│ 事件定级       │ <5m  │      │
│ 遏制措施       │ 正确  │      │
│ 沟通协调       │ 顺畅  │      │
│ 决策质量       │ 合理  │      │
└──────────────┴──────┴──────┘
```

## 五、实战演练

```
Live Fire = 真的"打"

常见形式:
  1. 红蓝对抗 → 红队模拟真实攻击
  2. 钓鱼测试 → 全员仿真钓鱼邮件
  3. DDoS 演练 → 测试环境压测

护网实战演练安全措施:
  ⚠️ 必须与业务确认时间窗口
  ⚠️ 必须有急停机制 (一键终止)
  ⚠️ 必须有回滚方案 (演练后恢复)
  ⚠️ 事前通知必要接口人
  ⚠️ 演练 ≠ 惩罚 (不扣绩效!)

演练检查单:
  □ 时间窗口确认 (业务低峰期)
  □ 急停方案就绪
  □ 回滚方案确认
  □ 通知受影响方
  □ 演练后复盘会安排
```

## 六、信息通报

```
事件分级:

P1 (严重 — 数据泄露/核心系统沦陷):
  通报: IC→CSO→CEO→法务→PR
  外部: 2小时内通报网信办/公安
  频率: 每30分钟更新进展

P2 (高 — 重要系统被入侵):
  通报: IC→CSO→受影响业务
  频率: 每2小时更新

P3 (中 — 普通安全事件):
  通报: IC→安全团队
  内部工单流转

通报模板:
  【安全事件通报】
  等级: P1
  类型: 疑似数据泄露
  时间: 2026-06-15 14:30
  影响: XX系统，约X万条数据
  状态: 已隔离，正在取证
  IC: 张三 138xxxx
```

## 八、完整案例

```
某金融企业护网 DESKTOP-ACCT-05 应急响应

Day 1 03:15 — 告警
  EDR: 发现可疑进程执行 (powershell -enc ...)
  主机: DESKTOP-ACCT-05
  → P1 告警 (护网期间所有外网告警=P1)

03:20 — 确认
  分析进程链: WINWORD.EXE → cmd.exe → powershell -enc
  → 确认: 宏文档下载执行 (真阳性)

03:22 — 遏制
  ✓ 安全组隔离 DESKTOP-ACCT-05
  ✓ 提取PowerShell编码 → 发现C2: 45.xxx.xxx.xxx
  ✓ 防火墙封禁 C2 IP

03:30 — 关联
  ✓ 45.xxx.xxx.xxx 还连接了 DESKTOP-ACCT-08
  ✓ 同一模版钓鱼邮件发给了 12 人

03:45 — 扩大
  ✓ 隔离 DESKTOP-ACCT-08
  ✓ 收件箱搜索 → 删除所有同主题邮件
  ✓ 全员警告

04:00 — 复盘
  根因: 缺少邮件附件沙箱检测
  整改: 启用附件沙箱 + 禁止宏执行

从告警→遏制: 7分钟
护网加分项: 快速响应 + 成功阻止横向扩散
```
