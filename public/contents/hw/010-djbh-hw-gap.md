## 1. 等保与护网的关系

护网中扣分 ≠ 等保不达标，但**等保达标是护网防守的底线**。

```
关系：
等保 = 最低安全标准（60分及格）
护网 = 实战检验（90分才是目标）

常见误区：过了等保测评 = 护网没问题 → 大错！

差距 = 护网要求 - 等保要求 = 从"合规"到"实战"的鸿沟
```

## 2. 差距分析方法论

### 2.1 分析维度

| 维度 | 等保要求 | 护网额外要求 |
|------|---------|-------------|
| 检测 | 有IDS/IPS即可 | 需EDR+NDR+SIEM联动 |
| 响应 | 有应急预案即可 | 需7x24值班+SOAR+5分钟响应 |
| 日志 | 保留6个月 | 需实时分析+关联 |
| 加固 | 安全基线达标 | 纵深防御+蜜罐 |
| 人员 | 有安全管理员 | 需专门的安全运营团队 |

### 2.2 差距检查脚本

```bash
#!/bin/bash
# djbh_hw_gap_check.sh - 等保vs护网差距自动化检查

echo "============================================"
echo "  等保vs护网差距检查"
echo "  时间: $(date)"
echo "============================================"

GAPS_FOUND=0

check() {
    local category="$1"
    local item="$2"
    local djbh_req="$3"
    local hw_req="$4"
    local current="$5"

    if [[ "$current" != "PASS" ]]; then
        echo ""
        echo "🔴 [${category}] ${item}"
        echo "   等保要求: ${djbh_req}"
        echo "   护网额外: ${hw_req}"
        echo "   当前状态: 不满足"
        GAPS_FOUND=$((GAPS_FOUND+1))
    fi
}

# 检测类差距
echo ""
echo "--- 检测类差距 ---"

# 1. EDR部署
edr_count=$(grep -c "active" /var/ossec/logs/alerts/alerts.json 2>/dev/null || echo "0")
check "检测" "EDR/HIDS全量覆盖" \
    "等保: 主机入侵防范" \
    "护网: EDR/HIDS ≥99%在线率，阻断模式" \
    "$([ $edr_count -gt 100 ] && echo "PASS" || echo "FAIL")"

# 2. NDR部署
ndr_status=$(systemctl is-active zeek 2>/dev/null || echo "inactive")
check "检测" "NDR网络流量分析" \
    "等保: 通信网络审计" \
    "护网: NDR全流量分析+DNS隧道检测" \
    "$([ "$ndr_status" = "active" ] && echo "PASS" || echo "FAIL")"

# 响应类差距
echo ""
echo "--- 响应类差距 ---"

# 3. SOAR自动化
soar_status=$(docker ps 2>/dev/null | grep shuffle | wc -l)
check "响应" "SOAR自动化编排" \
    "等保: 应急预案+处置流程" \
    "护网: SOAR≥60%自动化率+Playbook部署" \
    "$([ $soar_status -gt 0 ] && echo "PASS" || echo "FAIL")"

# 4. 7x24值班
check "响应" "7x24三班倒值守" \
    "等保: 工作日工作时间响应" \
    "护网: 7x24三班倒+15分钟SLA" \
    "FAIL"  # 如果日常没有7x24

# 防护类差距
echo ""
echo "--- 防护类差距 ---"

# 5. 蜜罐部署
honeypot_count=$(docker ps 2>/dev/null | grep -E "cowrie|dionaea|glastopf|hfish|tpot" | wc -l)
check "防护" "三层蜜罐部署" \
    "等保: 无要求" \
    "护网: 外网/DMZ/内网三层蜜罐" \
    "$([ $honeypot_count -ge 2 ] && echo "PASS" || echo "FAIL")"

# 6. 威胁情报
check "防护" "威胁情报实时联动" \
    "等保: 漏洞库+恶意代码库" \
    "护网: 商业/开源TI平台+MISP+IOC自动同步" \
    "$(curl -s http://misp:80 2>/dev/null >/dev/null && echo "PASS" || echo "FAIL")"

echo ""
echo "============================================"
echo "  发现差距: ${GAPS_FOUND} 项"
echo "============================================"
```

## 3. 常见差距Top 10

| # | 差距 | 等保现状 | 护网需要 | 修复难度 |
|---|------|---------|---------|---------|
| 1 | **EDR覆盖** | 仅核心服务器有AV | 全量EDR+HIDS，≥99%在线 | ⭐⭐ |
| 2 | **NDR缺失** | 无流量分析 | Zeek/Suricata全流量 | ⭐⭐⭐ |
| 3 | **SOAR缺失** | 手工处置 | 自动化≥60% | ⭐⭐⭐ |
| 4 | **蜜罐缺失** | 无 | 三层蜜罐 | ⭐⭐ |
| 5 | **7x24值班** | 仅工作时间 | 三班倒 | ⭐⭐⭐⭐ (人力) |
| 6 | **威胁情报** | 无或被动接收 | 主动订阅+自动化 | ⭐⭐ |
| 7 | **日志完整度** | 仅安全设备 | 全量日志(含应用+DB) | ⭐⭐ |
| 8 | **漏洞管理** | 季度/年度扫描 | 持续扫描+修复SLA | ⭐⭐⭐ (流程) |
| 9 | **红蓝演练** | 无 | 季度红蓝对抗 | ⭐⭐⭐⭐ |
| 10 | **安全意识** | 年度培训 | 季度培训+钓鱼测试 | ⭐ |
