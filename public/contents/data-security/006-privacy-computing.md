## 二、联邦学习 (Federated Learning)

### 2.1 横向联邦学习

```
场景：各参与方持有相同特征空间、不同样本的数据
示例：多家银行用户特征相同，但客户群体不同

架构：
  中心服务器
  │     │     │
客户端A 客户端B 客户端C

流程：
1. 服务器初始化全局模型并下发
2. 各客户端用本地数据训练，只上传模型参数/梯度
3. 服务器聚合参数（FedAvg算法）更新全局模型
4. 迭代至收敛

FedAvg算法（核心）：
  w_global = Σ(n_k / n_total) * w_k
  (按客户端数据量加权平均)
```

### 2.2 纵向联邦学习

```
场景：各参与方持有相同样本、不同特征空间的数据
示例：银行(A有金融特征) + 电商(B有行为特征) → 联合风控

流程：
1. 基于加密实体对齐（隐私集合求交, PSI）匹配共同用户
2. 各方分别训练本地子模型
3. 通过加密中间结果交换计算联合损失与梯度
4. 各方更新各自子模型

特点：训练过程中原始特征不出域
```

## 三、多方安全计算 (MPC)

### 3.1 核心概念

```
多方安全计算定义：
n个参与方 P1, P2, ..., Pn 各自持有私有数据 x1, x2, ..., xn，
共同计算函数 f(x1, x2, ..., xn)，
使得：
  - 正确性：各自得到正确的计算结果
  - 隐私性：各自不能获得其他参与方的私有数据

安全模型：
- 半诚实模型 (Semi-honest)：参与方遵循协议但好奇窥探
- 恶意模型 (Malicious)：参与方可能任意偏离协议
```

### 3.2 三大主流技术

#### (1) 秘密共享 (Secret Sharing)

```
Shamir秘密共享（(t, n)门限方案）：
- 将秘密S拆分为n个份额（shares）
- 任意t个份额可重构S
- 任意少于t个份额无法获得S的任何信息

加法秘密共享：
  秘密x → 随机拆分 x1 + x2 = x (mod p)
  A持有x1，B持有x2
  计算x+y：A计算 x1+y1, B计算 x2+y2 → 各自本地计算
  计算x*y：需要交互（Beaver三元组）

示例（2方加法）：
  A: x=5 → x1=3, x2=2（x1给B）
  B: y=8 → y1=5, y2=3（y2给A）
  A: z1=x1+y2=3+3=6
  B: z2=x2+y1=2+5=7
  结果: z1+z2=13 = 5+8 ✓
```

#### (2) 不经意传输 (Oblivious Transfer, OT)

```
1-out-of-2 OT：
  发送方: 持有消息 m0, m1
  接收方: 持有选择比特 b ∈ {0, 1}
  协议结束: 接收方获得 m_b，但不知 m_{1-b}
            发送方不知接收方选择了哪个

用于MPC：实现安全比较、安全选择、安全乘法等基本操作
```

#### (3) 混淆电路 (Garbled Circuit)

```
步骤：
1. 将函数编译为布尔电路
2. 混淆方（Garbler）对电路每根导线生成两个随机标签
   （对应0和1），混淆真值表
3. 评估方（Evaluator）用不经意传输获取自己输入对应的标签
4. 评估方逐门计算，最终得到输出标签
5. 输出映射回明文结果

性能：通信量大，适合小规模计算
```

## 四、可信执行环境 (TEE)

### 4.1 主流TEE技术

| TEE | 厂商 | 信任根 | 特点 |
|-----|------|--------|------|
| **Intel SGX** | Intel | Intel CPU | 飞地(Enclave)内存加密，支持128MB/512GB |
| **Intel TDX** | Intel | Intel CPU | 虚拟机级机密计算 |
| **AMD SEV-SNP** | AMD | AMD CPU | 虚拟机加密，内存完整性保护 |
| **ARM TrustZone** | ARM | ARM CPU | 移动/IoT设备 |
| **NVIDIA Confidential Computing** | NVIDIA | GPU | GPU机密计算(A100/H100) |
| **海光CSV** | 海光 | 国产CPU | 国密SM2/SM3/SM4加密 |
| **鲲鹏TrustZone** | 华为 | ARM | 华为服务器TEE |

### 4.2 SGX编程模型

```c
// Intel SGX Enclave基本结构
enclave {
    // 可信代码：运行在Enclave内部
    trusted {
        public int secure_compute([in] void* encrypted_data, 
                                   [out] void* result);
        
        // 密封(Sealing)：加密存储到外部
        public int seal_data([in] void* data, [out] void* sealed);
    };
    
    // 不可信代码：Enclave外部调用入口
    untrusted {
        // 只能通过ECALL/OCALL接口交互
    };
}

// 远程证明 (Remote Attestation)
// 1. Enclave生成度量报告(Measurement: MRSIGNER/MRENCLAVE)
// 2. 发送给Intel Attestation Service (IAS) 验证
// 3. 验证通过 → 建立安全通道 → 传输密钥/数据
```

## 五、技术对比与选型

| 维度 | 联邦学习 | MPC | TEE |
|------|---------|-----|-----|
| 计算性能 | ★★★★ | ★ | ★★★★★ |
| 通信开销 | ★★★ | ★ | ★★★★★ |
| 安全模型 | 半诚实 | 恶意/半诚实 | 硬件安全 |
| 适用规模 | 大 | 小-中 | 中-大 |
| 算法灵活度 | ML专用 | 任意函数 | 任意程序 |
| 合规认可度 | ★★★ | ★★★★ | ★★★★ |
| 成熟度 | ★★★★ | ★★★ | ★★★ |

### 选型建议

```
需要AI/ML联合训练 → 联邦学习（FATE/SecretFlow）
需要精准统计/查询 → MPC（CrypTen/SPDZ）
需要通用计算 + 高性能 → TEE（SGX/TDX/SEV）
最高安全需求 → MPC
最广泛适用性 → 联邦学习 + TEE 混合方案
```

## 六、行业实践案例

### 6.1 金融风控

```
场景：多家银行联合风控模型
方案：纵向联邦学习
工具：FATE
效果：在不泄露各银行客户数据的前提下，提升联合风控准确率

### 6.2 医疗科研

```
场景：多家医院联合疾病预测模型
方案：横向联邦学习
工具：OpenFL / NVIDIA FLARE
效果：联合多家医院数据训练，模型效果显著优于单中心
```

### 6.3 政务数据共享

```
场景：多部门联合统计分析（不泄露明细数据）
方案：MPC
工具：隐语(SecretFlow) / ABY
效果：实现跨部门数据"可用不可见"的联合分析
```

### 6.4 隐私计算与AI融合

```
场景：大模型微调/推理阶段的数据隐私保护
方案：TEE (NVIDIA Confidential Computing + H100)
效果：在GPU TEE中处理敏感数据，防止模型方/云厂商窃取数据

## 高分考点与知识巧记

> 🔑 **高分考点**：隐私计算是数据安全前沿技术考点，侧重三大技术路线对比和适用场景选型。联邦学习、MPC、TEE 的核心差异是高频考点。

| 考点 | 频次 | 核心记忆点 |
|:---|:---:|:---|
| 三大技术路线 | ⭐⭐⭐⭐⭐ | 联邦学习(FL)数据不动模型动、MPC 密码学协议、TEE 硬件隔离 |
| FL 分类 | ⭐⭐⭐⭐ | 横向联邦(特征相同)、纵向联邦(样本相同)、联邦迁移学习 |
| MPC 安全模型 | ⭐⭐⭐⭐ | 半诚实模型(遵循协议但好奇)、恶意模型(可任意偏离) |
| TEE 原理 | ⭐⭐⭐ | 硬件级可信执行环境(Intel SGX/AMD SEV/ARM TrustZone) |
| 场景选型 | ⭐⭐⭐⭐ | 联合建模→FL、联合查询→MPC、密钥保护→TEE |

> 💡 **知识巧记**：三大路线记"联方信"——联邦学习数据不动模型动、多方安全计算密码学协议、可信执行环境硬件隔离。FL 分类记"横特纵样"——横向特征同、纵向样本同。MPC 两模型记"半诚实守规矩但好奇，恶意模型可任意偏离"。

### 高分考点速查表

| 考察维度 | 关键结论 | 常见干扰项 |
|:---|:---|:---|
| FL 核心思想 | 数据不出本地，模型参数聚合 | "FL 需要把数据集中到一起" ❌ |
| MPC vs FL | MPC 用密码学协议，FL 用分布式训练 | "MPC 和 FL 是一回事" ❌ |
| TEE 信任根 | 依赖硬件厂商(Intel/AMD/ARM) | "TEE 完全不需要信任硬件厂商" ❌ |
| 半诚实 vs 恶意 | 半诚实遵循协议，恶意可攻击 | "半诚实模型足够安全" ❌ |

### 知识巧记口诀

> **隐私计算口诀**：
> 三大路线联方信，数据可用不可见。
> 联邦学习模型动数据不动，横向特征纵样本连。
> MPC 密码协议多方算，半诚实守规恶意偏。
> TEE 硬件隔离可信执行，SGX/SEV 信任根牵。
> 场景选型联合建模选 FL，联合查询 MPC 先。
