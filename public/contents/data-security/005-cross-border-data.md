## 二、数据出境概念

### 2.1 什么算"数据出境"

```
数据出境的三类情形：
1. 境内运营中收集和产生的数据传输、存储至境外
   例：境内用户数据 → 海外AWS服务器

2. 境内收集的数据存储在境内，但境外的机构/组织/个人可以查询/调取/下载/导出
   例：境内数据库 → 境外母公司VPN访问

3. 国家网信办规定的其他数据出境行为
   例：数据虽未出境但被境外主体实质控制
```

### 2.2 什么不算"数据出境"

```
- 仅在境内传输、存储、处理（境外不可访问）
- 境外处理境内自然人个人信息的活动，但该活动：
  · 不涉及境内个人信息处理者
  · 属于境外直接收集（如境外网站直接面向境外用户）
```

## 三、安全评估流程

### 3.1 评估流程图

```
┌────────────────────┐
│  1. 数据出境自评估  │ ← 企业自行完成
│  (数据出境安全风险  │
│   自评估报告)       │
└────────┬───────────┘
         ▼
┌────────────────────┐
│  2. 向省级网信办    │ ← 提交申报材料
│     提交申报        │
└────────┬───────────┘
         ▼
┌────────────────────┐
│  3. 省级网信办      │
│     完备性查验      │ ← 7个工作日内
│  通过？─否→补正    │
└────────┬───────────┘
         ▼ 是
┌────────────────────┐
│  4. 国家网信办      │
│     组织安全评估    │ ← 45个工作日内（可延长）
│  (一般情况)         │
└────────┬───────────┘
         ▼
┌────────────────────┐
│  5. 评估结果        │
│  通过→有效期3年     │
│  不通过→不得出境    │
└────────────────────┘
```

### 3.2 申报材料清单

```
1. 数据出境安全评估申报书（含承诺书）
2. 数据出境风险自评估报告
3. 数据处理者与境外接收方拟订立的法律文件
   （数据处理协议/标准合同/集团数据跨境协议等）
4. 安全评估工作需要的其他材料

格式要求：
- 纸质版 + 电子版（光盘）
- 所有材料需加盖公章
- 外文材料需附中文翻译件
```

## 四、自评估报告六大要点

### 4.1 报告框架

```
一、数据出境的基本情况
  1.1 数据处理者基本信息
  1.2 数据出境涉及业务和信息系统
  1.3 数据出境目的、范围、方式
  1.4 出境数据类型、规模、频次
  1.5 境外接收方基本情况

二、数据出境的影响评估
  2.1 对国家安全的影响
  2.2 对公共利益的影响
  2.3 对个人/组织合法权益的影响

三、数据安全保障能力评估
  3.1 数据处理者安全保障能力
  3.2 境外接收方安全保障能力
  3.3 数据安全事件应急响应能力

四、双方数据安全责任义务
  4.1 合同/协议中的数据安全条款
  4.2 违约与救济机制

五、数据安全风险综合评估
  5.1 风险识别
  5.2 风险分析
  5.3 风险等级判定

六、评估结论与建议
```

### 4.2 重点评估要素

```
数据出境必要性：
- 业务必要性论证（是否可选择境内方案？）
- 最小必要原则（出境字段是否为最小集合？）

境外接收方评估：
- 境外接收方所在国数据安全法律环境
- 境外接收方数据处理安全管理制度
- 境外接收方数据安全保障技术措施（加密/访问控制/审计/备份）
- 境外接收方历史数据安全事件记录

数据全生命周期保护：
- 收集：合法性基础、最小必要
- 传输：加密传输（TLS 1.3 / 国密SM2/SM4）
- 存储：加密存储、数据隔离
- 使用：最小授权、脱敏展示
- 删除：出境目的达成后的数据删除机制
```

## 五、标准合同备案（替代方案）

### 5.1 适用条件（非CIIO + 同时满足）

```
✅ 非关键信息基础设施运营者
✅ 处理个人信息不满100万人
✅ 自上年1月1日起累计向境外提供未满10万人个人信息
✅ 自上年1月1日起累计向境外提供未满1万人敏感个人信息

→ 可签订标准合同并备案，替代安全评估
```

### 5.2 标准合同核心条款

```
个人信息出境标准合同（网信办制式文本）：

第二条：个人信息处理者和境外接收方的基本情况
第三条：个人信息出境说明（类型/规模/频次/目的/方式/期限）
第四条：个人信息处理者和境外接收方的义务
  - 个人信息处理者义务：告知+同意+影响评估+个人信息保护
  - 境外接收方义务：最小必要+安全保障+应急响应+
    数据再转移限制+配合监管+个人信息主体权利保障
第五条：境外接收方所在国政策法规变化影响
第六条：个人信息主体权利
第七条：救济
第八条：合同解除
第九条：违约责任
第十条：生效、变更、终止
```

## 六、实操Checklist

### 6.1 数据出境摸底

- [ ] 梳理所有境外数据传输场景（业务系统/API/邮件/VPN/云服务）
- [ ] 逐场景确认出境数据类型（个人信息/重要数据/一般数据）
- [ ] 统计出境数据量级（个人信息数量/敏感个人信息数量）
- [ ] 识别境外接收方（母公司/供应商/SaaS服务商/境外分支机构）
- [ ] 判断CIIO身份（是否为关键信息基础设施运营者）

### 6.2 路径选择

- [ ] 是否触发安全评估？（CIIO/100万+/10万+/1万敏感）
- [ ] 是否适用标准合同？（不触发安全评估的个信出境）
- [ ] 是否需要个人信息保护认证？（跨国公司内部传输）
- [ ] 确定申报/备案路径

### 6.3 安全评估申报

- [ ] 完成数据出境风险自评估报告
- [ ] 起草/审查与境外接收方的法律文件
- [ ] 准备申报材料（纸质+电子版+翻译件）
- [ ] 法务/合规部门内部审核
- [ ] 向省级网信办提交申报
- [ ] 跟进进度（完备性查验→评估→结果）
- [ ] 评估通过后持续合规（出境后监管/重新评估触发条件监控）

## 七、常见问题Q&A

```
Q: 跨国公司内部数据传输是否需要安全评估？
A: 只要构成"数据出境"就需评估。跨国公司在华机构向境外总部
   传输个人信息，如触发条件，需申报安全评估。

Q: 使用境外云服务（AWS/Azure/GCP中国区以外）算数据出境吗？
A: 如果数据存储在境外数据中心，即构成数据出境。选择海外Region
   时需特别关注。

Q: 安全评估有效期多长？
A: 3年。有效期内出境情况变化需重新评估（目的/范围/类型变化、
   接收方所在国法律环境重大变化等）。

Q: 标准合同签订后多久需要备案？
A: 合同生效之日起10个工作日内向省级网信办备案。

Q: 境外接收方拒绝签署合同怎么办？
A: 不得向其提供个人信息。需寻找替代方案（境内本地化处理、
   更换供应商等）。
```

## 高分考点与知识巧记

> 🔑 **高分考点**：数据出境安全评估是数据合规的高频考点。三条出境路径、安全评估触发条件、标准合同备案时限是核心记忆点。

| 考点 | 频次 | 核心记忆点 |
|:---|:---:|:---|
| 三条出境路径 | ⭐⭐⭐⭐⭐ | 安全评估(网信办)、标准合同(备案)、认证(专业机构) |
| 安全评估触发 | ⭐⭐⭐⭐ | 重要数据/关键信息基础设施运营者/处理100万人以上个人信息 |
| 标准合同备案 | ⭐⭐⭐⭐ | 合同生效之日起 10 个工作日内向省级网信办备案 |
| 评估有效期 | ⭐⭐⭐ | 3 年，情况变化需重新评估 |
| PIC/PII 区分 | ⭐⭐⭐ | 重要数据必须评估，普通个人信息可选标准合同 |

> 💡 **知识巧记**：数据出境三路径记"评合认"——安全评估、标准合同、认证。安全评估触发条件记"重关百"——重要数据、关键信息基础设施、处理 100 万人以上。备案时限记"十工作日"——合同生效后 10 个工作日。

### 高分考点速查表

| 考察维度 | 关键结论 | 常见干扰项 |
|:---|:---|:---|
| 安全评估触发 | 重要数据/CIIO/100万人以上个人信息 | "所有数据出境都需安全评估" ❌ |
| 标准合同适用 | 非 CIIO + 处理不足 100 万人 | "CIIO 也可用标准合同" ❌ |
| 评估有效期 | 3 年，变化需重评 | "评估一次永久有效" ❌ |
| 备案时限 | 10 个工作日内向省级网信办 | "30 天内备案即可" ❌ |

### 知识巧记口诀

> **数据出境合规口诀**：
> 出境三路评合认，重关百必安全评估。
> 标准合同十工作日内备案，认证走专业机构。
> 评估有效期三年整，情况变化重新评。
> CIIO 和重要数据不可绕，境外接收方必须签。
