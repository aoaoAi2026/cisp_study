## 二、《数据安全法》核心制度

### 2.1 适用范围

```
境内数据处理活动 + 境外处理境内数据（损害国家安全/公共利益/公民权益的）
```

### 2.2 数据分类分级制度

| 级别 | 定义 | 示例 | 保护要求 |
|------|------|------|----------|
| 一般数据 | 非敏感、非重要 | 公开信息、内部通知 | 基本安全防护 |
| 重要数据 | 可能危害国家安全/经济运行/社会稳定 | 关键基础设施数据、基因数据、地理信息 | 重点保护+风险评估 |
| 核心数据 | 危害国家安全、国民经济命脉 | 国家秘密范围、核工业数据 | 最严格保护 |

### 2.3 数据安全保护义务

```
企业必须建立：
1. 数据安全管理制度 + 操作规程
2. 数据安全负责人/管理机构
3. 数据安全教育培训
4. 数据安全技术防护（加密/脱敏/审计/备份）
5. 数据安全风险评估与报告机制
6. 数据安全事件应急预案
```

### 2.4 数据出境安全评估

```
条件（满足任一即触发）：
- 向境外提供「重要数据」
- 关键信息基础设施运营者向境外提供个人信息
- 处理100万人以上个人信息者向境外提供个人信息
- 自上年1月1日起累计向境外提供10万人以上个人信息
```

## 三、《个人信息保护法》核心制度

### 3.1 个人信息处理原则

```
合法性基础（满足任一）：
☑ 取得个人同意
☑ 为订立/履行合同所必需
☑ 为履行法定职责/法定义务所必需
☑ 为应对突发公共卫生事件
☑ 紧急情况下保护自然人生命健康和财产安全
☑ 为公共利益实施新闻报道/舆论监督
☑ 在合理范围内处理已公开的个人信息
```

### 3.2 敏感个人信息

```
敏感个人信息（需单独同意 + 充分必要性说明）：
- 生物识别信息（人脸/指纹/声纹/掌纹/虹膜）
- 宗教信仰/特定身份/行踪轨迹
- 医疗健康/金融账户/不满14周岁未成年人信息
- 一旦泄露或非法使用，容易导致人格尊严受损或人身财产安全受危害
```

### 3.3 个人信息主体权利

| 权利 | 内容 |
|------|------|
| 知情权/决定权 | 知悉处理规则，自主决定 |
| 限制/拒绝权 | 有权限制或拒绝处理 |
| 查阅/复制权 | 查阅/复制个人信息 |
| 更正权 | 发现不准确/不完整→有权更正 |
| 删除权 | 特定情形下要求删除 |
| 可携带权 | 将个人信息转移至指定处理者 |

## 四、企业合规落地路径

### 4.1 合规路线图

```
Phase 1: 摸底排查（1-2月）
├── 数据资产盘点（依据GB/T 38667/37973）
├── 个人信息处理活动梳理
├── 数据流图绘制（收集→存储→使用→加工→传输→提供→公开→删除）
└── 现有制度/合同/协议合规差距分析

Phase 2: 制度建设（2-4月）
├── 数据分类分级管理制度
├── 数据安全管理制度 + 操作规程
├── 个人信息保护管理制度
├── 数据安全事件应急预案
├── 数据出境安全评估制度
└── 第三方数据处理合同模板

Phase 3: 技术落地（3-6月）
├── 数据分类分级标签平台
├── DLP数据防泄漏系统
├── 数据加密（传输+存储）/脱敏平台
├── 数据库审计/日志审计
├── 数据出境监测系统
└── 隐私计算平台（可选）

Phase 4: 持续运营
├── 定期合规审计（年度）
├── 数据安全风险评估
├── 员工安全培训
├── DPIA（个人信息保护影响评估）
└── 数据安全事件演练
```

### 4.2 组织保障

```
数据安全委员会
├── 数据安全负责人（管理层）
├── 数据安全管理部门
│   ├── 数据分类分级管理岗
│   ├── 数据安全技术岗
│   ├── 数据安全审计岗
│   └── 个人信息保护专员（DPO）
└── 各业务部门数据安全联络人
```

## 五、违法后果与案例

### 5.1 处罚力度

| 法律 | 罚款上限 | 其它处罚 |
|------|---------|---------|
| 网络安全法 | 100万元 | 责令暂停相关业务、停业整顿、关闭网站 |
| 数据安全法 | 1000万元 | 责令暂停相关业务、停业整顿、吊销许可证 |
| 个人信息保护法 | 5000万元/上年营业额5% | 停业整顿、吊销许可证、对直接责任人员罚款10-100万元 |

### 5.2 典型执法案例

```
- 滴滴出行：80.26亿元罚款（危害国家数据安全）
- 知网：5000万元（非法处理个人信息）
- 多家APP违规收集/使用个人信息被通报下架
```

## 高分考点与知识巧记

> 🔑 **高分考点**：《数据安全法》与《个人信息保护法》是 CISP 考试数据安全板块的基石。三法联动关系、数据分类分级制度、个人信息处理规则、数据出境要求是四大核心考点。

| 考点 | 频次 | 核心记忆点 |
|:---|:---:|:---|
| 三法联动 | ⭐⭐⭐⭐⭐ | 网安法(2017)→数安法(2021)→个保法(2021)，三法互补 |
| 数据分类分级 | ⭐⭐⭐⭐ | 数安法第21条：国家建立数据分类分级保护制度 |
| 个人信息处理原则 | ⭐⭐⭐⭐⭐ | 告知同意、目的限制、最小必要、公开透明、权责一致 |
| 数据出境 | ⭐⭐⭐⭐ | 安全评估/标准合同/认证三条路径 |
| 重要数据目录 | ⭐⭐⭐ | 各地区、各部门制定重要数据目录 |

> 💡 **知识巧记**：三法联动记"网数个"——网安法打底、数安法管数据、个保法护隐私。个人信息处理五原则记"告目最公责"——告知同意、目的限制、最小必要、公开透明、权责一致。

### 高分考点速查表

| 考察维度 | 关键结论 | 常见干扰项 |
|:---|:---|:---|
| 三法时间线 | 网安法2017→数安法2021→个保法2021 | "数安法早于网安法" ❌ |
| 个保法域外效力 | 在境外处理境内自然人个人信息的活动也适用 | "个保法只适用于境内" ❌ |
| 告知同意 | 处理个人信息需取得个人同意（例外情形除外） | "所有情形都需同意" ❌ |
| 数据出境路径 | 安全评估/标准合同/认证，三条路径并行 | "只能走安全评估" ❌ |

### 知识巧记口诀

> **数据安全法律口诀**：
> 网数个三法联动，一七年到二一年。
> 数安法分类分级管，个保法告知同意先。
> 数据出境三条路，评合认证并行选。
> 重要数据目录定，合规不是一次性完。
