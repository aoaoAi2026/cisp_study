## 1. 渗透测试报告结构

```
渗透测试报告 = 摘要 + 技术详情 + 修复建议 + 附录

标准结构：
1. 文档控制（版本/日期/密级）
2. 执行摘要（给管理层看的1页）
3. 测试概述（范围/方法/时间）
4. 漏洞汇总（统计+评级分布）
5. 漏洞详情（每个漏洞单独描述）
6. 修复建议（按优先级）
7. 附录（方法/POC/工具列表）
```

## 2. 漏洞评级标准

| 等级 | CVSS | 定义 | 示例 |
|------|------|------|------|
| **严重** | 9.0-10.0 | 直接获取系统权限，无需认证 | SQL注入RCE/文件上传WebShell |
| **高危** | 7.0-8.9 | 获取敏感数据/需简单条件 | SSRF内网/未授权访问 |
| **中危** | 4.0-6.9 | 需特定条件/信息泄露 | XSS/CSRF/目录遍历 |
| **低危** | 0.1-3.9 | 低风险信息泄露 | 版本泄露/不安全的Cookie |

## 3. 漏洞描述模板

```markdown
## [严重] SQL注入漏洞

### 基本信息
- **漏洞编号**: VULN-2026-001
- **CVSS评分**: 9.8 (CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H)
- **发现时间**: 2026-06-12
- **影响URL**: https://target.com/api/search

### 漏洞描述
在 `/api/search` 接口的 `keyword` 参数存在SQL注入漏洞。
攻击者无需认证即可通过构造特殊SQL语句获取数据库中的所有数据。

### 漏洞复现
```
GET /api/search?keyword=test' UNION SELECT 1,username,password,4,5 FROM users-- HTTP/1.1
Host: target.com
```

### 漏洞证明
（附截图：返回了所有用户名和密码）

### 影响范围
攻击者可以：
1. 获取所有用户账号密码
2. 篡改/删除数据库数据
3. 通过数据库写入WebShell

### 修复建议
1. 使用参数化查询（PreparedStatement）
2. 对用户输入进行严格的类型校验
3. 部署WAF规则

**修复代码示例**：
```java
// 错误写法
String sql = "SELECT * FROM products WHERE name='" + keyword + "'";

// 正确写法
String sql = "SELECT * FROM products WHERE name=?";
PreparedStatement pstmt = conn.prepareStatement(sql);
pstmt.setString(1, keyword);
```
```
