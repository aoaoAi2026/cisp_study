## 1. 渗透测试报告结构

```
渗透测试报告 = 摘要 + 技术详情 + 修复建议 + 附录

标准结构：
1. 文档控制（版本/日期/密级）
2. 执行摘要（给管理层看的1页）
3. 测试概述（范围/方法/时间）
4. 漏洞汇总（统计+评级分布）
5. 漏洞详情（每个漏洞单独描述）
6. 修复建议（按优先级）
7. 附录（方法/POC/工具列表）
```

## 2. 漏洞评级标准

| 等级 | CVSS | 定义 | 示例 |
|------|------|------|------|
| **严重** | 9.0-10.0 | 直接获取系统权限，无需认证 | SQL注入RCE/文件上传WebShell |
| **高危** | 7.0-8.9 | 获取敏感数据/需简单条件 | SSRF内网/未授权访问 |
| **中危** | 4.0-6.9 | 需特定条件/信息泄露 | XSS/CSRF/目录遍历 |
| **低危** | 0.1-3.9 | 低风险信息泄露 | 版本泄露/不安全的Cookie |

## 3. 漏洞描述模板

```markdown
## [严重] SQL注入漏洞

### 基本信息
- **漏洞编号**: VULN-2026-001
- **CVSS评分**: 9.8 (CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H)
- **发现时间**: 2026-06-12
- **影响URL**: https://target.com/api/search

### 漏洞描述
在 `/api/search` 接口的 `keyword` 参数存在SQL注入漏洞。
攻击者无需认证即可通过构造特殊SQL语句获取数据库中的所有数据。

### 漏洞复现
```
GET /api/search?keyword=test' UNION SELECT 1,username,password,4,5 FROM users-- HTTP/1.1
Host: target.com
```

### 漏洞证明
（附截图：返回了所有用户名和密码）

### 影响范围
攻击者可以：
1. 获取所有用户账号密码
2. 篡改/删除数据库数据
3. 通过数据库写入WebShell

### 修复建议
1. 使用参数化查询（PreparedStatement）
2. 对用户输入进行严格的类型校验
3. 部署WAF规则

**修复代码示例**：
```java
// 错误写法
String sql = "SELECT * FROM products WHERE name='" + keyword + "'";

// 正确写法
String sql = "SELECT * FROM products WHERE name=?";
PreparedStatement pstmt = conn.prepareStatement(sql);
pstmt.setString(1, keyword);
```
```

## 高分考点与知识巧记

### 高分考点速查表

| 考点 | 考察维度 | 记忆要点 |
|------|----------|----------|
| 渗透测试报告结构 | 规范标准 | 摘要(管理层)→技术详情(技术团队)→修复建议→附录 |
| CVSS评分体系 | 量化评估 | 3.0版本；基础分+时间分+环境分；9.0+严重/7.0+高危 |
| 漏洞描述五要素 | 报告规范 | 描述+复现步骤+PoC+影响评估+修复建议（缺一不可） |
| 修复优先级分类 | 风险管控 | P0(24h)严重漏洞→P1(7天)高危→P2(30天)中低危 |
| 执行摘要写法 | 沟通技巧 | 1页以内、非技术语言、核心风险+业务影响、建议措施 |
| 报告密级分类 | 合规要求 | 根据漏洞严重程度和业务影响确定密级和分发范围 |

### 知识巧记口诀

> **报告编写口诀**：
> 摘要写给管理层，技术细节给技术；
> CVSS评分要准确，3.0版本记清楚；
> 漏洞描述五要素，缺一不可要完整；
> P0严重P1高危，修复时间分三档；
> 修复建议可操作，代码示例最加分；
> 附录工具和方法，时间日志不能少。

> **CVSS口诀**：AV/AC/PR/UI/S/C/I/A，八字维度定分数，基础分+时间+环境，9分以上是严重。

### 考试陷阱提醒

| 陷阱 | 正确认知 |
|------|----------|
| ❌ 报告只需要技术细节 | ✅ 执行摘要是给管理层看的关键部分，CISP考试常考报告结构 |
| ❌ CVSS评分可以随意估 | ✅ 必须按照CVSS v3.0标准严格计算，考试会考计算公式 |
| ❌ 修复建议写"建议修复"就行 | ✅ 必须给出可操作的、具体的修复方案，最好附带代码示例 |

> 💡 **一句话总结**：一份专业的渗透测试报告是技术能力的最终体现——不仅要找到漏洞，更要说清楚风险、讲明白修法。CISP考试重点考查报告规范和CVSS评分。
