## 二、汽车攻击面全景

### 2.1 外部接口攻击面

```
车辆外部接口：
├── 物理接口
│   ├── OBD-II (车载诊断接口) — 直连CAN总线！
│   ├── USB接口
│   └── 充电接口(电动车/充电桩通信)
├── 短距离无线
│   ├── Bluetooth (BT/BLE)
│   ├── WiFi (车载热点)
│   ├── NFC (数字钥匙)
│   ├── UWB (超宽带数字钥匙)
│   └── TPMS (胎压监测 — 433MHz/315MHz)
├── 长距离无线
│   ├── 4G/5G 蜂窝 (T-Box)
│   ├── V2X (DSRC/C-V2X) — 车车/车路通信
│   └── GNSS (GPS/北斗 — 定位欺骗风险)
└── 云端
    ├── 远程信息处理(TSP平台)
    ├── OTA更新服务器
    ├── 手机APP(数字钥匙/远程控车)
    └── 第三方API开放平台
```

### 2.2 核心攻击目标

```
车辆关键安全组件：

T-Box (远程通信终端):
  4G/5G模块 + GPS + CAN网关
  → 攻破T-Box ≈ 获得CAN总线写入权限
  → 典型攻击：AT指令注入/基带漏洞/固件提取

IVI (车载信息娱乐系统):
  Android/Linux/QNX系统 + WiFi/BT/USB
  → 攻破IVI → 通过网关向CAN总线注入(若未隔离)
  → 典型攻击：浏览器漏洞/恶意APP/USB BadUSB

车载网关(CGW):
  CAN/以太网路由 + 防火墙
  → 域间隔离的最后防线
  → 典型攻击：网关固件篡改/诊断会话劫持

ADAS (高级驾驶辅助):
  摄像头/毫米波雷达/激光雷达 + AI模型
  → 传感器欺骗：让摄像头"看不到"行人
  → 对抗样本：修改路牌→ 让ADAS误识别
```

## 三、重大汽车安全事件

| 年份 | 事件 | 攻击路径 | 影响 |
|------|------|---------|------|
| 2015 | **Jeep Cherokee 远程攻击** | Miller & Valasek: Sprint蜂窝网络→Uconnect→CAN总线 | 140万辆召回,FCA被罚1.05亿$ |
| 2016 | **Tesla 远程控制** | KeenLab: 浏览器→IVI→CAN | Tesla 10秒内推送OTA修复 |
| 2019 | **Tesla Model 3 参赛者攻破** | Pwn2Own: 浏览器漏洞→MCU→网关 | 整车控制权限 |
| 2022 | **本田/Nissan密钥克隆** | Rolling-PWN: 重放滚动码攻击 | 多款车型可被任意解锁启动 |
| 2023 | **特斯拉UN R155合规** | - | Tesla全球首个过UN R155认证 |
| 2023 | **CAN注入攻击** | CAN Injection: 通过大灯/CAN连接器直接注入 | 无需钥匙即可开走车辆 |

### Jeep Cherokee 攻击详解

```
Charlie Miller & Chris Valasek (2015):

攻击链：
  1. 通过Sprint蜂窝网络扫描→发现Jeep的Uconnect系统(公网可达!)
  2. Uconnect系统使用开放端口6667(D-Bus调试接口)
  3. 通过D-Bus提权→获得Uconnect系统root权限
  4. Uconnect芯片(OMAP)通过SPI接口连接V850网关芯片
  5. 修改V850网关固件→CAN消息转发规则被篡改
  6. 攻击者可以从互联网发送任意CAN消息
     → 控制方向盘/刹车/加速/雨刷/空调...

关键教训：
  ✗ 信息娱乐系统直接连到公网(无防火墙)
  ✗ IVI和控制总线之间无有效隔离
  ✗ 未实施安全启动(Secure Boot)链
  ✓ 应该隔离IVI和动力CAN
  ✓ 应该实施OTA安全更新机制
```

## 四、国内汽车安全标准体系

```
中国智能网联汽车安全标准：

GB/T 体系 (推荐性国家标准)：
  GB/T 40855-2021 车载信息交互系统安全要求
  GB/T 40856-2021 车载网关安全要求
  GB/T 40857-2021 车载信息服务安全要求
  GB/T 43258-2023 汽车整车信息安全技术要求 ← ★核心
  GB/T 43267-2023 汽车软件升级技术要求

工信部《车联网网络安全和数据安全标准体系建设指南》：
  100+标准分3年建成

与UN R155/R156的关系：
  中国正在推进与UN法规的互认
  出口欧洲车辆：必须通过UN R155 CSMS认证
  国内车辆：满足GB/T标准 + 工信部入网要求
```

### 考试陷阱提醒

| 陷阱 | 正确理解 |
|:---|:---|
| "汽车安全只需要防OBD-II攻击" | ❌ 攻击面包括所有无线接口（蜂窝/WiFi/蓝牙/GNSS），远程攻击威胁更大 |
| "国内标准和国际标准不兼容" | ❌ GB/T 43258 对标 ISO 21434，两者兼容，出口车辆需同时满足 |
| "CAN总线有加密就安全" | ❌ CAN 设计之初无安全机制，加密只是补充，还需 IDS/网关白名单/域隔离 |

---

> **掌握车联网安全需要"从外到内"的攻击面思维——每一个无线接口都是潜在入口，每一辆智能汽车都是一个移动的 IoT 节点。**
